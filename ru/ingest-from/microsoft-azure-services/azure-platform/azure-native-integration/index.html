    Встроенный сервис Dynatrace для Azure¶

    Встроенный сервис Dynatrace для Azure¶

    • Последняя версия Dynatrace
    • Описание
    • Время чтения: 17 мин
    • Обновлено 23 июля 2024 г.

    Интеграция с Azure Marketplace позволяет создать единый процесс покупки, настройки и управления Dynatrace непосредственно в портале Azure. После настройки интеграции Dynatrace отображается как нативный сервис Azure, и вы можете управлять его конфигурацией из портала Azure.

    Возможности и ограничения

    Возможности:

    • Простое подключение: Лёгкое подключение к Dynatrace SaaS как к интегрированному сервису Azure через Azure Marketplace. Для настройки интеграции:

    • Не требуются концентраторы событий, облачные функции или дополнительная конфигурация

    • Единый счёт: Получайте единый счёт для всех ресурсов, потребляемых в Azure, включая все ресурсы Dynatrace SaaS.

    • Единый вход: Вам не нужны отдельные учётные данные для портала Dynatrace. Войдите один раз в портал Azure и плавно переходите в Dynatrace при необходимости.

    • Мониторинг логов: Обеспечивает пересылку логов активности подписки и логов ресурсов в Dynatrace. Подробнее см. в разделе Классический мониторинг логов.

    • Развёртывание OneAgent: Вы получаете единый интерфейс управления Dynatrace OneAgent. Вы можете устанавливать и удалять Dynatrace OneAgent в качестве расширений для Azure Virtual Machines и Azure App Services.

    Ограничения:

    • Интеграция создаёт новую среду и аккаунт Dynatrace; она не может работать в существующей среде Dynatrace.

    Azure Marketplace

    Сервис Azure Native Dynatrace доступен через Azure Marketplace.

    Чтобы получить индивидуальное частное предложение для Azure Native Dynatrace Service, свяжитесь с отделом продаж Dynatrace по электронной почте sales@dynatrace.com.

    Предварительные требования¶

    Активация частного плана для Azure Native Dynatrace Service

    Azure Native Dynatrace Service доступен по частному плану. Чтобы Dynatrace создал для вас частный план, свяжитесь с Dynatrace. После принятия предложения частный план для Azure Native Dynatrace Service станет доступен в Azure Marketplace.

    Регистрация поставщика ресурсов Dynatrace

    Чтобы создавать ресурсы Dynatrace и управлять ими через портал Azure, необходимо зарегистрировать поставщика ресурсов Dynatrace с именем Dynatrace.Observability в вашей подписке Azure.

    Как зарегистрировать поставщика ресурсов Dynatrace

    На портале Azure

    С помощью Azure CLI

    Следуйте инструкциям в разделе Поставщики ресурсов и типы Azure.

    Выполните команду ниже, заменив <subscription-id> на идентификатор вашей подписки.

    az provider register --namespace Dynatrace.Observability --subscription <subscription-id>

    Настройка разрешений

    Общие разрешения¶

    • Вам необходимо разрешение Contributor для подписки Azure.

    Первый пользователь, создающий первый ресурс Dynatrace и среду в подписке, становится владельцем аккаунта Dynatrace, созданного в ходе развёртывания интеграции. Для всех последующих ресурсов и сред Dynatrace, создаваемых в той же подписке другими пользователями, у владельца аккаунта также будут полные разрешения.

    Разрешения SSO¶

    Если вы используете Microsoft Entra ID в качестве поставщика удостоверений, вы можете настроить единый вход (SSO) из портала Azure в Dynatrace. Если вы используете другой поставщик удостоверений или не хотите настраивать SSO, вы можете пропустить этот раздел.

    Чтобы включить аутентификацию с единым входом для вашего ресурса Dynatrace, необходимо настроить SSO в Microsoft Entra ID.

    После настройки SSO в Active Directory вы можете включить SSO во время или после развёртывания интеграции Azure.

    Настройка интеграции¶

    При первом развёртывании Azure Native Dynatrace Service создаются новая среда Dynatrace на новом ресурсе Dynatrace.

    • Ресурс Dynatrace создаётся в подписке Azure и группе ресурсов, выбранных во время установки. Вы можете настраивать, управлять и устранять неполадки ресурса Dynatrace из портала Azure.

    • Среда Dynatrace создаётся в том же регионе Azure, в котором создаётся ресурс Dynatrace. В этой новой среде:

    • После установки OneAgent вы можете начать мониторинг метрик из ваших ресурсов Azure. Вы также можете собирать метрики из сервисов по умолчанию

    • Вы можете собирать логи из ваших ресурсов Azure
    • Для этой интеграции не требуется ActiveGate среды.

    Настройка интеграции

    1. Перейдите к подписке Azure, в которой хотите создать ресурс Dynatrace.
    2. Выберите Ресурсы, затем выберите Создать.
    3. Найдите Azure Native Dynatrace Service. Вы должны увидеть сообщение о доступности частного продукта.
    4. Выберите плитку Azure Native Dynatrace Service.
    5. В списке выбора плана выберите частный план, который вы приняли в разделе Активация частного плана для Azure Native Dynatrace Service, и нажмите Подписаться.
    6. В разделе Создать новую среду Dynatrace выберите Создать.
    7. В разделе Основные для параметра Группа ресурсов укажите, создать новую группу ресурсов или использовать существующую.

    Группа ресурсов — это контейнер для связанных ресурсов решения Azure. Дополнительные сведения см. в статье Обзор групп ресурсов Azure. 8. Введите Имя ресурса для ресурса Dynatrace, затем выберите Регион из раскрывающегося меню. Ресурс Dynatrace в Azure и среда Dynatrace будут созданы в выбранном регионе. 9. Выберите Тарифный план, затем выберите Далее: Метрики и логи. 10. Необязательно Укажите, следует ли Отправлять логи активности подписки и/или Отправлять логи ресурсов Azure. Подробнее см. в разделе Настройка метрик и логов.

    Если вы выберете **Отправлять логи ресурсов Azure для всех определённых сервисов**, логи ресурсов Azure отправляются для всех определённых ресурсов по умолчанию, то есть логи собираются для всех поддерживаемых ресурсов. Для фильтрации конкретного набора ресурсов Azure, отправляющих логи в Dynatrace, можно использовать теги ресурсов Azure. Правила тегов:

* Ресурсы Azure с тегами `Include` отправляют логи в Dynatrace.
* Ресурсы Azure с тегами `Exclude` не отправляют логи в Dynatrace.
* При конфликте правил включения и исключения приоритет имеют правила исключения.

  Существует ограничение в 20 записей для тегов `Include` или `Exclude`.
    1. Выберите Далее: Единый вход.

    2. Необязательно Укажите, следует ли Включить SSO через Microsoft Entra ID.

      • Если вы не хотите включать эту функцию, выберите Далее: Теги.
      • Если вы хотите включить эту функцию, выберите её, выберите отображаемый идентификатор приложения Dynatrace, затем выберите Далее: Теги.
      • Необязательно Укажите теги для нового ресурса Dynatrace, затем выберите Далее: Просмотр и создание.
      • Проверьте правильность введённой информации и выберите Создать. После завершения развёртывания можно выбрать Перейти к ресурсу для перехода к конкретному ресурсу Dynatrace.

    Привязка нескольких подписок Azure к одной среде Dynatrace Необязательно¶

    Убедитесь, что ваш аккаунт Azure имеет доступ к аккаунту Dynatrace со следующими разрешениями:

    • Просмотр аккаунта
    • Просмотр среды
    • Установка OneAgent
    • Управление настройками мониторинга

    Дополнительные сведения см. в разделе Разрешения среды.

    После развёртывания интеграции Azure вы можете:

    • Привязать дополнительные подписки Azure к только что созданной среде Dynatrace.
    • Привязать несколько сред Dynatrace к одной подписке Azure.

    При привязке нескольких подписок Azure и создании ресурса у вас должны быть привилегии аккаунта Dynatrace tenant-manage-settings и разрешение tenant-agent-install. Дополнительные сведения о настройке этих разрешений см. в разделе Разрешения на основе ролей.

    Выполните следующие шаги, повторяя процедуру для каждой подписки, которую необходимо привязать.

    1. Перейдите на портал Azure и найдите Azure Native Dynatrace Service в строке поиска.
    2. Выберите Azure Native Dynatrace Service.
    3. Выберите Создать.
    4. В разделе Основные для параметра Группа ресурсов укажите, создать новую группу ресурсов или использовать существующую.

    Среда Dynatrace для привязки и новый ресурс Dynatrace должны находиться в одном регионе. 6. Выберите среду Dynatrace для привязки к подписке Azure, затем выберите Далее: Метрики и логи. 7. Необязательно Укажите, следует ли Отправлять логи активности подписки и/или Отправлять логи ресурсов Azure. Подробнее см. в разделе Настройка метрик и логов.

    Если вы выберете Отправлять логи ресурсов Azure для всех определённых сервисов, логи ресурсов Azure отправляются для всех определённых ресурсов по умолчанию, то есть логи собираются для всех поддерживаемых ресурсов.     Доступ к среде Dynatrace¶

    После настройки интеграции вы можете получить доступ к среде Dynatrace непосредственно из портала Azure.

    На портале Azure перейдите к ресурсу Dynatrace и выберите Обзор. Там будут отображены все сведения о вашей среде Dynatrace, включая прямые ссылки на следующие страницы веб-интерфейса:

    • Панели мониторинга — Подробнее см. в разделе Классические панели мониторинга.
    • Просмотр логов — Подробнее см. в разделе Просмотр логов (Классические логи).
    • Топология Smartscape — Подробнее см. в разделе Визуализация среды через классический Smartscape.

    Настройка метрик и логов¶

    Метрики¶

    • Вы можете активировать метрики после развёртывания интеграции Azure.

    Сбор метрик с виртуальных машин и App Services

    Чтобы начать сбор метрик с виртуальных машин и App Services, необходимо установить Dynatrace OneAgent на эти ресурсы в качестве расширения.

    Сбор метрик из облачных сервисов

    Все сервисы и метрики включены по умолчанию. При необходимости их можно отключить, нажав кнопку удаления в списке сервисов. После подключения Dynatrace к вашей среде Azure он немедленно начинает мониторинг встроенных сервисов Azure для указанного субъекта-службы. Классические (ранее «встроенные») метрики Azure содержит перечень метрик облачных сервисов Azure, отслеживаемых по умолчанию.

    Управление облачными сервисами¶

    Все облачные сервисы отслеживаются по умолчанию, но вы можете быстро отключить или повторно включить их из списка по необходимости.

    Добавление сервисов в мониторинг

    1. Перейдите в Настройки > Облако и виртуализация > Azure.
    2. На странице обзора Azure найдите подключение, которое хотите изменить, и выберите Изменить в этой строке.
    3. В разделе Сервисы выберите Управление сервисами.
    4. Для каждого сервиса, который нужно добавить: выберите Добавить сервис, выберите сервис из списка, затем выберите Добавить сервис.
    5. Выберите Сохранить изменения для сохранения конфигурации.

    Настройка собираемых метрик для каждого сервиса

    После добавления сервиса Dynatrace автоматически начинает сбор набора метрик для этого сервиса.

    Рекомендуемые метрики:

    • Включены по умолчанию и не могут быть отключены.
    • Могут включать рекомендуемые измерения (включены по умолчанию, не могут быть отключены) и необязательные измерения (отключены по умолчанию, могут быть включены).

    Помимо рекомендуемых метрик, большинство сервисов предоставляют возможность включения необязательных метрик, которые можно добавлять и настраивать вручную.

    Список облачных сервисов Azure и собираемых метрик

    Для просмотра полного списка облачных сервисов Azure и метрик, собираемых для каждого из них, см. раздел Все облачные сервисы Azure.

    Кроме того, вы можете проверить список поддерживаемых Azure сервисов в Dynatrace Hub (выполните поиск по запросу Azure) или в веб-версии Dynatrace Hub.

    Добавление и настройка метрик

    1. Перейдите в Настройки > Облако и виртуализация > Azure.
    2. На странице обзора Azure найдите подключение, которое хотите изменить, и выберите Изменить в этой строке.
    3. В разделе Сервисы выберите Управление сервисами.
    4. Выберите сервис, для которого хотите добавить метрики. На странице сведений о сервисе будут показаны метрики, которые вы уже отслеживаете для данного сервиса.
    5. Выберите Добавить метрику.
    6. В списке Добавить новую метрику выберите метрику и нажмите Добавить метрику.
    7. Выберите для развёртывания сведений о метрике и её настройки.
    8. Выберите Применить для сохранения конфигурации.

    После выбора облачных сервисов и сохранения изменений мониторинг вновь добавленных сервисов начнётся автоматически.

    Логи¶

    • Вы можете активировать логи как во время развёртывания интеграции Azure, так и после развёртывания.

    Из Azure в Dynatrace можно отправлять два типа логов: логи активности подписки и логи ресурсов Azure.

    Логи активности подписки¶

    Логи активности подписки предоставляют сведения об операциях (PUT, POST, DELETE), выполняемых с каждым ресурсом Azure в вашей подписке (уровень управления). Для каждой подписки Azure существует один лог активности.

    Логи ресурсов Azure¶

    Логи ресурсов Azure предоставляют сведения об операциях, выполненных внутри ресурса Azure (уровень данных), например о получении секрета из хранилища ключей или выполнении запроса к базе данных. Содержимое логов ресурсов зависит от сервиса Azure и типа ресурса.

    Все сервисы Azure в категориях логов Azure Monitor принимают логи, включая Microsoft Entra ID и Azure Monitor Integration Service.

    Список логов ресурсов Azure см. в разделе Поддерживаемые категории для логов ресурсов Azure Monitor.

    Активация логов после развёртывания¶

    Активация логов активности подписки

    Активация логов ресурсов Azure

    1. На портале Azure перейдите к ресурсу Dynatrace и выберите Метрики и логи.
    2. Выберите Отправлять логи активности подписки.

    3. Выберите Сохранить.

    4. На портале Azure перейдите к ресурсу Dynatrace и выберите Метрики и логи.

    5. Выберите Отправлять логи ресурсов Azure для всех определённых источников.

    При выборе этого параметра логи ресурсов Azure отправляются для всех определённых ресурсов по умолчанию, то есть логи собираются для всех поддерживаемых ресурсов. 3. Необязательно Если вы хотите отфильтровать конкретный набор ресурсов Azure, отправляющих логи в Dynatrace, используйте теги ресурсов Azure. Правила тегов:

    • Ресурсы Azure с тегами Include отправляют логи в Dynatrace.
    • Ресурсы Azure с тегами Exclude не отправляют логи в Dynatrace.
    • При конфликте правил включения и исключения приоритет имеют правила исключения.
    • Выберите Сохранить.

    Azure Native Dynatrace Service автоматически включает пересылку логов в подписке Azure, в которой включён ресурс Dynatrace. Пересылка логов включается для всех поддерживаемых сервисов и ресурсов. Правила фильтрации по тегам позволяют включать или исключать определённые ресурсы Azure из отправки логов.

    Azure Native Dynatrace Service использует токен доступа Dynatrace с именем azure-native-integration, который ротируется каждые 24 часа.

    Создание тегов¶

    • Вы можете создавать теги как во время развёртывания интеграции Azure, так и после развёртывания.

    Вы можете применять теги к ресурсам Azure, группам ресурсов и подпискам для их логической организации в таксономию. Вы можете указать теги для нового ресурса Dynatrace в Azure, добавляя пользовательские пары ключ/значение:

    • В поле Имя введите имя тега, соответствующего ресурсу Azure Dynatrace (например, owner).
    • В поле Значение введите значение тега, соответствующего ресурсу Azure Dynatrace (например, адрес электронной почты владельца).

    Если вы не установите теги, все логи с отслеживаемых ресурсов в вашей подписке Azure будут отправляться в Dynatrace.

    Создание тегов после развёртывания¶

    На портале Azure перейдите к новому ресурсу Dynatrace и выберите Теги. Либо выберите Отслеживаемые ресурсы, затем выберите Изменить теги для нужных ресурсов.

    Управление отслеживаемыми ресурсами в портале Azure¶

    После развёртывания Azure Native Dynatrace Service вы можете просматривать, управлять и отслеживать ресурсы Azure, а также устанавливать OneAgent на виртуальные машины Azure и Azure App Services.

    Для просмотра списка ресурсов на портале Azure перейдите к ресурсу Dynatrace и выберите Отслеживаемые ресурсы. Список ресурсов можно фильтровать по Имени ресурса (имя ресурса Azure), Типу ресурса (тип ресурса Azure), Группе ресурсов (имя группы ресурсов для ресурса Azure), Региону (местоположение ресурса Azure) и Логи в Dynatrace (отправляет ли ресурс логи в Dynatrace).

    Развёртывание OneAgent на виртуальных машинах Azure и Azure App Services¶

    • Вы можете развернуть OneAgent после развёртывания интеграции Azure.

    Для мониторинга виртуальных машин Azure или Azure App Services вы можете установить Dynatrace OneAgent на эти ресурсы в качестве расширения.

    Установка OneAgent на виртуальную машину

    Установка OneAgent на App Service

    1. На портале Azure перейдите к ресурсу Dynatrace и выберите Виртуальные машины.
    2. Выберите виртуальную машину из списка, на которую хотите установить расширение OneAgent.
    3. Выберите Установить расширение.
    4. Необязательно Укажите, следует ли включить аналитику логов.
    5. Необязательно Укажите группу хостов для OneAgent.
    6. Выберите ОК для запуска процесса установки.

    7. После завершения установки Статус OneAgent для выбранной виртуальной машины отображается как Установлен.

    8. Чтобы просмотреть сведения об установленном OneAgent, выберите свою виртуальную машину и перейдите в раздел Расширения.

    9. Чтобы удалить OneAgent, выберите виртуальную машину и нажмите Удалить расширение.

    10. На портале Azure перейдите к ресурсу Dynatrace и выберите App Services.

    11. В списке выберите App Service, на который хотите установить расширение OneAgent.
    12. Выберите Установить расширение.
    13. Необязательно Укажите, следует ли включить аналитику логов.
    14. Необязательно Укажите группу хостов для OneAgent.
    15. Выберите ОК для запуска процесса установки.

    16. После завершения установки Статус OneAgent для выбранного App Service отображается как Установлен.

    17. Чтобы просмотреть сведения об установленном OneAgent, выберите свой App Service и перейдите в раздел Расширения.

    18. Чтобы удалить OneAgent, выберите App Service и нажмите Удалить расширение.

    Если App Service не отображается в Dynatrace после включения интеграции, перезапустите план App Service.

    Для запуска OneAgent на масштабируемых наборах виртуальных машин с интеграцией Dynatrace Azure используйте расширение Dynatrace OneAgent для виртуальных машин и создайте токен PaaS.

    Удаление Azure Native Dynatrace Service¶

    Чтобы удалить Azure Native Dynatrace Service, необходимо удалить ресурс Dynatrace. После удаления ресурса Dynatrace из Azure логи и метрики перестают отправляться в Dynatrace и все платежи за Dynatrace через Azure Marketplace прекращаются.

    Удаление ресурса Dynatrace

    1. На портале Azure перейдите к ресурсу Dynatrace и выберите Обзор.
    2. Выберите Удалить.
    3. Введите имя приложения, которое хотите удалить, и выберите Удалить.
