    Мониторинг Azure Database for MariaDB¶

    Мониторинг Azure Database for MariaDB¶

    • Последняя версия Dynatrace
    • Практическое руководство
    • Чтение: 2 мин
    • Опубликовано 25 июня 2020 г.

    На странице обзора Azure Database for MariaDB вы можете видеть, заканчивается ли у вас ресурс ЦП или наблюдается высокий процент использования ввода-вывода, поэтому вы всегда знаете, какие запросы необходимо оптимизировать.

    Предварительные условия¶

    • Dynatrace версии 1.196+
    • ActiveGate для среды версии 1.195+

    Включение мониторинга¶

    Чтобы узнать, как включить мониторинг сервиса, см. раздел Включение мониторинга сервиса.

    При необходимости для интеграции с OneAgent см. как отслеживается активность базы данных.

    Просмотр метрик сервиса¶

    Метрики сервиса можно просматривать в вашей среде Dynatrace либо на странице обзора пользовательского устройства, либо на странице Dashboards.

    Просмотр метрик на странице обзора пользовательского устройства¶

    Для доступа к странице обзора пользовательского устройства:

    1. Перейдите в Технологии и процессы (классическое представление).
    2. Отфильтруйте по имени сервиса и выберите соответствующую группу пользовательских устройств.
    3. После выбора группы пользовательских устройств вы попадёте на страницу обзора группы пользовательских устройств.
    4. На странице обзора группы пользовательских устройств перечислены все экземпляры (пользовательские устройства), принадлежащие группе. Выберите экземпляр для просмотра страницы обзора пользовательского устройства.

    Просмотр метрик на панели управления¶

    Если для сервиса имеется готовая панель управления, на вашей странице Dashboards будет отображаться готовая панель для соответствующего сервиса, содержащая все рекомендуемые метрики. Вы можете искать конкретные панели управления, фильтруя по Preset (Готовый), а затем по Name (Имя).

    Для существующих отслеживаемых сервисов может потребоваться повторно сохранить учётные данные, чтобы готовая панель управления отображалась на странице Dashboards. Для повторного сохранения учётных данных перейдите в Настройки > Облако и виртуализация > Azure, выберите нужный экземпляр Azure, затем нажмите Сохранить.

    Вы не можете вносить изменения непосредственно в готовую панель управления, но можете клонировать и редактировать её. Чтобы клонировать панель управления, откройте меню обзора (...) и выберите Клонировать. Чтобы удалить панель управления из списка, её можно скрыть. Для скрытия откройте меню обзора (...) и выберите Скрыть.

    Скрытие панели управления не влияет на других пользователей.

    Настройка зоны управления¶

    Для импорта панели управления Azure Database for MariaDB необходимо настроить зону управления для ограничения отображаемых на панели управления сущностей теми, которые относятся к этому сервису.

    При создании зоны управления для данной панели управления:

    1. Создайте правило, идентифицирующее сервисы на основе общего свойства:

    2. Тип сервиса: Database

    3. Технология: MariaDB
    4. Топология сервиса: Opaque service
    5. Добавьте условие для имени базы данных, содержащего строку mariadb (с учётом регистра).

    Пример

    После создания зоны управления выберите её на панели управления (Редактировать > Настройки > Зона управления по умолчанию). Дополнительную информацию см. в разделе Временной диапазон панели управления и зона управления.
