    Механизм устаревания файлов OneAgent

    • Latest Dynatrace

    OneAgent в режиме развертывания на основе установщика использует встроенный механизм устаревания, который обеспечивает поддержание разумного размера файлов OneAgent.

    Механизм устаревания не используется в режиме мониторинга только приложений (application-only mode).

    Оптимальное использование дискового пространства OneAgent определяется на основе следующих ограничений:

    • Минимальное необходимое пространство составляет 200 МБ * 3 = 600 МБ (журналы, оповещения, отчеты об аварийных сбоях).
    • Пространство по умолчанию составляет 1 ГБ * 3 = 3 ГБ (журналы, оповещения, отчеты об аварийных сбоях).

    Два типа файлов, которые вносят наибольший вклад в использование дискового пространства OneAgent во время его работы, -- это файлы журналов OneAgent и данные времени выполнения, такие как дампы аварийных сбоев и дампы памяти.

    Файлы журналов¶

    Максимальное дисковое пространство, занимаемое файлами журналов OneAgent, хорошо контролируется механизмом устаревания OneAgent. Журналы растут в размере медленно и равномерно. Непрерывный процесс очистки, запускаемый каждую минуту, поддерживает размер журналов в разумных пределах. Мы разработали механизм устаревания журналов с учетом внутреннего понимания того, как регистрируются события, связанные с OneAgent, поэтому устаревание файлов журналов не требует дополнительной настройки.

    Данные времени выполнения¶

    В отличие от файлов журналов, большие файлы данных времени выполнения, такие как дампы аварийных сбоев и дампы памяти, генерируются нерегулярно и могут вызывать резкие скачки использования диска. Чтобы смягчить эту проблему, используйте параметр установки DATA_STORAGE для указания пользовательского каталога для больших данных времени выполнения. Разместите пользовательский каталог на ресурсе, где ограничения на размер диска не так критичны, как на диске, где установлен OneAgent.

    Дополнительную информацию о настройке установки OneAgent см. в справке для конкретной ОС: Linux, Windows или AIX.

    Экстренная очистка¶

    В экстренной ситуации, когда свободное дисковое пространство достигает порога 3%, OneAgent очищает все файлы из каталогов, управляемых механизмом устаревания, за исключением последних журналов установщика и журнала модуля ОС OneAgent.

    Правила механизма устаревания OneAgent¶

    Механизм устаревания OneAgent обеспечивает постоянное поддержание дискового пространства, используемого OneAgent, в пределах заданных ограничений. Для этого к каталогам OneAgent применяется ряд правил. При необходимости вы можете связаться с экспертом по продукту Dynatrace через чат для изменения некоторых правил устаревания.

    Каталог журналов¶

    • Linux/AIX /var/log/dynatrace/oneagent
    • Windows %PROGRAMDATA%\dynatrace\oneagent\log

    Механизм устаревания файлов проверяет файлы и подкаталоги в основном каталоге журналов и удаляет самые старые файлы, когда:

    • Общий размер каталога превышает 1 ГБ
    • Любой файл в этом каталоге старше 14 дней

    Файлы удаляются из любого подкаталога, содержащего более 1000 файлов (самые новые файлы сохраняются; самые старые файлы удаляются).

    Для подкаталогов существуют дополнительные правила:

    • {log-dir}/process Файлы удаляются при выполнении любого из следующих условий:

    • Файлы в этом каталоге занимают более 300 МБ в сумме

    • Любой файл в этом каталоге старше 14 дней

    • {log-dir}/installer Файлы удаляются при выполнении любого из следующих условий:

    • Файлы в этом каталоге занимают более 30 МБ в сумме

    • Любой файл в этом каталоге старше 180 дней В Windows файл driver.log сохраняется.

    • {log-dir}/dumpproc Файлы удаляются при выполнении любого из следующих условий:

    • Файлы в этом каталоге занимают более 100 МБ в сумме

    • Любой файл в этом каталоге старше 14 дней

    Каталог хранения данных¶

    • Linux/AIX /var/lib/dynatrace/oneagent/datastorage
    • Windows %PROGRAMDATA%\dynatrace\oneagent\datastorage

    • {data_storage_dir}/supportalerts Файлы удаляются при выполнении любого из следующих условий:

    • Каталог содержит более 10 файлов

    • Файлы в этом каталоге занимают более 1 ГБ в сумме
    • Любой из файлов в этом каталоге старше 7 дней

    • {data_storage_dir}/memorydump Файлы удаляются при выполнении любого из следующих условий:

    • Любой из файлов в этом каталоге старше 2 часов

    • Файлы в этом каталоге занимают более 20 ГБ в сумме

    • {data_storage_dir}/crashreports Файлы удаляются при выполнении любого из следующих условий:

    • Они уже были отправлены в Dynatrace

    • Каталог содержит более 100 файлов
    • Файлы в этом каталоге занимают более 1 ГБ в сумме
    • Любой из файлов в этом каталоге старше 3 дней

    Каталог времени выполнения¶

    • Linux/AIX /var/lib/dynatrace/oneagent/agent/runtime
    • Windows %PROGRAMDATA%\dynatrace\oneagent\agent\runtime

    Механизм устаревания файлов OneAgent проверяет подкаталоги, начинающиеся с 0x. Проверки выполняются рекурсивно.

    • Если каталог 0x* содержит подкаталог dump и все файлы в нем старше 3 дней, то подкаталог dump удаляется.
    • Если все файлы и каталоги в каталоге 0x* старше 7 дней, весь каталог удаляется.
    • Если размер всех файлов в каталоге runtime превышает 1 ГБ, каталоги 0x* удаляются, начиная с самых старых, до тех пор, пока размер каталога runtime не станет менее 1 ГБ.
    • Этот каталог также полностью очищается при обновлении OneAgent.

    Каталог установочных бинарных файлов¶

    • Linux/AIX /opt/dynatrace/oneagent/bin
    • Windows %PROGRAMFILES%\dynatrace\oneagent\bin

    • {install-dir}/bin OneAgent развертывает ряд файловых артефактов в процессе обновления, особенно в случае внедряемых технологических модулей. Все файлы внедряемых модулей хранятся в папках с версиями. Когда OneAgent активен, он выполняет очистку файлов по следующим критериям:

    • Текущая используемая версия всегда сохраняется.

    • OneAgent сканирует все отслеживаемые процессы и определяет, какие библиотеки используются.
    • Список библиотек сравнивается с содержимым каталога bin. Неиспользуемые бинарные файлы из 32-битных и 64-битных каталогов удаляются независимо от того, включен или отключен мониторинг хоста. Для бинарных файлов в каталоге any очистка выполняется только при включенном мониторинге хоста. Механизм устаревания OneAgent сохраняет текущую версию и 10 последних предыдущих версий, всего 11 версий.

    Механизм устаревания для OneAgent в режиме мониторинга только приложений¶

    Если у вас нет доступа к инфраструктурному уровню, Dynatrace также предоставляет возможность мониторинга только приложений для Kubernetes, OpenShift, CloudFoundry или SAP Business Technology Platform.

    Механизм устаревания в режиме мониторинга только приложений управляет журналами модулей OneAgent. Все они расположены в соответствующих подкаталогах каталога журналов OneAgent по умолчанию:

    • Linux/AIX /var/log/dynatrace/oneagent
    • Windows %PROGRAMDATA%\dynatrace\oneagent\log

    Ротация журналов¶

    Каждый модуль OneAgent ограничивает количество своих журналов до 5. Каждый из этих 5 файлов ограничен 10 МБ.

    Устаревание журналов¶

    OneAgent очищает файлы журналов при запуске, когда:

    • Файл журнала старше 14 дней
    • Суммарный размер всех файлов журналов превышает 300 МБ
    • Количество файлов журналов превышает 1 000

    Чтобы предотвратить ненужные задержки при запуске вашего приложения, ограничение массовой очистки OneAgent установлено на удаление максимум 50 самых старых файлов.
