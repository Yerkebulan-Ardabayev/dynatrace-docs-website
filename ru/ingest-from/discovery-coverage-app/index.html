    Discovery & Coverage¶

    О приложении

    Независимо от того, являетесь ли вы единственным администратором Dynatrace, поддерживающим десятки пользователей, или частью центра компетенций по наблюдаемости (COE), поддерживающего тысячи, обеспечение комплексной наблюдаемости в масштабе требует интуитивной автоматизации. Discovery & Coverage позволяет обнаруживать слепые зоны и внедрять надлежащий уровень наблюдаемости в нужных местах, даже в большом масштабе.

    Предварительные требования

    Разрешения¶

    В следующей таблице описаны необходимые разрешения.

    app-settings:objects:read

    Требуется для чтения настроек приложения

    app-settings:objects:write

    Требуется для записи настроек приложения

    hub:catalog:read

    Требуется для чтения значков из Hub

    settings:objects:read

    Требуется для чтения настроек приёма логов

    settings:objects:write

    Требуется для создания учётных данных VMware (Settings 2.0)

    state:app-states:read

    Требуется для чтения состояния отключения облачных провайдеров покрытия хостов

    state:app-states:write

    Требуется для записи состояния отключения облачных провайдеров покрытия хостов

    storage:buckets:read

    Требуется для получения логов (правила Discovery findings)

    storage:entities:read

    Требуется для получения сущностей (покрытие хостов и Discovery findings)

    storage:filter-segments:read

    Требуется для функциональности сегментов фильтрации

    Начало работы

    Сценарии использования

    1 из 3 Добро пожаловать в Discovery & Coverage.

    Покрытие хостов¶

    Страница Покрытие хостов показывает краткий обзор подключённых облаков, количество обнаруженных хостов и количество хостов, мониторируемых Dynatrace. Приложение автоматически рассчитывает приоритет покрытия для каждого облака.

    Первый шаг к обеспечению полного покрытия вашей среды — убедиться, что все ваши облака и хосты охвачены.

    1. Перейдите в Discovery & Coverage.
    2. На панели слева выберите Покрытие хостов.

    На этой странице отображаются публичные и частные облачные провайдеры: AWS, Azure, GCP, Oracle, VMware, Nutanix и Hyper-V.

    3. Discovery & Coverage использует данные из Smartscape для формирования рекомендаций. На основе текущего мониторинга столбец Рекомендуемое действие предложит действие.

    • Действий не требуется

    • Подключить облако

      Например, если OneAgent установлен на хосте в облаке, которое в настоящее время не мониторится, это отображается в столбце Хосты в необнаруженных облаках, а рекомендуемое действие — Подключить облако. Это действие имеет Критический приоритет, потому что размер пробела в наблюдаемости не может быть определён до подключения всех облаков.

      Подключить облако направляет вас в соответствующее место для подключения конкретного облака, например, в Clouds или Hub. * Установить OneAgent

      Например, если мониторинг облака настроен и обнаружены немониторируемые хосты, рекомендуемое действие — Установить OneAgent, с приоритетом, определяемым размером пробела в наблюдаемости.

      Установить OneAgent открывает модальное окно Установить OneAgent, которое содержит IP-адреса немониторируемых хостов.

      • Чтобы установить OneAgent на подмножество немониторируемых хостов, раскройте строку облачного провайдера и используйте фильтры для сужения целевых хостов, например, по тегу облака.
      • Эти фильтры также можно использовать, если ваши хосты имеют несколько IP-адресов; например, вы можете фильтровать только частные IP-адреса. При применении фильтров текст кнопки Рекомендуемое действие изменяется, указывая, сколько экземпляров OneAgent будет установлено.
      • Вы также можете использовать селектор Сегментов в правом верхнем углу страницы для применения глобальных фильтров.

    По мере добавления дополнительных облаков и хостов, индикатор Покрытие OneAgent в верхней части показывает ваш прогресс к полному покрытию.

    Если вы не уверены, какой режим мониторинга хоста является правильным, вы можете начать с режима Foundation & Discovery, чтобы получить представление о том, какие приложения или инфраструктура могут находиться на хосте и какие взаимозависимости могут существовать.

    Результаты обнаружения¶

    После достижения широкого покрытия с помощью страницы Покрытие хостов выберите Результаты обнаружения для изучения глубины покрытия.

    OneAgent наполняет Smartscape обширными знаниями о приложениях и инфраструктуре на каждом хосте. Страница Результаты обнаружения использует эту информацию для определения того, установлен ли правильный уровень наблюдаемости в нужных местах.

    • Таблица в нижней части страницы отображает правила лучших практик для каждой технологии.
    • Графики в верхней части страницы обобщают данные из этих правил.

    Например, если на хосте обнаружена база данных, правило лучших практик рекомендует установить OneAgent как минимум в режим Infrastructure Monitoring и настроить соответствующее расширение для базы данных.

    Детали правил¶

    Для получения дополнительной информации о правиле лучших практик нажмите в строке таблицы, чтобы развернуть эту строку, как показано в следующем примере.

    Таблица, специфичная для правила, в развёрнутой строке показывает каждую применимую сущность.

    • Фильтрация здесь изменяет текст кнопки рекомендуемого действия.
    • Разверните раздел Объяснение правила для получения обзора того, почему правило считается лучшей практикой. Там также показан точный DQL-запрос, используемый для получения этой информации из Smartscape. Если вы хотите дополнительно изучить запрос, откройте меню Действия > для конкретной строки и выберите Открыть в, чтобы прикрепить запрос к дашборду или ноутбуку.

    При выборе рекомендуемого действия открывается окно с его описанием и списком необходимых разрешений, чтобы вы могли решить, Применить или Отменить действие.

    Если у вас есть дополнительные идеи для правил лучших практик, пожалуйста, оставьте свой отзыв на странице Канал обратной связи для приложения Discovery & Coverage в сообществе Dynatrace.

    Покрытие сети¶

    Страница Покрытие сети использует расширение SNMP Autodiscovery для сканирования вашей сети на наличие SNMP-устройств для мониторинга.

    Выбор Настроить сканирование проведёт вас через установку и настройку расширения. После настройки сканирование выполняется через регулярные интервалы.

    Обнаруженные устройства отображаются в соответствии с конфигурацией расширения.

    • Нажмите в строке таблицы, чтобы развернуть эту строку и отобразить все обнаруженные устройства. Покрытие основано на лучшей практике пингования каждого устройства с помощью Network Availability Monitoring и опроса устройства с помощью соответствующего SNMP-расширения.
    • Выберите Пинговать все для настройки Network Availability Monitoring и Опросить все для настройки соответствующего SNMP-расширения.

    Покрытие сервисов¶

    Страница Покрытие сервисов использует данные модуля eBPF Discovery OneAgent. Этот модуль предоставляет низкозатратный метод определения того, какие URL-адреса отправляются процессу, как часто и является ли источник публичным. Это позволяет быстро определить любые хосты, необходимые для ваших приложений и API. Обнаружение eBPF включено по умолчанию. Чтобы отключить его, в настройках найдите eBPF Service Discovery.

    Вы можете переключить OneAgent на обнаруженных хостах в режим мониторинга Full-Stack в столбце Включить Full-Stack.

    Установка OneAgent¶

    Помимо страницы покрытия хостов, вы можете установить OneAgent, выбрав > Установить OneAgent в правом верхнем углу. Подробнее см. Установка OneAgent.

    Установка ActiveGate¶

    Чтобы установить ActiveGate, выберите > Установить ActiveGate в правом верхнем углу. Подробнее см. Установка ActiveGate.
