    Действия коннектора обработки текста¶

    • Последняя версия Dynatrace
    • Справочник
    • 5 мин. чтения
    • Опубликовано 8 октября 2025 г.

    Ниже приведён список действий рабочего процесса, доступных для обработки текста.

    Подробности синтаксиса пути¶

    • Точечная нотация: используйте . для навигации по вложенным объектам или массивам.
    • Индексация массива: используйте .[index] для доступа к определённым элементам массива.
    • Экранирование специальных символов: используйте квадратные скобки и кавычки для имён свойств с пробелами или специальными символами, например .["some property"].
    • Обновление всего содержимого: используйте . для изменения всего содержимого. Возможность обновления всего содержимого полезна, если JSON или YAML является простой строкой или целым числом, а не объектом, например "simple-string".

    Экранирование синтаксиса шаблонизации¶

    При работе с файлами JSON или YAML вы можете столкнуться с синтаксисом шаблонизации в виде двойных фигурных скобок.

    Выражения рабочего процесса также используют нотацию двойных фигурных скобок.

    Можно экранировать {{ и }} с помощью {{ '{{' }} и {{ '}}' }} соответственно; однако действия, предоставляемые обработкой текста, поддерживают только корректный синтаксис JSON или YAML. Если результат вычисления выражения даёт входные данные действия, которые не считаются корректными с точки зрения JSON или YAML, состояние выполнения рабочего процесса переходит в Error.

    В официальной документации Jinja можно узнать подробнее о механизмах экранирования.

    Установить значение JSON¶

    Конфигурирует файл JSON.

    Входные данные¶

    Результаты¶

    {



"json": "<updated json content>"



}

    Пример¶

    Это пример входных данных для действия Установить значение JSON, в котором мы изменяем значение name с "Max" на "Michael".

    • Содержимое JSON

    {



"persons": [



{



"name": "Max"



},



{



"name": "John"



}



]



}
    * Путь: .persons[0].name * Значение: "Michael"

    Это результат выполнения действия Установить значение JSON в рамках рабочего процесса. Имя было изменено на Michael.

    {



"persons": [



{



"name": "Michael"



},



{



"name": "John"



}



]



}

    Установить значение YAML¶

    Управляет YAML-файлом. Поддерживает многодокументные YAML-файлы.

    Входные данные¶

    Результаты¶

    {



"yaml": "<updated yaml content>"



}

    Пример¶

    Это пример входных данных для действия Установить значение YAML, в котором мы изменяем значение name с "Max" на "Michael".

    • Содержимое YAML

    ---



persons:



- name: John



- name: Sarah



---



persons:



- name: Max



- name: Jeff
    * Путь .persons[0] * Индекс документа 1 * Значение { name: "Michael" }

    Пример результата¶

    Это результат выполнения действия Установить значение JSON в рамках рабочего процесса. Имя было изменено на Michael.

    persons:



- name: John



- name: Sarah



---



persons:



- name: Michael



- name: Jeff

    Получить значение JSON¶

    Извлекает значение из файла JSON.

    Поле Описание Обязательное
    Содержимое JSON Полное содержимое JSON. Обязательное
    Путь Путь к извлекаемому свойству. Обязательное

    Результат¶

    • Содержимое JSON

    {



"person": {



"name": "John Doe"



"age": 30,



}



}
    * Путь: .person

    {



"json": {



"name": "John Doe",



"age": 30



}



}

    Получить значение YAML¶

    Извлекает значение из файла YAML. Поддерживает многодокументные YAML-файлы.

    Поле Описание Обязательное
    Содержимое YAML Полное содержимое YAML. Обязательное
    Индекс документа Актуально только для YAML-файлов с несколькими документами. Индексация начинается с 0. Необязательное
    Путь Путь к извлекаемому свойству. Обязательное

    Результат¶

    • Содержимое YAML

    person:



name: "John doe"



age: 30
    * Путь: .person

    {



"yaml": "name: \"John Doe\"\nage: 30"



}
