    Действия для GitLab Connector

    Следующие действия рабочих процессов доступны для интеграции GitLab Connector . Каждое действие соответствует конечной точке API GitLab Connector.

    Подробнее о создании рабочих процессов см. в разделе Создание рабочих процессов в Dynatrace Workflows.

    Действия сгруппированы по следующим категориям:

    • API файлов репозитория
    • API мерж-реквестов
    • API задач
    • API заданий

    Создание файла¶

    Создаёт новый файл в новой ветке на основе указанной базовой ветки.

    Действие может работать только с одним файлом и приводит к созданию новой ветки. API GitLab не позволяет повторно использовать существующие ветки. Любая попытка сделать это приведёт к ошибке API GitLab.

    Входные данные¶

    Поле Описание Обязательное
    Start branch Начальная ветка, на основе которой создаётся новая ветка. Обязательное
    Branch Имя новой ветки. Обязательное
    File path Относительный путь к создаваемому файлу. Обязательное
    File content Содержимое нового файла. Обязательное
    Commit message Сообщение коммита для нового файла. Обязательное
    Author name Имя автора результирующего коммита. Необязательное

    Выходные данные¶

    Возвращает поля, описанные в документации GitLab Repository files API - Create new file in repository.

    Получение файла¶

    Получает файл из репозитория.

    Это действие следует использовать только для получения содержимого текстового файла.

    Входные данные¶

    Поле Описание Обязательное
    File path Относительный путь к файлу. Обязательное
    Ref Ссылка, из которой извлекается файл. Может быть именем ветки или хешем коммита. Обязательное

    Выходные данные¶

    Возвращает поля, описанные в документации GitLab Repository files API - Get file from repository.

    Обновление файла¶

    Обновляет файл в вашем репозитории GitLab. Это действие создаёт новую ветку на основе указанной базовой ветки.

    Действие может работать только с одним файлом и приводит к созданию новой ветки. API GitLab не позволяет повторно использовать существующие ветки. Любая попытка сделать это приведёт к ошибке API GitLab.

    Входные данные¶

    Поле Описание Обязательное
    Start branch Начальная ветка, на основе которой создаётся новая ветка. Необязательное
    Branch Имя новой ветки. Обязательное
    File path Относительный путь к обновляемому файлу. Обязательное
    File content Обновлённое содержимое файла. Обязательное
    Commit message Сообщение коммита для редактирования. Обязательное
    Author name Имя автора результирующего коммита. Необязательное

    Выходные данные¶

    Возвращает поля, описанные в документации GitLab Repository files API - Update existing file in repository.

    Удаление файла¶

    Действие может работать только с одним файлом и приводит к созданию новой ветки. API GitLab не позволяет повторно использовать существующие ветки. Любая попытка сделать это приведёт к ошибке API GitLab.

    Start branch Начальная ветка, на основе которой создаётся новая ветка. Обязательное
    Branch Имя новой ветки. Обязательное
    File path Относительный путь к удаляемому файлу. Обязательное
    Commit message Сообщение коммита для удаления. Обязательное
    Author name Имя автора результирующего коммита. Необязательное

    Создание мерж-реквеста¶

    Создаёт мерж-реквест в вашем репозитории GitLab.

    Входные данные¶

    Поле Описание Обязательное
    Source Branch Имя исходной ветки. Обязательное
    Target Branch Имя целевой ветки. Обязательное
    Title Заголовок нового мерж-реквеста. Обязательное
    Description Описание мерж-реквеста. Необязательное

    Выходные данные¶

    Возвращает поля, описанные в документации GitLab Merge requests API - Create MR.

    Получение мерж-реквеста¶

    Получает отдельный мерж-реквест в GitLab.

    Входные данные¶

    Поле Описание Обязательное
    Internal ID Внутренний идентификатор мерж-реквеста в GitLab. Обязательное
    Commits behind the target branch Включите переключатель для отображения коммитов, отстающих от целевой ветки. Необязательное
    Rebase operation in progress Включите переключатель для отображения текущей операции rebase. Необязательное

    Выходные данные¶

    Возвращает поля, описанные в документации GitLab Merge requests API - Get single MR.

    Список мерж-реквестов¶

    Получает список мерж-реквестов в вашем репозитории GitLab.

    Входные данные¶

    Поле Описание Обязательное
    Author Имя пользователя, создавшего мерж-реквесты. Необязательное
    State Статус интересующих вас мерж-реквестов. Необязательное

    Выходные данные¶

    Возвращает поля, описанные в документации GitLab Merge requests API - List merge requests.

    Слияние мерж-реквеста¶

    Выполняет слияние мерж-реквеста в вашем репозитории GitLab.

    Входные данные¶

    Поле Описание Обязательное
    Merge request Мерж-реквест для слияния. Обязательное
    SHA Хеш предполагаемого последнего коммита в мерж-реквесте. Если фактический последний коммит отличается, действие завершится ошибкой. Необязательное
    Enable squash merge Включите переключатель для использования стратегии squash merge или выключите для стратегии merge commit. Необязательное

    Выходные данные¶

    Возвращает поля, описанные в документации GitLab Merge requests API - Merge a merge request.

    Обновление мерж-реквеста¶

    Обновляет существующий мерж-реквест в вашем репозитории GitLab.

    Входные данные¶

    Поле Описание Обязательное
    Merge request Мерж-реквест для обновления. Обязательное
    Branch Имя целевой ветки. Обязательное
    Title Новый заголовок мерж-реквеста. Необязательное
    Description Описание мерж-реквеста. Необязательное

    Выходные данные¶

    Возвращает поля, описанные в документации GitLab Merge requests API - Update MR.

    Создание заметки к мерж-реквесту¶

    Создаёт новую заметку для отдельного мерж-реквеста в GitLab.

    Входные данные¶

    Поле Описание Обязательное
    Merge request Мерж-реквест, к которому будет добавлена заметка. Обязательное
    Content of the note Содержимое заметки. Обязательное
    SHA Хеш предполагаемого последнего коммита в мерж-реквесте. Если фактический последний коммит отличается, действие завершится ошибкой. Необязательное

    Выходные данные¶

    Возвращает поля, описанные в документации GitLab Merge requests API - Create MR Note.

    Создание новой задачи¶

    Создаёт новую задачу в вашем репозитории GitLab.

    Входные данные¶

    Поле Описание Обязательное
    Title Заголовок новой задачи. Обязательное
    Labels Список меток через запятую для добавления к задаче. Необязательное
    Issue type Тип задачи. Необязательное
    Description Описание задачи. Необязательное
    Assignees Пользователь, которому назначается задача. Если не задан, задача будет не назначена. Необязательное

    Выходные данные¶

    Возвращает поля, описанные в документации GitLab Issues API - New issue.

    Редактирование задачи¶

    Редактирует задачу в вашем репозитории GitLab.

    Входные данные¶

    Поле Описание Обязательное
    Title Новый заголовок задачи. Необязательное
    Labels Список меток через запятую для применения к задаче. Необязательное
    Issue type Тип задачи. Необязательное
    Description Описание задачи. Необязательное
    Close/Reopen Выберите для изменения статуса задачи. Необязательное

    Выходные данные¶

    Возвращает поля, описанные в документации GitLab Issues API - Edit an issue.

    Список задач¶

    Получает список задач из вашего репозитория GitLab.

    Входные данные¶

    Поле Описание Обязательное
    Issue Ids Показать только задачи с указанными идентификаторами, разделёнными запятыми. Необязательное
    Title or Description Показать только задачи, содержащие указанный текст в заголовке или описании. Необязательное
    Issue type Показать только задачи указанного типа. Необязательное
    Labels Показать только задачи, которым назначены все указанные через запятую метки. Необязательное
    State Укажите, хотите ли вы получить список открытых или закрытых задач. Необязательное

    Выходные данные¶

    Возвращает поля, описанные в документации GitLab Issues API - List issues.

    Получение статуса конвейера¶

    Получает статус конвейера, работающего в вашем репозитории GitLab.

    Входные данные¶

    Поле Описание Обязательное
    Branch Имя ветки. Обязательное
    Pipeline Внутренний идентификатор конвейера в GitLab. Обязательное

    Выходные данные¶

    Возвращает поля, описанные в документации GitLab Jobs API - Get a single job.

    Запуск нового конвейера¶

    Запускает новый запуск конвейера в вашем репозитории GitLab.

    Входные данные¶

    Поле Описание Обязательное
    Branch Имя ветки, для которой запускается конвейер. Обязательное

    Вы также можете задать любое количество параметров сборки.

    Выходные данные¶

    Возвращает поля, описанные в документации GitLab Jobs API - Run a job.
