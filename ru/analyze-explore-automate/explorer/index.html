    Data Explorer¶

    Data Explorer¶

    • Classic
    • Чтение: 7 мин
    • Обновлено 3 октября 2025 г.

    Эта страница относится к Data Explorer — классической функции Dynatrace, тесно интегрированной с классическими дашбордами, созданными с помощью Dashboards Classic. Обновитесь до последней версии Dynatrace, чтобы воспользоваться нашими новейшими возможностями исследования данных.

    Data Explorer значительно расширяет ваши возможности по запросу и визуализации метрик.

    • Анализируйте многомерные метрики — как встроенные в Dynatrace, так и полученные из StatsD, Prometheus, Telegraf и других каналов.
    • Выбирайте тип визуализации.
    • Добавляйте визуализацию прямо на классические дашборды для быстрого доступа и совместного использования.
    • Обеспечивает основу для расчёта метрик в визуализациях на классических дашбордах.

    Быстрый старт

    Если вы предпочитаете пока пропустить технические детали и учиться на практике, попробуйте Быстрый старт Data Explorer. Вы узнаете, как:

    • Использовать шаблон
    • Создать простую визуализацию с нуля
    • Использовать визуализацию непосредственно в Data Explorer
    • Закрепить визуализацию на дашборде для дальнейшего использования и совместного доступа

    Но если вы хотите узнать подробности прямо сейчас, читайте далее.

    Компоненты и концепции запросов¶

    Каждый запрос метрик состоит из нескольких необязательных компонентов. Например, этот запрос:

    содержит следующие компоненты:

    • Имя метрики: CPU usage % (builtin:host.cpu.usage)
    • Агрегация: Average (avg)
    • Разделение по: Host (dt.entity.host)
    • Фильтрация по: Host: OS type: Linux

    Описания этих и других возможных компонентов запроса приведены ниже.

    Редактор запросов помогает выбирать настройки запроса, совместимые с настраиваемым запросом.

    В примере ниже, если навести указатель мыши на значок i (информация) в списке выбора для Rate, редактор объяснит, почему эта настройка недоступна для текущего запроса.

    Имя метрики¶

    В редакторе запросов выберите имя метрики из списка, отображаемого в поле Select metric.... Это может быть встроенная метрика или метрика, полученная из канала, такого как StatsD, Prometheus или Telegraf, через наш API метрик.

    Чтобы выбрать метрику

    • Вы можете ввести или вставить имя метрики непосредственно в поле для поиска всех совпадающих метрик. В этом примере есть несколько совпадений. Мы выбираем метрику в категории Host, чтобы добавить её в наш запрос.

    • Если вы добавили какие-либо метрики в избранное в браузере метрик, эти метрики отображаются в верхней части списка в селекторе метрик.

    * Вы можете выбрать категорию метрик, чтобы сузить список метрик.

    • При наведении указателя мыши на любую метрику в списке боковая панель отображает подробную информацию об этой метрике.

    Чтобы увидеть больше информации об этой метрике, выберите View all metric information. Это откроет браузер метрик в новой вкладке (чтобы вы не потеряли свою работу в Data Explorer) с множеством полезных сведений о выбранной метрике.

    Пространственная агрегация¶

    Пространственная агрегация позволяет указать, как результирующие точки данных запроса метрик должны агрегироваться по измерениям.

    Запрос всегда предоставляет статистически наиболее точные результаты для данного запроса, даже если определённые метрики предоставляют различную статистику, что зависит от природы каждой метрики.

    Чтобы изменить эту агрегацию, выберите одну из списка, расположенного сразу после имени метрики в редакторе запросов:

    Каждая метрика предоставляет одинаковые возможные пространственные агрегации: Auto, Average, Count, Maximum, Minimum, Sum, Median, Percentile 10th, Percentile 75th и Percentile 90th.

    Разделение по (Split by)¶

    По умолчанию запрос не разделяет данные по каким-либо измерениям, используя агрегацию метрики. При разделении по измерению, например по хосту, агрегация используется для каждого хоста.

    Чтобы разделить по измерению

    1. Если Split by ещё не отображается в редакторе запросов, выберите его, а затем выберите Split by из списка.
    2. Установите Split by на измерение, по которому вы хотите разделить запрос.

    Пример¶

    Если метрика строки — CPU usage %, единственным доступным измерением является Host. В Split by выберите Host.

    Сортировка по (Sort by)¶

    По умолчанию результаты сортируются в порядке убывания на основе выбранной агрегации.

    Чтобы задать порядок сортировки

    1. Если Sort by ещё не отображается в редакторе запросов, выберите его, а затем выберите Sort by из списка.
    2. Установите Sort by на измерение, по которому вы хотите сортировать.
    3. Выберите порядок сортировки: ASC (по возрастанию) или DESC (по убыванию).

    Частота (Rate)¶

    Чтобы задать частоту

    1. Если Rate ещё не отображается в редакторе запросов, выберите его, а затем выберите Rate из списка.
    2. Установите Rate на None, Per second, Per minute или Per hour.

    Фильтрация по (Filter by)¶

    Область действия определяется любым установленным фильтром. По умолчанию область действия — (include all).

    Чтобы отфильтровать запрос (изменить область действия)

    1. Если Filter by ещё не отображается в редакторе запросов, выберите его, а затем выберите Filter by из списка.

    2. В Filter by:

    3. Выберите доступное измерение

    4. Укажите атрибут
    5. Укажите значение атрибута

    Вы можете добавить несколько фильтров.

    Пример¶

    Если метрика — Action count (by Apdex category) [web] (builtin:apps.web.actionCount.category) и вы хотите отфильтровать по конкретному веб-приложению с именем My web application

    1. Если Filter by ещё не отображается, отобразите его.
    2. В Filter by выберите Web application, затем выберите Name, а затем выберите My web application

    Смотрите также Автоматически расширенная фильтрация

    Лимит (Limit)¶

    По умолчанию количество метрик, которые вы видите при разделении по измерению, составляет 20.

    Чтобы задать явный лимит

    1. Если Limit ещё не отображается в редакторе запросов, выберите его, а затем выберите Limit из списка.
    2. Установите Limit на 1, 10, 20 или 100.

    Чтобы снять лимит, включите Advanced mode и удалите компонент :limit(n) из запроса.

    Глобальный лимит (Global limit)¶

    Глобальный лимит выравнивает записи по нескольким метрикам с одинаковой размерностью (преобразование Split by). Если Global limit установлен для метрики, в результат включаются только записи, соответствующие измерениям этой метрики.

    Global limit автоматически отображается, если выполнены все следующие условия:

    • Выбрана визуализация Table.
    • В таблицу добавлены как минимум две метрики.
    • Как минимум две метрики имеют одинаковое преобразование Split by.

    Только одна метрика может быть отмечена как Global limit. Метрика, отмеченная как Global limit, переопределяет преобразование Limit всех остальных метрик с тем же преобразованием Split by. В этом случае Limit для этих метрик будет отображаться как отключённый.

    В Advanced mode Global limit всегда виден независимо от условий.

    Значение по умолчанию (Default)¶

    Чтобы задать значение по умолчанию

    1. Если Default ещё не отображается в редакторе запросов, выберите его, а затем выберите Default из списка.
    2. Установите Default на желаемое значение по умолчанию.

    Сдвиг по времени (Timeshift)¶

    Чтобы задать значение сдвига по времени

    1. Если Timeshift ещё не отображается в редакторе запросов, выберите его, а затем выберите Timeshift из списка.
    2. Установите Timeshift на число (положительное или отрицательное).
    3. Выберите единицу измерения (секунда, минута, час, день, неделя) из соседнего списка.

    Пример¶

    Чтобы сдвинуть назад на две минуты:

    1. Установите Timeshift на -2
    2. Установите единицу измерения на minute

    Команды запросов¶

    Используйте эти команды в редакторе запросов для выбора компонентов запроса и установки значений.

    Визуализация¶

    Визуализация по умолчанию — график. Чтобы изменить визуализацию, выберите одну из списка в верхнем левом углу определения запроса.

    Доступны следующие типы визуализации:

    • Graph
    • Stacked column
    • Stacked area
    • Pie
    • Single value
    • Table
    • Top list
    • Heatmap
    • Honeycomb

    Плюс¶

    Чтобы добавить или удалить преобразования метрик для строки в редакторе запросов, выберите кнопку и затем установите или снимите флажки по необходимости.

    • All — отобразить все доступные поля (см. ниже)
    • Default
    • Filter by
    • Limit
    • Rate
    • Sort by
    • Split by
    • Timeshift

    Добавить метрику¶

    Чтобы добавить новую пустую строку, выберите Add metric и затем определите запрос этой строки.

    Дублировать метрику¶

    Чтобы создать копию уже добавленной в запрос метрики, выберите > Duplicate и затем отредактируйте копию по необходимости.

    Добавить событие метрики¶

    Если вы видите что-то на графике Data Explorer, что хотите продолжить отслеживать, можно легко создать событие метрики.

    Чтобы создать событие метрики из Data Explorer

    1. Выберите > Add metric event в редакторе запросов.

    Откроется страница Settings > Anomaly detection > Metric events в новом окне браузера (чтобы вы не потеряли свою работу в Data Explorer) с выбранным пунктом Add metric event и полями метрики, уже заполненными где это возможно из вашего запроса. 2. Завершите определение события метрики и сохраните изменения. 3. Закройте это окно браузера и вернитесь к первому окну, чтобы продолжить работу в Data Explorer.

    Подробнее о событиях метрик см. Metric events.

    Изменение порядка метрик¶

    Порядок, в котором метрики перечислены в вашем запросе, влияет на следующее:

    • Порядок, в котором элементы отрисовки рендерятся: метрики запроса рендерятся по порядку сверху вниз, поэтому последняя рендерится поверх остальных
    • Порядок столбцов в визуализации таблицы
    • Порядок, в котором настройки отображаются на панели Settings

    Чтобы изменить порядок метрик в запросе, выберите и перетащите метрику на новую позицию в списке метрик.

    Повторно выполните запрос, чтобы увидеть изменения.

    Переключение метрики¶

    Чтобы включать и выключать метрики, вы можете выбрать букву рядом с метрикой, которую хотите включить в запрос, или выбрать значок глаза.

    Удалить метрику¶

    Чтобы удалить метрику, выберите > Delete.

    Выполнить запрос¶

    Выберите Run query, чтобы выполнить текущий настроенный запрос и отобразить визуализацию результатов. Текст рядом с кнопкой Run query показывает статус последнего выполнения.

    Расширенный режим (Advanced mode)¶

    Чтобы в полной мере использовать возможности запросов Metrics API v2 из веб-интерфейса, включите Advanced mode для запроса.

    Advanced mode позволяет создавать выражения метрик, сдвиги по временным рамкам и многое другое непосредственно в веб-интерфейсе и, конечно, использовать эти возможности для создания визуализаций для ваших дашбордов. Начните с изучения примеров выражений метрик.

    Подробнее об Advanced mode см. Расширенный редактор запросов Data Explorer.

    Базовые линии (Baselines)¶

    Чтобы помочь вам выявлять аномалии, вы можете использовать базовые линии для добавления доверительного интервала к линии метрики на графике. Тогда вы сможете увидеть, когда значение выходит за пределы доверительного интервала. Расчёт базовой линии основан на модели Seasonal baseline, которая используется для создания событий метрик для обнаружения аномалий.

    • Базовые линии применяются только к визуализации Graph.
    • Базовые линии не добавляются к плитке дашборда при закреплении графика на дашборде.
    • Временной диапазон, используемый для вычисления базовой линии, определяется текущим выбранным разрешением:

    Добавить базовую линию¶

    1. На графике выберите линию метрики, для которой вы хотите построить базовую линию.
    2. Во всплывающем окне выберите Add baseline.
    3. Подождите, пока базовая линия будет рассчитана (Loading). Затем график будет перерисован с отображением базовой линии для выбранной метрики.

    Скрыть или показать базовую линию¶

    Базовые линии перечислены отдельно в легенде графика. Например, если вы добавите базовую линию к метрике CPU usage % в визуализации Graph, в легенде будут перечислены CPU usage % и CPU usage % - baseline. Выбирайте записи легенды, чтобы включать или выключать их отображение.

    Удалить базовую линию¶

    1. На графике выберите линию метрики, от которой вы хотите удалить базовую линию.
    2. Во всплывающем окне выберите Remove baseline. График будет перерисован с удалённой базовой линией.

    Сравнение с базовыми линиями событий метрик¶

    Вы можете заметить различия между базовыми линиями в Data Explorer и событиями метрик. Эти функции предлагают различные подходы для разных контекстов. В целом, конфигурация Data Explorer фиксирована, тогда как конфигурация событий метрик настраиваема.

    Подробнее о сезонных базовых линиях см. Seasonal baseline.

    Часто задаваемые вопросы о базовых линиях¶

    Как рассчитывается базовая линия?

    Расчёт базовой линии основан на модели сезонной базовой линии, используемой для создания событий метрик для обнаружения аномалий. Подробнее о внутренней работе модели см. Seasonal baseline.

    Почему базовая линия отличается от предпросмотра модели сезонной базовой линии?

    Хотя модель базовой линии основана на модели seasonal baseline, есть несколько причин, по которым результирующие базовые линии могут различаться:

    • Разрешение: Базовая линия в Data Explorer рассчитывается на основе данных в зависимости от текущего выбранного разрешения, как описано выше. Модель сезонной базовой линии конфигурации события метрики всегда обучается на данных с разрешением в 1 минуту. Если разрешения различаются, результирующая базовая линия отличается, так как значения метрик также различаются.
    • Временной диапазон базовой линии: Временной диапазон, используемый для вычисления базовой линии, определяется текущим выбранным разрешением, как описано выше. Поскольку конфигурации событий метрик всегда используют данные с разрешением в 1 минуту, временной диапазон обучения может различаться, что также может привести к различным базовым линиям.

    • Параметры модели: Базовая линия в Data Explorer использует фиксированные параметры по умолчанию для обучения модели базовой линии:

    • Tolerance = 4

    • Alert condition = 'Alert if metric is outside'
    • Нет оповещения при отсутствии данных Если параметры в конфигурации события метрики отличаются, результирующая базовая линия может быть другой.

    Коррелированные метрики¶

    Dynatrace Davis берёт в расчёт предметно-специфические знания и топологию при вычислении связанных сигналов наблюдаемости.     Что отображает анализатор?

    • Reference signal представляет серию данных, которую вы выбрали на графике. Формы других серий метрик сравниваются с формой этой серии.

    • Connected signals — это другие серии метрик с похожей формой, отсортированные от наиболее похожих к наименее похожим. Чем более похожа форма, тем ближе корреляция.

      Для каждой коррелированной метрики анализатор отображает:

      • Имя метрики
      • Измерение
      • ID сущности

      Корреляции иногда группируются. 5. На боковой панели выберите любую перечисленную метрику, чтобы автоматически добавить её в ваш текущий запрос.

    • Вы можете добавить несколько коррелированных метрик в запрос

    • Вы можете добавить одну и ту же метрику несколько раз, а затем отредактировать запрос
    • После добавления коррелированных метрик выберите Run query для обновления графика.

    Часто задаваемые вопросы о коррелированных метриках¶

    Что означает «корреляция» в данном контексте?

    Для определения корреляции анализатор проверяет форму серии данных, а не значения. Две серии с очень похожими формами считаются коррелированными.

    Почему опция «See correlated metrics» недоступна?

    Возможные причины, по которым вы не видите опцию «See correlated metrics»:

    • Вы не выбрали визуализацию Graph
    • Вы не использовали Split by по измерению в запросе
    • Вы не выполнили запрос для построения графика
    • Вы не выбрали (не щёлкнули) линию на графике

    Что означает «No connected signals found»?

    Если отображается No connected signals found, возможные причины:

    • Слишком малая вариация в выборке (например, метрика, представляющая прямую линию)
    • Слишком мало точек данных в выборке (например, при очень коротком временном диапазоне)

    Фокус¶

    Чтобы временно убрать потенциальный визуальный шум с графика и сфокусироваться на одной метрике, вы можете скрыть всё, кроме выбранной серии метрик.

    • Фокус применяется только к визуализации Graph.
    • Фокус не изменяет ваш запрос и не влияет на плитку дашборда при закреплении графика на дашборде.

    Фокусировка на серии метрик¶

    1. На линейном графике выберите линию метрики, на которой вы хотите сфокусироваться.
    2. Во всплывающем окне выберите Focus.

    График будет перерисован с отображением только выбранной метрики.

    Снять фокус¶

    1. На графике выберите линию метрики, на которой вы сфокусированы.
    2. Во всплывающем окне выберите Remove focus.

    График будет перерисован с отображением всех метрик.

    Результат¶

    Раздел Result отображает выбранную визуализацию результатов вашего запроса.

    Взаимодействие с визуализацией¶

    Вы можете наводить указатель мыши и выбирать элементы визуализации для просмотра деталей, перехода к соответствующим страницам Dynatrace и изменения визуализации для выявления проблем.

    В этом примере — визуализация Graph, показывающая линейный график метрики CPU usage % для хостов — визуализация показывала хост с нестабильным поведением, поэтому мы выбрали его, чтобы увидеть подробности о хосте во всплывающем окне.

    В этом примере доступны следующие опции:

    • Выберите Focus, чтобы временно сфокусировать график на одной метрике. Подробнее см. Фокус.
    • Выберите Add baseline, чтобы добавить базовую линию для выбранной метрики. Подробнее см. Базовые линии.
    • Выберите See correlated metrics, чтобы отобразить список коррелированных метрик и при желании добавить выбранные из них в ваш запрос. Подробнее см. Коррелированные метрики.
    • Выберите View host, чтобы перейти непосредственно на страницу деталей этого хоста.
    • Выберите View host list, чтобы перейти на страницу Hosts.

    Доступные опции во всплывающем окне зависят от настроенного запроса и визуализации.

    Экспорт в файл CSV¶

    Чтобы экспортировать в файл с разделением запятыми (CSV)

    1. Перейдите в Data Explorer и в разделе Result выберите > Export CSV.

    2. Экспорт CSV доступен для всех визуализаций, кроме honeycomb и single value

    3. Значения, экспортируемые в файл CSV, отражают форматирование, указанное в настройках Unit и Format в разделе Settings.
    4. Файл CSV с результатами сохраняется на вашем локальном компьютере.

    Имя файла указывает метрики, дату и временной диапазон.

    Например:

    • CPU usage % (May 24, 2022, 11_41 - 13_41).csv — содержит результаты метрики CPU usage %, запущенной 24 мая 2022 г. для двухчасового временного диапазона 11:41-13:41.
    • CPU usage % +1 (May 24, 2022, 13_19 - 13_49).csv — содержит результаты метрики CPU usage % и ещё одной метрики, запущенной 24 мая 2022 г. для получасового временного диапазона 13:19-13:49.

    Использование в API¶

    После выполнения запроса у вас есть возможность скопировать запрос для использования в API-запросе.

    1. Перейдите в Data Explorer и в разделе Result выберите > Copy request.
    2. Выберите, использовать ли временной диапазон результата.
    3. Выберите формат ответа: JSON или CSV.
    4. Выберите Copy, чтобы скопировать запрос в буфер обмена, или просто выберите и скопируйте нужные части запроса.
    5. Необязательно: выберите Get a token, чтобы перейти на страницу Generate access token и получить токен для запроса.

    Автоматически расширенная фильтрация¶

    Автоматически расширенные фильтры используют топологию Dynatrace (модель сущностей) для предоставления дополнительных измерений фильтрации, недоступных в исходной метрике. Они работают как на уровне плитки, так и на уровне дашборда.

    • На уровне плитки выбирайте их при настройке Filter by в Data Explorer, просто выбрав исходное измерение метрики, имеющей назначенную связь. Например, метрика, фиксирующая производительность событий Synthetic, имеет связь с монитором Synthetic. Используя топологию, вы можете сначала выбрать связанное событие Synthetic в фильтре, а затем, помимо имени, тега, идентификатора или состояния работоспособности, вы также получите дополнительную опцию для выбора связанного монитора.
    • На уровне дашборда, хотя вы не можете выбирать нужные связи, Dynatrace автоматически расширяет метрики, где это возможно, чтобы при передаче динамического фильтра он мог применяться к плитке с этой метрикой.

    Пример 1: Метрика шага Synthetic, расширенная монитором Synthetic¶

    Некоторые метрики производительности для событий Synthetic не имеют возможности фильтрации по монитору. Однако одно и то же событие может происходить в нескольких мониторах, и для просмотра производительности отдельного монитора необходима возможность фильтрации по ним.

    С автоматически расширенными фильтрами вы теперь можете фильтровать по шагу теста Synthetic.

    1. Выберите метрику (например, Action duration - load action (by event) [browser monitor]). Она имеет тип сущности SYNTHETIC_TEST_STEP.

    1. Добавьте фильтр.

    1. Выберите автоматически расширенное измерение.

    1. Результирующий фильтр.

    Более того, теперь вы можете использовать автоматически расширенные фильтры на дашборде, поэтому нет необходимости настраивать несколько плиток для просмотра одной и той же метрики для разных мониторов или разных хостов.

    1. На вкладке Dynamic filters страницы Dashboard settings вашего дашборда добавьте фильтр для Custom dimension.

    1. Выберите Synthetic monitor.

    1. Сохраните изменения и отобразите дашборд. Теперь вы можете фильтровать все плитки на дашборде по монитору Synthetic. Дашборд автоматически проверит для каждой плитки, существует ли такая связь. Таким образом, каждая плитка (без фильтров, установленных на уровне плитки), содержащая метрику события Synthetic, будет отфильтрована в тот момент, когда существует связь между шагом в плитке и выбранным вами монитором.

    Пример 2: Расширение метрики хоста¶

    Здесь мы расширяем метрику хоста экземпляром EC2.

    1. Создайте плитку, связанную с хостом, с метрикой хоста (например, CPU usage % - builtin:host.cpu.usage).
    2. Примените связанный фильтр, такой как EC2 instance (runsOn).

    Теперь плитка со всеми хостами отфильтрована только по хостам, работающим на этом экземпляре EC2. Это возможно, даже несмотря на то, что измерение EC2 instance не существует в исходной метрике хоста. Используя топологию (модель сущностей), Dynatrace может фильтровать хосты на основе этой связи.

    В этом варианте метрика хоста расширяется группой хостов.

    1. Установите фильтр для Host.Host Group (isInstanceOf) и закрепите плитку на дашборде.

    2. Теперь мы можем фильтровать плитки дашборда по группе хостов.

    Замечания и ограничения¶

    • Максимум 10 метрик на визуализацию
    • До 100 серий на метрику

    Для визуализации Honeycomb вы можете обойти этот лимит: включите Advanced mode и удалите :limit(100) из запроса. * В отличие от многомерного анализа, Data Explorer использует долгосрочные данные метрик, а не данные трассировок и запросов, поэтому значения в визуализациях могут отличаться от значений в многомерном анализе. * Для предотвращения проблем с производительностью на плитках дашбордов, созданных с помощью Data Explorer, максимальное количество точек данных для запроса на плитке дашборда составляет 4 000. На основе выбранного временного диапазона и применённого пользовательского разрешения Dynatrace прогнозирует количество точек данных для результата запроса. Если прогнозируемое количество точек данных превышает 4 000, Dynatrace автоматически переключается на разрешение, достаточно высокое для сохранения количества точек данных ниже 4 000.

    Обратите внимание, что это не применяется к визуализациям в самом Data Explorer, где может быть более 4 000 точек данных. Это применяется только к плиткам дашбордов, созданным с помощью Data Explorer, где комбинация разрешения/временного диапазона, выбранная на дашборде, приводит к более чем 4 000 точкам данных. * Примеры обозначений порядка величины в Dynatrace:

    Преобразование свёртки и разрешение¶

    Обратите внимание, что настройка Fold transformation влияет на разрешение.

    • Если Fold transformation установлено на Auto для визуализаций Table, Single value, Top list или Honeycomb, используется разрешение Inf (бесконечность) для обеспечения обратной совместимости. Если выбранный селектор метрик не поддерживает разрешение Inf, преобразование fold автоматически добавляется в конец запроса.
    • Если Fold transformation установлено на значение, отличное от Auto, используется fold.

    Поскольку все селекторы метрик запрашиваются с использованием одного и того же механизма общего значения (либо fold, либо Inf), добавление нового селектора, требующего fold, может изменить результат других селекторов.

    Чтобы проверить фактический запрос, используемый Data Explorer, перейдите в раздел Result в Data Explorer и выберите > Copy request.
