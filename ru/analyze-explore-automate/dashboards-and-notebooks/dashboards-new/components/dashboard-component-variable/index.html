    Добавление переменной на дашборд¶

    Используйте переменные для фильтрации дашбордов, в качестве значений переменных в плитках кода и как заполнители в заголовках плиток и тексте плиток Markdown.

    Устаревшие поля: dt.entity.*

    Если вы видите следующее сообщение:

    The dt.entity.* fields are deprecated. Please use dt.smartscape.* fields instead.

    рекомендуется использовать эквивалентное поле dt.smartscape.*.

    Устаревшие поля dt.entity.* будут в конечном итоге удалены.

    Добавление переменной¶

    Чтобы добавить переменную на дашборд

    1. В заголовке дашборда откройте меню и выберите Variables. Сочетание клавиш: Shift+V

    Сочетание клавиш: Shift+V

    Отображается панель Variable. 2. Определите переменную.

    • Name: имя переменной.

      • Допускаются только буквенно-цифровые значения
      • Не начинайте имя переменной с dt_

      • Type: может быть одним из следующих:

      • DQL: значение возвращается из запроса, который вы вводите при определении переменной.

      • Выберите Run, чтобы протестировать запрос.

      • Code: значение возвращается из кода, который вы вводите при определении переменной.

      • List: список значений, разделённых запятыми (CSV).

      • Чтобы определить возможные значения, введите их (через запятую) в поле под Data.

      • Чтобы определить значение по умолчанию, выберите его из списка Default value.
      • Чтобы разрешить одновременный выбор нескольких значений, включите Multi-select.
      • Free Text: свободный текст. Можно ввести Default value. Ваши изменения сохраняются автоматически.
      • Настройте Display settings.

    • Если выпадающий список переменной должен отображаться на дашборде, включите Display as filter on dashboard. Отключите, когда хотите скрыть переменную, например, когда она используется как статическое значение в плитках, но не должна отображаться пользователю дашборда.

    • Если пользователи должны иметь возможность одновременно выбирать несколько значений в выпадающем списке переменной, включите Multi-select. Отключите, когда хотите использовать только одиночные значения в выпадающем списке переменной.
    • Выберите Default value для выпадающего списка переменной. Если вы не введёте ничего в это поле, будет выбрано первое доступное значение.
    • Когда закончите, выберите < Variable вверху, чтобы перейти на панель Variables, или выберите , чтобы закрыть панель Variables.

    Переменные в дашбордах могут зависеть от других переменных.

    • Значение переменной пересчитывается, если её определение ссылается на другую переменную и значение другой переменной изменяется.

    Например, если значение переменной A изменяется, значение любой переменной, определение которой ссылается на переменную A, пересчитывается. * Циклы не допускаются.

    Например, если значение переменной A зависит от значения переменной B, значение переменной B не может зависеть от значения переменной A.

    Способ редактирования переменной зависит от Type. Подробности и примеры см. в:

    • Переменная DQL
    • Переменная List
    • Переменная Code
    • Переменная Free Text

    Использование с плиткой DQL¶

    1. Откройте меню и выберите DQL.
    2. Скопируйте и вставьте следующий запрос в поле DQL и выберите Run.

    Обратите внимание, что запрос ссылается на нашу новую переменную Host как $Host, где знак доллара указывает, что это имя переменной.

    fetch logs, scanLimitGBytes: 20


| filter in(host.name, array($Host))


| makeTimeseries count(), by:{ host.name }

    или при использовании переменной без множественного выбора можно ссылаться на неё так

    fetch logs, scanLimitGBytes: 20


| filter host.name == $Host


| makeTimeseries count(), by:{ host.name }
    3. Поскольку вы добавили переменную с множественным выбором, вы можете использовать селектор значений для определения того, какие значения отображаются в ваших визуализациях. 4. Вы, конечно, можете использовать переменную в нескольких запросах.

    Использование с плиткой Explore¶

    Чтобы использовать переменную в плитке Explore data

    1. Откройте меню и выберите Logs.
    2. Выберите и затем выберите host.name из Available filters.
    3. В добавленном поле фильтра введите $, чтобы получить предложения для всех доступных переменных, и выберите $Host.

    Примечание: добавление переменных в плитках Explore работает только для переменных с одиночным выбором в сочетании с оператором =.

    Чтобы применить значения по умолчанию к переменным, откройте меню и выберите Reset to default.

    2. В столбце Name выберите имя переменной, которую хотите отредактировать.

    3. Отредактируйте и проверьте переменную. Способ редактирования переменной зависит от Type. Подробности и примеры см. в:

    4. Переменная DQL

    5. Переменная List
    6. Переменная Code
    7. Переменная Free Text
    8. Выберите , чтобы закрыть панель Variables.

    Удаление переменной¶

    Чтобы удалить переменную с дашборда

    1. Откройте меню и выберите All variables.
    2. Найдите переменную, которую хотите удалить, в столбце Name, откройте меню для этой строки и выберите Delete.
    3. Выберите , чтобы закрыть панель Variables.

    Изменение значений переменных¶

    Если на дашборде есть одна или несколько переменных, они перечислены по имени в левом верхнем углу дашборда под его названием. Когда вы изменяете значения переменных, содержимое дашборда пересчитывается и отображается в соответствии с новыми значениями.

    Чтобы изменить значение переменной

    1. В левом верхнем углу дашборда найдите имя переменной.

    2. Используйте меню или поле ввода под именем переменной для изменения значения.

    3. Если переменная допускает только одно выбранное значение, выберите значение, которое хотите применить к дашборду.

    4. Если переменная допускает несколько выбранных значений, установите флажок для каждого значения, которое хотите применить к дашборду. Название меню этой переменной показывает, сколько значений выбрано.
    5. Для переменной Free Text вы можете отредактировать текст в поле под именем переменной.

    Изменение порядка переменных¶

    Чтобы изменить порядок переменных на дашборде

    1. Откройте меню и выберите All variables.
    2. Перетащите переменные в нужном порядке.

    Альтернатива: откройте меню для переменной, которую хотите переместить, и выберите Move up или Move down

    Типы переменных¶

    Переменная DQL¶

    Чтобы определить переменную DQL

    1. Задайте Name — имя, которое хотите дать переменной.

    2. Оно будет отображаться в верхней части дашборда, в списке на панели Variables и использоваться в коде.

    3. Может содержать только буквы (верхнего или нижнего регистра) и цифры (например, status, myHosts или Variable01) и не может начинаться с dt_.
    4. Установите Type в DQL.
    5. В Data введите запрос. Обязательно используйте summarize и collectDistinct для получения уникальных значений из источников данных, таких как логи.
    6. Выберите Run и проверьте результаты в разделе Preview, чтобы убедиться, что всё работает как ожидается.
    7. Если хотите иметь возможность выбирать более одного значения одновременно, включите Multi-select.

    Если вы добавите оба приведённых ниже примера на дашборд, вы сможете фильтровать дашборд по хосту и уровню серьёзности логов. Обязательно включите Multi-select, если хотите выбирать более одного значения одновременно.

    Пример переменной DQL¶

    • Name: Hosts
    • Type: DQL
    • Definition:
    fetch dt.entity.host


| fields id

    Если вы хотите использовать удобочитаемое имя, используйте | fields entity.name вместо | fields id.

    Подстановка переменных¶

    В плитках DQL у вас есть четыре варианта обёртывания значения переменной и замены заполнителя переменной перед выполнением запроса. Эти варианты применяются только к плиткам DQL и игнорируются для плиток Code и Markdown.

    $varName

    по умолчанию

    Если вы не указываете стратегию (например, $varName), значение переменной оборачивается в символы двойных кавычек ("). Это полезно для строк, таких как имена сущностей или категории.

    • Все кавычки внутри переменных экранируются и игнорируются
    • Чтобы экранировать символ двойной кавычки, используйте двойной обратный слеш:
    \\"

    Например, для сервиса с именем PaymentBackend, в DQL-запросе, где мы фильтруем по этому сервису, со стратегией по умолчанию мы заменяем заполнитель переменной после = в правой части фильтра следующим образом:

    | filter dt.entity.service = "PaymentBackend"

    $varName:noquote

    Если вы добавляете :noquote после имени переменной (например, $varName:noquote), значение переменной не оборачивается в кавычки. Это полезно для чисел, единиц измерения или имён функций.

    • Допускаются только буквенно-цифровые символы и точка (.), подчёркивание (_) и дефис (-)
    • Запрос завершится ошибкой, если значения содержат другие символы

    Например, для сервиса с именем PaymentBackend, в DQL-запросе, где мы фильтруем по этому сервису, со стратегией без кавычек мы заменяем заполнитель переменной после = в правой части фильтра следующим образом:

    | filter dt.entity.service = PaymentBackend

    $varName:backtick

    Если вы добавляете :backtick после имени переменной (например, $varName:backtick), значение переменной оборачивается в обратные кавычки.

    • Чтобы экранировать обратную кавычку, используйте двойной обратный слеш:

    \\`
    * Все обратные кавычки внутри переменных экранируются и игнорируются * Пример использования: идентификаторы полей

    Например, для сервиса с именем PaymentBackend, в DQL-запросе, где мы фильтруем по этому сервису, со стратегией обратных кавычек мы заменяем заполнитель переменной после = в правой части фильтра следующим образом:

    | filter dt.entity.service = `PaymentBackend`

    $varName:triplequote

    Если вы добавляете :triplequote после имени переменной (например, $varName:triplequote), значение переменной оборачивается в символы """. Это полезно для необработанного содержимого, такого как JSON, без экранирования.

    • Ничего не экранируется внутри значения
    • Запрос завершится ошибкой, если значение содержит """

    Например, для сервиса с именем PaymentBackend, в DQL-запросе, где мы фильтруем по этому сервису, со стратегией тройных кавычек мы заменяем заполнитель переменной после = в правой части фильтра следующим образом:

    | filter dt.entity.service = """PaymentBackend"""

    Ограничения при использовании переменных в плитках¶

    • Переменные в плитках могут быть строкового или числового типа. Случаи, требующие другого типа данных (например, duration), приводят к ошибкам запроса. Ниже приведены некоторые примеры обходных решений для таких ситуаций.
    • По соображениям безопасности при использовании переменных в плитках кода вы можете получить к ним доступ только внутри функции по умолчанию.

    Разрешение в команде DQL¶

    Прямое использование переменной $resolution (как в следующем запросе) не работает, потому что resolution требует предопределённого формата данных, тогда как переменная возвращает строковое значение.

    fetch logs


| ...


| summarize count(), by: {loglevel, bin(timestamp, $resolution)}

    В качестве обходного решения можно использовать функцию duration вместе с функцией конвертации DQL. Это обеспечивает требуемый вывод на основе значения $resolution.

    fetch logs


| ...


| summarize count(), by: {loglevel, bin(timestamp, duration(toLong($resolution), unit:"m"))}

    Преобразование значений переменных в другие типы данных¶

    Если вы хотите отфильтровать числовое значение, но сравнить его со строковым представлением, можно использовать нативную функцию конвертации DQL, такую как toString.

    fetch logs


| filter amount = toString($amount)


| ...

    Максимальный общий размер переменных в URL дашборда¶

    Если значения переменных превышают 30 КБ, они не могут быть сохранены в URL дашборда.

    • Если вы делитесь URL дашборда со значениями переменных, превышающими лимит размера, значения переменных не будут сохранены в URL, поэтому человек, открывающий дашборд по общему URL, не увидит выбранные значения.
    • Если вы сохраняете URL дашборда как закладку со значениями переменных, превышающими лимит размера, и открываете дашборд из этой закладки через 90 дней, выбранные значения не будут установлены.

    Устранение проблем с переменными¶

    Дашборд с одной или несколькими определёнными переменными отображает переменные в строке под названием дашборда. В этом примере вы можете видеть:

    • Меню переменных
    • Четыре переменные: LogLevels, MyFreeTextVariable, Variable1 и Variable2
    • Иконку предупреждения

    Чтобы исследовать предупреждение, выберите иконку предупреждения (или выберите > All variables), чтобы просмотреть все переменные.

    • Снова четыре переменные: LogLevels, MyFreeTextVariable, Variable1 и Variable2.
    • Две из этих переменных — Variable1 и Variable2 — отображают иконку предупреждения , указывающую на возможные проблемы с каждой из них. Чтобы узнать причину предупреждения для конкретной переменной, выберите её.

    В этом примере мы выбрали Variable1, чтобы отобразить определение переменной и любое связанное сообщение об ошибке или предупреждение. Здесь мы видим, что забыли сослаться на переменную где-либо.
