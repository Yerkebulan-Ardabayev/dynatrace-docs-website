    Визуализация категориальной диаграммы¶

    • Latest Dynatrace
    • 2 мин. чтения

    Когда использовать визуализацию категориальной столбчатой диаграммы

    Используйте категориальную столбчатую диаграмму:

    • Когда вы хотите сгруппировать данные по определённым категориям и сравнить их агрегированные значения, а не просматривать их во времени. Это особенно полезно, когда у вас более пяти отдельных категорий, и визуализация в виде круговой или кольцевой диаграммы становится недостаточной для сравнения различных пропорций.
    • Когда у вас есть данные о частоте или количестве для отдельных категорий. Например, запросы по коду ответа, ошибки по типу ошибки или строки логов по статусу.

    Пример¶

    Приведённая выше категориальная столбчатая диаграмма основана на следующем запросе.

    fetch logs


| summarize count(), by:{loglevel}

    В разделе Categorical bar chart options параметр Category axis label установлен на loglevel для отображения подписи по оси X.

    Эта категориальная столбчатая диаграмма показывает значения по вертикальной оси и категории по горизонтальной оси.

    Взаимодействие с диаграммой¶

    Взаимодействие при выборе¶

    Когда вы выбираете значение на диаграмме и закрепляете отображаемую подсказку, вы можете затем навести курсор на подсказку, чтобы отобразить меню с опциями, специфичными для выбора.

    Доступные взаимодействия с диаграммой зависят от вашего запроса и визуализации. Например, если вы выберете хост на линейной диаграмме и наведёте курсор на подсказку, вы увидите меню с такими элементами, как:

    • Copy name — копирование имени выбранного хоста.

    • Fields — раздел с подменю для каждого поля запроса. Подменю поля предлагает специфичные для поля опции, такие как:

    • Copy value — копирование значения поля. Также отображает тип поля.

    • Hide — скрытие поля на диаграмме.
    • Explain value — использование ИИ для объяснения поля.
    • Add command to query — раздел с командами, специфичными для поля, которые можно автоматически добавить в запрос.
    • Также может быть указано рекомендованное приложение.
    • Visual options — открытие панели редактирования для изменения параметров визуализации выбранного элемента.
    • Set color — открытие панели редактирования для изменения цвета выбранного элемента.
    • Go to host — открытие выбранного элемента в Infrastructure & Operations.

    В общем, если для выбранного элемента есть рекомендованные приложения, меню предлагает прямые ссылки на эти приложения, а затем опцию Open with для выбора другого целевого приложения. * Open with — подробнее см. Детализация и навигация.

    Заголовок¶

    Используйте поле заголовка в верхней части панели опций (изначально Untitled tile или Untitled section) для добавления заголовка к плитке дашборда или разделу блокнота.

    • Вы можете использовать переменные.

    Пример:

    1. Определите переменные Status и Emoji в вашем дашборде.
    2. Установите заголовок Current $Emoji status is $Status.
    3. Установите Status в значение Good.
    4. Установите Emoji в желаемое значение.

    Заголовок будет отображён соответствующим образом.

    Визуализация¶

    Если вы не уверены, что выбрали правильную визуализацию, используйте селектор визуализации для выбора различных визуализаций.

    Маппинг данных¶

    Раздел маппинга данных показывает, как столбец вашего результата сопоставлен с визуализацией.

    Общие правила настройки маппинга данных

    Разверните раздел Data mapping в настройках визуализации, чтобы увидеть, как данные в вашем результате сопоставлены с визуализацией, и при необходимости скорректировать эти настройки.

    • Обязательные поля отмечены звёздочкой (*). Пример:

    * Типы данных отображаются рядом с именами полей в выпадающих списках и сопоставленных полях. * Единицы измерения отображаются, когда назначена только одна. * Поля результатов сгруппированы в Suitable и Unsuitable. Поля помечаются как неподходящие, если их нельзя использовать для отображения данных в визуализации. Пример:

    * Автоматическое применение настроек маппинга данных по умолчанию:

    Dynatrace версии 1.319+

    • Уже существующие плитки и разделы считаются определёнными пользователем. Их конфигурации маппинга данных не обновляются автоматически.
    • Вновь созданные плитки и разделы применяют настройки маппинга данных по умолчанию. Если вы не изменяете эти настройки вручную, они могут измениться при новом выполнении плитки/раздела, если результаты изменятся и появятся отсутствующие или новые поля, лучше подходящие для маппинга данных.

    Настройки маппинга данных для визуализации¶

    Для категориальной столбчатой диаграммы раздел маппинга данных включает:

    • Values: значение (числовое) для отображения. Если указано несколько значений, вы можете выбрать более одного одновременно.
    • Categories: столбец результата, отображаемый как категория.

    Опции категориальной диаграммы¶

    Чтобы быстро изучить опции и решить, что лучше всего подходит, включайте и выключайте опции и сразу наблюдайте эффект на диаграмме. Например, лучше ли выглядит с подписью или без? Включите и выключите эту опцию и убедитесь сами.

    Оси¶

    • Category axis label
    • Value axis label
    • Min и Max
    • Relative values

    Общие¶

    • Color mode — выбор многоцветного или одноцветного режима.
    • Color palettes — выбор цветовой палитры из списка.

    • Legend position — выбор расположения легенды визуализации:

    • Automatic — Dynatrace выбирает подходящее расположение

    • Hidden — легенда не отображается
    • Bottom — легенда отображается внизу визуализации
    • Right — легенда отображается справа от визуализации

    • Vertical layout — определяет ориентацию диаграммы:

    • Horizontal — столбцы ориентированы горизонтально, слева направо

    • Vertical — столбцы ориентированы вертикально, снизу вверх

    • Category tick label layout — определяет ориентацию подписей делений категорий:

    • Horizontal — подписи делений категорий ориентированы горизонтально, слева направо

    • Vertical — подписи делений категорий ориентированы вертикально, снизу вверх

    Переопределения¶

    • Category colors — для переопределения цвета категории выберите из списка.

    Лимиты запросов¶

    Используйте раздел Query limits для проверки и настройки лимитов запросов Grail для каждого раздела блокнота или плитки дашборда. Эти настройки определяют максимальные лимиты при получении данных. Превышение любого лимита приведёт к предупреждению.

    Плитки дашбордов и разделы блокнотов, созданные в Dynatrace ранее версии 1.296, не затрагиваются. Эти существующие плитки/разделы будут возвращать те же результаты, что и раньше.

    • Read data limit (GB)

    Лимит в гигабайтах для объёма данных, которые будут сканированы при чтении. * Record limit

    Максимальное количество записей результатов, которое вернёт этот запрос. По умолчанию: 1000 записей. Чтобы увидеть больше записей, необходимо увеличить значение Record limit.

    • Если ваш запрос не содержит limit, например

      fetch logs

      применяется значение Record limit. По умолчанию вы увидите до 1000 записей. + Если ваш запрос также включает limit, например

      fetch logs


| limit 2000

      применяется меньшее из двух значений (либо limit в запросе, либо Record limit в веб-интерфейсе).

      В приведённом выше примере вы всё равно увидите только 1000 записей, если не увеличите значение Record limit. * Result size limit

    Максимальное количество байтов результата, которое вернёт этот запрос. Для лучшей производительности с типичными запросами и небольшими документами значение по умолчанию установлено на 1 МБ. * Sampling (только для Logs и Spans)

    Приводит к выборке подмножества записей Log или Span.

    Пороговые значения¶

    Настройка пороговых значений¶

    Для настройки пороговых значений в визуализации дашборда или блокнота

    1. Выберите для редактирования плитки визуализации.

    2. Выберите Thresholds.

    3. В Dashboards выберите вкладку Visual на боковой панели, затем выберите Thresholds.

    4. В Notebooks выберите Thresholds на боковой панели.
    5. Выберите Threshold.

    6. Определите пороговые значения. Для каждого диапазона:

    7. Выберите цвет для отображения этого диапазона

    8. Выберите оператор для определения порога этого диапазона
    9. Введите статическое значение для сравнения (с использованием выбранного оператора) с возвращаемым значением

    10. Введите метку для связи с определённым диапазоном.

      • Метки применяются только к диаграммам с осью Y, таким как диаграммы временных рядов и категориальная столбчатая диаграмма.
      • Метки не могут быть определены для таблиц и диаграммы одиночного значения.

    Пример настройки пороговых значений в Dashboards

    В этом примере используется столбчатая диаграмма, но опции применимы к другим визуализациям.

    1. В Dashboards создайте дашборд.
    2. Выберите и в разделе Snippets меню выберите Metrics > Chart top 10 hosts by CPU usage.
    3. На панели редактирования раздела выберите вкладку Visual и выберите Bar.
    4. Выберите Thresholds.
    5. Выберите Threshold.

    Отображается пустой набор полей пороговых значений. 6. Определите пороговые значения для отображаемой метрики. Вы можете наблюдать изменения на оси Y диаграммы.

    В этом примере мы определяем три диапазона использования CPU с соответствующими цветами и метками.

    Диапазоны отображаются на оси Y и в подсказке.

    Для сброса к значениям по умолчанию (отмена настроек пороговых значений) выберите значок корзины рядом с элементом.

    Редактирование пороговых значений¶

    1. Выберите для редактирования плитки визуализации.

    2. Выберите Thresholds.

    3. В Dashboards выберите вкладку Visual на боковой панели, затем выберите Thresholds.

    4. В Notebooks выберите Thresholds на боковой панели.
    5. После этого вы можете выполнить следующие действия (разверните строки для подробностей):

    Добавление пороговых значений

    Для добавления пороговых значений (если выбранная визуализация поддерживает дополнительные пороги) выберите Threshold.

    Отключение пороговых значений

    Для отключения (скрытия) существующих пороговых значений используйте переключатель. Настройки не теряются. Вы можете включить их обратно, если передумаете.

    Удаление пороговых значений

    Для удаления существующих пороговых значений выберите значок корзины.

    Редактирование настроек пороговых значений

    Для изменения существующих настроек пороговых значений просто отредактируйте поля.

    Добавление диапазона

    Для добавления диапазона к существующим пороговым значениям выберите Add range.

    Удаление диапазона

    Для удаления диапазона из существующих пороговых значений выберите кнопку удаления в соответствующей строке.

    Единицы измерения и форматы¶

    Для переопределения единиц измерения и форматов по умолчанию в визуализации дашборда или блокнота

    1. Выберите для редактирования плитки визуализации.
    2. Выберите вкладку Visual.
    3. Выберите Units and formats.
    4. Выберите Override.
    5. Выберите Override
    6. В выпадающем списке выберите элемент, для которого вы хотите добавить переопределение единицы.

    Это числовой столбец базового результата DQL, поэтому он варьируется в зависимости от запроса. Например:

    • Запрос fetch events возвращает события. Выпадающий список позволяет выбрать числовое поле (например, transfer_size) из результатов.
    • Запрос timeseries avg(dt.host.cpu.usage) возвращает один временной ряд для avg(dt.host.cpu.usage). Этот временной ряд является единственным доступным для выбора вариантом в списке.

    • Определите переопределение.

    • Default unit: Базовая единица, в которой были захвачены значения. Это None, если она не была включена в результат DQL, или автоматически определяется по единице, переданной из результата DQL. Это поле не приводит к конверсии.

    • Displayed unit: После определения базовой единицы вы можете использовать Displayed unit для конверсии. Например, если результат DQL определил ваше числовое значение в Bytes, Displayed unit предлагает подходящий список конверсий в байтах, такой как Kilobyte и Megabyte. В отличие от Default unit, Displayed unit всегда является числовой конверсией.
    • Decimals отображает количество десятичных знаков (степень точности) по умолчанию. Чтобы увидеть это в действии, измените выбор Decimals и наблюдайте изменения в визуализации.
    • Suffix отображает суффикс, добавляемый после единицы. Чтобы увидеть это в действии, введите строку и наблюдайте изменения в визуализации. Когда вы не можете найти нужную единицу, используйте Suffix для отображения желаемой единицы.
    • Включите Abbreviate large numbers, если хотите отображать большие числа в сокращённой форме. Например, 1053 становится 1.1K.

    Для сброса к значениям по умолчанию (отмена настроек переопределения для выбранного элемента) выберите значок корзины рядом с элементом.
