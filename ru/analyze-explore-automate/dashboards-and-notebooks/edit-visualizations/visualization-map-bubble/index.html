    Визуализация пузырьковой карты¶

    • Актуальная версия Dynatrace

    Dashboards версии 1.311+ Notebooks версии 1.311+

    Когда использовать

    Используйте пузырьковую карту для представления значений с помощью пузырьков различного размера и цвета. Примеры включают частоту ошибок различных сервисов, где размер пузырька отражает частоту, а цвет указывает на серьёзность, и распределение дохода по регионам, где размер пузырька отражает доход, а цвет указывает на темп роста.

    Примеры¶

    Чтобы попробовать пример

    1. Создайте DQL-плитку в Dashboards или DQL-секцию в Notebooks.
    2. Скопируйте пример данных и вставьте его в поле редактирования DQL.
    3. Выполните запрос.
    4. Выберите визуализацию и поэкспериментируйте с настройками визуализации.

    Взаимодействие с диаграммой¶

    Взаимодействие при выборе¶

    Когда вы выбираете значение на диаграмме и закрепляете отображаемую всплывающую подсказку, вы можете навести курсор на подсказку, чтобы отобразить меню с опциями для выбранного элемента.

    Доступные взаимодействия зависят от вашего запроса и визуализации. Например, если вы выбираете хост на линейной диаграмме и наводите курсор на подсказку, вы увидите меню с элементами, такими как:

    • Copy name — копировать имя выбранного хоста.

    • Fields — раздел с подменю для каждого поля запроса. Подменю поля предлагает специфичные для поля опции, такие как:

    • Copy value — копировать значение поля. Также отображает тип поля.

    • Hide — скрыть поле на диаграмме.
    • Explain value — использовать ИИ для объяснения поля.
    • Add command to query — раздел специфичных для поля команд, которые можно автоматически добавить к запросу.
    • Также может быть указано рекомендованное приложение.
    • Visual options — открывает панель редактирования для изменения параметров визуализации выбранного элемента.
    • Set color — открывает панель редактирования для изменения цвета выбранного элемента.
    • Go to host — открывает выбранный элемент в Infrastructure & Operations.

    В общем случае, если есть рекомендованные приложения для открытия выбранного элемента, меню предлагает прямые ссылки на эти приложения, а затем опцию Open with для выбора другого целевого приложения. * Open with — подробнее см. в разделе Переходы и навигация.

    Заголовок¶

    Используйте поле заголовка в верхней части панели параметров (изначально Untitled tile или Untitled section), чтобы добавить заголовок к плитке дашборда или секции ноутбука.

    • Можно использовать эмодзи, такие как 🟢 и 🔴 и ❤️.
    • Можно использовать переменные.

    Пример:

    1. Определите переменные Status и Emoji в вашем дашборде.
    2. Установите заголовок Current $Emoji status is $Status.
    3. Установите значение Status равным Good.
    4. Установите значение Emoji равным 🟢.

    Заголовок будет отображаться как Current 🟢 status is Good.

    Визуализация¶

    Если вы не уверены, что выбрали правильную визуализацию, используйте селектор визуализации, чтобы попробовать различные визуализации.

    Представление¶

    • Default zoom

    Установите уровень масштабирования по умолчанию для карты, выбрав один из следующих вариантов:

    • Data: автоматически подстраивает уровень масштабирования для отображения всех точек данных на карте.
    • World: устанавливает уровень масштабирования для отображения всего мира.
    • Custom: позволяет указать координаты центра карты и задать пользовательский уровень масштабирования.
    • Show country regions

    Включите для отображения контуров регионов внутри стран.

    Сопоставление данных¶

    Раздел сопоставления данных показывает, как столбец вашего результата сопоставлен с визуализацией.

    Развернуть для общих правил настройки сопоставления данных

    Разверните раздел Data mapping в настройках визуализации, чтобы увидеть, как данные вашего результата сопоставлены с визуализацией, и при необходимости скорректировать эти настройки.

    • Обязательные поля отмечены звёздочкой (*). Пример:

    * Типы данных отображаются рядом с именами полей в выпадающих списках и сопоставленных полях. * Единицы измерения отображаются, если назначена только одна. * Поля результатов группируются в категории Suitable и Unsuitable. Поля помечаются как неподходящие, если их нельзя использовать для отображения данных в визуализации. Пример:

    * Автоматическое применение настроек сопоставления данных по умолчанию:

    Dynatrace версии 1.319+

    • Уже существующие плитки и секции считаются пользовательскими. Их конфигурации сопоставления данных не обновляются автоматически.
    • Вновь созданные плитки и секции применяют настройки сопоставления данных по умолчанию. Если вы не изменяете эти настройки вручную, они могут измениться при новом выполнении плитки/секции, если результаты изменились и появились недостающие или более подходящие поля.

    Настройки сопоставления данных для конкретной визуализации¶

    Для визуализации пузырьковой карты раздел сопоставления данных включает:

    • Latitude: Обязательное. Выберите значение из запроса для использования в качестве широты каждой точки на карте.
    • Longitude: Обязательное. Выберите значение из запроса для использования в качестве долготы каждой точки на карте.
    • Radius value: Обязательное. Выберите значение из запроса для использования в качестве радиуса пузырька вокруг каждой точки на карте.
    • Color value: Необязательное. Выберите значение из запроса для использования в качестве цвета пузырька вокруг каждой точки на карте.

    Радиус¶

    Настройки радиуса определяют размер пузырька.

    • Zoom resize: изменение размера пузырька при масштабировании. По умолчанию: выключено.

    • Scale: определяет способ масштабирования радиуса пузырька.

    • Fixed: не масштабировать радиус пузырька.

    • Linear: масштабировать радиус пузырька линейно (по умолчанию).
    • Log: масштабировать радиус пузырька логарифмически.
    • Min radius: минимальный радиус пузырька. По умолчанию: 10.
    • Max radius: максимальный радиус пузырька. По умолчанию: 100.

    Легенда и подсказка¶

    • Show custom fields: для отображения пользовательских полей (имя и значение) при наведении на область карты включите Show custom fields и выберите каждое пользовательское поле, которое хотите отобразить.

    • Show legend: для отображения легенды карты включите Show legend и выберите Position легенды:

    • Auto: выбирает подходящее расположение в зависимости от размера карты и доступного пространства.

    • Bottom: отображает легенду под картой.
    • Right: отображает легенду справа от карты.

    • Text truncation: определяет способ обрезки текста, когда полный текст не может быть отображён.

    • A...: обрезка с правого конца текста (когда правый конец менее важен)

    • A...B: обрезка из середины текста (когда середина менее важна)
    • ...B: обрезка с левого конца текста (когда левый конец менее важен)

    • Min value: устанавливает минимальное значение в данных.

    • Auto: автоматически выбирает подходящий минимум на основе данных (0 или минимальное значение).

    • Custom: позволяет задать пользовательское минимальное значение.

    • Max value: устанавливает максимальное значение в данных.

    • Auto: автоматически выбирает подходящий максимум на основе данных (0 или максимальное значение). Custom позволяет задать пользовательское максимальное значение.

    • Custom: позволяет задать пользовательское максимальное значение.

    Цвет¶

    • Bubble colors

    Выберите способ окрашивания пузырьков:

    • Color palette: отображает все пузырьки в оттенке из выбранной цветовой палитры. Оттенок, используемый для каждого пузырька, соответствует значению Color value относительно других областей.

      Пример

      Если значения Color value, возвращённые вашим запросом, находятся в диапазоне от 0 до 100

      • Пузырёк со значением около 0 имеет оттенок цвета ближе к правому концу палитры.
      • Пузырёк со значением около 100 имеет оттенок цвета ближе к левому концу палитры.
      • Single-color: отображает все пузырьки одним цветом. Выберите цвет из списка или введите шестнадцатеричный код цвета.
      • Custom colors: отображает каждый пузырёк пользовательским цветом, определённым вами.

      Для каждого пользовательского цвета, который вы хотите добавить

      1. Выберите Color.
      2. Введите значение, оператор и цвет для использования с этим значением и оператором.

      Пример

      Предположим, вы хотите окрасить пузырьки по трём уровням Color value:

      • Зелёный, если Color value меньше 4 000
      • Жёлтый, если Color value достигает или превышает порог 4 000
      • Красный, если Color value достигает или превышает порог 5 000

      Для настройки

      • Выберите Color и добавьте строку пользовательского цвета со значением 0, оператором >= и нужным оттенком зелёного. Если Color value равно 0 или выше, пузырёк будет зелёным.
      • Выберите Color и добавьте строку пользовательского цвета со значением 4 000, оператором >= и нужным оттенком жёлтого. Если Color value равно 4 000 или выше, пузырёк будет жёлтым.
      • Выберите Color и добавьте строку пользовательского цвета со значением 5 000, оператором >= и нужным оттенком красного. Если Color value равно 5 000 или выше, пузырёк будет красным.

    Лимиты запросов¶

    Используйте раздел Query limits для проверки и настройки лимитов запросов Grail для каждой секции ноутбука или плитки дашборда. Эти настройки определяют максимальные лимиты при получении данных. Превышение любого лимита приведёт к генерации предупреждения.

    Плитки дашбордов и секции ноутбуков, созданные в Dynatrace ранее версии 1.296, не затрагиваются. Эти существующие плитки/секции будут возвращать те же результаты, что и раньше.

    • Read data limit (GB)

    Лимит в гигабайтах для объёма данных, которые будут просканированы при чтении. * Record limit

    Максимальное количество записей результата, которое вернёт данный запрос. По умолчанию: 1 000 записей. Чтобы увидеть больше записей, нужно увеличить значение Record limit.

    • Если ваш запрос не содержит limit, например

      fetch logs

      применяется значение Record limit. По умолчанию вы увидите до 1 000 записей. + Если ваш запрос также включает limit, например

      fetch logs


| limit 2000

      применяется меньшее из двух значений (либо limit в вашем запросе, либо Record limit в веб-интерфейсе).

      В приведённом выше примере вы всё равно увидите только 1 000 записей, если не увеличите значение Record limit. * Result size limit

    Максимальное количество байт результата, которое вернёт данный запрос. Для лучшей производительности при типичных запросах и небольших документах значение по умолчанию составляет 1 МБ. * Sampling (Logs and Spans only)

    Приводит к выборке подмножества записей Log или Span.

    Единицы измерения и форматы¶

    Чтобы переопределить единицы измерения и форматы по умолчанию в визуализации дашборда или ноутбука

    1. Выберите для редактирования плитки визуализации.
    2. Выберите вкладку Visual.
    3. Выберите Units and formats.
    4. Выберите Override.
    5. Выберите Override
    6. В выпадающем списке выберите элемент, для которого вы хотите добавить переопределение единицы измерения.

    Это числовой столбец базового результата DQL, поэтому он зависит от запроса. Например:

    • Запрос fetch events возвращает события. Выпадающий список здесь позволяет выбрать числовое поле (например, transfer_size) из результатов.
    • Запрос timeseries avg(dt.host.cpu.usage) возвращает один временной ряд для avg(dt.host.cpu.usage). Этот временной ряд является единственным доступным вариантом в списке.

    • Определите переопределение.

    • Default unit: базовая единица, в которой были зафиксированы значения. Это None, если она не была включена в результат DQL, или автоматически определяется единицей из результата DQL. Это поле не приводит к преобразованию.

    • Displayed unit: после определения единицы по умолчанию вы можете использовать Displayed unit для преобразования. Например, если результат DQL определил ваше числовое значение как Bytes, Displayed unit теперь предлагает подходящий список преобразований байтов, таких как Kilobyte и Megabyte. В отличие от Default unit, Displayed unit всегда является числовым преобразованием.
    • Decimals отображает количество десятичных знаков по умолчанию (степень точности). Чтобы увидеть это в действии, измените параметр Decimals и наблюдайте за изменением визуализации.
    • Suffix отображает суффикс после единицы измерения. Чтобы увидеть это в действии, введите строку и наблюдайте за изменением визуализации. Когда вы не находите нужную единицу, можно использовать Suffix для отображения желаемой единицы.
    • Включите Abbreviate large numbers, если хотите отображать большие числа в сокращённом виде. Например, 1053 становится 1.1K.

    Для сброса к значениям по умолчанию (отмена настроек переопределения для выбранного элемента) выберите значок корзины рядом с элементом.
