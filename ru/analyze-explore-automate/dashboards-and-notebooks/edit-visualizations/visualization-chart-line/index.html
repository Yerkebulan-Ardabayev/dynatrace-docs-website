    Визуализация линейной диаграммы

    Когда использовать визуализацию линейной диаграммы

    Используйте визуализацию линейной диаграммы:

    • Когда вы хотите анализировать измерения и их тренд во времени. Например, чтобы увидеть, быстро ли заканчивается ёмкость на всех дисках.
    • Когда вы хотите сравнить несколько серий во времени. Например, чтобы увидеть, одинаково ли загружены все ваши поды.
    • Когда вы хотите найти взаимосвязи или корреляции между несколькими сериями, представляющими разные переменные. Например, чтобы проанализировать, влияют ли определённые ошибки приложения напрямую на количество бронирований.

    Примеры¶

    Пример 1¶

    Диаграмма выше основана на следующем запросе.

    timeseries avg(dt.host.cpu.usage)
    • Тип визуализации установлен на Линейная диаграмма.
    • Остальные параметры установлены по умолчанию.

return {

'my series 2': Math.exp((index * 2) / 8),


timeframe: {


start: time - 1000 * 60 * invertedIndex,


end: time - 1000 * 60 * (invertedIndex - 1),


},


};


}),


types: [


{


indexRange: [0, sampleCount - 1],


mappings: {


timeframe: { type: 'timeframe' },


'my series': { type: 'double' },


'my series 2': {type: 'double'}


},


},


],


};


}

    Взаимодействие с диаграммой¶

    Для доступа к инструментам диаграммы:

    • Когда пространство ограничено, наведите курсор на диаграмму и выберите Инструменты диаграммы, чтобы открыть меню инструментов.
    • Когда достаточно места, инструменты диаграммы отображаются на панели инструментов.

    При наведении курсора на диаграмму панель инструментов по умолчанию отображается в правом нижнем углу диаграммы в свёрнутом виде.

    Параметры панели инструментов¶

    Правила масштабирования¶

    Для диаграмм временных рядов (Линейная, С областями, Столбчатая):

    • Вы не можете масштабировать диаграмму, если временной диапазон задан в запросе DQL.
    • Dashboards

    Если вы масштабируете плитку без пользовательского временного диапазона, глобальный временной диапазон дашборда обновляется соответственно, поэтому временные диапазоны остаются синхронизированными по всему дашборду, за исключением плиток с собственным переопределяющим временным диапазоном.

    Если вы масштабируете плитку с пользовательским временным диапазоном, но без временного диапазона в запросе, выбранный временной диапазон применяется к пользовательскому временному диапазону плитки, и изменённый временной диапазон отражается в дашборде. * Notebooks

    Если вы масштабируете временной ряд в секции блокнота, временной диапазон секции изменяется, и изменение сохраняется в блокноте.

    Взаимодействие при выделении¶

    Когда вы выбираете значение на диаграмме и закрепляете отображаемую всплывающую подсказку, вы можете навести курсор на подсказку, чтобы отобразить меню с параметрами, специфичными для выделения.

    Доступные взаимодействия с диаграммой зависят от вашего запроса и визуализации. Например, если вы выберете хост на линейной диаграмме и наведёте курсор на всплывающую подсказку, вы увидите меню с пунктами:

    • Копировать имя — копирует имя выбранного хоста.

    • Поля — раздел с подменю для каждого поля запроса. Подменю поля предлагает специфичные для поля параметры:

    • Копировать значение — копирует значение поля. Также отображает тип поля.

    • Скрыть — скрывает поле на диаграмме.
    • Объяснить значение — использует ИИ для объяснения поля.
    • Добавить команду к запросу — раздел команд, специфичных для поля, которые можно автоматически добавить в запрос.
    • Также может быть указано рекомендованное приложение.
    • Визуальные параметры — открывает панель редактирования для изменения параметров визуализации выбранного элемента.
    • Установить цвет — открывает панель редактирования для изменения цвета выбранного элемента.
    • Перейти к хосту — открывает выбранный элемент в Infrastructure & Operations.

    В целом, если есть рекомендованные приложения для открытия выбранного элемента, меню предлагает прямые ссылки на эти приложения, за которыми следует параметр Открыть с помощью для выбора другого целевого приложения. * Открыть с помощью — подробнее см. Детализация и навигация.

    Заголовок¶

    Используйте поле заголовка в верхней части панели параметров (изначально Untitled tile или Untitled section), чтобы добавить заголовок к плитке дашборда или секции блокнота.

    • Вы можете использовать эмодзи, такие как ð и ð и â¤ï¸.
    • Вы можете использовать переменные.

    Пример:

    1. Определите переменные Status и Emoji в вашем дашборде.
    2. Установите заголовок Current $Emoji status is $Status.
    3. Установите Status в значение Good.
    4. Установите Emoji в значение ð.

    Заголовок будет отображаться как Current ð status is Good.

    Визуализация¶

    Если вы не уверены, что выбрали правильную визуализацию, используйте селектор визуализации, чтобы попробовать разные визуализации.

    Сопоставление данных¶

    Раздел сопоставления данных показывает, как столбец вашего результата сопоставляется с визуализацией.

    Например, при работе с DQL и временным рядом в результате имеется столбец с массивом значений для каждого ряда (одна строка с одной ячейкой значений). Для временного ряда это сопоставление выполняется полностью автоматически, но раздел сопоставления данных показывает, какой базовый столбец сопоставлен.

    Развернуть: общие правила настройки сопоставления данных

    Разверните раздел Сопоставление данных в настройках визуализации, чтобы увидеть, как данные результата сопоставляются с визуализацией, и при необходимости настроить эти параметры.

    • Обязательные поля отмечены звёздочкой (*). Пример:

    * Типы данных отображаются рядом с именами полей в раскрывающихся списках и сопоставленных полях. * Единицы измерения отображаются, когда назначена только одна. * Поля результатов сгруппированы в Подходящие и Неподходящие. Поля отмечаются как неподходящие, если они не могут использоваться для отображения данных в визуализации. Пример:

    * Автоматическое применение настроек сопоставления данных по умолчанию:

    Dynatrace версии 1.319+

    • Уже существующие плитки и секции считаются пользовательскими. Их конфигурации сопоставления данных не обновляются автоматически.
    • Вновь созданные плитки и секции применяют настройку сопоставления данных по умолчанию. Если вы не изменяете эти настройки вручную, они могут измениться, если новое выполнение плитки/секции изменит результаты и появятся отсутствующие поля или новые поля, которые лучше подходят для сопоставления данных.

    Ограничения сопоставления данных для графических визуализаций на основе событий

    Dynatrace версии 1.322+

    Что изменилось?¶

    Чтобы сделать сопоставление данных проще и интуитивнее, мы ограничили определённые поля в интерфейсе для диаграмм на основе событий, показывая только наиболее релевантные параметры.

    • До: До версии Dynatrace 1.322 поля, такие как timestamp, interval и duration, могли быть сопоставлены с параметром «имена» в интерфейсе сопоставления данных.
    • После: Начиная с версии Dynatrace 1.322, эти поля больше недоступны для выбора в параметре «имена».

    Влияние¶

    Если ваши диаграммы в Dashboards или Notebooks используют эти поля, затронутые плитки будут отображать сообщение об ошибке, например:

    Selected field unavailable. The field "" is no longer available. Try adjusting the data, or select another field or visualization.

    Решение¶

    Для решения этой проблемы

    1. Замените существующую команду summarize на makeTimeseries.
    2. Замените параметр by: { ... 4h } на interval: 4h, используя тот же интервал, что и в предыдущей конфигурации в by.

    После применения этих изменений сопоставление данных будет корректно разрешать подходящие поля для параметра «имена», и ваши плитки будут работать как ожидается.

    Настройки сопоставления данных для конкретной визуализации¶

    Линейная диаграмма отображает одно или несколько значений во времени, поэтому сопоставление должно включать следующее:

    • Время: столбец результата, используемый для оси X (timestamp или timeframe).
    • Интервал: это значение сопоставляется автоматически и не может быть изменено. Оно показывает, какие поля сопоставлены для результатов на основе временных рядов. Берётся первое доступное поле интервала из набора результатов при использовании временного ряда (также включает любые данные на основе makeTimeseries).
    • Значения: выбор одного или нескольких значений, которые ваша диаграмма отображает во времени.
    • Имена: элементы, отображаемые, например, в легенде и именах серий.

    Параметры линейной диаграммы¶

    Чтобы быстро узнать о параметрах и решить, что лучше всего подходит, включайте и выключайте параметры и сразу видите эффект на диаграмме. Например, лучше ли с меткой или без? Включите и выключите этот параметр и убедитесь сами.

    Цветовые палитры¶

    Вы можете выбрать цветовую палитру из списка в разделе Цветовые палитры.

    Для линейной диаграммы, диаграммы с областями или столбчатой диаграммы вы можете по желанию переопределить выбранную цветовую палитру для любой серии.

    Положение легенды¶

    Выберите, где отображать легенду визуализации:

    • Автоматически — Dynatrace выбирает подходящее расположение
    • Скрыта — легенда не отображается
    • Снизу — легенда отображается внизу визуализации
    • Справа — легенда отображается справа от визуализации

    Поля легенды¶

    Выберите, какие элементы отображать в легенде.

    Пустые значения¶

    Соединять пустые значения — указывает, следует ли соединять пустые значения в визуализации.

    Ось времени¶

    Выбирает ось X.

    Левая ось¶

    Устанавливает масштаб левой оси:

    • Логарифмический
    • Линейный

    Минимум¶

    Минимум диапазона визуализации.

    • Авто — Dynatrace автоматически выбирает подходящий минимум на основе данных (0 или минимальное значение).
    • Минимальное значение — минимум устанавливается равным минимальному значению данных.
    • Пользовательский — отображает поле для ввода пользовательского минимального значения.

    Максимум¶

    Максимум диапазона визуализации.

    • Авто — Dynatrace автоматически выбирает подходящий максимум на основе данных (0 или максимальное значение).
    • Максимальное значение — максимум устанавливается равным максимальному значению данных.
    • Пользовательский — отображает поле для ввода пользовательского максимального значения.

    Метка¶

    Вертикальный текст, отображаемый как метка для оси Y.

    Переопределения серий¶

    Для линейной диаграммы, диаграммы с областями или столбчатой диаграммы вы можете по желанию переопределить выбранную цветовую палитру для любой серии.

    Раздел Переопределения серий перечисляет все настроенные переопределения.

    Чтобы добавить или изменить переопределение для серии

    1. Отредактируйте плитку визуализации.

    2. Выберите серию, для которой хотите переопределить цвет палитры. Есть два способа выбрать серию:

    3. На графике выберите серию, а затем выберите Редактировать серию из всплывающего окна

    4. Выберите Добавить переопределение в разделе Переопределения серий в правом нижнем углу страницы, а затем выберите серию из списка.
    5. Измените настройку Цвет на цвет, который хотите отображать для выбранной серии.
    6. Выберите Обновить, чтобы сохранить изменения в дашборде.

    Лимиты запросов¶

    Используйте раздел Лимиты запросов для проверки и настройки лимитов запросов Grail для секции блокнота или плитки дашборда. Эти настройки определяют максимальные лимиты при получении данных. Превышение любого лимита приведёт к генерации предупреждения.

    Плитки дашбордов и секции блокнотов, созданные в Dynatrace версии ранее 1.296, не затрагиваются. Эти существующие плитки/секции будут возвращать те же результаты, что и раньше.

    • Лимит чтения данных (ГБ)

    Лимит в гигабайтах для объёма данных, которые будут просканированы при чтении. * Лимит записей

    Максимальное количество записей результатов, которые вернёт этот запрос. По умолчанию: 1 000 записей. Чтобы увидеть больше записей, необходимо увеличить значение Лимит записей.

    • Если ваш запрос не содержит limit, например

      fetch logs

      применяется значение Лимит записей. По умолчанию вы увидите до 1 000 записей. + Если ваш запрос также включает limit, например

      fetch logs


| limit 2000

      применяется меньшее из двух значений (либо limit в вашем запросе, либо Лимит записей в веб-интерфейсе).

      В приведённом выше примере вы по-прежнему увидите только 1 000 записей, если не увеличите значение Лимит записей. * Лимит размера результата

    Максимальное количество байт результата, которое вернёт этот запрос. Для лучшей производительности с типичными запросами и небольшими документами значение по умолчанию установлено на 1 МБ. * Сэмплирование (только для логов и спанов)

    Приводит к выборке подмножества записей логов или спанов.

    Пороговые значения¶

    Настройка пороговых значений¶

    Для настройки пороговых значений в визуализации дашборда или блокнота

    1. Выберите для редактирования плитки визуализации.

    2. Выберите Пороговые значения.

    3. В Dashboards выберите вкладку Визуальное на боковой панели, а затем выберите Пороговые значения.

    4. В Notebooks выберите Пороговые значения на боковой панели.
    5. Выберите Пороговое значение.

    6. Определите пороговые значения. Для каждого диапазона:

    7. Выберите цвет для отображения этого диапазона

    8. Выберите оператор для определения порогового значения этого диапазона
    9. Введите статическое значение для сравнения (используя выбранный оператор) с возвращённым значением

    10. Введите метку для связи с определённым диапазоном.

      • Метки применяются только к диаграммам с осью Y, таким как диаграммы временных рядов и категориальная столбчатая диаграмма.
      • Метки не могут быть определены для таблиц и диаграммы одиночного значения.

    Пример настройки пороговых значений в Dashboards

    Этот пример использует столбчатую диаграмму, но параметры применимы к другим визуализациям.

    1. В Dashboards создайте дашборд.
    2. Выберите и в разделе Сниппеты меню выберите Метрики > Диаграмма топ-10 хостов по использованию CPU.
    3. На панели редактирования секции выберите вкладку Визуальное и выберите Столбчатая.
    4. Выберите Пороговые значения.
    5. Выберите Пороговое значение.

    Отображается пустой набор полей пороговых значений. 6. Определите пороговые значения для отображаемой метрики. Вы можете наблюдать свои изменения на оси Y диаграммы.

    В этом примере мы определяем три диапазона использования CPU с соответствующими цветами и метками.

    Вы можете видеть диапазоны, отображаемые на оси Y и во всплывающей подсказке.

    Чтобы сбросить до значений по умолчанию (отменить настройки пороговых значений), выберите значок корзины рядом с элементом.

    Редактирование пороговых значений¶

    Добавить пороговые значения

    Единицы измерения и форматы¶

    Для переопределения единиц измерения и форматов по умолчанию в визуализации дашборда или блокнота

    1. Выберите для редактирования плитки визуализации.
    2. Выберите вкладку Визуальное.
    3. Выберите Единицы измерения и форматы.
    4. Выберите Переопределение.
    5. Выберите Переопределение
    6. В раскрывающемся списке выберите элемент, для которого хотите добавить переопределение единицы измерения.

    Это числовой столбец базового результата DQL, поэтому он варьируется в зависимости от запроса. Например:

    • Запрос fetch events возвращает события. Раскрывающийся список здесь позволяет выбрать числовое поле (например, transfer_size) из результатов.
    • Запрос timeseries avg(dt.host.cpu.usage) возвращает один временной ряд для avg(dt.host.cpu.usage). Этот временной ряд является единственным доступным для выбора элементом в списке.

    • Определите переопределение.

    • Единица по умолчанию: базовая единица, в которой были записаны значения. Значение None, если она не была включена в результат DQL, или автоматически определяется единицей, переданной из результата DQL. Это поле не приводит к какому-либо преобразованию.

    • Отображаемая единица: после определения единицы по умолчанию вы можете использовать Отображаемую единицу для преобразования. Например, если результат DQL определил ваше числовое значение как Bytes, Отображаемая единица теперь предлагает подходящий список преобразований байтов, таких как Kilobyte и Megabyte. В отличие от Единицы по умолчанию, Отображаемая единица всегда является числовым преобразованием.
    • Десятичные знаки отображает количество десятичных знаков по умолчанию (степень точности). Чтобы увидеть в действии, измените выбор Десятичные знаки и наблюдайте изменение в визуализации.
    • Суффикс отображает суффикс после единицы измерения. Чтобы увидеть в действии, введите строку и наблюдайте изменение в визуализации. Когда вы не находите нужную единицу измерения, вы можете использовать Суффикс для отображения желаемой единицы.
    • Включите Сокращать большие числа, если хотите отображать большие числа в сокращённой форме. Например, 1053 становится 1.1K.
