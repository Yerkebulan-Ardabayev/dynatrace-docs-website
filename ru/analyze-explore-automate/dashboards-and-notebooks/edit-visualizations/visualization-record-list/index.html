    Список записей

    Когда использовать визуализацию в виде списка записей

    Используйте визуализацию в виде списка записей:

    • Когда вы хотите видеть фактические записи, возвращаемые запросом, с первого взгляда.
    • Когда вас интересует структура записей и распределение значений полей.
    • Когда вы хотите дополнительно исследовать данные с помощью взаимодействий.

    Отображает записи, возвращённые запросом. Некоторые данные в примере были скрыты.

    Взаимодействия со списком записей¶

    Действия с полями¶

    Выберите поле для отображения меню с параметрами, специфичными для данного поля. Доступные параметры могут различаться в зависимости от поля.

    • Copy field name копирует имя выбранного поля в буфер обмена.
    • Explain value — используйте ИИ для объяснения поля.

    • Add command to query — раздел команд, специфичных для поля, которые можно автоматически добавить к запросу, например:

    • Sort ascending добавляет команду к запросу для sort <fieldname> asc

    • Sort descending добавляет команду к запросу для sort <fieldname> desc
    • Summarize подсчитывает, как часто каждое из уникальных значений присутствует в результате (например, подсчитывает, как часто встречается каждый идентификатор хоста).
    • Extract fields
    • Open with передаёт всю запись в диалоговое окно Open with, не учитывая, какое поле вы выбрали.

    Действия со значениями¶

    Выберите значение для отображения меню с параметрами, специфичными для данного значения. Доступные параметры могут различаться в зависимости от типа значения.

    • Copy value копирует выбранное значение в буфер обмена. Этот пункт меню также отображает тип выбранного значения.
    • Explain value — используйте ИИ для объяснения значения.

    • View in fullscreen применяется только к строкам. Отображает значение в полноэкранном окне.

    • и позволяют переходить от записи к записи.

    • копирует текущее значение в буфер обмена.
    • закрывает окно и возвращает к таблице.

    • Раздел Add command to query в меню содержит связанные команды, которые можно автоматически добавить к запросу, например:

    • Less than

    • Less than or equal
    • Equal
    • Greater than
    • Greater than or equal
    • Not equal
    • Later than
    • Earlier than
    • ±5 seconds
    • Extract fields
    • Add entity names добавляет подстановку (результат добавляется в отдельное поле с именем dt.entity.<entity>.name) фактического имени сущности, связанной с данным идентификатором сущности.
    • Expand

    • Reduce to single value приводит результаты к формату, подходящему для определённых визуализаций, таких как Single value, Table или Categorical bar chart.

    • Рекомендуемое приложение — например, View Kubernetes workload — может быть отображено для быстрого доступа.

    • Open with передаёт всю запись в диалоговое окно Open with, не учитывая, какое значение вы выбрали.

    Заголовок¶

    Используйте поле заголовка в верхней части панели параметров (изначально Untitled tile или Untitled section), чтобы добавить заголовок к плитке дашборда или разделу блокнота.

    • Можно использовать эмодзи, такие как ð и ð и â¤ï¸.
    • Можно использовать переменные.

    Пример:

    1. Определите переменные с именами Status и Emoji в вашем дашборде.
    2. Задайте заголовок Current $Emoji status is $Status.
    3. Установите Status в значение Good.
    4. Установите Emoji в значение ð.

    Заголовок будет отображаться как Current ð status is Good.

    Визуализация¶

    Если вы не уверены, что выбрали правильную визуализацию, используйте селектор визуализации, чтобы попробовать различные визуализации.

    Лимиты запроса¶

    Используйте раздел Query limits, чтобы проверить и настроить лимиты запросов Grail для каждого раздела блокнота или плитки дашборда. Эти параметры определяют максимальные лимиты при получении данных. Превышение любого лимита приведёт к появлению предупреждения.

    Плитки дашбордов и разделы блокнотов, созданные в Dynatrace версии ранее 1.296, не затрагиваются. Существующие плитки/разделы будут возвращать те же результаты, что и раньше.

    • Read data limit (GB)

    Лимит в гигабайтах для объёма данных, которые будут сканироваться при чтении. * Record limit

    Максимальное количество записей результата, которое вернёт данный запрос. По умолчанию: 1 000 записей. Чтобы увидеть больше записей, необходимо увеличить значение Record limit.

    • Если ваш запрос не содержит limit, например

      fetch logs

      применяется значение Record limit. По умолчанию вы увидите до 1 000 записей. + Если ваш запрос также включает limit, например

      fetch logs


| limit 2000

      применяется меньшее из двух значений (либо limit в вашем запросе, либо Record limit в веб-интерфейсе).

      В приведённом выше примере вы всё равно увидите только 1 000 записей, если не увеличите значение Record limit. * Result size limit

    Максимальное количество байт результата, которое вернёт данный запрос. Для лучшей производительности при типичных запросах и небольших документах значение по умолчанию установлено на 1 МБ. * Sampling (Logs and Spans only)

    Обеспечивает выбор подмножества записей Log или Span.

    Единицы измерения и форматы¶

    Чтобы переопределить единицы измерения и форматы по умолчанию в визуализации дашборда или блокнота:

    1. Выберите для редактирования плитки визуализации.
    2. Выберите вкладку Visual.
    3. Выберите Units and formats.
    4. Выберите Override.
    5. Выберите Override
    6. В раскрывающемся списке выберите элемент, для которого вы хотите добавить переопределение единицы.

    Это числовой столбец базового результата DQL, поэтому он зависит от запроса. Например:

    • Запрос fetch events возвращает события. Раскрывающийся список позволяет выбрать числовое поле (например, transfer_size) из результатов.
    • Запрос timeseries avg(dt.host.cpu.usage) возвращает один временной ряд для avg(dt.host.cpu.usage). Этот временной ряд является единственным доступным вариантом в списке.

    • Определите переопределение.

    • Default unit: Базовая единица, в которой были получены значения. Значение None, если единица не была включена в результат DQL, или автоматически определяется единицей, переданной из результата DQL. Это поле не приводит к какому-либо преобразованию.

    • Displayed unit: После определения базовой единицы вы можете использовать Displayed unit для преобразования. Например, если результат DQL определил числовое значение как Bytes, Displayed unit предложит подходящий список преобразований байтов, таких как Kilobyte и Megabyte. В отличие от Default unit, Displayed unit всегда является числовым преобразованием.
    • Decimals отображает количество десятичных знаков (степень точности) по умолчанию. Чтобы увидеть результат, измените значение Decimals и наблюдайте за изменением в визуализации.
    • Suffix отображает суффикс, добавляемый после единицы. Чтобы увидеть результат, введите строку и наблюдайте за изменением в визуализации. Если вы не находите нужную единицу, вы можете использовать Suffix для отображения желаемой единицы.
    • Включите Abbreviate large numbers, если хотите отображать большие числа в сокращённой форме. Например, 1053 станет 1.1K.

    Чтобы сбросить настройки до значений по умолчанию (отменить переопределения для выбранного элемента), выберите значок корзины рядом с элементом.
