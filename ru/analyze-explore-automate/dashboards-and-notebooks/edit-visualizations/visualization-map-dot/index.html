    Визуализация точечной карты¶

    • Последнее Dynatrace
    • Руководство по настройке
    • 5-минутное чтение
    • Опубликовано 3 апреля 2025 г.

    Версия панелей управления 1.311+ Версия тетрадей 1.311+

    Когда использовать

    Используйте точечную карту, чтобы определить местоположения и представить конкретные значения. Примеры включают местоположения серверов и их метрики производительности, а также местоположения офисов и операционные метрики, такие как количество сотрудников или выручка.

    Примеры¶

    Чтобы попробовать пример

    1. Создайте плитку DQL в Панели управления или раздел DQL в Тетради.
    2. Скопируйте пример данных и вставьте их в редактор DQL.
    3. Запустите его.
    4. Выберите визуализацию и экспериментируйте с настройками визуализации.

    Взаимодействия с графиками¶

    Взаимодействия с выбором¶

    Когда вы выбираете значение на графике и фиксируете отображаемую подсказку, вы можете затем навести курсор на подсказку, чтобы отобразить меню вариантов, специфичных для выбора.

    Взаимодействия с графиками, доступные вам, зависят от вашего запроса и визуализации. Например, если вы выбираете хост на линейном графике и наводите курсор на подсказку, вы увидите меню элементов, таких как:

    • Копировать имя — копировать имя выбранного хоста.
    • Поля — раздел с подменю для каждого поля запроса. Подменю поля предлагает варианты, специфичные для поля, такие как:
    • Копировать значение — копировать значение поля. Также отображает тип поля.
    • Скрыть — скрыть поле на графике.
    • Объяснить значение — использовать ИИ для объяснения поля.
    • Добавить команду в запрос — раздел команд, специфичных для поля, которые можно автоматически добавить в ваш запрос.
    • Также может быть перечислено рекомендуемое приложение.
    • Визуальные варианты — открывает панель редактирования, чтобы изменить варианты визуализации для выбранного элемента.
    • Установить цвет — открывает панель редактирования, чтобы изменить цвет выбранного элемента.
    • Перейти к хосту — открывает выбор в Инфраструктура и операции.

    В целом, если есть рекомендуемые приложения для открытия выбранного элемента, меню предлагает прямые ссылки на эти приложения, за которыми следует вариант Открыть с для выбора другой целевой приложения. * Открыть с — для подробностей см. Переходы и навигация.

    3. Установите Status на Good.

    Визуализация¶

    Если вы не уверены, что выбрали правильную визуализацию, используйте селектор визуализаций для попытки разных визуализаций.

    Вид¶

    • Уровень масштабирования по умолчанию

    Установите уровень масштабирования по умолчанию для карты, выбрав один из следующих вариантов:

    • Данные: Автоматически регулирует уровень масштабирования, чтобы все точки данных поместились в пределах вида карты.
    • Мир: Устанавливает уровень масштабирования для отображения всего мира.
    • Пользовательский: Позволяет указать координаты центра карты и установить пользовательский уровень масштабирования.
    • Отображать регионы стран

    Включите эту опцию, чтобы отображать контуры регионов внутри стран.

    Привязка данных¶

    Раздел привязки данных показывает, как столбец вашего результата привязан к визуализации.

    Разверните для общих правил настроек привязки данных

    Разверните раздел Привязка данных настроек вашей визуализации, чтобы увидеть, как данные в вашем результате привязаны к вашей визуализации, и чтобы скорректировать эти настройки, если необходимо.

    • Обязательные поля помечены звездочкой (*). Пример:

    * Типы данных отображаются рядом с именами полей в выпадающих списках и привязанных полях. * Единицы измерения отображаются, когда есть только одна назначенная. * Поля результатов сгруппированы в Подходящие и Неподходящие. Поля, которые не могут быть использованы для отображения данных в визуализации, помечены как неподходящие. Пример:

    * Автоматическое применение настроек привязки данных по умолчанию:

    Dynatrace версия 1.319+

    • Существующие плитки и разделы считаются пользовательскими. Их конфигурации привязки данных не обновляются автоматически.
    • Новые плитки и разделы применяют настройку привязки данных по умолчанию. Если вы не измените эти настройки вручную, они могут измениться, если новое выполнение плитки/раздела изменит результаты и есть поля, отсутствующие или новые, которые лучше подходят для привязки данных.

    Настройки привязки данных, специфичные для визуализации¶

    • Широта: Выберите значение из вашего запроса для использования в качестве широты каждой точки на карте.
    • Долгота: Выберите значение из вашего запроса для использования в качестве долготы каждой точки на карте.
    • Значение радиуса: Выберите значение из вашего запроса для использования в качестве радиуса пузыря вокруг каждой точки на карте.
    • Значение цвета: Выберите значение из вашего запроса для использования в качестве цвета пузыря вокруг каждой точки на карте.

    Форма¶

    • Стиль: Выберите символ, который будет отображаться для каждой сопоставленной точки.

    • Форма: Выберите стандартную форму из списка.

    • Иконка: Выберите иконку из списка.
    • Эмодзи: Добавьте эмодзи с помощью комбинации клавиш WIN+. или Win+; или символа.
    • Размер: Установите размер отображения (в пикселях) для каждого отображаемого символа.

    • Направление: Установите угол, под которым данные точки на карте будут визуализированы.

    • Фиксированный: Фиксированный угол (0-360) для поворота символа. Например, 0 не повернёт символ, а 180 перевернёт символ вверх тормашками.

    • Данные: Значение (0-360) сопоставленное из вашего набора данных. Выберите из списка.

    Легенда и подсказка¶

    • Показать пользовательские поля: Чтобы отобразить пользовательские поля (имя и значение) при наведении курсора на область карты, включите Показать пользовательские поля и выберите каждое пользовательское поле, которое вы хотите отобразить.

    • Показать легенду: Чтобы отобразить легенду карты, включите Показать легенду и выберите положение Легенды:

    • Авто: Выбирает подходящее положение на основе размера карты и доступного пространства.

    • Низ: Отображает легенду под картой.
    • Право: Отображает легенду справа от карты.

    • Обрезка текста: Определяет, как обрезать текст, когда полный текст не может быть отображён.

    • A…: Обрезает текст с правого конца (когда правый конец менее важен)

    • A…B: Обрезает текст из середины (когда середина менее важна)
    • …B: Обрезает текст с левого конца (когда левый конец менее важен)

    • Минимальное значение: Устанавливает минимальное значение в данных.

    • Авто: Автоматически выбирает подходящее минимальное значение на основе данных (0 или минимальное значение).

    • Пользовательское: Позволяет установить пользовательское минимальное значение.

    • Максимальное значение: Устанавливает максимальное значение в данных.

    • Авто: Автоматически выбирает подходящее максимальное значение на основе данных (0 или максимальное значение). Пользовательское позволяет установить пользовательское максимальное значение.

    • Пользовательское: Позволяет установить пользовательское максимальное значение.

    Цвет¶

    • Цвета пузырей

    Выберите, как раскрасить пузыри:

    • Палитра цветов: Отображает все пузыри в оттенке цвета из выбранной палитры цветов. Оттенок, используемый для каждого пузыря, соответствует значению Цветового значения по отношению к другим областям.

      Пример

      Если значения Цветового значения, возвращаемые вашим запросом, варьируются от 0 до 100

      • Пузырь с значением около 0 имеет оттенок цвета от правого конца палитры.
      • Пузырь с значением около 100 имеет оттенок цвета от левого конца палитры.
      • Одиночный цвет: Отображает все пузыри в одном и том же цвете. Выберите цвет из списка или введите код цвета в шестнадцатеричном формате.
      • Пользовательские цвета: Отображает каждый пузырь в пользовательском цвете, определённом вами.

      Для каждого пользовательского цвета, который вы хотите добавить

      1. Выберите Цвет.
      2. Введите значение, оператор и цвет, который будет использоваться для этого значения и оператора.

      Пример

      Предположим, вы хотите раскрасить пузыри по трем уровням Цветового значения:

      • Зелёный, если Цветовое значение меньше 4,000
      • Жёлтый, если Цветовое значение достигает или превышает пороговое значение 4,000
      • Красный, если Цветовое значение достигает или превышает пороговое значение 5,000

      Чтобы настроить это

      • Выберите Цвет и добавьте строку пользовательского цвета с значением 0, оператором ≥, и желаемым оттенком зелёного. Если Цветовое значение равно 0 или выше, пузырь будет зелёным.
      • Выберите Цвет и добавьте строку пользовательского цвета с значением 4,000, оператором ≥, и желаемым оттенком жёлтого. Если Цветовое значение равно 4,000 или выше, пузырь будет жёлтым.
      • Выберите Цвет и добавьте строку пользовательского цвета с значением 5,000, оператором ≥, и желаемым оттенком красного. Если Цветовое значение равно 5,000 или выше, пузырь будет красным.

      Ограничения запросов¶

    Используйте раздел Ограничения запросов, чтобы проверить и скорректировать ограничения запросов Grail для каждой секции заметок или плитки на панели мониторинга. Эти настройки определяют максимальные ограничения при получении данных. Превышение любого ограничения приведет к генерации предупреждения.

    Плитки на панели мониторинга и секции заметок, созданные в Dynatrace ранее версии 1.296, не затронуты. Эти существующие плитки/секции будут возвращать те же результаты, что и раньше.

    • Ограничение чтения данных (ГБ)

    Ограничение в гигабайтах для количества данных, которые будут прочитаны во время чтения. * Ограничение записей

    Максимальное количество записей результатов, которое этот запрос вернет. По умолчанию: 1 000 записей. Чтобы увидеть больше записей, необходимо увеличить значение Ограничение записей.

    • Если ваш запрос не содержит limit, например

      fetch logs

      значение Ограничение записей применяется. По умолчанию вы увидите до 1 000 записей. + Если ваш запрос также содержит limit, например

      fetch logs



| limit 2000

      применяется меньшее из двух значений (либо limit в вашем запросе, либо Ограничение записей в веб-интерфейсе).

      В примере выше вы увидите только 1 000 записей, если не увеличите значение Ограничение записей. * Ограничение размера результата

    Максимальное количество байтов результата, которое этот запрос вернет. Для лучшей производительности с типичными запросами и меньшими документами значение по умолчанию установлено в 1 МБ. * Отбор проб (только для журналов и спанов)

    Результатом является выборка подмножества записей журнала или спана.

    Единицы и форматы¶

    Чтобы переопределить значения по умолчанию единиц и форматов в визуализации панели мониторинга или заметок

    1. Выберите для редактирования плитки визуализации.
    2. Выберите вкладку Визуализация.
    3. Выберите Единицы и форматы.
    4. Выберите Переопределить.
    5. Выберите Переопределить
    6. В выпадающем списке выберите элемент, для которого вы хотите добавить переопределение единицы.

    Это числовой столбец основного результата DQL, поэтому он варьируется в зависимости от запроса. Например:

    • Запрос fetch events возвращает события. Выпадающий список здесь позволяет выбрать числовое поле (например, transfer_size) из результатов.
    • Запрос timeseries avg(dt.host.cpu.usage) возвращает один временной ряд для avg(dt.host.cpu.usage). Этот временной ряд является единственной доступной опцией в списке.

    • Определите переопределение.

    • Единица по умолчанию: Основная единица, в которой были захвачены значения. Она равна None, если не была включена в результат DQL, или автоматически определена единицей, переданной из результата DQL. Это поле не приводит к какому-либо преобразованию.

    • Отображаемая единица: Как только вы определите единицу по умолчанию, вы можете использовать Отображаемую единицу для преобразования. Например, если результат DQL определил ваше числовое значение в результате как Байты, Отображаемая единица теперь предлагает подходящий список преобразований байтов, таких как Килобайт и Мегабайт. В отличие от Единицы по умолчанию, Отображаемая единица всегда является числовым преобразованием.
    • Десятичные знаки отображают количество десятичных знаков (степень точности) для отображения. Чтобы увидеть это в действии, измените выбор Десятичные знаки и наблюдайте за изменением визуализации.
    • Суффикс отображает суффикс, который следует отображать после единицы. Чтобы увидеть это в действии, введите строку и наблюдайте за изменением визуализации. Когда вы не найдете единицу, которую ищете, вы можете использовать Суффикс, чтобы отобразить желаемую единицу.
    • Включите Сокращать большие числа, если вы хотите отображать большие цифры в сокращенной форме. Например, 1053 становится 1,1K.

    Чтобы сбросить значения по умолчанию (отменить настройки переопределения для выбранного элемента), выберите значок корзины рядом с элементом.
