    Визуализация «Ленточный график»¶

    Визуализация «Ленточный график»¶

    • Latest Dynatrace
    • Практическое руководство
    • 1 мин. чтения
    • Опубликовано 08 июля 2022

    Когда использовать визуализацию «Ленточный график»

    Используйте визуализацию «Ленточный график»:

    • Когда вы хотите увидеть диапазон или вариативность точек данных. Например, для прогнозирования данных или отображения доверительных интервалов.
    • Когда вы хотите показать несколько статистических показателей для одной и той же точки данных для их статистического анализа. Например, при сравнении максимума и минимума использования подов во времени для определения подходящих лимитов.

    Пример¶

    Приведённый выше график построен на основе следующего запроса.

    timeseries avg=avg(dt.host.cpu.load),



max=max(dt.host.cpu.load),



min=min(dt.host.cpu.load),



by:dt.entity.host



| limit 1

    В общем случае для создания ленточного графика, аналогичного приведённому выше, используйте timeseries для формирования запроса DQL, содержащего следующее:

    • max (или upper) для определения верхней границы полосы
    • min (или lower) для определения нижней границы полосы
    • третий ряд данных для отрисовки линии

    Взаимодействие с графиком¶

    Взаимодействие при выборе¶

    Когда вы выбираете значение на графике и закрепляете отображаемую всплывающую подсказку, вы можете навести курсор на всплывающую подсказку, чтобы увидеть меню с параметрами, специфичными для выбора.

    Доступные взаимодействия с графиком зависят от вашего запроса и визуализации. Например, если вы выберете хост на линейном графике и наведёте курсор на всплывающую подсказку, вы увидите меню со следующими пунктами:

    • Копировать имя — копирование имени выбранного хоста.

    • Поля — раздел с подменю для каждого поля запроса. Подменю поля предлагает специфичные для поля параметры, такие как:

    • Копировать значение — копирование значения поля. Также отображает тип поля.

    • Скрыть — скрытие поля на графике.
    • Объяснить значение — использование ИИ для объяснения поля.
    • Добавить команду в запрос — раздел специфичных для поля команд, которые можно автоматически добавить к вашему запросу.
    • Также может быть указано рекомендуемое приложение.
    • Визуальные параметры — открытие панели редактирования для изменения параметров визуализации выбранного элемента.
    • Задать цвет — открытие панели редактирования для изменения цвета выбранного элемента.
    • Перейти к хосту — открытие выбранного элемента в Infrastructure & Operations.

    В общем случае, если для открытия выбранного элемента существуют рекомендуемые приложения, меню предлагает прямые ссылки на эти приложения, а затем параметр Открыть в, чтобы выбрать другое целевое приложение. * Открыть в — подробнее см. Переходы и навигация.

    Заголовок¶

    Используйте поле заголовка в верхней части панели параметров (изначально Untitled tile или Untitled section) для добавления заголовка к плитке дашборда или разделу ноутбука.

    • Вы можете использовать эмодзи, такие как 📊 и 📈 и ❤️.
    • Вы можете использовать переменные.

    Пример:

    1. Определите переменные с именами Status и Emoji в вашем дашборде.
    2. Задайте заголовок Current $Emoji status is $Status.
    3. Установите Status в значение Good.
    4. Установите Emoji в значение 📊.

    Заголовок будет отображаться как Current &#x1F4CA; status is Good.

    Визуализация¶

    Если вы не уверены, что выбрали правильную визуализацию, используйте селектор визуализации для пробы различных визуализаций.

    Маппинг данных¶

    Раздел маппинга данных показывает, как столбец вашего результата сопоставлен с визуализацией.

    Общие правила настройки маппинга данных

    Разверните раздел Маппинг данных в настройках визуализации, чтобы увидеть, как данные в результате сопоставлены с визуализацией, и при необходимости скорректировать эти настройки.

    • Обязательные поля отмечены звёздочкой (*). Пример:

    * Типы данных отображаются рядом с именами полей в выпадающих списках и сопоставленных полях. * Единицы измерения отображаются, когда назначена только одна. * Поля результатов сгруппированы в Подходящие и Неподходящие. Поля помечаются как неподходящие, если они не могут быть использованы для отображения данных в визуализации. Пример:

    * Автоматическое применение настроек маппинга данных по умолчанию:

    Dynatrace версии 1.319+

    • Существующие плитки и разделы считаются определёнными пользователем. Их конфигурации маппинга данных не обновляются автоматически.
    • Вновь созданные плитки и разделы применяют настройки маппинга данных по умолчанию. Если вы не изменяете эти настройки вручную, они могут измениться при повторном выполнении плитки/раздела, если результаты изменятся и появятся отсутствующие поля или новые поля, лучше подходящие для маппинга данных.

    Специфичные настройки маппинга данных для визуализации¶

    Ленточный график отображает значение во времени с верхней и нижней границами полосы.

    В общем случае используйте timeseries для формирования запроса DQL, содержащего следующее:

    • max (или upper) для определения верхней границы полосы.
    • min (или lower) для определения нижней границы полосы.
    • Третий ряд данных для отрисовки линии

    Таким образом, маппинг должен включать следующее:

    • Time: столбец результата, используемый для оси X (timestamp или timeframe).
    • Interval: это значение сопоставляется автоматически и не может быть изменено. Оно сообщает, какие поля сопоставлены для результатов на основе timeseries. Берётся первое доступное поле интервала из набора результатов при использовании timeseries (также включает данные на основе makeTimeseries).
    • Band min values: верхняя граница полосы.
    • Band max values: нижняя граница полосы.
    • Values: выбор одного или нескольких значений, которые график отображает во времени.
    • Names: элементы, отображаемые, например, в легенде и именах рядов.

    Параметры ленточного графика¶

    Чтобы быстро изучить параметры и определить, что лучше всего подходит, включайте и выключайте параметры и сразу наблюдайте результат на графике. Например, выглядит ли лучше с подписью или без? Включите и выключите этот параметр и убедитесь сами.

    Цветовые палитры¶

    Вы можете выбрать цветовую палитру из списка в разделе Цветовые палитры.

    Для линейного графика, графика с областями или столбчатого графика вы можете при необходимости переопределить выбранную цветовую палитру для любого ряда.

    Положение легенды¶

    Выберите, где отображать легенду визуализации:

    • Автоматически — Dynatrace выбирает подходящее расположение
    • Скрыта — легенда не отображается
    • Внизу — легенда отображается в нижней части визуализации
    • Справа — легенда отображается с правой стороны визуализации

    Нулевые значения¶

    Соединять нулевые значения — указывает, следует ли соединять нулевые значения в визуализации.

    Левая ось¶

    Задаёт масштаб левой оси:

    • Логарифмический
    • Линейный

    Минимум¶

    Минимум диапазона визуализации.

    • Авто — Dynatrace автоматически выбирает подходящий минимум на основе данных (0 или минимальное значение).
    • Мин. значение — минимум устанавливается равным минимальному значению данных.
    • Пользовательский — отображает поле ввода для задания пользовательского минимального значения.

    Максимум¶

    Максимум диапазона визуализации.

    • Авто — Dynatrace автоматически выбирает подходящий максимум на основе данных (0 или максимальное значение).
    • Макс. значение — максимум устанавливается равным максимальному значению данных.
    • Пользовательский — отображает поле ввода для задания пользовательского максимального значения.

    Подпись¶

    Вертикальный текст, отображаемый как подпись для оси Y.

    Лимиты запросов¶

    Используйте раздел Лимиты запросов для проверки и настройки лимитов запросов Grail для каждого раздела ноутбука или плитки дашборда. Эти настройки определяют максимальные лимиты при получении данных. Превышение любого лимита приведёт к предупреждению.

    Плитки дашбордов и разделы ноутбуков, созданные в Dynatrace ранее версии 1.296, не затрагиваются. Существующие плитки/разделы будут возвращать те же результаты, что и раньше.

    • Лимит чтения данных (ГБ)

    Лимит в гигабайтах для объёма данных, которые будут просканированы при чтении. * Лимит записей

    Максимальное количество записей результата, которое вернёт запрос. По умолчанию: 1 000 записей. Чтобы увидеть больше записей, необходимо увеличить значение Лимита записей.

    • Если ваш запрос не содержит limit, например

      fetch logs

      применяется значение Лимита записей. По умолчанию вы увидите до 1 000 записей. + Если ваш запрос также включает limit, например

      fetch logs



| limit 2000

      применяется меньшее из двух значений (либо limit в запросе, либо Лимит записей в веб-интерфейсе).

      В приведённом выше примере вы увидите только 1 000 записей, если не увеличите значение Лимита записей. * Лимит размера результата

    Максимальное количество байт результата, которое вернёт запрос. Для лучшей производительности с типичными запросами и документами меньшего размера значение по умолчанию установлено в 1 МБ. * Выборка (только для логов и спанов)

    Результат — выбор подмножества записей логов или спанов.

    Пороги¶

    Настройка порогов¶

    Для настройки порогов в визуализации дашборда или ноутбука:

    1. Выберите для редактирования плитки визуализации.

    2. Выберите Пороги.

    3. В Dashboards выберите вкладку Визуальные на боковой панели, затем выберите Пороги.

    4. В Notebooks выберите Пороги на боковой панели.
    5. Выберите Порог.

    6. Определите пороги. Для каждого диапазона:

    7. Выберите цвет для отображения этого диапазона

    8. Выберите оператор для определения порога диапазона
    9. Введите статическое значение для сравнения (с использованием выбранного оператора) с возвращённым значением

    10. Введите подпись для связи с определённым диапазоном.

      • Подписи применяются только к графикам с осью Y, таким как графики timeseries и категориальные столбчатые графики.
      • Подписи нельзя задать для таблиц и графика единичного значения.

    Пример настройки порогов в Dashboards

    В этом примере используется столбчатый график, но параметры применимы к другим визуализациям.

    1. В Dashboards создайте дашборд.
    2. Выберите и в разделе Snippets меню выберите Metrics > Chart top 10 hosts by CPU usage.
    3. На панели редактирования раздела выберите вкладку Визуальные и выберите Bar.
    4. Выберите Пороги.
    5. Выберите Порог.

    Отображается пустой набор полей порога. 6. Определите пороги для отображаемой метрики. Вы можете наблюдать изменения на оси Y графика.

    В этом примере мы определяем три диапазона использования CPU с соответствующими цветами и подписями.

    Вы можете видеть диапазоны, отображённые на оси Y и во всплывающей подсказке.

    Для сброса к значениям по умолчанию (отмены настроек порогов) выберите значок корзины рядом с элементом.

    Редактирование порогов¶

    1. Выберите для редактирования плитки визуализации.

    2. Выберите Пороги.

    3. В Dashboards выберите вкладку Визуальные на боковой панели, затем выберите Пороги.

    4. В Notebooks выберите Пороги на боковой панели.
    5. Далее вы можете выполнить следующие действия (разверните строки для подробностей):

    Добавить пороги

    Чтобы добавить пороги (если выбранная визуализация поддерживает дополнительные пороги), выберите Порог.

    Выключить пороги

    Чтобы выключить (скрыть) существующие пороги, используйте переключатель. Настройки не теряются. Вы можете снова включить их при необходимости.

    Удалить пороги

    Чтобы удалить существующие пороги, выберите значок корзины .

    Редактировать настройки порогов

    Чтобы изменить существующие настройки порогов, просто отредактируйте поля.

    Добавить диапазон

    Чтобы добавить диапазон к существующим порогам, выберите Добавить диапазон.

    Удалить диапазон

    Чтобы удалить диапазон из существующих порогов, выберите кнопку удаления в этой строке.

    Единицы измерения и форматы¶

    Для переопределения единиц измерения и форматов по умолчанию в визуализации дашборда или ноутбука:

    1. Выберите для редактирования плитки визуализации.
    2. Выберите вкладку Визуальные.
    3. Выберите Единицы измерения и форматы.
    4. Выберите Переопределение.
    5. Выберите Переопределение
    6. В выпадающем списке выберите элемент, для которого хотите добавить переопределение единиц измерения.

    Это числовой столбец базового результата DQL, поэтому он зависит от запроса. Например:

    • Запрос fetch events возвращает события. Выпадающий список позволяет выбрать числовое поле (например, transfer_size) из результатов.
    • Запрос timeseries avg(dt.host.cpu.usage) возвращает один timeseries для avg(dt.host.cpu.usage). Этот timeseries является единственным доступным для выбора элементом в списке.

    • Определите переопределение.

    • Единица по умолчанию: базовая единица, в которой были захвачены значения. None, если она не была включена в результат DQL, или автоматически определяется из единицы, переданной из результата DQL. Это поле не приводит к конвертации.

    • Отображаемая единица: после определения единицы по умолчанию вы можете использовать Отображаемую единицу для конвертации. Например, если результат DQL определил ваше числовое значение как Bytes, Отображаемая единица теперь предлагает подходящий список конвертаций байтов, таких как Kilobyte и Megabyte. В отличие от Единицы по умолчанию, Отображаемая единица всегда является числовой конвертацией.
    • Десятичные знаки отображает количество десятичных знаков по умолчанию (степень точности). Чтобы увидеть в действии, измените выбор Десятичных знаков и наблюдайте изменение в визуализации.
    • Суффикс отображает суффикс после единицы измерения. Чтобы увидеть в действии, введите строку и наблюдайте изменение в визуализации. Когда вы не находите нужную единицу, вы можете использовать Суффикс для отображения желаемой единицы.
    • Включите Сокращать большие числа, если хотите отображать большие значения в сокращённой форме. Например, 1053 станет 1.1K.

    Для сброса к значениям по умолчанию (отмены настроек переопределения для выбранного элемента) выберите значок корзины рядом с элементом.

    В этом примере используется линейный график, но параметры применимы к другим визуализациям.

    1. В Dashboards создайте дашборд.
    2. Выберите и в разделе Snippets меню выберите Metrics > Chart average CPU across all hosts.
    3. На панели редактирования раздела выберите вкладку Визуальные и выберите Line.
    4. Выберите Единицы измерения и форматы.

    5. Выберите Переопределение.

    6. В выпадающем списке выберите метрику, для которой хотите добавить переопределение. В этом примере доступна только одна метрика. 7. Определите переопределение для отображаемой метрики. Вы можете наблюдать изменения на оси Y графика.
