    Визуализация «Кольцевая диаграмма»¶

    Визуализация «Кольцевая диаграмма»¶

    • Последняя версия Dynatrace
    • Практическое руководство
    • Чтение: 1 мин
    • Опубликовано 8 июля 2022

    Когда использовать визуализацию «Кольцевая диаграмма»

    Используйте визуализацию «Кольцевая диаграмма»:

    • Когда в данных имеется лишь несколько (не более 5-7) чётко различимых категорий/сегментов.
    • Когда нужно сэкономить пространство и совместить одно значение с разбивкой.

    Пример¶

    Приведённая выше кольцевая диаграмма основана на следующем запросе.

    fetch logs



| summarize count(), by:{loglevel}

    В разделе Donut chart options включена опция Show total, чтобы отобразить значение 1.6M в центре кольца.

    Взаимодействие с диаграммой¶

    Взаимодействие при выборе¶

    При выборе значения на диаграмме и закреплении отображаемой подсказки можно навести курсор на подсказку, чтобы увидеть меню параметров, специфичных для выбранного элемента.

    Доступные варианты взаимодействия зависят от вашего запроса и визуализации. Например, если вы выберете хост на линейной диаграмме и наведёте курсор на подсказку, вы увидите меню со следующими пунктами:

    • Copy name -- копирование имени выбранного хоста.

    • Fields -- раздел с подменю для каждого поля запроса. Подменю поля предлагает специфичные параметры, такие как:

    • Copy value -- копирование значения поля. Также отображает тип поля.

    • Hide -- скрытие поля на диаграмме.
    • Explain value -- использование ИИ для объяснения поля.
    • Add command to query -- раздел команд, специфичных для поля, которые можно автоматически добавить в запрос.
    • Также может быть указано рекомендуемое приложение.
    • Visual options -- открытие панели редактирования для изменения параметров визуализации выбранного элемента.
    • Set color -- открытие панели редактирования для изменения цвета выбранного элемента.
    • Go to host -- открытие выбранного элемента в Infrastructure & Operations.

    Как правило, если для выбранного элемента есть рекомендуемые приложения, меню предлагает прямые ссылки на эти приложения, а также опцию Open with для выбора другого целевого приложения. * Open with -- подробнее см. Детализация и навигация.

    Заголовок¶

    Используйте поле заголовка в верхней части панели настроек (изначально Untitled tile или Untitled section), чтобы добавить заголовок к плитке дашборда или разделу блокнота.

    • Можно использовать эмодзи.
    • Можно использовать переменные.

    Пример:

    1. Определите переменные с именами Status и Emoji в вашем дашборде.
    2. Установите заголовок Current $Emoji status is $Status.
    3. Установите Status в значение Good.
    4. Установите Emoji в значение эмодзи.

    Заголовок будет отображаться как Current <эмодзи> status is Good.

    Визуализация¶

    Если вы не уверены, что выбрали подходящую визуализацию, используйте селектор визуализации для выбора других типов визуализации.

    Сопоставление данных¶

    Раздел сопоставления данных показывает, как столбец результата сопоставляется с визуализацией.

    Общие правила настройки сопоставления данных

    Разверните раздел Data mapping в настройках визуализации, чтобы увидеть, как данные результата сопоставлены с визуализацией, и при необходимости изменить эти настройки.

    • Обязательные поля отмечены звёздочкой (*). Пример:

    * Типы данных отображаются рядом с именами полей в раскрывающихся списках и сопоставленных полях. * Единицы измерения отображаются, когда назначена только одна. * Поля результатов сгруппированы в Suitable и Unsuitable. Поля помечаются как непригодные, если их нельзя использовать для отображения данных в визуализации. Пример:

    * Автоматическое применение настроек сопоставления данных по умолчанию:

    Dynatrace версии 1.319+

    • Существующие плитки и разделы считаются определёнными пользователем. Их конфигурации сопоставления данных не обновляются автоматически.
    • Вновь созданные плитки и разделы применяют настройку сопоставления данных по умолчанию. Если вы не изменяете эти настройки вручную, они могут измениться при повторном выполнении плитки/раздела, если результаты содержат отсутствующие поля или новые поля, лучше подходящие для сопоставления данных.

    Настройки сопоставления данных для конкретной визуализации¶

    Для кольцевой диаграммы раздел сопоставления данных включает:

    • Values: значение, отражаемое в размере сегмента.
    • Categories: столбец результата, отображаемый как сегмент кольца.

    Параметры кольцевой диаграммы¶

    Чтобы быстро изучить параметры и решить, что лучше подходит, включайте и выключайте параметры и сразу наблюдайте эффект на диаграмме. Например, лучше ли выглядит с подписью или без? Включите и выключите эту опцию и убедитесь сами.

    Цветовые палитры¶

    Вы можете выбрать цветовую палитру из списка в разделе Color palettes.

    Сегмент¶

    Установите Slice в единицу, по которой вы хотите разделить диаграмму. Slice и Value совместно определяют сопоставление результатов запроса с диаграммой.

    Значение¶

    Установите Value в значение, определяющее размер сегментов. Slice и Value совместно определяют сопоставление результатов запроса с диаграммой.

    Диаграмма¶

    Используйте Hide labels для отображения или скрытия подписей на диаграмме.

    Расположение легенды¶

    Выберите, где отображать легенду визуализации:

    • Automatic -- Dynatrace выбирает подходящее расположение
    • Hidden -- легенда не отображается
    • Bottom -- легенда отображается внизу визуализации
    • Right -- легенда отображается справа от визуализации

    Общее значение¶

    Чтобы показать общее значение в центре кольцевой диаграммы, включите Show total.

    Используйте Relative values (%) для отображения значений диаграммы в абсолютных значениях или относительных значениях (процент от общего).

    Объединение сегментов¶

    Используйте настройки Merge slices для определения способа объединения сегментов на диаграмме.

    • Absolute

    Например, если у вас три сегмента со следующими значениями:

    • Сегмент 1: 40
    • Сегмент 2: 5
    • Сегмент 3: 5

    Если вы выберете Absolute и установите Absolute limit в значение 10, сегменты 2 и 3 будут объединены в одну группу с меткой, заданной в Group label. * Relative (%)

    Например, если у вас три сегмента со следующими значениями:

    • Сегмент 1 - 40 (80% от общего)
    • Сегмент 2 - 5 (10% от общего)
    • Сегмент 3 - 5 (10% от общего)

    Если вы выберете Relative (%) и установите Relative limit в значение 20 (20%), сегменты 2 и 3 (каждый из которых ниже относительного значения 20%) будут объединены в одну группу с меткой, заданной в Group label. * Number of slices

    Например, если у вас три сегмента со следующими значениями:

    • Сегмент 1: 40
    • Сегмент 2: 5
    • Сегмент 3: 5

    Если вы выберете Number of slices и установите Slice limit в значение 2, первые два сегмента будут отображаться отдельно, а остальные будут объединены в одну группу с меткой, заданной в Group label.

    Переопределение цвета¶

    Чтобы переопределить цвет элемента диаграммы, выберите Add color override, выберите элемент из списка, а затем установите Color в нужный цвет.

    Ограничения запроса¶

    Используйте раздел Query limits для проверки и настройки ограничений запроса Grail для раздела блокнота или плитки дашборда. Эти настройки определяют максимальные ограничения при выборке данных. Превышение любого ограничения приведёт к генерации предупреждения.

    Плитки дашборда и разделы блокнота, созданные в Dynatrace до версии 1.296, не затрагиваются. Эти существующие плитки/разделы будут возвращать те же результаты, что и раньше.

    • Read data limit (GB)

    Ограничение в гигабайтах на объём данных, которые будут сканироваться при чтении. * Record limit

    Максимальное количество записей результата, которое вернёт этот запрос. По умолчанию: 1 000 записей. Чтобы увидеть больше записей, нужно увеличить значение Record limit.

    • Если ваш запрос не содержит limit, например

      fetch logs

      применяется значение Record limit. По умолчанию вы увидите до 1 000 записей. + Если ваш запрос также включает limit, например

      fetch logs



| limit 2000

      применяется меньшее из двух значений (либо limit в запросе, либо Record limit в веб-интерфейсе).

      В приведённом выше примере вы по-прежнему увидите только 1 000 записей, если не увеличите значение Record limit. * Result size limit

    Максимальный объём в байтах результата, который вернёт этот запрос. Для лучшей производительности при типичных запросах и небольших документах значение по умолчанию установлено в 1 МБ. * Sampling (Logs and Spans only)

    Выборка подмножества записей Log или Span.

    Единицы измерения и форматы¶

    Чтобы переопределить единицы измерения и форматы по умолчанию в визуализации дашборда или блокнота:

    1. Выберите для редактирования плитки визуализации.
    2. Выберите вкладку Visual.
    3. Выберите Units and formats.
    4. Выберите Override.
    5. Выберите Override.
    6. В раскрывающемся списке выберите элемент, для которого вы хотите добавить переопределение единицы измерения.

    Это числовой столбец базового результата DQL, поэтому он зависит от запроса. Например:

    • Запрос fetch events возвращает события. Раскрывающийся список позволяет выбрать числовое поле (например, transfer_size) из результатов.
    • Запрос timeseries avg(dt.host.cpu.usage) возвращает один временной ряд для avg(dt.host.cpu.usage). Этот временной ряд является единственным доступным вариантом в списке.

    • Определите переопределение.

    • Default unit: базовая единица измерения, в которой были захвачены значения. Значение None, если она не была включена в результат DQL, или автоматически определяется единицей, переданной из результата DQL. Это поле не приводит к конвертации.

    • Displayed unit: после определения единицы измерения по умолчанию можно использовать Displayed unit для конвертации. Например, если результат DQL определил числовое значение как Bytes, Displayed unit предлагает подходящий список конвертаций, таких как Kilobyte и Megabyte. В отличие от Default unit, Displayed unit всегда является числовой конвертацией.
    • Decimals отображает количество десятичных знаков по умолчанию (степень точности). Чтобы увидеть действие, измените выбор Decimals и наблюдайте за изменениями в визуализации.
    • Suffix отображает суффикс после единицы измерения. Чтобы увидеть действие, введите строку и наблюдайте за изменениями в визуализации. Если вы не нашли нужную единицу измерения, можно использовать Suffix для отображения нужной единицы.
    • Включите Abbreviate large numbers, если хотите отображать большие числа в сокращённой форме. Например, 1053 станет 1.1K.

    Чтобы сбросить настройки по умолчанию (отменить переопределение для выбранного элемента), нажмите значок корзины рядом с элементом.
