    Визуализация «Гистограмма»¶

    • Latest Dynatrace

    Когда использовать визуализацию «Гистограмма»

    Используйте гистограмму для визуализации распределения числовых значений.

    Примеры¶

    Пример 1¶

    Визуализация гистограммы выше основана на следующем запросе.

    fetch spans


| filter span.kind == "server"


| filter duration < 1s


| summarize count(), by:{range(duration, 10ms)}

    Пример 2¶

    Визуализация гистограммы выше основана на следующем запросе бизнес-событий.

    fetch bizevents


| filter event.type == "com.easytrade.trade-closed"


| filter direction == "longsell"


| filter amount <= 10000


| summarize count = count(), by:{range(amount, 300)}

    Пример 3¶

    Визуализация гистограммы выше основана на следующем запросе. Она показывает распределение оценок риска для открытых событий безопасности.

    Запрос ниже обновлён в соответствии с новой таблицей событий безопасности Grail. Полный список изменений и действий, необходимых для миграции, см. в руководстве по миграции таблиц безопасности Grail.

    fetch security.events


| filter event.status == "OPEN"


| summarize count = count(), by:{range(vulnerability.risk.score, 0.5)}

    Пример 4¶

    Визуализация гистограммы выше основана на следующем запросе. Это вариация предыдущего примера, которая показывает дополнительное разделение по статусу события безопасности.

    Запрос ниже обновлён в соответствии с новой таблицей событий безопасности Grail. Полный список изменений и действий, необходимых для миграции, см. в руководстве по миграции таблиц безопасности Grail.

    fetch security.events


| summarize count = count(), by:{range(vulnerability.risk.score, 0.5), event.status}

    Взаимодействие с графиком¶

    Взаимодействие при выборе¶

    Когда вы выбираете значение на графике и закрепляете отображаемую подсказку, вы можете навести курсор на подсказку, чтобы отобразить меню с опциями, специфичными для выбора.

    Доступные взаимодействия с графиком зависят от вашего запроса и визуализации. Например, если вы выберете хост на линейном графике и наведёте курсор на подсказку, вы увидите меню с пунктами, такими как:

    • Copy name — скопировать имя выбранного хоста.

    • Fields — раздел с подменю для каждого поля запроса. Подменю поля предлагает специфичные для поля опции, такие как:

    • Copy value — скопировать значение поля. Также отображает тип поля.

    • Hide — скрыть поле на графике.
    • Explain value — использовать ИИ для объяснения поля.
    • Add command to query — раздел специфичных для поля команд, которые можно автоматически добавить в запрос.
    • Также может быть указано рекомендуемое приложение.
    • Visual options — открывает панель редактирования для изменения параметров визуализации выбранного элемента.
    • Set color — открывает панель редактирования для изменения цвета выбранного элемента.
    • Go to host — открывает выбранный элемент в Infrastructure & Operations.

    В целом, если есть рекомендуемые приложения для открытия выбранного элемента, меню предлагает прямые ссылки на эти приложения, а затем опцию Open with для выбора другого целевого приложения. * Open with — подробнее см. Детализация и навигация.

    Заголовок¶

    Используйте поле заголовка в верхней части панели опций (изначально Untitled tile или Untitled section), чтобы добавить заголовок к плитке дашборда или секции блокнота.

    • Вы можете использовать эмодзи.
    • Вы можете использовать переменные.

    Пример:

    1. Определите переменные Status и Emoji в вашем дашборде.
    2. Установите заголовок Current $Emoji status is $Status.
    3. Установите Status в значение Good.
    4. Установите Emoji в нужное значение.

    Заголовок будет отображён соответствующим образом.

    Визуализация¶

    Если вы не уверены, что выбрали правильную визуализацию, используйте селектор визуализации, чтобы попробовать разные визуализации.

    Сопоставление данных¶

    Раздел сопоставления данных показывает, как столбец вашего результата сопоставлен с визуализацией.

    Общие правила настроек сопоставления данных

    Разверните раздел Data mapping в настройках визуализации, чтобы увидеть, как данные в вашем результате сопоставлены с визуализацией, и при необходимости скорректировать эти настройки.

    • Обязательные поля отмечены звёздочкой (*). Пример:

    * Типы данных отображаются рядом с именами полей в выпадающих списках и сопоставленных полях. * Единицы отображаются, когда назначена только одна единица. * Поля результата сгруппированы в Suitable и Unsuitable. Поля отмечены как неподходящие, если они не могут быть использованы для отображения данных в визуализации. Пример:

    * Автоматическое применение настроек сопоставления данных по умолчанию:

    Dynatrace версии 1.319+

    • Существующие плитки и секции считаются пользовательскими. Их конфигурации сопоставления данных не обновляются автоматически.
    • Вновь созданные плитки и секции применяют настройки сопоставления данных по умолчанию. Если вы не изменяете эти настройки вручную, они могут измениться, если новое выполнение плитки/секции изменит результаты и появятся отсутствующие или новые поля, лучше подходящие для сопоставления данных.

    Настройки сопоставления данных для гистограммы¶

    Гистограмма определяется массивом корзин (bins), где каждая корзина представляет непрерывный диапазон значений. Раздел сопоставления данных включает:

    • Range: ширина (размер) отдельных столбцов
    • Values: количество или частота (значение), определяющие высоту отдельных столбцов
    • Names: элементы, отображаемые, например, в легенде и именах серий.

    Параметры графика «Гистограмма»¶

    Чтобы быстро ознакомиться с опциями и определить, что лучше всего подходит, включайте и выключайте опции и сразу наблюдайте эффект на графике. Например, лучше ли выглядит с меткой или без? Включите и выключите эту опцию и посмотрите сами.

    Цветовые палитры¶

    Вы можете выбрать цветовую палитру из списка в разделе Color palettes.

    Положение легенды¶

    Выберите, где отображать легенду визуализации:

    • Automatic — Dynatrace выбирает подходящее расположение
    • Hidden — легенда не отображается
    • Bottom — легенда отображается внизу визуализации
    • Right — легенда отображается справа от визуализации

    Ось X¶

    Устанавливает масштаб оси X:

    • Logarithmic
    • Linear

    Min¶

    Минимальное значение оси X визуализации.

    • Auto — Dynatrace автоматически выбирает подходящий минимум на основе данных (0 или min-value).
    • Min-value — минимум устанавливается равным минимальному значению данных.
    • Custom — отображает поле ввода для указания пользовательского минимального значения.

    Max¶

    Максимальное значение оси X визуализации.

    • Auto — Dynatrace автоматически выбирает подходящий максимум на основе данных (0 или max-value).
    • Max-value — максимум устанавливается равным максимальному значению данных.
    • Custom — отображает поле ввода для указания пользовательского максимального значения.

    Label¶

    Текст, отображаемый в качестве подписи для оси Y.

    Ось Y¶

    Устанавливает масштаб оси Y:

    • Logarithmic
    • Linear

    • Min-value — минимум устанавливается равным минимальному значению данных.
    • Custom — отображает поле ввода для указания пользовательского минимального значения.

    • Max-value — максимум устанавливается равным максимальному значению данных.

    Лимиты запросов¶

    Используйте раздел Query limits для проверки и настройки лимитов запросов Grail для секции блокнота или плитки дашборда. Эти настройки определяют максимальные лимиты при получении данных. Превышение любого лимита приведёт к генерации предупреждения.

    Плитки дашбордов и секции блокнотов, созданные в Dynatrace версии ранее 1.296, не затрагиваются. Эти существующие плитки/секции будут возвращать те же результаты, что и раньше.

    • Read data limit (GB)

    Лимит в гигабайтах на объём данных, которые будут просканированы при чтении. * Record limit

    Максимальное количество записей результата, которые вернёт запрос. По умолчанию: 1000 записей. Чтобы увидеть больше записей, необходимо увеличить значение Record limit.

    • Если в вашем запросе нет limit, например:

      fetch logs

      применяется значение Record limit. По умолчанию вы увидите до 1000 записей. + Если ваш запрос также включает limit, например:

      fetch logs


| limit 2000

      применяется меньшее из двух значений (либо limit в вашем запросе, либо Record limit в веб-интерфейсе).

      В примере выше вы всё равно увидите только 1000 записей, если не увеличите значение Record limit. * Result size limit

    Максимальное количество байт результата, которые вернёт запрос. Для лучшей производительности при типичных запросах и небольших документах значение по умолчанию установлено на 1 МБ. * Sampling (только для Logs и Spans)

    Приводит к выборке подмножества записей Log или Span.

    Пороговые значения¶

    Настройка пороговых значений¶

    Для настройки пороговых значений в визуализации дашборда или блокнота

    1. Выберите для редактирования плитки визуализации.

    2. Выберите Thresholds.

    3. В Dashboards выберите вкладку Visual в боковой панели, затем выберите Thresholds.

    4. В Notebooks выберите Thresholds в боковой панели.
    5. Выберите Threshold.

    6. Определите пороговые значения. Для каждого диапазона:

    7. Выберите цвет для отображения этого диапазона

    8. Выберите оператор для определения порога этого диапазона
    9. Введите статическое значение для сравнения (с помощью выбранного оператора) с возвращённым значением

    10. Введите метку для связи с определённым диапазоном.

      • Метки применяются только к графикам с осью Y, таким как графики временных рядов и категориальные столбчатые диаграммы.
      • Метки не могут быть определены для таблиц и графика с одним значением.

    Пример настройки пороговых значений в Dashboards

    Этот пример использует столбчатую диаграмму, но опции применимы к другим визуализациям.

    1. В Dashboards создайте дашборд.
    2. Выберите и в разделе Snippets меню выберите Metrics > Chart top 10 hosts by CPU usage.
    3. В панели редактирования секции выберите вкладку Visual и выберите Bar.
    4. Выберите Thresholds.
    5. Выберите Threshold.

    Отобразится пустой набор полей пороговых значений. 6. Определите пороговые значения для отображаемой метрики. Вы можете наблюдать изменения на оси Y графика.

    В этом примере мы определяем три диапазона использования CPU с соответствующими цветами и метками.

    Вы можете видеть диапазоны, отображаемые на оси Y и в подсказке.

    Чтобы сбросить значения по умолчанию (отменить настройки пороговых значений), выберите значок корзины рядом с элементом.

    Редактирование пороговых значений¶

    1. Выберите для редактирования плитки визуализации.

    2. Выберите Thresholds.

    3. В Dashboards выберите вкладку Visual в боковой панели, затем выберите Thresholds.

    4. В Notebooks выберите Thresholds в боковой панели.
    5. С этого момента вы можете выполнить следующие действия (разверните строки для подробностей):

    Добавить пороговые значения

    Чтобы добавить пороговые значения (если выбранная визуализация поддерживает дополнительные пороги), выберите Threshold.

    Отключить пороговые значения

    Чтобы отключить (скрыть) существующие пороговые значения, используйте переключатель. Никакие настройки не теряются. Вы можете включить их снова, если передумаете.

    Удалить пороговые значения

    Чтобы удалить существующие пороговые значения, выберите значок корзины.

    Изменить настройки пороговых значений

    Чтобы изменить существующие настройки пороговых значений, просто отредактируйте поля.

    Добавить диапазон

    Чтобы добавить диапазон к существующим пороговым значениям, выберите Add range.

    Удалить диапазон

    Чтобы удалить диапазон из существующих пороговых значений, выберите кнопку удаления в этой строке.

    Единицы и форматы¶

    Для переопределения единиц и форматов по умолчанию в визуализации дашборда или блокнота

    1. Выберите для редактирования плитки визуализации.
    2. Выберите вкладку Visual.
    3. Выберите Units and formats.
    4. Выберите Override.
    5. Выберите Override.
    6. В выпадающем списке выберите элемент, для которого вы хотите добавить переопределение единиц.

    Это числовой столбец базового результата DQL, поэтому он зависит от запроса. Например:

    • Запрос fetch events возвращает события. Выпадающий список здесь позволяет выбрать числовое поле (например, transfer_size) из результатов.
    • Запрос timeseries avg(dt.host.cpu.usage) возвращает один временной ряд для avg(dt.host.cpu.usage). Этот временной ряд является единственным доступным для выбора вариантом в списке.

    • Определите переопределение.

    • Default unit: базовая единица, в которой были захвачены значения. Она равна None, если не была включена в результат DQL, или автоматически определяется единицей из результата DQL. Это поле не приводит к преобразованию.

    • Displayed unit: после определения единицы по умолчанию вы можете использовать Displayed unit для преобразования. Например, если результат DQL определил ваше числовое значение как Bytes, Displayed unit теперь предлагает подходящий список преобразований байтов, таких как Kilobyte и Megabyte. В отличие от Default unit, Displayed unit всегда выполняет числовое преобразование.
    • Decimals отображает количество десятичных знаков по умолчанию (степень точности). Чтобы увидеть это в действии, измените выбор Decimals и наблюдайте за изменением в визуализации.
    • Suffix отображает суффикс, добавляемый после единицы. Чтобы увидеть это в действии, введите строку и наблюдайте за изменением в визуализации. Когда вы не находите нужную единицу, вы можете использовать Suffix для отображения желаемой единицы.
    • Включите Abbreviate large numbers, если хотите отображать большие числа в сокращённой форме. Например, 1053 становится 1.1K.

    Чтобы сбросить значения по умолчанию (отменить настройки переопределения для выбранного элемента), выберите значок корзины рядом с элементом.

      На это:

      * Suffix отображает необязательный суффикс после единицы. Чтобы увидеть это в действии, введите строку и наблюдайте за изменением в визуализации.

    Чтобы сбросить значения по умолчанию (отменить настройки переопределения для выбранной метрики), выберите значок корзины рядом с метрикой.
