    Live Debugger

    • Latest Dynatrace
    • 8 мин. чтения

    Live Debugger -- это облачное приложение для сбора данных в реальном времени, которое обеспечивает мгновенный доступ к данным на уровне кода, необходимым для быстрого устранения неполадок и отладки в любой среде, от разработки до продакшена.

    • Получайте необходимые и актуальные данные без добавления дополнительного кода, ожидания повторного развёртывания или попыток воспроизвести проблемы локально.
    • С помощью неблокирующих точек останова мгновенно просматривайте полное состояние вашего приложения, включая трассировку стека, значения переменных и многое другое -- без остановки или прерывания работающего кода.

    Предварительные требования¶

    • Среда Dynatrace с подпиской Dynatrace Platform Subscription (DPS), включающей Code Monitoring.")
    • OneAgent версии 1.309+ с включёнными функциями OneAgent features: Java Live-Debugger и Node.js Live-Debugger
    • Режимы мониторинга: Full-Stack, Infrastructure или Discovery (с внедрением модуля кода контейнера)
    • Модули кода: Java и Node.js

    Начало работы¶

    1 из 3 Снимок отладочных данных в реальном времени, созданный после срабатывания неблокирующей точки останова.

    Активация функции Live Debugger¶

    Чтобы активировать функцию Live Debugger

    1. Перейдите в Settings > Collect and capture > General monitoring settings > OneAgent features.
    2. Включите Java Live-Debugger, Node.js Live-Debugger или оба варианта в зависимости от ваших потребностей.
    3. Перезапустите все затронутые процессы.

    Включение Observability for Developers¶

    Для использования Live Debugger необходимо подключить Observability for Developers. Вы можете включить её для определённых групп процессов, пространств имён Kubernetes, кластеров или всей среды.

    На уровне среды

    1. Перейдите в Settings > Collect and capture > Observability for Developers > Enable Observability for Developers.
    2. Включите Enable Observability for Developers.

    На уровне группы процессов

    1. Перейдите в Technologies & Processes Classic.
    2. Выберите категорию, а затем нужную группу процессов.
    3. На странице обзора группы процессов выберите Settings.
    4. Перейдите в Observability for Developers > Enable Observability for Developers.
    5. Включите Enable Observability for Developers.

    На уровне пространств имён и кластеров Kubernetes

    1. Перейдите в Kubernetes.
    2. Выберите нужное пространство имён или кластер.
    3. В правом верхнем углу выберите > Connection settings или, если недоступно, Anomaly detection settings.
    4. Закройте оверлей Kubernetes namespace anomaly detection или Connection settings. Должна отобразиться страница настроек пространства имён или кластера.
    5. Перейдите в Collect and capture > Observability for Developers > Enable Observability for Developers.
    6. Включите Enable Observability for Developers.

    Включение Live Debugging в ActiveGate¶

    ActiveGate версии 1.311+

    ActiveGate не является строго необходимым для работы Live Debugger, но значительно упрощает процесс, особенно в средах Kubernetes. Он позволяет свести взаимодействие с Dynatrace к одной точке, доступной локально. Помимо удобства, это решение оптимизирует объём трафика и снижает сложность сети и затраты.

    Развёртывание на хосте¶

    Модуль Live Debugger включён по умолчанию для развёртываний ActiveGate на хосте начиная с версии ActiveGate 1.311.

    Развёртывание через Kubernetes Operator¶

    Для развёртываний через Kubernetes Operator включите модуль Live Debugger через раздел возможностей ActiveGate в конфигурации DynaKube.

    Чтобы включить модуль Live Debugger

    1. Установите возможность debugging в файле DynaKube.yaml.
    activeGate:


capabilities:


- debugging

    Если вы используете прокси-сервер, настройте его в Environment ActiveGate, как описано в разделе Прокси для ActiveGate.

    Начало отладки приложения в реальном времени¶

    Чтобы начать сеанс отладки в реальном времени

    1. Перейдите в Live Debugger.
    2. Выберите (Debug Configuration) под заголовком приложения, чтобы открыть оверлей Customize your debug session.

    3. Определите область экземпляров, которые вы хотите отлаживать.

    4. Добавьте один или несколько фильтров из списка фасетов.

    5. Выберите фильтр в свойствах приложения.

    Получение исходного кода¶

    Каждый сеанс отладки должен содержать исходный код приложения, которое вы собираетесь отлаживать. Live Debugger позволяет быстро загружать исходные коды из локальной файловой системы и различных Git-провайдеров. При интеграции исходного кода в Live Debugger он остаётся между вашим репозиторием кода и локальным браузером.

    Существует два основных способа импорта исходного кода:

    • Автоматическое получение
    • Ручное получение

    Автоматическое получение¶

    Live Debugger может подключиться к репозиторию и автоматически получить исходный код для выбранного экземпляра.

    Ваш репозиторий может быть автоматически получен для выбранного экземпляра на основе ваших Git-разрешений, если ваш администратор уже настроил это.

    Чтобы узнать больше об автоматическом получении, см. Интеграция с системой контроля версий.

    Ручное получение¶

    Live Debugger позволяет легко загрузить соответствующий исходный код в приложение. Выберите в разделе Source Code, чтобы открыть список поддерживаемых Git-провайдеров, из которых можно получить исходные коды.

    Интеграция с локальными Git-серверами¶

    Live Debugger интегрируется напрямую с облачными версиями следующих Git-провайдеров как в локальном, так и в облачном варианте:

    • GitHub
    • Bitbucket
    • GitLab
    • Azure DevOps

    Чтобы включить интеграцию Live Debugger с вашим локальным Git-провайдером

    1. Перейдите в Settings > Apps > Git On-Premise Servers.

    2. Выберите Add item.

    3. Git Provider: укажите вашего Git-провайдера.

    4. Server URL: укажите HTTP/HTTPS URL вашего сервера.
    5. Выберите Save and close.

    Для использования Bitbucket On-Premises установите настольное приложение Dynatrace.

    Локальная файловая система¶

    Live Debugger включает настольное приложение (Dynatrace Desktop app), которое может получать доступ к репозиториям исходного кода из вашей локальной файловой системы. Это позволяет получать исходный код от локально размещённых Git-провайдеров.

    Windows: версии 7, 8 и 8.1 не поддерживаются. MacOS: все версии поддерживаются.

    1. В разделе Source Code выберите > Local Filesystem.
    2. Выберите Download desktop app, чтобы получить установочный файл для настольного приложения Dynatrace.
    3. Установите приложение в соответствии с инструкциями для вашей операционной системы.

    Установка неблокирующей точки останова¶

    Чтобы получить данные на уровне кода в реальном времени

    1. Выберите сервис, который вы хотите отладить.
    2. Перейдите к файловой системе.
    3. Установите неблокирующую точку останова на строке кода, которую вы хотите отладить.

    Неблокирующие точки останова -- это контрольные точки для строк кода, из которых вы хотите собирать отладочные данные. Эти точки останова не останавливают выполнение вашего кода. Вместо этого они содержат запрос на сбор отладочных данных. Установка этих точек останова и получение полученных снимков не влияют на ваше приложение или его пользователей.

    После установки неблокирующей точки останова убедитесь, что ваш код был вызван, чтобы данные были собраны и сразу отобразились в Live Debugger.

    Просмотр отладочных данных¶

    После срабатывания неблокирующей точки останова собранные данные отображаются в панели снимков в нижней части Live Debugger.

    Выберите нужный снимок для просмотра следующей информации:

    • Все локальные переменные и их значения.
    • Информация о процессе.
    • Полная трассировка стека до точки останова.
    • Информация о трассировке из работающего приложения.

    Вы также можете отлаживать код вверх по стеку из представления трассировки стека, включая сторонние компоненты.

    Концепции¶

    Application Observability : Невозможность доступа к данным на уровне кода по запросу -- это проблема, с которой сталкивается большинство разработчиков при устранении неполадок и отладке приложений. Вместо этого им приходится проходить длительные циклы развёртывания и исследовать логи для поиска нужных данных, чтобы понять, что происходит в работающем коде.

    Live Debugger : Live Debugger -- это облачное решение для отладки и сбора данных в реальном времени, обеспечивающее мгновенный доступ к отладочным данным на уровне кода, необходимым для устранения неполадок и понимания сложных современных приложений без дополнительного кодирования, повторного развёртывания или перезапуска.

    Вы можете использовать ![Live Debugger](https://dt-cdn.net/images/live-debugger-256-b934a2bad5.png "Live Debugger") **Live Debugger** в любой среде, от разработки до продакшена, и в любое время для мгновенного доступа к данным в реальном времени, необходимым для быстрого определения первопричины проблем.

    Динамическое логирование : Команды разработчиков часто стараются логировать как можно больше, поскольку зачастую не уверены, какие данные понадобятся для отладки проблемы. Это генерирует множество ненужных логов и ужасное соотношение сигнал/шум.

    Переходя к подходу Live Debugging, команды могут снизить затраты на логирование и, что ещё важнее, значительные усилия, необходимые для поддержки интенсивных конвейеров логирования и их очистки.

    Совместная разработка : Собранные отладочные данные можно совместно использовать между членами команды при отладке проблем приложения. Это позволяет каждому члену команды иметь собственное представление собранных отладочных данных и делиться ими с другими членами команды для улучшения совместной работы.

    В результате ![Live Debugger](https://dt-cdn.net/images/live-debugger-256-b934a2bad5.png "Live Debugger") **Live Debugger** может стать единственным источником истины, основным местом, где разработчики видят реальное положение дел и совместно работают над ним.

    Инспекция трассировок по запросу : Live Debugger собирает объектное представление трассировок, используемых вашим приложением. Разработчики могут затем соотносить захваченные отладочные данные с данными трассировки из вашего приложения.

    Обучение и адаптация разработчиков : Понимание нового кода может быть сложной задачей. Это вдвойне сложно, когда речь идёт об устаревшем коде или, как часто бывает, о коде, написанном кем-то другим.

    ![Live Debugger](https://dt-cdn.net/images/live-debugger-256-b934a2bad5.png "Live Debugger") **Live Debugger** позволяет разработчикам ускорить процесс обучения, давая им возможность наблюдать за незнакомым кодом, работающим в его родной среде.

    Сценарии использования¶

    Сбор отладочных данных в реальном времени¶

    • Мгновенно понимайте состояние вашего приложения с помощью неблокирующих точек останова и получайте мгновенный вывод, включая полную трассировку стека, локальные и глобальные переменные, данные трассировки и контекст, а также метрики сервера (процессор, память, измерения времени и многое другое).

    Более быстрое устранение неполадок¶

    • Мгновенно отлаживайте самые сложные потоки.
    • Исправляйте ошибки без каких-либо сложностей, накладных расходов или рисков.
    • Получайте данные в реальном времени без остановки вашего приложения.
    • Получайте видимость сторонних зависимостей и преодолевайте проблемы отладки, связанные с использованием компонентов с открытым исходным кодом, внешних зависимостей или устаревшего кода.

    Удобная отладка для разработчиков¶

    • Предоставьте разработчикам доступ к данным на уровне кода по запросу.
    • Получите полную видимость вашего кода с помощью простого и интуитивно понятного интерфейса. Оптимизируйте эффективность R&D: тратьте меньше времени на исправление ошибок и больше времени на создание ценности для вашей организации и клиентов.
    • Не покидайте зону комфорта, выбирая между плагином для IDE или веб-приложением Live Debugger.

    Более быстрое тестирование и оптимизация кода¶

    • Мгновенно определяйте, была ли достигнута определённая строка кода, с помощью исследовательской аналитики.
