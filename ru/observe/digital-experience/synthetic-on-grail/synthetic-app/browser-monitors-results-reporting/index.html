    Отчёты о результатах браузерных мониторов

    • Latest Dynatrace

    Отчёты браузерных мониторов предоставляют:

    • Панель предварительного просмотра — отображает результаты выполнения монитора, проблемы, ошибки и свойства монитора. Содержит краткую информацию с готовыми ссылками для быстрого перехода к деталям проблем и настройкам монитора.
    • Страница результатов отчётов — отображает более подробную информацию, включая визуализации результатов, такие как графики и таблицы, показывающие метрики доступности и производительности монитора за выбранный период.
    • Анализ выполнений — позволяет детально анализировать отдельные выполнения монитора.

    Для отображения результатов браузерного монитора

    1. Перейдите в Synthetic.
    2. Необязательно. Для фильтрации таблицы по браузерным мониторам выберите Browser в разделе Filter options слева.
    3. В списке браузерных мониторов выберите монитор, который хотите изучить. Справа появится панель предварительного просмотра.
    4. Для доступа к странице деталей выберите View details в правом верхнем углу панели предварительного просмотра.

    Визуализация метрик¶

    Верхняя панель отображает общую доступность и производительность монитора за выбранный период.

    Используйте панель фильтров в верхней части страницы для фильтрации выполнений браузерного монитора по одной или нескольким локациям.

    Под панелью фильтров отображаются плитки с важной информацией:

    • Сбой/Доступен/Нет данных — статус доступности монитора на момент последнего выполненного теста.
    • Активные проблемы — количество проблем, наблюдаемых в данный момент.
    • Затронутые локации — количество локаций, в которых произошли сбои за определённый период.
    • Доступность — процент доступности монитора за определённый период.
    • Общее время простоя — общее время сбоев для всех локаций за определённый период.
    • Длительность — среднее время ответа для всех локаций за определённый период.

    В правом верхнем углу страницы вы можете перейти к Анализу выполнений, Выполнению по требованию, а также к дополнительным настройкам браузерного монитора:

    • Редактировать
    • дополнительные опции (Отключить и Удалить)

    Доступность¶

    Карточка Availability отображает доступность монитора за выбранный период, агрегированную для всех локаций или для каждой локации отдельно. Она содержит информацию обо всех проблемах доступности, связанных с данным монитором, окнах обслуживания и незавершённых выполнениях. Проблемы отображаются в виде полосок-уведомлений над графиком доступности. Они показывают детали проблемы и отмечают период, когда проблема произошла.

    Также, если монитор не работает (недоступен) в одной или нескольких локациях, длительность сбоя (простоя) и затронутые локации отображаются в визуализации метрик над карточкой Availability.

    Монитор может быть недоступен в одной или во всех локациях. За определённый период может произойти несколько простоев. Их длительности суммируются для расчёта общей продолжительности простоя. Перекрывающиеся простои не учитываются повторно.

    Расчёт доступности¶

    Метрики доступности рассчитываются путём деления количества успешных выполнений ("работает") на общее количество выполнений.

    1. Определите количество успешных выполнений за период.

    Это общее количество выполнений за период минус количество неудачных ("не работает") выполнений. 2. Определите доступность.

    Разделите количество успешных выполнений на общее количество выполнений, а затем умножьте на 100, чтобы получить процент.

    Пример

    Предположим, у нас 5 неудачных выполнений за 35-минутный период.

    • Всего выполнений = 35.
    • Неудачных выполнений = 5.
    • Успешных выполнений = 35-5 = 30.
    • Доступность = (30/35) \* 100 = 0.8571 \* 100 = 85,71 процента.

    Исключение выполнений в период обслуживания из расчёта метрик¶

    Вы можете настроить окно обслуживания так, чтобы выполнения тестов не подавлялись в периоды обслуживания. Тогда, если во время обслуживания произойдут сбои, неудачные выполнения будут включены в расчёт метрик.

    В примере ниже доступность ниже 100%, потому что неудачные выполнения по умолчанию включаются в расчёт метрик.

    timeseries av = avg(dt.synthetic.browser.availability), by: {dt.entity.synthetic_test}


| fields dt.entity.synthetic_test, avgAV=arrayAvg(av)


| filter dt.entity.synthetic_test == "SYNTHETIC_TEST-2891B733D75330FA"

    Чтобы исключить выполнения периода обслуживания из расчёта метрик

    1. Добавьте измерение dt.maintenance_window_ids в запрос.
    2. Отфильтруйте выполнения, не покрытые окном обслуживания, с помощью isNull(dt.maintenance_window_ids).

    В примере ниже мы обнаружили неудачные выполнения в период обслуживания. Однако они не были включены в расчёт метрик, потому что мы использовали измерение dt.maintenance_window_ids вместе с условием фильтра isNull(dt.maintenance_window_ids).

    timeseries av = avg(dt.synthetic.browser.availability), by: {dt.entity.synthetic_test, dt.maintenance_window_ids}


| filter dt.entity.synthetic_test == "SYNTHETIC_TEST-2891B733D75330FA"


| filter isNull(dt.maintenance_window_ids)


| fields avgAV=arrayAvg(av)

    Интерполяция¶

    Описанный выше подход предполагает, что все выполнения происходят с одинаковой частотой. Однако в реальных условиях это не всегда так, поскольку частота выполнения монитора может меняться, а также могут инициироваться дополнительные выполнения по требованию.

    Для повышения точности расчёта введён механизм интерполяции: общее время мониторинга делится на точки данных на уровне минут. Все точки данных учитываются как выполнения, хотя они не обязательно совпадают с фактическими выполнениями.

    Чтобы увидеть интерполированные точки данных, добавьте измерение interpolated в запрос.

    timeseries avg(dt.synthetic.browser.availability), by:{dt.entity.synthetic_test, interpolated}


| filter dt.entity.synthetic_test == "SYNTHETIC_TEST-7E0A1CF206C4A1C5"

    Результат ниже показывает синие точки данных, совпадающие с фактическими выполнениями, и белые точки данных, которые не совпадают.

    Представим, что монитор настроен на выполнение тестов каждые пять минут. Таким образом, каждая пятая точка данных (синяя) совпадает с фактическим выполнением. Все точки данных после первого неудачного выполнения и до первого успешного выполнения считаются "неудачными". Поэтому при расчёте метрик доступности количество неудачных выполнений рассчитывается следующим образом:

    Неудачные выполнения = первое фактическое неудачное выполнение + все последующие "неудачные" точки данных.

    Уведомления над графиком доступности¶

    Также карточка Availability отображает уведомления над графиком доступности. Уведомления показывают детали окна обслуживания, сбоев и невыполненных или незавершённых тестов. Уведомление отображается в виде полоски над периодом, когда было применено окно обслуживания, произошёл сбой или некоторые тесты не были выполнены или завершены. Наведите курсор на полоску, чтобы увидеть детали:

    • Локальный сбой браузерного монитора — сбой произошёл в одной или нескольких локациях, из которых монитор выполнял тесты.
    • Глобальный сбой браузерного монитора — сбой произошёл во всех локациях, из которых монитор выполнял тесты.

    Локальный и глобальный сбой — просмотр деталей проблемы

    Для просмотра деталей проблемы

    1. Нажмите на полоску, чтобы закрепить всплывающую подсказку Локальный сбой браузерного монитора или Глобальный сбой браузерного монитора.
    2. Наведите курсор на подсказку.
    3. Выберите Просмотреть детали проблемы.

    4. Выберите один из доступных вариантов:

    5. Problems для просмотра деталей проблемы в Latest Dynatrace

    6. Problems Classic для просмотра деталей проблемы в Dynatrace Classic
    7. Окно обслуживания — в данном периоде было активно окно обслуживания.

    Как посмотреть окно обслуживания и что происходит при включённом/выключенном выполнении монитора

    Чтобы увидеть окно обслуживания

    1. Нажмите на полоску, чтобы закрепить всплывающую подсказку Окно обслуживания.
    2. Наведите курсор на подсказку Окно обслуживания.
    3. Выберите Просмотреть настройки окна обслуживания.

    4. Выберите Settings Classic.

    5. Когда опция Отключить выполнение синтетического монитора включена в окне обслуживания, запланированные автоматические тесты подавляются во время периода обслуживания, и результаты тестов за этот период отсутствуют. Только тесты по требованию не подавляются.

    6. Когда опция Отключить выполнение синтетического монитора выключена в окне обслуживания, тесты выполняются во время периода обслуживания, но из-за обслуживания результаты тестов могут отличаться от тех, которые наблюдаются в нормальных условиях.

    Невыполненные или незавершённые тесты¶

    Если некоторые тесты не были выполнены или завершены из-за неправильных конфигураций монитора или проблем на стороне Dynatrace, вы можете увидеть причину:

    • Нет результата — выполнение началось, но результаты не были доставлены. Обратитесь в службу поддержки Dynatrace.
    • Нет возможности — вы пытаетесь выполнить мониторы, которые не поддерживаются локацией (например, NAM ICMP мониторы с локации с неактивным ICMP).
    • Отброшен, не выполнен — вероятная причина — перегруженная приватная локация. Проверьте метрики ёмкости. Если проблем с метриками нет, обратитесь в службу поддержки Dynatrace.

    Производительность¶

    Карточка Performance отображает:

    • Среднюю длительность каждого шага по всем локациям (вкладка Step performance).
    • Общую длительность всех шагов для каждой локации (вкладка Split by location).

    Она помогает выявить медленные шаги или региональные проблемы производительности.

    Отображение проблем в разделе «Производительность»¶

    • Красная полоска над графиком указывает на текущую проблему.
    • Серая полоска над графиком указывает на закрытую проблему.

    В этом разделе отмечаются только проблемы производительности.

    Если монитор нарушает пороговое значение производительности — как для общей длительности всех шагов, так и для длительности отдельного шага — сплошная красная линия появляется над графиком производительности на время существования проблемы. Нажмите на красную полоску, чтобы отобразить ссылку на страницу обзора проблемы.

    Просмотр деталей проблемы

    1. Наведите курсор на полоску проблемы, чтобы увидеть детали текущей/закрытой проблемы во всплывающей подсказке.
    2. Нажмите на полоску проблемы и наведите курсор на подсказку для отображения дополнительных опций.
    3. Выберите Просмотреть детали проблемы для открытия деталей проблемы в Problems .

    Проблемы¶

    Карточка Problems показывает проблемы производительности (нарушение порогового значения) и доступности (локальный или глобальный сбой).

    Существуют три основных типа проблем для браузерных мониторов:

    • Глобальный сбой (доступность).
    • Локальный сбой (доступность).
    • Нарушение порогового значения производительности (проблема производительности для суммы всех шагов или отдельных шагов).

    Проблемы производительности могут объединять нарушения пороговых значений на уровне монитора и отдельных шагов.

    Раздел проблем перечисляет все проблемы за выбранный период, как открытые, так и закрытые. Кроме того, вы можете:

    • Просматривать детали проблемы в виде таблицы с датой начала и длительностью.
    • Раскрывать каждую проблему для просмотра затронутых сущностей и профилей оповещений.

    Проблемы вместе с составляющими их событиями дают полную картину количества и масштаба проблем вашего монитора.

    Синтетические шаги и действия¶

    Карточка Synthetic steps and actions показывает детали для каждого браузерного шага, который должен выполнить монитор. Это может быть загрузка страницы, нажатие на кнопку или любой другой элемент управления и т.д. Для каждого шага вы можете увидеть такие детали, как:

    • Тип действия — например, Click.
    • Средняя длительность — среднее время ответа за определённый период, например, за последний час.
    • Снимок экрана — снимок экрана, сделанный во время выполнения шага.

    Нажмите на значок раскрытия для шага, чтобы увидеть время ответа отдельных выполнений за определённый период (например, за последний час). Это позволяет оценить общую производительность шага. Значок раскрытия недоступен и не может быть выбран, если данные о производительности шага отсутствуют.

    События¶

    Карточка Events показывает все события, составляющие проблемы. События для активных и разрешённых проблем отображаются в списке и на временной шкале.

    Наведите курсор на временной слот на временной шкале событий, чтобы увидеть тип и количество событий, сгенерированных за этот интервал. Нажмите на временной слот, чтобы просмотреть подсказку с событиями, произошедшими в нём.

    Выберите тип события, чтобы увидеть список событий. Событие создаётся для каждой локации, где, например, ваш монитор нарушает пороговые значения производительности на уровне шага или монитора. Выберите отдельное событие для просмотра деталей.

    Ошибки¶

    Карточка Errors отображает ошибки, обнаруженные за выбранный период.

    На карточке вы можете:

    • Просматривать детали ошибки в виде таблицы с шагом и количеством вхождений.
    • Выбрать Анализировать ошибки.

    Фронтенды¶

    Карточка Frontends отображает фронтенды, к которым привязан монитор.

    Локации¶

    Карточка Locations показывает статус последнего выполнения монитора для каждой локации. Статус отображается цветами:

    • Зелёный — локация была доступна во время последнего выполнения монитора.
    • Красный — во время последнего выполнения монитора был обнаружен сбой.
    • Серый — данные о выполнениях для этой локации отсутствуют.

    Свойства и теги¶

    Эта информационная карточка показывает количество шагов, количество и список локаций, частоту выполнения монитора и примененные теги.

    Выберите Редактировать для редактирования или добавления дополнительных тегов.

    Анализ выполнений¶

    При использовании браузерного монитора вы можете детально анализировать отдельные выполнения монитора.

    Анализ выполнений без нового опыта браузерного мониторинга¶

    Не применимо к новым Environment Dynatrace, созданным после 26 января 2026 года.

    Если новый опыт браузерного мониторинга не активирован, вы можете анализировать выполнения на странице Многомерный анализ в Synthetic Classic.

    1. Выберите Анализировать выполнения в верхней части страницы деталей монитора.

    Вы будете перенаправлены на страницу Многомерный анализ в Dynatrace Classic. 2. В разделе Детальный временной фрейм анализа выберите выполнение, чтобы закрепить его подсказку. 3. Наведите курсор на подсказку и выберите Просмотреть выполнение.

    Анализ выполнений с новым опытом браузерного мониторинга¶

    Лицензия DPS

    Если новый опыт браузерного мониторинга активирован, вы можете анализировать выполнения через каскадную диаграмму (Waterfall chart) в Latest Dynatrace.

    Во время выполнения монитора может быть загружена одна или несколько страниц. Набор компонентов (ресурсов) загружается вместе со страницей. Когда вы анализируете отдельные выполнения монитора с помощью каскадной диаграммы, вы можете увидеть подробную информацию о каждом ресурсе, загруженном со страницей во время выполнения монитора. Подробности включают:

    • Тип ресурса, например, скрипт, изображение, шрифт, данные.
    • Имя ресурса.
    • Длительность загрузки ресурса в мс.
    • Инициатор ресурса, например, fetch, script, link. Подробнее см. Инициатор ресурса.
    • Статус блокировки рендеринга.
    • Время начала и окончания загрузки ресурса.

    Каскадная диаграмма может помочь обнаружить и изолировать конкретные ресурсы в конкретных выполнениях, которые снижают общую производительность. Также она может помочь проанализировать причины сбоев монитора.

    Чтобы проанализировать выполнения монитора через каскадную диаграмму

    1. Выберите Анализировать выполнения в правом верхнем углу страницы деталей монитора.
    2. Найдите нужное выполнение в таблице Выполнения. Каждая строка таблицы — это запись выполнения. Вы можете использовать фильтр для сужения списка выполнений. Можно фильтровать по локации или временному фрейму.

    Если включён автоматический повтор при ошибке, монитор выполняет вторую попытку в случае неудачи первой. В таблице Выполнения вторая попытка отображается по умолчанию, и в записи выполнения появляется значок — нажмите на него, чтобы увидеть также первую попытку. 3. Выберите выполнение.

    Появляется таблица Pages с деталями выполнения. В строке Steps вы можете увидеть все шаги, выполненные во время выполнения монитора. Период выполнения шага выделен синей полоской. Наведите курсор на полоску для деталей шага, таких как длительность, тип и элемент интерфейса. Вы можете отфильтровать конкретный шаг. В этом случае только полоска, связанная с отфильтрованным шагом, остаётся синей. Все остальные полоски становятся серыми.

    Под строкой Steps отображаются страницы, загруженные во время выполнения монитора.

    Во время выполнения монитора страницы могут открываться в отдельных окнах или вкладках браузера. В этом случае для каждого окна появляется запись в таблице со значком. Нажмите на значок, чтобы увидеть страницы, загруженные в этом окне.

    Под строками Steps и окна (при наличии) каждая последующая строка таблицы — это запись загруженной страницы. Период загрузки страницы выделен синей полоской. 4. Для нужной страницы выберите запись в таблице.

    Появляется карточка Requests waterfall с каскадной диаграммой для выбранной страницы. Диаграмма показывает все ресурсы, загруженные со страницей во время выполнения:

    • Шага монитора, когда страница была загружена.
    • Всех последующих шагов монитора, когда браузерный монитор взаимодействовал со страницей (например, клики, нажатия клавиш).

    Вы можете выбрать ресурс для просмотра деталей его производительности. Ресурсы, загруженные с ошибками, имеют красную метку в колонке Type.

    Инициатор ресурса¶

    Инициатор ресурса — это функция веб-платформы, которая инициировала загрузку ресурса. Зная инициатора ресурса, вы можете, например, обнаружить XHR-действия — их инициаторами обычно являются xmlhttprequest и fetch.

    Инициатор ресурса не представляет тип загруженного контента. Например, файл .css может быть загружен элементом <link> — в этом случае link указывается как инициатор ресурса.

    Инициатор Запрос инициирован
    audio Атрибут src элемента <audio>
    beacon Метод navigator.sendBeacon()
    body Атрибут background элемента <body>
    css Функция CSS url()
    early-hint Ответ 103 Early Hint
    embed Атрибут src элемента <embed>
    fetch Метод fetch()
    frame Загрузка элемента <frame>
    icon Favicon; нестандартный, сообщается только Safari
    iframe Атрибут src элемента <iframe>
    image Элемент <image>
    img Атрибут src или srcset элемента <img>
    input Элемент <input> типа image
    link Элемент <link>
    navigation Запрос навигации
    object Элемент <object>
    other (ни одно условие не совпало)
    ping Ping элемента <a>
    script Элемент <script>
    track Атрибут src элемента <track>
    video Атрибут poster или src элемента <video>
    xmlhttprequest XMLHttpRequest
