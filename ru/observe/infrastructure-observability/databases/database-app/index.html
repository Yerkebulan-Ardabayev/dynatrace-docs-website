    Приложение Databases¶

    Приложение Databases¶

    • Последняя версия Dynatrace
    • Приложение
    • Чтение: 4 мин
    • Обновлено 26 февраля 2026 г.

    Dynatrace Databases позволяет отслеживать, анализировать и оптимизировать среды баз данных с ясностью и контролем. Приложение предоставляет детальную информацию о производительности, состоянии и конфигурации, помогая выявлять проблемы на ранних стадиях и поддерживать надежную работу.

    Благодаря интеграции с экосистемой наблюдаемости Dynatrace, Databases связывает метрики баз данных с данными приложений и инфраструктуры, обеспечивая более быстрое устранение неполадок и более обоснованное принятие решений.

    Предварительные требования¶

    Прежде чем начать, убедитесь в следующем:

    • Конфигурация ActiveGate

    • Назначьте одну или несколько групп ActiveGate для удаленного подключения к целевым серверам баз данных.

    • Необходимые компоненты

    • Для баз данных, требующих дополнительных компонентов (например, драйверов JDBC), установите их на все назначенные ActiveGate в соответствии с рекомендациями по расширениям.

    • Сеть и разрешения

    • Проверьте сетевое подключение и правила межсетевого экрана для каждого типа базы данных. ActiveGate должен иметь возможность установить прямое сетевое соединение с хостом базы данных.

    • Создайте учетную запись для мониторинга с соответствующими разрешениями (например, доступ к системным представлениям, метрикам производительности и схемам).

    Разрешения¶

    В следующей таблице описаны необходимые разрешения.

    davis:analyzers:execute

    Выполнение анализатора Davis для проблем сущностей

    settings:objects:read

    Чтение объектов настроек из settings V2 для Ownership и SLO

    settings:objects:write

    Запись объектов настроек из settings V2 для Ownership и SLO

    state:user-app-states:read

    Чтение состояния пользовательского приложения

    state:user-app-states:write

    Сохранение состояния UI

    storage:buckets:read

    Чтение бакетов из Grail buckets

    storage:entities:read

    Чтение сущностей из Grail

    storage:events:read

    Чтение событий из Grail

    storage:logs:read

    Чтение логов из Grail

    storage:metrics:read

    Чтение метрик из Grail

    Ограничение доступа к данным¶

    В общем случае пользователям приложения Databases необходимы базовые разрешения storage:** для доступа к данным из Grail.

    Однако для строгого контроля доступа вы можете создать пользовательскую политику, ограничивающую доступ к бакету Grail default_database_monitoring.

    Бакет default_database_monitoring -- это место, где хранятся все данные, полученные расширениями баз данных, включая логи. Вы можете использовать следующее выражение для создания политики:

    ALLOW storage:buckets:read WHERE storage:bucket-name = 'default_database_monitoring'

    Для получения дополнительной информации см. Работа с политиками и Предоставление доступа к Grail.

    Шаги установки¶

    1. Установите Databases из Dynatrace Hub.
    2. Настройте приложение для мониторинга поддерживаемых экземпляров баз данных. Инструкции по настройке для конкретных поставщиков см. в разделе Начало работы с мониторингом баз данных.
    3. Добавьте необходимые конфигурации мониторинга (например, учетные данные, конечные точки) для каждого экземпляра.

    Начало работы¶

    Используйте Databases для мониторинга, анализа и оптимизации сред баз данных. Приложение интегрируется с экосистемой наблюдаемости Dynatrace и предоставляет практические рекомендации, помогающие обеспечить надежность, улучшить производительность и сократить время простоя.

    Обзор приложения Databases¶

    Централизованная панель мониторинга¶

    Вкладка Overview предоставляет высокоуровневую сводку вашей среды баз данных:

    • Состояние баз данных: Визуализация здоровых и нездоровых экземпляров баз данных для быстрого выявления проблемных областей.
    • Метрики хостов: Доступ к данным с хостов, на которых работает OneAgent, для более глубокого анализа зависимостей ресурсов.

    Исследование экземпляров баз данных¶

    Выберите вкладку Explorer для просмотра списка всех отслеживаемых экземпляров. Таблица Database instances отображает эти экземпляры.

    • Состояние: Унифицированные статусы и проблемы, обнаруженные Dynatrace Intelligence, помогают быстро выявлять критические проблемы.
    • Утилизация: Метрики, такие как использование CPU, потребление памяти, пользовательские вызовы и активные сессии, дают представление об эффективности использования ресурсов.
    • Детали хоста: Детализация метрик хоста или доступ к представлению Infrastructure and Operations для комплексного анализа.
    Панель деталей сущности¶

    Для обзора отдельного экземпляра выполните одно из следующих действий:

    • Перейдите на вкладку Explorer и выберите имя экземпляра в таблице Database instances.
    • В крайнем правом столбце выберите значок (Statement performance).

    Здесь вы можете получить доступ к панели Entity details и проанализировать все метрики, собранные расширением.

    Анализ производительности запросов¶

    Анализируйте ресурсоемкие запросы для оптимизации производительности:

    • Фильтруйте запросы по времени, CPU, диску или метрикам ожидания.
    • Получите доступ к планам выполнения, чтобы понять взаимодействие запросов и определить возможности для оптимизации.
    Производительность запросов¶

    Если расширение предоставляет соответствующую информацию, вы можете отобразить Statement performance analysis для отслеживания производительности запросов, потребляющих наибольшее количество ресурсов.

    Для фокусировки анализа:

    • Установите Filter statements на строку поиска.
    • Установите Contextual analysis на контекст предопределенных метрик (Time, CPU, Disk или Waits), или выберите значок настроек столбцов для настройки столбцов и адаптации контекста под ваши нужды.
    • Выберите Request execution plan, чтобы понять, как база данных выполняет запрос для оптимизации.

    Расширенные возможности¶

    Приложение помогает проактивно устранять потенциальные проблемы до того, как они повлияют на работу, сокращать время простоя и повышать надежность.

    • Обнаружение и устранение аномалий с помощью анализа на основе ИИ.
    • Определение первопричин снижения производительности или сбоев.
    • Получение практических рекомендаций по оптимизации запросов и конфигураций.

    Преимущества¶

    Databases помогает поддерживать надежную, эффективную и оптимизированную работу баз данных. Приложение проактивно решает проблемы и предоставляет практические рекомендации, позволяя вам сосредоточиться на стратегических приоритетах.

    • Минимизация простоев: Обнаружение и устранение проблем до того, как они повлияют на работу.
    • Улучшение производительности: Оптимизация запросов, схем и конфигураций для повышения эффективности.
    • Обеспечение надежности: Поддержка стабильной работы баз данных в различных средах.
    • Повышение продуктивности: Автоматизация задач наблюдаемости и сокращение ручного устранения неполадок.

    Концепции¶

    Оценка состояния (Health score)¶

    Предопределенная, ненастраиваемая метрика оценивает доступность, производительность, конфигурацию и использование ресурсов.

    Оповещения о состоянии и предупреждающие сигналы¶

    Оповещения о состоянии и предупреждающие сигналы помогают отслеживать инфраструктуру, предоставляя четкую и практическую информацию. Эти функции снижают информационный шум от проблем инфраструктуры и улучшают возможности оповещения, позволяя сосредоточиться на самом важном. Это достигается за счет лучшей категоризации обнаруженных неисправностей.

    • Для критических событий создается оповещение о состоянии (Health alert), которое инициирует расследование Dynatrace Problems.
    • Для некритических ситуаций предупреждающий сигнал (Warning signal) информирует вас о потенциальной проблеме.

    Вы можете настроить готовые оповещения о состоянии и предупреждающие сигналы через вкладку Alert templates.

    На вкладке Alert templates предоставлены предопределенные шаблоны оповещений для наиболее популярных поставщиков баз данных. Легко создайте новое оповещение, выбрав шаблон и нажав New Alert. Затем либо настройте оповещение в мастере Anomaly Detection, либо создайте оповещение в один шаг.

    Все пользовательские оповещения, подробное описание возможностей и ограничений см. в разделе Anomaly Detection.

    Инсайты¶

    Инсайты выявляют закономерности, аномалии и тенденции на основе встроенной экспертизы в предметной области. Они выходят за рамки необработанных метрик, предоставляя значимые выводы, такие как снижение производительности, узкие места ресурсов и нарушения лучших практик. Эти инсайты помогают сосредоточиться на самом важном и быстро предпринять обоснованные корректирующие действия.

    Гибкость развертывания¶

    Databases поддерживает различные модели развертывания:

    • Локальное (On-premises): Мониторинг традиционных конфигураций баз данных.
    • Облачное (Cloud): Обеспечение видимости облачных баз данных (например, AWS RDS).
    • Гибридное (Hybrid): Обеспечение согласованной наблюдаемости в смешанных средах.

    Сценарии использования¶

    Понимание состояния баз данных¶

    Databases предоставляет обзор состояния баз данных в реальном времени. Приложение непрерывно оценивает ключевые метрики для формирования оценки состояния, отражающей текущее состояние вашей среды баз данных. Эта оценка состояния помогает быстро определить области, требующие внимания.

    • Отслеживает показатели производительности, такие как время выполнения запросов, использование ресурсов и проблемы с подключением.
    • Выявляет аномалии и потенциальные риски, влияющие на надежность баз данных.

    Анализ и оптимизация производительности запросов¶

    Запросы часто являются основной причиной неэффективности баз данных. Приложение предоставляет инструменты для анализа выполнения запросов и определения областей для улучшения. Это гарантирует, что ваши базы данных смогут эффективно справляться с нагрузками.

    • Обнаруживает медленные или неэффективные запросы, влияющие на производительность базы данных.
    • Предоставляет рекомендации по оптимизации запросов, такие как переписывание запросов или добавление индексов для улучшения производительности.
    • Предлагает детальные планы выполнения, помогающие понять, как запросы взаимодействуют с вашей базой данных.

    Обнаружение и устранение проблем баз данных¶

    Databases помогает решать проблемы до того, как они повлияют на работу баз данных. Приложение использует возможности ИИ Dynatrace для выявления проблем, анализа их первопричин и предложения практических решений.

    • Обнаруживает аномалии в поведении баз данных с помощью анализа на основе ИИ.
    • Определяет первопричины снижения производительности или сбоев.
    • Рекомендует шаги по устранению, такие как изменение конфигурации или оптимизация запросов, на основе практических рекомендаций.

    Интеграция с Dynatrace для сквозной наблюдаемости¶

    Databases является частью экосистемы Dynatrace и обеспечивает комплексную наблюдаемость всего технологического стека. Приложение интегрируется с другими инструментами Dynatrace для предоставления единого представления базы данных и ее зависимостей.

    • Grail: Обработка больших объемов данных для масштабируемой аналитики.
    • Smartscape: Построение карты зависимостей в реальном времени для контекстно-зависимого анализа.
    • Dynatrace Intelligence AI: Обнаружение аномалий и предоставление интеллектуальных рекомендаций.
