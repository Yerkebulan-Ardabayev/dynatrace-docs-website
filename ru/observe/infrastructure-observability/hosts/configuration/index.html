    Настройки на уровне хоста

    Во многих случаях вы можете настраивать параметры мониторинга на уровне среды, группы хостов или хоста.

    • Настройки на уровне хоста переопределяют настройки на уровне группы хостов и среды.
    • Настройки на уровне группы хостов переопределяют настройки на уровне среды.

    Для настройки параметров на уровне хоста

    1. Перейдите в Hosts Classic.
    2. Найдите и выберите ваш хост для отображения страницы обзора хоста.
    3. В правом верхнем углу страницы обзора хоста выберите More (...) > Settings.

    Здесь выбирайте элементы на левой панели для навигации по страницам настроек уровня хоста. В этом примере вы увидите страницу настроек General для HOST-001.

    Навигация по иерархии

    Изменения настроек на уровне хоста переопределяют настройки на уровне группы хостов и среды.

    • Если настройки на уровне хоста не были изменены, отображается сообщение со ссылкой вверх от настроек уровня хоста к аналогичным настройкам на уровне группы хостов. Если настройки для группы хостов также не настроены, аналогичное сообщение предлагает ссылку вверх от настроек группы хостов к настройкам среды.
    • Если настройки уровня хоста были изменены, отображается сообщение: «These settings are overriding [host group name] (Host group) settings.» Вы можете выбрать имя группы хостов для перехода к этим настройкам или выбрать Remove override, если хотите удалить настройки уровня хоста и использовать настройки группы хостов.

    General¶

    Страница General в Host settings отображает таблицу отслеживаемых технологий.

    1. На странице Host settings уровня хоста выберите General.

    Таблица Monitored technologies содержит технологии мониторинга на выбранном хосте:

    • Technology — название технологии
    • Type — тип мониторинга, например JMX monitoring, OneAgent extension, Custom extension и Service insights
    • Configuration — текущий уровень конфигурации для выбранного хоста
    • Monitoring — состояние мониторинга этой технологии на выбранном хосте

    • Выберите в столбце Edit для просмотра параметров конфигурации:

    • Use host configuration определяет, использовать ли настройки мониторинга уровня хоста, переопределяя аналогичные настройки среды и группы хостов.

    • При включении Use host configuration становятся доступны специфичные для технологии настройки.

    Для управления настройками мониторинга хоста перейдите в Мониторинг хоста.

    Мониторинг хоста¶

    OneAgent автоматически отслеживает хост и его процессы, сервисы и приложения, но вы можете отключить мониторинг, полностековый мониторинг или автоинъекцию на уровне хоста.

    1. На странице Host settings выберите Host monitoring. На этой странице три вкладки:

    2. Monitoring

    3. Monitoring Mode
    4. Advanced Settings
    5. Перейдите в Monitoring и установите Monitor this host для включения или отключения мониторинга выбранного хоста.

    6. Перейдите в Monitoring Mode и установите Full-Stack, Infrastructure или Discovery для включения или отключения выбранного режима мониторинга для выбранного хоста.

    7. Подробнее см. Режимы мониторинга OneAgent.

    8. Режим мониторинга OneAgent автоматически перезаписывает эту настройку при каждом изменении через oneagentctl или при подключении OneAgent.
    9. Перейдите в Advanced settings и установите ProcessAgent injection или CodeModule injection для включения или отключения автоматической инъекции для выбранного хоста.
    10. Выберите Save changes.

    Контейнерные технологии¶

    Dynatrace OneAgent автоматически отслеживает все процессы, работающие на ваших отслеживаемых хостах. В контейнерных средах (например, Kubernetes, OpenShift, Cloud Foundry или Docker) OneAgent автоматически внедряет модули кода в контейнеризованные процессы для обеспечения полностековой видимости приложений, работающих в контейнерах.

    1. На странице Host settings выберите Container technologies.

    2. Для каждого типа контейнера включите или отключите Enabled для определения автоматической инъекции модулей кода.

    3. При включении автоинъекция обеспечивает глубокий мониторинг всех процессов в контейнерах на уровне запросов и распределённых трассировок для выбранного хоста.

    4. При отключении OneAgent не будет выполнять инъекцию в контейнер выбранного типа на выбранном хосте.

    Параметры дисков¶

    OneAgent автоматически обнаруживает и отслеживает все ваши точки монтирования, но вы можете создать правила-исключения для удаления определённых дисков из списка мониторинга.

    1. На странице Host settings выберите Disk options.

    2. Настройте специфичные для хоста параметры дисков по необходимости.

    3. Show all NFS disks Linux

      При отключении OneAgent пытается дедуплицировать NFS-диски. Отключено по умолчанию. * Exclude disks

      Вы можете создать правила-исключения для удаления дисков из мониторинга.

      Подробнее см. Исключение дисков и сетевого трафика из мониторинга хоста.

    Расширение Disk Analytics¶

    Linux

    Установите расширение Disk Analytics для получения более детальной видимости локальных хранилищ данных и их томов, разделов и RAID-массивов на хостах Linux.

    1. На странице Host settings выберите Disk Analytics Extension.
    2. Включите или отключите Enable Disk Analytics data collection для определения сбора данных Disk Analytics на выбранном хосте.

    Если вы включите сбор данных без добавления расширения, данные будут видны только в Data Explorer.

    Подробнее об установке и использовании расширения Disk Analytics см. Расширение Disk Analytics.

    Трафик NetTracer¶

    Linux

    NetTracer — это инструмент с открытым исходным кодом для трассировки TCP-событий и сбора метрик сетевых подключений в Linux.

    1. На странице Host settings выберите NetTracer traffic.
    2. Включите или отключите Enable NetTracer traffic network monitoring для определения мониторинга сетевого трафика NetTracer на выбранном хосте.
    3. Выберите Save changes.

    Подробнее см. Расширенный мониторинг сети.

    Исключение сетевого трафика¶

    OneAgent автоматически обнаруживает и отслеживает весь ваш сетевой трафик, но вы можете исключить трафик на определённых сетевых интерфейсах или хостах из мониторинга.

    1. На странице Host settings выберите Exclude network traffic.

    На этой странице два раздела:

    • Exclude NIC — этот раздел содержит все сетевые интерфейсы, исключённые из мониторинга сетевого трафика на определённом интерфейсе.

      • Для добавления записи выберите Add item
      • Для удаления записи выберите Delete row
      • Для редактирования записи разверните Details

      • Exclude IP — этот раздел содержит все IP-адреса хостов для исключения при расчёте связности (другие метрики всё равно будут рассчитываться).

      • Для добавления записи выберите Add item

      • Для удаления записи выберите Delete row
      • Для редактирования записи разверните Details
      • Выберите Save changes.

    Подробнее см. Исключение дисков и сетевого трафика из мониторинга хоста.

    Обнаруженные процессы¶

    Страница Detected processes представляет собой таблицу процессов, обнаруженных на выбранном хосте, доступную только для чтения.

    • Для управления настройками мониторинга процессов выберите ссылку Process group monitoring.
    • Для включения или отключения глубокого мониторинга определённых групп процессов на хосте выберите ссылку Process group deep monitoring.

    Подробнее см. Глубокий мониторинг процессов.

    Мониторинг групп процессов¶

    Для включения или отключения глубокого мониторинга определённой группы процессов на хосте

    1. На странице Host settings выберите Process group monitoring.

    Таблица содержит группы процессов с настройками мониторинга, специфичными для хоста.

    • Для добавления записи выберите Add process group, выберите Process group и задайте Monitoring state для выбранной группы процессов.
    • Для удаления записи выберите Delete row
    • Для редактирования записи разверните Details и измените настройки.
    • Выберите Save changes.

    Подробнее см. Глубокий мониторинг процессов.

    Флаги обнаружения групп процессов¶

    Страница Built-in detection rules содержит настройки флагов обнаружения групп процессов для выбранного хоста.

    Для включения или отключения флагов обнаружения групп процессов

    1. На странице Host settings выберите Process group detection flags.

    Если настройки уровня хоста не настроены, отображается сообщение со ссылкой на аналогичные настройки на уровне группы хостов и далее на уровень среды. 2. Включайте или отключайте правила обнаружения по необходимости для установки специфичных для хоста флагов обнаружения групп процессов.

    Наведите курсор на значок информации рядом с правилом для просмотра деталей правила. 3. Выберите Save changes.

    Подробнее см. Обнаружение групп процессов.

    Декларативная группировка процессов¶

    Dynatrace автоматически отслеживает группы процессов известных типов технологий или потребляющие значительные ресурсы. С помощью декларативной группировки процессов вы можете автоматически отслеживать дополнительные технологии.

    Страница Declarative process grouping содержит все группы процессов, определённые через декларативную группировку.

    Для добавления декларативной группы процессов в таблицу

    1. На странице Host settings выберите Declarative process grouping.
    2. Выберите Add process group.
    3. Введите имя, идентификатор и параметр отчётности.
    4. Выберите Add detection rule для описания способа обнаружения группы процессов: свойство и условие. См. инструкции на экране для помощи с условиями.
    5. Выберите Save changes.

    Подробнее см. Обнаружение групп процессов.

    Доступность процессов¶

    Используйте Process availability для мониторинга наличия хотя бы одного процесса, соответствующего указанному правилу мониторинга, на вашем хосте. Если ни один процесс на вашем хосте не соответствует правилу, вы получите оповещение. Если вы также включите снимки экземпляров процессов, вы получите подробный отчёт об активности наиболее ресурсоёмких процессов, а также о последней активности процессов, соответствующих правилу.

    Для добавления правила мониторинга процессов

    1. На странице Host settings выберите Process availability.
    2. Выберите Add monitoring rule.
    3. Введите имя правила.
    4. Выберите Add detection rule для описания способа обнаружения процесса: свойство и условие. См. инструкции на экране для помощи с условиями.
    5. Выберите Add property для определения дополнительных свойств ключ-значение, прикрепляемых к инициированному событию.
    6. Выберите Save changes.

    Подробнее см. Доступность процессов.

    Снимки экземпляров процессов¶

    Если вы включите снимки экземпляров процессов, Dynatrace изучает наиболее ресурсоёмкие процессы, работающие на вашем хосте, и процессы, отслеживаемые через Доступность процессов. При возникновении инициирующего события метрики, полученные за 10 минут до и 10 минут после события для этих процессов, отправляются в кластер.

    Для включения снимков экземпляров процессов

    1. На странице Host settings выберите Process instance snapshots.
    2. Включите Enable Process instance snapshots.
    3. Установите Reported processes limit на максимальное количество процессов, которые хост будет передавать.
    4. Выберите Save changes.

    Business Observability¶

    OneAgent может захватывать бизнес-события из входящих HTTP-запросов на основе правил захвата, которые указывают OneAgent захватывать бизнес-события при вызове определённых веб-сервисов или эндпоинтов.

    • Правило захвата состоит из правил-триггеров, обязательной информации о бизнес-событии (например, тип и провайдер) и необязательных полей данных события.
    • Триггер определяет критерии, при выполнении которых захватывается бизнес-событие (например, вызывается эндпоинт «/api/buy»).

    Для настройки правила захвата на уровне хоста

    1. На странице Host settings выберите Business Observability > OneAgent.
    2. Выберите Add new capture rule.
    3. Определите правило. Подробнее см. Захват бизнес-событий.
    4. Выберите Save changes.

    Подробнее см. Захват бизнес-событий.

    Обнаружение аномалий¶

    Используйте страницы обнаружения аномалий на уровне хоста для настройки чувствительности обнаружения, установки пороговых значений оповещений или отключения оповещений для выбранного хоста.

    Для настройки обнаружения аномалий для хоста

    1. На странице Host settings выберите Anomaly detection > Infrastructure.
    2. Настройте специфичные для хоста пороговые значения по необходимости.
    3. Выберите Save changes.

    Для настройки обнаружения аномалий для дисков хоста

    1. На странице Host settings выберите Anomaly detection > Disks.
    2. Настройте специфичные для хоста пороговые значения по необходимости.
    3. Выберите Save changes.

    Подробнее см. Настройка чувствительности обнаружения аномалий для инфраструктуры.

    Обновления OneAgent¶

    Используйте OneAgent updates для настройки поведения обновления OneAgent на выбранном хосте.

    • Автоматическое обновление при первой возможности
    • Автоматическое обновление во время окон обновления
    • Без автоматических обновлений

    Обновления среды, инициированные вручную, переопределяют настройки обновления отдельных хостов. Подробнее об обновлениях среды см. Обновления OneAgent.

    Для настройки автоматического обновления OneAgent на выбранном хосте

    1. На странице Host settings выберите OneAgent updates.
    2. Установите Update mode.
    3. Выберите Save changes.

    Для ручного обновления OneAgent на выбранном хосте

    1. Установите Update OneAgent to a specific version на версию OneAgent, которую вы хотите установить на выбранный хост.
    2. Выберите Update now.

    Мониторинг служб ОС¶

    Используйте OS services monitoring для настройки оповещений о службах ОС в нежелательных состояниях.

    1. На странице Host settings выберите OS services monitoring.
    2. Выберите Add policy.
    3. Установите System (Linux или Windows).
    4. Установите Rule name.
    5. Определите условия оповещения.
    6. Выберите Save changes.

    Подробнее см. Мониторинг служб ОС.

    Extension Execution Controller¶

    Используйте Extension Execution Controller для настройки Extension Execution Controller (EEC) для развёртывания OneAgent.

    1. На странице Host settings выберите Extension Execution Controller.

    2. Настройте параметры EEC:

    3. Установите Enable Extension Execution Controller.

    4. Установите Enable local HTTP Metric, Log and Event Ingest API.
    5. Установите Enable Dynatrace StatsD.
    6. Установите Performance profile. Подробнее см. Профиль производительности — потребление ресурсов.
    7. Выберите Save changes.

    Подробнее о EEC см. О расширениях.

    Мониторинг логов¶

    Используйте страницу Log Monitoring для настройки специфичных для хоста параметров мониторинга логов.

    1. На странице Host settings выберите Log Monitoring.

    2. Откройте страницу конфигурации для настроек, которые хотите настроить на этом хосте.

    3. Конфигурация OneAgent

    4. Конфигурация пользовательского источника логов — подробнее см. Пользовательский источник логов

    5. Конфигурация хранения логов — подробнее см. Правила приёма логов
    6. Маскирование конфиденциальных данных — подробнее см. Маскирование конфиденциальных данных в OneAgent
    7. Конфигурация временных меток — подробнее см. Конфигурация временных меток/разделения
    8. Выберите Save changes.

    Анализ аварийных дампов¶

    Используйте Crash dump analytics для управления автоматическим обнаружением аварийных завершений приложений.

    1. На странице Host settings выберите Crash dump analytics.
    2. Включите или отключите Crash dump analytics для определения, будет ли Dynatrace автоматически обнаруживать аварийные завершения приложений на выбранном хосте.
    3. Выберите Save changes.

    Подробнее о дампах аварийных завершений в Linux и Windows см. Анализ аварийных завершений.
