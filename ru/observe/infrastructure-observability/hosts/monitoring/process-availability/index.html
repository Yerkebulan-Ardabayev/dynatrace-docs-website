    Доступность процессов

    OneAgent версии 1.237+

    Для мониторинга доступности ключевых процессов на ваших хостах необходимо определить правила мониторинга. После создания правила, когда соответствующий процесс отсутствует на хосте, Dynatrace создаёт событие оповещения.

    Вы можете анализировать последнюю активность процессов, определённых для мониторинга доступности, в разделе Снимки экземпляров процессов на странице обзора хоста.

    Определение области применения¶

    Вы можете создавать правила для применения на уровне среды, группы хостов и отдельного хоста. Правила нижнего уровня переопределяют правила верхнего уровня. Например, правило, созданное на уровне хоста, переопределяет правило с тем же именем, созданное на уровне среды.

    Определение правила мониторинга¶

    1. Перейдите на страницу Process availability для уровня, на котором вы хотите применить правило:

    2. Среда: перейдите в Settings и выберите Processes and containers > Process availability.

    3. Группа хостов: перейдите на страницу группы хостов по адресу https://your-environment/ui/settings/HOST_GROUP-NAME и выберите Process availability.
    4. Хост: перейдите на страницу обзора хоста, выберите More (...), перейдите в Settings и выберите Process availability.
    5. На странице Process availability выберите Add monitoring rule. Правило доступности процесса может включать несколько отдельных правил обнаружения. Процесс считается идентифицированным, если все отдельные правила обнаружения совпадают.
    6. В поле Monitoring rule name введите имя, под которым правило будет отображаться в списке.

    7. В разделе Operating system (OneAgent версии 1.287+) выберите операционные системы, на которых должно применяться правило мониторинга. Вы можете выбрать несколько.

    8. Windows

    9. Linux
    10. AIX
    11. Установите Minimum number of matching processes (OneAgent версии 1.287+) — минимальное количество процессов, которые должны соответствовать этому правилу. Если на каком-либо хосте совпадает меньше процессов, срабатывает оповещение.
    12. Выберите Add detection rule, чтобы определить правило обнаружения.

    Одно правило мониторинга может содержать несколько правил обнаружения. Если вы добавите более одного правила обнаружения, процесс идентифицируется при совпадении всех правил (связь AND).

    • Rule scope — выбор Process или Host определяет последующие параметры конфигурации. Разверните ниже для подробностей.

      Process

      • Select process property — объект, по которому будет проверяться правило обнаружения:

      • Command line - single argument — каждый параметр командной строки оценивается отдельно. Оценка чувствительна к регистру.

      • Command line — OneAgent версии 1.333+ Вся командная строка оценивается целиком. Оценка чувствительна к регистру.
      • Executable path — правила не чувствительны к регистру.
      • User — OneAgent версии 1.287+ Пользователь чувствителен к регистру для Linux и AIX, не чувствителен к регистру для Windows.

      Компараторы оценивают каждый параметр командной строки отдельно для свойства Command line - single argument (в версиях до 1.333 называлось Command line). Например, процесс python my.py -ab -cd -ef будет соответствовать условию $contains(cd), $eq(-ab), но не будет соответствовать $suffix(-cd -ef), поскольку -cd и -ef являются отдельными аргументами, которые обрабатываются раздельно.

      OneAgent версии 1.307+ Исполняемый файл также рассматривается как часть командной строки, поскольку является первым аргументом.

      • Condition — в зависимости от того, что должно совпадать в правиле, вы можете определить строку, используя:

      • $contains — совпадение, если свойство содержит указанное значение. Например, $contains(keepalived) совпадает, если keepalived встречается в любом месте свойства.

      • $eq — совпадение, если свойство точно соответствует указанному значению. Например, $eq(-d) совпадает, если -d точно соответствует свойству.
      • $prefix — совпадение, если свойство начинается с указанного значения. Например, $prefix(/usr/sbin/keepalived) совпадает со свойством, начинающимся с /usr/sbin/keepalived.
      • $suffix — совпадение, если свойство заканчивается указанным значением. Например, $suffix(keepalived) совпадает со свойством, заканчивающимся на keepalived.

      Host

      OneAgent версии 1.287+

      Custom metadata — это определяемые пользователем пары ключ-значение, которые вы можете назначать хостам, мониторинг которых осуществляется Dynatrace.

      Определяя пользовательские метаданные, вы можете обогатить данные мониторинга контекстом, специфичным для потребностей вашей организации, таким как имена сред, владельцы команд, версии приложений или любые другие релевантные детали.

      • Key — указывает ключ метаданных, по которому вы хотите выполнить сопоставление.

      • Condition — позволяет определить строку:

      • $contains(production) — совпадение, если production встречается в любом месте значения метаданных хоста.

      • $eq(production) — совпадение, если production точно соответствует значению метаданных хоста.
      • $prefix(production) — совпадение, если production соответствует началу значения метаданных хоста.
      • $suffix(production) — совпадение, если production соответствует окончанию значения метаданных хоста.

      Доступные логические операции:

      • $not($eq(production)) — совпадение, если значение метаданных хоста отличается от production.
      • $and($prefix(production),$suffix(main)) — совпадение, если значение метаданных хоста начинается с production и заканчивается на main.
      • $or($prefix(production),$suffix(main)) — совпадение, если значение метаданных хоста начинается с production или заканчивается на main.

      Экранирование специальных символов: при включении специальных символов, таких как ( и ), в выражения сопоставления экранируйте их символом тильды ~. Например, чтобы сопоставить значение метаданных my(amazing)property, введите $eq(my~(amazing~)property). 7. Если вам нужно добавить ещё одно правило обнаружения к этому правилу мониторинга, повторите предыдущий шаг. 8. Выберите Add property (OneAgent версии 1.249+ Dynatrace версии 1.249+), чтобы указать пользовательское свойство ключ-значение для события.

    • Key: введите dt. в поле Key для получения подсказок.

    • Value: введите { в поле Value для получения подсказок.

    Вы можете использовать только значения, предложенные в подсказках.

    Пример пользовательского сообщения в деталях события:

    • Key = custom.message
    • Value = The {dt.entity.host} is deployed on: {dt.os.type}

    В этом примере обратите внимание, что Value содержит два ключа:

    • {dt.entity.host}
    • {dt.os.type}

    Значения хоста и типа ОС будут извлечены при срабатывании правила. Если подстановка ключа не удаётся, и ключ, и значение будут недоступны. 9. Когда вы закончите определение правила мониторинга, включая все правила обнаружения, входящие в состав правила мониторинга, выберите Save changes.

    После сохранения изменений:

    • Ваше правило мониторинга добавляется в список правил мониторинга на странице Process availability. Отображаемое имя — то, что вы ввели в поле Monitoring rule name.
    • Ваше правило мониторинга применяется на уровне, который вы выбрали на первом шаге: среда, группа хостов или хост.

    Управление правилами¶

    Правила мониторинга перечислены на странице Process availability. Каждое правило мониторинга, в свою очередь, содержит список из одного или нескольких правил обнаружения.

    • Чтобы просмотреть или отредактировать детали любого правила мониторинга или обнаружения, выберите в столбце Details для этого правила.
    • Чтобы изменить порядок правил, перетащите любое правило на другое место в списке.
    • Чтобы удалить правило, выберите в столбце Delete и затем подтвердите действие.
    • Чтобы включить или отключить правило (только для правил мониторинга), используйте переключатель в столбце Enabled.
