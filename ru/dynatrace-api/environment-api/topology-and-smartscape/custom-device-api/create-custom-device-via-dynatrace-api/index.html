    Создание пользовательского устройства через Dynatrace API¶

    Создание пользовательского устройства через Dynatrace API¶

    • Справочник
    • Обновлено 22 марта 2023 г.
    • Устарело

    Эндпоинт Custom device API Topology and Smartscape позволяет создать пользовательское устройство с указанным именем в вашей среде Dynatrace.

    На этой странице описано, как создать пользовательское устройство без отправки каких-либо данных.

    Чтобы узнать, как отправлять данные на пользовательское устройство, см. Отправка метрик пользовательского устройства через REST API.

    Запрос принимает и возвращает полезную нагрузку в формате application/json.
    POST SaaS https://{your-environment-id}.live.dynatrace.com/api/v1/entity/infrastructure/custom/{customDeviceId}
    POST Environment ActiveGate https://{your-activegate-domain}:9999/e/{your-environment-id}/api/v1/entity/infrastructure/custom/{customDeviceId}

    Аутентификация¶

    Для выполнения этого запроса необходим токен доступа с областью DataExport.

    Чтобы узнать, как получить и использовать его, см. Токены и аутентификация.

    Параметры¶

    Для данного варианта использования элемент series в теле запроса должен быть опущен.

    Параметр Тип Описание Расположение Обязательность
    customDeviceId string Идентификатор пользовательского устройства. Если вы используете идентификатор существующего устройства, соответствующие параметры будут обновлены. Не используйте здесь идентификатор сущности Dynatrace. path Обязательный
    body CustomDevicePushMessage JSON-тело запроса. Содержит параметры пользовательского устройства. body Необязательный

    Объекты тела запроса¶

    Объект CustomDevicePushMessage¶

    Конфигурация пользовательского устройства.

    Элемент Тип Описание Обязательность
    configUrl string URL веб-страницы конфигурации пользовательского устройства, например страница входа для межсетевого экрана или маршрутизатора. Необязательный
    displayName string Имя пользовательского устройства, которое будет отображаться в пользовательском интерфейсе. Необязательный
    favicon string Значок, который будет отображаться для вашего пользовательского компонента в Smartscape. Укажите полный URL файла значка. Необязательный
    group string Определённый пользователем идентификатор группы сущности. Идентификатор группы помогает поддерживать согласованную картину связей устройство-группа. Один из многих случаев, когда правильная группа важна — это обнаружение сервисов: вы можете определить, какие пользовательские устройства должны вести к одному и тому же сервису, задав одинаковый идентификатор группы. Если вы зададите идентификатор группы, он будет хеширован в идентификатор сущности Dynatrace пользовательского устройства. В этом случае пользовательское устройство может быть только частью одной группы пользовательских устройств. Если вы не зададите идентификатор группы, Dynatrace создаст его на основе идентификатора или типа пользовательского устройства. Также группа не будет хеширована в идентификатор устройства, что означает, что устройство может переключаться между группами. Необязательный
    hostNames string[] Список имён хостов, связанных с пользовательским устройством. Эти имена используются для автоматического обнаружения горизонтальных коммуникационных связей между этим компонентом и всеми другими наблюдаемыми компонентами в Smartscape. После обнаружения соединения оно автоматически отображается в Smartscape. Если вы отправите значение, существующие значения будут перезаписаны. Если вы отправите null или пустое значение, либо опустите это поле, существующие значения будут сохранены. Необязательный
    ipAddresses string[] Список IP-адресов, принадлежащих пользовательскому устройству. Эти адреса используются для автоматического обнаружения горизонтальных коммуникационных связей между этим компонентом и всеми другими наблюдаемыми компонентами в Smartscape. После обнаружения соединения оно автоматически отображается в Smartscape. Если вы отправите значение (включая пустое значение), существующие значения будут перезаписаны. Если вы отправите null или опустите это поле, существующие значения будут сохранены. Необязательный
    listenPorts integer[] Список портов, которые прослушивает пользовательское устройство. Эти порты используются для обнаружения горизонтальных коммуникационных связей между этим компонентом и всеми другими наблюдаемыми компонентами в Smartscape. После обнаружения соединения оно автоматически отображается в Smartscape. Если указаны порты, следует также добавить хотя бы один IP-адрес или имя хоста для пользовательского устройства. Если вы отправите значение, существующие значения будут перезаписаны. Если вы отправите null, пустое значение или опустите это поле, существующие значения будут сохранены. Необязательный
    properties object Список свойств в виде пар ключ-значение, которые будут отображаться под инфографикой вашего пользовательского устройства. Необязательный
    series EntityTimeseriesData[] Список значений метрик, которые передаются для пользовательского устройства. Метрика, которую вы передаёте, должна уже существовать в Dynatrace. Чтобы узнать, как создать пользовательскую метрику, см. Timeseries API v1 - PUT a custom metric. Dynatrace агрегирует все значения, которые вы передаёте для пользовательского устройства. Если вы отправите значение (включая пустое значение), оно будет добавлено к набору существующих значений. Если вы отправите null или опустите это поле, набор существующих значений не изменится. Необязательный
    tags string[] Список пользовательских тегов, которые вы хотите прикрепить к вашему пользовательскому устройству. Необязательный
    type string Определение типа технологии пользовательского устройства. Оно должно совпадать с типом технологии метрики, которую вы передаёте. Если вы отправите значение, существующее значение будет перезаписано. Если вы отправите null, пустое значение или опустите это поле, существующее значение будет сохранено. Необязательный

    Объект EntityTimeseriesData¶

    Информация о метрике и её точках данных.

    Элемент Тип Описание Обязательность
    dataPoints array[] Список точек данных. Каждая точка данных — это массив, содержащий метку времени и значение. Метка времени указывается в миллисекундах UTC в виде числа, например: 1520523365557. Гарантированный временной интервал составляет 30 минут в прошлое. Пользовательская метрика должна быть зарегистрирована до того, как вы сможете передать значение метрики. Поэтому метка времени для передачи значения должна быть после времени регистрации метрики. Обязательный
    dimensions object Измерения точек данных, которые вы передаёте. Ключ измерения метрики должен быть определён ранее в определении метрики. Необязательный
    timeseriesId string Идентификатор метрики, в которую вы хотите передать точки данных. Обязательный

    Ответ¶

    Коды ответа¶

    Код Тип Описание
    200 CustomDevicePushResult Успешно. Пользовательское устройство было создано или обновлено.
    400 ErrorEnvelope Ошибка. Входные данные некорректны.
    4XX ErrorEnvelope Ошибка на стороне клиента.
    5XX ErrorEnvelope Ошибка на стороне сервера.

    Объекты тела ответа¶

    Объект CustomDevicePushResult¶

    Результат запроса на создание пользовательского устройства. Идентификатор сущности вычисляется автоматически.

    Элемент Тип Описание
    entityId string Идентификатор сущности Dynatrace пользовательского устройства.
    groupId string Идентификатор сущности Dynatrace группы пользовательских устройств.

    Объект ErrorEnvelope¶

    Элемент Тип Описание
    error Error -

    Объект Error¶

    Элемент Тип Описание
    code integer Код состояния HTTP
    constraintViolations ConstraintViolation[] Список нарушений ограничений
    message string Сообщение об ошибке

    Объект ConstraintViolation¶

    Список нарушений ограничений

    Элемент Тип Описание
    location string -
    message string -
    parameterLocation string - Элемент может содержать следующие значения: * HEADER * PATH * PAYLOAD_BODY * QUERY
    path string -

    Пример¶

    В этом примере запрос создаёт пользовательское устройство idOfmyCustomDevice типа F5-Firewall с IP-адресом 172.16.115.211 и портом прослушивания 9999. Запрос также задаёт некоторые дополнительные параметры.

    См. Отправка метрик пользовательского устройства через Dynatrace API, чтобы узнать, как отправлять данные на вновь созданное пользовательское устройство.

    Токен API передаётся в заголовке Authorization.

    Запрос возвращает идентификаторы пользовательского устройства (см. entityId) и его группы (см. groupId) в качестве подтверждения.

    Вы можете скачать или скопировать пример тела запроса, чтобы попробовать его самостоятельно.

    Curl¶

    curl -X POST \



https://mySampleEnv.live.dynatrace.com/api/v1/entity/infrastructure/custom/idOfmyCustomDevice \



-H 'Authorization: Api-Token dt0c01.abc123.abcdefjhij1234567890'



-H 'Content-Type: application/json' \



-d '{



"displayName" : "F5 Firewall 24",



"ipAddresses" : ["172.16.115.211"],



"listenPorts" : ["9999"],



"type" : "F5-Firewall",



"favicon" : "http://assets.dynatrace.com/global/icons/f5.png",



"configUrl" : "http://192.128.0.1:8080",



"tags": [



"REST example"



],



"properties" : {



"Sample Property 1": "Sample value 1"



}



}'
