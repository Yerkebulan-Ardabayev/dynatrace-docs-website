    Отправка метрик пользовательского устройства через Dynatrace API¶

    • Deprecated

    Эндпоинт Custom device API Topology and Smartscape позволяет отправлять точку данных пользовательской метрики на пользовательское устройство в Dynatrace. Этот запрос также может обновлять метаданные пользовательского устройства.

    Метрика, которую вы отправляете, должна уже существовать в Dynatrace.

    См. Create custom device via the Dynatrace API, чтобы узнать, как создать пользовательское устройство без отправки данных на него.

    Вы можете отправлять данные на пользовательское устройство ретроспективно — эндпоинт custom device поддерживает отправку данных за период до одного часа в прошлое. Однако для обеспечения правильной работы анализа первопричин ИИ и оповещений на основе метрик мы рекомендуем отправлять данные в реальном времени.

    Для этого сценария использования элемент series тела запроса является обязательным.

    Запрос принимает и возвращает полезную нагрузку application/json.
    POST Managed https://{your-environment-id}.live.dynatrace.com/api/v1/entity/infrastructure/custom/{customDeviceId}
    POST Environment ActiveGate https://{your-activegate-domain}:9999/e/{your-environment-id}/api/v1/entity/infrastructure/custom/{customDeviceId}

    Аутентификация¶

    Для выполнения этого запроса вам необходим токен доступа с областью действия DataExport.

    Чтобы узнать, как его получить и использовать, см. Tokens and authentication.

    Параметры¶

    Параметр Тип Описание Расположение Обязательный
    customDeviceId string Идентификатор пользовательского устройства. Если вы используете идентификатор существующего устройства, соответствующие параметры будут обновлены. Не используйте здесь идентификатор сущности Dynatrace. path Обязательный
    body CustomDevicePushMessage JSON-тело запроса. Содержит параметры пользовательского устройства. body Необязательный

    Объекты тела запроса¶

    Объект CustomDevicePushMessage¶

    Конфигурация пользовательского устройства.

    Элемент Тип Описание Обязательный
    configUrl string URL страницы конфигурации пользовательского устройства, например страницы входа для файрвола или маршрутизатора. Необязательный
    displayName string Имя пользовательского устройства, которое будет отображаться в пользовательском интерфейсе. Необязательный
    favicon string Иконка, отображаемая для вашего пользовательского компонента в Smartscape. Укажите полный URL файла иконки. Необязательный
    group string Пользовательский идентификатор группы сущности. Идентификатор группы помогает поддерживать согласованную картину связей между устройствами и группами. Один из множества случаев, когда правильная группа важна — это обнаружение сервисов: вы можете определить, какие пользовательские устройства должны приводить к одному и тому же сервису, задав для них одинаковый идентификатор группы. Если вы установите идентификатор группы, он будет хеширован в идентификатор сущности Dynatrace пользовательского устройства. В этом случае пользовательское устройство может быть частью только одной группы пользовательских устройств. Если вы не установите идентификатор группы, Dynatrace создаст его на основе идентификатора или типа пользовательского устройства. Кроме того, группа не будет хеширована в идентификатор устройства, что означает, что устройство может переключаться между группами. Необязательный
    hostNames string[] Список имён хостов, связанных с пользовательским устройством. Эти имена используются для автоматического обнаружения горизонтальных связей между этим компонентом и всеми другими наблюдаемыми компонентами в Smartscape. Как только связь обнаружена, она автоматически отображается в Smartscape. Если вы отправляете значение, существующие значения будут перезаписаны. Если вы отправляете null или пустое значение, или опускаете это поле, существующие значения будут сохранены. Необязательный
    ipAddresses string[] Список IP-адресов, принадлежащих пользовательскому устройству. Эти адреса используются для автоматического обнаружения горизонтальных связей между этим компонентом и всеми другими наблюдаемыми компонентами в Smartscape. Как только связь обнаружена, она автоматически отображается в Smartscape. Если вы отправляете значение (включая пустое), существующие значения будут перезаписаны. Если вы отправляете null или опускаете это поле, существующие значения будут сохранены. Необязательный
    listenPorts integer[] Список портов, которые прослушивает пользовательское устройство. Эти порты используются для обнаружения горизонтальных связей между этим компонентом и всеми другими наблюдаемыми компонентами в Smartscape. Как только связь обнаружена, она автоматически отображается в Smartscape. Если указаны порты, следует также добавить хотя бы один IP-адрес или имя хоста для пользовательского устройства. Если вы отправляете значение, существующие значения будут перезаписаны. Если вы отправляете null, пустое значение или опускаете это поле, существующие значения будут сохранены. Необязательный
    properties object Список свойств в виде пар ключ-значение, которые будут отображаться под инфографикой вашего пользовательского устройства. Необязательный
    series EntityTimeseriesData[] Список значений метрик, отправляемых для пользовательского устройства. Метрика, которую вы отправляете, должна уже существовать в Dynatrace. Чтобы узнать, как создать пользовательскую метрику, см. Timeseries API v1 - PUT a custom metric. Dynatrace агрегирует все значения, которые вы отправляете для пользовательского устройства. Если вы отправляете значение (включая пустое), оно будет добавлено к набору существующих значений. Если вы отправляете null или опускаете это поле, набор существующих значений не изменится. Необязательный
    tags string[] Список пользовательских тегов, которые вы хотите прикрепить к вашему пользовательскому устройству. Необязательный
    type string Определение типа технологии пользовательского устройства. Должно совпадать с типом технологии метрики, которую вы отправляете. Если вы отправляете значение, существующее значение будет перезаписано. Если вы отправляете null, пустое значение или опускаете это поле, существующее значение будет сохранено. Необязательный

    Объект EntityTimeseriesData¶

    Информация о метрике и её точках данных.

    Элемент Тип Описание Обязательный
    dataPoints array[] Список точек данных. Каждая точка данных — это массив, содержащий временную метку и значение. Временная метка указывается в миллисекундах UTC в виде числа, например: 1520523365557. Гарантированный временной диапазон составляет 30 минут в прошлое. Пользовательская метрика должна быть зарегистрирована до того, как вы сможете отправлять значения. Поэтому временная метка для отправки значения должна быть после времени регистрации метрики. Обязательный
    dimensions object Измерения точек данных, которые вы отправляете. Ключ измерения метрики должен быть заранее определён в определении метрики. Необязательный
    timeseriesId string Идентификатор метрики, куда вы хотите отправить точки данных. Обязательный

    Ответ¶

    Коды ответа¶

    Код Тип Описание
    200 CustomDevicePushResult Успех. Пользовательское устройство было создано или обновлено.
    400 ErrorEnvelope Ошибка. Входные данные недопустимы.
    4XX ErrorEnvelope Ошибка на стороне клиента.
    5XX ErrorEnvelope Ошибка на стороне сервера.

    Объекты тела ответа¶

    Объект CustomDevicePushResult¶

    Результат запроса на отправку данных пользовательского устройства. Идентификатор сущности вычисляется автоматически.

    Элемент Тип Описание
    entityId string Идентификатор сущности Dynatrace пользовательского устройства.
    groupId string Идентификатор сущности Dynatrace группы пользовательских устройств.

    Объект ErrorEnvelope¶

    Элемент Тип Описание
    error Error -

    Объект Error¶

    Элемент Тип Описание
    code integer HTTP-код статуса
    constraintViolations ConstraintViolation[] Список нарушений ограничений
    message string Сообщение об ошибке

    Объект ConstraintViolation¶

    Список нарушений ограничений

    Элемент Тип Описание
    location string -
    message string -
    parameterLocation string - Элемент может содержать следующие значения: * HEADER * PATH * PAYLOAD_BODY * QUERY
    path string -

    JSON-модели тела ответа¶

    {
"entityId": "string",
"groupId": "string"
}

    {
"error": {
"code": 1,
"constraintViolations": [
{
"location": "string",
"message": "string",
"parameterLocation": "HEADER",
"path": "string"
}
],
"message": "string"
}
}

    Пример¶

    В этом примере запрос отправляет две точки данных метрики custom:firewall.connections.dropped для устройства idOfmyCustomDevice. Точки данных (со значением 460 для временной метки 1539860400000 и значением 456 для временной метки 1539860460000) принадлежат значению ethernetcard1 измерения nic.

    Запрос также отправляет ещё две точки данных той же метрики, но для ethernetcard2 в том же измерении, и обновляет метаданные устройства, добавляя свойство и тег.

    Токен API передаётся в заголовке Authorization.

    Запрос возвращает идентификаторы пользовательского устройства (см. entityId) и его группы (см. groupId) в качестве подтверждения.

    Вы можете скачать или скопировать пример тела запроса, чтобы опробовать его самостоятельно.

    Curl¶

    curl -X POST \
https://mySampleEnv.live.dynatrace.com/api/v1/entity/infrastructure/custom/idOfmyCustomDevice \
-H 'Authorization: Api-Token dt0c01.abc123.abcdefjhij1234567890' \
-H 'Content-Type: application/json' \
-d '{
"tags": [
"tag2"
],
"type": "F5-Firewall",
"properties" : {
"Sample Property 2": "Sample value 2"
},
"series" : [
{
"timeseriesId" : "custom:firewall.connections.dropped",
"dimensions" : {
"nic" : "ethernetcard1"
},
"dataPoints" : [
[ 1539860400000, 460 ],
[ 1539860460000, 456 ]
]
},
{
"timeseriesId" : "custom:firewall.connections.dropped",
"dimensions" : {
"nic" : "ethernetcard2"
},
"dataPoints" : [
[ 1539860430000, 439 ],
[ 1539860490000, 460 ]
]
}
]
}'
