# Управление атаками

# Управление атаками

* Explanation
* Updated on Apr 22, 2024

После [включения и настройки Runtime Application Protection](/managed/secure/application-security/application-protection "Настройка и конфигурирование Dynatrace Runtime Application Protection для мониторинга атак и уязвимостей на уровне кода, генерируемых атаками.") Dynatrace начинает обнаруживать атаки на все отслеживаемые приложения в вашей среде. Атака — это любой запрос (обращение) от определённого клиентского IP-адреса к коду вашего приложения с вредоносным намерением (например, для доступа к защищённой информации или её удаления с помощью SQL injection), направленный на уязвимость на уровне кода.

## Список атак

Чтобы просмотреть список атак в вашей среде, перейдите в **Attacks**. Отображается следующая информация.

### Инфографика ключевых функций

![infographic-attacks](https://dt-cdn.net/images/2023-03-14-09-09-23-1579-7f13f156a2.png)

infographic-attacks

* Общий обзор количества атак в соответствии с [выбранным периодом](#global) и [management zone](#mz). Существует три типа атак; выберите любой из них для перехода к атакам, отфильтрованным по соответствующему типу:

  + **Exploited:** Атака была осуществлена. Это происходит при активированном режиме мониторинга.
  + **Blocked:** Атака была заблокирована. Это происходит при активированном режиме блокировки.
  + **Allowlisted:** Вы разрешили прохождение атаки. Это происходит, когда атака добавляется в allowlist.
* Текущий глобальный режим управления атаками (`Monitoring`, `Blocking`, `Off`). При выборе выполняется переход на страницу **Application Protection: General settings**, где можно изменить конфигурацию. Подробнее см. в разделе [Настройка Runtime Application Protection](/managed/secure/application-security/application-protection#config "Настройка и конфигурирование Dynatrace Runtime Application Protection для мониторинга атак и уязвимостей на уровне кода, генерируемых атаками.").
* **Attack source locations:** Карта стран, из которых производились атаки, за выбранный период.

  + Карта окрашена в соответствии с количеством атак из определённых стран; подробнее см. в легенде карты. Выберите страну для просмотра дополнительной информации об атаках из этой страны:

    - **IPs:** Количество уникальных [IP-адресов](#client-ip-address), с которых производились атаки.
    - **Locations:** Количество уникальных мест в стране, из которых производились атаки.
    - **Attacks:** Общее количество атак из данного местоположения.
    - **Amount:** Индикатор количества атак: от левого края (ноль атак) до правого (несколько атак). Индикатор, расположенный крайне справа, означает, что из выбранной страны зафиксировано больше всего атак относительно других стран.
    - **Last attacks** — список последних зафиксированных атак с указанием типа инъекции, исходного IP злоумышленника, временной метки и города.

  Чтобы отфильтровать список атак по выбранной стране, нажмите **Filter for attacks from this country**.

  + Серое местоположение — это то, из которого не было атак за выбранный период, и поэтому подробная информация недоступна.
* **Attacks over time:** График количества атак (exploited, blocked, allowlisted) за выбранный период. По умолчанию период — `Last two hours`. Другой фильтр можно задать в [глобальном периоде](#global).

### Обнаруженные атаки

![attacks detected](https://dt-cdn.net/images/2024-06-18-11-40-25-1670-17779ad624.png)

attacks detected

Список обнаруженных атак в вашей среде.
Для оптимизации производительности одновременно отображается не более 500 атак. Можно сузить результаты с помощью [фильтров](#filter).
Для сортировки списка по любому элементу выберите соответствующий заголовок столбца. Для добавления или удаления заголовков столбцов нажмите **Format table**.

#### Атака

* Dynatrace-идентификатор атаки (пример: `A-2JPGJXQF`)
* Краткая версия расположения точки входа в коде атаки (пример: `org.springframework.web.filter.DelegatingFilterProxy.invokeDelegate():135`).
* Название атакованной группы процессов (пример: `BloatedJavaSoftwareGroup-IG-1`)
* Символ публичной доступности из интернета, если таковая имеется. Если символ серый и перечёркнутый — публичная доступность из интернета не обнаружена. Если символ отсутствует — данных нет.
* Символ доступных данных, если затронуты какие-либо доступные активы данных. Если символ серый и перечёркнутый — доступных активов данных в диапазоне нет. Если символ отсутствует — данных нет.

#### Точка входа

HTTP-путь запроса (пример: `/image`).

#### Исходный IP

IPv4 или IPv6 адрес злоумышленника.

#### Местоположение

Страна, из которой была произведена атака.

#### Тип

Тип эксплойта:

* **SQL injection:** Направлен на базу данных.
* **Command injection:** Направлен на хост.
* **JNDI injection:** Направлен на хост.
* **SSRF:** Направлен на хост/сервис.

#### Статус

Управление атакой в соответствии с [настройками конфигурации](/managed/secure/application-security/application-protection#config "Настройка и конфигурирование Dynatrace Runtime Application Protection для мониторинга атак и уязвимостей на уровне кода, генерируемых атаками."):

* **Exploited:** Атака была осуществлена. Это происходит при активированном режиме мониторинга.
* **Blocked:** Атака была заблокирована. Это происходит при активированном режиме блокировки.
* **Allowlisted:** Вы разрешили прохождение атаки. Это происходит, когда атака добавляется в allowlist.

#### Временная метка

Время совершения атаки.

#### Сведения

Разверните ![Expand row](https://dt-cdn.net/images/expand-row-icon-9c4950fc2e.svg "Expand row") строки атак для просмотра подробностей или выполнения следующих действий:

* Нажмите **Add to allowlist**, если необходимо добавить и настроить новое правило исключения для атаки. Подробнее см. в разделе [Определение правил исключений](/managed/secure/application-security/application-protection/application-protection-rules#exception-rules "Создание, изменение и удаление правил для конкретных атак.").
* Нажмите **Block attack**, если необходимо добавить и определить новое правило блокировки атаки. Подробнее см. в разделе [Определение правил управления конкретными атаками](/managed/secure/application-security/application-protection/application-protection-rules#handling-rules "Создание, изменение и удаление правил для конкретных атак.").
* Нажмите **View attack details** для перехода на [страницу сведений об атаке](#details).

## Сведения об атаке

Чтобы просмотреть подробную информацию об атаке, перейдите в **Attacks** и выберите нужную атаку.
Отображается следующая информация.

### Заголовок атаки

Пример заголовка:

![attack title](https://dt-cdn.net/images/2024-06-18-11-43-33-498-d7d7345219.png)

attack title

* Тип эксплойта (`SQL injection`, `Command injection`, `JNDI injection` или `SSRF`)
* Dynatrace-идентификатор атаки (пример: `A-2JPGJXQF`)
* Название атакованной группы процессов (пример: `MembershipService.dll unguard-membership-service-*`)

### Инфографика ключевых функций

![infographic-attack-details](https://dt-cdn.net/images/2022-11-11-12-27-00-1355-c6f3eb1c6a.png)

infographic-attack-details

* Тип атаки (`Exploited`, `blocked`, `allowlisted`)
* **Public internet exposure:** Наличие публичной доступности из интернета. Возможные состояния:

  + **Public network:** Публичная доступность из интернета присутствует.
  + **Not detected:** Публичная доступность из интернета не обнаружена.
  + **Not available:** Данные недоступны, так как связанные хосты работают в режиме Infrastructure Monitoring. Подробнее см. в разделе [Режимы мониторинга](/managed/secure/application-security#monitoring-modes "Доступ к функциям Dynatrace Application Security.").
* **Reachable data assets:** Наличие затронутых доступных активов данных. Возможные состояния:

  + **Within range:** Доступные активы данных затронуты.
  + **None within range:** Доступных активов данных в диапазоне нет.
  + **Not available:** Данные недоступны, так как связанные хосты работают в режиме Infrastructure Monitoring. Подробнее см. в разделе [Режимы мониторинга](/managed/secure/application-security#monitoring-modes "Доступ к функциям Dynatrace Application Security.").
* **Process group instance:** Название затронутого процесса.
* **Vulnerability:** Тип инъекции и расположение уязвимого кода.
* **Timestamp:** Время совершения атаки.
* **Source IP:** IP-адрес, с которого была произведена атака.

Доступны следующие действия:

* Нажмите **Add to allowlist** для [добавления атаки в allowlist](/managed/secure/application-security/application-protection/application-protection-rules#exception-rules "Создание, изменение и удаление правил для конкретных атак.").
* Нажмите **Block attack** для [блокировки атаки](/managed/secure/application-security/application-protection/application-protection-rules#handling-rules "Создание, изменение и удаление правил для конкретных атак.").

### Путь атаки

![attack-path](https://dt-cdn.net/images/2022-11-11-12-30-25-1570-938080fffb.png)

attack-path

Путь атаки — это визуальное представление атаки, содержащее следующие шаги:

1. **Source IP:** IP-адрес, с которого была произведена атака.
2. **Entry point:** Путь запроса, содержащего вредоносный payload, использованный в этой атаке.
3. **Vulnerability:** Использованная уязвимость на уровне кода.
4. В зависимости от типа инъекции:

* **Target:** Отображает атакованный хост (для command- и JNDI-инъекций) или целевой хост/сервис (для SSRF).
* **Database:** Отображает атакованную базу данных (для SQL-инъекций).

### Точка входа

![Entry point section](https://dt-cdn.net/images/2024-02-14-07-54-51-782-2d98081aea.png)

Entry point section

* **HTTP path:** HTTP-путь, использованный для атаки.
* **Entry point function:** Функция, в которой вредоносный payload впервые получил доступ к атакованному процессу.
* **Payload:** HTTP-параметр, содержащий вредоносный payload и его значение.

### Уязвимость

![vulnerability-attacks](https://dt-cdn.net/images/2022-11-11-12-34-22-786-0e08b656b0.png)

vulnerability-attacks

* **Name:** Использованная уязвимость на уровне кода.
* **Code location:** Расположение в коде, где вызывается функция точки входа.

  Если расположение недоступно, **Code location** не отображается.
* **Vulnerable function:** Функция, использовавшая часть payload злоумышленника, что привело к эксплуатации уязвимости.
* В зависимости от типа инъекции:

  + **SQL statement** (в случае SQL injection)
  + **Command** (в случае command injection)
  + **JNDI lookup name** (в случае JNDI injection)
  + **Request URL** (в случае SSRF)

Контролируемые пользователем данные выделены.

Нажмите **View vulnerability** для перехода на [страницу подробностей соответствующей уязвимости на уровне кода](/managed/secure/application-security/vulnerability-analytics/code-level-vulnerabilities/manage-code-level-vulnerabilities#details "Мониторинг уязвимостей на уровне кода в вашей среде.").

Этот вариант доступен только при включённом [обнаружении уязвимостей на уровне кода](/managed/secure/application-security/vulnerability-analytics#clv-detection "Мониторинг, визуализация, анализ и устранение уязвимостей сторонних компонентов и уязвимостей на уровне кода, отслеживание прогресса устранения и создание правил мониторинга.").

## Определение клиентского IP-адреса

Когда веб-запросы направляются непосредственно на отслеживаемый сервер, Dynatrace определяет IP-адреса устройств конечных пользователей через сокетные соединения. Однако при использовании неотслеживаемых компонентов — балансировщиков нагрузки, CDN или прокси-серверов — удалённый IP-адрес отличается от исходного IP-адреса. В таких случаях Dynatrace также учитывает определённые HTTP-заголовки. Эти заголовки наиболее часто используются для идентификации исходного IP-адреса при подключении клиента к веб-серверу через HTTP-прокси, CDN или балансировщик нагрузки и не поддаются настройке.

## Фильтрация атак

Существует несколько способов фильтрации атак, как показано ниже.

### Фильтрация по глобальному периоду

Можно использовать глобальный выбор периода для фильтрации атак, произошедших в определённый период.

### Фильтрация по management zone

При фильтрации по конкретной management zone будут отображаться только атаки из этой management zone. Это ограничение также влияет на карту **Attack source locations**, график **Attacks over time** и сам список атак.

После возникновения атаки она назначается существующим management zones. Атака может быть частью только тех management zones, которые существовали в момент её совершения. Создание management zone для уже произошедшей атаки не повлияет на неё.

Сведения о настройке и применении management zones, а также о правилах, определяющих и ограничивающих доступные сущности в management zone, см. в разделе [Management zones](/managed/manage/identity-access-management/permission-management/management-zones "Узнайте о концепциях management zones, как их определять и использовать максимально эффективно.").

### Фильтрация по сведениям об атаке

В строке фильтра доступны следующие фильтры.

* **Process group name:** Отображает атаки по названию группы процессов.
* **Entry point:** Отображает атаки по HTTP-пути запроса.
* **Source IP:** Отображает атаки по точному IPv4 или IPv6 адресу злоумышленника. Подробнее см. в разделе [Клиентский IP-адрес](#client-ip-address).
* **Attack source location:** Отображает атаки по местоположению (стране), из которого они были произведены.
* **Technology:** Отображает атаки по поддерживаемым технологиям (`Java`, `.NET`).
* **Type:** Отображает атаки по типу эксплойта (SQL injection, command injection, JNDI injection или SSRF).
* **Status:** Отображает атаки по статусу атаки (`blocked`, `allowlisted` или `exploited`). Подробнее см. в разделе [настройки конфигурации](/managed/secure/application-security/application-protection#config "Настройка и конфигурирование Dynatrace Runtime Application Protection для мониторинга атак и уязвимостей на уровне кода, генерируемых атаками.").

Также можно фильтровать по статусу атаки, выбрав любой тип атаки на общей обзорной панели в верхней части страницы **Attacks**.

### Фильтрация по местоположению источника атаки

На карте **Attack source locations** выберите любое местоположение, из которого были произведены атаки, затем нажмите **Filter for attacks from this country**.
