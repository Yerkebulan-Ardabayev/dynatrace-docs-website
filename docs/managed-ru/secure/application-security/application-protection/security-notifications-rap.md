# Уведомления безопасности об атаках

* Overview
* Updated on Aug 18, 2022

Dynatrace автоматически обнаруживает проблемы безопасности в вашей среде и предоставляет контекстуальные автоматические оценки рисков и сведения через веб-интерфейс Dynatrace и [API](/managed/dynatrace-api/environment-api/application-security/vulnerabilities "Узнайте о возможностях API уязвимостей.") в вашей инфраструктуре.

Для передачи проблем безопасности вашим командам в целях оповещения и устранения необходимо интегрировать уведомления безопасности с Dynatrace. Интеграция атак Dynatrace предоставляет быстрое встроенное решение для оповещения и уведомлений о продуктовых атаках, обнаруженных в вашей среде.

Пользователи с [правами только на просмотр](/managed/secure/application-security#restrict-permissions "Доступ к функциям Dynatrace Application Security.") также могут настраивать оповещения об атаках.

Инструкции по интеграции уведомлений безопасности с Dynatrace приведены в любом из следующих разделов.

[### Интеграция через webhook](/managed/secure/application-security/application-protection/security-notifications-rap/webhook-integration "Интеграция уведомлений безопасности об уязвимостях и/или атаках с помощью webhook.")[### Интеграция с Jira](/managed/secure/application-security/application-protection/security-notifications-rap/jira-integration "Интеграция уведомлений безопасности об уязвимостях и/или атаках с помощью Jira.")[### Интеграция по email](/managed/secure/application-security/application-protection/security-notifications-rap/email-integration "Интеграция уведомлений безопасности об атаках по email.")

Для более широкого контроля и дополнительных параметров конфигурации ознакомьтесь с нашим [open-source проектом Rautee](https://github.com/dynatrace-oss/rautee). С помощью Rautee можно создавать собственные автоматизированные рабочие процессы по вашим потребностям. Подробности, инструкции и примеры см. в документации на GitHub.
