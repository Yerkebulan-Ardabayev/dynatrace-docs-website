# Request naming API

* Reference
* Published Mar 05, 2019

API **Request naming** позволяет управлять конфигурацией правил именования запросов.

[### Список всех правил

Обзор всех правил именования запросов, определённых в вашем окружении.](/managed/dynatrace-api/configuration-api/service-api/request-naming-api/get-all "Просмотр всех правил именования запросов вашего окружения через Dynatrace API.")[### Просмотр правила

Получить параметры правила именования запросов по его ID.](/managed/dynatrace-api/configuration-api/service-api/request-naming-api/get-a-rule "Просмотр правила именования запросов через Dynatrace API.")

[### Создание правила

Создать новое правило именования запросов с нужными вам параметрами.](/managed/dynatrace-api/configuration-api/service-api/request-naming-api/post-new-rule "Создание правила именования запросов через Dynatrace API.")[### Редактирование правила

Обновить правило именования запросов. Также можно создать новое правило с указанным ID.](/managed/dynatrace-api/configuration-api/service-api/request-naming-api/put-a-rule "Редактирование правила именования запросов через Dynatrace API.")[### Удаление правила

Удалить правило именования запросов, которое вам больше не нужно.](/managed/dynatrace-api/configuration-api/service-api/request-naming-api/delete-a-rule "Удаление правила именования запросов через Dynatrace API.")
