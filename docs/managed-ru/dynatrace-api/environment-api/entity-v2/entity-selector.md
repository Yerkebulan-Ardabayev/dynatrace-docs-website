# Environment API v2 - Селектор сущностей

# Environment API v2 - Селектор сущностей

* Справочник
* Обновлено 27 января 2026 г.

Селектор сущностей (entity selector), это мощный инструмент для указания того, какие сущности должны попасть в область вызовов Environment API v2. Он используется в нескольких API, поэтому достаточно один раз изучить синтаксис, чтобы затем переиспользовать его для разных сценариев.

Необходимо указать один из следующих критериев:

* [Тип сущности](#type)
* [ID сущности](#id)

Дополнительно можно указать любую комбинацию следующих критериев:

* [Имя сущности](#name)
* [Атрибут сущности](#attribute)
* [Тег](#tag)
* [ID зоны управления](#mzid)
* [Имя зоны управления](#mzname)
* [Состояние работоспособности](#health)
* [Включение удалённых сущностей](#deletedEntities)

Если указано несколько критериев, в ответ попадают только результаты, соответствующие **всем** критериям сразу.

* Например: `type(HOST),entityName.equals(Server)`

Если текстовое значение в критерии содержит специальные символы (например, скобки или запятую), такой текст нужно заключить в кавычки.

* Например: `type(HOST),entityName.equals("Server(prod),1")`

## Ограничения

Общая длина строки **entitySelector** ограничена 2 000 символами.

В одном запросе можно выбрать только один тип сущности.

## Тип сущности

Тип сущности, которую нужно получить.

Список доступных типов сущностей можно получить вызовом [GET all entity types](/managed/dynatrace-api/environment-api/entity-v2/get-all-entity-types "Получить все типы отслеживаемых сущностей в вашем окружении через Dynatrace API.").

|  |  |
| --- | --- |
| Синтаксис | `type("value")` |
| Несколько значений | Неприменимо |
| Оператор значения | `EQUALS` |
| Чувствительность к регистру | Неприменимо |

## ID сущности Dynatrace

Идентификатор сущности Dynatrace, которую нужно получить.

Чтобы указать несколько ID, перечислите их через запятую (`,`). Все запрашиваемые сущности должны быть одного типа.

Список доступных сущностей можно получить вызовом [GET entities list](/managed/dynatrace-api/environment-api/entity-v2/get-entities-list "Получить список отслеживаемых сущностей через Dynatrace API.").

|  |  |
| --- | --- |
| Синтаксис | `entityId("id-1","id-2")` |
| Несколько значений | Применимо |
| Оператор значения | `EQUALS` |
| Чувствительность к регистру | Неприменимо |

## Имя сущности

Имя запрашиваемой сущности.

Список доступных сущностей можно получить вызовом [GET entities list](/managed/dynatrace-api/environment-api/entity-v2/get-entities-list "Получить список отслеживаемых сущностей через Dynatrace API.").

|  |  |
| --- | --- |
| Синтаксис | `entityName("name")` |
| Несколько значений | Неприменимо |
| Оператор значения | `CONTAINS` |
| Чувствительность к регистру | Неприменимо |

### Модификатор «начинается с»

Меняет оператор значения критерия «имя сущности» на `BEGINS WITH`.

|  |  |
| --- | --- |
| Синтаксис | `entityName.startsWith("name")` |
| Несколько значений | Неприменимо |
| Оператор значения | `BEGINS WITH` |
| Чувствительность к регистру | Неприменимо |

### Модификатор «равно»

Меняет оператор значения критерия «имя сущности» на `EQUALS`.

|  |  |
| --- | --- |
| Синтаксис | `entityName.equals("name")` |
| Несколько значений | Неприменимо |
| Оператор значения | `EQUALS` |
| Чувствительность к регистру | Неприменимо |

### Модификатор «одно из значений»

Позволяет указать несколько значений для критерия «имя сущности».

|  |  |
| --- | --- |
| Синтаксис | `entityName.in("name1","name2")` |
| Несколько значений | Применимо |
| Оператор значения | `EQUALS` |
| Чувствительность к регистру | Неприменимо |

### Модификатор чувствительности к регистру

По умолчанию имена сущностей сравниваются без учёта регистра. Сделать критерий строже можно через модификатор `caseSensitive`. Он принимает любой критерий по имени сущности (в том числе уже модифицированный через `.startsWith` или `.equals`) и сравнивает значения с учётом регистра.

|  |  |
| --- | --- |
| Синтаксис | `caseSensitive(<критерий по имени сущности>)` |
| Несколько аргументов | Неприменимо |

## Атрибут сущности

Пара «имя атрибута, значение атрибута», которой должна обладать запрашиваемая сущность.

Список доступных атрибутов можно получить вызовом [GET entity type](/managed/dynatrace-api/environment-api/entity-v2/get-entity-type "Получить детали типа отслеживаемой сущности через Dynatrace API."), посмотрев поле **properties**. Можно использовать атрибуты, значения которых представимы строкой.

|  |  |
| --- | --- |
| Синтаксис | `<имя атрибута>("значение атрибута")` |
| Несколько значений | Применимо |
| Оператор значения | `EQUALS` |
| Регистрозависимое имя атрибута | Неприменимо |
| Регистрозависимое значение атрибута | Применимо |

### Модификатор «существует»

Меняет оператор критерия «атрибут сущности» на `EXISTS`. В этом случае условие выбирает сущности, у которых есть данный атрибут, независимо от его значения.

|  |  |
| --- | --- |
| Синтаксис | `<имя атрибута>.exists()` |
| Оператор | `EXISTS` |

## Тег

Тег запрашиваемых сущностей. Теги в форматах `[context]key:value`, `key:value` и `value` распознаются и разбираются автоматически. Если тег-значение содержит двоеточие (`:`), его необходимо экранировать обратной косой (`\`). Иначе тег будет разобран как `key:value`.

Чтобы указать несколько тегов, перечислите их через запятую (`,`). В ответ попадёт сущность, у которой есть **любой** из указанных тегов.

Список доступных тегов можно получить вызовами [GET custom tags](/managed/dynatrace-api/environment-api/custom-tags/get-tags "Получить пользовательские теги отслеживаемых сущностей через Dynatrace API.") и [GET auto-tags](/managed/dynatrace-api/configuration-api/automatically-applied-tags-api/get-all "Получить все автоматически применяемые теги окружения через Dynatrace API.").

|  |  |
| --- | --- |
| Синтаксис | `tag("[context]key1:value-1","key2:value-2","value-3")` |
| Несколько значений | Применимо |
| Оператор значения | `EQUALS` |
| Чувствительность к регистру | Применимо |

| Несколько значений | Применимо |
| Оператор значения | `EQUALS` |

Чтобы указать несколько имён, перечислите их через запятую (`,`).

Список доступных зон управления можно получить вызовом [GET all management zones](/managed/dynatrace-api/configuration-api/management-zones-api/get-all "Получить все зоны управления окружения через Dynatrace API.").

|  |  |

## Состояние работоспособности

Состояние работоспособности запрашиваемых сущностей. Возможные значения: `HEALTHY` и `UNHEALTHY`.

|  |  |
| --- | --- |
| Синтаксис | `healthState("HEALTHY")` |
| Несколько значений | Неприменимо |
| Оператор значения | `EQUALS` |
| Чувствительность к регистру | Применимо |

## Первое появление

Метка времени (UTC, миллисекунды), когда сущность была впервые обнаружена.

|  |  |
| --- | --- |
| Синтаксис | `firstSeenTms.<operator>(timestamp)` |
| Несколько значений | Неприменимо |
| Оператор значения | * `lte`: раньше указанного времени или в это же время * `lt`: раньше указанного времени * `gte`: позже указанного времени или в это же время * `gt`: позже указанного времени |
| Чувствительность к регистру | н/п |

## Включение удалённых сущностей

Включить или исключить сущности, помеченные как удалённые. Поддерживаются только следующие типы сущностей:

* `HTTP_CHECK`
* `SYNTHETIC_TEST`
* `EXTERNAL_SYNTHETIC_TEST`
* `MULTIPROTOCOL_MONITOR`
* `APPLICATION`
* `MOBILE_APPLICATION`
* `CUSTOM_APPLICATION`
* `DCRUM_APPLICATION`
* `CUSTOM_DEVICE`

|  |  |
| --- | --- |
| Синтаксис | `deletedEntities.<operator>()` |
| Оператор | * `include`: включить удалённые сущности * `exclude`: исключить удалённые сущности |
| Значения | Неприменимо |
| Пример | `type(HTTP_CHECK), deletedEntities.include()`: выбрать все сущности типа `HTTP_CHECK`, в том числе помеченные как удалённые. |
| Примечание | * По умолчанию удалённые сущности исключаются из всех результатов. * Этот критерий не поддерживается в [правилах зон управления](/managed/manage/identity-access-management/permission-management/management-zones/management-zone-rules#text "Задайте правила для ограничения сущностей, доступных в пределах зоны управления.") и в [автоматическом тегировании](/managed/manage/tags-and-metadata/setup/how-to-define-tags#automatic "Узнайте, как задавать и применять теги вручную и автоматически."). |

## Связи

Связи (relationships), которыми должна обладать запрашиваемая сущность.

Список доступных связей можно получить вызовом [GET entity type](/managed/dynatrace-api/environment-api/entity-v2/get-entity-type "Получить детали типа отслеживаемой сущности через Dynatrace API."), проверив поля **fromRelationships** и **toRelationships**.

|  |  |
| --- | --- |
| Синтаксис | * `fromRelationships.<relationship>(<entitySelector>)` * `toRelationships.<relationship>(<entitySelector>)` |
| Несколько аргументов | Неприменимо |
| Примечание | Принимает в качестве аргумента селектор сущностей. |

## Инверсия критерия

Чтобы инвертировать любой критерий (кроме **type**), используется модификатор `not`. Он принимает критерий аргументом и инвертирует условие. Например, критерий `not(tag("Infrastructure:Linux"))` выбирает сущности, у которых *нет* тега **Infrastructure:Linux**.

Инвертированные критерии можно использовать как часть составных селекторов, точно так же, как обычные критерии.

|  |  |
| --- | --- |
| Синтаксис | `not (<критерий>)` |
| Несколько аргументов | Неприменимо |
| Примечание | Не поддерживает критерий [**type**](#type). |
