# API отслеживаемых сущностей - POST пользовательское устройство

# API отслеживаемых сущностей - POST пользовательское устройство

* Справочник
* Опубликовано 28 мая 2020 г.

Создаёт или обновляет пользовательское устройство (custom device).

Запрос принимает и возвращает payload в формате `application/json`.

|  |  |  |
| --- | --- | --- |
| POST | ManagedDynatrace for Government | `https://{your-domain}/e/{your-environment-id}/api/v2/entities/custom` |
| POST | Environment и Cluster ActiveGate (порт по умолчанию 9999) | `https://{your-activegate-domain}:9999/e/{your-environment-id}/api/v2/entities/custom` |

## Аутентификация

Для выполнения запроса нужен access-токен со scope `entities.write`.

О том, как получить и использовать токен, смотрите [Tokens and authentication](/managed/discover-dynatrace/references/dynatrace-api/basics/dynatrace-api-authentication).

## Параметры

| Параметр | Тип | Описание | В | Обязательный |
| --- | --- | --- | --- | --- |
| uiBased | boolean | Если `true`, устройство будет считаться созданным через UI. Оно будет автоматически обновляться и не устаревать. | query | Необязательный |
| body | [CustomDeviceCreation](#openapi-definition-CustomDeviceCreation) | JSON-тело запроса. Содержит параметры пользовательского устройства. | body | Обязательный |

### Объекты тела запроса

#### Объект `CustomDeviceCreation`

Конфигурация пользовательского устройства.

| Поле | Тип | Описание | Обязательность |
| --- | --- | --- | --- |
| configUrl | string | URL веб-страницы конфигурации устройства, например страницы входа для файрвола или маршрутизатора. | Необязательный |
| customDeviceId | string | Внутренний ID пользовательского устройства. Если указан ID уже существующего устройства, его параметры будут обновлены. | Обязательный |
| displayName | string | Имя пользовательского устройства, отображаемое в интерфейсе. | Обязательный |
| dnsNames | string[] | Список DNS-имён, связанных с пользовательским устройством. Эти имена используются для автоматического обнаружения горизонтальных связей между этим компонентом и другими наблюдаемыми компонентами в Smartscape. Как только связь обнаружена, она автоматически отображается на Smartscape. Непубличные DNS-адреса тоже могут быть сопоставлены внутри среды, если доменное имя состоит хотя бы из двух частей, например `hostname.xyz`. Если передать значение, существующие значения будут перезаписаны. Если передать `null`, пустое значение или вовсе опустить это поле, существующие значения сохранятся. | Необязательный |
| faviconUrl | string | Иконка, отображаемая для вашего пользовательского компонента в Smartscape. Укажите полный URL файла иконки. | Необязательный |
| group | string | Пользовательский ID группы сущности. ID группы помогает поддерживать согласованную картину связей устройство-группа. Один из многих случаев, где правильная группа важна, это детекция сервисов: задавая один и тот же ID группы для разных пользовательских устройств, можно указать, что они должны давать один и тот же сервис. Если ID группы задан, он будет захеширован в Dynatrace entity ID этого устройства. В этом случае устройство может принадлежать только одной группе. Если ID группы не указан, Dynatrace создаст его на основе ID или типа устройства. При этом группа не будет включена в хеш entity ID, и устройство сможет менять группу. | Необязательный |
| ipAddresses | string[] | Список IP-адресов, принадлежащих устройству. Эти адреса используются для автоматического обнаружения горизонтальных связей между этим компонентом и другими наблюдаемыми компонентами в Smartscape. Как только связь обнаружена, она автоматически отображается на Smartscape. Если передать значение (в том числе пустое), существующие значения будут перезаписаны. Если передать `null` или вовсе опустить это поле, существующие значения сохранятся. | Необязательный |
| listenPorts | integer[] | Список портов, на которых слушает пользовательское устройство. Эти порты используются для обнаружения горизонтальных связей между этим компонентом и другими наблюдаемыми компонентами в Smartscape. Как только связь обнаружена, она автоматически отображается на Smartscape. Если указаны порты, нужно также указать хотя бы один IP-адрес или DNS-имя устройства. Если передать значение, существующие значения будут перезаписаны. Если передать `null`, пустое значение или вовсе опустить это поле, существующие значения сохранятся. | Необязательный |
| properties | object | Список свойств в формате «ключ-значение», которые будут показаны под инфографикой пользовательского устройства. | Необязательный |
| type | string | Тип технологии пользовательского устройства. Должен совпадать с типом технологии метрики, которую вы отправляете. Если передать значение, существующее будет перезаписано. Если передать `null`, пустое значение или вовсе опустить это поле, существующее значение сохранится. | Необязательный |

### JSON-модель тела запроса

Это модель тела запроса, показывающая возможные поля. Для реального запроса её нужно адаптировать.

```
{



"configUrl": "http://coffee-machine.dynatrace.internal.com/coffeemachine/manage",



"customDeviceId": "customDeviceId",



"displayName": "coffeeMachine",



"dnsNames": [



"coffee-machine.dynatrace.internal.com"



],



"faviconUrl": "https://www.freefavicon.com/freefavicons/food/cup-of-coffee-152-78475.png",



"group": "myCustomDeviceGroup",



"ipAddresses": [



"10.0.0.1"



],



"listenPorts": [



80



],



"properties": {},



"type": "coffee machine"



}
```

## Ответ

### Коды ответа

| Код | Тип | Описание |
| --- | --- | --- |
| **201** | [CustomDeviceCreationResult](#openapi-definition-CustomDeviceCreationResult) | Успех |
| **204** | - | Успех |
| **400** | - | Ошибка. Входные данные некорректны. |
| **4XX** | [ErrorEnvelope](#openapi-definition-ErrorEnvelope) | Ошибка на стороне клиента. |
| **5XX** | [ErrorEnvelope](#openapi-definition-ErrorEnvelope) | Ошибка на стороне сервера. |

### Объекты тела ответа

#### Объект `CustomDeviceCreationResult`

Краткое представление только что созданного пользовательского устройства.

| Поле | Тип | Описание |
| --- | --- | --- |
| entityId | string | Dynatrace entity ID пользовательского устройства. |
| groupId | string | Dynatrace entity ID группы пользовательского устройства. |

### JSON-модели тела ответа

```
{



"entityId": "string",



"groupId": "string"



}
```

## Пример

В этом примере запрос создаёт пользовательское устройство с ID `restExample` и следующими параметрами:

* тип: `F5-Firewall`
* IP-адрес `172.16.115.211`
* порт прослушивания `9999`

Запрос также задаёт несколько дополнительных параметров.

API-токен передаётся в заголовке **Authorization**.

В ответ возвращаются ID устройства (поле **entityId**) и его группы (поле **groupId**) как подтверждение.

Тело примера можно скачать или скопировать и попробовать у себя.

#### Curl

```
curl -L -X POST 'https://mySampleEnv.live.dynatrace.com/api/v2/entities/custom' \



-H 'Authorization: Api-Token dt0c01.abc123.abcdefjhij1234567890' \



-H 'Content-Type: application/json' \



--data-raw '{



"customDeviceId": "restExample",



"displayName" : "F5 Firewall 24",



"ipAddresses" : ["172.16.115.211"],



"listenPorts" : ["9999"],



"type" : "F5-Firewall",



"favicon" : "http://assets.dynatrace.com/global/icons/f5.png",



"configUrl" : "http://192.128.0.1:8080",



"properties" : {



"Sample Property 1": "Sample value 1"



}



}



'
```

#### URL запроса

```
https://mySampleEnv.live.dynatrace.com/api/v2/entities/custom
```

#### Тело запроса

```
{



"customDeviceId": "restExample",



"displayName": "F5 Firewall 24",



"ipAddresses": ["172.16.115.211"],



"listenPorts": ["9999"],



"type": "F5-Firewall",



"favicon": "http://assets.dynatrace.com/global/icons/f5.png",



"configUrl": "http://192.128.0.1:8080",



"properties": {



"Sample Property 1": "Sample value 1"



}



}
```

#### Тело ответа

```
{



"entityId": "CUSTOM_DEVICE-1525F193C0578E2C",



"groupId": "CUSTOM_DEVICE_GROUP-FC2E2ABF54F513D8"



}
```

#### Код ответа

201
