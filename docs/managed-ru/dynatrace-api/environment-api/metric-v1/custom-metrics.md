# Timeseries API v1 - Пользовательские метрики

* Справочник
* Опубликовано 19 июля 2017 г.

**Metrics API v1** позволяет создавать новые типы сетевых компонентов, а также регистрировать и отправлять пользовательские метрики для этих устройств.

Пользовательское сетевое устройство, это любая часть вашей среды, на которой нельзя запустить Dynatrace OneAgent. Примеры: межсетевые экраны, шлюзы DataPower, облачные базы данных и любое другое сетевое устройство, например прокси или шлюз. С помощью Dynatrace API ваше собственное сетевое устройство может отправлять пользовательские метрики в Dynatrace на основе нативных свойств этих устройств, либо вы можете написать собственные скрипты, которые извлекают метрики с вашего сетевого или облачно-сетевого устройства.

Dynatrace отображает пользовательское устройство на уровне хостов и процессов в Smartscape, так как само устройство может содержать сетевой IP-адрес, а также прослушиваемые порты. Когда обнаруживается коммуникация между хостом с инструментированным OneAgent и пользовательским устройством, соединение автоматически отображается в Smartscape. Когда ваши пользовательские устройства начинают отправлять значения метрик, эти метрики можно использовать на плитке пользовательских диаграмм.

Поскольку Dynatrace AI, а также интеллектуальное обнаружение и корреляция проблем зависят от топологической информации, каждое пользовательское устройство должно сообщать о своём расположении в вашей сетевой инфраструктуре. Указав правильный IP-адрес вместе с подробным описанием свойств пользовательского устройства, вы позволяете Dynatrace автоматически разместить ваше пользовательское устройство в вашей существующей среде [Smartscape](/managed/analyze-explore-automate/smartscape-classic "Узнайте, как Smartscape визуализирует все сущности и зависимости в вашей среде.").
