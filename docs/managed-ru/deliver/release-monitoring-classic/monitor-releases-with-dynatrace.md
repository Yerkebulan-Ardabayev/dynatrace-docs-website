# Monitor releases with Dynatrace

# Monitor releases with Dynatrace

* Практическое руководство
* Чтение 4 мин
* Обновлено 11 авг. 2025 г.

После того как настроены [переменные окружения для определения версии](/managed/deliver/release-monitoring-classic/version-detection-strategies "Metadata for version detection in different technologies") и, опционально, [интегрирована система отслеживания задач и настроены динамические запросы](/managed/deliver/release-monitoring-classic/issue-tracking-integration "Integrate your issue tracker into Dynatrace to pull statistics for monitored entities."), можно начинать анализировать данные, относящиеся к каждой релизной версии программного обеспечения.

## Список релизов

Чтобы посмотреть список релизов программного обеспечения в окружении, перейти в **Releases**.

Пример страницы Releases

![Release list](https://dt-cdn.net/images/2021-05-25-12-02-34-1641-8ea61fbf08.png)

Список релизов

Сортировка **Release version** использует лексикографическую (строковую) сортировку, а не сортировку по семантической версии. При лексикографической сортировке цифры трактуются как отдельные символы, и порядок определяется на основе значений ASCII или Unicode этих символов. Сортировка **Release version** использует лексикографическую сортировку, потому что для релизных версий нет фиксированного шаблона, и пользователи могут вводить любое значение.

Страница **Releases** отображает следующую информацию об указанных релизах:

### Release inventory

Показывает все обнаруженные релизы. Каждая запись представляет процессы, экземпляры групп процессов, сгруппированные по версии релиза и сборки, стадии и продукту, к которому они логически относятся.

### Release events

Показывает все события, соответствующие релизам, такие как перезапуск процесса и события развёртывания.

### Tracked issues

Показывает задачи из внешних систем отслеживания задач, связанные с релизами.

## Фильтры

* Чтобы задать простые фильтры, использовать быстрые фильтры в левой части страницы.
* Чтобы задать более детальные фильтры, использовать панель фильтров над таблицей.  
  Например, можно использовать фильтр `tag`, чтобы искать по тегу экземпляра группы процессов.
* Чтобы увидеть только релизы, затронутые проблемами, выбрать **Apply filter** над таблицей **Release inventory**.

  + Фильтрация по конкретному релизу показывает обзор стадий и контекстов продукта, в которых развёрнут релиз.
  + Фильтрация по стадии показывает обзор всех релизов, развёрнутых на этой стадии.

## Список деталей релиза

Чтобы посмотреть детали релиза, выбрать ссылку в столбце **Name** на странице **Releases**.

Пример страницы Release details

![Release details](https://dt-cdn.net/images/2021-05-25-12-41-08-1636-73e6cdccd9.png)

Детали релиза

Страница **Release details** отображает следующую информацию о выбранном релизе:

### Release name

Показывает сводку захваченных метаданных (имя группы процессов, версия, стадия, продукт, технологии, экземпляры, пропускная способность, проблемы и уязвимости сторонних компонентов) и ссылки на группу процессов, в которой развёрнут релиз.

* **Instances** показывает количество развёртываний этого релиза на конкретной стадии в контексте конкретного продукта.
* **Throughput** показывает, сколько трафика направляется на выбранный релиз.
* **Problems** показывает количество открытых затрагивающих проблем, связанных с экземпляром группы процессов релиза.

  Если Dynatrace обнаруживает проблемы в релизе, **Problems** отображается красным. Выбрать это, чтобы открыть страницу **Problems** и собрать больше информации. Подробнее см. [Problem overview](/managed/dynatrace-intelligence/root-cause-analysis/concepts "Get acquainted with root cause analysis concepts.").
* **Third-party vulnerabilities** показывает количество уязвимостей сторонних компонентов, связанных с выбранным релизом. Это помогает проверить, например, не внесёт ли новый релиз известную уязвимость, и на основе этого решить, как это влияет на график релизов и насколько это релевантно.

  Если Dynatrace обнаруживает уязвимости в релизе, **Third-party vulnerabilities** отображается красным. Выбрать это, чтобы открыть страницу **Security** и собрать больше информации. Подробнее см. [Application Security monitoring](/managed/secure/application-security "Access the Dynatrace Application Security functionalities.").

### Process details

Показывает хосты с базовыми метриками, на которых развёрнут релиз, и ссылки на страницу хоста для дальнейшего анализа.

### Release events

Показывает все события, соответствующие выбранному релизу, такие как перезапуск процесса и события развёртывания.

### Tracked issues

Показывает задачи из внешних систем отслеживания задач, связанные с выбранным релизом.

### Release validation

Чтобы оптимизировать таблицу release inventory, когда не выделен ни один экземпляр Cloud Automation, столбец с результатом release validation не отображается.

Показывает список release validation, обнаруженных в пределах глобального периода времени.

* Список можно сортировать по оценке и времени окончания оценки.
* Выбрать любую release validation в списке, чтобы перейти к тепловой карте соответствующей оценки в экземпляре Cloud Automation.

Кроме того, график показывает последние 20 результатов release validation:

* Ось Y отображает оценку
* Ось X отображает время окончания каждой оценки quality gate
* Цвет столбца показывает статус quality gate (`Failed`, `Warning` или `Passed`)
