# Monitor releases with Dynatrace

# Monitor releases with Dynatrace

* How-to guide
* 4-min read
* Updated on Aug 11, 2025

После [настройки переменных окружения для обнаружения версий](/managed/deliver/release-monitoring/version-detection-strategies "Metadata for version detection in different technologies") и, при необходимости, [интеграции системы отслеживания задач и настройки динамических запросов](/managed/deliver/release-monitoring/issue-tracking-integration "Integrate your issue tracker into Dynatrace to pull statistics for monitored entities."), вы можете приступить к анализу данных по каждой версии релиза вашего ПО.

## Список релизов

Для отображения списка программных релизов в вашем окружении перейдите в **Releases**.

Пример страницы Releases

![Release list](https://dt-cdn.net/images/2021-05-25-12-02-34-1641-8ea61fbf08.png)

Список релизов

Сортировка **Release version** использует лексикографическую (строковую) сортировку, а не семантическую. При лексикографической сортировке числа обрабатываются как отдельные символы, а порядок определяется на основе ASCII- или Unicode-значений этих символов. Лексикографическая сортировка используется потому, что нет единого шаблона для версий релизов, и пользователи могут вводить любые значения.

На странице **Releases** отображается следующая информация о перечисленных релизах:

### Инвентарь релизов

Отображает все обнаруженные релизы. Каждая запись представляет процессы, экземпляры групп процессов, сгруппированные по версии релиза и сборки, стейджу и продукту, к которому они логически относятся.

### События релизов

Отображает все события, соответствующие релизам, например перезапуск процессов и события развёртывания.

### Отслеживаемые задачи

Отображает задачи из внешних трекеров, связанные с релизами.

## Фильтры

* Для установки простых фильтров используйте быстрые фильтры слева на странице.
* Для более детальной фильтрации используйте панель фильтров над таблицей.  
  Например, можно использовать фильтр `tag` для поиска по тегу экземпляра группы процессов.
* Для отображения только релизов, затронутых проблемами, выберите **Apply filter** над таблицей **Release inventory**.

  + Фильтрация по конкретному релизу показывает обзор стейджей и контекстов продуктов, в которых он развёрнут.
  + Фильтрация по стейджу показывает обзор всех релизов, развёрнутых на этом стейдже.

## Сведения о релизе

Для просмотра сведений о релизе выберите ссылку в столбце **Name** на странице **Releases**.

Пример страницы Release details

![Release details](https://dt-cdn.net/images/2021-05-25-12-41-08-1636-73e6cdccd9.png)

Сведения о релизе

На странице **Release details** отображается следующая информация о выбранном релизе:

### Имя релиза

Показывает сводку захваченных метаданных (имя группы процессов, версия, стейдж, продукт, технологии, экземпляры, пропускная способность, проблемы и уязвимости сторонних компонентов) и ссылки на группу процессов, в которой развёрнут релиз.

* **Instances** показывает количество развёртываний этого релиза на конкретном стейдже в контексте конкретного продукта.
* **Throughput** показывает объём трафика, маршрутизируемого к выбранному релизу.
* **Problems** показывает количество открытых проблем, затрагивающих экземпляр группы процессов данного релиза.

  Если Dynatrace обнаруживает проблемы в релизе, **Problems** отображается красным. Выберите его для перехода на страницу **Problems** и получения дополнительной информации. Подробнее см. в разделе [Обзор проблем](/managed/dynatrace-intelligence/root-cause-analysis/concepts "Get acquainted with root cause analysis concepts.").
* **Third-party vulnerabilities** показывает количество уязвимостей сторонних компонентов, связанных с выбранным релизом. Это позволяет проверить, например, не введёт ли новый релиз известную уязвимость, и на основе этого принять решение о его влиянии на расписание релизов.

  Если Dynatrace обнаруживает уязвимости в релизе, **Third-party vulnerabilities** отображается красным. Выберите его для перехода на страницу **Security** и получения дополнительной информации. Подробнее см. в разделе [Application Security monitoring](/managed/secure/application-security "Access the Dynatrace Application Security functionalities.").

### Сведения о процессе

Отображает хосты с базовыми метриками, на которых развёрнут релиз, и ссылки на страницу хоста для дальнейшего анализа.

### События релизов

Отображает все события, соответствующие выбранному релизу, например перезапуск процессов и события развёртывания.

### Отслеживаемые задачи

Отображает задачи из внешних трекеров задач, связанные с выбранным релизом.

### Валидация релиза

Для оптимизации таблицы инвентаря релизов, когда экземпляр Cloud Automation не подготовлен, столбец для отображения результата валидации релиза не показывается.

Перечисляет валидации релиза, обнаруженные в рамках глобального временного диапазона.

* Список можно сортировать по оценке и времени окончания оценки.
* Выберите любую валидацию релиза в списке для перехода к тепловой карте соответствующей оценки в вашем экземпляре Cloud Automation.

Дополнительно на графике отображаются последние 20 результатов валидации релиза:

* По оси Y отображается оценка
* По оси X отображается время окончания каждой оценки контроля качества (quality gate)
* Цвет столбца указывает статус контроля качества (`Failed`, `Warning` или `Passed`)
