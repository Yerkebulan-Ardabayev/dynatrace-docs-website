# SLO management with management-zone permissions

# SLO management with management-zone permissions

* Reference
* 3-min read
* Published Feb 16, 2022

Вы можете управлять доступом к SLO в вашем окружении, задавая разрешения на запись и/или чтение для пользователей или групп пользователей на уровне окружения или зоны управления.
Подробнее о настройке разрешений см. в разделе [Role-based permissions](/managed/manage/identity-access-management/permission-management/role-based-permissions "Role-based permissions").

Проверки разрешений основаны только на глобальном селекторе сущностей.

## Доступ на чтение на основе уровней разрешений

* На уровне окружения пользователь с разрешением **View environment** может просматривать любой SLO. Для управления разрешениями групп можно [создавать политики](/managed/manage/identity-access-management/permission-management/manage-user-permissions-policies "Working with policies").

  **Пример политики:** `ALLOW settings:objects:read, settings:schemas:read WHERE settings:schemaId = "builtin:monitoring.slo";`
* На уровне зоны управления пользователь с разрешением **View environment** может читать все глобальные SLO и все SLO, принадлежащие соответствующей зоне управления.

  В настоящее время политику нельзя назначить на уровне зоны управления.

## Доступ на запись на основе уровней разрешений

* На уровне окружения пользователь с разрешениями **Change monitoring settings** может создавать и обновлять любой SLO, включая глобальные SLO без указанных зон управления. Для управления разрешениями групп можно [создавать политики](/managed/manage/identity-access-management/permission-management/manage-user-permissions-policies "Working with policies").

  **Пример политики:** `ALLOW settings:objects:read, settings:schemas:read, settings:objects:write WHERE settings:schemaId = "builtin:monitoring.slo";`.
* На уровне зоны управления пользователь с разрешениями **Change monitoring settings** может создавать и обновлять любой SLO для соответствующей зоны управления, но не может создавать или обновлять глобальные SLO.

## Просмотр и редактирование SLO на основе уровней разрешений

При оценке SLO учитываются как выбранная зона управления, так и зона управления из селектора сущностей.

* На уровне окружения

  + Вы можете просматривать, создавать, обновлять и удалять SLO. Указание селектора сущностей в SLO необязательно.
* На уровне зоны управления

  + Вы можете просматривать все SLO без селекторов сущностей или с селекторами сущностей, в которых зона управления не указана; однако они будут оцениваться только для вашей выбранной зоны управления.
  + Вы можете просматривать SLO, в селекторе сущностей которых указана одна из ваших зон управления.
  + Вы можете редактировать SLO, в селекторе сущностей которых указана одна из ваших зон управления, при наличии разрешений на запись для этой конкретной зоны управления.
  + Вы можете создавать новые SLO, в которых в селекторе сущностей указана хотя бы одна зона управления, к которой у вас есть доступ.

## Просмотр и добавление плиток SLO на дашборде на основе уровней разрешений

* Вы можете добавлять SLO, видимые на странице **Service-level objectives**, на свои классические дашборды, нажав **Pin to dashboard** в **Actions**.
* Новые SLO можно добавлять непосредственно на дашборде. Список фильтруется на основе SLO, видимых в выбранной зоне управления.
* Глобальные SLO (без селекторов сущностей или с селекторами сущностей, в которых зона управления не указана) можно добавлять при фильтрации по **All** в фильтре зоны управления.
* Фильтрация по плиткам SLO недоступна. В зависимости от вашего доступа к зонам управления SLO в плитке, для выбранной зоны управления вы либо получаете доступ к плитке, либо видите результат **N/A**.
* Если на дашборде есть SLO с зоной управления, к которой у вас нет доступа, плитка отображает **Access denied**.

Начиная с Dynatrace версии 1.233+, для конкретной плитки также можно задать пользовательскую зону управления, аналогично временному диапазону. В этом случае глобальный фильтр зоны управления не актуален для соответствующей плитки.
