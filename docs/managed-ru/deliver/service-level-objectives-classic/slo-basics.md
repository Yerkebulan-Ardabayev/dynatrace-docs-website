# Service-Level Objectives basics

# Service-Level Objectives basics

* Explanation
* 2-min read
* Updated on Mar 25, 2024

Dynatrace предоставляет всю необходимую информацию в реальном времени, которая требуется командам **Site Reliability Engineering (SRE)** для мониторинга определённых ими целей.

Команда SRE отвечает за выбор подходящих **показателей уровня обслуживания** (**SLI**) для конкретного сервиса с целью тщательного мониторинга надёжности его предоставления.
SLI могут различаться от сервиса к сервису, так как не все сервисы одинаково критичны с точки зрения временных ограничений и ограничений по ошибкам.  
Dynatrace предлагает более 2000 различных метрик, готовых к использованию в качестве специализированных SLI.

После того как команда SRE выбрала набор индикаторных метрик, остаётся определить и контролировать операционную цель в рамках **цели уровня обслуживания** (**SLO**).

Типичная цель уровня обслуживания (SLO) состоит из следующих основных данных:

* **Показатель уровня обслуживания (SLI)**: индикатор, используемый для измерения успешного предоставления сервиса. SLI, как правило, ссылаются на метрики, такие как процент успешных запросов к сервису, доля пользователей мобильного приложения без сбоев, доля успешных запусков синтетических тестов или время отклика.
* **Целевое значение (Target)**: цель определяет планируемый показатель для достижения в части предоставления сервиса. Цель может быть, например, такой: 99,99% всех вызовов сервиса должны возвращаться без ошибки, или 95% всех запросов к сервису должны выполняться за время отклика менее 2 секунд.
* **Период оценки**: период оценки необходим для стандартизации коммуникации по результату SLO. Без определённого периода оценки понятия доступности субъективны.

Каждое определение SLO может оцениваться по следующим результирующим метрикам:

* **Статус SLO**: текущий результат оценки SLO, выраженный в процентах. Семантика этого процента (например, 99,3% всех запросов к сервису успешны, или 99,99% всех пользователей веб-сайта «удовлетворены» по рейтингу Apdex) и целевое значение для этого процента определяются командой SRE.

  Статус SLO должен быть нормализован к процентному значению в диапазоне от 0 до 100%.
* **Бюджет ошибок SLO**: оставшийся запас до момента, когда определённая цель SLO считается нарушенной. Например, если SLO определяет целевое значение 95%, а текущий статус SLO оценивается как 98%, оставшийся бюджет ошибок — это разница между статусом SLO и целевым значением SLO, в данном случае 3%.

Подробнее о нормализованном бюджете ошибок SLO и скорости сжигания бюджета ошибок SLO см. в разделе [Настройка и мониторинг целей уровня обслуживания с помощью Dynatrace](/managed/deliver/service-level-objectives-classic/configure-and-monitor-slo "Create, configure, and monitor service-level objectives with Dynatrace.").
