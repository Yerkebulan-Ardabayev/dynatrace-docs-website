# Release monitoring

# Release monitoring

* Обзор
* Чтение за 1 мин
* Опубликовано 14 сентября 2020 г.

Жизненный цикл релиза программного продукта требует тщательного управления рисками релиза. Кроме того, по мере того как в организации разворачивается всё больше компонентов и версий, частота релизов растёт, а ручной сбор релиз-релевантных данных легко становится узким местом в конвейере автоматизации релизов и автоматизированном жизненном цикле ПО.

Dynatrace предлагает встроенное решение для анализа релизов, которое помогает определить:

* Какие версии развёрнуты на этапах развёртывания и в производственных средах на основе нескольких стратегий определения версий.
* Стадии релиза развёрнутых версий.
* Список изменений для новой версии.
* Известные ошибки и являются ли они блокерами релиза.
* Риски, связанные с конкретными версиями.
* Какая версия испытывает избыточную нагрузку (например, если временно перенаправляется нагрузка с помощью canary-развёртывания).
* Как новая версия ведёт себя по сравнению с предыдущими версиями.
* Статистику проблем, связанных с отслеживаемыми сущностями.
