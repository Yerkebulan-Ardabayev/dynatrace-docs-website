# Backstage integration

# Backstage integration

* Tutorial
* 4-min read
* Updated on Nov 17, 2025

Интегрируйте Dynatrace в Backstage для повышения качества опыта разработчиков.

## Введение

Богатая контекстом наблюдаемость и аналитика безопасности всегда под рукой — разработчики хорошо знают поведение своих сервисов в среде разработки или тестовой среде, однако им не хватает информации о производительности сервиса на этапе стейджинга или в продуктивной среде. Плагины Dynatrace для Backstage позволяют разработчикам получать данные об наблюдаемости и безопасности из Dynatrace и отображать их в программных компонентах, управляемых через [каталог программного обеспечения Backstage](https://backstage.io/docs/features/software-catalog/). Данные представлены в табличном формате с умными ссылками на приложения Dynatrace для более глубокого анализа и расследования первопричин в случае связанных проблем или уязвимостей безопасности.

Для интеграции Dynatrace в Backstage доступны: Backstage Dynatrace plugin for Managed и Backstage Dynatrace plugin.

Отказ от ответственности

Плагин Backstage Dynatrace plugin for Managed не описан на данной странице. Подробнее см. [Backstage Dynatrace plugin for Managed](https://github.com/backstage/community-plugins/tree/main/workspaces/dynatrace/plugins/dynatrace).

Плагин Backstage Dynatrace plugin for Managed предоставляет:

* Отображение связанных с сервисом проблем и результатов синтетических тестов
* Основан на Dynatrace v2 API

Плагин Backstage Dynatrace plugin предоставляет:

* Kubernetes-аналитику по умолчанию с богатым контекстом и глубокими ссылками
* Поддержку нескольких окружений
* Подход на основе DQL для получения любых данных наблюдаемости и безопасности
* Табличное представление результатов DQL
* Просмотр валидаций Site Reliability Guardian

Подробнее см. [плагины Backstage Dynatrace](https://github.com/Dynatrace/backstage-plugin).

## Целевая аудитория

Вы — платформенный инженер, который стремится снизить когнитивную нагрузку на разработчиков и выстроить процессы, обеспечивающие им быструю обратную связь. Необходимо знание Backstage и DQL для удовлетворения требований разработчиков к обзору безопасности и производительности, а также к точкам входа для более глубоких расследований.

## Сценарий

Цель
:   Предоставить разработчикам необходимые сведения об наблюдаемости и безопасности для совершенствования и эксплуатации их компонентов.

[Backstage](https://backstage.io/) стал популярным порталом разработчиков для построения внутренней платформы разработки (IDP). Сфокусированный на различных возможностях для разработчиков, централизованный [каталог программного обеспечения Backstage](https://backstage.io/docs/features/software-catalog/) является фундаментальным элементом Backstage. Он позволяет управлять всем вашим программным обеспечением: микросервисами, библиотеками, пайплайнами данных, веб-сайтами, моделями машинного обучения и другими компонентами. Backstage восстанавливает порядок в ваших микросервисах и инфраструктуре, позволяя продуктовым командам быстро выпускать высококачественный код без ущерба для автономии.

Помимо централизованного управления программными компонентами, разработчики хотят получать данные мониторинга в реальном времени и одним кликом выявлять проблемы или уязвимости безопасности для своих развёртываний. Плагин Backstage Dynatrace plugin предоставляет эту информацию разработчикам через обзорные таблицы с богатым контекстом в Backstage. В результате они получают обзор производительности сервиса и важные показатели: счётчики проблем или уязвимости безопасности. Умные глубокие ссылки направляют разработчика в Dynatrace, где он может провести более глубокий анализ и расследование первопричин без перегрузки этого обзора.

На следующем снимке экрана показаны развёртывания сервиса на трёх стейджах с глубокой ссылкой на Kubernetes-рабочую нагрузку и логи, а также отображаются окружения Dynatrace, которые их мониторят.

## Предварительные условия

* Backstage версии 1.18.0 или выше
* Клиент OAuth 2.0:

1. Перейдите в [**Account Management**](https://myaccount.dynatrace.com/). Если у вас несколько аккаунтов, выберите нужный.
2. Выберите **Identity & access management** > **OAuth clients**.
3. Нажмите **Create client**.
4. Введите описание клиента и адрес электронной почты пользователя.
5. Выберите как минимум следующие области доступа:

   * `storage:metrics:read`
   * `storage:entities:read`
   * `storage:events:read`
   * `storage:buckets:read`
   * `storage:security.events:read`
6. Прокрутите вниз и нажмите **Create client**.
7. Скопируйте идентификатор клиента, секрет клиента и URN аккаунта Dynatrace. Эти настройки необходимы для [конфигурации](https://github.com/Dynatrace/backstage-plugin?tab=readme-ov-file#add-dynatrace-environment-connection) плагина Backstage.

## Настройка и использование

### Шаг 1: начало работы

Руководство по установке, интеграции, настройке и запуску плагина Dynatrace для Backstage описано и поддерживается в [руководстве по началу работы](https://github.com/Dynatrace/backstage-plugin?tab=readme-ov-file#getting-started).

Вкратце, необходимо:

* Установить плагины в Backstage
* Интегрировать фронтенд-элемент на страницу компонента
* Интегрировать бэкенд-плагин
* Настроить подключение к окружению Dynatrace
* Перезапустить/запустить Backstage

### Шаг 2: использование дополнительных возможностей

Плагины Backstage Dynatrace предоставляют ряд возможностей кастомизации для различных требований.

* Поддержка нескольких окружений: подключите несколько окружений Dynatrace к плагину Backstage Dynatrace для отображения данных из разных окружений. Распространённый вариант использования — разделение на pre-production и production мониторинг с различными окружениями Dynatrace.
* Сценарий Kubernetes: плагины Backstage Dynatrace поддерживают Kubernetes-сущности по умолчанию с предварительно настроенным запросом для Kubernetes-развёртываний и отдельным компонентом для представления данных.
* Пользовательские запросы: используйте любой пользовательский DQL-запрос для получения данных из Dynatrace и их отображения в табличном формате. Результаты с единственным значением не поддерживаются.
* Обратная ссылка в Dynatrace: обогащайте DQL-запросы умными глубокими ссылками, направляющими пользователя Backstage в Dynatrace для более глубокого анализа наблюдаемости и безопасности.

Подробнее см. [возможности кастомизации](https://github.com/Dynatrace/backstage-plugin?tab=readme-ov-file#additional-features).

Следуя этой лучшей практике, ваши разработчики получают сведения об наблюдаемости и безопасности всегда под рукой, что позволяет им немедленно получать обратную связь от Dynatrace о поведении своих сервисов в процессе стейджинга и в продуктивной среде.
