# Release monitoring

# Release monitoring

* Overview
* 1-min read
* Published Sep 14, 2020

Жизненный цикл программного продукта в рамках релиза требует тщательного управления рисками. Кроме того, по мере развёртывания всё большего числа компонентов и версий частота релизов в вашей организации растёт, и ручной сбор данных, связанных с релизами, может легко стать узким местом в вашем пайплайне автоматизации и автоматизированном жизненном цикле ПО.

Dynatrace предлагает встроенное решение для анализа релизов, которое помогает определить:

* Какие версии развёрнуты на ваших стейджах и в производственных окружениях — на основе нескольких стратегий обнаружения версий.
* Стейджи релиза для развёрнутых версий.
* Список изменений для новой версии.
* Известные ошибки и являются ли они блокировщиками релиза.
* Риски, связанные с конкретными версиями.
* Какая версия испытывает избыточную нагрузку (например, при временном перенаправлении нагрузки с помощью canary-развёртывания).
* Как ведёт себя новая версия по сравнению с предыдущими.
* Статистику задач, связанных с отслеживаемыми сущностями.
