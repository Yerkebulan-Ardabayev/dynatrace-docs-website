# Lens

# Lens

* Чтение 5 мин
* Обновлено 24 окт 2025 г.

Функция **Lens** в Управлении аккаунтом предоставляет аналитику по использованию платформы и её работоспособности, помогая понять, как Dynatrace используется в вашей организации, и рекомендует области для оптимизации.

Lens имеет две основные страницы: [Adoption](#adoption) и [Environments](#environments).

Lens не имеет доступа к данным самомониторинга тенантов на кластерах Managed.

Чтобы просматривать данные самомониторинга вашего кластера Managed в Lens, необходимо загружать эти данные в общий или выделенный тенант SaaS.

Выделенные тенанты SaaS доступны для клиентов Managed с Enterprise Success & Support.
За дополнительной информацией обратитесь к специалисту Dynatrace через live chat в вашем окружении.

## Adoption

Страница **Lens** > **Adoption** даёт представление о принятии платформы и её работоспособности:

* Активность пользователей
* Какие технологии отслеживаются в Dynatrace
* Способ развёртывания мониторируемых хостов (тип лицензии, тип платформы)
* Количество запущенных синтетических тестов
* Количество настроенных интеграций уведомлений о проблемах

### Активность пользователей и использование функций

Диаграмма **User activity** показывает, сколько пользователей обращаются к Dynatrace с течением времени и сколько сессий открывается.

Диаграмма **Feature usage** помогает понять, какие функции Dynatrace используются наиболее часто.

* Чем больше площадь функции на диаграмме, тем чаще она используется при обращении к Dynatrace (сессии)
* Чем темнее область функции, тем больше у неё просмотров страниц (независимо от количества сессий)

Например, пользователь, вошедший в Dynatrace и перешедший на **Dashboards** 10 раз, учитывается как 1 сессия и 10 просмотров страниц.

![Пример использования функций](https://dt-cdn.net/images/lens-001-1839-ba5a894223.png)

Пример использования функций

Данные для этих диаграмм основаны на автоматизированных возможностях самоуправления Dynatrace.
Это включает сбор данных мониторинга реальных пользователей (на базе Dynatrace) с вашей платформы Dynatrace.

* Для клиентов Dynatrace SaaS это выполняется автоматически
* Для клиентов Dynatrace Managed требуется [включение проактивной поддержки](/managed/managed-cluster/basics/mission-control-proactive-support "Mission Control проактивно мониторит ваш Managed Cluster.").

  + Если данные не отображаются, обратитесь к специалисту Dynatrace через live chat для проверки конфигурации самомониторинга.
  + Подробнее о самомониторинге Dynatrace для клиентов Managed см. раздел **Pro-active support** в [Обмене данными Mission Control](/managed/managed-cluster/basics/mission-control-data-exchange "Ознакомьтесь с данными, которыми ваш Managed Cluster обменивается с Mission Control.").

## Environments

**Lens** > **Environments** предлагает:

* Обзор количества технологий, обнаруженных и отслеживаемых Dynatrace с течением времени.
* Сводку по хостам окружений в вашем развёртывании Dynatrace.
* Сводку о количестве мониторируемых хостов, развёрнутых в облаке.
* Сводку о количестве публичных и частных синтетических тестов, запущенных в ваших окружениях.
* Общую сводку о количестве определённых API tokens, интеграций уведомлений о проблемах, используемых PaaS-интеграций и экспортов сессий.

### Окружения

Обзор количества технологий, обнаруженных и отслеживаемых Dynatrace с течением времени.

### Сведения о мониторинге хостов

Предоставляет сводку по хостам окружений в вашем развёртывании Dynatrace с разбивкой мониторируемых хостов по операционным системам, режиму мониторинга и облачному развёртыванию. Данные основаны на общем числе хостов, наблюдавшихся за предыдущие 24 часа. В высокоэластичных окружениях, где в течение дня запускается и останавливается много хостов, это число может быть выше ожидаемого.

### Облако и виртуализация

Сводка о количестве мониторируемых хостов, развёрнутых в облаке.

### Синтетические тесты

Сводное представление о количестве публичных и частных синтетических тестов, запущенных в ваших окружениях. Период по умолчанию — 3 месяца; можно изменить в блоке.

### Входящие/исходящие интеграции данных

Общая сводка о количестве определённых API tokens, интеграций уведомлений о проблемах, используемых PaaS-интеграций и экспортов сессий.

* **API tokens:** Dynatrace предоставляет мощные API для интеграции Dynatrace в ваши жизненные циклы процессов и извлечения информации. Значение `0` токенов указывает на то, что организация не использует эти API.
* **Problem notifications:** Уведомления о проблемах помогают определить, интегрирован ли Dynatrace с какой-либо внутренней системой. Если блок показывает `0` уведомлений о проблемах и организация использует хелпдеск (например, ServiceNow) или систему пейджинга (например, PagerDuty), возможно, вы упускаете возможность интеграции.
* **PaaS tokens:** Быстрый показатель того, настроен ли Dynatrace с каким-либо из ваших PaaS-окружений (например, Microsoft Azure, Cloud Foundry, IBM Bluemix).
* **Session Export:** Показывает, настроено ли в каком-либо из ваших окружений потоковое кодирование данных мониторинга реальных пользователей из Dynatrace во внешний источник данных.
