# Обзор лицензии (классическое лицензирование Dynatrace)

* Пояснение
* Чтение 3 мин
* Опубликовано 24 ноя 2025 г.

Представление **License** в Управлении аккаунтом предоставляет актуальное представление потребления продуктов по классической схеме лицензирования Dynatrace.
Классическое лицензирование Dynatrace охватывает использование таких продуктов, как host units, host unit hours, Digital Experience Monitoring, Davis data units (DDUs) и Application Security units (ASUs).
Исторический анализ доступен на ежедневном или почасовом уровне.

Доступ к этой информации осуществляется через **Account Management** > **License**.

Если вы видите **Account Management** > **Subscription**, обратитесь к [документации Dynatrace Platform Subscription](/managed/manage/account-management/license-subscription/subscription-overview-dps "Просмотр сводки бюджета и анализа стоимости Dynatrace Platform Subscription.").

## Общее использование лицензии

Раздел **Total license usage** отображает актуальную информацию о потреблении лицензии.

![Представление License в Управлении аккаунтом для классической модели лицензирования Dynatrace, с выделением Total license usage.](https://dt-cdn.net/images/account-management-classic-total-license-usage-1857-f5c4b93a02.png)

Представление License в Управлении аккаунтом для классической модели лицензирования Dynatrace, с выделением Total license usage.

Каждый лицензируемый продукт отображается в виде отдельной инфографики, показывающей, сколько каждого продукта было использовано.
Цветные полосы показывают, находитесь ли вы в пределах квоты или превысили её:

* Синий: Использование ниже квоты.
* Коричневый: Квота превышена, и использование влечёт сверхлимитные расходы (если это разрешено для вашей лицензии).
* Красный: Достигнута максимальная квота, мониторинг деактивирован.

Наведите курсор на инфографику продукта для отображения детальной разбивки использования.

## Сведения об использовании

Раздел **Usage details** отображает историческое потребление лицензии.

![Представление License в Управлении аккаунтом для классической модели лицензирования Dynatrace, с выделением Usage details.](https://dt-cdn.net/images/account-management-classic-usage-details-1857-acaa5cff26.png)

Представление License в Управлении аккаунтом для классической модели лицензирования Dynatrace, с выделением Usage details.

Информацию можно фильтровать по **Cluster**, **Product**, **Resolution** и **Timeframe**.

Данные можно экспортировать в формате CSV.
Для этого выберите ![Export](https://dt-cdn.net/images/dashboards-app-dashboard-export-5ed2168e10.svg "Export") **Export data**.

Используйте **Resolution** для выбора детализации:

* **Daily**: Ежедневное суммарное использование.
* **Hourly**: Почасовые накопленные итоги за выбранный день.
* **Total**: Накопленное использование для лицензий на основе потребления (включает host unit hours, Digital Experience Monitoring, Davis data units и Application Security units).
* **Linear forecast**: Прогноз использования на основе линейной экстраполяции данных за последние 30 дней (только для Digital Experience Monitoring и Davis data units).
* **Seasonal forecast**: Прогноз использования на основе сезонного прогноза из данных за последние 365 дней (только для Digital Experience Monitoring и Davis data units).

## ЧаВо

### Почему потребление лицензии в **Usage details** не совпадает с информацией в **Total license usage**?

* **Total license usage** предоставляет точное и актуальное измерение потребления лицензии.
  Эти измерения разграничивают бесплатное, платное и сверхлимитное использование.
* **Usage details** предоставляет историческую разбивку общего потребления.
  Эта разбивка включает все единицы и не разграничивает платное, бесплатное и сверхлимитное использование.

### Когда следует использовать линейный прогноз, а когда — сезонный?

* Линейная модель:

  + Для новых клиентов линейная модель обеспечивает более точные прогнозы в первый год потребления.
  + Линейная модель также точна, если потребление за предыдущие 30 дней является показательным для будущего использования.
* Сезонная модель: если ваш бизнес носит более сезонный характер (например, розничное использование возрастает в праздничные периоды) и у вас есть данные за год или более, сезонная модель может дать более точную оценку будущего использования.

### Почему прогноз показывает, что я израсходовал больше DDUs, чем предусмотрено текущей лицензией?

Прогнозы отображают суммарное потребление, накопленное с первого дня действия вашей первой лицензии Dynatrace.
Суммарное потребление может включать потребление по лицензии, срок действия которой уже истёк.

Поэтому даже если прогноз показывает, что вы израсходовали больше DDUs, чем предусмотрено текущей лицензией, это не означает автоматически, что вы уже израсходовали DDUs текущей лицензии.
