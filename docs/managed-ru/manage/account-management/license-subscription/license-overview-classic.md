# Обзор (классическое лицензирование Dynatrace)

* Пояснение
* 3 мин на чтение
* Опубликовано 24 ноября 2025 г.

Раздел **License** в Account Management показывает потребление продуктов классического лицензирования Dynatrace в реальном времени.
Классическое лицензирование Dynatrace охватывает использование таких продуктов, как host units, host unit hours, Digital Experience Monitoring, Davis data units (DDUs) и Application Security units (ASUs).
Исторический анализ доступен на дневном или почасовом уровне.

Эта информация доступна в **Account Management** > **License**.

Если вместо этого отображается **Account Management** > **Subscription**, нужно обратиться к документации [Dynatrace Platform Subscription](/managed/manage/account-management/license-subscription/subscription-overview-dps "View your Dynatrace Platform Subscription (DPS) budget summary and cost analysis.").

## Общее потребление лицензии

Раздел **Total license usage** показывает информацию о потреблении лицензии в реальном времени.

![Раздел License в Account Management для классической модели лицензирования Dynatrace, с выделенным общим потреблением лицензии.](https://dt-cdn.net/images/account-management-classic-total-license-usage-1857-f5c4b93a02.png)

Раздел License в Account Management для классической модели лицензирования Dynatrace, с выделенным общим потреблением лицензии.

Каждый лицензированный продукт отображается на отдельной инфографике, которая показывает объём его использования.
Цвет полосы показывает, находится потребление в рамках квоты или превышает её:

* Синий: потребление ниже квоты.
* Коричневый: квота превышена, потребление приводит к дополнительным начислениям (если это допустимо для лицензии).
* Красный: достигнута максимальная квота, мониторинг деактивирован.

При наведении на инфографику продукта отображается подробная разбивка потребления.

## Детали потребления

Раздел **Usage details** показывает историческое потребление лицензии.

![Раздел License в Account Management для классической модели лицензирования Dynatrace, с выделенными деталями потребления.](https://dt-cdn.net/images/account-management-classic-usage-details-1857-acaa5cff26.png)

Раздел License в Account Management для классической модели лицензирования Dynatrace, с выделенными деталями потребления.

Эту информацию можно фильтровать по **Cluster**, **Product**, **Resolution** и **Timeframe**.

Эти данные можно экспортировать в формате CSV.
Для этого нужно выбрать ![Export](https://dt-cdn.net/images/dashboards-app-dashboard-export-5ed2168e10.svg "Export") **Export data**.

С помощью **Resolution** выбирается детализация:

* **Daily**: суммарное потребление по дням.
* **Hourly**: почасовые накопительные итоги за выбранный день.
* **Total**: накопительное потребление для лицензий с оплатой по факту потребления (включает host unit hours, Digital Experience Monitoring, Davis data units и Application Security units).
* **Linear forecast**: прогноз потребления на основе линейной экстраполяции данных за последние 30 дней (доступно только для Digital Experience Monitoring и Davis data units).
* **Seasonal forecast**: прогноз потребления на основе сезонного прогноза по данным за последние 365 дней (доступно только для Digital Experience Monitoring и Davis data units).

## Часто задаваемые вопросы

### Почему потребление лицензии в **Usage details** не совпадает с информацией в **Total license usage**?

* **Total license usage** обеспечивает точное измерение потребления лицензии в реальном времени.
  Эти измерения различают бесплатное, платное потребление и потребление сверх квоты.
* **Usage details** показывает историческую разбивку общего потребления.
  Эта разбивка включает все единицы и не различает платное, бесплатное потребление и потребление сверх квоты.

### Когда использовать линейные и сезонные прогнозы?

* Линейная модель:

  + Если это новый клиент, линейная модель даёт более точные прогнозы в первый год потребления.
  + Линейная модель также точна, если потребление за предыдущие 30 дней отражает будущее использование.
* Сезонная модель: если бизнес более сезонный (например, всплески потребления в розничной торговле в праздничные периоды) и есть данные о потреблении за год и более, сезонная модель может дать более точную оценку будущего использования.

### Почему прогноз показывает, что потребление DDU превышает объём текущей лицензии?

Прогнозы показывают общее потребление, накопленное с первого дня действия первой лицензии Dynatrace.
Общее потребление может включать потребление по лицензии, срок действия которой уже истёк.

Поэтому, даже если прогноз показывает потребление DDU больше, чем предусмотрено текущей лицензией, это не означает, что DDU текущей лицензии уже исчерпаны.
