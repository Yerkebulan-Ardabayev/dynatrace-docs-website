# Центр уведомлений

# Центр уведомлений

* Пояснение
* Опубликовано 29 мая 2026 г.

Account Management отправляет уведомления при приближении к пороговым значениям расходов или при обнаружении неожиданных изменений в потреблении.
На этой странице описано, как работают уведомления в разделах [Оповещения о бюджете](/managed/manage-your-costs/control/budgets "Узнайте, как настроить бюджеты в Dynatrace.") и [Мониторы затрат](/managed/manage-your-costs/control/cost-monitors "Узнайте, как использовать функцию Cost Monitors для прогнозирования и отслеживания событий затрат.").

## Обзор

В Account Management уведомления генерируют две функции.

* **Оповещения о бюджете** уведомляют о том, что фактическое или прогнозируемое потребление достигло настроенного порога.
* **Мониторы затрат** уведомляют об обнаружении прогнозных условий или аномалий в дневном потреблении.

Обе функции отправляют уведомления через **Центр уведомлений** и по электронной почте.
Мониторы затрат дополнительно поддерживают уведомления на основе API.

## Сценарии использования

* Понять, где в Account Management отображаются уведомления.
* Проверить, какие права доступа нужны для получения и открытия уведомлений.
* Решить, стоит ли интегрировать уведомления мониторов затрат через API.
* Сравнить поведение уведомлений бюджета и мониторов затрат перед настройкой пороговых значений.

| Канал | Доступен для |

### Центр уведомлений

**Центр уведомлений** предоставляет историю всех уведомлений, отправленных из Account Management, включая события бюджета и мониторов затрат.

Чтобы открыть **Центр уведомлений**, выберите  > **Notification Center** на панели инструментов Account Management.

Таблица отображает события, обнаруженные в течение текущего периода обязательств, с самыми последними событиями сверху.
Таблица включает:

* **Date**: когда было обнаружено событие; используйте фильтр **Timeframe**, чтобы ограничить результаты по дате.
* **Message**: текст события, сгенерированный Dynatrace; используйте фильтр **Search**, чтобы отфильтровать результаты по конкретным текстовым строкам.
* **Event type**: фильтрация по конкретному типу события.
* **Levels**: выберите, чтобы просматривать события по уровню серьёзности.
* **Environments**: выберите отдельное имя окружения, чтобы отфильтровать события конкретного окружения, или выберите **Account**, чтобы отфильтровать события на уровне аккаунта.
* **Capability**: указывает функцию, к которой относится событие. Выберите конкретную функцию, чтобы отфильтровать таблицу, или выберите **DPS Cost Total**, чтобы просмотреть все прогнозные события.

### Уведомления по электронной почте

Можно настроить, когда отправляются уведомления по электронной почте и кто их получает.
Инструкции по настройке получателей электронной почты для каждого типа оповещений см. на соответствующей странице:

* [Уведомления по электронной почте мониторов затрат](/managed/manage-your-costs/control/cost-monitors#email-notifications "Узнайте, как использовать функцию Cost Monitors для прогнозирования и отслеживания событий затрат.")
* [Уведомления по электронной почте оповещений о бюджете](/managed/manage-your-costs/control/budgets#notification "Узнайте, как настроить бюджеты в Dynatrace.")

Уведомления отправляются с адреса `no-reply@dynatrace.com`.

Письма с уведомлениями содержат ссылки на Account Management.
Получателям нужно право доступа **View account**, чтобы открывать связанные страницы.

Для настройки параметров уведомлений пользователю Dynatrace нужно право доступа к аккаунту **View and manage account and billing information**.

### Уведомления API

Уведомления API доступны для событий мониторов затрат. Подробности см. в разделе [Notifications API - POST filter notifications](/managed/dynatrace-api/account-management-api/post-notifications "Список уведомлений для вашего аккаунта.").
