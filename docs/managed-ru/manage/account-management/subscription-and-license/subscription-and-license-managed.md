# Управление лицензией Dynatrace Managed

* Опубликовано 01 фев 2022 г.

Развёртывание Dynatrace Managed может включать несколько кластеров, каждый со своей лицензией. Каждая лицензия определяет продукты, которые можно использовать (продукты могут включать host units, host unit hours, Digital Experience Monitoring, Davis data units и Application security units). Управление аккаунтом предоставляет представление об использовании лицензий по всем кластерам (**Total license usage**) и почасовую/ежедневную информацию об использовании каждого кластера (**Usage details**).

Эта страница предоставлена исключительно в информационных целях. Условия предложения бесплатной пробной версии Dynatrace и/или вашей лицензии Dynatrace применяются к любому использованию продуктов или сервисов Dynatrace. Дополнительные сведения см. в разделе [Лицензирование Dynatrace](/managed/license "О Dynatrace Platform Subscription (DPS), модели лицензирования.").

## Общее использование лицензии

Раздел Total license usage показывает актуальную информацию о потреблении лицензии в реальном времени.

![Управление аккаунтом: Dynatrace Managed: License usage](https://dt-cdn.net/images/account-management-managed-license-usage-1145-bbcf3e666e.png)

Управление аккаунтом: Dynatrace Managed: License usage

Селектор позволяет выбрать и просмотреть использование лицензии для каждого кластера. Параметр **All** позволяет агрегировать использование лицензий по всем кластерам.

* Каждый лицензируемый продукт отображается в виде отдельной инфографики, показывающей, сколько каждого продукта было использовано.
* Инфографики продуктов отображают использование квоты лицензии в виде числа (например, 100 / 1k) и процента использования.

  Инфографики продуктов используют цвет для обозначения статуса:

  + Синий — использование ниже квоты
  + Жёлтый — квота превышена, и сейчас начисляются сверхлимитные расходы (если это разрешено)
  + Красный — достигнута максимальная квота, мониторинг отключён

  Наведите курсор на инфографику продукта для отображения детальной разбивки использования.

Таблица **Total environment usage** предоставляет текстовое представление использования лицензии по окружениям и продуктам.

## Сведения об использовании

Используйте диаграммы **Usage details** для анализа исторического потребления лицензии, выбирая нужные параметры **Cluster**, **Product**, **Resolution** и **Timeframe**.

**Resolution** позволяет просматривать данные о потреблении разными способами:

* `Daily` — ежедневное суммарное использование
* `Hourly` — почасовые накопленные итоги за выбранный день
* `Total` — накопленное использование для лицензий на основе потребления (host unit hours, Digital Experience Monitoring, Davis data units, Application security units)
* `Linear forecast` — прогноз использования на основе линейной экстраполяции данных за последние 30 дней (только для Digital Experience Monitoring и Davis data units)
* `Seasonal forecast` — прогноз суммарного использования на основе сезонного прогноза из данных за последние 365 дней (только для Digital Experience Monitoring и Davis data units)

Изменение параметров диаграммы обновляет таблицу **Cluster usage breakdown** ниже. Таблица отражает суммарное потребление по кластерам на основе выбранных кластера, продукта и временного периода.

Выберите ![Export](https://dt-cdn.net/images/dashboards-app-dashboard-export-5ed2168e10.svg "Export") **Export data** для загрузки данных диаграммы в формате CSV, что удобно для дополнительных табличных отчётов и агрегации.

## ЧаВо

Почему данные об использовании в Consumption usage details не совпадают с Total license usage?

* License usage предоставляет точное и актуальное измерение потреблённой лицензии и разграничивает бесплатное, платное и сверхлимитное использование.
* Consumption usage details предоставляет историческую разбивку общего потребления, включающего платные, бесплатные и сверхлимитные единицы, поэтому может не совпадать с license usage.

Когда следует использовать Linear forecast, а когда — Seasonal forecast?

* Если вы новый клиент, возможно, придётся полагаться на линейную модель до тех пор, пока не накопится история потребления за полный год. Линейная модель также точна, если данные за последние 30 дней отражают будущее использование.
* Если ваш бизнес носит более сезонный характер (например, розничные всплески в праздничные периоды) и у вас есть данные за год или более, сезонная модель может дать лучшую оценку будущего использования.
