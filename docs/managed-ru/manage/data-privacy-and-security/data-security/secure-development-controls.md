## Оценка внешних сервисов и библиотек

Команды безопасности проводят аудит безопасности всех внешних сторонних поставщиков и сервисов перед их использованием. Все сторонние библиотеки оцениваются по качеству, производительности, лицензированию и уязвимостям и требуют одобрения перед использованием.

![Оценка внешних сервисов и библиотек](https://cdn.bfldr.com/B686QPH3/as/pq9zwckn6b66vr36xc4nng6/Evaluation_of_external_services_and_libraries_-_Light_Mode?auto=webp&format=png&position=1)

## Рецензирование кода

Каждое изменение кода утверждается коллегой-разработчиком. Изменения в критически важных областях безопасности продукта должны дополнительно утверждаться специалистами по безопасности.

Изменения в основную кодовую ветку требуют pull request, который проходит многочисленные автоматизированные тесты, включая выбранный набор тестов статического анализа кода на безопасность.

![Рецензирование кода](https://cdn.bfldr.com/B686QPH3/as/rthtkcpfxn9bjx74mtf4328/Code_reviews_-_Light_Mode?auto=webp&format=png&position=1)

## Статический анализ кода

Статический анализ кода и статическое тестирование безопасности приложений (SAST) выполняются ежедневно. Правила и плагины активно поддерживаются командой качества кода Dynatrace, состоящей из разработчиков программного обеспечения и экспертов по безопасности.

Плагины включают предопределённые и самостоятельно разработанные правила обнаружения уязвимостей безопасности и ошибок.

![Статический анализ кода](https://cdn.bfldr.com/B686QPH3/as/36c74k58fvpfrtxbpzn8/Static_code_analysis_-_Light_Mode?auto=webp&format=png&position=1)

## Сканирование сторонних библиотек

Сторонние библиотеки централизованно управляются с помощью инструмента анализа состава программного обеспечения (SCA). Ежедневно выполняются сканирования, обнаруживаются уязвимости безопасности и риски лицензирования, создаются заявки на устранение.

## Подпись кода

Пакеты установщиков автоматически подписываются в конвейере сборки с использованием сертификатов подписи кода. Установщики Windows подписываются сертификатами подписи кода Extended Validation (EV).

Кроме того, при установке и обновлении автоматически выполняется проверка подписи.

Плагины и расширения, созданные Dynatrace, подписываются, и подпись проверяется при их активации на хостах. Любое изменение их содержимого делает подпись недействительной и предотвращает активацию.

![Подпись кода](https://cdn.bfldr.com/B686QPH3/as/6f6ttzwkwnjvr9wkp3qmxgq7/Code_Signing_-_Light_Mode?auto=webp&format=png&position=1)

## Отслеживание уязвимостей и ключевые показатели эффективности

Все проблемы и уязвимости безопасности отслеживаются в централизованной системе управления заявками, которая также используется другими командами для всех рабочих задач. Команды безопасности классифицируют и оценивают все уязвимости с использованием Common Vulnerability Scoring System (CVSS). Сроки устранения для каждого уровня серьёзности уязвимостей определены и непрерывно контролируются.

Централизованные панели мониторинга безопасности и ежеквартальные отчёты доступны всем командам. Для выявленных проблемных областей планируются и реализуются улучшения.

![Отслеживание уязвимостей и ключевые показатели эффективности](https://cdn.bfldr.com/B686QPH3/as/7m2pck8z487zpnckr53rfxv/Vulnerability_tracking_and_KPIs_-_Light_Mode?auto=webp&format=png&position=1)

## Программы обучения и адаптации сотрудников по безопасности

Все сотрудники Dynatrace обязаны посещать и успешно завершать ежегодные программы повышения осведомлённости о безопасности, охватывающие корпоративную и продуктовую политику безопасности.

Для новых сотрудников ежегодная программа повышения осведомлённости о безопасности и дополнительное обучение безопасности продукта являются частью программы адаптации.
