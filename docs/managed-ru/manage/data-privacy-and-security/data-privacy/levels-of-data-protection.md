# Уровни защиты данных

* Чтение 2 мин
* Updated on Dec 13, 2023

В зависимости от настроек окружения вы можете собирать данные, в отношении которых у вас есть юридические обязательства, обусловленные законами, стандартами и нормативными актами о защите данных, такими как GDPR, CCPA, HIPAA, LGPD, PCI-DSS и другими. Для решения подобных ситуаций Dynatrace предлагает несколько уровней защиты, помогающих точно настроить потоки данных.

[**Маскирование при захвате**](/managed/manage/data-privacy-and-security/data-privacy/levels-of-data-protection#at-capture "Узнайте, как Dynatrace защищает данные конечных пользователей, применяя ситуативные уровни защиты.")[**Маскирование при хранении**](/managed/manage/data-privacy-and-security/data-privacy/levels-of-data-protection#at-storage "Узнайте, как Dynatrace защищает данные конечных пользователей, применяя ситуативные уровни защиты.")[**Маскирование при отображении**](/managed/manage/data-privacy-and-security/data-privacy/levels-of-data-protection#at-display "Узнайте, как Dynatrace защищает данные конечных пользователей, применяя ситуативные уровни защиты.")

Для маскирования при захвате и маскирования при хранении любые связанные функции маскирования в Dynatrace применяются только после их включения; такое маскирование не применяется ретроактивно.

## Защита персональных данных путём отказа от их захвата (маскирование при захвате)

При маскировании на этапе захвата данные маскируются при первом контакте с Dynatrace. Исходные данные не покидают мониторируемое окружение.

Dynatrace может автоматически маскировать чувствительные точки данных при первом контакте с технологией Dynatrace. Это происходит внутри приложения, мониторируемого процесса или браузера, поэтому точки данных заменяются заполнителем до отправки в кластер Dynatrace. Для таких заполнителей используются звёздочки (`*`). Например, адрес электронной почты `john.smith@domain.com` преобразуется в `*********` и никогда не отправляется и не хранится в исходном виде.

Используйте следующие параметры для управления маскированием при захвате.

* [Маскирование на стороне OneAgent](/managed/manage/data-privacy-and-security/configuration/configure-global-privacy-settings#oneagent-side-masking "Узнайте, как настроить маскирование конфиденциальности данных для IP-адресов, геолокации и имён действий пользователей.")
* [Маскирование логов](/managed/manage/data-privacy-and-security/configuration/configure-global-privacy-settings#log-masking "Узнайте, как настроить маскирование конфиденциальности данных для IP-адресов, геолокации и имён действий пользователей.")
* [Маскирование IP-адресов конечных пользователей](/managed/manage/data-privacy-and-security/configuration/configure-global-privacy-settings#mask-ip-and-gps "Узнайте, как настроить маскирование конфиденциальности данных для IP-адресов, геолокации и имён действий пользователей.")
* [Маскирование действий пользователей](/managed/manage/data-privacy-and-security/configuration/configure-global-privacy-settings#mask-user-actions "Узнайте, как настроить маскирование конфиденциальности данных для IP-адресов, геолокации и имён действий пользователей.")

## Защита персональных данных путём отказа от их хранения (маскирование при хранении)

При маскировании на этапе хранения данные отправляются на сервер Dynatrace для оптимального анализа и маскируются перед сохранением.

Используйте следующие параметры для управления маскированием при хранении.

* [Маскирование персональных данных в URI](/managed/manage/data-privacy-and-security/configuration/configure-global-privacy-settings#mask-uris "Узнайте, как настроить маскирование конфиденциальности данных для IP-адресов, геолокации и имён действий пользователей.")

## Защита персональных данных путём ограничения их отображения (маскирование при отображении)

При маскировании на этапе отображения данные хранятся в исходном виде, но доступны только выбранным вами пользователям.

Ознакомьтесь со следующей информацией, чтобы узнать, как настроить маскирование при отображении.

* [Ограниченный доступ к просмотру персональных данных](/managed/manage/data-privacy-and-security/configuration/configure-global-privacy-settings#restrict-view-access "Узнайте, как настроить маскирование конфиденциальности данных для IP-адресов, геолокации и имён действий пользователей.")
* [Конфиденциальные атрибуты запросов](/managed/manage/data-privacy-and-security/configuration/configure-global-privacy-settings#conf-attribute "Узнайте, как настроить маскирование конфиденциальности данных для IP-адресов, геолокации и имён действий пользователей.")
