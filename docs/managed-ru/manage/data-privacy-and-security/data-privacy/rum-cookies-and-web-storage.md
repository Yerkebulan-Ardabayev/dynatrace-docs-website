# Куки и клиентское хранилище для RUM и Session Replay

# Куки и клиентское хранилище для RUM и Session Replay

* Справочник
* Updated on May 04, 2026

Dynatrace Real User Monitoring (RUM) использует HTTP-заголовки, куки, веб-хранилище и IndexedDB. Используемые заголовки перечислены в разделе [Ограничения брандмауэра для RUM](/managed/observe/digital-experience/web-applications/initial-setup/firewall-constraints-for-rum "Узнайте, как обеспечить прохождение данных Real User Monitoring через ваш брандмауэр."). На этой странице описываются используемые куки, веб-хранилище и записи IndexedDB.

Куки, веб-хранилище и IndexedDB используются для следующих целей:

* Агрегация пользовательских сессий
* Идентификация повторных пользователей
* Сохранение состояния функций RUM, таких как [управление стоимостью и трафиком](/managed/observe/digital-experience/web-applications/additional-configuration/configure-cost-and-traffic-control-web "Используйте параметр управления стоимостью и трафиком в Dynatrace для сокращения использования сессий для веб-приложений.")
* Корреляция действий пользователей с соответствующими серверными трассами
* Обеспечение согласованности записи Session Replay между страницами
* Кэширование данных о захвате ресурсов Session Replay для исключения избыточных загрузок

Данные, хранящиеся в куки, веб-хранилище и IndexedDB, не содержат персональных или конфиденциальных данных. Они состоят из случайных идентификаторов, временных меток, булевых флагов, конфигурации, хешей и кэшированных ресурсов.

## Куки

В следующей таблице приведён обзор куки, используемых в RUM и Session Replay. Все они являются куки первой стороны.

`<суффикс>`, используемый в таблице, описан в разделе [Суффикс для имён куки и ключей веб-хранилища](#cookie-and-web-storage-suffix).

| Куки | Структура | Истекает | Макс. размер | Назначение |
| --- | --- | --- | --- | --- |
| `dtCookie<суффикс>` | Состояние сессии v4: v\_4\_key1\_value1\_key2\_value2\_keyN\_valueN  Возможные ключи:  * srv * sn * mvisitor * msn * perc * ol * mul * app:<appID>  Пример v4: v\_4\_srv\_7\_sn\_4D3133F359A76AB05AAF39691696858A | Сессия | Ограничений нет, обычно менее 100 байт | Отслеживает посещение в нескольких запросах. |
| `dtPC<суффикс>` | <serverID>$<randomValue>\_<currentMillis>v<randomValue>e<eventCount> | Сессия | 58 байт | Требуется для маршрутизации маяков RUM; включает идентификатор сессии для агрегации пользовательских сессий. |
| `dtSa<суффикс>` | <Имя действия в кодировке URL> | Сессия | Максимальное количество символов в URL | Служит промежуточным хранилищем для действий, охватывающих несколько страниц. Используется для сохранения имён действий пользователей, таких как `Click on Login`, между разными страницами. Это необходимо, поскольку загрузка страницы вызывает перезапуск JavaScript-кода, и вся контекстная информация должна храниться в куки. |
| `dtValidationCookie<суффикс>` | Строка `dTValidationCookieValue`. | Удаляется через несколько миллисекунд; срок действия не устанавливается | Длина строки `dTValidationCookieValue`, то есть `23` | Используется для определения домена верхнего уровня. |
| `rxVisitor<суффикс>` | <visitorID> | Сессия или постоянный при включённом [отслеживании пользователей](/managed/observe/digital-experience/web-applications/additional-configuration/configure-real-user-monitoring-according-to-gdpr#user-tracking "Узнайте о параметрах конфиденциальности Dynatrace для обеспечения соответствия ваших веб-приложений требованиям по защите данных.") | 45 байт | Содержит идентификатор посетителя для корреляции сессий. |
| `rxvt<суффикс>` | <timestamp>|<timestamp> | Сессия | 27 байт | Хранит тайм-аут сессии. |
| `dtsrVID<суффикс>` [1](#fn-1-1-def) | <viewID> | Сессия | 20 байт | При включённом Session Replay `dtsrVID` хранит идентификатор текущего записываемого представления. |
| `dtSR<суффикс>` [2](#fn-1-2-def) | Формат v4  Возможные ключи:  * vwid * rfnsr * vid * st  Пример: v\_4\_rfnsr\_150\_vid\_CFEOMVJRAQLDWHDQPLMORLTRGUFRLFUM-0\_st\_1\_vwid\_1764935792511 | Сессия | 81 байт | При включённом Session Replay хранит значения, необходимые для обеспечения согласованности записи между страницами. |

1

Куки `dtsrVID` существует с версии RUM JavaScript 1.325+ по версию RUM JavaScript 1.333.

2

Дополнительный куки `dtSR` доступен с версии RUM JavaScript 1.335+.

### Настройка атрибутов куки

Dynatrace RUM не поддерживает атрибут `HttpOnly`. Поскольку куки с `HttpOnly` недоступны JavaScript, RUM JavaScript не может их читать или изменять. Убедитесь, что ваша инфраструктура не добавляет атрибут `HttpOnly`, так как это нарушит работу мониторинга.

Если политика безопасности вашей компании требует атрибутов куки `Secure` и `SameSite`, необходимо настроить RUM для их установки, поскольку они не устанавливаются по умолчанию; см. раздел [Настройка атрибутов куки RUM](/managed/observe/digital-experience/web-applications/additional-configuration/configure-cookie-attributes#data-privacy-relevant-attributes "Узнайте, какие атрибуты куки RUM можно настраивать и как это делать.").

## Веб-хранилище

RUM и Session Replay используют как `sessionStorage`, так и `localStorage`. В следующей таблице перечислены возможные записи. Некоторые из них служат резервными копиями куки RUM и Session Replay, поскольку браузеры могут удалять куки при наличии большого их количества.

| Ключ | Хранилище | Назначение |
| --- | --- | --- |
| `rxVisitor<суффикс>` | Хранится в `localStorage` при включённом [отслеживании пользователей](/managed/observe/digital-experience/web-applications/additional-configuration/configure-real-user-monitoring-according-to-gdpr#user-tracking "Узнайте о параметрах конфиденциальности Dynatrace для обеспечения соответствия ваших веб-приложений требованиям по защите данных."), в противном случае — в `sessionStorage`. | Резервная копия куки `rxVisitor<суффикс>`. |
| `rxvisitid<суффикс>` | `sessionStorage` | Резервная копия идентификатора сессии из куки `dtPC<суффикс>`. |
| `rxvt<суффикс>` | `sessionStorage` | Резервная копия куки `rxvt<суффикс>`. |
| `dtCookie<суффикс>` | `sessionStorage` | Резервная копия куки `dtCookie<суффикс>`. |
| `dtSa<суффикс>` | `sessionStorage` | Резервная копия куки `dtSa<суффикс>`. |
| `ruxitagentjs_<appID или пусто>_Store` | `localStorage` | Хранит последний ответ маяка, содержащий конфигурацию RUM JavaScript. |
| `dt-pVO` | `localStorage` | Указывает, что постоянные значения были удалены с помощью метода API [`dtrum.disablePersistentValues`](https://docs.dynatrace.com/javascriptapi/doc/types/dtrum.html#disablepersistentvalues). |
| `dtsrNOSR` | `localStorage`. | Хранит серьёзность последнего сообщения о причине отсутствия Session Replay и `visitId`. Существует до версии RUM JavaScript 1.333. |
| `dtSR<суффикс>` | По умолчанию в `localStorage`, опционально в куки. | Хранит значения, необходимые для обеспечения согласованности записи между страницами при включённом Session Replay. Существует с версии RUM JavaScript 1.335. |
| `dtsrTID` | `sessionStorage` | Хранит идентификатор текущей вкладки для Session Replay. |
| `dtsrTIDrSt` | `sessionStorage` | Булев флаг для Session Replay, указывающий, было ли окно открыто программным способом недавно. |

## IndexedDB

Session Replay использует IndexedDB для сохранения состояния захвата ресурсов между сессиями. В следующей таблице перечислены возможные записи.

| Имя хранилища | Назначение |
| --- | --- |
| `dT_store#global` | При включённом Session Replay `dT_store#global` хранит хеши содержимого CSS-ресурсов, уже захваченных Dynatrace. Используется для исключения повторной загрузки неизменённых ресурсов в будущих сессиях. |
| `dT_store#visit` | При включённом Session Replay `dT_store#visit` буферизирует захваченное содержимое ресурсов и метаданные, ожидающие передачи, а также отслеживает ресурсы, которые не удалось получить в рамках текущего посещения. |

## Суффикс для имён куки и ключей веб-хранилища

В окружениях, созданных в Dynatrace версии 1.294+, к именам куки и ключам определённых записей веб-хранилища добавляется суффикс. В таблицах выше этот суффикс обозначен как `<суффикс>`. Он предотвращает конфликты при использовании нескольких окружений Dynatrace с пересекающимися [доменами куки](/managed/observe/digital-experience/web-applications/additional-configuration/configure-cookie-attributes#cookie-domain "Узнайте, какие атрибуты куки RUM можно настраивать и как это делать.").

Если вам нужны полные имена куки для настройки брандмауэров и другой инфраструктуры, вы можете получить их с помощью конечной точки API [Имена куки RUM API — GET cookie names](/managed/dynatrace-api/environment-api/rum/rum-cookie-names-get-cookie-names "Просмотр полных имён куки RUM."). Чтобы найти их в интерфейсе:

1. Перейдите в **Web**.
2. Выберите приложение, которое необходимо настроить.
3. В правом верхнем углу страницы обзора приложения выберите **More** (**…**) > **Edit**.
4. В настройках приложения выберите **Injection** > **Cookie**.
5. Имена куки отображаются в верхней части страницы.

## Предоставление пользователям возможности согласия с использованием куки, веб-хранилища и IndexedDB

Куки, веб-хранилище и IndexedDB Dynatrace необходимы для работы Real User Monitoring. По умолчанию Dynatrace создаёт их автоматически. Для защиты конфиденциальности конечных пользователей вы можете предоставить им возможность принять или отклонить куки, веб-хранилище и мониторинг в целом. Это реализуется через [режим opt-in](/managed/observe/digital-experience/web-applications/additional-configuration/configure-real-user-monitoring-according-to-gdpr#user-opt-in-mode-gdpr "Узнайте о параметрах конфиденциальности Dynatrace для обеспечения соответствия ваших веб-приложений требованиям по защите данных.").

При включённом режиме opt-in RUM отключён по умолчанию. В результате Dynatrace не устанавливает куки, не использует веб-хранилище и не записывает в IndexedDB. После того как конечный пользователь принимает вашу политику конфиденциальности данных, ваше приложение должно вызвать `dtrum.enable()` через JavaScript API для начала сбора данных. Dynatrace создаёт куки и записывает в веб-хранилище и IndexedDB только после вызова этого метода.

Подробнее о включении режима opt-in см. в разделе [Настройка параметров конфиденциальности данных для веб-приложений](/managed/observe/digital-experience/web-applications/additional-configuration/configure-real-user-monitoring-according-to-gdpr#user-opt-in-mode-gdpr "Узнайте о параметрах конфиденциальности Dynatrace для обеспечения соответствия ваших веб-приложений требованиям по защите данных.").

Если пользователи не дают согласия, Real User Monitoring остаётся отключённым.
