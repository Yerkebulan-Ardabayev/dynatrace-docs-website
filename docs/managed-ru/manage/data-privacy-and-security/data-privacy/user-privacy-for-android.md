# Руководство по безопасности данных для Android

* Чтение 5 мин
* Updated on Sep 15, 2025

Начиная с 20 июля 2022 года Google обязывает предоставлять пользователям информацию о том, как ваше мобильное приложение собирает, защищает и передаёт их данные. Это также касается данных, собираемых сторонними партнёрами, такими как Dynatrace.

Данная страница поможет вам заполнить [форму Data safety](https://support.google.com/googleplay/android-developer/answer/10787469#complete_form_steps) в Google Play Console. Вопросы и типы данных соответствуют форме Data safety, однако ответы отражают исходное состояние по умолчанию.

OneAgent может собирать дополнительные данные через вашу [ручную инструментацию](/managed/observe/digital-experience/mobile-applications/instrument-android-app/instrumentation-via-oneagent-sdk "Узнайте, что такое OneAgent SDK для Android."). Если вы настроили инструментацию для сбора дополнительных данных, обязательно отразите это в разделе **Data safety** в Google Play Console.

Чтобы заполнить форму Data safety:

1. Войдите в Google Play Console и выберите нужное мобильное приложение.
2. В меню слева перейдите в **Policy** > **App content**.
3. В разделе **Data safety** выберите **Manage**.
4. Заполните форму Data safety, используя информацию на этой странице.

## Сбор и безопасность данных

Используйте приведённую ниже таблицу для ответа на вопросы этого раздела.

| Вопрос | Ответ | Примечание |
| --- | --- | --- |
| Собирает или передаёт ли ваше приложение какие-либо из требуемых типов пользовательских данных? | Применимо — Да |  |
| Все ли пользовательские данные, собираемые вашим приложением, зашифрованы при передаче? | Применимо — Да |  |
| Какие методы создания аккаунта поддерживает ваше приложение? | Применимо — Моё приложение не позволяет пользователям создавать аккаунт | Dynatrace не предоставляет пользовательских аккаунтов |
| Могут ли пользователи войти в ваше приложение с аккаунтами, созданными вне приложения? | Не применимо — Нет |  |
| Предоставляете ли вы пользователям возможность запросить удаление их данных? | Не применимо — Нет |  |

## Типы данных

Выберите все типы данных, которые ваше мобильное приложение собирает или передаёт.

В таблице ниже указаны все данные, которые OneAgent собирает по умолчанию.

| Категория | Тип данных | Собирается по умолчанию? | Примечание |
| --- | --- | --- | --- |
| Местоположение |  | Не применимо — Нет | Выберите **Approximate location** (приблизительное местоположение), если вы включили [мониторинг местоположения](/managed/observe/digital-experience/mobile-applications/instrument-android-app/instrumentation-via-plugin/monitoring-capabilities#location-monitoring "Настройте плагин Dynatrace Android Gradle для управления возможностями мониторинга OneAgent.") и ваше мобильное приложение имеет разрешение на использование геолокационных данных устройства.  Dynatrace может захватывать [приблизительное местоположение](/managed/manage/data-privacy-and-security/configuration/configure-global-privacy-settings#mask-ip-and-gps "Узнайте, как настроить маскирование конфиденциальности данных для IP-адресов, геолокации и имён действий пользователей."), но никогда не собирает точное местоположение. |
| Персональные данные |  | Не применимо — Нет | Выберите **User IDs** или другие типы персональных данных, если вы собираете персональную информацию ваших пользователей через [разметку пользователей](/managed/observe/digital-experience/mobile-applications/instrument-android-app/instrumentation-via-oneagent-sdk/oneagent-sdk-for-android#tag-specific-users "Узнайте, как обогатить мониторинг пользовательского опыта в Android с помощью OneAgent SDK."). |
| Финансовые данные |  | Не применимо — Нет |  |
| Здоровье и фитнес |  | Не применимо — Нет |  |
| Сообщения |  | Не применимо — Нет |  |
| Фотографии и видео |  | Не применимо — Нет |  |
| Аудиофайлы |  | Не применимо — Нет |  |
| Файлы и документы |  | Не применимо — Нет |  |
| Календарь |  | Не применимо — Нет |  |
| Контакты |  | Не применимо — Нет |  |
| Активность приложения | Взаимодействия с приложением | Применимо — Да | Dynatrace фиксирует взаимодействия пользователей, например запуски приложения или касания, и сообщает о них как о действиях пользователей.  Сведения о настройке или отключении этих функций см. в разделах [Мониторинг действий пользователей](/managed/observe/digital-experience/mobile-applications/instrument-android-app/instrumentation-via-plugin/monitoring-capabilities#action-monitoring "Настройте плагин Dynatrace Android Gradle для управления возможностями мониторинга OneAgent.") и [Мониторинг жизненного цикла](/managed/observe/digital-experience/mobile-applications/instrument-android-app/instrumentation-via-plugin/monitoring-capabilities#lifecycle-monitoring "Настройте плагин Dynatrace Android Gradle для управления возможностями мониторинга OneAgent."). |
| Просмотр веб-страниц |  | Не применимо — Нет |  |
| Информация и производительность приложения | Журналы сбоев | Применимо — Да | Dynatrace собирает информацию о [сбоях](/managed/observe/digital-experience/rum-concepts/user-and-error-events#crash "Узнайте о событиях пользователей и ошибках и типах событий, собираемых Dynatrace."). Если вы также включили [Session Replay при сбоях](/managed/observe/digital-experience/session-replay/session-replay-android "Настройка Session Replay для Android-приложений для отслеживания действий пользователей."), Dynatrace собирает несколько снимков экрана до сбоя.  Снимите флажок **Crash logs**, если вы отключили [отчётность о сбоях](/managed/observe/digital-experience/mobile-applications/instrument-android-app/instrumentation-via-plugin/monitoring-capabilities#crash-reporting "Настройте плагин Dynatrace Android Gradle для управления возможностями мониторинга OneAgent."). |
| Информация и производительность приложения | Диагностика | Применимо — Да | Dynatrace собирает некоторую информацию о производительности, например время загрузки приложений и операций или время выполнения веб-запросов. |
| Информация и производительность приложения | Другие данные производительности приложения | Применимо — Да | Dynatrace собирает некоторые данные о производительности приложения, например уровень заряда батареи устройства или разрешение экрана. В Google Play Console нельзя указать, какие именно данные о производительности приложения собираются. Данные производительности приложения, собираемые Dynatrace: * Уровень заряда батареи * Встроенная оперативная память * Свободная оперативная память * Модель устройства * Тип процессора * Название оператора * Тип сетевого подключения (например, мобильная сеть, Wi-Fi или LAN) * Технология сети (например, 2G, 3G, 4G, 5G, 802.11x) * Разрешение экрана * Ориентация (вертикальная или горизонтальная) * Версия приложения * Название приложения * Язык пользователя * Версия Android * Новый пользователь (для первой сессии) |
| Идентификаторы устройства или другие |  | Не применимо — Нет | Dynatrace не захватывает и не сообщает никаких идентификаторов устройства. Рекомендуется избегать передачи идентификаторов устройства через ручную инструментацию. |

## Использование и обработка данных

Также необходимо предоставить информацию об использовании и обработке данных для каждого типа данных, выбранного в разделе **Data types**. Выберите **Start** для продолжения.

Используйте приведённую ниже таблицу для всех типов данных, захватываемых OneAgent.

| Вопрос | Ответ | Примечание |
| --- | --- | --- |
| Эти данные собираются, передаются или и то, и другое? | Собираются |  |
| Эти данные обрабатываются временно? | Не применимо — Нет |  |
| Эти данные обязательны для вашего приложения, или пользователи могут выбрать, собирать ли их? | Сбор данных обязателен (пользователи не могут его отключить) | Выберите **Users can choose whether this data is collected** (Пользователи могут выбрать, собирать ли эти данные), если вы включили [opt-in пользователей](/managed/observe/digital-experience/mobile-applications/additional-configuration/configure-rum-privacy-mobile#opt-in-mode-mobile "Воспользуйтесь параметрами конфиденциальности Dynatrace для обеспечения соответствия ваших мобильных приложений требованиям по защите данных.") для вашего мобильного приложения. |
| Для чего собираются эти пользовательские данные? | Аналитика |  |
