# Системные уведомления

* Пояснение
* Чтение 2 мин
* Опубликовано 23 фев 2022 г.

Таблица **Notifications** отображает уведомления об обновлениях системы Dynatrace.

## Отображение уведомлений

Чтобы открыть страницу **Notifications**, перейдите в **Notifications**.

Страница **Notifications** доступна только системным администраторам. Если вы не видите **Notifications** в Dynatrace, у вас нет прав системного администратора.

## Фильтрация таблицы

Для фильтрации таблицы задайте фильтр в строке фильтра:

* `Severity` можно установить в `SEVERE`, `WARNING` или `INFO`.
* `Message` — строка поиска по содержимому сообщения. Например, введите `agent` для отображения обновлений OneAgent и других уведомлений, содержащих `agent` в сообщении.

## Сортировка таблицы

Для сортировки таблицы выберите соответствующий заголовок столбца. Доступна сортировка по:

* **Received** — дата и время получения уведомления
* **Severity** — уровень серьёзности уведомления (`SEVERE`, `WARNING` или `INFO`)

## Устранение проблем

Некоторые уведомления содержат прямую ссылку на другую страницу в веб-интерфейсе Dynatrace. Выберите ссылку, чтобы перейти на страницу и проверить или изменить соответствующие настройки.

## Очистка уведомлений

Некоторые уведомления истекают и удаляются из списка, когда система восстанавливается до нормального состояния. Для всех уведомлений нет единого срока истечения, поэтому если система не восстанавливается, уведомление не будет удалено из списка до тех пор, пока вы не выберете ![Remove](https://dt-cdn.net/images/remove-icon-105c6a04c2.svg "Remove") в столбце **Clear**.

* Для очистки одного уведомления выберите ![Remove](https://dt-cdn.net/images/remove-icon-105c6a04c2.svg "Remove") в столбце **Clear**.
* Для очистки нескольких уведомлений установите соответствующие флажки и выберите **Clear selected**.

### Примечания

* Уведомления очищаются на уровне пользователя: если страницей **Notifications** пользуются несколько человек, очистка ваших уведомлений не влияет на то, что видят другие.
* Если вы очистите уведомление, но условия, породившие его, сохранятся или повторятся, уведомление (или его новая копия) может появиться снова.
