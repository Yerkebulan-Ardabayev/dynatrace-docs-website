# Применение и использование зон управления

* How-to guide
* 4-min read
* Updated on Aug 20, 2025

После [настройки зон управления](/managed/manage/identity-access-management/permission-management/management-zones/set-up-management-zones "Создание зон управления и назначение прав доступа к ним.") можно [назначать их группам пользователей](#apply) и предоставлять необходимые разрешения уровня зоны управления. Члены этих групп смогут [фильтровать окружения по назначенным им зонам управления](#filter). Зоны управления также можно [использовать для настройки окон обслуживания](#mw).

## Применение зоны управления

Следующие шаги описывают процесс назначения зоны управления пользователю через настройки группы.

1. [Настройте зону управления](/managed/manage/identity-access-management/permission-management/management-zones/set-up-management-zones "Создание зон управления и назначение прав доступа к ним.") (**Settings** > **Preferences** > **Management zones**) или проверьте имя зоны управления и [правила](/managed/manage/identity-access-management/permission-management/management-zones/management-zone-rules "Определение правил для ограничения объектов, доступных в зоне управления."), определяющие, какие объекты и данные доступны в зоне.
2. В Cluster Management Console перейдите в **User authentication** > **User groups**.
3. Создайте или выберите группу, в которую нужно добавить пользователя.

   Назначьте группе необходимые [разрешения](/managed/manage/identity-access-management/user-and-group-management/user-groups-and-permissions "Узнайте о поддерживаемых разрешениях и политиках, о том, как назначать их группам и управлять пользователями и группами.") уровня окружения и зоны управления для объектов, доступных в зоне. Например, если пользователю должна быть разрешена возможность создавать или редактировать синтетические мониторы в зоне управления, необходимо предоставить разрешение **Manage monitoring settings** на уровне зоны управления или окружения.
4. Убедитесь, что пользователь имеет статус **ACTIVE**.
5. Назначьте пользователю группу (с доступом к зоне управления) из предыдущего шага.

После входа в окружение пользователь может использовать кнопку **Filter** в строке меню для доступа к назначенным им зонам управления.

## Фильтрация по зоне управления

Зоны управления доступны в виде фильтров (через кнопку **Filter** в строке меню) всем пользователям, назначенным в эти зоны — выберите нужную зону из списка.

![Фильтр по зонам управления](https://dt-cdn.net/images/mz-filter-2560-89a3233cbf.png "Фильтр по зонам управления")

Фильтры зон управления применяются ко всем представлениям, отображающим несколько объектов, включая отдельные [дашборды](/managed/analyze-explore-automate/dashboards-classic "Создание, управление и использование Dynatrace Dashboards Classic."), страницы со списками (например, хостов, групп процессов, сервисов, синтетических мониторов, приложений и [SLO](/managed/deliver/service-level-objectives-classic/configure-and-monitor-slo#mz "Создание, настройка и мониторинг сервисных уровней в Dynatrace.")), [Logs](/managed/analyze-explore-automate/log-monitoring/analyze-log-data/management-zones-and-log-monitoring "Узнайте, как принятые данные журналов назначаются зонам управления."), [Smartscape](/managed/analyze-explore-automate/smartscape-classic "Узнайте, как Smartscape визуализирует все объекты и зависимости в вашем окружении.") и страницу Technologies. Это позволяет каждому пользователю видеть только те объекты, которые имеют к нему отношение.

Обратите внимание, что при создании объектов и их назначении тегами через API может наблюдаться задержка перед появлением объектов в зонах управления. Советы по оптимизации тегирования и ускорению назначения зон управления см. в разделе [Лучшие практики масштабирования правил тегирования и зон управления](/managed/manage/tags-and-metadata/basic-concepts/best-practice-tagging-at-scale "Оптимизация правил автоматического тегирования и зон управления.").

В рамках зоны управления доступны только те объекты, к которым у вас есть доступ в Dynatrace. На страницах, к объектам которых у вас нет доступа, данные не отображаются.

Вы можете просматривать [**Dashboards**](/managed/analyze-explore-automate/dashboards-classic "Создание, управление и использование Dynatrace Dashboards Classic."), созданные другими пользователями, и создавать собственные для доступных вам объектов.

Обратите внимание, что для просмотра данных [Real User Monitoring (RUM)](/managed/observe/digital-experience/rum-concepts/rum-overview "Узнайте о Real User Monitoring, ключевых метриках производительности, мониторинге мобильных приложений и т.д.") требуется [разрешение уровня окружения](/managed/manage/identity-access-management/permission-management/role-based-permissions#permissions "Разрешения на основе ролей") **View environment**.

## Зоны управления и окна обслуживания

Можно [определять окна обслуживания с фильтрацией по зонам управления](/managed/analyze-explore-automate/notifications-and-alerting/maintenance-windows/define-maintenance-window#scope "Создание окон обслуживания и определение их области действия.") — объекты, соответствующие зонам управления, считаются находящимися на техническом обслуживании.
