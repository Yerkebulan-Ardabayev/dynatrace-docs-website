# Справочник политик IAM

* Reference
* 1-min read
* Published Mar 25, 2021

Ниже приведён полный справочник разрешений IAM и соответствующих условий, применимых к сервисам Dynatrace. Используйте его при необходимости определить политики доступа на основе детального набора разрешений и условий, применяемых для каждого сервиса.

## environment

Разрешения пользователей окружения и зоны управления. Подробнее см. в разделе [Миграция разрешений на основе ролей в Dynatrace IAM](https://dt-url.net/3s23539).

Разрешения ролей IAM работают так же, как и классические роли: разрешение `environment:roles:viewer` является частью любого другого разрешения роли. Например, политика, предоставляющая разрешение `environment:roles:manage-settings`, также разрешает пользователю доступ к веб-интерфейсу.

### environment:roles:viewer

Предоставляет пользователю разрешение **Access environment**.

#### Условия:

* `environment:management-zone` — строка, однозначно идентифицирующая зону управления. Применяет разрешение на уровне зоны управления для указанной зоны.
  Операторы: `IN`, `startsWith`, `NOT startsWith`, `=`, `!=`, `MATCH`

### environment:roles:manage-settings

### environment:roles:agent-install

## extensions

Сервис расширений

### extensions:definitions:read

Предоставляет разрешение на чтение конфигураций расширений и окружений

#### Условия:

* `extensions:extension-name` — строка, однозначно идентифицирующая отдельное расширение
  Операторы: `IN`, `NOT IN`, `startsWith`, `NOT startsWith`, `!=`, `=`

## settings

Сервис настроек

### settings:objects:read

Разрешает чтение объектов настроек, принадлежащих схеме

#### Условия:

* `settings:schemaId` — строка, однозначно идентифицирующая отдельную схему настроек. Идентификатор схемы можно найти через выделенную конечную точку схем в Dynatrace Environment API или в информационном блоке экрана настроек. Условие выполняется, если свойство schemaId объекта совпадает.
  Операторы: `IN`, `=`, `!=`, `startsWith`, `NOT startsWith`
* `settings:schemaGroup` — группа схем, позволяющая обращаться к нескольким отдельным схемам одновременно. Группу схемы можно найти через выделенную конечную точку схем в Dynatrace Environment API или в информационном блоке экрана настроек. Условие выполняется, если у схемы объекта есть совпадающее свойство schemaGroup.
  Операторы: `IN`, `=`
* `settings:entity.hostGroup` — атрибут группы хостов объекта, для которого хранится настройка. Полезно для предоставления доступа к областям настроек всех хостов, принадлежащих одной группе.
  Операторы: `IN`, `=`, `!=`
* `settings:scope` — точный идентификатор области объекта настроек. Позволяет предоставить доступ к области отдельного хоста. В этом случае область равна идентификатору объекта, например `HOST-48B8F52F33098830`.
  Операторы: `IN`, `=`, `!=`, `startsWith`, `NOT startsWith`
* `environment:management-zone` — имя зоны управления. Применимо к объектам настроек, допустимым в области объекта, который может быть сопоставлен с зоной управления, или к объектам настроек схем `builtin:alerting.maintenance-window`, `builtin:alerting.profile`, `builtin:anomaly-detection.metric-events`, `builtin:monitoring.slo` и `builtin:problem.notifications`.
  Операторы: `IN`, `=`, `startsWith`, `MATCH`

### settings:objects:write

Разрешает запись объектов настроек, принадлежащих схеме

#### Условия:

* `settings:schemaId` — строка, однозначно идентифицирующая отдельную схему настроек. Идентификатор схемы можно найти через выделенную конечную точку схем в Dynatrace Environment API или в информационном блоке экрана настроек. Условие выполняется, если свойство schemaId объекта совпадает.
  Операторы: `IN`, `=`, `!=`, `startsWith`, `NOT startsWith`
* `settings:schemaGroup` — группа схем, позволяющая обращаться к нескольким отдельным схемам одновременно.
  Операторы: `IN`, `=`
* `settings:entity.hostGroup` — атрибут группы хостов объекта, для которого хранится настройка.
  Операторы: `IN`, `=`, `!=`
* `settings:scope` — точный идентификатор области объекта настроек.
  Операторы: `IN`, `=`, `!=`, `startsWith`, `NOT startsWith`
* `environment:management-zone` — имя зоны управления.
  Операторы: `IN`, `=`, `startsWith`, `MATCH`

### settings:schemas:read

Разрешает чтение схем настроек

#### Условия:

* `settings:schemaId` — строка, однозначно идентифицирующая отдельную схему настроек.
