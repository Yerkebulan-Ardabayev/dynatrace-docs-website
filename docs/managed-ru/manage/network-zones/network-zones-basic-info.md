# Начало работы с сетевыми зонами

* Руководство
* Чтение 3 мин
* Обновлено 10 дек 2025 г.

Сетевые зоны — это объекты Dynatrace, представляющие вашу сетевую структуру. Они помогают эффективно маршрутизировать трафик, избегая ненужного трафика между центрами обработки данных и сетевыми регионами.

### Без сетевых зон

![Маршрутизация — без сетевых зон](https://dt-cdn.net/images/routing-1-1-800-a92215b7e5.png)

Маршрутизация — без сетевых зон

Все ваши OneAgent имеют доступ ко всем ActiveGate, которые маршрутизируют трафик в Dynatrace. Если вы введёте межсетевой экран, который помешает некоторым OneAgent достигать ActiveGate, которые они использовали, это может привести к ненужному трафику.

Кроме того, поскольку каждый OneAgent может подключаться к любому ActiveGate, возможна ситуация, когда OneAgent подключается к ActiveGate из другого региона, и трафику приходится идти вокруг всего мира.

### С сетевыми зонами

![Маршрутизация — с сетевыми зонами](https://dt-cdn.net/images/routing-2-1-800-dcdfba1450.png)

Маршрутизация — с сетевыми зонами

Сетевые зоны предотвращают неэффективные соединения, определяя, какой ActiveGate должен использовать каждый OneAgent. Вы по-прежнему можете использовать ActiveGate из других зон в качестве [резервного варианта](/managed/manage/network-zones/network-zones-basic-info#alternative "Узнайте, как начать работу с сетевыми зонами.") на случай недоступности основных ActiveGate.

### Развёртывание сетевых зон

Развёртывание сетевых зон зависит от того, начинаете ли вы новое развёртывание Dynatrace (сценарий новой установки) или добавляете сетевые зоны к существующему развёртыванию (сценарий миграции).

В обоих случаях общие шаги одинаковы. Следуйте шагам своего сценария, чтобы начать использовать сетевые зоны.

## Активация сетевых зон

Активация сетевых зон

1. Перейдите в **Settings** и выберите **Preferences** > **Network zones**.
2. Включите **Enable network zones in this environment**.

Либо вы можете использовать вызов API [PUT global configuration](/managed/dynatrace-api/environment-api/network-zones/put-global-config "Редактирование глобальной конфигурации сетевых зон через Dynatrace API.").

## Организация сетевых зон

Сетевые зоны организованы по кластеру Managed.

Обзор сетевых зон можно получить в [Cluster Management Console](/managed/managed-cluster/basics/managed-components#cluster-management-console "Изучите архитектуру Dynatrace Managed, включая Managed Cluster, Cluster Management Console и Mission Control.") или через [Cluster API](/managed/dynatrace-api/cluster-api/cluster-api-v2 "Узнайте об управлении окружениями, сетевыми зонами, Synthetic-расположениями, узлами и токенами в Dynatrace Managed с помощью Cluster API v2.").

## Именование сетевых зон

Имя сетевой зоны — это строка из буквенно-цифровых символов. Также можно использовать дефисы (`-`) и подчёркивания (`_`). Длина строки ограничена 256 символами.

Точка (`.`) служит разделителем в имени сетевой зоны (см. раздел **Шаблон именования** ниже). Её нельзя использовать в других целях, и имя сетевой зоны не должно начинаться с точки.

Имена сетевых зон не чувствительны к регистру. Dynatrace хранит их в нижнем регистре.

Существует предварительно созданная сетевая зона **default** (по умолчанию). OneAgent использует её, когда сетевая зона не указана или настройка сброшена. Имя **default** равнозначно пустому имени.

### Шаблон именования

Для удобства работы с сетевыми зонами рекомендуется следующий шаблон именования:

```
provider.region.availability-zone.datacenter.tier
```

Если вы используете собственную инфраструктуру, укажите название своей компании в части `provider`.

В этот шаблон можно добавлять дополнительные поля.

## Альтернативная сетевая зона

Для каждой сетевой зоны можно настроить одну или несколько альтернативных зон — сетевых зон, которые OneAgent должны использовать при недоступности ActiveGate из основной зоны. Убедитесь, что ActiveGate из альтернативной сетевой зоны доступны для OneAgent основной сетевой зоны.

Настройка альтернативных сетевых зон

1. Перейдите в **Deployment Status** и выберите **Network zones**.
2. Выберите сетевую зону, которую нужно настроить, и нажмите **Edit** в правом верхнем углу страницы.
3. Выберите нужные сетевые зоны.

Либо можно использовать вызов [PUT a network zone](/managed/dynatrace-api/environment-api/network-zones/put-network-zone "Обновление сетевой зоны через Dynatrace API.") Environment API для настройки списка альтернативных зон.

## Режим резервирования

Режим резервирования определяет, как сеть обрабатывает трафик, если в данной сетевой зоне недоступны ActiveGate и альтернативные сетевые зоны не настроены.

Доступные режимы резервирования:

* **Any ActiveGate (default)**: трафик направляется к любому доступному ActiveGate, если основная сетевая зона недоступна.
* **Only DefaultZone**: трафик направляется только в зону по умолчанию.
* **None (drop traffic)**: трафик отбрасывается, если основная сетевая зона недоступна.

### Настройка режима резервирования

1. Перейдите в **Deployment Status** и выберите **Network zones**.
2. Выберите сетевую зону, для которой нужно установить режим резервирования.

   Обратите внимание, что сетевая зона по умолчанию не настраивается.
3. Нажмите **Edit**.
4. В разделе **Fallback mode** выберите режим резервирования из списка.
5. Нажмите **Save changes**.

## Ограничения

Максимальное количество сетевых зон на окружение по умолчанию составляет 10 000. Если вам нужно больше, свяжитесь со специалистом Dynatrace через live chat в вашем окружении Dynatrace.
