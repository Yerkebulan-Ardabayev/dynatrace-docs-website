# Теги и метаданные

# Теги и метаданные

* Пояснение
* Чтение 1 мин
* Опубликовано 19 июл 2017 г.

[Dynatrace OneAgent](/managed/platform/oneagent "Узнайте о возможностях мониторинга OneAgent.") автоматически обнаруживает компоненты в вашем окружении: группы процессов, сервисы и приложения для реальных пользователей. Помимо автоматического обнаружения, точно определяются базовые технологии (например, Apache HTTP server, IBM WebSphere и многое другое). В рамках этого автообнаружения Dynatrace также выявляет сетевую топологию. После начала мониторинга трафика Dynatrace обнаруживает балансировщики нагрузки и прокси. Dynatrace также поддерживает сторонние интеграции, дополняя данные топологией и информацией мониторинга из AWS, VMware, Azure, OpenStack и других источников.

Однако управление таким большим объёмом информации и организация крупных окружений мониторинга представляют собой реальную проблему. Для её решения Dynatrace поддерживает теги и метаданные. Они позволяют организовывать мониторируемые окружения осмысленным образом. Теги в Dynatrace — это метки или маркеры, а метаданные — пары ключ/значение, присущие любой мониторируемой сущности. Метаданные в основном используются для определения дополнительной информации о сущностях, а теги — для их организации. На основе метаданных можно создавать теги. В целом, несмотря на тесную связь, теги и метаданные — разные концепции, которые создаются и используются по-разному.
