# Отправка уведомлений Dynatrace в Jira

* Чтение: 3 мин
* Обновлено 25 апреля 2024 г.

Благодаря интеграции Dynatrace и Jira тикеты автоматически создаются для всех новых проблем, автоматически обнаруженных в средах Dynatrace.

Atlassian Jira — наиболее популярная система отслеживания задач, доступная сегодня. Большинство отделов разработки и эксплуатации по всему миру используют Jira для управления задачами. Возможность Jira управлять задачами, запускать рабочие процессы и отслеживать изменения в коде и в производственной среде критически важна для современных DevOps-подразделений.

Простая и прямая интеграция Dynatrace и Atlassian Jira значительно сокращает ручной труд и полностью автоматизирует передачу проблем, обнаруженных Dynatrace, в проект Jira вашей организации.

## Настройка интеграции с Jira

Интеграция поддерживает Jira REST API v2. Jira REST API v3 не поддерживается.

Чтобы настроить интеграцию Jira с Dynatrace:

1. Перейдите в **Settings** > **Integration** > **Problem notifications**.
2. Выберите **Add notification**.
3. Выберите **Jira** из доступных типов уведомлений.
4. Настройте уведомление:

   * Введите **Display name** (отображаемое имя) для этой интеграции — произвольное имя, которое будет отображаться в Dynatrace в разделе **Settings** > **Integration** > **Problem notifications** после завершения настройки.
   * Введите **Jira endpoint URI**.
   * Введите **Username** (имя пользователя) и **API token** (токен API) для этого проекта Jira.

     Personal Access Tokens

     **Personal Access Tokens** не поддерживаются для классической интеграции Jira. Можно использовать только метод базовой аутентификации.
   * Введите **Project key** (ключ проекта) — не название проекта — проекта Jira, в котором должны создаваться и отслеживаться новые задачи.

     + Чтобы найти все доступные ключи проектов, в Jira перейдите в **Projects** > **View all projects**.
   * Введите **Issue type** (тип задачи) Jira, который должен использоваться для проблем, обнаруженных Dynatrace.

     + Необходимо указать тип задачи, который уже настроен в Jira, иначе интеграция не будет работать.
     + Чтобы найти все доступные типы задач или создать новый, в Jira перейдите в **Options** > **Issues**.
   * Введите краткое **Summary** (описание) задачи и более подробное **Issue description** (описание проблемы).
     Помимо обычного текста, описание может содержать заполнители.

     Заполнители

     В разделе **Available placeholders** на странице настройки перечислены заполнители, доступные для этой интеграции. Заполнители автоматически заменяются информацией о проблеме.

   Пример рабочей конфигурации интеграции с Jira:

   ![Пример конфигурации интеграции Jira для уведомлений и оповещений](https://dt-cdn.net/images/example-of-jira-integration-1686-50bc46b6d7.png)

   Пример конфигурации интеграции Jira для уведомлений и оповещений
5. Выберите **Send test notification**, чтобы убедиться, что интеграция с Jira работает.
6. **Save changes**.

## После интеграции

После завершения интеграции с Jira тикеты автоматически создаются в вашем проекте Jira для всех проблем, автоматически обнаруженных Dynatrace.

Пример тикета

![Пример тикета Jira](https://dt-cdn.net/images/jiraintegration-4-1646-ae8a48e8c0.png)

Пример тикета Jira

Dynatrace автоматически обновляет уровни серьёзности задач в Jira в зависимости от их влияния на приложения и клиентский опыт.

Dynatrace не закрывает разрешённые задачи автоматически. Задачи Jira необходимо закрывать вручную.
