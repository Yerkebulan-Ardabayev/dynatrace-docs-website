# Отправка уведомлений Dynatrace по электронной почте

* Чтение: 2 мин
* Обновлено 25 апреля 2024 г.

Dynatrace предлагает несколько готовых интеграций для автоматической отправки уведомлений о проблемах в сторонние системы обмена сообщениями или управления инцидентами. Если ваша сторонняя система не поддерживается готовой интеграцией, можно легко настроить интеграцию по электронной почте. При таком подходе письмо отправляется каждый раз, когда Dynatrace обнаруживает проблему, влияющую на реальных пользователей.

Кратковременные проблемы

Кратковременные проблемы не генерируют уведомления об открытых проблемах. Для таких проблем Dynatrace отправляет только уведомления о разрешённых проблемах, информируя о том, что такая проблема имела место.

Можно настроить строку темы писем с уведомлениями о проблемах, определив текстовый шаблон с заполнителями, которые динамически подставляют релевантные данные проблемы — такие как идентификатор проблемы, её влияние или состояние. По умолчанию тело писем содержит HTML-описание обнаруженной проблемы и прямую ссылку на соответствующую страницу [сведений о проблеме](/managed/dynatrace-intelligence/root-cause-analysis/concepts "Ознакомьтесь с концепциями анализа первопричин.") в Dynatrace.

Чтобы настроить уведомления о проблемах по электронной почте:

1. Перейдите в **Settings** > **Integration** > **Problem notifications**.
2. Выберите **Add notification**.
3. Выберите **Email** из доступных типов уведомлений.
4. Настройте уведомление:

   * Введите **Display name** (отображаемое имя) для этой интеграции — произвольное имя, которое будет отображаться в Dynatrace в разделе **Settings** > **Integration** > **Problem notifications** после завершения настройки.
   * В разделе **To** выберите **Add recipient** для добавления адреса электронной почты, на который следует отправлять уведомления. Для добавления дополнительных получателей выбирайте **Add recipient** для каждого адреса.
   * В поле **CC** выберите **Add recipient** для добавления дополнительных адресов электронной почты.
   * В поле **BCC** выберите **Add recipient** для добавления адресов, которые должны получать уведомления, но не отображаться в заголовке письма.
   * В поле **Subject** введите текст и/или вставьте заполнители, которые будут автоматически подставлять данные проблемы — идентификатор, состояние или влияние.
   * В поле **Body** настройте текст сообщения оповещения, который появится в теле писем.

     Заполнители

     В разделе **Available placeholders** на странице настройки перечислены заполнители, доступные для этой интеграции. Заполнители автоматически заменяются реальными значениями в сообщении.
   * Выберите [профиль оповещений](/managed/analyze-explore-automate/notifications-and-alerting/alerting-profiles "Узнайте, как создавать профили оповещений и управлять ими.") для фильтрации потока проблем.
5. Выберите **Send test notification**, чтобы убедиться, что интеграция по электронной почте работает.
6. **Save changes**.
