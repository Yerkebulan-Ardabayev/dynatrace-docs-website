# Push-уведомления через мобильное приложение Dynatrace

* Чтение: 8 мин
* Обновлено 14 января 2026 г.

Dynatrace предоставляет несколько каналов для получения оповещений об обнаруженных проблемах в среде. Помимо традиционных методов, таких как электронная почта, Dynatrace интегрируется напрямую с популярными сторонними сервисами — Slack, Jira и Opsgenie. Кроме того, можно получать оповещения через мобильное приложение Dynatrace.

Прекращение поддержки нативных мобильных приложений (iOS и Android) для Dynatrace

В рамках стратегии обеспечения единообразного пользовательского опыта Dynatrace прекращает поддержку нативных мобильных приложений для iOS и Android. Это согласуется со стратегией предоставления единого адаптивного веб-интерфейса и использования современных гибких каналов уведомлений.

#### Ключевые даты

* **Последнее обновление приложения**: конец 2025 г.
* **Дата прекращения поддержки**: 30 июня 2026 г.
* **Окончание поддержки**: 30 июня 2026 г.

#### Что изменится

После даты прекращения поддержки нативные мобильные приложения Dynatrace будут недоступны для загрузки и не будут поддерживаться.

Рекомендуется использовать Dynatrace через адаптивный веб-интерфейс, предоставляющий полный функционал на всех типах устройств и экранах любого размера.

Push-уведомления теперь будут доставляться через сторонние интеграции — Slack, Teams, PagerDuty, электронную почту и ntfy.

#### Почему это изменение?

* **Единый опыт**: единая адаптивная веб-платформа обеспечивает согласованную функциональность на всех устройствах.
* **Гибкость**: вы можете выбрать предпочитаемые инструменты коммуникации для оповещений и обновлений.

#### Рекомендуемые действия

* Начните использовать адаптивный веб-интерфейс.
* Настройте предпочитаемые интеграции уведомлений:

  + Dynatrace SaaS: [Недоступно в Dynatrace Managed](/managed/upgrade/unavailable-in-managed "Выбранная функция недоступна в Dynatrace Managed.")
  + Dynatrace Managed: [Уведомления о проблемах](/managed/analyze-explore-automate/notifications-and-alerting/problem-notifications "Узнайте, как интегрировать сторонние системы уведомлений о проблемах с Dynatrace.")
* Удалите нативное приложение с устройств после даты прекращения поддержки.

#### Нужна помощь?

Для помощи с переходом или настройкой интеграций обратитесь к документации по ссылкам выше или свяжитесь со службой поддержки.

Остальная часть этой страницы относится к мобильному приложению Dynatrace, поддержка которого прекращается 30 июня 2026 г. Подробнее см. выше.

Мобильное приложение Dynatrace доступно для устройств [iOS](https://dt-url.net/ue02yx1) и [Android](https://dt-url.net/uf22yp6). Его основная функция — доставка push-уведомлений о проблемах, обнаруженных в ваших средах мониторинга. Уведомления извлекаются и обновляются в реальном времени, обеспечивая информирование об актуальных проблемах. Это делает мобильное приложение удобным и эффективным инструментом управления оповещениями и мониторинга среды в дороге.

Push-уведомления для каждой среды фильтруются на основе зон управления, для которых у пользователя есть как минимум права на чтение. Если у пользователя нет прав на чтение для определённой зоны управления, push-уведомления для этой зоны не отправляются.

Помимо фильтра зон управления для отдельных пользователей, push-уведомления также фильтруются по **Default alerting profile** среды. Изменение **Default alerting profile** затрагивает все push-уведомления о проблемах, отправляемые всем пользователям среды.

**Ключевые функции**

* Оповещения в реальном времени: немедленные push-уведомления об обнаруженных проблемах.
* Поддержка нескольких сред: мониторинг нескольких сред Dynatrace из одного приложения.
* Совместимость: поддержка push-уведомлений из сред Dynatrace SaaS и Dynatrace Managed.

## Назначение и контекст

Это руководство описывает использование мобильного приложения Dynatrace для получения push-уведомлений об обнаруженных проблемах в средах мониторинга.

## Целевая аудитория

Руководство предназначено для:

* Инженеров по эксплуатации
* Инженеров конвейеров
* Системных инженеров
* Инженеров по надёжности сайтов (SRE)
* Инженеров по автоматизации сборки

## Предварительные требования

### Android

* Android 9.0 и выше
* [Мобильное приложение Dynatrace для Android](https://dt-url.net/uf22yp6) установлено на мобильном устройстве. Убедитесь, что устройство имеет постоянный доступ к интернету.

### iOS

* iOS 16 и выше
* [Мобильное приложение Dynatrace для iOS](https://dt-url.net/ue02yx1) установлено на мобильном устройстве. Убедитесь, что устройство имеет постоянный доступ к интернету.

### Сеть

* Если сервер Dynatrace Managed работает за корпоративным брандмауэром, необходимо настроить Cluster ActiveGate, **доступный из интернета**. Cluster ActiveGate обслуживает REST API, необходимый для мобильного приложения. Дополнительные сведения о маршрутизации трафика OneAgent в Dynatrace см. в разделе [Маршрутизация трафика OneAgent в Dynatrace, мониторинг облачных сред и удалённых технологий с помощью расширений](/managed/ingest-from/dynatrace-activegate/capabilities/routing-monitoring-purpose "Узнайте о возможностях маршрутизации и мониторинга ActiveGate.").
* Брандмауэр должен разрешать входящие сетевые подключения к настроенному домену Cluster ActiveGate через настроенный порт.
* Если перед Cluster ActiveGate установлен брандмауэр, он должен отправлять правильные сертификаты безопасности.
* Используйте [проверку DigiCert](https://www.digicert.com/help/), чтобы убедиться, что сертификат вашего публично доступного ActiveGate проходит все проверки. iOS и Android блокируют небезопасные запросы без действительного сертификата домена.

## Токен доступа API мобильного приложения

Мобильное приложение использует токен доступа API с назначенной областью **Mobile** (`mobile`). Для просмотра токена перейдите в [меню пользователя](/managed/discover-dynatrace/get-started/dynatrace-ui#user "Навигация по платформе Dynatrace Managed") > **Personal Access Tokens**.

## Коммуникация мобильного приложения Dynatrace с Dynatrace Managed

Между кластером Dynatrace и мобильным приложением существует двусторонняя связь.

1. Кластер Dynatrace Managed передаёт минимальную информацию (только заголовок и идентификатор проблемы) через [Mission Control](/managed/managed-cluster/basics/mission-control-proactive-support "Mission Control проактивно отслеживает ваш Managed Cluster, предоставляет обновления программного обеспечения и поддерживает безопасность и надёжность вашей установки.").
2. Мобильное приложение может затем получать всю подробную информацию о проблеме по запросу. Выберите уведомление, чтобы получить обновлённую информацию о проблеме.

Для доступа к API сведений о проблемах необходимо установить [публично доступный ActiveGate](/managed/managed-cluster/basics/managed-deployments#scenario-2-pure-dynatrace-managed-setup "Узнайте, как развёртывания Dynatrace Managed эволюционируют от базовой внутренней установки до глобально распределённой высокодоступной архитектуры."). Доступ защищён автоматически сгенерированным пользовательским токеном API, упомянутым выше. Этот токен ограничен API мобильного приложения и не может использоваться для других вызовов API или входа в кластер.

## Начало работы

### Шаг 1: Вход в мобильное приложение

1. В управляемой среде перейдите в меню учётной записи пользователя в правом верхнем углу и выберите **Receive alerts via mobile app**.
2. В мобильном приложении Dynatrace выберите **Connect Managed Environment** для сканирования сгенерированного QR-кода среды.

   При каждой перезагрузке или повторном посещении страницы QR-кода в меню учётной записи предыдущий код удаляется, создаётся новый, и необходимо повторно подключить мобильное приложение.

### Шаг 2: Просмотр проблем и уязвимостей

Мобильное приложение Dynatrace отображает все подключённые среды и выделяет общее количество активных проблем и уязвимостей для каждой с помощью красного индикатора. Этот глобальный индикатор помогает командам эксплуатации приоритизировать среды с активными проблемами.

Для каждой уязвимости отображается её [оценка безопасности Dynatrace (DSS)](/managed/secure/application-security/vulnerability-analytics/third-party-vulnerabilities/davis-security-score "Рассчитайте оценку безопасности Davis и спрогнозируйте потенциальные риски уязвимостей с помощью Davis AI."), количество затронутых сущностей и номер CVE.

* Для поиска выберите **CVE ID**, **External ID** или **Component** и введите строку поиска.
* Для изменения порядка сортировки выберите **Sorted by time** или **Sorted by DSS**.

### Шаг 5: Получение push-уведомлений

После входа в Dynatrace push-уведомления поступают для всех вновь обнаруженных проблем во всех подключённых средах.

В настройках мобильного приложения можно отключать или включать push-уведомления для каждой среды отдельно.

Все push-уведомления фильтруются по зонам управления, для которых у вас есть как минимум права наблюдателя.

Для получения push-уведомлений роль пользователя должна включать разрешение `View environment`. Обратите внимание, что назначения на основе политик в настоящее время не поддерживаются для мобильного приложения.

Если вы не получаете push-уведомления:

* Убедитесь, что у вас есть необходимые права для среды и конкретной зоны управления, в которой была обнаружена проблема.
* Убедитесь, что мобильное приложение Dynatrace имеет необходимые разрешения Android и iOS для получения push-уведомлений.
* Выйдите из системы и войдите снова, чтобы повторить регистрацию push-уведомлений у поставщика push-уведомлений.
