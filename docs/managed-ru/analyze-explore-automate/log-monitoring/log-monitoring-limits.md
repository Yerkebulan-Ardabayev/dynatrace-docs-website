# Стандартные ограничения Log Monitoring (Logs Classic)

# Стандартные ограничения Log Monitoring (Logs Classic)

* Чтение: 1 мин
* Обновлено 04 марта 2026 г.

Log Monitoring Classic

На этой странице перечислены стандартные ограничения для последней версии Dynatrace Log Monitoring.

Текущие ограничения применяются как к приёму файлов журналов, так и к общему приёму журналов через API.

## Ограничения приёма журналов

В таблице ниже приведены наиболее важные стандартные ограничения, связанные с приёмом журналов. Все указанные ограничения относятся к данным в кодировке UTF-8.

| Тип | Ограничение | Описание |
| --- | --- | --- |
| Содержимое | 8 kB | Максимальный размер тела записи журнала |
| Ключ атрибута | 100 байт | Ключ значения атрибута |
| Длина значения атрибута | 250 байт | Максимальная длина значения атрибута |
| Количество атрибутов журнала | 50 | Максимальное количество атрибутов в журнале |
| Событий журнала в минуту | 1k/мин на кластер по умолчанию | Максимальное количество событий журнала в минуту |
| Возраст журналов | 24 часа | Максимальный возраст записей журнала при приёме |
| Журналы с будущими датами | Без ограничений[1](#fn-1-1-def) | Насколько далеко в будущее могут уходить записи журнала |
| Значений на атрибут | 32 значения | Максимальное количество отдельных значений в атрибуте |
| Размер запроса | 10 MB | Максимальный размер данных payload |
| Количество записей журнала | 50 000 записей | Максимальное количество записей журнала на запрос |
| Вложенные объекты | 5 уровней | Максимальное количество уровней при приёме вложенных объектов |

1

Ограничений на приём записей журнала с будущими временными метками нет, однако записи с временными метками, выходящими более чем на 10 минут в будущее, получают временную метку, равную моменту приёма.

Если поток событий журнала в кластере превышает лимит, все события сверх лимита игнорируются. При увеличении ресурсов (RAM) узлов можно увеличить лимит на основе размера ресурсов кластера с помощью вызова API или Cluster Management Console (CMC).

Есть два способа обновить лимит кластера:

* С помощью вызова API. Подробнее см. в разделе [Обновление событий журнала на кластер для Log Monitoring](/managed/dynatrace-api/cluster-api/cluster-api-v2/log-monitoring/post-update-log-events-per-cluster-for-log-monitoring "Узнайте, как обновить общий лимит событий журнала на кластер на основе размера ресурсов кластера через API.").
* Обновление лимита кластера через Cluster Management Console (CMC):

  1. В CMC выберите **Environments** и окружение, для которого нужно обновить общий лимит событий журнала на кластер.
  2. На странице сведений об окружении в разделе **Cluster overload prevention settings** нажмите **Refresh cluster limit**.

## Событий журнала в минуту

По умолчанию приём данных журналов ограничен 1 000 событиями журнала в минуту на кластер.

* Если поток данных журналов в кластере превышает лимит, все события сверх лимита игнорируются.
* При увеличении ресурсов (RAM) узлов можно увеличить лимит на основе размера ресурсов кластера с помощью вызова API или Cluster Management Console (CMC).

Это ограничение применяется ко всем источникам событий (OneAgent и общий приём журналов). Для получения количества входящих событий журнала и событий, отклонённых из-за превышения лимита, можно использовать метрики самомониторинга. Подробнее см. в разделе [Метрики самомониторинга](/managed/analyze-explore-automate/metrics-classic/self-monitoring-metrics#log-self-monitoring-metrics "Просмотрите полный список метрик самомониторинга Dynatrace.").

**Обновление лимита кластера через API**  
См. [Обновление событий журнала на кластер для Log Monitoring](/managed/dynatrace-api/cluster-api/cluster-api-v2/log-monitoring/post-update-log-events-per-cluster-for-log-monitoring "Узнайте, как обновить общий лимит событий журнала на кластер на основе размера ресурсов кластера через API.").

**Обновление лимита кластера через Cluster Management Console (CMC)**

1. В CMC выберите **Environments** и окружение, для которого нужно обновить общий лимит событий журнала на кластер.
2. На странице сведений об окружении в разделе **Cluster overload prevention settings** нажмите **Refresh cluster limit**.
