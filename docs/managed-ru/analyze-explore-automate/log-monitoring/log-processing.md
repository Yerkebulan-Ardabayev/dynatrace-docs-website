# Обработка журналов (Logs Classic)

# Обработка журналов (Logs Classic)

* Пояснение
* Чтение: 3 мин
* Обновлено 18 января 2023 г.

Log Monitoring Classic

Dynatrace Log Monitoring может преобразовывать входящие данные журналов для лучшего понимания, анализа или дальнейшей обработки на основе создаваемых правил обработки журналов.

Информация может записываться в журналы в самых разных форматах в зависимости от приложения или процесса, операционной системы и других факторов. Обработка журналов предлагает централизованный и гибкий способ извлечения ценности из необработанных строк журналов.

Например, с помощью обработки журналов можно извлекать числовые значения из строк журнала, преобразовывать их в метрики на Dynatrace Platform и включать в дашборды и обнаружение проблем.

Обработка журналов не влияет на потребление [DDU](/managed/license/monitoring-consumption-classic/davis-data-units/log-monitoring-consumption "Узнайте, как рассчитывается объём потребления DDU для Dynatrace Log Monitoring Classic.") при приёме журналов.

## Правила обработки журналов

Перейдите в **Settings** > **Log Monitoring** > **Processing**, чтобы просмотреть действующие правила обработки журналов, изменить порядок существующих правил и создать новые. Правила выполняются в том порядке, в котором они перечислены, сверху вниз. Этот порядок критически важен, поскольку предыдущее правило может влиять на данные журнала, используемые последующим правилом.

Разверните **Details**, чтобы проверить определение правила. Правило обработки журналов состоит из следующих элементов:

* **Rule name** (имя правила)
* **Matcher** (условие сопоставления)
* **Rule definition** (определение правила)

Любое правило можно включить или отключить в столбце **Active**.

## Встроенные правила

По умолчанию обработка журналов включает множество включённых встроенных правил, отвечающих за очистку или нормализацию данных журналов. Название каждого встроенного правила начинается с `[Built-in]`.

Эти правила нельзя изменить напрямую, однако их можно отключить, скопировать их определения и создать новые правила с нужными изменениями.

## Добавление правила обработки журналов

Чтобы создать правило обработки журналов:

1. Перейдите в **Settings** > **Log Monitoring** > **Processing**.
2. Нажмите **Add rule**.
3. Введите название правила обработки журналов.
4. Введите поисковый запрос журнала в разделе **Matcher**.  
   Поисковый запрос сужает доступные данные журнала для выполнения данного конкретного правила. Это тот же поисковый запрос, который используется в [строке поиска просмотрщика журналов](/managed/analyze-explore-automate/log-monitoring/analyze-log-data/log-viewer#query-syntax "Узнайте, как использовать просмотрщик журналов Dynatrace для анализа данных.").

   Сопоставление на основе предыдущих правил не поддерживается

   Matcher работает с исходным набором данных до применения каких-либо правил обработки. Сопоставление записей, изменённых предыдущими правилами, не поддерживается. Например, поле, изменённое в правиле 1 и используемое для сопоставления в правиле 2, будет содержать исходное значение этого поля, а не изменённое значение из правила 1.
5. Введите определение правила обработки.  
   Определение правила обработки — это инструкция по обработке журналов о том, как Dynatrace должен преобразовывать или изменять данные журнала, суженные с помощью **Log query**.

   Определение правила создаётся с использованием [команд](/managed/analyze-explore-automate/log-monitoring/log-processing/log-processing-commands "Используйте команды обработки журналов для преобразования входящих данных журнала для лучшего анализа или дальнейшей обработки."), [функций](/managed/analyze-explore-automate/log-monitoring/log-processing/log-processing-functions "Используйте типы данных обработки журналов для преобразования входящих данных журнала для лучшего анализа или дальнейшей обработки.") обработки журналов и сопоставления шаблонов, позволяющих добавлять, преобразовывать или удалять входящие записи журнала. Это даёт полный контроль над тем, как данные журнала представляются в Dynatrace Log Monitoring.
6. Протестируйте правило обработки журналов.  
   Можно протестировать определение правила, загрузив пример журнала или вручную введя его фрагмент в текстовое поле **Log sample**.

   1. Выберите пример журнала.

      * При загрузке примера журнала данные для тестирования правила — это результат совпадения **Log query**.
      * При ручном вводе фрагмента данных журнала убедитесь, что он в формате JSON. Любые текстовые данные журнала должны быть помещены в поле `content` JSON-объекта.
   2. Запустите тест.

      Нажмите **Test the rule** и просмотрите результат в текстовом поле **Test result**.
7. Нажмите **Save changes**.
