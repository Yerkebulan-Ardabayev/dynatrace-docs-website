# Анализ данных журналов (Logs Classic)

* Обзор
* Чтение: 4 мин
* Обновлено 18 января 2023 г.

Log Monitoring Classic

Log Monitoring предоставляет прямой доступ к содержимому журналов всех критически важных процессов системы. Данные журналов, как правило, содержат большое количество информации. Один из способов работы с большим объёмом данных — сужение выборки записей журналов и их парсинг.

## Просмотрщик журналов

Просмотрщик журналов позволяет представить данные журналов в фильтруемой таблице, удобной для работы, и просматривать данные за определённый период с использованием обнаруженных аспектов содержимого журналов. Можно использовать **Available attributes** для сужения представления журналов и фокусировки на конкретном аспекте содержимого.

* Ознакомьтесь с разделом [Просмотрщик журналов (Logs Classic)](/managed/analyze-explore-automate/log-monitoring/analyze-log-data/log-viewer "Узнайте, как использовать просмотрщик журналов Dynatrace для анализа данных.").

Права пользователя для Log Monitoring

Журналы часто содержат чувствительную информацию, которую могут видеть не все пользователи. По этой причине администратор Dynatrace должен добавить утверждённых пользователей Log Monitoring в группу **Log viewer**, имеющую разрешение безопасности учётной записи **View logs**. Пользователи без прав администратора по умолчанию НЕ входят в эту группу. Для доступа к содержимому журналов их необходимо явно добавить.

## События журналов

После создания событий журналов на основе содержимого журналов ИИ Dynatrace автоматически соотносит релевантные события журналов с обнаруживаемыми проблемами в среде. Релевантные события журналов, связанные с проблемами, учитываются при анализе первопричин.

* Ознакомьтесь с разделом [События журналов (Logs Classic)](/managed/analyze-explore-automate/log-monitoring/analyze-log-data/log-events "Узнайте, как создавать и использовать события журналов Dynatrace для анализа данных.").

## Метрики журналов

Dynatrace Log Monitoring позволяет не только просматривать и анализировать журналы, но и создавать метрики на основе данных журналов и использовать их в Dynatrace наравне с любой другой метрикой. Их можно добавлять на панель мониторинга, включать в анализ и создавать пользовательские оповещения.

* Ознакомьтесь с разделом [Метрики журналов (Logs Classic)](/managed/analyze-explore-automate/log-monitoring/analyze-log-data/log-metrics "Узнайте, как создавать и использовать метрики журналов Dynatrace для анализа данных.").

## Пользовательские атрибуты журналов

В Dynatrace Log Monitoring можно определять собственные пользовательские атрибуты данных журналов, соответствующие конкретному формату данных. Аналогично автоматически обнаруживаемым атрибутам журналов, пользовательские атрибуты извлекаются из данных журналов при приёме и становятся доступными в Dynatrace.

* Ознакомьтесь с разделом [Пользовательские атрибуты журналов (Logs Classic)](/managed/analyze-explore-automate/log-monitoring/analyze-log-data/log-custom-attributes "Узнайте, как создавать и использовать пользовательские атрибуты при приёме данных журналов.").

## Анализ обогащённых данных журналов

С обогащёнными данными журналов можно проверить конкретного пользователя внутри приложения. Используйте просмотрщик журналов и ссылку на распределённые трассировки PurePath® из конкретной записи журнала. Можно просмотреть все журналы для конкретной пользовательской сессии, а через вкладку **Logs** в распределённых трассировках — быстро понять, что произошло.

* Ознакомьтесь с разделом [Использование обогащения журналов для трассировок при решении проблем](/managed/observe/application-observability/distributed-traces/use-cases/problems-logs-traces "Используйте обогащение журналов для просмотра связанных записей в распределённых трассировках.").
