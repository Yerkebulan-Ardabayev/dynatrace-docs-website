# Пользовательские атрибуты журналов (Logs Classic)

* Учебное руководство
* Чтение: 3 мин
* Обновлено 03 июля 2023 г.

Log Monitoring Classic

Dynatrace [автоматически обнаруживает атрибуты](/managed/upgrade/unavailable-in-managed "Выбранная функция недоступна в Dynatrace Managed.") принятых данных журналов. **Available attributes** позволяют быстро фильтровать данные таблицы результатов по конкретному атрибуту данных журнала.

Также можно определять собственные пользовательские атрибуты данных журналов, соответствующие конкретному формату. Аналогично автоматически обнаруживаемым атрибутам журналов, пользовательские атрибуты извлекаются из данных журналов при приёме и становятся доступными в Dynatrace.

Их можно использовать в качестве фильтров в просмотрщике журналов (параметры таблицы и атрибуты детального просмотра записей журналов), в качестве измерений при создании метрик журналов и свойств при создании событий журналов.

Эти пользовательские атрибуты должны совпадать с атрибутами журналов в принятых данных, иначе они будут проигнорированы.

Dynatrace версии 1.226+

Dynatrace автоматически распознаёт атрибуты журналов, видимые в деталях записей и параметрах таблицы.

## Создание пользовательского атрибута журнала

1. Перейдите в **Settings** > **Log Monitoring** > **Custom attributes**.
2. Нажмите **Add custom attribute**.
3. Выберите **Log Monitoring** > **Custom attributes** и нажмите **Add custom attribute**.
4. Включите параметр **Show attribute values in side bar**. Это обеспечивает индексирование значений атрибута и поиск журналов по ним.
5. Введите **Key**.
   Правила для ключей:

   * Ключ может содержать только буквенно-цифровые символы, знаки подчёркивания (`_`), дефисы (`-`), точки (`.`) и двоеточия (`:`).
   * Не может начинаться с дефиса.
   * Все символы должны быть из латинского алфавита, без диакритических знаков.

Можно проверить, входят ли планируемые пользовательские атрибуты в список [атрибутов ключ-значение](/managed/analyze-explore-automate/log-monitoring/acquire-log-data/logs-classic-ingestion-api#semantics "Узнайте, как Dynatrace принимает данные журналов.") для автоматического обнаружения. Если они присутствуют там, добавлять их в пользовательский список не нужно.

## Пример

В этом примере выполняется API-вызов приёма журналов с JSON, содержащим следующие атрибуты журнала:

```
{



"timestamp": "2021-07-29T10:54:40.962165022Z",



"level": "error",



"source": "Skynet",



"application.id": "PaymentService-Prod",



"message": "PaymentService-Prod failure.",



"data": {}



}
```

Затем создаётся пользовательский атрибут журнала и используется для создания метрики журнала и события журнала.

1. Выполните API-вызов.
2. Создайте пользовательский атрибут.
