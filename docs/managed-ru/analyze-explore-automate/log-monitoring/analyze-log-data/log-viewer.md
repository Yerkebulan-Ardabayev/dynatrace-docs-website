# Просмотрщик журналов (Logs Classic)

* Пояснение
* Чтение: 9 мин
* Обновлено 18 января 2023 г.

Log Monitoring Classic

Просмотрщик журналов позволяет просматривать журналы за определённый период с использованием обнаруженных аспектов содержимого журналов. Можно использовать **Available attributes** для сужения представления и фокусировки на конкретном аспекте содержимого.

Для доступа к просмотрщику журналов перейдите в **Logs**. Просмотрщик состоит из четырёх разделов:

## Поиск

В **Filter by** можно устанавливать фильтры для сужения отображаемых событий журнала. Выберите **Advanced query** для ручного редактирования запроса.

* С отображаемым **Filter by** (по умолчанию) фильтр работает в режиме автодополнения, где можно выбирать из обнаруженных полей данных журнала (лимит 10 различных атрибутов). Для фильтров с одинаковыми атрибутами достаточно выполнения одного условия. Для фильтров с разными атрибутами все условия должны быть выполнены.
* С **Advanced query** можно задавать более сложные критерии с использованием комбинаций ключевых слов, фраз, логических операторов и скобок (лимит 10 различных атрибутов).

При фильтрации содержимого журналов можно использовать только полные токены. Содержимое журнала разбивается на токены согласно правилам [Unicode Text Segmentation](https://unicode.org/reports/tr29/). Альтернативно можно использовать подстановочный символ `*` в конце токена.

Язык поисковых запросов Dynatrace

| Категория | Описание | Пример |
| --- | --- | --- |
| Текстовый поиск | Поиск отдельных вхождений слов. Пробелы интерпретируются как операторы AND. Поиск нечувствителен к регистру для значений атрибутов. | `error`, `search words`, `"search phrase" more` |
| Атрибуты | Поиск записей с указанным атрибутом и значением. | `host.name="HOST1"`, `host.name!="HOST1"`, `http.code=123` |
| Фразы | Группа слов в двойных кавычках. Поиск по точному совпадению фразы в поле содержимого. | `content="search text"` |
| Булевы операторы | Допустимые операторы: `AND`, `OR`. **Приоритет**: `AND`, `OR`. | `status="INFO" AND host.name="HOST1"`, `status="INFO" OR host.name="HOST1"` |
| Группировка | Скобки `( )` для группировки условий в подзапросы. | `status="INFO" AND (host.name="HOST1" OR host.name="HOST2")` |
| Подстановочные символы | Звёздочка `*` для представления произвольной строки. Допускается только в конце поискового термина. | `host.name="host*"` |
| Специальные символы | Экранирование специальных символов в именах или значениях атрибутов. | `` `attri=bute`="\"abc"` `` |
| Entity Selector | Поиск записей с использованием entity selector. | `dt.source_entity inEntitySelector "type(\"HOST\")"` |

## График

График журналов — это гистограмма событий журнала по времени, дающая быстрый обзор журналов и их серьёзности за выбранный период.

## Таблица результатов

Таблица результатов под графиком отображает события журнала, соответствующие запросу и фильтру за выбранный период. Каждая строка представляет запись журнала и может быть развёрнута для детального просмотра. Отображаются первые 100 совпадающих записей; для просмотра дополнительных результатов нажмите **Show 100 more** внизу таблицы.

По умолчанию принятые записи журнала сортируются по временной метке.

По умолчанию просмотрщик отображает максимум 1 000 событий журнала. При отсутствии ожидаемых результатов уточните запрос или сузьте временной диапазон.

Чтобы показать или скрыть конкретные столбцы:

1. Нажмите **Format table**.
   Отображается список атрибутов журнала, генерируемых Dynatrace, которые можно добавить в таблицу.
2. Установите или снимите флажки для отображения или скрытия соответствующих столбцов.

## Экспорт данных журнала

Чтобы экспортировать данные таблицы:

1. Нажмите **Actions**.
2. Выберите **Download**.
3. Выберите формат: **JSON** или **CSV**.
4. Выберите лимит записей журнала для загрузки.
   При лимите до 1 000 записей создаётся один файл в выбранном формате.
   При лимите более 1 000 записей создаётся ZIP-файл с выбранным файлом и метаданными в формате .txt.

Функция экспорта полного набора данных для заданного запроса доступна только на кластерах Managed.

## Available attributes

**Available attributes** (отображаются слева от таблицы) дают обзор и возможность фильтрации данных журнала. Это автоматически обнаруживаемые атрибуты данных, представленных в таблице. Каждый атрибут отображает до десяти наиболее популярных значений. Для фильтрации всех значений определённого атрибута создайте и запустите запрос в поисковой строке просмотрщика.

## Детали журналов

Уникальные атрибуты данных журнала (атрибуты с высокой кардинальностью), такие как `span_id` и `trace_id`, создают избыточно длинные списки доступных атрибутов, что может влиять на производительность просмотрщика. По этой причине они не отображаются в **Available attributes**. Их по-прежнему можно использовать в расширенных поисковых запросах.
