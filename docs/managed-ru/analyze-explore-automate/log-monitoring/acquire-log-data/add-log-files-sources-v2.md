# Источники логов и хранилище (Logs Classic)

* Чтение: 2 мин
* Обновлено 18 января 2023 г.

Log Monitoring Classic

Начиная с OneAgent версии 1.243 и Dynatrace Cluster версии 1.252, настоятельно рекомендуется переключиться на [Хранилище логов](/managed/analyze-explore-automate/log-monitoring/acquire-log-data/log-storage "Настройте хранение файлов логов, уже известных OneAgent.").

Начиная с версии Dynatrace 1.285 (март 2024 г.), Dynatrace автоматически конвертирует конфигурации [источников логов](/managed/upgrade/unavailable-in-managed "Ваш выбор недоступен в Dynatrace Managed.") и [хранилища логов](/managed/upgrade/unavailable-in-managed "Ваш выбор недоступен в Dynatrace Managed.") в последнюю версию.

Также можно перейти на новую конфигурацию вручную, нажав **Upgrade configuration**. Все текущие параметры будут полностью перенесены.

Обновлённая конфигурация (см. [Правила приёма логов](/managed/analyze-explore-automate/log-monitoring/acquire-log-data/log-storage "Настройте хранение файлов логов, уже известных OneAgent.")), определённая на страницах **Custom log source configuration** и **Log sources and storage**, предоставит:

* Большую гибкость при определении источников логов (например, путь к логу, уровень лога, пространство имён Kubernetes, развёртывание Kubernetes)
* Гибкость в определении источников логов (с использованием областей среды, группы хостов и хоста)
* Детализацию при управлении правами доступа к источникам логов
* Использование REST API для управления источниками логов
* Возможность фильтровать и маскировать данные логов при захвате

Чтобы включить или исключить определённые источники логов из хранилища

1. Перейдите в **Settings** > **Log Monitoring** > **Log sources and storage**.
2. Выберите из списка **Include all logs**, **Include the following logs** или **Exclude the following logs**.
3. Переключайтесь между вкладками для выбора логов из **Hosts perspective** или **Process groups perspective**.

Переключение вкладок

Сохранятся только источники логов на текущей активной (выбранной) вкладке. Источники логов, отмеченные на другой вкладке, будут проигнорированы.

4. Нажмите **Save changes**.

## Переход на новую конфигурацию хранилища

После перехода в **Settings** > **Log Monitoring** > **Log storage** происходит автоматическая миграция со старого формата конфигурации хранилища на новый. В текущей конфигурации произойдут следующие изменения:

* **Перспектива хостов**  
  Все элементы, настроенные в **Hosts perspective**, переносятся как набор сопоставителей в соответствующую область хоста.
* **Перспектива групп процессов**  
  Только правила, применяемые ко всей группе процессов, переносятся в область тенанта. Если группа процессов включена только для подмножества хостов, соответствующие правила должны быть созданы на уровне хоста.

После успешной миграции конфигурации источников логов можно использовать новые элементы конфигурации и добавлять сопоставители.

## Часто задаваемые вопросы

Обратимо ли это изменение?

Нет. После изменения все старые конфигурации удаляются, поэтому убедитесь в своём решении перед выполнением.
