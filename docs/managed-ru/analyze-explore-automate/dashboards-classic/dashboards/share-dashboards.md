# Предоставление доступа к дашбордам Dynatrace

* Практическое руководство
* Чтение: 7 мин
* Опубликовано 19 июля 2017 г.

[Dashboards Classic](/managed/analyze-explore-automate/dashboards-classic "Узнайте, как создавать дашборды Dynatrace Classic, управлять ими и использовать их.")

Вы можете делиться дашбордами с кем угодно, даже если у них нет учётных записей в той же среде Dynatrace.

Ограничения

* Невозможно предоставить доступ к дашборду пользователю или группе, у которых есть только политики IAM (без разрешений на основе ролей).

* Невозможно использовать анонимную ссылку на дашборд для доступа к данным Grail.

## Быстрый старт

Чтобы поделиться дашбордом

1. Откройте дашборд, которым хотите поделиться.

   * Чтобы открыть дашборд, перейдите в **Dashboards** и выберите его имя.
2. Нажмите **More** (**…**) > **Share** в правом верхнем углу дашборда.
3. Нажмите **Advanced settings**, чтобы открыть **Dashboard settings** на вкладке **Manage access**.
4. Включите **Share dashboard** и укажите параметры доступа, как описано ниже.

   * [Предоставить доступ конкретному пользователю](#access-user)
   * [Предоставить доступ конкретной группе](#access-group)
   * [Предоставить доступ любому пользователю по ссылке](#access-all-authenticated)
   * [Предоставить анонимный доступ](#access-anonymous)

   Эти варианты можно комбинировать. Например, можно предоставить доступ `Edit` разработчикам дашборда и доступ `View` другим пользователям или группам в организации.
5. Нажмите **Save changes**.

Чтобы прекратить доступ к дашборду

1. Перейдите в **Dashboards** и выберите имя дашборда.
2. Нажмите **More** (**…**) > **Share** в правом верхнем углу дашборда.
3. В появившемся окне отключите **Share this dashboard**.

Ранее настроенные параметры доступа сохраняются и снова активируются при повторном включении **Share this dashboard**.

## Настройка глобальных параметров доступа к дашбордам

При публичном доступе дашборд могут просматривать (но не изменять) даже пользователи, не имеющие доступа к соответствующей среде Dynatrace или вообще не зарегистрированные в Dynatrace.

1. Перейдите в **Settings** и выберите **Dashboards** > **General settings**.
2. Включите или отключите **Allow anonymous access**, чтобы на глобальном уровне (учётной записи) определить, можно ли открывать дашборды публично.
3. Нажмите **Save changes**.

## Предоставление доступа конкретному пользователю

Чтобы предоставить доступ аутентифицированному пользователю Dynatrace

1. Откройте дашборд, которым хотите поделиться.

   * Чтобы открыть дашборд, перейдите в **Dashboards** и выберите его имя.
2. Нажмите **More** (**…**) > **Share** в правом верхнем углу дашборда.
3. Нажмите **Advanced settings**, чтобы открыть **Dashboard settings** на вкладке **Manage access**.

1. Включите **Share dashboard**.
2. В разделе **Access permissions** нажмите **Grant access permission**.
3. В списке **Who to grant access to** выберите **Specific user**.
4. В списке **User** выберите пользователя, которому нужно предоставить доступ.
5. В поле **What they should be able to do with the dashboard** выберите `Edit` или `View`.
6. Нажмите **Save changes**.  
   Указанный пользователь добавится в список под **Grant access permission**.

   * Когда **Share dashboard** включён, у выбранного пользователя есть доступ к дашборду в соответствии с указанным выше типом (`Edit` или `View`).
   * Чтобы изменить тип доступа (`Edit` или `View`), нажмите **Details** для этого пользователя и измените **What they should be able to do with the dashboard**.
   * Чтобы отозвать доступ, нажмите **Delete** для этого пользователя.

   Пользователи с разрешением `Edit` могут редактировать общие дашборды. Будьте осторожны при предоставлении разрешения `Edit`:

   * Изменения, внесённые другим пользователем, затронут всех пользователей, имеющих доступ к этому дашборду.
   * Если два пользователя редактируют один дашборд одновременно, последние сохранённые изменения перезапишут более ранние. При попытке редактировать дашборд, который в данный момент редактирует другой пользователь, вы получите уведомление.

## Предоставление доступа конкретной группе

Чтобы предоставить доступ всем пользователям группы Dynatrace

1. Откройте дашборд, которым хотите поделиться.

   * Чтобы открыть дашборд, перейдите в **Dashboards** и выберите его имя.
2. Нажмите **More** (**…**) > **Share** в правом верхнем углу дашборда.
3. Нажмите **Advanced settings**, чтобы открыть **Dashboard settings** на вкладке **Manage access**.

1. Включите **Share dashboard**.
2. В разделе **Access permissions** нажмите **Grant access permission**.
3. В поле **Who to grant access to** выберите **Specific group**.
4. В поле **User group** выберите группу, которой нужно предоставить доступ к дашборду.
5. В поле **What they should be able to do with the dashboard** выберите `Edit` или `View`.
6. Нажмите **Save changes**.  
   Указанная группа пользователей добавится в список под **Grant access permission**.

   * Когда **Share dashboard** включён, у всех участников выбранной группы есть доступ к дашборду в соответствии с указанным выше типом (`Edit` или `View`).
   * Чтобы изменить тип доступа (`Edit` или `View`), нажмите **Details** для этой группы и измените **What they should be able to do with the dashboard**.
   * Чтобы отозвать доступ, нажмите **Delete** для этой группы.

   Пользователи с разрешением `Edit` могут редактировать общие дашборды. Будьте осторожны при предоставлении разрешения `Edit`:

   * Изменения, внесённые другим пользователем, затронут всех пользователей, имеющих доступ к этому дашборду.
   * Если два пользователя редактируют один дашборд одновременно, последние сохранённые изменения перезапишут более ранние. При попытке редактировать дашборд, который в данный момент редактирует другой пользователь, вы получите уведомление.

## Предоставление доступа любому пользователю по ссылке

Чтобы предоставить доступ любому аутентифицированному пользователю Dynatrace, имеющему копию общей ссылки

1. Откройте дашборд, которым хотите поделиться.

   * Чтобы открыть дашборд, перейдите в **Dashboards** и выберите его имя.
2. Нажмите **More** (**…**) > **Share** в правом верхнем углу дашборда.
3. Нажмите **Advanced settings**, чтобы открыть **Dashboard settings** на вкладке **Manage access**.

1. Включите **Share dashboard**.
2. В разделе **Access permissions** нажмите **Grant access permission**.
3. В поле **Who to grant access to** выберите **Any user with the link**.
4. Нажмите **Copy link**, чтобы скопировать ссылку для доступа к дашборду.
5. Нажмите **Save changes**.  
   В список разрешений под кнопкой **Grant access permission** будет добавлено **Any user with the link has permission**.
6. Поделитесь ссылкой (адресом) с пользователями, которые должны просматривать дашборд.

   * Когда **Share dashboard** включён, любой пользователь, имеющий копию ссылки, может использовать её для просмотра дашборда.
   * Дашборд не будет отображаться в таблице **Dashboards** пользователя до первого перехода по ссылке.
   * Чтобы отозвать этот доступ, нажмите **Delete** в строке **Any user with the link has permission**.

## Предоставление анонимного доступа

Вы можете предоставить **анонимный доступ** к дашборду через распространяемые ссылки:

* Анонимный доступ даёт только право просмотра
* Можно создавать ссылки для отдельных зон управления
* Для включения или отключения анонимного доступа для вашей среды см. **Settings** > **Dashboards** > **General settings**.

* Невозможно использовать анонимную ссылку на дашборд для доступа к данным Grail.

### Создание публичной ссылки на дашборд

Чтобы создать публичную ссылку на дашборд, доступную всем (без аутентификации)

1. Откройте дашборд, которым хотите поделиться.

   * Чтобы открыть дашборд, перейдите в **Dashboards** и выберите его имя.
2. Нажмите **More** (**…**) > **Share** в правом верхнем углу дашборда.
3. Нажмите **Advanced settings**, чтобы открыть **Dashboard settings** на вкладке **Manage access**.

1. Включите **Share dashboard**.
2. В разделе **Anonymous access** нажмите **Add anonymous access link**.
3. В поле **Management Zone for anonymous access link** выберите зону управления или оставьте значение `default`.

   * Доступные зоны управления зависят от ваших разрешений на момент создания ссылки.
4. Нажмите **Save changes**.  
   В список под **Add anonymous access link** добавится публичная ссылка.
5. Разверните запись (**Details**) и нажмите **Copy link**, чтобы скопировать ссылку в буфер обмена. Вставьте ссылку в любое сообщение со ссылкой на дашборд.

Повторите процедуру для создания нескольких анонимных ссылок.

* Если для дашборда заданы [временной диапазон и зона управления по умолчанию](/managed/analyze-explore-automate/dashboards-classic/dashboards/dashboard-timeframe#default "Узнайте о параметрах временного диапазона и зоны управления дашборда Dynatrace."), ссылка будет распространяться с этими параметрами. В противном случае ссылка создаётся с временным диапазоном **last 2 hours** и зоной управления **All**.
* Если дашборд доступен для зоны управления **All**, получатель ссылки получает доступ к тем зонам управления, которые были доступны вам на момент создания ссылки.

### Отзыв публичной ссылки на дашборд

Чтобы отозвать публичную ссылку на дашборд (доступную без аутентификации)

1. В разделе **Anonymous access** найдите нужную ссылку.
2. Нажмите **X** в столбце **Delete** для этой строки, затем нажмите **Save changes**.

### Внимание: общие ссылки

Общая ссылка имеет разрешения создателя, а не получателя.

* Любой, кто использует созданную вами ссылку, видит всё, что видите вы через эту ссылку (например, зоны управления).
* При потере ваших разрешений пользователи вашей ссылки также теряют эти разрешения.

Для более строгого контроля над тем, кто может просматривать дашборд, используйте варианты аутентифицированного доступа — для отдельного пользователя, группы или среды.
