Глобальные селекторы временного диапазона и зоны управления доступны на всех страницах и представлениях. Их можно найти в правом верхнем углу.

  Элементы управления селектором временного диапазона

  Глобальный селектор временного диапазона служит временны́м фильтром, который в большинстве случаев позволяет выбрать конкретный временной диапазон анализа, сохраняющийся на всех страницах и представлениях при навигации.

  ![Селектор временного диапазона: предустановки](https://dt-cdn.net/images/timeframe-selector-basic-355-f0a835da1e.png)

  Селектор временного диапазона: предустановки

  + Вкладка **Presets** содержит все стандартные временны́е диапазоны. Выберите любой для его применения.
  + Вкладка **Custom** отображает календарь. Нажмите начальный день, затем конечный день и нажмите **Apply** для выбора диапазона.

    - Выбранные интервалы устанавливаются с окончанием в начале следующего дня (время `00:00`). Например, если вы выбрали 3–4 сентября, диапазон начинается 3 сентября в `00:00` и заканчивается **5** сентября в `00:00`, чтобы охватить последнюю минуту диапазона. Отображаемое время можно редактировать.
  + Вкладка **Recent** отображает недавно использованные временны́е диапазоны. Выберите любой для возврата к нему.
  + Кнопки **<** и **>** сдвигают диапазон назад или вперёд во времени. Шаг равен длине исходного диапазона. Например, при текущем диапазоне `Last 2 hours` нажатие **<** сдвинет диапазон на 2 часа назад: `-4h to -2h`.
  + Наведите курсор на временно́й диапазон, чтобы увидеть время начала, продолжительность и время окончания.

    ![Селектор временного диапазона: наведение курсора](https://dt-cdn.net/images/timeframe-selector-hover-168-cfb13dc777.png)

    Селектор временного диапазона: наведение курсора

  Выражения временного диапазона

  При выборе текущего временного диапазона в строке меню отобразится редактируемое выражение.

  + Слева направо выражение содержит время начала, оператор `to` и время окончания.
  + Если явное время окончания отсутствует, `to` и `now` подразумеваются. Например, `-2h` эквивалентно `-2h to now`.
  + Поддерживаемые единицы: `s`, `m`, `h`, `d`, `w`, `M`, `q`, `y` (также можно использовать полные слова: `minutes`, `quarter`)

  | **Примеры выражений** | **Значение** |
  | --- | --- |
  | `today` | От начала сегодняшнего дня до начала следующего. |
  | `yesterday` | От начала вчерашнего дня до начала сегодняшнего. |
  | `yesterday to now` | От начала вчерашнего дня до текущего момента. |
  | `previous week` | Предыдущие семь полных дней. |
  | `this year` | Текущий календарный год от 1 января `00:00` до 1 января следующего года `00:00`. |
  | `last 6 weeks` | Последние 42 дня, заканчивающиеся сейчас. |
  | `-2h` | От 2 часов назад до текущего времени. |
  | `-4d to -1h30m` | От 4 дней назад до 1,5 часов назад. |
  | `-1w` | Последние 7 дней. |
  | `-1w/w` | От начала предыдущей календарной недели до текущего времени. |
  | `-1w/w + 8h` | От начала прошлой недели плюс 8 часов (08:00 в понедельник). |
  | `-1d/d+9h00m to -1d/d+17h00m` | Рабочие часы вчера, с 09:00 до 17:00. |
  | `2020-08-16 21:28 to 2020-08-19 10:02` | Абсолютный диапазон в формате `YYYY-MM-DD hh:mm`. |
  | `1598545932346 to 1598837052346` | Временны́е метки Unix в миллисекундах. |

## Сохранение временного диапазона и зоны управления

Выборки временного диапазона и зоны управления сохраняются и передаются на все посещаемые страницы. Например, после изменения этих параметров на дашборде они сохранятся при переходе с плитки **Applications** к страницам отдельных приложений.

Исключение: открытие нового дашборда или возврат к текущему через кнопку **Dashboard** в верхнем левом углу. В этом случае временной диапазон и зона управления сбрасываются к [значениям по умолчанию](#default) дашборда.

Селектор временного диапазона запоминает до 10 последних значений.

## Временной диапазон и зона управления по умолчанию

Для каждого дашборда можно задать временной диапазон и зону управления по умолчанию — они применяются при каждом открытии дашборда и используются при [предоставлении доступа](/managed/analyze-explore-automate/dashboards-classic/dashboards/share-dashboards "Узнайте, как делиться дашбордами Dynatrace с другими.").

Можно также задать временной диапазон и зону управления по умолчанию для каждой плитки.

### Временной диапазон дашборда

Чтобы задать временной диапазон дашборда, переопределяющий глобальный

1. Перейдите на вкладку **Settings**.
2. Включите **Default timeframe**.
3. Выберите временной диапазон по умолчанию для этого дашборда.

### Зона управления дашборда

Чтобы задать зону управления дашборда, переопределяющую глобальную

1. Перейдите на вкладку **Settings**.
2. Включите **Default management zone**.
3. Выберите зону управления по умолчанию для этого дашборда.

### Временной диапазон плитки

Чтобы задать временной диапазон плитки, переопределяющий временной диапазон дашборда

1. Перейдите в **Dashboards**.
2. Выберите имя дашборда, чтобы открыть его.
3. Нажмите **Edit** в правом верхнем углу дашборда.

   * Если кнопка **Edit** недоступна, у вас нет прав на редактирование.
4. Выберите плитку для настройки.
5. Включите **Custom timeframe**.
6. Выберите временной диапазон по умолчанию для этой плитки.
7. Нажмите **Done**.  
   При установке временного диапазона плитки в её правом верхнем углу отобразится фильтр.

### Зона управления плитки

Чтобы задать зону управления плитки, переопределяющую зону управления дашборда

1. Перейдите в **Dashboards**.
2. Выберите имя дашборда, чтобы открыть его.
3. Нажмите **Edit** в правом верхнем углу дашборда.

   * Если кнопка **Edit** недоступна, у вас нет прав на редактирование.
4. Выберите плитку для настройки.
5. Включите **Custom management zone**.
6. Выберите зону управления по умолчанию для этой плитки.
7. Нажмите **Done**.  
   При установке зоны управления плитки в её правом верхнем углу отобразится фильтр.
