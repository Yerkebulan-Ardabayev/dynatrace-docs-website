# Создание дашбордов Dynatrace для нескольких сред

# Создание дашбордов Dynatrace для нескольких сред

* Практическое руководство
* Чтение: 8 мин
* Обновлено 4 июня 2024 г.

[Dashboards Classic](/managed/analyze-explore-automate/dashboards-classic "Узнайте, как создавать дашборды Dynatrace Classic, управлять ими и использовать их.")

Дашборд Dynatrace может включать мониторинговые артефакты (такие как метрики, логи, события, пользовательские сессии и серверные трассировки) из нескольких сред Dynatrace и даже поддерживать удалённые зоны управления (для плиток, поддерживающих пользовательские зоны управления).

## Обзор

### Преимущества

Дашборды Dynatrace служат единой точкой мониторинга для таких артефактов, как метрики, логи, события и пользовательские сессии. Дашборды для нескольких сред позволяют объединять эти артефакты из разных сред Dynatrace.

После настройки удалённого подключения в локальной среде Dynatrace [вы можете быстро направить плитки дашборда к удалённой среде](#tile).

![Пример: выбор удалённой среды для плитки](https://dt-cdn.net/images/select-tile-environment-example-317-72348cc81f.png)

Пример: выбор удалённой среды для плитки

Для плиток, допускающих настройку зоны управления (ищите параметр **Custom management zone** на панели конфигурации плитки), можно указать другую зону управления.

Поскольку возможности удалённой среды для плиток дашборда позволяют построить общий обзор проблем и других данных, дашборды могут служить центром для более глубокого анализа. Один щелчок по любой плитке дашборда, отображающей удалённую информацию, переведёт вас прямо в специальное представление удалённой среды, где можно продолжить анализ.

### Ограничения

* Плитка удалённой среды обратно совместима не более чем на пять версий.

  + Плитка удалённой среды, работающая в окружении с версией Dynatrace *x*, будет корректно функционировать с версией Dynatrace *x*-5 или более поздней, развёрнутой в удалённой среде.

    Если разница между локальной и удалённой версиями превышает пять версий, плитка может работать, но такая конфигурация не поддерживается.
  + Плитка удалённой среды отображает данные на основе функций, включённых в версию кластера удалённой среды.

    Например, если плитка удалённой среды зависит от функции, добавленной в версии Dynatrace *x*, и эта версия развёрнута локально, но удалённая среда всё ещё работает на версии Dynatrace *x*-1, функция не будет отображаться на локальном дашборде, поскольку удалённая среда её не поддерживает.

    Если плитка удалённой среды зависит от функции, не поддерживаемой в удалённой среде, плитка отображает сообщение об ошибке, объясняющее расхождение между локальной и удалённой средами.
  + Для обеспечения максимальной совместимости между локальной и удалённой средами рекомендуется использовать одну версию Dynatrace.
* Удалённые соединения не передают пользовательский контекст и права доступа через границы сред.

  По этой причине рекомендуется использовать зоны управления для сегментирования/ограничения плиток дашборда при просмотре удалённой информации.

  С учётом вышесказанного дашборды для удалённых сред могут настраивать только администраторы среды. Обычные пользователи по-прежнему могут просматривать дашборды для удалённых сред и взаимодействовать с ними без ограничений.
* Плитка дашборда «Карта мира» не предназначена для сценариев с удалёнными средами (и поэтому их не поддерживает).

## Настройка

Чтобы создать плитку дашборда, запрашивающую данные из удалённой среды:

* В удалённой среде Dynatrace [создайте токен доступа](#token), позволяющий запрашивать данные из этой среды
* В локальной среде Dynatrace [добавьте удалённую среду](#link) в таблицу удалённых сред
* В локальной среде Dynatrace [создайте одну или несколько плиток, отображающих удалённые данные](#tile), запрашиваемые из удалённой среды

Альтернативы через API

Описанные ниже процедуры используют веб-интерфейс Dynatrace. Для выполнения аналогичных задач через API см.:

* [Access tokens API](/managed/dynatrace-api/environment-api/tokens-v2/api-tokens "Управление токенами аутентификации Dynatrace API.") — для создания токена в удалённой среде
* [Remote environments API](/managed/dynatrace-api/configuration-api/remote-environments "Управление конфигурациями удалённых сред Dynatrace через API конфигурации.") — для создания ссылки на удалённую среду из локальной
* [Dashboards API](/managed/dynatrace-api/configuration-api/dashboards-api "Узнайте, как управлять конфигурацией дашборда через классический API конфигурации Dynatrace.") — для настройки дашборда с плитками, запрашивающими удалённую среду

### Создание токена доступа

Данная процедура позволяет получить токен доступа из удалённой среды, необходимый для остальных шагов.

Чтобы создать токен доступа для удалённой среды Dynatrace

1. Войдите в удалённую среду.

   * Это среда, из которой вы получаете данные.
   * Если у вас нет доступа к удалённой среде, эту процедуру может выполнить кто-либо с соответствующими правами.
2. Перейдите в **Access Tokens**.
3. Нажмите **Generate new token**.
4. Введите имя токена.
5. Найдите область действия **Fetch data from a remote environment** (`RestRequestForwarding`) и установите её флажок.
6. Нажмите **Generate token**.  
   Будет создан токен, дающий вашей локальной среде право на получение данных из этой удалённой среды.
7. Нажмите **Copy** и сохраните токен в надёжном месте.  
   Это длинная строка, которую потребуется скопировать и вставить в Dynatrace позже.

### Добавление удалённой среды

Чтобы добавить удалённую среду Dynatrace в список доступных удалённых сред

1. Войдите в локальную среду Dynatrace.
2. Перейдите в **Settings**.
3. Выберите **Integration** > **Remote environments**.
4. Нажмите **Connect environment**.
5. Определите удалённую среду, из которой локальная среда получает данные, и нажмите **Save changes**.

   * **Name** — имя, под которым эта среда будет отображаться в локальной среде Dynatrace при настройке плитки для запроса данной удалённой среды. Произвольный текст; не влияет на удалённую среду.
   * **Remote environment URI**

     + Для Dynatrace SaaS формат должен быть следующим:

       `https://<ENVIRONMENTID>.live.dynatrace.com/`

       Замените `<ENVIRONMENTID>` на фактический идентификатор среды.
     + Для Dynatrace Managed допустим любой URI.
     + Для подключения среды Dynatrace (SaaS) к развёртыванию Dynatrace Managed через URI за пределами домена `dynatrace-managed.com` обратитесь к специалисту Dynatrace через живой чат в своей среде.
   * **Network scope**

     + `External`: удалённая среда находится в другой сети. При наличии используются глобально настроенные параметры прокси. Это область по умолчанию.
     + `Internal`: удалённая среда находится в той же сети. Глобально настроенные параметры прокси не используются.
     + `Cluster`: удалённая среда находится в том же кластере. Запрос направляется на `localhost`.

     Dynatrace SaaS может использовать только область `External`.
   * **Token** — токен, созданный в предыдущей процедуре. Он должен включать область **Fetch data from a remote environment** (`RestRequestForwarding`).
   * **Test connection** — проверяет соединение из локальной среды с удалённой.

     Убедитесь, что получено сообщение `connection successfully established`, прежде чем продолжить.

Теперь, когда соединение с удалённой средой установлено, можно создавать плитки дашборда, запрашивающие эту среду.

### Создание плитки для отображения удалённых данных

Чтобы создать плитку, получающую данные из удалённой среды

1. Откройте дашборд, на котором будет отображаться плитка.
2. Нажмите **Edit**.
3. Выберите или добавьте плитку, на которой нужно отобразить данные из удалённой среды.  
   Раздел **Environment** панели настройки плитки отображает имя среды, из которой плитка получает данные мониторинга.

   * **Default (local)** — плитка получает данные из локальной среды Dynatrace.
   * Все остальные перечисленные среды — удалённые среды, для которых установлено соединение.
4. Выберите удалённую среду, к которой должна обращаться выбранная плитка.  
   Если при добавлении удалённой среды в предыдущей процедуре вы назвали её `Boston`, она должна быть в этом списке.

   ![Пример: выбор удалённой среды для плитки](https://dt-cdn.net/images/select-tile-environment-example-317-72348cc81f.png)

   Пример: выбор удалённой среды для плитки
5. Нажмите **Done**, чтобы отобразить готовый дашборд.
6. Наведите курсор на значок фильтра плитки, чтобы увидеть выбранную среду.

   ![Пример: отображение фильтров плитки для просмотра выбранной удалённой среды](https://dt-cdn.net/images/tile-remote-environment-tooltip-example-495-96be8d3ec4.png)

   Пример: отображение фильтров плитки для просмотра выбранной удалённой среды

## Примеры

### Одинаковые плитки, разные среды

В этом примере мы создадим дашборд, отображающий состояние хостов и состояние сети для локальной и удалённой среды рядом. Предполагается, что удалённая среда уже добавлена в локальную.

1. Перейдите в **Dashboards**, нажмите **Create dashboard**, задайте имя и нажмите **Create**.
2. Перетащите две плитки **Header** на дашборд (или перетащите одну, а затем клонируйте её).
3. Отредактируйте один заголовок, введя **Local**, а другой — **Remote**.
4. Перетащите две плитки **Host health** на дашборд (или перетащите одну, а затем клонируйте её).  
   Разместите одну под заголовком **Local**, а другую — под заголовком **Remote**.
5. Перетащите две плитки **Network status** на дашборд (или перетащите одну, а затем клонируйте её).  
   Разместите одну под заголовком **Local**, а другую — под заголовком **Remote**.

   На этом этапе все плитки запрашивают локальную среду, то есть у вас две одинаковые плитки **Host health** и две одинаковые плитки **Network status**.

   ![Одинаковые плитки, одинаковые среды](https://dt-cdn.net/images/same-tiles-same-environments-595-51063546a9.png)

   Одинаковые плитки, одинаковые среды
6. Под заголовком **Remote** отредактируйте плитки **Host health** и **Network status** так, чтобы обе обращались к удалённой среде (в данном примере — `Boston`).

   ![Пример: выбор удалённой среды для плитки](https://dt-cdn.net/images/select-tile-environment-example-317-72348cc81f.png)

   Пример: выбор удалённой среды для плитки
7. Нажмите **Done**, чтобы отобразить дашборд.

   * Плитки под заголовком **Local** по-прежнему отображают локальные данные о хостах и сети.
   * Плитки под заголовком **Remote** теперь обращаются к удалённой среде:

     + Удалённые плитки отображают состояние хостов и сети для удалённой среды, а не для локальной.
     + На каждой удалённой плитке отображается значок фильтра. Наведите курсор на значок, чтобы увидеть имя среды.

   ![Одинаковые плитки, разные среды](https://dt-cdn.net/images/same-tiles-different-environments-595-d747473897.png)

   Одинаковые плитки, разные среды

Разумеется, можно добавлять другие типы плиток и указывать дополнительные удалённые среды.
