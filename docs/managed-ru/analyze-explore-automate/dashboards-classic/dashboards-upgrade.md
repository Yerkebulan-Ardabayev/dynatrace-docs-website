# Обновите пользовательские диаграммы прямо сейчас

* Практическое руководство
* Чтение: 11 мин
* Updated on Aug 02, 2023

[Dashboards Classic](/managed/analyze-explore-automate/dashboards-classic "Узнайте, как создавать дашборды Dynatrace Classic, управлять ими и использовать их.")

Если вы используете **Custom charts** в своих дашбордах, прочитайте следующий текст.

## Требуется немедленное действие

* **Что:**

  Если вы используете пользовательские диаграммы, необходимо **обновить их прямо сейчас**.

  + Пользовательские диаграммы больше не получают исправления и улучшения.
  + Пользовательские диаграммы больше нельзя редактировать.

  При обновлении пользовательских диаграмм до диаграмм [Data Explorer](/managed/analyze-explore-automate/explorer "Выполняйте запросы метрик и преобразуйте результаты для получения нужных аналитических данных.") вы сможете редактировать диаграммы, пользоваться исправлениями и улучшениями Data Explorer, а также быть готовы к переходу на новую платформу.
* **Когда:**

  + Dynatrace версии 1.265 была последним выпуском, поддерживавшим пользовательские диаграммы
  + Dynatrace SaaS версии 1.271+ и Dynatrace Managed версии 1.272+ полностью отключили редактирование пользовательских диаграмм и прекратили поддержку их кода
* **Как:**

  Обновление можно выполнять плитку за плиткой, дашборд за дашбордом, или сразу для всех через API.

Подробнее см. ниже.

## Относится ли это ко мне?

Это не относится к визуализациям и дашбордам, созданным с помощью Data Explorer. Это касается только старой функциональности **Custom charts**, которая предшествовала Data Explorer и теперь полностью устарела.

### Проверка таблицы дашбордов

Чтобы узнать, есть ли у вас устаревшие пользовательские диаграммы для обновления, отфильтруйте таблицу **Dashboards** по `Custom charts: Yes`.

1. Перейдите в **Dashboards**.
2. Необязательно Чтобы отобразить только свои дашборды, добавьте фильтр `Ownership: Mine`.

   ![Фильтрация таблицы Dashboards по Ownership: Mine](https://dt-cdn.net/images/dashboards-filter-ownership-mine-2-286-23183a2be4.png)

   Фильтрация таблицы Dashboards по Ownership: Mine

   Чтобы проверить все дашборды, к которым у вас есть доступ, пропустите этот шаг.
3. Добавьте фильтр `Custom charts: Yes`.

   ![Фильтрация таблицы Dashboards по Custom charts: Yes](https://dt-cdn.net/images/dashboards-filter-custom-charts-select-404-6e177a0487.png)

   Фильтрация таблицы Dashboards по Custom charts: Yes
4. Проверьте таблицу

   * **Если после фильтрации по `Custom charts: Yes` дашборды не отображаются — вам нечего обновлять. Можно остановиться здесь.**
   * Если вы не фильтровали по владельцу дашборда, могут отображаться некоторые предустановленные дашборды с владельцем `Dynatrace`. Их можно игнорировать.
   * Если отображаются другие дашборды, у каждого должна быть метка `Custom charts`, так как вы установили фильтр `Custom charts: Yes`.

     В приведённом примере необходимо обновить два дашборда.

     ![Метка пользовательской диаграммы в таблице Dashboards с фильтрами Ownership: Mine и Custom charts: Yes](https://dt-cdn.net/images/dashboards-filter-custom-charts-effect-693-134256be1b.png)

     Метка пользовательской диаграммы в таблице Dashboards

### Проверка плиток на дашборде

Найдя дашборд с пользовательскими диаграммами, откройте его и обратите внимание на следующие признаки необходимости обновления:

* Плитка, требующая обновления, отображает значок предупреждения и подсказку.

  ![Подсказка о необходимости обновить плитку](https://dt-cdn.net/images/tile-tooltip-740-69c663200c.png)

  Подсказка о необходимости обновить плитку
* Если на текущем дашборде есть хотя бы одна плитка для обновления, в меню дашборда (правый верхний угол) будет доступен параметр **Upgrade**.

  ![Параметр Upgrade в меню дашборда](https://dt-cdn.net/images/custom-charts-upgrade-in-dashboard-menu-126-8eb637983a.png)

  Параметр Upgrade в меню дашборда

## Как выполнить обновление?

Прежде всего рекомендуется создать резервные копии наиболее важных дашбордов. Затем попробуйте обновить копии, чтобы освоить процесс перед обновлением всех остальных дашбордов.

Например, выберите любой популярный дашборд, от которого вы зависите, возможно тот, где есть несколько сложных плиток **Custom chart** (используются несколько метрик, настроены единицы, скорости и т. д.), и клонируйте его. После этого можно безопасно работать с клоном и проверять корректность обновления.

Сортировка дашбордов по популярности

1. Перейдите в **Dashboards**.  
   Отобразятся все дашборды, которые вы можете просматривать или редактировать.
2. Отсортируйте таблицу по столбцу **Popularity**.

Клонирование существующего дашборда

1. Перейдите в **Dashboards**.
2. В таблице дашбордов для нужного дашборда выберите **More** (**…**) > **Clone**.

   * Копия откроется в режиме редактирования.
   * Исходный дашборд останется без изменений.

Варианты обновления:

* [По одной плитке](#web-one-tile)
* [Весь дашборд целиком (все плитки)](#web-entire-dashboard)
* [Все дашборды сразу через Dynatrace API](#api)

Во всех случаях создавайте резервные копии перед началом работы.

## Обновление через веб-интерфейс

В веб-интерфейсе можно обновлять по одной плитке или по одному дашборду.

### Обновление одной плитки

Чтобы обновить одну плитку

1. Перейдите в **Dashboards**.
2. Откройте дашборд с плиткой, требующей обновления.

   ![Подсказка о необходимости обновить плитку](https://dt-cdn.net/images/tile-tooltip-740-69c663200c.png)

   Подсказка о необходимости обновить плитку
3. В меню плитки выберите **Preview upgraded chart**.
   Конвертированная диаграмма откроется в Data Explorer в новой вкладке браузера, чтобы вы могли сравнить старую и новую визуализации.

   ![Меню плитки: Preview upgraded chart](https://dt-cdn.net/images/tile-preview-upgrade-280-8ea1aabb99.png)

   Меню плитки: Preview upgraded chart
4. Проверьте визуализацию и убедитесь в корректности конвертации. При необходимости внесите изменения.
5. Если результат вас устраивает, нажмите **Pin to dashboard** и закрепите обновлённую визуализацию на другом или том же дашборде.

   * При закреплении на том же дашборде у вас будут обе плитки. Это даст ещё одну возможность сравнить их. Если всё устраивает, удалите старую плитку (со значком предупреждения) и перетащите новую на её место.

### Обновление всего дашборда

Эти процедуры можно выполнять с созданием клонов (резервных копий) или без.

#### Быстрее, но без создания резервных копий

Обновление всего дашборда (всех плиток сразу) без создания клона.

1. Перейдите в **Dashboards**.
2. Найдите дашборд в таблице.
3. В строке дашборда выберите > **Upgrade**. При отсутствии ошибок весь дашборд будет обновлён.

   ![Параметр Upgrade в таблице Dashboards для конкретного дашборда](https://dt-cdn.net/images/custom-charts-upgrade-in-table-menu-140-971e7f8c53.png)

   Параметр Upgrade в таблице Dashboards для конкретного дашборда

#### Медленнее, но с созданием резервной копии

Создание клона (резервной копии) дашборда и обновление всего клонированного дашборда.

1. Перейдите в **Dashboards**.
2. Найдите дашборд в таблице.
3. В строке дашборда выберите > **Clone**. Будет создана копия дашборда и открыта в режиме редактирования. Исходный дашборд останется без изменений.
4. Нажмите **Done** для выхода из режима редактирования.
5. В правом верхнем углу дашборда выберите > **Upgrade**.

   ![Параметр Upgrade в таблице Dashboards для конкретного дашборда](https://dt-cdn.net/images/custom-charts-upgrade-in-table-menu-140-971e7f8c53.png)

   Параметр Upgrade в таблице Dashboards для конкретного дашборда
6. Проверьте обновлённый дашборд.

   Для сравнения двух дашбордов бок о бок

   1. Перейдите в **Dashboards**.
   2. Щёлкните правой кнопкой мыши по имени первого дашборда и откройте ссылку в новом окне.
   3. Щёлкните правой кнопкой мыши по имени второго дашборда и откройте ссылку в новом окне.
   4. Расположите окна браузера бок о бок и визуально сравните дашборды.
7. Убедившись в корректности обновления, можно удалить исходный дашборд.

#### Альтернативная процедура через меню дашборда

Эту процедуру можно выполнять с созданием резервной копии (шаг 3) или без.

1. Перейдите в **Dashboards**.
2. Выберите имя дашборда для его открытия.
3. Рекомендуется Если резервная копия ещё не создана, можно сделать это сейчас: в правом верхнем углу дашборда выберите > **Clone** для создания и отображения копии. При клонировании отображается клон, исходный дашборд остаётся без изменений.
4. В правом верхнем углу дашборда выберите > **Upgrade**.

   * Если на предыдущем шаге вы клонировали дашборд, на этом шаге обновляется клон.
   * Если клонирование не выполнялось, на этом шаге обновляется исходный дашборд без создания резервной копии.

   ![Параметр Upgrade в меню дашборда](https://dt-cdn.net/images/custom-charts-upgrade-in-dashboard-menu-126-8eb637983a.png)

   Параметр Upgrade в меню дашборда

После завершения обновления дашборда:

* В нижней части дашборда кратко отобразится сообщение: **The dashboard was successfully upgraded!**
* Плитки на этом дашборде больше не будут отображать значок предупреждения, так как все они обновлены до плиток Data Explorer
* Параметр **Upgrade** в меню дашборда больше не будет доступен

### Проверка наличия других дашбордов для обновления

1. Перейдите в **Dashboards**.
2. Отфильтруйте страницу по `Custom charts: Yes`.

* Уже обновлённые дашборды не должны отображаться
* Дашборды с меткой `Custom charts` нуждаются в обновлении.

Подробнее об идентификации дашбордов для обновления см. в разделе [Относится ли это ко мне?](#does-this-apply-to-me) выше.

## Обновление через API

Dynatrace версии 1.251+

Через Dynatrace API можно обновить один дашборд или все сразу.
API удобно пропускает дашборды без **Custom charts**, что избавляет от необходимости предварительной фильтрации.

### Обновление одного дашборда

Для обновления одного дашборда через API используйте конечную точку migrate.

```
https://{your-environment-id}/rest-api-doc/index.jsp?urls.primaryName=Configuration%20API#/Dashboards/migrateDashboard
```

Замените `{your-environment-id}` на ваш фактический [ID среды](/managed/discover-dynatrace/get-started/monitoring-environment "Узнайте о средах мониторинга и работе с ними.").

### Обновление всех дашбордов

Чтобы обновить все дашборды через API

1. Используйте конечную точку GET для получения всех ID дашбордов.

   ```
   https://{your-environment-id}/rest-api-doc/index.jsp?urls.primaryName=Configuration%20API#/Dashboards/getDashboardStubsList
   ```

   Замените `{your-environment-id}` на ваш фактический [ID среды](/managed/discover-dynatrace/get-started/monitoring-environment "Узнайте о средах мониторинга и работе с ними.").
2. Переберите все ID, как описано в разделе [Обновление одного дашборда](#api-single) выше.

## Что делать, если обновление завершается неудачно?

В зависимости от сложности плиток обновление дашборда может завершиться частичной неудачей (некоторые плитки обновляются успешно, другие нет) или полной неудачей (ни одна плитка не обновлена).

В любом случае не стоит беспокоиться о потере данных. Дашборды и плитки — это визуализации данных, а не сами данные. Данные никогда не подвергаются риску при обновлении.
