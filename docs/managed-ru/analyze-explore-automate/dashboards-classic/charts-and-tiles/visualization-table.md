## Смена визуализации

При переключении между визуализациями учтите, что некоторые параметры характерны для конкретной визуализации.

* При переключении часть настроек может быть проигнорирована. Значок информации предупредит об этом.

  ![Data Explorer: предупреждение о смене визуализации](https://dt-cdn.net/images/change-visualization-warning-1277-9438aaceea.png)

  Data Explorer: предупреждение о смене визуализации
* При возврате к исходной визуализации может потребоваться повторная настройка.

## Изменение набора метрик

Визуализация может отображать любой набор метрик в запросе с несколькими метриками.

Чтобы включать или отключать метрики, выберите букву рядом с нужной метрикой или нажмите значок глаза.

![Data Explorer: переключение метрики](https://dt-cdn.net/images/eye-toggle-metric-1238-d343b9cc0e.png)

Data Explorer: переключение метрики

## Сортировка по столбцу и порядку

По умолчанию таблицы Data Explorer сортируются по первой метрике в запросе (по убыванию на основе выбранной агрегации).

Чтобы отсортировать таблицу по другому столбцу, необходимо изменить порядок столбцов.

1. Найдите метрику, по которой хотите отсортировать таблицу.
2. Перетащите эту метрику на первую позицию в определении запроса.
3. Запустите запрос.

   Таблица будет отсортирована по метрике, перемещённой на первое место.

Чтобы задать порядок сортировки

1. Если **Sort by** ещё не отображается для метрики, выберите и затем `Sort by` из списка.
2. Установите **Sort by** для нужного измерения.
3. Выберите порядок сортировки: `ASC` (по возрастанию) или `DESC` (по убыванию).

## Настройки

Раздел **Settings** — один из разворачиваемых разделов на правой панели Data Explorer.

![Data Explorer: раздел Settings](https://dt-cdn.net/images/data-explorer-settings-settings-322-7b365c52e5.png)

Data Explorer: раздел Settings

### Трансформация свёртки

Трансформация свёртки объединяет список точек данных (временной ряд) в одну точку данных.

* По умолчанию — `Auto`.
* Доступные варианты: `Last value`, `Average`, `Count`, `Maximum`, `Minimum`, `Sum`, `Median`, `Value`, `Percentile 10th`, `Percentile 75th`, `Percentile 90th`.

При `Auto` для `Table`, `Single value`, `Top list` или `Honeycomb` используется разрешение `Inf`. Чтобы проверить запрос — перейдите в раздел **Result** и выберите > **Copy request**.

## Настройки по метрике

Имена столбцов таблицы совпадают с именами метрик по умолчанию. Можно изменить заголовки столбцов.

Чтобы переименовать столбец

1. В разделе **Settings** выберите для метрики/столбца, который хотите переименовать.

   ![Переименование метрики](https://dt-cdn.net/images/single-value-rename-metric-232-266-b358f19b2b.png)

   Переименование метрики
2. Отредактируйте имя и выберите галочку для сохранения.

### Единица измерения и формат

* **Unit**: `None`, `Auto`, или конкретная единица. В **Advanced mode** — `:setUnit(<unit>)`.
* **Format**: `None`, `Auto`, `0`, `0.0`, `0.00`, `0.000`.

| Обозначение | Множитель | Значение |
| --- | --- | --- |
| k | 10^3 | кило, тысяча |
| M | 10^6 | мега, миллион |
| G | 10^9 | гига, миллиард |
| T | 10^12 | тера, триллион |

## Столбцы

Используйте флажки в разделе **Columns** визуализации таблицы Data Explorer, чтобы включать и отключать отдельные столбцы. Эти настройки отражаются в итоговых плитках дашборда.

## Строки

### Data Explorer

По умолчанию количество результатов ограничено 100.

Чтобы задать явный лимит

1. Если **Limit** ещё не отображается в редакторе запросов, выберите и затем `Limit` из списка.
2. Установите **Limit** в одно из доступных значений.

Используйте **Limit** > **Maximum** для снятия ограничения, либо включите **Advanced mode** и удалите `:limit(100)` из запроса.

### Плитка таблицы на дашборде

Максимальное количество строк на плитке определяется лимитом запроса в Data Explorer. Результаты отображаются постранично.

## Страницы

Используйте элементы управления в нижней части раздела **Result** для перелистывания страниц таблицы.

* Плитки таблицы на дашборде также имеют элементы управления страницами.
* Размер страницы плитки автоматически подстраивается под размер плитки.

## Часто задаваемые вопросы

Почему отображаются не все ряды метрики?

* *По умолчанию* отображается не более `20` рядов на метрику. Некоторые ряды могут отсутствовать в Data Explorer. Для отображения нужных рядов укажите более конкретные фильтры: зону управления или фильтр по имени объекта.
* *Максимальное* количество отображаемых рядов на метрику — `100`. Этот лимит применяется даже при удалении [трансформации **limit**](/managed/dynatrace-api/environment-api/metric-v2/metric-selector#limit "Настройте селектор метрик для API Metric v2.") из селектора метрик на [вкладке **Code**](/managed/analyze-explore-automate/explorer/explorer-advanced-query-editor "Создавайте расширенные запросы с помощью расширенного режима Data Explorer.").

Если данные для выражения метрики отсутствуют, см. раздел [Почему результат моего выражения метрики пуст?](/managed/dynatrace-api/environment-api/metric-v2/metric-faq#empty-result-metric-expression "Часто задаваемые вопросы по Metrics API v2.").

Почему некоторые ячейки таблицы пусты, хотя должны содержать значения?

Первопричина этой проблемы часто та же, что и в предыдущем вопросе.

Ряды метрик ограничены определённым числом.

Например, при запросе `builtin:host.cpu.usage` и `builtin:host.cpu.idle` с разбивкой по `dt.entity.host` по умолчанию запрашивается топ-100 хостов для каждой метрики. Но топ-100 хостов для `builtin:host.cpu.usage` вероятно отличается от топ-100 для `builtin:host.cpu.idle`, что приводит к пустым ячейкам для некоторых хостов.
