## Смена визуализации

При переключении между визуализациями учтите, что некоторые параметры настройки характерны для конкретной визуализации.

* Если выбрать визуализацию, настроить для неё **Settings**, а затем переключиться на другую визуализацию, часть настроек первой визуализации может быть проигнорирована, так как они неприменимы к новой. Значок информации в списке визуализаций предупредит об этой возможности.

  ![Data Explorer: предупреждение о смене визуализации](https://dt-cdn.net/images/change-visualization-warning-1277-9438aaceea.png)

  Data Explorer: предупреждение о смене визуализации
* При возврате к исходной визуализации может потребоваться повторная настройка некоторых параметров.

## Изменение набора метрик

По умолчанию эта визуализация отображает первую метрику из запроса с несколькими метриками.

Чтобы выбрать другую метрику из запроса с несколькими метриками, выберите букву рядом с нужной метрикой или нажмите значок глаза.

## Настройки

Раздел **Settings** — один из разворачиваемых разделов на правой панели Data Explorer. Содержимое раздела **Settings** может варьироваться в зависимости от выбранной визуализации.

![Data Explorer: раздел Settings](https://dt-cdn.net/images/data-explorer-settings-settings-322-7b365c52e5.png)

Data Explorer: раздел Settings

### Разрешение

Разрешение — это детализация оси X (времени) визуализации.

* Чтобы Data Explorer автоматически выбирал подходящее разрешение для выбранного временного диапазона, в разделе **Settings** установите **Resolution** в `Auto`
* Чтобы задать конкретное разрешение, выберите его из списка

Некоторые разрешения недоступны для определённых временных диапазонов. Если выбрана несовместимая комбинация, Dynatrace автоматически подберёт разрешение и выведет пояснительное сообщение, например: `Auto-resolution applied. Resolution value of [6 hours] applied. Selected timeframe doesn't allow for [5 minutes] resolution.` Чтобы переопределить автоматическое разрешение, выберите другое из списка.

Умное разрешение на плитках дашборда

Во избежание проблем с производительностью на плитках дашборда, созданных с помощью Data Explorer, максимальное количество точек данных для запроса на плитке составляет 4 000. На основе выбранного временного диапазона и заданного пользовательского разрешения Dynatrace оценивает количество точек данных. Если прогнозируемое количество превышает 4 000, Dynatrace автоматически переключается на более высокое разрешение.

Обратите внимание, что это не применяется к визуализациям в самом Data Explorer. Ограничение распространяется только на плитки дашборда, созданные с помощью Data Explorer.

### Показать легенду

Определяет, отображать ли легенду, сопоставляющую цвета с диапазонами значений.

### Показать метки

Определяет, отображать ли значение (количество) для каждого диапазона.

### Ось Y

Определяет, отображать ли ось Y по значению или с разбивкой по измерениям.

### Диапазоны

Количество диапазонов, отображаемых на каждой оси.

* Значение по умолчанию для каждой оси — `Auto`.
* Чтобы задать количество диапазонов вручную, введите число. Чтобы вернуть значение `Auto`, очистите поле.

## Настройки по метрике

Раздел **Settings** также отображает параметры визуализации для каждой метрики, выбранной в запросе.

### Переименование метрики

Можно изменить имя метрики, отображаемое на диаграмме и в её легенде. Определение запроса сохранит исходное имя метрики.

1. В разделе **Settings** выберите для метрики, которую хотите переименовать.
2. Отредактируйте имя и выберите галочку для сохранения нового имени.

### Изменение цветовой палитры

Чтобы изменить цветовую палитру для метрики, в разделе **Settings** выберите новую палитру из списка.

![Data Explorer: список цветовых палитр метрик](https://dt-cdn.net/images/metric-color-palette-list-67-fd8f7395d1.png)

Data Explorer: список цветовых палитр метрик

### Единица измерения и формат

Используйте параметры **Unit** и **Format** для определения способа отображения данных. При экспорте в CSV-файл параметры также применяются к экспортируемым значениям.

#### Единица измерения

Используйте параметр **Unit** для задания единицы измерения.

* `None` = Единица измерения не отображается
* `Auto` = Dynatrace выбирает подходящую единицу измерения
* Другие варианты задают конкретную единицу для отображения.
* Чтобы добавить пользовательскую единицу/суффикс, введите строку в поле **Unit** и выберите её из списка.
* В **Advanced mode** можно использовать `:setUnit(<unit>)` для выбора из более широкого диапазона единиц.

Примеры обозначений порядков величин:

| Обозначение | Множитель | Значение |
| --- | --- | --- |
| k | 10^3 | кило, тысяча |
| M | 10^6 | мега, миллион |
| G | 10^9 | гига, миллиард |
| T | 10^12 | тера, триллион |

#### Формат

Используйте параметр **Format** для настройки количества десятичных знаков.

* `None` = Без форматирования.
* `Auto` = Dynatrace выбирает подходящий формат.
* Другие варианты задают количество десятичных знаков: `0`, `0.0`, `0.00`, `0.000`

#### Примеры

Пример: байты, доллары и центы, счётчики, пороги — аналогичны описанию в других визуализациях.

## Пороговые значения

Раздел **Threshold** — один из разворачиваемых разделов на правой панели Data Explorer. Используйте настройки порогов для улучшения визуализаций и плиток.

![Data Explorer: раздел Threshold](https://dt-cdn.net/images/data-explorer-settings-threshold-323-3ddaf51ec3.png)

Data Explorer: раздел Threshold

Устанавливайте значения порогов после выбора **Unit**:

* Если сначала выбрать **Unit**, параметры порогов будут подготовлены для соответствия выбранной единице.
* Если изменить **Unit** после задания значений порогов, значения порогов не будут автоматически пересчитаны.

### Установка порогов

1. В разделе **Thresholds** введите значения порогов

   ![Установка значений порогов](https://dt-cdn.net/images/threshold-definition-set-threshold-values-graph-249-e0895fe995.png)

   Установка значений порогов
2. Настройте цвета порогов Необязательно

   ![Настройка цветов порогов](https://dt-cdn.net/images/threshold-definition-adjust-threshold-color-269-2e5d1f3fb4.png)

   Настройка цветов порогов

### Скрытие и отображение цветов порогов

Чтобы скрыть или показать цвета порогов без удаления настроек, в разделе **Thresholds** нажмите.
