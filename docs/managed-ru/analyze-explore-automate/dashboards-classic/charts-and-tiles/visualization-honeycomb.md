Чтобы визуализировать результаты запроса в виде сот, выберите `Honeycomb` из списка над определением запроса в левом верхнем углу страницы.

Ограничение ячеек

По умолчанию количество ячеек в визуализации «Соты» ограничено 100. Если нужно отобразить больше ячеек, используйте **Limit** > **Maximum** или включите **Advanced mode** и удалите `:limit(100)` из запроса.

## Смена визуализации

При переключении между визуализациями учтите, что некоторые параметры настройки характерны для конкретной визуализации.

* Если выбрать визуализацию, настроить для неё **Settings**, а затем переключиться на другую визуализацию, часть настроек первой визуализации может быть проигнорирована. Значок информации предупредит об этой возможности.

  ![Data Explorer: предупреждение о смене визуализации](https://dt-cdn.net/images/change-visualization-warning-1277-9438aaceea.png)

  Data Explorer: предупреждение о смене визуализации
* При возврате к исходной визуализации может потребоваться повторная настройка некоторых параметров.

## Настройки

Раздел **Settings** — один из разворачиваемых разделов на правой панели Data Explorer.

![Data Explorer: раздел Settings](https://dt-cdn.net/images/data-explorer-settings-settings-322-7b365c52e5.png)

Data Explorer: раздел Settings

### Показать легенду и показать соты

Отображение сот и легенды можно настраивать независимо.

* Если выбрать только **Show legend**, легенда расширяется и становится единственным визуальным представлением результатов запроса.
* Если выбрать только **Show hive**, соты расширяются и становятся единственным визуальным представлением.
* При выборе обоих параметров (как в примере выше) соты являются основной визуализацией, а легенда отображается под ними.

### Показать метки

При выборе **Show labels** каждая ячейка в сотах отображает метку и значение, как в примере выше.

### Трансформация свёртки

Трансформация свёртки объединяет список точек данных (временной ряд: набор точек данных за период) в одну точку данных.

* Трансформация свёртки по умолчанию — `Auto`, которая автоматически выбирает наиболее подходящую временну́ю агрегацию на основе метрики.
* Можно вручную переопределить `Auto` одним из следующих вариантов: `Last value`, `Average`, `Count`, `Maximum`, `Minimum`, `Sum`, `Median`, `Value`, `Percentile 10th`, `Percentile 75th`, `Percentile 90th`.
* Если нужно видеть последнее зафиксированное значение метрики (а не агрегацию), выберите `Last value`.

Трансформация свёртки и разрешение

Обратите внимание, что параметр **Fold transformation** влияет на разрешение.

* Если **Fold transformation** установлен в `Auto` для визуализации `Table`, `Single value`, `Top list` или `Honeycomb`, используется разрешение `Inf` (бесконечность) для обеспечения обратной совместимости. Если выбранный селектор метрик не поддерживает разрешение `Inf`, трансформация `fold` автоматически добавляется в конец запроса.
* Если **Fold transformation** задан в значении, отличном от `Auto`, используется `fold`.

Поскольку все селекторы метрик запрашиваются с использованием одного и того же механизма суммарного значения (`fold` или `Inf`), добавление нового селектора, требующего `fold`, может изменить результаты других селекторов.

Чтобы проверить фактический запрос, используемый Data Explorer, перейдите в раздел **Result** в Data Explorer и выберите > **Copy request**.

## Настройки по метрике

Раздел **Settings** также отображает параметры визуализации для каждой метрики, выбранной в запросе.

### Переименование метрики

1. В разделе **Settings** выберите для метрики, которую хотите переименовать.
2. Отредактируйте имя и выберите галочку для сохранения.

### Изменение цветовой палитры

Чтобы изменить цветовую палитру для метрики, в разделе **Settings** выберите новую палитру из списка.

![Data Explorer: список цветовых палитр метрик](https://dt-cdn.net/images/metric-color-palette-list-67-fd8f7395d1.png)

Data Explorer: список цветовых палитр метрик

### Единица измерения и формат

Используйте параметры **Unit** и **Format** для определения способа отображения данных. При экспорте в CSV-файл параметры также применяются к экспортируемым значениям.

#### Единица измерения

* `None` = Единица измерения не отображается
* `Auto` = Dynatrace выбирает подходящую единицу измерения
* Другие варианты задают конкретную единицу для отображения.
* Для пользовательской единицы/суффикса введите строку в поле **Unit** и выберите её из списка.
* В **Advanced mode** можно использовать `:setUnit(<unit>)`.

| Обозначение | Множитель | Значение |
| --- | --- | --- |
| k | 10^3 | кило, тысяча |
| M | 10^6 | мега, миллион |
| G | 10^9 | гига, миллиард |
| T | 10^12 | тера, триллион |

#### Формат

* `None` = Без форматирования.
* `Auto` = Dynatrace выбирает подходящий формат.
* Другие варианты: `0`, `0.0`, `0.00`, `0.000`
