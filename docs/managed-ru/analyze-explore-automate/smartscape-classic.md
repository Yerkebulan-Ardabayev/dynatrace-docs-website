# Визуализация среды с помощью Smartscape

* Чтение: 8 мин
* Опубликовано 19 июля 2017 г.

Smartscape — наш инструмент визуализации топологии среды, работающий практически в реальном времени, — является одной из самых мощных функций Dynatrace.

Автообнаружение Smartscape обеспечивает быструю и эффективную визуализацию всех топологических зависимостей в инфраструктуре, процессах и сервисах:

* По вертикальной оси отображаются зависимости полного стека на всех уровнях.
* По горизонтальной оси визуализируются все входящие и исходящие связи вызовов на каждом уровне.

Несколькими щелчками мыши Smartscape предоставляет доступ к детальному топологическому представлению всей среды, давая больше информации и контроля над ней.

* Принимайте более взвешенные решения — например, о корректировке архитектуры сервисов или инфраструктуры для улучшения производительности приложений.
* Изучайте межуровневые и одноуровневые взаимозависимости процессов, хостов и сервисов для лучшего понимания влияния зависимостей на производительность приложений.
* Выполняйте детализацию для более глубокого анализа проблем. Например, Dynatrace может выявить проблему со сторонними зависимостями и помочь понять влияние проблемы на производительность приложения.

## Доступ к Smartscape

Для доступа к классическому представлению топологии среды Smartscape перейдите в Smartscape.

Для начала выберите нужное представление:

* Выберите **Show problems** для отображения топологии [проблем](#problems).
* Выберите **Show third-party vulnerabilities** и укажите уровни риска (можно выбрать несколько) для отображения топологии [сторонних уязвимостей](#vulnerabilities).

## Проблемы

При выборе **Show problems** каждый уровень Smartscape включает отдельную вкладку с индикатором работоспособности. Выберите уровень, на котором нужно просмотреть топологию:

* Applications (Приложения)
* Services (Сервисы)
* Processes (Процессы)
* Hosts (Хосты)
* Data centers (Центры обработки данных)

Каждый уровень отображает разный тип сущностей: приложение, сервис, процесс, хост или центр обработки данных.

* Для просмотра другого уровня выберите его вкладку.
* Для масштабирования используйте кнопки **+/-** в правом верхнем углу или прокрутку мыши.
* Для перемещения по представлению нажмите и перетащите в любом месте Smartscape.
* Для просмотра имени сущности наведите курсор на символ сущности. Пока отображается имя, можно нажать стрелку рядом с ним для перехода на страницу обзора сущности.

В первом примере ниже:

* Уровень **Hosts** отображает статус работоспособности **1/33**: мониторинг ведётся для 33 хостов, из которых 1 имеет проблему.
* Остальные четыре уровня (**Applications**, **Services**, **Processes** и **Data centers**) в этом примере показывают только количество сущностей на каждом уровне, так как все они работоспособны.

Smartscape отображает данные за последние 72 часа и поэтому не может быть настроен с помощью селектора временного диапазона. Для просмотра зависимостей между сущностями одного уровня выберите соответствующую вкладку слева.

### Applications

В этом примере выбрана вкладка **Applications** и справа отображаются **12** обнаруженных приложений в данной среде.

![Приложения в Smartscape](https://dt-cdn.net/images/smartscape-applications-1310-dc4704a941.png)

Приложения в Smartscape

Dynatrace поддерживает мониторинг веб-приложений и мобильных приложений. Символ узла указывает тип приложения:

| Значок | Значение |
| --- | --- |
|  | **Веб-приложение** |
|  | **Мобильное приложение** |

Связи или зависимости между приложениями отсутствуют, так как в Dynatrace приложения рассматриваются с точки зрения пользователя и являются лишь точками входа. Несколько приложений могут использовать одни и те же сервисы, но приложения не могут быть напрямую связаны друг с другом.

### Services

Вкладка **Services** отображает топологию всех сервисов, работающих в среде.

![Сервисы в Smartscape](https://dt-cdn.net/images/smartscape-services-1621-d595e2d107.png)

Сервисы в Smartscape

**Узлы**

Каждый узел представляет отдельный сервис с символом, обозначающим лежащий в основе тип сервиса.

| Значок | Значение |
| --- | --- |
| см. пример выше | Коммерческие логотипы — дельфин MySQL, перо Apache, кот Tomcat — обозначают [технологии сервисов, поддерживаемые Dynatrace](/managed/ingest-from/technology-support "Технические подробности поддержки Dynatrace для конкретных платформ и фреймворков разработки."). |
|  | Сервисы нераспознанных технологий обозначаются универсальным символом сервиса. Как правило, это [непрозрачные сервисы](/managed/observe/application-observability/services/service-detection/service-detection-v1/service-types/opaque-services "Что такое непрозрачные сервисы."). |

**Связи**

* Стрелки на линиях связей к выбранному сервису и от него указывают, являются ли они входящими или исходящими вызовами.
* Пунктирная линия связи означает, что между двумя сервисами не было запросов за последние два часа или сервис не перенаправлял и не получал ничего за последние два часа.
* Связь устаревает и больше не отображается в Smartscape, если она была неактивна более 72 часов или сервис не получал нагрузки в течение последних 72 часов.

**Фильтрация отображения**
Откройте меню Browse (**…**) в правом верхнем углу Smartscape для скрытия или отображения:

* Неактивных запросов
* Сервисов, не связанных с другими сервисами

Обратите внимание, что количество сервисов на вкладке **Services** в Smartscape может отличаться от количества на странице **Services**, так как Smartscape учитывает более широкий охват категорий сервисов.

### Processes

Вкладка **Processes** обеспечивает визуализацию, в которой каждый узел соответствует процессу, а каждая связь представляет TCP/IP-запрос.

* Не все процессы, выполняемые в среде, отображаются, так как их количество часто очень велико. Для наглядности показаны только [наиболее важные процессы](/managed/observe/infrastructure-observability/process-groups/basic-concepts/which-are-the-most-important-processes "Отображение наиболее важных процессов для мониторинга и группировки.").
* Пунктирные линии показывают неактивные или прерванные связи между процессами.

![Процессы в Smartscape](https://dt-cdn.net/images/smartscape-processes-1623-2321abf3d3.png)

Процессы в Smartscape

### Hosts

Вкладка **Hosts** показывает топологию инфраструктуры с точки зрения хостов.

* Связи представляют TCP/IP-запросы.
* Хосты, как физические, так и виртуальные машины, обозначаются коммерческими логотипами операционных систем.

![Хосты в Smartscape](https://dt-cdn.net/images/smartscape-hosts-1659-8c03fceb20.png)

Хосты в Smartscape

Универсальные символы хостов используются для обозначения кандидатов на мониторинг (неизвестных хостов, обнаруженных, поскольку они получают TCP/IP-запросы от отслеживаемых хостов). Рекомендуется устанавливать Dynatrace OneAgent на кандидатов на мониторинг в среде при каждой возможности.

| Значок | Значение |
| --- | --- |
|  | кандидат на мониторинг |

Неактивные кандидаты на мониторинг — это кандидаты, которые не взаимодействовали с хостом. Неактивные связи между хостами и прерванные связи визуализируются пунктирными линиями.

### Data centers

Вкладка **Data centers** отображает узлы, указывающие расположение хостов. При наличии физических серверов в инфраструктуре соответствующие узлы центров обработки данных указывают города, в которых расположены эти центры, и обозначаются универсальными значками хостов.

При использовании виртуальных серверов или инфраструктуры на основе PaaS узлы центров обработки данных помечаются соответствующим образом (например, VMware data center, AWS Availability Zone или Azure region) и визуализируются с соответствующими логотипами компаний.

![Центры обработки данных в Smartscape](https://dt-cdn.net/images/smartscape-datacenters-1625-63d687b49e.png)

Центры обработки данных в Smartscape

### Межуровневые взаимосвязи

Для просмотра межуровневых связей выберите любую сущность на любом уровне Smartscape. Вертикальные зависимости сущности отображаются слева.

В этом примере выбрано веб-приложение `easytravel-dynatrace-dev`. Слева можно видеть, как зависимости приложения распространяются вниз до уровня центров обработки данных.

![Межуровневые связи в Smartscape 1](https://dt-cdn.net/images/smartscape-inter1-1687-680a65579e.png)

Межуровневые связи в Smartscape 1

Приложение `easytravel-dynatrace-dev` выделено красным, так как оно испытывает проблему. Следуя вниз по связанным зависимостям, можно увидеть, что это приложение вызывает 27 сервисов Tomcat и 4 сервиса ASP/.NET. Эти сервисы выполняются в процессах тех же типов технологий (выделенные красным испытывают проблемы). Видно, что эти процессы размещены на хосте под управлением Windows с именем `lr-ws-l02v` (для просмотра этой детали наведите курсор на узел хоста). Поскольку неработоспособные процессы выполняются на этом хосте, он также неработоспособен (обратите внимание на красную цифру **1** на плитке **Hosts** слева).

Этот пример показывает, как легко обнаружить первопричину проблемы и сократить время её устранения, изучая связанные сущности в Smartscape.

## Сторонние уязвимости

При выборе **Show third-party vulnerabilities** в Smartscape выберите **Risk level** для указания уровней риска, которые нужно отображать. Можно выбрать несколько флажков.

* Слева вертикальный столбец показывает количество затронутых узлов красным цветом и общее количество узлов серым на уровнях процессов и хостов.
* На топологии затронутые узлы отображаются в цвете, соответствующем уровню риска:

  + Тёмно-красный для `Critical`
  + Красный для `High`
  + Жёлтый для `Medium`
  + Синий для `Low`

Для просмотра уязвимостей затронутых сущностей переключитесь на уровень **Processes** или **Hosts**. (Уязвимости могут быть связаны с приложением, сервисом или центром обработки данных, но не влияют на них напрямую. Непосредственно уязвимостям подвержены только процессы или хосты.)

* Для масштабирования используйте кнопки **+/-** в правом верхнем углу или прокрутку мыши.
* Для перемещения по представлению нажмите и перетащите в любом месте Smartscape.
* Для просмотра имени сущности наведите курсор на символ сущности. Пока отображается имя, можно нажать стрелку рядом с ним для перехода на страницу обзора.
* Для просмотра межуровневых связей выберите любую сущность. Панель слева разворачивается для отображения вертикальных зависимостей сущности.

Дополнительные сведения об управлении сторонними уязвимостями см. в разделе [Сторонние уязвимости](/managed/secure/application-security/vulnerability-analytics/third-party-vulnerabilities "Мониторинг, визуализация, анализ и устранение сторонних уязвимостей, отслеживание прогресса устранения и создание правил мониторинга.").
