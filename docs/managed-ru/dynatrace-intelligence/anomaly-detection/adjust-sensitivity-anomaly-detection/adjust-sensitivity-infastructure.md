# Настройка чувствительности обнаружения аномалий для инфраструктуры

* How-to guide
* 3-min read
* Updated on Apr 10, 2026

Dynatrace обнаруживает множество аномалий производительности, связанных с инфраструктурой, которые могут привести к проблемам в вашей инфраструктуре (конкретный список зависит от типа инфраструктуры). Каждый тип аномалии обнаруживается независимо и формирует собственные проблемы и оповещения.

Чтобы настроить глобальную конфигурацию обнаружения аномалий для инфраструктуры

1. Перейдите в раздел **Settings** > **Anomaly detection**.
2. В разделе **Infrastructure** выберите тип инфраструктуры, для которого необходимо настроить обнаружение аномалий.

Обнаружение аномалий можно настроить для всех хостов, отслеживаемых Dynatrace, независимо от их типа (физические или виртуальные). Dynatrace также обнаруживает сетевые проблемы и проблемы с дисками хостов.

Для каждого типа аномалии можно выбрать автоматическое обнаружение — тогда Davis будет оповещать вас о проблемах — или задать фиксированный порог. Выберите нужный вариант из соответствующего списка.

По умолчанию Dynatrace оповещает только о неожиданных сбоях.

* При плановом завершении работы отключение хоста является ожидаемым: операционная система отправила сигнал завершения работы, уведомив OneAgent о том, что оператор намеренно выключает хост.
* Если OneAgent не получает сигнала завершения работы, отключение классифицируется как неожиданное.

Вы можете включить уведомления о плановых завершениях работы.

## Пороги для конкретных дисков

Серверные оповещения по дискам для новых тенантов

Начиная с версии SaaS 1.308, серверные оповещения по дискам для новых тенантов по умолчанию отключены. Рекомендуется использовать [Disk Edge alerting](/managed/observe/infrastructure-observability/hosts/configuration/anomaly-detection#disk-edge-alerting "Настройте обнаружение аномалий хоста, включая пороги проблем и событий."). Disk Edge alerting позволяет создавать более сложные и конкретные правила с использованием:

* метрик для оповещения (доступное дисковое пространство, файловая система только для чтения, время чтения, время записи и доступные inode);
* операционной системы, к которой следует применять политику;
* фильтров по имени диска;
* условий по пользовательским метаданным хоста;
* пользовательских свойств, прикреплённых к формируемому событию.

Обратите внимание, что Disk Edge alerting требует OneAgent версии 1.293 и выше.

Davis автоматически обнаруживает аномалии дисков, такие как малое доступное дисковое пространство или медленные диски. На хосте могут быть разные типы дисков: загрузочный диск, диск для хранения всех логов или диск для хранения бизнес-данных. Если оповещения о малом дисковом пространстве бессмысленны для загрузочного диска фиксированного размера, то для диска с критически важными бизнес-данными они вполне уместны.

С помощью пользовательских правил обнаружения дисков можно задать детально настроенные правила для отдельных групп дисков (группы формируются на основе шаблонов имён дисков и/или тегов хостов). Пороги на уровне диска переопределяют глобальные пороги для соответствующих дисков, тогда как для остальных дисков продолжают действовать глобальные настройки.

Чтобы изменить настройки порогов для группы дисков

1. Перейдите в раздел **Settings** > **Anomaly detection**.
2. В разделе **Infrastructure** выберите **Custom disk-detection rules**.
3. Нажмите **Add item**.
4. Выберите метрику для мониторинга и задайте понятное имя для правила.
5. Укажите порог для метрики и количество выборок, которые должны нарушить порог для формирования оповещения.
6. Необязательно: укажите шаблон имени диска.

   Отдельные правила не связаны логически и применяются независимо. Например, если одно правило соответствует всем дискам, не содержащим `A`, а другое — всем дискам, не содержащим `B`, то каждый диск будет соответствовать либо первому, либо второму, либо обоим правилам одновременно. Обратите внимание, что в одном фильтре правила нельзя указывать несколько значений, подстановочные знаки или регулярные выражения.
7. Необязательно: чтобы дополнительно сузить область применения правила дисков, перечислите теги, которые должен иметь хост.
8. Нажмите **Save changes**.

### Иерархия конфигурации обнаружения аномалий

Обнаружение аномалий можно настроить на нескольких уровнях: среды, группы хостов, хоста и т. д. При наличии нескольких правил, затрагивающих один и тот же объект, наиболее конкретное правило имеет приоритет над более общими, как показано на схеме ниже.

## Пороги для конкретной группы хостов

Чтобы настроить правила дисков для конкретной группы хостов

1. Перейдите в раздел **Deployment Status**.
2. Отфильтруйте таблицу по **Host group** и выберите имя группы хостов, для которой необходимо настроить пользовательские правила обнаружения дисков.
3. Для одного из перечисленных хостов выберите группу хостов.
4. Разверните раздел **Anomaly detection** и выберите **Custom disk-detection rules**.
5. Нажмите **Add item**.
6. Выберите метрику для мониторинга и задайте понятное имя для правила.
7. Укажите порог для метрики и количество выборок, которые должны нарушить порог для формирования оповещения.
8. Необязательно: укажите шаблон имени диска.
9. Необязательно: перечислите теги, которые должен иметь хост.
10. Нажмите **Save changes**.

## Пороги для конкретного хоста

В качестве альтернативы глобальному определению порогов для всей среды можно задать детально настроенные пороги для отдельных хостов. Пороги на уровне хоста переопределяют глобальные пороги для данного хоста, тогда как для остальных хостов продолжают действовать глобальные настройки. Вернуться к глобально определённым порогам можно в любой момент.

Чтобы изменить настройки порогов для конкретного хоста

1. Перейдите в раздел **Hosts**.
2. Выберите хост, который необходимо настроить.
3. В правом верхнем углу страницы выберите **More** (**…**) > **Settings**.
4. Перейдите в раздел **Anomaly detection** > **Infrastructure** для настройки конфигурации.
