# Настройка чувствительности обнаружения аномалий

* Explanation
* 2-min read
* Published Sep 06, 2021

К типичным аномалиям, обнаруживаемым Dynatrace, относятся рост частоты ошибок, деградация времени отклика, а также резкие скачки или падения трафика приложений. Наблюдая за вашей средой, Dynatrace изучает эталонные значения, соответствующие нормальному поведению, и адаптирует обнаружение аномалий соответствующим образом. Этот процесс называется *автоматическим построением базиса*. Помимо использования автоматического базиса, можно задать *фиксированные пороги*, определяющие условия для формирования проблемы и оповещения. Эти конфигурации можно применять глобально или для конкретных объектов.

*Чувствительность обнаружения проблем* определяет уровень статистической достоверности, необходимый для формирования оповещения.

| Чувствительность | Статистическая достоверность | Примечания |
| --- | --- | --- |
| Низкая | Высокая | Полезна на стадиях разработки и предпроизводственного тестирования — позволяет избежать избыточных оповещений. |
| Высокая | Низкая | Проблема формируется даже при нарушении порога лишь несколькими точками данных. Подходит для критически важных производственных сервисов. |

Для некоторых критериев Dynatrace различает абсолютные и относительные пороги. В приведённом ниже примере (часть конфигурации обнаружения аномалий для приложения) пороги деградации ключевых показателей производительности установлены на уровне `100 мс` (абсолютный) и `50%` (относительный) выше автоматически определённого базиса. Порог для 10% наиболее медленных запросов задан равным `1 000 мс` (абсолютный) и `100%` (относительный) выше автоматически определённого базиса.

Кроме того, обнаружение аномалий учитывает количество действий в минуту, которые должны происходить в отслеживаемом приложении (`10 действий в минуту` в примере выше). С помощью этой настройки можно отключить оповещения для малонагруженных приложений и сервисов — построение базиса и оповещения для таких приложений зачастую приводят к избыточным уведомлениям.

Чтобы настроить чувствительность обнаружения на глобальном уровне

1. Перейдите в раздел **Settings**.
2. Разверните раздел **Anomaly detection**.
3. Выберите нужный тип объекта. Подробные инструкции см. в одной из следующих тем.

* [Приложения](/managed/dynatrace-intelligence/anomaly-detection/adjust-sensitivity-anomaly-detection/adjust-sensitivity-applications "Узнайте, как настроить чувствительность обнаружения проблем для приложений.")
* [Сервисы](/managed/dynatrace-intelligence/anomaly-detection/adjust-sensitivity-anomaly-detection/adjust-sensitivity-services "Узнайте, как настроить чувствительность обнаружения проблем для сервисов.")
* [Сервисы баз данных](/managed/dynatrace-intelligence/anomaly-detection/adjust-sensitivity-anomaly-detection/adjust-sensitivity-services-database "Узнайте, как настроить чувствительность обнаружения проблем для сервисов баз данных.")
* [Инфраструктура](/managed/dynatrace-intelligence/anomaly-detection/adjust-sensitivity-anomaly-detection/adjust-sensitivity-infastructure "Настройка чувствительности обнаружения проблем для инфраструктуры.")
