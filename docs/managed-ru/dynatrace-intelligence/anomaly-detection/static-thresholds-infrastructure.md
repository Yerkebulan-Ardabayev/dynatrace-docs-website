Мониторинг инфраструктуры Dynatrace основан на многочисленных встроенных предопределённых статических порогах. Эти пороги относятся к конкурентному использованию ресурсов, таким как всплески CPU, использование памяти и дискового пространства. Вы можете изменить эти пороги по умолчанию в разделе **Settings** > **Anomaly Detection** > **Infrastructure**.

Для приложений и сервисов можно в любой момент отключить обнаружение на основе автоматического построения базиса и переключиться на статические пороги, заданные пользователем. Если задать статический порог для времени отклика и частоты ошибок на уровне приложения или сервиса, события будут формироваться при нарушении статического порога. Событие замедления формируется при нарушении статических порогов для медианного или 90-го перцентиля времени отклика.

### Предопределённые статические пороги для хостов

| Хосты | Статический порог по умолчанию |
| --- | --- |
| Насыщение CPU | Оповещение, если использование CPU выше **95**% в 18 из 30 10-секундных интервалов. |
| Использование памяти | * Оповещение, если использование памяти выше **90**% на Windows или **80**% на Linux * И частота ошибок страниц памяти выше **100** ошибок/с на Windows или **20** ошибок/с на Linux в 18 из 30 10-секундных интервалов. |
| Активность GC | * Оповещение, если время GC выше **40**% * ИЛИ задержка GC выше **25**% в 18 из 30 10-секундных интервалов. |
| Нехватка памяти Java | Оповещение, если число исключений Java out-of-memory равно **1** в минуту или выше. |
| Высокая системная нагрузка AIX | Требуется OneAgent версии 1.261+. Оповещение, если системная нагрузка превышает заданный порог в течение заданного числа выборок. Это оповещение специфично для хостов AIX. |

### Предопределённые статические пороги для сетей

| Сеть | Статический порог по умолчанию |
| --- | --- |
| Число потерянных пакетов | * Оповещение, если доля потерянных пакетов при приёме/передаче выше **10**% * И суммарная скорость пакетов выше **10** пакетов/с в 18 из 30 10-секундных интервалов. |
| Использование сети | Оповещение, если утилизация отправки/приёма трафика выше **90**% в 18 из 30 10-секундных интервалов. |
| TCP-соединимость для процесса | * Оповещение, если доля сбоев новых соединений выше **3**% * И число неудачных соединений выше **10** соединений/мин в 18 из 30 10-секундных интервалов. |
| Частота повторных передач | * Оповещение, если частота повторных передач выше **10**% * И число повторно переданных пакетов выше **10** пакетов/мин в 18 из 30 10-секундных интервалов. |

### Предопределённые статические пороги для дисков

| Диск | Статический порог по умолчанию |
| --- | --- |
| Мало свободного места | Оповещение, если свободное дисковое пространство менее **3**% в 18 из 30 10-секундных интервалов. |
| Медленные диски | Оповещение, если время чтения или записи диска выше **200** мс в 18 из 30 10-секундных интервалов. |
| Доступное число inode | Оповещение, если доля доступных inode менее **5**% в 18 из 30 10-секундных интервалов. |

Поскольку статические пороги не адаптируются к изменяющейся среде и требуют значительных ручных усилий, Dynatrace также предлагает интеллектуальное [автоматическое многомерное построение базиса](/managed/dynatrace-intelligence/anomaly-detection/automated-multidimensional-baselining "Узнайте, как ИИ Dynatrace автоматически вычисляет базисы по многомерной схеме."), которое работает «из коробки» без ручной настройки порогов. Самое важное — оно автоматически адаптируется к изменениям в паттернах трафика.
