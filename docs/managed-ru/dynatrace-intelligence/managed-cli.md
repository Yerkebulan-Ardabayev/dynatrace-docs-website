# Dynatrace Managed CLI

* Пояснение
* Чтение за 2 минуты
* Обновлено 11 июня 2026 г.

Dynatrace Managed CLI (`dtmgd`), это инструмент командной строки с открытым исходным кодом, доступный только для чтения, предназначенный для запросов данных наблюдаемости из Managed-кластеров. Он предоставляет доступ из терминала к Managed classic API. С его помощью можно скриптовать доступ к проблемам, сущностям, метрикам, логам, событиям, целевым показателям уровня сервиса (SLO) и данным безопасности без использования веб-интерфейса Dynatrace. CLI следует той же модели, что и `dtctl`, аналогичный инструмент для Dynatrace SaaS.

## Сценарии использования

* **Скриптование и автоматизация**: используй форматы вывода table, wide, JSON и YAML для интеграции вывода CLI в shell-скрипты, CI-конвейеры и рабочие процессы мониторинга. Автоматическая постраничная разбивка обрабатывает большие наборы результатов без дополнительного скриптования.
* **Запросы к нескольким окружениям**: выполняй одну команду параллельно в нескольких настроенных окружениях и объединяй результаты. Это полезно организациям, которые запускают отдельные Managed-кластеры по регионам или командам и которым нужен единый обзор по всем ним.
* **Мониторинг в реальном времени**: используй режим наблюдения (watch mode), чтобы повторно выполнять запрос через настраиваемые интервалы и получать в терминале живое представление изменяющихся данных, таких как открытые проблемы или недавние события.
* **Интеграция с ИИ-ассистентами**: используй CLI как источник данных для ИИ-ассистентов и конвейеров больших языковых моделей, потребляющих структурированные данные наблюдаемости. CLI распознаёт окружения ИИ-ассистентов и оборачивает вывод в структурированные JSON-конверты.

## Доступные команды

CLI организует доступ к данным Managed в четыре категории команд:

* **get**: список ресурсов с параметрами фильтрации и сортировки
* **describe**: получение подробного вывода по отдельным ресурсам
* **query**: получение данных за временной диапазон с агрегацией и настраиваемым разрешением
* **config**: управление контекстами окружений и сохранёнными учётными данными

Следующие типы ресурсов доступны во всех категориях команд:

* Проблемы
* Сущности и типы сущностей
* События
* Метрики
* Логи
* Целевые показатели уровня сервиса
* Уязвимости безопасности
* Журналы аудита
* Сетевые зоны
