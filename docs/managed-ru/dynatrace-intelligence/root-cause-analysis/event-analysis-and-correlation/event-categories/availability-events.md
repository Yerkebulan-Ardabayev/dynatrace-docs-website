На этой странице представлена информация о поддерживаемых событиях доступности и логике их возникновения.

## Неожиданно низкий трафик

`UNEXPECTED_LOW_LOAD`

Dynatrace собирает многомерный базис для трафика приложений и пользовательских действий и тем самым изучает типичные паттерны трафика всех ваших приложений и пользовательских действий.

Аномально низкий трафик приложения может указывать на полные сбои приложения, поэтому оповещения об аномально низком трафике включены по умолчанию.

Перейдите в раздел **Settings** > **Anomaly detection** > **Applications**, чтобы настроить чувствительность оповещений о низком трафике. При включении Dynatrace использует паттерн сравнения временных интервалов (ежедневный или еженедельный) для оповещения об аномальных ситуациях с трафиком. Фактический отслеживаемый трафик приложения сравнивается с предыдущим интервалом, и оповещение формируется, если сравнение выявляет необычно низкий трафик.

### Применимые объекты Dynatrace

К этому событию применимы следующие объекты Dynatrace:

* Сервис
* Приложение
* Мобильное приложение

## Хост или мониторинг недоступен

`OSI_UNEXPECTEDLY_UNAVAILABLE`

Это событие обнаруживается, когда хост внезапно отключается или Dynatrace теряет сетевое соединение с OneAgent хоста. Если операционная система завершает работу корректно, Dynatrace не открывает проблему. Dynatrace отображает хост в недоступном состоянии.

Условие:

* Сетевое соединение с отслеживаемым хостом теряется неожиданно, пока OneAgent и хост продолжают работать. Соединение должно быть потеряно более чем на 5 минут, прежде чем OneAgent начнёт снова отправлять сигналы и кешированные данные метрик заполнят отсутствующие данные на графике.
* OneAgent не может перехватить и отправить обычное сообщение об остановке операционной системы.
* Вывод из эксплуатации виртуальных хостов (например, в группах автомасштабирования AWS), когда операционная система не завершает работу, из-за чего Dynatrace обнаруживает это как событие потери соединения (событие задерживается до 10 минут после потери соединения).

Условия закрытия:

* Событие закрывается, когда хост снова становится доступным.
* Таймаут события наступает через 12 часов.

## Процесс недоступен

`PROCESS_UNAVAILABLE`, `PROCESS_GROUP_LOW_INSTANCE_COUNT`

OneAgent Dynatrace автоматически обнаруживает каждый запущенный процесс на хостах и сообщает о состоянии доступности всех процессов. Davis автоматически обнаруживает доступность процессов и хостов, поэтому мониторинг доступности для групп процессов по умолчанию отключён.

Однако если группа процессов и её доступность имеют высокую важность, можно включить мониторинг доступности на уровне группы процессов. Для формирования события на уровне группы процессов есть два подхода:

* Если какой-либо процесс становится недоступным. В этом случае событие формируется немедленно, как только любой процесс группы становится недоступным по любой причине.
* Если число доступных процессов опускается ниже заданного порога. В этом случае можно вручную указать минимально допустимое число процессов в группе. Событие формируется, если число активных процессов опускается ниже порога на **2 последовательные минуты**.

Если вы намеренно останавливаете процесс, событие также будет сформировано при выполнении этого условия. В этом случае просто закройте созданную проблему вручную.

Для настройки мониторинга доступности группы процессов нажмите **Edit** на странице сведений о группе процессов и выберите пункт **Availability monitoring**.

Условия закрытия:

* Событие закрывается, когда процесс снова становится доступным.
* Таймаут события наступает через 5 дней.

### Применимые объекты Dynatrace

К этому событию применимы следующие объекты Dynatrace:

* Группа процессов (`PROCESS_GROUP_LOW_INSTANCE_COUNT`)
* Экземпляр группы процессов (`PROCESS_UNAVAILABLE`)

## Глобальный сбой синтетического монитора

`WEB_CHECK_GLOBAL_OUTAGE`

Синтетические мониторы проверяют ваши сайты из нескольких географических регионов по регулярному расписанию. Dynatrace формирует событие глобального сбоя синтетического монитора, если ваш сайт перестаёт отвечать из всех настроенных географических регионов.

### Применимые объекты Dynatrace

К этому событию применимы следующие объекты Dynatrace:

* Синтетический монитор

## Локальный сбой синтетического монитора

`WEB_CHECK_LOCAL_OUTAGE`

Синтетические мониторы регулярно проверяют ваши сайты из нескольких географических регионов. Dynatrace формирует событие локального сбоя синтетического монитора в случае, если ваш сайт не отвечает хотя бы из одного настроенного географического региона. Как только ваше веб-приложение перестаёт отвечать из всех настроенных регионов, проблема повышается до статуса глобального сбоя.

### Применимые объекты Dynatrace

К этому событию применимы следующие объекты Dynatrace:

* Синтетический монитор

### Применимые объекты Dynatrace

К этому событию применимы следующие объекты Dynatrace:

* Внешний синтетический монитор

## Обнаружен паттерн лога доступности

`HOST_LOG_AVAILABILITY`

Log Monitoring Dynatrace позволяет определять паттерны логов, указывающие на проблемы с доступностью на уровне хоста. Все обнаруженные вхождения паттерна регистрируются, и если число обнаруженных паттернов превышает ваш критический порог, формируется событие паттерна лога доступности.

### Применимые объекты Dynatrace

К этому событию применимы следующие объекты Dynatrace:

* Хост

## Обнаружен паттерн лога доступности

`PROCESS_LOG_AVAILABILITY`

Log Monitoring Dynatrace позволяет определять паттерны логов, указывающие на проблемы с доступностью на уровне процесса. Каждое вхождение настроенного паттерна подсчитывается со временем, и если число обнаруженных паттернов превышает ваш критический порог, формируется событие паттерна лога доступности.

### Применимые объекты Dynatrace

К этому событию применимы следующие объекты Dynatrace:

* Экземпляр группы процессов

## Пользовательское событие доступности

`AVAILABILITY_EVENT`

Это общее событие доступности может использоваться плагинами мониторинга или через [Dynatrace API](/managed/dynatrace-api/environment-api/events-v1 "Узнайте, что можно сделать с помощью Dynatrace Events API.") для формирования настроенного события доступности с пользовательским заголовком (например, `Batch process schedule outage`).

### Применимые объекты Dynatrace

К этому событию применимы все объекты Dynatrace.
