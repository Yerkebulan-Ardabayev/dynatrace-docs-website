# События замедления

* Explanation
* 3-min read
* Published Dec 28, 2018

На этой странице представлена информация о поддерживаемых событиях замедления и логике их возникновения.

## Деградация длительности пользовательского действия

`USER_ACTION_DURATION_DEGRADATION`

События деградации длительности пользовательских действий обнаруживаются в Dynatrace несколькими способами. По умолчанию Dynatrace использует автоматическое построение базиса для обнаружения событий деградации 50-го перцентиля (медианы) или 90-го перцентиля производительности пользовательских действий приложения.

На уровне приложения базис различает три основные категории пользовательских действий:

* действия загрузки;
* XHR-действия;
* пользовательские действия.

Если производительность одной из этих категорий деградирует, событие формируется со ссылкой на одну из них.

Dynatrace автоматически собирает многомерные базисы для пользовательских действий, где измерения — это пользовательское действие, географический регион, операционная система и тип браузера. Если замедление обнаруживается для конкретной комбинации этих измерений (например, платёж, Браузер: Firefox), результирующее событие будет содержать ссылку на нарушающие измерения.

### Затронутые типы объектов

Следующие типы объектов могут быть затронуты событиями замедления:

* Сервисы
* Мобильные приложения
* Веб-приложения

## Деградация времени отклика

`SERVICE_RESPONSE_TIME_DEGRADED`

События деградации времени отклика обнаруживаются в Dynatrace несколькими способами. По умолчанию Dynatrace использует автоматическое построение базиса для обнаружения событий деградации 50-го перцентиля (медианы) или 90-го перцентиля времени отклика сервиса и метода сервиса. Перейдите в раздел **Settings** > **Anomaly detection** > **Services** для поиска настроек автоматического построения базиса. Также можно использовать статический порог вместо динамического базиса. Статические пороги сравниваются с отслеживаемыми 50-м и 90-м перцентилями.

### Применимые объекты Dynatrace

К этому событию применимы следующие объекты Dynatrace:

* Сервис

## Обнаружен паттерн лога производительности

`HOST_LOG_PERFORMANCE`

Log Monitoring Dynatrace позволяет определять паттерны логов, указывающие на проблемы с производительностью на уровне хоста. Каждое вхождение настроенного паттерна подсчитывается со временем. Если число обнаруженных паттернов превышает ваш критический порог, будет сформировано событие паттерна лога производительности.

### Применимые объекты Dynatrace

К этому событию применимы следующие объекты Dynatrace:

* Хост

## Обнаружен паттерн лога производительности

`PROCESS_LOG_PERFORMANCE`

Log Monitoring Dynatrace позволяет определять паттерны логов, указывающие на проблемы с производительностью на уровне процесса или группы процессов. Каждое вхождение настроенного паттерна подсчитывается со временем, и если число обнаруженных паттернов превышает ваш критический порог, будет сформировано событие паттерна лога производительности.

### Применимые объекты Dynatrace

К этому событию применимы следующие объекты Dynatrace:

* Экземпляр группы процессов

## Нарушение порога производительности синтетического монитора

`SYNTHETIC_SLOWDOWN`

Можно определить конкретные пороги производительности для синтетических мониторов, которые проверяют приложение из нескольких глобальных местоположений через регулярные интервалы. Если пользовательский порог производительности для одного из настроенных местоположений не выполняется, будет сформировано событие нарушения порога производительности синтетического монитора.

### Применимые объекты Dynatrace

К этому событию применимы следующие объекты Dynatrace:

* Синтетический монитор
* Внешний синтетический монитор

## Пользовательское событие производительности

`PERFORMANCE_EVENT`

Это общее событие производительности может использоваться плагинами мониторинга или через Dynatrace REST API для формирования настроенного события производительности с пользовательским заголовком (например, `Batch process schedule slowdown`).

### Применимые объекты Dynatrace

К этому событию применимы все объекты Dynatrace.
