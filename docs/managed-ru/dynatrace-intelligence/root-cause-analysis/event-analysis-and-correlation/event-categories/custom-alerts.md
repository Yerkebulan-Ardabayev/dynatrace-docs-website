# Пользовательские оповещения

* Explanation
* 1-min read
* Updated on Jan 28, 2026

На этой странице представлена информация о поддерживаемых пользовательских оповещениях и логике их возникновения.

Пользовательские оповещения можно определить, когда требуется уведомление о превышении конкретного порога для выбранной метрики.

Тип события `CUSTOM_ALERT` предоставляет простой способ определить порог для заданной метрики. Dynatrace отправляет оповещения, когда метрика нарушает пользовательский порог. Можно определить оповещения как для случаев, когда фактические значения метрики превышают, так и опускаются ниже пользовательского порога. Поскольку метрика может регистрироваться несколькими компонентами в вашей среде, Dynatrace всегда оповещает со ссылкой на компонент, демонстрирующий нарушающее значение метрики.

Как и для других типов событий, можно настроить функциональность Davis AI для пользовательских событий с помощью чётко определённых свойств событий, таких как `dt.davis.allow_merge`. Подробнее о свойствах см. в разделе [Semantic Dictionary](/managed/upgrade/unavailable-in-managed "Выбранный раздел недоступен в Dynatrace Managed.").
