## Рост частоты ошибок JavaScript

`JAVASCRIPT_ERROR_RATE_INCREASED`

По умолчанию события роста частоты ошибок JavaScript обнаруживаются автоматическим построением базиса, которое изучает типичные частоты ошибок JavaScript для каждого приложения и пользовательского действия. Если частота ошибок JavaScript увеличивается по сравнению с изученным базисом, Dynatrace формирует событие роста частоты ошибок JavaScript. Перейдите в настройки построения базиса в разделе **Settings** > **Anomaly detection** > **Web applications**, чтобы настроить чувствительность оповещений.

Если вам нужны статические пороги, а не основанный на базисе подход к оповещениям, переключитесь с автоматического режима на фиксированные пороги, как показано ниже. Чувствительность управляет уровнем статистической достоверности, необходимым для формирования события. Низкая чувствительность означает необходимость высокой достоверности, и наоборот. Например, чтобы немедленно видеть события, даже если порог нарушили лишь несколько точек данных, выберите высокую чувствительность.

## Рост частоты сбоев мобильного приложения

`MOBILE_APP_CRASH_RATE_INCREASED`

Сбои мобильных приложений (Android и iOS) регистрируются вместе со связанными трассировками стека сбоев и контекстной информацией. Dynatrace строит базис числа сбоев на версию приложения и оповещает, когда одна из версий вашего мобильного приложения деградирует по числу сбоев. Период наблюдения для оповещений о сбоях мобильных приложений составляет 10 минут. Если Dynatrace наблюдает необычно высокую частоту сбоев одной из версий вашего мобильного приложения в скользящем окне 10 минут, будет сформировано событие роста частоты сбоев мобильного приложения. Чтобы избежать избыточных оповещений в периоды низкой нагрузки, перед формированием события должны быть активны не менее 10 одновременных пользователей с одинаковой версией приложения.

## Высокая частота потерянных пакетов

`HIGH_DROPPED_PACKETS_RATE`

По умолчанию Dynatrace оповещает, если доля потерянных пакетов на уровне TCP-сети превышает 10%, а общее число потерянных пакетов превышает 10 пакетов/с в 3 из 5 одноминутных интервалов наблюдения. Перейдите в настройки обнаружения аномалий инфраструктуры в разделе **Settings** > **Anomaly detection** > **Host**, чтобы настроить чувствительность оповещений.

### Применимые объекты Dynatrace

К этому событию применимы следующие объекты Dynatrace:

* Хост
* Гипервизор

## Высокое число сетевых ошибок

`HIGH_NETWORK_ERROR_RATE`

По умолчанию Dynatrace оповещает, если доля неудачных попыток соединения на уровне TCP-сети превышает 3%, а общее число неудачных соединений превышает 10 соединений/мин в 3 из 5 одноминутных интервалов наблюдения. Перейдите в настройки обнаружения аномалий инфраструктуры в разделе **Settings** > **Anomaly detection** > **Host**, чтобы настроить чувствительность оповещений.

### Применимые объекты Dynatrace

К этому событию применимы следующие объекты Dynatrace:

* Хост

## Высокая частота ошибок Lambda

`LAMBDA_FUNCTION_HIGH_ERROR_RATE`

Событие высокой частоты ошибок Lambda информирует о высокой частоте неудачных вызовов конкретной функции AWS Lambda. По умолчанию это событие формируется, если частота неудачных вызовов превышает 5% в 3 из 5 одноминутных интервалов. Перейдите в раздел **Settings** > **Anomaly detection** > **Infrastructure** > **AWS**, чтобы настроить чувствительность оповещений о частоте ошибок Lambda.

### Применимые объекты Dynatrace

К этому событию применимы следующие объекты Dynatrace:

* Функция AWS Lambda

## Высокая частота ошибок бэкенда Elastic load balancer

`ELASTIC_LOAD_BALANCER_HIGH_BACKEND_FAILURE_RATE`

Dynatrace автоматически сообщает о числе неудачных попыток подключения к бэкенду балансировщика нагрузки AWS. По умолчанию Dynatrace формирует оповещение, если число неудачных попыток подключения к бэкенду превышает 10 в минуту в течение не менее 3 из 5 одноминутных интервалов.

### Применимые объекты Dynatrace

К этому событию применимы следующие объекты Dynatrace:

* Elastic load balancer

## Обнаружен паттерн лога ошибок

`HOST_LOG_ERROR`

Log Monitoring Dynatrace позволяет определять паттерны логов, указывающие на проблемы, связанные с ошибками, на уровне хоста. Каждое вхождение настроенного паттерна подсчитывается со временем, и если число обнаруженных паттернов превышает ваш критический порог, формируется событие паттерна лога ошибок.

### Применимые объекты Dynatrace

* Хост

## Обнаружен паттерн лога ошибок

`PROCESS_LOG_ERROR`

Каждое вхождение настроенного паттерна подсчитывается со временем, и если число обнаруженных паттернов превышает ваш критический порог, формируется событие паттерна лога ошибок.

## Последовательность перезапусков

`RDS_RESTART_SEQUENCE`

Dynatrace обнаруживает и оповещает об аномальных перезапусках службы реляционных баз данных (RDS). По умолчанию событие последовательности перезапусков формируется, если экземпляр RDS показывает общее число перезапусков выше 2 в течение периода наблюдения 20 минут.

### Применимые объекты Dynatrace

К этому событию применимы следующие объекты Dynatrace:

* Служба реляционных баз данных
