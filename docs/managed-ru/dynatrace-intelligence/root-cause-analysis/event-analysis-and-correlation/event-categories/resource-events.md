## Неожиданно высокий трафик

`UNEXPECTED_HIGH_LOAD`

Dynatrace собирает многомерный базис для трафика приложений и пользовательских действий и тем самым изучает типичные паттерны трафика всех ваших приложений и пользовательских действий. Оповещения об аномально высоком трафике приложения являются опциональной функцией в настройках приложений: **Settings** > **Anomaly detection** > **Applications**.

При включении Dynatrace использует двухсезонный паттерн (ежедневный и еженедельный) для оповещения об аномальных ситуациях с трафиком. Фактический отслеживаемый трафик приложения сравнивается с тем же периодом прошлой недели, и оповещение формируется, если сравнение выявляет необычно высокий трафик.

## Насыщение CPU

`CPU_SATURATED`

Исключение для AIX

Dynatrace не поддерживает события насыщения CPU на AIX. В качестве обходного решения можно настроить пользовательские оповещения на основе метрик хоста. Примеры:

* `CPU idle`: например, оповещение, если метрика ниже 10% в течение 5 минут за 5-минутный период.
* `CPU wait`: например, оповещение, если метрика превышает 90%.

Указанные значения являются примерами. Замените их по необходимости в соответствии с вашими требованиями.

События насыщения CPU формируются на уровне хоста, когда использование CPU превышает критический порог.

* По умолчанию Dynatrace оповещает, когда среднее использование CPU превышает 95% в 18 из 30 последовательных 10-секундных интервалов.
* Помимо типичного события насыщения CPU, связанного с OneAgent, управление облаком AWS может сообщать о событиях насыщения CPU для экземпляров EC2. В случае экземпляра EC2, отслеживаемого OneAgent, Dynatrace отдаёт приоритет событиям OneAgent над событиями AWS. Если один из экземпляров EC2 не отслеживается OneAgent, но у вас есть активная интеграция AWS и настройка Dynatrace, событие насыщения CPU будет сформировано при использовании выше 90% за один пятиминутный интервал.

## Насыщение памяти

`MEMORY_SATURATED`

Исключение для AIX

Dynatrace не поддерживает события насыщения памяти на AIX.

* **Windows**: По умолчанию Dynatrace оповещает, если использование памяти превышает 90% и частота ошибок страниц памяти превышает 100 ошибок/с в течение 18 из 30 10-секундных интервалов.
* **Linux**: По умолчанию Dynatrace оповещает, если использование памяти превышает 80% и частота ошибок страниц памяти превышает 20 ошибок/с в течение 18 из 30 10-секундных интервалов.

## Мало свободного места на диске

`LOW_DISK_SPACE`

По умолчанию Dynatrace оповещает, если свободное дисковое пространство на любом из ваших дисков опускается ниже 3% в течение не менее 18 из 30 10-секундных интервалов наблюдения.

Пороги дисков в Dynatrace широко настраиваемы как на уровне хоста, так и на уровне глобальных настроек. Для поддержки большого числа дисков можно определить глобальные правила порогов дисков с гибкими фильтрами по тегам для группировки подмножеств хостов.

### Применимые объекты Dynatrace

## Медленный диск

`SLOW_DISK`

По умолчанию Dynatrace оповещает, если время чтения или записи на любом из ваших дисков превышает 200 мс в течение не менее 18 из 30 10-секундных интервалов наблюдения.

Пороги дисков в Dynatrace широко настраиваемы как на уровне хоста, так и на уровне глобальных настроек. Для поддержки большого числа дисков можно определить глобальные правила порогов дисков с гибкими фильтрами по тегам для группировки подмножеств хостов.

## Длительная сборка мусора

`HIGH_GC_ACTIVITY`

По умолчанию Dynatrace оповещает о длительной сборке мусора, если в 18 из 30 10-секундных интервалов наблюдения время сборки мусора превышает 40% или задержка превышает 25%.

Это событие не формируется для процессов, запущенных в [развёртываниях без модуля ОС](/managed/ingest-from/technology-support/oneagent-platform-and-capability-support-matrix#other-modules "Узнайте, какие возможности поддерживаются OneAgent на различных операционных системах и платформах."), таких как Solaris и z/OS.

## Пользовательское событие конкурентного использования ресурсов

`RESOURCE_EVENT`

Это общее событие конкурентного использования ресурсов может использоваться плагинами мониторинга или через Dynatrace REST API для формирования настроенного события ресурса с пользовательским заголовком.

Пример: пользовательское событие конкурентного использования ресурсов с пользовательским заголовком `Low batch job pool`.
