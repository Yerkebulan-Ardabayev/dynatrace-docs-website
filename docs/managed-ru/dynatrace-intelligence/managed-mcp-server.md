# MCP-сервер Dynatrace Managed

# MCP-сервер Dynatrace Managed

* Разъяснение
* Чтение: 2 мин
* Обновлено 16 июня 2026 г.

MCP-сервер Dynatrace Managed, это сервер с открытым исходным кодом, реализующий Model Context Protocol (MCP). Он позволяет ИИ-ассистентам запрашивать данные observability из самостоятельно размещённых окружений Dynatrace Managed. Вместо ручной навигации по веб-интерфейсу Dynatrace можно исследовать проблемы, запрашивать метрики и анализировать результаты проверки безопасности. Это можно делать прямо из ИИ-ассистента, например Claude Desktop или Visual Studio Code с GitHub Copilot.

Сервер является проектом с community-поддержкой, выпущенным по лицензии MIT. Для окружений Dynatrace SaaS существует отдельный MCP-сервер.

## Сценарии использования

MCP-сервер наиболее полезен, когда нужен контекст observability без переключения между инструментами. Типичные применения:

* Расследование инцидентов, когда ИИ-ассистента просят получить детали проблемы и данные связанных сущностей
* Запрос показателей производительности и логов на естественном языке без ручного составления вызовов API
* Сравнение производственного и тестового окружений для подтверждения того, улучшает ли новая версия производительность, нагрузку или частоту ошибок
* Анализ уязвимостей безопасности и результатов проверки соответствия требованиям в окружении Managed
* Понимание связей между отслеживаемыми сущностями через их графы зависимостей
* Выполнение многоэтапных расследований инцидентов, объединяющих проблемы, логи и метрики в рамках одного диалога

### Поддержка нескольких окружений

Один экземпляр MCP-сервера подключается одновременно к нескольким окружениям Managed. У каждого окружения есть короткое имя, на которое можно ссылаться в запросах. Это полезно для организаций, использующих отдельные Managed Cluster для разных регионов, команд или стадий жизненного цикла.

MCP-сервер Managed также можно запускать вместе с MCP-сервером Dynatrace SaaS. Это поддерживает сценарии гибридной миграции, когда исторические данные observability остаются на Managed Cluster, а актуальные данные находятся в Dynatrace SaaS.

## Доступные инструменты

MCP-сервер предоставляет следующие данные observability из каждого подключённого окружения Managed:

* **Проблемы**: активные проблемы и их детали
* **Сущности**: отслеживаемые сущности и их связи
* **Метрики**: показатели производительности через V2 Metrics API
* **Логи**: содержимое логов с фильтрацией по содержимому и по времени
* **События**: системные события
* **Service level objectives**: детали, оценки и error budget
* **Безопасность**: уязвимости и проблемы безопасности
