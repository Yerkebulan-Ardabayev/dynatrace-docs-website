# Нотация порядков величин

* Explanation
* 2-min read
* Published Dec 19, 2025

Dynatrace отображает значения метрик с использованием нотации порядков величин, основанной на [Международной системе единиц (СИ)](https://dt-url.net/os03419).

## Нотация

Примеры нотации порядков величин в Dynatrace:

| Нотация | Множитель | Значение |
| --- | --- | --- |
| k | 10^3 | кило, тысяча |
| M | 10^6 | мега, миллион |
| G | 10^9 | гига, миллиард |
| T | 10^12 | тера, триллион |

## Примеры

### Браузер метрик

В этом примере из [браузера метрик](/managed/analyze-explore-automate/dashboards-classic/metrics-browser "Browse metrics with the Dynatrace metrics browser.") видны значения, отображаемые в миллионах. Это нотация порядков величин (`7.5M` означает «приблизительно 7,5 миллиона», а не «ровно 7,5 миллиона»).

Порядок величины здесь выбирается автоматически на основе размера значений. Например, та же метрика за более короткий период времени может отображаться в значениях `k` вместо `M`.

![Пример значений в нотации порядков величин в браузере метрик.](https://dt-cdn.net/images/magnitude-metrics-browser-1284-02225955bf.png)

Пример значений в нотации порядков величин в браузере метрик.

### Data Explorer и плитки дашборда

В [Data Explorer](/managed/analyze-explore-automate/explorer "Query for metrics and transform results to gain desired insights.") можно разрешить Dynatrace автоматически выбирать порядок величины, как это происходит в **браузере метрик**, однако Data Explorer также позволяет задать конкретный порядок величины, переопределяя автоматический выбор: установите **Unit** для отображения значений выбранной метрики в определённом порядке величины.

![Настройка Unit в Data Explorer](https://dt-cdn.net/images/explorer-set-unit-296-7dfcc5b953.png)

Настройка Unit в Data Explorer

Три плитки ниже идентичны, за исключением того, что при их настройке в Data Explorer для **Unit** были установлены разные значения. Слева направо:

* **Auto** — при **Unit** равном `Auto` для первой плитки Dynatrace автоматически выбирает подходящий порядок величины для значений: `1.32M` (примерно 1,32 миллиона), `428k` (примерно 428 тысяч) и т.д.
* **k (thousand)** — при **Unit** равном `k (thousand)` для второй плитки Dynatrace выражает каждое значение в тысячах.
* **M (million)** — при **Unit** равном `M (million)` для третьей плитки Dynatrace выражает каждое значение в миллионах.

![Плитки с разными настройками Unit](https://dt-cdn.net/images/dashboard-tiles-unit-examples-1006-61e950f821.png)

Плитки с разными настройками Unit

В **Advanced mode** можно использовать `:setUnit(<unit>)` для выбора из более широкого диапазона единиц.

Кроме того, с помощью параметра **Format** можно задать количество отображаемых десятичных знаков: `0`, `0.0`, `0.00`, `0.000`.
