# Обзор Real User и Synthetic Monitoring (DPS)

* Обзор
* 7 мин чтения
* Обновлено 25 июня 2026 г.

Real User и Synthetic Monitoring в Dynatrace состоит из двух областей возможностей, которые работают вместе, обеспечивая полную видимость того, как пользователи взаимодействуют с приложениями:

* **Real User Monitoring (RUM)** фиксирует и анализирует реальные взаимодействия конечных пользователей в веб-, мобильных и гибридных приложениях.
* **Synthetic Monitoring** использует автоматизированные, сценарные тесты для имитации поведения пользователей и проактивного обнаружения проблем с производительностью или доступностью.

На этой странице приведён обзор того, как эти возможности работают вместе. Подробнее о том, как рассчитывается потребление и как просматривать и контролировать затраты, см. на страницах, посвящённых конкретным возможностям, ссылки на которые приведены ниже.

## Обзор Real User Monitoring

Начать стоит с Real User Monitoring, при необходимости добавив session replay и свойства RUM в соответствии с потребностями.
Каждая возможность выставляется по отдельной строке карты тарификации.

* **Real User Monitoring (RUM)**: фокус на трендах производительности, ошибках и поведении в масштабе.
* **Real User Monitoring с Session Replay**: ускоряет устранение проблем с UX/UI, возникающих из-за некорректно отображаемых страниц и бесконечных индикаторов загрузки.
* **Real User Monitoring Property**: свойства RUM используются, когда нужны метаданные уровня бизнес-события или действия для фильтрации, сегментации и выполнения запросов или построения дашбордов, сравнивающих производительность и позволяющих разбираться с конкретными пользователями.

Подробнее см. [Понимание и управление потреблением для Real User Monitoring (DPS)](/managed/license/capabilities/real-user-synthetic-monitoring/real-user-monitoring "Learn how your consumption of the Dynatrace Real User Monitoring (RUM) DPS capabilities are billed and charged.").

## Обзор Synthetic Monitoring

Dynatrace Synthetic Monitoring имитирует посещения или взаимодействия пользователей с сайтом или приложением.
Он состоит из трёх самостоятельных возможностей, которые можно использовать и комбинировать в соответствии с потребностями.

* **Browser Monitor или Clickpath**: имитирует посещение приложения пользователем с помощью современного, обновлённого веб-браузера для мониторинга критически важных для бизнеса рабочих процессов приложения.

  Подробнее см. [Понимание и управление потреблением для Browser Monitor или Clickpath (DPS)](/managed/license/capabilities/real-user-synthetic-monitoring/browser-monitor-clickpath "Learn how your consumption of the Browser Monitor or Clickpath DPS capability is billed and charged.").
* **HTTP Monitor**: проверяет доступность ресурсов, таких как веб-сайты или конечные точки API.

  Подробнее см. [Понимание и оценка потребления для HTTP Monitor (DPS)](/managed/license/capabilities/real-user-synthetic-monitoring/http-monitor "Learn how your consumption of the Dynatrace HTTP Monitor DPS capability is billed and charged.").
* **Third-Party Synthetic Integration**: передаёт сторонние синтетические данные и события в Dynatrace.

  Подробнее см. [Понимание и оценка потребления для Third-Party Synthetic API Integration (DPS)](/managed/license/capabilities/real-user-synthetic-monitoring/third-party-api "Learn how your consumption of the the Dynatrace Third-Party Synthetic API Integration DPS capability is billed and charged.").
