# Обзор Digital Experience Monitoring (DEM) (DPS)

* Overview
* 7-min read
* Updated on May 16, 2025

Dynatrace Digital Experience Monitoring (DEM) обеспечивает мониторинг доступности и производительности, помогая гарантировать, что ваши приложения и сервисы доступны, функциональны, быстры и эффективны по всем каналам, включая мобильные устройства, веб, IoT и API.

## Real User Monitoring (RUM)

Dynatrace RUM — это процесс мониторинга производительности, собирающий подробные данные о каждом взаимодействии пользователя с приложением: как браузерным, так и мобильным на Android или iOS.
Real User Monitoring собирает данные по различным метрикам.
Например, данные о действиях загрузки могут включать метрики навигации, начала запроса и speed index.

## Real User Monitoring with Session Replay

Session Replay помогает захватывать и визуально воспроизводить полный цифровой опыт конечных пользователей.
Взаимодействия пользователей с приложением записываются и могут быть воспроизведены в виде видеозаписи.
Session Replay упрощает воспроизведение производственных проблем и ошибок, а также понимание таких проблем, как некорректное отображение страниц и бесконечные спиннеры загрузки.

## Real User Monitoring Property

Dynatrace собирает обширную информацию о производительности приложений. Эта информация может быть обогащена ценными метаданными и преобразована в свойства пользовательских действий и сессий. Эти свойства позволяют выполнять мощные запросы, сегментирование и агрегацию по захваченным метаданным.

Любое свойство RUM может быть настроено для приложения.

## Browser Monitor or Clickpath

Браузерный монитор имитирует посещение приложения пользователем через современный обновлённый браузер для мониторинга критически важных бизнес-сценариев работы приложения.
Браузерные мониторы можно настроить для запуска из любых публичных или частных локаций Dynatrace с частотой до каждых пяти минут, и они отправляют оповещения при недоступности приложения или значительном снижении производительности.
Кроме того, они могут использоваться для проверки доступности внутренних ресурсов, недоступных извне сети.

Для записи точной последовательности кликов и пользовательских вводов, доступности и производительности используется Dynatrace recorder.
Как вариант, определённую последовательность можно задать с помощью сценария.

## Third-Party Synthetic API Ingestion

Конечные точки Third-Party Synthetic API позволяют передавать синтетические данные и события из сторонних систем в Dynatrace.
API также обеспечивает корректировку ранее переданных в Dynatrace результатов.

## HTTP Monitor

Синтетические HTTP-мониторы можно создавать для проверки доступности ресурсов — веб-сайтов или API-эндпоинтов.
Кроме того, они могут использоваться для проверки доступности внутренних ресурсов, недоступных извне сети.

## Synthetic NAM Monitoring

Мониторы сетевой доступности (NAM) не имеют отдельной строки в прайс-листе Dynatrace. Вместо этого тарификация основывается на [количестве точек данных метрик](/managed/upgrade/unavailable-in-managed "Your selection is unavailable in Dynatrace Managed."), генерируемых при каждом выполнении NAM-теста. За дополнительной информацией обратитесь к менеджеру аккаунта Dynatrace.
