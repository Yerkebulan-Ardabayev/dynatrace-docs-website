# Ключевые понятия

* Справочник
* Чтение за 3 мин
* Опубликовано 25 июня 2026

Краткий справочник по ключевым терминам и понятиям, используемым в биллинге, использовании и управлении учётной записью Dynatrace Platform Subscription (DPS).

Account Management
:   Account Management предоставляет прогнозирование, оповещения, детализацию и данные об использовании в реальном времени. Здесь также можно управлять лицензией и подписками Dynatrace, пользователями Dynatrace, доступом SSO, а также отслеживать внедрение платформы Dynatrace и состояние окружений.

Annual commitment (годовое обязательство)
:   Годовое обязательство, это минимальная сумма, которую нужно тратить ежегодно в рамках контракта DPS. Соглашения DPS обычно заключаются на срок от 1 до 3 лет.

Billing usage events (события использования для биллинга)
:   Использование возможностей Dynatrace фиксируется как события использования для биллинга (BUE) в Grail. Дополнительные сведения об использовании хранятся там же, в виде атрибутов BUE. Доступ к ним можно получить через Account Management либо через собственные дашборды с запросами DQL.

Capability (возможность)
:   Dynatrace рассчитывает стоимость лицензирования на основе потребления окружением возможностей Dynatrace. Например, категория Application & Infrastructure Observability включает четыре возможности: Full-Stack Monitoring, Infrastructure Monitoring, Mainframe Monitoring и Foundation & Discovery. Их можно найти в прайс-листе (rate card).

Consumption (потребление)
:   Использование Dynatrace приводит к потреблению возможностей, которое измеряется в разных единицах измерения.

Costs (затраты)
:   Всё использование собирается и оценивается по прайс-листу (rate card) подписки. В результате использование преобразуется в затраты, выраженные в денежном эквиваленте. Затраты, это результат умножения использования на цену соответствующей возможности Dynatrace по прайс-листу.

Forecast (прогноз)
:   Подписка Dynatrace Platform Subscription предоставляет сводку бюджета, прогнозирующую затраты до конца периода подписки. Алгоритмы используют методы сезонного прогнозирования, анализируя прошлое использование и предсказывая будущее использование на основе повторяющихся паттернов использования и затрат.

Metered usage (измеряемое использование)
:   Dynatrace измеряет использование каждый раз при использовании возможности. Использование учитывает количество единиц, потребляемых для конкретной возможности Dynatrace. У каждой возможности своя единица измерения, например, "memory-gibibyte-hours" для Full-Stack Monitoring.

On-demand usage (использование по требованию)
:   Использование по требованию наступает после достижения минимального годового обязательства. Dynatrace никогда не взимает штрафную плату за превышение лимита. Вместо этого клиенты, потребляющие больше своего минимального годового обязательства по расходам, могут продолжать использовать платформу по требованию, с ежемесячной оплатой по тем же тарифам, что и предоплаченное потребление.

Rate card (прайс-лист)
:   Прайс-лист, это структурированный список цен, определяющий стоимость по возможностям в рамках модели DPS. Индивидуальный прайс-лист является частью заказа на покупку и доступен на портале Account Management. Для каждой возможности платформы в прайс-листе определена цена.

Subscription term (срок подписки)
:   Соглашение Dynatrace Platform Subscription обычно заключается на срок от 1 до 3 лет, с минимальным годовым обязательством. Для каждой возможности платформы в прайс-листе, входящем в соглашение, определена цена.
