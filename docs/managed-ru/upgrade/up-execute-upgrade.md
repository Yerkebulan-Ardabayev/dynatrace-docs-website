# Выполнение обновления развёртывания

* Updated on Aug 03, 2023

![Выполнение обновления с Managed на SaaS](https://dt-cdn.net/images/upgrade-to-saas-upgrade-3454-35eeb82133.png "Выполнение обновления с Managed на SaaS")

Выполнение обновления с Managed на SaaS

На этом этапе выполняется миграция компонентов платформы, инфраструктуры и мониторинга. Это включает лицензирование, среды SaaS, инфраструктуру, конфигурации Dynatrace, интеграции, а также все источники мониторинга приложений и инфраструктуры.

Сначала необходимо подготовить новую платформу Dynatrace SaaS и сопутствующую инфраструктуру. Затем следует последовательно перенести конфигурацию, интеграции и приложения.

## Шаг 1 — Подготовка

Основные цели этого шага:

* Прояснить возможные изменения в лицензировании
* Подготовить среду Dynatrace
* Развернуть необходимую инфраструктуру

Этот шаг сосредоточен на подготовке новой платформы Dynatrace (включая среду SaaS), всех вспомогательных компонентов инфраструктуры и конфигураций для поддержки ActiveGate, а также необходимых сетевых потоков.

### Запрос подготовки сред Dynatrace

Dynatrace создаёт новые среды SaaS в вашем аккаунте в выбранном гиперскейлере и регионе. Для этого требуется настройка надлежащего лицензирования для ваших сред SaaS, включая параллельную эксплуатацию кластера Managed. Предоставьте информацию о планируемых сроках миграции и ожидаемом росте нагрузки.

### Предварительная настройка новых сред

После подготовки среды SaaS выполняется её предварительная настройка, которая включает:

* Настройку аккаунта — SSO, пользователи и разрешения
* Настройку прокси, DNS и корпоративных фаерволов при необходимости
* Проверку с командой поддержки Dynatrace необходимости установки пользовательских флагов функций
* Проверку корректности настройки лицензирования для всех ваших сред

## Шаг 2 — Выполнение

Основные цели этого шага:

* Миграция конфигураций
* Миграция ActiveGate
* Миграция OneAgent

На этом шаге применяется вся конфигурация мониторинга к среде Dynatrace. Помимо этого, рекомендуется выполнить следующие действия.

### Миграция конфигурации

Среда Dynatrace может содержать большое количество отдельных элементов конфигурации. Ручная миграция может вызвать задержки в проекте и увеличить вероятность ошибок. Следуйте процедуре, описанной в разделе [Миграция конфигурации](/managed/upgrade/up-execute-upgrade/up-migrate-cfg "Migrate your Dynatrace Managed environment configuration using Monaco."), чтобы выполнить миграцию конфигурации наиболее эффективным способом.

#### Валидация конфигурации

После переноса конфигураций важно проверить, что все необходимые конфигурации успешно перенесены. Это гарантирует, что среда содержит все необходимые конфигурации для продолжения обработки данных после переключения OneAgent на платформу SaaS и обеспечивает плавное переключение для пользователей Dynatrace.

Для наиболее критичных конфигураций выполните ручную проверку, сравнив среды. Можно также использовать JSON `diff` в сочетании с REST API Dynatrace.

После валидации перенесённой конфигурации среда Dynatrace SaaS готова к приёму трафика от OneAgent.

### Миграция ActiveGate

Основная задача ActiveGate — маршрутизация трафика OneAgent к Dynatrace, мониторинг облачных сред или мониторинг удалённых технологий с использованием расширений. Поэтому перед миграцией OneAgent убедитесь, что они смогут безопасно и успешно подключиться к среде Dynatrace, [выполнив миграцию экземпляров ActiveGate](/managed/upgrade/up-execute-upgrade/up-migrate-ag "Migrate your ActiveGate configuration to SaaS."). На втором этапе выполните миграцию ActiveGate, отвечающих за мониторинг Synthetic и z/OS.

### Миграция OneAgent

Миграция OneAgent включает перенастройку или переустановку всех экземпляров OneAgent с Dynatrace Managed на новую среду SaaS. Этот шаг требует тщательного планирования и координации. Учитывайте процедуры развёртывания и обслуживания более широкой прикладной среды.

OneAgent следует мигрировать сразу после завершения миграции конфигурации. Это минимизирует расхождение между средами и обеспечит максимально плавный переход на новую платформу.

Серия шагов повторяется для каждого мигрируемого приложения. Используя план на основе приоритетов, выполните миграцию каждого приложения, следуя приведённым ниже шагам.

#### Перенастройка OneAgent на среду SaaS

Все OneAgent, развёрнутые в среде клиента, должны быть перенастроены на новую среду Dynatrace. OneAgent может быть развёрнут на облачных платформах, контейнерных платформах или операционных системах.

Существуют два подхода к перенаправлению OneAgent на новую среду SaaS:

* Обновление и перенастройка OneAgent на месте (рекомендуется) — подробнее см. в разделе [Миграция OneAgent](/managed/upgrade/up-execute-upgrade/up-migrate-oa "Migrate your OneAgent configuration to SaaS.")
* Переустановка OneAgent

Рекомендуется использовать автоматизированное решение. Также убедитесь, что параметры Host Group, Network Zone и Custom Metadata заданы корректно при установке в соответствии с архитектурным дизайном, разработанным в руководстве по обновлению.

#### Перезапуск процессов приложений и проверка их отслеживания в среде Dynatrace

Внедрение OneAgent в отслеживаемые процессы происходит только при запуске приложения. До перезапуска процессов приложений OneAgent продолжает указывать на среду Dynatrace Managed. После перенастройки OneAgent необходимо перезапустить все процессы приложений, в которых OneAgent настроен в режиме полного стека для глубокого мониторинга приложений.

### Валидация мониторинга

В качестве заключительного шага миграции проверьте, что данные приложений поступают в новую среду без перебоев: хосты, процессы, сервисы и приложения отображаются корректно, пользовательские настройки применены, а оповещения и уведомления настроены так же, как в исходной среде.

Проверьте данные в среде SaaS, чтобы убедиться в их корректности и нормальном поступлении метрик.

Рекомендуется регулярно проверять состояние обновления следующими способами:

* Сверять конфигурацию в старой и новой средах, проверять работу интеграций и корректность настройки ActiveGate
* Привлекать команды, использующие Dynatrace, чтобы они убедились в корректности настройки мониторинга

### Миграция Dynatrace Operator

Если Dynatrace Operator развёрнут в кластере Kubernetes, можно [выполнить миграцию мониторинга Kubernetes с кластера Managed в среду SaaS](/managed/upgrade/up-execute-upgrade/up-migrate-dto "Migrate your Dynatrace Operator configuration to SaaS."), перенастроив развёртывание Dynatrace Operator.

### Миграция расширений Dynatrace

Если для расширения возможностей наблюдаемости Dynatrace используются расширения, для сбора данных с них в среде SaaS потребуются дополнительные шаги. Шаги миграции расширений различаются в зависимости от типа и версии расширения.

* [Миграция расширения OneAgent](/managed/upgrade/up-execute-upgrade/up-migrate-oa-extension "Migrate your OneAgent extension to SaaS.")
* [Миграция расширения ActiveGate](/managed/upgrade/up-execute-upgrade/up-migrate-ag-extension "Migrate your ActiveGate extension to SaaS.")

## Шаг 3 — Интеграция

Основные цели этого шага:

* Обновление внешних интеграций для взаимодействия со средами SaaS
* Обеспечение повторной интеграции ландшафта мониторинга и оповещений после миграции

Все сторонние интеграции, ранее использовавшиеся с кластером Dynatrace Managed, необходимо обновить и проверить. Настройте и проверьте все интеграции и убедитесь, что сквозные рабочие процессы функционируют. Примеры таких интеграций: платформы управления сервисами, платформы автоматизации для автоматического устранения неполадок, CI/CD-конвейеры и системы отчётности.

### Обновление автоматизации, использующей API Dynatrace

Весь доступ через API необходимо обновить для работы с новой средой. Это требует генерации новых токенов API, поскольку существующие токены не могут быть перенесены по соображениям безопасности.
