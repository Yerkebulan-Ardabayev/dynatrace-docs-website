# Недоступно в Dynatrace Managed

# Недоступно в Dynatrace Managed

* Published Nov 28, 2023

Выбранная вами документация относится к функциям Dynatrace, доступным только в модели SaaS (программное обеспечение как услуга). Подробнее о Dynatrace см. в разделе [Что такое Dynatrace Managed](/managed/discover-dynatrace/what-is-dynatrace "Dynatrace Managed is an on-premises observability platform for analyzing, automating, and innovating with the power of AI. Learn how it helps deliver more secure software faster.").

## Почему только в SaaS?

Ряд инноваций использует возможности Grail, который доступен только в Dynatrace в модели SaaS.

Grail — это дата-лейкхаус Dynatrace, разработанный специально для данных наблюдаемости. Это единое решение для хранения логов, метрик, трейсов, событий и других данных. Все данные в Grail взаимосвязаны в рамках модели реального времени, отражающей топологию и зависимости в отслеживаемой среде.

* Grail спроектирован как дата-лейкхаус с массово-параллельной обработкой (MPP), масштабируемый до тысяч узлов.
* Локальные развёртывания Managed чаще всего рассчитаны на кластеры из 3, 12 или до 30 узлов.

При масштабировании до больших объёмов атрибуты базовой инфраструктуры должны быть последовательно реализованы, контролируемы и сбалансированы, что нецелесообразно в физических дата-центрах.

Grail доступен для всех клиентов Dynatrace SaaS, развёрнутых на AWS или Azure.

## Если вы готовы перейти на SaaS

Отлично.

Чтобы в полной мере воспользоваться облаком с Dynatrace SaaS, начните с раздела [Переход с Managed на развёртывание SaaS](/managed/upgrade "Start your upgrade from Dynatrace Managed deployment to Dynatrace SaaS deployment.") для получения обзора процесса миграции.

## Если вы не готовы перейти на SaaS

Не беспокойтесь.

Dynatrace Managed по-прежнему остаётся лидирующим решением для клиентов с обязательными требованиями к локальному развёртыванию.

Dynatrace продолжает поддерживать клиентов Managed и будет выпускать ежемесячные обновления продукта со множеством инноваций, не требующих возможностей Grail. Дорожная карта Managed Dynatrace не предусматривает поддержки Grail, поскольку Grail разработан исключительно для SaaS и массово-параллельной обработки данных.

Важным действием, которое можно предпринять сейчас, является точное определение препятствий для перехода вашей организации с Dynatrace Managed на Dynatrace SaaS.

* Составьте список всех препятствий и связанных с ними приложений. Возможно, не все отслеживаемые сущности имеют одинаковые требования: некоторые можно перенести на SaaS раньше, другие — позже. Временное гибридное развёртывание (SaaS и Managed) может оказаться хорошим решением.
* Свяжитесь с нами, чтобы совместно пройти по чек-листу и обеспечить плавный переход.
