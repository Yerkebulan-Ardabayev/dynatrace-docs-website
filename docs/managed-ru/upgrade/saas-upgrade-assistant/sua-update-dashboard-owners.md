# Обновление владельцев дашбордов в SaaS Upgrade Assistant

# Обновление владельцев дашбордов в SaaS Upgrade Assistant

* Published Dec 19, 2024

Latest Dynatrace

У каждого дашборда должен быть владелец. В Managed владельцы дашбордов настраиваются с использованием ID пользователя. Однако ID пользователя в Managed не обязательно будет действительным ID пользователя в аккаунте SaaS. Чтобы обеспечить видимость дашбордов после миграции, ![SaaS Upgrade Assistant](https://dt-cdn.net/images/saas-upgrade-assistant-61dc5b83c0.svg "SaaS Upgrade Assistant") **SaaS Upgrade Assistant** помогает в обновлении владения дашбордами следующим образом.

* Проверяет, какие владельцы дашбордов отсутствуют в аккаунте SaaS, помечает соответствующие дашборды, визуализирует результаты на столбчатой диаграмме и создаёт предупреждения для отслеживания таких случаев.
* Автоматически обновляет владельцев дашбордов, заменяя ID пользователей адресами электронной почты, заданными в кластере Managed. Если у некоторых пользователей отсутствует адрес электронной почты, можно задать домен электронной почты для добавления к ID пользователя.
* Предоставляет возможность отказаться от нежелательных функций в разделе **Dashboard ownership** в ![Edit](https://dt-cdn.net/images/share-settings-1859c5a34c.svg "Edit") **Settings**.

## Развёртывание

На главной странице выберите среду для перехода на страницу **Upgrade overview**, затем выберите тип развёртывания для этой среды. Если хотя бы у одного дашборда из конфигураций, включённых в развёртывание, отсутствует владелец для аккаунта SaaS, в процедуре развёртывания появится новый шаг **Update dashboard owners** со следующей информацией:

* Сведения о текущих настройках
* Раздел с диаграммой, показывающей количество:

  + Владельцев дашбордов, отсутствующих в аккаунте SaaS.
  + Дашбордов, затронутых проблемой отсутствия владельцев.

  Этот раздел скрыт, если создание предупреждений в ![Edit](https://dt-cdn.net/images/share-settings-1859c5a34c.svg "Edit") **Settings** отключено.
* **Automatic adjustments** показывает изменения, которые будут применены автоматически при развёртывании, и позволяет:

  + Проверить количество применяемых изменений.
  + Просмотреть все корректировки пользователей и при необходимости применить их немедленно.
  + Узнать количество пользователей в маппинге, у которых не настроен адрес электронной почты. Используйте ссылку для перехода в Settings и задайте домен электронной почты, который будет автоматически добавляться к ID пользователя.

  Этот раздел скрыт, если автоматическое обновление владельцев дашбордов отключено.
* **Manual adjustments** показывает количество пользователей, которые не будут обновлены автоматически.

  + Выберите **Show users without replacement suggestions**, чтобы просмотреть список таких пользователей.
  + Выберите пользователя из списка, чтобы открыть автоматически заполненную форму массового редактирования свойств для удобного ручного редактирования.

  Когда необходимы ручные корректировки?

  + Если у пользователя нет адреса электронной почты и не задан домен электронной почты.
  + Если автоматическое обновление владельцев дашбордов отключено.
  + Если некоторые пользователи отсутствуют в маппинге — это означает, что у них есть дашборды, однако они больше не существуют в кластере Managed. Как правило, это касается уволившихся сотрудников. В таком случае можно либо переназначить дашборды другим пользователям, либо исключить их из развёртывания.

  Этот раздел скрыт, если ручные корректировки не требуются.

## Предварительный просмотр корректировок владельцев дашбордов

Выберите **Preview dashboard owner adjustments**, чтобы открыть всплывающее окно с предложенными корректировками. В этой таблице можно:

* Просмотреть все корректировки владельцев дашбордов.
* Найти корректировки по предыдущим или новым значениям.
* Просмотреть изменения в дашбордах с выбранным владельцем.
* Применить все изменения немедленно.

Выберите **Confirm**, чтобы применить все перечисленные изменения автоматически в процессе развёртывания, или **Apply all now**, чтобы применить все изменения немедленно.

## Предупреждения

![SaaS Upgrade Assistant](https://dt-cdn.net/images/saas-upgrade-assistant-61dc5b83c0.svg "SaaS Upgrade Assistant") **SaaS Upgrade Assistant** проверяет наличие владельцев дашбордов в аккаунте SaaS и выводит предупреждения в нескольких местах, помогая пользователям в процессе обновления владения.

Предупреждения могут отображаться в следующих местах:

* Вкладка **Review errors** — для каждого дашборда формируется одно предупреждение. Их можно просмотреть и перейти к деталям конфигурации.
* Вкладка **Upgrade details: configuration** — значок предупреждения рядом с типом конфигурации `dashboard` означает, что для одного или нескольких дашбордов требуется действие. Тот же значок отображается рядом с именами затронутых конфигураций дашбордов.

Выберите затронутую конфигурацию и перейдите на вкладку **Errors** в этой конфигурации, чтобы просмотреть предупреждение с инструкциями.

Отсутствующие владельцы дашбордов помечены значком предупреждения на вкладке **Edit properties**.

## Предложения

Если нет необходимости обновлять всех владельцев дашбордов автоматически, ![SaaS Upgrade Assistant](https://dt-cdn.net/images/saas-upgrade-assistant-61dc5b83c0.svg "SaaS Upgrade Assistant") **SaaS Upgrade Assistant** также может отображать предложения на основе маппинга адресов электронной почты, экспортированного из кластера Managed. Предложения — это фактически те корректировки, которые были бы применены автоматически, но их можно использовать по отдельности: для этого не нужно включать автоматическое обновление владельцев дашбордов.

Чтобы применить предложения, выберите **Use value** в:

* Форме массового редактирования свойств
* Вкладке **Edit properties** отдельной конфигурации

## Настройки владения дашбордами

Автоматическое обновление владельцев дашбордов также можно настроить в разделе ![Edit](https://dt-cdn.net/images/share-settings-1859c5a34c.svg "Edit") **Settings** > **Dashboard ownership**, который предоставляет следующие возможности.

* Переключатель **Automatically update dashboard owners** — включён по умолчанию. При отключении владельцы дашбордов не будут обновляться в процессе развёртывания.
* Переключатель **Create a warning if dashboard ownership is set to a user that doesn't exist in your SaaS account** — включён по умолчанию. При отключении ![SaaS Upgrade Assistant](https://dt-cdn.net/images/saas-upgrade-assistant-61dc5b83c0.svg "SaaS Upgrade Assistant") **SaaS Upgrade Assistant** не будет проверять наличие владельца и не будет отображать предупреждения или диаграмму затронутых дашбордов.
* Поле ввода домена электронной почты — по умолчанию пустое. Если заданный домен корректен, он будет добавляться к ID существующих пользователей без адреса электронной почты. Эта опция особенно полезна при использовании LDAP-интеграции (например, ID пользователей в формате `firstname.lastname` и отсутствие адресов электронной почты на LDAP-сервере). Добавляя домен компании к имени и фамилии, ![SaaS Upgrade Assistant](https://dt-cdn.net/images/saas-upgrade-assistant-61dc5b83c0.svg "SaaS Upgrade Assistant") **SaaS Upgrade Assistant** может сформировать корректный адрес электронной почты для пользователей SaaS.

  Домен электронной почты не будет добавляться к отсутствующим владельцам дашбордов из маппинга пользователей кластера Managed.

## Возможные сценарии

### Развёртывание в один клик с настройками по умолчанию

#### Допущения

* Настройки не изменялись.
* Загружен корректный архив конфигураций и выполнен переход к исходной среде.
* Все действующие пользователи имеют настроенные адреса электронной почты в Managed.

#### Шаги

1. Выберите шаг развёртывания **Deploy all configurations** и нажмите **Next**.
2. На шаге **Update dashboard owners** нажмите **Next**.
3. На шаге **Prepare authorization token** при необходимости выберите **Generate token for 24 hours**, затем нажмите **Next**.
4. На шаге **Run deployment** выберите **Run full deployment** и подтвердите.
5. В разделе **Upgrade results** дождитесь завершения развёртывания и проверьте результаты.
   **Примечание:** после завершения развёртывания может потребоваться обновить страницу для актуализации свойств пользователей и связанных предупреждений.

Диагностика

* Если изменения на шаге **Update dashboard owners** не отображаются, обновите страницу.
* Если владельцы обновлены, однако предупреждения по-прежнему сигнализируют об их отсутствии, пригласите пользователей Managed в аккаунт SaaS, используя их адрес электронной почты из Managed в качестве логина.
* Если все активные пользователи приглашены, однако часть предупреждений сохраняется, это может быть вызвано старыми дашбордами удалённых пользователей. Для решения проблемы можно:

  + Вернуться на шаг **Update dashboard owners** и выбрать **Show users without replacement suggestions**. Для каждого отображаемого недействительного владельца дашборда выберите ссылку и отредактируйте значение в форме массового редактирования. В поле **Replace with** следует указать адрес электронной почты существующего пользователя. После внесения всех изменений запустите выборочное развёртывание для отредактированных дашбордов.
  + Найти все затронутые дашборды и удалить их вручную из среды SaaS.

### Исправление дашбордов перед развёртыванием

#### Допущения

* Настройки не изменялись.
* Загружен корректный архив конфигураций и выполнен переход к исходной среде.
* Все действующие пользователи имеют настроенные адреса электронной почты в Managed.

#### Шаги

1. На шаге **Select deployment type** выберите **Deploy all configurations**, затем нажмите **Next**.
2. На шаге **Update dashboard owners**:

   1. В разделе **Automatic adjustments** выберите **Preview dashboard owner adjustments**.
   2. В всплывающем окне **Preview dashboard owner adjustments** проверьте изменения и нажмите **Apply all now**.

      Вместо кнопки **Preview** должно появиться следующее сообщение об успехе:

      `All existing automatic owner replacement suggestions have already been applied.`
   3. Если количество предупреждений не уменьшается после обновления, убедитесь, что пользователи Managed приглашены в аккаунт SaaS с их адресом электронной почты из Managed в качестве логина.
   4. В разделе **Manual adjustments** выберите **Show users without replacement suggestions** и исправьте всех таких пользователей.
   5. Нажмите **Next**, если шаг ещё виден. Если нет — перейдите непосредственно к следующему шагу.
3. На шаге **Prepare authorization token** при необходимости выберите **Generate token for 24 hours**, затем нажмите **Next**.
4. На шаге **Run deployment** выберите **Run full deployment** и подтвердите.
5. В разделе **Upgrade results** дождитесь завершения развёртывания.
6. Ожидаемый результат: все предупреждения были устранены до развёртывания.

### Выборочное развёртывание

1. На шаге **Select deployment** выберите **Choose configurations to deploy**, затем нажмите **Next**.
2. Шаг выбора конфигураций для включения в развёртывание.

   1. Перейдите на вкладку **Upgrade details: configuration** и включите несколько дашбордов в следующее развёртывание.
   2. Нажмите **Next**.
3. На шаге **Update dashboard owners** проверьте корректировки и нажмите **Next**.
4. На шаге **Prepare authorization token** при необходимости выберите **Generate token for 24 hours**, затем нажмите **Next**.
5. На шаге **Run deployment** выберите **Run selective deployment** и подтвердите.
6. В разделе **Upgrade results** дождитесь завершения развёртывания.
7. Ожидаемый результат: только выбранные конфигурации были изменены и развёрнуты. Остальные дашборды не затронуты. Владельцы остальных дашбордов также не изменены, за исключением случаев, когда они являлись владельцами некоторых из выбранных дашбордов.
8. Необязательно. Если требуется мигрировать дополнительные конфигурации, выберите **More configurations** и повторите выборочное развёртывание несколько раз. При каждом развёртывании владельцы дашбордов будут автоматически обновляться только для дашбордов, включённых в развёртывание.

### Пользователи Managed не имеют адресов электронной почты

#### Допущения

* В кластере Managed используется LDAP-интеграция, и на LDAP-сервере нет адресов электронной почты, поэтому у всех пользователей отсутствуют адреса электронной почты, однако их ID легко идентифицировать в формате `firstname.lastname`.
* Настройки не изменялись.
* Загружен корректный архив конфигураций и выполнен переход к исходной среде.

#### Шаги

1. На шаге **Select deployment** выберите **Deploy all configurations**, затем нажмите **Next**.
2. На шаге **Update dashboard owners**:

   1. В разделе **Automatic adjustments** выберите **Settings**.
   2. Перейдите в раздел **Dashboard ownership**, задайте домен электронной почты и нажмите **Save**.
   3. Вернитесь к исходной среде.
   4. Проверьте корректировки и нажмите **Next**.
3. На шаге **Prepare authorization token** при необходимости выберите **Generate token for 24 hours**, затем нажмите **Next**.
4. На шаге **Run deployment** выберите **Run full deployment** и подтвердите.
5. В разделе **Upgrade results** дождитесь завершения развёртывания.
6. Ожидаемый результат: владельцы дашбордов теперь представлены корректными адресами электронной почты. Предупреждения могут сохраняться, и может потребоваться пригласить пользователей в аккаунт SaaS или исправить некоторые адреса электронной почты, как описано в базовом сценарии.

Домен электронной почты не был добавлен к некоторым владельцам.

Возможны две причины:

* Пользователь был удалён из кластера Managed и отсутствует в маппинге.
* ID пользователя содержит недопустимые символы, и добавление домена электронной почты не образует корректный адрес.

В любом из этих случаев необходимо скорректировать таких владельцев вручную.

### Полностью ручное обновление владельцев

#### Допущения

* Отключена функция **Automatically update dashboard owners**, и требуется вручную проверить и отредактировать конфигурации.
* Загружен корректный архив конфигураций и выполнен переход к исходной среде.

#### Шаги

1. На шаге **Select deployment** выберите **Choose configurations to deploy**, затем нажмите **Next**.
2. Шаг выбора конфигураций для включения в развёртывание.

   1. Перейдите на вкладку **Upgrade details: configuration** и выберите **Include all**.
   2. Просмотрите конфигурации и исключите ненужные.

      1. При просмотре дашбордов обратите внимание на значок предупреждения и ошибку для каждого дашборда. Если это происходит для дашборда, который требуется включить, выберите имя пользователя в сообщении об ошибке.
      2. Выберите **Use value** или введите корректный адрес электронной почты пользователя вручную.
      3. Выберите **Preview changes** > **Save**. Если появится сообщение `Changed properties were found in other configs`, выберите **Preview changes** и **Save** ещё раз, затем нажмите **Done**.
   3. Вернитесь на вкладку **Overview** и нажмите **Next**.
3. На шаге **Prepare authorization token** при необходимости выберите **Generate token for 24 hours**, затем нажмите **Next**.
4. На шаге **Run deployment** выберите **Run selective deployment** и подтвердите.
5. В разделе **Upgrade results** дождитесь завершения развёртывания.

Если в дальнейшем требуется развернуть другие дашборды и исправить всех владельцев:

1. Перейдите в **Upgrade details: configuration** > **Bulk edit**.
2. Выберите `Dashboard owner` в качестве группы свойств.
3. Выпадающий список **Find** будет содержать всех владельцев. Выберите тех, которых требуется отредактировать.
4. Необязательно. Если требуется использовать предложения, выберите **Use value**.

## FAQ

Что происходит, если маппинг пользователей отсутствует?

Автоматическое обновление владельцев дашбордов невозможно без маппинга пользователей. Этот маппинг был добавлен в кластеры Managed версии 1.298. Если используется старый пакет конфигурации без маппинга пользователей, приложение отобразит предупреждение.

Рекомендуется повторно экспортировать пакет из обновлённого кластера Managed.

Маппинг пользователей — это файл `users.ndjson`, расположенный в пакете экспорта конфигурации.

Что делать, если шаг Update dashboard owners не отображается?

Это может означать, что все владельцы ваших дашбордов уже являются действительными пользователями SaaS. Если это не так, убедитесь, что тип развёртывания выбран. При планировании выборочного развёртывания включите хотя бы один дашборд, чтобы шаг появился.

Что делать, если владельцы были обновлены автоматически, но предупреждения всё равно сигнализируют об их отсутствии?

Корректировка владельцев дашбордов в соответствии с автоматическими предложениями не гарантирует, что все пользователи смогут видеть свои дашборды. При возникновении этой проблемы:

* Убедитесь, что адрес электронной почты затронутого пользователя в кластере Managed указан корректно.
* Создайте пользователя в аккаунте SaaS с корректным адресом электронной почты.
