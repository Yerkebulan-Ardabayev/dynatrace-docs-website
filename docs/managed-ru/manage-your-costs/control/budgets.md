# Budget alerts

# Budget alerts

* Пояснение
* 3 мин чтения
* Обновлено 03 нояб. 2025 г.

Вкладка **Budgets** в Account Management позволяет настраивать бюджеты подписки Dynatrace Platform Subscription (DPS). Можно:

* определять бюджеты на уровне аккаунта, окружения и capability;
* настраивать уведомления, информирующие о превышении пороговых значений.

Примеры поддерживаемых сценариев:

* получать уведомления при достижении подпиской заданного процента от годового обязательства;
* установить пороговое значение бюджета в размере 10% от общей стоимости подписки для тестового окружения более низкого уровня («песочница»);
* установить фиксированное пороговое значение для новой capability и получать уведомление при достижении затрат в 1000 USD.

Budgets дополняют существующие [Cost monitors](/managed/manage-your-costs/control/cost-monitors "Learn how to use the Cost Monitors feature to make forecasts and cost events."), позволяя определять произвольные пороговые значения на уровне окружения и capability.

Чтобы перейти к разделу **Budgets**, откройте **Account Management** > **Subscription** > **Cost management** и выберите вкладку **Budgets**.

Уведомления о бюджете недоступны для пробных лицензий и Proof-of-Concept.

## Обзор

Budgets, это лимиты расходов, привязанные к годовому обязательству по DPS.
Их задают на уровне аккаунта, окружения или capability, и Dynatrace уведомляет о приближении фактического или прогнозируемого потребления к каждому пороговому значению (значения по умолчанию: 75%, 90%, 100%).
Budgets представляют собой плановые ограничители, дополняющие cost monitors, которые обнаруживают аномалии по мере их возникновения.

## Сценарии использования

* Установить предельный уровень расходов на уровне аккаунта, соответствующий годовому обязательству.
* Ограничить отдельные окружения долей от общего обязательства.
* Применить бюджет для отдельной capability, если именно она (например, логи или метрики) формирует основную часть затрат.
* Уведомлять финансовый отдел, FinOps или владельцев capability при приближении фактических затрат к пороговому значению.

## Рекомендации по работе с бюджетами

Подписка Dynatrace Platform Subscription автоматически включает три исходных пороговых значения бюджета, которые уведомляют администраторов лицензии при достижении общим потреблением на уровне аккаунта 75%, 90% и 100% от годового обязательства.

По умолчанию уведомления при превышении бюджета получают все пользователи с правом аккаунта **View and manage account and billing information**.
Эти пороговые значения можно скорректировать под свои нужды, а также добавить дополнительных получателей, чтобы нужные пользователи были в курсе.

## Просмотр бюджетов

Вкладка **Budgets** отображает все активные бюджеты, определённые для аккаунта.
Каждый бюджет отображается в виде карточки, которая показывает:

* общую сумму бюджета, израсходованную за период выставления счёта;
* пороговое значение стоимости для бюджета;
* цветной индикатор прогресса, сравнивающий фактическое потребление с пороговым значением стоимости.

  + Синий: израсходовано менее 75% от порогового значения стоимости.
  + Коричневый: израсходовано 75% или более от порогового значения стоимости.
  + Красный: израсходовано 100% или более от порогового значения стоимости.
* количество настроенных окружений;
* количество настроенных capabilities.

## Создание и редактирование бюджетов

Чтобы просмотреть и изменить настройки бюджета, выберите его на странице сводки **Budgets**, либо выберите **+ New budget** для создания нового бюджета.
Можно иметь до 20 активных бюджетов.

![Edit budget alerts in Account Management](https://dt-cdn.net/images/budgets-edit-5760-1e178f41f3.png)

Редактирование уведомлений о бюджете в Account Management

Создавать бюджеты и изменять их настройки могут только пользователи с правом аккаунта **View and manage account and billing information**.

Для каждого бюджета требуется следующая информация:

* **Name**: используется для идентификации бюджета в портале Account Management и в письмах уведомлений.
* **Monitored environment**: бюджет можно определить для всех окружений или для конкретных окружений.
  Выберите окружения, к которым применяется бюджет.
* **Monitored capabilities**: бюджет можно определить для всех capabilities или для одной либо нескольких выбранных capabilities.
  Выберите capabilities, к которым применяется бюджет.
* **Cost threshold type**: тип порогового значения, фиксированное значение или процент от годового обязательства по DPS.
* **Threshold amount**: значение, фиксированное или в виде процента, как определено в **Cost threshold type**.
* **Email notification details**: для каждого бюджета можно задать список email-адресов для отправки уведомлений при превышении бюджета:

  + **Notify license administrators** отправляет уведомления всем пользователям с правом аккаунта **View and manage account and billing information**.
    Чтобы определить пользователей с этим правом, откройте **Identity & access management** > **People**.
  + **Send to others** позволяет указать другие email-адреса пользователей, которые должны получать уведомления о бюджете.
    Письма с уведомлениями содержат ссылку на Account Management, поэтому получателям требуется право **View account access**.
  + **Subject**: используется в теме письма (необязательно; если не задано, используется текст по умолчанию).
  + **Message**: используется в описании письма (необязательно; если не задано, используется текст по умолчанию).

Новая карточка бюджета станет видна примерно через 20 минут.
До этого момента будет отображаться сообщение **We're creating your budget. Check back later to see cost data.**

Каждую карточку бюджета можно удалить.
Наведите курсор на карточку и выберите  >  **Delete**.

## Уведомления о бюджете

[Cost notifications](/managed/manage/account-management/notifications "Learn about Account Management notifications, for example budget alerts and cost monitor events.") информируют о достижении пороговых значений бюджета.
Уведомления можно получать в **Notification Center** и по email.

Когда затраты достигают или превышают заданное пороговое значение для выбранных окружений и capabilities, письмо отправляется пользователям, указанным в определении бюджета.

* События и письма по бюджету срабатывают только при превышении заданного пользователем порогового значения.
* Письмо отправляется только один раз, в день превышения порогового значения.
  Если задать пороговое значение, которое уже превышено, письмо-уведомление отправлено не будет.
* При одновременном превышении нескольких бюджетов пользователь получит отдельное письмо по каждому превышенному бюджету.

Чтобы перейти в **Notification Center** Account Management, выберите **View your account** в любом письме-уведомлении.
