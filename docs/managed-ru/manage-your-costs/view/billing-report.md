# Billing report

# Billing report

* Пояснение
* Чтение 1 мин
* Опубликовано 15 ноября 2023

Отчёт по биллингу DPS содержит все детали состояния бюджета DPS за каждый день текущего периода. В нём показано, какие затраты были учтены в какой день, с учётом даты учёта. Между датой потребления и датой учёта есть разница: потребление всегда происходит в момент использования возможностей Dynatrace. Учёт происходит, когда потребление признаётся и вычитается из годового обязательства DPS. Поэтому дата учёта может отличаться от даты потребления.

## Обзор

Отчёт по биллингу, это представление учтённых затрат с ежедневной периодичностью за период DPS. Он дополняет представления потребления в реальном времени (которые показывают использование по мере его возникновения), отражая то, что признано в счёт годового обязательства. Те же исходные данные доступны через API Account Management для интеграций с финансовыми системами.

## Сценарии использования

* Сверка учтённых затрат с обязательством по подписке по дням.
* Выявление дневных колебаний учтённых затрат.
* Программное извлечение данных отчёта по биллингу для финансовых процессов.

* Подтверждение, какие кластеры сообщили об использовании в конкретный день.

## Доступ к отчёту по биллингу

Чтобы получить доступ к отчёту по биллингу в Dynatrace, перейти в **Account Management** > **Subscription** > **Billing report**.

## Диаграммы отчёта

Отчёт по биллингу состоит из следующих диаграмм:

* **Booked costs to date:** эта диаграмма показывает накопленные затраты по дню учёта. Она также показывает годовое минимальное обязательство по подписке.
* **Booked costs per day:** эта диаграмма показывает затраты, учтённые за каждый отдельный день. Она не содержит накопленного состояния бюджета. Эта диаграмма помогает выявлять колебания в дневных затратах.

* **Number of reporting clusters:** эта диаграмма показывает количество отдельных кластеров, сообщивших об использовании в каждый день. На этой диаграмме можно выявить периоды, когда некоторые кластеры были офлайн. Когда кластеры временно офлайн, дневные затраты обычно снижаются.
