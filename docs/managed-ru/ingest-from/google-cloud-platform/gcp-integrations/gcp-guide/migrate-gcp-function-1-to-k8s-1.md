# Миграция с Google Cloud Functions 1.0 на Google Cloud k8s 1.0

* Практическое руководство
* Чтение: 2 мин
* Опубликовано 29 ноября 2022 г.

Dynatrace версии 1.230+

Поскольку интеграция Google Cloud на основе функций (v1.0) выводится из эксплуатации, пользователям следует перейти на интеграцию Google Cloud на основе k8s (v1.0).

Оба вида интеграции имеют одинаковые возможности. Меняются только развёртываемые компоненты. Непрерывность данных сохраняется, за исключением промежутка времени между развёртываниями. Вам не нужно ничего менять в дашбордах или оповещениях, поскольку ключи метрик остаются прежними.

## Перед началом

Рекомендуется иметь файл `activation-config.yaml`, который вы использовали при развёртывании функции. Большинство параметров, необходимых для нового развёртывания, можно повторно использовать на основе этого файла.

Файл `activation-config.yaml` задаёт область мониторинга (сервисы и наборы функций). В новом развёртывании необходимо указать ту же область мониторинга, чтобы охват мониторинга не изменился.

Если файла у вас нет, вы можете проверить текущую область мониторинга в развёртывании функции:

1. Найдите свою функцию мониторинга в Google Cloud. По умолчанию используется имя `dynatrace-gcp-monitor`.
2. Перейдите на вкладку **Variables**.
3. Скопируйте переменную `ACTIVATION_CONFIG` и сохраните её. Её можно использовать как образец при настройке области мониторинга (сервисы и наборы функций) для нового развёртывания.

## Руководство по миграции

1. Удалите интеграцию Google Cloud на основе функций.

   Последовательное обновление (развёртывание новой интеграции до удаления старой) не рекомендуется, так как метрики будут передаваться дважды, что может повлиять на ваши данные.

   См. [Удаление](/managed/ingest-from/google-cloud-platform/legacy/deploy-with-google-cloud-function-legacy#uninstall "Настройка мониторинга сервисов Google Cloud с помощью Google Cloud Function.").

   Убедитесь, что вы отключили все расширения Google Cloud в интерфейсе Dynatrace. Либо вы можете добавить параметр `--upgrade-extensions` во время развёртывания на шаге 3 при запуске скрипта развёртывания `./deploy-helm.sh`.
2. Разверните интеграцию Google Cloud на основе k8s с нуля.

   См. [Настройка интеграции метрик и логов Dynatrace с Google Cloud в новом кластере GKE Autopilot](/managed/ingest-from/google-cloud-platform/gcp-integrations/gcp-guide/deploy-k8 "Настройка мониторинга логов и метрик для сервисов GCP в новом кластере GKE Autopilot.").
3. Убедитесь, что интеграция Google Cloud на основе k8s работает.

   См. [Проверка установки](/managed/ingest-from/google-cloud-platform/gcp-integrations/gcp-guide/deploy-k8#verify "Настройка мониторинга логов и метрик для сервисов GCP в новом кластере GKE Autopilot.").
