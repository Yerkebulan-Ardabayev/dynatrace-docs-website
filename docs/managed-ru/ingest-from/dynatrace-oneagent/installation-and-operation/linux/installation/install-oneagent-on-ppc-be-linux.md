# Установка OneAgent на PPC BE Linux

* Чтение: 3 мин
* Обновлено 22 января 2026 г.

Чтобы установить Dynatrace OneAgent при развёртывании Dynatrace SaaS, перейдите на [Dynatrace.com](https://www.dynatrace.com) и выполните **Login**, используя имя пользователя и пароль, которые вы получили от Dynatrace в письме с подтверждением регистрации. При развёртывании Dynatrace Managed [откройте Cluster Management Console и выберите окружение](/managed/managed-cluster/operation/manage-your-monitoring-environments "Узнайте, как создавать, настраивать, открывать, удалять, отключать окружения мониторинга и переключаться между ними."), которое нужно мониторить. Затем перейдите к шагам установки ниже.

## Требования

* Вам нужны [разрешения](/managed/ingest-from/dynatrace-oneagent/installation-and-operation/linux/installation/linux-non-privileged "Узнайте, когда Dynatrace OneAgent требует привилегий root на Linux.") для следующих действий:

  + Создать каталог, в который вы хотите установить OneAgent
  + Изменить настройки брандмауэра (необходимо только если ваша внутренняя политика маршрутизации может препятствовать доступу программного обеспечения Dynatrace в Интернет).
  + Перезапустить сервисы ваших приложений
* Ознакомьтесь с [требованиями к дисковому пространству](/managed/ingest-from/dynatrace-oneagent/installation-and-operation/aix/installation/disk-space-requirements-for-oneagent-installation-and-update-on-aix "Узнайте о требованиях к дисковому пространству для установки OneAgent на AIX.").
* Все хосты, которые должны мониториться, должны иметь возможность отправлять данные в кластер Dynatrace. В зависимости от того, является ли ваше развёртывание Dynatrace SaaS или Managed, а также от схемы сети и настроек безопасности можно либо предоставить прямой доступ к кластеру Dynatrace, либо [настроить ActiveGate](/managed/ingest-from/dynatrace-activegate "Изучите базовые концепции, связанные с ActiveGate.").

* На PPC BE Linux OneAgent поддерживает только Java и Apache/IHS.
* Для установки OneAgent на PPC BE Linux доступ root не требуется.
* OneAgent можно установить в любой каталог.

### Разрешите подключения через брандмауэр

Убедитесь, что настройки вашего брандмауэра разрешают связь с Dynatrace.  
В зависимости от политики вашего брандмауэра вам может потребоваться явно разрешить определённые исходящие подключения. **Удалённые адреса Dynatrace, которые нужно добавить в список разрешённых, указаны на странице установки OneAgent.**

## Установка

1. Перейдите в **Deploy Dynatrace**.
2. Выберите **Start installation** > **Linux**.
3. Выберите тип установщика **PowerPC (BE)** из списка. Скопируйте команду из текстового поля **Use this command on the target host to download OneAgent for Linux PowerPC (BE)**
4. Войдите на ваш хост PPC BE Linux и выполните команду, скопированную из Dynatrace.

   * Если вы планируете загружать Dynatrace OneAgent напрямую на сервер, учтите, что устаревшие или отсутствующие библиотеки (например, CA-сертификаты или OpenSSL) не позволят выполнить загрузку.
   * Dynatrace использует зашифрованные подключения. OpenSSL необходим, чтобы `wget` мог получить доступ к серверу. ZIP-архив OneAgent также можно скачать, нажав ссылку **Download OneAgent installer** в нижнем колонтитуле страницы и сохранив скрипт установщика в любое место; это полностью исключает необходимость в команде `wget`.
5. В файловой системе создайте папку для установки OneAgent. Например, `/opt/dynatrace/oneagent`.
6. Распакуйте ZIP-архив OneAgent в только что созданную папку.

   Все мониторируемые приложения должны иметь возможность читать библиотеку OneAgent. Убедитесь, что права это разрешают.
7. Теперь у вас есть два варианта: мониторить каждое приложение на хосте или только одно приложение.

   Все приложения

   Одно приложение

   Чтобы автоматически мониторить каждое приложение на хосте, включите компонент `liboneagentproc.so` OneAgent. Он находится в системном каталоге библиотек (`/lib` или `/lib64`, в зависимости от вашей архитектуры), в `/etc/ld.so.preload`.

   ## Вы на месте!

   Отлично, настройка завершена! Теперь можно осмотреться в своём новом окружении мониторинга.

   Доступ к окружению мониторинга можно получить через [Cluster Management Console](/managed/managed-cluster/operation/manage-your-monitoring-environments "Узнайте, как создавать, настраивать, открывать, удалять, отключать окружения мониторинга и переключаться между ними.").

   ![Вы на месте](https://dt-cdn.net/images/arrive-1533-e7eb3573a6.png)

   Вы на месте

   И последнее: чтобы мониторить ваши процессы, их нужно перезапустить. Перезапустить процессы можно в любое время, даже во время следующего планового периода обслуживания в вашей организации.

   Чтобы мониторить одно приложение, сначала перезапустите его. Предварите команду запуска приложения следующими командами:

   ```
   DT_HOME=<installation directory>



   export DT_HOME



   LD_PRELOAD=$DT_HOME/agent/<system library>/liboneagentproc.so



   export LD_PRELOAD
   ```

   Где:

   * `<installation directory>` обозначает каталог, в который установлен OneAgent
   * `<system library>` обозначает `/lib` или `/lib64`, в зависимости от вашей архитектуры

   ## Вы на месте!

   Отлично, настройка завершена! Теперь можно осмотреться в своём новом окружении мониторинга. При развёртывании SaaS можно в любое время открыть окружение мониторинга, перейдя на [Dynatrace.com](https://www.dynatrace.com) и нажав кнопку **Login** в правом верхнем углу. При развёртывании Dynatrace Managed можно [открыть окружение мониторинга через Cluster Management Console](/managed/managed-cluster/operation/manage-your-monitoring-environments "Узнайте, как создавать, настраивать, открывать, удалять, отключать окружения мониторинга и переключаться между ними.").
