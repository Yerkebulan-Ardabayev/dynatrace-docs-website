# Обновление Dynatrace OneAgent на AIX

* Практическое руководство
* Чтение: 1 мин
* Опубликовано 19 сентября 2018 г.

OneAgent, установленный в режиме full-stack, имеет встроенный настраиваемый механизм автоматического обновления.

Общие сведения об обновлении OneAgent, включая отслеживание обновлений и создание окон обновления, см. в разделе [Обновление OneAgent](/managed/ingest-from/dynatrace-oneagent/oneagent-update "Узнайте, как обновлять OneAgent.").

## Настройка обновлений OneAgent

Обновление OneAgent можно настроить глобально, для группы хостов и для отдельного хоста.

Настройка глобального обновления OneAgent

Когда автоматические обновления включены глобально, при появлении новой версии OneAgent все установленные экземпляры OneAgent, кроме тех, для которых вы отключили автообновления на уровне группы хостов или отдельного хоста, будут автоматически скачивать обновление и обновлять свои бинарные и конфигурационные файлы.

Настройки автоматического обновления на уровне группы хостов и отдельного хоста переопределяют глобальные настройки.

1. Перейдите в **Settings** > **Updates** > **OneAgent updates**.
2. Выберите один из вариантов обновления:

   * **Automatic updates at earliest convenience**  
     Обновлять все OneAgent автоматически, независимо от окон обновления.
   * **Automatic updates during update windows**  
     Обновлять все OneAgent автоматически в течение выбранного окна обновления.

     + При выборе этого параметра отображается список доступных окон обновления. Выберите одно из них.
     + Чтобы настроить новое окно обновления для обновлений OneAgent, перейдите в **Settings** > **Updates** > **Update windows for OneAgent updates**.
   * **No automatic updates**  
     Не обновлять OneAgent автоматически.

Ручной запуск **Update now to target version** обновит все хосты с выбранной комбинацией ОС и архитектуры, независимо от их статуса автоматического обновления.

Настройка обновления OneAgent для группы хостов

Настройки обновления OneAgent на уровне группы хостов переопределяют глобальные настройки и переопределяются настройками на уровне хоста.

1. Откройте страницу **Host group settings**.  
   Способы доступа:

   * Перейдите в **Settings** > **Monitoring** > **Monitoring overview**, найдите любой хост из группы хостов, которую требуется настроить, и выберите имя группы хостов (не имя хоста).
   * Перейдите в **Hosts**, откройте страницу хоста и разверните раздел **Properties and tags**. Если хост принадлежит группе, под пунктом **Host group** будет ссылка на неё.
   * Перейдите в **Deployment Status** > **OneAgents**. Если хост принадлежит группе хостов, под именем хоста отображается ссылка на страницу настроек группы хостов.
2. На странице **Host group settings** выберите **OneAgent updates** в левой части страницы.
3. Выберите один из вариантов обновления:

   * **Inherit global update settings**  
     Следовать глобальным настройкам обновления для OneAgent в этой группе хостов.

     + Текущая глобальная настройка отображается в скобках в этой строке.
     + Чтобы перейти на страницу глобальной конфигурации, выберите ссылку `global`.
   * **Automatic updates at earliest convenience**  
     Обновлять все OneAgent в этой группе хостов автоматически, независимо от окон обновления. Игнорировать глобальные настройки обновления.
   * **Automatic updates during update windows**  
     Обновлять все OneAgent в этой группе хостов автоматически в течение выбранного окна обновления. Игнорировать глобальные настройки обновления.

     + При выборе этого параметра отображается список доступных окон обновления. Выберите одно из них.
     + Чтобы настроить новое окно обновления для обновлений OneAgent, перейдите в **Settings** > **Updates** > **Update windows for OneAgent updates**.
   * **No automatic updates**  
     Не обновлять OneAgent в этой группе хостов автоматически. Игнорировать глобальные настройки обновления.

Ручной запуск **Update now to target version** обновит все хосты с выбранной комбинацией ОС и архитектуры, независимо от их статуса автоматического обновления.

Настройка обновления OneAgent на уровне хоста

Настройки обновления OneAgent на уровне хоста переопределяют настройки обновления OneAgent на глобальном уровне и на уровне группы хостов.

1. Откройте страницу **Host** для хоста, который требуется настроить.  
   Способы доступа:

   * Перейдите в **Hosts** и выберите хост.
   * Перейдите в **Settings** > **Monitoring** > **Monitoring overview**, откройте вкладку **Hosts** и выберите хост.
2. На странице **Host** откройте меню обзора (**…**) и выберите **Settings**.
3. Выберите **OneAgent updates** в левой части страницы.
4. Выберите один из вариантов обновления:

   * **Inherit…**  
     Следовать настройкам обновления группы хостов или глобальным настройкам для обновления этого OneAgent.

     + Если выбранный хост принадлежит группе хостов, текущая настройка группы хостов отображается в скобках в этой строке, а имя группы является ссылкой на страницу конфигурации группы хостов.
     + Если выбранный хост не принадлежит группе хостов, глобальная настройка отображается в скобках в этой строке, а «global» является ссылкой на страницу глобальной конфигурации (**Settings** > **Updates** > **OneAgent updates**).
   * **Automatic updates at earliest convenience**  
     Обновлять этот OneAgent автоматически, независимо от окон обновления. Игнорировать настройки обновления группы хостов.
   * **Automatic updates during update windows**  
     Обновлять этот OneAgent автоматически в течение выбранного окна обновления. Игнорировать глобальные настройки обновления.

     + При выборе этого параметра отображается список доступных окон обновления. Выберите одно из них.
     + Чтобы настроить новое окно обновления для обновлений OneAgent, перейдите в **Settings** > **Updates** > **Update windows for OneAgent updates**.
   * **No automatic updates**  
     Не обновлять этот OneAgent автоматически. Игнорировать глобальные настройки обновления.

Ручное обновление OneAgent на отдельных хостах

Чтобы вручную обновить OneAgent, работающий на отдельном хосте:

1. Перейдите в **Settings > Monitoring > Monitoring overview**.
2. Откройте вкладку **Hosts**.
3. Чтобы обновить OneAgent или скачать последнюю версию, выберите **Update** рядом с именем интересующего вас хоста.

   Кнопка **Update** появляется, только если установленная версия OneAgent на конкретном хосте устарела и если это OneAgent в режиме full-stack. Эта кнопка не отображается для PaaS- и standalone-OneAgent.
4. Выберите **Update now**.  
   Если кнопка **Update now** недоступна, у вас нет прав на скачивание установщика.

Как вариант, можно скачать последнюю версию установщика OneAgent, вручную скопировать его на целевой хост и выполнить установку непосредственно на целевом хосте.

Выбор версии OneAgent для установки на новых хостах

Чтобы управлять тем, какая версия OneAgent автоматически устанавливается на все новые хосты:

1. Перейдите в **Settings** > **Updates** > **OneAgent updates**.
2. В **Update mode** выберите **No automatic updates**, чтобы отключить автоматические обновления OneAgent.

   Подробнее об отключении автоматических обновлений OneAgent на PaaS/Kubernetes см. в [Параметры DynaKube для Dynatrace Operator на Kubernetes/OpenShift](/managed/ingest-from/setup-on-k8s/reference/dynakube-parameters "Список доступных параметров для настройки Dynatrace Operator на Kubernetes.").
3. В **Target version** выберите версию OneAgent для установки на все новые хосты.

Выбранная версия OneAgent также используется для интеграций PaaS.

Выбор версии OneAgent для обновления

Чтобы выбрать, до какой версии OneAgent выполнять обновление

1. Перейдите в **Settings** > **Updates** > **OneAgent updates**.
2. В списке **Target version** можно указать конкретную версию по номеру версии OneAgent или выбрать относительную целевую версию:

   * **Latest stable version**  
     Самая последняя стабильная версия OneAgent, доступная в вашем окружении. Фактический номер версии отображается в скобках.
   * **Previous stable version**  
     Стабильная версия перед **Latest stable version**. Соответствующий ей на данный момент номер версии отображается в скобках. Номер версии OneAgent увеличивается на *два* с каждым релизом, поэтому это число будет на *два* меньше последнего номера версии.
   * **Older stable version**  
     Стабильная версия перед **Previous stable version**. Соответствующий ей на данный момент номер версии отображается в скобках. Номер версии OneAgent увеличивается на *два* с каждым релизом, поэтому это число будет на *четыре* меньше последнего номера версии.

* **Specific OneAgent version**

Целевая версия используется для:

* Автоматических обновлений
* Автоматических обновлений в течение окон обслуживания
* Ручных обновлений, когда вы выбираете версию, до которой требуется обновиться

Целевую версию и режим обновления можно задать на уровне:

* **Environment level**
  Влияет на все OneAgent в окружении. Также используется для Deployment API.
* **Host group**
  Влияет на все OneAgent в группе хостов. Переопределяет уровень окружения. Не влияет на Deployment API.
* **Host**
  Влияет на OneAgent на этом хосте. Переопределяет конфигурацию уровня группы хостов и окружения. Не влияет на Deployment API.

Если вы выберете версию более раннюю, чем развёрнутая в настоящее время, понизить версию OneAgent будет невозможно. Потребуется установить более новую версию поверх существующей версии OneAgent.

## Системные требования

### Дисковое пространство

Подробнее см. [Файлы OneAgent и требования к дисковому пространству](/managed/ingest-from/dynatrace-oneagent/installation-and-operation/aix/installation/disk-space-requirements-for-oneagent-installation-and-update-on-aix "Узнайте о требованиях к дисковому пространству для установки OneAgent на AIX.")

### Свободная память

Для запуска обновления OneAgent хосту требуется 200 МБ свободной памяти.

## Проверка установленной версии OneAgent

Используйте один из этих методов, чтобы проверить, какая версия OneAgent установлена у вас в настоящее время.

### Интерфейс командной строки OneAgent

Запустите `oneagentctl` с параметром `--version`. Дополнительные сведения см. в [Настройка OneAgent через интерфейс командной строки](/managed/ingest-from/dynatrace-oneagent/oneagent-configuration-via-command-line-interface#display-oneagent-version "Узнайте, как выполнять некоторые задачи настройки OneAgent без переустановки OneAgent.").

### Обзор хоста

1. Перейдите в **Hosts**.
2. Щёлкните интересующий вас хост.
3. Разверните **Properties** под именем хоста. Установленная версия OneAgent указана в перечисленных свойствах.

### Статус развёртывания

1. Перейдите в **Deployment Status**.
2. Откройте вкладку **All hosts** или **Recently connected hosts**.
3. Разверните интересующую вас запись хоста. Установленная версия OneAgent включена в отображаемую информацию.
