# Сообщения модулей z/OS - системные сообщения модуля IMS

# Сообщения модулей z/OS - системные сообщения модуля IMS

* Чтение: 3 мин
* Опубликовано 22 марта 2019 г.

### ZDTI032W

|  |  |
| --- | --- |
| Полное сообщение | ZDTI032W Recovery routine entered. |
| Пояснение | Подпрограмма восстановления после ABEND модуля IMS была вызвана RTM. |
| Действие системы | Процесс восстановления после ABEND продолжается. |
| Реакция пользователя | Обратитесь к специалисту по продуктам Dynatrace через чат в вашем окружении. |

### ZDTI033W

|  |  |
| --- | --- |
| Полное сообщение | ZDTI033W Successful ABEND recovery, agent disabled. |
| Пояснение | Процесс восстановления после ABEND в модуле IMS завершился успешно. |
| Действие системы | Модуль IMS отключён, а система IMS продолжает работу. |
| Реакция пользователя | Обратитесь к специалисту по продуктам Dynatrace через чат в вашем окружении. |

### ZDTI034E

|  |  |
| --- | --- |
| Полное сообщение | ZDTI034E Unable to obtain dynamic storage. |
| Пояснение | Подпрограмма восстановления после ABEND модуля IMS не смогла получить динамическую память, необходимую для формирования диагностики ABEND. |
| Действие системы | Процесс восстановления после ABEND продолжается, но часть диагностической информации может быть недоступна. Программная (`SFT`) запись об ошибке с дополнительным описанием ABEND записывается в системный набор данных z/OS `SYS1.LOGREC`. |
| Реакция пользователя | Запустите служебную программу z/OS EREP, чтобы распечатать программную (`SFT`) запись об ошибке, связанную с ABEND. Обратитесь к специалисту по продуктам Dynatrace через чат в вашем окружении. |

### ZDTI035W

|  |  |
| --- | --- |
| Полное сообщение | ZDTI035W Retry not permitted, must percolate. |
| Пояснение | Система z/OS не разрешает процессу восстановления после ABEND в модуле IMS продолжать работу. |
| Действие системы | Процесс восстановления завершается и выполняет percolate (передаёт управление обратно RTM). |
| Реакция пользователя | Обратитесь к специалисту по продуктам Dynatrace через чат в вашем окружении. |

### ZDTI036W

|  |  |
| --- | --- |
| Полное сообщение | ZDTI036W ZDTIIInn zzzzzzzz yyyymmdd hh.mm VER vv.rr.mm ABEND at offset xxxxxx. |
| Пояснение | ABEND произошёл по смещению `xxxxxx` в модуле `ZDTIIInn` модуля IMS, где `nn` соответствует версии IMS (например, `12`). Остальные поля описывают уровень обслуживания модуля IMS. |
| Действие системы | Программная (`SFT`) запись об ошибке с дополнительным описанием ABEND записывается в системный набор данных z/OS `SYS1.LOGREC`. Опционально может быть снят SVC dump. Процесс восстановления после ABEND продолжается. |
| Реакция пользователя | Запустите служебную программу z/OS EREP, чтобы распечатать программную (SFT) запись об ошибке, связанную с ABEND. Обратитесь к специалисту по продуктам Dynatrace через чат в вашем окружении. |

### ZDTI037W

|  |  |
| --- | --- |
| Полное сообщение | ZDTI037W ZDTIIInn zzzzzzzz yyyymmdd hh.mm VER vv.rr.mm ABEND offset N/A, no SDWA provided. |
| Пояснение | ABEND произошёл в модуле `ZDTIIInn` модуля IMS, где `nn` соответствует версии IMS (например, `12`). Остальные поля описывают уровень обслуживания модуля IMS. Смещение ABEND определить не удалось, так как RTM не предоставил SDWA. |
| Действие системы | Процесс восстановления завершается и выполняет percolate (передаёт управление обратно RTM). |
| Реакция пользователя | Обратитесь к специалисту по продуктам Dynatrace через чат в вашем окружении. |

### ZDTI038W

|  |  |
| --- | --- |
| Полное сообщение | ZDTI038W Recovery routine of recovery routine entered. |
| Пояснение | Во время восстановления после ABEND в модуле IMS подпрограмма восстановления сама столкнулась с ABEND. |
| Действие системы | Процесс восстановления после ABEND продолжается, но дальнейшая диагностика не формируется. Модуль IMS отключён, а система IMS продолжает работу. |
| Реакция пользователя | Обратитесь к специалисту по продуктам Dynatrace через чат в вашем окружении. |
