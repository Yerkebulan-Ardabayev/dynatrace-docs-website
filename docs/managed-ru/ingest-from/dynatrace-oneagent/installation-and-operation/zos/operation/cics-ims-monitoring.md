# Настройка мониторинга CICS и IMS

* Обновлено 5 декабря 2025 г.

Определите дополнительные настройки мониторинга для транзакций CICS и IMS. Перейдите в **Settings** > **Mainframe** и найдите следующие пункты меню:

## Мониторинг транзакций {#transaction-monitoring}

Определите дополнительные настройки мониторинга для транзакций CICS и IMS.

| Параметр | Описание |
| --- | --- |
| Monitor all incoming web requests | Dynatrace автоматически трассирует входящие веб-запросы, когда они вызываются мониторируемыми сервисами. Включите эту настройку, чтобы трассировать все входящие веб-запросы независимо от того, вызываются они мониторируемыми или немониторируемыми сервисами.  Рекомендуем включать её только на короткий период времени. |
| Group CICS regions that belong to the same CICSPlex | Включите эту настройку, чтобы группировать регионы CICS, принадлежащие одному CICSPlex, в одну группу процессов. Если отключена, группа процессов создаётся для каждого региона CICS.  Можно указать региону CICS дожидаться готовности CICSPlex перед продолжением. См. [Группировка по имени CICSPlex](/managed/ingest-from/dynatrace-oneagent/installation-and-operation/zos/installation/install-cics#plexname-grouping "Установка модуля Dynatrace CICS.").  Эта настройка включена по умолчанию. |
| Create CICS services based on transaction IDs | Включите эту настройку, чтобы создавать CICS-сервис для каждого мониторируемого ID транзакции внутри группы процессов. Если отключена, CICS-сервис создаётся для каждого мониторируемого региона CICS внутри группы процессов.  Эта настройка включена по умолчанию. Рекомендуем включать её только когда регионы CICS сгруппированы по их CICSPlex. |
| Group IMS regions that belong to the same subsystem | Включите эту настройку, чтобы группировать регионы IMS, принадлежащие одной подсистеме, в одну группу процессов. Если отключена, группа процессов создаётся для каждого региона IMS.  Эта настройка включена по умолчанию. |
| Create IMS services based on transaction IDs | Включите эту настройку, чтобы создавать IMS-сервис для каждого мониторируемого ID транзакции внутри группы процессов. Если отключена, IMS-сервис создаётся для каждого мониторируемого региона IMS внутри группы процессов.  Эта настройка включена по умолчанию. Рекомендуем включать её только когда регионы IMS сгруппированы по их подсистеме. |
| PurePathÂ® distributed trace node limit | Максимальное количество узлов в распределённой трассировке, которое будет захвачено для одного вызова программы CICS или IMS. Рекомендуем лимит в 500, это значение по умолчанию. Увеличение лимита по умолчанию может привести к большим накладным расходам на обработку дополнительных данных.  Значение `0` означает неограниченное число узлов. |

## Фильтры старта транзакций {#transaction-start-filters}

Dynatrace трассирует транзакции CICS и IMS, когда они вызываются мониторируемыми сервисами. Dynatrace также трассирует транзакции, запускаемые по поддерживаемому протоколу через CICS Transaction Gateway, z/OS Connect Enterprise Edition или IBM MQ (если они не ограничены фильтрами IBM MQ).

Транзакции, которые стартуют на мейнфрейме или на терминале (например, на терминале «зелёный экран» IBM 3270) или вызываются немониторируемыми сервисами, нужно явно перечислить, чтобы они трассировались.

### Фильтр старта терминальных транзакций CICS

Чтобы трассировать транзакции CICS, которые стартуют на терминале (например, на терминале «зелёный экран» IBM 3270), добавьте их ID транзакций в фильтр старта терминальных транзакций CICS. Можно использовать звёздочку (`*`) как подстановочный знак. Например, используйте `TN*`, чтобы трассировать все ID транзакций, начинающиеся с `TN`, или `*`, чтобы трассировать все транзакции, стартующие на терминале.

Активируйте требуемую [возможность OneAgent](/managed/ingest-from/dynatrace-oneagent/oneagent-features "Управление возможностями OneAgent глобально и для каждой группы процессов.") **z/OS CICS terminal transaction sensor**.

### Фильтр старта транзакций CICS

Чтобы трассировать транзакции CICS, которые стартуют на мейнфрейме или вызываются немониторируемыми сервисами, добавьте их ID транзакций в фильтр старта транзакций CICS. В качестве суффикса можно использовать звёздочку (`*`) как подстановочный знак. Например, используйте `TN*`, чтобы трассировать все ID транзакций, начинающиеся с `TN`.

Этот сенсор игнорирует транзакции, которые захватываются другим сенсором в момент старта транзакции (CICS Transaction Gateway и z/OS Connect Enterprise Edition, а также запросы старта транзакций и DPL-запросы от других транзакций CICS). Он мешает транзакциям, которые распознаются позже в цепочке: если транзакция из списка вызывается через IBM MQ, SOAP, JSON, HTTP или CICS/IMS SDK, для неё стартует новая трассировка, не связанная с предыдущей.

Если в этом списке есть ID транзакций, относящихся к терминалу (например, к терминалу «зелёный экран» IBM 3270), рекомендуем перенести их в список **CICS terminal transaction start filter**.

В фильтре старта транзакций CICS:

* В суффиксе ID транзакции можно использовать звёздочку (`*`), но она не может быть единственным символом.
* Не рекомендуем задавать ID транзакции как `C*`, потому что это захватит внутренние транзакции CICS.

### Фильтр старта терминальных транзакций IMS

Чтобы трассировать транзакции IMS, которые стартуют на терминале (например, на терминале «зелёный экран» IBM 3270), добавьте их ID транзакций в фильтр старта терминальных транзакций IMS. Можно использовать звёздочку (`*`) как подстановочный знак. Например, используйте `TN*`, чтобы трассировать все ID транзакций, начинающиеся с `TN`, или `*`, чтобы трассировать все транзакции, стартующие на терминале.

Чтобы трассировать такие транзакции, нужно включить [возможность OneAgent](/managed/ingest-from/dynatrace-oneagent/oneagent-features "Управление возможностями OneAgent глобально и для каждой группы процессов.") **z/OS IMS terminal transaction sensor**.

### Фильтр старта транзакций IMS

Чтобы трассировать транзакции IMS, которые стартуют на мейнфрейме или вызываются немониторируемыми сервисами, добавьте их ID транзакций в фильтр старта транзакций IMS. В качестве суффикса можно использовать звёздочку (`*`) как подстановочный знак. Например, используйте `TN*`, чтобы трассировать все ID транзакций, начинающиеся с `TN`.

В отличие от сенсора старта транзакций CICS, сенсор старта транзакций IMS не мешает трассировке, захваченной другим сенсором (MQ, SOAP, ITRA, IMS Connect и zOS Connect).

Если в этом списке есть ID транзакций, относящихся к терминалу (например, к терминалу «зелёный экран» IBM 3270), перенесите их в список **IMS terminal transaction start filter**.

## Фильтры IBM MQ {#ibm-mq-filters}

Dynatrace трассирует транзакции CICS и IMS, инициируемые из очередей IBM MQ. Чтобы ограничить трассировку определёнными очередями, укажите их имена в include-списках. Чтобы исключить очереди из трассировки, укажите их имена в exclude-списках. Для IMS эти списки применяются к регионам обработки сообщений.

| Параметр | Описание |
| --- | --- |
| CICS: Included MQ queues | Список имён очередей MQ, связанных с CICS, которые могут запускать трассировку. |
| CICS: Excluded MQ queues | Список имён очередей MQ, связанных с CICS, которые не должны запускать трассировку. |
| IMS: Included MQ queues | Список имён очередей MQ, связанных с регионом обработки сообщений IMS, которые могут запускать трассировку. |
| IMS: Excluded MQ queues | Список имён очередей MQ, связанных с регионом обработки сообщений IMS, которые не должны запускать трассировку. |
| IMS bridge: Included transaction IDs | Чтобы трассировать только конкретные транзакции, отправленные через IMS bridge, укажите их ID транзакций в include-списке. |
| IMS bridge: Excluded transaction IDs | Чтобы исключить из трассировки конкретные транзакции, отправленные через IMS bridge, укажите их ID транзакций в exclude-списке. |
