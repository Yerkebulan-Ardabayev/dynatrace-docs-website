# Обновление OneAgent

# Обновление OneAgent

* 3-min read
* Updated on Nov 06, 2025

OneAgent имеет встроенный настраиваемый механизм автоматического обновления.

Автоматические обновления OneAgent просты

При включённом автообновлении вам не нужно беспокоиться о ручном обновлении OneAgent в вашем окружении. При каждом выпуске новой версии OneAgent автоматически загружает все необходимые бинарные файлы и библиотеки и устанавливает их. Ручная настройка не требуется.

После автоматических обновлений OneAgent необходимо перезапустить все серверные процессы. Это связано с тем, что некоторые компоненты OneAgent продолжают работать в процессах, мониторируемых Dynatrace (например, Java, .NET, Apache и IIS).

* До перезапуска эти процессы будут продолжать мониторироваться с предыдущей версией OneAgent.
* После перезапуска эти процессы будут мониторироваться с последней версией OneAgent.

Автоматические обновления OneAgent гибки

По умолчанию глобальные автоматические обновления OneAgent включены, но это можно настроить на глобальном уровне, уровне группы хостов и уровне хоста. На каждом уровне можно выбрать одну из следующих стратегий обновления OneAgent:

* Обновлять автоматически независимо от окон обновления.
* Обновлять автоматически в течение выбранного [окна обновления](#maintenance-windows).
* Не обновлять автоматически. При этом варианте вы не получаете уведомлений об устаревших экземплярах OneAgent.

Дополнительно:

* На уровне группы хостов можно выбрать наследование глобальных настроек обновления OneAgent.
* На уровне хоста можно выбрать наследование настроек обновления OneAgent группы хостов.

Настройки хоста переопределяют настройки группы хостов, а настройки группы хостов переопределяют глобальные настройки.

Автоматические обновления OneAgent безопасны

* Коммуникация OneAgent зашифрована.
* Установщики OneAgent подписаны, и подпись проверяется для каждого загруженного пакета установщика. В случае сбоя проверки попытка автообновления прерывается.

Обновления OneAgent также управляются через Dynatrace API

Помимо описанных здесь процедур через веб-интерфейс Dynatrace, для управления обновлениями OneAgent можно использовать [API конфигурации OneAgent](/managed/dynatrace-api/configuration-api/oneagent-configuration "Управляйте конфигурацией OneAgent через Dynatrace API.").

* [Уровень хоста](/managed/dynatrace-api/configuration-api/oneagent-configuration/oneagent-on-host "Управляйте конфигурацией экземпляров OneAgent на хостах через Dynatrace API.")
* [Уровень группы хостов](/managed/dynatrace-api/configuration-api/oneagent-configuration/oneagent-in-host-group "Управляйте конфигурацией экземпляров OneAgent в группах хостов через Dynatrace API.")
* [Уровень окружения](/managed/dynatrace-api/configuration-api/oneagent-configuration/oneagent-environment-wide "Управляйте конфигурацией OneAgent для всего окружения через Dynatrace API.")

OneAgent не будет автоматически обновлён сразу после первой установки. По умолчанию Dynatrace ждёт 45 минут перед обновлением хостов (как EC2, так и не EC2).

Эта задержка не применяется при ручных обновлениях через **Update now**, что означает немедленное применение обновлений, инициированных таким способом.

## Мониторинг обновлений OneAgent

Для мониторинга обновлений OneAgent перейдите в **Settings** > **Updates** > **OneAgent updates**.

* Проверка или изменение настроек автообновления на уровне окружения
* Мониторинг статуса обновления по операционным системам (Linux, AIX и Windows)
* Проверка или изменение статуса автообновления библиотеки JavaScript для Real User Monitoring

Мониторинг глобального обновления OneAgent

Для мониторинга обновлений OneAgent для всего окружения перейдите в **Settings** > **Updates** > **OneAgent updates**.

#### Обновления OneAgent

В этом разделе можно проверить или изменить настройку уровня окружения:

* **Automatic updates at earliest convenience**  
  Обновлять все OneAgent автоматически независимо от окон обновления.
* **Automatic updates during update windows**  
  Обновлять все OneAgent автоматически в течение выбранного окна обновления.

  + При выборе этой настройки отображается список доступных окон обновления. Выберите одно.
  + Для настройки нового окна обновления OneAgent перейдите в **Settings** > **Updates** > **Update windows for OneAgent updates**.
* **No automatic updates**  
  Не обновлять OneAgent автоматически.

#### Статус автоматических обновлений

В этом разделе можно проверить статус автоматических обновлений по операционным системам. Для каждой ОС на странице отображается количество экземпляров OneAgent с разбивкой по статусу обновления.

#### Библиотека JavaScript для Real User Monitoring

Включите **Automatically update real user monitoring JavaScript library** для автоматического обновления библиотеки JavaScript для Real User Monitoring. Текущая версия библиотеки указана под переключателем.

Статусы обновления OneAgent

* Incompatible — автообновление невозможно. Для этого хоста/версии может потребоваться ручное обновление.
* Update available — доступно обновление для данной версии OneAgent.
* Suppressed — загрузка не выполнена, поскольку автообновление не поддерживается для данного развёртывания.
* Unknown — например, хост не найден или для него нет автообновления.
* Up to date — версия установщика актуальна.
* Update in progress — хост запросил и загрузил установщик, но ещё не переподключился с ожидаемой новой версией.
* Update pending — для данного хоста настроено обновление. Оно имеет приоритет над другими действиями для вновь обнаруженных хостов, но может не начаться немедленно по причинам, таким как внутренние обновления кластера, ограничение скорости и проверки стабильности.
* Update problem — при выполнении обновления возникла проблема. Пожалуйста, ознакомьтесь с подробностями.
* Scheduled — обновление запланировано на настроенное окно обновления.

## Настройка обновлений OneAgent

Обновление OneAgent можно настроить глобально, для каждой группы хостов и для каждого хоста.

Настройка глобального обновления OneAgent

При включённых автоматических обновлениях глобально каждый раз при выходе новой версии OneAgent все установленные экземпляры OneAgent, кроме тех, у которых автообновление отключено на уровне группы хостов или хоста, автоматически загружают обновление и обновляют бинарные файлы и конфигурационные файлы.

Настройки автоматического обновления на уровне группы хостов и отдельного хоста переопределяют глобальные настройки.

1. Перейдите в **Settings** > **Updates** > **OneAgent updates**.
2. Выберите один из вариантов обновления:

   * **Automatic updates at earliest convenience**  
     Обновлять все OneAgent автоматически независимо от окон обновления.
   * **Automatic updates during update windows**  
     Обновлять все OneAgent автоматически в течение выбранного окна обновления.

     + При выборе этой настройки отображается список доступных окон обновления. Выберите одно.
     + Для настройки нового окна обновления OneAgent перейдите в **Settings** > **Updates** > **Update windows for OneAgent updates**.
   * **No automatic updates**  
     Не обновлять OneAgent автоматически.

Ручной запуск **Update now to target version** обновит все хосты с выбранной ОС и архитектурой независимо от статуса автоматического обновления.

Настройка обновления OneAgent для группы хостов

Настройки обновления OneAgent на уровне группы хостов переопределяют глобальные настройки и перекрываются настройками на уровне хоста.

1. Откройте страницу **Host group settings**.  
   Варианты доступа:

   * Перейдите в **Settings** > **Monitoring** > **Monitoring overview**, найдите любой хост в нужной группе и выберите название группы хостов (не имя хоста).
   * Перейдите в **Hosts**, откройте страницу хоста и разверните раздел **Properties and tags**. Если хост принадлежит группе, там будет ссылка на неё в разделе **Host group**.
   * Перейдите в **Deployment Status** > **OneAgents**. Если хост принадлежит группе хостов, ссылка на страницу настроек группы отображается под именем хоста.
2. На странице **Host group settings** выберите **OneAgent updates** в левой части страницы.
3. Выберите один из вариантов обновления:

   * **Inherit global update settings**  
     Следовать глобальным настройкам обновления OneAgent в этой группе хостов.

     + Текущая глобальная настройка отображается в скобках на этой строке.
     + Для перехода на страницу глобальной конфигурации выберите ссылку `global`.
   * **Automatic updates at earliest convenience**  
     Обновлять все OneAgent в этой группе хостов автоматически независимо от окон обновления. Глобальные настройки обновления игнорируются.
   * **Automatic updates during update windows**  
     Обновлять все OneAgent в этой группе хостов автоматически в течение выбранного окна обновления. Глобальные настройки обновления игнорируются.

     + При выборе этой настройки отображается список доступных окон обновления. Выберите одно.
     + Для настройки нового окна обновления OneAgent перейдите в **Settings** > **Updates** > **Update windows for OneAgent updates**.
   * **No automatic updates**  
     Не обновлять OneAgent в этой группе хостов автоматически. Глобальные настройки обновления игнорируются.

Ручной запуск **Update now to target version** обновит все хосты с выбранной ОС и архитектурой независимо от статуса автоматического обновления.

Настройка обновления OneAgent на уровне хоста

Настройки обновления OneAgent на уровне хоста переопределяют настройки обновления на глобальном уровне и уровне группы хостов.

1. Откройте страницу **Host** для хоста, который хотите настроить.  
   Варианты доступа:

   * Перейдите в **Hosts** и выберите хост.
   * Перейдите в **Settings** > **Monitoring** > **Monitoring overview**, выберите вкладку **Hosts** и выберите хост.
2. На странице **Host** откройте меню браузера (**...**) и выберите **Settings**.
3. Выберите **OneAgent updates** в левой части страницы.
4. Выберите один из вариантов обновления:

   * **Inherit...**  
     Следовать настройкам обновления группы хостов или глобальным настройкам для этого OneAgent.

     + Если выбранный хост принадлежит группе хостов, текущая настройка группы отображается в скобках, а имя группы является ссылкой на страницу конфигурации группы.
     + Если выбранный хост не принадлежит группе хостов, глобальная настройка отображается в скобках, а «global» является ссылкой на глобальную страницу конфигурации (**Settings** > **Updates** > **OneAgent updates**).
   * **Automatic updates at earliest convenience**  
     Обновлять этот OneAgent автоматически независимо от окон обновления. Настройки обновления группы хостов игнорируются.
   * **Automatic updates during update windows**  
     Обновлять этот OneAgent автоматически в течение выбранного окна обновления. Глобальные настройки обновления игнорируются.

     + При выборе этой настройки отображается список доступных окон обновления. Выберите одно.
     + Для настройки нового окна обновления OneAgent перейдите в **Settings** > **Updates** > **Update windows for OneAgent updates**.
   * **No automatic updates**  
     Не обновлять этот OneAgent автоматически. Глобальные настройки обновления игнорируются.

Ручное обновление OneAgent на отдельных хостах

Для ручного обновления OneAgent на отдельном хосте:

1. Перейдите в **Settings > Monitoring > Monitoring overview**.
2. Выберите вкладку **Hosts**.
3. Для обновления OneAgent или загрузки последней версии выберите **Update** рядом с именем интересующего хоста.

   Кнопка **Update** появляется только если установленная версия OneAgent на конкретном хосте устарела и это полнофункциональный OneAgent (Full-Stack). Кнопка не появляется для PaaS и standalone OneAgent.
4. Выберите **Update now**.  
   Если кнопка **Update now** недоступна, у вас нет разрешений на загрузку установщика.

Альтернативно, вы можете загрузить последнюю версию установщика OneAgent, вручную скопировать её на целевой хост и выполнить установку непосредственно на нём.

Выбор версии OneAgent для установки на новые хосты

Для управления версией OneAgent, автоматически устанавливаемой на все новые хосты:

1. Перейдите в **Settings** > **Updates** > **OneAgent updates**.
2. В разделе **Update mode** выберите **No automatic updates** для отключения автоматических обновлений OneAgent.

   Подробнее об отключении автоматических обновлений OneAgent на PaaS/Kubernetes смотрите в разделе [DynaKube parameters for Dynatrace Operator on Kubernetes/OpenShift](/managed/ingest-from/setup-on-k8s/reference/dynakube-parameters "Список доступных параметров для настройки Dynatrace Operator на Kubernetes.").
3. В разделе **Target version** выберите версию OneAgent для установки на все новые хосты.

Выбранная версия OneAgent также используется для PaaS-интеграций.

Выбор версии OneAgent для обновления

Для выбора версии OneAgent, до которой выполняется обновление:

1. Перейдите в **Settings** > **Updates** > **OneAgent updates**.
2. В списке **Target version** можно указать конкретную версию по номеру или выбрать относительную целевую версию:

   * **Latest stable version**  
     Наиболее свежая стабильная версия OneAgent, доступная в вашем окружении. Фактический номер версии отображается в скобках.
   * **Previous stable version**  
     Стабильная версия, предшествующая **Latest stable version**. Соответствующий номер версии отображается в скобках. Номер версии OneAgent увеличивается на *два* при каждом выпуске, поэтому это число будет на *два* меньше последней версии.
   * **Older stable version**  
     Стабильная версия, предшествующая **Previous stable version**. Соответствующий номер версии отображается в скобках. Номер версии OneAgent увеличивается на *два* при каждом выпуске, поэтому это число будет на *четыре* меньше последней версии.

* **Specific OneAgent version**

Целевая версия используется для:

* Автоматических обновлений
* Автоматических обновлений в течение окон обслуживания
* Ручных обновлений при выборе версии для обновления

Целевую версию и режим обновления можно задать на:

* **Уровне окружения** — влияет на все OneAgent окружения. Также используется для Deployment API.
* **Уровне группы хостов** — влияет на все OneAgent группы хостов. Переопределяет уровень окружения. Не влияет на Deployment API.
* **Уровне хоста** — влияет на OneAgent на этом хосте. Переопределяет настройки группы хостов и уровня окружения. Не влияет на Deployment API.

Если вы выберете версию, более старую чем текущая развёрнутая, откатить OneAgent не удастся. Для перехода на более старую версию необходимо установить более новую версию поверх существующей.

## Настройка окон обновления

Окна обновления OneAgent используются для планирования автоматических обновлений OneAgent при выборе варианта **Automatic updates during update windows** на глобальном уровне, уровне группы хостов или уровне хоста.

Окна обновления в настоящее время не применяются в окружениях Kubernetes.

Просмотр всех окон обновления OneAgent

1. Перейдите в **Settings** > **Updates** > **Update windows for OneAgent updates**.
2. Просмотрите список всех существующих окон обновления OneAgent.

Включение или отключение окна обновления OneAgent

1. Перейдите в **Settings** > **Updates** > **Update windows for OneAgent updates**.
2. Найдите окно обновления, которое хотите включить или отключить.
3. Включите или отключите переключатель.

Удаление окна обновления OneAgent

1. Перейдите в **Settings** > **Updates** > **Update windows for OneAgent updates**.
2. Найдите окно обновления, которое хотите удалить.
3. Выберите **X** в столбце **Delete**.

Создание или изменение окна обновления OneAgent

1. Перейдите в **Settings** > **Updates** > **Update windows for OneAgent updates**.
2. Откройте окно обновления для редактирования:

   * Для создания нового окна обновления выберите **Create update window**.
   * Для редактирования существующего окна разверните **Details** для нужного окна.
3. Задайте **Name** — имя, под которым это окно будет отображаться в веб-интерфейсе и API Dynatrace.
4. Опишите окно обновления:

   * Для разового, неповторяющегося окна обновления:

     + Установите **Recurrence** в `Once`
     + Установите **Update time** на дату и время разового обновления OneAgent
   * Для планирования окна обновления каждые x дней:

     + Установите **Recurrence** в `Daily intervals`
     + Установите **Every x days** на количество дней между обновлениями (`1` = каждый день, `2` = каждые два дня, `3` = каждые три дня и т.д.)
     + Установите **Update time** на время в указанные дни для обновления OneAgent
   * Для планирования окна обновления каждые x недель в определённые дни недели:

     + Установите **Recurrence** в `Weekly intervals`
     + Включите каждый день недели, который нужно добавить в окно обновления
     + Установите **Every x weeks** на количество недель между обновлениями (`1` = каждую неделю, `2` = каждые две недели и т.д.)
     + Установите **Update time** на время в указанные дни для обновления OneAgent
   * Для планирования окна обновления каждые x месяцев в определённый день месяца:

     + Установите **Recurrence** в `Monthly intervals`
     + Установите **Day of the month** на день месяца для обновления OneAgent
     + Установите **Every x months** на количество месяцев между обновлениями (`1` = каждый месяц, `2` = каждые два месяца и т.д.)
5. Выберите **Save changes**.

После настройки окна обновления его можно применить для настройки автоматических обновлений OneAgent на глобальном уровне, уровне группы хостов или уровне хоста.

Применение окна обновления OneAgent

Окно обновления OneAgent можно применить к автоматическим обновлениям на уровне окружения, группы хостов или хоста.

**Окружение**

1. Перейдите в **Settings** > **Updates** > **OneAgent updates**.
2. Выберите **Automatic updates during update windows**.
3. В появившемся списке выберите окно обновления для применения на этом уровне.

**Группа хостов**

1. Перейдите в **Settings** > **Monitoring** > **Monitoring overview**.
2. Найдите любой хост в нужной группе хостов.
3. Выберите название группы хостов (не имя хоста).
4. На странице **Host group settings** выберите **OneAgent updates** в левой части страницы.
5. Выберите **Automatic updates during update windows**.
6. В появившемся списке выберите окно обновления для применения на этом уровне.

**Хост**

1. Откройте страницу **Host** для хоста, который хотите настроить.  
   Варианты доступа:

   * Перейдите в **Hosts** и выберите хост.
   * Перейдите в **Settings** > **Monitoring** > **Monitoring overview**, выберите вкладку **Hosts** и выберите хост.
2. На странице **Host** откройте меню браузера (**...**) и выберите **Settings**.
3. Выберите **OneAgent updates** в левой части страницы.
4. Выберите **Automatic updates during update windows**.
5. В появившемся списке выберите окно обновления для применения на этом уровне.

## Отключение автоматических обновлений

Отключать автоматические обновления OneAgent не рекомендуется. В целях диагностики временно отключить автоматические обновления можно в Dynatrace в разделе **Settings** > **Updates** > **OneAgent updates**. Однако если у вас очень строгие правила развёртывания программного обеспечения, автообновления можно отключить постоянно с помощью [интерфейса командной строки OneAgent](/managed/ingest-from/dynatrace-oneagent/oneagent-configuration-via-command-line-interface#automatic-updates "Узнайте, как выполнять задачи настройки OneAgent без переустановки.").
