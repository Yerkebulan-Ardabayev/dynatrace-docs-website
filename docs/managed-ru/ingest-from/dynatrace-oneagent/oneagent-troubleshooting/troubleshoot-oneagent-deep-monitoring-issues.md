# Устранение проблем глубокого мониторинга OneAgent

# Устранение проблем глубокого мониторинга OneAgent

* 3-min read
* Published Jul 16, 2018

Dynatrace OneAgent поддерживается на многих [платформах](/managed/ingest-from/technology-support/oneagent-platform-and-capability-support-matrix "Узнайте, какие возможности поддерживает OneAgent на различных операционных системах и платформах.") и предлагает глубокий мониторинг для многих [технологий и фреймворков](/managed/ingest-from/technology-support "Найдите технические подробности о поддержке Dynatrace конкретных платформ и фреймворков разработки."). Команда Dynatrace тестирует все возможные сценарии глубокого мониторинга, но для упрощения поддержки в ситуациях, когда глубокий мониторинг может не работать как ожидается, вы можете воспользоваться страницей [Функции OneAgent](/managed/ingest-from/dynatrace-oneagent/oneagent-features "Управление функциями OneAgent глобально и для каждой группы процессов.").

## Сценарии использования

Настройки на странице **OneAgent features** позволяют отключать определённые функции OneAgent на детальном уровне для упрощения устранения проблем. Это может быть полезно для определения первопричины проблемы вплоть до конкретной функции.

Эти настройки также позволяют временно отключить определённые функции, которые могут вызывать проблемы, сохраняя при этом все остальные функции. В это время команда поддержки Dynatrace может работать над постоянным исправлением.

Перед деактивацией настроек OneAgent

Не рекомендуется деактивировать настройки функций OneAgent без предварительной консультации со службой поддержки Dynatrace. При отключении определённой функции OneAgent в вашем окружении OneAgent прекращает собирать данные наблюдаемости для этой функции, поэтому вы можете пропустить важную информацию.

## Настройки

Каждую функцию на странице **OneAgent features** можно отключить для всего окружения, определённой группы процессов или отдельного процесса.

### Настройки окружения

Чтобы включить или отключить функцию для всего окружения:

1. Перейдите в **Settings**.
2. Выберите **Preferences** > **OneAgent features**.
3. Найдите функцию. Для поиска можно ввести строку фильтра над таблицей.
4. Разверните запись, чтобы:

   * Необязательно: просмотреть описание функции
   * Необязательно: просмотреть метаданные
   * Необязательно: перейти по ссылке на соответствующие примечания к выпуску
5. Измените настройку **Enabled** по мере необходимости.

Все функции инструментирования можно включить или отключить главным переключателем, а некоторые из них имеют два дополнительных параметра:

* **Activate this feature also in OneAgents only fulfilling the minimum Opt-In version**  
  Этот параметр активирует функцию с минимальной версией для opt-in при первом выпуске. Обычно функцию можно активировать, если она удовлетворяет минимальной версии OneAgent, но некоторые функции изначально выпускаются как opt-in с минимальной версией. Это полезно, когда версия OneAgent для общей доступности (GA) выше минимальной версии opt-in. Если вы уже включили функцию во время opt-in, вы можете продолжать использовать OneAgent без обновления до новой GA-версии.
* **Instrumentation enabled**  
  Этот параметр действует немедленно. Однако при перезапуске или запуске процесса функция не будет добавлена в ваше приложение. Это означает, что функция не будет влиять на приложение. Недостаток: при повторном включении функции позднее потребуется перезапустить затронутые процессы.

### Настройки для конкретного процесса

Если конкретная функция является проблемной только в контексте определённой группы процессов, вы можете отключить её только для этой группы процессов, не затрагивая остальную часть окружения.

Чтобы включить или отключить функцию для конкретного процесса или группы процессов:

1. Перейдите в **Smartscape Topology**.
2. Выберите **Processes**.
3. На карте топологии наведите курсор на процесс и выберите значок ссылки для перехода на страницу сведений о процессе. Например:

   ![Ссылка из топологии Smartscape на страницу сведений о процессе](https://dt-cdn.net/images/link-to-process-page-377-da0282e435.png)

   Ссылка из топологии Smartscape на страницу сведений о процессе
4. На странице сведений о процессе выберите **More** (**...**) > **Settings**, затем выберите вкладку **OneAgent features**.
5. Выберите **Add override**, чтобы добавить специфичную для процесса настройку, которая переопределяет настройку окружения.
6. Введите строку поиска в поле **Feature**, чтобы найти и выбрать функцию.
7. Установите переключатели по мере необходимости для переопределения, затем выберите **Save changes**.
