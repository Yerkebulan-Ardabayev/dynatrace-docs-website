## Доверенные корневые сертификаты для OneAgent

OneAgent поставляется с доверенными SSL-сертификатами Dynatrace, которые используются для проверки успешного подключения OneAgent к серверу Dynatrace или ActiveGate.

Если в вашем окружении используется прокси с пользовательским сертификатом (что требует обновления SSL-сертификата удалённого сервера) или у вас есть Environment ActiveGate с собственным пользовательским сертификатом, необходимо добавить дополнительные сертификаты в OneAgent.

### ActiveGate с пользовательским сертификатом

1. Определите SSL-сертификат, который использует ваш ActiveGate.
2. Получите CA-сертификат (вместе с промежуточными сертификатами при их наличии).
3. Сохраните CA-сертификаты как `custom.pem`.
4. Разместите файл `custom.pem` в соответствующей директории в зависимости от операционной системы:

   **Linux**: `/var/lib/dynatrace/oneagent/agent/customkeys`

   **Windows**: `%PROGRAMDATA%\dynatrace\oneagent\agent\customkeys`

Убедитесь, что файл `custom.pem` содержит сертификат ActiveGate вместе с промежуточными сертификатами при их наличии.

### Прокси с пользовательским сертификатом

1. Определите SSL-сертификат, который использует ваш прокси.
2. Получите CA-сертификат (вместе с промежуточными сертификатами при их наличии).
3. Сохраните CA-сертификаты как `custom-proxy.pem`.
4. Разместите файл `custom-proxy.pem` в соответствующей директории в зависимости от операционной системы:

   **Linux**: `/var/lib/dynatrace/oneagent/agent/customkeys`

   **Windows**: `%PROGRAMDATA%\dynatrace\oneagent\agent\customkeys`

Убедитесь, что файл `custom-proxy.pem` содержит сертификат прокси вместе с промежуточными сертификатами при их наличии.

### ActiveGate и прокси с пользовательскими сертификатами

Если ваша конфигурация включает и HTTPS-прокси с пользовательским сертификатом, и Environment ActiveGate с собственным пользовательским сертификатом, вам необходимо предоставить файл `custom.pem` и файл `custom-proxy.pem`.
