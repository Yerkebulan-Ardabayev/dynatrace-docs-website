# Механизм устаревания файлов OneAgent

* 6-min read
* Updated on Feb 25, 2026

OneAgent в инсталляторном развёртывании использует встроенный механизм устаревания, который гарантирует, что файлы OneAgent занимают разумный объём дискового пространства.

Механизм устаревания не используется в режиме application-only.

Оптимальное использование диска OneAgent определяется на основе следующих ограничений:

* Минимально необходимое пространство: 200 МБ \* 3 = 600 МБ (логи, оповещения, отчёты о сбоях).
* Стандартно необходимое пространство: 1 ГБ \* 3 = 3 ГБ (логи, оповещения, отчёты о сбоях).

Два типа файлов, вносящих наибольший вклад в использование дискового пространства OneAgent в процессе работы, — это файлы логов OneAgent и данные среды выполнения, такие как дампы при сбоях и дампы памяти.

## Файлы логов

Максимальное дисковое пространство, занимаемое файлами логов OneAgent, хорошо управляется механизмом устаревания. Логи растут медленно и постепенно. Непрерывный процесс очистки, запускаемый каждую минуту, удерживает размер логов в разумных пределах. Механизм устаревания логов разработан с учётом внутреннего знания о том, как регистрируются события, связанные с OneAgent, поэтому дополнительная настройка не требуется.

## Данные среды выполнения

В отличие от файлов логов, крупные файлы данных среды выполнения, такие как дампы при сбоях и дампы памяти, генерируются внезапно и могут вызывать резкие скачки использования диска. Для снижения этого риска используйте параметр установки `DATA_STORAGE`, чтобы указать пользовательскую директорию для крупных данных среды выполнения. Разместите пользовательскую директорию на ресурсе, где жёсткие ограничения дискового пространства не столь критичны, как на диске с установленным OneAgent.

Дополнительные сведения о настройке установки OneAgent смотрите в справке для конкретной ОС: [Linux](/managed/ingest-from/dynatrace-oneagent/installation-and-operation/linux/installation/customize-oneagent-installation-on-linux#data-storage "Узнайте, как использовать установщик Linux с параметрами командной строки."), [Windows](/managed/ingest-from/dynatrace-oneagent/installation-and-operation/windows/installation/customize-oneagent-installation-on-windows#data-storage "Узнайте, как использовать установщик OneAgent для Windows.") или [AIX](/managed/ingest-from/dynatrace-oneagent/installation-and-operation/aix/installation/customize-oneagent-installation-on-aix#data-storage "Узнайте, как использовать установщик AIX с параметрами командной строки.").

## Экстренная очистка

В экстренной ситуации, когда свободное дисковое пространство достигает порога в `3%`, OneAgent очищает все файлы из директорий, управляемых механизмом устаревания, за исключением последних логов установщика и лога модуля ОС OneAgent.

## Правила механизма устаревания OneAgent

Механизм устаревания OneAgent гарантирует, что используемое OneAgent дисковое пространство постоянно находится в пределах заранее определённых ограничений. Для этого к директориям OneAgent применяется ряд правил. При необходимости вы можете обратиться к специалисту Dynatrace через онлайн-чат для изменения некоторых правил устаревания.

### Директория логов

* Linux/AIX: `/var/log/dynatrace/oneagent`
* Windows: `%PROGRAMDATA%\dynatrace\oneagent\log`

Механизм устаревания файлов проверяет файлы и поддиректории в основной директории логов и удаляет старейшие файлы, когда:

* Общий размер директории превышает 1 ГБ
* Любой файл в этой директории старше 14 дней

Файлы удаляются из любой поддиректории, содержащей более 1000 файлов (сохраняются новейшие файлы, удаляются старейшие).

Для поддиректорий действуют дополнительные правила:

* `{log-dir}/process`
  Файлы удаляются при выполнении любого из условий:

  + Файлы в этой директории занимают более 300 МБ в общей сложности
  + Любой файл в этой директории старше 14 дней
* `{log-dir}/installer`
  Файлы удаляются при выполнении любого из условий:

  + Файлы в этой директории занимают более 30 МБ в общей сложности
  + Любой файл в этой директории старше 180 дней. На Windows файл `driver.log` сохраняется.
* `{log-dir}/dumpproc`
  Файлы удаляются при выполнении любого из условий:

  + Файлы в этой директории занимают более 100 МБ в общей сложности
  + Любой файл в этой директории старше 14 дней

### Директория хранения данных

* Linux/AIX: `/var/lib/dynatrace/oneagent/datastorage`
* Windows: `%PROGRAMDATA%\dynatrace\oneagent\datastorage`
* `{data_storage_dir}/supportalerts`
  Файлы удаляются при выполнении любого из условий:

  + Директория содержит более 10 файлов
  + Файлы в директории занимают более 1 ГБ в общей сложности
  + Любой файл в директории старше 7 дней
* `{data_storage_dir}/memorydump`
  Файлы удаляются при выполнении любого из условий:

  + Любой файл в директории старше 2 часов
  + Файлы в директории занимают более 20 ГБ в общей сложности
* `{data_storage_dir}/crashreports`
  Файлы удаляются при выполнении любого из условий:

  + Они уже были переданы в Dynatrace
  + Директория содержит более 100 файлов
  + Файлы в директории занимают более 1 ГБ в общей сложности
  + Любой файл в директории старше 3 дней

### Директория среды выполнения

* Linux/AIX: `/var/lib/dynatrace/oneagent/agent/runtime`
* Windows: `%PROGRAMDATA%\dynatrace\oneagent\agent\runtime`

Механизм устаревания файлов OneAgent проверяет поддиректории, начинающиеся с `0x`. Проверки выполняются рекурсивно.

* Если директория `0x*` содержит поддиректорию dump и все файлы в ней старше 3 дней, поддиректория dump удаляется.
* Если все файлы и директории в директории `0x*` старше 7 дней, вся директория удаляется.
* Если размер всех файлов в директории среды выполнения превышает 1 ГБ, директории `0x*` удаляются, начиная со старейших, до тех пор, пока размер директории среды выполнения не станет ниже 1 ГБ.
* Эта директория также полностью очищается при обновлении OneAgent.

### Директория bin установки

* Linux/AIX: `/opt/dynatrace/oneagent/bin`
* Windows: `%PROGRAMFILES%\dynatrace\oneagent\bin`
* `{install-dir}/bin`
  OneAgent развёртывает ряд файловых артефактов в процессе обновления, особенно в случаях инъектируемых технологических модулей. Все инъектируемые файлы модулей хранятся в версионированных папках. Когда OneAgent активен, он выполняет очистку файлов согласно следующим критериям:

  + Используемая в данный момент версия всегда сохраняется.
  + OneAgent сканирует все мониторируемые процессы и определяет, какие библиотеки используются.
  + Список библиотек сравнивается с содержимым директории bin. Неиспользуемые бинарные файлы из директорий 32-bit и 64-bit удаляются независимо от того, включён или отключён мониторинг хоста. Для бинарных файлов в директории `any` очистка происходит только при включённом мониторинге хоста. Механизм устаревания OneAgent сохраняет текущую версию и 10 наиболее поздних предыдущих версий — итого 11 версий.

## Механизм устаревания для OneAgent в режиме мониторинга только приложений

Если у вас нет доступа к уровню инфраструктуры, Dynatrace также предоставляет возможность мониторинга только приложений для [Kubernetes](/managed/ingest-from/setup-on-k8s/deployment/app-obs-managed "Развёртывание Dynatrace Operator в режиме мониторинга приложений на Kubernetes"), [OpenShift](/managed/ingest-from/setup-on-k8s/deployment/app-obs-managed "Развёртывание Dynatrace Operator в режиме мониторинга приложений на Kubernetes"), [CloudFoundry](/managed/ingest-from/setup-on-container-platforms/cloud-foundry/deploy-oneagent-on-cloud-foundry-for-application-only-monitoring "Установка OneAgent на Cloud Foundry.") или [SAP Business Technology Platform](/managed/ingest-from/setup-on-container-platforms/cloud-foundry/deploy-oneagent-on-sap-cloud-platform-for-application-only-monitoring "Установка OneAgent на SAP Business Technology Platform.").

Механизм устаревания в режиме мониторинга только приложений управляет логами модулей OneAgent. Все они находятся в соответствующих поддиректориях директории логов OneAgent по умолчанию:

* Linux/AIX: `/var/log/dynatrace/oneagent`
* Windows: `%PROGRAMDATA%\dynatrace\oneagent\log`

### Ротация логов

Каждый модуль OneAgent ограничивает количество своих логов до 5. Каждый из этих 5 файлов ограничен 10 МБ.

### Устаревание логов

OneAgent удаляет файлы логов при запуске, когда:

* Файл лога старше 14 дней
* Суммарный размер всех файлов логов превышает 300 МБ
* Количество файлов логов превышает 1000

Для предотвращения излишних задержек при запуске приложения ограничение массового удаления OneAgent установлено на удаление максимум 50 старейших файлов.
