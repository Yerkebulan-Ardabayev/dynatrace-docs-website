# Пользовательская топология

* 4-min read
* Updated on Mar 16, 2023

Пошаговое руководство по созданию пользовательских типов сущностей и связей между ними.

## Шаги настройки

### 1. Определите тип сущности

Перейдите в **Settings > Topology model > Generic types** и создайте новый тип сущности.

Укажите:
* Имя типа (например, `truck`).
* Отображаемое имя (например, `Truck`).
* Идентификационные правила (extraction rules).

### 2. Настройте правила извлечения

Правила извлечения определяют, как Dynatrace идентифицирует экземпляры сущностей на основе данных метрик и логов.

Пример правила:
* Источник: поле `truck.id` в данных метрик.
* Шаблон: `truck-{id}`.

### 3. Определите связи между сущностями

Для настройки связей используйте роли:
* **Caller** (вызывающий) — сущность, инициирующая взаимодействие.
* **Callee** (вызываемый) — сущность, получающая взаимодействие.

Связи между сущностями одного типа (например, между грузовиками) также поддерживаются.

## Результат

После настройки пользовательская топология отображается в Smartscape, и метрики отображаются в контексте определённых вами бизнес-сущностей.
