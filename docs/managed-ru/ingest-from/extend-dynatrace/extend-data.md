# Обогащение данных, поступающих в систему, полями, специфичными для Dynatrace

# Обогащение данных, поступающих в систему, полями, специфичными для Dynatrace

* Чтение: 4 мин
* Опубликовано 26 апр. 2021 г.

В отличие от автоматического приёма данных через OneAgent, данные, отправляемые напрямую в ActiveGate (например, приём API), не обогащаются автоматически информацией, связанной с хостом. Это может привести к дополнительным расходам, поскольку не учитываются возможные включённые [квоты DDU](/managed/license/classic-licensing/davis-data-units "Понять, как рассчитывается потребление мониторинга Dynatrace на основе Davis data units (DDU)."), которые могли бы применяться.

Различные варианты развёртывания Dynatrace предоставляют несколько файлов свойств в Java-стиле и файлов JSON с наборами атрибутов, которые можно использовать для обогащения запросов к Dynatrace и обеспечения того, что Dynatrace сможет сопоставить данные с инфраструктурой.

## Каталог обогащения

Dynatrace использует следующие каталоги для предоставления файлов `.json` и `.properties` с данными обогащения:

* В Unix: `/var/lib/dynatrace/enrichment`
* В Windows: `%ProgramData%\dynatrace\enrichment`

## Dynatrace OneAgent

Стандартная установка OneAgent предоставляет следующие файлы со сведениями, специфичными для хоста, в [каталоге обогащения](#enrichment-directory):

* `dt_host_metadata.json`
* `dt_host_metadata.properties`

Оба файла содержат один и тот же набор атрибутов ресурсов на уровне хоста, которые OneAgent использует для обогащения артефактов мониторинга для данного хоста. Сюда также входят теги в формате ключ-значение и свойства, заданные через [oneagentctl](/managed/observe/infrastructure-observability/hosts/configuration/define-tags-and-metadata-for-hosts "Узнать, как назначать теги и задавать дополнительные свойства для отслеживаемого хоста.") или через [удалённое управление конфигурацией](/managed/ingest-from/bulk-configuration "Выполнить конфигурацию OneAgent и ActiveGate на хостах со страницы состояния развёртывания или в массовом порядке с помощью Dynatrace API.").

Если телеметрические данные поставляются по каналам, отличным от OneAgent (например, с использованием OpenTelemetry), важно обогащать телеметрические данные этими атрибутами вручную, чтобы обеспечить корректную привязку к хосту.

Пример того, как загрузить файл JSON, см. в [примере на Python](#python-example) ниже.

## Dynatrace Kubernetes Operator

Следующие файлы доступны в [каталоге обогащения](#enrichment-directory), если включено обогащение метаданными.

* `dt_metadata.json`
* `dt_metadata.properties`

Оба файла содержат одни и те же данные в разных форматах. Пример того, как загрузить файл JSON, см. в [примере на Python](#python-example) ниже.

Инструкции по настройке приведены в руководстве [Настройка каталога обогащения](/managed/ingest-from/setup-on-k8s/guides/metadata-automation/metadata-enrichment "Настроить обогащение метаданными в Dynatrace Operator для привязки метаданных Kubernetes к телеметрическим сигналам с помощью OneAgent, экспортёра OTLP или автономного обогащения.").

## Виртуальные файлы OneAgent

Когда OneAgent отслеживает приложение, приложение также получает доступ к следующим виртуальным файлам:

* `dt_metadata_e617c525669e072eebe3d0f08212e8f2.json`
* `dt_metadata_e617c525669e072eebe3d0f08212e8f2.properties`

Эти файлы не привязаны к каталогу обогащения и физически не существуют в файловой системе, а предоставляются инструментацией OneAgent. Оба файла возвращают одни и те же данные, а расширение файла (`.json`/`.properties`) определяет только формат вывода.

В контексте [Kubernetes Operator](#dynatrace-kubernetes-operator) виртуальный файл также содержит атрибуты файлов `dt_metadata.{json,properties}`.

### Как получить доступ к виртуальным файлам

1. Использовать стандартную функцию чтения файлов платформы языка программирования, чтобы открыть и прочитать один из этих файлов:

   * `dt_metadata_e617c525669e072eebe3d0f08212e8f2.json`
   * `dt_metadata_e617c525669e072eebe3d0f08212e8f2.properties`

   Выбранное расширение файла имеет значение для шага 4. Использовать только имя файла, не указывая дополнительный путь.
2. Содержимое файла, это одна текстовая строка с абсолютным путём к файлу (следует считать это временным значением и не сохранять, не кэшировать этот путь для дальнейшего использования).
3. Открыть файл по пути, полученному на предыдущем шаге, и прочитать его содержимое целиком.
4. Формат содержимого, полученного на предыдущем шаге чтения, будет соответствовать расширению файла, выбранному на шаге 1 (свойства в Java-стиле или JSON).

Если шаг 1 возвращает ошибку «файл не найден», нужно проверить, что приложение инструментировано OneAgent.

### Ограничения

* Поддерживается для процессов, отслеживаемых в режиме полного стека и глубокого мониторинга только приложения.
* Проверки `stat` и другие проверки `if (exists)` для этих файлов завершаются неудачно. Эти проверки не требуются для работы механизма.
* Использование `syscalls` напрямую для доступа к файлам не поддерживается. Это также означает, что приложения на Go, используемые для приёма метрик, не поддерживаются, если не используется OneAgent SDK, как описано в [Инструментирование приложения на Go с помощью OpenTelemetry](/managed/ingest-from/opentelemetry/walkthroughs/go "Узнать, как инструментировать приложение на Go с помощью OpenTelemetry и Dynatrace.").

## Пример на Python

Следующий пример показывает, как загрузить информацию обогащения в виде JSON на Python в Unix.

```
# Initialize dictionary variable



enrich_attrs = dict()



# Iterate over the potential data files and try reading them



for name in ["dt_metadata_e617c525669e072eebe3d0f08212e8f2.json", "/var/lib/dynatrace/enrichment/dt_metadata.json", "/var/lib/dynatrace/enrichment/dt_host_metadata.json"]:



try:



data = ''



with open(name) as f:



data = json.load(f if name.startswith("/var") else open(f.read()))



enrich_attrs.update(data)



except:



pass # An exception indicates the file was not available



# Use enrich_attrs here to enrich your requests to Dynatrace.



# For example, when instrumenting with OpenTelemetry, add the data as resource attributes.
```

Пример кода инициализирует пустой словарь для импортируемых атрибутов. Затем он перебирает массив имён файлов `.json` и загружает содержимое каждого файла как документ JSON, добавляя ключи в словарь. Исключения при работе с файлами означают, что конкретный файл недоступен, и игнорируются.
