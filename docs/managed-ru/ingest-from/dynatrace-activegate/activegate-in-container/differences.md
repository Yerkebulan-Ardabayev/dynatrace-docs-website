# Различия между контейнерным и хост-основанным ActiveGate

# Различия между контейнерным и хост-основанным ActiveGate

* 1-min read
* Published Sep 01, 2023

ActiveGate, развёрнутый на хосте с помощью установщика, в зависимости от выбранного [назначения](/managed/ingest-from/dynatrace-activegate/capabilities "Узнайте о возможностях и применениях ActiveGate."), состоит из нескольких процессов, обеспечивающих различные функциональные возможности. Образ контейнера охватывает лишь их подмножество, предоставляемое основным процессом ActiveGate.

## Назначения

Образ контейнера ActiveGate в настоящее время поддерживает только подмножество возможностей [маршрутизации и мониторинга](/managed/ingest-from/dynatrace-activegate/capabilities#functional_tbl "Узнайте о возможностях и применениях ActiveGate."), а также [частного синтетического мониторинга](/managed/ingest-from/dynatrace-activegate/capabilities#synthetic "Узнайте о возможностях и применениях ActiveGate.").

Полный обзор смотрите в разделе [Назначения и функциональность ActiveGate](/managed/ingest-from/dynatrace-activegate/capabilities "Узнайте о возможностях и применениях ActiveGate.").

|  | **Установщик** | **Контейнер** |
| --- | --- | --- |
| Маршрутизация OneAgent | Применимо | Применимо |
| API | Применимо | Применимо (кроме логов) |
| Infrastructure Monitoring (VMware, CloudFoundry, DBInsights, Kubernetes) | Применимо | Применимо |
| Мониторинг облачных сред | Применимо | Применимо (Azure), не применимо (AWS) |
| Extensions | Применимо | Не применимо |
| Синтетические мониторы | Применимо | Применимо |
| ZRemote | Применимо | Не применимо |

## Автоматическое обновление

[Автоматическое обновление ActiveGate](/managed/ingest-from/dynatrace-activegate/operation/update-activegate "Узнайте, как определить установленную версию ActiveGate и загрузить и установить последнюю версию.") поддерживается только для хост-основанных ActiveGate, развёрнутых с помощью установщика.

В окружениях Kubernetes обновления ActiveGate управляются средой выполнения контейнеров.

ActiveGate обновляется автоматически при перезапуске пода, если доступна новая версия, за исключением случаев, когда в образе явно указана определённая версия.

## Удалённое управление конфигурацией

[Удалённое управление конфигурацией](/managed/ingest-from/bulk-configuration "Выполните настройку OneAgent и ActiveGate на хостах со страницы Deployment status или в масштабе с помощью Dynatrace API.") не поддерживает контейнерные ActiveGate.

Конфигурация контейнерного ActiveGate не сохраняется постоянно. Она является декларативной и использует собственные механизмы конфигурации Kubernetes, поэтому любые изменения, инициированные механизмом удалённого управления конфигурацией, будут потеряны при перезапуске контейнера.
