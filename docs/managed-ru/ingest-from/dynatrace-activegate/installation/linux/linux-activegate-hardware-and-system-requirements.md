Запуск ActiveGate на выделенной системе

Для оптимальной производительности и повышенной безопасности рекомендуется устанавливать и запускать ActiveGate на выделенной системе.
Использование ActiveGate на выделенной системе не только минимизирует риск компрометации данных аутентификации ActiveGate, но и снижает вероятность злонамеренного изменения конфигурации.

Подробные характеристики пропускной способности логов для Environment ActiveGate при приёме логов API смотри на странице [Лимиты по умолчанию для Log Monitoring (Logs Classic)](/managed/analyze-explore-automate/log-monitoring/log-monitoring-limits "Лимиты по умолчанию для последней версии Dynatrace Log Monitoring.").

## Требования к оборудованию

Нужна выделенная под ActiveGate машина, обладающая следующим:

* 4 ГБ свободного места на диске для установки ActiveGate и Extensions, конфигурации и логов, для нужд автообновления.
* 4 ГБ для кэшированных установщиков ActiveGate и OneAgent, а также образов контейнеров, если их нужно хранить.
* Место для файлов дампов, если их нужно хранить. Эта функциональность по умолчанию отключена, но её можно включить в конфигурации ActiveGate. Максимальный размер места для хранения [настраивается](/managed/observe/application-observability/profiling-and-optimization/memory-dump-analysis/configure-an-activegate-for-memory-dump-storage "Как включить хранение дампов памяти на ActiveGate.") и по умолчанию составляет 100 ГБ.
* 600 МБ + 1,5 ГБ (буфер) свободного места на диске для файла постоянного хранения при повторной передаче логов Extension Execution Controller.
* Место для загрузки extensions, зависит от используемых extensions.
* 2 ГБ ОЗУ (рекомендуется 4 ГБ).
* 1 двухъядерный процессор.

Для крупных сред может понадобиться машина с дополнительными процессором и памятью.

## Детали постоянного хранения

Механизм надёжности обеспечивает сохранность логов Extension Execution Controller (EEC) в случаях, когда ActiveGate или OneAgent недоступны, возникают проблемы с сетью или EEC испытывает перегрузку при приёме данных. Это минимизирует пробелы в покрытии логами.

### Общая информация

* Постоянное хранение данных требует 2136 МБ свободного места на диске:

  + 600 МБ свободного места на диске используется механизмом надёжности
  + 1,5 ГБ свободного места на диске используется в качестве буфера
* Соответствие требованию проверяется периодически, и если оно не выполняется, постоянное хранение отключается, а приём логов будет передаваться без механизма надёжности.
* Объём используется пропорционально нагрузке при приёме логов.
* Если требование не может быть выполнено на хосте, можно изменить конфигурацию постоянного хранения логов. Подробнее смотри в разделе [Конфигурация постоянного хранения](#persistence_config).

### Конфигурация

Файл конфигурации Windows: `C:\ProgramData\dynatrace\remotepluginmodule\agent\conf\extensionsuser.conf`

Файл конфигурации Linux: `/var/lib/dynatrace/remotepluginmodule/agent/conf/extensionsuser.conf`

| **Переменная** | **Описание** |
| --- | --- |
| `persistence.reliable_mode` | `true`, надёжный режим включён; при невыполнении требования к месту генерируются логи SFM `false`, надёжный режим выключен; приём логов будет передаваться без механизма надёжности |
| `persistence.total_limit_kb` | Максимальный лимит объёма для Log Persistence Extensions в килобайтах. По умолчанию: 600 МБ. Можно изменить вручную, если требование не может быть выполнено на хосте. |

## Системные требования

* Убедиться в правильной [настройке сетевых портов](/managed/ingest-from/dynatrace-activegate/supported-connectivity-schemes-for-activegates "Learn about the connectivity priorities between ActiveGate types as well as the priorities between ActiveGates and OneAgents.").
* Операционная система должна поддерживать не менее 500 000 открытых файлов для пользователя `dtuserag`.
  Чтобы посмотреть системный лимит, выполнить команду:

  ```
  [user@host]# cat /proc/sys/fs/file-max
  ```

  Также возможно, что превышено [допустимое число открытых файлов в Linux](/managed/ingest-from/dynatrace-activegate#too-many-open-files-in-linux "Understand the basic concepts related to ActiveGate.").
* В операционной системе должно быть доступно не менее 20 000 процессов для пользователя `dtuserag`.
  Чтобы посмотреть системный лимит, выполнить команду:

  ```
  [user@host]# cat /proc/sys/kernel/pid_max
  ```
* Установщик ActiveGate для Linux не поддерживает ACL (Access Control List). Правила ACL могут блокировать доступ к директориям и файлам, созданным установщиком, из-за чего ActiveGate не сможет запуститься. При использовании ACL нужно отключить правила, относящиеся к директориям установки, заданным в следующих параметрах:

  ```
  INSTALL=



  CONFIG=



  LOG=



  TEMP=



  PACKAGES_DIR=
  ```

## Руководство по подбору размера

В следующей таблице показаны требования к размеру инстанса машины в зависимости от количества OneAgentов, взаимодействующих с ActiveGate. На каждом хосте OneAgent выполняет восемь задач мониторинга:

* мониторинг инфраструктуры;
* мониторинг логов;
* full-stack-мониторинг 3 инстансов Apache Tomcat;
* full-stack-мониторинг 2 инстансов Apache HTTP Server;
* мониторинг расширений (extension monitoring).

Реальное количество хостов может отличаться в зависимости от отслеживаемых технологий в окружении. Рекомендуется, чтобы машина, на которой работает ActiveGate, не превышала 50% загрузки CPU и 80% использования памяти. Кроме того, нужно учитывать, что ActiveGateы могут быть неработоспособны во время обновлений, перезапусков или кратковременных проблем со связью. Для обеспечения высокой доступности работающие ActiveGateы должны быть способны принять трафик недоступных ActiveGateов.
