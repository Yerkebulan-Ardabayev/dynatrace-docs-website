# Обновление ActiveGate

* 1-min read
* Published Jul 17, 2018

Ограничения

Функциональность **автообновления** и **обновления одним кликом** доступна только для развёртывания на основе хоста (с помощью установщика).

* [Поведение при обновлении контейнерного ActiveGate](/managed/ingest-from/dynatrace-activegate/activegate-in-container/differences#auto-update "Узнайте, чем контейнерный ActiveGate отличается от ActiveGate на основе хоста.")

Для просмотра списка установленных ActiveGate перейдите в **Deployment Status** > **ActiveGates**. Для каждого ActiveGate в списке отображается текущая **Version** и **Update status** (актуальный, ожидает обновления или обновляется).

Жёлтый значок предупреждения указывает на то, что ваш ActiveGate отстаёт более чем на пять версий. Такие ActiveGate следует обновить как можно скорее.

ActiveGate в контейнерах развёртываются и загружаются с помощью специальных облачных инструментов. Например, Kubernetes использует [пользовательские определения ресурсов](/managed/ingest-from/setup-on-k8s/guides/deployment-and-configuration/monitoring-and-instrumentation/k8s-api-monitoring "Мониторинг Kubernetes API с помощью Dynatrace.").

### Автоматическое обновление

Для управления функцией обновления конкретного ActiveGate в списке выберите ActiveGate, чтобы развернуть подробности.

Перейдите в **Settings**, чтобы открыть настройки **ActiveGate updates** для конкретного ActiveGate.

* **Auto-update**
  Для использования функции автообновления убедитесь, что переключатель **Automatic updates at earliest convenience** для ActiveGate включён. Это означает, что при выходе новой версии ActiveGate на соответствующий хост будет загружен новый установочный пакет и установлена новая версия ActiveGate. Это настройка по умолчанию для новых окружений. Для существующих окружений настройка остаётся неизменной.
  Проверка доступности обновлений выполняется с интервалом 30 минут.
* **One-click update**
  Для немедленного обновления нажмите **Update**. Эта функция доступна только если переключатель **Automatic updates at earliest convenience** выключен для конкретного ActiveGate.

### Ручная загрузка и обновление

Также можно загрузить и обновить ActiveGate вручную. Удалять текущую версию ActiveGate не нужно. Просто установите новую версию поверх старой: конфигурация ActiveGate будет перенесена.

* [Установка Environment ActiveGate](/managed/ingest-from/dynatrace-activegate/installation "Узнайте, как настроить ActiveGate.").

Во время обновления конфигурация ActiveGate сохраняется в файлах `custom.properties` и `launcheruserconfig.conf`. Эти два файла не перезаписываются при обновлении, однако перед обновлением рекомендуется создать их резервные копии.

* Свойства файла `custom.properties` см. [Настройка ActiveGate](/managed/ingest-from/dynatrace-activegate/configuration/configure-activegate "Узнайте, какие свойства ActiveGate можно настроить.").
* Свойства файла `launcheruserconfig.conf` см. [Настройка лончера ActiveGate](/managed/ingest-from/dynatrace-activegate/configuration/configure-activegate "Узнайте, какие свойства ActiveGate можно настроить.").

## Статус обновления

Статус обновления ActiveGate, отображаемый на странице **Deployment Status**, может принимать следующие значения:

**Up to date**
Последняя доступная версия ActiveGate установлена на соответствующем хосте.

**Update available**
Для данной версии ActiveGate доступно обновление.

**Update pending**
Сразу после нажатия **Update ActiveGate** статус меняется на **Update pending** и сохраняется до начала процесса обновления.
Статус также может отображаться как `pending` в следующих случаях:

* Кластер в настоящее время обновляется.
* Достигнуто максимальное количество одновременных загрузок обновлений, и ActiveGate ожидает продолжения загрузки.

**Update in progress**
ActiveGate запросил и загрузил новый установочный пакет с сервера и в данный момент устанавливает его или восстанавливает подключение к серверу.

**Update problem**
В этом случае для ActiveGate отображается старый номер версии.
Возможные проблемы:

* ActiveGate загрузил новый установщик, но установка не была предпринята или не прошла успешно; ActiveGate по-прежнему работает на старой версии.
  Проверьте журналы автообновления ActiveGate и журналы установщика для выяснения причины.
* ActiveGate загрузил новый установщик, но затем не смог восстановить подключение к серверу (потерян).
  Проверьте журналы установщика ActiveGate для выяснения причины сбоя установки.
* Для ActiveGate нет доступных установщиков.
* Обновление приостановлено, так как в другом окружении возникли проблемы с обновлением предложенной версии ActiveGate.

**Unknown**
Соединение с этим ActiveGate потеряно и статус не может быть определён.
Этот статус также может отображаться при успешном удалении ActiveGate: в этом случае на странице **Deployment Status** удалённый ActiveGate будет отображаться со статусом обновления `Unknown` в течение семи дней.
