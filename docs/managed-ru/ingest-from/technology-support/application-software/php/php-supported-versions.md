# Поддерживаемые версии PHP

* Чтение: 2 мин
* Опубликовано: 12 июня 2019

Сроки поддержки разных версий PHP определяются проектом PHP. Чтобы узнать, какие версии в настоящее время поддерживаются PHP и когда для каждой версии запланировано окончание поддержки, обратитесь к разделу [Поддерживаемые версии PHP](https://php.net/supported-versions.php) на [php.net](https://php.net/).

* Устаревшие версии не получают обновлений или патчей безопасности, поэтому не следует запускать их в производственной среде.
* Dynatrace обязуется поддерживать каждую версию PHP как минимум столько же, сколько её поддерживает вендор (в большинстве случаев поддержка продлевается ещё минимум на полгода после этого момента).

## Поддержка релизов Early Access

Если для версии PHP есть поддержка Early Access, она имеет отдельный переключатель, который позволяет включить или отключить мониторинг этой версии.

Чтобы включить мониторинг PHP для релиза Early Access в Dynatrace

1. Перейдите в **Settings** и выберите **Monitoring** > **Monitored technologies**.
2. На вкладке **Supported technologies** найдите **PHP** и откройте его для редактирования.
3. На странице **PHP** включите **Enable PHP *x* monitoring on every host**.

   Убедитесь, что ваше окружение мониторинга соответствует требованиям, указанным под переключателем.

## Сообщения для неподдерживаемых версий

Когда OneAgent обнаруживает неподдерживаемую версию PHP, она исключается из мониторинга, а на странице обзора процесса отображается предупреждение:

```
Activation of deep monitoring was unsuccessful



Process version is not supported
```

![PHP Monitoring Process version not supported](https://dt-cdn.net/images/php-monitoring-1668-782cbab381.png)

Мониторинг PHP: версия процесса не поддерживается

Это сообщение также записывается в журналы ошибок на хост-машине.

Ознакомьтесь со [средами и версиями, поддерживаемыми Dynatrace](/managed/ingest-from/technology-support#php "Технические подробности о поддержке Dynatrace конкретных платформ и фреймворков разработки.") применительно к PHP, а также с [поддерживаемыми версиями PHP](https://php.net/supported-versions.php).

## Известные ограничения

* Мониторинг CLI по умолчанию отключён, поскольку при кратковременном выполнении CLI накладные расходы на инъекцию могут составлять значительную долю всего времени выполнения. Не следует использовать мониторинг CLI, если в отслеживаемом приложении применяются многопоточность и форкинг, так как эти возможности не поддерживаются.
* На данный момент нет различия между CGI и Fast-CGI; они регистрируются как единый тип: CGI.
