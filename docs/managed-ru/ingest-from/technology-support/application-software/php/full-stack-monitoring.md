# Full-stack PHP monitoring

# Full-stack PHP monitoring

* Чтение: 4 мин
* Обновлено: 8 ноября 2024

Можно мониторить PHP в своих средах Windows (IIS/Apache): полностековый мониторинг теперь доступен в Windows для приложений Apache mod\_php и PHP CGI.

## Включение мониторинга PHP

Чтобы включить мониторинг PHP в Dynatrace

1. Перейдите в **Settings** и выберите **Monitoring** > **Monitored technologies**.
2. На вкладке **Supported technologies** найдите **PHP** и откройте его для редактирования.
3. Включите настройки, необходимые для вашего мониторинга:

   * **Monitor PHP on every host**  
     Этот переключатель включает мониторинг PHP на всех хостах. Чтобы мониторить часть хостов, отключите этот переключатель, а затем включите мониторинг PHP на выбранных хостах.
   * **Enable FastCGI PHP processes launched by Apache HTTP Server on every host**  
     Эта настройка постоянно включена начиная с OneAgent версии 1.191.
   * **Enable PHP *x* monitoring on every host**  
     Если доступен [релиз Early Access](/managed/whats-new/preview-releases "Узнайте о наших Preview-релизах и о том, как вы можете в них участвовать."), этот переключатель позволяет попробовать его.

     Убедитесь, что ваше окружение мониторинга соответствует требованиям, указанным под переключателем.

## Запуск PHP на веб-сервере

Наиболее распространённые способы запуска PHP на веб-сервере:

* `mod_PHP`, вариант по умолчанию на чистых серверах Apache HTTP. PHP выполняется как модуль Apache, то есть запрос PHP обрабатывается в рамках процесса Apache. Это обеспечивает хорошую производительность PHP, но гибкость ограничена конфигурацией, разрешениями, ограничениями Apache и т. д.
* `PHP_FPM`, FastCGI Process Manager (FPM), использует управление пулами для оптимизации производительности. Каждый пул действует как отдельный экземпляр PHP, что позволяет создавать независимую конфигурацию для каждого пула и обеспечивает детальный контроль и большую гибкость. Это самый производительный способ запуска PHP.

## Мониторинг PHP-CLI

Early Access

Мониторинг PHP-CLI включает глубокий мониторинг ваших приложений PHP-CLI. Он обеспечивает видимость баз данных, поддержку многопоточности, мониторинг исходящих вызовов и многое другое, что позволяет:

* Анализировать производительность вашего приложения PHP-CLI
* Легко находить узкие места в вашем коде
* Понять, почему обмен данными с базой данных занимает именно столько времени
* Выяснить, правильно ли приложение управляет памятью

### Полная видимость на уровне кода для вашего приложения PHP-CLI

С мониторингом PHP-CLI вы получаете:

* Поддержка PHP-CLI в потокобезопасном и непотокобезопасном режимах для всех версий PHP, поддерживаемых Dynatrace
* Все существующие функции PHP в OneAgent, а также функции только для CLI:

  + Поддержка `pthreads` для потокобезопасного PHP-CLI
  + Горячие точки методов с использованием амбиентного сэмплирования
  + Поддержка атрибутов запросов
  + Глубокий мониторинг (после включения [обнаружения пользовательских сервисов](/managed/observe/application-observability/services/service-detection/service-detection-v1/service-types/custom-services "Определите точки входа (метод, класс или интерфейс) для пользовательских сервисов, не использующих стандартные протоколы."))

### Требования для мониторинга PHP-CLI

* OneAgent версии 1.181+, установленный на всех отслеживаемых хостах
* Включён мониторинг PHP NG  
  Чтобы проверить или изменить эту настройку

  1. Перейдите в **Settings** и выберите **Monitoring** > **Monitored technologies**.
  2. Найдите **PHP** и откройте его для редактирования.

### Ограничения мониторинга PHP-CLI

* Инъекция OneAgent создаёт небольшие накладные расходы при запуске процесса. Её не следует включать для короткоживущих процессов.
* Инструментация процессов `php-cli` с кодом, указанным в качестве аргумента командной строки, не поддерживается.
* Инструментация `pthreads` поддерживается с PHP 7.2 по PHP 7.3. Более ранние релизы PHP не полностью потокобезопасны, и их не следует использовать с `pthreads`.

### Начало работы с мониторингом PHP-CLI

Чтобы включить глубокий мониторинг ваших приложений PHP-CLI

1. Перейдите в **Settings** и выберите **Processes and containers** > **Custom process monitoring rules**.
2. Выберите **Add custom rule**, чтобы начать определение правила мониторинга процессов для PHP-CLI.  
   Можно построить правило на основе таких условий, как:

   * `EXE name`
   * `EXE path`
   * `PHP script`
   * `PHP working directory`

   Подробнее о правилах мониторинга процессов см.:

   * запись в блоге [Автоматизированное и детальное управление мониторингом в масштабе](https://www.dynatrace.com/news/blog/automated-fine-grained-control-of-monitoring-at-scale/)
   * [Настройка мониторинга групп процессов](/managed/observe/infrastructure-observability/process-groups/configuration/pg-monitoring "Способы настройки мониторинга групп процессов")
3. Выберите **Save changes**.
   Ваше правило появится на странице **Custom process monitoring rules**.
4. Добавьте дополнительные правила при необходимости.
5. Перезапустите ваше приложение PHP.

### Извлекаем пользу из мониторинга PHP-CLI

Получите полное представление о PHP: соединения с базами данных и многое другое

Проверьте, правильно ли ваше приложение управляет памятью

С метриками сборки мусора и OPcache вы получаете представление о том, как ваше приложение управляет памятью. Вы также получаете подробные метрики процессов, включая использование CPU, использование памяти, рабочие процессы и информацию ввода-вывода из коробки.
