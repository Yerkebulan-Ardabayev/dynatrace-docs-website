# Node.js

* Справочник
* Чтение: 4 мин
* Обновлено 23 апреля 2026 г.

[Node.js](https://nodejs.org), это серверный фреймворк на основе [движка JavaScript V8](https://developers.google.com/v8/) от Google. Node.js имеет асинхронную модель выполнения и часто используется для связывания компонентов или как прокси-уровень в корпоративных средах.

## Возможности

Dynatrace предоставляет широкие возможности мониторинга Node.js:

* Метрики Heap и процессов
* Дампы Heap
* Сэмплирование CPU
* Метрики цикла событий
* Анализ входящих и исходящих HTTP-вызовов
* Выделенная поддержка различных баз данных (включая захват запросов)
* [Поддержка OpenTelemetry](https://github.com/open-telemetry/opentelemetry-js-api/) для захвата трассировок и приёма метрик.  
  Подробнее см. в разделе [Инструментирование JavaScript-приложения на Node.js с OpenTelemetry](/managed/ingest-from/opentelemetry/walkthroughs/nodejs "Узнайте, как инструментировать JavaScript-приложение на Node.js с помощью OpenTelemetry и Dynatrace.")
* [OneAgent SDK](/managed/ingest-from/extend-dynatrace/extend-tracing/oneagent-sdk "Dynatrace OneAgent SDK позволяет вручную инструментировать приложение, чтобы расширить сквозную видимость для фреймворков и технологий, для которых ещё нет готового модуля кода.") для пользовательской трассировки
* [Непрерывный анализ потоков для рабочих потоков](#worker-threads)

Подробности о поддерживаемых технологиях, используемых вместе с Node.js, см. в [матрице поддерживаемых технологий](/managed/ingest-from/technology-support#nodejs "Технические подробности о поддержке Dynatrace для конкретных платформ и фреймворков разработки.").

## Поддержка и прекращение поддержки

Node.js следует [модели выпусков LTS](https://github.com/nodejs/Release).

Для Node.js v25 и более ранних версий нечётные версии достигают конца жизненного цикла (EOL) вскоре после выпуска следующей чётной версии, тогда как чётные версии в итоге становятся LTS-выпусками. Для корпоративных production-сред мы настоятельно рекомендуем использовать LTS-выпуски.

Начиная с Node.js v27 модель выпусков меняется: каждая мажорная версия проходит сначала нестабильную, затем стабильную стадию, при этом все версии в итоге становятся LTS-выпусками, а в год выходит одна мажорная версия. Подробнее см. [Evolving the Node.js Release Schedule](https://nodejs.org/en/blog/announcements/evolving-the-nodejs-release-schedule).

При выпуске каждой новой мажорной версии Node.js мы добавляем её поддержку.

Dynatrace продлевает поддержку каждой версии Node.js минимум на шесть месяцев после официальной даты окончания жизненного цикла. Для LTS-выпусков поддержка длится минимум год после окончания жизненного цикла, давая вам больше времени на обновление.

Обратите внимание, что поддерживаются только [официальные версии Node.js](https://nodejs.org/en/about/previous-releases).

## Непрерывный анализ потоков для рабочих потоков {#worker-threads}

Node.js версии 12+ OneAgent версии 1.251+ Dynatrace версии 1.256+

[Непрерывный анализ потоков](/managed/observe/application-observability/profiling-and-optimization/continuous-thread-analysis "Непрерывно анализируйте состояние потоков и их развитие, чтобы быстро выявлять и устранять проблемы производительности в процессах Java и Node.js.") для [рабочих потоков](https://nodejs.org/api/worker_threads.html#worker-thread) может автоматически выявлять интенсивно нагружающие CPU потоки и точно определять проблемы масштабируемости, когда работа распределена между множеством потоков, чтобы вы могли устранить узкие места производительности до того, как это затронет ваших конечных пользователей.

Ограничения

Метрики, специфичные для Node.js (например, память, сборка мусора и циклы событий), сообщаются только для потока `main`.

Просмотр классов (необходимый для пользовательских сервисов обмена сообщениями kafkajs) ограничен потоком `main`.

Автоматическое трассирование транзакций между потоками `main` и `worker` отсутствует. Для трассирования транзакций между потоками можно использовать либо [трассирование OpenTelemetry](/managed/ingest-from/dynatrace-oneagent/oneagent-and-opentelemetry/oneagent-otel "Узнайте, как отправлять данные OpenTelemetry в Dynatrace OneAgent."), либо [OneAgent SDK](/managed/ingest-from/extend-dynatrace/extend-tracing/oneagent-sdk "Dynatrace OneAgent SDK позволяет вручную инструментировать приложение, чтобы расширить сквозную видимость для фреймворков и технологий, для которых ещё нет готового модуля кода.").

## Известные ограничения

* Из-за платформенных ограничений JavaScript и Node.js видимость на уровне кода ограничена по сравнению с .NET или Java.
* В сочетании с неподдерживаемыми сторонними модулями контекст может теряться в асинхронных колбэках. В таких случаях обратитесь к эксперту по продукту Dynatrace через онлайн-чат в вашем окружении Dynatrace.
* OneAgent версии 1.279+ время CPU для сервисов Node.js не сообщается. Эти показатели вводили в заблуждение, поскольку по своей конструкции значительная часть любой операции обрабатывается асинхронно внутри среды выполнения Node.js без возможности соотнести фактическое время CPU с конкретным сервисом.
* Web Streams, WebSocket Client не поддерживаются.
* Функции Node.js, помеченные как «experimental», не поддерживаются.
* Использование NPM-модуля [esm](https://github.com/standard-things/esm) в [варианте 1 для пакетов](https://github.com/standard-things/esm/tree/3.2.25#getting-started) может привести к снижению видимости (особенно если он используется для главного скрипта приложения). Предпочтительнее использовать вариант 2 для предзагрузки `esm` через параметр командной строки `-r`.
* В настоящее время поддержка [модулей ECMAScript](https://nodejs.org/api/esm.html) (так называемых «ES6 modules») ограничена:

  + Если сам главный файл скрипта является модулем ECMAScript, то для внедрения OneAgent в процесс Node.js требуется OneAgent версии 1.219+ с включённым [предзагрузчиком агента](/managed/whats-new/preview-releases#oneagent-1-219-nodejs-agent-preloading "Узнайте о наших Preview-выпусках и о том, как в них участвовать.").
  + Инструментирование модулей ECMAScript в настоящее время недоступно. Это ограничивает поддержку `kafkajs`, если заданная пользователем точка входа для сенсора KafkaJs находится внутри модуля ECMAScript.
* **Webpack** по умолчанию объединяет все модули в один файл. OneAgent не может инструментировать объединённые модули. Чтобы обойти это ограничение, все модули, которые должны быть инструментированы OneAgent (такие как `express`, `mongodb` и `pg`), нужно вынести как внешние (externals) в конфигурации webpack. Подробнее см. документацию по [webpack externals](https://webpack.js.org/configuration/externals/).
* Использование **Webpack** или других сборщиков (bundler) также может повлиять на автоматическое обнаружение уязвимостей. Причина в том, что программные компоненты не удаётся обнаружить, так как они скрыты за конфигурацией сборщика и недоступны во время выполнения. Обнаружить и сообщить можно только о пакетах, развёрнутых как внешние.
