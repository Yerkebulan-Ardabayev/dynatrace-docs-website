# Поддерживаемые версии Go

* Чтение: 7 мин
* Обновлено: 13 апреля 2026

Каждый раз, когда выходит новая мажорная версия Go, Dynatrace добавляет её поддержку. Поддержка каждой минорной версии
также добавляется: подробнее см. [Матрица версий](#go-version-matrix).

[Политика релизов Go](https://dt-url.net/uos3rmi) поддерживает две последние мажорные версии Go. Однако в Dynatrace решили
поддерживать каждую версию Go как минимум на полгода дольше, чтобы у вас было достаточно времени на обновления.

## Внешние метаданные

OneAgent может поддерживать более новые версии Go, запрашивая внешние метаданные у Dynatrace Cluster. После того как OneAgent
получит внешние метаданные, он сможет инструментировать приложение, основанное на версии Go, для которой у OneAgent
нет встроенной поддержки. Требуется перезапуск процесса Go.

Поэтому [матрица версий](#go-version-matrix) имеет две разные верхние границы в каждом диапазоне версий. Встроенные
версии Go поддерживаются всегда, тогда как версия из верхней границы в скобках может поддерживаться, если OneAgent сможет получить нужные
внешние метаданные от Dynatrace Cluster.

Внешние метаданные публикуются независимо от цикла релизов OneAgent, и OneAgent всегда получает последние
доступные внешние метаданные от Dynatrace Cluster. Это также означает, что в поэтапном окружении приложения могут
быть инструментированы на более позднем этапе, хотя не были инструментированы на более раннем, потому что в промежутке появились новые
метаданные. Если такое поведение нежелательно, отключите функцию OneAgent **Go external metadata**, чтобы внешние
метаданные не использовались. Функции OneAgent находятся в **Settings** > **Preferences** > **OneAgent features**.
