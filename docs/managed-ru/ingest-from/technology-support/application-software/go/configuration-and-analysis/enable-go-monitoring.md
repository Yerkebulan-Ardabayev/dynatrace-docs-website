# Включение мониторинга Go

* Чтение: 2 мин
* Обновлено: 23 октября 2025

Dynatrace OneAgent для Go может мониторить ваши Golang-приложения, а также любые сторонние Go-приложения, от которых могут зависеть ваши собственные приложения. Эта технология мониторинга позволяет собирать данные, которые далеко выходят за рамки возможностей традиционных решений мониторинга. Dynatrace извлекает внутреннюю информацию среды выполнения Go, недоступную через публичные API среды выполнения Go.

Мониторинг Go включён по умолчанию для всех новых окружений. Однако мониторятся только приложения, указанные как разрешённые в предустановленных правилах мониторинга процессов. Чтобы включить мониторинг дополнительных процессов, потребуется задать собственные правила мониторинга.

## Включение мониторинга Go

Чтобы активировать мониторинг Go

1. Перейдите в **Settings** > **Monitoring** > **Monitored technologies**.
2. Найдите **Go** в списке поддерживаемых технологий и нажмите **Edit** (иконка карандаша).
3. Включите **Monitor Go on every host**.

   ![Включение мониторинга Go](https://dt-cdn.net/images/enable-go-monitoring-1839-e2e67e5563.png)

   Включение мониторинга Go
4. [Создайте правило мониторинга процессов](/managed/observe/infrastructure-observability/process-groups/configuration/pg-monitoring#rules "Способы настройки мониторинга групп процессов"), чтобы включить глубокий мониторинг выбранных процессов.

## Включение статического мониторинга Go

Поддержка мониторинга статически слинкованных Go-приложений доступна начиная с версии Dynatrace OneAgent 1.203.

Если нет зависимостей от пакетов, использующих [cgo](https://blog.golang.org/cgo), инструментарий Go по умолчанию создаёт статически слинкованные исполняемые файлы Go. Такие приложения не допускают динамическую загрузку дополнительного кода, поэтому большинство агентов мониторинга не могут автоматически внедриться в такие приложения.

С Dynatrace не нужно принудительно делать приложение динамически слинкованным или менять исходный код, чтобы мониторить статически слинкованные Go-приложения. Однако есть [несколько ограничений для этого решения](/managed/ingest-from/technology-support/application-software/go/support/go-known-limitations#static-monitoring "Узнайте об ограничениях поддержки Go и способах их обхода.").

Чтобы активировать статический мониторинг Go-приложений

1. Перейдите в **Settings** > **Monitoring** > **Monitored technologies**.
2. Найдите **Go** в списке поддерживаемых технологий и нажмите **Edit** (иконка карандаша).
3. Убедитесь, что **Monitor Go on every host** включён.
4. Включите **Enable Go static application monitoring on every host**.

   ![Включение статического мониторинга Go](https://dt-cdn.net/images/enable-go-static-monitoring-1838-33cac1f1b9.png)

   Включение статического мониторинга Go
5. [Создайте правило мониторинга процессов](/managed/observe/infrastructure-observability/process-groups/configuration/pg-monitoring#rules "Способы настройки мониторинга групп процессов"), чтобы включить глубокий мониторинг каждого статически слинкованного исполняемого файла Go.

## Встроенные правила мониторинга процессов

У OneAgent есть набор встроенных правил мониторинга процессов, которые определяют, какие процессы мониторит OneAgent. Чтобы проверить текущие правила мониторинга процессов, перейдите в **Settings** > **Processes and containers** > **Built-in process monitoring rules**.

По умолчанию OneAgent мониторит предопределённый список Go-приложений (например, Gorouter и InfluxDB) и динамически слинкованные Go-приложения, запущенные в контейнере.
Чтобы мониторить произвольное Go-приложение, [задайте пользовательское правило мониторинга процессов](/managed/observe/infrastructure-observability/process-groups/configuration/pg-monitoring#rules "Способы настройки мониторинга групп процессов").
