# Полная видимость на уровне кода

* Чтение: 1 мин
* Опубликовано: 15 апреля 2019

Go-приложения компилируются напрямую в машинный код процессора. В Go нет абстрагирующего слоя виртуальной машины с заданными точками подключения мониторинга, как Java JVM или .NET CLR. Поэтому Dynatrace OneAgent работает на уровне инструкций процессора, чтобы перехватывать вызовы функций Go. Возможности параллельной обработки в Go уникальны, и OneAgent оптимизирован так, чтобы избегать неявных точек синхронизации между горутинами при сборе данных мониторинга.

Страницы обзора процессов и сервисов, а также представления hotspot и распределённые трассировки дают ценную информацию о деталях обработки в ваших Golang-приложениях и могут помочь вовремя обнаружить критические проблемы. Профилировщик CPU позволяет исследовать код, выполняемый вне контекста веб-запроса, например фоновые активности и планировщики.

![Профилировщик CPU](https://dt-cdn.net/images/cpu-profiler-1680-826439a3cc.png)

Профилировщик CPU

Dynatrace OneAgent умеет отличать код приложения, выполняемый в контексте распределённой трассировки (например, входящего веб-запроса), от кода, выполняемого в фоне. **Background thread CPU** и **Service request CPU** отслеживаются отдельно. Например, посмотрите на скриншот ниже: пакет Go **net/http** порождает новую горутину для каждого входящего веб-запроса или соединения. Именно поэтому горутина, выполняющая функцию-обработчик веб-запроса, помечается как связанная с сервисом. Другие горутины, запущенные до обработки веб-запроса, также считаются частью выполнения сервиса и поэтому тоже помечаются. С другой стороны, обработка, выполняемая горутинами без метки связи с сервисом, учитывается как фоновая активность.
