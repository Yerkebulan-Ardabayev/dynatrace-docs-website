Dynatrace собирает метрики для нескольких предварительно выбранных пространств имён, включая Amazon Cognito. Вы можете просматривать метрики для каждого экземпляра сервиса, разделять метрики по нескольким измерениям и создавать пользовательские графики, которые можно закрепить на дашбордах.

## Предварительные требования

Для включения мониторинга этого сервиса вам необходимо:

* ActiveGate версии 1.181+, а именно:

  + Для развёртываний Dynatrace SaaS вам необходим Environment ActiveGate или Multi-environment ActiveGate.
  + Для развёртываний Dynatrace Managed можно использовать любой тип ActiveGate.

    Для доступа на основе ролей (как в [SaaS](../../../../../ingest-from/amazon-web-services/integrate-with-aws/cloudwatch-metrics.md#role-based-access "Интеграция метрик из Amazon CloudWatch."), так и в [Managed](https://docs.dynatrace.com/managed/shortlink/aws-managed-deployment) развёртывании) вам необходим [Environment ActiveGate](../../../../../ingest-from/dynatrace-activegate/installation.md "Узнайте, как настроить ActiveGate"), установленный на хосте Amazon EC2.
* Dynatrace версии 1.182+
* Обновлённая [политика мониторинга AWS](../../../../../ingest-from/amazon-web-services/integrate-with-aws/cloudwatch-metrics.md#aws-policy-and-authentication "Интеграция метрик из Amazon CloudWatch.") для включения дополнительных сервисов AWS.
  Для [обновления политики AWS IAM](https://docs.aws.amazon.com/IAM/latest/UserGuide/access_policies_manage-edit.html#edit-managed-policy-console) используйте JSON ниже, содержащий политику мониторинга (разрешения) для всех поддерживаемых сервисов.

* `"apigateway:GET"` для **Amazon API Gateway**

* Для сбора метрик необходимо включить режим аудита для экземпляра Amazon Cognito. В режиме аудита расширенные функции безопасности публикуют метрики в Amazon CloudWatch.

## Просмотр метрик сервиса

Вы можете просматривать метрики сервиса в среде Dynatrace на **странице обзора пользовательского устройства** или на странице **Dashboards**.

### Просмотр метрик на странице обзора пользовательского устройства

Для доступа к странице обзора пользовательского устройства:

1. Перейдите в ![Technologies](https://dt-cdn.net/images/technologies-512-977161d83c.png "Technologies") **Technologies & Processes Classic**.
2. Отфильтруйте по имени сервиса и выберите соответствующую группу пользовательских устройств.
3. После выбора группы пользовательских устройств вы окажетесь на **странице обзора группы пользовательских устройств**.
4. **Страница обзора группы пользовательских устройств** отображает все экземпляры (пользовательские устройства), принадлежащие группе. Выберите экземпляр для просмотра **страницы обзора пользовательского устройства**.

### Просмотр метрик на дашборде

Вы также можете просматривать метрики в веб-интерфейсе Dynatrace на дашбордах. Для этого сервиса нет предустановленного дашборда, но вы можете [создать собственный](../../../../../analyze-explore-automate/dashboards-classic/dashboards/create-dashboards.md "Узнайте, как создавать и редактировать дашборды Dynatrace.").

Чтобы проверить наличие предустановленных дашбордов для каждого сервиса AWS, см. список ниже.

## Доступные метрики

| Название | Описание | Единица | Статистика | Измерения | Рекомендуемая |
| --- | --- | --- | --- | --- | --- |
| AccountTakeOverRisk | Запросы, в которых Amazon Cognito обнаружил риск захвата аккаунта | Count | Sum | Region, RiskLevel | Да |
| CompromisedCredentialsRisk | Запросы, в которых Amazon Cognito обнаружил скомпрометированные учётные данные | Count | Sum | Region, Operation, UserPoolId, RiskLevel | Да |
| NoRisk | Запросы, в которых Amazon Cognito не обнаружил никакого риска | Count | Sum | Region, Operation, UserPoolId | Да |
| OverrideBlock | Запросы, которые Amazon Cognito заблокировал на основе конфигурации разработчика | Count | Sum | Region, RiskLevel | Да |
| Risk | Запросы, которые Amazon Cognito отметил как рискованные | Count | Sum | Region, Operation, UserPoolId | Да |
