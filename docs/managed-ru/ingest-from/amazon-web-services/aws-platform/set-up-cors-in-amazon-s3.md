# Настройка CORS в Amazon S3

* Практическое руководство
* Чтение: 3 мин
* Опубликовано 19 июля 2017 г.

Используйте эти рекомендации, чтобы настроить CORS (cross-origin HTTP requests) в AWS для бакетов Amazon S3 (Simple Storage Service). Подробнее о CORS и принципах его работы см. в статье [HTTP access control (CORS)](https://developer.mozilla.org/en-US/docs/Web/HTTP/Access_control_CORS).

## Введение

Современные веб-браузеры не предоставляют доступ к информации об ошибках в `window.onerror` для скриптов, загруженных с других доменов. Это серьёзное ограничение, которое снижает полезность движка обнаружения JavaScript-ошибок Dynatrace.

Без CORS вместо подробностей ошибки и стек-трейса при необработанном JavaScript-исключении появляется сообщение вида:

«Script error» обычно возникает, когда исключение нарушает same-origin-policy браузера, то есть когда ошибка происходит в скрипте, размещённом на домене, отличном от домена текущей страницы.
Если скрипт отдаётся с установленными CORS HTTP-заголовками, в большинстве браузеров можно получить полные детали ошибки, добавив атрибут `crossorigin="anonymous"` к тегу скрипта. К сожалению, Internet Explorer 11 и Edge сейчас не поддерживают атрибут `crossorigin`. Также важно устанавливать атрибут `crossorigin` только для скриптов, у которых присутствуют CORS-заголовки, иначе скрипт перестанет выполняться.

Чтобы получить полную видимость JavaScript-ошибок, на серверах с другим доменом нужно настроить CORS HTTP-заголовки, а к тегу скрипта применить атрибут `crossorigin`. Перед этим шагом необходимо настроить Amazon S3.

### Настройка Amazon S3

Сначала задайте параметры CORS для ваших ресурсов в S3. Настройка выполняется на уровне бакета. К сожалению, задать параметры CORS для отдельных ресурсов внутри бакета нельзя.

1. Откройте консоль Amazon S3.
2. Выберите бакет, в котором находятся ваши ресурсы.
3. Перейдите в **Permissions**.
4. Прокрутите вниз до **Cross-origin resource sharing (CORS)** и нажмите **Edit**.
5. Вставьте конфигурацию CORS в формате JSON.

   Пример JSON:

   ```
   [



   {



   "AllowedHeaders": [



   "*"



   ],



   "AllowedMethods": [



   "GET",



   "HEAD"



   ],



   "AllowedOrigins": [



   "*"



   ],



   "ExposeHeaders": []



   }



   ]
   ```

   Подробности см. в [Creating a cross-origin resource sharing (CORS) configuration](https://docs.aws.amazon.com/AmazonS3/latest/userguide/ManageCorsUsing.html).
6. Нажмите **Save changes**, чтобы сохранить конфигурацию.

### Настройка CloudFront

После настройки Amazon S3 включите CORS-заголовки в CloudFront. Иначе заголовки будут отфильтрованы, и механизм не сработает.

1. Откройте консоль CloudFront. Откроется окно, похожее на показанное ниже. Найдите свою дистрибуцию в списке и кликните по ссылке её **ID**. При необходимости используйте поле поиска (выделено красным ниже), чтобы найти нужную дистрибуцию.

   ![AWS cors 3](https://dt-cdn.net/images/aws-cors-3-1652-d619805ef5.png)

   AWS cors 3
2. Теперь должна отображаться ваша дистрибуция. Нажмите вкладку **Behaviors**. Найдите запись поведения для вашего ресурса, описанную в колонке **Path Pattern**. Выберите шаблон и нажмите **Edit**. Если запись поведения отсутствует, создайте её.

   ![AWS cors 4](https://dt-cdn.net/images/aws-cors-4-968-4f0dee4542.png)

   AWS cors 4
3. Теперь можно править настройки поведения. Все элементы, относящиеся к CORS, выделены ниже красным. Сначала выберите, какой HTTP-метод будет доступен для этого поведения. Для **Allowed HTTP Methods** рекомендуется выбрать второй или третий вариант и установить флажок **OPTIONS**, иначе использовать CORS для REST-вызовов с preflighted-запросами не получится.
4. В списке **Forward Headers** выберите **Allowlist**. Появится меню **Allowlist headers**. Выберите конкретный заголовок в левом списке и нажмите **Add**. Если нужного заголовка нет, введите его в текстовое поле **Filter headers...** и нажмите **Add custom**. Для CORS важнее всего заголовки `Origin` и `Access-Control-Allow-Origin`. Остальные заголовки опциональны и зависят от того, что вы хотите получить.
5. Подтвердите изменения, нажав **Yes, Edit**.

   AWS cors 5
