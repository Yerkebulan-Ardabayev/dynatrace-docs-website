# Миграция расширений JMX

* Практическое руководство
* Чтение: 3 мин
* Опубликовано 16 июня 2025 г.

Платформа [Extensions Framework](/managed/ingest-from/extensions "Узнайте, как создавать расширения Dynatrace и управлять ими.") для JMX доступна начиная с версии 1.213 и открывает новые возможности визуализации и запросов к данным. Следуйте этому руководству, чтобы узнать, как использовать Visual Studio Code для автоматического преобразования расширений версии 1.0 в новый формат.

## Предварительные требования

1. Выполните первоначальную настройку редактора

   Для простоты это руководство предполагает, что редактор уже настроен для первоначального использования. Если Dynatrace Extensions для VS Code ещё не использовались, следуйте [руководству по началу работы](/managed/ingest-from/extensions/develop-your-extensions/addon-for-vscode/getting-started "Настройте редактор Visual Studio Code и создайте первое расширение, загрузив его в Dynatrace за 5 минут."), чтобы выполнить первоначальную настройку.

   В руководстве предполагается наличие доступа к разработческим учётным данным. Если следовали руководству [по началу работы](/managed/ingest-from/extensions/develop-your-extensions/addon-for-vscode/getting-started "Настройте редактор Visual Studio Code и создайте первое расширение, загрузив его в Dynatrace за 5 минут."), сохраните созданные учётные данные в глобальных настройках VS Code. Пример приведён на странице [Настройки](/managed/ingest-from/extensions/develop-your-extensions/addon-for-vscode/settings#when-using-your-credentials "Сведения о настройках, доступных для конфигурации Dynatrace Extensions").
2. Включите расширения JMX 2.0 в своём окружении

   Откройте **Settings > Preferences > OneAgent features** и включите **Java Metric Extensions (JMX)**, затем перезапустите отслеживаемые процессы Java.

## Преобразование расширения JMX как нового проекта

### Инициализация рабочего пространства

1. Создайте новую папку на компьютере и откройте её в VS Code.
2. Нажмите F1 и выберите команду **Dynatrace extensions: Initialize workspace**
3. Затем выберите из списка версию схемы **1.275.0**
4. При запросе о сертификатах выберите **Use existing**
5. При запросе о типе проекта выберите **JMX 1.0 Conversion**

Рабочее пространство инициализировано, и можно приступать к преобразованию старого расширения.

### Преобразование расширения

1. Необходимо загрузить расширение JMX 1.0 для преобразования. Можно выбрать файл `.zip` или `plugin.json` в локальной файловой системе либо просмотреть подключённое окружение Dynatrace. В рамках этого руководства выберем второй вариант. Выберите загрузку **Remotely**.
2. Отобразится список расширений из тенанта. Выберите расширение, которое требуется преобразовать.

   После выбора расширения может появиться запрос на ввод нового имени расширения, если старое слишком длинное.
3. Затем выберите технологию, чтобы страницы Process можно было настроить автоматически. Выберите подходящую для процесса Java. В противном случае выберите **All other**.
4. При желании данные также можно отображать на странице хоста. Чтобы продолжить работу с этим руководством, выберите **Yes**.

В результате преобразования новый манифест расширения будет создан по пути `extension/extension.yaml`.

## Автономное преобразование расширения JMX

Если инициализировать новое рабочее пространство не требуется или работа уже ведётся в зарегистрированном рабочем пространстве, автоматизацию можно запустить как отдельную команду.

Нажмите F1 и выберите команду **Dynatrace Extensions: Convert JMX**. Этот рабочий процесс начинается с запроса на загрузку расширения и выполняет те же шаги, что описаны выше.

По завершении новый манифест будет помещён в `extension/extension.yaml` либо появится запрос на выбор места сохранения, если данная папка не существует в текущем открытом рабочем пространстве.

## Развёртывание и настройка нового расширения

### Сборка и загрузка в Dynatrace

1. Выполните сборку расширения: нажмите F1 и выберите команду **Dynatrace extensions: Build**.
2. Затем выберите загрузку в Dynatrace, нажав **Yes**.
3. Затем активируйте расширение, нажав **Yes**.

### Добавление конфигурации мониторинга

1. Откройте расширение из запроса или из меню Extensions в Dynatrace.
2. Добавьте конфигурацию мониторинга, нажав кнопку.
3. Выберите хост, на котором выполняется процесс Java, затем нажмите **Next step**.
4. На следующей странице снова нажмите **Next step**, затем добавьте описание и нажмите **Activate**.

После активации конфигурации мониторинга сбор данных начинается автоматически.

## Поиск данных расширения

Откройте страницу сведений одного из хостов, на которых выполняется отслеживаемый процесс Java. Внизу страницы должна отображаться карточка с заголовком, начинающимся с **JMX Metrics**.

В этой карточке выберите любой из перечисленных процессов. Затем нажмите `...` и выберите **Metrics and logs analysis**.

На открывшейся странице отобразятся несколько карточек из вашего расширения.

Карточки на странице Process добавляются только в том случае, если технология была выбрана в процессе преобразования.
