## Что такое расширения?

Расширения — это модульные пакеты, определяющие порядок сбора и структурирования Dynatrace телеметрических данных из внешних источников. Каждое расширение выполняется контроллером выполнения расширений (Extension Execution Controller, EEC), который работает в вашем окружении.

## Как работают расширения?

Расширения позволяют выйти за рамки встроенных возможностей мониторинга Dynatrace. Они позволяют собирать такие телеметрические данные, как метрики, события и логи, из внешних технологий.

С помощью расширений вы можете:

* Принимать данные от технологий, не поддерживаемых нативно.
* Создавать индивидуальную логику мониторинга для вашего окружения.
* Визуализировать и анализировать пользовательские данные.

## Зачем использовать расширения?

Расширения полезны, когда необходимо мониторить технологию или сервис, который Dynatrace не обнаруживает автоматически. Они обеспечивают расширенную настройку и наблюдаемость практически для любого источника данных.

### Ключевые преимущества

* Автоматическое распространение на ActiveGate и OneAgent со встроенным переключением при отказе.
* Полное моделирование топологии, включая пользовательские объекты и сопоставление связей.
* Расширенные метаданные метрик для согласованного контекстно-зависимого мониторинга.
* Возможность создавать пользовательские расширения, адаптированные под ваши потребности:

  + Декларативный, удобочитаемый формат YAML (без программирования для поддерживаемых источников данных).
  + Программируемые расширения с использованием источника данных Python для большей гибкости.

### Варианты использования

* Мониторинг на основе SNMP: мониторинг сетевых устройств.
* Мониторинг на основе SQL: мониторинг SQL-баз данных различных производителей.
* Мониторинг на основе Prometheus: мониторинг экспортёров Prometheus.
* Мониторинг на основе WMI: мониторинг Windows-устройств.
* Мониторинг на основе JMX: получение данных из JMX MBeans.
