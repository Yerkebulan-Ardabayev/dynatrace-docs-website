На этой странице представлены новые функции, изменения и исправления ошибок в Dynatrace Managed версии 1.308.

## По всей платформе

Несовместимое изменение · Platform | Settings

### Изменения в журнале аудита настроек

Скорректирована генерация журнала аудита настроек для улучшения читаемости и снижения количества избыточных записей.

* Обновления через API, не изменяющие значение, больше не будут отображаться в журнале аудита. Это типично для случаев, когда скрипт часто записывает идентичные значения, что могло приводить к большому количеству записей без полезного содержимого.

  Несмотря на то что это функциональное улучшение, технически это является несовместимым изменением.

* Классический веб-интерфейс не затронут этим изменением для большинства существующих схем. Если пользователь повторно сохраняет существующее значение (как правило, изменяет значение, возвращает его обратно и сохраняет), записи `NO_CHANGE` всё ещё могут появляться.

Обновление функции · Cluster

### Обновление Cassandra до версии 4.1

В рамках данного выпуска узлы Cassandra обновляются до версии 4.1.

Вмешательство пользователя или время простоя не требуются; обновление выполняется посредством поэтапного (rolling) обновления в рамках обычных обновлений версии.

Обновление функции · Infrastructure Observability | Hosts

### Проверка стабильности для автоматических обновлений OneAgent

Значения по умолчанию для автоматических обновлений/проверки стабильности снижены до 45 минут для хостов EC2 и не-EC2. Это изменение улучшает процесс автоматических обновлений при использовании окон обслуживания.

Обновление функции · Infrastructure Observability | Hosts

### Управление удалённой конфигурацией (RCM): автоматический перезапуск при изменении тегов и свойств хоста

OneAgent будет автоматически перезапускаться при изменении тегов и свойств хоста через RCM.

Обновление функции · Platform

### Обновление Jetty до версии 10

В рамках данного выпуска встроенная функциональность веб-сервера Jetty обновлена до версии 10.0.24.

Вмешательство пользователя или время простоя не требуются; обновление должно выполняться посредством поэтапного (rolling) обновления в рамках обычных обновлений версии.

В определённых сценариях высокой нагрузки наблюдается незначительное увеличение использования CPU и памяти, вызванное изменениями реализации в Jetty 10. Если экземпляры ActiveGate уже достигли своих ресурсных лимитов, рекомендуется их увеличить во избежание возможной перегрузки.

Обновление функции · Delivery | Cluster

### Обновление установщика Dynatrace Managed для использования Adoptium JRE версий 11.0.25 / 17.0.13

Установщик Dynatrace Managed обновлён для использования Adoptium JRE версии 11.0.25 (для Cassandra) и 17.0.13 (для всех остальных компонентов).

Обновление функции · Platform

### Незначительные изменения в описаниях метрик сети

Внесены незначительные улучшения в описания сетевых метрик, отображаемые в веб-интерфейсе.
