## Улучшённая функция предварительного просмотра для правил тегирования и управленческих зон

При создании правил тегирования или управленческих зон важно проверять затронутые отслеживаемые сущности и, если их много, эффективно их фильтровать. Теперь в настройках реализована улучшённая функция предварительного просмотра с опциями фильтрации по имени сущности, типу, а также датам первого и последнего обнаружения.

Улучшённая функция предварительного просмотра доступна для следующих настроек:

* **Tags** > **Automatically applied tags**
* **Preferences** > **Management zones**
* **Cloud Automation** > **SLO definition**

## Обновления функций

Platform

### Обновление установщика Dynatrace Managed для использования Adoptium Java версий 11.0.30 / 17.0.18 / 21.0.10

Установщик Dynatrace Managed обновлён для использования Adoptium Java версий 11.0.30 (для Cassandra), 17.0.18 (для ActiveGate) и 21.0.10 (для всех остальных компонентов). Вмешательство пользователя или время простоя не требуются; обновление выполняется посредством поэтапного (rolling) обновления в рамках обычного обновления версии.

Platform

### Индикатор прогресса при загрузке пакета установки

Пакеты установки могут быть достаточно большими, поэтому их загрузка вручную в **Cluster Management Console** через страницу настроек **Automatic updates** может занять некоторое время. Теперь отображается индикатор прогресса с оставшимся временем, чтобы вы всегда были информированы о текущем статусе.

## Исправления и обслуживание

### Исправленные ошибки в этом выпуске

* Улучшены правила инструментирования контейнеров для предотвращения случайного инструментирования компонентов платформы OpenShift 4.14+. Это повышает производительность и снижает накладные расходы. (OA-61720)
* Исправлена проблема, при которой несколько потоков могли инициировать избыточные переподключения к хранилищу данных после сброса соединения, что потенциально приводило к излишней нагрузке и нестабильности. (MGD-11060)
* На странице **Process group** метрика JVM **Garbage collection time** теперь корректно отображается как сумма, а не как среднее значение. Однако это изменение может повлиять на существующий базовый уровень метрики. (MGD-10582)
* Улучшена обработка некорректных запросов в Data Explorer для предоставления более полезной обратной связи, когда запросы возвращают слишком большой набор результатов или когда пользователь выполняет слишком много запросов. (MGD-9727)
* Добавлена аннотация `dynatrace.com/split-mounts` в шаблон пода synthetic. Эта аннотация была введена в Dynatrace Operator 1.8.0 для предотвращения конфликтов с образами приложений, которые уже содержат директорию `/var/lib/dynatrace`, обеспечивая корректное внедрение в поды ActiveGate. Добавление этой аннотации в синтетические развёртывания согласует шаблон синтетического пода с поведением Operator по умолчанию для подов, управляемых ActiveGate. (DEM-22172)
* Исправлено поведение, при котором изменения размера страницы не учитывались при воспроизведении RUM-сессий через Session Replay (перемотка рендерера назад во времени). (DEM-21951)
* Исправлена проблема, при которой атрибуты `dt.source_entity` и `loglevel` перезаписывались правилами обработки логов Azure. Теперь эти атрибуты сохраняют свои исходные значения, если они установлены источником. (APPOBS-33744)
