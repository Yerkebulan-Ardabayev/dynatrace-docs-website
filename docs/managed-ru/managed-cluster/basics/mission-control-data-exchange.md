# Обмен данными с Mission Control

# Обмен данными с Mission Control

* Reference
* 5-min read
* Updated on May 08, 2026

Managed Cluster обменивается данными с Mission Control как минимум однократно или периодически. Обмен данными с Mission Control можно настроить через [параметры кластера](/managed/managed-cluster/configuration/configure-cluster-preferences "Configure cluster preferences and privacy settings") в соответствии с потребностями вашей организации и нормативными требованиями по защите данных.

Для обеспечения соответствия нормативным требованиям по защите данных:

* Ознакомьтесь с тем, [какие персональные данные собирает Dynatrace](/managed/manage/data-privacy-and-security/data-privacy/personal-data-captured-by-dynatrace "Find out what types of end-user data may be captured during Dynatrace monitoring and the methods that are available for masking personal end-user data.").
* [Настройте Dynatrace для защиты персональных данных](/managed/manage/data-privacy-and-security/configuration/configure-global-privacy-settings "Learn how to set up data privacy masking for end user IP addresses, geolocations, and user action names.").

## Обмен данными с Mission Control

В следующей таблице задокументированы данные, которыми Managed Cluster обменивается с Mission Control. Всё взаимодействие между Managed Cluster и Mission Control зашифровано и инициируется только кластером. Часть данных можно исключить из взаимодействия, например проактивный доступ Dynatrace к вашим кластерам. Однако некоторые данные являются обязательными для Dynatrace и не могут быть отключены (например, данные лицензии).

| Тип | Периодичность | Запрос Managed Cluster | Ответ Mission Control | Отказ |

| Установка | Однократно при установке; при каждом обновлении | Лицензионный ключ | Имя лицензии, имя учётной записи, URL загрузки OneAgent, флаг установки OneAgent, флаг проверки OneAgent | Нет |
| Регистрация | Однократно после первого запуска | Идентификатор кластера, лицензионный ключ | Статус регистрации, имя пользователя, пароль | Нет |
| Проверка лицензии | Каждые 5 минут | Идентификатор кластера | Идентификатор кластера, лицензионный ключ, имя лицензии, тип лицензии, статус лицензии, сведения о лицензии, имя учётной записи, дата создания лицензии, дата истечения лицензии, лимиты лицензионных единиц, лимиты превышения лицензионных единиц, модель ценообразования, флаг удалённого доступа, флаг тестовой среды, код страны, категории отслеживаемых хостов | Нет |
| Данные лицензии | Каждый час | Идентификатор кластера, начало временного периода, конец временного периода, данные лицензии *сред*; для каждой *среды* см. [Экспорт данных лицензии](/managed/managed-cluster/operation/export-license-data "Learn how to export license data from the Cluster Management Console.") | Статус лицензии, лицензионный ключ, использование лицензионных единиц, превышение лицензионных единиц, оставшиеся лицензионные единицы, данные лицензии *сред*; для каждой *среды*: UUID среды, использование лицензионных единиц, превышение лицензионных единиц, оставшиеся лицензионные единицы | Нет |
| Проверка работоспособности | Каждые 2 минуты | Идентификатор кластера, версия кластера, флаг автообновления, флаг отчётности о событиях, флаг удалённого доступа, часовой пояс, объём трафика, email-адрес администратора, *узлы кластера*, *Cluster ActiveGate*, окна обслуживания; для каждого *узла кластера*: идентификатор узла, IP-адрес, состояние узла, состояние сервера, временная метка запуска, имя ОС, версия ОС, информация JVM, статистика CPU, статистика памяти, статистика хранилища, версия Cassandra, файлы данных Cassandra, разделы Cassandra, тип сертификата, пользовательские настройки; для каждого *Cluster ActiveGate*: идентификатор, IP-адрес, тип, состояние, имя ОС, версия ОС, тип сертификата | Статус работоспособности, описание статуса, минимальное количество узлов, минимальное количество ядер CPU, минимальный объём памяти | Нет |
| Heartbeat | 20 запросов каждые 10 минут; 12 запросов каждые 10 минут при отключённом удалённом доступе | Идентификатор кластера, идентификатор узла, тип источника | Флаг удалённого доступа, URL WebSocket | Нет |
| Синхронизация сред | Разница синхронизации каждые 10 минут; полная синхронизация раз в день | UUID среды, имя среды | — | Нет |
| Обновления | Каждый час | — | Метка пакета установки, тип, версия, минимально требуемая версия, *файлы*; для каждого *файла*: тип, размер, URL загрузки | Нет |
| Dynatrace Hub | Каждые 30–60 минут | — | Изображения и текстовые описания возможностей и расширений, доступных в Dynatrace Hub | Нет |
| [Потоки уязвимостей Application Security](/managed/secure/application-security#vulnerability-feeds "Access the Dynatrace Application Security functionalities.") | При появлении обновлений | Данные потоков уязвимостей | Данные потоков уязвимостей | Нет |

## Отслеживаемые технологии и использование продукта

Dynatrace может получать информацию об отслеживаемых технологиях и использовании продукта в Managed Cluster, включая перечисленные ниже объекты, чтобы информировать вас о несовместимостях и технологических рисках, связанных с вашим кластером.

Компании Dynatrace (Dynatrace LLC и дочерние компании) [работают по всему миру](https://www.dynatrace.com/company/locations). Данные могут передаваться и обрабатываться в облачных сервисах в США и других регионах. Подробности см. в разделе [Конфиденциальность и безопасность данных](/managed/manage/data-privacy-and-security "Learn how Dynatrace applies various security measures required to protect private data.").

Соответствующие журналы доступны на каждом узле Managed Cluster по пути `<datastore_dir>/log/server/audit.rest.proxy.log`. Dynatrace не отправляет имена хостов и никакую информацию, которая могла бы поставить под угрозу безопасность вашего кластера.

| Тип | Данные |
| --- | --- |
| ActiveGate | Тип ActiveGate, версия сборки, операционная система, поддерживаемые возможности, статус обновления |
| Application configs | Конфигурации на уровне приложений, охватывающие контроль затрат, ключевые действия пользователей и синтетические проверки |
| Dashboard configs | Конфигурации на уровне панелей управления, включая базовые настройки общего доступа и используемые типы плиток |
| Digital Experience | Пользовательские сеансы Real User Monitoring, включая мобильные сеансы и использование Session Replay, а также диагностические метрики |
| Distributed traces | Диагностические данные по распределённым трассировкам и хранилищу данных на уровне кода |
| Dynatrace API | Агрегированная статистика по конечным точкам Dynatrace API, включая количество вызовов и время ответа |
| JavaScript framework | Агрегированная статистика по JavaScript-фреймворкам, обнаруженным в ходе мониторинга |
| Log Monitoring | Агрегированная статистика по использованию журналов и индексам |
| Monitored entities | Агрегированная статистика по отслеживаемым процессам, хостам, сервисам, приложениям и связанным объектам |
| Network zones | Агрегированная статистика по сетевым зонам на уровне кластера и среды, включая количество OneAgent и ActiveGate |
| Requests | Агрегированная статистика по ключевым запросам и атрибутам запросов |
| RUM | Версия RUM JavaScript для мобильного мониторинга и метаданные об отслеживаемых версиях ОС и сбоях агента |
| OneAgent | Подключённые OneAgent с версией установщика, операционной системой, информацией об инжекции, технологией процессов и хостов, метриками памяти хоста |
| Extension | Активные расширения с типом, версией, размером, статусом, метриками расширения и информацией о связи с хостами |
| Problem detection | Проблемы со статусом, серьёзностью, уровнем воздействия, идентификаторами затронутых объектов и сведениями о событиях |
| Security | Метаданные технических проблем безопасности и уязвимостей |
| Synthetic monitors | Активные конфигурации синтетических мониторов с типом монитора, идентификатором и именем местоположения (для пользовательских местоположений имя не отправляется) |
| Tags | Агрегированная статистика по количеству тегов в разбивке по типу тега и типу объекта |

| Параметр | Данные |
| --- | --- |
| Alerting profile | Конфигурации профилей оповещений |
| API token permissions | Агрегированная статистика по токенам API в разбивке по уровню разрешений |
| Conditional procedures | Агрегированная статистика по правилам для зон управления и автоматически применяемых тегов |
| Deep monitoring | Настройки Early Access и функций диагностики OneAgent |
| Environmental configs | Агрегированная статистика по настройкам предпочтений и расширенным конфигурациям среды |
| Feature flags | Настройки функций среды, отличающиеся от значений по умолчанию |
| Integrations | Настройки интеграции, включая уведомления о проблемах, Dynatrace API, модули Dynatrace, экспорт пользовательских сеансов и интеграции с системами отслеживания задач |
| Maintenance windows | Агрегированная статистика по настроенным окнам обслуживания и их параметрам: тип обслуживания, длина текста, расписание, фильтры |
| Metric events | Агрегированная статистика по количеству конфигураций, типам моделей и фильтров |
| Metrics | Метаданные метрик среды, настроенные типы данных метрик, статистика использования метрик |
| Preferences | Настройки обновлений OneAgent и конфиденциальности данных |
| Service Level Objectives | Агрегированная статистика по использованию Service Level Objectives |
| Tile filters | Агрегированная статистика по фильтрам плиток для оптимизации масштабируемости |
| Virtualization | Настроенные типы виртуализации: AWS, Azure, VMware |
