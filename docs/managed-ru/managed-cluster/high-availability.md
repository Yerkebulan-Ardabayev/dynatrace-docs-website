# Высокая доступность

# Высокая доступность

* 1 мин на чтение
* Обновлено 07 июля 2026 г.

Dynatrace Managed высокая доступность защищает от разных масштабов сбоев. Единственный Managed Cluster переносит потерю отдельных узлов. Premium High Availability (PHA) добавляет второй дата-центр, поэтому мониторинг продолжается, даже если целый дата-центр выходит из строя. Поверх любой из этих топологий развёртывание с учётом стоек группирует узлы в домены отказа, поэтому Managed Cluster переживает также отключение целой стойки. Сначала выбирается базовая топология, единственный Managed Cluster либо PHA, затем при необходимости добавляется развёртывание с учётом стоек. Следующие страницы помогают сравнить эти варианты, добавить дата-центр, преобразовать существующий Managed Cluster и восстановиться после сбоя.

## Premium High Availability

[### Высокая доступность с несколькими дата-центрами

Понять, как Premium High Availability (PHA) использует избыточность, размещение оборудования, планирование ёмкости, репликацию данных и автоматическое переключение при отказе между дата-центрами.](/managed/managed-cluster/high-availability/multi-data-centers "Понять, как Dynatrace Managed Premium High Availability обеспечивает переключение при отказе, устойчивость данных и маршрутизацию данных между дата-центрами.")[### Добавление дата-центра

Реплицировать узлы Managed Cluster между двумя дата-центрами и подготовить Cassandra, Elasticsearch и компоненты сервера для PHA.](/managed/managed-cluster/high-availability/add-data-center "Узнать, как реплицировать узлы Managed Cluster между двумя дата-центрами для настройки развёртывания Premium High Availability с репликацией между дата-центрами.")[### Переключение при отказе между дата-центрами

Рассмотреть, как PHA обнаруживает отключения узлов Elasticsearch и Cassandra, направляет решения о переключении при отказе и восстанавливает сервис после восстановления.](/managed/managed-cluster/high-availability/failover "Узнать, как механизм переключения при отказе между дата-центрами Premium High Availability обнаруживает отключения узлов и передаёт ответственность исправному дата-центру.")[### Восстановление дата-центра из другого дата-центра

Восстановить дата-центр в развёртывании PHA, используя другой дата-центр в качестве источника восстановления.](/managed/managed-cluster/high-availability/recover-from-data-center "Восстановить дата-центр, если Premium High Availability не может исправить его в течение 72 часов, путём восстановления или пересоздания из другого дата-центра.")[### Восстановление дата-центра из резервной копии

Восстановить утраченный дата-центр из резервной копии и восстановить топологию дата-центра в развёртывании с несколькими дата-центрами.](/managed/managed-cluster/high-availability/recover-from-backup "Узнать, как восстановить дата-центр Premium High Availability из резервной копии в развёртывании Dynatrace Managed с несколькими дата-центрами после потери дата-центра.")[### Восстановление дата-центра

Восстановить утраченный дата-центр, переустановить узлы, перенести хранилища данных и восстановить репликацию между обоими дата-центрами.](/managed/managed-cluster/high-availability/rebuild-data-center "Узнать, как восстановить утраченный дата-центр в развёртывании Dynatrace Managed Premium High Availability и восстановить репликацию между обоими дата-центрами.")
