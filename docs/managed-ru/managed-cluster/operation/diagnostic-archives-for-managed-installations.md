# Диагностические архивы для установок Dynatrace Managed

# Диагностические архивы для установок Dynatrace Managed

* Published Sep 05, 2018

При выполнении диагностики кластера Dynatrace Managed доступны два варианта анализа:

* **Автоматический анализ:** Dynatrace Managed собирает диагностические данные кластера и автоматически отправляет их в Dynatrace, чтобы эксперты Dynatrace могли быстро и эффективно проанализировать ваш запрос в службу поддержки.
* **Локальный сбор и проверка:** Dynatrace Managed собирает диагностические данные кластера, после чего вы можете самостоятельно проанализировать данные и при необходимости приложить их к запросу в службу поддержки для проверки экспертами Dynatrace.

## Автоматический анализ

По умолчанию Dynatrace собирает диагностические данные кластера и немедленно их анализирует.

Если вы предпочитаете собрать и проверить данные перед их ручной отправкой в Dynatrace для анализа, см. раздел [Локальный сбор и проверка](#cluster-collect-and-review-locally).

1. На странице **Home** Dynatrace Managed выберите **More** (**…**) > **Run Cluster diagnostics**.

   Пример выбора пункта меню

   ![Cluster Management Console home page displaying Run Cluster diagnostics button.](https://dt-cdn.net/images/man-diag-enter-1401-b9076df6dc.png)

   Cluster Management Console home page displaying Run Cluster diagnostics button.
2. На странице **Run Cluster diagnostics** кратко опишите, что, на ваш взгляд, работает не так, как ожидалось.

   Пример страницы диагностики кластера

   ![Cluster diagnostics page.](https://dt-cdn.net/images/cluster-diags-1545-9e20e56075.png)

   Cluster diagnostics page.
3. (Необязательно) По умолчанию для анализа собираются данные за 7 дней. Если требуется больше данных, выберите ссылку **Advanced options**, измените количество дней и выберите **Apply**.
4. Выберите **Start analysis**.

### Что происходит дальше

Dynatrace выполняет следующие действия:

* Собирает диагностические данные за последние 7 дней (если значение по умолчанию не изменено) для затронутых узлов кластера и ActiveGate
* Сохраняет собранные диагностические данные
* Загружает диагностические данные в региональное хранилище Mission Control для дальнейшего анализа

Столбец **State** описывает текущий этап процесса.

**State** не обновляется автоматически. Выберите **Refresh** для проверки изменения состояния.

Collecting

Сбор данных выполняется.
В процессе сбора данных можно:

* **Refresh** — обновить страницу для проверки прогресса.
* **Cancel** — отменить сбор диагностических данных.

Collected

Dynatrace завершил сбор диагностических данных.
После сбора данных можно:

* **Analyze** — отправить собранные данные в Dynatrace для анализа.
* **Download** — загрузить собранные данные локально для самостоятельного изучения.
* **Delete** — удалить проблему, включая собранные диагностические данные.

Sending in progress

Диагностические данные передаются в Dynatrace для анализа.
В процессе отправки данных можно:

* **Refresh** — обновить страницу для проверки прогресса.
* **Download** — загрузить собранные диагностические данные.
* **Delete** — удалить проблему, включая собранные диагностические данные.

Sent to Dynatrace cloud

Диагностические данные переданы в Dynatrace для анализа.

Analyzing

Dynatrace анализирует диагностические данные.  
В процессе анализа данных можно:

* **Refresh** — обновить страницу для проверки прогресса.
* **Download** — загрузить собранные диагностические данные.
* **Delete** — удалить проблему, включая собранные диагностические данные.

Analyzed

Анализ завершён. Количество связанных оповещений указано в скобках.  
После анализа можно:

* **Download** — загрузить собранные диагностические данные.
* **Delete** — удалить проблему, включая собранные диагностические данные.

Delete in progress

Диагностические данные удаляются. В процессе удаления данных можно:

* **Refresh** — обновить страницу для проверки прогресса.

Deleted

Диагностические данные удалены. Dynatrace сохраняет только небольшой набор сведений о том, кто, когда, где и по какой причине собирал диагностические данные.

Canceled

Процесс диагностики был отменён вручную до завершения.

### Просмотр результатов анализа Dynatrace

По завершении анализа Dynatrace отправляет результаты обратно в ваш кластер. Если было выявлено потенциальное решение, Dynatrace перечисляет его в разделе **Alerts**.

## Локальный сбор и проверка

В данной процедуре описывается сбор диагностических данных для хоста или процесса локально. Используйте этот вариант, если предпочитаете собрать и проверить данные перед их ручной отправкой в Dynatrace для анализа.

Если вы хотите собрать данные и отправить их в Dynatrace автоматически для анализа, см. раздел [Автоматический анализ](#cluster-analyze-automatically).

1. На странице **Home** Dynatrace Managed выберите **More** (**…**) > **Run Cluster diagnostics**.

   Пример выбора пункта меню

   ![Cluster Management Console home page displaying Run Cluster diagnostics button.](https://dt-cdn.net/images/man-diag-enter-1401-b9076df6dc.png)

   Cluster Management Console home page displaying Run Cluster diagnostics button.
2. На странице **Run Cluster diagnostics** кратко опишите, что, на ваш взгляд, работает не так, как ожидалось.

   Пример страницы диагностики кластера

   ![Cluster diagnostics page.](https://dt-cdn.net/images/cluster-diags-1545-9e20e56075.png)

   Cluster diagnostics page.
3. Выберите ссылку **Advanced options**.
4. Выберите **and store locally**.

   * Здесь также можно изменить количество дней сбора данных (по умолчанию — `7 дней`).
5. Выберите **Apply**.
6. Выберите **Start collection** для сбора диагностических данных и их локального сохранения.

### Что происходит дальше

Dynatrace выполняет следующие действия:

* Собирает диагностические данные за последние 7 дней (если значение по умолчанию не изменено) для затронутых узлов кластера и ActiveGate
* Сохраняет собранные диагностические данные

Столбец **State** описывает текущий этап процесса.

**State** не обновляется автоматически. Выберите **Refresh** для проверки изменения состояния.

Collecting

Сбор данных выполняется.
В процессе сбора данных можно:

* **Refresh** — обновить страницу для проверки прогресса.
* **Cancel** — отменить сбор диагностических данных.

Collected

Dynatrace завершил сбор диагностических данных.
После сбора данных можно:

* **Analyze** — отправить собранные данные в Dynatrace для анализа.
* **Download** — загрузить собранные данные локально для самостоятельного изучения.
* **Delete** — удалить проблему, включая собранные диагностические данные.

### Что делать с собранными данными

После сбора данных можно:

* **Загрузить** собранные данные.

  + Проанализировать данные самостоятельно. Краткое описание содержимого загрузки см. в разделе [Содержимое диагностических данных](#cluster-contents).
  + Приложить данные к запросу в службу поддержки.
* **Проанализировать** данные.
* **Удалить** проблему, включая собранные диагностические данные.

## Обзор диагностики кластера

Для просмотра обзора всех сеансов диагностики кластера в вашей среде:

1. На странице **Home** Dynatrace Managed выберите **More** (**…**) > **Run Cluster diagnostics**.
2. На странице **Run Cluster diagnostics** отображается список всей диагностической активности в вашем кластере.

   * Разверните любую запись для просмотра сведений.
   * Если данные не были удалены (все диагностические данные удаляются автоматически через 30 дней), их можно загрузить или удалить.

## Содержимое диагностических данных

Все собранные диагностические данные сжимаются в ZIP-файл `SupportArchive<ID number>`, включающий следующие папки и файлы:

* `COLLECTOR` (ActiveGate)  
  Содержит список всех **ActiveGate** кластера. Диагностические данные расположены в отдельной папке для каждого ActiveGate. Папки ActiveGate названы по UUID ActiveGate. Каждая папка ActiveGate содержит следующие подпапки:

  | Папка | Описание |
  | --- | --- |
  | `autoupdater` | Содержит журналы процесса автообновления. |
  | `config` | Содержит снимок [директории конфигурации](/managed/ingest-from/dynatrace-activegate/configuration/where-can-i-find-activegate-files "Find out where ActiveGate files are stored on Windows and Linux systems.") ActiveGate. |
  | `debugui` | Содержит снимок внутренней конфигурации среды, связанной с ActiveGate. |
  | `install` | Содержит журналы установки ActiveGate. |
  | `log` | Содержит снимок [директории журналов](/managed/ingest-from/dynatrace-activegate/configuration/where-can-i-find-activegate-files "Find out where ActiveGate files are stored on Windows and Linux systems.") ActiveGate. |
* `SERVER` (узлы)  
  Содержит список всех **узлов** кластера. Диагностические данные расположены в отдельной папке для каждого узла. Папки узлов названы по уникальному номеру узла. Каждая папка узла содержит следующие подпапки:

  | Папка | Описание |
  | --- | --- |
  | `autoupdater` | Содержит журналы процесса автообновления. |
  | `config` | Содержит конфигурационные файлы компонентов узлов кластера. |
  | `log` | Содержит файлы журналов, включая журналы аудита серверного процесса. |
  | `managed-logs` | Содержит файлы журналов других компонентов узла кластера, сгруппированные по отдельным папкам. |
  | `nginx-error-logs` | Содержит журналы ошибок NGINX-сервера. |
* `download details.txt` (TXT)  
  Содержит информацию и статистику о диагностическом архиве, включая количество файлов и размер каждой папки.

## Конфиденциальность данных

Для соблюдения региональных нормативных актов о защите данных и конфиденциальности Dynatrace выполняет следующее:

* Автоматически маскирует часть персональных данных (например, IBAN и учётные данные URI) перед их сохранением.
* Автоматически удаляет все диагностические данные через 30 дней после их сбора. Это применяется к данным в вашей среде Dynatrace и на кластере Dynatrace.

Вы можете удалить собранные диагностические данные раньше. Для обеспечения прозрачности Dynatrace сохраняет лишь небольшой набор сведений о том, кто, когда, где и по какой причине собирал диагностические данные.

Дополнительные сведения о конфиденциальности данных Dynatrace см. в следующих разделах:

* [Персональные данные, собираемые Dynatrace](/managed/manage/data-privacy-and-security/data-privacy/personal-data-captured-by-dynatrace#oneagent-diagnostics "Find out what types of end-user data may be captured during Dynatrace monitoring and the methods that are available for masking personal end-user data.")
* [Сроки хранения данных](/managed/manage/data-privacy-and-security/data-privacy/data-retention-periods#diagnostics "Check retention times for various data types.")

## Устранение неполадок

Статус: `Collecting of the diagnostic data wasn't possible within 20 minutes.`

* Если Dynatrace не может собрать диагностические данные в течение 20 минут, он автоматически предпринимает повторную попытку.
* Если повторная попытка также завершается неудачей, рекомендуется обратиться к эксперту Dynatrace через онлайн-чат.

`State` выглядит зависшим.

**State** не обновляется автоматически. Выберите **Refresh** для проверки изменения состояния.
