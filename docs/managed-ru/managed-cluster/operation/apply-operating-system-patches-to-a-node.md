# Применение патчей операционной системы к узлу

* Updated on Oct 17, 2025

Рекомендуется выполнять обновление операционной системы узла или применение патчей по одному узлу за раз. Программное обеспечение узла Dynatrace Managed состоит из ряда взаимозависимых служб Dynatrace, которые необходимо останавливать и запускать в определённом порядке. Для корректного запуска, остановки или перезапуска служб Dynatrace используйте официальную команду `dynatrace.sh` с соответствующим параметром.

Для применения патча операционной системы к узлу Dynatrace:

1. Войдите на Linux-машину, где установлен узел Dynatrace Managed, с правами пользователя root.
2. Выполните команду `dynatrace.sh` с соответствующим параметром (`start`, `stop` или `restart`).  
   По умолчанию скрипт расположен в директории `/opt/dynatrace-managed/launcher`.

   ```
   [root@localhost]# ./dynatrace.sh stop
   ```
3. Примените все необходимые патчи к операционной системе узла.

   Перезагрузка системы

   Некоторые патчи операционной системы могут потребовать перезагрузки системы. После перезагрузки службы Dynatrace возобновятся автоматически, если только они не были явно остановлены. Если планируется установка дополнительных патчей после перезагрузки, необходимо снова остановить службы Dynatrace перед продолжением.  
   Несоблюдение этого требования может привести к непоследовательному приёму данных наблюдаемости узлом, что вызовет пробелы в данных и ошибочные оповещения. Всегда убеждайтесь, что Dynatrace полностью остановлен перед применением изменений на уровне операционной системы к узлам кластера.
4. Если перезагрузки системы не было, перезапустите службы Dynatrace.

   ```
   [root@localhost]# ./dynatrace.sh start
   ```
