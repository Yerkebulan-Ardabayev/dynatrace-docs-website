# Remove a cluster node

# Remove a cluster node

* Updated on May 24, 2024

Удалить узел кластера можно с помощью Cluster Management Console для Dynatrace Managed. Поскольку удаление узла фактически означает деинсталляцию Dynatrace Managed, в качестве альтернативы можно воспользоваться командной строкой. Оба подхода описаны ниже.

## Удаление узла кластера через Cluster Management Console

Recommended

Прежде чем удалить узел из кластера Dynatrace Managed, необходимо сначала отключить на нём трафик OneAgent, чтобы предотвратить поступление данных на узел в процессе удаления.

### Отключение трафика OneAgent

Чтобы отключить узел кластера

1. Войдите в систему с правами администратора. Используйте URL Dynatrace Managed или узел кластера, который вы хотите сохранить (нельзя удалить узел, находясь в системе с его адреса).
2. Перейдите в раздел **Deployment Status**.
3. Выберите **Cluster nodes** и в списке узлов выберите узел, на котором хотите отключить трафик OneAgent.
4. Нажмите **Configure** для просмотра страницы конфигурации узла.
5. Нажмите кнопку меню (**…**) в правом верхнем углу.
6. Нажмите **Disable OneAgent traffic**, чтобы прекратить обработку данных мониторинга на узле. Если необходимо временно исключить узел из кластера, остановитесь на этом шаге и включите узел позже. Dynatrace Managed продолжит работу на машине, но не будет обрабатывать данные мониторинга.

### Удаление отключённого узла кластера

Перед удалением узла рекомендуется дождаться истечения срока хранения данных хранилища транзакций (до 35 дней). Это объясняется тем, что, хотя все метрики реплицируются в кластере, необработанные данные транзакций не хранятся в избыточном режиме. Удаление узла до истечения срока хранения данных хранилища транзакций может повлиять на анализ данных уровня кода и пользовательских сессий.

Удаляйте не более одного узла за раз. Во избежание потери данных выждите 24 часа перед удалением каждого последующего узла. Это связано с тем, что для автоматического перераспределения реплик долгосрочных метрик на оставшихся узлах требуется до 24 часов.

Если резервное копирование кластера включено и вы удаляете узел, файлы резервной копии этого узла остаются в вашем хранилище резервных копий. Чтобы удалить резервную копию именно удалённого узла, удалите папку `<path-to-backup>/<UUID>/node_<node_id>`.

Пример удаления резервной копии конкретного узла

В данном примере пути:

```
rmdir -r /mnt/backup/bckp/c9dd47f0-87d7-445e-bbeb-26429fac06c6/node_3/
```

части пути имеют следующий смысл:

* `/mnt/backup/bckp/` = `<path-to-backup>`
* `c9dd47f0-87d7-445e-bbeb-26429fac06c6` = `<UUID>`
* `3` = `<node_id>`

После удаления существующего узла кластера необходимо выждать 72 часа перед установкой нового узла на хост с тем же IP-адресом (при переустановке на тот же хост).

Чтобы удалить отключённый узел кластера

1. Войдите в систему с правами администратора. Используйте URL Dynatrace Managed или узел кластера, который вы хотите сохранить (нельзя удалить узел, находясь в системе с его адреса).
2. Перейдите на **Home** для навигации на страницу узлов кластера Dynatrace Managed.
3. Выберите узел кластера, который хотите удалить, чтобы открыть страницу обзора узла.
4. Нажмите кнопку меню (**…**) в правом верхнем углу.
5. Нажмите **Remove node**. Узел будет остановлен и полностью деинсталлирован с вашего сервера.

Dead nodes

Dynatrace Managed отображает удалённые и недоступные узлы в течение 7 дней. Однако даже после того, как недоступный узел перестаёт отображаться на странице кластеров, он по-прежнему остаётся зарегистрированным. Для полного удаления узла воспользуйтесь вызовом Dynatrace API. Выполните вызов Cluster Management API к следующей конечной точке: `onpremise/nodeManagement/deadNodeCleaning`.

Подробности см. в Dynatrace API explorer в CMC.

1. В правом верхнем углу CMC откройте пользовательское меню и выберите **Cluster Management API**.
2. Разверните раздел **Cluster node** и выполните **DELETE** для `/nodeManagement/deadNodeCleaning`.

## Удаление узла кластера через REST API

Recommended

Рекомендуется удалять узел кластера через Cluster Management Console. Однако существует и альтернативный вариант — через публичный API: `/api/v1.0/onpremise/nodeManagement/triggerRemoveNode`.

Как удалить узлы, если кластер недоступен

Если рекомендуемые методы удаления узлов кластера ([Cluster Management Console](#remove-node-cmc) и [вызов API](#remove-node-api)) недостаточны и проблемы сохраняются, воспользуйтесь следующей командой:

1. Войдите на Linux-машину, где установлен Dynatrace Managed.
2. Перейдите в основную директорию `PRODUCT_PATH` с бинарными файлами Dynatrace Managed (по умолчанию: `/opt/dynatrace-managed`).
3. С правами root выполните следующую команду:

   `/bin/sh uninstall-dynatrace.sh`

После успешного выполнения скрипта все данные Dynatrace Managed будут удалены с машины.
