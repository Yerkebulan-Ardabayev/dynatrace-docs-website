# Reconfigure the IP address of a cluster node

* Updated on Nov 03, 2025

Cluster version 1.320+

По мере развития инфраструктуры Dynatrace Managed стремится адаптировать развёртывание к динамической природе облачных центров обработки данных.

Dynatrace Managed использует статические IP-адреса, однако функция перенастройки IP позволяет динамически изменять IP-адрес узла без необходимости удалять узел и устанавливать новый.

Для успешной перенастройки IP-адреса в кластере **должно быть не менее трёх узлов**.

Если старый и новый IP-адреса назначены одной и той же машине, убедитесь, что таблица маршрутизации настроена таким образом, что весь исходящий трафик к другим узлам кластера направляется исключительно через новый IP-адрес.

Перенастройка IP-адреса также может служить альтернативой:

* [Миграции кластера](/managed/managed-cluster/operation/migrate-a-cluster "Learn how to migrate a cluster from one data center to another.")
* [Применению патчей операционной системы на узле](/managed/managed-cluster/operation/apply-operating-system-patches-to-a-node "Find out how to perform a graceful server shut down and apply OS patches to a node.")

## Шаги

### Генерация токена

1. Перейдите в Cluster Management Console (CMC) > **Settings** > **API tokens**.
2. Прокрутите вниз и выберите **Cluster tokens**.
3. Нажмите **Generate token** и введите **Token name**.
4. Включите переключатель **Service Provider API**.
5. Скопируйте токен и сохраните его в надёжном месте, так как повторный доступ к нему будет невозможен.

### Проверка данных узла

Убедитесь, что все данные узла доступны на узле, который вы хотите перенастроить. Это можно сделать:

* Скопировав данные на узел.
* Подключив диски.

### Запуск скрипта

Запустите приведённый ниже скрипт на узле, который вы хотите перенастроить:

```
/opt/dynatrace-managed/installer/migrate-ip-local.sh --seed-auth <cluster-token>
```

Выполнение скрипта занимает некоторое время — до одного часа. После завершения узел должен отображаться в кластере под новым IP-адресом.

Кроме того, администратор кластера получит уведомление о выполнении перенастройки IP в виде события кластера.

### Часто задаваемые вопросы

Где находится журнал процесса перенастройки?

Основной журнал находится в `/var/log/dynatrace/install.log` на перенастраиваемой машине. В этом процессе участвуют многие компоненты на всех узлах, поэтому подробности каждого шага можно найти в соответствующих журналах.

Можно ли запускать скрипт несколько раз?

Да, скрипт можно запускать несколько раз, но не одновременно.

Что делать, если имеется несколько сетевых интерфейсов?

Необходимо указать IP-адрес, который вы хотите использовать для Dynatrace, с помощью параметра `--cluster-ip <ip-for-dynatrace>`.

Можно ли изменить IP-адрес нескольких узлов одновременно?

Нет, изменять IP-адрес можно только у одного узла за раз.

Что делать, если какой-либо из шагов завершится с ошибкой?

Проверьте журнал и устраните проблему. После этого можно запустить скрипт повторно.
