# Кастомизация установки Managed Cluster

# Кастомизация установки Managed Cluster

* Reference
* 9-min read
* Updated on May 08, 2026

Используйте параметры командной строки для переопределения настроек по умолчанию при установке Managed Cluster. По умолчанию никакие параметры не являются обязательными — установщик Managed работает в интерактивном режиме, предлагая подтвердить или переопределить настройки по умолчанию. Значения, введённые интерактивно, всегда имеют приоритет над значениями параметров командной строки.

При запуске установщика в интерактивном режиме с параметрами командной строки значения параметров отображаются в подсказках вместо значений по умолчанию. Любые введённые в этот момент значения имеют приоритет.

Для установки Managed Cluster в неинтерактивном режиме с настройками по умолчанию используйте `--install-silent` совместно с `--license` для передачи лицензионного ключа.

## Настройки по умолчанию

* Путь установки (бинарные файлы): `/opt/dynatrace-managed`
* Файлы данных Dynatrace Cluster: `/var/opt/dynatrace-managed`
* Системный пользователь, запускающий процессы Dynatrace: `dynatrace`
* Системная группа, запускающая процессы Dynatrace: `dynatrace`

Для вывода списка всех доступных параметров командной строки запустите установщик с `--help` от имени root:

```
[root@localhost]# ./dynatrace-managed-installer.sh --help
```

## Параметры начальной настройки

Используйте эти параметры для начальной настройки сервера. При их указании установщик после успешной установки и регистрации кластера генерирует токен аутентификации для публичного REST API.

| Параметр | Описание |
| --- | --- |
| `--initial-environment <name>` | Имя первой среды мониторинга. |
| `--initial-first-name <name>` | Имя администратора. |
| `--initial-last-name <name>` | Фамилия администратора. |
| `--initial-email <email>` | Адрес электронной почты администратора. |
| `--initial-pass <pass>` | Пароль администратора. |

## Параметры режима установки

| Параметр | Описание |
| --- | --- |
| `--install` | Стандартная интерактивная установка. |
| `--install-silent` | Устанавливает с настройками по умолчанию в неинтерактивном режиме. Используйте другие параметры командной строки для переопределения отдельных настроек по умолчанию. Полезно для автоматизации установки. |
| `--install-new-dc-silent` | Только для Premium High Availability. Устанавливает в новом центре обработки данных в неинтерактивном режиме. Используйте другие параметры командной строки для переопределения отдельных настроек по умолчанию. |
| `--upgrade` | Запускает процесс обновления из командной строки. |
| `--system-check` | Проверяет, соответствует ли машина и операционная система требованиям новой версии. |
| `--restore` | Восстанавливает установку Dynatrace Managed из файла резервной копии, указанного через `--backup-file`. Резервное копирование выполняется ежедневно после настройки в Dynatrace Managed. |
| `--reconfigure` | Перенастраивает существующую установку с использованием параметров установки. |
| `--uninstall` | Удаляет Dynatrace Managed. |
| `--self-check` | Проверяет целостность файла установщика. |
| `--backup-file <path>` | Полный путь к файлу резервной копии, содержащей конфигурацию и данные. Используется совместно с `--restore`. |
| `--timeouts <component:value, ..>` | Увеличивает тайм-ауты для указанных компонентов. Используйте в следующих случаях:  * Конкретному компоненту требуется увеличенный тайм-аут (например, запуск фаервола ОС занимает несколько минут). * У вас много сред и требуется больше времени для запуска Server. |
| `--drop-x-forwarded-for` | Dynatrace использует заголовок `X-Forwarded-For` для таких функций, как ведение журнала. Обрабатывайте эти заголовки в соответствии с передовыми практиками безопасности — неправильная обработка может привести к отображению и записи некорректных данных, если злоумышленник добавит поддельные заголовки `X-Forwarded-For` в запросы. |

Конфигурация прокси

Крайний внешний обратный прокси, через который возможен доступ к Dynatrace, должен быть настроен следующим образом.

* Все запросы, направленные к Dynatrace, очищаются от всех заголовков X-Forwarded-For.
* Следующие пути запросов должны быть исключены из этого поведения. Разрешите запросы к этим путям сохранять заголовки X-Forwarded-For, поскольку соответствующие функции требуют их.

  + `/bf`
  + `/mbeacon`
  + `/communication`

Nginx-прокси, установленный вместе с Dynatrace Managed, следует этим правилам по умолчанию. Однако вы можете отключить это поведение во время установки, если Nginx не является крайним внешним прокси, через который клиенты получают доступ к Dynatrace. В этом случае необходимо настроить крайний внешний прокси вручную.

Следующие компоненты можно настроить с помощью `--timeouts`:

| Компонент | Описание |
| --- | --- |
| `proc` | Базовый тайм-аут в секундах. Это значение прибавляется к специфическим для компонентов тайм-аутам ниже для Nodekeeper, Cassandra, Elasticsearch, Server, ActiveGate и NGINX. |
| `ndk` | Тайм-аут запуска Nodekeeper в секундах = `proc` + `ndk` |
| `cas` | Тайм-аут запуска Cassandra в секундах = `proc` + `cas` |
| `els` | Тайм-аут запуска Elasticsearch в секундах = `proc` + `els` |
| `srv` | Тайм-аут запуска Server в секундах = `proc` + `svr` |
| `ag` | Тайм-аут запуска ActiveGate в секундах = `proc` + `ag` |
| `ngx` | Тайм-аут запуска NGINX в секундах = `proc` + `ngx` |
| `fw` | Тайм-аут запуска фаервола в секундах = `fw` |

## Параметры установки

Вы можете изменить путь установки и другие параметры с помощью следующих настроек:

### Бинарные файлы

| Параметр | Описание |
| --- | --- |
| `--binaries-dir <path>` | Полный путь к директории бинарных файлов Dynatrace. При использовании этого параметра с включённым SELinux требуется наличие утилиты semanage в системе. **Значение по умолчанию**: `/opt/dynatrace-managed` |

### Лицензирование

| Параметр | Описание |
| --- | --- |
| `--license <license-key>` | Лицензионный ключ, полученный от Dynatrace. |
| `--license-file <path>` | Файл лицензии Dynatrace, используется при отсутствии лицензионного ключа. |

### Хранилище данных

Если SELinux включён, следующие параметры требуют наличия утилиты semanage в системе.

| Параметр | Описание |
| --- | --- |
| `--datastore-dir <path>` | Полный путь к директории установки Dynatrace. Используйте для этой директории выделенный диск или раздел. Здесь хранятся только данные Dynatrace, не настроенные в других хранилищах. Если специальные пути для других хранилищ не настроены, все данные Dynatrace хранятся здесь. **Значение по умолчанию**: `/var/opt/dynatrace-managed` |
| `--rpl-datastore-dir <path>` | Полный путь к хранилищу Session Replay Dynatrace. **Значение по умолчанию**: `/var/opt/dynatrace-managed/server/replayData` |
| `--cas-datastore-dir <path>` | Полный путь к директории репозитория метрик Dynatrace. При указании этого пути Dynatrace хранит данные метрик здесь, а не в основном хранилище данных. **Значение по умолчанию**: `/var/opt/dynatrace-managed/cassandra` |
| `--els-datastore-dir <path>` | Полный путь к директории хранилища Elasticsearch Dynatrace. **Значение по умолчанию**: `/var/opt/dynatrace-managed/elasticsearch` |
| `--svr-datastore-dir <path>` | Полный путь к директории хранилища необработанных транзакций Dynatrace. При указании этого пути Dynatrace хранит необработанные данные транзакций здесь, а не в основном хранилище данных. **Значение по умолчанию**: `/var/opt/dynatrace-managed/server/` |

Путь по умолчанию `/var/opt/dynatrace-managed/server/replayData` не может быть точкой монтирования.

### Системный пользователь

Для максимальной безопасности конфигурации Dynatrace создаёт уникального системного пользователя Linux для сервисов Dynatrace Managed.

Системный пользователь:

* Имеет только права чтения, записи и выполнения, необходимые для нужных файлов
* Не имеет домашней директории
* Использует оболочку `nologin`
* Имеет отключённый вход в систему

Эти меры ограничивают действия злоумышленника при компрометации системы безопасности. При использовании учётной записи по умолчанию настройте её по тем же принципам.

Следующие параметры задают системного пользователя, уполномоченного запускать процессы Dynatrace.

| Параметр | Описание |
| --- | --- |
| `--system-user <user:group>` | Имя пользователя и имя группы системного пользователя, уполномоченного запускать процессы Dynatrace. **Значение по умолчанию**: `dynatrace:dynatrace` |
| `--system-user-id <UID:GID>` | UID и GID системного пользователя, уполномоченного запускать процессы Dynatrace. **Значение по умолчанию**: `1011:1011` |

Можно указать один или оба параметра.

* Имя пользователя/группы и UID/GID должны быть одинаковыми на всех узлах кластера для обеспечения надлежащего доступа к общему хранилищу (например, резервным копиям).
* При переопределении настроек пользователя и группы по умолчанию изменённые значения автоматически передаются как новые значения по умолчанию на последующие узлы, добавляемые в кластер.

### Подключение к Mission Control

| Параметр | Описание |
| --- | --- |
| `--network-proxy` | Если ваша машина использует сетевой прокси для подключения к интернету, укажите адрес здесь в следующем формате: `protocol://[user:password@]server-address:port`. **Значение по умолчанию**: `none` |
| `--network-proxy-cert-file` | Если ваша машина использует HTTPS-прокси с самоподписанным сертификатом, расширьте доверенное хранилище сертификатов. Укажите полный путь к файлу публичного SSL-сертификата в формате PEM. |
| `--registration-token` | Токен для регистрации в Mission Control (необязателен при стандартной установке). |

### Managed Cluster

| Параметр | Описание |
| --- | --- |
| `--cluster-ip` | Если ваша машина имеет более одного сетевого интерфейса, необходимо выбрать интерфейс для трафика Managed Cluster и указать его IPv4-адрес здесь. |
| `--cluster-nodes <id:ip, ..>` | Можно указать идентификатор узла с IP-адресом. Используйте это при восстановлении Managed Cluster, когда необходимо подключить узел замены с IP-адресом, отличным от исходного. |
| `--seed-ip <ip>` | IPv4-адрес seed-узла в Managed Cluster. |
| `--seed-auth <auth-token>` | Токен аутентификации для подключения к seed-узлу. Находится после входа в seed-узел на странице **Download node installer**. Токен действителен три часа. |
| `--rack-dc <dc>` | Для rack-aware развёртываний укажите центр обработки данных, содержащий стойку, в которую необходимо добавить узел. |
| `--rack-name <rack>` | Для rack-aware развёртываний укажите имя стойки, в которую необходимо добавить узел. |

### OneAgent

| Параметр | Описание |
| --- | --- |
| `--install-agent <on|off>` | Включить/отключить установку OneAgent для самомониторинга. **Значение по умолчанию**: `on` |
| `--agent-system-user <user:group>` | Имя пользователя и группа системного пользователя, запускающего агент самомониторинга. Используйте, только если пользователь агента по умолчанию недоступен. |
| `--agent-dir <path>` | Позволяет установить OneAgent для самомониторинга в другую директорию. Например, в Linux: `/bin/sh Managed-installer.sh --agent-dir /opt/dt/self-monitoring`.  При значении `--agent-dir` равном `/data/dynatrace/` установщик создаёт символическую ссылку `/opt/dynatrace/oneagent` -> `/data/dynatrace` и помещает все файлы OneAgent в указанную директорию (в данном примере `/data/dynatrace`). После удаления OneAgent удалите эту символическую ссылку вручную. При использовании этого параметра в Linux с включённым SELinux требуется наличие бинарного файла `semanage` в системе. |

## Параметры SSL/TLS-сертификата

Параметры командной строки также помогают установить или обновить SSL/TLS-сертификаты на Dynatrace Managed. Подробнее об установке полностью квалифицированных цифровых сертификатов см. в разделе [Можно ли использовать собственный SSL-сертификат?](/managed/managed-cluster/installation/ssl-certificate-managed-cluster "Configure your own SSL certificate for a Managed Cluster instead of using the built-in Dynatrace-managed certificate automation.")

| Параметр | Описание |
| --- | --- |
| `--ssl-protocols "<protocols>"` | Список протоколов, разделённых пробелами, принимаемых SSL-соединениями. Заменяет список по умолчанию. |
| `--ssl-ciphers "<ciphers>"` | Определение шифров, принимаемых SSL-соединениями. Заменяет определение по умолчанию. Это определение должно сначала быть проверено командой `openssl ciphers`. |
| `--ciphers-autoupdate <on|off>` | Включить/отключить автоматическое обновление шифров, которое устанавливает значения по умолчанию для протоколов и шифров, принимаемых SSL-соединениями, при каждом обновлении/перенастройке. **Значение по умолчанию для новых узлов**: `on` |

### Прочее

| Параметр | Описание |
| --- | --- |
| `--hosts <on|off>` | Включить/отключить изменение файла `/etc/hosts`. **Значение по умолчанию**: `on` |
| `--sudo-cmd "<cmd>"` | Команда, которая должна использоваться для выполнения системных команд с правами суперпользователя. Должна содержать переменную `$CMD` (вводится как `\$CMD`). По умолчанию используется: `/usr/bin/sudo \$CMD`. Например:  ```  dynatrace-managed.sh --sudo-cmd  "/usr/bin/pbrun \$CMD" ``` |
| `--no-start` | Только для обновления. Используйте этот параметр, если не нужно запускать Dynatrace Managed сразу после обновления. |
| `--unregister` | Используйте этот параметр при удалении Managed Cluster для отмены регистрации кластера и освобождения лицензии в Mission Control. |
| `--version` | Вывести информацию о версии. |
| `--tmp-dir <path>` | Полный путь к директории для временных файлов установщика. |
