# Порты узлов кластера

* Reference
* 3-min read
* Updated on May 08, 2026

Dynatrace Managed требует определённых сетевых портов для работы. Настройте фаервол так, чтобы все перечисленные ниже порты были открыты для двунаправленной связи. В типичном развёртывании Managed все порты должны быть открыты между узлами кластера.

| Порт | Используется | Примечания |
| --- | --- | --- |
| 443 | Веб-интерфейс Dynatrace, OneAgent и REST API | Переадресуется на локальный порт 8022 через правило iptables prerouting. **Этот порт должен оставаться открытым** для приёма входящего трафика из вашего центра обработки данных. Вся связь в кластере использует HTTPS со стойким шифрованием. |
| 5701-5710 | Платформа распределённых данных Hazelcast In-memory | Равномерно распределяет данные по узлам кластера и обеспечивает горизонтальное масштабирование. Требуется только для внутрикластерного взаимодействия. Порты могут быть заблокированы для трафика извне кластера. |
| 5711, 8018 | Nodekeeper | Требуется только для внутрикластерного взаимодействия. Порты могут быть заблокированы для трафика извне кластера. |
| 7000, 7001, 9042 | Hypercube storage на основе Cassandra | Требуется только для внутрикластерного взаимодействия. Порты могут быть заблокированы для трафика извне кластера. |
| 7199 | Cassandra JMX | Требуется только для внутрикластерного взаимодействия. Порт может быть заблокирован для трафика извне кластера. |
| 8019 | Интерфейс обновления | Требуется только для внутрикластерного взаимодействия. Порт может быть заблокирован для трафика извне кластера. |
| 8020, 8021 | Веб-интерфейс и REST API Dynatrace Managed | Требуется только для внутрикластерного взаимодействия. Порты могут быть заблокированы для трафика извне кластера. |
| 8022 | Веб-интерфейс и REST API Dynatrace Managed (NGINX) | Может использоваться вместо порта 443, когда требуется непривилегированный порт. Порты могут быть заблокированы для трафика извне кластера. |
| 8443[1](#fn-1-1-def) | Данные мониторинга от OneAgent, узлы внутри кластера Dynatrace Managed | OneAgent отправляет данные исходящим трафиком только на Dynatrace Cluster — на отслеживаемом хосте входящий порт не открывается. Каждый хост с OneAgent должен иметь доступ к этому порту, который также должен оставаться открытым для межузлового взаимодействия. |
| 9200, 9300 | Поисковый движок на основе Elasticsearch | Требуется только для внутрикластерного взаимодействия. Порты могут быть заблокированы для трафика извне кластера. |
| 9998 | Встроенный ActiveGate | Только для внутрикластерного взаимодействия. |

1

Среды, использующие версию кластера ниже 1.166, используют порт `8443`. Новые среды по-прежнему используют порт `8443`, однако его не нужно открывать за пределами узлов кластера. Обновлённые среды сохраняют настройки портов из предыдущей версии.

## Исходящая связь с Mission Control

Все узлы кластера должны иметь возможность взаимодействовать с Mission Control для получения обновлений программного обеспечения и обмена данными. Разрешите исходящий трафик на следующие IPv4-адреса:

* `13.228.109.33`
* `46.137.81.205`
* `52.5.224.56`
* `52.48.91.146`
* `52.200.165.10`
* `52.221.165.63`

Используемые домены:

* `mcsvc.dynatrace.com`
* `mcsvc-us.dynatrace.com`
* `mcsvc-eu.dynatrace.com`
* `mcsvc-ap.dynatrace.com`

Все данные кластера, получаемые Mission Control через указанные выше домены, хранятся в регионах США.

Для соблюдения ограничений на экспорт США[2](#fn-2-2-def) при планировании размещения Managed Cluster или мониторинга сервисов Huawei Cloud (или любого аффилированного лица Huawei) обратитесь в службу поддержки Dynatrace для настройки конечной точки в Европе или Азиатско-Тихоокеанском регионе.

2

Программное обеспечение Dynatrace не может быть экспортировано или реэкспортировано из США в (а) любую страну группы E, перечисленную в SUPPLEMENT NO. 1 TO PART 740 — COUNTRY GROUPS (в настоящее время Сирия, Северная Корея, Иран и Куба) или Крымский регион Украины, либо (б) любую компанию, организацию или лицо, указанное как сторона, вызывающую озабоченность, по [ссылке](https://2016.export.gov/ecr/eg_main_023148.asp) (в настоящее время включая Huawei Technologies и её дочерние компании по всему миру), либо (3) для любого конечного использования, связанного с разработкой, производством или применением ядерного, химического или биологического оружия или ракет. Наши контракты с клиентами требуют соблюдения этих и других законов об экспортном контроле.

Связь с Mission Control использует HTTPS и WSS через порт `443` с TLS v1.2. SSL/TLS-сертификат предоставляется Amazon и автоматически обновляется ежегодно.

Связь с Mission Control может быть маршрутизирована через прокси. Прокси должен поддерживать WebSockets и расширение TLS SNI.
