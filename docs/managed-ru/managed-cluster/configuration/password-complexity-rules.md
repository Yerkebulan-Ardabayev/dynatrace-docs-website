# Правила сложности пароля

* Published Nov 18, 2020

На этой странице описаны сведения о конфигурации, рекомендации и значения по умолчанию для параметра политики безопасности `Password must meet complexity requirements`. Правила сложности пароля применяются только к встроенной учётной записи администратора и внутренним учётным записям пользователей (см. [Группы пользователей и права доступа](/managed/manage/identity-access-management/user-and-group-management/user-groups-and-permissions "Learn about the supported permissions and policies, how you can assign them to groups, and how you can manage your users and groups.")). При обновлении политики паролей всем пользователям будет предложено обновить пароли при следующем входе в систему.

Параметр политики `Passwords must meet complexity requirements` обязывает пароли соответствовать набору требований к надёжности. Можно настроить следующие требования к паролям:

1. Пароль содержит минимальное количество буквенно-цифровых символов.
2. Пароль содержит минимальное количество символов из следующих категорий:

   * Прописные буквы латинского алфавита (от `A` до `Z`, включая символы с диакритическими знаками)
   * Строчные буквы латинского алфавита (от `a` до `z`, эсцет, включая символы с диакритическими знаками)
   * Цифры по основанию 10 (от `0` до `9`)
   * Не буквенно-цифровые символы (специальные символы): (`` ~!@#$%^&*_-+=`|\(){}[]:;"'<>,.?/ ``) Символы валют, такие как евро или британский фунт стерлингов, не засчитываются как специальные символы в рамках данного параметра политики.

Требования к сложности применяются при изменении или создании паролей.

## Значения по умолчанию

В следующей таблице перечислены фактические и эффективные значения политики по умолчанию. Значения по умолчанию также указаны на странице свойств политики. Допустимый диапазон значений свойств политики: от `8` до `128` для свойства `Minimum password length` и от `0` до `128` для остальных свойств.

| Название свойства политики | Значение по умолчанию | Рекомендуемое |
| --- | --- | --- |
| Minimum password length | `8` для существующих кластеров, установленных до версии 1.206; `12` для новых кластеров, установленных начиная с версии 1.206 | `12` |
| Minimum number of uppercase characters | `1` | `1` |
| Minimum number of lowercase characters | `1` | `1` |
| Minimum number of digits | `1` | `1` |
| Minimum number of non-alphanumeric characters | `0` | `any` |

## Рекомендации

С 2013 года «Специальная публикация NIST 800-63, Приложение A» рекомендует включать в пароли нестандартное использование заглавных букв, специальные символы и хотя бы одну цифру. Также рекомендуется регулярно менять пароли — не реже одного раза в 90 дней. Этот совет принят большинством систем и включён в ряд стандартов, которым должны следовать организации.

1. Соблюдайте минимальную длину пароля в 12 символов (большая длина не всегда означает лучшую безопасность).
2. Применяйте комбинацию прописных и строчных букв, цифр и специальных символов.
3. Обучайте пользователей не использовать пароли Dynatrace в других системах.
4. По возможности используйте корпоративный централизованный сервис управления пользователями — LDAP или SSO.
