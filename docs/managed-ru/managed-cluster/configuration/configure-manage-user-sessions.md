# Настройка пользовательских сессий и управление ими

* Published Apr 02, 2020

К Dynatrace Managed могут обращаться многочисленные пользователи, принадлежащие к различным группам. Количество одновременных сессий для каждой учётной записи пользователя следует ограничивать до минимума, необходимого для выполнения рабочих задач. Dynatrace Managed позволяет контролировать и ограничивать количество одновременных пользовательских сессий для каждой учётной записи.

Перейдите в **User authentication** > **User sessions** для просмотра текущих пользовательских сессий или их завершения. В представлении пользовательских сессий указывается тип входа, время входа и IP-адрес или устройство, с которого произошёл вход.

Типы входа:

* `LOCAL` — пользователи, существующие только в базе данных кластера и локальные для него
* `LDAP` — пользователи LDAP
* `SSO` — вход через провайдера идентификации
* `DEVOPSTOKEN` — вход из проактивной поддержки Mission Control

Для завершения сессии выберите ![Remove](https://dt-cdn.net/images/remove-icon-105c6a04c2.svg "Удалить") и подтвердите действие. Сессия немедленно завершится и пользователь будет выведен из системы.

Для изменения максимально допустимого количества одновременных пользовательских сессий выберите **Configure concurrent sessions**. Можно задать разные ограничения для обычных пользователей и администраторов. Учётные записи администраторов — это пользователи, принадлежащие к любой группе с глобальными правами администратора кластера.

Для снятия всех ограничений по сессиям для администраторов и обычных пользователей включите параметр **Unlimited concurrent sessions**.

Для обновления политики автоматического завершения сессий используйте REST API. По умолчанию автоматический выход пользователей, остающихся на страницах с автообновлением, отсутствует. См. раздел [Обновление конфигурации пользовательских сессий кластера](/managed/dynatrace-api/cluster-api/cluster-api-v2/user-management/update-cluster-user-sessions-configuration "Learn how to update Dynatrace Cluster user sessions configuration using API.")
