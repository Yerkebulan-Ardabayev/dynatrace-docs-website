# Настройка возможностей кластера

* Практическое руководство
* Чтение: 2 мин
* Обновлено 18 июня 2026 г.

Некоторые развёртывания могут требовать, чтобы определённые узлы обслуживали только UI-трафик и не обрабатывали данные OneAgent. Или наоборот, когда узлы обрабатывают только данные OneAgent и не обслуживают UI-трафик. Настроив тот или иной из этих вариантов, можно добиться более высокой стабильности, лучшего поведения кэша и более быстрого времени отклика UI.

Трафик Web UI

Для повышения производительности балансировки нагрузки DNS, начиная с Dynatrace Managed 1.214, трафик web UI по умолчанию отключён при установке 13-го и последующих узлов (или 7-го и последующих узлов в дата-центре для Premium HA). Трафик web UI для узла можно включить или отключить через Cluster REST API.

Чтобы настроить трафик OneAgent или web UI на узле, используй CMC или Cluster REST API.

## Рабочие процессы UI

Используй CMC для настройки трафика OneAgent и трафика web UI для узла Managed Cluster.

### Настройка трафика OneAgent

Чтобы отключить трафик OneAgent (то есть перевести узел Managed Cluster в режим ожидания):

1. Перейди в раздел **Deployment status**.
2. Разверни узел Managed Cluster, который нужно отключить, и выбери **Configure**.
3. Выбери кнопку **Browse** (**…**) в правом верхнем углу.
4. Выбери **Disable OneAgent traffic**, чтобы остановить обработку данных мониторинга на узле.

Чтобы включить трафик OneAgent на узле, повтори шаги с 1 по 3, затем выбери **Enable OneAgent traffic**, чтобы запустить обработку данных мониторинга на узле.

### Настройка трафика web UI

Чтобы отключить трафик web UI (то есть перевести узел Managed Cluster в режим ожидания):

1. Перейди в раздел **Deployment status**.
2. Разверни узел Managed Cluster, который нужно отключить, и выбери **Configure**.
3. Выбери кнопку **Browse** (**…**) в правом верхнем углу.
4. Выбери **Disable Web UI traffic**, чтобы остановить обработку данных мониторинга на узле.

Чтобы включить трафик Web UI на узле, повтори шаги с 1 по 3, затем выбери **Enable Web UI traffic**, чтобы запустить обработку данных мониторинга на узле.

## Рабочие процессы API

Используй Cluster REST API для настройки трафика OneAgent или web UI для узла Managed Cluster.

### Настройка параметров трафика узла

Этот параметр можно настроить с помощью вызова API. См. [POST /cluster/configuration](/managed/dynatrace-api/cluster-api/cluster-api-v1/cluster-v1/post-cluster-nodes-responsibilities "Узнай, как настроить обязанности узлов кластера.").

### Аутентификация

Для выполнения этого запроса нужно разрешение **Cluster token management**, назначенное токену API. Если у тебя нет токена API, его можно сгенерировать.

1. Перейди в раздел **Access Tokens**.
2. Выбери **Generate token** и укажи имя токена.
3. Выбери разрешения **Service Provider API** и нажми **Generate**.

Используй сгенерированный токен API для выполнения вызовов API.

### Конечная точка

Можно выполнить `POST` на конечную точку `/api/v1.0/onpremise/cluster/configuration` со следующими параметрами:

| Параметр | Тип | Описание | В | Обязателен |
| --- | --- | --- | --- | --- |
| ID | integer | ID узла | body | Да |
| webUI | boolean | Задаёт webUI для этого узла. Определяет, должен ли этот узел использоваться для обслуживания запросов WebUI (`true`/`false`). | body | Да |
| OneAgent | boolean | Задаёт трафик OneAgent для этого узла. Определяет, должен ли этот узел получать трафик OneAgent (`true`/`false`). | body | Да |
| datacenter | string | Не используется. | body | Нет |

Пример полезной нагрузки JSON для вызова API:

```
{



"clusterNodes": [



{



"id": 1,



"webUI": true,



"agent": true,



"datacenter": "dc-1"



},



{



"id": 2,



"webUI": true,



"agent": true,



"datacenter": "dc-1"



}



]



}
```
