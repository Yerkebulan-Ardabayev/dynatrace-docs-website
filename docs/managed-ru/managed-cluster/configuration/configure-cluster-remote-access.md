# Настройка удалённого доступа к Cluster

* Практическое руководство
* Чтение за 4 минуты
* Обновлено 18 июня 2026 г.

Специалисты по продукту Dynatrace могут удалённо помогать с обновлениями Dynatrace Managed Cluster и устранением проблем, когда возникают неполадки. Обязательное условие: специалистам по продукту Dynatrace нужны права на удалённый доступ к Managed Cluster. Можно настроить права удалённого доступа к Managed Cluster, чтобы разрешить специалистам по продукту Dynatrace предоставлять обновления и проактивную поддержку.

Требуются права администратора

Нужны права администратора кластера для **Cluster Management Console (CMC)**.

## Рабочие процессы в UI

Используй CMC для настройки прав удалённого доступа для специалистов по продукту Dynatrace.

### Настройка прав удалённого доступа

Чтобы настроить уровень прав в Managed Cluster, в **CMC** перейди в **Settings** > **Remote access permissions**.

На этой странице можно разрешить специалистам по продукту Dynatrace удалённый доступ к Managed Cluster. Если эта настройка включена и обнаружены события, специалисты по продукту Dynatrace могут удалённо корректировать настройки Managed Cluster для обеспечения оптимальной производительности и стабильности.

Безопасность

Всё взаимодействие с Mission Control защищено и выполняется по HTTPS с проверками сертификатов, как в браузере. Все изменения конфигурации Dynatrace Managed полностью фиксируются в журнале аудита, а каждый удалённый доступ регистрируется как отдельное событие (открой **Events**, чтобы посмотреть список зафиксированных событий). Команда Mission Control не имеет доступа к сертификатам или учётным данным пользователей. Команда Mission Control также не может получить root-доступ к каким-либо серверам.

## Рабочие процессы API

Используй Remote Access REST API для настройки параметров и прав удалённого доступа. Подробности см. в [Dynatrace Cluster API](/managed/dynatrace-api/cluster-api/cluster-api-v2#remote-access "Узнай об управлении окружениями, сетевыми зонами, синтетическими локациями, нодами и токенами в Dynatrace Managed с помощью Cluster API v2.").

## Справочник по удалённому доступу

Используй этот справочник, чтобы сравнить области удалённого доступа и права ролей перед тем, как предоставить доступ специалистам по продукту Dynatrace.

### Области удалённого доступа

После включения удалённого доступа Dynatrace можно настроить права удалённого доступа для специалистов по продукту Dynatrace с одной из следующих областей.

* **All**

  Вся команда специалистов по продукту Dynatrace может получить доступ к Managed Cluster, чтобы предоставить полноценную проактивную поддержку и оптимизировать настройки Managed Cluster.
* **Read-only access to all**

  Вся команда специалистов по продукту Dynatrace может получить доступ к Managed Cluster, но не может изменять настройки Managed Cluster. Опция доступа только для чтения существенно ограничивает уровень проактивной поддержки. При этой опции для пользователя с удалённым доступом доступна только роль **Viewer**. Специалисты по продукту Dynatrace свяжутся при необходимости внесения изменений.
* **Approved**

  Доступ к Managed Cluster получают только одобренные специалисты по продукту Dynatrace. Администраторы кластера получают уведомления по email о запросах на удалённый доступ, ожидающих рассмотрения. Администратор кластера должен одобрить каждый запрос, чтобы предоставить права. Можно настроить продолжительность и роль, которую предоставляешь. Также можно заранее предоставить права известным специалистам по продукту Dynatrace.

  Область Approved даёт максимальный контроль над тем, кто может получить доступ к Managed Cluster, но существенно снижает возможности команды специалистов по продукту Dynatrace по предоставлению проактивной поддержки.

### Права ролей удалённого доступа

Пользователю с удалённым доступом можно назначить роль **Admin**, **User** или **Viewer**. В таблице ниже приведены подробности о правах, назначаемых каждой роли.

| Права | Admin | User | Viewer | Описание |
| --- | --- | --- | --- | --- |
| Environment | Применимо | Применимо | Применимо | Разрешает доступ только для чтения к окружению. В частности, сотрудники Dynatrace имеют доступ к:  * Настройкам, связанным с пользователем: профиль вошедшего пользователя и его настройки (например, запланированные отчёты, избранные дашборды и пункты меню). * Страницам Dynatrace Hub: страницы установки OneAgent или ActiveGate. * Настройкам в режиме только для чтения. * Отчётам. * **CMC** в режиме только для чтения. * Конфигурации правил захвата данных запросов в режиме только для чтения. * Внутренним диагностическим данным. * Чтению журнала аудита. * Доступу к архиву поддержки. * Хранилищу учётных данных Synthetic и действиям обновления/удаления учётных данных, принадлежащих пользователю. Пользователи без этого права видят, что точка данных существует, но персональные данные замаскированы звёздочками (\*\*\*\*\*). Также разрешает пользователю вручную запускать дампы памяти. |
