# Настройка интернет-прокси

* Практическое руководство
* Чтение 2 мин
* Обновлено 18 июня 2026 г.

Настройка интернет-соединения для [проактивной поддержки Mission Control (MC)](/managed/managed-cluster/basics/mission-control-proactive-support "Mission Control проактивно отслеживает Managed Cluster, предоставляет обновления ПО и обеспечивает безопасность и надёжность установки."), получения обновлений MC и использования внешних уведомлений о проблемах через такие инструменты, как ServiceNow, Jira и вебхуки.

Протоколы аутентификации

Доступны следующие протоколы аутентификации:

* Basic
* NTLMv1 (устарел)

Настраивай аутентификацию Basic вместе с TLS. Поверх TLS транспортное шифрование защищает учётные данные.

Существует три способа настройки прокси-соединения для Managed Cluster. Выбери подход, который лучше всего подходит под задачу.

## Настройка прокси во время установки

Используй следующие параметры командной строки для настройки прокси-соединения с MC во время установки Managed:

```
--network-proxy
```

Если машина использует сетевой прокси для подключения к интернету, введи адрес в следующем формате:

```
protocol://[user:password@]server-address:port
```

Значение по умолчанию, `none`.

```
--network-proxy-cert-file
```

Если машина использует сетевой HTTPS-прокси с самоподписанным сертификатом, расширь хранилище доверенных сертификатов. После этого параметра укажи полный путь к файлу публичного SSL-сертификата в формате PEM.

## Настройка прокси в Cluster Management Console

1. Войди в **Cluster Management Console**.
2. Перейди в **Settings** > **Internet proxy** и отредактируй **Proxy configuration** для конкретного дата-центра.
3. Выбери **Connect via proxy** и укажи данные прокси-сервера:

   * **Scheme**
   * **Proxy address** и **Port**
   * **Username** и **Password**, если анонимный доступ невозможен.

### Исключение хостов из интернет-прокси

Чтобы исключить хосты из прокси, например когда интеграции проблем через вебхуки указывают на хосты внутренней сети, добавь их в список исключений. Используй символ подстановки (`*`) в начале или конце каждой записи хоста, чтобы охватить все URL в пределах домена.

## Настройка прокси через Cluster API

Используй [Cluster API](/managed/dynatrace-api/cluster-api/cluster-api-v1/internet-proxy-v1/put-cluster-proxy-configuration "Узнай, как использовать Dynatrace API для настройки или обновления конфигурации прокси кластера."), чтобы задать или обновить конфигурацию интернет-прокси Managed Cluster.

## Часто задаваемые вопросы

Можно ли использовать прозрачный прокси?

Да, Dynatrace поддерживает настройку прозрачного прокси.

Прозрачный прокси (также известный как перехватывающий прокси, встроенный прокси или принудительный прокси) может маршрутизировать и перехватывать обмен данными между Managed Cluster и MC. Прозрачный прокси обычно располагается между Managed Cluster и MC. С прозрачным прокси можно аудировать и проверять содержимое всех обменов данными.

Dynatrace не обязательно знать о наличии прокси. Настрой Dynatrace Managed на доверие корневому сертификату, закрытый ключ которого известен прокси. При такой настройке прокси анализирует содержимое SSL/TLS-транзакций. Эта схема представляет собой конфигурацию man-in-the-middle, которую Dynatrace авторизует за счёт доверия к корневому сертификату, принадлежащему прокси.

Как обновить SSL-сертификат

Запусти скрипт реконфигурации со следующими параметрами:

```
/opt/dynatrace-managed/installer/reconfigure.sh --update-cert --network-proxy-cert-file <proxy_cert_file>
```
