# Настройка подключения к SMTP-серверу

* Updated on Nov 13, 2023

Используйте эти параметры для управления способом доставки Dynatrace email-уведомлений, отчётов и других сообщений пользователям и администраторам.

1. Перейдите в **Settings**.
2. Разверните раздел **Emails** и выберите **SMTP server** для отображения страницы **Configure SMTP server**.
3. Начните с выбора способа доставки email.

   * **SMTP server with Mission Control fallback** (настройка по умолчанию)  
     Выберите этот вариант для доставки email через SMTP-сервер с резервным использованием Mission Control при недоступности SMTP.

     + Пока ваш SMTP-сервер доступен, настройки SMTP управляют отправкой почты (почтовый сервер, безопасность соединения и т. д.).
     + Если ваш SMTP-сервер становится недоступным, Mission Control доставляет email с адресом отправителя `notifications-noreply@dynatrace-managed.com`.
   * **SMTP server**  
     Выберите этот вариант для доставки email только через SMTP-сервер (без резервного метода доставки).

     + Настройки SMTP управляют отправкой почты (почтовый сервер, безопасность соединения и т. д.).
     + Если ваш SMTP-сервер становится недоступным, email отправляться не будет.
   * **Mission Control**  
     Выберите этот вариант для отправки всей почты через Mission Control.

     + Все описанные ниже шаги настройки SMTP-сервера недоступны, поскольку настраивать SMTP-сервер не требуется. Dynatrace берёт всё на себя.
     + Адрес отправителя — `notifications-noreply@dynatrace-managed.com`.
4. Задайте **Connection security** — протокол безопасности для вашего SMTP-сервера. По умолчанию используется `No encryption` на порту `25`.

   * При изменении **Connection security** значение **Port** автоматически меняется на стандартный порт для выбранного протокола (см. номера портов по умолчанию ниже).
   * При ручном изменении значения **Port** ваши настройки сохраняются, и изменение протокола **Connection security** не меняет значение **Port** автоматически.

   * `No encryption`: небезопасно. Порт по умолчанию: `25`.  
     Отправка email без шифрования. Используйте этот вариант только если SMTP-сервер не поддерживает другие способы защиты.
   * `SSL/TLS required`: безопасно. Порт по умолчанию: `465`.

     + Если удаётся установить SSL/TLS-соединение — отправка зашифрованного email по SSL/TLS.
     + Если SSL/TLS-соединение не удаётся установить — email не отправляется.
   * `STARTTLS optional`: небезопасно. Порт по умолчанию: `587`.

     + Если удаётся установить STARTTLS-соединение — отправка зашифрованного email по STARTTLS.
     + Если STARTTLS-соединение не удаётся установить — отправка email без шифрования.
   * `STARTTLS required`: безопасно. Порт по умолчанию: `587`.

     + Если удаётся установить STARTTLS-соединение — отправка зашифрованного email по STARTTLS.
     + Если STARTTLS-соединение не удаётся установить — email не отправляется.
5. Задайте **Mail server** — адрес SMTP-сервера.
6. Задайте **Port** — номер порта SMTP-сервера. Подробности см. выше в разделе **Connection security**.
7. Задайте **Username** и **Password**, если сервер требует аутентификации.
8. Задайте **Sender email address** — адрес, который будет отображаться как адрес отправителя.
9. Задайте **Email test address** — адрес для отправки тестовых сообщений.
10. Выберите **Send test message**, чтобы проверить конфигурацию SMTP отправкой тестового сообщения на **Email test address**.

    * Через некоторое время, если отправка прошла успешно, отображается статусное сообщение `Sending succeeded`.
    * Проверьте входящую почту учётной записи **Email test address**, чтобы убедиться, что пришло сообщение «Dynatrace Managed: Mail server setup» с ожидаемым адресом отправителя.
11. Выберите **Save changes**, когда конфигурация будет корректной.
