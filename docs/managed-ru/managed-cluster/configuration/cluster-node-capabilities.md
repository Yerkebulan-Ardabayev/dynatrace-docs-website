# Возможности узлов кластера

* Published Jul 17, 2018

В ряде развёртываний может потребоваться, чтобы определённые узлы обслуживали только трафик веб-интерфейса, не обрабатывая данные OneAgent. Или наоборот — узлы обрабатывают только данные OneAgent, не обслуживая трафик веб-интерфейса. Настройка одного из этих вариантов позволяет повысить стабильность, улучшить поведение кэша и ускорить работу веб-интерфейса.

Трафик веб-интерфейса

Для улучшения производительности DNS-балансировки нагрузки начиная с Dynatrace Managed 1.214 трафик веб-интерфейса по умолчанию отключается при установке 13-го и последующих узлов (или 7-го и последующих узлов в центре обработки данных для Premium HA). Включить или отключить трафик веб-интерфейса для узла можно через Cluster REST API.

Для включения или отключения трафика OneAgent на узле можно воспользоваться веб-интерфейсом, командной строкой или вызовом API.

## Включение/отключение трафика OneAgent через Cluster Management Console

Для отключения трафика OneAgent (то есть перевода узла кластера в режим ожидания)

1. Перейдите в **Deployment Status**.
2. Разверните нужный узел кластера и нажмите **Configure**.
3. Нажмите кнопку Browse (**…**) в правом верхнем углу.
4. Выберите **Disable OneAgent traffic**, чтобы остановить обработку данных мониторинга на узле.

Для включения трафика OneAgent на узле повторите первые три шага, затем выберите **Enable OneAgent traffic**, чтобы возобновить обработку данных мониторинга на узле.

## Включение/отключение трафика веб-интерфейса через Cluster Management Console

Для отключения трафика веб-интерфейса (то есть перевода узла кластера в режим ожидания)

1. Перейдите в **Deployment Status**.
2. Разверните нужный узел кластера и нажмите **Configure**.
3. Нажмите кнопку Browse (**…**) в правом верхнем углу.
4. Выберите **Disable Web UI traffic**, чтобы остановить обработку данных мониторинга на узле.

Для включения трафика веб-интерфейса на узле повторите первые три шага, затем выберите **Enable Web UI traffic**, чтобы возобновить обработку данных мониторинга на узле.

## Включение/отключение трафика OneAgent или веб-интерфейса с помощью вызова API

Данный параметр можно настроить через вызов API. См. раздел [Настройка обязанностей узлов кластера](/managed/dynatrace-api/cluster-api/cluster-api-v1/cluster-v1/post-cluster-nodes-responsibilities "Learn how to configure cluster nodes responsibilities.")

### Аутентификация

Для выполнения этого запроса необходимо разрешение **Cluster token management**, назначенное вашему токену API. Если у вас нет токена API, его можно создать.

1. Перейдите в **Access Tokens**.
2. Выберите **Generate token** и введите имя токена.
3. Выберите разрешения **Service Provider API** и нажмите **Generate**.

Используйте сгенерированный токен API для выполнения вызовов API.

### Конечная точка

Можно выполнить `POST` к конечной точке `/api/v1.0/onpremise/cluster/configuration` со следующими параметрами:

| Параметр | Тип | Описание | В | Обязателен |
| --- | --- | --- | --- | --- |
| ID | integer | ID узла | body | обязателен |
| webUI | boolean | Задаёт трафик веб-интерфейса для данного узла. Определяет, должен ли этот узел обслуживать запросы WebUI (`true`/`false`). | body | обязателен |
| OneAgent | boolean | Задаёт трафик OneAgent для данного узла. Определяет, должен ли этот узел получать трафик OneAgent (`true`/`false`). | body | обязателен |
| datacenter | string | не используется. | body | не обязателен |

Пример JSON-payload для вызова API:

```
{



"clusterNodes": [



{



"id": 1,



"webUI": true,



"agent": true,



"datacenter": "dc-1"



},



{



"id": 2,



"webUI": true,



"agent": true,



"datacenter": "dc-1"



}



]



}
```
