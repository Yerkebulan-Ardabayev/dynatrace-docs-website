# Настройка контроля затрат и трафика для пользовательских приложений

* How-to guide
* 1-min read
* Updated on Feb 28, 2024

По умолчанию Dynatrace фиксирует все пользовательские действия и сессии для анализа. Такой подход обеспечивает полную видимость производительности вашего приложения и клиентского опыта. Настройка **Cost and traffic control** позволяет снизить детализацию анализа действий и сессий пользователей, указав Dynatrace фиксировать меньший процент сессий.

Хотя эта настройка позволяет снизить [затраты на мониторинг](/managed/license/monitoring-consumption-classic/digital-experience-monitoring-units "Узнайте, как рассчитывается потребление Dynatrace Digital Experience Monitoring на основе DEM-единиц."), она также снижает видимость того, как клиенты используют ваши приложения. Например, при значении `10%` Dynatrace анализирует лишь каждую десятую сессию пользователя.

Чтобы ограничить количество анализируемых сессий пользователей:

1. Перейдите в **Frontend**.
2. Выберите приложение, которое нужно настроить.
3. Нажмите **More** (**…**) > **Edit** в правом верхнем углу плитки с именем вашего приложения.
4. В настройках приложения выберите **General** > **Enablement and cost control**.
5. В разделе **Real User Monitoring** введите значение меньше `100` в поле **Cost and traffic control**. Значение по умолчанию — `100`.

После указания этой настройки Dynatrace анализирует равномерно распределённое количество сессий, соответствующее заданному проценту от общего числа сессий.

Функция контроля затрат и трафика не поддерживается в [Dynatrace OpenKit](/managed/ingest-from/extend-dynatrace/openkit/dynatrace-openkit-libraries-on-github "Ознакомьтесь с библиотеками Dynatrace OpenKit на GitHub.") для JavaScript.
