# Настройка определения IP-адресов в пользовательских приложениях

# Настройка определения IP-адресов в пользовательских приложениях

* How-to guide
* 1-min read
* Updated on Mar 21, 2024

Когда веб-запросы и RUM beacon напрямую отправляются на инструментированный сервер, Dynatrace определяет IP-адреса устройств конечных пользователей через socket-соединения.

При использовании неинструментированных компонентов, таких как балансировщики нагрузки, CDN или прокси-серверы, удалённый IP-адрес отличается от исходного. В таких случаях Dynatrace также анализирует определённые HTTP-заголовки. Эти заголовки чаще всего используются для определения исходного IP-адреса, когда клиент подключается к веб-серверу через HTTP-прокси, CDN или балансировщик нагрузки.

Вы можете просмотреть список заголовков, которые Dynatrace использует для определения IP-адресов клиентов ваших приложений. Для этого перейдите в **Settings** > **Web and mobile monitoring** > **IP determination**. Dynatrace обрабатывает эти заголовки в определённом порядке, который можно изменить, а также добавить собственные заголовки.

## Управление заголовками по умолчанию

Вы можете обновлять имена заголовков, удалять ненужные и изменять порядок их обработки в Dynatrace.

1. Перейдите в **Settings** > **Web and mobile monitoring** > **IP determination**.
2. Управляйте заголовками:

   * Чтобы переименовать заголовок: нажмите **Expand row** ![Expand row](https://dt-cdn.net/images/expand-row-icon-9c4950fc2e.svg "Expand row") и введите новое имя заголовка.
   * Чтобы удалить заголовок: нажмите **Delete row** ![Remove](https://dt-cdn.net/images/remove-icon-105c6a04c2.svg "Remove").
   * Чтобы изменить приоритет заголовка: нажмите и удерживайте **Drag row** ![Drag handle](https://dt-cdn.net/images/drag-handle-turquoise-600-1aa0e5ea00.svg "Drag handle") рядом с именем заголовка и переместите его вверх или вниз.

## Добавление пользовательских заголовков клиентского IP-адреса

Если IP-адреса клиентов находятся в другом заголовке, создайте пользовательское правило, чтобы Dynatrace мог корректно определять IP-адреса.

Чтобы добавить пользовательский заголовок клиентского IP-адреса:

1. Перейдите в **Settings** > **Web and mobile monitoring** > **IP determination**.
2. Нажмите **Create custom client IP header**.
3. Укажите имя заголовка.

Для одной среды можно создать до 10 пользовательских заголовков клиентского IP-адреса.

Добавление пользовательского заголовка клиентского IP-адреса может потребоваться в следующих случаях.

* Заголовок, содержащий IP-адреса клиентов, отсутствует в нашем списке заголовков в **Settings** > **Web and mobile monitoring** > **IP determination**.
* IP-адреса клиентов для ваших приложений хранятся в проприетарном заголовке.
* IP-адреса клиентов определяются некорректно: например, в Dynatrace отображаются IP-адреса балансировщиков нагрузки.

## Причины некорректного определения IP-адресов клиентов

Существует несколько причин, по которым в вашем приложении могут отображаться некорректные IP-адреса клиентов. Подробные инструкции см. в разделах выше.

| Проблема | Возможное решение |
| --- | --- |
| IP-адреса клиентов расположены в заголовке, которого нет в нашем списке. | Добавьте пользовательский заголовок клиентского IP-адреса. |
| Указан некорректный пользовательский заголовок клиентского IP-адреса. | Обновите имя заголовка. |
| Указанный пользовательский заголовок не содержит IP-адресов. | Укажите другой пользовательский заголовок. |
| Порядок заголовков задан неверно. | Измените порядок обработки заголовков. |
| Инфраструктура не настроена на добавление заголовка, передающего IP-адреса клиентов. | Перенастройте инфраструктуру. |
| Заголовок, содержащий IP-адреса клиентов, не является частью beacon-запросов.[1](#fn-1-1-def) | Перенастройте инфраструктуру. |

1

Заголовки определения IP-адресов в обычных запросах приложения не влияют на определение IP-адресов в захваченных RUM-данных, даже если они захватываются как трейсы и коррелируют с пользовательскими действиями.
