# Создание вычисляемых метрик для пользовательских приложений

# Создание вычисляемых метрик для пользовательских приложений

* How-to guide
* 1-min read
* Updated on May 10, 2024

В Dynatrace вы можете создавать вычисляемые метрики, чтобы сделать текущий анализ доступным для [построения диаграмм](/managed/analyze-explore-automate/explorer "Запрашивайте метрики и преобразовывайте результаты для получения нужных аналитических данных.") и [использования через API](/managed/dynatrace-api/environment-api/metric-v2 "Получайте информацию о метриках через Metrics v2 API."). Вычисляемые метрики также можно использовать для добавления пользовательских оповещений.

Выбрав нужное приложение, вы можете использовать **Multidimensional Analysis** для выбора аспектов пользовательских действий и создания вычисляемой метрики. Можно выбрать разбивку выбранных метрик производительности по другому измерению (геолокация, браузер, тип ошибки) или использовать только одно измерение, например [свойства пользовательских действий](/managed/observe/digital-experience/web-applications/additional-configuration/define-user-action-and-session-properties "Определяйте пользовательские строковые, числовые и датовые свойства для ваших веб-приложений.").

## Создание вычисляемой метрики

Чтобы создать вычисляемую метрику из вашего приложения:

1. Перейдите в **Frontend** и выберите приложение, для которого нужно создать метрику.
2. В разделе **Top 3 actions** нажмите **Analyze performance**.
3. Выберите нужные **OS family**, **Contribution**, временной диапазон и параметры **Analyze by**.
4. Необязательно: используйте строку фильтров для добавления фильтров по геолокации, производителю, Apdex и другим параметрам, чтобы сосредоточиться на нужных данных.
5. Нажмите **Create metric**.

   ![Создание метрики на странице Multidimensional analysis](https://dt-cdn.net/images/mobile-metric-create-metric-mda-1758-a913b7f692.png)

   Создание метрики на странице Multidimensional analysis
6. Необязательно: измените имя и ключ метрики, включите **Split by <dimension name>**.

   Для [мобильных](/managed/observe/digital-experience/mobile-applications/analyze-and-use/action-and-session-properties-mobile "Свойства действий и сессий — метаданные в формате «ключ-значение», обеспечивающие дополнительную видимость и более глубокий анализ взаимодействия конечных пользователей.") и [пользовательских приложений](/managed/observe/digital-experience/custom-applications/analyze-and-use/action-and-session-properties-custom "Свойства действий и сессий — метаданные в формате «ключ-значение», обеспечивающие дополнительную видимость и более глубокий анализ взаимодействия конечных пользователей.") разбивка метрики по свойству действия пользователя недоступна. Разбивка по свойству действия доступна только для веб-приложений.
7. Нажмите **Create metric**.

   ![Оверлей для создания метрики](https://dt-cdn.net/images/mobile-metric-create-metric-overlay-330-88ec98fac1.png)

   Оверлей для создания метрики

Используйте метрику для создания пользовательской диаграммы или оповещения.

В вычисляемые метрики записываются только новые данные; ретроспективные данные не включаются.

Для всех приложений одной среды можно иметь до 500 включённых вычисляемых метрик, и до 100 включённых метрик на одно приложение.

### Пример

В этом примере проанализируем `Price` — [свойство пользовательского действия](/managed/observe/digital-experience/web-applications/additional-configuration/define-user-action-and-session-properties#custom-properties "Определяйте пользовательские строковые, числовые и датовые свойства для ваших веб-приложений.") — и отфильтруем его по `Loyalty status` — другому свойству действия.

На странице **Multidimensional analysis** выберите временной диапазон для анализа. Чтобы отфильтровать цены, уплаченные только platinum-клиентами, выберите `Price` в списке **Analyze by**, затем добавьте дополнительные фильтры: `String property`, `Loyalty status` и `Platinum`.

## Создание пользовательских диаграмм на основе вычисляемых метрик

Создание диаграмм помогает анализировать комбинации метрик приложений непосредственно на дашборде. Вы можете разбивать и фильтровать доступные сущности для точной настройки измерений метрик.

Подробнее о создании диаграмм и их закреплении на дашбордах см. [Data Explorer](/managed/analyze-explore-automate/explorer "Запрашивайте метрики и преобразовывайте результаты для получения нужных аналитических данных.").

## Управление метриками

После создания вычисляемой метрики вы можете просматривать её свойства, удалять, временно отключать или создавать диаграмму или метрическое событие.

После создания метрики изменить её свойства невозможно.

1. Перейдите в **Frontend**.
2. Выберите приложение, которое нужно настроить.
3. Нажмите **More** (**…**) > **Edit** в правом верхнем углу плитки с именем вашего приложения.
4. В настройках приложения выберите **Metrics**.
5. Выберите метрику для управления и ознакомьтесь с её свойствами или выполните одно из следующих действий.

   * **Enable or disable** ![Toggle icon](https://dt-cdn.net/images/icon-toggle-barista-701-35879d6adf.png "Toggle icon") — включить или отключить метрику
   * **Copy** — скопировать API URL метрики
   * **Create a chart** — создать диаграмму в [Data Explorer](/managed/analyze-explore-automate/explorer "Запрашивайте метрики и преобразовывайте результаты для получения нужных аналитических данных.")
   * **Create alert** — создать [метрическое событие](/managed/dynatrace-intelligence/anomaly-detection/metric-events "Узнайте о метрических событиях в Dynatrace")
   * **Delete metric** — удалить метрику
