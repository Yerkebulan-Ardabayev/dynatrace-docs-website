# Просмотр отчётов о сбоях в пользовательских приложениях

# Просмотр отчётов о сбоях в пользовательских приложениях

* How-to guide
* 1-min read
* Updated on Mar 20, 2024

Анализ и устранение [сбоев](/managed/observe/digital-experience/rum-concepts/user-and-error-events#crash "Узнайте о событиях пользователей и ошибках, а также о типах событий, фиксируемых Dynatrace.") критически важны для улучшения пользовательского опыта. Dynatrace захватывает сбои и их стек-трейсы, что позволяет оценить критичность сбоя и определить его первопричину. Рабочий процесс анализа сбоев помогает оценить их влияние, выявить затронутые группы пользователей и быстро добраться до первопричины. Проактивное устранение проблем обеспечивает стабильное достижение бизнес-целей вашими приложениями.

При работе с клиент-серверными конфигурациями некоторые сбои могут иметь серверную первопричину. Dynatrace использует технологию PurePath® и OneAgent®, чтобы коррелировать пользовательские действия с веб-запросами, обеспечивая полную видимость: от отдельных действий пользователя до конкретных серверных запросов к базе данных, способствующих сбоям.

Чтобы просмотреть базовую информацию о сбоях в вашем приложении:

1. Перейдите в **Frontend**.
2. Выберите приложение для анализа.
3. На странице обзора приложения выберите плитку **Crashes & Errors**.

Плитка **Crashes & Errors** показывает количество сбоев, процент пользователей без сбоев и частоту сбоев за текущий выбранный период. Диаграмма **Crashes by version** отображает общее количество сбоев по версиям за выбранный период.

![Плитка Crashes на странице обзора приложения](https://dt-cdn.net/images/crashes-tile-mobile-app-852-77531749a6.png)

Плитка Crashes на странице обзора приложения

Чтобы получить доступ к отчётам о сбоях в вашем приложении:

1. На странице обзора приложения выберите плитку **Crashes & Errors**.
2. Под диаграммой **Crashes by version** нажмите **Analyze crashes**. Откроется страница **Crash analysis**.

![Страница Crash analysis](https://dt-cdn.net/images/crash-analysis-mobile-app-2519-daa9f932f4.png)

Страница Crash analysis

## Страница анализа сбоев

Страница **Crash analysis** даёт обзор всех групп сбоев за выбранный период и позволяет сосредоточиться на диапазоне свойств и измерений, специфичных для паттерна сбоя. Вы можете анализировать сбои по версии операционной системы, версии приложения, региону пользователя, длительности сессии, типу соединения и другим измерениям.

### Последние версии приложения

Раздел **Latest app versions** содержит информацию о сбоях для четырёх последних версий приложения. Для каждой версии можно проверить процент пользователей без сбоев, количество затронутых пользователей и количество сбоев на определённое число сессий.

### Статистика сбоев

В разделе **Crash statistics** можно проверить общее количество сбоев, количество групп сбоев и количество затронутых версий за выбранный период. В разделе **Latest app versions** нажмите **Filter** ![Filter](https://dt-cdn.net/images/filter-icon-41ddd02d66.svg "Filter") рядом с нужной версией для получения информации по конкретной версии.

Также можно использовать другие фильтры: **Connection type**, **Country**, **Manufacturer**.

### Группы сбоев

Dynatrace группирует сбои по схожести стек-трейса и месту возникновения в исходном коде. Как правило, это работает и для разных версий приложения, что упрощает определение наличия сбоя в последнем выпуске.

## Страница деталей группы сбоев

В разделе **Crash groups** на странице **Crash analysis** выберите группу сбоев для просмотра всех её экземпляров. Используйте фильтры для изучения конкретной части группы сбоев.

![Информация о группе сбоев](https://dt-cdn.net/images/crash-group-mobile-app-2518-9324fa74b5.png)

Информация о группе сбоев

На странице деталей группы сбоев нажимайте **Next / previous crash occurrence** ![Left and right arrows](https://dt-cdn.net/images/analysisarrowbuttons-49-c49391e1bc.png "Left and right arrows") для перемещения между экземплярами сбоев. Выберите интересующий экземпляр, затем переключайтесь между вкладками для просмотра подробной информации.

Stack trace

Device information

Session information

На вкладке **Stack trace** изучите стек-трейс или скачайте его для совместного использования с командой.

![Стек-трейс для экземпляра сбоя](https://dt-cdn.net/images/crash-occurence-stack-trace-tab-2520-f9c5b285f0.png)

Стек-трейс для экземпляра сбоя

На вкладке **Device information** просмотрите подробную информацию об устройстве, на котором произошёл текущий экземпляр сбоя: геолокацию, версию ОС, язык пользователя и многое другое.

![Информация об устройстве для экземпляра сбоя](https://dt-cdn.net/images/crash-occurence-device-info-tab-2520-8e77ca1904.png)

Информация об устройстве для экземпляра сбоя

На вкладке **Session information** нажмите **View full session** для перехода на страницу деталей сессии. Там доступна подробная информация о сессии со сбоем, список всех пользовательских действий и событий, произошедших до сбоя, а также выбор экземпляра сбоя. Подробнее см. [Анализ сессий пользователей](/managed/observe/digital-experience/session-segmentation/new-user-sessions "Узнайте о сегментации и атрибутах фильтрации сессий пользователей.").

![Информация о сессии для экземпляра сбоя](https://dt-cdn.net/images/crash-occurence-session-info-tab-2520-e40fca17fd.png)
