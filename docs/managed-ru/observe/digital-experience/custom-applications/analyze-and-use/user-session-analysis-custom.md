# Анализ сессий пользователей в пользовательских приложениях

* How-to guide
* 1-min read
* Published Jan 30, 2023

В современной цифровой среде возможность просматривать и анализировать сессии пользователей в ваших пользовательских приложениях является необходимостью.

По умолчанию анализ сессий пользователей охватывает все сессии в вашей среде: веб-, мобильные и пользовательские приложения. Чтобы сосредоточиться только на пользовательских приложениях, необходимо добавить один фильтр.

Чтобы просмотреть сессии пользователей, захваченные в пользовательских приложениях:

1. Перейдите в **Session Segmentation**. Откроется страница **User sessions**.
2. В поле **Filter by** выберите **Application type:** `Custom`, чтобы получить сессии пользователей, захваченные в пользовательских приложениях.
3. Необязательно: для более точного анализа добавьте дополнительные фильтры, например по версии операционной системы, производителю, количеству ошибок и оценке пользовательского опыта.
4. Выберите временную метку нужной сессии для перехода на страницу её деталей.

Dynatrace захватывает сессии пользователей из пользовательских приложений как мобильные сессии. Встроенные метрики захватываются без указания того, является ли источник мобильным или пользовательским приложением. Подробнее см. [Встроенные метрики](/managed/analyze-explore-automate/metrics-classic/built-in-metrics "Изучите полный список встроенных метрик Dynatrace.").

Подробнее см. [Анализ сессий пользователей](/managed/observe/digital-experience/session-segmentation/new-user-sessions "Узнайте о сегментации и атрибутах фильтрации сессий пользователей.").
