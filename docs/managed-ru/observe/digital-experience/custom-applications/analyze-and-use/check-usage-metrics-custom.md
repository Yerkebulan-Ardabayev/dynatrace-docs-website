# Проверка метрик пользовательского опыта в пользовательских приложениях

# Проверка метрик пользовательского опыта в пользовательских приложениях

* How-to guide
* 1-min read
* Published Jan 30, 2023

Метрики пользовательского опыта вашего приложения дают быстрый обзор количества пользователей, активно использующих приложение за выбранный временной диапазон.

Чтобы получить доступ к метрикам пользовательского опыта вашего приложения:

1. Перейдите в **Frontend**.
2. Выберите приложение для анализа.
3. Изучите плитку **User experience**, выбранную по умолчанию.

Плитка **User experience** показывает общее количество пользователей, средний рейтинг Apdex и частоту пользовательских действий за выбранный временной диапазон.

![Плитка User experience на странице обзора приложения](https://dt-cdn.net/images/user-experience-tile-mobile-app-852-0baf6bdc02.png)

Плитка User experience на странице обзора приложения

Подробная информация о тенденциях рейтинга Apdex, активных и новых пользователях, а также распределении версий приложения доступна в соответствующих разделах.

## Рейтинг Apdex

Диаграмма **Apdex rating** показывает тенденции среднего [рейтинга Apdex](/managed/observe/digital-experience/rum-concepts/scores-and-ratings/apdex-ratings "Узнайте, как Dynatrace использует Apdex для измерения удовлетворённости пользователей производительностью приложения.") вашего приложения за выбранный временной диапазон.

![Раздел User experience - Apdex rating](https://dt-cdn.net/images/user-experience-apdex-rating-847-ca725a074c.png)

Раздел User experience - Apdex rating

## Пользовательские действия и пользователи

Диаграмма **User actions and Users** сравнивает количество пользовательских действий с количеством активных пользователей за выбранный временной диапазон.

![Раздел User experience - User actions and Users](https://dt-cdn.net/images/user-experience-user-actions-and-users-848-27d7bac217.png)

Раздел User experience - User actions and Users

## Новые пользователи

Диаграмма **New users** показывает количество пользователей, впервые запустивших ваше приложение за выбранный временной диапазон. Эта метрика учитывает только первую сессию после запуска инструментированного приложения. Обновления приложения и обновления библиотеки Dynatrace не сбрасывают счётчик для возвращающихся пользователей. Метрика привязана к конкретным устройствам, поэтому пользователи считаются повторно при установке приложения на нескольких устройствах.

![Раздел User experience - New users](https://dt-cdn.net/images/user-experience-new-users-2-848-df286583cc.png)

Раздел User experience - New users

Метрика **New users** не разграничивает нескольких пользователей, использующих одно устройство с одной установкой приложения.

## Распределение по версиям приложения

Диаграмма **App version distribution** отражает темп принятия версий, сравнивая количество сессий приложения, исходящих из разных версий. Особенно полезна после выпуска новых версий — позволяет отслеживать распространение последней версии.

![Раздел User experience - App versions](https://dt-cdn.net/images/app-version-distribution-848-03883e6518.png)

## Топ-3 действия

В списке отображаются три топовых действия по длительности и частоте пользовательских действий.

![Раздел User experience - Top 3 actions](https://dt-cdn.net/images/user-experience-top-3-actions-848-b804aa48cd.png)

### Страница обзора пользовательского действия

Из списка **Top 3 actions** на странице обзора приложения выберите пользовательское действие для открытия страницы его обзора.

![Страница обзора пользовательского действия](https://dt-cdn.net/images/user-action-detail-page-2518-a51f30ad70.png)

Страница обзора пользовательского действия

На этой странице вы можете пометить действие как ключевое, выполнить waterfall-анализ и просмотреть рейтинг Apdex и производительность действия.

Также доступны списки зафиксированных ошибок и ошибок веб-запросов для текущего действия. Выберите интересующую ошибку, чтобы открыть её [страницу деталей](#error-details-page).

### Многомерный анализ пользовательских действий

Под списком **Top 3 actions** на странице обзора приложения нажмите **Analyze performance** для перехода на многомерную страницу **User action analysis**.

Многомерный анализ позволяет детально изучить пользовательские действия и просматривать данные в нескольких измерениях. На странице **User action analysis** вы можете создавать метрики, просматривать ключевые действия пользователей и изучать топ-100 действий по различным критериям. Также можно анализировать приложение по операционной системе, геолокации, версии приложения и другим параметрам.

### Ошибки веб-запросов и зафиксированные ошибки

На многомерной странице **User action analysis** также доступен список ошибок веб-запросов или зафиксированных [ошибок](/managed/observe/digital-experience/rum-concepts/user-and-error-events#error "Узнайте о событиях пользователей и ошибках, а также о типах событий, фиксируемых Dynatrace."): выберите **Web request errors over time** или **Reported errors over time** в поле **Analyze by**. Список ошибок отображается под диаграммой. Разверните интересующую ошибку и нажмите **Analyze error** для открытия страницы её деталей.

![Анализ ошибки из раздела MDA](https://dt-cdn.net/images/analyze-error-from-mda-2518-90cecae0cc.png)

Анализ ошибки из раздела MDA

#### Страница деталей ошибки

Страница деталей ошибки предоставляет ценную информацию об ошибках вашего приложения.

Страница деталей ошибки веб-запроса

Страница деталей зафиксированной ошибки

![Страница деталей ошибки веб-запроса](https://dt-cdn.net/images/web-request-error-details-page-1772-928bb94d69.png)

Страница деталей ошибки веб-запроса

![Страница деталей зафиксированной ошибки](https://dt-cdn.net/images/reported-error-details-page-new-1644-60289f33a5.png)

Страница деталей зафиксированной ошибки

На странице отображаются детали ошибки: расчётное количество ошибок, провайдер (для ошибки запроса), технология (для зафиксированной ошибки) и другое. Также перечислены затронутые сессии и действия пользователей — выберите действие или сессию для просмотра их деталей. Разбивка по распределению показывает относительную частоту операционных систем, версий ОС, версий приложения и устройств; разбивка по странам показывает все затронутые страны и соответствующую частоту ошибок.

Для ошибок веб-запросов также можно [настроить правила исключения ошибок](/managed/observe/digital-experience/custom-applications/additional-configuration/web-request-errors-custom "Прекратите считать определённые HTTP-коды ответов ошибками в пользовательских приложениях."). Нажмите **Edit error exclusion rules** для перехода на соответствующую страницу настроек.

## Географические регионы

Карта мира **Geographic regions** на странице обзора приложения визуализирует все географические регионы, из которых приходят ваши пользователи. Для каждой страны можно просмотреть количество активных сессий, активных пользователей и сбоев, а также общий рейтинг Apdex.

![Раздел User experience - Geo regions](https://dt-cdn.net/images/user-experience-geo-regions-848-7ae15779b9.png)

Раздел User experience - Geo regions

Если на карте мира отсутствуют данные, возможно, необходимо [сопоставить внутренние IP-адреса с местоположениями](/managed/observe/digital-experience/custom-applications/additional-configuration/map-internal-ip-addresses-to-locations-custom "Настройте Dynatrace на использование локальных адресов для определения местонахождения пользователей ваших пользовательских приложений.").
