# Анализ веб-запросов в пользовательских приложениях

* How-to guide
* 1-min read
* Published Jan 30, 2023

С помощью клиентского мониторинга Dynatrace вы можете оценивать производительность сервисов первой и третьей сторон с точки зрения доступности, времени отклика и частоты ошибок.

Чтобы отслеживать веб-запросы вашего приложения:

1. Перейдите в **Frontend**.
2. Выберите приложение для анализа.
3. На странице обзора приложения выберите плитку **Web requests**.

Плитка **Web requests** отображает общее количество веб-запросов и частоту ошибок за выбранный временной диапазон.

![Плитка Web requests на странице обзора приложения](https://dt-cdn.net/images/web-requests-tile-mobile-app-2-852-01bbe5846b.png)

Плитка Web requests на странице обзора приложения

Информация о частоте запросов, времени запросов и топ-провайдерах доступна в разделах **Web requests** и **Top providers**.

## Web requests - Error rate

Эта диаграмма сравнивает частоту веб-запросов (количество запросов в минуту) с частотой [ошибок](/managed/observe/digital-experience/rum-concepts/user-and-error-events#error "Узнайте о событиях пользователей и ошибках, а также о типах событий, фиксируемых Dynatrace.") за выбранный временной диапазон.

![Вкладка Web requests - Error rate](https://dt-cdn.net/images/web-requests-error-rate-790-fb758ee8a4.png)

Вкладка Web requests - Error rate

## Web requests - Request time

Эта диаграмма сравнивает частоту веб-запросов со временем запросов за выбранный временной диапазон.

![Вкладка Web requests - Request time](https://dt-cdn.net/images/web-requests-request-time-790-aa8db1c006.png)

Вкладка Web requests - Request time

## Топ-провайдеры

Этот список включает HTTP-домены с наибольшим количеством исходящих запросов, генерируемых вашим приложением за выбранный временной диапазон.

![Список топ-провайдеров](https://dt-cdn.net/images/top-providers-mobile-app-790-bb9125360d.png)

Список топ-провайдеров

Чтобы просмотреть дополнительную информацию о провайдере, выберите его из списка. Откроется страница деталей провайдера с данными о частоте запросов, времени запросов и частоте ошибок.

* Перейдите на вкладку **Web requests** для изучения информации об отдельном веб-запросе: частота запросов, продолжительность, размер, частота ошибок и другие детали.
* Перейдите на вкладку **Errors** для просмотра списка ошибок веб-запросов данного провайдера. Выберите нужную ошибку для открытия [страницы деталей ошибки веб-запроса](/managed/observe/digital-experience/session-segmentation/new-user-sessions#error-details-page "Узнайте о сегментации и атрибутах фильтрации сессий пользователей.").
