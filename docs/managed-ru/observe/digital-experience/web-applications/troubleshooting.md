# Устранение неполадок RUM для веб-приложений

# Устранение неполадок RUM для веб-приложений

* Reference
* 2-min read
* Updated on Jul 07, 2023

На этой странице описано, что делать, если Real User Monitoring (RUM) не работает в вашей среде.

Прежде чем приступать к устранению неполадок, проверьте [страницу проверки работоспособности](/managed/observe/digital-experience/web-applications/initial-setup/app-health-check "The Application health check page allows you to analyze the health of your application, see which RUM JavaScript versions are currently in use, or confirm that the RUM JavaScript is injected correctly.") для вашего приложения.

## Общее устранение неполадок

Для общего устранения неполадок обратитесь к разделу [Web applications: Issues with RUM JavaScript](https://dt-url.net/up03l19) в сообществе Dynatrace.

## Устранение неполадок, связанных с пользовательскими действиями

Для устранения неполадок, связанных с пользовательскими действиями, обратитесь к следующим страницам в сообществе Dynatrace.

* [User action data is missing](https://dt-url.net/zqa3lef)
* [Web request isn't linked to its user action](https://dt-url.net/io63ls2)
* [Sessions are randomly split into shorter sessions](https://dt-url.net/cb23ley)
* [Source actions are missing for subdomain navigations](https://dt-url.net/xi43lit)

## Устранение неполадок, связанных с метаданными

Если вы столкнулись с проблемой, связанной с метаданными, обратитесь к разделу [Web applications: Sessions are not correctly tagged or metadata is missing](https://dt-url.net/rp83l5k) в сообществе Dynatrace.
