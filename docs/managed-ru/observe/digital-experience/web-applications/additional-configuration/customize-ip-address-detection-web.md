# Настройка определения IP-адресов для веб-приложений

# Настройка определения IP-адресов для веб-приложений

* How-to guide
* 1-min read
* Updated on Mar 21, 2024

Когда веб-запросы и RUM-маяки отправляются напрямую на инструментированный сервер, Dynatrace определяет IP-адреса устройств конечных пользователей через сокетные соединения.

При использовании неинструментированных компонентов, таких как балансировщики нагрузки, CDN или прокси, удалённый IP-адрес отличается от исходного. В таких случаях Dynatrace также проверяет определённые HTTP-заголовки. Эти заголовки чаще всего используются для идентификации исходного IP-адреса, когда клиент подключается к веб-серверу через HTTP-прокси, CDN или балансировщик нагрузки.

Список заголовков, которые Dynatrace использует для определения клиентских IP-адресов приложений, доступен в разделе **Settings** > **Web and mobile monitoring** > **IP determination**. Dynatrace обрабатывает эти заголовки в определённом порядке, который можно изменить, а также добавить собственные заголовки.

## Управление заголовками по умолчанию

Можно обновлять имена заголовков, удалять ненужные заголовки и изменять порядок их обработки Dynatrace.

1. Перейдите в **Settings** > **Web and mobile monitoring** > **IP determination**.
2. Управляйте заголовками:

   * Чтобы переименовать заголовок, выберите **Expand row** ![Expand row](https://dt-cdn.net/images/expand-row-icon-9c4950fc2e.svg "Expand row") и введите новое имя заголовка.
   * Чтобы удалить заголовок, выберите **Delete row** ![Remove](https://dt-cdn.net/images/remove-icon-105c6a04c2.svg "Remove").
   * Чтобы изменить приоритет заголовка, зажмите **Drag row** ![Drag handle](https://dt-cdn.net/images/drag-handle-turquoise-600-1aa0e5ea00.svg "Drag handle") рядом с именем заголовка и переместите его вверх или вниз.

## Добавление пользовательских заголовков клиентского IP-адреса

Если клиентские IP-адреса находятся в другом заголовке, создайте пользовательское правило, чтобы Dynatrace мог правильно их определять.

Чтобы добавить пользовательский заголовок клиентского IP-адреса:

1. Перейдите в **Settings** > **Web and mobile monitoring** > **IP determination**.
2. Выберите **Create custom client IP header**.
3. Введите имя заголовка.

Можно создать до 10 пользовательских заголовков клиентского IP-адреса на среду.

Добавление пользовательского заголовка клиентского IP-адреса может потребоваться в одном из следующих случаев.

* Заголовок, содержащий клиентские IP-адреса, отсутствует в списке заголовков в разделе **Settings** > **Web and mobile monitoring** > **IP determination**.
* Клиентские IP-адреса для приложений хранятся в проприетарном заголовке.
* Клиентские IP-адреса определяются неправильно, например, в Dynatrace отображаются IP-адреса балансировщиков нагрузки.

## Причины некорректного определения клиентских IP-адресов

Существует несколько причин, по которым для приложения могут отображаться неправильные клиентские IP-адреса. Подробные инструкции приведены в разделах выше.

| Проблема | Возможное решение |
| --- | --- |
| Клиентские IP-адреса для приложений находятся в заголовке, отсутствующем в нашем списке. | Добавьте пользовательский заголовок клиентского IP-адреса. |
| Указан неверный пользовательский заголовок клиентского IP-адреса. | Обновите имя заголовка. |
| Указанный пользовательский заголовок клиентского IP-адреса фактически не содержит IP-адресов. | Укажите другой пользовательский заголовок клиентского IP-адреса. |
| Порядок обработки заголовков неверен. | Измените порядок обработки заголовков. |
| Инфраструктура не настроена для добавления заголовка с клиентскими IP-адресами. | Перенастройте инфраструктуру. |
| Заголовок с клиентскими IP-адресами не входит в состав запросов маяков.[1](#fn-1-1-def) | Перенастройте инфраструктуру. |

1

Заголовки определения IP-адреса в обычных запросах приложения не используются для определения IP-адресов собранных данных RUM, даже если они фиксируются как трассировки и эти трассировки коррелируют с пользовательскими действиями.
