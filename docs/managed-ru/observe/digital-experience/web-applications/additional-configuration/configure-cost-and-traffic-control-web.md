# Настройка управления стоимостью и трафиком для веб-приложений

* How-to guide
* 1-min read
* Published Sep 29, 2021

По умолчанию Dynatrace фиксирует все пользовательские действия и пользовательские сессии для анализа. Такой подход обеспечивает полную видимость производительности приложения и клиентского опыта. С помощью параметра **Cost and traffic control** можно снизить детализацию анализа пользовательских действий и сессий, указав Dynatrace фиксировать меньший процент пользовательских сессий.

Несмотря на то что этот параметр позволяет сократить [стоимость мониторинга](/managed/license/monitoring-consumption-classic/digital-experience-monitoring-units "Understand how Dynatrace Digital Experience Monitoring consumption is calculated based on DEM units."), он также снижает видимость того, как клиенты используют ваши веб-приложения. Например, значение `10%` означает, что Dynatrace будет анализировать только каждую десятую пользовательскую сессию.

Чтобы ограничить количество анализируемых пользовательских сессий:

1. Перейдите в **Web**.
2. Выберите приложение, которое нужно настроить.
3. В правом верхнем углу страницы обзора приложения выберите **More** (**…**) > **Edit**.
4. В настройках приложения выберите **General settings** > **Enablement and cost control**.
5. В разделе **Real User Monitoring** введите значение менее `100` в поле **Cost and traffic control**. Значение по умолчанию в этом поле — `100`.

При заданном параметре Dynatrace анализирует равномерно распределённое количество пользовательских сессий, соответствующее указанному вами проценту.
