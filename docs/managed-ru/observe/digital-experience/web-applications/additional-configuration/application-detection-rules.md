# Проверка правил обнаружения приложений

* Explanation
* 1-min read
* Published Apr 03, 2020

С помощью правил обнаружения приложений можно определить транзакции, составляющие ваши RUM-приложения. Пользовательские действия, обнаруженные Dynatrace, группируются для представления приложения с точки зрения пользователя, что обеспечивает автономный мониторинг и анализ в Dynatrace.

Теперь можно проверить, обнаруживают ли заданные правила обнаружения трафик с нужных URL и правильно ли они отражают предназначенные для них приложения. Проверив URL любого автоматически инжектированного RUM-приложения, вы можете:

* увидеть, какие именно правила обнаружения соответствуют введённому URL;
* убедиться, что RUM включён для отслеживаемого приложения;
* быстро получить доступ к настройке правила обнаружения, нажав кнопку **Edit** этого правила;
* управлять настройкой и тестированием правил обнаружения приложений.

Чтобы лучше понять заданные правила обнаружения приложений и их влияние на ваше приложение:

1. Перейдите в **Settings** > **Web and mobile monitoring** > **Application detection**.
2. В разделе **Check your existing detection rules** введите URL и нажмите **Check URL**.

Можно создать до 1000 правил обнаружения приложений на одну среду.
