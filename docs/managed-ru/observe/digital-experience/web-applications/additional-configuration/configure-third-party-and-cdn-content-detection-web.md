# Настройка обнаружения ресурсов первой, сторонних сторон и CDN для веб-приложений

* How-to guide
* 1-min read
* Updated on Jun 30, 2022

Dynatrace распознаёт более 1000 сторонних поставщиков контента, включая Google, Amazon, Twitter и LinkedIn. Типичный сторонний контент включает виджеты Facebook и Twitter, gravatars и аналогичные ресурсы. Крайне важно видеть, как сторонние поставщики контента влияют на ваши приложения, и понимать, является ли такой контент первопричиной замедлений.

Сети доставки контента (CDN), как правило, находятся под вашим контролем. Ваше приложение может использовать CDN для хранения и предоставления часто запрашиваемого статического и динамического контента, что необходимо для ускорения загрузки в определённых географических регионах.

## Разница между сторонними поставщиками и CDN

Разница между сторонними поставщиками контента и CDN заключается в степени вашего контроля над данными.

* Если вы используете внешние данные, которыми не управляете, контент поступает от стороннего поставщика.
* Если вы используете внешний ресурс для доставки статического или динамического контента в приложение, например изображений или CSS, рассматривайте этот сервис как CDN.

## Просмотр ведущих поставщиков приложения

Ведущих поставщиков приложения и подробную информацию о каждом поставщике можно просмотреть на странице обзора приложения. Подробнее см. раздел [Анализ производительности > Ресурсы](/managed/observe/digital-experience/web-applications/analyze-and-use/performance-analysis#resources "Understand the available types of performance analysis that are provided by Dynatrace.").

## Управление поставщиками среды

### Просмотр полного списка поставщиков

Dynatrace обнаруживает множество сторонних поставщиков контента. Если Dynatrace не находит имя хоста в списке известных сторонних поставщиков, он помечает ресурс как CDN. Dynatrace также может определить, какие домены обслуживает ваша организация, и классифицировать их как первую сторону.

Чтобы увидеть полный список поставщиков, обнаруженных для вашей среды, перейдите в **Settings** > **Web and mobile monitoring** > **Provider breakdown**.

* Добавленные вручную поставщики отображаются непосредственно под кнопкой **Add custom provider item**. Можно [добавить поставщика](#add-provider-manually) и пометить его как первую сторону, стороннего поставщика или CDN.
* Автоматически обнаруженные CDN и сторонние поставщики отображаются в разделе **Auto detected providers**.

### Добавление поставщика вручную

Если поставщик не найден в списке автоматически обнаруженных поставщиков или необходимо переопределить правила автообнаружения, можно добавить поставщика вручную.

Чтобы добавить поставщика вручную:

1. Перейдите в **Settings** > **Web and mobile monitoring** > **Provider breakdown**.
2. Выберите **Add custom provider item**.
3. Введите **Resource name** и задайте **Resource type**, указав, является ли ресурс первой стороной, сторонним поставщиком или CDN.
4. Необязательно: укажите URL для значка бренда поставщика.
5. Выберите **Add name pattern** и введите шаблон для доменного имени поставщика. При необходимости можно указать несколько шаблонов.
6. Необязательно: включите **Submit this provider-pattern to improve auto-detection**, если считаете, что Dynatrace должен автоматически обнаруживать этот ресурс как сторонний.

Начиная с версии Dynatrace 1.240, автообнаружение поставщиков первой стороны включено по умолчанию и не может быть деактивировано. Если необходимо переопределить автоматически обнаруженных поставщиков первой стороны, [добавьте поставщика вручную](#add-provider-manually).

### Изменение порядка добавленных вручную поставщиков

Список добавленных вручную поставщиков обрабатывается в порядке их появления в списке. Поэтому поставщик с более конкретным шаблоном домена должен стоять первым. Например, `api.example.com` должен стоять перед `example.com`.

Чтобы изменить порядок добавленных вручную поставщиков:

1. Перейдите в **Settings** > **Web and mobile monitoring** > **Provider breakdown**.
2. Найдите список добавленных вручную поставщиков под кнопкой **Add custom provider item**.
3. Используйте **Drag handle** ![Drag handle](https://dt-cdn.net/images/drag-handle-turquoise-600-1aa0e5ea00.svg "Drag handle") для изменения приоритета поставщика.
