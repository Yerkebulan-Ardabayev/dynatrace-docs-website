# Определение целей конверсии

* How-to guide
* 1-min read
* Published Jul 15, 2019

Для конкретных пользовательских действий можно определять цели конверсии, чтобы понять, насколько успешно достигаются ключевые показатели конверсии — например, успешное оформление заказа, подписка на рассылку или запрос демо. Можно добавлять цели для достижения определённых пользовательских действий или URL назначения, а также для информации о сессии — например, для сессий с более чем 10 пользовательскими действиями. В последнем примере пользователю потребуется выполнить не менее 10 пользовательских действий в рамках одной сессии для достижения этой цели конверсии.

Рекомендуется тщательно отслеживать эти пользовательские действия — не только с технической точки зрения, чтобы убедиться, что всё работает корректно и продолжительность действий приемлема, но и чтобы видеть, сколько клиентов фактически использует данные функции. Это можно сделать с помощью подхода [ключевых действий](/managed/observe/digital-experience/rum-concepts/user-actions#key-user-actions "Learn what user actions are and how they help you understand what users do with your application.") и целей конверсии.

Цели конверсии определяются для каждого приложения отдельно. Они могут основываться на URL назначения, конкретном пользовательском действии, длительности сессии или количестве действий в сессии.

Чтобы определить цель конверсии для приложения:

1. Перейдите в **Web**.
2. Выберите приложение, которое нужно настроить.
3. В правом верхнем углу страницы обзора приложения выберите **More** (**…**) > **Edit**.
4. В настройках приложения выберите **Behavior analytics** > **Conversion goals**.
5. Выберите **Add goal**.
6. Введите имя для цели конверсии и настройте остальные параметры.
7. Выберите **Add goal**.

На одно приложение можно определить не более 20 целей конверсии.
