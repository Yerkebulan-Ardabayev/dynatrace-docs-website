# Работа с ключевыми метриками производительности

* How-to guide
* 3-min read
* Published Jun 05, 2019

Dynatrace предлагает расширенный набор [ключевых метрик производительности](/managed/observe/digital-experience/rum-concepts/user-action-metrics#kpm "Learn what metrics Dynatrace calculates for user actions and find out what each metric indicates."), на основе которых можно настраивать Real User Monitoring.

## Выбор правильной метрики производительности

Для корректной аналитики мониторинга важно выбирать правильную метрику производительности для каждого условия приложения.

Для многих традиционных веб-приложений **User action duration** считается наилучшей метрикой для оптимизации веб-производительности, поскольку она фокусируется на времени от ввода пользователя до полной загрузки страницы. Однако одностраничные приложения (SPA) на основе Angular, Ember, React и других JavaScript-фреймворков не зависят от загрузки новой страницы после каждого нажатия. Поэтому стремление к быстрой загрузке «видимой без прокрутки части» может привести к высокому общему времени пользовательских действий. В результате такие метрики, как **Visually complete** или **Speed index**, могут точнее отражать реальное время, в которое конечный пользователь может выполнить следующее действие. С точки зрения конечного пользователя веб-приложения становятся удобны для использования после того, как готовы к работе определённые элементы — например, выбор даты или автодополнение в поле поиска.

Также существуют приложения, в которых интерфейс не так важен, как взаимодействие с приложением. Пример — приложение для операторов call-центра или справочного стола, где пользователи являются экспертами и работают с горячими клавишами. Для таких приложений время взаимодействия важнее полной загрузки изображений, поэтому метрика **Long tasks** и **DOM interactive** могут быть наиболее подходящими.

Правильный выбор ключевой метрики производительности также позволяет адаптироваться к различным уровням производительности разных функций приложения. Например, клиент может быть готов ждать пять секунд после нажатия **Confirm booking**, но разочаруется, ожидая пять секунд появления поля поиска.

С ключевыми метриками производительности и [ключевыми пользовательскими действиями](/managed/observe/digital-experience/rum-concepts/user-actions#key-user-actions "Learn what user actions are and how they help you understand what users do with your application.") можно выбирать оптимальную метрику производительности и ожидаемые цели производительности для каждого отдельного пользовательского действия. Ключевые метрики производительности Dynatrace позволяют реагировать на такие переменные условия для каждого отслеживаемого приложения и пользовательского действия.

## Использование ключевых метрик производительности для мониторинга производительности приложений

Ключевые метрики производительности доступны по умолчанию, при этом **Visually complete** выбрана в качестве ключевой метрики для действий загрузки и XHR-действий. **User action duration** установлена в качестве метрики по умолчанию для пользовательских действий. Ключевые метрики производительности можно изменить для каждого типа действий.

Чтобы выбрать ключевые метрики производительности для приложения:

1. Перейдите в **Web**.
2. Выберите приложение, которое нужно настроить.
3. В правом верхнем углу страницы обзора приложения выберите **More** (**…**) > **Edit**.
4. Выберите **General settings** > **Load actions** / **XHR actions** / **Custom actions**.
5. В разделе **Key performance metric** выберите ключевую метрику производительности, наилучшим образом отражающую пользовательский опыт для действий загрузки и XHR-действий.

В ситуациях, когда необходимо адаптировать ключевую метрику производительности для различных функций приложения, можно выбрать другую ключевую метрику для ключевых пользовательских действий приложения.

Чтобы выбрать ключевые метрики производительности для ключевых пользовательских действий:

1. Перейдите в **Web**.
2. Выберите приложение для настройки.
3. В разделе **Top 3 user actions** выберите **View full details**.
4. Прокрутите страницу вниз и перейдите на вкладку **Key user actions**.
5. Выберите нужное ключевое пользовательское действие.
6. На странице сведений о ключевом пользовательском действии выберите **More** (**…**) > **Edit**.
7. Выберите ключевую метрику производительности, наилучшим образом отражающую пользовательский опыт для данного ключевого пользовательского действия.
