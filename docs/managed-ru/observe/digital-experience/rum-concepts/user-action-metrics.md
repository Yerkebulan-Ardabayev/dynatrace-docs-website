# Метрики пользовательских действий

# Метрики пользовательских действий

* Explanation
* 8-min read
* Updated on Mar 20, 2023

Для веб-приложений Dynatrace рассчитывает широкий набор метрик [пользовательских действий](/managed/observe/digital-experience/rum-concepts/user-actions "Узнайте, что такое пользовательские действия и как они помогают понять, что пользователи делают в вашем приложении.").

Core web vitals

Следующие метрики являются Core web vitals и доступны только для действий загрузки:

* **Largest contentful paint**
* **Cumulative layout shift**
* **First input delay**

## Метрики milestone

Метрики milestone представляют события, происходящие в определённый момент времени в ходе пользовательского действия. Однако эти метрики измеряются в секундах или миллисекундах, поскольку рассчитываются относительно времени начала пользовательского действия. Иными словами, время начала пользовательского действия служит точкой отсчёта для измерения метрик milestone. Например, значение 47 мс для **Navigation start** означает, что это событие произошло через 47 мс после времени начала действия.

| Метрика | Описание | Применимо к |
| --- | --- | --- |
| **Navigation start** | Момент, когда пользовательский агент завершает подсказку о выгрузке предыдущей страницы. Если предыдущей страницы нет, значение совпадает с `PerformanceTiming.fetchStart`. | Действия загрузки |
| **Request start** | Время до того, как пользовательский агент отправит запрос на получение страницы с сервера, из кэша приложения или локального ресурса. Если транспортный уровень даёт сбой и соединение восстанавливается, метрика устанавливается в значение, соответствующее новому запросу. | Действия загрузки, XHR-действия |
| **Time to first byte** | Время до получения первого байта ответа от сервера, кэша приложения или локального ресурса | Действия загрузки, XHR-действия |
| **HTML downloaded** | Время до получения пользовательским агентом последнего байта ответа или закрытия транспортного соединения (в зависимости от того, что наступит раньше). Для XHR-действий эта метрика называется **Response end**. | Действия загрузки, XHR-действия |
| **DOM interactive** | Время до перехода страницы в состояние «interactive» и готовности к приёму ввода | Действия загрузки |
| **Speed index**[1](#fn-1-1-def) | Показатель, измеряющий скорость отрисовки видимых частей страницы | Действия загрузки |
| **DOM content loaded** | Время до срабатывания пользовательским агентом события `DOMContentLoaded` на странице. Событие `DOMContentLoaded` выполняется после загрузки и полного разбора базового HTML страницы. Оно не ожидает завершения загрузки дополнительных ресурсов (например, стилей, субфреймов и изображений). | Действия загрузки |
| **First paint**[2](#fn-1-2-def) | Время до отрисовки первого элемента, отличного от фона по умолчанию | Действия загрузки |
| **First contentful paint**[3](#fn-1-3-def) | Время до отрисовки первого контента (текста или изображений) | Действия загрузки |
| **Largest contentful paint**[4](#fn-1-4-def) | Время до отрисовки наибольшего элемента в области просмотра | Действия загрузки |
| **Visually complete**[1](#fn-1-1-def) | Время полной отрисовки контента в области просмотра | Действия загрузки, XHR-действия |
| **DOM complete** | Время до перехода страницы в состояние «complete» | Действия загрузки |
| **Load event start** | Время до начала события загрузки страницы | Действия загрузки |
| **Load event end** | Время до завершения события загрузки страницы | Действия загрузки |
| **User action duration** | Время, необходимое для завершения загрузки страницы. Включает время загрузки XHR-запросов, инициированных до `loadEventEnd`, а также время загрузки динамических ресурсов и выполнения скриптов, запущенных изменениями DOM. | Действия загрузки, XHR-действия, пользовательские действия |
| **First input start** | Момент первого взаимодействия пользователя со страницей, например, клик по элементу управления | Действия загрузки |
| **First input delay**[5](#fn-1-5-def) | Время от первого взаимодействия со страницей до момента, когда пользовательский агент может ответить на это взаимодействие | Действия загрузки |

1

**Speed index** и **Visually complete** доступны только в браузерах, поддерживающих [mutation observers](https://developers.google.com/web/updates/2012/02/Detect-DOM-changes-with-Mutation-Observers): Microsoft Internet Explorer 11, Microsoft Edge 15+, Firefox 57+ и Google Chrome 61+.

2

**First paint** поддерживается во всех браузерах, кроме Internet Explorer.

3

**First contentful paint** доступна только для [браузеров на основе Chromium](https://www.chromium.org/Home).

4

**Largest contentful paint** доступна для браузеров на основе Chromium. Измеряется с использованием [API Google](https://web.dev/lcp/#measure-lcp-in-javascript).

5

Для браузеров на основе Chromium **First input delay** измеряется с использованием [API Google](https://web.dev/fid/#measure-fid-in-javascript). Для Firefox, Safari и Internet Explorer 9+ метрика измеряется через реализацию Dynatrace, которая прослушивает события типа `click`, `mousedown`, `keydown` или `touchstart` и вычисляет задержку с момента регистрации таких событий RUM JavaScript до их срабатывания.

## Метрики phase

В отличие от метрик milestone, метрики phase подразумевают длительность.

| Метрика | Описание | Применимо к |
| --- | --- | --- |
| **Blocking** | Время между получением пользовательским агентом запроса на загрузку ресурса и фактическим началом загрузки. Включает время ожидания свободного TCP-сокета, создания записей в дисковом кэше и согласования прокси. | Действия загрузки |
| **Application cache** | Время проверки соответствующих кэшей приложения. Включает время до установки соединения с сервером. | Действия загрузки |
| **DNS lookup** | Время разрешения имени хоста для целевого URL | Действия загрузки |
| **TCP connect** | Время установки TCP-соединения с сервером (включая SSL) | Действия загрузки |
| **Secure connect** | Время защиты установленного соединения с сервером. Включает SSL-рукопожатие и SOCKS. | Действия загрузки |
| **Request** | Время запроса страницы с сервера до получения первого байта | Действия загрузки |
| **Response** | Время получения ответа | Действия загрузки |
| **Processing** | Время между загрузкой DOM и началом события загрузки | Действия загрузки |
| **OnLoad** | Время обработки события загрузки | Действия загрузки |
| **OnDOMContentLoaded** | Время выполнения обработчиков `onDomContentLoaded` | Действия загрузки |
| **Callback** | Время выполнения XHR-коллбэков | Действия загрузки |
| **Redirect time** | Время следования HTTP-перенаправлениям | Действия загрузки |
| **Long tasks**[1](#fn-2-1-def) | Суммарное время всех длинных JavaScript-задач (более 50 мс). Метрика измеряется от **Navigation start** на протяжении всей длительности пользовательского действия. | Действия загрузки |

1

Вместо метрики [**Total blocking time**](https://web.dev/tbt/), которую мы решили не вводить из-за технических ограничений, можно использовать метрику **Long tasks**. Также обратите внимание, что метрика **Blocking**, отображаемая на waterfall, не связана с **Total blocking time** и доступна только для определённых запросов.

## Ключевые метрики производительности

Из перечисленных выше метрик Dynatrace выделяет некоторые в качестве ключевых. Ключевые метрики производительности позволяют выбрать оптимальную метрику пользовательского опыта для каждого приложения. Эти ключевые метрики дают ценные сведения о производительности, способствующие успеху вашего цифрового бизнеса. Ключевые метрики производительности включают:

* User action duration
* Visually complete
* Speed index
* DOM interactive
* Load event end
* Load event start

* HTML downloaded / Response end
* Time to first byte
* Largest contentful paint
* Cumulative layout shift[1](#fn-3-1-def)
* First input delay (только RUM web)

1

**Cumulative layout shift** — показатель, измеряющий непредвиденное смещение видимых элементов веб-страницы. Доступна для [браузеров на основе Chromium](https://www.chromium.org/Home) и измеряется с использованием [API Google](https://web.dev/cls/#measure-cls-in-javascript).

Dynatrace позволяет [выбрать подходящую метрику производительности для каждого приложения](/managed/observe/digital-experience/web-applications/analyze-and-use/work-with-key-performance-metrics "Узнайте, как использовать подходящие ключевые метрики производительности для оптимизации данных о пользовательском опыте в каждом приложении.").

## Отсутствующие измерения метрик

Измерения метрик могут отсутствовать в следующих случаях:

#### Visually complete

* В ходе действия загрузки пользователь выбирает другую ссылку, которая запускает новое действие загрузки до завершения исходного события загрузки страницы. Второе действие загрузки прерывает расчёт **Visually complete** для первого, поэтому измерение недоступно.  
  В более ранней версии **Visually complete** Dynatrace использовала длительность пользовательского действия в качестве резервного измерения. Это искажало временной ряд, поэтому такие случаи теперь исключаются для обеспечения более точных агрегатов.
* Сразу после действия загрузки выполняется позднее перенаправление.
* В процессе расчёта достигается [тайм-аут Visually complete](/managed/observe/digital-experience/web-applications/analyze-and-use/how-to-use-visually-complete-and-speed-index-metrics#config-enhanced-vc "Узнайте, как использовать метрики «Visually complete» и «Speed index».").

#### Core web vitals

* API браузера не срабатывает и не предоставляет измерение метрики до завершения пользовательского действия (см. метрику **User action duration**).  
  В более ранних версиях Dynatrace Real User Monitoring эти значения отправлялись после завершения пользовательского действия. Такие значения не были значимыми и зачастую не были репрезентативными. Например, **First input delay (FID)** измеряет отзывчивость при загрузке страницы. Основной причиной высокого значения FID является выполнение JavaScript. Если пользователь ждал минуту после загрузки страницы перед выбором новой ссылки, большая часть обработки JavaScript уже завершилась бы, и полученное небольшое значение FID не было бы репрезентативным для реальной загрузки страницы. Такие измерения искажали общий агрегат метрики.
* Браузер пользователя не поддерживает API, используемые для измерения Core web vitals. Подробнее о поддержке браузерами см. в разделе [Метрики milestone](#milestone-metrics).
* Страница загружается или перезагружается в неактивной вкладке, и API браузера не возвращает значения.
* API браузера возвращает отрицательные значения.
