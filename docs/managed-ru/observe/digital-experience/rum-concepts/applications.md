# Приложения

# Приложения

* Explanation
* 4-min read
* Updated on Aug 18, 2025

В мониторинге реальных пользователей (RUM) мониторируемые приложения — это логические конструкции, на которые отображаются приложения клиентов: веб-сайты, мобильные приложения и другие объекты для мониторинга трафика реальных пользователей. Таким образом, подразумевается, что подобные клиентские приложения имеют интерфейс конечного пользователя. Типичные интерфейсы конечного пользователя включают браузерные интерфейсы веб-приложений и сайтов, а также интерфейсы мобильных приложений на iOS или Android, работающих на смартфонах или планшетах.

Интерфейс конечного пользователя определяет тип приложения, создаваемого в Dynatrace. Способ получения данных мониторинга и тип собираемых данных различаются для каждого типа приложения. Например, для веб-приложения, работающего в браузере, [RUM JavaScript](/managed/observe/digital-experience/web-applications/initial-setup/snippet-formats "Выберите формат RUM JavaScript-сниппета, наиболее подходящий для вашего случая") [внедряется автоматически](/managed/observe/digital-experience/web-applications/initial-setup/rum-injection "Настройте автоматическое внедрение RUM JavaScript в страницы ваших приложений") или [вставляется вручную](/managed/observe/digital-experience/web-applications/initial-setup/set-up-agentless-real-user-monitoring "Настройте мониторинг без агента для ваших веб-приложений.") в HTML-страницы приложения и начинает отслеживать каждое пользовательское действие.

## Определение RUM-приложения

Определение RUM-приложения для веб-приложений генерируется автоматически. В частности, если ваши веб-приложения работают на системах, где можно установить [OneAgent](/managed/ingest-from/dynatrace-oneagent "Ознакомьтесь с ключевыми концепциями, связанными с OneAgent, и узнайте, как установить и эксплуатировать OneAgent на различных платформах."), Dynatrace автоматически внедряет RUM JavaScript в HTML-страницы. Поэтому каждая [среда мониторинга](/managed/discover-dynatrace/get-started/monitoring-environment "Ознакомьтесь со средами мониторинга и узнайте, как с ними работать.") автоматически получает приложение по умолчанию с именем [«My web application».](/managed/observe/digital-experience/web-applications/initial-setup/define-your-applications-via-the-my-web-application-placeholder "Узнайте, как определять приложения с использованием предложенного, ручного подхода или правил обнаружения приложений.") Все RUM-данные по умолчанию назначаются этому приложению.

Для мобильных и пользовательских приложений определение генерируется при создании приложения и настройке мониторинга Dynatrace. Эти типы приложений не мониторируются автоматически.

## Поддерживаемые типы приложений

Dynatrace поддерживает различные типы приложений: веб- и мобильные, а также rich client-приложения, приложения для автомобилей и IoT-приложения с пользовательским взаимодействием. Каждый тип имеет разные возможности мониторинга и разный интерфейс в Dynatrace. Однако все типы объединяют общие концепции: [сессии пользователей](/managed/observe/digital-experience/rum-concepts/user-session "Узнайте, как определяется сессия пользователя, когда она начинается и заканчивается, как рассчитывается её продолжительность и многое другое.") и [пользовательские действия](/managed/observe/digital-experience/rum-concepts/user-actions "Узнайте, что такое пользовательские действия и как они помогают понять, что пользователи делают в вашем приложении.").

| **Тип внедрения** | Внедрение не требуется: события жизненного цикла, пользовательские действия и веб-запросы мониторируются автоматически. |

Пользовательские приложения охватывают все цифровые точки взаимодействия: от традиционных rich client-приложений до интеллектуальных IoT-приложений и даже Alexa Skills. Поддержка таких приложений реализована через [Dynatrace OpenKit](/managed/ingest-from/extend-dynatrace/openkit "Узнайте, как инструментировать приложение с помощью OpenKit, как использовать методы Dynatrace OpenKit API и многое другое.").

|  |  |
| --- | --- |
| **Интерфейс пользователя** | Любая цифровая точка взаимодействия: от rich client-приложений до IoT-приложений с пользовательским взаимодействием. Интерфейсом может быть даже аппаратное устройство, например Amazon Alexa. |
| **Подход к мониторингу** | Dynatrace RUM JavaScript |
| **Тип внедрения** | Внедрение выполняется клиентом с использованием [Dynatrace OpenKit](/managed/ingest-from/extend-dynatrace/openkit "Узнайте, как инструментировать приложение с помощью OpenKit, как использовать методы Dynatrace OpenKit API и многое другое.") для их технологии. |
| **Как начать** | [Инструментирование приложений с использованием Dynatrace OpenKit](/managed/ingest-from/extend-dynatrace/openkit/instrument-your-application-using-dynatrace-openkit "Узнайте, как использовать Dynatrace OpenKit для инструментирования всех нативных нетрадиционных и немобильных цифровых точек взаимодействия в вашей среде.") |

Dynatrace захватывает сессии пользователей пользовательских приложений как мобильные сессии. Встроенные метрики захватываются без указания того, является ли источник мобильным или пользовательским приложением. Подробнее см. [Встроенные метрики](/managed/analyze-explore-automate/metrics-classic/built-in-metrics "Изучите полный список встроенных метрик Dynatrace.").
