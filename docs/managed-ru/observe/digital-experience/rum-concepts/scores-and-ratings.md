# Оценки и рейтинги Dynatrace

* Overview
* 1-min read
* Published Jul 19, 2017

Dynatrace использует различные рейтинги и оценки для измерения производительности вашего приложения и качества пользовательского опыта.

Эти рейтинги особенно удобны, когда у вас много приложений и нужно быстро получить обзор их состояния или сравнить производительность. Существует ряд метрик, таких как **Visually complete** или количество ошибок, для подобных сравнений. Однако, поскольку приложения различаются по многим параметрам, прямое сравнение по конкретным метрикам не всегда информативно. Например, медианное значение **Visually complete** в 5 секунд для галереи изображений может быть вполне приемлемым для конечных пользователей, тогда как задержка в 2 секунды для текстового новостного приложения была бы неприемлемой.

В таких случаях сравнение только по сырым метрикам не оптимально. Оценки и индексы дают более полную картину, поскольку объединяют различные метрики и правила в единое число с учётом индивидуальных порогов, заданных для каждого приложения. Они позволяют по одному числу оценить, работают ли ваши приложения нормально и удовлетворены ли пользователи.

* [Рейтинги Apdex](/managed/observe/digital-experience/rum-concepts/scores-and-ratings/apdex-ratings "Узнайте, как Dynatrace использует Apdex для измерения удовлетворённости пользователей производительностью приложения.") — оценки уровня приложения для определения его производительности.
* [Оценка пользовательского опыта](/managed/observe/digital-experience/rum-concepts/scores-and-ratings/user-experience-score "Оценка пользовательского опыта — метрика, используемая для классификации сессий пользователей как Frustrating, Tolerable или Satisfying.") — оценка на уровне сессии, используемая для определения качества опыта в рамках единой сессии в нескольких приложениях.
