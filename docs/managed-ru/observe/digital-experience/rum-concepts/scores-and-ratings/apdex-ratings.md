# Рейтинги Apdex

# Рейтинги Apdex

* Explanation
* 3-min read
* Updated on Jan 27, 2023

Dynatrace рассчитывает [рейтинги Apdex](https://en.wikipedia.org/wiki/Apdex), предоставляя единую метрику, отражающую производительность вашего приложения и ошибки, влияющие на пользовательский опыт.

Apdex рассчитывается для каждого отдельного [пользовательского действия](/managed/observe/digital-experience/rum-concepts/user-actions "Узнайте, что такое пользовательские действия и как они помогают понять, что пользователи делают в вашем приложении.") и для каждого приложения в целом. Это обеспечивает быстрое понимание пользовательского опыта в вашем приложении.

Рейтинги Apdex по умолчанию в Dynatrace основаны на пороговых значениях, специфичных для каждого приложения.

| Балл | Уровень производительности |
| --- | --- |
| 0.94-1.0 | Excellent |
| 0.85-0.94 | Good |
| 0.7-0.85 | Fair |
| 0.5-0.7 | Poor |
| < 0.5 | Unacceptable |

Рейтинги Apdex можно использовать в качестве контрольных точек для сравнения двух приложений во времени, даже если временны́е пороги для этих приложений различаются.

Приложения обычно состоят из множества различных типов пользовательских действий. Например, может быть приемлемо, что сложный поиск занимает до 6 секунд, тогда как загрузка главной страницы того же приложения должна происходить менее чем за 2 секунды. Такие различия можно учесть, настроив разные пороги Apdex для различных типов пользовательских действий.

Подробнее о стандарте Apdex см. [Apdex References](https://www.apdex.org/index.php/documents/).

## Настройка параметров Apdex в Dynatrace

Вы можете настроить рейтинги Apdex в Dynatrace под конкретные требования вашего приложения. После настройки они дают быстрый и удобный способ оценки производительности всех отслеживаемых действий пользователей: значение `1.0` — идеальное; значения ниже `0.5` — неприемлемые. Рекомендуется определить подходящие временные пороги удовлетворённости пользователей и конфигурации влияния ошибок для каждого мониторируемого пользовательского действия.

Вы можете настроить параметры Apdex для вашего приложения и для его ключевых пользовательских действий.

### Настройка параметров Apdex для приложений

Вы можете изменить пороги Apdex для ваших [веб-](/managed/observe/digital-experience/web-applications/additional-configuration/configure-apdex-web "Настройте пороги производительности удовлетворённости пользователей для вашего веб-приложения и его ключевых действий."), [мобильных](/managed/observe/digital-experience/mobile-applications/additional-configuration/configure-apdex-mobile "Настройте пороги производительности удовлетворённости пользователей для вашего мобильного приложения и его ключевых действий.") и [пользовательских приложений](/managed/observe/digital-experience/custom-applications/additional-configuration/configure-apdex-custom "Настройте пороги производительности удовлетворённости пользователей для вашего пользовательского приложения и его ключевых действий.").

### Настройка параметров Apdex для ключевых пользовательских действий

Вы также можете изменить пороги Apdex для конкретного ключевого пользовательского действия в ваших [веб-](/managed/observe/digital-experience/web-applications/additional-configuration/configure-key-user-actions-web "Обозначьте действие как ключевое и настройте рейтинг Apdex для ключевых действий в веб-приложениях."), [мобильных](/managed/observe/digital-experience/mobile-applications/additional-configuration/configure-key-user-actions-mobile "Обозначьте действие как ключевое и настройте рейтинг Apdex для ключевых действий в мобильных приложениях.") и [пользовательских приложениях](/managed/observe/digital-experience/custom-applications/additional-configuration/configure-key-user-actions-custom "Обозначьте действие как ключевое и настройте рейтинг Apdex для ключевых действий в пользовательских приложениях.").

## Влияние ошибок на Apdex

Пользовательские действия с JavaScript-[ошибками](/managed/observe/digital-experience/rum-concepts/user-and-error-events#error "Узнайте о событиях пользователей и ошибках, а также о типах событий, фиксируемых Dynatrace.") оцениваются как **Frustrated**. Иногда пользовательские действия, которые на самом деле быстрые и не превышают порог Apdex, отображаются красным и оцениваются как **Frustrated**, потому что часть из них содержит JavaScript-ошибки.

То же касается ошибок запросов. HTTP-код ответа, нарушение CSP или ресурсный запрос, настроенный в параметрах как ошибка, приводит к оценке пользовательского действия как **Frustrated**.

Вы можете исключить ошибки из расчётов Apdex. См. [Настройка обнаружения ошибок для веб-приложений](/managed/observe/digital-experience/web-applications/additional-configuration/configure-errors "Настройте захват или игнорирование ошибок запросов, пользовательских и JavaScript-ошибок."), [Настройка параметров Apdex для мобильных приложений](/managed/observe/digital-experience/mobile-applications/additional-configuration/configure-apdex-mobile "Настройте пороги производительности удовлетворённости пользователей для вашего мобильного приложения и его ключевых действий.") и [Настройка параметров Apdex для пользовательских приложений](/managed/observe/digital-experience/custom-applications/additional-configuration/configure-apdex-custom "Настройте пороги производительности удовлетворённости пользователей для вашего пользовательского приложения и его ключевых действий.") для получения подробностей.

## Анализ на основе Apdex

Dynatrace упрощает анализ Apdex вашего приложения в различных измерениях. Вы можете проверить рейтинг Apdex для конкретного пользовательского действия, местоположения и приложения, а также просмотреть рейтинг Apdex для каждого пользовательского действия в одной сессии пользователя.

Подробнее см. [Контекстный анализ Apdex](/managed/observe/digital-experience/session-segmentation/apdex-analysis "Проверьте рейтинг Apdex для пользовательского действия, местоположения и приложения.").
