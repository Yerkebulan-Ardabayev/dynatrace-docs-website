# Session Replay

# Session Replay

* Overview
* 3-min read
* Updated on Jan 27, 2023

Session Replay — мощный инструмент, способный модернизировать вашу стратегию Digital Experience Monitoring (DEM). Вы можете использовать его для захвата и визуального воспроизведения полного цифрового опыта каждого пользователя.

Вы можете записывать взаимодействия клиентов с вашим приложением и воспроизводить каждый клик или тап в виде фильма. Session Replay также упрощает воспроизведение проблем в production для ваших команд QA, что позволяет разработчикам устранить разрыв между кодом и пользовательским опытом.

Session Replay помогает выявлять ошибки, требующие немедленного исправления, а также другие проблемы, например некорректно отображаемые страницы и бесконечные индикаторы загрузки. Вы также можете использовать Session Replay для обнаружения и анализа проблемных зон в приложении и улучшения его общей удобности.

Session Replay доступен для [веб-приложений](/managed/observe/digital-experience/session-replay/configure-session-replay-web "Настройте потребление мониторинга и параметры конфиденциальности данных для Session Replay."), [Android](/managed/observe/digital-experience/session-replay/session-replay-android "Настройте Session Replay для Android-приложений, чтобы узнать, какие действия выполняют пользователи.") и [iOS](/managed/observe/digital-experience/session-replay/session-replay-ios "Предварительные условия и процедура включения Session Replay для iOS-приложений.").

Session Replay не поддерживается для Cordova, React Native, Flutter, Xamarin и .NET MAUI.

## Применение Session Replay

### Детальный анализ ошибок

С помощью Session Replay вы можете детально изучить обнаруженные ошибки:

* Обнаруживать ошибки и другие проблемы.
* Понимать точную последовательность действий пользователя, приведших к ошибке.
* Оценивать серьёзность проблемы и её влияние на пользовательский опыт.
* Наблюдать за влиянием на клиентов, воспроизводя сессию, когда проблема не очевидна.

Разработчики могут использовать Session Replay для просмотра, анализа, воспроизведения и исправления ошибок.

Для детального анализа ошибок не требуется записывать все сессии. Вы можете использовать [управление стоимостью и трафиком для веб-приложений](/managed/observe/digital-experience/session-replay/configure-session-replay-web#cost-traffic-control "Настройте потребление мониторинга и параметры конфиденциальности данных для Session Replay."), чтобы записывать лишь часть сессий. Если анализируемая ошибка не слишком редкая, её можно обнаружить даже при записи 20% сессий.

К этим сессиям применяется стандартный [период хранения данных](/managed/manage/data-privacy-and-security/data-privacy/data-retention-periods "Проверьте сроки хранения данных различных типов.").

Возможность воспроизводить записанные сессии с настройками [маскирования](/managed/observe/digital-experience/session-replay/configure-session-replay-web#sr-masking "Настройте потребление мониторинга и параметры конфиденциальности данных для Session Replay.") или без них контролируется разрешениями. Разрешения доступны на уровне среды и зоны управления. Подробнее см. [Управление группами пользователей и разрешениями](/managed/manage/identity-access-management/permission-management/role-based-permissions#permissions "Разрешения на основе ролей").

### Разрешение жалоб

Поскольку Session Replay — лучший способ показать, что именно делал пользователь, он предоставляет средства для урегулирования жалоб клиентов. Используйте Session Replay для:

* Просмотра точного пути клиента в вашем приложении.
* Выявления конкретной проблемы, с которой столкнулся пользователь.
* Предоставления правильных инструкций.

Для урегулирования жалоб клиентов идеально записывать все сессии. Однако для экономии места рекомендуется установить меньший срок хранения.

### Анализ удобства использования

С помощью Session Replay вы можете обнаруживать и анализировать следующие проблемы:

* Дизайн UX недостаточно интуитивно понятен.
* Процесс слишком сложен, и пользователи покидают приложение на полпути.
* Приложение работает медленно, и пользователь многократно кликает или тапает, чтобы перейти на следующую страницу или экран.
* Приложение работает не так, как ожидается, во всех браузерах или на всех устройствах.
* Приложение предлагает пользователю изменить ориентацию устройства, но пользователь не понимает эту подсказку.

К этим сессиям применяется стандартный [период хранения данных](/managed/manage/data-privacy-and-security/data-privacy/data-retention-periods "Проверьте сроки хранения данных различных типов.").

## Технические ограничения для веб-приложений

Для веб-приложений Session Replay совместим со страничными приложениями, одностраничными приложениями (SPA) и приложениями, использующими IFrame. Однако применяются [определённые ограничения](/managed/observe/digital-experience/session-replay/session-replay-restrictions-web "Узнайте об ограничениях, применяемых к Session Replay.").
