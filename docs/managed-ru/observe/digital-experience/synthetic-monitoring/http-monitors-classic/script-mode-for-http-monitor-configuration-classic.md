# Script mode для настройки HTTP-мониторов

# Script mode для настройки HTTP-мониторов

* How-to guide
* 1-min read
* Published Jul 17, 2019

Помимо настройки в интерфейсе (**Visual mode**), вы можете использовать **Script mode** для настройки [HTTP-мониторов](/managed/observe/digital-experience/synthetic-monitoring/http-monitors-classic/create-an-http-monitor-classic "Узнайте, как создать HTTP-монитор для проверки производительности и доступности сайта."). В этом режиме доступен базовый JSON-скрипт монитора. Для опытных пользователей Synthetic Monitoring это значительно упрощает работу и ускоряет создание и управление мониторами. Используйте редактор скриптов, чтобы быстро находить конкретные события (шаги) или изменять локаторы по всему скрипту.

Режимы не ограничены: можно переключаться между интерфейсным режимом и режимом скрипта, нажимая **Visual mode**/**Script mode**.

## Доступ к скрипту

Чтобы изменить HTTP-монитор в режиме скрипта:

1. Перейдите в **Synthetic Classic** и переключитесь в режим отображения списком.
2. Установите флажок рядом с монитором, который хотите изменить, > нажмите **Edit** в нижнем левом углу.
3. Выберите вкладку **HTTP Requests** в меню настроек монитора слева.
4. Нажмите **Script mode** вверху.

## Редактирование скрипта

Вы можете скачать скрипт (**Download script as .json**) или скопировать его из редактора и отредактировать в любом текстовом редакторе. Также можно редактировать скрипт прямо в браузере. Это даёт следующие преимущества:

* Автодополнение: нажмите **Ctrl+Spacebar**, чтобы увидеть список подсказок.
* Подсветка синтаксиса упрощает написание кода скрипта.
* Мгновенная проверка ошибок: редактор сразу отображает предупреждение при любой ошибке. Наведите курсор на ошибку, чтобы узнать причину и получить рекомендацию по исправлению. Сохранение изменений невозможно до устранения всех ошибок.

Обратите внимание: все специальные символы и переносы строк необходимо экранировать обратным слешем (например, `\n` для новой строки, `\"` для двойной кавычки, `\t` для табуляции).

## Структура скрипта

Параметры скрипта

Модель скрипта

#### Главный объект скрипта

Содержит скрипт монитора.

| Элемент | Тип | Описание | Обязательный |
| --- | --- | --- | --- |
| `version` | string | Версия скрипта: используйте значение `1.0`. | Обязательный |
| `requests` | Массив из:  [request](#request) | Список HTTP-запросов, выполняемых монитором. Запросы выполняются в порядке их появления в скрипте. | Обязательный |

#### Объект `request`

Содержит параметры HTTP-запроса.

| Элемент | Тип | Описание | Обязательный |
| --- | --- | --- | --- |
| `description` | string | Краткое описание события, отображаемое в интерфейсе | Опциональный |
| `url` | string | URL для проверки | Обязательный |
| `method` | string | HTTP-метод запроса | Обязательный |
| `requestBody` | string | Тело HTTP-запроса: необходимо экранировать все JSON-символы. Устанавливается в null, если метод запроса GET, HEAD или OPTIONS. | Опциональный |
| `validation` | [requestValidation](#validation) | Конфигурация валидации запроса. Валидация помогает проверить, загружает ли HTTP-монитор ожидаемое содержимое. | Опциональный |
| `configuration` | [requestConfiguration](#configuration) | [Настройка](/managed/observe/digital-experience/synthetic-monitoring/http-monitors-classic/configure-http-monitors-classic#setup "Узнайте о настройке HTTP-мониторов.") монитора | Опциональный |
| `preProcessingScript` | string | [Скрипт](/managed/observe/digital-experience/synthetic-monitoring/http-monitors-classic/pre-and-post-scripting-for-http-monitors-classic "Узнайте, как применять скрипты пред- и пост-выполнения к запросам."), выполняемый перед запросом. Необходимо экранировать все JSON-символы и обозначать переносы строк как `/n`. | Опциональный |
| `postProcessingScript` | string | [Скрипт](/managed/observe/digital-experience/synthetic-monitoring/http-monitors-classic/pre-and-post-scripting-for-http-monitors-classic "Узнайте, как применять скрипты пред- и пост-выполнения к запросам."), выполняемый после запроса. Необходимо экранировать все JSON-символы и обозначать переносы строк как `/n`. | Опциональный |

#### Объект `requestValidation`

Содержит конфигурацию валидации запроса.

| Элемент | Тип | Описание | Обязательный |
| --- | --- | --- | --- |
| `rules` | Массив из:  [validationRule](#validation-rule) | Список правил валидации | Опциональный |

При валидации HTTP-монитора оцениваются следующие типы правил (см. [validationRule](#validation-rule)):

* Валидация кода статуса ответа
* Текстовая валидация: тело ответа проверяется на наличие строки текста или регулярного выражения
* Валидация срока действия SSL-сертификата

Это означает, что монитор **завершится с ошибкой**, если выполнено хотя бы одно правило с `passIfFound` = `false`, или хотя бы одно правило с `passIfFound` = `true` не выполнено.

Монитор **завершится успешно**, если ни одно правило с `passIfFound` = `false` не выполнено и все правила с `passIfFound` = `true` выполнены.

#### Объект `validationRule`

Содержит правило валидации.

| Элемент | Тип | Описание | Обязательный |
| --- | --- | --- | --- |
| `type` | string | Тип правила. Возможные значения:  * `patternConstraint` — выполняет простое сопоставление содержимого. * `regexConstraint` — интерпретирует сопоставление содержимого как [регулярное выражение](/managed/manage/tags-and-metadata/reference/regular-expressions-in-dynatrace "Узнайте об использовании регулярных выражений в Dynatrace."). * `httpStatusesList` — проверяет список HTTP-кодов статуса. * `certificateExpiryDateConstraint` — проверяет, не истекает ли сертификат в течение заданного количества дней. | Обязательный |
| `passIfFound` | boolean | Условие валидации:  * `true` — валидация **успешна**, если указанное содержимое/элемент найден. * `false` — валидация **неуспешна**, если указанное содержимое/элемент найден.  Для `certificateExpiryDateConstraint` всегда указывайте `false`, чтобы монитор завершался с ошибкой, если до истечения SSL-сертификата осталось меньше заданного количества дней. | Обязательный |
| `value` | string | Искомое содержимое | Обязательный |

#### Объект `requestConfiguration`

Содержит [настройку](/managed/observe/digital-experience/synthetic-monitoring/http-monitors-classic/configure-http-monitors-classic#setup "Узнайте о настройке HTTP-мониторов.") монитора.

| Элемент | Тип | Описание | Обязательный |
| --- | --- | --- | --- |
| `userAgent` | string | [**User agent**](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/User-Agent) запроса | Опциональный |
| `acceptAnyCertificate` | boolean | Принимать любой (`true`) или только доверенный (`false`) SSL-сертификат. При значении `false` монитор завершается с ошибкой при недействительных SSL-сертификатах. Если не задано, используется значение `false`. | Опциональный |
| `followRedirects` | boolean | Следовать (`true`) или не следовать (`false`) редиректам. При значении `false` редиректы отображаются как успешные запросы с кодом ответа `3xx`. Если не задано, используется значение `false`. | Опциональный |
| `requestHeaders` | Массив из:  [requestHeader](#header) | Список дополнительных заголовков запроса. По умолчанию задан только заголовок **User-Agent**. Задать или изменить этот заголовок здесь нельзя: используйте для этого поле `userAgent`. | Опциональный |

#### Объект `requestHeader`

Содержит HTTP-заголовок запроса.

| Элемент | Тип | Описание | Обязательный |
| --- | --- | --- | --- |
| `name` | string | Ключ заголовка | Обязательный |
| `value` | string | Значение заголовка | Обязательный |
