# Types of synthetic monitors

# Types of synthetic monitors

* Explanation
* 3-min read
* Published Oct 06, 2017

Синтетический мониторинг — это проактивное моделирование пользовательских визитов независимо от того, посещают ли реальные пользователи ваш сайт в данный момент. Dynatrace Synthetic Monitoring обеспечивает круглосуточную глобальную видимость ваших приложений. HTTP-монитор использует простые HTTP-запросы. Браузерный монитор представляет собой значительно большее: он управляет реальными сессиями веб-браузера с полной поддержкой HTML5/AJAX.

Dynatrace предлагает следующие типы синтетических мониторов: браузерные мониторы по одному URL, браузерные clickpath-ы, HTTP-мониторы и NAM-мониторы.

## Браузерные мониторы по одному URL

[Браузерный монитор по одному URL](/managed/observe/digital-experience/synthetic-monitoring/browser-monitors/create-a-single-url-browser-monitor "Узнайте, как настроить браузерный монитор по одному URL для проверки доступности сайта.") — это эквивалент имитации посещения вашего приложения пользователем с использованием современного обновлённого веб-браузера. Браузерные мониторы можно настроить для запуска из любого из [глобальных публичных](/managed/observe/digital-experience/synthetic-monitoring/general-information/public-synthetic-locations "Узнайте обо всех доступных публичных расположениях Synthetic Monitoring.") или [частных расположений](/managed/observe/digital-experience/synthetic-monitoring/private-synthetic-locations/create-a-private-synthetic-location "Узнайте, как создать частное расположение для синтетического мониторинга.") с частотой до каждых пяти минут. Браузерные мониторы оповещают при недоступности приложения или значительном снижении базовой производительности.

## Браузерные clickpath-ы

[Браузерные clickpath-ы](/managed/observe/digital-experience/synthetic-monitoring/browser-monitors/record-a-browser-clickpath "Узнайте, как записать браузерный clickpath для мониторинга доступности и производительности приложения.") — это имитируемые пользовательские визиты для мониторинга бизнес-критичных рабочих процессов приложения. Можно использовать рекордер Dynatrace (или [режим скрипта](/managed/observe/digital-experience/synthetic-monitoring/browser-monitors/script-mode-for-browser-monitor-configuration "Создавайте и редактируйте браузерные мониторы в формате JSON.")) для захвата точной последовательности кликов и пользовательского ввода, которые вы хотите отслеживать на предмет доступности и производительности. После захвата щелчков мышью и дополнительных пользовательских действий можно настроить автоматическое выполнение браузерного clickpath-а через регулярные интервалы из глобальных публичных или частных расположений для тестирования доступности и производительности сайта.

## HTTP-мониторы

[HTTP-мониторы](/managed/observe/digital-experience/synthetic-monitoring/http-monitors-classic/create-an-http-monitor-classic "Узнайте, как настроить HTTP-монитор для проверки производительности и доступности сайта.") состоят из простых HTTP-запросов. Их можно использовать для мониторинга доступности API-эндпоинтов или выполнения простых HTTP-проверок доступности одного ресурса. Для HTTP-мониторов также можно задавать пороговые значения производительности. Как и браузерные мониторы, HTTP-мониторы выполняются автоматически через регулярные интервалы из глобальных публичных или частных расположений.

HTTP-мониторы, выполняемые [ActiveGate](/managed/observe/digital-experience/synthetic-monitoring/private-synthetic-locations/create-a-private-synthetic-location "Узнайте, как создать частное расположение для синтетического мониторинга.") из частного расположения Synthetic, можно использовать для проверки доступности внутренних ресурсов, недоступных снаружи сети. Обратите внимание: в частных расположениях потребление ресурсов отслеживается отдельно для подтипа [**high-resource HTTP monitors**](/managed/observe/digital-experience/synthetic-monitoring/private-synthetic-locations/manage-private-synthetic-locations "Анализируйте и управляйте использованием ресурсов в частных расположениях Synthetic."). Эти мониторы имеют специальные ресурсоёмкие функции, такие как скрипты до/после выполнения, авторизация OAuth2 или аутентификация Kerberos.

## Мониторинг доступности сети

[Мониторинг доступности сети](/managed/observe/digital-experience/synthetic-monitoring/network-availability-monitors/nam-for-managed) (NAM) позволяет отслеживать доступность удалённых хостов или сервисов по сети, когда HTTP/HTTPS-эндпоинт недоступен.

NAM позволяет создавать синтетические мониторы доступности сети следующих типов:

* ICMP — отправляет ping-запросы с настраиваемым количеством пакетов или размером для проверки наличия сетевого соединения с хостом или устройством. Также проверяет качество этого соединения.
* TCP — устанавливает TCP-соединение с определённым портом. Проверяет, открыт ли порт и принимает ли он TCP-соединения. Также проверяет доступность хоста через сеть.
* DNS — проверяет, может ли имя хоста быть разрешено в IP-адрес.

NAM-мониторы работают только с частными расположениями. Подробнее о частных расположениях см. в разделе [Частные расположения Synthetic](/managed/observe/digital-experience/synthetic-monitoring/private-synthetic-locations "Узнайте о частных расположениях Synthetic.").

## Поддерживаемые браузеры

Поддерживаемый браузер для Dynatrace Synthetic Recorder — Google Chrome (последняя версия с обратной совместимостью).

Браузер, используемый для выполнения браузерных мониторов из публичных расположений, указывается на странице Frequency and locations при создании или редактировании браузерного монитора.

Версии браузеров, которые следует установить на ActiveGate с поддержкой Synthetic для выполнения частных браузерных мониторов, см. в разделе [Браузерные мониторы в частных расположениях](/managed/observe/digital-experience/synthetic-monitoring/private-synthetic-locations/create-a-private-synthetic-location "Узнайте, как создать частное расположение для синтетического мониторинга.").

## Безопасность синтетических мониторов

Для повышения безопасности синтетических мониторов Dynatrace блокирует отправку мониторами запросов на локальный хост (например, `localhost` или `127.0.0.1`).

Дополнительно можно ознакомиться с [архитектурой безопасности хранилища учётных данных](/managed/manage/credential-vault#security "Храните и управляйте учётными данными в хранилище.") для учётных данных синтетического мониторинга.
