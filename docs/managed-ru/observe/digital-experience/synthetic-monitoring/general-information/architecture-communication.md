# Архитектура и коммуникация Synthetic

* 3-min read

Узнайте об архитектуре Synthetic и о том, как данные безопасно проходят через неё. Ознакомьтесь с основными компонентами и тем, как они взаимодействуют между собой для обеспечения бесперебойной работы. Анализ потоков данных позволяет понять, как информация перемещается, обрабатывается и шифруется, обеспечивая эффективную работу системы.

## Компоненты архитектуры

В потоке коммуникации Synthetic участвуют следующие компоненты:

* **Dynatrace Cluster** — хранит конфигурацию синтетических мониторов, планирует выполнения мониторов для частных расположений и при необходимости отправляет конфигурацию мониторов в публичную инфраструктуру.
* **Частное расположение** — группа Synthetic ActiveGate, назначенных на ваш монитор. Dynatrace Cluster планирует выполнения для частного расположения.
* **ActiveGate** — компонент, используемый движком Synthetic для регистрации и взаимодействия с Dynatrace Cluster.
* **Публичная инфраструктура Synthetic** — инфраструктура, отвечающая за планирование и конфигурирование публичных расположений.
* **Публичное расположение** — группа движков Synthetic, назначенных на ваш монитор. Публичная инфраструктура Synthetic планирует выполнения для публичного расположения.
* **Движок Synthetic** — движок, отвечающий за выполнение мониторов.
* **Beacon forwarder** — пересылает результаты от движков Synthetic в Dynatrace Cluster.

## Коммуникация и поток данных

В зависимости от используемого расположения поток данных различается. Ознакомьтесь с тем, как работает коммуникация для частных и публичных расположений и как в этом процессе задействованы токены, ориентируясь на диаграмму выше.

Каждый номер соответствует шагу на диаграмме выше.

Коммуникация через частное расположение

Коммуникация через публичное расположение

1. **Dynatrace Cluster к ActiveGate**

   Кластер предоставляет конфигурацию, определения мониторов и планирует выполнения. Данные шифруются с помощью [токена ActiveGate](/managed/ingest-from/dynatrace-activegate/activegate-security "Защитите ActiveGate с помощью выделенных токенов.").
2. **Движок Synthetic к ActiveGate**

   Движок Synthetic запрашивает конфигурацию, определения мониторов и выполнения от кластера через ActiveGate. Данные шифруются с помощью `synthetic token`, который создаётся при запуске ActiveGate и хранится в файле `synthetic.token` в вашей папке `config`. Этот файл токена используется совместно ActiveGate и движком Synthetic.
3. **Движок Synthetic к beacon forwarder**

   Движок Synthetic отправляет результаты выполнений. Данные шифруются с помощью токена `synthetic monitors`. Dynatrace Cluster генерирует этот токен на уровне клиента, и он ротируется каждые 24 часа. Раз в сутки токены старше 48 часов удаляются. Токен передаётся движку вместе с деталями выполнения, а коммуникация защищена [токеном ActiveGate](/managed/ingest-from/dynatrace-activegate/activegate-security "Защитите ActiveGate с помощью выделенных токенов.").
4. **Beacon forwarder к Dynatrace Cluster**

   Beacon forwarder пересылает результаты, полученные от движка, в кластер. Данные шифруются с помощью [токена ActiveGate](/managed/ingest-from/dynatrace-activegate/activegate-security "Защитите ActiveGate с помощью выделенных токенов.").

5. **Dynatrace Cluster к публичной инфраструктуре Synthetic**

   Кластер предоставляет определения мониторов, детали окон обслуживания и состояние клиента, включая состояние лицензии. Данные шифруются с помощью токена `synthetic monitors`. Dynatrace Cluster генерирует этот токен на уровне клиента, и он ротируется каждые 24 часа. Раз в сутки токены старше 48 часов удаляются. Токен передаётся движку вместе с деталями выполнения, а коммуникация защищена [токеном ActiveGate](/managed/ingest-from/dynatrace-activegate/activegate-security "Защитите ActiveGate с помощью выделенных токенов.").
6. **Движок Synthetic к публичной инфраструктуре**

   Движок Synthetic запрашивает конфигурацию, определения мониторов и выполнения из публичной инфраструктуры, а также передаёт диагностические данные. Данные защищены на сетевом уровне.
7. **Движок Synthetic к beacon forwarder**

   Движок Synthetic отправляет результаты выполнений. Данные шифруются с помощью токена `synthetic monitors`. Dynatrace Cluster генерирует этот токен на уровне клиента, и он ротируется каждые 24 часа. Раз в сутки токены старше 48 часов удаляются. Токен передаётся движку вместе с деталями выполнения, а коммуникация защищена [токеном ActiveGate](/managed/ingest-from/dynatrace-activegate/activegate-security "Защитите ActiveGate с помощью выделенных токенов.").
