# Инструментирование мобильных приложений с помощью плагина Cordova Dynatrace в RUM Classic

* Практическое руководство
* Чтение 5 мин
* Обновлено 12 октября 2022 г.

Гибридные мобильные приложения сочетают преимущества нативных мобильных приложений (доступ к аппаратной части телефона, развёртывание через магазин приложений) с гибкостью и платформенной независимостью современных веб-технологий.

Плагин Cordova Dynatrace позволяет инструментировать приложения, построенные на базе Apache Cordova. Это касается и приложений, использующих Capacitor, Ionic или IBM mobile first. Плагин Cordova упрощает мониторинг гибридных приложений с помощью следующих технологий:

* OneAgent for Mobile, который отслеживает нативную часть приложения
* RUM JavaScript, который отслеживает производительность веб-части приложения

Подробную техническую документацию см. на странице [плагина Cordova Dynatrace﻿](https://www.npmjs.com/package/@dynatrace/cordova-plugin) на сайте npm.

## Настройка мониторинга гибридных мобильных приложений

1. В Dynatrace перейти в раздел **Mobile**.
2. Выбрать **Create mobile app**.
3. Ввести имя приложения и выбрать **Create mobile app**. Откроется страница настроек приложения.
4. На странице настроек приложения выбрать **Instrumentation wizard** и указать платформу мобильного приложения, например **Cordova**.

   ![Создание приложения в Dynatrace](https://dt-cdn.net/images/setup-wizard-1764-763c621790.png)

   Создание приложения в Dynatrace
5. Ввести команду `cordova plugin add @dynatrace/cordova-plugin --save` в корневом каталоге проекта на базе Cordova, чтобы установить плагин Cordova Dynatrace.
6. Выбрать **Monitor the web view**, чтобы включить мониторинг веб-представления.
7. Выбрать **Download dynatrace.config.js** и добавить скачанный файл в корень рабочей области приложения Cordova рядом с файлом `config.xml`.
8. Собрать проект с помощью команды `cordova build android` или `cordova build ios`.

При сборке приложения для Android или iOS плагин Cordova автоматически инструментирует приложение с помощью OneAgent for Mobile, который отслеживает нативную часть гибридного приложения. Это даёт видимость запусков приложения, действий пользователей, отчётов о сбоях и метаданных устройства, таких как модель и версия ОС. Данные мониторинга, собранные OneAgent for Mobile, инкапсулируются в мобильном приложении в Dynatrace.

Для захвата действий пользователя в веб-представлениях Dynatrace использует [RUM JavaScript](/managed/observe/digital-experience/rum-classic/web-applications/initial-setup/snippet-formats "Выбрать формат сниппета RUM JavaScript, наиболее подходящий для конкретного случая использования"), который также применяется для всех веб-приложений, [настроенных для безагентного мониторинга Dynatrace](/managed/observe/digital-experience/rum-classic/web-applications/initial-setup/set-up-agentless-real-user-monitoring "Настройка безагентного мониторинга для веб-приложений."). Плагин Cordova автоматически внедряет RUM JavaScript непосредственно в исходный код HTML. Данные мониторинга, собранные RUM JavaScript, инкапсулируются в веб-приложении в Dynatrace. Это приложение получает все данные мониторинга и генерирует токен API, чтобы плагин Cordova мог получить RUM JavaScript из REST API.

## Запуск мониторинга и анализа приложения

1. Запустить гибридное приложение в эмуляторе или на тестовом устройстве и пройти по сценариям использования, чтобы сгенерировать трафик, который сможет отслеживать Dynatrace. По завершении закрыть приложение, чтобы убедиться, что все данные отправлены в Dynatrace. OneAgent for Mobile может кэшировать данные до 2 минут.
2. В Dynatrace перейти в раздел **Mobile**.
3. Выбрать новое гибридное приложение из списка приложений.

   Все данные метрик анализа производительности, относящиеся к приложению, отображаются на странице обзора гибридного приложения. Раздел **Hybrid app** показывает общие сведения о производительности веб-части приложения.

   ![Анализ мобильных приложений](https://dt-cdn.net/images/analyze-mobile-app-1954-6c4199ccdd.png)

   Анализ мобильных приложений
4. Выбрать **View web application**, чтобы проанализировать данные о производительности, которые Dynatrace захватывает через внедрённый RUM JavaScript.

   Страница обзора веб-приложения содержит действия пользователей, захваченные в веб-представлении, поэтому здесь будет отображаться значительная доля действий пользователей приложения.

   ![Анализ веб-приложений](https://dt-cdn.net/images/analyze-web-app-1954-d5e81e0ff9.png)

   Анализ веб-приложений

   Данные о производительности разделены между двумя представлениями приложений. Однако Dynatrace объединяет все обнаруженные действия пользователей из этих представлений приложений в единого пользователя и единую пользовательскую сессию.
5. Перейти в **Session Segmentation**, чтобы открыть страницу **User sessions**. В поле **Filtered by** ввести **Application** и выбрать гибридное мобильное приложение, что позволит сосредоточить анализ на только что созданном приложении.
6. Выбрать пользователя по умолчанию, созданного для гибридного приложения, в столбце **User**. Обратить внимание, что этот пользователь обращался к двум приложениям, мобильному и веб-приложению.

   ![Анализ пользователя](https://dt-cdn.net/images/analyze-user-1950-e23a310611.png)

   Анализ пользователя
7. Выбрать пользовательскую сессию. Все действия пользователя, захваченные в мобильной и веб-части гибридного приложения, объединены в единую сессию вместе с метаданными, которые позволяют Dynatrace показывать, какие действия пользователя были захвачены OneAgent for Mobile, а какие, RUM JavaScript.

   Следующая диаграмма показывает последовательность действий пользователя, момент их возникновения в течение пользовательской сессии и время ожидания между действиями.

   ![Объединённая пользовательская сессия](https://dt-cdn.net/images/joined-user-session-1571-0684cc8268.png)

   Объединённая пользовательская сессия

Далее можно продолжить анализ пользовательского опыта. Можно добавить дополнительные фильтры, чтобы сосредоточиться на нужных сессиях, или углубиться в водопады действий пользователя веб- и мобильной сессий приложения.
