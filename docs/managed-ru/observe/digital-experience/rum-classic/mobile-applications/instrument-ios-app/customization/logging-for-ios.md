# OneAgent для отладочного логирования iOS в RUM Classic

* Практическое руководство
* Чтение за 1 минуту
* Обновлено 24 октября 2023 г.

Чтобы включить отладочное логирование для iOS, добавь [ключ конфигурации `DTXLogLevel`](/managed/observe/digital-experience/rum-classic/mobile-applications/instrument-ios-app/customization/ios-configuration-keys#general "С помощью ключей конфигурации можно точно настроить автоматическую инструментацию iOS-приложений.") в файл [`Info.plist`](/managed/observe/digital-experience/rum-classic/mobile-applications/instrument-ios-app/instrumentation/info-plist-file "Файл Info.plist хранит идентификацию приложения и ключи конфигурации. С его помощью можно точно настроить конфигурацию инструментации.") приложения:

```
<key>DTXLogLevel</key>



<string>ALL</string>
```

Доступные уровни логирования перечислены ниже в порядке возрастания детализации, каждый уровень включает информацию предыдущих уровней логирования.

| Уровень логирования | Собираемая информация |
| --- | --- |
| `OFF` | Только версия OneAgent |
| `SEVERE` | События, влияющие на функциональность OneAgent, которые могут отключить мониторинг |
| `WARNING` | События, которые могут привести к проблемам |
| `INFO` | Дополнительные информационные сообщения |
| `ALL` | Вся отладочная информация |

Во время работы приложения логи можно найти в консоли в области отладки Xcode. Обрати внимание, что уровень логирования предназначен для устранения неполадок и никак не влияет на данные, собираемые OneAgent.

Уровень логирования по умолчанию, `OFF`, поскольку интенсивное логирование может повлиять на производительность приложения.

Кроме того, нужно убедиться, что переменная окружения `OS_ACTIVITY_MODE` не установлена в значение `disabled`, так как это отключит отладочное логирование.

Вывод логирования в консоли Xcode 15+ больше не показывает временные метки. При копировании логов, например, для отправки специалисту продукта Dynatrace, нужно выделить в консоли все строки логов и выбрать **Copy Rows with All Metadata**.
