# Проверка метрик пользовательского опыта для мобильных приложений в RUM Classic

* Практическое руководство
* 1 минута на чтение
* Опубликовано 19 июля 2017 г.

Метрики пользовательского опыта приложения дают быстрый обзор количества пользователей, активно использующих приложение за выбранный период.

Чтобы получить доступ к метрикам пользовательского опыта приложения

1. Перейти в **Frontend**.
2. Выбрать приложение, которое нужно проанализировать.
3. Изучить плитку **User experience**, которая выбрана по умолчанию.

Плитка **User experience** показывает общее количество пользователей, общий рейтинг Apdex и частоту пользовательских действий за выбранный период.

![Плитка User experience на странице обзора приложения](https://dt-cdn.net/images/user-experience-tile-mobile-app-852-0baf6bdc02.png)

Плитка User experience на странице обзора приложения

Подробная информация о трендах рейтинга Apdex приложения, активных пользователях, новых пользователях и распределении версий приложения доступна в соответствующих разделах.

## New users

График **New users** показывает, сколько пользователей запустили приложение впервые за выбранный период. Эта метрика учитывает только первую сессию после запуска инструментированного приложения. Обновления приложения и обновления библиотеки Dynatrace не вызывают сброс состояния для возвращающихся пользователей. Эта метрика привязана к конкретным устройствам, поэтому пользователи учитываются несколько раз, если они устанавливают приложение на несколько устройств.

![Раздел User experience — New users](https://dt-cdn.net/images/user-experience-new-users-2-848-df286583cc.png)

Раздел User experience — New users

Метрика **New users** не различает нескольких пользователей, использующих одно и то же устройство и одну и ту же установку приложения.

## App version distribution

График **App version distribution** отражает скорость перехода на новую версию, сравнивая количество сессий приложения, происходящих из разных версий приложения. Этот график особенно интересен после выпуска новых версий приложения, поскольку позволяет отслеживать переход на последнюю версию.

### Страница обзора пользовательского действия

В списке **Top 3 actions** на странице обзора приложения нужно выбрать пользовательское действие, чтобы открыть страницу обзора пользовательского действия.

![Страница обзора пользовательского действия](https://dt-cdn.net/images/user-action-detail-page-2518-a51f30ad70.png)

Страница обзора пользовательского действия

На этой странице можно отметить действие как ключевое пользовательское действие, выполнить waterfall-анализ, а также посмотреть рейтинг Apdex действия и его производительность.

Также можно изучить списки зарегистрированных ошибок и ошибок веб-запросов для текущего действия. Нужно выбрать интересующую ошибку, чтобы открыть [страницу с подробностями](#error-details-page).

### Многомерный анализ пользовательских действий

Под списком **Top 3 actions** на странице обзора приложения нужно выбрать **Analyze performance**, чтобы перейти на страницу многомерного анализа **User action analysis**.

Многомерный анализ позволяет глубоко изучить пользовательские действия и посмотреть данные по множеству измерений. На странице **User action analysis** можно создавать метрики, просматривать ключевые пользовательские действия приложения и изучать топ-100 пользовательских действий по разным критериям. Также можно анализировать приложение по операционной системе, геолокации, версии приложения и другим параметрам.

### Ошибки веб-запросов и зарегистрированные ошибки

На странице многомерного анализа **User action analysis** также можно изучить список ошибок веб-запросов или зарегистрированных [ошибок](/managed/observe/digital-experience/rum-classic/rum-concepts/user-and-error-events#error "Узнать о пользовательских событиях и событиях ошибок, а также о типах пользовательских событий и событий ошибок, фиксируемых Dynatrace."): нужно выбрать **Web request errors over time** или **Reported errors over time** в текстовом поле **Analyze by**. Список ошибок появится под графиком. Нужно развернуть интересующую ошибку и затем выбрать **Analyze error**, чтобы открыть страницу с подробностями об ошибке.

![Analyze error в разделе MDA](https://dt-cdn.net/images/analyze-error-from-mda-2518-90cecae0cc.png)

Analyze error в разделе MDA

Страница с подробностями об ошибке веб-запроса

![Страница с подробностями о зарегистрированной ошибке](https://dt-cdn.net/images/reported-error-details-page-new-1644-60289f33a5.png)

Страница с подробностями о зарегистрированной ошибке

На ней отображаются подробности об ошибке, такие как оценочное количество ошибок, провайдер (для ошибки запроса), технология (для зарегистрированной ошибки) и другое. На странице также перечислены затронутые пользовательские сессии и затронутые пользовательские действия: нужно выбрать пользовательское действие или пользовательскую сессию, чтобы посмотреть подробности. Разбивка по распределению показывает информацию об относительной частоте операционных систем, версий ОС, версий приложения и устройств, а разбивка по странам показывает все затронутые страны и соответствующую им частоту ошибок.

Для ошибок веб-запросов также можно [настроить правила исключения ошибок](/managed/observe/digital-experience/rum-classic/mobile-applications/additional-configuration/web-request-errors-mobile "Прекратить рассматривать определённые HTTP-коды ответа как ошибки для мобильных приложений."), если не нужно, чтобы Dynatrace помечал определённый HTTP-код состояния как ошибочный. Нужно выбрать **Edit error exclusion rules**, чтобы перейти на соответствующую страницу настроек.

## Geographic regions

Карта мира **Geographic regions** на странице обзора приложения визуализирует все географические регионы, из которых происходят пользователи. Для каждой страны можно посмотреть количество активных сессий, активных пользователей и сбоев, а также общий рейтинг Apdex.

![Раздел User experience — Geo regions](https://dt-cdn.net/images/user-experience-geo-regions-848-7ae15779b9.png)

Раздел User experience — Geo regions
