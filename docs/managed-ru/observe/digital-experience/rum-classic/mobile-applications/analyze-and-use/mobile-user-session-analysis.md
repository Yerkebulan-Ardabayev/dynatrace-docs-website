# Анализ пользовательских сессий мобильных приложений в RUM Classic

* Практическое руководство
* 1 минута на чтение
* Опубликовано 19 июля 2017 г.

В современной цифровой потребительской среде наличие mobile-first стратегии для маркетинга бизнеса обязательно. Независимо от того, заходят пользователи на веб-предложения через мобильный браузер или используют загружаемое нативное или гибридное мобильное приложение, успех бизнеса зависит от высокой производительности пользовательского опыта. Именно поэтому Dynatrace ориентирован на стратегию мониторинга mobile-first, предлагающую широкий спектр технологических данных о мобильном пользовательском опыте.

По умолчанию анализ пользовательских сессий показывает все пользовательские сессии в среде: веб, мобильные и пользовательские приложения. Чтобы сфокусировать анализ только на мобильных приложениях, нужно добавить один фильтр.

Чтобы просмотреть пользовательские сессии, зафиксированные в мобильных приложениях

1. Перейти в **Session Segmentation**. Откроется страница **User sessions**.
2. В поле **Filter by** выбрать **Application type:** `Mobile`, чтобы отобразить мобильные пользовательские сессии.
3. Опционально Для дальнейшей фокусировки анализа можно добавить больше фильтров, таких как версия операционной системы, производитель, количество ошибок и оценка пользовательского опыта.
4. Выбрать временную метку нужной пользовательской сессии, чтобы перейти на страницу с деталями пользовательской сессии.

Dynatrace фиксирует пользовательские сессии из пользовательских приложений как мобильные пользовательские сессии. Встроенные метрики фиксируются без указания, мобильное это приложение или пользовательское. Подробнее см. [Встроенные метрики](/managed/analyze-explore-automate/metrics-classic/built-in-metrics "Explore the complete list of built-in Dynatrace metrics.").
