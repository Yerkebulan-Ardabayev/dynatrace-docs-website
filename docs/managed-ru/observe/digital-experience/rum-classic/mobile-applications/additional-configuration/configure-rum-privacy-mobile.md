# Настройка параметров конфиденциальности данных для мобильных приложений в RUM Classic

* Практическое руководство
* 4 минуты чтения
* Обновлено 16 апр. 2024 г.

Для многих компаний обеспечение конфиденциальности персональных данных клиентов, это важная составляющая успеха цифрового бизнеса. Dynatrace предоставляет многочисленные улучшения в области конфиденциальности, с помощью которых можно настроить параметры конфиденциальности данных в приложениях. Правильная настройка этих параметров помогает защитить персональные данные клиентов и обеспечить соответствие организации требованиям Общего регламента по защите данных (GDPR), требованиям раскрытия данных магазинов приложений и другим законам и нормам о защите данных.

Хотя Dynatrace предлагает множество настроек конфиденциальности данных, настраиваемых как на уровне [окружения](/managed/manage/data-privacy-and-security/configuration/configure-global-privacy-settings "Узнайте, как настроить маскирование данных конфиденциальности для IP-адресов конечных пользователей, геолокации и названий действий пользователей."), так и на уровне приложения, ответственность за правильную настройку этих параметров и принятие мер предосторожности для защиты персональных данных клиентов лежит только на вас.

## Режим согласия пользователя (opt-in)

В режиме opt-in каждый пользователь приложения может задать свои настройки конфиденциальности данных и решить, хочет ли он делиться своей информацией. Пользователь может выбрать именно те типы информации, которыми готов поделиться. Например, один пользователь может разрешить только отправку отчётов о сбоях, а другой, дополнительно разрешить сбор данных о производительности и пользовательских данных.

### Возможный сценарий работы режима opt-in

Ниже описаны стандартные шаги настройки режима согласия пользователя (opt-in) для мобильных приложений.

1. [Режим opt-in включается](#enable-opt-in-mode) с помощью специального флага или ключа, после чего приложение переинструментируется.
2. При запуске OneAgent проверяет, настроил ли пользователь уже свои параметры конфиденциальности данных.

   По умолчанию для всех пользователей, ещё не задавших параметры конфиденциальности данных, применяются следующие настройки по умолчанию:

   * Уровень сбора данных: `Off` (данные мониторинга не отправляются в Dynatrace)
   * Отчёты о сбоях: `Off` (отчёты о сбоях не отправляются в Dynatrace)

   Благодаря этому при первом запуске приложения данные в Dynatrace не передаются.
3. Если пользователь ещё не настроил параметры конфиденциальности данных, показывается диалоговое окно (пример ниже) с запросом разрешения на сбор данных о производительности (соответствует [уровню сбора данных](#data-collection-levels) **Performance**), включение персональных данных в информацию, передаваемую в Dynatrace (соответствует уровню сбора данных **User behavior**), и отправку отчётов о сбоях.

   Не забудьте создать и показать пользователям диалоговое окно с настройками конфиденциальности данных. Пользователи приложения будут использовать это диалоговое окно для настройки параметров конфиденциальности данных.
4. Пользователь задаёт свои параметры конфиденциальности данных; вызовы API используются для сохранения выбранных пользователем настроек.
5. При следующем запуске приложения OneAgent применяет новые параметры конфиденциальности данных и передаёт ровно столько данных, сколько конкретный пользователь согласился передавать в Dynatrace.

### Включение режима opt-in

Чтобы включить режим opt-in для мобильных приложений:

1. Перейдите в **Mobile**.
2. Выберите мобильное приложение, которое нужно настроить.
3. Выберите **More** (**…**) > **Edit** в правом верхнем углу плитки с названием приложения.
4. В настройках приложения выберите **General** > **Data privacy**.
5. Включите **Enable user opt-in mode**.
6. Обновите файл конфигурации приложения (build-файл для Android, [`Info.plist`](/managed/observe/digital-experience/rum-classic/mobile-applications/instrument-ios-app/instrumentation/info-plist-file "Файл Info.plist хранит идентификационные данные приложения и ключи конфигурации. Используйте его для точной настройки конфигурации инструментирования.") для iOS и `dynatrace.config.<extension>` для кросс-платформенных фреймворков), добавив специальный флаг или ключ, включающий режим opt-in.

   Проверьте мастер инструментирования на предмет обновлённого фрагмента кода конфигурации.
7. Пересоберите приложение, чтобы новая конфигурация вступила в силу.

### Уровни сбора данных

В таблице ниже описаны доступные уровни сбора данных и указано, передаются ли [пользовательские теги](/managed/observe/digital-experience/rum-classic/rum-concepts/user-and-error-events#user-tagging "Узнайте о пользовательских событиях и событиях ошибок, а также о типах таких событий, фиксируемых Dynatrace.") и пользовательские действия, события, значения и ошибки для конкретного уровня.

| Уровень | Описание | Пользовательские теги, пользовательские события и пользовательские значения | Пользовательские действия и ошибки |
| --- | --- | --- | --- |
| **Off**  Данные мониторинга не отправляются | Персональные данные не передаются; все идентификаторы рандомизируются при каждом запуске.[1](#fn-1-1-def) | Не применимо | Не применимо |
| **Performance**  Передаются только данные о производительности, собираемые автоматически | Персональные данные не передаются; все идентификаторы рандомизируются при каждом запуске. | Не применимо | Применимо |
| **User behavior**  Передаются данные о производительности и данные пользователя | Персональные данные передаются; OneAgent распознаёт и учитывает пользователей, возвращающихся в будущем.[2](#fn-1-2-def) | Применимо | Применимо |

1

Отправляется единое событие `Loading <App>` для учёта количества пользователей, отказавшихся от передачи данных.

2

Если пользовательские теги и передача пользовательских событий или значений не настроены, уровень **User behavior** работает аналогично уровню **Performance**.

## Отслеживание пользователей

Для RUM Classic OneAgent для Mobile помечает HTTP-запросы заголовком `x-dynatrace`. Dynatrace использует этот заголовок для связывания мобильной части веб-запроса с частью сервиса, зафиксированной другим OneAgent. Для RUM на последней версии Dynatrace OneAgent для Mobile помечает HTTP-запросы с помощью заголовков `traceparent` и `tracestate`.

Для гибридных приложений cookie `dtAdk` позволяет объединить сессию от OneAgent для Mobile и сессию от RUM JavaScript так, чтобы эти сессии отображались как единая сессия, а cookie `dtAdkSettings` используется для синхронизации настроек между OneAgent для Mobile и RUM JavaScript.

## Маскирование пользовательских действий

Если нужно избежать сбора персональной информации для всех пользовательских действий в мобильном приложении, ознакомьтесь с соответствующими разделами о маскировании пользовательских действий для [Android](/managed/observe/digital-experience/rum-classic/mobile-applications/instrument-android-app/instrumentation-via-plugin/monitoring-capabilities#mask-user-actions "Настройте плагин Gradle для Android Dynatrace, чтобы отрегулировать возможности мониторинга OneAgent.") или [iOS](/managed/observe/digital-experience/rum-classic/mobile-applications/instrument-ios-app/customization/oneagent-sdk-for-ios#mask-user-actions "Обогатите мониторинг пользовательского опыта на мобильных устройствах с помощью OneAgent SDK для iOS.").

После включения маскирования пользовательских действий в мобильном приложении OneAgent заменяет все названия действий вида `Touch on <control title>` на имя класса или тип элемента управления, которого коснулся пользователь. Например, `Touch on Account 123456` заменяется на `Touch on Button`.
