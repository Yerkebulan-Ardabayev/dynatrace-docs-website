# Игнорирование ошибок веб-запросов для мобильных приложений в RUM Classic

* Практическое руководство
* Чтение за 1 минуту
* Опубликовано 30 авг. 2021 г.

По умолчанию Dynatrace помечает все коды состояния ответа `4xx` и `5xx` как [ошибки](/managed/observe/digital-experience/rum-classic/rum-concepts/user-and-error-events#error "Узнайте о пользовательских событиях и событиях ошибок, а также о типах пользовательских событий и событий ошибок, фиксируемых Dynatrace.") веб-запросов. Однако возможно, что некоторые неуспешные коды HTTP на самом деле не указывают на проблему в приложении. Например, иногда код состояния `401 Unauthorized` просто показывает, что пользователь решил не входить в систему. Другой пример, код `429 Too Many Requests`, который может указывать на проблемы ограничения скорости запросов (throttling), не вызванные приложением.

Чтобы игнорировать определённые коды состояния HTTP, нужно настроить, какие коды состояния не должны помечаться как ошибки. При создании правила исключения можно добавить отдельные коды состояния HTTP, например `401`, или диапазоны кодов состояния, например `400-499`.

Чтобы создать правило исключения

1. Перейдите в **Frontend**.
2. Выберите приложение, которое нужно настроить.
3. В правом верхнем углу плитки с названием приложения выберите **More** (**…**) > **Edit**.
4. В настройках приложения выберите **Request errors**.
5. Выберите **Add exclusion rule** и укажите код состояния HTTP или диапазон кодов, которые следует считать успешными.

   ![Настройка правила исключения](https://dt-cdn.net/images/mobile-exclusion-rules-2124-75133d34c8.png)

   Настройка правила исключения

После настройки правила исключения веб-запросы с ранее ошибочными кодами состояния будут учитываться как успешные веб-запросы при расчёте частоты ошибок. Это повлияет на рейтинг Apdex приложения, частоту ошибок запросов и список ошибок веб-запросов, а также на некоторые метрики пользовательских действий и пользовательские метрики.

Например, на скриншотах страницы обзора приложения ниже видно снижение частоты ошибок веб-запросов. Это произошло после того, как для этого приложения было настроено правило игнорирования кода состояния ответа `401 Unauthorized`. Также обратите внимание на снижение частоты ошибок для основных провайдеров приложения, а также на более высокий рейтинг Apdex и более низкую частоту ошибок для одного из пользовательских действий.
