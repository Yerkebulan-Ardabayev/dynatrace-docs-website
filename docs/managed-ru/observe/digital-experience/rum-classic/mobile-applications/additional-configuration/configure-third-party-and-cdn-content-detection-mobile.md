# Настройка обнаружения собственных, сторонних и CDN-ресурсов для мобильных приложений в RUM Classic

* Практическое руководство
* Чтение 1 мин
* Опубликовано 30 января 2023 г.

Dynatrace распознаёт более 1000 сторонних поставщиков контента, включая Google, Amazon, Twitter и LinkedIn. К типичному стороннему контенту относятся виджеты Facebook и Twitter, граватары и подобные ресурсы. Важно понимать, как сторонние поставщики контента влияют на приложения, и определять, не является ли такой контент первопричиной замедлений.

Сети доставки контента (CDN) обычно находятся под собственным контролем. Приложение может использовать CDN для хранения и обслуживания часто запрашиваемого статического и динамического контента, что требуется для ускорения загрузки в определённых географических регионах.

## Разница между сторонними поставщиками и CDN

Разница между сторонними поставщиками контента и CDN заключается в степени контроля над данными.

* Если используются внешние данные, над которыми нет контроля, контент поступает от стороннего поставщика.
* Если для доставки статического или динамического контента приложению (например, изображений или CSS) используется внешний ресурс, такой сервис стоит рассматривать как CDN.

## Управление поставщиками среды

### Просмотр полного списка поставщиков

Dynatrace обнаруживает множество сторонних поставщиков контента. Когда Dynatrace не может найти имя хоста в списке известных сторонних поставщиков, он помечает ресурс как CDN. Кроме того, Dynatrace может определить, какие домены принадлежат организации, и классифицировать эти домены как собственные.

Чтобы увидеть полный список поставщиков, обнаруженных для среды, перейдите в **Settings** > **Web and mobile monitoring** > **Provider breakdown**.

* Вручную добавленные поставщики перечислены непосредственно под кнопкой **Add custom provider item**. Поставщика можно [добавить](#add-provider-manually) и пометить как собственный, сторонний или CDN.
* Автоматически обнаруженные CDN и сторонние поставщики перечислены в разделе **Auto detected providers**.

### Добавление поставщика вручную

Если поставщика нет в списке автоматически обнаруженных или нужно переопределить правила автоматического обнаружения, его можно добавить вручную.

Чтобы добавить поставщика вручную

1. Перейдите в **Settings** > **Web and mobile monitoring** > **Provider breakdown**.
2. Выберите **Add custom provider item**.
3. Введите **Resource name** и укажите **Resource type**, чтобы определить, является ли ресурс собственным, сторонним или CDN.
4. Необязательно: укажите URL значка бренда поставщика.
5. Выберите **Add name pattern**, а затем введите шаблон для доменного имени поставщика. При необходимости можно указать несколько шаблонов.
6. Необязательно: включите **Submit this provider-pattern to improve auto-detection**, если считается, что Dynatrace должен автоматически распознавать этот ресурс как сторонний.

Начиная с версии Dynatrace 1.240, автоматическое обнаружение собственных поставщиков включено по умолчанию и не может быть отключено. Если нужно переопределить автоматически обнаруженных собственных поставщиков, [добавьте поставщика вручную](#add-provider-manually).

### Изменение порядка вручную добавленных поставщиков

Список вручную добавленных поставщиков обрабатывается в порядке их следования в списке. По этой причине поставщик с более специфичным шаблоном домена должен идти первым. Например, `api.example.com` должен идти перед `example.com`.

Чтобы изменить порядок вручную добавленных поставщиков

1. Перейдите в **Settings** > **Web and mobile monitoring** > **Provider breakdown**.
2. Найдите список вручную добавленных поставщиков под кнопкой **Add custom provider item**.
3. Используйте **Drag handle** ![Drag handle](https://dt-cdn.net/images/drag-handle-turquoise-600-1aa0e5ea00.svg "Drag handle"), чтобы изменить приоритет поставщика.
