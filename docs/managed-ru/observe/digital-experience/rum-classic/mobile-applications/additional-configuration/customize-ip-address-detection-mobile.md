# Настройка определения IP-адреса для мобильных приложений в RUM Classic

* Практическое руководство
* Чтение 1 мин
* Обновлено 21 марта 2024 г.

Когда веб-запросы и RUM-маячки отправляются напрямую на инструментированный сервер, Dynatrace определяет IP-адреса устройств конечных пользователей через сокетные соединения.

При использовании неинструментированных компонентов, таких как балансировщики нагрузки, CDN или прокси-серверы, удалённый IP-адрес отличается от исходного IP-адреса. В таких случаях Dynatrace также отслеживает определённые HTTP-заголовки. Эти заголовки чаще всего используются для определения исходного IP-адреса, когда клиент подключается к веб-серверу через HTTP-прокси, CDN или балансировщик нагрузки.

Список заголовков, которые Dynatrace использует для определения IP-адресов клиентов для приложений, можно посмотреть в разделе **Settings** > **Web and mobile monitoring** > **IP determination**. Dynatrace обрабатывает эти заголовки в определённом порядке, но порядок обработки можно изменить и добавить собственные заголовки.

## Управление стандартными заголовками

Можно обновлять названия заголовков, удалять ненужные заголовки и менять порядок, в котором Dynatrace обрабатывает заголовки.

1. Перейти в **Settings** > **Web and mobile monitoring** > **IP determination**.
2. Управление заголовками:

   * Чтобы переименовать заголовок, выбрать **Expand row** ![Expand row](https://dt-cdn.net/images/expand-row-icon-9c4950fc2e.svg "Expand row"), а затем ввести новое название заголовка.
   * Чтобы удалить заголовок, выбрать **Delete row** ![Remove](https://dt-cdn.net/images/remove-icon-105c6a04c2.svg "Remove").
   * Чтобы изменить приоритет заголовка, выбрать и удерживать **Drag row** ![Drag handle](https://dt-cdn.net/images/drag-handle-turquoise-600-1aa0e5ea00.svg "Drag handle") рядом с названием заголовка и переместить заголовок вверх или вниз.

## Добавление собственных заголовков IP клиента

Если IP-адреса клиентов находятся в другом заголовке, нужно создать собственное правило, чтобы Dynatrace мог корректно определять IP-адреса.

Чтобы добавить собственный заголовок IP клиента

1. Перейти в **Settings** > **Web and mobile monitoring** > **IP determination**.
2. Выбрать **Create custom client IP header**.
3. Указать название заголовка.

В одной среде можно создать до 10 собственных заголовков IP клиента.

Добавление собственного заголовка IP клиента может понадобиться в одном из следующих случаев.

* Когда заголовок, содержащий IP-адреса клиентов, отсутствует в списке заголовков в разделе **Settings** > **Web and mobile monitoring** > **IP determination**.
* Когда IP-адреса клиентов для приложений хранятся в собственном (проприетарном) заголовке.
* Когда IP-адреса определяются некорректно, например, когда в Dynatrace отображаются IP-адреса балансировщиков нагрузки.

## Причины некорректного определения IP-адреса клиента

Может быть несколько причин, по которым для приложения не отображаются корректные IP-адреса клиентов. Подробные инструкции приведены в разделах выше.

| Проблема | Возможное решение |
| --- | --- |
| IP-адреса клиентов для приложений находятся в заголовке, которого нет в списке. | Добавить собственный заголовок IP клиента. |
| Указан некорректный собственный заголовок IP клиента. | Обновить название заголовка. |
| Указанный собственный заголовок IP клиента фактически не содержит IP-адреса. | Указать другой собственный заголовок IP клиента. |
| Неверный порядок заголовков. | Изменить порядок обработки заголовков. |
| Инфраструктура не настроена на добавление заголовка, передающего IP-адреса клиентов. | Перенастроить инфраструктуру. |
| Заголовок, содержащий IP-адреса клиентов, не входит в состав запросов-маячков.[1](#fn-1-1-def) | Перенастроить инфраструктуру. |

1

Заголовки определения IP в обычных запросах приложения не имеют значения для определения IP собранных RUM-данных, даже если они фиксируются как трассировки и эти трассировки сопоставляются с действиями пользователя.
