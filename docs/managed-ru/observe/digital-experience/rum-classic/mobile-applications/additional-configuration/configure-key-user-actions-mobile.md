# Настройка ключевых пользовательских действий для мобильных приложений в RUM Classic

* Практическое руководство
* 1 минута на чтение
* Опубликовано 30 января 2023 г.

Большинство приложений включают пользовательские действия, которые особенно важны для успеха цифрового бизнеса. Примеры таких действий: регистрация, оформление заказа, поиск товара. Такие [ключевые пользовательские действия](/managed/observe/digital-experience/rum-classic/rum-concepts/user-actions#key-user-actions "Узнайте, что такое пользовательские действия и как они помогают понять, что пользователи делают в приложении.") могут выполняться дольше остальных или, наоборот, должны укладываться в более короткую, чем в среднем, продолжительность.

С помощью функции ключевых пользовательских действий можно [настроить пороговые значения Apdex](/managed/observe/digital-experience/rum-classic/rum-concepts/scores-and-ratings/apdex-ratings#key-user-actions "Узнайте, как Dynatrace использует Apdex для измерения удовлетворённости пользователей производительностью приложения.") для каждого из этих пользовательских действий. Эта функция позволяет отслеживать ключевые действия с помощью выделенного плитки дашборда и следить за историческими трендами.

## Пометить пользовательское действие как ключевое

1. Перейти в **Frontend**.
2. Выбрать приложение и прокрутить вниз до **Top 3 user actions** или **Top 3 actions**.
3. Выбрать **View full details** или **Analyze performance**.
4. Прокрутить вниз, чтобы увидеть список пользовательских действий, и выбрать нужное пользовательское действие.  
   Откроется страница с подробностями о пользовательском действии.
5. В правом верхнем углу страницы с подробностями о пользовательском действии выбрать **Mark as key user action**.

Можно определить до 500 ключевых пользовательских действий на среду для всех приложений и до 100 ключевых пользовательских действий на приложение.

При достижении максимального лимита ключевых пользовательских действий стоит рассмотреть использование расчётных метрик (доступны для [веб-приложений](/managed/observe/digital-experience/rum-classic/web-applications/additional-configuration/rum-calculated-metrics-web "Создавайте расчётные метрики, а также пользовательские графики на основе расчётных метрик для веб-приложений."), [мобильных приложений](/managed/observe/digital-experience/rum-classic/mobile-applications/additional-configuration/rum-calculated-metrics-mobile "Создавайте расчётные метрики, а также пользовательские графики на основе расчётных метрик для мобильных приложений.") и [пользовательских приложений](/managed/observe/digital-experience/rum-classic/custom-applications/additional-configuration/rum-calculated-metrics-custom "Создавайте расчётные метрики, а также пользовательские графики на основе расчётных метрик для пользовательских приложений.")), которые предлагают похожие возможности.

## Закрепить ключевое пользовательское действие на дашборде

[Dashboards Classic](/managed/analyze-explore-automate/dashboards-classic "Узнайте, как создавать, управлять и использовать Dynatrace Dashboards Classic.")

1. Перейти в **Frontend**.
2. Выбрать приложение и прокрутить вниз до **Top 3 user actions** или **Top 3 actions**.
3. Выбрать **View full details** или **Analyze performance**.
4. Найти нужное ключевое пользовательское действие и выбрать его.  
   Откроется страница с подробностями о пользовательском действии.
5. В верхней части страницы с подробностями о пользовательском действии выбрать **Pin to dashboard**. Подробнее см. [Закрепление плиток на дашборде](/managed/analyze-explore-automate/dashboards-classic/charts-and-tiles/pin-tiles-to-your-dashboard "Узнайте, как закреплять плитки на дашбордах.").

## Настроить рейтинг Apdex для ключевого пользовательского действия

Чтобы изменить пороговые значения Apdex для ключевого пользовательского действия

1. Перейти в **Frontend**.
2. Выбрать приложение и прокрутить вниз до **Top 3 user actions** или **Top 3 actions**.
3. Выбрать **View full details** или **Analyze performance**.
4. Найти нужное ключевое пользовательское действие и выбрать его.  
   Откроется страница с подробностями о пользовательском действии.
5. В правом верхнем углу страницы с подробностями о пользовательском действии выполнить одно из следующего:

   * Для веб-приложений выбрать **More** (**…**) > **Edit** > **Key performance metric**.
   * Для мобильных и пользовательских приложений выбрать ![Expand](https://dt-cdn.net/images/expandbutton-40-e1f11ff81d.png "Expand") (**Expand**) > **Edit** > **Key performance metric**.
6. С помощью ползунка настроить пороговые значения Apdex.
