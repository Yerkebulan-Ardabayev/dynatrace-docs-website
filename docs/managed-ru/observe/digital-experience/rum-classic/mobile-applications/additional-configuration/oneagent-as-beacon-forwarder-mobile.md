# Использование OneAgent в качестве конечной точки маяка для мобильных приложений в RUM Classic

* Практическое руководство
* Чтение за 1 минуту
* Опубликовано 21 января 2020 г.

OneAgent можно использовать как альтернативную конечную точку маяка для плавного перехода на мобильный RUM Dynatrace.

Это особенно полезно в случаях, когда кластер ActiveGate доступен только внутри сети и не может использовать общедоступный IP-адрес. Dynatrace Managed нельзя установить в таких средах, поэтому он не может управлять доменом и создавать доверенный сертификат.

Инструментированный сервер OneAgent должен быть настроен с использованием одной из следующих технологий:

* Apache HTTP Server
* IBM HTTP Server
* Oracle HTTP Server
* Веб-приложения на основе Java servlet (например, Tomcat и Wildfly)
* IIS
* NGINX
* Node.js

Чтобы использовать OneAgent в качестве конечной точки маяка для приложения

1. Перейти в **Frontend**.
2. Выбрать приложение, которое нужно настроить.
3. Выбрать **More** (**…**) > **Edit** в верхнем правом углу плитки с названием приложения.
4. В настройках приложения выбрать **General** > **Beacon Endpoint**.
5. В раскрывающемся списке **Type** выбрать **Instrumented web server**.
6. Ввести URL-адрес веб-сервера или сервера приложений Java (на основе servlet) в следующем формате:  
   `http(s)://<my-instrumented-server>:port/dtmb`
7. Выбрать **Save changes**.
8. В настройках приложения выбрать **Instrumentation wizard** и выбрать нужную операционную систему.
9. Собрать и протестировать мобильное приложение, чтобы проверить изменение конфигурации.
