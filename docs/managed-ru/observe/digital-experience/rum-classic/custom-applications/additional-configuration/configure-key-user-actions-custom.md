# Настройка ключевых пользовательских действий для пользовательских приложений в RUM Classic

* Практическое руководство
* 1 мин на чтение
* Опубликовано 30 января 2023 г.

Большинство приложений включают пользовательские действия, которые особенно важны для успеха цифрового бизнеса. Примеры таких действий, это регистрации, оформление заказов и поиск товаров. Такие [ключевые пользовательские действия](/managed/observe/digital-experience/rum-classic/rum-concepts/user-actions#key-user-actions "Узнайте, что такое пользовательские действия и как они помогают понять, что пользователи делают с приложением.") могут выполняться дольше, чем другие, или к ним может предъявляться требование более короткой, чем в среднем, длительности.

С помощью функции ключевых пользовательских действий можно [настроить пороговые значения Apdex](/managed/observe/digital-experience/rum-classic/rum-concepts/scores-and-ratings/apdex-ratings#key-user-actions "Узнайте, как Dynatrace использует Apdex для измерения удовлетворённости пользователей производительностью приложения.") для каждого из таких пользовательских действий. Эту функцию можно использовать для мониторинга ключевых действий с помощью выделенного плитки дашборда и отслеживания исторических трендов.

## Пометка пользовательского действия как ключевого

1. Перейдите в **Frontend**.
2. Выберите приложение и прокрутите вниз до **Top 3 user actions** или **Top 3 actions**.
3. Выберите **View full details** или **Analyze performance**.
4. Прокрутите вниз, чтобы увидеть список пользовательских действий, и выберите нужное пользовательское действие.  
   Откроется страница с подробностями пользовательского действия.
5. В верхнем правом углу страницы с подробностями пользовательского действия выберите **Mark as key user action**.

Можно определить до 500 ключевых пользовательских действий на среду по всем приложениям и до 100 ключевых пользовательских действий на приложение.

При достижении максимального лимита ключевых пользовательских действий стоит рассмотреть использование вычисляемых метрик (доступны для [веб-приложений](/managed/observe/digital-experience/rum-classic/web-applications/additional-configuration/rum-calculated-metrics-web "Создавайте вычисляемые метрики, а также пользовательские графики на основе вычисляемых метрик для веб-приложений."), [мобильных приложений](/managed/observe/digital-experience/rum-classic/mobile-applications/additional-configuration/rum-calculated-metrics-mobile "Создавайте вычисляемые метрики, а также пользовательские графики на основе вычисляемых метрик для мобильных приложений.") и [пользовательских приложений](/managed/observe/digital-experience/rum-classic/custom-applications/additional-configuration/rum-calculated-metrics-custom "Создавайте вычисляемые метрики, а также пользовательские графики на основе вычисляемых метрик для пользовательских приложений.")), которые предлагают аналогичные возможности.

## Закрепление ключевого пользовательского действия на дашборде

[Dashboards Classic](/managed/analyze-explore-automate/dashboards-classic "Узнайте, как создавать, управлять и использовать Dynatrace Dashboards Classic.")

1. Перейдите в **Frontend**.
2. Выберите приложение и прокрутите вниз до **Top 3 user actions** или **Top 3 actions**.
3. Выберите **View full details** или **Analyze performance**.
4. Найдите нужное ключевое пользовательское действие и выберите его.  
   Откроется страница с подробностями пользовательского действия.
5. В верхней части страницы с подробностями пользовательского действия выберите **Pin to dashboard**. Подробнее см. [Закрепление плиток на дашборде](/managed/analyze-explore-automate/dashboards-classic/charts-and-tiles/pin-tiles-to-your-dashboard "Узнайте, как закреплять плитки на дашбордах.").

## Настройка оценки Apdex для ключевого пользовательского действия

Чтобы изменить пороговые значения Apdex для ключевого пользовательского действия

1. Перейдите в **Frontend**.
2. Выберите приложение и прокрутите вниз до **Top 3 user actions** или **Top 3 actions**.
3. Выберите **View full details** или **Analyze performance**.
4. Найдите нужное ключевое пользовательское действие и выберите его.  
   Откроется страница с подробностями пользовательского действия.
5. В верхнем правом углу страницы с подробностями пользовательского действия выполните одно из следующих действий:

   * Для веб-приложений выберите **More** (**…**) > **Edit** > **Key performance metric**.
   * Для мобильных и пользовательских приложений выберите ![Expand](https://dt-cdn.net/images/expandbutton-40-e1f11ff81d.png "Expand") (**Expand**) > **Edit** > **Key performance metric**.
6. С помощью ползунка настройте пороговые значения Apdex.
