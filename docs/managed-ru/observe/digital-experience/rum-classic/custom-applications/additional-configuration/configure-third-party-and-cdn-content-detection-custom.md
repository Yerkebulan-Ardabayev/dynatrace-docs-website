# Настройка обнаружения ресурсов первой стороны, сторонних поставщиков и CDN для пользовательских приложений в RUM Classic

# Настройка обнаружения ресурсов первой стороны, сторонних поставщиков и CDN для пользовательских приложений в RUM Classic

* Практическое руководство
* Чтение за 1 минуту
* Опубликовано 30 января 2023 г.

Dynatrace распознаёт более 1000 сторонних поставщиков контента, включая Google, Amazon, Twitter и LinkedIn. Типичный сторонний контент включает виджеты Facebook и Twitter, граватары и подобные ресурсы. Важно видеть, как сторонние поставщики контента влияют на приложения, и понимать, является ли такой контент первопричиной замедлений.

Сети доставки контента (CDN), как правило, находятся под собственным контролем. Приложение может использовать CDN для хранения и предоставления часто запрашиваемого статического и динамического контента, что требуется для ускорения загрузки в определённых географических регионах.

## Разница между сторонними поставщиками и CDN

Разница между сторонними поставщиками контента и CDN заключается в степени контроля над данными.

* Если используются внешние данные, над которыми нет контроля, контент поступает от стороннего поставщика.
* Если приложение полагается на внешний ресурс для доставки статического или динамического контента, например изображений или CSS, этот сервис нужно рассматривать как CDN.

## Управление поставщиками окружения

### Просмотр полного списка поставщиков

Dynatrace обнаруживает множество сторонних поставщиков контента. Если Dynatrace не может найти имя хоста в списке известных сторонних поставщиков, ресурс помечается как CDN. Также Dynatrace может определить, какие домены принадлежат организации, и классифицировать такие домены как первую сторону.

Чтобы увидеть полный список поставщиков, обнаруженных для окружения, перейдите в **Settings** > **Web and mobile monitoring** > **Provider breakdown**.

* Вручную добавленные поставщики перечислены непосредственно под кнопкой **Add custom provider item**. Можно [добавить поставщика](#add-provider-manually) и пометить его как первую сторону, стороннего поставщика или CDN.
* Автоматически обнаруженные CDN и сторонние поставщики перечислены в разделе **Auto detected providers**.

### Добавление поставщика вручную

Если поставщик не найден в списке автоматически обнаруженных или нужно переопределить правила автоматического обнаружения, поставщика можно добавить вручную.

Чтобы добавить поставщика вручную

1. Перейдите в **Settings** > **Web and mobile monitoring** > **Provider breakdown**.
2. Выберите **Add custom provider item**.
3. Введите **Resource name** и укажите **Resource type**, чтобы определить, является ли ресурс первой стороной, сторонним поставщиком или CDN.
4. Необязательно. Укажите URL значка бренда поставщика.
5. Выберите **Add name pattern**, а затем введите шаблон для доменного имени поставщика. При необходимости можно указать несколько шаблонов.
6. Необязательно. Включите **Submit this provider-pattern to improve auto-detection**, если считается, что Dynatrace должен автоматически распознавать этот ресурс как сторонний.

Начиная с версии Dynatrace 1.240, автоматическое обнаружение поставщиков первой стороны включено по умолчанию и не может быть отключено. Если нужно переопределить автоматически обнаруженных поставщиков первой стороны, [добавьте поставщика вручную](#add-provider-manually).

### Изменение порядка вручную добавленных поставщиков

Список вручную добавленных поставщиков обрабатывается в порядке их появления в списке. По этой причине поставщик с более специфичным шаблоном домена должен идти раньше. Например, `api.example.com` должен идти перед `example.com`.

Чтобы изменить порядок вручную добавленных поставщиков

1. Перейдите в **Settings** > **Web and mobile monitoring** > **Provider breakdown**.
2. Найдите список вручную добавленных поставщиков под кнопкой **Add custom provider item**.
3. Используйте **Drag handle** ![Drag handle](https://dt-cdn.net/images/drag-handle-turquoise-600-1aa0e5ea00.svg "Drag handle"), чтобы изменить приоритет поставщика.
