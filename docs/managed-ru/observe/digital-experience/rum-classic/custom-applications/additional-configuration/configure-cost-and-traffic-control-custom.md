# Настройка контроля стоимости и трафика для пользовательских приложений в RUM Classic

* Практическое руководство
* Чтение: 1 минута
* Обновлено 28 февраля 2024 г.

По умолчанию Dynatrace собирает данные обо всех действиях пользователей и пользовательских сессиях для анализа. Такой подход обеспечивает полное понимание производительности приложения и качества клиентского опыта. С помощью настройки **Cost and traffic control** можно снизить детализацию анализа действий пользователей и пользовательских сессий, указав Dynatrace собирать данные по меньшему проценту пользовательских сессий.

Хотя эта настройка позволяет снизить [затраты на мониторинг](/managed/license/classic-licensing/digital-experience-monitoring-units "Узнайте, как рассчитывается потребление Dynatrace Digital Experience Monitoring на основе DEM-юнитов."), она также снижает видимость того, как клиенты используют приложения. Например, при значении `10%` Dynatrace анализирует только каждую десятую пользовательскую сессию.

Чтобы ограничить количество анализируемых пользовательских сессий

1. Перейти в **Frontend**.
2. Выбрать приложение, которое нужно настроить.
3. Выбрать **More** (**…**) > **Edit** в правом верхнем углу плитки с названием приложения.
4. В настройках приложения выбрать **General** > **Enablement and cost control**.
5. В разделе **Real User Monitoring** ввести значение меньше `100` в поле **Cost and traffic control**. Значение по умолчанию для этого поля, `100`.

При заданной настройке Dynatrace анализирует равномерно распределённое количество пользовательских сессий, соответствующее указанному проценту пользовательских сессий.

Функция контроля стоимости и трафика не поддерживается в [Dynatrace OpenKit](/managed/ingest-from/extend-dynatrace/openkit/dynatrace-openkit-libraries-on-github "Ознакомьтесь с библиотеками Dynatrace OpenKit на GitHub.") для JavaScript.
