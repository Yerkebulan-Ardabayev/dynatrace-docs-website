# Создание вычисляемых метрик для пользовательских приложений в RUM Classic

* Руководство
* Чтение 1 минута
* Обновлено 10 мая 2024 г.

В Dynatrace можно создавать вычисляемые метрики, чтобы сделать текущий анализ доступным для [построения диаграмм](/managed/analyze-explore-automate/explorer "Запрос метрик и преобразование результатов для получения нужных выводов.") и [использования API](/managed/dynatrace-api/environment-api/metric-v2 "Получение информации о метриках через Metrics v2 API."). Вычисляемые метрики также можно использовать для добавления пользовательских оповещений.

После выбора нужного приложения можно использовать **Multidimensional Analysis**, чтобы выбрать аспекты действий пользователей и создать вычисляемую метрику. Можно разбить выбранные метрики производительности по другому измерению, например по геолокации, браузеру и типу ошибки, либо использовать только отдельные измерения, такие как [свойства действий пользователя](/managed/observe/digital-experience/rum-classic/web-applications/additional-configuration/define-user-action-and-session-properties "Определение пользовательских строковых, числовых и датовых свойств для отслеживаемых веб-приложений.").

## Создание вычисляемой метрики

Чтобы создать вычисляемую метрику для своего приложения

1. Перейти в **Frontend** и выбрать приложение, для которого нужно создать метрику.
2. В разделе **Top 3 actions** выбрать **Analyze performance**.
3. Выбрать нужные параметры **OS family**, **Contribution**, временной интервал и **Analyze by**.
4. Необязательно. Использовать панель фильтров, чтобы добавить фильтры по геолокации, производителю, Apdex и другим параметрам и сфокусировать итоговую метрику на нужных данных.
5. Выбрать **Create metric**.

   ![Creating a metric on the Multidimensional analysis page](https://dt-cdn.net/images/mobile-metric-create-metric-mda-1758-a913b7f692.png)

   Создание метрики на странице Multidimensional analysis
6. Необязательно. Изменить имя и ключ метрики и включить **Split by <dimension name>**.

   Для [мобильных](/managed/observe/digital-experience/rum-classic/mobile-applications/analyze-and-use/action-and-session-properties-mobile "Свойства действий пользователя и сессии, представляющие собой пары ключ-значение метаданных, обеспечивают дополнительную видимость и более глубокий анализ опыта конечных пользователей. Используя эти свойства для приложений, можно фильтровать пользовательские сессии, добавлять вычисляемые метрики, создавать диаграммы и многое другое.") и [пользовательских приложений](/managed/observe/digital-experience/rum-classic/custom-applications/analyze-and-use/action-and-session-properties-custom "Свойства действий пользователя и сессии, представляющие собой пары ключ-значение метаданных, обеспечивают дополнительную видимость и более глубокий анализ опыта конечных пользователей. Используя эти свойства для приложений, можно фильтровать пользовательские сессии, добавлять вычисляемые метрики, создавать диаграммы и многое другое.") разбивка метрики по свойству действия пользователя недоступна. Разбивка по свойству действия пользователя доступна только для веб-приложений.
7. Выбрать **Create metric**.

   ![Overlay for creating a metric](https://dt-cdn.net/images/mobile-metric-create-metric-overlay-330-88ec98fac1.png)

   Оверлей создания метрики

Использовать метрику для создания пользовательской диаграммы или оповещения.

В вычисляемые метрики записываются только новые данные, ретроспективные данные не включаются.

Можно иметь до 500 включённых вычисляемых метрик на среду по всем приложениям и до 100 включённых вычисляемых метрик на приложение.

### Пример

В этом примере проанализируем `Price`, это [свойство действия пользователя](/managed/observe/digital-experience/rum-classic/web-applications/additional-configuration/define-user-action-and-session-properties#custom-properties "Определение пользовательских строковых, числовых и датовых свойств для отслеживаемых веб-приложений."), и отфильтруем его по `Loyalty status`, ещё одному свойству действия пользователя.

На странице **Multidimensional analysis** выбрать временной интервал для анализа. Чтобы отфильтровать цены, оплаченные только платиновыми клиентами, выбрать `Price` в списке **Analyze by**, а затем задать дополнительные фильтры, выбрав `String property`, `Loyalty status` и `Platinum`.

## Создание пользовательских диаграмм на основе вычисляемых метрик

Создание диаграмм помогает анализировать комбинации метрик приложения прямо на дашборде. Можно разбивать и фильтровать доступные сущности, чтобы точно настроить измерения метрики, отображаемые на диаграммах, и отфильтровать нужные сущности.

Подробнее о создании диаграмм и их закреплении на дашбордах см. в разделе [Data Explorer](/managed/analyze-explore-automate/explorer "Запрос метрик и преобразование результатов для получения нужных выводов.").

## Управление метриками

После создания вычисляемой метрики можно просматривать её свойства, удалять её, временно отключать или создавать для неё диаграмму или событие метрики.

После создания метрики изменить её свойства нельзя.

1. Перейти в **Frontend**.
2. Выбрать приложение, которое нужно настроить.
3. Выбрать **More** (**…**) > **Edit** в верхнем правом углу плитки с именем приложения.
4. В настройках приложения выбрать **Metrics**.
5. Выбрать метрику, которой нужно управлять, и проверить её свойства или выполнить одно из следующих действий.

   * **Включить или отключить** ![Toggle icon](https://dt-cdn.net/images/icon-toggle-barista-701-35879d6adf.png "Toggle icon") метрику
   * **Скопировать** URL API для метрики
   * **Создать диаграмму** с помощью [Data Explorer](/managed/analyze-explore-automate/explorer "Запрос метрик и преобразование результатов для получения нужных выводов.")
   * **Create alert**, чтобы создать [событие метрики](/managed/dynatrace-intelligence/anomaly-detection/metric-events "Информация о событиях метрик в Dynatrace")
   * **Delete metric**
