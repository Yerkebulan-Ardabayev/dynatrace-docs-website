# Проверка метрик пользовательского опыта для пользовательских приложений в RUM Classic

* Практическое руководство
* 1 минута на чтение
* Опубликовано 30 января 2023 г.

Метрики пользовательского опыта приложения дают быстрый обзор количества пользователей, активно использующих приложение за выбранный период времени.

Для доступа к метрикам пользовательского опыта приложения

1. Перейти в **Frontend**.
2. Выбрать приложение, которое нужно проанализировать.
3. Изучить плитку **User experience**, которая выбрана по умолчанию.

Плитка **User experience** показывает общее количество пользователей, общий рейтинг Apdex и частоту пользовательских действий за выбранный период времени.

![Плитка User experience на странице обзора приложения](https://dt-cdn.net/images/user-experience-tile-mobile-app-852-0baf6bdc02.png)

Плитка User experience на странице обзора приложения

Подробная информация о трендах рейтинга Apdex приложения, активных пользователях, новых пользователях и распределении версий приложения доступна в соответствующих разделах.

## Новые пользователи

График **New users** показывает, сколько пользователей запустили приложение впервые за выбранный период времени. Эта метрика учитывает только первый сеанс после запуска инструментированного приложения. Обновления приложения и обновления библиотеки Dynatrace не вызывают сброс состояния для возвращающихся пользователей. Эта метрика привязана к конкретным устройствам, поэтому пользователи учитываются несколько раз, если они устанавливают приложение на нескольких устройствах.

![Раздел User experience - новые пользователи](https://dt-cdn.net/images/user-experience-new-users-2-848-df286583cc.png)

Раздел User experience - новые пользователи

Метрика **New users** не различает нескольких пользователей, которые используют одно и то же устройство и одну и ту же установку приложения.

## Распределение версий приложения

График **App version distribution** отражает темп внедрения версий, сравнивая количество сеансов приложения, происходящих от разных версий приложения. Этот график особенно интересен после выпуска новых версий приложения, поскольку позволяет отслеживать внедрение последней версии.

### Страница обзора пользовательского действия

В списке **Top 3 actions** на странице обзора приложения нужно выбрать пользовательское действие, чтобы открыть страницу обзора пользовательского действия.

![Страница обзора пользовательского действия](https://dt-cdn.net/images/user-action-detail-page-2518-a51f30ad70.png)

Страница обзора пользовательского действия

На этой странице можно отметить действие как ключевое пользовательское действие, выполнить waterfall-анализ, а также посмотреть рейтинг Apdex и производительность действия.

Также можно изучить списки зарегистрированных ошибок и ошибок веб-запросов для текущего действия. Нужно выбрать интересующую ошибку, чтобы открыть её [страницу с подробностями](#error-details-page).

### Многомерный анализ пользовательских действий

Под списком **Top 3 actions** на странице обзора приложения нужно выбрать **Analyze performance**, чтобы перейти на страницу многомерного анализа **User action analysis**.

Многомерный анализ позволяет детально изучить пользовательские действия и просмотреть данные по множеству измерений. На странице **User action analysis** можно создавать метрики, просматривать ключевые пользовательские действия приложения и изучать топ-100 пользовательских действий на основе разных критериев. Также можно анализировать приложение по операционной системе, геолокации, версии приложения и другим параметрам.

### Ошибки веб-запросов и зарегистрированные ошибки

На странице многомерного анализа **User action analysis** также можно изучить список ошибок веб-запросов или зарегистрированных [ошибок](/managed/observe/digital-experience/rum-classic/rum-concepts/user-and-error-events#error "Узнать о пользовательских событиях и событиях ошибок, а также о типах пользовательских событий и событий ошибок, фиксируемых Dynatrace."): нужно выбрать **Web request errors over time** или **Reported errors over time** в текстовом поле **Analyze by**. Список ошибок появится под графиком. Нужно развернуть интересующую ошибку и выбрать **Analyze error**, чтобы просмотреть страницу с подробностями ошибки.

![Analyze error в разделе MDA](https://dt-cdn.net/images/analyze-error-from-mda-2518-90cecae0cc.png)

Страница с подробностями ошибки предоставляет ценную информацию об ошибках приложения.

Страница с подробностями ошибки веб-запроса

Страница с подробностями зарегистрированной ошибки

![Страница с подробностями ошибки веб-запроса](https://dt-cdn.net/images/web-request-error-details-page-1772-928bb94d69.png)

Страница с подробностями ошибки веб-запроса

![Страница с подробностями зарегистрированной ошибки](https://dt-cdn.net/images/reported-error-details-page-new-1644-60289f33a5.png)

Страница с подробностями зарегистрированной ошибки

Здесь отображаются подробности ошибки, такие как оценочное количество ошибок, провайдер (для ошибки запроса), технология (для зарегистрированной ошибки) и другое. На странице также перечислены затронутые пользовательские сеансы и затронутые пользовательские действия, нужно выбрать пользовательское действие или пользовательский сеанс, чтобы посмотреть его подробности. Разбивка по распределению показывает информацию об относительной частоте операционных систем, версий ОС, версий приложения и устройств, а разбивка по странам показывает все затронутые страны и соответствующий им уровень ошибок.

Для ошибок веб-запросов также можно [настроить правила исключения ошибок](/managed/observe/digital-experience/rum-classic/custom-applications/additional-configuration/web-request-errors-custom "Прекратить обработку определённых кодов ответа HTTP как ошибок для пользовательских приложений."), если нужно, чтобы Dynatrace не отмечал определённый статус-код HTTP как ошибочный. Нужно выбрать **Edit error exclusion rules**, чтобы перейти на соответствующую страницу настроек.

Карта мира **Geographic regions** на странице обзора приложения визуализирует все географические регионы, из которых происходят пользователи. Для каждой страны можно посмотреть количество активных сеансов, активных пользователей и сбоев, а также общий рейтинг Apdex.
