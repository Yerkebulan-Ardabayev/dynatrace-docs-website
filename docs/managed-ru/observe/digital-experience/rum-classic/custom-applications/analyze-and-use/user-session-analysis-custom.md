# Анализ пользовательских сессий пользовательских приложений в RUM Classic

* Практическое руководство
* 1 минута на чтение
* Опубликовано 30 января 2023 г.

Возможность просматривать и анализировать пользовательские сессии в пользовательских приложениях необходима в современной среде цифрового потребления.

По умолчанию анализ пользовательских сессий показывает все пользовательские сессии в среде: для веб-, мобильных и пользовательских приложений. Чтобы сосредоточить анализ только на пользовательских приложениях, нужно добавить фильтр.

Чтобы просмотреть пользовательские сессии, зафиксированные в пользовательских приложениях

1. Перейти на страницу **Session Segmentation**. Откроется страница **User sessions**.
2. В поле **Filter by** выбрать **Application type:** `Custom`, чтобы получить пользовательские сессии, зафиксированные в пользовательских приложениях.
3. Дополнительно Чтобы сузить анализ, можно добавить дополнительные фильтры, такие как версия операционной системы, производитель, количество ошибок и оценка пользовательского опыта.
4. Выбрать временную метку нужной пользовательской сессии, чтобы перейти на страницу с подробностями пользовательской сессии.

Dynatrace фиксирует пользовательские сессии из пользовательских приложений как мобильные пользовательские сессии. Встроенные метрики фиксируются без указания, относятся ли они к мобильному или пользовательскому приложению. Подробнее см. [Встроенные метрики](/managed/analyze-explore-automate/metrics-classic/built-in-metrics "Explore the complete list of built-in Dynatrace metrics.").
