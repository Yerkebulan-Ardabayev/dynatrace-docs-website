Dynatrace предлагает расширенный набор [ключевых показателей производительности](/managed/observe/digital-experience/rum-classic/rum-concepts/user-action-metrics#kpm "Узнайте, какие метрики рассчитывает Dynatrace для действий пользователей, и какой информацией обладает каждая метрика."), на которых можно основывать Real User Monitoring.

## Выбор подходящей метрики производительности

Для корректной аналитики мониторинга важно выбрать подходящую метрику производительности для каждого условия работы приложения.

Для многих традиционных веб-приложений **User action duration** считается лучшей доступной метрикой для оптимизации веб-производительности, так как эта метрика отражает время от действия пользователя до полной загрузки страницы. Но одностраничные приложения (SPA) на базе Angular, Ember, React и других JavaScript-фреймворков не зависят от загрузки новой страницы после каждого клика. Поэтому гонка за скоростью «above the fold» может приводить к высоким общим значениям user action duration. В результате другие метрики, такие как **Visually complete** или **Speed index**, точнее отражают реальное время, когда конечный пользователь может выполнить следующее действие. С точки зрения конечного пользователя веб-приложение становится пригодным для использования после того, как определённые элементы, например выбор даты или автозаполнение поля поиска, готовы к использованию.

Есть и приложения, где интерфейс не так важен, как взаимодействие с приложением. Пример: приложение call-центра или приложение для службы поддержки документации, где пользователи являются экспертами и работают с горячими клавишами. Для таких приложений время до начала взаимодействия с приложением важнее, чем полная загрузка всех изображений, поэтому метрика **Long tasks** важна, а **DOM interactive** может быть лучшей метрикой для рассмотрения.

Выбор подходящей ключевой метрики производительности также позволяет подстраиваться под разные уровни производительности различных функций приложения. Например, клиент может спокойно ждать пять секунд после нажатия **Confirm booking**, но раздражаться, если поле поиска появляется пять секунд.

С помощью ключевых показателей производительности и [ключевых действий пользователей](/managed/observe/digital-experience/rum-classic/rum-concepts/user-actions#key-user-actions "Узнайте, что такое действия пользователей и как они помогают понять, что пользователи делают в приложении.") можно подобрать оптимальную метрику производительности и ожидаемые цели производительности, наиболее подходящие для каждого отдельного действия пользователя. Ключевые показатели производительности Dynatrace позволяют реагировать на такие переменные условия для каждого приложения и действия пользователя, за которыми ведётся наблюдение.

## Использование ключевых показателей производительности для мониторинга производительности приложения

Ключевые показатели производительности доступны по умолчанию, и **Visually complete** выбрана в качестве ключевой метрики производительности для действий загрузки (load) и XHR. **User action duration** установлена как метрика по умолчанию для пользовательских действий (custom actions). Ключевые показатели производительности можно изменить для каждого типа действия.

Чтобы выбрать ключевые показатели производительности для приложения

1. Перейти в **Web**.
2. Выбрать приложение, которое нужно настроить.
3. В правом верхнем углу страницы обзора приложения выбрать **More** (**…**) > **Edit**.
4. Выбрать **General settings** > **Load actions** / **XHR actions** / **Custom actions**.
5. В разделе **Key performance metric** выбрать ключевую метрику производительности, которая лучше всего отражает пользовательский опыт для действий загрузки и XHR-действий пользователя.

Для случаев, когда нужно настроить ключевую метрику производительности отдельно для разных функций приложения, можно выбрать другую ключевую метрику производительности для ключевых действий пользователей приложения.

Чтобы выбрать ключевые показатели производительности для ключевых действий пользователей

1. Перейти в **Web**.
2. Выбрать приложение, которое нужно настроить.
3. В разделе **Top 3 user actions** выбрать **View full details**.
4. Прокрутить вниз и перейти на вкладку **Key user actions**.
5. Выбрать нужное ключевое действие пользователя.
6. На странице сведений о ключевом действии пользователя выбрать **More** (**…**) > **Edit**.
7. Выбрать ключевую метрику производительности, которая лучше всего отражает пользовательский опыт для этого ключевого действия пользователя.
