# Определение целей конверсии в RUM Classic

* Практическое руководство
* Чтение за 1 минуту
* Опубликовано 15 июля 2019 г.

Можно определить цели конверсии для конкретных пользовательских действий, чтобы понять, насколько успешно достигаются вехи конверсии, например успешные оформления заказа, подписки на рассылку или заявки на демоверсию. Можно добавить цели для достижения конкретных пользовательских действий или целевых URL, а также для информации о сессии, например для сессий с более чем 10 пользовательскими действиями. В этом примере пользователю нужно выполнить как минимум 10 пользовательских действий в рамках одной сессии, чтобы достичь этой цели конверсии.

Такие пользовательские действия стоит внимательно отслеживать не только с технической точки зрения, чтобы убедиться, что всё работает как ожидается и продолжительность действий приемлема, но и для того, чтобы увидеть, сколько клиентов на самом деле пользуется теми или иными функциями. Этого можно достичь, используя подход [ключевых действий](/managed/observe/digital-experience/rum-classic/rum-concepts/user-actions#key-user-actions "Learn what user actions are and how they help you understand what users do with your application.") и целей конверсии.

Цели конверсии можно определить для каждого приложения отдельно. В основу цели конверсии можно положить целевой URL, конкретное пользовательское действие, продолжительность сессии или количество действий за сессию.

Чтобы определить цель конверсии для приложения

1. Перейти в **Web**.
2. Выбрать приложение, которое нужно настроить.
3. В правом верхнем углу страницы обзора приложения выбрать **More** (**…**) > **Edit**.
4. В настройках приложения выбрать **Behavior analytics** > **Conversion goals**.
5. Выбрать **Add goal**.
6. Ввести название цели конверсии и настроить остальные параметры.
7. Выбрать **Add goal**.

Для одного приложения можно определить не более 20 целей конверсии.
