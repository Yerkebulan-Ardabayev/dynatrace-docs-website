# Настройка автоматического внедрения в RUM Classic

# Настройка автоматического внедрения в RUM Classic

* Практическое руководство
* Обновлено 01 июля 2025 г.

После развёртывания OneAgent в режиме полного мониторинга стека (full-stack monitoring) на хосте, он по умолчанию автоматически внедряет RUM JavaScript в HTML веб-приложений для технологий, перечисленных в разделе [Поддержка технологий - Real User Monitoring - веб-серверы и приложения](/managed/ingest-from/technology-support#rum-auto-injection "Найти технические детали, связанные с поддержкой Dynatrace для конкретных платформ и фреймворков разработки."). Чтобы инициировать внедрение, достаточно перезапустить процессы веб-сервера после установки OneAgent.

По умолчанию внедрение следует набору [встроенных правил автоматического внедрения](#where-is-rum-js-injected), но бывают сценарии, в которых требуется определить [пользовательское правило внедрения](#custom-injection-rule) для достижения внедрения. Пользовательские правила внедрения также поддерживают [исключение отдельных страниц из внедрения](#disable-injection), что позволяет настроить [ручную вставку для страниц приложения с автоматическим внедрением](#manual-insertion-using-oneagent).

RUM JavaScript отвечает за сбор данных RUM из веб-браузеров пользователей. Он отправляет эти данные в виде XHR POST-запросов на конечную точку beacon, которая выполняет первичную проверку перед пересылкой данных в кластер Dynatrace. Конечная точка beacon по умолчанию в сценариях автоматического внедрения предоставляется OneAgent, инструментирующим веб-сервер или сервер приложений, но возможны и альтернативные настройки, см. [Настройка конечной точки beacon для веб-приложений в RUM Classic](/managed/observe/digital-experience/rum-classic/web-applications/additional-configuration/beacon-endpoint "Изменить URL конечной точки beacon по умолчанию и отправлять beacon-запросы RUM в инфраструктуру Dynatrace или на другой инструментированный веб-сервер.").

Фрагмент RUM JavaScript, внедряемый в HTML приложения, может отличаться по формату, каждый из которых предназначен для удовлетворения конкретных требований, описанных в разделе [Выбор формата фрагмента в RUM Classic](/managed/observe/digital-experience/rum-classic/web-applications/initial-setup/snippet-formats "Выбрать формат фрагмента RUM JavaScript, наиболее подходящий для конкретного сценария использования").
Для настройки формата, используемого при автоматическом внедрении, см. [Настройка формата фрагмента](#configure-snippet-format).

OneAgent внедряет RUM JavaScript на каждую страницу на основе текущей конфигурации всякий раз, когда инструментированный веб-сервер или сервер приложений доставляет страницу. Однако политика кэширования страницы определяет, как часто у OneAgent появляется возможность выполнить это внедрение. Для решения этой проблемы OneAgent включает функцию, оптимизирующую заголовки управления кэшем. Подробнее о том, как работает эта функция, и о её ограничениях см. в разделе [Оптимизации заголовков управления кэшем](#cache-header-optimization).

Если нет доступа к веб-серверу, на котором размещено приложение, и поэтому нет возможности установить OneAgent, можно отслеживать приложение с помощью [агентless RUM](/managed/observe/digital-experience/rum-classic/web-applications/initial-setup/set-up-agentless-real-user-monitoring "Настройка мониторинга без агента для веб-приложений.").

## Встроенные правила автоматического внедрения

OneAgent обнаруживает ответы, содержащие HTML-контент, и внедряет RUM JavaScript в раздел head каждой страницы. Он не изменяет другие ресурсы, такие как изображения, файлы CSS, документы JSON или XML, либо простой текст.

OneAgent пытается внедрить RUM JavaScript как первый скрипт на странице, что помогает обеспечить надёжные результаты мониторинга. Внедрение работает только с корректным HTML, соответствующим основным передовым практикам, например, все элементы должны быть правильно открыты и закрыты. OneAgent следует указанным ниже встроенным правилам, чтобы определить наиболее подходящее место в HTML-документе для внедрения RUM JavaScript.

### Спецификация правил автоматического внедрения

## Явное указание места внедрения RUM JavaScript

В некоторых случаях OneAgent может быть не в состоянии выполнить внедрение с использованием [встроенных правил](#where-is-rum-js-injected). Обычно это связано с некорректным HTML, хотя другие редкие условия также могут помешать набору правил определить подходящую точку внедрения. В таких сценариях можно добиться автоматического внедрения, определив пользовательское правило внедрения, явно указывающее, куда должен быть внедрён RUM JavaScript в HTML-документе.

Чтобы создать пользовательское правило внедрения, указывающее точку внедрения в HTML-документе

1. Перейти в **Web**.
2. Выбрать приложение, которое нужно настроить.
3. В правом верхнем углу страницы обзора приложения выбрать **More** (**…**) > **Edit**.
4. В настройках приложения выбрать **Injection** > **Custom injection rules**.
5. В разделе **Define custom injection rules** выбрать **Add custom rule**.
6. Определить **Operator** и, при необходимости, соответствующий **URL pattern**, чтобы указать, к каким страницам применяется пользовательское правило внедрения.
7. В разделе **Rule** выбрать **Before specific HTML** или **After specific HTML** и ввести соответствующий HTML-паттерн.
8. Выбрать **Save changes**, чтобы создать пользовательское правило внедрения.

## Отключение внедрения для отдельных страниц

Чтобы отключить внедрение для одной или нескольких страниц приложения

1. Перейти в **Web**.
2. Выбрать приложение, которое нужно настроить.
3. В правом верхнем углу страницы обзора приложения выбрать **More** (**…**) > **Edit**.
4. В настройках приложения выбрать **Injection** > **Custom injection rules**.
5. В разделе **Define custom injection rules** выбрать **Add custom rule**.
6. Определить **Operator** и, если требуется, соответствующий **URL pattern**, чтобы указать, к каким страницам применяется пользовательское правило внедрения.
7. В разделе **Rule** выбрать **Do not inject**.
8. Выбрать **Save changes**, чтобы создать пользовательское правило внедрения.

Это правило отключает только внедрение и [оптимизации заголовков управления кешем](#cache-header-optimization), оно не отключает функциональность RUM полностью. Например, cookie всё равно могут устанавливаться. Если нужен способ выборочно включать RUM, см. инструкции в разделе [Выборочное развёртывание RUM Classic для приложений](/managed/observe/digital-experience/rum-classic/web-applications/initial-setup/selective-rum-rollout "Roll out RUM selectively after installing OneAgent in full-stack monitoring mode on your hosts").

## Настройка формата фрагмента

Чтобы настроить формат фрагмента для автоматического внедрения

1. Перейти в **Web**.
2. Выбрать приложение, которое нужно настроить.
3. В правом верхнем углу страницы обзора приложения выбрать **More** (**…**) > **Edit**.
4. В настройках приложения выбрать **Injection** > **Automatic injection**.
5. В разделе **Snippet format** выбрать нужный формат.

Подробнее о разных форматах фрагмента см. в разделе [Выбор формата фрагмента в RUM Classic](/managed/observe/digital-experience/rum-classic/web-applications/initial-setup/snippet-formats "Select a format for the RUM JavaScript snippet that best fits your specific use case").

## Оптимизации заголовков управления кешем

🟢 Включено по умолчанию

Политика кеширования приложения напрямую влияет на то, как часто OneAgent может внедрять RUM JavaScript на страницу при текущей конфигурации. Чтобы обеспечить своевременное применение изменений конфигурации, функция оптимизации заголовков управления кешем изменяет HTTP-заголовки так, чтобы условные запросы приводили к промаху кеша при каждом изменении конфигурации RUM, давая OneAgent возможность внедрения. Подробности этих оптимизаций и конкретных изменений заголовков приведены ниже.

Спецификация оптимизаций заголовков управления кешем

Когда оптимизация заголовков управления кешем включена и конфигурация RUM была изменена, запросы, которые в противном случае привели бы к `304 Not Modified`, отвечают `200 OK`, давая OneAgent возможность внедрить RUM JavaScript в HTML-код приложения. Если конфигурация RUM не изменялась, запросы приводят к ответу `304 Not Modified`.

### Изменённые заголовки

OneAgent изменяет следующие заголовки ответа и запроса:

#### Заголовки ответа

* `ETag`
* `Last-Modified`

#### Заголовки запроса

* `If-None-Match`
* `If-Modified-Since`
* `If-Unmodified-Since`[1](#fn-1-1-def)
* `If-Match`[1](#fn-1-1-def)

1

Не изменяется для запросов диапазона (range requests), которые, как правило, исключены из мониторинга RUM.

Ниже описано, как OneAgent изменяет заголовки управления кешем.

#### Когда конфигурация RUM изменена

* Входящие заголовки запроса `If-None-Match` и `If-Modified-Since` удаляются.
* Изменения, внесённые в заголовки `If-Unmodified-Since` и `If-Match`, удаляются.
* Когда используются оба заголовка, `ETag` и `Last-Modified`:

  + К заголовку `ETag` добавляется суффикс.
  + Из заголовка `Last-Modified` вычитается одна секунда.
* Когда используется только заголовок `ETag`:

  + К заголовку `ETag` добавляется суффикс.
* Когда используется только заголовок `Last-Modified`:

  + Создаётся заголовок `ETag`.
  + Из заголовка `Last-Modified` вычитается одна секунда.

#### Когда конфигурация RUM не изменена

* Перед тем как запрос обрабатывается веб- или прикладным сервером, изменения, внесённые в заголовки, удаляются, чтобы веб- или прикладной сервер не видел этих изменений.

Когда приложение состоит из нескольких инструментированных уровней, изменения заголовков `ETag` и `Last-Modified` применяются на каждом уровне.

OneAgent не изменяет заголовки `Expires` и `Cache-Control`, которые явно определяют время истечения срока действия ресурса. Для обеспечения своевременных обновлений рекомендуется устанавливать более короткие сроки действия в этих заголовках.

### Отключение оптимизаций заголовков управления кешем

В редких случаях оптимизации заголовков управления кешем могут мешать работе приложения. Если это происходит, функцию можно отключить.
При отключении развёртывание новых конфигураций RUM будет полностью зависеть от существующего поведения кеширования приложения.

Чтобы отключить оптимизации заголовков управления кешем

1. Перейти в **Web**.
2. Выбрать приложение, которое нужно настроить.
3. В правом верхнем углу страницы обзора приложения выбрать **More** (**…**) > **Edit**.
4. В настройках приложения выбрать **Injection** > **Automatic injection**.
5. Отключить **Optimize the value of cache control headers for use with Dynatrace Real User Monitoring Classic**.
6. Выбрать **Save changes**.

### Отключение оптимизаций заголовков управления кешем для не-HTML-ресурсов

В некоторых случаях OneAgent может изменять заголовки управления кешем не-HTML-ресурсов. Это происходит потому, что изменения заголовков применяются до того, как можно определить точный тип содержимого. Хотя OneAgent исключает некоторые ресурсы из оптимизаций заголовков управления кешем на основе суффикса URI, учесть все не-HTML типы содержимого невозможно.

Если нужно предотвратить изменения заголовков управления кешем для конкретных не-HTML-ресурсов, можно создать [пользовательское правило внедрения, которое отключает внедрение для этих ресурсов](#disable-injection).

## Использование ручной вставки для страниц приложения с автоматическим внедрением

Если OneAgent развёрнут на группах процессов и автоматическое внедрение поддерживается для используемой технологии, OneAgent по умолчанию внедряет RUM JavaScript автоматически. Но при желании его можно вставить вручную вместо этого на все или отдельные страницы.

Сначала нужно подавить автоматическое внедрение, как описано в разделе [Отключение внедрения для отдельных страниц](#disable-injection). Затем, чтобы вставить RUM JavaScript вручную

1. Перейти в **Web**.
2. Выбрать приложение, которое нужно настроить.
3. В правом верхнем углу страницы обзора приложения выбрать **More** (**…**) > **Edit**.
4. В настройках приложения выбрать **Injection** > **Manual insertion**.
5. Выбрать нужный [формат фрагмента](/managed/observe/digital-experience/rum-classic/web-applications/initial-setup/snippet-formats "Select a format for the RUM JavaScript snippet that best fits your specific use case") и скопировать предоставленный фрагмент.
6. Вставить фрагмент в HTML-код перед любыми другими скриптами.

Также фрагмент можно получить через [API](/managed/dynatrace-api/environment-api/rum/real-user-monitoring-javascript-code "Learn how you can use the Dynatrace API to set up and maintain your manually injected applications using the Real User Monitoring JavaScript API."), что позволяет интегрировать его вставку в процесс сборки. Это обеспечивает постоянную работу приложения с актуальной конфигурацией.
