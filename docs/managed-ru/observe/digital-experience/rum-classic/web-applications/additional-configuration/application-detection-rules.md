# Проверка правил определения приложений в RUM Classic

* Пояснение
* Чтение 1 мин
* Опубликовано 3 апр 2020

С помощью правил определения приложений можно задать транзакции, из которых складываются RUM-приложения. Действия пользователя, обнаруженные Dynatrace, группируются вместе, представляя приложение с точки зрения пользователя, что даёт возможность отдельного мониторинга и анализа в Dynatrace.

Теперь можно проверить, действительно ли заданные правила определения обнаруживают трафик с нужных URL и точно отражают приложения, для обнаружения которых они предназначены. Проверив URL любого автоматически внедрённого RUM-приложения, можно:

* увидеть, какие именно правила определения соответствуют введённому URL
* убедиться, что RUM включён для отслеживаемого приложения
* быстро перейти к настройке правила определения, выбрав кнопку **Edit** у нужного правила
* получить контроль над настройкой и тестированием правил определения приложений

Чтобы лучше понять заданные правила определения приложений и их влияние на приложение

1. Перейти в **Settings** > **Web and mobile monitoring** > **Application detection**.
2. В разделе **Check your existing detection rules** ввести URL и выбрать **Check URL**.

В одной среде можно создать до 1000 правил определения приложений.
