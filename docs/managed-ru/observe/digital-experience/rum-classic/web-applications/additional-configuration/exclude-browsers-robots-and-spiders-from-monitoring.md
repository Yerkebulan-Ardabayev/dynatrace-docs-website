# Исключение IP-адресов, браузеров, ботов и пауков из мониторинга для веб-приложений в RUM Classic

# Исключение IP-адресов, браузеров, ботов и пауков из мониторинга для веб-приложений в RUM Classic

* Практическое руководство
* Чтение за 2 мин
* Обновлено 21 марта 2024 г.

Dynatrace позволяет исключать из Real User Monitoring отдельные IP-адреса, типы браузеров, роботов и пауков.

С помощью настроек, описанных ниже, нельзя заблокировать трафик, создаваемый [синтетическими мониторами Dynatrace](/managed/observe/digital-experience/synthetic-monitoring/general-information/types-of-synthetic-monitors "Узнайте о типах синтетических мониторов Dynatrace."). В частности, исключение IP-адресов синтетических локаций, а также исключение ботов и пауков не блокирует трафик Synthetic Monitoring.

Хотя приложения, назначенные вручную, можно отвязать от синтетических мониторов, отвязать от синтетических мониторов приложения, назначенные автоматически, нельзя.

Чтобы исключить синтетические данные из анализа, используй фильтр **User type: Real users**, доступный на большинстве страниц с деталями приложения, в [Data Explorer](/managed/analyze-explore-automate/explorer "Запрашивай метрики и преобразуй результаты, чтобы получить нужные выводы."), [сегментации сессий](/managed/observe/digital-experience/rum-classic/session-segmentation/user-sessions "Узнай о сегментации пользовательских сессий и атрибутах фильтрации."), [USQL](/managed/observe/digital-experience/rum-classic/session-segmentation/custom-queries-segmentation-and-aggregation-of-session-data "Узнай, как получать доступ и запрашивать данные пользовательских сессий по ключевым словам, синтаксису, функциям и прочему.") и других инструментах.

## Исключение IP-адресов

В настройках приложения можно исключить один IP-адрес или диапазон IP-адресов.

Чтобы исключить IP-адрес

1. Перейди в **Web**.
2. Выбери приложение, которое нужно настроить.
3. В правом верхнем углу страницы обзора приложения выбери **More** (**…**) > **Edit**.
4. В настройках приложения выбери **Capturing** > **Exclusions** > **IP address exclusions**.
5. Выбери **Add an IP address to exclude**.
6. Укажи IP-адреса, которые нужно исключить из Real User Monitoring.

## Исключение браузеров

Если нужно исключить определённые устаревшие типы браузеров из списка отслеживаемых браузеров, создай правила исключения браузеров.

Чтобы создать правило исключения браузера

1. Перейди в **Web**.
2. Выбери приложение, которое нужно настроить.
3. В правом верхнем углу страницы обзора приложения выбери **More** (**…**) > **Edit**.
4. В настройках приложения выбери **Capturing** > **Exclusions** > **Browser exclusions**.
5. Выбери **Add a browser exclusion rule**.
6. В раскрывающемся списке **Browser** выбери браузер, который нужно исключить, а затем укажи точную версию и тип.

## Исключение трафика ботов и пауков

В отслеживаемом приложении может встречаться трафик, создаваемый автоматическими ботами или пауками. Такой трафик может вызывать оповещения **Unexpected low traffic**, из-за которых сложно увидеть действительно важные проблемы. Ботов и пауков можно исключить из RUM.

Чтобы исключить ботов и пауков из RUM

1. Перейди в **Web**.
2. Выбери приложение, которое нужно настроить.
3. В правом верхнем углу страницы обзора приложения выбери **More** (**…**) > **Edit**.
4. В настройках приложения выбери **Capturing** > **Exclusions** > **Browser exclusions**.
5. Выбери **Add a browser exclusion rule**.
6. В раскрывающемся списке **Browser** выбери **Bots & Spiders**.

Обновление всех экземпляров OneAgent с учётом внесённых изменений занимает несколько минут.

В отличие от [классификации пользователей, которую Dynatrace добавляет к собранным данным RUM](/managed/observe/digital-experience/rum-classic/rum-concepts/detection-of-ip-addresses-locations-and-user-agents#browsers "Dynatrace определяет IP-адреса и геолокацию, например город, регион и страну, а также браузеры, устройства и операционные системы."), эта функция не полагается на запросы к базе данных, поскольку применяется на стороне OneAgent. Вместо этого [строка user agent﻿](https://developer.mozilla.org/docs/Web/HTTP/Headers/User-Agent) сопоставляется с набором шаблонов, что менее точно, чем запрос к базе данных. По этой причине часть трафика ботов и пауков всё же может быть захвачена.
