# Настройка обнаружения ресурсов первой стороны, третьих сторон и CDN для веб-приложений в RUM Classic

* Практическое руководство
* Чтение за 1 минуту
* Обновлено 30 июня 2022 г.

Dynatrace распознаёт более 1000 провайдеров стороннего контента, включая Google, Amazon, Twitter и LinkedIn. Типичный сторонний контент, это виджеты Facebook и Twitter, граватары и похожие ресурсы. Важно видеть, как провайдеры стороннего контента влияют на приложения, и понимать, является ли такой контент первопричиной замедлений.

Сети доставки контента (CDN) обычно находятся под собственным контролем. Приложение может использовать CDN для хранения и предоставления часто запрашиваемого статического и динамического контента, что требуется для ускорения загрузки в определённых географических регионах.

## Различие между сторонними провайдерами и CDN

Различие между провайдерами стороннего контента и CDN заключается в степени контроля над данными.

* Если используются внешние данные, над которыми нет контроля, контент поступает от стороннего провайдера.
* Если приложение полагается на внешний ресурс для доставки статического или динамического контента, например изображений или CSS, такой сервис стоит рассматривать как CDN.

## Управление провайдерами окружения

### Просмотр полного списка провайдеров

Dynatrace обнаруживает множество провайдеров стороннего контента. Когда Dynatrace не может найти имя хоста в списке известных сторонних провайдеров, он помечает ресурс как CDN. Кроме того, Dynatrace может определить, какие домены хостит организация, и отнести такие домены к первой стороне.

Чтобы увидеть полный список провайдеров, обнаруженных для окружения, перейдите в **Settings** > **Web and mobile monitoring** > **Provider breakdown**.

* Провайдеры, добавленные вручную, перечислены прямо под кнопкой **Add custom provider item**. Провайдера можно [добавить](#add-provider-manually) и пометить как первую сторону, третью сторону или CDN.
* Автоматически обнаруженные CDN и сторонние провайдеры перечислены в разделе **Auto detected providers**.

### Добавление провайдера вручную

Если провайдер не найден в списке автоматически обнаруженных или требуется переопределить правила автообнаружения, провайдера можно добавить вручную.

Чтобы добавить провайдера вручную

1. Перейдите в **Settings** > **Web and mobile monitoring** > **Provider breakdown**.
2. Выберите **Add custom provider item**.
3. Введите **Resource name** и укажите **Resource type**, чтобы определить, является ли ресурс первой стороной, третьей стороной или CDN.
4. Необязательно: укажите URL значка бренда провайдера.
5. Выберите **Add name pattern**, а затем введите шаблон для доменного имени провайдера. При необходимости можно указать несколько шаблонов.
6. Необязательно: включите **Submit this provider-pattern to improve auto-detection**, если считаете, что Dynatrace должен автоматически распознавать этот ресурс как сторонний.

Начиная с версии Dynatrace 1.240, автообнаружение провайдеров первой стороны включено по умолчанию и не может быть отключено. Если нужно переопределить автоматически обнаруженных провайдеров первой стороны, [добавьте провайдера вручную](#add-provider-manually).

### Изменение порядка вручную добавленных провайдеров

Список вручную добавленных провайдеров обрабатывается в порядке их появления в списке. По этой причине провайдер с более конкретным шаблоном домена должен идти первым. Например, `api.example.com` должен идти перед `example.com`.

Чтобы изменить порядок вручную добавленных провайдеров

1. Перейдите в **Settings** > **Web and mobile monitoring** > **Provider breakdown**.
2. Найдите список вручную добавленных провайдеров под кнопкой **Add custom provider item**.
3. Используйте **Drag handle** ![Drag handle](https://dt-cdn.net/images/drag-handle-turquoise-600-1aa0e5ea00.svg "Drag handle"), чтобы изменить приоритет провайдера.
