# Настройка определения IP-адресов для веб-приложений в RUM Classic

* Практическое руководство
* 1 минута на чтение
* Обновлено 21 марта 2024 г.

Когда веб-запросы и RUM beacons отправляются напрямую на инструментированный сервер, Dynatrace определяет IP-адреса устройств конечных пользователей через сокетные соединения.

При использовании неинструментированных компонентов, таких как балансировщики нагрузки, CDN или прокси, удалённый IP-адрес отличается от исходного. В таких случаях Dynatrace также отслеживает определённые заголовки HTTP. Эти заголовки чаще всего используются для определения исходного IP-адреса, когда клиент подключается к веб-серверу через HTTP-прокси, CDN или балансировщик нагрузки.

Список заголовков, которые Dynatrace использует для определения IP-адресов клиентов в приложениях, можно посмотреть в разделе **Settings** > **Web and mobile monitoring** > **IP determination**. Dynatrace обрабатывает эти заголовки в определённом порядке, но порядок обработки можно изменить и добавить собственные заголовки.

## Управление заголовками по умолчанию

Можно изменять названия заголовков, удалять ненужные заголовки и менять порядок, в котором Dynatrace их обрабатывает.

1. Перейти в **Settings** > **Web and mobile monitoring** > **IP determination**.
2. Управление заголовками:

   * Чтобы переименовать заголовок, выбрать **Expand row** ![Expand row](https://dt-cdn.net/images/expand-row-icon-9c4950fc2e.svg "Expand row") и ввести новое название заголовка.
   * Чтобы удалить заголовок, выбрать **Delete row** ![Remove](https://dt-cdn.net/images/remove-icon-105c6a04c2.svg "Remove").
   * Чтобы изменить приоритет заголовка, выбрать и удерживать **Drag row** ![Drag handle](https://dt-cdn.net/images/drag-handle-turquoise-600-1aa0e5ea00.svg "Drag handle") рядом с названием заголовка и переместить его вверх или вниз.

## Добавление собственных заголовков IP-адреса клиента

Если IP-адреса клиентов находятся в другом заголовке, нужно создать собственное правило, чтобы Dynatrace мог корректно определять IP-адреса.

Чтобы добавить собственный заголовок IP-адреса клиента

1. Перейти в **Settings** > **Web and mobile monitoring** > **IP determination**.
2. Выбрать **Create custom client IP header**.
3. Указать название заголовка.

На одну среду можно создать до 10 собственных заголовков IP-адреса клиента.

Добавление собственного заголовка IP-адреса клиента может потребоваться в одном из следующих случаев.

* Заголовок с IP-адресами клиентов отсутствует в списке заголовков в разделе **Settings** > **Web and mobile monitoring** > **IP determination**.
* IP-адреса клиентов для приложений хранятся в собственном (proprietary) заголовке.
* IP-адреса определяются некорректно, например, когда в Dynatrace отображаются IP-адреса балансировщиков нагрузки.

## Причины некорректного определения IP-адреса клиента

Может быть несколько причин, по которым в приложении не отображаются корректные IP-адреса клиентов. Подробные инструкции см. в разделах выше.

| Проблема | Возможное решение |
| --- | --- |
| IP-адреса клиентов для приложений находятся в заголовке, которого нет в нашем списке. | Добавить собственный заголовок IP-адреса клиента. |
| Указан неверный собственный заголовок IP-адреса клиента. | Обновить название заголовка. |
| Указанный собственный заголовок IP-адреса клиента фактически не содержит IP-адреса. | Указать другой собственный заголовок IP-адреса клиента. |
| Порядок заголовков указан неверно. | Изменить порядок обработки заголовков. |
| Инфраструктура не настроена на добавление заголовка, передающего IP-адреса клиентов. | Перенастроить инфраструктуру. |
| Заголовок, содержащий IP-адреса клиентов, не входит в состав beacon-запросов.[1](#fn-1-1-def) | Перенастроить инфраструктуру. |

1

Заголовки определения IP в обычных запросах приложения не имеют значения для определения IP захваченных данных RUM, даже если они захватываются как трассировки и эти трассировки соотносятся с действиями пользователя.
