# Настройка ключевых пользовательских действий для веб-приложений в RUM Classic

* Практическое руководство
* Чтение 1 мин
* Опубликовано 27 января 2023 г.

Большинство приложений включают пользовательские действия, особенно важные для успеха цифрового бизнеса. Примерами таких действий служат регистрация, оформление заказа и поиск товаров. Выполнение таких [ключевых пользовательских действий](/managed/observe/digital-experience/rum-classic/rum-concepts/user-actions#key-user-actions "Узнайте, что такое пользовательские действия и как они помогают понять, что пользователи делают в приложении.") может занимать больше времени, чем других, либо к ним может предъявляться требование выполняться быстрее среднего.

С помощью функции ключевых пользовательских действий можно [настроить пороги Apdex](/managed/observe/digital-experience/rum-classic/rum-concepts/scores-and-ratings/apdex-ratings#key-user-actions "Узнайте, как Dynatrace использует Apdex для измерения удовлетворённости пользователей производительностью приложения.") для каждого из этих пользовательских действий. С помощью этой функции можно отслеживать ключевые действия с помощью отдельного плитки дашборда и следить за историческими трендами.

## Отметить пользовательское действие как ключевое

1. Перейти в **Frontend**.
2. Выбрать приложение и прокрутить вниз до **Top 3 user actions** или **Top 3 actions**.
3. Выбрать **View full details** или **Analyze performance**.
4. Прокрутить вниз, чтобы увидеть список пользовательских действий, и выбрать нужное пользовательское действие.  
   Откроется страница с подробностями о пользовательском действии.
5. В правом верхнем углу страницы с подробностями о пользовательском действии выбрать **Mark as key user action**.

Можно определить до 500 ключевых пользовательских действий на среду для всех приложений и до 100 ключевых пользовательских действий на приложение.

При достижении максимального лимита ключевых пользовательских действий стоит рассмотреть использование расчётных метрик (доступны для [веб](/managed/observe/digital-experience/rum-classic/web-applications/additional-configuration/rum-calculated-metrics-web "Создание расчётных метрик, а также пользовательских графиков на основе расчётных метрик для веб-приложений."), [мобильных](/managed/observe/digital-experience/rum-classic/mobile-applications/additional-configuration/rum-calculated-metrics-mobile "Создание расчётных метрик, а также пользовательских графиков на основе расчётных метрик для мобильных приложений.") и [пользовательских приложений](/managed/observe/digital-experience/rum-classic/custom-applications/additional-configuration/rum-calculated-metrics-custom "Создание расчётных метрик, а также пользовательских графиков на основе расчётных метрик для пользовательских приложений.")), которые предлагают похожие возможности.

## Закрепить ключевое пользовательское действие на дашборде

[Dashboards Classic](/managed/analyze-explore-automate/dashboards-classic "Узнайте, как создавать и использовать Dynatrace Dashboards Classic, а также управлять ими.")

1. Перейти в **Frontend**.
2. Выбрать приложение и прокрутить вниз до **Top 3 user actions** или **Top 3 actions**.
3. Выбрать **View full details** или **Analyze performance**.
4. Найти нужное ключевое пользовательское действие и выбрать его.  
   Откроется страница с подробностями о пользовательском действии.
5. В верхней части страницы с подробностями о пользовательском действии выбрать **Pin to dashboard**. Подробности см. в разделе [Закрепление плиток на дашборде](/managed/analyze-explore-automate/dashboards-classic/charts-and-tiles/pin-tiles-to-your-dashboard "Узнайте, как закреплять плитки на дашбордах.").

## Настройка рейтинга Apdex для ключевого пользовательского действия

Чтобы изменить пороги Apdex для ключевого пользовательского действия

1. Перейти в **Frontend**.
2. Выбрать приложение и прокрутить вниз до **Top 3 user actions** или **Top 3 actions**.
3. Выбрать **View full details** или **Analyze performance**.
4. Найти нужное ключевое пользовательское действие и выбрать его.  
   Откроется страница с подробностями о пользовательском действии.
5. В правом верхнем углу страницы с подробностями о пользовательском действии выполнить одно из следующего:

   * Для веб-приложений выбрать **More** (**…**) > **Edit** > **Key performance metric**.
   * Для мобильных и пользовательских приложений выбрать ![Expand](https://dt-cdn.net/images/expandbutton-40-e1f11ff81d.png "Expand") (**Expand**) > **Edit** > **Key performance metric**.
6. С помощью ползунка настроить пороги Apdex.
