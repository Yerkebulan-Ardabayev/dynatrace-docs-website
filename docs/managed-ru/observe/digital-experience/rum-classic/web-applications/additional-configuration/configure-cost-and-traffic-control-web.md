# Настройка контроля затрат и трафика для веб-приложений в RUM Classic

* Практическое руководство
* Чтение за 1 минуту
* Опубликовано 29 сент. 2021 г.

По умолчанию Dynatrace фиксирует все действия пользователей и пользовательские сессии для анализа. Такой подход обеспечивает полное представление о производительности приложения и клиентском опыте. С помощью настройки **Cost and traffic control** можно снизить детализацию анализа действий пользователей и пользовательских сессий, указав Dynatrace фиксировать меньший процент пользовательских сессий.

Эта настройка позволяет снизить [расходы на мониторинг](/managed/license/classic-licensing/digital-experience-monitoring-units "Понять, как рассчитывается потребление Dynatrace Digital Experience Monitoring на основе единиц DEM."), однако при этом снижается видимость того, как клиенты используют веб-приложения. Например, значение `10%` приводит к тому, что Dynatrace анализирует только каждую десятую пользовательскую сессию.

Чтобы ограничить число анализируемых пользовательских сессий

1. Перейти в **Web**.
2. Выбрать приложение, которое нужно настроить.
3. В верхнем правом углу страницы обзора приложения выбрать **More** (**…**) > **Edit**.
4. В настройках приложения выбрать **General settings** > **Enablement and cost control**.
5. В разделе **Real User Monitoring** ввести значение меньше `100` в поле **Cost and traffic control**. Значение по умолчанию для этого поля, `100`.

При заданной настройке Dynatrace анализирует равномерно распределённое число пользовательских сессий, соответствующее указанному проценту пользовательских сессий.
