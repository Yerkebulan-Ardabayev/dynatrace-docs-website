# Устранение неполадок RUM Classic для веб-приложений

* Справка
* Чтение: 2 мин
* Обновлено 07 июля 2023 г.

На этой странице объясняется, что делать, если Real User Monitoring Classic (RUM Classic) не работает в среде.

Перед тем как приступить к устранению неполадок, стоит проверить [страницу проверки состояния](/managed/observe/digital-experience/rum-classic/web-applications/initial-setup/app-health-check "The Application health check page allows you to analyze the health of your application, see which RUM JavaScript versions are currently in use, or confirm that the RUM JavaScript is injected correctly.") для приложения.

## Общее устранение неполадок

По общим вопросам устранения неполадок см. [Web applications: Issues with RUM JavaScript﻿](https://dt-url.net/up03l19) в сообществе Dynatrace.

## Устранение неполадок, связанных с пользовательскими действиями

Для устранения неполадок, связанных с пользовательскими действиями, см. следующие страницы в сообществе Dynatrace.

* [User action data is missing﻿](https://dt-url.net/zqa3lef)
* [Web request isn't linked to its user action﻿](https://dt-url.net/io63ls2)
* [Sessions are randomly split into shorter sessions﻿](https://dt-url.net/cb23ley)
* [Source actions are missing for subdomain navigations﻿](https://dt-url.net/xi43lit)

## Устранение неполадок, связанных с метаданными

Если возникла проблема, связанная с метаданными, см. [Web applications: Sessions are not correctly tagged or metadata is missing﻿](https://dt-url.net/rp83l5k) в сообществе Dynatrace.
