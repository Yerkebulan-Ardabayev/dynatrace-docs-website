# Метрики пользовательских действий в RUM Classic

# Метрики пользовательских действий в RUM Classic

* Пояснение
* Чтение за 8 минут
* Обновлено 20 марта 2023 г.

Для веб-приложений Dynatrace рассчитывает множество метрик [пользовательских действий](/managed/observe/digital-experience/rum-classic/rum-concepts/user-actions "Learn what user actions are and how they help you understand what users do with your application."), пользовательских действий.

Основные веб-показатели

Следующие метрики относятся к основным веб-показателям (core web vitals) и доступны только для действий загрузки:

* **Largest contentful paint**
* **Cumulative layout shift**
* **First input delay**

## Метрики контрольных точек

Метрики контрольных точек представляют собой события, которые происходят в определённый момент времени во время пользовательского действия. При этом эти метрики измеряются в секундах или миллисекундах, поскольку рассчитываются относительно времени начала пользовательского действия. Иными словами, время начала пользовательского действия служит опорной точкой, от которой отсчитывается время для метрик контрольных точек. Например, значение 47 мс для **Navigation start** означает, что это событие произошло через 47 мс после времени начала действия.

| Метрика | Описание | Применимо к |
| --- | --- | --- |
| **Navigation start** | Момент, когда пользовательский агент завершает запрос на выгрузку предыдущей страницы. Если предыдущей страницы нет, это значение совпадает с `PerformanceTiming.fetchStart`. | Действия загрузки |
| **Request start** | Время до того, как пользовательский агент отправляет запрос на получение страницы с сервера, из соответствующих кэшей приложения или локального ресурса. Если транспортный уровень даёт сбой и соединение переоткрывается, эта метрика устанавливается на время, соответствующее новому запросу. | Действия загрузки, действия XHR |
| **Time to first byte** | Время, затраченное до получения первого байта ответа от сервера, соответствующих кэшей приложения или локального ресурса | Действия загрузки, действия XHR |
| **HTML downloaded** | Время, затраченное до получения пользовательским агентом последнего байта ответа или до закрытия транспортного соединения, в зависимости от того, что произойдёт раньше. Для действий XHR эта метрика называется **Response end**. | Действия загрузки, действия XHR |
| **DOM interactive** | Время, затраченное до установки статуса страницы «interactive», когда она готова принимать ввод | Действия загрузки |
| **Speed index**[1](#fn-1-1-def) | Оценка, измеряющая, насколько быстро отрисовываются видимые части страницы | Действия загрузки |
| **DOM content loaded** | Время, затраченное до того, как пользовательский агент вызовет событие `DOMContentLoaded` на странице. Событие `DOMContentLoaded` выполняется, когда базовый HTML страницы загружен и его разбор завершён. Событие не ожидает завершения загрузки дополнений (например, таблиц стилей, вложенных фреймов и изображений). | Действия загрузки |
| **First paint**[2](#fn-1-2-def) | Время, затраченное на отрисовку первого не заданного по умолчанию элемента фона | Действия загрузки |
| **First contentful paint**[3](#fn-1-3-def) | Время, затраченное на отрисовку первого фрагмента контента, например текста или изображений | Действия загрузки |
| **Largest contentful paint**[4](#fn-1-4-def) | Время, затраченное на отрисовку самого крупного элемента в области просмотра | Действия загрузки |
| **Visually complete**[1](#fn-1-1-def) | Время, затраченное на полную отрисовку контента в области просмотра | Действия загрузки, действия XHR |
| **DOM complete** | Время, затраченное до установки статуса страницы «complete» | Действия загрузки |
| **Load event start** | Время, затраченное до начала события загрузки страницы | Действия загрузки |
| **Load event end** | Время, затраченное до завершения события загрузки страницы | Действия загрузки |
| **User action duration** | Время, затраченное на завершение загрузки страницы. Сюда входит время загрузки XHR-запросов, инициированных до `loadEventEnd`, а также время загрузки динамических ресурсов и выполнения скриптов, запущенных изменениями DOM. | Действия загрузки, действия XHR, пользовательские действия |
| **First input start** | Момент, когда пользователь впервые взаимодействует со страницей, например, нажимает элемент управления интерфейса | Действия загрузки |
| **First input delay**[5](#fn-1-5-def) | Время от первого взаимодействия со страницей до момента, когда пользовательский агент может отреагировать на это взаимодействие | Действия загрузки |

1

**Speed index** и **Visually complete** доступны только в браузерах, поддерживающих [mutation observers﻿](https://developers.google.com/web/updates/2012/02/Detect-DOM-changes-with-Mutation-Observers): Microsoft Internet Explorer 11, Microsoft Edge 15+, Firefox 57+ и Google Chrome 61+.

2

**First paint** поддерживается во всех браузерах, кроме Internet Explorer.

3

**First contentful paint** доступна только для [браузеров на базе Chromium﻿](https://www.chromium.org/Home).

4

**Largest contentful paint** доступна для браузеров на базе Chromium. Измеряется с помощью [предоставленных Google API﻿](https://web.dev/lcp/#measure-lcp-in-javascript).

5

Для браузеров на базе Chromium **First input delay** измеряется с помощью [предоставленных Google API﻿](https://web.dev/fid/#measure-fid-in-javascript). Для Firefox, Safari и Internet Explorer 9+ метрика измеряется с помощью реализации Dynatrace, которая отслеживает события типа `click`, `mousedown`, `keydown` или `touchstart` и вычисляет задержку от момента регистрации таких событий JavaScript-кодом RUM до момента их фактического срабатывания.

## Метрики фаз

В отличие от метрик контрольных точек, метрики фаз подразумевают длительность.

| Метрика | Описание | Применимо к |
| --- | --- | --- |
| **Blocking** | Время между получением пользовательским агентом запроса на скачивание ресурса и фактическим началом скачивания ресурса. Сюда входит время, затраченное на ожидание свободного TCP-сокета, создание записей дискового кэша и согласование прокси. | Действия загрузки |
| **Application cache** | Время, затраченное на проверку соответствующих кэшей приложения. Сюда входит время до установления соединения с сервером. | Действия загрузки |
| **DNS lookup** | Время, затраченное на разрешение имени хоста для целевого URL | Действия загрузки |
| **TCP connect** | Время, затраченное на установление TCP-соединения с сервером (включая SSL) | Действия загрузки |
| **Secure connect** | Время, затраченное на защиту соединения, установленного с сервером. Сюда входят SSL-рукопожатие и SOCKS. | Действия загрузки |
| **Request** | Время, затраченное на запрос страницы с сервера до получения первого байта | Действия загрузки |
| **Response** | Время, затраченное на получение ответа | Действия загрузки |
| **Processing** | Время между загрузкой DOM и Load event start | Действия загрузки |
| **OnLoad** | Время, затраченное на обработку события загрузки | Действия загрузки |
| **OnDOMContentLoaded** | Время, затраченное на выполнение обработчиков `onDomContentLoaded` | Действия загрузки |
| **Callback** | Время, затраченное на выполнение обратных вызовов XHR | Действия загрузки |
| **Redirect time** | Время, затраченное на выполнение HTTP-перенаправлений | Действия загрузки |
| **Long tasks**[1](#fn-2-1-def) | Суммарное время всех длительных задач JavaScript (более 50 мс). Метрика измеряется от **Navigation start** за всю продолжительность пользовательского действия. | Действия загрузки |

1

Метрику **Long tasks** можно использовать вместо метрики [**Total blocking time**﻿](https://web.dev/tbt/), которую было решено не вводить из-за определённых технических ограничений. Также обратите внимание, что метрика **Blocking**, показанная в waterfall, не связана с **Total blocking time** и доступна только для конкретных запросов.

## Ключевые метрики производительности

Из перечисленных выше метрик Dynatrace выделяет некоторые как ключевые. Ключевые метрики производительности позволяют выбрать оптимальную метрику пользовательского опыта для каждого из приложений. Эти ключевые метрики предоставляют ценные сведения о производительности, способствующие успеху цифрового бизнеса. Ключевые метрики производительности включают:

* User action duration
* Visually complete
* Speed index
* DOM interactive
* Load event end
* Load event start

* HTML downloaded / Response end
* Time to first byte
* Largest contentful paint
* Cumulative layout shift[1](#fn-3-1-def)
* First input delay (только для RUM web)

1

**Cumulative layout shift**, это оценка, измеряющая неожиданное смещение видимых элементов веб-страницы. Она доступна для [браузеров на базе Chromium﻿](https://www.chromium.org/Home) и измеряется с помощью [предоставленных Google API﻿](https://web.dev/cls/#measure-cls-in-javascript).

Dynatrace позволяет [выбрать подходящую метрику производительности для каждого условия применения приложения](/managed/observe/digital-experience/rum-classic/web-applications/analyze-and-use/work-with-key-performance-metrics "Learn how to use the right key performance metrics to optimize user experience data for each of your applications."), для каждого условия применения приложения.

## Отсутствующие измерения метрик

Метрические измерения могут отсутствовать в следующих случаях:

#### Visually complete

* Во время загрузочного действия пользователь выбирает другую ссылку, которая запускает новое загрузочное действие до завершения события загрузки исходной страницы. Второе загрузочное действие прерывает вычисление **Visually complete** первого загрузочного действия, поэтому измерение недоступно.
  В более ранней версии **Visually complete** Dynatrace использовал длительность действия пользователя как резервное измерение. Это искажало временной ряд, поэтому такие случаи теперь исключаются для обеспечения более точных агрегатов.
* Сразу после загрузочного действия срабатывает поздний редирект.
* Во время вычисления достигается [тайм-аут Visually complete](/managed/observe/digital-experience/rum-classic/web-applications/analyze-and-use/how-to-use-visually-complete-and-speed-index-metrics#config-enhanced-vc "Learn how to use 'Visually complete' and 'Speed index' metrics.").

#### Core web vitals

* Браузерный API не срабатывает и не предоставляет измерение метрики до завершения действия пользователя (см. метрику **User action duration**).
  В более ранних версиях Dynatrace Real User Monitoring Classic эти значения отправлялись после завершения действия пользователя. Такие значения были неинформативны и часто нерепрезентативны. Например, **First input delay (FID)** измеряет отзывчивость при загрузке страницы. Основной фактор высокого значения FID, это выполнение JavaScript. Поэтому, если пользователь ждал минуту после загрузки страницы перед выбором новой ссылки, большая часть, если не вся, обработки JavaScript уже произошла бы, и итоговое малое измерение FID не было бы репрезентативным значением для реальной загрузки страницы. Такие измерения искажали общий агрегат метрики.
* Браузер пользователя не поддерживает API, которые используются для измерения core web vitals. Подробности см. в разделе [Milestone metrics](#milestone-metrics), где указано, какие браузеры поддерживают Core web vitals.
* Страница загружается или перезагружается в неактивной вкладке, и браузерный API не сообщает значения.
* Браузерный API предоставляет отрицательные значения.
