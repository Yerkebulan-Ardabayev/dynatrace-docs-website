# Приложения в RUM Classic

* Пояснение
* 4 мин на чтение
* Обновлено 18 авг. 2025 г.

В Real User Monitoring Classic (RUM Classic) отслеживаемые приложения, это логические конструкции, на которые сопоставляются пользовательские приложения (веб-сайты, мобильные приложения и другое) для мониторинга трафика от реальных пользователей. Соответственно, подразумевается, что такие пользовательские приложения имеют пользовательский интерфейс. Типичные пользовательские интерфейсы включают браузерные интерфейсы веб-приложений и сайтов, а также интерфейсы на базе iOS или Android мобильных приложений, работающих на смартфонах или планшетах.

Пользовательский интерфейс определяет тип приложения, создаваемого в Dynatrace. Способ, которым Dynatrace получает данные мониторинга, и тип собираемых данных различаются для каждого типа приложения. Например, для веб-приложения, работающего в браузере, [JavaScript Real User Monitoring Classic (RUM Classic)](/managed/observe/digital-experience/rum-classic/web-applications/initial-setup/snippet-formats "Выберите формат сниппета RUM JavaScript, который лучше всего подходит для вашего конкретного случая использования") [автоматически внедряется](/managed/observe/digital-experience/rum-classic/web-applications/initial-setup/rum-injection "Настройте автоматическое внедрение RUM JavaScript на страницы ваших приложений") или [вставляется вручную](/managed/observe/digital-experience/rum-classic/web-applications/initial-setup/set-up-agentless-real-user-monitoring "Настройте безагентный мониторинг для ваших веб-приложений.") в HTML-страницы приложения и начинает мониторинг каждого действия пользователя.

## Определение RUM-приложения

Определение RUM-приложения для веб-приложений генерируется автоматически. Точнее, если веб-приложения работают в системах, где можно установить [OneAgent](/managed/ingest-from/dynatrace-oneagent "Разберитесь в основных концепциях, связанных с OneAgent, и узнайте, как устанавливать и эксплуатировать OneAgent на разных платформах."), Dynatrace автоматически внедряет RUM JavaScript в HTML-страницы. Поэтому каждая [среда мониторинга](/managed/discover-dynatrace/get-started/monitoring-environment "Узнайте, что такое среда мониторинга Dynatrace, как найти идентификатор среды и как настроить и подключить несколько сред.") автоматически получает приложение по умолчанию с названием [«My web application»](/managed/observe/digital-experience/rum-classic/web-applications/initial-setup/define-your-applications-via-the-my-web-application-placeholder "Узнайте, как определить приложения по рекомендуемому, ручному подходу или подходу с правилами обнаружения приложений."). Все данные RUM по умолчанию присваиваются этому приложению.

Для мобильных и пользовательских приложений определение генерируется при создании приложения и настройке мониторинга Dynatrace. Эти типы приложений не отслеживаются автоматизированным способом.

## Поддерживаемые типы приложений

Dynatrace поддерживает различные типы приложений: веб и мобильные, а также rich client-приложения, приложения, работающие в автомобиле, и IoT-приложения с пользовательскими взаимодействиями. Каждый тип имеет свои возможности мониторинга и свой пользовательский интерфейс в Dynatrace. Однако все типы объединены общими концепциями, такими как [пользовательские сессии](/managed/observe/digital-experience/rum-classic/rum-concepts/user-session "Узнайте, как определяется пользовательская сессия, когда она начинается и заканчивается, как рассчитывается длительность пользовательской сессии и многое другое.") и [действия пользователя](/managed/observe/digital-experience/rum-classic/rum-concepts/user-actions "Узнайте, что такое действия пользователя и как они помогают понять, что пользователи делают в вашем приложении.").

| **Тип внедрения** | Внедрение RUM JavaScript может выполняться автоматически с помощью OneAgent или вручную для [безагентного мониторинга](/managed/observe/digital-experience/rum-classic/web-applications/initial-setup/set-up-agentless-real-user-monitoring "Настройте безагентный мониторинг для ваших веб-приложений.") |

Мобильные приложения, это нативные мобильные приложения на iOS или Android, а также гибридные приложения, доступ к которым осуществляется через браузер.

|  |  |
| --- | --- |
| **Пользовательский интерфейс** | Нативное мобильное приложение на iOS или Android |
| **Подход к мониторингу** | OneAgent для iOS или Android; для гибридных приложений браузерная часть отслеживается с помощью RUM JavaScript |
| **Тип внедрения** | Внедрение не требуется, события жизненного цикла, действия пользователя и веб-запросы отслеживаются «из коробки». |
| **С чего начать** | [Инструментируйте ваши Android-приложения](/managed/observe/digital-experience/rum-classic/mobile-applications/instrument-android-app "Узнайте, как инструментировать мониторинг мобильных приложений на Android, как настроить инструментирование и многое другое.")[Инструментируйте ваши iOS-приложения](/managed/observe/digital-experience/rum-classic/mobile-applications/instrument-ios-app "Инструментируйте мониторинг мобильных приложений для iOS-приложений, настройте автоинструментирование и собирайте дополнительные данные с помощью ручного инструментирования.") |

Пользовательские приложения относятся ко всем цифровым точкам взаимодействия в вашей среде: от традиционных rich client-приложений до умных IoT-приложений или даже Alexa Skills. Такие приложения поддерживаются через [Dynatrace OpenKit](/managed/ingest-from/extend-dynatrace/openkit "Узнайте, как инструментировать приложение с помощью OpenKit, как использовать методы API Dynatrace OpenKit и многое другое.").

|  |  |
| --- | --- |
| **Пользовательский интерфейс** | Любая цифровая точка взаимодействия, которую клиент хочет отслеживать: от rich client-приложений через приложения, работающие в автомобиле, до IoT-приложений с пользовательскими взаимодействиями. Пользовательским интерфейсом может быть даже аппаратное обеспечение, например Amazon Alexa. |
| **Подход к мониторингу** | RUM JavaScript Dynatrace |
| **Тип внедрения** | Внедрение выполняется клиентом с помощью [Dynatrace OpenKit](/managed/ingest-from/extend-dynatrace/openkit "Узнайте, как инструментировать приложение с помощью OpenKit, как использовать методы API Dynatrace OpenKit и многое другое.") для его технологии. |
| **С чего начать** | [Инструментируйте ваши приложения с помощью Dynatrace OpenKit](/managed/ingest-from/extend-dynatrace/openkit/instrument-your-application-using-dynatrace-openkit "Узнайте, как использовать Dynatrace OpenKit для инструментирования всех не-веб и не-мобильных цифровых точек взаимодействия в вашей среде, будь то традиционные rich client-приложения, умные IoT-приложения или даже Alexa Skills.") |

Dynatrace фиксирует пользовательские сессии из пользовательских приложений как мобильные пользовательские сессии. Встроенные метрики фиксируются без указания того, мобильное это приложение или пользовательское. Подробнее см. [Встроенные метрики](/managed/analyze-explore-automate/metrics-classic/built-in-metrics "Изучите полный список встроенных метрик Dynatrace.").
