# Инструментирование мобильных приложений с помощью плагина Dynatrace Cordova

* How-to guide
* 5-min read
* Updated on Oct 12, 2022

Гибридные мобильные приложения сочетают преимущества нативных мобильных приложений — доступ к аппаратным ресурсам телефона и распространение через магазин приложений — с гибкостью и платформонезависимостью современных веб-технологий.

Плагин Dynatrace Cordova позволяет инструментировать приложения, построенные на базе Apache Cordova. Это включает приложения с использованием Capacitor, Ionic или IBM Mobile First. Плагин Cordova упрощает мониторинг гибридных приложений с использованием следующих технологий:

* OneAgent для мобильных устройств — выполняет мониторинг нативной части приложения
* RUM JavaScript — отслеживает производительность браузерной части приложения

Подробная техническая документация доступна на странице [Dynatrace Cordova plugin](https://www.npmjs.com/package/@dynatrace/cordova-plugin) на сайте npm.

## Настройка мониторинга гибридного мобильного приложения

1. В Dynatrace перейдите в раздел **Mobile**.
2. Выберите **Create mobile app**.
3. Введите имя приложения и нажмите **Create mobile app**. Откроется страница параметров приложения.
4. В параметрах приложения выберите **Instrumentation wizard** и укажите платформу мобильного приложения, например **Cordova**.

   ![Creating an app in Dynatrace](https://dt-cdn.net/images/setup-wizard-1764-763c621790.png)

   Создание приложения в Dynatrace
5. Выполните команду `cordova plugin add @dynatrace/cordova-plugin --save` в корневом каталоге проекта на базе Cordova для установки плагина Dynatrace Cordova.
6. Выберите **Monitor the web view** для включения мониторинга веб-представления.
7. Выберите **Download dynatrace.config.js** и добавьте загруженный файл в корень рабочего пространства приложения Cordova рядом с файлом `config.xml`.
8. Соберите проект с помощью `cordova build android` или `cordova build ios`.

При сборке Android- или iOS-приложения плагин Cordova автоматически инструментирует приложение с помощью OneAgent для мобильных устройств, который отслеживает нативную часть гибридного приложения. Это обеспечивает видимость запусков приложения, пользовательских действий, отчётов о сбоях и метаданных устройства, таких как модель и версия ОС. Данные мониторинга, собранные OneAgent для мобильных устройств, инкапсулируются в мобильном приложении в Dynatrace.

Для записи пользовательских действий в веб-представлениях Dynatrace использует [RUM JavaScript](/managed/observe/digital-experience/web-applications/initial-setup/snippet-formats "Выберите формат фрагмента RUM JavaScript, наиболее подходящий для вашего конкретного случая использования."), который также применяется для всех веб-приложений, [настроенных для агентлесс-мониторинга Dynatrace](/managed/observe/digital-experience/web-applications/initial-setup/set-up-agentless-real-user-monitoring "Настройте агентлесс-мониторинг для веб-приложений."). Плагин Cordova автоматически внедряет RUM JavaScript непосредственно в HTML-источники. Данные мониторинга, собранные RUM JavaScript, инкапсулируются в веб-приложении в Dynatrace. Это приложение получает все данные мониторинга и генерирует API-токен для извлечения RUM JavaScript плагином Cordova через REST API.

![Cordova app instrumentation](https://dt-cdn.net/images/instrumentcordovaapp-1913-f1c90c1409.png)

## Начало мониторинга и анализа приложения

1. Запустите гибридное приложение в эмуляторе или на тестовом устройстве и воспроизведите сценарии использования для генерации трафика, который Dynatrace сможет отслеживать. По завершении закройте приложение, чтобы все данные были переданы в Dynatrace. Обратите внимание, что OneAgent для мобильных устройств может кэшировать данные до 2 минут.
2. В Dynatrace перейдите в раздел **Mobile**.
3. Выберите новое гибридное приложение из списка.

   На странице обзора гибридного приложения отображаются все данные анализа производительности. В разделе **Hybrid app** показаны общие сведения о производительности веб-части приложения.

   ![Analyze mobile apps](https://dt-cdn.net/images/analyze-mobile-app-1954-6c4199ccdd.png)

   Анализ мобильных приложений
4. Выберите **View web application**, чтобы проанализировать данные производительности, собранные Dynatrace через внедрённый RUM JavaScript.

   На странице обзора веб-приложения содержатся пользовательские действия, записанные в веб-представлении, поэтому здесь отображается большая часть пользовательских действий приложения.

   ![Analyze web apps](https://dt-cdn.net/images/analyze-web-app-1954-d5e81e0ff9.png)

   Анализ веб-приложений

   Данные производительности разделены по двум представлениям приложения. Однако Dynatrace объединяет все обнаруженные пользовательские действия из этих представлений в одного пользователя и одну пользовательскую сессию.
5. Перейдите в раздел **Session Segmentation**, чтобы открыть страницу **User sessions**. В поле **Filtered by** введите **Application** и выберите гибридное мобильное приложение — это позволит сосредоточить анализ на новом приложении.
6. В столбце **User** выберите пользователя по умолчанию, созданного для гибридного приложения. Обратите внимание, что этот пользователь обращался к двум приложениям — мобильному и веб-приложению.

   ![Analyze user](https://dt-cdn.net/images/analyze-user-1950-e23a310611.png)

   Анализ пользователя
7. Выберите пользовательскую сессию. Все пользовательские действия, записанные в мобильной и веб-части гибридного приложения, должны быть объединены в одну сессию вместе с метаданными, позволяющими Dynatrace показать, какие действия были записаны OneAgent для мобильных устройств, а какие — RUM JavaScript.

   На следующем графике показана последовательность пользовательских действий, время их совершения в ходе пользовательской сессии и время ожидания между действиями.

   ![Joined user session](https://dt-cdn.net/images/joined-user-session-1571-0684cc8268.png)

   Объединённая пользовательская сессия

Отсюда можно продолжить анализ пользовательского опыта: добавить дополнительные фильтры для исследования интересующих сессий или детально изучить водопадный анализ пользовательских действий веб- и мобильных сессий приложения.
