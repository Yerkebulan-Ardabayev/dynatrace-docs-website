# Настройка параметров конфиденциальности данных для мобильных приложений

* How-to guide
* 4-min read
* Updated on Apr 16, 2024

Для многих компаний обеспечение конфиденциальности персональных данных клиентов является важным элементом успеха цифрового бизнеса. Dynatrace предоставляет многочисленные средства защиты конфиденциальности, которые вы можете использовать для настройки параметров конфиденциальности данных ваших приложений. При правильной настройке эти параметры помогают защитить персональные данные клиентов и обеспечить соответствие организации требованиям Общего регламента по защите данных (GDPR), требованиям к раскрытию данных в магазинах приложений, а также другим законам и нормативным актам о защите данных.

Хотя Dynatrace предлагает многочисленные параметры конфиденциальности данных, настраиваемые как на уровне [окружения](/managed/manage/data-privacy-and-security/configuration/configure-global-privacy-settings "Узнайте, как настроить маскировку конфиденциальности данных для IP-адресов конечных пользователей, геолокаций и имён пользовательских действий."), так и на уровне приложения, именно вы несёте ответственность за правильную настройку этих параметров и принятие мер по защите персональных данных ваших клиентов.

## Режим согласия пользователя

В режиме согласия каждый пользователь вашего приложения может настроить свои предпочтения конфиденциальности и решить, хочет ли он делиться своей информацией. Пользователи могут выбирать, какими именно типами информации они готовы делиться. Например, один пользователь может разрешить только отправку отчётов о сбоях, тогда как другой может также разрешить захват данных о производительности и пользовательских данных.

### Возможный сценарий режима согласия пользователя

Следующие шаги описывают стандартный рабочий процесс настройки режима согласия пользователя для ваших мобильных приложений.

1. Вы [включаете режим согласия пользователя](#enable-opt-in-mode) с помощью специального флага или ключа и повторно инструментируете приложение.
2. При запуске OneAgent проверяет, настроил ли пользователь свои предпочтения конфиденциальности.

   По умолчанию для всех пользователей, которые ещё не задали свои предпочтения, применяются следующие настройки:

   * Уровень сбора данных: `Off` (данные мониторинга не отправляются в Dynatrace)
   * Отчёты о сбоях: `Off` (отчёты о сбоях не отправляются в Dynatrace)

   Таким образом, при первом запуске приложения никакие данные не передаются в Dynatrace.
3. Если пользователь ещё не настроил свои предпочтения конфиденциальности, вы показываете диалог (см. пример ниже) с запросом разрешения на захват данных о производительности (соответствует уровню сбора данных **Performance**), включение личных данных в информацию, передаваемую в Dynatrace (соответствует уровню **User behavior**), и отправку отчётов о сбоях.

   Не забудьте создать и показать пользователям диалог с настройками конфиденциальности данных. Именно через этот диалог пользователи вашего приложения будут задавать свои предпочтения конфиденциальности.
4. Пользователь задаёт свои предпочтения конфиденциальности, а вы с помощью вызовов API сохраняете их.
5. При следующем запуске приложения OneAgent применяет новые предпочтения конфиденциальности и передаёт только тот объём данных, с которым конкретный пользователь согласился.

### Включение режима согласия

Чтобы активировать режим согласия для мобильных приложений:

1. Перейдите в **Mobile**.
2. Выберите мобильное приложение, которое нужно настроить.
3. Нажмите **More** (**…**) > **Edit** в правом верхнем углу плитки с именем вашего приложения.
4. В настройках приложения выберите **General** > **Data privacy**.
5. Включите **Enable user opt-in mode**.
6. Обновите файл конфигурации вашего приложения (файл сборки для Android, [`Info.plist`](/managed/observe/digital-experience/mobile-applications/instrument-ios-app/instrumentation/info-plist-file "Файл Info.plist содержит идентификаторы и ключи конфигурации вашего приложения. Используйте его для точной настройки конфигурации инструментирования.") для iOS и `dynatrace.config.<extension>` для кроссплатформенных фреймворков), добавив специальный флаг или ключ, включающий режим согласия.

   Проверьте мастер инструментирования для получения обновлённого фрагмента кода конфигурации.
7. Пересоберите приложение, чтобы новая конфигурация вступила в силу.

### Уровни сбора данных

В таблице ниже описаны доступные уровни сбора данных и указано, передаются ли [пользовательские теги](/managed/observe/digital-experience/rum-concepts/user-and-error-events#user-tagging "Узнайте о событиях пользователей и ошибок, а также о типах событий, захватываемых Dynatrace.") и пользовательские действия, события, значения и ошибки для каждого уровня.

| Уровень | Описание | Пользовательские теги, события и значения | Пользовательские действия и ошибки |
| --- | --- | --- | --- |
| **Off**  Данные мониторинга не отправляются | Личные данные не передаются; все идентификаторы случайно изменяются при каждом запуске.[1](#fn-1-1-def) | Не применяется | Не применяется |
| **Performance**  Отправляются только данные о производительности, захватываемые автоматически | Личные данные не передаются; все идентификаторы случайно изменяются при каждом запуске. | Не применяется | Применяется |
| **User behavior**  Отправляются данные о производительности и пользовательские данные | Личные данные передаются; OneAgent распознаёт и фиксирует пользователей при повторных посещениях.[2](#fn-1-2-def) | Применяется | Применяется |

1

Одно событие `Loading <App>` отправляется для отслеживания числа пользователей, отказавшихся от передачи данных.

2

Если вы не настроили пользовательские теги и отчёты о пользовательских событиях или значениях, уровень **User behavior** работает аналогично уровню **Performance**.

## Отслеживание пользователей

В RUM Classic OneAgent для мобильных устройств добавляет к HTTP-запросам заголовок `x-dynatrace`. Dynatrace использует этот заголовок для связи мобильной части веб-запроса с серверной частью, захватываемой другим OneAgent. В New RUM Experience OneAgent для мобильных устройств добавляет к HTTP-запросам заголовки `traceparent` и `tracestate`.

Для гибридных приложений файл cookie `dtAdk` позволяет объединить сессию от OneAgent для мобильных устройств с сессией от RUM JavaScript, чтобы они отображались как единая сессия, тогда как файл cookie `dtAdkSettings` используется для синхронизации настроек между OneAgent для мобильных устройств и RUM JavaScript.

## Маскировка пользовательских действий

Если вы хотите исключить захват личных данных для всех пользовательских действий в мобильном приложении, ознакомьтесь с соответствующими разделами о маскировке пользовательских действий для [Android](/managed/observe/digital-experience/mobile-applications/instrument-android-app/instrumentation-via-plugin/monitoring-capabilities#mask-user-actions "Настройте плагин Dynatrace Android Gradle для управления возможностями мониторинга OneAgent.") или [iOS](/managed/observe/digital-experience/mobile-applications/instrument-ios-app/customization/oneagent-sdk-for-ios#mask-user-actions "Расширьте мониторинг мобильного пользовательского опыта с помощью OneAgent SDK для iOS.").

После включения маскировки пользовательских действий в мобильном приложении OneAgent заменяет все имена действий вида `Touch on <control title>` именем класса или типом элемента управления, к которому прикоснулся пользователь. Например, `Touch on Account 123456` изменяется на `Touch on Button`.
