# Настройка определения IP-адресов для мобильных приложений

# Настройка определения IP-адресов для мобильных приложений

* How-to guide
* 1-min read
* Updated on Mar 21, 2024

Когда веб-запросы и RUM-beacon отправляются напрямую на инструментированный сервер, Dynatrace определяет IP-адреса устройств конечных пользователей через соединения сокетов.

При использовании неинструментированных компонентов, таких как балансировщики нагрузки, CDN или прокси, удалённый IP-адрес отличается от исходного. В таких случаях Dynatrace также анализирует определённые HTTP-заголовки. Эти заголовки чаще всего используются для определения исходного IP-адреса, когда клиент подключается к веб-серверу через HTTP-прокси, CDN или балансировщик нагрузки.

Вы можете просмотреть список заголовков, которые Dynatrace использует для определения IP-адресов клиентов для ваших приложений. Для этого перейдите в **Settings** > **Web and mobile monitoring** > **IP determination**. Dynatrace обрабатывает эти заголовки в определённом порядке, но вы можете изменить порядок обработки и добавить собственные заголовки.

## Управление заголовками по умолчанию

Вы можете обновлять имена заголовков, удалять ненужные и изменять порядок их обработки Dynatrace.

1. Перейдите в **Settings** > **Web and mobile monitoring** > **IP determination**.
2. Управляйте заголовками:

   * Чтобы переименовать заголовок, нажмите **Expand row** ![Expand row](https://dt-cdn.net/images/expand-row-icon-9c4950fc2e.svg "Expand row") и введите новое имя.
   * Чтобы удалить заголовок, нажмите **Delete row** ![Remove](https://dt-cdn.net/images/remove-icon-105c6a04c2.svg "Remove").
   * Чтобы изменить приоритет заголовка, нажмите и удерживайте **Drag row** ![Drag handle](https://dt-cdn.net/images/drag-handle-turquoise-600-1aa0e5ea00.svg "Drag handle") рядом с именем заголовка и переместите его вверх или вниз.

## Добавление пользовательских заголовков IP клиента

Если IP-адреса клиентов хранятся в другом заголовке, создайте пользовательское правило, чтобы Dynatrace мог правильно их определять.

Чтобы добавить пользовательский заголовок IP клиента:

1. Перейдите в **Settings** > **Web and mobile monitoring** > **IP determination**.
2. Нажмите **Create custom client IP header**.
3. Укажите имя вашего заголовка.

Для одного окружения можно создать до 10 пользовательских заголовков IP клиента.

Добавление пользовательского заголовка IP клиента может потребоваться в одном из следующих случаев.

* Когда заголовок, содержащий IP-адреса клиентов, отсутствует в нашем списке заголовков в **Settings** > **Web and mobile monitoring** > **IP determination**.
* Когда IP-адреса клиентов для ваших приложений хранятся в проприетарном заголовке.
* Когда IP-адреса клиентов определяются некорректно, например когда в Dynatrace отображаются IP-адреса балансировщиков нагрузки.

## Причины неправильного определения IP-адресов клиентов

Существует несколько причин, по которым IP-адреса клиентов вашего приложения могут определяться неверно. Ознакомьтесь с разделами выше для получения подробных инструкций.

| Проблема | Возможное решение |
| --- | --- |
| IP-адреса клиентов для ваших приложений расположены в заголовке, которого нет в нашем списке. | Добавьте пользовательский заголовок IP клиента. |
| Указан неверный пользовательский заголовок IP клиента. | Обновите имя заголовка. |
| Указанный пользовательский заголовок IP клиента на самом деле не содержит IP-адресов. | Укажите другой пользовательский заголовок IP клиента. |
| Порядок заголовков неверный. | Измените порядок обработки заголовков. |
| Инфраструктура не настроена на добавление заголовка с IP-адресами клиентов. | Перенастройте инфраструктуру. |
| Заголовок с IP-адресами клиентов не включён в beacon-запросы.[1](#fn-1-1-def) | Перенастройте инфраструктуру. |

1

Заголовки определения IP в обычных запросах приложения не имеют значения для определения IP захваченных RUM-данных, даже если они захватываются как трассировки и коррелируются с пользовательскими действиями.
