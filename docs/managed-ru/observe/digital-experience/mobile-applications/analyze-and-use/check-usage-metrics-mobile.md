# Проверка метрик пользовательского опыта для мобильных приложений

* How-to guide
* 1-min read
* Published Jul 19, 2017

Метрики пользовательского опыта приложения дают быстрый обзор числа пользователей, активно использующих приложение за выбранный период.

Чтобы получить доступ к метрикам пользовательского опыта приложения:

1. Перейдите в **Frontend**.
2. Выберите приложение, которое нужно проанализировать.
3. Изучите плитку **User experience**, которая выбрана по умолчанию.

Плитка **User experience** показывает общее число пользователей, суммарный рейтинг Apdex и частоту пользовательских действий за выбранный период.

![User experience tile on the application overview page](https://dt-cdn.net/images/user-experience-tile-mobile-app-852-0baf6bdc02.png)

User experience tile on the application overview page

Детальная информация о тенденциях рейтинга Apdex, активных и новых пользователях, а также о распределении версий приложения доступна в соответствующих разделах.

## Рейтинг Apdex

График **Apdex rating** показывает тенденции среднего [рейтинга Apdex](/managed/observe/digital-experience/rum-concepts/scores-and-ratings/apdex-ratings "Узнайте, как Dynatrace использует Apdex для измерения удовлетворённости пользователей производительностью приложения.") вашего приложения за выбранный период.

User experience section - Apdex rating

## Новые пользователи

График **New users** показывает, сколько пользователей запустили ваше приложение впервые за выбранный период. Эта метрика учитывает только первую сессию после запуска вашего инструментированного приложения. Обновления приложения и обновления библиотеки Dynatrace не сбрасывают состояние для возвращающихся пользователей. Метрика привязана к конкретным устройствам, поэтому пользователи учитываются несколько раз, если они устанавливают приложение на нескольких устройствах.

![User experience section - New users](https://dt-cdn.net/images/user-experience-new-users-2-848-df286583cc.png)

User experience section - New users

Метрика **New users** не различает нескольких пользователей, использующих одно устройство и одну установку приложения.

## Распределение версий приложения

График **App version distribution** отражает скорость распространения версий, сравнивая количество сессий приложения из различных версий. Этот график особенно интересен после выпуска новых версий приложения, так как позволяет отслеживать распространение последней версии.

### Страница обзора пользовательских действий

Из списка **Top 3 actions** на странице обзора приложения выберите пользовательское действие, чтобы открыть его страницу обзора.

![User action overview page](https://dt-cdn.net/images/user-action-detail-page-2518-a51f30ad70.png)

User action overview page

На этой странице вы можете отметить действие как ключевое, выполнить анализ водопада, а также просмотреть рейтинг Apdex и производительность действия.

Вы также можете изучить списки зафиксированных ошибок и ошибок веб-запросов для текущего действия. Выберите интересующую вас ошибку, чтобы открыть её [страницу с подробными сведениями](#error-details-page).

### Многомерный анализ пользовательских действий

Под списком **Top 3 actions** на странице обзора приложения нажмите **Analyze performance**, чтобы перейти на многомерную страницу **User action analysis**.

Многомерный анализ позволяет детально изучать пользовательские действия и просматривать данные по множеству измерений. На странице **User action analysis** вы можете создавать метрики, просматривать ключевые пользовательские действия и анализировать 100 лучших действий по различным критериям. Вы также можете анализировать приложение по операционной системе, геолокации, версии приложения и другим параметрам.

### Ошибки веб-запросов и зафиксированные ошибки

На многомерной странице **User action analysis** вы также можете просматривать список ошибок веб-запросов или зафиксированных [ошибок](/managed/observe/digital-experience/rum-concepts/user-and-error-events#error "Узнайте о событиях пользователей и ошибок, а также о типах событий, захватываемых Dynatrace."): выберите **Web request errors over time** или **Reported errors over time** в поле **Analyze by**. Список ошибок отобразится под графиком. Разверните интересующую вас ошибку и нажмите **Analyze error**, чтобы просмотреть страницу с подробными сведениями.

![Analyze error from MDA section](https://dt-cdn.net/images/analyze-error-from-mda-2518-90cecae0cc.png)

Analyze error from MDA section

#### Страница с подробными сведениями об ошибке

Страница с подробными сведениями об ошибке предоставляет ценную информацию об ошибках приложения.

Страница с подробными сведениями об ошибке веб-запроса

Страница с подробными сведениями о зафиксированной ошибке

![Web request error details page](https://dt-cdn.net/images/web-request-error-details-page-1772-928bb94d69.png)

Web request error details page

![Reported error details page](https://dt-cdn.net/images/reported-error-details-page-new-1644-60289f33a5.png)

Reported error details page

На ней отображаются сведения об ошибке: расчётное число ошибок, поставщик (для ошибки запроса), технология (для зафиксированной ошибки) и другое. На странице также перечислены затронутые пользовательские сессии и затронутые пользовательские действия — выберите действие или сессию, чтобы просмотреть её подробности. Разбивка по распределению отображает информацию об относительной частоте операционных систем, версий ОС, версий приложения и устройств, а разбивка по странам показывает все затронутые страны и соответствующую частоту ошибок.

Для ошибок веб-запросов вы также можете [настроить правила исключения ошибок](/managed/observe/digital-experience/mobile-applications/additional-configuration/web-request-errors-mobile "Прекратите считать определённые коды HTTP-ответов ошибками для мобильных приложений."), если не хотите, чтобы Dynatrace отмечал конкретный HTTP-код состояния как ошибочный. Нажмите **Edit error exclusion rules**, чтобы перейти на соответствующую страницу настроек.

## Географические регионы

Карта мира **Geographic regions** на странице обзора приложения визуализирует все географические регионы, из которых приходят пользователи. Для каждой страны можно просмотреть количество активных сессий, активных пользователей и сбоев, а также общий рейтинг Apdex.

![User experience section - Geo regions](https://dt-cdn.net/images/user-experience-geo-regions-848-7ae15779b9.png)

User experience section - Geo regions

Если данные не отображаются на карте мира, возможно, вам нужно [сопоставить внутренние IP-адреса с местоположениями](/managed/observe/digital-experience/mobile-applications/additional-configuration/map-internal-ip-addresses-to-locations-mobile "Настройте Dynatrace для использования локальных адресов, чтобы понять местонахождение пользователей ваших мобильных приложений.").
