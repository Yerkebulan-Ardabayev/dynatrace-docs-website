# Анализ пользовательских сессий мобильных приложений

* How-to guide
* 1-min read
* Published Jul 19, 2017

В современной цифровой среде наличие стратегии mobile-first в маркетинге бизнеса является обязательным условием. Независимо от того, обращаются ли пользователи к вашим предложениям через мобильный браузер или используют нативное или гибридное мобильное приложение, успех бизнеса зависит от высокопроизводительного пользовательского опыта. Именно поэтому Dynatrace придерживается стратегии мониторинга mobile-first, предлагая широкий спектр технологических инсайтов по мобильному пользовательскому опыту.

По умолчанию анализ пользовательских сессий отображает все сессии в вашем окружении: для веб-, мобильных и пользовательских приложений. Чтобы сосредоточить анализ только на мобильных приложениях, необходимо добавить один фильтр.

Чтобы просмотреть пользовательские сессии, захваченные в мобильных приложениях:

1. Перейдите в **Session Segmentation**. Откроется страница **User sessions**.
2. В поле **Filter by** выберите **Application type:** `Mobile`, чтобы отображать только мобильные пользовательские сессии.
3. Необязательно: для более детального анализа добавьте дополнительные фильтры, такие как версия операционной системы, производитель, количество ошибок и оценка пользовательского опыта.
4. Выберите временную метку нужной пользовательской сессии, чтобы перейти на страницу с её подробностями.

Dynatrace захватывает пользовательские сессии пользовательских приложений как мобильные. Встроенные метрики захватываются без указания того, является ли приложение мобильным или пользовательским. Подробнее см. в разделе [Встроенные метрики](/managed/analyze-explore-automate/metrics-classic/built-in-metrics "Ознакомьтесь с полным списком встроенных метрик Dynatrace.").

Дополнительные сведения см. в разделе [Новый: анализ пользовательских сессий](/managed/observe/digital-experience/session-segmentation/new-user-sessions "Узнайте о сегментации пользовательских сессий и атрибутах фильтрации.").
