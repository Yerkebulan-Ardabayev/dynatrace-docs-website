# Отладочное журналирование OneAgent for iOS

* How-to guide
* 1-min read
* Updated on Oct 24, 2023

Для включения отладочного журналирования для iOS добавьте [конфигурационный ключ `DTXLogLevel`](/managed/observe/digital-experience/mobile-applications/instrument-ios-app/customization/ios-configuration-keys#general "С помощью конфигурационных ключей можно точно настроить авто-инструментирование iOS-приложений.") в файл [`Info.plist`](/managed/observe/digital-experience/mobile-applications/instrument-ios-app/instrumentation/info-plist-file "Файл Info.plist хранит ключи идентификации и конфигурации приложения. Используйте его для точной настройки конфигурации инструментирования.") вашего приложения:

```
<key>DTXLogLevel</key>



<string>ALL</string>
```

Доступные уровни журналирования перечислены ниже в порядке возрастания детализации. Каждый уровень включает информацию всех предыдущих уровней.

| Уровень журнала | Захватываемая информация |
| --- | --- |
| `OFF` | Только версия OneAgent |
| `SEVERE` | События, влияющие на функциональность OneAgent и способные отключить мониторинг |
| `WARNING` | События, которые могут привести к проблемам |
| `INFO` | Дополнительные информационные сообщения |
| `ALL` | Вся отладочная информация |

При запуске приложения журналы должны отображаться в консоли в области отладки Xcode. Обратите внимание, что уровень журналирования предназначен для устранения неполадок и не влияет на данные, захватываемые OneAgent.

Уровень журналирования по умолчанию — `OFF`, так как интенсивное журналирование может снизить производительность приложения.

Кроме того, убедитесь, что переменная окружения `OS_ACTIVITY_MODE` не установлена в значение `disabled`, иначе отладочное журналирование будет отключено.

В консоли Xcode 15+ результаты журналирования больше не отображают временные метки. При копировании журналов, например для их отправки специалисту по продукту Dynatrace, выберите все строки журнала в консоли и нажмите **Copy Rows with All Metadata**.
