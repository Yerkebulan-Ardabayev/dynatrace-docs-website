# Анализ исключений

* Explanation
* 4-min read
* Updated on Sep 13, 2022

Представление **Exceptions overview** — диагностическая функция, позволяющая видеть, какие исключения возникли в вашем окружении за выбранный период и в пределах выбранной зоны управления.

Чтобы перейти на страницу анализа **Exceptions overview**

1. Перейдите в **Multidimensional Analysis**.
2. Выберите плитку **Exception analysis**.

![Страница анализа исключений](https://dt-cdn.net/images/exception-analysis-3562-8213d4c4bc.png)

Страница анализа исключений

## Настройка представления

На странице **Exceptions overview** перечислены все исключения, возникшие за выбранный период в пределах выбранной зоны управления. В разделе **Configure view** можно настроить несколько возможностей фильтрации. Представление обновляется автоматически при изменении параметров.

| Параметр | Описание |
| --- | --- |
| Metric | Метрика для анализа. По умолчанию установлено **Exception count**. |
| Aggregation | Способ агрегации значений метрики. Доступные агрегации зависят от выбранной метрики. |
| Split mode | Способ обработки измерений, указанных в **Split by dimension**.  * **Split by services** — каждое измерение отображается отдельно для каждого сервиса. * **Merge by services** — одинаковые измерения из разных сервисов объединяются в одно. |
| Split by dimension | Список измерений, по которым разбиваются запросы. По умолчанию установлено измерение класса исключения (`{Exception:Class}`).  Можно указать несколько измерений. Поместите курсор в поле ввода, чтобы увидеть доступные параметры. Запросы разбиваются по измерениям в указанном порядке. |
| Filter requests | Фильтрация запросов для включения в представление. По умолчанию установлен следующий критерий:  * **Exception**: `Any exception`  Можно добавить дополнительные критерии. Поместите курсор в поле ввода, чтобы увидеть доступные параметры.  Критерии одного типа объединяются по логике OR. Критерии разных типов объединяются по логике AND. |

После настройки представление можно сохранить для быстрого доступа в будущем. Просто выберите **Save view** и укажите имя.

Также можно сохранить конфигурацию как вычисляемую метрику сервиса и использовать её как любую другую метрику в Dynatrace (например, для [оповещений](/managed/dynatrace-intelligence/anomaly-detection/metric-events "Узнайте о событиях метрик в Dynatrace")).

Данные таблицы можно экспортировать в файл CSV.

1. В правом нижнем углу страницы выберите **Show export menu** ![More](https://dt-cdn.net/images/more-icon-01c8b008ca.svg "More").

   ![Меню Show export](https://dt-cdn.net/images/show-export-menu-107-2a8a76c9a2.png)

   Меню Show export
2. Выберите **Export visible data** или **Export table data**.

   | Параметр | Экспортируемые данные | Поля | Количество записей |
   | --- | --- | --- | --- |
   | **Export visible data** | Текущая видимая область таблицы с учётом применённых фильтров | Только видимые данные | До 100 топовых измерений |
   | **Export table data** | Все данные таблицы | Все доступные данные, связанные с топовыми измерениями | До 100 топовых измерений |

## Представление

* На диаграмме показаны 15 топовых измерений (все остальные измерения агрегируются в одно), а таблица ниже содержит до 85 дополнительных измерений, что в сумме составляет до 100 отображаемых на диаграмме измерений. Представление мгновенно адаптируется к изменениям в панели **Configure view**.

  В столбце **Actions** таблицы можно выбрать:

  + **Filter** ![Filter](https://dt-cdn.net/images/filter-icon-41ddd02d66.svg "Filter") — для фильтрации представления по указанному измерению.
  + **More** (**…**) — для доступа к дополнительным параметрам анализа из [меню **Analyze**](/managed/observe/application-observability/services-classic/context-specific-drill-down#analyze-menu "Узнайте об удобной навигации и фильтрации для анализа сервисов.").
* Диаграмма использует [данные трассировок и запросов](/managed/observe/application-observability/multidimensional-analysis#data-source "Настройте представление многомерного анализа и сохраните его как вычисляемую метрику.") с разными периодами хранения. Для временных периодов, содержащих данные старше 10 дней, можно включить переключатель **Show data retention**, чтобы лучше понять, какие данные доступны за какой период непосредственно на диаграмме.

## Детали исключения

Страница **Exception details** помогает понять, как исключение влияет на ваше цифровое окружение. Чтобы перейти на неё, выберите **Exception details** из меню действий для нужного исключения.

![Детали исключения](https://dt-cdn.net/images/exception-details-1066-b96d1325ad.png)

Детали исключения

Страница предоставляет обзор сообщений об исключениях; трассировку стека на уровне кода, где можно углубиться в код вплоть до метода, выбросившего исключение; а также запросы, затронутые исключением. Выберите сообщение об исключении, чтобы отфильтровать трассировку стека и затронутые запросы по выбранному сообщению.

Можно дополнительно сузить область анализа, применив фильтры. Поместите курсор в поле **Filtered by**, чтобы увидеть доступные параметры. Быстрый способ фильтрации по конкретному запросу — открыть вкладку **Affected request** и выбрать кнопку фильтрации ![Filter](https://dt-cdn.net/images/filter-icon-41ddd02d66.svg "Filter") нужного запроса.

## Примеры использования

Ниже приведены некоторые сценарии использования представления **Exception analysis**.

### Поиск конкретного исключения

С помощью представления **Exception analysis** можно сосредоточить анализ на конкретном классе исключения. Просто найдите нужное исключение в таблице и выберите кнопку фильтра ![Filter](https://dt-cdn.net/images/filter-icon-41ddd02d66.svg "Filter").

Если вас интересует редкое исключение, которого нет в таблице, его всё равно можно найти по классу исключения.

Необходимо указать точный класс исключения, так как фильтр использует оператор `EQUALS`.

1. Откройте страницу анализа **Exceptions overview** и при необходимости установите дополнительные фильтры.
2. В поле **Filter requests** удалите критерий **Exception**: `Any exception`.
3. Снова добавьте тип критерия **Exception** и вставьте класс исключения в качестве значения.  
   Представление отобразит данные только по выбранному исключению.

### Поиск затронутых сервисов

Данные об исключении можно просматривать как агрегированное значение по всем сервисам или отдельно для каждого сервиса. Этот параметр позволяет легко определить, какие сервисы затронуты определённым исключением.

1. Отфильтруйте представление по нужному исключению, как описано выше.
2. В списке **Split mode** выберите **Split by services**.

Представление отображает данные по выбранному исключению с разбивкой по сервисам. Отсюда можно углубиться в **Exception** details или к дальнейшим [параметрам анализа](/managed/observe/application-observability/services-classic "Узнайте о классическом мониторинге сервисов в Dynatrace.") интересующего сервиса.
