# Временные показатели анализа сервисов

# Временные показатели анализа сервисов

* Reference
* 5-min read
* Updated on Jul 24, 2023

Анализ сервисов оперирует множеством различных временных показателей, описывающих поведение сервиса. В таблице ниже приведён обзор этих показателей. Временные показатели различаются в зависимости от типа анализа:

* DT: [Распределённые трассировки](/managed/observe/application-observability/distributed-traces/use-cases/segment-request#pp-code-level-tab "Улучшайте производительность распределённой системы, сегментируя запросы с медленным временем отклика через поток сервисов и анализируя их распределённые трассировки.").
* RT: [Время отклика](/managed/observe/application-observability/services-classic/service-response-time-hotspots "Определяйте действия, потребляющие наибольшую часть времени отклика для каждого сервиса.").
* SF: [Поток сервисов](/managed/observe/application-observability/services-classic/service-flow "Узнайте, как Dynatrace помогает отслеживать последовательность вызовов сервисов, инициируемых каждым запросом в вашем окружении.").
* MH: [Горячие точки методов](/managed/observe/application-observability/services-classic/service-response-time-hotspots "Определяйте действия, потребляющие наибольшую часть времени отклика для каждого сервиса.").

Независимо от того, какой анализ выполняется, не гарантируется отображение всех перечисленных здесь временных показателей. Фактически отображаемые показатели полностью зависят от того, как запущены ваши сервисы. Например, если распределённая трассировка выполняется целиком на одном хосте без какого-либо сетевого взаимодействия, сетевые показатели отображаться не будут.

| Показатель | Отображается в | Описание |
| --- | --- | --- |
| Response time | DT SF | Общее время выполнения первого узла распределённой трассировки.  * Server side Время между моментом запуска распределённой трассировки на стороне сервера и моментом отправки ответа клиенту. * Client side Время между моментом, когда клиент инициирует запрос, и моментом получения ответа. |
| Response time | RT | Общее время выполнения метода.  * Server side Время обработки метода. * Client side Время между моментом, когда клиент инициирует запрос, и моментом получения ответа. |
| Processing time | DT RT | Длительность распределённой трассировки от начала до конца.  Это *wall-clock time* (разница между временем начала и конца), а не сумма всех асинхронных выполнений. |
| Execution time | RT | Общее время выполнения кода.  Это **сумма всех асинхронных выполнений**, поэтому она может быть длиннее, чем processing time. |
| Suspension | DT RT | Время, в течение которого выполнение любого кода приостановлено. Обычно вызывается сборкой мусора. |
| Wait time | DT | Время, в течение которого код активно ожидает чего-либо (например, `Object.wait()` или аналогичной функциональности). |
| Lock wait | RT | Время, в течение которого код заблокирован. Обычно вызывается временем ожидания перед входом в синхронизированный блок кода или временем ожидания получения спинлока. |
| Active wait | RT | Время, в течение которого код *данного узла* ожидает чего-либо. Ожидание, вызванное дочерними вызовами, не включается. |
| Lock time | DT | Время, в течение которого код заблокирован ожиданием синхронного блока кода. |
| Network I/O | DT RT | Время, в течение которого код активно ожидает *нативных* сетевых функций (например, `java.net.SocketInputStream.socketRead0`).  При анализе времени отклика ожидание, вызванное дочерними вызовами, не включается. |
| Disk I/O | DT RT | Время, в течение которого код активно читает с диска или пишет на диск, либо ожидает ввода-вывода диска.  При анализе времени отклика ожидание, вызванное дочерними вызовами, не включается. |
| Total I/O | - | В узле — **сумма Network I/O и Disk I/O**.  Если эти метрики недоступны и узел является непосредственным родителем синхронно вызываемого дочернего сервиса, не являющегося пользовательским, Total I/O узла можно оценить как **Duration узла минус CPU Self time узла**. |
| CPU time | DT RT | Время, в течение которого CPU выполняет код, связанный с распределённой трассировкой. Измерение предоставляется OneAgent. |
| Self time | DT RT | Время обработки конкретного узла в распределённой трассировке. |
| Other | DT | Любая неклассифицированная доля self time. |
| Elapsed time | DT | Время между выполнениями — от момента создания распределённой трассировки до момента входа в метод. |
| Duration | DT | Временной интервал, представляющий длительность узла, включая его синхронные дочерние узлы, от момента начала до момента конца. |
| Code execution | MH | Процент измеренных выборок, в которых код метода активно выполняется. |
| Disk I/O | MH | Процент измеренных выборок, в которых метод активно читает с диска или пишет на диск либо ожидает ввода-вывода диска. |
| Network I/O | MH | Процент измеренных выборок, в которых метод активно ожидает *нативных* сетевых функций. |
| Waiting | MH | Процент измеренных выборок, в которых метод активно ожидает чего-либо (например, `Object.wait()` или аналогичной функциональности). |
| Locking | MH | Процент измеренных выборок, в которых метод заблокирован ожиданием синхронного блока кода. |
