# Горячие точки времени отклика сервиса

* How-to guide
* 5-min read
* Published Jul 19, 2017

При включённом глубоком мониторинге процессов Dynatrace анализирует время отклика каждого сервиса, работающего в каждом процессе. Это применимо к Java, .NET, Node.js, PHP, Apache webserver, IIS, NGINX и другим технологиям.

Горячие точки времени отклика указывают на то, какие операции потребляют больше всего времени для конкретного сервиса. На каждой странице **Service** отображается обзор связанных горячих точек времени отклика в разделе **Current hotspot**, включающий:

* Топ-выводы, такие как запросы с медленным временем отклика, высокой частотой отказов или высоким потреблением CPU.
* Запросы с высоким потреблением ресурсов запроса. Процент показывает долю запроса в общем времени отклика сервиса. В примере ниже `85%` времени отклика `JourneyService` тратится на выполнение запроса `findJourneys`.

![Горячие точки 0](https://dt-cdn.net/images/hotspots0-1309-9b8dfdbea1.png)

Горячие точки 0

Чтобы просмотреть анализ горячих точек

1. Перейдите в **Services**.
2. Выберите сервис для анализа.
3. Выполните одно из следующих действий:

   * Для анализа топ-вывода горячей точки выберите вывод в разделе **Current hotspots** для отображения деталей сервиса, отфильтрованных по проблемному запросу.
   * Для анализа всех горячих точек времени отклика сервиса на странице обзора сервиса выберите **View requests**, **View dynamic requests** или **View resource requests**. Dynatrace отобразит общие сведения о сервисе.
4. Выберите **View response time hotspots** для перехода на страницу **Response time analysis**.

![Горячие точки 1](https://dt-cdn.net/images/hotspots1-1309-752c2e2bd8.png)

Горячие точки 1

На странице **Response time analysis** отображается среднее время отклика за анализируемый период. С левой стороны инфографики в разделе **Distribution** видно, какая доля времени приходится на вызовы других сервисов, вызовы баз данных и выполнение кода. С правой стороны в разделе **Top findings** перечислены наиболее значимые горячие точки, определённые Dynatrace. Можно выбрать любую из этих записей для просмотра дополнительных сведений.

## Вызовы других сервисов

Выберите **Interaction with services and queues** для отображения сведений о том, как эти вызовы вносят вклад в общее время отклика.

![Горячие точки 2](https://dt-cdn.net/images/hotspots2-1889-e6dec0cd21.png)

Горячие точки 2

С левой стороны видно, как часто анализируемый сервис вызывает другие сервисы и в какой мере эти вызовы влияют на время отклика. С правой стороны эта информация представлена более детально. В приведённом выше примере видно, что наибольший вклад в время отклика вносит сервис `Easytravel Customer Frontend`. Выберите имя сервиса для просмотра следующего уровня иерархии. Достигнув нижнего уровня (уровня запроса), вы увидите полные сведения о времени отклика для выбранного запроса.

![Горячие точки 3](https://dt-cdn.net/images/hotspots3-1887-d7ea5fdd55.png)

Горячие точки 3

В приведённом выше примере видно, что `EasytravelWebserver:8079` вызывает URL `/orange.jsf` на `easytravel Customer Frontend` только в 7,46% от всех инициируемых им запросов — ровно по одному разу. Среднее время отклика `/orange.jsf` составляет `255 мс`. Однако поскольку этот вызов происходит редко, он вносит лишь `19 мс` в среднее время отклика `EasytravelWebserver:8079`.

При нажатии кнопки со стрелкой на левой границе раздела **Interaction with services and queues** можно снова переместиться вверх по иерархии.

## Запросы к базам данных

Выберите раздел **Database usage** инфографики для доступа к сведениям о запросах к базам данных анализируемого сервиса и просмотра их вклада в время отклика. Верхний раздел предоставляет ту же навигацию и информацию, что и раздел **Interaction with services and queues**. Ниже можно просмотреть все запросы к базам данных.

![Горячие точки 4](https://dt-cdn.net/images/hotspots4-1252-be6d5ddef6.png)

Горячие точки 4

Выбрав раскрывающийся список в правом верхнем углу, можно сортировать операторы базы данных по **average execution time**, **percentage of calls**, **invocations** и **contribution**. По умолчанию в начале списка находятся операторы, вносящие наибольший вклад в время отклика. Выберите оператор SQL для просмотра значений этих четырёх атрибутов.

![Горячие точки 5](https://dt-cdn.net/images/hotspots5-1228-0f1d8e37b5.png)

## Видимость на уровне кода

Выберите **Service execution** для просмотра времени, затраченного сервисом на выполнение собственного кода.

![Время выполнения сервиса](https://dt-cdn.net/images/service-execution-time-1445-780d85d724.png)

Время выполнения сервиса

Длина полос представляет время, затраченное в каждой конкретной области. Тёмно-синий цвет соответствует времени CPU. Выберите любое имя строки, отображаемое как ссылка, для просмотра горячих точек методов соответствующего уровня.

![Видимость на уровне кода](https://dt-cdn.net/images/hotspots-1446-6c931f44ae.png)

Видимость на уровне кода

Нажмите кнопку **View method hotspots** и проверьте все классы и методы, выполненные для работы сервиса, в разделе **Call tree**. В столбце **Stacktrace samples** показано, сколько раз данный класс/метод был выполнен для сервиса в течение выбранного периода. В столбце **Contribution** показана доля потребления, которую класс/метод вносит в общее выполнение. Это позволяет определить, какой класс и/или метод потребляет большую часть времени выполнения, и оптимизировать код.

### Загрузка кода

Предварительные условия

Убедитесь, что у вас есть [разрешение](/managed/manage/identity-access-management/permission-management/role-based-permissions "Разрешения на основе ролей") **View sensitive request data** в вашем окружении.

Ограничения

Загрузка кода поддерживается для Java, .NET и Node.js.

Чтобы загрузить код для выполнения

1. Перейдите в **Actions** и выберите  > **Source code**.
2. Выберите процесс, из которого хотите загрузить код.
3. Выберите **Download** для интересующего вас выполнения.
4. Откройте исходный код.

   Node.js

   Java

   .NET

   Исходный код отображается непосредственно в Dynatrace.

   Для преобразования кода в исходный код используйте внешний декомпилятор, например [Bytecode Viewer](https://bytecodeviewer.com).

   Выберите декомпилятор, наиболее подходящий для конкретной версии Java, с которой вы работаете.

   Для преобразования кода в исходный код используйте внешний декомпилятор, например [dotPeak](https://www.jetbrains.com/decompiler/).

   Выберите декомпилятор, наиболее подходящий для конкретной версии .NET, с которой вы работаете.

## Автоматические горячие точки

В разделе **Top findings** инфографики анализа **Response time** перечислены горячие точки, являющиеся основными факторами для анализируемого запроса.

Если запрос медленный, основная причина будет указана в разделе **Top findings**. Выберите вывод для просмотра дополнительных сведений. В данном примере видно, что из 639 мс в вызове `PaymentLogic.DoPay` было потрачено 226 мс!

![Горячие точки 7](https://dt-cdn.net/images/hotspots7-1254-e581fe35b2.png)

Горячие точки 7

## Распределение времени отклика

Выберите круговой раздел **Response time** инфографики для открытия диаграммы, показывающей распределение времени отклика по всем анализируемым запросам. Это позволяет понять, имеет ли ваш сервис широкий диапазон времени отклика или работает равномерно.

![Горячие точки 8](https://dt-cdn.net/images/hotspots8-1248-286099cfc1.png)

Горячие точки 8

Как правило, сервисы веб-запросов имеют широкий диапазон времени отклика. Анализ отдельного запроса зачастую показывает более равномерную картину.
