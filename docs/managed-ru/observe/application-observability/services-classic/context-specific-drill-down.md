# Контекстно-зависимая детализация

# Контекстно-зависимая детализация

* Reference
* 4-min read
* Published Nov 13, 2018

Меню контекстно-зависимой детализации упрощают навигацию и фильтрацию результатов, представленных в представлениях анализа сервисов Dynatrace. Можно быстро просматривать огромные объёмы данных всего за несколько кликов.

## Табличные макеты

Табличные макеты содержат следующий набор кнопок в правой части каждой строки таблицы: ![Chart](https://dt-cdn.net/images/blue-chart-icon-4c5e910a12.svg "Chart") ![Filter](https://dt-cdn.net/images/filter-icon-41ddd02d66.svg "Filter") ![More](https://dt-cdn.net/images/more-icon-01c8b008ca.svg "More")

### Контекстно-зависимые фильтры

![Filter](https://dt-cdn.net/images/filter-icon-41ddd02d66.svg "Filter") Контекстно-зависимые фильтры доступны практически на всех страницах анализа сервисов с табличными макетами.

Нажатие кнопки фильтра для конкретного экземпляра сервиса в таблице экземпляров фильтрует всю страницу, показывая только результаты этого конкретного экземпляра. Рассмотрим пример страницы **Details** сервиса ниже. Выбор кнопки фильтра для конкретного **экземпляра сервиса** в таблице экземпляров отфильтрует всю страницу, показывая только результаты этого конкретного экземпляра.

![Контекстно-зависимый фильтр](https://dt-cdn.net/images/context-specific-filter-2994-197bdffd08.png)

Контекстно-зависимый фильтр

Аналогично, выбор кнопки фильтра для конкретного атрибута запроса на вкладке **Request attributes** фильтрует всю страницу, показывая только результаты, связанные с выбранным атрибутом запроса.

### Создание контекстно-зависимых диаграмм

![Chart](https://dt-cdn.net/images/blue-chart-icon-4c5e910a12.svg "Chart") Создание контекстно-зависимых диаграмм доступно практически на всех страницах анализа сервисов с табличными макетами.

Выберите кнопку диаграммы для любой строки, чтобы создать диаграмму на основе содержимого выбранной строки. Это упрощает создание диаграмм на основе пользовательских метрик или атрибутов запросов.

![Контекстно-зависимые диаграммы](https://dt-cdn.net/images/context-specific-charts-2994-7b2cccd651.png)

Контекстно-зависимые диаграммы

Следующая диаграмма представляет время отклика экземпляра сервиса `Custom Frontend`. То же самое можно сделать на вкладке **CPU** для отображения суммарного времени CPU экземпляра.

![Время отклика в контекстно-зависимых диаграммах](https://dt-cdn.net/images/context-specific-charts-3012-2c8eaa171b.png)

Время отклика в контекстно-зависимых диаграммах

### Контекстно-зависимые меню детализации

![More](https://dt-cdn.net/images/more-icon-01c8b008ca.svg "More") Меню **Analyze**, доступные для результатов на всех страницах анализа сервисов с табличными макетами, открывают все возможные варианты детализации и анализа на основе текущего контекста.

Посмотрите на развёрнутое меню **Analyze** ниже. В данном примере можно видеть параметры для контекстно-зависимого анализа на основе **Method hotspots**, **Outliers**, **Distributed traces**, **Response time hotspots**, **Service backtrace**, **Comparison**, **Service flow** и **Top web requests**.

![Меню Analyze](https://dt-cdn.net/images/analyze-menu-3026-03d65e04ae.png)

Меню Analyze

В приведённом примере выбран сервис `Customer Frontend` в разделе **Incoming requests to this service** и запрос `orange-booking-payment.jsf` в разделе **Requests from**. Такая комбинация детализации фокусирует анализ на всех запросах `orange-booking-payment.jsf` сервиса `Customer Frontend`, вызывающих базу данных `easyTravel-Business` через другие сервисы дерева. Кроме того, фильтры, применённые через **Filter requests**, передаются в любой параметр анализа, выбранный из меню **Analyze**.

## Многомерный анализ

Контекстно-зависимое меню для [многомерного анализа](/managed/observe/application-observability/multidimensional-analysis "Настройте представление многомерного анализа и сохраните его как вычисляемую метрику.") позволяет выполнять контекстно-зависимый анализ на основе отдельных метрик временных рядов, включённых в диаграммы. Это меню включает все функции фильтрации, доступные на страницах анализа сервисов, вместе с расширенными возможностями для построения диаграмм и разбивки по нескольким измерениям.

## Поток сервисов

Хотя **Service flow** не включает таблицы, контекстно-зависимые меню детализации доступны и в этом представлении: ![Distributed traces](https://dt-cdn.net/images/purepaths-icon-790bea38ba.svg "Distributed traces") ![Filter](https://dt-cdn.net/images/filter-icon-41ddd02d66.svg "Filter") ![More](https://dt-cdn.net/images/more-icon-01c8b008ca.svg "More") или ![Database statements](https://dt-cdn.net/images/database-statements-icon-62baf10731.svg "Database statements") ![Filter](https://dt-cdn.net/images/filter-icon-41ddd02d66.svg "Filter") ![More](https://dt-cdn.net/images/more-icon-01c8b008ca.svg "More")

Выберите любой сервис в потоке сервисов, и появится всплывающее меню детализации.

![Контекстно-зависимая детализация в потоке сервисов](https://dt-cdn.net/images/service-flow-context-specific-options-3036-4afddf7a92.png)

Контекстно-зависимая детализация в потоке сервисов

### Контекстно-зависимые распределённые трассировки PurePath®

![Distributed traces](https://dt-cdn.net/images/purepaths-icon-790bea38ba.svg "Distributed traces") Контекстно-зависимые распределённые трассировки доступны для всех небазовых сервисов.

Выберите **Distributed traces**, чтобы просмотреть страницу **Distributed traces**, отфильтрованную для отображения только тех распределённых трассировок, которые инициируют отображаемый поток сервисов (`easyTravel Customer Frontend` в примере ниже) и впоследствии вызывают выбранный сервис (`Credit Card Verification` в примере ниже).

![Распределённые трассировки из потока сервисов](https://dt-cdn.net/images/distributed-traces-2976-2399031360.png)

Распределённые трассировки из потока сервисов

### Контекстно-зависимые операторы БД

![Database statements](https://dt-cdn.net/images/database-statements-icon-62baf10731.svg "Database statements") Контекстно-зависимые операторы БД доступны для большинства сервисов баз данных.

Выберите **Top database statements**, чтобы просмотреть страницу **Top database statements**, отфильтрованную для отображения только тех операторов в базе данных (``` DB1`` на рисунке ниже), которые выполняются в результате вызовов, исходящих из сервиса, инициировавшего поток сервисов (```easyTravel Customer Frontend` на рисунке ниже).

![Доступ к топ операторов БД из потока сервисов](https://dt-cdn.net/images/top-database-statatement-button-3088-1104748825.png)

Доступ к топ операторов БД из потока сервисов

### Контекстно-зависимые фильтры

![Filter](https://dt-cdn.net/images/filter-icon-41ddd02d66.svg "Filter") Контекстно-зависимые фильтры доступны для всех сервисов.

Они фильтруют поток сервисов по вызовам, исходящим из сервиса, инициировавшего поток сервисов, и впоследствии вызывающим выбранный сервис — так же, как кнопка **Filter service flow** в верхней плитке [боковой панели](/managed/observe/application-observability/services-classic/service-flow/service-flow-metrics#side-pane "Узнайте о метриках потока сервисов, измеряющих производительность вызовов сервисов, инициируемых каждым запросом в вашем окружении.").

### Контекстно-зависимые меню детализации

![More](https://dt-cdn.net/images/more-icon-01c8b008ca.svg "More") Меню **Analyze**, доступные для всех сервисов, открывают все возможные варианты детализации и анализа на основе текущего выбранного сервиса.
