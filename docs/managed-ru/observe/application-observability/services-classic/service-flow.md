# Поток сервисов

* How-to guide
* 2-min read
* Published Jul 19, 2017

Dynatrace понимает транзакции ваших приложений от начала до конца. Это транзакционное понимание визуализируется через **Service flow** — представление, иллюстрирующее последовательность вызовов сервисов, инициируемых каждым запросом к сервису в вашем окружении. В потоке сервисов виден поток вызовов сервисов с точки зрения одного сервиса, запроса или их отфильтрованного подмножества. Помимо конкретных инициируемых сервисов, можно видеть, как каждый компонент запроса вносит вклад в общее время отклика.

Чтобы просмотреть поток сервисов, инициируемый конкретным сервисом

1. Перейдите в **Services**.
2. Выберите сервис для анализа.
3. На странице обзора сервиса в разделе **Understand dependencies** выберите **View service flow**.

На изображении ниже показан поток сервисов, инициируемый сервисом `EasyTravelWebServer`, а также вклад каждого инициируемого сервиса в общее время отклика. Сервис `EasyTravelWebServer` вызывает сервис Tomcat. Сервис Tomcat, в свою очередь, вызывает два других сервиса, которые вызывают ещё два сервиса. Сразу очевидно, что наибольший вклад в общее время отклика вносит сервис `Credit Card Verification`, хотя его вызывающий сервис `BookingService` вызывается только `27%` запросов. Такой уровень анализа позволяет понять более сложные взаимосвязи внутри системы.

**Service flow** предоставляет аналитику последовательности цепочек вызовов сервисов. Чтобы понять порядок выполнения вызовов сервисов относительно друг друга, используйте вместо этого **Service backtrace**.

Потоки сервисов могут становиться весьма сложными. Для улучшения читаемости данных потока сервисов Dynatrace агрегирует сервисы, которые вносят незначительный вклад в общее время отклика. Агрегированные сервисы имеют имена вида `2 services` и `4 instances`. Обратите внимание на агрегат `4 services` в среднем столбце ниже. Выберите его, чтобы увидеть, какие сервисы здесь агрегированы. При выборе одного из агрегированных сервисов **Service flow** отображает путь запросов выбранного сервиса.

Агрегации рассчитываются динамически в зависимости от размера окна браузера. Можно использовать функцию масштабирования браузера для освобождения дополнительного пространства.

Как видно в приведённом выше примере, некоторые сервисы в потоке сервисов встречаются несколько раз (`easyTravel Customer Frontend` и `EasyTravelWebserver`). Это происходит потому, что данные сервисы вызывались в нескольких точках в ходе выполнения этой транзакции. Каждый вызов выполняет разные действия и вносит разный вклад в общее время отклика. Dynatrace не агрегирует эти различные обстоятельства в одну метрику, поскольку каждое из них само по себе значимо и даёт ценное представление о том, что происходит на глубоком уровне в вашем окружении.
