# Анализ сервисов

* How-to guide
* 6-min read
* Updated on Jun 13, 2025

Этот раздел посвящён странице **Services** и представляет собой редизайн классической страницы.

Все сервисы, обнаруженные Dynatrace в вашем окружении, отображаются на странице **Services**. Можно анализировать каждый сервис и переходить к детализации на уровне кода.

![Список сервисов](https://dt-cdn.net/images/services-page-1504-133c9f1b72.png)

Список сервисов

Как перейти:

1. Перейдите в **Services**.
2. Выберите имя сервиса в списке для перехода на страницу обзора этого сервиса.

![Обзор сервиса с новым дизайном Unified Analysis](https://dt-cdn.net/images/service-overview-new-design-1768-fe1ab99c85.png)

Обзор сервиса с новым дизайном Unified Analysis

На каждой странице **Service** перечислены наиболее важные сведения о данном сервисе. Все соответствующие метрики сервиса отображаются на одной странице, разделённой на несколько логических разделов. Другие панели страницы обзора сервиса показывают производительность сервиса и служат точками входа для более глубокого анализа.

## Панель уведомлений

Панель уведомлений сервиса даёт быстрый обзор состояния сервиса. Выберите элемент уведомления для отображения дополнительной информации.

### Properties and tags

Выберите **Properties and tags** на панели уведомлений для отображения панели **Properties and tags**, которая показывает метаданные о выбранном сервисе:

* **Tags** перечисляет теги, применённые к сервису в данный момент.  
  Выберите **Add Tag** для добавления тега к метаданным сервиса.
* **Properties** перечисляет различные свойства сервиса, такие как имя приложения, тип сервиса, технологии и зоны управления.

### Problems

* На панели уведомлений **Problems** указывает активные и закрытые проблемы, связанные с выбранным сервисом.
* Выберите **Problems** на панели уведомлений для отображения панели **Problems** со списком проблем.

  + Выберите проблему для просмотра сведений.
  + Выберите **Go to problems** для перехода на страницу [Problems](/managed/dynatrace-intelligence "Ознакомьтесь с возможностями Davis AI."), отфильтрованную по выбранному сервису.

### SLOs

* На панели уведомлений **SLOs** указывает текущее количество [SLO](/managed/deliver/service-level-objectives-classic "Мониторинг и оповещение о целях уровня сервиса с Dynatrace в Service-Level Objectives Classic."), связанных с выбранным сервисом.
* Выберите **SLOs** на панели уведомлений для отображения панели **Service-level objectives** со списком SLO, прямо или косвенно связанных с сервисом.

#### Прямо связанные SLO

* SLO прямо связано с сервисом, когда [селектор сущностей](/managed/dynatrace-api/environment-api/entity-v2/entity-selector "Настройте селектор сущностей для конечных точек Environment API.") SLO соответствует следующим критериям:

  + Тип сущности установлен как `"SERVICE"`.
  + Идентификатор сущности установлен как идентификатор сервиса.
* Чтобы видеть только SLO, прямо связанные с сервисом, убедитесь, что **Show only directly connected SLOs** включено.

#### Косвенно связанные SLO

* SLO не прямо связано с сервисом, когда в [селекторе сущностей](/managed/dynatrace-api/environment-api/entity-v2/entity-selector "Настройте селектор сущностей для конечных точек Environment API.") SLO идентификатор сущности не указан.

  Пример: при указании общих значений, таких как `type("SERVICE"),tag("slo")`, запрос возвращает все SLO для всех сервисов, включая текущий.
* Чтобы видеть SLO, не связанные прямо с сервисом, отключите **Show only directly connected SLOs**.

#### Параметры

* Разверните **Details** для просмотра диаграммы соответствующих метрик SLO.
* В **Actions** выберите:

  + **View in Data Explorer** для [просмотра метрик SLO в Data Explorer](/managed/deliver/service-level-objectives-classic/configure-and-monitor-slo#explorer "Создавайте, настраивайте и отслеживайте цели уровня сервиса с Dynatrace.").
  + **Pin to Dashboard** для [закрепления SLO на дашборде](/managed/deliver/service-level-objectives-classic/configure-and-monitor-slo#dash "Создавайте, настраивайте и отслеживайте цели уровня сервиса с Dynatrace.").
  + **SLO definition** для редактирования SLO в **Service-level objective definitions**.
  + **Clone** для [клонирования SLO](/managed/deliver/service-level-objectives-classic/configure-and-monitor-slo#clone "Создавайте, настраивайте и отслеживайте цели уровня сервиса с Dynatrace.").
  + **Create alert** для [создания оповещения для SLO](/managed/deliver/service-level-objectives-classic/configure-and-monitor-slo#alerts "Создавайте, настраивайте и отслеживайте цели уровня сервиса с Dynatrace.").

#### Нет SLO

Если SLO не найдены, можно:

* Выбрать другой временной период в правом верхнем углу.

  ![Селектор временного периода: строка меню](https://dt-cdn.net/images/timeframe-selector-menu-bar-264-8193110c8c.png)

  Селектор временного периода: строка меню
* Выбрать **Add SLO** для создания SLO в [мастере SLO](/managed/deliver/service-level-objectives-classic/configure-and-monitor-slo#wizard "Создавайте, настраивайте и отслеживайте цели уровня сервиса с Dynatrace.").

### Owners

Выберите **Owners** на панели уведомлений для отображения панели **Ownership** со списком [владельцев](/managed/deliver/ownership "Сопоставляйте права собственности команды с отслеживаемыми сущностями для улучшения совместной работы, распределения задач, реагирования на инциденты и уязвимости, а также управления уровнями сервиса.") выбранного сервиса.

* Выберите  для получения дополнительной информации о текущем праве собственности.
* Чтобы добавить право собственности, выберите **Add a new Ownership tag**.

## Производительность

### Service overview

Карточку **Service overview** можно настроить для фокусировки на различных метриках производительности сервиса.

* В **Service overview** можно:

  + Просматривать сервис в Smartscape, выбрав **Smartscape view**.
  + Анализировать **Response time hotspots**, **Details of failures** и горячие точки методов.
  + Сравнивать показатели производительности запросов сервиса — время отклика, сбои, CPU и нагрузку — за разные временные периоды.
* Для каждой метрики можно выбрать **More** (**…**) и:

  + Анализировать метрику в Data Explorer.
  + Создать событие метрики.
  + Закрепить метрику на классическом дашборде. Подробности см. в разделе [Закрепление плиток на дашборде](/managed/analyze-explore-automate/dashboards-classic/charts-and-tiles/pin-tiles-to-your-dashboard "Узнайте, как закрепить плитки на ваших дашбордах.").

### Endpoints

Unified service

Карточка **Endpoints** предоставляет обзор отслеживаемых конечных точек сервиса.

* Для анализа метрик конечной точки в столбце **Details** выберите .
* Для анализа распределённых трассировок конечной точки в столбце **Actions** выберите **More** (**…**) > **View distributed traces**.

Доступность данных зависит от метрик конечных точек, потребляющих DDU. Для начала работы с мониторингом конечных точек см. раздел [Управление мониторингом конечных точек](/managed/observe/application-observability/services/service-detection/service-detection-v1/service-types/unified-service#endpoint-monitoring "Определяйте сервисы на основе сигналов наблюдаемости, принятых через Trace ingest API.").

### Service mesh metrics

Unified service

Если в приложении обнаружен сервисный меш, метрики сервисного меша отображаются для связанного сервиса. [Логи и метрики ISTIO](/managed/observe/infrastructure-observability/container-platform-monitoring/kubernetes-monitoring/monitor-istio-metrics "Приём метрик Istio и обнаружение топологии.") обогащаются идентификатором сервисного меша, помещая данные в контекст. Обратите внимание, что метрики сервисного меша создаются из принятых трассировок сервисного меша.

### Key requests

Карточка **Key requests** предоставляет обзор ключевых запросов, найденных для сервиса в выбранном временном периоде. Можно быстро получить доступ к анализу других запросов, связанных с сервисами:

* [**View all requests**](/managed/observe/application-observability/services-classic#charts "Узнайте о классическом мониторинге сервисов в Dynatrace.")
* [**Top web requests**](/managed/observe/application-observability/multidimensional-analysis/top-web-requests "Узнайте, как анализировать все веб-запросы по всем вашим сервисам с помощью Dynatrace.")
* [**View outliers**](/managed/observe/application-observability/services-classic/response-time-distribution-and-outlier-analysis#outlier-analysis "Получите представление о распределении времени отклика по всем запросам, включая аномально высокие и аномально низкие.")

### Topology

В карточке **Topology** можно узнать:

* Какие сервисы вызывают данный сервис и какие вызываются им.  
  Выберите **Related services** для понимания связи сервисов. Разверните **Details** для просмотра диаграммы соответствующих метрик сервиса. Можно быстро получить доступ к дальнейшим параметрам анализа, таким как [**View backtrace**](/managed/observe/application-observability/services-classic/service-backtrace "Отследите последовательность вызовов сервисов вплоть до нажатия в браузере, инициировавшего эту последовательность.") и [**Service flow**](/managed/observe/application-observability/services-classic/service-flow "Узнайте, как Dynatrace помогает отслеживать последовательность вызовов сервисов, инициируемых каждым запросом в вашем окружении.").
* На каких процессах и хостах запущен сервис.  
  Разверните **Details** для просмотра диаграммы соответствующих метрик процесса. Выберите имя процесса для более глубокого анализа.
* Какие HTTP-мониторы и (мобильные) приложения вызывают сервис.  
  Выберите имя вызывающей сущности для более глубокого анализа.

![Топология — обзор сервиса](https://dt-cdn.net/images/topology-services-2886-fc9dda7f82.png)

### Distributed traces

Карточка **Distributed traces** предоставляет обзор наиболее актуальных трассировок в зависимости от выбранного временного периода. Выберите **Full search** для непосредственного перехода к [обзору распределённых трассировок сервиса](/managed/observe/application-observability/distributed-traces/analysis/get-started "Начало работы с анализом распределённых трассировок в Dynatrace.").

### Events

Содержит список [событий](/managed/dynatrace-intelligence/root-cause-analysis/event-analysis-and-correlation "Ознакомьтесь с разделом Events на странице обзора каждого хоста, процесса и сервиса."), влияющих на сервис в текущем временном периоде.

### Related logs

Содержит список [логов](/managed/analyze-explore-automate/log-monitoring "Узнайте, как включить мониторинг логов, какие сведения он предоставляет, и многое другое."), связанных с сервисом в текущем временном периоде.

* Чтобы проанализировать все логи для связанного сервиса, выберите **Go to logs**.
* Чтобы проанализировать конкретный лог, разверните **Details**. Если для строки лога найдена трассировка или пользовательская сессия, вы можете получить к ним прямой доступ из этого представления.
