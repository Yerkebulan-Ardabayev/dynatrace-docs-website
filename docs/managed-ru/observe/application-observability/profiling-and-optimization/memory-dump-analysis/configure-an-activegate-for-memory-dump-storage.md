# Настройка ActiveGate для хранения дампов памяти

* How-to guide
* 4-min read
* Published Jan 08, 2019

Применимо к Environment ActiveGate и Cluster ActiveGate

Следующая конфигурация ActiveGate для хранения дампов памяти применима как к Environment ActiveGate, так и к Cluster ActiveGate.

Когда приложение испытывает утечки памяти или высокую частоту смены объектов, важно получить дампы памяти для анализа этих проблем. В производственных средах это часто затруднено, когда нет возможности войти в окружение и нет других способов запустить создание дампов памяти. Dynatrace позволяет как запускать создание дампов памяти, так и безопасно скачивать их в выбранный инструмент анализа. Для включения дампов памяти необходимо настроить ActiveGate, как описано ниже.

Если вы настраиваете ActiveGate для сбора дампов памяти, а затем перезапускаете службу ActiveGate, все экземпляры OneAgent автоматически переподключатся и распознают, что ActiveGate теперь может хранить дампы памяти. С этого момента при каждом успешном создании дампа памяти он будет автоматически загружаться на соответствующий ActiveGate. После безопасного сохранения на ActiveGate OneAgent удаляет локальный файл из своего хранилища. Обратите внимание, что OneAgent автоматически удаляет дамп памяти на исходном хосте через один час.

## Перед началом

Ознакомьтесь с [основными правилами работы с конфигурацией ActiveGate](/managed/ingest-from/dynatrace-activegate/configuration/configure-activegate#basic-rules "Узнайте, какие свойства ActiveGate можно настроить в соответствии с вашими потребностями и требованиями.").

## Активация функции создания дампов

Создайте раздел `[collector]` в файле `custom.properties`, если он ещё не существует.
Укажите свойство `DumpSupported` в этом разделе и установите значение `true`.

```
[collector]



DumpSupported = true
```

## Указание выделенного каталога для дампов (необязательно)

Если не настроено иное, ActiveGate сохраняет дампы памяти в [каталоге по умолчанию](/managed/ingest-from/dynatrace-activegate/configuration/where-can-i-find-activegate-files "Узнайте, где хранятся файлы ActiveGate в системах Windows и Linux.").

Чтобы использовать пользовательский путь, укажите путь в файле свойств и создайте каталог с соответствующими разрешениями:

Создайте раздел `[dump]` в `custom.properties` (если он ещё не существует), затем укажите свойство `dumpDir` в этом разделе. Например:

```
[dump]



dumpDir = dump
```

Свойство `dumpDir` может содержать относительный путь, например `dumpDir = mydir`. В этом случае каталог будет подкаталогом расположения по умолчанию. Также может быть задан абсолютный путь, например `dumpDir = C:\data\memory-dumps`.

Создайте каталог, если он ещё не существует, и убедитесь, что у него правильные права доступа для пользователя, запускающего службу ActiveGate. По умолчанию это `dtuserag` для Linux и `Local Service` для Windows:

* Системы Linux:  
  `Read`, `Write` и `Execute`
* Системы Windows:  
  `Read`, `Write` и `Modify`

### Использование `/temp` или `/tmp` в Linux

Из-за возможной автоматической очистки рекомендуется не использовать каталоги Linux `/temp`, `/tmp` или их подкаталоги для установки ActiveGate или хранения данных, генерируемых или используемых ActiveGate.

## Указание параметров управления файлами дампов (необязательно)

Также в разделе `[dump]` можно указать следующие свойства:

| Свойство | Значение по умолчанию | Описание |
| --- | --- | --- |
| `dumpDir` | `dump` | Допустимый путь к каталогу хранения для хранения дампов |
| `maxSizeGb` | `100` | Квота хранилища в ГБ. При заполнении самые старые дампы перезаписываются до освобождения достаточного места для нового дампа. |
| `maxAgeDays` | `7` | Максимальный возраст дампа памяти в днях до его автоматической перезаписи |
| `maxConcurrentUploads` | `5` | Максимальное количество одновременных загрузок дампов от OneAgent |
| `downloadUrl` | не задан | Пользовательский URL для скачивания. Особенно полезен, если ActiveGate находится за балансировщиком нагрузки для управления трафиком конечных пользователей или если ActiveGate обнаруживает только IP-адрес, а пользователю требуется FQDN. Обратите внимание, что балансировщик нагрузки не следует размещать между ActiveGate и OneAgent, так как это может нарушить функциональность дампов памяти. URL должен содержать правильную схему и порт. |

## Перезапуск ActiveGate

При изменении конфигурации ActiveGate необходимо [перезапустить основную службу ActiveGate](/managed/ingest-from/dynatrace-activegate/operation/stop-restart-activegate "Узнайте, как можно запускать, останавливать и перезапускать ActiveGate в Windows или Linux."), чтобы изменения вступили в силу.

## Пример содержимого файла custom.properties

```
[collector]



DumpSupported = true



[dump]



# relative to collector root dir or absolute path



dumpDir = dump



# maximum size in GB for stored memory dump



maxSizeGb = 100



# maximum age in days for keeping stored memory dumps



maxAgeDays = 7



# maximum number of concurrent file uploads supported



maxConcurrentUploads = 5
```
