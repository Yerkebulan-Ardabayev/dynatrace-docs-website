# Профилирование CPU

* How-to guide
* 2-min read
* Published Nov 22, 2018

Dynatrace предлагает следующие возможности для расширенного анализа CPU на уровне кода:

* Получение сведений о горячих точках процессов и групп процессов
* Доступ к фоновой активности, активности сервисов, выполнению отдельных запросов и конкретным запросам
* Разбивка и фильтрация данных по выполнению кода, сетевым I/O, дисковым I/O, времени блокировки и времени ожидания
* Разбивка и фильтрация данных по API
* Просмотр прямых трассировок стека и горячих точек с обратными трассировками стека
* Поддержка Java, .NET, PHP, Node.js и Golang

Чтобы выполнить анализ CPU на уровне кода:

1. Перейдите в **Profiling & Optimization** и выберите **Continuous CPU profiling**.
2. Выберите группу процессов, содержащую нужный процесс для анализа.
3. Найдите нужный процесс и в столбце **Actions** выберите соответствующий значок **Method hotspots** для перехода на страницу **Method hotspots**.

## Параметры анализа

Для анализа группы процессов удалите фильтры.

Цветовая кодировка круговой диаграммы, диаграммы с областями и полос в трассировке стека уровня кода облегчает диагностику на уровне кода.

Для доступа к разбивке процессов выберите вкладку **Top APIs**. Вы можете [настроить список API](/managed/observe/application-observability/services/customize-api-definitions "Установите правила обнаружения для настройки API в вашем окружении.").

![Вкладка Top APIs в профилировании CPU](https://dt-cdn.net/images/cpu-analysis-top-apis-1861-9b7ace016f.png)

Вкладка Top APIs в профилировании CPU

Чтобы просмотреть прямую трассировку стека, выберите вкладку **Call hierarchy**. Каждый перечисленный метод помечен именем и цветом API.

![Вкладка Call hierarchy](https://dt-cdn.net/images/tab-call-hierarchy-1599-8547c4ef49.png)

Вкладка Call hierarchy

Выберите вкладку **Hotspots** для просмотра трёх топовых горячих точек с обратными трассировками стека.

## Фильтрация

При фильтрации времени выполнения и топовых API представления **Call hierarchy** и **Hotspots** наследуют определённые фильтры.

В следующем примере фильтрация по выполнению кода и Built-In JRE (путём выбора **Code execution** на вкладке **Execution time** и **Built-In JRE** на вкладке **Top APIs**) показывает на вкладке **Hotspots**, что большая часть времени выполнения встроенного JRE API приходится на вызов `ConcurrentLinkedQueue.iterator`. Открыв кадры стека, можно выявить виновника.

## Скачивание кода

Предварительные требования

Убедитесь, что у вас есть [разрешение](/managed/manage/identity-access-management/permission-management/role-based-permissions "Разрешения на основе ролей") **View sensitive request data** в вашем окружении.

Ограничения

Скачивание кода поддерживается для Java, .NET и Node.js.

Чтобы скачать код для выполнения:

1. Перейдите в **Actions** и выберите > **Source code**.
2. Выберите процесс, из которого нужно скачать код.
3. Выберите **Download** для нужного выполнения.
4. Откройте исходный код.
