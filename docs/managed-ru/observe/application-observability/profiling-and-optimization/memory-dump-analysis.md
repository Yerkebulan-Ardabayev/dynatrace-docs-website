# Анализ дампов памяти

# Анализ дампов памяти

* How-to guide
* 5-min read
* Updated on Jan 19, 2026

Dynatrace может хранить и анализировать дампы памяти для приложений Java, .NET и Node.js.

Дампы памяти хранятся OneAgent локально в течение ограниченного времени на диске машины отслеживаемого сервера приложений.

Если настроен ActiveGate, OneAgent автоматически загружает дампы памяти на ActiveGate, который выступает в роли центрального долгосрочного хранилища дампов памяти. Этот подход гарантирует, что дампы памяти доступны только пользователям, имеющим доступ к сетевому расположению вашего ActiveGate. Эта мера предосторожности обеспечивает дополнительный уровень безопасности, гарантируя, что никакие персональные данные не покинут ваш дата-центр, если только вы специально не настроите иначе.

## Перед началом

* Для запуска дампов памяти необходимо разрешение пользователя [**View sensitive request data**](/managed/manage/identity-access-management/permission-management/role-based-permissions#environment "Разрешения на основе ролей").
* Для хранения дампов памяти на вашем сервере приложений должно быть достаточно места.
* Для доступа к сохранённым дампам памяти через веб-интерфейс Dynatrace [настройте ActiveGate для хранения дампов памяти в централизованном расположении](/managed/observe/application-observability/profiling-and-optimization/memory-dump-analysis/configure-an-activegate-for-memory-dump-storage "Узнайте, как включить хранение дампов памяти на ActiveGate."). Дополнительные сведения о сроках хранения дампов памяти см. в разделе [Периоды хранения данных](/managed/manage/data-privacy-and-security/data-privacy/data-retention-periods#memory-dumps "Проверьте сроки хранения для различных типов данных.").

## Создание дампов памяти

Чтобы создать дамп памяти

1. Перейдите на страницу **Memory dumps**:

   * На странице сущности, которую нужно проанализировать, выберите **More** (**…**) > **Memory dump details**.
   * Перейдите в **Profiling & Optimization** > **Memory dumps**.
2. Выберите нужный процесс и выберите **Trigger new dump** для создания нового дампа памяти.  
   Создание дампа памяти занимает несколько минут. Необходимое время сильно варьируется в зависимости от типа приложения. Для Java-приложений с несколькими ГБ памяти кучи это может занять несколько минут; небольшие дампы создаются почти мгновенно.

   Java

   Dynatrace использует [JVM Tool Interface (JVM TI)](https://docs.oracle.com/javase/8/docs/platform/jvmti/jvmti.html) для создания дампов памяти. По этой причине ваша JVM может зависнуть в процессе создания дампа памяти. Перезапустите Java-приложение после создания дампа памяти.
3. Через несколько минут обновите страницу. Только что созданный дамп появится в списке.

## Скачивание и просмотр дампов памяти

Чтобы скачать дампы памяти

1. Перейдите на страницу **Memory dumps**:

   * На странице сущности, которую нужно проанализировать, выберите **More** (**…**) > **Memory dump details**.
   * Перейдите в **Profiling & Optimization** > **Memory dumps**.
2. Разверните запись дампа памяти.
3. В списке **Download link** выберите ActiveGate, с которого нужно скачать дамп памяти, и выберите **Download**.

   Если скачать дамп памяти через интерфейс не удаётся, скачайте файл вручную по локальному пути, указанному в веб-интерфейсе. Обратите внимание, что дампы памяти, хранящиеся локально в OneAgent, доступны в течение ограниченного времени; при периодической очистке каталога OneAgent размер файла может составлять 0 байт.

В случае Java-приложений скачивание предоставляет дамп памяти в формате [hprof](https://dt-url.net/kh03s1r), который можно анализировать с помощью ряда инструментов, включая [Eclipse Memory analyzer](https://dt-url.net/uq23syk) и [VisualVM](https://dt-url.net/mz43sws). IBM JVM не поддерживает hprof, но поддерживает собственный формат IBM Portable Heap Dump (PHD). Его также можно анализировать с помощью Eclipse Memory analyzer.

Дампы памяти Node.js можно открывать во встроенном инструменте анализа снимков кучи памяти Google Chrome.

Дампы памяти .NET можно открывать в [PerfView](https://dt-url.net/fb03syb) или Visual Studio.

## Ограничения

* Дампы памяти .NET не поддерживаются в контейнерах на базе Alpine Linux.
* Загрузка дампов памяти не поддерживается для мониторинга [Kubernetes Application-only](/managed/ingest-from/setup-on-k8s/deployment/app-obs-managed "Развёртывание Dynatrace Operator в режиме мониторинга приложений в Kubernetes") и [Kubernetes Cloud Native Full Stack](/managed/ingest-from/setup-on-k8s/deployment/full-stack-managed "Развёртывание Dynatrace Operator в режиме облачного полного стека в Kubernetes").

  Это связано с тем, что OneAgent работает в контейнере с файловой системой только для чтения (бинарный каталог OneAgent `/opt/dynatrace/oneagent-paas` смонтирован только для чтения), поэтому Dynatrace не может записывать необходимые файлы дампов памяти. В то же время параметр установщика `DATA_STORAGE`, используемый для переопределения каталога дампов, [не поддерживается в контейнерных развёртываниях](/managed/ingest-from/setup-on-container-platforms/docker/set-up-dynatrace-oneagent-as-docker-container#limitations "Установка и обновление Dynatrace OneAgent в качестве Docker-контейнера."). Поэтому невозможно изменить расположение дампов на путь с возможностью записи. В результате сбор дампов памяти невозможен в средах Kubernetes, отслеживаемых с помощью Application-only или Cloud Native Full Stack.

## Часто задаваемые вопросы

Если я включу анализ дампов памяти на нескольких ActiveGate, какой из них будет выполнять дамп памяти?

ActiveGate имеют автоматически назначаемый приоритет. Если несколько ActiveGate имеют одинаковый приоритет, конечная точка выбирается случайным образом.

Что происходит при сбое передачи файла на ActiveGate?

OneAgent пытается отправить список дампов на все доступные конечные точки, пока не найдёт работающую. Этот процесс повторяется до успешного завершения или до удаления дампов в рамках задач устаревания (например, если их слишком много или они слишком старые).

Что происходит, если у ActiveGate заканчивается место для хранения дампов памяти?

ActiveGate сначала удаляет устаревшие дампы. Если устаревших дампов нет, ActiveGate удаляет самые старые дампы первыми.

Можно ли настроить расположение хранения дампов памяти OneAgent?

Да. OneAgent хранит дампы памяти локально и обеспечивает, чтобы они не выходили за пределы локальной сети. Можно [настроить расположение дампов памяти](/managed/ingest-from/dynatrace-oneagent/installation-and-operation/linux/installation/customize-oneagent-installation-on-linux#data-storage "Узнайте, как использовать установщик Linux с параметрами командной строки.").

Конечный пользователь не имеет доступа ни к одной из конечных точек ActiveGate. Можно ли всё равно предоставить доступ к файлу дампа памяти?

Да. Поскольку иногда конечная точка ActiveGate может быть недоступна для конечного пользователя, у ActiveGate может быть несколько IP-адресов и, следовательно, несколько конечных точек. Если все существующие конечные точки одновременно недоступны для конечного пользователя, можно всё равно предоставить доступ к файлу дампа памяти.

* Можно включить удалённый доступ к ActiveGate, изменив публичные конечные точки.  
  Чтобы узнать, как настроить новую конечную точку, см.:

  + [Включение модуля дампов памяти](/managed/ingest-from/dynatrace-activegate/configuration/configure-activegate#mem_dump_mod "Узнайте, какие свойства ActiveGate можно настроить в соответствии с вашими потребностями и требованиями.")
  + [Настройка ActiveGate для хранения дампов памяти](/managed/observe/application-observability/profiling-and-optimization/memory-dump-analysis/configure-an-activegate-for-memory-dump-storage "Узнайте, как включить хранение дампов памяти на ActiveGate.")
* Если удалённый доступ к ActiveGate невозможен, можно скачать файл дампа памяти вручную с хоста ActiveGate.

  + Для доступа к хосту ActiveGate используйте протокол, позволяющий передавать файлы (например, sFTP или SSH).
  + Для скачивания файла дампа памяти вам нужно [узнать расположение файла](/managed/observe/application-observability/profiling-and-optimization/memory-dump-analysis/configure-an-activegate-for-memory-dump-storage#specify-dedicated-dump-directory "Узнайте, как включить хранение дампов памяти на ActiveGate.").
  + Для идентификации дампа памяти распакуйте его файл с помощью протокола, включающего `summary.json` (например, sFTP или SSH).
