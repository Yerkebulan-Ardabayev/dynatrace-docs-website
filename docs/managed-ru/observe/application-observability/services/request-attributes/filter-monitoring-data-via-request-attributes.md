# Фильтрация данных мониторинга с помощью атрибутов запросов

# Фильтрация данных мониторинга с помощью атрибутов запросов

* 2-min read
* Updated on Jul 28, 2023

После определения атрибутов запросов они перечисляются в соответствующем анализе сервисов и указываются как метки для соответствующих запросов.

![Атрибуты запросов](https://dt-cdn.net/images/request-attribute-filter-1446-3c51f44605.png)

Атрибуты запросов

* Чтобы отфильтровать всё представление страницы, показывая только те запросы, которые несут определённый атрибут, выберите атрибут запроса из списка **Request attribute**.
* Чтобы проверить значения атрибута запроса, разверните его строку.

  Выбрав значение, можно отфильтровать страницу до запросов, несущих указанное значение для выбранного атрибута запроса.
* Применённый фильтр по атрибуту запроса или значению атрибута запроса сохраняется в дальнейших параметрах анализа, таких как [**Service flow**](/managed/observe/application-observability/services-classic/service-flow "Узнайте, как Dynatrace помогает отслеживать последовательность вызовов сервисов, инициируемых каждым запросом в вашем окружении.") и [**Multidimensional analysis**](/managed/observe/application-observability/multidimensional-analysis "Настройте представление многомерного анализа и сохраните его как вычисляемую метрику.").

  Пример

  Чтобы отфильтровать запросы **BookingService** по значению атрибута запроса, выберите **Request attributes** > **Booking** и значение **checkCreditCard**. Результаты отражают текущие настройки фильтра и показывают те же метрики, что и таблица запросов.

  ![Список атрибутов запросов](https://dt-cdn.net/images/request-attribute-filter-1-1439-2762e66325.png)

  Список атрибутов запросов

  + **Median response time** — медианное время отклика всех запросов, содержащих атрибут запроса.
  + **Total time consumption** — сумма времён отклика всех запросов в выбранном временном периоде, имеющих выбранный атрибут запроса.
  + Также можно просматривать соответствующие метрики пропускной способности.

  Выберите **Create analysis view** для визуализации пользовательского представления многомерного анализа, отфильтрованного по выбранному значению атрибута запроса.

  ![Многомерный анализ, отфильтрованный по значению атрибута запроса](https://dt-cdn.net/images/request-attribute-filter-2-1423-91835f8d9f.png)

  Многомерный анализ, отфильтрованный по значению атрибута запроса
* Для фильтрации [пользовательских диаграмм](/managed/analyze-explore-automate/dashboards-classic/dashboards-upgrade "Обновите пользовательские диаграммы Dynatrace до визуализаций Data Explorer.") по атрибуту запроса или значению атрибута запроса создайте [вычисляемую метрику](/managed/observe/application-observability/services/calculated-service-metric "Узнайте, как создать вычисляемую метрику на основе веб-запросов.") на основе этих условий.

  Без вычисляемой метрики, если атрибут запроса обнаружен для сервиса, все точки данных метрики сервиса отображаются в пользовательских диаграммах.
