# Захват атрибутов запросов на основе аргументов методов

# Захват атрибутов запросов на основе аргументов методов

* 6-min read
* Published May 11, 2018

Dynatrace позволяет создавать атрибуты запросов на основе аргументов методов.

## Создание атрибутов запросов

Чтобы создать атрибут запроса на основе аргумента метода

1. Перейдите в **Settings** > **Server-side service monitoring** > **Request attributes**.
2. Выберите **Define a new request attribute**.
3. Укажите уникальное **Request attribute name**. Атрибут можно переименовать в любой момент в будущем.
4. Выберите **Add new data source**.
5. Необязательно Определите область применения атрибута запроса.
6. В списке **Request attribute source** выберите **Java method parameter(s)**, **.NET method parameter(s)** или **PHP method parameter(s)**.
7. Нажмите кнопку **Select method sources** для открытия мастера классов. Здесь можно выбрать класс метода, на основе аргумента которого нужно задать атрибут.
8. Выберите группу процессов, содержащую интересующие вас классы или интерфейсы, и выберите **Continue**.
9. Найдите класс, содержащий интересующий метод. Начните вводить имя класса и нажмите кнопку **Search**. Заполнение списка может занять несколько секунд.
10. Выберите класс из отображаемого списка. Если список не содержит нужный класс, уточните строку поиска.

    ![Атрибуты запросов .NET](https://dt-cdn.net/images/request-attributes-dotnet-1-1326-b3cbc95405.png)

    Атрибуты запросов .NET
11. Наконец, выберите один или несколько методов, параметры которых нужно захватить, затем выберите **Finish**.
12. Выбранные методы перечислены в правиле захвата аргументов методов (см. ниже). Для каждого метода в раскрывающемся списке под **Capture** выберите аргумент или возвращаемое значение метода для захвата.

    ![Атрибуты запросов .NET](https://dt-cdn.net/images/request-attributes-dotnet-2-1325-6354f7c2ab.png)

    Атрибуты запросов .NET
13. Список всегда можно расширить или удалить методы позднее. После сохранения перезапустите процессы, к которым применяется это правило.

### Примечания

* Помимо аргументов и возвращаемых значений любого метода, можно также захватывать количество вхождений. При захвате вхождений следует использовать только одно правило метода. Dynatrace будет считать количество вызовов этого метода в рамках одного запроса.
* Можно [захватывать непримитивные и нестроковые объекты](#numerical). Однако это может привести к повышенным накладным расходам производительности. Некоторые объекты могут непреднамеренно изменять своё состояние в зависимости от вызываемого метода, поэтому используйте эту опцию с осторожностью.
* Базовые классы Java (например, `javax.` и `java.`) или .NET (например, `System.`) выдают предупреждение **Not instrumentable**. Эти классы использовать нельзя, поскольку их инструментирование означало бы инструментирование значительной части вашего окружения.

## Постобработка

В большинстве случаев захваченное значение будет содержать именно то, что нужно. Однако может не потребоваться всё значение или каждое значение. С помощью постобработки можно обработать захваченное значение.

Разверните опцию **Optionally restrict or process the captured parameter(s) further**, чтобы увидеть шаги обработки. Шаги выполняются в указанном порядке — каждый шаг применяется к результату предыдущего.

Применять все шаги необязательно. Каждый шаг становится активным после указания значения или выбора флажка.

![Опции постобработки для определения правила атрибута запроса](https://dt-cdn.net/images/post5steps-1353-6e753964a1.png)

Опции постобработки для определения правила атрибута запроса

Шаг 1 позволяет извлечь часть результирующей строки на основе символов-разделителей.

Шаг 2 может разбить захваченное значение на несколько значений на основе символа-разделителя.

Шаг 3 удаляет пробельные символы.

Шаг 4 позволяет отфильтровать захваченные значения, не соответствующие указанному критерию.

Шаг 5 позволяет извлечь часть результирующей строки на основе [регулярного выражения](/managed/manage/tags-and-metadata/reference/regular-expressions-in-dynatrace "Узнайте, как использовать регулярные выражения в контексте Dynatrace.").

## Числовые значения и агрегации

Может потребоваться захватить аргумент метода, который вызывается несколько раз. Иногда интересует только конкретное значение. В других случаях может понадобиться только подсчитать выполнения или усреднить захваченные значения. Помимо прочего, аргумент может быть сложным объектом, и интересует только один аспект этого объекта.

Для решения таких ситуаций можно использовать опции **Data type** и **Aggregation on request** при определении или настройке атрибута запроса.

### Агрегации

Агрегация применяется не только в рамках одного источника данных (например, правила метода), но и по нескольким источникам данных. Порядок определяется:

* Порядком правил источников данных.
* Порядком правил методов в рамках источника данных метода.
* Порядком выполнений метода в приложении (когда метод выполняется несколько раз).

#### Текстовые атрибуты

Для текстовых атрибутов можно выбрать одну из следующих опций:

* First occurrence
* Last occurrence
* Set of distinct values

Опция **Set of distinct values** позволяет использовать все уникальные значения, найденные в запросе, для фильтрации. Например, если в запросе захвачен атрибут запроса `Product` с двумя значениями — `Book` и `Video`, можно фильтровать по любому из этих значений для нахождения соответствующего запроса.

#### Числовые атрибуты

Для числовых значений можно использовать следующие агрегации:

* Minimum of captured values
* Maximum of captured values
* Average of captured values
* Sum of captured values
* Count occurrences
* Count distinct

При выборе одной из этих опций тип данных автоматически меняется на integer. Эти настройки полезны при подсчёте чего-либо.

## Доступ к вложенным объектам

Нужное значение может быть недоступно как простой аргумент, а быть частью сложного объекта-аргумента или даже переменной-членом рассматриваемого класса. Такие значения всё равно доступны. Можно обращаться не только к значениям аргументов и возвращаемым значениям, но и к самому объекту. Когда захватываемый элемент (будь то аргумент или объект) является сложным объектом, есть возможность определить метод (цепочку), обеспечивающий глубокий доступ к объекту. В примере ниже используется метод `getBookingCode`. Можно даже выполнить цепочку, например `getBookingCode().getCustomerCode()`.

![Атрибуты запросов](https://dt-cdn.net/images/deep-object-access-1157-bad06a137b.png)

Атрибуты запросов

Функция доступа к вложенным объектам вводит новый код в ваше приложение, который должен выполняться. Таким образом, она может изменить состояние приложения или оказать влияние на производительность. Используйте эту функцию с осторожностью.

### Ограничения

К доступу к вложенным объектам применяются следующие ограничения:

* Можно обращаться только к одному полю за раз. При обращении к полю оно должно быть в начале цепочки.
* Параметры методов не допускаются.
* Необходимо использовать корректную нотацию Java, .NET или PHP.
