# Атрибуты запросов

* 3-min read
* Updated on Aug 02, 2024

Dynatrace отслеживает все запросы от начала до конца и автоматически мониторит сервисы, лежащие в основе каждой транзакции. Производительность и атрибуты каждого запроса можно анализировать детально. Вы не ограничены только предопределёнными атрибутами. Можно также настраивать пользовательские *атрибуты запросов*, которые можно использовать для улучшения фильтрации и анализа веб-запросов.

Атрибуты запросов — это по существу пары ключ/значение, связанные с конкретным запросом к сервису. Например, если у вас есть туристический веб-сайт, отслеживающий направления бронирований каждого клиента, можно настроить атрибут направления для каждого запроса к сервису. Конкретное значение атрибута направления каждого запроса автоматически заполняется для всех вызовов, включающих атрибут направления (см. пример атрибута `destination` ниже). Один запрос может иметь несколько атрибутов запросов.

Несколько запросов в рамках одной распределённой трассировки могут иметь один и тот же атрибут, но с разными значениями.

## Конфиденциальные атрибуты запросов

Поскольку атрибуты запросов могут содержать конфиденциальные значения, Dynatrace позволяет пометить атрибут запроса как конфиденциальный. Для этого:

1. Перейдите в **Settings** > **Server-side service monitoring** > **Request attributes**.
2. Выберите **Edit** для соответствующего атрибута запроса.
3. Выберите **Request attribute contains confidential data**.

При включённом параметре пользователи Dynatrace, не имеющие доступа к конфиденциальным данным, видят только скрытое представление маскированных данных. Например, они могут видеть все метрики производительности, связанные с выполнением определённого SQL-оператора, но все конфиденциальные значения в операторе заменены звёздочками (`*****`) и скрыты от несанкционированного доступа.

## Ограничения

### Количество атрибутов запросов

Максимальное количество атрибутов запросов на запрос — 100.

### Количество значений атрибута запроса

Максимальное количество значений атрибута запроса на запрос — 10.

### Количество значений атрибута запроса в числовых расчётах

Для каждого запроса максимальное количество значений атрибута запроса, оцениваемых в числовом расчёте (таком как `avg`, `sum`, `count` или `max`), составляет 1 000.

### Количество захватываемых атрибутов запросов на распределённую трассировку

Максимальное количество атрибутов запросов, которые OneAgent может захватить для распределённой трассировки, — 1 000. К этому ограничению относятся атрибуты запросов, захватываемые несколько раз в рамках распределённой трассировки, и атрибуты запросов, захватываемые в отдельных запросах. После достижения ограничения последующие атрибуты запросов не захватываются.
