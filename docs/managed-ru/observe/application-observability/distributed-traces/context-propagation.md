## Контекст трассировки между сервисами

Dynatrace обеспечивает непрерывную видимость потоков сервисов, распространяя контекст трассировки по мере перемещения транзакций между сервисами и компонентами. Это распространение необходимо для сквозного мониторинга распределённых приложений.

## Автоматическое распространение с помощью OneAgent

При установке OneAgent он автоматически:

* Внедряет контекст трассировки в исходящие запросы.
* Считывает контекст трассировки из входящих запросов.
* Поддерживает контекст транзакций через границы сервисов.
* Использует различные механизмы для корреляции распределённых трассировок, что позволяет Dynatrace отображать полный поток транзакций через ваше приложение.

## Механизмы распространения

Dynatrace использует несколько механизмов для поддержания контекста трассировки:

* **x-dynatrace** — для различных протоколов связи. Этот формат является собственностью Dynatrace.
* **traceparent** и **tracestate** — стандартный формат W3C, используемый как OneAgent, так и OpenTelemetry.
* **dtdTraceTagInfo** — пользовательское свойство для различных систем обмена сообщениями.

Конкретная реализация варьируется в зависимости от технологии: иногда используются HTTP-заголовки HTTPS, свойства сообщений SQS или внедрение данных в payload AWS EventBridge.

## Смешанные среды с OpenTelemetry

В смешанных средах, где используются как OneAgent, так и инструментирование OpenTelemetry, распространение контекста трассировки устраняет разрыв между ними:

* Как OneAgent, так и OpenTelemetry используют формат W3C Trace Context.
* Контекст должен последовательно распространяться по каналу связи.
* Для HTTP это стандартизировано в заголовках.
* Для других протоколов, таких как обмен сообщениями и события, необходимо соблюдать единообразие размещения между OneAgent и OpenTelemetry.

## Включение контекста трассировки W3C

Существует несколько причин для включения контекста трассировки W3C в Dynatrace:

* Совместимость с отраслевым стандартом — соответствует спецификации W3C.
* Независимая от поставщика трассировка — работает в гетерогенных средах с несколькими решениями мониторинга.
* Перспективность — соответствует направлению развития отрасли в области распределённой трассировки.

### Особенности использования контекста трассировки W3C

* Проблемы совместимости — несмотря на стандартизацию, реализации в реальных условиях могут различаться.
* Поведение браузера и приложений — некоторые клиенты могут многократно отправлять один и тот же traceId, что влияет на качество трассировки.
* Конфликты инструментов — несколько инструментов APM в одном процессе могут перезаписывать контекст друг друга.

### Настройка контекста трассировки W3C

Чтобы включить контекст трассировки W3C

1. Перейдите в **Settings** > **Preferences** > **OneAgent features**.
2. Включите **Send W3C Trace Context HTTP headers** и **Send W3C Trace Context gRPC headers**.

Хотя стандарт W3C формально регламентирует распространение через HTTP, Dynatrace и отрасль в целом применяют эти концепции к другим протоколам связи.
