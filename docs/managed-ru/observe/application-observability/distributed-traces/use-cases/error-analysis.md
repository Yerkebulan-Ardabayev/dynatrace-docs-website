# Детализация до причин сбоев сервиса

* Tutorial
* 1-min read
* Updated on May 17, 2024

Анализ отдельных запросов — полезный способ получить более глубокое понимание обнаруженных ошибок. В этой статье вы узнаете, как определить ошибку, лежащую в основе растущей частоты отказов сервиса, с помощью распределённой трассировки.

## Сценарий

На изображении ниже видно, что запросы к `Redis` начали завершаться с ошибкой примерно в `10:45` на временной шкале.

## Шаги

1. Чтобы перейти на вкладку **Failure rate**, откройте страницу сведений о сервисе и выберите кнопку **View** (например, **View requests**, **View dynamic requests** или **View resource requests**).

   ![Распределённая трассировка 11](https://dt-cdn.net/images/purepath11-1574-ab48d3472e.png)

   Распределённая трассировка 11
2. Выберите **Analyze backtrace**, чтобы узнать, откуда поступали эти запросы.

   Запросы исходят из сервиса `weather-express`, и почти все неудачные запросы к `Redis` имеют одинаковое исключение — `AbortError`, вызванный закрытым соединением.

   ![Распределённая трассировка 12](https://dt-cdn.net/images/purepath12-1591-7acd7c6a02.png)

   Распределённая трассировка 12
3. Чтобы детализировать до затронутых трассировок `Node.js`, выберите **More** (**…**) > **Distributed traces**.

   Глядя на трассировку `Node.js` и дерево выполнения кода ниже, можно увидеть, что запрос к `Redis` приводит к ошибке. Видно, где именно эта ошибка возникает в потоке кода `Node.js`.

   ![Распределённая трассировка 13](https://dt-cdn.net/images/purepath13-1017-a26a6a6140.png)

   Распределённая трассировка 13
4. Выберите вкладку **Errors** для анализа исключения.

   ![Распределённая трассировка 14](https://dt-cdn.net/images/purepath14-1015-5ae03d05ff.png)

   Распределённая трассировка 14

## Заключение

Каждая распределённая трассировка на вкладке **Errors** показывает уникальный набор параметров, предшествующих ошибке. При таком подходе к анализу представление распределённых трассировок может быть очень полезным для понимания причин возникновения тех или иных исключений.
