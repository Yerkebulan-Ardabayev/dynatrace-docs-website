# Process group availability monitoring and alerting

# Мониторинг доступности и оповещения для групп процессов

* How-to guide
* Published May 12, 2018

Если у вас есть критически важные процессы, доступность которых необходимо отслеживать, или если нужно обеспечить, чтобы количество процессов кластера никогда не опускалось ниже определённого порога, Dynatrace позволяет выбрать наиболее подходящую стратегию оповещений о доступности. Это означает, что Dynatrace можно настроить для проактивного оповещения при отключении или аварийном завершении любого процесса или нескольких процессов в конкретной [группе процессов](/managed/observe/infrastructure-observability/process-groups "Analyze process groups and customize process group naming, detection, and monitoring."). Оповещения содержат ссылки на соответствующие [страницы проблем](/managed/dynatrace-intelligence "Get familiar with the capabilities of Davis AI.") Dynatrace, что упрощает доступ ко всем деталям, связанным с недоступным процессом, для быстрого устранения проблемы.

## Включение мониторинга доступности группы процессов

Вместо стандартной передачи событий доступности процессов для групп, обслуживающих сервисы, можно настроить получение оповещений при недоступности процесса или несоблюдении минимального порога.

1. Перейдите в **Hosts**.
2. Выберите интересующий хост.
3. Прокрутите вниз и выберите **Consuming processes**.
4. В списке **Process** выберите интересующую группу процессов (или отдельный процесс).
5. В меню просмотра (**…**) выберите **Settings** для доступа к **Process group settings**.
6. Выберите **Availability monitoring**.
7. По умолчанию оповещение отключено. Для его включения необходимо активировать переключатель **Enable process group availability monitoring**.

## Когда активировать этот параметр

После включения **Enable process group availability monitoring** доступны два варианта.

Получать оповещения только о наиболее важных проблемах доступности группы процессов

Этот вариант инициирует событие доступности при недоступности любого процесса в выбранной группе процессов.

![Pg settings](https://dt-cdn.net/images/2021-03-12-11-53-24-1127-df5e27dbf8.png)

Создавать оповещение при недоступности любого процесса в выбранной группе процессов

Этот вариант инициирует событие доступности при несоблюдении заданного пользователем порогового значения минимального количества работающих процессов в выбранной группе.

В примере ниже минимальное необходимое количество работающих процессов установлено равным `2`. Это означает, что всякий раз, когда в данной группе процессов будет менее двух работающих процессов, Dynatrace будет инициировать событие уровня доступности для группы.
