# Переход на правила группировки процессов

* Практическое руководство
* 3 минуты чтения
* Опубликовано 10 нояб. 2025 г.

Это обновление объединяет следующее в единые правила группировки процессов:

* Простые правила обнаружения
* Расширенные правила обнаружения
* Декларативные правила группировки процессов

## Преимущества правил группировки процессов

Теперь можно переопределять встроенные правила обнаружения и заново определять группировку и обнаружение процессов, активируя разделение, объединение или отключение мониторинга. Такое объединение теперь позволяет выполнять группировку на основе переменных окружения, процессов, свойств и пользовательских строк. Одни и те же правила теперь применяются ко всем агентам и режимам, что снижает неопределённость при выборе нужных настроек для изменения группировки по умолчанию.

## Сценарии перехода

В зависимости от предыдущих настроек групп процессов возможны два сценария перехода:

| Сценарий | Что происходит |
| --- | --- |
| Предыдущих правил обнаружения нет | Переход выполняется автоматически. |
| Есть существующие декларативные правила группировки, простые или расширенные правила обнаружения | Существует [риск перехода](#migration-risks), при котором идентификаторы сущностей групп процессов могут измениться. Переход нужно выполнить вручную. |

### Ограничения перехода

| Общее количество правил в области действия | Лимит |
| --- | --- |
| Декларативная группировка + простые правила обнаружения + расширенные правила обнаружения | < 400 |

## Риски перехода

Переход на правила группировки процессов сопряжён со следующими рисками:

* Идентификаторы сущностей изменятся для процессов, сгруппированных в данный момент по простым и расширенным правилам обнаружения.
* Автоматический переход может не покрыть все потребности клиента, поэтому могут потребоваться ручные корректировки.

## Переход на правила группировки процессов вручную

### Предварительные условия

* **Права доступа**: для запуска перехода нужно одно из следующего:

  + Роль `environment:roles:manage-settings`.
  + Безусловные права одновременно на `settings:objects:read` и `settings:objects:write`.

  Подробнее о ролях и правах доступа см. в разделе [Управление идентификацией и доступом](/managed/manage/identity-access-management "Настройка пользователей, групп и прав доступа.").

  Без необходимых прав доступа кнопка **Migrate** недоступна.
* **Версия OneAgent**: на всех хостах нужен OneAgent версии 1.329 и выше.

### Процесс перехода

Если существуют предыдущие правила обнаружения, они конвертируются из старых настроек в виде предварительного просмотра только для чтения, что позволяет выполнить проверку вручную. После проверки можно нажать кнопку **Migrate**, чтобы переключиться на новую функцию.
