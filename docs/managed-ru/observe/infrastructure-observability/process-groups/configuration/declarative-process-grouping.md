# Declarative process grouping

# Декларативная группировка процессов

* How-to guide
* 5-min read
* Updated on Jan 07, 2026

OneAgent version 1.217+

В крупных динамических средах количество процессов, выполняющихся на хостах, может стать трудно управляемым. По этой причине Dynatrace автоматически отслеживает [важные типы групп процессов](/managed/observe/infrastructure-observability/process-groups/basic-concepts/which-are-the-most-important-processes "Display the most important processes for monitoring and process grouping.") — группы процессов, относящиеся либо к известной технологии, либо потребляющие значительные ресурсы.

Однако Dynatrace также предоставляет возможность мониторинга конкретных процессов, не попадающих ни в одну из этих двух категорий. Это означает, что можно отслеживать процессы технологий, неизвестных Dynatrace, которые не потребляют значительных ресурсов.

Начиная с OneAgent version 1.307, глубоко отслеживаемые процессы поддерживаются декларативной группировкой в режимах Full-Stack Monitoring, Infrastructure Monitoring и Discovery при развёртывании через установщик OneAgent и в классическом развёртывании Kubernetes full-stack. Для других развёртываний декларативная группировка поддерживается для процессов без глубокого мониторинга.

После внесения изменений конфигурации перезапустите глубоко отслеживаемые процессы для применения обновлений.

Используйте декларативную группировку процессов осторожно, поскольку она может вызвать проблемы совместимости с расширениями или плагинами, зависящими от имён экземпляров групп процессов (PGI) или технологий. Такие изменения могут привести к сбоям и потере данных или метрик в отслеживаемых процессах.

Для минимизации риска проблем совместимости при настройке декларативной группировки процессов рекомендуется:

* Убедиться, что расширения или плагины не будут затронуты новыми правилами группировки.
* Протестировать правила декларативной группировки в контролируемой среде перед применением в production, чтобы исключить нежелательные побочные эффекты.
* Регулярно проверять и обновлять правила декларативной группировки в соответствии с изменениями в среде или настройках приложения.

Функция декларативной группировки позволяет добавлять детальные правила для сопоставления с любой конкретной группой процессов. Декларативная группировка основана на фреймворке Settings 2.0, поэтому правила можно создавать через веб-интерфейс Dynatrace или [Settings API](#api) — см. [Пример JSON-payload для конфигурации декларативной группы процессов](#eg3).

## Создание декларативной группы процессов

Чтобы создать группу процессов через веб-интерфейс Dynatrace:

1. Перейдите в **Settings**.
2. Выберите **Processes and containers** > **Declarative process grouping**.
3. Выберите **Add monitored technology**. Технология — это логический контейнер для группировки правил обнаружения. Она будет отображаться как основная технология группы процессов.
4. Выберите **Add process group**. Группа процессов может включать несколько отдельных правил обнаружения. Группа процессов идентифицируется, если все отдельные правила обнаружения совпадают.
5. Перейдите к **Process group display name** и выберите имя группы процессов.
6. Введите **Process group identifier** — уникальную строку для идентификации группы процессов в Dynatrace.
7. OneAgent version 1.259+ Выберите параметр **Report process group**:

   * **Always** — созданная группа процессов всегда передаётся в Dynatrace.
   * **Never** — созданная группа процессов никогда не передаётся в Dynatrace.
   * **Only when resource usage is high** — группа процессов передаётся при использовании CPU, памяти или сетевого трафика свыше 5%. Подробнее см. в разделе [Most important processes](/managed/observe/infrastructure-observability/process-groups/basic-concepts/which-are-the-most-important-processes "Display the most important processes for monitoring and process grouping.").
8. Выберите **Add detection rule**.

   1. Выберите объект (свойство процесса), по которому будет проверяться правило обнаружения:

      * **Command line arguments**
      * **Executable**
      * **Executable path**

      При добавлении пути к исполняемому файлу необходимо учитывать различия между Windows и Linux. Windows использует обратную косую черту в качестве разделителя пути, Linux — прямую. Автоматической нормализации путей на стороне OneAgent нет. Поэтому необходимо указывать правильный формат пути для используемой ОС.
   2. Введите **Condition**.

      На предыдущем шаге вы выбрали свойство процесса для проверки (например, путь к исполняемому файлу). На этом шаге задайте условие (оператор и строку), которое сравнивается с выбранным свойством процесса. Можно использовать один из следующих операторов:

      * `$prefix` = начинается с

        Например, `$prefix(/usr/sbin/keepalived)` истинно, если выбранное свойство процесса начинается с `/usr/sbin/keepalived`.
      * `$suffix` = заканчивается на

        Например, `$suffix(keepalived)` истинно, если выбранное свойство процесса заканчивается на `keepalived`.
      * `$eq` = равно; точное совпадение

        Например, `$eq(-d)` истинно, если выбранное свойство процесса равно `-d`.
      * `$contains` = содержит; включает данную подстроку

        Например, `$contains(keepalived)` истинно, если `keepalived` встречается в любом месте выбранного свойства процесса.

      Операторы сравнения работают для каждого параметра командной строки отдельно. Например, процесс `python my.py -ab -cd -ef` будет соответствовать условию `$contains(cd)`, `$eq(-ab)`, но не будет соответствовать `$suffix(-cd -ef)`, поскольку `-cd` и `-ef` — отдельные аргументы, обрабатываемые независимо.

      Обратите внимание, что исполняемый файл также является частью командной строки в качестве первого аргумента.

      Все условия нечувствительны к регистру.
   3. Необязательно: добавьте ещё одно правило обнаружения.

      * При добавлении нескольких правил обнаружения процесс идентифицируется только при совпадении всех правил (отношение AND).
      * Если нескольким правилам соответствует один и тот же процесс, применяется только первое правило. Это зависит от порядка правил.
9. Завершив определение всех правил, выберите **Save changes**.

После сохранения изменений Dynatrace автоматически обнаружит и будет отслеживать новую пользовательскую группу процессов на всех хостах — в том числе на хостах, запущенных после определения пользовательской группы.

## Управление декларативной группировкой процессов

Для управления декларативными группами процессов:

1. Перейдите в **Declarative process grouping** для нужного уровня настройки.

   Уровень хоста

   1. Перейдите в **Hosts**.
   2. Найдите и выберите хост для отображения страницы его обзора.
   3. В правом верхнем углу страницы обзора хоста выберите **More** (**…**) > **Settings**.

   4. В настройках хоста выберите **Declarative process grouping**.

   Уровень группы хостов

   1. Перейдите в **Deployment Status** > **OneAgents**.
   2. На странице **OneAgent deployment** отключите **Show new OneAgent deployments**.
   3. В поле **Filter by** введите **Host group** и выберите нужную группу хостов из раскрывающегося списка.

      Список хостов теперь отфильтрован по выбранной группе хостов. Каждый перечисленный хост имеет ссылку **Host group:** `<имя группы>`.

      Свойство **Host group** не отображается, если выбранный хост не принадлежит ни одной группе хостов.
   4. Выберите имя группы хостов в любой строке.

      Так как фильтр применён по группе хостов, все отображаемые хосты относятся к одной группе.

   5. В настройках группы хостов выберите **Declarative process grouping**.

   Уровень среды

   Перейдите в **Settings** и выберите **Processes and containers** > **Declarative process grouping**.
2. Определённые декларативные группы процессов отображаются в таблице под кнопкой **Add monitored technology**.

   * Чтобы прекратить мониторинг указанной группы процессов, отключите **Enabled**.
   * Для удаления группы процессов из таблицы выберите ![Remove](https://dt-cdn.net/images/remove-icon-105c6a04c2.svg "Remove") в столбце **Delete**.
   * Для просмотра и редактирования деталей группы процессов выберите ![Expand row](https://dt-cdn.net/images/expand-row-icon-9c4950fc2e.svg "Expand row") в столбце **Details**.
