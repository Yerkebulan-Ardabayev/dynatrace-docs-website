# Анализ сервисов баз данных (новая страница)

* Explanation
* 5-min read
* Published Jun 20, 2023

Страница обзора базы данных была полностью переработана.

* Данная документация описывает новый дизайн.
* Чтобы вернуться к классической странице базы данных, на странице обзора базы данных выберите **More** (**…**) > **Return to classic page**, а затем обратитесь к [документации по классической странице базы данных](/managed/observe/infrastructure-observability/databases/database-services-classic/analyze-database-services "Анализируйте сервисы баз данных с помощью Dynatrace (классическая страница).").

Все базы данных, обнаруженные Dynatrace в вашей среде, отображаются на странице **Databases**. Можно анализировать каждую базу данных и детализировать информацию до уровня кода.

Как перейти:

1. Перейдите в **Database Services**.
2. Выберите имя базы данных в списке, чтобы перейти на страницу обзора этой базы данных.

На каждой странице **Database** перечислена наиболее важная информация по данной базе данных.

![Обзор базы данных | Unified analysis](https://dt-cdn.net/images/database-ua-overview-3502-2a520ae771.png)

Обзор базы данных | Unified analysis

Все актуальные метрики базы данных отображаются на одной странице, которая разделена на несколько логических разделов. Другие панели страницы обзора базы данных показывают производительность базы данных и служат точками входа для более глубокого анализа.

## Панель уведомлений

Панель уведомлений базы данных даёт быстрый обзор состояния базы данных. Выберите элемент уведомления для отображения дополнительной информации.

### Свойства и теги

Выберите **Properties and tags** на панели уведомлений, чтобы открыть панель **Properties and tags**, которая отображает метаданные выбранной базы данных:

* **Tags** — список тегов, применённых к базе данных в настоящее время.  
  Выберите **Add Tag**, чтобы добавить тег в метаданные базы данных.
* **Properties** — список различных свойств базы данных, таких как имя приложения, тип базы данных, технологии и зоны управления.

### Проблемы

* На панели уведомлений **Problems** отображает активные и закрытые проблемы, связанные с выбранной базой данных.
* Выберите **Problems** на панели уведомлений, чтобы открыть панель **Problems** со списком проблем.

  + Выберите проблему для просмотра деталей.
  + Выберите **Go to problems**, чтобы перейти на страницу [Problems](/managed/dynatrace-intelligence "Ознакомьтесь с возможностями Davis AI."), отфильтрованную по выбранной базе данных.

### SLO

* На панели уведомлений **SLOs** показывает текущее количество [SLO](/managed/deliver/service-level-objectives-classic "Мониторинг и оповещения по целевым показателям уровня обслуживания с помощью Dynatrace в Service-Level Objectives Classic."), связанных с выбранной базой данных.
* Выберите **SLOs** на панели уведомлений, чтобы открыть панель **Service-level objectives** со списком SLO, напрямую или косвенно связанных с базой данных.

#### Напрямую связанные SLO

* SLO напрямую связан с сервисом, когда [селектор объектов](/managed/dynatrace-api/environment-api/entity-v2/entity-selector "Настройка селектора объектов для конечных точек Environment API.") SLO соответствует следующим критериям:

  + Тип объекта установлен как `"DATABASE"`.
  + Идентификатор объекта установлен как идентификатор базы данных.
* Чтобы видеть только SLO, напрямую связанные с базой данных, убедитесь, что **Show only directly connected SLOs** включён.

#### Косвенно связанные SLO

* SLO не связан напрямую с базой данных, если в [селекторе объектов](/managed/dynatrace-api/environment-api/entity-v2/entity-selector "Настройка селектора объектов для конечных точек Environment API.") SLO не указан идентификатор объекта.

  Пример: когда указаны общие значения, такие как `type("DATABASE"),tag("slo")`, запрос возвращает все SLO для всех баз данных, включая текущую.
* Чтобы видеть SLO, не связанные напрямую с базой данных, отключите **Show only directly connected SLOs**.

#### Параметры

* Разверните **Details** для просмотра графика соответствующих метрик SLO.
* В разделе **Actions** выберите:

  + **View in Data Explorer**, чтобы [просмотреть метрики SLO в Data Explorer](/managed/deliver/service-level-objectives-classic/configure-and-monitor-slo#explorer "Создание, настройка и мониторинг целевых показателей уровня обслуживания с помощью Dynatrace.").
  + **Pin to Dashboard**, чтобы [закрепить SLO на дашборде](/managed/deliver/service-level-objectives-classic/configure-and-monitor-slo#dash "Создание, настройка и мониторинг целевых показателей уровня обслуживания с помощью Dynatrace.").
  + **SLO definition**, чтобы изменить SLO в **Service-level objective definitions**.
  + **Clone**, чтобы [клонировать SLO](/managed/deliver/service-level-objectives-classic/configure-and-monitor-slo#clone "Создание, настройка и мониторинг целевых показателей уровня обслуживания с помощью Dynatrace.").
  + **Create alert**, чтобы [создать оповещение для SLO](/managed/deliver/service-level-objectives-classic/configure-and-monitor-slo#alerts "Создание, настройка и мониторинг целевых показателей уровня обслуживания с помощью Dynatrace.").

#### Нет SLO

Если SLO не найдены, можно:

* Выбрать другой временной интервал в правом верхнем углу.

  ![Выбор временного интервала: строка меню](https://dt-cdn.net/images/timeframe-selector-menu-bar-264-8193110c8c.png)

  Выбор временного интервала: строка меню
* Выбрать **Add SLO**, чтобы создать SLO в [мастере SLO](/managed/deliver/service-level-objectives-classic/configure-and-monitor-slo#wizard "Создание, настройка и мониторинг целевых показателей уровня обслуживания с помощью Dynatrace.").

### Доступность базы данных

Выберите **Database availability** на панели уведомлений, чтобы отобразить график с обзором обнаруженных проблем с доступностью базы данных за выбранный временной интервал.

### Владельцы

Выберите **Owners** на панели уведомлений, чтобы открыть панель **Ownership** со списком [владельцев](/managed/deliver/ownership "Сопоставление ответственности команды с мониторируемыми объектами для улучшения совместной работы, назначения задач, реагирования на инциденты и уязвимости, а также управления уровнями обслуживания.") выбранной базы данных.

* Выберите ℹ, чтобы узнать больше о текущей ответственности.
* Для добавления тега ответственности выберите **Add Ownership tag**.

## Производительность

### Обзор сервиса базы данных

Раздел **Database service overview** можно настроить для фокусировки на различных метриках производительности базы данных. Для каждой метрики можно выбрать **More** (**…**) и:

* Проанализировать метрику в Data Explorer.
* Создать событие метрики.
* Закрепить метрику на классическом дашборде. Подробности см. в разделе [Закрепление плиток на дашборде](/managed/analyze-explore-automate/dashboards-classic/charts-and-tiles/pin-tiles-to-your-dashboard "Узнайте, как закреплять плитки на дашбордах.").

### Топология

В разделе **Topology** можно узнать:

* Сервисы, обращающиеся к базе данных, и сервисы, к которым обращается база данных.  
  Выберите **Related services**, чтобы понять связи между сервисами. Разверните **Details** для просмотра графика соответствующих метрик сервиса. Для продолжения анализа можно выбрать [**View backtrace**](/managed/observe/application-observability/services-classic/service-backtrace "Трассировка последовательности вызовов сервисов вплоть до клика в браузере, который инициировал цепочку вызовов.").
* Процессы и хосты, на которых работает сервис.  
  Разверните **Details** для просмотра графика соответствующих метрик процесса. Выберите имя процесса для его анализа.

### Типы операторов

Содержит обзор типов операторов, найденных для базы данных за выбранный временной интервал.

* Разверните **Details** для просмотра графика соответствующего оператора.
* Выберите имя типа оператора для анализа операторов БД с фильтрацией по выбранному типу.
* Выберите **View all statements** для анализа всех операторов для базы данных.

### Распределённые трассировки

Раздел **Distributed traces** предоставляет обзор последних трассировок за выбранный временной интервал. Выберите **Full search**, чтобы перейти к [обзору распределённых трассировок для базы данных](/managed/observe/application-observability/distributed-traces/analysis/get-started "Начало работы с анализом распределённых трассировок в Dynatrace.").

### События

Список [событий](/managed/dynatrace-intelligence/root-cause-analysis/event-analysis-and-correlation "Понимание раздела Events на каждой странице обзора хоста, процесса и сервиса."), влияющих на базу данных в текущем временном интервале.

### Связанные журналы

Список [журналов](/managed/analyze-explore-automate/log-monitoring "Узнайте, как включить мониторинг журналов, какие аналитические данные он предоставляет и многое другое."), связанных с базой данных в текущем временном интервале.

* Для анализа всех журналов связанной базы данных выберите **Go to logs**.
* Для анализа конкретного журнала разверните **Details**. Если для строки журнала найдена трассировка или пользовательская сессия, к ней можно обратиться непосредственно из этого представления.
