# Поддержка переменных связывания SQL

* How-to guide
* 4-min read
* Updated on Apr 26, 2023

Переменные связывания — это способ параметризации SQL-операторов, при котором в их предложениях **where** используются знаки вопроса или параметры, например:

* SQL server: `select count (*) from report where tenant = @tenant`
* Java JDBC: `select count (*) from report where tenant = ?`

Переменные связывания позволяют серверу базы данных один раз подготовить оператор и выполнять его многократно без повторного разбора и анализа.

Переменные связывания неприменимы к операторам, использующим литералы, например:

```
select count (*) from report where tenant = 'xxxx'
```

Такие операторы не могут быть параметризованы и повторно разбираются и анализируются сервером базы данных при каждом выполнении.

Переменные связывания создают высокую нагрузку на сеть и хранилище. Для получения дополнительных сведений о поддержке переменных связывания и их доступности см. раздел [ЧЗВ](#faq) ниже.

## Включение захвата переменных связывания SQL

Для включения захвата значений SQL-переменных связывания:

1. Перейдите в **Settings** > **Server-side service monitoring** > **Deep monitoring**.
2. Разверните **Database** и включите/отключите **Capture SQL bind values** на глобальном уровне.
3. Необязательно Чтобы переопределить глобальную настройку, перейдите в **Process group override**.

   1. Чтобы добавить переопределение, выберите **Add process group override** и выберите затронутую группу процессов.
   2. Необязательно Чтобы выбрать конкретный процесс из выбранной группы процессов, выберите процесс из выпадающего списка.
   3. Выберите **Add**.
4. Выберите **Save changes**.

![Захват значений SQL-переменных связывания](https://dt-cdn.net/images/capture-sql-bind-values-942-067033b12f.png)

Захват значений SQL-переменных связывания

Независимо от того, включена ли эта настройка на уровне всего сайта или для отдельных групп процессов, можно использовать Dynatrace OneAgent для захвата значений переменных связывания. Это применимо к следующим технологиям:

* ADO.net
* JDBC
* Фреймворки баз данных PHP

Если массив, возвращаемый методом `executeBatch()`, содержит более одного элемента, что указывает на выполнение нескольких команд, Dynatrace маскирует значения переменных связывания для обеспечения конфиденциальности данных. Это связано с тем, что различные выполнения `executeBatch()` могут объединять несколько команд, что требует маскировки значений переменных связывания для предотвращения раскрытия конфиденциальной информации.

## ЧЗВ

Имеет ли захват переменных связывания негативные последствия?

Эта функция может захватывать большой объём конфиденциальных данных — поэтому использовать её следует осторожно.
Функция также может захватывать большой объём данных и увеличивать сетевой трафик и требования к хранилищу для распределённых трассировок.
Переменные связывания также могут привести к увеличению нагрузки на обработку и хранение для вашего Managed-кластера.
