# Обнаружение и мониторинг

* Explanation
* 4-min read
* Updated on Jul 26, 2024

Dynatrace предоставляет аналитические данные о нагрузке на базу данных, временах ответа и частоте ошибок. Можно мониторить даже время получения соединения. Поскольку Dynatrace является решением с полным стеком, оно также предоставляет аналитику с инфраструктурной точки зрения для ваших баз данных — просто за счёт развёртывания OneAgent на хостах баз данных. Это добавляет второй ракурс для метрик работоспособности, таких как загрузка CPU или памяти, и даже сетевое состояние. В анализ включены даже файлы журналов.

С мониторингом баз данных на уровне кода можно:

* Видеть влияние операторов БД на времена ответа сервисов.
* Находить затратные операторы БД на основе вызовов сервисов или действий пользователей, из которых они исходят.
* Выяснять, какие сервисы обращаются к базам данных чаще всего.
* Видеть, какую нагрузку отдельные сервисы создают на базы данных.
* Понимать, почему некоторые операторы работают медленно.
* Получать уведомления об увеличении стоимости и частоты выполнения SQL-операторов.

Вызовы баз данных и технология PurePath®

Распределённая трассировка запроса к базе данных показывает конкретный оператор БД, выполненный приложением. Поскольку трассировки фиксируются внутри приложения, а не в самой базе данных, трассировки операторов БД отображаются только в том случае, если приложение, их выполняющее, либо отправляет распределённые трассировки в [API приёма трассировок OpenTelemetry](/managed/ingest-from/opentelemetry/otlp-api "Узнайте о конечных точках OTLP API, которые ваше приложение использует для экспорта данных OpenTelemetry в Dynatrace."), либо мониторится через OneAgent. Если приложение ещё не запускало распределённые трассировки, можно определить пользовательский сервис. Посмотрите страницу **Method hotspots**, чтобы выбрать метод, вызываемый до запроса к базе данных в том же потоке.

Атрибуты, используемые для обнаружения, отмечены звёздочкой ✱ на странице обзора [базы данных](/managed/observe/infrastructure-observability/databases/database-services-classic/analyze-database-services-new "Анализируйте сервисы баз данных с помощью Dynatrace (новая страница).") в разделе **Properties and tags**.

## Как мониторится активность базы данных

При мониторинге баз данных Dynatrace рассматривает не методы, а фиксации, запросы и изменения, связанные с сервисами баз данных. Мониторя такие вызовы, Dynatrace обеспечивает автоматический анализ первопричин проблем производительности сервисов баз данных. Например, если запросы или фиксации в базу данных замедляются, вы немедленно получите уведомление. Будет показано, какие сервисы затронуты проблемой.

Вызовы баз данных, выполняемые через мониторируемые процессы Java, .NET, PHP и Node.js, мониторятся автоматически при условии, что взаимодействие с базой данных основано на [поддерживаемом фреймворке баз данных](/managed/ingest-from/technology-support#applications-services-and-databases "Технические сведения о поддержке Dynatrace конкретных платформ и фреймворков разработки."), таком как JDBC, ADO.NET или PDO. Когда OneAgent установлен на хосте, запускающем сервер приложений, Dynatrace обеспечивает запись всех операторов БД при условии, что для вызывающих процессов активен глубокий мониторинг и запрос к базе данных добавлен к существующей распределённой трассировке PurePath®. Как только начнут мониториться первые обращения к базе данных, страница **Databases** обновится с новой записью базы данных.

При мониторинге активности базы данных Dynatrace показывает, какие операторы БД выполняются чаще всего и какие операторы занимают больше всего времени. Также можно видеть, какие сервисы выполняют операторы БД. Например, Dynatrace автоматически узнаёт, когда сервис начинает отправлять слишком много операторов к базе данных и перегружает сервер базы данных.

Когда возникает проблема, связанная с базой данных, например из-за снижения времени ответа, Dynatrace сообщает, есть ли корреляция между проблемой и ростом нагрузки на связанный сервис, который может быть первопричиной проблемы.

## Включение мониторинга

Dynatrace мониторит ряд технологий баз данных с помощью развёртывания OneAgent, а также с помощью расширений ActiveGate, запускаемых с ActiveGate, удалённо подключающегося к серверу базы данных.
