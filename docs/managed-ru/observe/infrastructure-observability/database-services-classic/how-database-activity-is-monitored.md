# Обнаружение и мониторинг

* Разъяснение
* Чтение 4 мин
* Обновлено 26 июля 2024 г.

Dynatrace предоставляет данные о нагрузке на базу данных, времени отклика и частоте сбоев. Можно отслеживать даже время получения соединения. Поскольку Dynatrace это полностековое решение, оно также даёт представление об инфраструктуре баз данных, достаточно развернуть OneAgent на хостах баз данных. Это добавляет второй ракурс: метрики состояния вроде использования CPU или памяти и даже состояние сети. В анализ включаются даже файлы логов.

С мониторингом баз данных на уровне кода можно:

* Видеть влияние операторов базы данных на время отклика сервисов.
* Находить затратные операторы базы данных на основе вызовов сервисов или пользовательских действий, из которых они происходят.
* Узнать, какие сервисы чаще всего обращаются к базам данных.
* Видеть, какую нагрузку создают на базы данных отдельные сервисы.
* Понимать, почему некоторые операторы выполняются медленно.
* Получать уведомления о возросшей стоимости и частоте выполнения SQL-операторов.

Вызовы базы данных и технология PurePath®

Распределённая трасса запроса к базе данных показывает конкретный оператор базы данных, выполненный приложением. Поскольку трассы фиксируются внутри приложения, а не на самой базе данных, трассы операторов базы данных видны только в том случае, если приложение, которое их делает, либо отправляет распределённые трассы на [эндпоинт приёма трасс OpenTelemetry API](/managed/ingest-from/opentelemetry/otlp-api "Узнайте об эндпоинтах OTLP API, которые приложение использует для экспорта данных OpenTelemetry в Dynatrace."), либо отслеживается через OneAgent. Если указанное приложение ещё не запустило распределённые трассы, можно определить пользовательский сервис. На странице **Method hotspots** нужно выбрать метод, который вызывается перед запросом к базе данных в том же потоке.

Атрибуты, используемые для обнаружения, отмечены звёздочкой ✱ на странице обзора [базы данных](/managed/observe/infrastructure-observability/database-services-classic/analyze-database-services-new "Анализируйте сервисы баз данных с помощью Dynatrace (новая страница).") в разделе **Properties and tags**.

## Как отслеживается активность базы данных

При мониторинге баз данных Dynatrace учитывает не методы, а коммиты, запросы и изменения, относящиеся к сервисам баз данных. Отслеживая такие вызовы, Dynatrace способен выполнять автоматический анализ первопричин проблем производительности сервисов баз данных. Например, если запросы или коммиты к базе данных замедляются, уведомление приходит немедленно. Также показывается, какие сервисы затронуты проблемой.

Вызовы базы данных, выполняемые через отслеживаемые процессы Java, .NET, PHP и Node.js, отслеживаются автоматически, пока взаимодействие с базой данных опирается на [поддерживаемый фреймворк базы данных](/managed/ingest-from/technology-support#applications-services-and-databases "Узнайте технические подробности о поддержке Dynatrace для конкретных платформ и фреймворков разработки."), такой как JDBC, ADO.NET или PDO. Когда OneAgent установлен на хосте, где выполняется сервер приложений, Dynatrace обеспечивает журналирование всех операторов базы данных, пока для вызывающих процессов активен глубокий мониторинг, а запрос к базе данных добавлен к существующей распределённой трассе PurePath®. Как только фиксируются первые вызовы к базе данных, страница **Databases** обновляется новой записью базы данных.

При мониторинге активности базы данных Dynatrace показывает, какие операторы базы данных выполняются чаще всего и какие занимают больше всего времени. Также видно, какие сервисы выполняют операторы базы данных. Например, Dynatrace автоматически определяет, когда сервис начинает отправлять слишком много операторов базы данных и выводит сервер базы данных за пределы его возможностей.

Когда возникает проблема, связанная с базой данных, например из-за падения времени отклика, Dynatrace сообщает, есть ли корреляция между проблемой и ростом нагрузки на связанный сервис, что может быть первопричиной проблемы.

## Включение мониторинга

Dynatrace отслеживает ряд технологий баз данных через развёртывание OneAgent, а также с помощью расширений ActiveGate, которые можно запускать с ActiveGate, подключающегося к серверу базы данных удалённо.
