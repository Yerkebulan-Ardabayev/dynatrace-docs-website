# Поддержка переменных привязки SQL

* Практическое руководство
* Чтение 4 мин
* Обновлено 26 апр. 2023 г.

Переменные привязки, это способ параметризации SQL-инструкций, при котором инструкции содержат вопросительные знаки или параметры в предложении **where**, например:

* SQL server: `select count (*) from report where tenant = @tenant`
* Java JDBC: `select count (*) from report where tenant = ?`

Переменные привязки позволяют серверу базы данных подготовить инструкцию один раз и выполнять её многократно без повторного разбора или анализа.

Переменные привязки неприменимы к инструкциям, использующим литералы, например:

```
select count (*) from report where tenant = ‘xxxx’
```

Такие инструкции нельзя параметризовать, и сервер базы данных выполняет их повторный разбор и анализ при каждом выполнении.

Переменные привязки создают высокую нагрузку на сеть и хранилище. Подробнее о поддержке переменных привязки и её доступности см. в разделе [FAQ](#faq) ниже.

## Включение захвата переменных привязки SQL

Для включения захвата значений привязки SQL

1. Перейти в **Settings** > **Server-side service monitoring** > **Deep monitoring**.
2. Развернуть **Database** и включить/выключить **Capture SQL bind values** на глобальном уровне.
3. Опционально Чтобы переопределить глобальную настройку, перейти в **Process group override**.

   1. Чтобы добавить переопределение, выбрать **Add process group override** и указать нужную группу процессов.
   2. Опционально Чтобы выбрать конкретный процесс из выбранной группы процессов, выбрать процесс из выпадающего списка.
   3. Выбрать **Add**.
4. Выбрать **Save changes**.

![Capture SQL bind values](https://dt-cdn.net/images/capture-sql-bind-values-942-067033b12f.png)

Capture SQL bind values

Независимо от того, включена ли эта настройка для всего сайта или для отдельных групп процессов, Dynatrace OneAgent можно использовать для захвата значений переменных привязки. Это применимо к следующим технологиям:

* ADO.net
* JDBC
* Фреймворки баз данных PHP

Если массив, возвращаемый `executeBatch()`, содержит больше одного элемента, что указывает на выполнение нескольких команд, Dynatrace маскирует значения переменных привязки для обеспечения конфиденциальности данных. Это связано с тем, что разные выполнения `executeBatch()` могут агрегировать несколько команд, что требует маскирования значений переменных привязки для предотвращения раскрытия конфиденциальной информации.

## FAQ

Есть ли негативные последствия у захвата переменных привязки?

Эта функция может захватывать много конфиденциальных данных, поэтому её использование стоит тщательно продумать.
Функция также может захватывать большой объём данных и увеличивать сетевой трафик, а также потребность в хранилище для распределённых трасс.
Переменные привязки также могут приводить к повышенной нагрузке на обработку и хранение данных для Managed Cluster.
