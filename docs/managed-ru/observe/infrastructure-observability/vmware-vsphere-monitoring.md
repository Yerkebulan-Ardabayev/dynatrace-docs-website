# Мониторинг VMware vSphere

* Практическое руководство
* Опубликовано 12 августа 2021 г.

Настроить мониторинг Dynatrace для платформы VMware легко, используя ActiveGate в качестве коммуникационного шлюза.

* ActiveGate получает данные от VMware и отправляет их в Dynatrace Cluster.
* OneAgent, установленный на каждой виртуальной машине, предоставляет дополнительные данные о состоянии вашей инфраструктуры.

**Поток данных мониторинга с вашей платформы VMware в Dynatrace:**

![Virtualization data flow](https://dt-cdn.net/images/virtualization-flow-1280-93a1053e89.png)

Поток данных виртуализации

Следующее относится только к VMware. Для других платформ виртуализации достаточно установить OneAgent для мониторинга виртуализированных хостов, так как мониторинг уровней управления виртуализацией поддерживается только для VMware.

После установки Dynatrace OneAgent и активации мониторинга процессов на виртуальной машине вы видите, что происходит в вашей операционной системе, в частности, как ведут себя и взаимодействуют процессы на хосте.

Dynatrace собирает информацию об использовании виртуализированного CPU, потреблении памяти и операциях с хранилищем. Dynatrace также обнаруживает миграции виртуальных машин (vMotion) и создание новых виртуальных машин.

Выполните приведённые ниже шаги, чтобы настроить мониторинг на уровне управления виртуализацией вашего VMware vCenter или автономных хостов ESXi.

## Предварительные требования

* Доступ только для чтения к серверу vCenter или доступ к автономному хосту ESXi.

## Установка и настройка ActiveGate

[Установите Environment ActiveGate](/managed/ingest-from/dynatrace-activegate/installation "Узнайте, как настроить ActiveGate") в своём дата-центре перед подключением Dynatrace к платформе VMware.  
Для **Dynatrace Managed** можно использовать встроенный ActiveGate, работающий на узле кластера. Однако Cluster ActiveGate обычно используется для пересылки данных мониторинга RUM и/или Synthetic в Dynatrace Cluster. Рекомендуем не перегружать этот ActiveGate другим типом данных мониторинга. В зависимости от размера VMware стоит рассмотреть выделенный ActiveGate для каждого окружения.

Для мониторинга виртуализации свойство `vmware_monitoring_enabled` в `custom.properties` должно иметь значение `true` (значение по умолчанию).

Подробнее см. в разделе [Настройка свойств ActiveGate](/managed/ingest-from/dynatrace-activegate/configuration/configure-activegate#vmware "Узнайте, какие свойства ActiveGate можно настроить под свои нужды и требования.").

## Подключение Dynatrace к платформе VMware

Чтобы подключить Dynatrace к платформе VMware

1. Откройте **Settings** > **Cloud and virtualization** > **VMware** и нажмите **Connect new instance**.
2. Укажите IP-адрес или имя сервера vCenter либо автономного хоста ESXi, который нужно отслеживать (без префикса протокола `http://` или `https://`).
3. Проверьте настройки сети/прокси.  
   Если возникает ошибка связи, хотя предоставленные данные верны, причиной могут быть настройки сети/прокси. Рекомендуем проверять настройки сети/прокси при добавлении новой интеграции VMware.

   Необязательно. При настройке интеграции VMware вы также можете обойти прокси для подключения к vCenter или ESXi. Измените [конфигурацию ActiveGate](/managed/ingest-from/dynatrace-activegate/configuration/set-up-proxy-authentication-for-activegate#exclude-hosts "Узнайте, как настроить свойства ActiveGate для установки прокси."), чтобы исключить определённый хост из прокси.
4. Введите учётные данные пользователя, чтобы ActiveGate мог войти и собирать данные мониторинга. Требуемые для этого пользователя права: **просмотр и доступ только для чтения**. Доступ уровня администратора для включения мониторинга не требуется (изменения в настройках VMware не нужны).

   Не нужно добавлять хосты ESXi по отдельности, если они управляются сервером vCenter.
5. ActiveGate версии 1.268+ Укажите условие фильтра, чтобы ограничить число отслеживаемых кластеров:

   * `$prefix(parameter)`: значение свойства начинается с `parameter`
   * `$eq(parameter)`: значение свойства точно совпадает с `parameter`
   * `$suffix(parameter)`: значение свойства заканчивается на `parameter`
   * `$contains(parameter)`: значение свойства содержит `parameter`
6. Нажмите **Test connection**, чтобы убедиться, что введённые данные успешно подключились к вашему vCenter.

   Учётные данные

   Учётные данные больше не проверяются автоматически, поэтому важно указывать действительные учётные данные для подключения к вашему vCenter. Если указать недействительные учётные данные, Dynatrace всё равно попытается подключиться к вашему vCenter, что может создавать ненужный сетевой трафик.

   Если учётные данные для конкретного vCenter со временем меняются, а вы забываете обновить их в настройках, Dynatrace обнаружит пять неудачных попыток подключения к вашему vCenter. После этого данная настройка будет отключена, чтобы предотвратить блокировку вашего аккаунта VMware.
7. Нажмите **Save changes**.

   Синхронизация времени

   Различия в системном времени могут приводить к пропуску метрик VMware. Чтобы Dynatrace корректно отображал данные мониторинга, синхронизируйте настройки времени на всех отслеживаемых хостах и vCenter с NTP-сервером.

Чтобы охватить всю виртуальную инфраструктуру, повторите эти шаги для всех остальных серверов vCenter или автономных хостов ESXi в вашем окружении.

## Ограничение мониторинга инфраструктуры VMware

После настройки мониторинга VMware вам может потребоваться ограничить, какие элементы инфраструктуры (например, хосты и виртуальные машины) должны фактически отслеживаться Dynatrace. Для этого можно использовать механизм разрешений, доступный в VMware. Подробнее см. в разделе [Ограничение мониторинга инфраструктуры VMware с помощью разрешений](/managed/observe/infrastructure-observability/vmware-vsphere-monitoring/limit-infrastructure-monitoring-using-permissions "Ограничьте размер отслеживаемой инфраструктуры VMware с помощью механизма разрешений VMware.").

## Настройка мониторинга vSphere с помощью Settings API

Для настройки мониторинга VMware vSphere можно использовать [Settings API](/managed/dynatrace-api/environment-api/settings "Узнайте, что предлагает Dynatrace Settings API.").

1. Чтобы узнать схему, используйте [GET a schema](/managed/dynatrace-api/environment-api/settings/schemas/get-schema "Просмотр схемы настроек через Dynatrace API.") с `builtin:virtualization.vmware` в качестве schemaId.
2. На основе схемы `builtin:virtualization.vmware` создайте объект конфигурации.
3. Чтобы создать конфигурацию, используйте [POST an object](/managed/dynatrace-api/environment-api/settings/objects/post-object "Создание или проверка объекта настроек через Dynatrace API.").
