# Ограничение мониторинга инфраструктуры VMware с помощью разрешений

* How-to guide
* Published Feb 10, 2021

Следующее относится только к VMware. Для других платформ виртуализации достаточно установить OneAgent для мониторинга виртуализированных хостов, поскольку мониторинг уровней управления виртуализацией поддерживается только для VMware.

При настройке мониторинга VMware можно управлять инфраструктурными элементами (такими как хосты и виртуальные машины), которые должны мониториться Dynatrace, путём установки или изменения разрешений в vCenter.

Ниже приведены примеры управления разрешениями пользователей для виртуальных машин.

## Предварительные требования

* Роль администратора с доступом к управлению разрешениями в vCenter

## Мониторинг всех виртуальных машин в пуле ресурсов

Для мониторинга всех виртуальных машин в пуле ресурсов (или любом другом «родительском» элементе иерархии инфраструктуры VMware) необходимо:

Назначить пользователям разрешения только для чтения для хостов VMware

Для назначения пользователям разрешений только для чтения для хостов VMware:

1. В vCenter перейдите в представление хоста.
2. Выберите **Permissions**, затем значок `+`.
3. Выберите имя пользователя, затем роль **Read-only**.
4. Нажмите **OK** для сохранения изменений.

Поскольку виртуальная машина может позднее перенестись на другой хост, рекомендуется включить мониторинг Dynatrace (добавить разрешения только для чтения) на всех хостах, на которые может перенестись виртуальная машина.

Назначить пользователям разрешения только для чтения для пула ресурсов

Для назначения пользователям разрешений только для чтения для пула ресурсов:

1. В vCenter перейдите в представление пула ресурсов.
2. Выберите **Permissions**, затем значок `+`.
3. Выберите имя пользователя, затем роль **Read-only**.
4. Выберите **Propagate to children**.
5. Нажмите **OK** для сохранения изменений.

После назначения пользователям разрешений только для чтения для хостов VMware и пула ресурсов хосты и виртуальные машины, для которых предоставлены разрешения, станут видимыми в Dynatrace.

## Исключение отдельной виртуальной машины из мониторинга

Для исключения отдельной виртуальной машины из мониторинга Dynatrace, например виртуальной машины, унаследовавшей разрешения только для чтения от родительского элемента, необходимо удалить разрешения на чтение для этой виртуальной машины.

Для удаления разрешений пользователя на чтение для виртуальной машины:

1. В vCenter перейдите в представление виртуальной машины.
2. Выберите **Permissions**, затем значок `+`.
3. Выберите имя пользователя, затем выберите **No access** для роли.
4. Нажмите **OK** для сохранения изменений.

## Добавление отдельной виртуальной машины в мониторинг

Для добавления отдельной виртуальной машины в мониторинг Dynatrace необходимо назначить пользователям разрешения только для чтения для этой виртуальной машины.

Для назначения пользователям разрешений только для чтения для виртуальной машины:

1. В vCenter перейдите в представление виртуальной машины.
2. Выберите **Permissions**, затем значок `+`.
3. Выберите имя пользователя, затем роль **Read-only**.
4. Нажмите **OK** для сохранения изменений.
