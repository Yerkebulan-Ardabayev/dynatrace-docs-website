# Мониторинг Heroku

С включённым облачно-нативным мониторингом Dynatrace для приложений Heroku доступно следующее:

* Глубокий мониторинг приложений и подробности на уровне кода для Java, PHP, Node.js и других, с помощью всего одного независимого от языка buildpack
* [Автоматический анализ первопричин](/managed/dynatrace-intelligence/root-cause-analysis/concepts "Ознакомьтесь с концепциями анализа первопричин.") веб-приложений Heroku
* Информация о том, как приложения Heroku используют [сервисы баз данных](/managed/observe/infrastructure-observability/databases "Узнайте, как автоматически обнаруживать сервисы баз данных, анализировать сервисы баз данных и многое другое."), включая подробные метрики для каждого запроса к базе данных
* Данные [Real User Monitoring](/managed/observe/digital-experience/rum-classic/rum-concepts/rum-overview "Узнайте о Real User Monitoring Classic, ключевых показателях производительности, мониторинге мобильных приложений и многом другом.") о поведении клиентов в веб-браузере и на мобильных устройствах
* Автоматизированный мониторинг внешних и сторонних сервисов (например, вызовов внешних REST API)

## Просмотр результатов мониторинга

После привязки учётной записи Dynatrace к приложению Heroku становится доступен весь спектр возможностей мониторинга приложений и сервисов, которые предоставляет Dynatrace (например, **Smartscape** и данные на уровне сервисов с помощью **Service flow**). Dynatrace автоматически определяет, что приложение работает на Heroku, а также сервисы, связанные с этим приложением Heroku.

![Мониторинг Heroku](https://dt-cdn.net/images/heroku1-2874-778cc74486.png)

Мониторинг Heroku

Dynatrace автоматически запускает глубокий мониторинг приложений Heroku и предоставляет видимость на уровне кода для сервисов этих приложений. **Service flow** Dynatrace позволяет отслеживать, как запросы к сервисам, предоставляемым приложением Heroku, распространяются по системе. Трассировка сервисов также помогает выявлять узкие места производительности и неудачные запросы в цепочке взаимодействия сервисов друг с другом. С Dynatrace ещё никогда не было так просто определить первопричину低кой производительности в гетерогенных стеках микросервисов.

## Тегирование приложений Heroku

Можно использовать мощный [механизм тегирования](/managed/manage/tags-and-metadata "Используйте теги и метаданные для организации данных в среде Dynatrace.") Dynatrace для автоматической организации и фильтрации всех отслеживаемых компонентов приложений Heroku. Dynatrace позволяет применять теги к процессам и хостам на основе переменных окружения.

`heroku config:set DT_TAGS=owner=team-easytravel`
