# Container monitoring rules

# Правила мониторинга контейнеров

* How-to guide
* 2-min read
* Published Jun 25, 2021

Для управления всеми настройками, связанными с контейнерами, следуйте приведённым ниже инструкциям.

## Определение правил мониторинга контейнеров

Правила мониторинга контейнеров определяют, включён ли углублённый мониторинг для процессов, работающих в контейнерах.

**Добавление правила мониторинга**

1. Перейдите в **Settings** > **Processes and containers** > **Container monitoring rules**.
2. Нажмите **Add item**.
3. Выберите **Mode**: следует ли Dynatrace отслеживать контейнер.
4. Выберите **Container property** для сравнения с вашим правилом.
5. Выберите оператор условия (например, `begins with`).
6. Задайте значение условия.
7. Нажмите **Save changes**, чтобы сохранить конфигурацию и добавить новое правило в список правил мониторинга контейнеров.

**Редактирование пользовательского правила мониторинга**

1. Перейдите в **Settings** > **Processes and containers** > **Container monitoring rules**.
2. Найдите правило и внесите изменения.

   * Чтобы изменить условия правила, откройте раздел **Details** и внесите необходимые изменения.
   * Чтобы удалить правило, нажмите в столбце **Delete**.
   * Чтобы отключить или включить правило, измените переключатель в столбце **Enabled**.
   * Чтобы изменить порядок правила в списке, в столбце **Order** перетащите правило на новую позицию.

     Правила мониторинга контейнеров выполняются в том порядке, в котором они отображаются в списке.
3. Нажмите **Save changes**.

## Включение и отключение правил мониторинга контейнеров

Существует две категории правил мониторинга контейнеров, которые можно включать или отключать по умолчанию:

* Пользовательские правила — созданные вами.
* Встроенные правила — определённые Dynatrace.

**Включение/отключение пользовательского правила мониторинга**

1. Перейдите в **Settings** > **Processes and containers** > **Container monitoring rules**.
2. Найдите правило и измените переключатель в столбце **Enabled**.
3. Нажмите **Save changes**, чтобы сохранить конфигурацию.

**Включение/отключение встроенных правил мониторинга**

1. Перейдите в **Settings** > **Processes and containers** > **Built-in container monitoring rules**.
2. Включите или отключите следующие правила по необходимости:

   * Do not monitor containers where Kubernetes container name equals `POD`
   * Do not monitor containers where Docker stripped image name contains `pause-amd64`
   * Do not monitor containers where Kubernetes namespaces equals `openshift-sdn`
3. Нажмите **Save changes**, чтобы сохранить конфигурацию.

Встроенные правила включены по умолчанию. Их можно отключить, но нельзя редактировать.

## Ограничения

* Правила мониторинга контейнеров действуют только при установке OneAgent на хостах.
* Интеграции только для приложений без полной установки OneAgent не поддерживают правила мониторинга. Однако в таких случаях сами интеграции обеспечивают аналогичный уровень контроля над настройкой мониторинга контейнеров.
* В Kubernetes правила мониторинга контейнеров применяются только к режиму внедрения `classicFullStack`.

  Правила мониторинга контейнеров игнорируются для режимов внедрения на основе webhook (`cloudNativeFullStack` или `applicationMonitoring`). Для этих режимов следует использовать конфигурацию на основе аннотаций, описанную в разделе [Настройка мониторинга для пространств имён и подов](/managed/ingest-from/setup-on-k8s/guides/deployment-and-configuration/monitoring-and-instrumentation/annotate "Configure monitoring for namespaces and pods").
