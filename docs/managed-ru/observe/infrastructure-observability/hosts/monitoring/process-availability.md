# Process availability

# Доступность процессов

* How-to guide
* 5-min read
* Updated on Jan 29, 2026

OneAgent version 1.237+

Для мониторинга доступности ключевых процессов на хостах необходимо определить правила мониторинга. После создания правила Dynatrace генерирует оповещающее событие при отсутствии совпадающего процесса на хосте.

Последнюю активность процессов, определённых для мониторинга доступности, можно анализировать в разделе [Process instance snapshots](/managed/observe/infrastructure-observability/hosts/monitoring/host-monitoring#snapshots "Monitor hosts with Dynatrace.") на странице обзора хоста.

## Определение области применения

Правила можно создавать для применения на уровне среды, группы хостов и хоста. Правила на более низком уровне переопределяют правила на более высоком уровне. Например, правило, созданное на уровне хоста, переопределяет правило с тем же именем, созданное на уровне среды.

## Определение правила мониторинга

1. Перейдите на страницу **Process availability** для уровня, на котором должно применяться правило:

   * **Environment**: перейдите в **Settings** и выберите **Processes and containers** > **Process availability**.
   * **Host group**: перейдите на страницу группы хостов по адресу `https://your-environment/ui/settings/HOST_GROUP-NAME` и выберите **Process availability**.
   * **Host**: перейдите на страницу обзора хоста, выберите **More** (**…**), перейдите в **Settings** и выберите **Process availability**.
2. На странице **Process availability** выберите **Add monitoring rule**. Мониторинг доступности процессов может включать несколько отдельных правил обнаружения. Процесс идентифицируется, если все отдельные правила обнаружения совпадают.
3. В поле **Monitoring rule name** введите имя, под которым правило будет отображаться в списке.
4. В разделе **Operating system** (OneAgent version 1.287+) выберите операционные системы, к которым должно применяться правило мониторинга. Можно выбрать несколько систем.

   * Windows
   * Linux
   * AIX
5. Задайте **Minimum number of matching processes** (OneAgent version 1.287+) — минимальное количество процессов, которые должны совпадать с данным правилом. Если на отдельном хосте совпадает меньше процессов, создаётся оповещение.
6. Выберите **Add detection rule**, чтобы определить правило обнаружения.

   Одно правило мониторинга может содержать несколько правил обнаружения. При добавлении нескольких правил обнаружения процесс идентифицируется только при совпадении всех правил (отношение AND).

   * **Rule scope** — выбор **Process** или **Host** определяет последующие параметры конфигурации.

     Process

     + **Select process property** — объект, по которому проверяется правило обнаружения:

       - **Command line - single argument** — каждый параметр командной строки оценивается отдельно. Оценка чувствительна к регистру.
       - **Command line** — OneAgent version 1.333+ Вся командная строка оценивается целиком. Оценка чувствительна к регистру.
       - **Executable** — оценка чувствительна к регистру для Linux и AIX и **не** чувствительна для Windows.
       - **Executable path** — оценка чувствительна к регистру для Linux и AIX и **не** чувствительна для Windows.
       - **User** — OneAgent version 1.287+ Оценка чувствительна к регистру для Linux и AIX и **не** чувствительна для Windows.

     Операторы сравнения оценивают каждый параметр командной строки отдельно для свойства **Command line - single argument** (в версиях до 1.333 — **Command line**). Например, процесс `python my.py -ab -cd -ef` будет соответствовать условию `$contains(cd)`, `$eq(-ab)`, но не будет соответствовать `$suffix(-cd -ef)`, поскольку `-cd` и `-ef` — отдельные аргументы, обрабатываемые независимо.

     OneAgent version 1.307+ Исполняемый файл также рассматривается как часть командной строки в качестве первого аргумента.

     + **Condition** — в зависимости от условий совпадения правила можно определить строку с использованием:

       - `$contains` — совпадает, если свойство содержит указанное значение. Например, `$contains(keepalived)` совпадает, если `keepalived` встречается в любом месте свойства.
       - `$eq` — совпадает, если свойство точно соответствует указанному значению. Например, `$eq(-d)` совпадает, если `-d` точно соответствует свойству.
       - `$prefix` — совпадает, если свойство начинается с указанного значения. Например, `$prefix(/usr/sbin/keepalived)` совпадает со свойством, начинающимся с `/usr/sbin/keepalived`.
       - `$suffix` — совпадает, если свойство заканчивается на указанное значение. Например, `$suffix(keepalived)` совпадает со свойством, заканчивающимся на `keepalived`.

     Host

     OneAgent version 1.287+

     **Custom metadata** — это определяемые пользователем пары «ключ–значение», которые можно назначать хостам, отслеживаемым Dynatrace.

     Определяя пользовательские метаданные, можно обогатить данные мониторинга контекстом, специфичным для нужд организации: именами сред, принадлежностью команде, версиями приложений и другими сведениями.

     + **Key** — ключ метаданных для сопоставления.
     + **Condition** — строка, которая:

       - `$contains(production)` — совпадает, если `production` встречается в значении метаданных хоста.
       - `$eq(production)` — совпадает, если `production` точно соответствует значению метаданных хоста.
       - `$prefix(production)` — совпадает, если `production` является префиксом значения метаданных хоста.
       - `$suffix(production)` — совпадает, если `production` является суффиксом значения метаданных хоста.

       Доступные логические операции:

       - `$not($eq(production))` — совпадает, если значение метаданных хоста не равно `production`.
       - `$and($prefix(production),$suffix(main))` — совпадает, если значение начинается с `production` и заканчивается на `main`.
       - `$or($prefix(production),$suffix(main))` — совпадает, если значение начинается с `production` или заканчивается на `main`.

       **Экранирование специальных символов**: при включении специальных символов, например `(` и `)`, в выражения сопоставления, экранируйте их тильдой `~`. Например, для сопоставления со значением метаданных `my(amazing)property` используйте `$eq(my~(amazing~)property)`.
7. Для добавления ещё одного правила обнаружения к данному правилу мониторинга повторите предыдущий шаг.
8. Выберите **Add property** (OneAgent version 1.249+ Dynatrace version 1.249+), чтобы задать пользовательское свойство «ключ–значение» для события.

   * **Key**: введите `dt.` в поле **Key** для получения подсказок.
   * **Value**: введите `{` в поле **Value** для получения подсказок.

   Можно использовать только значения, предложенные в подсказках.

   Пример пользовательского сообщения в деталях события:

   * **Key** = `custom.message`
   * **Value** = `The {dt.entity.host} is deployed on: {dt.os.type}`

   В данном примере **Value** содержит два ключа:

   * `{dt.entity.host}`
   * `{dt.os.type}`

   Значения хоста-сущности и типа ОС будут извлечены при срабатывании правила. Если подстановка ключа не удастся, ключ и значение будут недоступны.
9. Завершив определение правила мониторинга, включая все правила обнаружения, выберите **Save changes**.

После сохранения изменений:

* Правило мониторинга добавляется в список правил на странице **Process availability**. В качестве отображаемого имени используется значение, введённое в поле **Monitoring rule name**.
* Правило мониторинга применяется на уровне, выбранном на первом шаге: среды, группы хостов или хоста.

## Управление правилами

Правила мониторинга отображаются на странице **Process availability**. Каждое правило мониторинга содержит список из одного или нескольких правил обнаружения.

* Для просмотра или редактирования деталей правила мониторинга или обнаружения выберите ![Expand row](https://dt-cdn.net/images/expand-row-icon-9c4950fc2e.svg "Expand row") в столбце **Details** для нужного правила.
* Для изменения порядка правил перетащите ![Drag handle](https://dt-cdn.net/images/drag-handle-turquoise-600-1aa0e5ea00.svg "Drag handle") правило на нужную позицию в списке.
* Для удаления правила выберите ![Remove](https://dt-cdn.net/images/remove-icon-105c6a04c2.svg "Remove") в столбце **Delete** и подтвердите действие.
* Для включения или отключения правила (только для правил мониторинга) используйте переключатель в столбце **Enabled**.
