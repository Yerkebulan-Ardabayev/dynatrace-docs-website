# Exclude disks and network traffic from host monitoring

# Исключение дисков и сетевого трафика из мониторинга хостов

* How-to guide
* 4-min read
* Updated on Jul 08, 2024

OneAgent автоматически обнаруживает и отслеживает точки монтирования и сетевой трафик хоста, однако вы можете исключить выбранные точки монтирования или сетевой трафик из мониторинга.

## Исключение дисков

Используйте настройки **Disk options** для создания правил исключений, позволяющих убрать диски из мониторинга.

* Некоторые файловые системы (например, `autofs`, `proc`, `cgroup`, `tmpfs`) всегда исключены, поскольку их мониторинг не имеет смысла.
* Правила исключения дисков можно создавать на уровне среды, группы хостов или хоста. Исключения, заданные на более низком уровне, переопределяют исключения на более высоком уровне (например, исключения уровня хоста переопределяют исключения уровня среды).

Чтобы создать правило исключения диска:

1. Перейдите на страницу **Disk options** для нужного уровня:

   * Среда

     Перейдите в **Settings** > **Preferences** > **Disk options**.
   * Группа хостов

     1. Перейдите в **Deployment Status** > **OneAgents**.
     2. Отфильтруйте таблицу по `Host group` и выберите группу хостов, для которой нужно создать правило исключения диска.
     3. Для любого перечисленного хоста (все они принадлежат выбранной группе хостов) выберите ссылку **Host group**.
     4. Выберите **Disk options**.
   * Хост

     1. Перейдите в **Hosts** и выберите хост, для которого нужно создать правило исключения диска.
     2. На странице обзора хоста выберите > **Settings**.
     3. Выберите **Disk options**.

   После перехода на страницу **Disk options** шаги для создания правила исключения диска одинаковы. Важное различие — уровень, к которому применяется правило: среда, группа хостов или хост.
2. Нажмите **Add item**.
3. Выберите **Operating system** для исключаемого диска.
4. Задайте в поле **Disk or mount point path** путь к точке монтирования диска, исключаемого из мониторинга.

   **Примеры:**

   * `/mnt/my_disk`
   * `/staff/emp1`
   * `C:\`
   * `/staff/*`
   * `/disk*`

   * Пути к точкам монтирования чувствительны к регистру.
   * Подстановочный знак в `/staff/*` означает исключение всех дочерних папок `/staff`.
   * Подстановочный знак в `/disk*` означает исключение всех точек монтирования, начинающихся с `/disk` (например, `/disk1`, `/disk99` и `/diskabc`).
5. Задайте в поле **File system type** тип файловой системы, исключаемой из мониторинга.

   **Примеры:**

   * `ext4`
   * `ext3`
   * `btrfs`
   * `ext*`

   * Типы файловых систем чувствительны к регистру.
   * Подстановочный знак в `ext*` означает исключение соответствующих файловых систем (например, типов `ext4` и `ext3`).
6. Нажмите **Save changes**.

## Исключение сетевого трафика

Используйте настройки **Exclude network traffic** для исключения трафика на определённых сетевых интерфейсах или хостах из мониторинга.

### Исключение сетевых интерфейсов (NIC)

Весь сетевой трафик от всех выбранных NIC исключается из мониторинга.

1. На странице **Exclude network traffic** в разделе **Exclude NIC** нажмите **Add item**.
2. Задайте в поле **Operating system** операционную систему сетевого интерфейса.
3. Задайте в поле **Name** имя сетевого интерфейса для выбранной операционной системы.
4. Нажмите **Save changes**.

### Исключение IP-адресов

Весь сетевой трафик от всех выбранных IP-адресов хостов исключается из вычисления связности (другие метрики по-прежнему рассчитываются). Это может быть полезно, например, для предотвращения ложных оповещений о связности.

Чтобы исключить IP-адрес из расчётов связности:

1. На странице **Exclude network traffic** в разделе **Exclude IP** нажмите **Add item**.
2. Введите IP-адрес, трафик которого нужно исключить из расчётов связности. Подстановочные знаки и диапазоны не допускаются.
3. Нажмите **Save changes**.

## Диски, не отслеживаемые OneAgent

Следующие диски не отслеживаются OneAgent. Чтобы исключить дополнительный тип файловой системы или имя точки монтирования, см. раздел [Исключение дисков](#disk-options).

| Операционная система | Тип | Исключённые диски |
| --- | --- | --- |
| Все поддерживаемые ОС | Файловые системы | `hsfs`  `devtmpfs`  `sysfs`  `rootfs`  `ramfs`  `proc`  `procfs`  `devpts`  `securityfs`  `cgroup`  `cpuset`  `pstore`  `mqueue`  `debugfs`  `autofs`  `hugetlbfs`  `fusectl`  `fuse.gvfsd-fuse`  `binfmt_misc`  `iso9660`  `none`  `rpc_pipefs` |
| Linux | Файловые системы | `tmpfs`  `udf`  `squashfs` |
| Linux | Точка монтирования | `/dev` |
| AIX | Файловые системы | `cdrfs` |
| Solaris | Сетевой интерфейс | `mac` |
