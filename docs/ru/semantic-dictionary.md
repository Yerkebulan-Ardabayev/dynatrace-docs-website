# Семантический Словарь

* Последнее Dynatrace
* Обзор
* Обновлено 23 февраля 2026 г.

Семантический Словарь определяет стандартизированные имена полей, используемые во всех типах данных мониторинга, таких как журналы, события, спаны, метрики и сущности, а также различные доменные модели данных, которые используют эти имена полей. Это является предварительным условием для стабильных и надёжных приложений и автоматизаций.

В Grail данные организованы в записи, каждая из которых состоит из списка полей. Поля в Grail категоризируются на два основных типа: глобальные и доменные.

Глобальные поля поддерживают последовательную семантику во всём продукте Grail, предлагая стандартизированные ссылки на данные. [Справочник глобальных полей](/docs/semantic-dictionary/fields "Ознакомьтесь со списком глобальных полей, имеющих хорошо определённое семантическое значение в Dynatrace и которые можно использовать в различных типах мониторинга.") содержит этот список.

Модели данных представляют собой группу полей, адаптированных к различным случаям использования и доменам. Раздел [Модели данных](/docs/semantic-dictionary/model "Обзор Семантического Словаря, дающий контекст отдельным полям.") содержит список всех моделей данных, используемых в Grail.

Модели данных могут требовать дополнительных полей, уникальных для конкретных моделей данных или проблемных доменов и не могут быть определены глобально. Эти поля не входят в справочник глобальных полей. Вместо этого они определяются только в справочнике модели данных.

### Уровни стабильности

Семантический Словарь также использует уровни стабильности для указания на надёжность и постоянство полей. Каждое задокументированное поле должно быть присвоено одному из следующих уровней:

* экспериментальный: поля, находящиеся в фазе тестирования и подверженные потенциальным изменениям или удалению.
* стабильный: поля, имеющие подтверждённый и надёжный статус, обеспечивающий последовательную основу для управления данными.
* устаревший: поля, которые больше не рекомендуются для использования и могут быть исключены в будущих выпусках.

Эти уровни стабильности помогают пользователям принимать обоснованные решения о выборе и использовании полей, обеспечивая последовательность и долговечность данных в экосистеме Grail.
