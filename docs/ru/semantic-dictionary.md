# Семантический Словарь

* Последнее Dynatrace
* Обзор
* Обновлено 11 февраля 2026 г.

Семантический Словарь определяет стандартизированные имена полей, используемые во всех типах данных мониторинга, таких как журналы, события, спаны, метрики и сущности, а также различные доменные модели данных, которые используют эти имена полей. Это является предварительным условием для стабильных и надежных приложений и автоматизаций.

В Grail данные организованы в записи, каждая из которых состоит из списка полей. Поля в Grail категоризируются на два основных типа: глобальные и доменные.

Глобальные поля поддерживают последовательную семантику во всем продукте Grail, предлагая стандартизированные ссылки на данные. [Справочник глобальных полей](/docs/semantic-dictionary/fields "Ознакомьтесь со списком глобальных полей, имеющих четко определенное семантическое значение в Dynatrace и которые можно использовать в разных типах мониторинга.") содержит этот список.

Модели данных представляют собой группу полей, адаптированных для различных случаев использования и доменов. Раздел [Модели данных](/docs/semantic-dictionary/model "Обзор Семантического Словаря, предоставляющий контекст для отдельных полей.") содержит список всех моделей данных, используемых в Grail.

Модели данных могут требовать дополнительных полей, уникальных для конкретных моделей данных или проблемных доменов и не определяемых глобально. Эти поля не входят в справочник глобальных полей. Вместо этого они определяются только в справочнике модели данных.

### Уровни стабильности

Семантический Словарь также использует уровни стабильности для указания на надежность и постоянство полей. Каждое задокументированное поле должно быть присвоено одному из следующих уровней:

* экспериментальный: поля, находящиеся в стадии тестирования и подлежащие потенциальным изменениям или удалению.
* стабильный: поля, имеющие подтвержденный и надежный статус, обеспечивающий последовательную основу для управления данными.
* устаревший: поля, которые больше не рекомендуются для использования и могут быть исключены в будущих выпусках.

Эти уровни стабильности помогают пользователям принимать обоснованные решения о выборе и использовании полей, обеспечивая последовательность и долговечность данных в экосистеме Grail.
