# Операторы DQL

* Latest Dynatrace
* Справочник
* 22 мин. чтения
* Обновлено 28 окт. 2025

В следующей таблице представлен список всех операторов DQL.

Приоритет операторов следующий (от наивысшего к наименьшему):

* `-` (арифметическое отрицание)
* `*`, `/`, `%`
* `@`
* `+`, `-` (вычитание)
* `~`
* `==`, `!=`, `>`, `>=`, `<`, `<=`
* `in`
* `not`
* `and`
* `xor`
* `or`

## Арифметические операторы

Арифметические операторы можно использовать с числами, представленными типами `long` или `double`. Кроме того, некоторые операторы поддерживают типы `timestamp`, `timeframe`, `duration` или `ip`.

### ДЕЛЕНИЕ

Целочисленное деление

При делении значения типа `long` на другое значение типа `long` с помощью оператора `/` результат также является значением типа `long`, и дробная часть отбрасывается. Чтобы получить результат с дробной частью (значение типа `double`), необходимо преобразовать хотя бы один из операндов в тип `double` (например, с помощью функции [toDouble](/docs/platform/grail/dynatrace-query-language/functions/conversion-and-casting-functions#toDouble "A list of DQL conversion and casting functions.")).

## Операторы равенства

Сравнения на равенство (`==`, `!=`) используют трехзначную булеву алгебру (`true`, `false`, `null`). Это означает, что если любая сторона сравнения на равенство равна `null`, общий результат сравнения также будет `null`.
Существуют четыре функции DQL, которые охватывают сценарии, когда необходимо получить отсутствующие записи или записи со значением `null`:

* [Функция `isTrueOrNull`](/docs/platform/grail/dynatrace-query-language/functions/boolean-functions#isTrueOrNull "A list of DQL boolean functions.")
* [Функция `isFalseOrNull`](/docs/platform/grail/dynatrace-query-language/functions/boolean-functions#isFalseOrNull "A list of DQL boolean functions.")
* [Функция `isNull`](/docs/platform/grail/dynatrace-query-language/functions/boolean-functions#isNull "A list of DQL boolean functions.")
* [Функция `isNotNull`](/docs/platform/grail/dynatrace-query-language/functions/boolean-functions#isNotNull "A list of DQL boolean functions.")

Например, приведенный ниже запрос с базовой фильтрацией не возвращает записи со значениями `null` или отсутствующими значениями:

```
fetch logs



| filter log.source != "logsourcename"  // does not provide the records where `log.source` is null or missing
```

Однако использование функции `isTrueOrNull` включает записи со значениями `null` и отсутствующими значениями:

```
fetch logs



| filter isTrueOrNull(log.source != "logsourcename") // also provides the records where `log.source` is null or missing
```

## Логические операторы

Поведение логических операторов следует трехзначной булевой логике.

* **AND**

  + `true` AND `null` = `null`
  + `null` AND `true` = `null`
  + `false` AND `null` = `false`
  + `null` AND `false` = `false`
  + `null` AND `null` = `null`
* **OR**

  + `true` OR `null` = `true`
  + `null` OR `true` = `true`
  + `false` OR `null` = `null`
  + `null` OR `false` = `null`
  + `null` OR `null` = `null`
* **XOR**

  + `true` XOR `null` = `null`
  + `null` XOR `true` = `null`
  + `false` XOR `null` = `null`
  + `null` XOR `false` = `null`
  + `null` XOR `null` = `null`
* **NOT**

  + NOT `null` = `null`

## Итеративные выражения

Итеративные выражения можно использовать для вычисления каждого элемента данного массива или каждого i-го элемента одного или нескольких массивов.

### iAny

Проверяет итеративное булево выражение и возвращает `true`, если выражение было истинным хотя бы один раз, и `false`, если нет. Например:

```
fetch logs



| fieldsAdd a = array(1, 2, 3)



| filter iAny(a[] > 2)
```

### iCollectArray

Собирает результаты итеративного выражения в массив. Например:

```
fetch logs



| fieldsAdd a = array(1, 2, 3), b = array(10, 11, 12)



| fieldsAdd iCollectArray(a[] + b[])
```

### iIndex

Позволяет получить доступ к индексу элемента итеративного выражения. Например, вы можете добавить индекс значения в массиве и развернуть массив.

```
data record(a = array(2, 3, 7, 7, 1))



| fields a = record(value = a[], index = iIndex())



| expand a



| fields value = a[value], index = a[index]
```

iIndex работает только в выражениях, где присутствует хотя бы одно итеративное выражение.

## Выравнивание по времени

Оператор `@` выравнивает метку времени по указанной единице времени. Он округляет метку времени вниз до начала единицы времени.

#### Синтаксис

`[timestamp|duration|calendarDuration] @ unit`

#### Левая сторона

На левой стороне оператора `@` можно использовать выражение типа `timestamp`, выражение типа `duration` или календарную длительность.
Если вы используете оператор `@` без выражения на левой стороне, оператор будет использовать выражение метки времени `now()` и выровняет текущее время по единице времени. Например, `@h` -- это начало текущего часа, что эквивалентно `now()@h`. Выражения типа `duration` и календарные длительности рассматриваются как смещение относительно `now()`.
Например, `-2M@..` эквивалентно `(now() - 2M)@...`.

#### Правая сторона

Единица времени может быть любой поддерживаемой DQL [единицей длительности](/docs/platform/grail/dynatrace-query-language/data-types#duration "A list of DQL data types."), включая `s` (секунда), `m` (минута), `h` (час), или единицей календарной длительности, такой как `d` (день), `w` (неделя), `M` (месяц), `q` (квартал) и `y` (год).

Единицы длительности (`h`, `m`, `s`, `ms`, `us` и `ns`) позволяют добавлять множитель, например, `@3h`.
Отсутствие множителя эквивалентно установке его в `1`. Обратите внимание на следующие ограничения при добавлении множителя:

* `h` поддерживает все делители `24`: `1h`, `2h`, `3h`, `4h`, `6h`, `8h`, `12h`, `24h`. `@24h` аналогично `@1d`, но может отличаться при переходе на летнее/зимнее время.
* `m` и `s` поддерживают все делители `60`: `1m`, `2m`, `3m`, `4m`, `5m`, `6m`, `10m`, `12m`, `15m`, `20m`, `30m`, `60m` и аналогично для `s`.
* `ms`, `us` и `ns` поддерживают все делители `1000`.

По умолчанию единица недели `w` использует первый день недели, определенный настроенной локалью.
Для явного указания первого дня недели можно использовать следующие единицы времени:

* `w0` (воскресенье)
* `w1` (понедельник)
* `w2` (вторник)
* `w3` (среда)
* `w4` (четверг)
* `w5` (пятница)
* `w6` (суббота)
* `w7` (воскресенье)

Например, `@w1` означает полночь понедельника текущей недели.

##### Примеры

Для следующих примеров текущее время -- среда, 04 сентября 2024, 14:47:05+0200.

## Поиск

Вы можете использовать оператор `~` в выражениях для сопоставления значения выражения с заданной строкой поиска. Выполняемое сравнение нечувствительно к регистру и поддерживает сопоставление с шаблонами с использованием подстановочных символов. Оператор `~` возвращает значение типа `boolean`: `true` в случае совпадения и `false` в противном случае.

#### Синтаксис

`expression ~ "string literal"`

#### Левая сторона

На левой стороне оператора `~` можно использовать любое выражение. Подробнее о поведении с различными типами данных см. в разделе [Возвращаемые значения](#search-returns).

#### Правая сторона

Строковый литерал для поиска. Может быть одним из следующих:

* Строка поиска без подстановочного символа (`*`). Например:

  ```
  content ~ "error"
  ```
* Строка поиска с подстановочным символом (`*`) в качестве первого и/или последнего символа. Например:

  ```
  user ~ "*dynatrace.com"
  ```

#### Возвращаемые значения

##### Строки поиска без подстановочных символов

Оператор `~` выполняет поиск значения как строкового токена внутри строки. Его поведение зависит от типа данных выражения на левой стороне:

* Если выражение имеет тип `string`, оператор выполняет поиск значения как токена внутри строки. Поиск нечувствителен к регистру. Например, `"Hello World"` соответствует `~"world"`, но `"HelloWorld"` -- нет.
* Если выражение имеет тип `long`, `double`, Smartscape ID, IP-адрес или `UID`, оператор совпадает только если строковое представление его значения равно строке поиска. Например, IP-адрес `192.0.0.1` соответствует `~"192.0.0.1"`, но не `~"192"`.
* Если выражение имеет тип `array`, проверяется каждый элемент. Оператор совпадает, если хотя бы один элемент массива совпадает.
* Если выражение имеет тип `record`, оператор совпадает, если совпадает любое имя поля или значение.
* Если выражение имеет тип `boolean`, `timestamp`, `duration` или `binary`, результат всегда false.

##### Строки поиска с подстановочными символами

Оператор выполняет поиск шаблона в токенах строки. Его поведение зависит от типа данных выражения на левой стороне:

* Если выражение имеет тип `string`, результат равен true, если хотя бы один из токенов соответствует шаблону.
* Если выражение имеет тип `array`, результат равен true, если один из элементов массива соответствует шаблону.
* Если выражение имеет тип `record`, результат равен true, если имя или значение вложенного поля соответствует шаблону.
* Если выражение имеет любой другой тип (`long`, `double`, Smartscape ID, IP-адрес, `UID`, `boolean`, `timestamp`, `duration` или `binary`), шаблоны не поддерживаются, и результат всегда `false`.
