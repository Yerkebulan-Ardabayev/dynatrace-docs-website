Последняя версия Dynatrace

Инструмент DPL Architect помогает вам

* Извлекать поля из записей.
* Создавать правильный шаблон данных для экономии времени при разработке DPL-шаблонов.
* Получать мгновенную обратную связь об эффективности и охвате ваших шаблонов для конкретного сценария использования.
* Сохранять и повторно использовать существующие DPL-шаблоны для быстрого доступа к сценариям аналитики данных.
* Использовать предустановленные шаблоны для наиболее популярных технологий.

## Как это работает

DPL Architect обеспечивает мгновенную обратную связь по вашему DPL-выражению без необходимости повторного выполнения DQL-запроса. Это экономит ваше время и усилия при определении нужного DPL-выражения. Обратная связь предоставляется в двух контекстах: базовый набор данных и набор данных предварительного просмотра совпадений.

### Базовый набор данных

Базовый набор данных создается из исходного запроса, выполненного в Notebooks. Тот же запрос выполняется в DPL Architect и сохраняется как базовый набор данных, чтобы показать вам, какая часть соответствует созданному вами шаблону.

![Базовый набор данных](https://dt-cdn.net/images/image-2023-07-09-23-26-30-663-2294-742f00c80e.png)

### Набор данных предварительного просмотра совпадений

Набор данных предварительного просмотра совпадений состоит из записей, отображаемых в DPL Architect. Когда вы открываете DPL Architect, запись, из которой вы начали извлекать дополнительные поля, отображается в редакторе предварительного просмотра совпадений. Вы можете добавить дополнительные строки из базового набора данных, выбрав [**Добавить в предпросмотр**](#visual-feedback), и при необходимости создать дополнительные записи вручную. Часть записи, соответствующая DPL-шаблону, выделяется, чтобы вы могли визуализировать прогресс вашего шаблона.

![Набор данных предварительного просмотра совпадений](https://dt-cdn.net/images/image-2023-07-09-23-29-23-054-2352-31d5669db5.png)

### Ввод шаблона схемы

Используйте редактор выражений для ввода шаблона схемы и начала извлечения полей. Начните вводить текст и используйте подсказки автозаполнения.

* Если шаблон в правильном формате, отображается **Pattern is valid**.
* Если формат шаблона содержит ошибки, **Pattern error: ...** указывает строку и позицию недопустимого элемента.

![Редактор DPL-выражений](https://dt-cdn.net/images/2023-04-21-18-36-49-819-d897a39622.png)

### Просмотр совпадающих записей

[**Базовый набор данных**](#base-dataset) отображает, сколько записей из результатов исходного запроса соответствуют вашему шаблону.

![Совпадающие записи](https://dt-cdn.net/images/2023-04-21-19-19-31-1839-c862944146.png)

### Предварительный просмотр с подсветкой синтаксиса

* [**Редактор предварительного просмотра совпадений**](#match-preview) проверяет, соответствует ли введённый шаблон схемы записи. Часть записи, соответствующая DPL-шаблону, выделяется.

![Предварительный просмотр совпадений](https://dt-cdn.net/images/match-preview-1522-b64f0262ce.png)

* **Add log record** добавляет строку, в которую вы можете ввести пользовательскую запись данных для предварительного просмотра в дополнение к существующим данным для проверки шаблона.

### Просмотр извлечённых полей

**Results** отображает извлечённые поля как новые столбцы, если в вашем шаблоне вы добавите имена к извлечённым полям после двоеточия (например, `IPADDR:client_ip`).

![Результаты запроса](https://dt-cdn.net/images/results-1585-68c93a5fb3.png)

### Получение визуальной обратной связи о качестве шаблона

Выберите **Add to preview** (расположена справа от базового набора данных), чтобы

* Добавить записи из результатов запроса в компонент предварительного просмотра совпадений и получить визуальную обратную связь о качестве шаблона.
* Узнать, какие записи из исходного набора результатов соответствуют вашему шаблону.
* Добавить несовпадающие записи в набор предварительного просмотра для дальнейшего улучшения DPL-шаблона.

### Экспериментирование с несколькими шаблонами

Вы можете одновременно открыть несколько DPL-шаблонов в разных вкладках. Таким образом вы можете

* Повторно использовать части существующих шаблонов
* Пробовать различные сценарии
* Сравнивать производительность шаблонов

При работе с несколькими шаблонами используйте следующие команды:

* Выберите **New**, чтобы создать новый шаблон.
* Выберите **Save**, чтобы сохранить шаблон.
* Выберите **Saved patterns**, чтобы показать или скрыть список сохранённых шаблонов.

Несохранённые шаблоны сохраняются после закрытия DPL Architect и удаляются только при ручном закрытии вкладок.

### Применение шаблона к запросу

* Выберите **Insert pattern**, чтобы добавить шаблон к команде parse в конце исходного запроса.
* Выберите **Close**, чтобы вернуться к разделу запроса, и повторно выполните запрос, чтобы увидеть извлечённые поля в результатах.

## Предустановленные шаблоны

DPL Architect предоставляет множество предустановленных шаблонов для наиболее популярных технологий в отрасли, таких как AWS, Microsoft и Google Cloud.

Эти шаблоны можно

* Использовать сразу для извлечения всех полей из определённого события.
* Настраивать в соответствии с вашими предпочтениями.

### Доступ к предустановленным шаблонам

Для доступа к предустановленным шаблонам выберите **Saved patterns** > **Dynatrace patterns**.
Для быстрого доступа предустановленные шаблоны логически разделены по структуре папок.
