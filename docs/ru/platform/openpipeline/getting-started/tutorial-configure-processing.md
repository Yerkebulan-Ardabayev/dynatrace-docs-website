* Учебное руководство

Управляйте настройкой конечных точек загрузки, динамических маршрутов и конвейеров через OpenPipeline.

## Для кого предназначена эта статья

Эта статья предназначена для администраторов, управляющих конфигурацией загрузки данных, хранением и обогащением данных, а также политиками преобразования.

## Что вы узнаете

В этой статье вы узнаете, как создать новую конфигурацию для записей данных через OpenPipeline — от настройки загрузки до хранения в Grail.

## Прежде чем начать

Необходимые знания

* [Dynatrace Query Language](../../grail/dynatrace-query-language.md "Использование Dynatrace Query Language.")
* [Поток данных в OpenPipeline](../concepts/data-flow.md "Узнайте, как данные проходят через платформу Dynatrace — от загрузки до хранения — через OpenPipeline.")
* [Обработка в OpenPipeline](../concepts/processing.md "Узнайте основные концепции обработки в OpenPipeline.")

Предварительные требования

* Среда Dynatrace SaaS на базе Grail и AppEngine.
* Вы обладаете разрешениями `openpipeline:configurations:write` и `openpipeline:configurations:read`. Информацию о настройке разрешений см. в разделе [Разрешения в Grail](../../grail/organize-data/assign-permissions-in-grail.md "Узнайте, как назначать разрешения для бакетов и таблиц в Grail.").
* Если вы уже используете конвейер обработки логов, убедитесь, что [условия сопоставления преобразованы в DQL](../../../analyze-explore-automate/logs/logs-upgrade/lma-dql-conversion.md "Преобразуйте текущие правила мониторинга логов в DQL.").

Ключевые термины

Источники загрузки
:   Источник загрузки для области конфигурации, собирающий данные от провайдера в платформу Dynatrace, например, конечные точки API или OneAgent.

Маршрутизация
:   Назначение данных конвейеру на основе условий сопоставления (динамическая маршрутизация) или прямой настройки (статическая).

Конвейер
:   Набор инструкций обработки для структурирования, разделения и хранения данных.

Этап
:   Фаза в последовательности конвейера, ориентированная на задачу и определяемая процессорами.

Процессор
:   Предварительно отформатированная инструкция обработки.

### Шаг 1. Настройка конвейера

OpenPipeline хранит данные в бакетах Grail. Если вам нужен бакет с определёнными разрешениями или пользовательским сроком хранения данных, [создайте пользовательский бакет](../../grail/organize-data.md#custom-grail-buckets "Информация о модели данных Grail, состоящей из бакетов, таблиц и представлений.").

В разделе **Pipelines** отображаются встроенные и пользовательские конвейеры для области конфигурации в вашей среде. Чтобы настроить собственные конвейеры для группировки обработки и извлечения по технологии или команде:

1. Перейдите в ![Настройки](https://dt-cdn.net/images/settings-icon-256-38e1321b51.webp "Настройки") **Settings** > **Process and contextualize** > **OpenPipeline** и выберите область конфигурации.
2. Выберите **Pipelines** > ![Добавить](https://dt-cdn.net/images/dashboards-app-menu-plus-7e9b7c3547.svg "Добавить") **Pipeline**, чтобы добавить новый конвейер.
3. Обязательно введите имя конвейера.
4. Для каждого этапа настройте процессоры.

   1. Выберите ![Добавить](https://dt-cdn.net/images/dashboards-app-menu-plus-7e9b7c3547.svg "Добавить") **Processor** и выберите процессор.
   2. Для каждого процессора укажите имя и условие сопоставления. Дополнительные обязательные поля зависят от процессора и указаны в пользовательском интерфейсе.
5. Нажмите **Save**.

Успешно настроенные конвейеры отображаются в списке конвейеров. Выберите ![Дополнительные действия](https://dt-cdn.net/images/dashboards-app-menu-kebab-c39eda426b.svg "Дополнительные действия") > **Edit** для изменения пользовательского конвейера.

### Шаг 2 (необязательный). Настройка пользовательских источников загрузки

В разделе **Ingest sources** отображаются встроенные и пользовательские источники загрузки для области конфигурации в вашей среде. Чтобы настроить собственные источники загрузки, например конечные точки API для сервиса, технологии или команды:

1. Перейдите в ![Настройки](https://dt-cdn.net/images/settings-icon-256-38e1321b51.webp "Настройки") **Settings** > **Process and contextualize** > **OpenPipeline** и выберите область конфигурации.
2. Выберите **Ingest sources** > ![Добавить](https://dt-cdn.net/images/dashboards-app-menu-plus-7e9b7c3547.svg "Добавить") **Source**.
3. Введите имя источника и путь URI, затем выберите вариант маршрутизации.

   * Для статической маршрутизации данных в конвейер укажите имя конвейера, затем продолжите настройку **Ingest sources**.
   * Для динамической маршрутизации данных завершите настройку **Ingest sources**, а затем [укажите условия сопоставления](#route).
4. Необязательно: чтобы настроить хранилище для источника загрузки, раскройте **Advanced options** и выберите существующий бакет Grail.
5. Необязательно: чтобы предварительно обработать данные перед их маршрутизацией в конвейер, перейдите в раздел **Pre-processing** и настройте процессоры. Для каждого процессора укажите имя и условие сопоставления. Дополнительные обязательные поля зависят от процессора и указаны в пользовательском интерфейсе.
6. Нажмите **Save**.

Успешно настроенные пользовательские источники загрузки отображаются в списке источников загрузки. Выберите ![Дополнительные действия](https://dt-cdn.net/images/dashboards-app-menu-kebab-c39eda426b.svg "Дополнительные действия") > **Edit** для изменения пользовательского источника загрузки.

### Шаг 3. Динамическая маршрутизация данных в конвейер

Вы можете маршрутизировать входящие данные в конвейеры на основе значения любого поля или источника загрузки. Для динамической маршрутизации данных:

1. Перейдите в ![Настройки](https://dt-cdn.net/images/settings-icon-256-38e1321b51.webp "Настройки") **Settings** > **Process and contextualize** > **OpenPipeline** и выберите область конфигурации.
2. Выберите **Dynamic routing** > ![Добавить](https://dt-cdn.net/images/dashboards-app-menu-plus-7e9b7c3547.svg "Добавить") **Dynamic route**.
3. Введите условие сопоставления.
4. Нажмите **Save**.

Успешно настроенные маршруты отображаются в списке динамической маршрутизации. Выберите ![Дополнительные действия](https://dt-cdn.net/images/dashboards-app-menu-kebab-c39eda426b.svg "Дополнительные действия") > **Edit** для изменения конфигурации маршрутизации.

## Заключение

Вы настроили источники загрузки, маршрутизацию и обработку записей области конфигурации через OpenPipeline. После того как вы [начнёте загрузку данных](how-to-ingestion.md "Как загружать данные для области конфигурации в OpenPipeline."), ваши данные будут обработаны в соответствии с настройками, сохранены в бакете Grail и доступны для анализа через возможности Grail.
