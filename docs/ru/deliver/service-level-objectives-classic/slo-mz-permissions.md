# Управление SLO с помощью разрешений зон управления


* Классический вариант

Вы можете управлять доступом к SLO в своей среде, устанавливая разрешения на запись и/или чтение для пользователей или групп пользователей на уровне среды или зоны управления.
Подробнее о настройке разрешений см. в разделе Разрешения на основе ролей.

Проверка разрешений основана исключительно на глобальном селекторе сущностей.

## Доступ на чтение в зависимости от уровня разрешений

* На уровне среды пользователь с разрешением **View environment** может просматривать любой SLO. Для управления разрешениями групп можно создавать политики.

  **Пример политики:** `ALLOW settings:objects:read, settings:schemas:read WHERE settings:schemaId = "builtin:monitoring.slo";`
* На уровне зоны управления пользователь с разрешением **View environment** может читать все глобальные SLO и все SLO, принадлежащие соответствующей зоне управления.

  В настоящее время политика не может быть назначена на уровне зоны управления.

## Доступ на запись в зависимости от уровня разрешений

* На уровне среды пользователь с разрешением **Change monitoring settings** может создавать и обновлять любые SLO, включая глобальные SLO без указанных зон управления. Для управления разрешениями групп можно создавать политики.

  **Пример политики:** `ALLOW settings:objects:read, settings:schemas:read, settings:objects:write WHERE settings:schemaId = "builtin:monitoring.slo";`.
* На уровне зоны управления пользователь с разрешением **Change monitoring settings** может создавать и обновлять любые SLO для соответствующей зоны управления, но не может создавать или обновлять глобальные SLO.

## Просмотр и редактирование SLO в зависимости от уровня разрешений

При оценке SLO учитываются как выбранная зона управления, так и зона управления из селектора сущностей.

* На уровне среды:

  + Вы можете просматривать, создавать, обновлять и удалять SLO. Указание селектора сущностей в SLO необязательно.
* На уровне зоны управления:

  + Вы можете просматривать все SLO без селекторов сущностей или с селекторами сущностей, в которых не указана зона управления, однако они будут оцениваться только в рамках выбранной зоны управления.
  + Вы можете просматривать SLO, в селекторе сущностей которых указана одна из ваших зон управления.
  + Вы можете редактировать SLO, в селекторе сущностей которых указана одна из ваших зон управления, если у вас есть разрешение на запись для этой конкретной зоны управления.
  + Вы можете создавать новые SLO, в селекторе сущностей которых указана хотя бы одна зона управления, к которой у вас есть доступ.

## Просмотр и добавление плиток SLO на дашборд в зависимости от уровня разрешений

* Вы можете добавлять SLO, видимые на странице **Service-level objectives**, на свои классические дашборды, выбрав **Pin to dashboard** в разделе **Actions**.
* Вы можете добавлять только что созданные SLO непосредственно на дашборд. Выборка фильтруется на основе SLO, видимых в выбранной зоне управления.
* Глобальные SLO (без селекторов сущностей или с селекторами сущностей, в которых не указана зона управления) можно добавить при выборе значения **All** в фильтре зон управления.
* Фильтрация по плиткам SLO недоступна. В зависимости от наличия доступа к зонам управления SLO в плитке при выбранной зоне управления вы либо получаете доступ к плитке, либо видите результат **N/A**.
* Если на дашборде присутствует SLO с зоной управления, к которой у вас нет доступа, на плитке отображается **Access denied**.

Начиная с Dynatrace версии 1.233+, для отдельной плитки также можно задать пользовательскую зону управления, аналогично временному периоду. В этом случае глобальный фильтр зон управления не применяется к соответствующей плитке.
