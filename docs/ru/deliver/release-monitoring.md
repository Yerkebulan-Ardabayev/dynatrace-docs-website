# Release monitoring Classic


Жизненный цикл программного продукта при выпуске релиза требует тщательного управления рисками, связанными с релизом. Кроме того, по мере развертывания все большего числа компонентов и версий частота релизов в вашей организации возрастает, и ручной сбор данных, относящихся к релизам, может легко стать узким местом в вашем конвейере автоматизации релизов и автоматизированном жизненном цикле программного обеспечения.

Dynatrace предлагает встроенное решение для анализа релизов, которое помогает определить:

* Какие версии развернуты на ваших стадиях развертывания и в производственных средах на основе нескольких стратегий определения версий.
* Стадии релиза развернутых версий.
* Журнал изменений (changelog) для новой версии.
* Известные ошибки и являются ли они блокирующими для релиза.
* Риски, связанные с конкретными версиями.
* Какая версия испытывает чрезмерную нагрузку (например, если вы временно перенаправляете нагрузку с помощью канареечного развертывания).
* Как новая версия ведет себя по сравнению с предыдущими версиями.
* Статистику проблем, связанных с отслеживаемыми сущностями.

## Настройка

* Узнайте, как настроить переменные окружения для определения версий.
* При необходимости вы можете интегрировать системы отслеживания задач и настроить динамические запросы.

## Анализ

После настройки вашего программного обеспечения/системы отслеживания задач вы можете анализировать жизненный цикл программного продукта ваших релизов.
