# Команды Terraform CLI


Terraform CLI предлагает широкий набор команд для управления вашими ресурсами. В этом разделе мы приводим общее описание основных команд. Поскольку эти команды не являются специфичными для Dynatrace Terraform провайдера, подробную информацию можно найти в [документации Terraform](https://developer.hashicorp.com/terraform/docs).

## Основные команды Terraform

Вы можете использовать следующие встроенные команды с Dynatrace Terraform провайдером:

* `init`: Подготавливает рабочий каталог для других команд.
* `validate`: Проверяет корректность конфигурации.
* `plan`: Показывает изменения, необходимые для текущей конфигурации.
* `apply`: Создает или обновляет инфраструктуру.
* `destroy`: Уничтожает ранее созданную инфраструктуру.

Кроме того, Dynatrace реализует следующую команду специально для Dynatrace Terraform провайдера:

* `export`: Экспортирует существующие ресурсы Dynatrace из заданного окружения Dynatrace.
  [Примеры Terraform CLI](terraform-cli-commands.md#terraform-cli-examples "This is a list of Terraform CLI commands.") содержат примеры использования команды `export`.

## Экспорт конфигурации из окружения Dynatrace с помощью Dynatrace Terraform Provider

В дополнение к встроенной функциональности Terraform, Dynatrace Terraform Provider может быть запущен как отдельный исполняемый файл для экспорта существующей конфигурации из окружения Dynatrace. Эта функциональность предоставляет альтернативу ручному созданию конфигурации Terraform и удобный способ создания шаблонов на основе существующей конфигурации.

При экспорте ресурсов учитывайте соответствующие области разрешений для получения нужных элементов конфигурации.

### Предварительные требования

* Установите Terraform CLI и настройте Configuration as Code через Terraform, а также убедитесь, что он доступен в вашем `PATH`.
* Создайте токен платформы или OAuth-клиент с необходимыми правами доступа.
  Необходимые разрешения зависят от того, какие API вы используете.
  Для получения дополнительной информации ознакомьтесь с документацией по API или справочником политик IAM.

### Использование утилиты экспорта

1. Определите переменные окружения для идентификации тенанта Dynatrace, из которого будет извлечена конфигурация, в соответствии с разделом Поддержка API и управление правами доступа.

   При необходимости задайте переменную окружения `DYNATRACE_TARGET_FOLDER` для указания выходного каталога (папки экспорта). Если переменная не задана, используется каталог по умолчанию `.configuration`.
2. Перейдите к исполняемому файлу Dynatrace Terraform Provider в терминале. Этот исполняемый файл не является универсальным, таким как `terraform.exe` или `./terraform`.

   Обычно его можно найти в каталоге `.terraform/providers/registry.terraform.io/dynatrace-oss/dynatrace/{provider_version}/{os_version}/terraform-provider-dynatrace_x.y.z/`.
3. Запустите исполняемый файл напрямую с нужными параметрами. Экспорт является дополнительной функциональностью; прямой вызов плагина допустим, и предупреждение можно игнорировать.

   Windows

   Linux

   ```
   terraform-provider-dynatrace.exe -export [-ref] [-migrate] [-import-state] [-id] [-flat] [-exclude] [<resourcename>[=<id>]]'
   ```

   ```
   ./terraform-provider-dynatrace -export [-ref] [-migrate] [-import-state] [-id] [-flat] [-exclude] [<resourcename>[=<id>]]'
   ```

### Команды

Команда `list-exclusions` предоставляет обзор ресурсов, которые не будут экспортированы, если они не указаны явно.
Эта команда не выполняет экспорт.
Она не предназначена для использования совместно с другими параметрами.

Windows

Linux

```
terraform-provider-dynatrace.exe -export -list-exclusions
```

```
./terraform-provider-dynatrace -export -list-exclusions
```

По умолчанию ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** (`dynatrace_json_dashboard`) и другие ресурсы исключаются из экспорта, если ресурс не указан напрямую. Используйте команду `-list-exclusions` для получения полного списка исключенных ресурсов.

### Параметры и флаги

* `-ref`: Включает ресурсы с источниками данных и зависимостями.
* `-migrate`: Включает ресурсы с зависимостями, исключая источники данных.
* `-import-state`: Инициализирует модули Terraform и импортирует ресурсы в состояние.
* `-id`: Отображает закомментированный вывод идентификатора в файлах ресурсов.
* `-flat`: Сохраняет все ресурсы непосредственно в целевой папке без модульной структуры.
* `-exclude`: Исключает указанные ресурсы из экспорта.

## Дополнительная информация

Во время экспорта Terraform возможны ситуации, когда некоторые файлы не могут быть обработаны корректно.
Такие файлы перемещаются в следующие каталоги в папке экспорта, см. [Использование утилиты экспорта](terraform-cli-commands.md#export "This is a list of Terraform CLI commands.").
Причины добавляются в виде комментариев в начале каждого файла.

* `.flawed`: Файлы в этом каталоге либо устарели, либо требуют модификации.
* `.required_attention`: Файлам в этом каталоге не хватает существенной информации для применения в Terraform.
  Это может быть связано с конфиденциальными данными или случаями, когда файлы требуют дополнительного внимания.
  (Например, конфиденциальные строки `dynatrace_credentials` недоступны через API.)
