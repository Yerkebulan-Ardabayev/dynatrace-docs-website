# Концепции Monaco

* Актуальная версия Dynatrace
* Объяснение
* Время чтения: 1 мин
* Обновлено 07 октября 2025 г.

Monaco был разработан в рамках Dynatrace для обеспечения надежной и безопасной настройки сред Dynatrace в масштабе.
Он представляет собой легковесный инструмент командной строки (CLI), который использует нативный JSON для описания конфигурационных ресурсов Dynatrace.

Monaco — это проприетарное решение Dynatrace, которое может работать автономно; следовательно, для его использования не требуется никаких дополнительных сторонних инструментов.

## Преимущества использования Dynatrace Monaco

* Не требуется ручное редактирование в продакшене.
* Масштабирование конфигураций по командам и стадиям.
* Поддержание синхронизации конфигураций или сред.
* Воссоздание конфигураций (например, для целей аварийного восстановления).

## Концепции Monaco

Манифест развертывания
:   Манифест развертывания определяет проекты и папку с конфигурационными файлами, которые вы развертываете в указанной целевой среде.
    Манифесты развертывания — это YAML-файлы, которые сообщают CLI Dynatrace Monaco, какие проекты развертывать и куда именно они должны быть развернуты.
    Чтобы CLI Dynatrace Monaco знал, что развертывать, должен присутствовать файл манифеста.
    Этот файл содержит сведения о том, что и куда развертывать.
    Манифест сохраняется в виде YAML-файла.

Проекты
:   Проекты Monaco — это директории (папки), которые логически группируют конфигурации API. Примером проекта может служить сервис, для которого все конфигурации находятся в соответствующей папке. Проекты могут состоять из нескольких файлов и директорий. Дополнительную информацию см. в разделе [Управление проектом Monaco](configuration/projects.md "Manage a project folder with Dynatrace Configuration as Code via Monaco.").

    Директории проектов группируют конфигурационные файлы, которые развертываются в одну и ту же целевую среду.

Конфигурация ресурсов Dynatrace
:   Конфигурация ресурсов Dynatrace состоит из нескольких элементов.

    * YAML-файл с определяющими параметрами, зависимостями, именем и связанным JSON-шаблоном.
    * JSON-файл шаблона содержит необходимую полезную нагрузку (payload) API, включая заполнители для параметров из YAML-файла.

    Конфигурационный YAML-файл содержит основную информацию о развертываемой конфигурации.

    Основная информация включает имя конфигурации, расположение файла шаблона и параметры, которые можно использовать в файле шаблона.
    Параметры могут быть переопределены в зависимости от того, какая группа или среда развертывается в данный момент.

    JSON-файл шаблона содержит полезную нагрузку (payload), которая будет загружена в конечные точки API Dynatrace. Он позволяет ссылаться на все определенные параметры конфигурации с помощью синтаксиса `{{ .PARAMETER_NAME }}`.
