# Работа с командами Monaco CLI для Latest Dynatrace


* Latest Dynatrace

Monaco CLI предлагает несколько встроенных команд для управления файлами конфигурации Dynatrace.

* `deploy`: развертывание конфигураций в среде Dynatrace.
* `download`: загрузка конфигураций из среды Dynatrace.
* `delete`: удаление конфигураций из среды Dynatrace и исключение их из последующих развертываний.
* `generate`: генерация шаблонов файлов на основе заданной конфигурации.
* `account`: операции, связанные с ресурсами управления учетными записями.
* Глобальные флаги: модификация команд.

## Deploy

Команда `deploy` развертывает конфигурации в средах, определенных в выбранном файле манифеста развертывания.

### Использование

```
monaco deploy [ARGS] [OPTIONS]
```

Простейший способ использования deploy -- запустить команду без каких-либо флагов и передать имя файла манифеста развертывания.
При этом все конфигурации из раздела проекта файла манифеста развертывания применяются ко всем средам, определенным в разделе сред файла.

#### Указание прокси-сервера

В средах, где доступ к конечным точкам API Dynatrace возможен только через прокси-сервер, Monaco CLI предоставляет возможность указать адрес вашего прокси-сервера при выполнении команды:

Для этого установите переменную окружения `HTTPS_PROXY` в адрес вашего прокси-сервера.

```
# Linux or Mac OS


HTTPS_PROXY=localhost:5000


monaco deploy example.yaml


# Windows


$env:HTTPS_PROXY="localhost:5000"


monaco deploy example.yaml
```

## Download

Команда `download` позволяет загружать конфигурации из среды Dynatrace в виде файлов Monaco CLI.

Эта команда позволяет:

* Создавать резервные копии существующих конфигураций Dynatrace.
* Переносить (загружать) конфигурации Dynatrace из одной среды в другую.
* Создавать шаблоны файлов конфигурации Dynatrace на основе существующих (созданных через UI) конфигураций.

Monaco может выполнять:

* Загрузку через локальный YAML-файл манифеста, в котором указана среда.
* Загрузку напрямую из URL-адреса среды Dynatrace.

### Использование

```
monaco download [ARGS] [OPTIONS]
```

### Параметры

Доступные параметры зависят от метода загрузки.

#### Аргументы для загрузки через манифест

Эти аргументы используются при загрузке через YAML-файл манифеста.

#### Аргументы для загрузки через URL

Эти аргументы используются при загрузке через URL-адрес среды Dynatrace.

#### Параметры

Эти параметры можно использовать с любым методом загрузки (через манифест или URL).

Вы можете комбинировать несколько флагов `--only-<type>` в одной команде `monaco download` для одновременной загрузки нескольких типов.

### Дополнительная настройка через переменные окружения

Вы можете использовать переменные окружения для установки дополнительных параметров конфигурации.

#### Фильтрация загружаемых файлов

По умолчанию определенные типы конфигураций или объекты конфигурации не включаются в загрузку.
Это связано с тем, что Dynatrace API не возвращает полную информацию для этих типов конфигураций.

* Как правило, эти типы содержат секреты, которые не должны экспортироваться, например:

  + `aws-credentials`
  + `azure-credentials`
  + `kubernetes-credentials`
  + `credential-vault`
  + `extension`
* Конфигурации, доступные только для чтения, также исключаются.

Начиная с Monaco CLI версии 2.2.0+ вы можете отключить эту фильтрацию.
Для этого установите одну из переменных окружения, описанных в таблице ниже.
Затем используйте `monaco download` для загрузки конфигурации настроек.

| Переменная окружения | Описание |
| --- | --- |
| `MONACO_FEAT_DOWNLOAD_FILTER` | Управляет всей фильтрацией загрузки. Если установлена в `false`, фильтры не применяются, и все настройки загружаются. Имеет приоритет над всеми другими переменными окружения. |
| `MONACO_FEAT_DOWNLOAD_FILTER_SETTINGS` | Управляет фильтрацией загрузки Settings 2.0. Если установлена в `false`, все настройки Settings 2.0 загружаются без фильтрации. Имеет приоритет над переменными окружения `MONACO_FEAT_DOWNLOAD_FILTER_SETTINGS_UNMODIFIABLE`. |
| `MONACO_FEAT_DOWNLOAD_FILTER_SETTINGS_UNMODIFIABLE` | Управляет фильтрацией загрузки Settings 2.0. Если установлена в `false`, настройки только для чтения также загружаются. |
| `MONACO_FEAT_DOWNLOAD_FILTER_CLASSIC_CONFIGS` | Управляет фильтрацией загрузки типов классического Configuration API. Если установлена в `false`, все конфигурации классического Configuration API загружаются без фильтрации. |

Хотя это позволяет загрузить проект, автоматическое развертывание будет невозможно.
Потребуется некоторая ручная постобработка.

```
# Linux or macOS


MONACO_FEAT_DOWNLOAD_FILTER=false


# MONACO_FEAT_DOWNLOAD_FILTER_SETTINGS=false


# MONACO_FEAT_DOWNLOAD_FILTER_SETTINGS_UNMODIFIABLE=false


# MONACO_FEAT_DOWNLOAD_FILTER_CLASSIC_CONFIGS=false


monaco download


# Windows


$env:MONACO_FEAT_DOWNLOAD_FILTER="false"


# $env:MONACO_FEAT_DOWNLOAD_FILTER_SETTINGS="false"


# $env:MONACO_FEAT_DOWNLOAD_FILTER_SETTINGS_UNMODIFIABLE="false"


# $env:MONACO_FEAT_DOWNLOAD_FILTER_CLASSIC_CONFIGS="false"


monaco download
```

#### Параллельная загрузка

При загрузке конфигурации Monaco выполняет несколько одновременных API-запросов для максимально быстрой загрузки больших наборов конфигураций.
По умолчанию выполняется до пяти одновременных запросов.

Если вам нужна более быстрая загрузка и ваша среда или сеть это позволяют, вы можете настроить Monaco для выполнения дополнительных одновременных запросов.
Для этого используйте переменную окружения `MONACO_CONCURRENT_REQUESTS`, установив её значение равным желаемому количеству одновременных запросов.
Например, `MONACO_CONCURRENT_REQUESTS=15` разрешает до 15 одновременных запросов.

```
# Linux or Mac OS


MONACO_CONCURRENT_REQUESTS=15


monaco download


# Windows


$env:MONACO_CONCURRENT_REQUESTS="15"


monaco download
```

Если вы заметили проблемы с загрузкой конфигураций, например, если внутренняя сеть ограничивает и отклоняет запросы, уменьшите количество одновременных запросов.

#### Разрешение зависимостей

Конфигурация Dynatrace может содержать зависимости от других конфигураций.
Monaco разрешает эти зависимости автоматически:

* Стандартный резолвер запускается автоматически как часть `monaco download`.
* Быстрый резолвер доступен начиная с Monaco CLI версии 2.0.2+.
  Он ускоряет выполнение `monaco download`, но требует больше ресурсов: необходимо 16+ ГБ оперативной памяти и несколько сотен ГБ хранилища, доступного в качестве файла подкачки.

Для активации быстрого резолвера установите переменную окружения `MONACO_FEAT_FAST_DEPENDENCY_RESOLVER` в значение `true`.

```
# Linux or Mac OS


MONACO_FEAT_FAST_DEPENDENCY_RESOLVER=true


monaco download


# Windows


$env:MONACO_FEAT_FAST_DEPENDENCY_RESOLVER="true"


monaco download
```

## Delete

Команда `delete` -- это удобный способ удаления конфигураций из сред Dynatrace.

* Удаление долгоживущих конфигураций в производственных средах не идеально, но иногда необходимо.
* Очистка временных конфигураций в средах разработки.

В качестве входных данных команда `delete` требует два YAML-файла:

* Файл манифеста, определяющий соответствующие среды Dynatrace.
* Файл удаления, определяющий соответствующие конфигурации.

Файл удаления генерируется с помощью `monaco generate deletefile`, см. [Generate](commands-saas.md#generate-deletefile "Как использовать приложение Monaco CLI, включая аргументы и параметры.").

### Использование

```
monaco delete [--manifest <file>] [--file <file>] [OPTIONS]
```

## Generate

Команда `monaco generate` позволяет генерировать различные вспомогательные файлы на основе вашей конфигурации:

* Файл удаления
* DOT-граф
* Файлы JSON-схем

### Использование

```
monaco generate [ARGS] [OPTIONS]
```

### Аргументы

### Параметры

Доступные параметры зависят от конкретного используемого аргумента.

#### deletefile

Эти параметры используются с командой `monaco generate deletefile`.

Команда `generate deletefile` не может обрабатывать конфигурации, содержащие ссылки в поле имени, и завершится ошибкой при обнаружении таких конфигураций.

**Пример проблемной конфигурации:**

```
configs:


- id: appRule


config:


name:


configId: application


configType: application-web


property: name


type: reference


template: rule.json


skip: false


type:


api: app-detection-rule
```

**Обходное решение:** Создайте файл удаления вручную или обновите сгенерированный файл, указав имя конфигурации без ссылок. См. формат файла удаления для правильного синтаксиса.

#### graph

Эти параметры используются с командой `monaco generate graph`.

#### schema

Эти параметры используются с командой `monaco generate schema`.

Команда `account` используется для операций, затрагивающих ресурсы управления учетными записями.

### Использование

```
monaco account [ARGS] [OPTIONS]
```

### Аргументы

### Параметры

Доступные параметры зависят от конкретного используемого аргумента.

#### deploy

Эти параметры используются с командой `monaco account deploy`.

#### download

Эти параметры используются с командой `monaco account download`.

##### Параметры для загрузки через манифест

Эти параметры используются при загрузке через YAML-файл манифеста.

##### Параметры для загрузки через UUID

Эти параметры используются при загрузке через UUID учетной записи Dynatrace.

##### Дополнительные параметры

Эти параметры можно использовать с любым методом загрузки (через манифест или UUID).

#### delete

Эти параметры используются с командой `monaco account delete`.

##### Пример

В этом примере мы удаляем пользователя, сервисного пользователя, группу, политику и границу, используя соответствующие записи в файле `delete.yaml`.

```
delete:


- type: user


email: the.user@dynatrace.com


- type: serviceUser


name: Monaco service user


- type: group


name: My Group


- type: policy


name: My Policy


level:


type: account


- type: boundary


name: My Boundary
```

## Глобальные параметры

Эти параметры можно использовать с любым аргументом `monaco`.
Они полезны для устранения ошибок, с которыми вы можете столкнуться.
