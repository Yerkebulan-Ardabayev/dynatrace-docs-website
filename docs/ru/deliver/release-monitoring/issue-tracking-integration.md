# Интеграция с системой отслеживания задач

* Classic
* How-to guide
* 2-min read
* Published Sep 14, 2020

Чтобы получать статистику по проблемам релизов для отслеживаемых объектов и настраивать динамические запросы, вы можете интегрировать вашу систему отслеживания задач с Dynatrace.

## Поддерживаемые интеграции

В настоящее время Dynatrace поддерживает интеграцию с

* Jira on-premises
* Jira cloud
* GitHub
* GitLab
* ServiceNow

## Интеграция вашей системы отслеживания задач

Чтобы интегрировать систему отслеживания задач

1. Перейдите в **Settings Classic** > **Cloud Automation** > **Issue-tracking for releases**.
2. Выберите **Add issue-tracking query**.
3. Введите необходимую информацию (**Issue label**, [**Issue query**](#query), **Issue type**, **Issue-tracking system**, **Target URL**[1](#fn-1-1-def), [**Username** и **Password** или **Token**](#credentials)).
4. При наличии ошибок конфигурации в нижней части страницы отобразится сообщение об ошибке (`Please resolve errors before saving...`). Выберите **Show errors**, чтобы просмотреть ошибки конфигурации (выделены красным цветом).
5. Необязательно. Выберите **Check configuration**, чтобы проверить связь между Dynatrace и системой отслеживания задач.
6. Выберите **Save changes**, чтобы сохранить конфигурацию.

1

Для GitLab, чтобы определить запросы к нескольким проектам, можно ввести конечную точку API `/groups`.

### Запрос задач

В поле **Issue query** можно указывать запросы с заполнителями, которые подставляются во время выполнения (для динамической фильтрации).
Примеры:

* **Jira on-premises/Jira cloud:** `issueType = Bug and component in ({PRODUCT}) and affectedVersion in ({VERSION})`
* **GitHub:** `is:issue is:open label:{PRODUCT} label:{VERSION}`
* **GitLab:** `search={PRODUCT} {VERSION}&state=opened&scope=issues`

#### Исключение

Для **ServiceNow** заполнители не поддерживаются. Вы можете запрашивать инциденты по значениям атрибутов инцидента в формате `<col_name><operator><value>`.
Пример: `correlation_displayLIKEDYNATRACE^active=true`. В этом случае фильтрация будет применяться к записям, содержащим `DYNATRACE` в столбце `correlation_display`, которые в данный момент активны.
Для других операторов обратитесь к [документации ServiceNow APIï»¿](https://dt-url.net/0w03qc9).

Поддерживается любой запрос, который можно записать в виде параметра запроса `sysparm_query`.

### Учётные данные

Поля **Username**, **Password** и **Token** заполняются следующим образом:

* Для **GitHub** введите имя пользователя и OAuthToken
* Для **GitLab** введите API-токен с правами на чтение
* Для **Jira on-premises** введите имя пользователя и пароль
* Для **Jira cloud** введите имя пользователя и OAuthToken
* Для **ServiceNow** введите имя пользователя и пароль

После добавления трекера задач в Dynatrace вы сможете видеть статистику задач, связанную с отслеживаемыми объектами, в разделе **Release inventory** на странице **Releases**. Например, если инвентарь релизов показывает записи для приложения **Cassandra** с версией `3.11`, интеграция с системой отслеживания задач предоставит количество ошибок для Cassandra версии 3.11.
