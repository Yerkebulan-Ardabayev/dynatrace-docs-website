# Мониторинг релизов с помощью Dynatrace


* Classic
* Практическое руководство
* Чтение: 4 мин
* Обновлено 11 августа 2025 г.

После того как вы [настроите переменные окружения для определения версий](version-detection-strategies.md "Metadata for version detection in different technologies") и, при необходимости, [интегрируете свою систему отслеживания задач и настроите динамические запросы](issue-tracking-integration.md "Integrate your issue tracker into Dynatrace to pull statistics for monitored entities."), вы можете приступить к анализу данных, связанных с каждой версией релиза вашего программного обеспечения.

## Список релизов

Чтобы просмотреть список релизов программного обеспечения в вашей среде, перейдите в раздел **Releases**.

Пример страницы Releases

![Список релизов](https://dt-cdn.net/images/2021-05-25-12-02-34-1641-8ea61fbf08.png)

Сортировка по **Release version** использует лексикографическую (строковую) сортировку, а не сортировку по семантическим версиям. При лексикографической сортировке числа рассматриваются как отдельные символы, и порядок определяется на основе значений этих символов в таблице ASCII или Unicode. Сортировка по **Release version** использует лексикографический порядок, поскольку для версий релизов не существует единого шаблона, и пользователи могут вводить любое значение.

Страница **Releases** отображает следующую информацию о перечисленных релизах:

### Инвентаризация релизов

Показывает все обнаруженные релизы. Каждая запись представляет процессы и экземпляры групп процессов, сгруппированные по версиям релиза и сборки, стадии и продукту, к которому они логически относятся.

### События релизов

Показывает все события, соответствующие релизам, такие как перезапуск процессов и события развертывания.

### Отслеживаемые задачи

Показывает задачи из внешних систем отслеживания задач, связанные с релизами.

## Фильтры

* Для установки простых фильтров используйте быстрые фильтры в левой части страницы.
* Для установки более детальных фильтров используйте панель фильтрации над таблицей.
  Например, вы можете использовать фильтр `tag` для поиска по тегу экземпляра группы процессов.
* Чтобы увидеть только релизы, затронутые проблемами, выберите **Apply filter** над таблицей **Release inventory**.

  + Фильтрация по конкретному релизу показывает обзор стадий и контекстов продуктов, в которых развернут данный релиз.
  + Фильтрация по стадии показывает обзор всех релизов, развернутых на данной стадии.

## Просмотр деталей релиза

Чтобы увидеть подробности о релизе, выберите ссылку в столбце **Name** на странице **Releases**.

Пример страницы деталей релиза

![Детали релиза](https://dt-cdn.net/images/2021-05-25-12-41-08-1636-73e6cdccd9.png)

Страница **Release details** отображает следующую информацию о выбранном релизе:

### Название релиза

Показывает сводку собранных метаданных (название группы процессов, версия, стадия, продукт, технологии, экземпляры, пропускная способность, проблемы и уязвимости сторонних компонентов) и ссылки на группу процессов, в которой развернут релиз.

* **Instances** показывает количество развертываний данного релиза на определенной стадии в контексте конкретного продукта.
* **Throughput** показывает, какой объем трафика направляется на выбранный релиз.
* **Problems** показывает количество открытых проблем, затрагивающих экземпляр группы процессов данного релиза.

  Если Dynatrace обнаруживает проблемы в релизе, **Problems** отображается красным цветом. Выберите его, чтобы перейти на страницу **Problems** и получить дополнительную информацию. Подробнее см. в разделе [Обзор проблем](../../dynatrace-intelligence/root-cause-analysis/concepts.md "Get acquainted with root cause analysis concepts.").
* **Third-party vulnerabilities** показывает количество уязвимостей сторонних компонентов, связанных с выбранным релизом. Это помогает проверить, например, не вносит ли новый релиз известную уязвимость, и на основе этого принять решение о её влиянии на ваш график релизов.

  Если Dynatrace обнаруживает уязвимости в релизе, **Third-party vulnerabilities** отображается красным цветом. Выберите его, чтобы перейти на страницу **Security** и получить дополнительную информацию. Подробнее см. в разделе [Мониторинг безопасности приложений](../../secure/application-security.md "Access the Dynatrace Application Security functionalities.").

### Детали процесса

Показывает хосты с базовыми метриками, на которых развернут релиз, и ссылки на страницу хоста для дальнейшего анализа.

### События релизов

Показывает все события, соответствующие выбранному релизу, такие как перезапуск процессов и события развертывания.

### Отслеживаемые задачи

Показывает задачи из внешних систем отслеживания задач, связанные с выбранным релизом.

### Валидация релиза

Для оптимизации таблицы инвентаризации релизов, если экземпляр Cloud Automation не подготовлен, столбец для отображения результата валидации релиза не показывается.

Отображает список валидаций релизов, обнаруженных в пределах глобального временного интервала.

* Вы можете сортировать список по оценке и времени завершения проверки.
* Выберите любую валидацию релиза в списке, чтобы перейти к тепловой карте соответствующей проверки в вашем экземпляре Cloud Automation.

Кроме того, график отображает последние 20 результатов валидации релизов:

* Ось Y отображает оценку
* Ось X отображает время завершения каждой проверки quality gate
* Цвет столбца указывает на статус quality gate (`Failed`, `Warning` или `Passed`)
