# Мониторинг релизов с помощью Dynatrace

После настройки переменных окружения и интеграции систем отслеживания задач можно анализировать данные релизов.

## Список релизов

Перейдите в **Releases**. Страница содержит:

* **Инвентаризация релизов** — все обнаруженные релизы (процессы, сгруппированные по версии, стадии и продукту).
* **События релизов** — перезапуски процессов и события развертывания.
* **Отслеживаемые задачи** — задачи из внешних систем.

Сортировка по **Release version** — лексикографическая (строковая).

## Фильтры

* Быстрые фильтры — в левой части страницы.
* Детальные фильтры — панель над таблицей (например, фильтр `tag`).
* Фильтр по проблемам — **Apply filter** над таблицей.

## Детали релиза

Выберите ссылку в столбце **Name**. Страница отображает:

### Название релиза

Сводка метаданных: версия, стадия, продукт, технологии, экземпляры, пропускная способность, проблемы, уязвимости.

* **Instances** — количество развертываний на стадии
* **Throughput** — объём трафика на версию
* **Problems** — открытые проблемы (красный при наличии)
* **Third-party vulnerabilities** — уязвимости (красный при наличии)

### Детали процесса

Хосты с метриками и ссылки на страницу хоста.

### События релизов

События, соответствующие выбранному релизу.

### Отслеживаемые задачи

Задачи из внешних систем для выбранного релиза.

### Валидация релиза

Список валидаций в пределах глобального временного интервала. График последних 20 результатов: ось Y — оценка, ось X — время, цвет — статус (`Failed`, `Warning`, `Passed`).
