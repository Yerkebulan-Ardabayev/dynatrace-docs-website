* 8 мин. чтения

Владение мониторируемыми сущностями определяется в рамках **команд**. Вы можете настраивать команды владения в веб-интерфейсе, через API и с помощью [Configuration as Code](../configuration-as-code.md "Используйте Dynatrace Configuration as Code через Monaco или Terraform."). Каждая команда имеет **уникальный идентификатор**, который является основой для [назначения владения сущностям](assign-ownership.md "Назначайте владельцев сущностям, используя метаданные сущностей, такие как метки, переменные окружения и теги.") различными методами (метки и аннотации Kubernetes, метаданные хоста, переменные окружения или теги). Кроме того, определения команд включают имена, описания, важную маршрутную информацию для уведомлений (через Microsoft Teams, Slack, Jira и электронную почту), зоны ответственности (например, эксплуатация или безопасность) и дополнительные полезные ссылки.

Настройки команд основаны на [фреймворке Settings 2.0](../../manage/settings/settings-20.md "Введение в фреймворк Settings 2.0"), который определяет каждую конфигурацию команды как объект настроек на основе схемы владения, независимо от того, используете ли вы API, Configuration as Code, импортированные данные из сторонних баз данных или веб-интерфейс в разделе **Settings** > **Ownership**.

Информация о владении, прикреплённая к мониторируемым сущностям, помогает сопоставлять вашу среду Dynatrace с нужными командами для совместной работы, решения проблем и эскалации уязвимостей.

## Разрешения

Для создания, редактирования или удаления команды через API вам необходимо **любое** из следующих разрешений:

* Разрешение токена `settings.write` [разрешение токена](../../manage/identity-access-management/access-tokens-and-oauth-clients/access-tokens.md "Узнайте о концепции токена доступа и его областях действия.")
* [Политика IAM](../../manage/identity-access-management/permission-management/manage-user-permissions-policies.md "Работа с политиками") `ALLOW settings:objects:write, settings:schemas:read WHERE settings:schemaId = "builtin:ownership.teams";`

Для создания команд через веб-интерфейс вам необходимо разрешение **Manage monitoring settings** [разрешение](../../manage/identity-access-management/permission-management/role-based-permissions.md "Разрешения на основе ролей") на уровне среды или зоны управления.

## Создание команды через веб-интерфейс

Чтобы создать команду через веб-интерфейс Dynatrace

1. Перейдите в **Settings** > **Ownership** > **Teams**.

   ![Страница настроек команд владения](https://dt-cdn.net/images/ownership-teams-page-2212-be3abe3c7d.png)
2. Выберите **Add team**.
3. Обязательно Начните с указания **Team name** (имени команды).

В данных команд владения, [импортированных из Microsoft Entra ID](../ownership-app.md#import-teams "Предоставляет настраиваемые действия для определения рабочих процессов, интегрирующих владельцев сущностей и их контактную информацию."), параметр `displayName` из `get_groups` устанавливается как **Team name**.

4. Обязательно **Team identifier** (идентификатор команды) автоматически заполняется на основе имени команды. Используйте идентификатор по умолчанию или укажите свой.

   **Требования**

   Идентификаторы команд являются основой для [назначения владения сущностям](assign-ownership.md "Назначайте владельцев сущностям, используя метаданные сущностей, такие как метки, переменные окружения и теги."), независимо от метода. Основные идентификаторы команд должны быть уникальными в пределах вашей среды. Однако вы можете повторно использовать идентификатор ранее удалённой команды.

   * Идентификаторы должны быть уникальными; два основных идентификатора команд не могут совпадать.
   * Идентификаторы должны содержать от 1 до 100 символов.
   * Они должны начинаться и заканчиваться буквой (`[a-zA-Z]`).
   * Между начальным и конечным символами допускаются дефисы (`-`), символы подчёркивания (`_`) и буквенно-цифровые символы. Пробелы и другие специальные символы не поддерживаются.
   * После создания основной **идентификатор команды не может быть изменён**.

   В информации о команде, [импортированной из Microsoft Entra ID](../ownership-app.md#import-teams "Предоставляет настраиваемые действия для определения рабочих процессов, интегрирующих владельцев сущностей и их контактную информацию."), это поле устанавливается равным **Object Id** соответствующей группы в Entra ID и не может быть изменено. **External ID** импортированной команды владения в Dynatrace также устанавливается равным этому значению.
5. Добавьте **Description** (описание), которое помогает отличить команду, например, по управляемым сущностям или по зоне ответственности. Это описание добавляется к имени команды для отображения на странице **Ownership teams**.
6. При необходимости укажите **Supplementary identifiers** (дополнительные идентификаторы). Выберите **Add supplementary identifier**, чтобы указать до трёх дополнительных идентификаторов, например, для обозначения подкоманд. При назначении владения сущностям вы можете использовать основной идентификатор команды или любой из дополнительных идентификаторов для назначения сущности данной команде.

   * Дополнительные идентификаторы должны соответствовать тем же требованиям к символам, что и основной идентификатор команды.
   * Дополнительный идентификатор не может совпадать с основным идентификатором той же команды.
   * В пределах команды дополнительные идентификаторы должны быть уникальными.
   * Дополнительный идентификатор может совпадать с основным или дополнительным идентификатором другой команды.
   * В отличие от основного идентификатора команды, дополнительные идентификаторы **могут быть изменены и удалены**.

   В данных команд владения, [импортированных из Microsoft Entra ID](../ownership-app.md#import-teams "Предоставляет настраиваемые действия для определения рабочих процессов, интегрирующих владельцев сущностей и их контактную информацию."), параметр `mailNickname` из `get_groups` устанавливается как уникальный, понятный человеку **Supplementary Identifier**.
7. Выберите все **Responsibilities** (зоны ответственности), применимые к команде. Хотя эти переключатели необязательны, они предоставляют важные метаданные и помогают указать соответствующие контактные данные.

   * **Development** — команды, ответственные за разработку и поддержку программного кода
   * **Security** — команды, отвечающие за оценку влияния и приоритетности уязвимостей и реагирование на них
   * **Operations** — команды, которые развёртывают и управляют программным обеспечением, с фокусом на производительность и доступность
   * **Infrastructure** — команды, ответственные за установку и обслуживание ИТ-оборудования
   * **Line of business** — команды, ответственные за достижение бизнес-целей и целей клиентов
8. Добавьте все **Contact details** (контактные данные), применимые к команде. Вы можете предоставить конкретную маршрутную информацию для целевых уведомлений по электронной почте, Jira, Microsoft Teams и Slack.

   Вы можете добавить столько записей, сколько необходимо. Создайте отдельную запись для каждого адреса электронной почты, канала Slack, команды Microsoft Teams или проекта Jira, который вы хотите добавить.

   1. Выберите **Add contact information**.
   2. Выберите **Integration type** (тип интеграции).
   3. Укажите данные следующим образом:

      * **Email** — укажите один **Email** адрес.
      * **Jira** — введите имя проекта Jira **Project**, **Default assignee** Обязательно и **URL** проекта.
      * **MS Teams** — укажите имя команды **Team** и **URL** (вы можете предоставить ссылку на канал команды из MS Teams).
      * **Slack** — укажите имя канала **Channel** и **URL**.
9. Укажите любые другие **Links** (ссылки), описывающие команду или её обязанности.
10. Выберите **Add link**.
11. Выберите **Type** (тип) ссылки, например, **Documentation** или **Wiki**.
12. Укажите **URL**.

    Вы можете добавить столько записей, сколько необходимо. Создайте отдельную запись для каждой ссылки, которую вы хотите добавить.
13. Выберите **Add additional information**, чтобы предоставить дополнительные пары ключ-значение в полях **Name** и **Value**, например, наименования центров затрат или информацию об организационной иерархии. Вы также можете указать необязательный **URL**.

    Ключи для дополнительной информации:

    * Должны содержать от 1 до 100 символов.
    * Могут содержать буквы, цифры, специальные символы и пробелы.
14. Укажите **External ID**, который используется только для целей автоматизации, таких как импорт или обновление информации о команде. Например, используйте это поле для хранения уникального идентификатора, присвоенного команде во внешней системе, такой как Active Directory.

    * External ID может быть указан только при создании команды.
    * После создания external ID **не может быть изменён**.
    * Эта строка может начинаться и заканчиваться специальными символами, пробелами, цифрами и буквами, а также содержать их.

    * External ID нельзя использовать для назначения информации о владении сущностям в парах ключ-значение; для назначения сущностей командам-владельцам используйте основной идентификатор команды и дополнительные идентификаторы.

    * В информации о команде, [импортированной из Microsoft Entra ID](../ownership-app.md#import-teams "Предоставляет настраиваемые действия для определения рабочих процессов, интегрирующих владельцев сущностей и их контактную информацию."), это поле устанавливается равным **Object Id** соответствующей группы из Entra ID и не может быть изменено. **Team identifier** импортированной команды владения также устанавливается равным этому значению.
15. **Save changes** (сохраните изменения).

Дополнительные сведения об оптимальной настройке команд см. в разделе [Лучшие практики для владения сущностями](best-practices.md "Советы и лучшие практики для обеспечения адекватного покрытия владения сущностями").

## Создание команды через API и Configuration as Code

Создание команды через [**Settings API**](../../dynatrace-api/environment-api/settings.md "Узнайте, что предлагает Dynatrace Settings API.") требует [JSON-схемы](../../dynatrace-api/environment-api/settings/schemas/get-schema.md "Просмотрите схему настроек через Dynatrace API.") конфигурации владения (`builtin:ownership.config`), предоставляемой Dynatrace. Вы можете получить схему с помощью Settings API. Предоставьте конфигурацию команды в виде JSON-полезной нагрузки.

[Configuration as Code](../configuration-as-code.md "Используйте Dynatrace Configuration as Code через Monaco или Terraform.") позволяет использовать JSON-шаблоны и YAML-файлы для создания одной или нескольких команд одновременно.

## Импорт команд через Workflows Workflows

Вы можете импортировать группы пользователей из **Microsoft Entra ID** и **ServiceNow** в качестве команд владения в Dynatrace.
Для этого см. описание использования действия [`import_teams`](../ownership-app.md#import-teams "Предоставляет настраиваемые действия для определения рабочих процессов, интегрирующих владельцев сущностей и их контактную информацию.") и рабочих процессов для импорта команд.

## Управление командами

Вы можете управлять командами владения (создавать, удалять, редактировать) через **Settings** > **Ownership** > **Teams**, а также через [Settings API](../../dynatrace-api/environment-api/settings.md "Узнайте, что предлагает Dynatrace Settings API.").

Страница настроек **Ownership teams** содержит список всех команд (и их описания) в вашей среде, независимо от того, были ли они созданы через веб-интерфейс или API. Используйте поле **Filter** для фильтрации этого списка по любой строке в имени или описании команды.

Вы можете **Delete** (удалить) команду из этого представления и развернуть **Details** для просмотра и редактирования информации о команде. Однако вы не можете изменить или удалить основной идентификатор команды.

### История изменений

Выберите ![Ещё](https://dt-cdn.net/images/more-icon-01c8b008ca.svg "Ещё") в правом верхнем углу страницы **Ownership teams**, а затем выберите **Revision history** для просмотра подробностей изменений, включая создание, изменение и удаление команд. Ревизии отображаются с **Timestamp** (временной меткой), **Source** (источником) изменения (**Web UI** или **API**) и **User** (пользователем), внёсшим изменение.

![История команд владения](https://dt-cdn.net/images/ownership-teams-revisions-1462-e55719add4.png)

Разверните для просмотра **Details** любой ревизии. Изменения выделены цветом, чтобы показать **Original value** (исходное значение) и **New value** (новое значение). На изображении ниже показан удалённый дополнительный идентификатор.

![Упрощённый вид ревизии](https://dt-cdn.net/images/ownership-teams-revision1-1849-055f7f5d2e.png)

**Switch to advanced mode** (переключитесь в расширенный режим) и выберите **Show diff**, чтобы просмотреть те же изменения в виде JSON-объектов. **Key** (ключ), отображаемый в расширенном режиме, — это уникальный идентификатор настройки, назначенный каждой команде. Выберите ключ, чтобы просмотреть подробности соответствующей команды.

![YAML-вид ревизии](https://dt-cdn.net/images/ownership-teams-revision2-1848-06b3770890.png)

Вы также можете фильтровать историю изменений для конкретной команды — вставьте любой **Key** идентификатора настройки в поле **Filter**.

### Дополнительные возможности управления командами

Существуют дополнительные удобства для управления каждой отдельной командой.

1. Разверните подробности команды на странице **Ownership teams**.
2. Выберите ![Ещё](https://dt-cdn.net/images/more-icon-01c8b008ca.svg "Ещё") в правом верхнем углу подробностей команды для доступа к специфичным для команды [**Revision history**](#revision-history) (истории изменений), полезным фрагментам **API** для управления командой и возможности **Duplicate** (дублировать) данные команды в качестве основы для создания новой команды.

   ![Ссылки для управления командой](https://dt-cdn.net/images/ownership-team-specific-links-1462-4d8ea5d2df.png)

Фрагменты API для чтения, создания, обновления и удаления команд доступны как для Windows, так и для Linux через интерфейс командной строки. Хотя фрагменты API специфичны для конкретной команды, вы можете изменять их, например, удалив идентификатор команды для создания новой команды. Обязательно укажите свой токен доступа.

![Фрагменты API для управления командами](https://dt-cdn.net/images/ownership-teams-api-snippets-763-e9a0f5eb88.png)
