# Ownership


Предварительные требования

### Разрешения

В следующей таблице описаны необходимые разрешения.

Кроме того, вам потребуются следующие основные разрешения в Workflows (перейдите в **Workflows** > **Settings** > **Authorization Settings**).

* `app-engine:apps:run` — позволяет просматривать и запускать приложения; предоставляет базовый доступ к Launcher.
* `app-engine:functions:run` — позволяет использовать исполнитель функций.

Также вы можете настроить пользователей с помощью [политики пользователя AppEngine](https://www.dynatrace.com/news/blog/tailored-access-management-part-2-onboard-users-to-grail-and-appengine/).

settings:objects:write

Начало работы

Сценарии использования

![Ownership](https://dt-cdn.net/images/ownership-w-background-512-99cc966544.webp "Ownership") **Ownership** предоставляет действия для построения рабочего процесса, запрашивающего команду-владельца сущности и соответствующую контактную информацию. С помощью этих действий вы можете извлекать данные о владении сущностью и интегрировать их с другими приложениями, например, для отправки уведомлений в Slack или обновлений в Jira владельцам сущностей на основе различных триггеров.

Кроме того, ![Ownership](https://dt-cdn.net/images/ownership-w-background-512-99cc966544.webp "Ownership") **Ownership** позволяет импортировать команды из следующих источников с помощью рабочих процессов автоматизации:

* Группы из Microsoft Entra ID
* Группы из ServiceNow
* Любой источник данных в формате JSON

Подробнее см. в разделе Действия для Ownership.

![Получение информации о команде-владельце выбранных сущностей выполняется с помощью действия рабочего процесса "get_owners".
Один идентификатор сущности или список идентификаторов сущностей можно использовать напрямую или через выражение для получения результата ранее выполненного действия рабочего процесса для извлечения информации о команде-владельце.

1 из 4 — Получение информации о команде-владельце выбранных сущностей выполняется с помощью действия рабочего процесса "get\_owners".
Один идентификатор сущности или список идентификаторов сущностей можно использовать напрямую или через выражение для получения результата ранее выполненного действия рабочего процесса для извлечения информации о команде-владельце.
Если контактные данные указаны в настройках команд-владельцев, полученная информация может использоваться с действиями уведомлений рабочего процесса, такими как Slack, MS Teams, E-Mail или JIRA.

* Предоставление контактной информации на уровне сущности для улучшения автоматизированных действий по устранению проблем, таких как целевые уведомления, например, через Workflows
* Обеспечение прозрачности команд и зависимостей
* Сокращение среднего времени восстановления (MTTR) для автоматического информирования нужных команд о новых инцидентах.
* Автоматический импорт и хранение информации о командах-владельцах для поддержания актуальности и синхронизации данных.
