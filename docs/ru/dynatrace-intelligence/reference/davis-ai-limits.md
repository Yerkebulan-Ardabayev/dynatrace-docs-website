# Ограничения Dynatrace Intelligence


* Последняя версия Dynatrace
* Справочник
* Чтение: 5 мин
* Обновлено 4 февраля 2026 г.

На этой странице перечислены ограничения по умолчанию для компонентов Dynatrace Intelligence.

## Проблемы и события

Некоторые ограничения проблем и событий применяются для каждого провайдера. Следующие провайдеры подпадают под ограничения:

* `AGENT_LOCAL_REST_API_INGEST`
* `AVAILABILITY`
* `BASELINING`
* `EVENTS_REST_API_INGEST`
* `KUBERNETES_ANOMALY_DETECTION`
* `KUBERNETES_EVENT`
* `LOG_EVENTS`
* `METRIC_EVENTS`
* `ONEAGENT`
* `OPENPIPELINE_DATA_EXTRACTION`
* `REAL_USER_MONITORING`
* `SYNTHETIC`

### Количество одновременно активных проблем Davis

Максимальное количество одновременно активных проблем Davis в рамках одного окружения составляет 10 000.

### Количество одновременно активных событий Davis

Максимальное количество одновременно активных событий Davis в рамках одного окружения по всем провайдерам событий составляет 15 000.

### Количество одновременно активных событий Davis на провайдера

Максимальное количество одновременно активных событий Davis на провайдера событий составляет 4 000. Пример провайдера событий: `event.provider = "AVAILABILITY"`.

### Количество событий Davis, обрабатываемых каждый час

Максимальное количество событий Davis на провайдера событий, которое может быть обработано в течение часа, составляет 100 000. Этот лимит восполняется каждый час.

Некоторые события переопределяют ограничения по умолчанию. Подробнее см. в таблице ниже.

### Максимальное время жизни события

Максимальное время жизни активных событий составляет 60 дней. Если базовая аномалия не исчезает в течение 60 дней, соответствующее событие закрывается и создается новое.

### Временной интервал объединения проблем

Максимальный временной интервал объединения проблем составляет 90 минут. Если проблема остается нерешенной более 90 минут, новые события больше не будут объединяться с ней.

Эта мера гарантирует, что Dynatrace Intelligence не будет собирать несвязанную информацию для длительных инцидентов.
