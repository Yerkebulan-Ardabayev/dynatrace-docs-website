# Problems app

**Problems** автоматически анализирует инциденты, определяет первопричину и область воздействия, помогая сократить MTTR.

## Предварительные требования

### Разрешения

* `business-analytics:business-flows:read` -- чтение бизнес-потоков
* `davis-copilot:conversations:execute` -- диалоги с помощником
* `davis-copilot:document-search:execute` -- поиск документов
* `davis:analyzers:execute` -- анализатор деталей проблемы
* `document:documents:read/write/delete` -- работа с документами
* `notification:notifications:read/write` -- уведомления
* `settings:objects:read` -- чтение настроек

### Установка

Убедитесь, что приложение [установлено](../manage/hub.md#install).

## Лента проблем

По умолчанию отображаются проблемы за 2 часа. Открытые проблемы всегда вверху. Диаграмма вверху визуализирует аномалии.

### Фильтрация

Быстрые фильтры слева: **Status** (`Active`/`Closed`), **Category** (замедления, ошибки, ресурсы, доступность), **Impact** (фронтенды, сервисы, инфраструктура).

Панель фильтров поддерживает булеву логику (AND/OR, скобки). Текстовый поиск: `* ~ <текст>`.

Поля текстового поиска: `event.id`, `event.name`, `event.description`, `event.status`, `display_id`, `labels.alerting_profile`, `entity_tags`, `root_cause_entity_name`.

### Сегменты

Предопределённые фильтры для быстрой фильтрации по командам. Создаются на основе фильтров событий:

```
cloud.region = "us-east-1c" AND event.kind = "DAVIS_PROBLEM"
```

### Автообновление

Настройки обновления в правом верхнем углу -- выбор частоты или ручное обновление.

## Исследование проблем

### Детали проблемы

Выберите ID проблемы. Страница показывает первопричину (красная метка), затронутые сущности, диаграммы аномальных метрик.

### Сравнение

Выберите несколько проблем флажками > **Show details** для быстрого переключения.

### Свойства событий

Полезные свойства: `event.description` (поддерживает Markdown), `dt.query` (DQL-запрос), `dt.entity.*` (связанные сущности), `dt.settings.object_id` (ссылка на настройки).

### Экспорт CSV

Экспорт данных ленты (до 1000 записей) из таблицы, Notebooks или Dashboards.

## Первопричина без переключения контекста

Информация о проблемах доступна в других приложениях (Kubernetes, и т.д.) без перехода в Problems.

## Логи

Вкладка **Logs** -- все строки логов по сущностям, затронутым проблемой, с уровнями логирования.

## Уведомления

Глобальный индикатор активных проблем в доке. Персонализация через фильтры **Category** с сохранением. Email-уведомления -- личная настройка, запускается через OpenPipeline.

## Представление Deployment

Визуализация затронутых развертываний (облачные регионы, кластеры K8s) в формате UML-диаграммы. Первопричина автоматически выделена. Максимум 5 уровней иерархии.

## Пользовательские поля проблем

**Settings** > **Dynatrace Intelligence** > **Root cause analysis** > **Problem fields**. Можно добавлять пользовательские поля из событий, переименовывать и удалять.

Ограничения: только строковые значения, нельзя дублировать имена полей Semantic Dictionary.

## Руководства по устранению неполадок

Создание через Dashboards/Notebooks из деталей проблемы: **Troubleshooting** > **New**. При общем доступе -- ИИ автоматически предлагает для аналогичных проблем.

## Варианты углублённого анализа

В деталях проблемы, зависят от типа сущности:
* Анализ сбоев
* Просмотр связанных логов
* Просмотр неуспешных трассировок
* Переход к странице сущности

## Обзор проблемы (Overview)

Вкладка **Overview** -- резюме с разделами: **Impact**, **Root cause**, **Visual resolution path** (граф зависимостей), **Automation and remediation** (запущенные рабочие процессы).
