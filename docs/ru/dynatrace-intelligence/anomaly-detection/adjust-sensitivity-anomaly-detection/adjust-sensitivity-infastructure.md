# Настройка чувствительности обнаружения аномалий для инфраструктуры

* Latest Dynatrace
* Руководство
* Чтение: 3 мин.
* Обновлено 28 января 2026 г.

Dynatrace обнаруживает множество аномалий производительности, связанных с инфраструктурой, которые могут привести к проблемам с вашей инфраструктурой (конкретный список зависит от типа вашей инфраструктуры). Каждый тип аномалии обнаруживается независимо и инициирует соответствующие проблемы и оповещения.

Чтобы настроить глобальную конфигурацию обнаружения аномалий для инфраструктуры

1. Перейдите в **Settings** > **Anomaly detection**.
2. В разделе **Infrastructure** выберите тип инфраструктуры, для которого вы хотите настроить обнаружение аномалий.

Обнаружение аномалий может быть настроено для всех хостов, отслеживаемых Dynatrace, независимо от их типа (физические или виртуальные). Dynatrace также обнаруживает сетевые проблемы и проблемы с дисками ваших хостов.

Для каждого типа аномалии вы можете выбрать автоматическое обнаружение, чтобы положиться на Dynatrace Intelligence для оповещения о проблемах, или задать фиксированный порог. Просто выберите нужный вариант из соответствующего списка.

По умолчанию Dynatrace оповещает только о неожиданных сбоях.

* Во время плавного завершения работы отключение хоста является ожидаемым, и операционная система отправила сигнал завершения, уведомляющий OneAgent о том, что оператор намеренно завершает работу хоста.
* Если OneAgent не получает сигнала завершения, отключение классифицируется как неожиданное.

Вы можете дополнительно включить получение уведомлений о плановых отключениях.

## Пороги для конкретных дисков

Серверное оповещение о дисках для новых тенантов

Начиная с версии SaaS 1.308, серверное оповещение о дисках по умолчанию отключено для новых тенантов. Мы рекомендуем использовать [оповещение Disk Edge](/docs/observe/infrastructure-observability/hosts/configuration/anomaly-detection#disk-edge-alerting "Настройка обнаружения аномалий хоста, включая пороги проблем и событий.") вместо этого. Оповещение Disk Edge позволяет создавать более сложные и специфичные правила с использованием:

* Метрик для оповещения (доступное дисковое пространство, файловая система только для чтения, время чтения, время записи и доступные inode)
* Операционной системы, к которой должна применяться политика
* Фильтров по имени диска
* Условий пользовательских метаданных хоста
* Пользовательских свойств, прикреплённых к инициированному событию

Имейте в виду, что оповещение Disk Edge требует OneAgent версии 1.293+.

Dynatrace Intelligence автоматически обнаруживает аномалии дисков, такие как недостаток свободного дискового пространства или медленные диски. На хосте существуют различные типы дисков: загрузочный диск, диск для хранения логов или диск для хранения бизнес-данных. Хотя оповещение о недостатке дискового пространства не имеет смысла для загрузочного образа фиксированного размера, оно вполне целесообразно для диска с критически важными бизнес-данными.

С помощью пользовательских правил обнаружения дисков вы можете настроить точные правила для отдельных групп дисков (группы формируются на основе шаблонов имён дисков и/или тегов хостов). Пороги на уровне дисков переопределяют глобальные пороги для соответствующих дисков, в то время как глобальные настройки по-прежнему применяются к другим дискам.

Чтобы изменить настройки порогов для группы дисков

1. Перейдите в **Settings** > **Anomaly detection**.
2. В разделе **Infrastructure** выберите **Custom disk-detection rules**.
3. Нажмите **Add item**.
4. Выберите метрику для мониторинга и укажите осмысленное имя для правила.
5. Укажите порог для метрики и количество выборок, которые должны нарушить порог для инициирования оповещения.
6. Необязательно: укажите шаблон имени диска.

   Отдельные правила не связаны логически и применяются независимо. Например, если одно правило соответствует всем дискам, не содержащим `A`, а другое -- всем дискам, не содержащим `B`, то каждый диск будет соответствовать либо первому, либо второму, либо обоим правилам одновременно. Обратите внимание, что добавление нескольких значений, подстановочных знаков или регулярных выражений в рамках одного фильтра правила также невозможно.
7. Необязательно: для дополнительного сужения использования диска укажите теги, которые должен иметь хост.
8. Нажмите **Save changes**.

### Иерархия конфигурации обнаружения аномалий

Вы можете настроить обнаружение аномалий на нескольких уровнях -- среда, группа хостов, хост и т. д. Когда несколько правил затрагивают одну и ту же сущность, наиболее специфичное правило имеет приоритет над более общими правилами, как показано на диаграмме ниже.

## Пороги для конкретной группы хостов

Чтобы настроить правила для дисков конкретной группы хостов

1. Перейдите в **Deployment Status**.
2. Отфильтруйте таблицу по **Host group** и выберите имя группы хостов, для которой вы хотите настроить пользовательские правила обнаружения дисков.
3. Для одного из перечисленных хостов выберите группу хостов.
4. Разверните **Anomaly detection** и выберите **Custom disk-detection rules**.
5. Нажмите **Add item**.
6. Выберите метрику для мониторинга и укажите осмысленное имя для правила.
7. Укажите порог для метрики и количество выборок, которые должны нарушить порог для инициирования оповещения.
8. Необязательно: укажите шаблон имени диска.
9. Необязательно: для дополнительного сужения использования диска укажите теги, которые должен иметь хост.
10. Нажмите **Save changes**.

## Пороги для конкретного хоста

В качестве альтернативы определению порогов глобально для всей среды вы можете задать точные пороги для отдельных хостов. Пороги на уровне хоста переопределяют глобальные пороги для этого хоста, в то время как глобальные настройки по-прежнему применяются к другим хостам. Вы можете в любой момент вернуться к глобально определённым порогам.

Чтобы изменить настройки порогов для конкретного хоста

1. Перейдите в ![Hosts](https://dt-cdn.net/images/hosts-512-59f5d2dd7f.png "Hosts") **Hosts Classic**.
2. Выберите хост, который хотите настроить.
3. В правом верхнем углу страницы нажмите **More** (**...**) > **Settings**.
4. Перейдите в **Anomaly detection** > **Infrastructure**, чтобы настроить конфигурацию.
