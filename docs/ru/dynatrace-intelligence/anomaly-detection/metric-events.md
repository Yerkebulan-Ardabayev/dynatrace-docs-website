# Метрические события

Dynatrace Classic

Рекомендуется перейти на ![Anomaly Detection - new](https://dt-cdn.net/images/davis-anomalydetection-256-105da91594.png "Anomaly Detection - new") **Anomaly Detection** -- поддержка DQL-запросов к Grail, оповещения на логи/спаны/бизнес-события, расширенные запросы.

## Типы метрических событий

* **Ключ метрики** -- оценка одной метрики. Только статические пороги.
* **Селектор метрик** -- сложные запросы с историческими данными и арифметическими операциями.

## Стратегии мониторинга

* **Статический порог** -- фиксированный.
* **Авто-адаптивный порог** -- динамически адаптируется (только для селектора метрик).
* **Сезонная базовая линия** -- доверительный интервал для сезонных паттернов (только для селектора метрик).

**Ограничение:** 10 000 конфигураций метрических событий.

## Обзор

**Settings** > **Anomaly detection** > **Metric events** -- обзор всех событий с информацией об ограничениях и сбоях за 24 часа.

Отслеживаемые проблемы:
* `THROTTLED` -- достигнуты лимиты запросов.
* `QUERY_FAILED` -- сбой запроса (недоступные зоны/теги/данные).
* `CONFIG_BLOCKED` -- конфигурация заблокирована из-за высокой нагрузки.

## Зоны управления

При наличии разрешений на зону управления можно создавать привязанные к ней конфигурации. Для конфигураций без привязки нужен доступ администратора.

Зоны управления через [правила размерных данных](../../manage/identity-access-management/permission-management/management-zones/management-zone-rules.md#logs-metrics) не поддерживаются.

## Осведомлённость о топологии

Метрики содержат ссылки на топологию (хосты, процессы и т.д.). Davis AI использует эту информацию для определения первопричины. При создании события Dynatrace автоматически выбирает наиболее релевантную сущность.

Сценарии привязки к сущностям:

* **Нет сущности** -- событие на уровне окружения.
* **Одна сущность** -- событие на этой сущности.
* **Несколько сущностей** -- выбирается наиболее специфичная (процесс > хост).
