# Автоадаптивные пороговые значения

Динамический подход, при котором эталонное значение обнаружения аномалий меняется со временем. Преимущества перед статическим порогом:
* Не нужно заранее знать пороговое значение.
* Не нужно вручную настраивать пороги при изменении поведения метрик.

Каждая сущность в конфигурации получает собственный независимый порог. Например, 5 хостов = 5 независимых пороговых значений.

**Ограничение:** 100 конфигураций событий метрик на среду.

## Автоадаптивный vs статический порог

Пример: время записи на диск -- волатильная метрика. Статический порог (20 мс) даёт ложноположительные при росте нагрузки. Автоадаптивный порог автоматически адаптируется ежедневно на основе данных за 7 дней.

## Расчёт порогового значения

Эталон -- данные за последние 7 дней:
* **Базовый уровень** -- 99-й перцентиль всех минутных измерений.
* **Флуктуация сигнала** -- межквартильный размах (25-й -- 75-й перцентиль).
* **Порог** = базовый уровень + (n x флуктуация сигнала), где n настраивается параметром `number of signal fluctuation`.

Скользящее окно определяет частоту нарушений для генерации события (не обязательно последовательных). По умолчанию: 3 из 5 минут. Максимум: 60 минут.
