# Автоадаптивные пороговые значения для обнаружения аномалий

Автоадаптивные пороговые значения -- это динамический подход к определению базовых показателей, при котором эталонное значение для обнаружения аномалий меняется со временем. Основное преимущество перед [статическим пороговым значением](static-thresholds.md "When to use a static threshold for your anomaly detection.") состоит в том, что эталонное значение динамически адаптируется со временем, и вам не нужно заранее знать пороговое значение. Кроме того, вам не нужно вручную настраивать несколько статических пороговых значений для метрик, поведение которых меняется со временем.

Когда конфигурация обнаружения аномалий включает несколько сущностей, каждая сущность получает собственное автоадаптивное пороговое значение, и каждое пороговое значение оценивается независимо. Например, если область действия конфигурации включает пять хостов, Dynatrace рассчитывает и оценивает пять независимых пороговых значений.

Существует **ограничение в 100 конфигураций событий метрик на среду** независимо от того, сколько отдельных пороговых значений содержит каждая конфигурация.

## Автоадаптивное и статическое пороговое значение

Рассмотрим пример, в котором адаптивное пороговое значение имеет преимущество перед статически определённым пороговым значением. На приведённой ниже диаграмме показано измеренное время записи на диск в миллисекундах. Это волатильная метрика, которая демонстрирует всплески в зависимости от интенсивности записи на диск. Если бы мы определяли пороговое значение для каждого диска в этой ИТ-системе на основе начальных данных (начало диаграммы), мы бы установили статическое пороговое значение на уровне 20 миллисекунд. Однако впоследствии нагрузка на диск изменится в сторону увеличения, и установленное таким образом статическое пороговое значение будет генерировать множество ложноположительных оповещений. Чтобы избежать этого, пришлось бы определить новое пороговое значение и вручную адаптировать конфигурацию.

![Пример обнаружения аномалий со статическим пороговым значением для времени записи на диск в приложении Notebooks.](https://dt-cdn.net/images/notebooks-disk-static-threshold-ad-1743-2441063182.png)

Автоадаптивные пороговые значения, напротив, автоматически адаптируют эталонные пороговые значения ежедневно на основе измерений за предыдущие семь дней. Если метрика меняет своё поведение, пороговое значение адаптируется автоматически.

![Пример обнаружения аномалий с автоадаптивным пороговым значением для времени записи на диск в приложении Notebooks.](https://dt-cdn.net/images/notebooks-disk-auto-adaptive-threshold-ad-1742-71e913aa5e.png)

## Расчёт порогового значения

Эталонными значениями для расчёта порогового значения являются данные метрик за последние семь дней.

* Измерения за каждую минуту используются для расчёта 99-го перцентиля всех измерений. Это определяет соответствующий **базовый уровень**.
* Межквартильный размах между 25-м и 75-м перцентилями затем используется в качестве **флуктуации сигнала**, которая может быть добавлена к базовому уровню. С помощью параметра `number of signal fluctuation` (n x флуктуация сигнала) вы можете контролировать, сколько раз флуктуация сигнала добавляется к базовому уровню для формирования фактического порогового значения для оповещения.

Ещё одним важным параметром для динамических пороговых значений является скользящее окно, которое используется для сравнения текущих измерений с рассчитанным пороговым значением. Оно определяет, как часто рассчитанное пороговое значение должно быть нарушено в пределах скользящего окна времени, чтобы было сгенерировано событие (нарушения не обязательно должны быть последовательными). Такой подход помогает избежать слишком агрессивного оповещения при единичных нарушениях. Скользящее окно можно установить максимум на 60 минут.

По умолчанию любые 3 минуты из скользящего окна в 5 минут должны нарушить пороговое значение, чтобы было сгенерировано событие. То есть событие должно иметь 3 минуты с нарушениями в пределах любого 5-минутного скользящего окна.
