* Пояснение

Некоторые типичные аномалии, обнаруживаемые Dynatrace, включают увеличение частоты ошибок, ухудшение времени отклика, а также всплески или падения трафика приложений. Наблюдая за вашей средой, Dynatrace изучает эталонные значения, представляющие её нормальное поведение, и соответствующим образом адаптирует обнаружение аномалий. Этот процесс называется *автоматическим определением базовых показателей*. Помимо использования автоматического базового уровня, вы можете задать *фиксированные пороговые значения*, определяющие, когда следует регистрировать проблему и отправлять оповещение. Эти параметры можно настроить глобально или для конкретных сущностей.

*Чувствительность обнаружения проблем* определяет уровень статистической достоверности, необходимый для отправки оповещения.

| Чувствительность | Статистическая достоверность | Примечания |
| --- | --- | --- |
| Низкая | Высокая | Полезна на этапах разработки и предпродакшена, позволяя избежать слишком большого количества оповещений. |
| Высокая | Низкая | Проблема регистрируется, даже если лишь несколько точек данных превысили пороговое значение. Полезна для критически важных производственных сервисов. |

Для некоторых критериев Dynatrace различает абсолютные и относительные пороговые значения. В примере ниже (показана часть конфигурации обнаружения аномалий для приложения) пороговые значения для ухудшения ключевых показателей производительности установлены на `100 ms` (абсолютное) и `50%` (относительное) выше автоматически определённого базового уровня. Пороговое значение для самых медленных 10% запросов установлено на `1,000 ms` (абсолютное) и `100%` (относительное) выше автоматически определённого базового уровня.

Кроме того, обнаружение аномалий учитывает количество действий в минуту, которые должны происходить в отслеживаемом приложении (`10 actions per minute` в примере выше). С помощью этой настройки вы можете отключить оповещения для приложений и сервисов с низким трафиком — определение базовых показателей и оповещение для приложений с низким трафиком часто приводят к ненужным оповещениям.

Чтобы настроить чувствительность обнаружения на глобальном уровне

1. Перейдите в **Settings**.
2. Разверните **Anomaly detection**.
3. Выберите нужный тип сущности. Подробные инструкции см. в одной из тем ниже.

* Приложения
* Сервисы
* Сервисы баз данных
* Инфраструктура
* События расширений
