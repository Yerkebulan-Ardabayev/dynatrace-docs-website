Dynatrace обнаруживает аномалии: рост частоты ошибок, ухудшение времени отклика, всплески/падения трафика. Система автоматически изучает эталонные значения (*автоматическое определение базовых показателей*). Дополнительно можно задать *фиксированные пороговые значения*. Настройка доступна глобально или для конкретных сущностей.

*Чувствительность обнаружения проблем* определяет требуемый уровень статистической достоверности:

| Чувствительность | Статистическая достоверность | Примечания |
| --- | --- | --- |
| Низкая | Высокая | Для dev/staging -- меньше оповещений. |
| Высокая | Низкая | Оповещение даже при нескольких точках выше порога. Для критичных production-сервисов. |

Dynatrace различает абсолютные (`100 ms`) и относительные (`50%`) пороговые значения. Также учитывается минимальное количество действий в минуту для отключения оповещений при низком трафике.

### Настройка глобальной чувствительности

1. **Settings** > **Anomaly detection**.
2. Выберите тип сущности: Приложения, Сервисы, Сервисы баз данных, Инфраструктура, События расширений.
