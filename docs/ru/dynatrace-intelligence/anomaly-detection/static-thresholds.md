# Статические пороговые значения для обнаружения аномалий

* Последняя версия Dynatrace
* Пояснение
* Время чтения: 2 мин
* Обновлено 05 апр. 2024 г.

Статическое пороговое значение представляет собой жёсткий предел, который метрика не должна нарушать. Поскольку статические пороговые значения не меняются со временем, они являются важным инструментом мониторинга для определения критических границ нормальной работы.

Важно выбрать между статическим и [адаптивным пороговым значением](auto-adaptive-threshold.md "How Dynatrace adapts thresholds for multiple entities within the scope of an anomaly detection configuration.") в зависимости от вашего сценария использования.

Например, вы можете использовать статическое пороговое значение для установки предела общего потребления памяти известным процессом. В этом случае статическое пороговое значение превосходит адаптивное, поскольку если потребление памяти медленно растёт со временем, адаптивное пороговое значение просто изменяется вместе с ним, не сигнализируя о проблемах и в конечном итоге приводя к скрытой утечке памяти.

На иллюстрациях ниже потребление памяти стабильно увеличивается в течение 30 дней. Статически определённое пороговое значение в 40 МБ обнаружит аномальное поведение процесса, тогда как адаптивное пороговое значение будет увеличиваться вместе со значением метрики.

![Статическое пороговое значение](https://dt-cdn.net/images/static-threshold-1-1041-d8667870ee.png)

![Адаптивное пороговое значение](https://dt-cdn.net/images/static-threshold-2-1041-538cab4669.png)

Помимо порогового значения, вы можете указать, как часто порог должен быть нарушен в пределах скользящего временного окна для создания события (нарушения не обязательно должны быть последовательными). Это помогает избежать чрезмерно агрессивного оповещения при единичных нарушениях порога. Вы можете задать скользящее окно длительностью до 60 минут.

По умолчанию любые 3 минуты из скользящего окна в 5 минут должны нарушать ваше пороговое значение для создания события. То есть для создания события требуется 3 минуты нарушения в пределах любого 5-минутного скользящего окна.
