Статический порог -- жёсткий предел, который метрика не должна нарушать. Не меняется со временем.

**Когда использовать:** Для критических границ, где адаптивный порог неуместен. Пример: лимит потребления памяти процессом -- адаптивный порог просто растёт вместе с утечкой памяти, а статический (40 МБ) обнаружит аномалию.

Скользящее окно определяет, как часто порог должен быть нарушен для создания события (нарушения не обязательно последовательные). По умолчанию: **3 из 5 минут**. Максимум: 60 минут.
