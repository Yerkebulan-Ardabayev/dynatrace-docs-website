# Метрики гистограмм

* Latest Dynatrace
* Explanation
* 7-min read
* Published Jul 09, 2025

Метрики гистограмм — это мощный способ отразить распределение значений для данного измерения, например времени отклика, размеров запросов или длительности. В отличие от счётчика, измеряющего общий итог одним значением, гистограмма подсчитывает количество вхождений при разных пороговых значениях (называемых корзинами). Это многозначный счётчик, агрегирующий данные в корзины, что позволяет анализировать персентили, средние значения и общую форму распределения данных.

## Варианты использования метрик гистограмм

* Измерение распределения времени отклика для сервисов или эндпоинтов
* Отслеживание размеров запросов или полезной нагрузки
* Анализ метрик задержки или продолжительности в распределённых системах

Для более глубокого изучения гистограмм и их преимуществ см. следующие публикации в блоге OpenTelemetry:

* [Why histograms?](https://opentelemetry.io/blog/2023/why-histograms/)
* [Histograms vs. summaries](https://opentelemetry.io/blog/2023/histograms-vs-summaries/)
* [Exponential histograms](https://opentelemetry.io/blog/2023/exponential-histograms/)

## Приём метрик гистограмм

Для приёма гистограмм из источников OpenTelemetry (OTLP ingest API) или Prometheus (Dynatrace Operator) специальная настройка не требуется. Метрики гистограмм отправляются автоматически, когда ваша среда получает данные OpenTelemetry или Kubernetes.

Экспоненциальная гистограмма OpenTelemetry не поддерживается как гистограмма: данные `min|max|sum|count` гистограммы принимаются, но корзины — нет.

Если происходит одно из следующего, OpenTelemetry ingest API возвращает ответы `400` или `200 with partial success`.

* Кумулятивные гистограммы не принимаются (аналогично кумулятивным счётчикам).
* Точки данных гистограммы без `sum` не принимаются. Это происходит при записи отрицательных значений.
* Корзины гистограммы не отсортированы.
* Граничные значения корзин гистограммы `NaN` или `Infinite` недопустимы.

## Запрос персентилей с помощью DQL

Dynatrace Query Language (DQL) позволяет анализировать метрики гистограмм с помощью агрегации персентилей. Функция вычисляет запрошенный персентиль по всем корзинам для каждого временного слота.

Например:

```
timeseries percentile(http_request_duration_seconds_bucket, 99)
```

Запрос вычисляет 99-й персентиль значений на основе принятых корзин гистограммы.

Гистограммы представляют данные в виде серии корзин, каждая из которых содержит количество значений, попадающих в определённый диапазон. Например, `http_request_duration_seconds_bucket` содержит счётчики длительности запросов, сгруппированных по заранее определённым диапазонам (измерение `le`).

Функция `percentile` использует данные корзин для оценки запрошенного персентиля (например, 99-го персентиля). Она интерполирует значение, ниже которого находятся 99% наблюдаемых точек данных, на основе накопленных счётчиков в корзинах гистограммы.

Этот запрос особенно полезен для анализа метрик задержки или производительности, поскольку персентили дают представление о наихудших сценариях, с которыми сталкиваются пользователи.

## Примеры визуализации и важные предупреждения

Вы можете визуализировать метрики гистограмм в Dynatrace Notebooks или Dashboards с помощью запросов DQL. Однако учитывайте следующие ограничения:

* Dynatrace в настоящее время отображает метрики гистограмм **в виде персентилей с линейными диаграммами по времени** (с использованием команды `timeseries`). Визуализация распределения вхождений по корзинам за определённый период времени (обычно с использованием столбчатой диаграммы) в данный момент **НЕ** поддерживается.

Дополнительную информацию см. в разделе [Редактирование визуализаций для Notebooks и Dashboards](/docs/analyze-explore-automate/dashboards-and-notebooks/edit-visualizations "Создавайте, редактируйте и просматривайте визуализации на ваших дашбордах и в ноутбуках Dynatrace.").
