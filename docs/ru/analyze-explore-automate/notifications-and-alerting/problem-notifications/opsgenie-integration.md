# Отправка уведомлений Dynatrace в Opsgenie

* Classic
* 2 мин. чтения
* Обновлено 10 окт. 2022 г.

Dynatrace предлагает готовую интеграцию с Opsgenie, которая автоматически отправляет уведомления о проблемах Dynatrace в вашу среду Opsgenie в нескольких регионах.

Настройка интеграции уведомлений о проблемах с Opsgenie

1. Перейдите в **Settings** > **Integration** > **Problem notifications**.
2. Выберите **Add notification**.
3. Выберите **Opsgenie** из доступных типов уведомлений.
4. Настройте уведомление:

   * Введите **Display name** для этой интеграции. Это произвольное имя интеграции, которое будет отображаться в Dynatrace в разделе **Settings** > **Integration** > **Problem notifications** после завершения настройки.
   * **Opsgenie API key**
     Перейдите в свою учётную запись Opsgenie и настройте интеграцию с Dynatrace. Вставьте сгенерированный API-ключ Opsgenie в это поле.
   * **Opsgenie region domain**
     Выберите регион Opsgenie, начав вводить конечную точку региона. Если это поле оставить пустым, по умолчанию будет выбран регион US.
   * **Message**
     Введите текст и заполнители, связанные с проблемой.

     Заполнители

     В разделе **Available placeholders** на странице конфигурации перечислены заполнители, которые можно использовать для этой интеграции. Уведомления о проблемах автоматически подставляют значения заполнителей и включают их в соответствующие инциденты Opsgenie. Инциденты Opsgenie создаются автоматически при обнаружении проблем и закрываются при их устранении. Сведения об инциденте содержат подробную информацию о проблеме, а также прямую ссылку на проблему в Dynatrace.
5. Назначьте [Alerting profile](../alerting-profiles.md "Узнайте, как создавать профили оповещений и управлять ими.").
6. Выберите **Send test notification**, чтобы убедиться, что интеграция с Opsgenie работает.
7. **Save changes**.

## Уровни приоритетов

Проблемы с доступностью в Dynatrace сопоставляются с инцидентами Opsgenie уровня P1, проблемы с ошибками — с уровнем P2, замедления — с уровнем P3 и так далее. Dynatrace также отправляет теги всех затронутых проблемой компонентов вместе с каждым инцидентом Opsgenie.

URL-адрес проблемы содержит прямую обратную ссылку на среду Dynatrace, из которой был отправлен инцидент с проблемой.

* ссылку на проблему в Dynatrace
* степень серьёзности проблемы, обнаруженной Dynatrace, сопоставленную с соответствующим значением приоритета Opsgenie.
