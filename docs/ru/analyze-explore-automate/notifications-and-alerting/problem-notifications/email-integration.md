# Отправка уведомлений Dynatrace по электронной почте

* Classic
* 2-min read
* Updated on Apr 25, 2024

Для расширенных возможностей и автоматизации рабочих процессов (например, для включения целевых уведомлений и устранения проблем) см. [Workflows Connectors](/docs/analyze-explore-automate/workflows/actions "Use Dynatrace ready-made actions for your workflows and integrate Dynatrace with third-party systems.").

Dynatrace предлагает несколько готовых интеграций, которые автоматически отправляют уведомления о проблемах Dynatrace в ваши сторонние системы обмена сообщениями или управления инцидентами. Если же ваша сторонняя система не поддерживается готовой интеграцией, вы можете легко настроить интеграцию по электронной почте. При таком подходе электронное письмо отправляется каждый раз, когда Dynatrace обнаруживает проблему в вашей среде, затрагивающую реальных пользователей.

Кратковременные проблемы

Кратковременные проблемы не генерируют уведомления об открытых проблемах. Для таких проблем Dynatrace отправляет только уведомления об устранённых проблемах, чтобы сообщить вам о том, что такая проблема возникла.

Вы можете настроить строку темы писем с уведомлениями о проблемах, определив текстовый шаблон, включающий заполнители, которые автоматически заполняются соответствующими деталями проблемы, такими как идентификатор проблемы, её влияние или состояние. По умолчанию тело уведомлений по электронной почте содержит HTML-форматированное описание обнаруженной проблемы и прямую ссылку на соответствующую страницу [сведений о проблеме](/docs/dynatrace-intelligence/root-cause-analysis/concepts "Get acquainted with root cause analysis concepts.") в Dynatrace.

Настройка уведомлений о проблемах по электронной почте

1. Перейдите в **Settings** > **Integration** > **Problem notifications**.
2. Выберите **Add notification**.
3. Выберите **Email** из доступных типов уведомлений.
4. Настройте уведомление:

   * Введите **Display name** для этой интеграции. Это произвольное имя интеграции, которое будет отображаться в Dynatrace в разделе **Settings** > **Integration** > **Problem notifications** после завершения настройки.
   * В разделе **To** выберите **Add recipient**, чтобы добавить адрес электронной почты для получения уведомлений. Чтобы добавить дополнительных получателей, выбирайте **Add recipient** для каждого дополнительного адреса **To**.
   * В поле **CC** выберите **Add recipient**, чтобы добавить дополнительные адреса электронной почты для получения уведомлений.
   * В поле **BCC** выберите **Add recipient**, чтобы добавить дополнительные адреса электронной почты для получения уведомлений, которые не должны отображаться в заголовке письма.
   * В поле **Subject** введите текст и/или вставьте заполнители, которые будут автоматически заполнены соответствующими деталями проблемы, такими как идентификатор, состояние или влияние проблемы.
   * В поле **Body** настройте сообщение оповещения о проблеме, которое будет отображаться в теле писем с уведомлениями о проблемах.

     Заполнители

     Раздел **Available placeholders** на странице конфигурации содержит список заполнителей, которые можно использовать для этой интеграции. Заполнители автоматически заменяются фактическими значениями в сообщении.
   * Выберите [профиль оповещения](/docs/analyze-explore-automate/notifications-and-alerting/alerting-profiles "Learn how to create and manage alerting profiles.") для фильтрации потока проблем.
5. Выберите **Send test notification**, чтобы убедиться, что ваша интеграция по электронной почте работает.
6. **Save changes**.
