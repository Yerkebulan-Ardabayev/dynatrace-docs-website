# Наблюдение за журналами в режиме реального времени

* Последняя версия Dynatrace
* Учебное руководство
* Чтение: 2 мин
* Обновлено 28 января 2026 г.

В этом руководстве описывается, как принимать, изучать и визуализировать данные журналов без сложной настройки или знания языков запросов.

## Целевая аудитория

Разработчики или инженеры по надёжности сайтов (SRE), начинающие использовать Dynatrace для анализа журналов.

## Результат обучения

Используя платформенные приложения, вы сможете немедленно получать сведения о поведении приложений и инфраструктуры на основе журналов.

Благодаря аналитике журналов в режиме реального времени и с контекстом вы сможете быстрее обнаруживать проблемы, коррелировать события, эффективно сотрудничать и создавать пользовательскую аналитику для своих сценариев использования — всё в единой платформе наблюдаемости.

## Предварительные условия

* Права на установку OneAgent
* Права на чтение журналов

## Наблюдение за данными журналов в режиме реального времени

### 1. Приём данных журналов

Самый простой способ начать отправку журналов в Dynatrace — использовать OneAgent.

1. Установите OneAgent на хост.
2. Перейдите в ![Settings](https://dt-cdn.net/images/settings-icon-256-38e1321b51.webp "Settings") **Настройки** > **Сбор и захват** > **Мониторинг журналов** > **Правила приёма журналов** и включите **[Встроенный] Принимать все журналы**.

Подробнее о поддерживаемых источниках журналов см. в разделе [Приём журналов](../lma-log-ingestion.md "Потоковая передача данных журналов в Dynatrace.").

### 2. Изучение записей журналов в приложении Logs

Для быстрого поиска и изучения принятых журналов используйте ![Logs](https://dt-cdn.net/images/logs-256-ae0a9ca67f.png "Logs") **Logs**. Оно помогает находить журналы с помощью быстрых фильтров и автодополнения, а также может объяснить значение выбранной записи.

Подробнее см. в разделе [Приложение Logs](../lma-logs-app.md "Поиск, фильтрация и анализ журналов с помощью приложения Dynatrace Logs для быстрого изучения и обмена выводами.").

### 3. Совместное использование информации через приложение Notebooks

В ![Notebooks](https://dt-cdn.net/images/notebooks-768-046137830a.webp "Notebooks") **Notebooks** создавайте совместно используемые документы для пользовательской аналитики. Создайте новую записную книжку и [добавьте раздел Prompt](../../dashboards-and-notebooks/notebooks.md#section-create-davis-copilot "Анализируйте, визуализируйте и делитесь выводами из данных наблюдаемости — всё в одном совместном настраиваемом рабочем пространстве.") для запросов на естественном языке.

Подробнее см. в разделе [Notebooks](../../dashboards-and-notebooks/notebooks.md "Анализируйте, визуализируйте и делитесь выводами из данных наблюдаемости — всё в одном совместном настраиваемом рабочем пространстве.").

### 4. Мониторинг с помощью приложения Dashboards

В ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** используйте различные плитки для визуализации данных журналов и их мониторинга в своих сценариях использования.

Подробнее см. в разделе [Dashboards](../../dashboards-and-notebooks/dashboards-new.md "Создавайте интерактивные настраиваемые представления для визуализации, анализа и совместного использования данных наблюдаемости в режиме реального времени.").

## Следующие шаги

Чтобы начать наблюдение за журналами, не требуется сложная настройка или знание языков запросов. После получения данных вы сможете изучить другие сценарии использования для аналитики и наблюдаемости с журналами.

Для получения дополнительной информации ознакомьтесь с разделом **Связанные темы** и см. [Сценарии использования Log Management and Analytics](../lma-use-cases.md "Изучите распространённые сценарии использования Log Management and Analytics в Dynatrace.").
