# Обработка логов с помощью классического конвейера

* Актуальная версия Dynatrace
* Объяснение
* Чтение: 4 мин
* Обновлено 11 декабря 2025 г.

Dynatrace может преобразовывать входящие строки логов для улучшения ясности, анализа и дальнейшей трансформации на основе определённых вами правил обработки логов. Этот подход известен как **обработка логов с помощью классического конвейера** или **классический конвейер обработки логов**.

Переход на обработку логов с OpenPipeline

Несмотря на то, что классический конвейер обработки логов по-прежнему доступен для некоторых сред, мы рекомендуем переключиться на [обработку логов с OpenPipeline](lma-log-processing/lma-openpipeline.md "Обработка логов с помощью Dynatrace OpenPipeline.") как мощное решение для управления, обработки и анализа логов. Обработка логов с помощью классического конвейера будет прекращена в будущем.

Обработка логов происходит по мере поступления данных логов в среду Dynatrace SaaS и до их записи на диск (сохранения). Устанавливая правила обработки логов, вы можете обрабатывать данные логов сразу после их поступления в Dynatrace. После обработки данные логов отправляются в хранилище и доступны для дальнейшего анализа. Этот метод позволяет обрабатывать данные логов из всех каналов [приёма логов](lma-log-ingestion.md "Потоковая передача данных логов в Dynatrace.").

Например, вы можете извлекать числовые значения из строк логов с помощью классического конвейера обработки логов, преобразовывать их в метрики на платформе Dynatrace и включать их в дашборды и обнаружение проблем.

Потребление DDU

Обработка логов не влияет на потребление [DDU](../../license/monitoring-consumption-classic/davis-data-units/log-management-and-analytics.md "Узнайте, как рассчитывается объём потребления DDU для Dynatrace Log Management and Analytics.") при приёме логов.

Обработка логов с помощью классического конвейера основана на правилах, которые содержат условие совпадения (matcher) и определение правила обработки.

* Условие совпадения сужает доступные данные логов для выполнения конкретного правила.
* Правило обработки — это инструкция по обработке логов, определяющая, как Dynatrace должен преобразовывать или изменять данные логов, соответствующие условию совпадения.

## Этапы обработки логов

Классический конвейер обработки логов включает следующие этапы:

1. Автоматическая обработка логов при приёме

2. [Обработка логов с помощью классического конвейера](lma-classic-log-processing.md "Используйте правила обработки логов для преобразования входящих данных логов для лучшего понимания, анализа или дальнейшей трансформации.")

## Правила обработки логов

Перейдите в **Settings** (Dynatrace Classic) или **Settings Classic** > **Log Monitoring** > **Processing**, чтобы просмотреть действующие правила обработки логов, изменить порядок существующих правил и создать новые. Правила выполняются в порядке их расположения, сверху вниз. Этот порядок критически важен, поскольку предшествующее правило может повлиять на данные логов, которые использует последующее правило в своём определении.

Все правила обработки, соответствующие записи лога, применяются сверху вниз. Выходные данные одного правила являются входными данными для следующего.

Разверните раздел **Details**, чтобы изучить определение правила. Правило обработки логов состоит из следующих элементов:

* **Rule name** (Имя правила)
* **Matcher** (Условие совпадения)
* **Rule definition** (Определение правила)

Вы можете включить или выключить любое правило в столбце **Active**.

## Встроенные правила

По умолчанию обработка логов с помощью классического конвейера включает множество активированных встроенных правил, отвечающих за очистку или нормализацию данных логов. Название каждого встроенного правила начинается с `[Built-in]`.

Вы не можете изменять эти правила напрямую, но у вас есть возможность отключать их, копировать их определения и создавать новые правила с вашими изменениями.

## Добавление правила обработки логов

Чтобы создать правило обработки логов:

1. Перейдите в **Settings** (Dynatrace Classic) или **Settings Classic** > **Log Monitoring** > **Processing**.
2. Нажмите **Add rule**.
3. Укажите имя для правила обработки логов.
4. Укажите запрос для логов в разделе **Matcher**.
   Поисковый запрос для логов сужает доступные данные логов для выполнения конкретного правила. Добавьте **Matcher** к вашему правилу, вставив ваш [DQL-запрос для условия совпадения](lma-classic-log-processing/lma-log-processing-matcher.md "Изучите специфические DQL-функции и логические операторы для обработки логов.").

   Совпадение на основе предыдущих правил не поддерживается

   Условие совпадения работает с исходным набором данных до применения каких-либо правил обработки. Совпадение записей, изменённых предшествующими правилами, не поддерживается. Например, поле, изменённое в правиле 1 и используемое для совпадения в правиле 2, будет содержать исходное значение этого поля и не будет использовать изменённое поле из правила 1.
5. Укажите определение правила обработки.
   Определение правила обработки — это инструкция по обработке логов, определяющая, как Dynatrace должен преобразовывать или изменять ваши данные логов.

   Определение правила создаётся с использованием [команд](lma-classic-log-processing/lma-log-processing-commands.md "Изучите сценарии использования команд обработки логов в Dynatrace на базе Grail.") обработки логов, [функций](lma-classic-log-processing/lma-log-processing-functions.md "Изучите сценарии использования функций обработки логов в Dynatrace на базе Grail.") и сопоставления с шаблонами ([Dynatrace Pattern Language](../../platform/grail/dynatrace-pattern-language.md "Используйте Dynatrace Pattern Language для описания шаблонов с помощью средств сопоставления.")), которые позволяют добавлять, преобразовывать или удалять входящие записи логов. Это даёт вам полный контроль над тем, как ваши данные логов представляются мониторингу логов Dynatrace.
6. Протестируйте правило обработки логов.

   1. Предоставьте образец лога.

      Вы можете протестировать определение правила, предоставив фрагмент образца лога вручную в текстовом поле **Paste a log / JSON sample**. Убедитесь, что он в формате JSON. Любые текстовые данные логов должны быть вставлены в поле `content` JSON.
   2. Запустите тест.

      Нажмите **Test the rule** и просмотрите результат в текстовом поле **Test result**.
7. Нажмите **Save changes**.
