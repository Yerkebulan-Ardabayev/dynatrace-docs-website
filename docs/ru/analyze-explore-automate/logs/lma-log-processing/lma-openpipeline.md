# Обработка логов с помощью OpenPipeline


* Latest Dynatrace
* Explanation
* 4-min read
* Updated on Oct 15, 2025

[OpenPipeline](../../../platform/openpipeline.md "Масштабируйте обработку данных платформы с помощью Dynatrace OpenPipeline.") -- это решение Dynatrace для обработки данных из различных источников. Оно обеспечивает удобную обработку данных любого масштаба и формата на платформе Dynatrace. Использование OpenPipeline при обработке логов в Dynatrace предоставляет мощное решение для управления, обработки и анализа логов. Этот подход сочетает традиционные возможности обработки логов с расширенными функциями обработки данных OpenPipeline для получения более глубокого понимания ваших данных логов.

Мы рекомендуем использовать обработку логов с помощью OpenPipeline как масштабируемое и мощное решение для управления, обработки и анализа логов. Если у вас нет доступа к OpenPipeline, используйте [классический конвейер обработки логов](../lma-classic-log-processing.md "Используйте правила обработки логов для преобразования входящих данных логов для лучшего понимания, анализа или дальнейшей трансформации.").

## Преимущества OpenPipeline

OpenPipeline предоставляет следующие преимущества:

* Контекстная трансформация данных: OpenPipeline извлекает данные с учётом контекста и преобразует их в более эффективные форматы, например, конвертирует логи в бизнес-события.
* Единый язык обработки: [Dynatrace Query Language (DQL)](../../../platform/grail/dynatrace-query-language.md "Как использовать Dynatrace Query Language.") используется в качестве языка обработки, предлагая единый синтаксис для всех функций Dynatrace и более продвинутые параметры обработки.
* Концепция конвейеров: вы можете разделить трафик приёма логов на различные конвейеры с выделенной обработкой, извлечением данных и метрик, разрешениями и хранилищем.
* Дополнительные процессоры: вы можете использовать дополнительные процессоры, например, для добавления или удаления полей. Полный список см. в разделе [Обработка в OpenPipeline](../../../platform/openpipeline/concepts/processing.md "Изучите основные концепции обработки Dynatrace OpenPipeline.").
* Расширенное извлечение данных: извлекайте бизнес-события из логов с расширенными возможностями извлечения данных.
* Увеличенные лимиты: воспользуйтесь увеличенными [лимитами](../lma-limits.md "Лимиты по умолчанию для последней версии Dynatrace Log Management and Analytics.") по умолчанию, включая размер содержимого до 524 288 байт, размер атрибута до 2 500 байт и до 250 атрибутов логов.
* Улучшенная производительность и более высокая пропускная способность.

## Этапы обработки логов

Ниже описаны этапы обработки логов с помощью OpenPipeline.

![logs-openpipeline](https://dt-cdn.net/images/logs-openpipeline-1566-74807e1b77.png)

Определённые поля исключаются из сопоставления и обработки или ограничены. Подробнее см. в разделе [Лимиты, специфичные для полей](../../../platform/openpipeline/reference/limits.md#fields "Справочные лимиты Dynatrace OpenPipeline.").

1

Данные остаются в исходной структурированной форме. Это важно для детального анализа и устранения неполадок, так как гарантирует, что никакая информация не будет потеряна или изменена.

2

Извлечённые метрики отправляются только в Grail, за исключением областей событий безопасности (новые) и конфигурации span.

Принятые логи направляются маршрутом по умолчанию во встроенные конвейеры для обеспечения хранения в Grail. Для обработки логов с помощью OpenPipeline встроенный конвейер -- это конвейер **default**, обеспечивающий хранение в Grail. Создавайте пользовательские маршруты и конвейеры для настройки обработки и хранения в OpenPipeline. Обработка основана на доступных записях и не учитывает обогащение записей из внешних сервисов.

Если вы создали пользовательские конвейеры и ваши логи направляются к ним по определению динамического маршрута, эти логи не обрабатываются конвейером по умолчанию. Если логи не направляются в пользовательские конвейеры, они обрабатываются конвейером по умолчанию.

## Включение технологических пакетов

OpenPipeline предоставляет [технологические пакеты](../../../platform/openpipeline/use-cases/tutorial-technology-processor.md "Настройте конвейер обработки для структурирования технологически специфичных логов в соответствии с семантическим словарём Dynatrace.") для распространённых технологий и форматов логов. Вы можете вручную включить необходимые технологические пакеты.

Чтобы включить технологические пакеты в OpenPipeline

1. Перейдите в ![Settings](https://dt-cdn.net/images/settings-icon-256-38e1321b51.webp "Settings") **Settings** > **Process and contextualize** > **OpenPipeline** > **Logs**.
2. Перейдите на вкладку **Pipelines** и выберите ![Add](https://dt-cdn.net/images/dashboards-app-menu-plus-7e9b7c3547.svg "Add")**Pipeline**, чтобы добавить новый конвейер.

   Если вы видите **Classic pipeline** на вкладке **Pipelines**, это означает, что вы всё ещё используете [классический конвейер обработки логов](../lma-classic-log-processing.md "Используйте правила обработки логов для преобразования входящих данных логов для лучшего понимания, анализа или дальнейшей трансформации."). Мы рекомендуем перенести все ваши правила обработки логов (то есть классический конвейер) на обработку логов с помощью OpenPipeline, создав необходимые конвейеры, как описано на этой странице.
3. Введите имя для нового конвейера.
4. Выберите ![Add](https://dt-cdn.net/images/dashboards-app-menu-plus-7e9b7c3547.svg "Add")**Processor** на вкладке **Processing** и выберите **Technology bundle**.
5. Выберите нужную технологию, а затем нажмите **Choose**.
6. Выберите **Run sample data**, чтобы протестировать конфигурацию и просмотреть результат.
7. Нажмите **Save**.

Для обработки логов необходимо включить динамическую маршрутизацию. Подробнее см. в разделе [Маршрутизация данных](../../../platform/openpipeline/getting-started/how-to-routing.md "Узнайте, как направлять данные в конвейер обработки OpenPipeline.").

## Добавление пользовательского конвейера

Чтобы создать пользовательский конвейер в OpenPipeline

1. Перейдите в ![Settings](https://dt-cdn.net/images/settings-icon-256-38e1321b51.webp "Settings") **Settings** > **Process and contextualize** > **OpenPipeline** > **Logs**.
2. Перейдите на вкладку **Pipelines** и выберите ![Add](https://dt-cdn.net/images/dashboards-app-menu-plus-7e9b7c3547.svg "Add")**Pipeline**, чтобы добавить новый конвейер.
3. Введите имя для нового конвейера.
4. Выберите одну из вкладок, представляющих этапы обработки логов: **Processing**, **Metric Extraction**, **Data extraction**, **Permission** или **Storage**.
5. Выберите ![Add](https://dt-cdn.net/images/dashboards-app-menu-plus-7e9b7c3547.svg "Add")**Processor** и выберите из доступных процессоров.
6. Для каждого процессора укажите имя и условие сопоставления. Дополнительные обязательные поля зависят от процессора и указаны в пользовательском интерфейсе.
7. Если доступно, выберите **Run sample data**, чтобы протестировать конфигурацию и просмотреть результат.
8. Нажмите **Save**.

Вы можете просмотреть или отредактировать любой конвейер, выбрав запись и внеся необходимые изменения.
