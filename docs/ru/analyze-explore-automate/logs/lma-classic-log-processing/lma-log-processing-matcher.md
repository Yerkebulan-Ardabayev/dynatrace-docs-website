С [Dynatrace на Grail](/docs/platform/grail/dynatrace-grail "Grail — это хранилище данных Dynatrace, специально разработанное для данных наблюдаемости и безопасности. Оно служит единым унифицированным хранилищем для логов, метрик, трейсов, событий и многого другого.") вы можете использовать функции и логические операторы [Dynatrace Query Language](/docs/platform/grail/dynatrace-query-language "Как использовать Dynatrace Query Language.") (DQL) в матчерах.

Матчер фильтрует входящие данные и сужает область данных, обрабатываемых создаваемым правилом. Вы можете использовать матчер в обработке логов и событий, метриках логов, событиях логов и бакетах логов для следующих целей:

* Фильтрация записей, содержащих указанную фразу.
* Поиск в данных логов определённого значения в заданном атрибуте.
* Проверка, является ли значение NULL.
* Использование логических операторов для соединения двух или более выражений.

  Чтобы узнать об использовании логических операторов в DQL, см. [Логические операторы и операторы равенства](/docs/platform/grail/dynatrace-query-language/operators#dql-logical-or-equality-operators "Список операторов DQL.").

## Функции

### matchesPhrase

Фильтрует записи, содержащие указанную фразу. Возвращает только совпадающие записи. Эта функция нечувствительна к регистру для символов ASCII, работает с многозначными атрибутами (совпадает с любым из значений), а символ звёздочки (`*`) является подстановочным знаком, относящимся только к одному слову, а не ко всему значению поля.

* **Валидация**
  Функция `matchesPhrase` выполняет нечувствительный к регистру поиск [contains](/docs/platform/grail/dynatrace-query-language/functions#contains "Список функций DQL.") для всей строки запроса и не поддерживает подстановочные знаки внутри строки.
  Для найденных результатов выполняется дополнительная валидация:

  + если запрос начинается с буквенно-цифрового символа, предшествующий символ должен быть небуквенно-цифровым.
  + если запрос заканчивается буквенно-цифровым символом, последующий символ должен быть небуквенно-цифровым.
  + если запрос начинается со звёздочки, валидация предшествующего символа не выполняется.
  + если запрос заканчивается звёздочкой, валидация последующего символа не выполняется.
* **Синтаксис**
  `matchesPhrase(expression, phrase [, caseSensitive])`
* **Параметры**
* **Пример**
  В этом примере вы добавляете фильтр, который находит записи логов, содержащие фразу `error` в их содержимом.

  ```
  matchesPhrase(content, "error")
  ```

### matchesValue

Ищет в записях определённое значение в заданном атрибуте. Возвращает только совпадающие записи. Эта функция нечувствительна к регистру для символов ASCII, работает с многозначными атрибутами (совпадает с любым из значений) и не поддерживает подстановочные знаки внутри значения.

* **Синтаксис**
  `matchesValue(expression, value [, caseSensitive])`
* **Параметры**
* **Пример**
  В этом примере вы добавляете фильтр записей, где атрибут `process.technology` содержит значение `nginx`.

  ```
  matchesValue(process.technology, "nginx")
  ```

### isNotNull

Проверяет, что значение не является NULL.

* **Синтаксис**
  `isNotNull(<value>)`
* **Пример**
  В этом примере мы фильтруем (выбираем) данные, где поле `host.name` содержит значение.

  ```
  isNotNull(`host.name`)
  ```

## Операторы

Логические операторы могут использоваться для соединения двух или более выражений. Ознакомьтесь с [Логическими операторами и операторами равенства](/docs/platform/grail/dynatrace-query-language/operators#dql-logical-or-equality-operators "Список операторов DQL."), чтобы узнать больше о поведении логических операторов в DQL.

### OR

Логическое сложение (ИЛИ).

* **Синтаксис**
  `<expression_1> or <expression_2>`
* **Пример**
  В этом примере вы добавляете матчер для фильтрации записей, где содержимое включает либо фразу `timestamp`, либо фразу `trigger`.

  ```
  matchesPhrase(content, "timestamp") or matchesPhrase(content, "trigger")
  ```

### AND

Логическое умножение (И).

* **Синтаксис**
  `<expression_1> and <expression_2>`
* **Пример**
  В этом примере вы добавляете матчер для фильтрации записей, где содержимое включает фразу `timestamp` и фразу `trigger`.

  ```
  matchesPhrase(content, "timestamp") and matchesPhrase(content, "trigger")
  ```

### NOT

Логическое отрицание (НЕ).

* **Синтаксис**
  `not <expression>`
* **Пример**
  В этом примере вы добавляете матчер для фильтрации записей, где содержимое не включает фразу `timestamp`.

  ```
  not matchesPhrase(content, "timestamp")
  ```

### Строгое равенство

[Логический оператор](/docs/platform/grail/dynatrace-query-language/operators "Список операторов DQL.") (`==`), указывающий на точное совпадение.

Типы данных должны быть идентичными. Однако если десятичное значение равно `0`, числа с плавающей запятой можно сравнивать с целочисленными данными. Например, `1==1.0`
Для строк поиск чувствителен к регистру.

В отличие от функции `matchesValue`, оператор `strict equality` выполняет сравнение с учётом регистра, не поддерживает подстановочные знаки и не работает с элементами, являющимися частью многозначных атрибутов.

* **Синтаксис**
  `<expression1> == <expression2>`

## Группировка

Вы можете создавать условные группировки с помощью скобок `( )`.

```
matchesValue(process.technology, "nginx") and ( matchesPhrase(content, "error") or matchesPhrase(content, "warn") )
```

## Повторное использование выражений

Все выражения матчера, используемые в событиях логов, метриках, обработке или конфигурациях бакетов, являются допустимым DQL. Это означает, что вы также можете использовать эти выражения вместе с командой DQL filter, например, в [просмотрщике логов](/docs/analyze-explore-automate/log-monitoring/analyze-log-data/log-viewer "Узнайте, как использовать просмотрщик логов Dynatrace для анализа данных логов.").
