## Приём через OneAgent

Рекомендуется

OneAgent — это рекомендуемый, мощный инструмент, который автоматически находит источники логов из широкого спектра технологий на множестве различных платформ, систем контейнерной оркестрации и операционных систем. Обратитесь к [матрице поддержки платформ и возможностей OneAgent](../../../ingest-from/technology-support/oneagent-platform-and-capability-support-matrix.md#other-modules "Узнайте, какие возможности поддерживаются OneAgent на различных операционных системах и платформах.") для просмотра поддерживаемых операционных систем.

Ознакомьтесь с вариантом использования OneAgent для приёма логов, чтобы узнать, как настроить мониторинг логов с помощью OneAgent для автоматического обнаружения и приёма логов с ваших хостов.

Мы рекомендуем использовать OneAgent для работы с логами, так как он обеспечивает следующие преимущества:

* Упрощённое инструментирование для хостов, процессов и кластеров Kubernetes.

  + Бесшовная установка на хостах, а Operator для Kubernetes обеспечивает первоклассную работу со встроенной наблюдаемостью логов.
  + Автоматическое обогащение логов контекстной информацией, такой как топология и метаданные Kubernetes.
  + Включение контекста трассировки в логи одним кликом для улучшения отслеживаемости.
* Автоматическое обнаружение критических логов в сочетании с гибкой настройкой пользовательских источников логов, обеспечивающее всестороннюю наблюдаемость.
* Расширенные возможности управления логами в масштабе, предлагающие конфигурации для форматов логов, маскирования конфиденциальных данных и фильтрации захвата и обработки.

## Автообнаружение данных логов

OneAgent автоматически обнаруживает файлы логов, обеспечивая сбор и анализ соответствующих логов для всех отслеживаемых процессов. OneAgent сканирует файловую систему и приложения, работающие на хосте, для обнаружения файлов и источников логов. Перейдите на страницу Автообнаружение контента логов, чтобы узнать о процессе автообнаружения.

После обнаружения источников логов OneAgent применяет соответствующие правила приёма логов. Эти правила определяют, как логи должны собираться, анализироваться и пересылаться на платформу мониторинга Dynatrace. Автообнаружение включает паттерны ротации логов.

OneAgent автоматически обнаруживает логи с хостов и собирает логи из систем контейнерной оркестрации Kubernetes и контейнеров Docker.

OneAgent начинает приём логов, как только его модуль логов впервые считывает файл логов. Фактическое время начала может зависеть от интервалов приёма и времени, необходимого для распространения конфигурации из среды в модуль логов.

### OneAgent для логов хостов

OneAgent упрощает управление логами, автоматически обогащая логи на основе контекста инфраструктуры и источника логов, а также обеспечивая включение обогащения трассировки одним кликом для улучшения устранения неполадок. Для начала мониторинга логов достаточно установки и настройки централизованных правил приёма логов в Dynatrace. OneAgent также предлагает расширенные функции для масштабируемого управления логами, включая фильтрацию, маскирование конфиденциальных данных, определение пользовательских источников логов, обнаружение паттернов ротации логов и централизованную конфигурацию для упрощения управления жизненным циклом.

## Пользовательские источники логов

Многие приложения генерируют логи в форматах или местоположениях, не охваченных стандартным механизмом автообнаружения. Вы можете добавить пользовательские источники логов, когда автоматическое обнаружение не распознаёт определённые файлы логов или когда необходимо мониторить логи приложений, не покрытых настройками по умолчанию. Настройте пользовательские источники логов при возникновении проблем с паттерном ротации или когда файл логов не соответствует требованиям детектора. Подробнее см. Пользовательский источник логов.

### Правила приёма логов

Обязательно

Настройка правил приёма логов — самый важный шаг в процессе конфигурации. Правила позволяют указать, какие автоматически обнаруженные и пользовательские логи принимаются, фильтруются и сохраняются. Правила приёма логов допускают настройку в соответствии с заданными условиями соответствия, такими как группа процессов или файл источника логов. Это обеспечивает правильное управление логами, принимаемыми из различных источников, и их интеграцию в систему мониторинга логов Dynatrace. (Включает автоматически обнаруженные и пользовательские логи).

Вы можете просмотреть источники логов, обнаруженные OneAgent, на странице **Host** или **Process** в Dynatrace. Для новых тенантов некоторые встроенные правила включены по умолчанию. Подробнее см. на странице Правила приёма логов.

Правила приёма логов применяются исключительно к OneAgent. Эти правила не распространяются на другие механизмы сбора логов.

## Настройки OneAgent

Мониторинг логов Dynatrace использует модуль логов OneAgent, включённый по умолчанию при всех установках OneAgent. Хотя мониторинг логов не требует специальной конфигурации, вы можете изменить некоторые параметры, доступные для модуля логов OneAgent.

### Включение мониторинга логов с помощью `oneagentctl`

Чтобы включить мониторинг логов в OneAgent, используйте `oneagentctl` с параметром `--set-app-log-content-access=true`.
Подробнее см. [Конфигурация мониторинга логов](../../../ingest-from/dynatrace-oneagent/oneagent-configuration-via-command-line-interface.md#log-monitoring "Узнайте, как выполнять некоторые задачи конфигурации OneAgent без необходимости переустановки.").

### Глобальные настройки OneAgent для мониторинга логов

1. Перейдите в **Settings** > **Log Monitoring** > **Advanced log settings**.
2. Настройте параметры и **сохраните изменения**.

### Настройки OneAgent для мониторинга логов на уровне хоста

1. Перейдите в **Hosts** и выберите ваш хост Linux.
2. На странице обзора хоста выберите **Ещё** (**...**) > **Настройки** в правом верхнем углу.
3. На странице **Настройки хоста** выберите **Log Monitoring** и **Advanced log settings**.
4. Настройте параметры и **сохраните изменения**.

## Проверка включения мониторинга логов

**Мониторинг логов Dynatrace** включён по умолчанию, но контролирует только возможность модуля логов OneAgent. Для фактического приёма и просмотра логов необходимо также настроить правила приёма логов. Если у вас возникли проблемы со сбором логов, убедитесь, что эта настройка не была отключена.

Выполните следующие шаги в вашей среде Dynatrace, чтобы проверить, включён ли **Мониторинг логов Dynatrace**:

1. Перейдите в ![Settings](https://dt-cdn.net/images/settings-icon-256-38e1321b51.webp "Settings") **Settings**.
2. Далее перейдите в **Collect and capture** > **General monitoring settings** > **Monitored technologies**.
3. Найдите **Log Monitoring** в списке поддерживаемых технологий и выберите  **Редактировать**.
4. Проверьте, включён ли переключатель **Включить мониторинг логов для всех OneAgent в вашей среде**, и включите его, если нет.

Если эта настройка отключена на глобальном уровне, вы можете включить **Мониторинг логов Dynatrace** на уровне хоста:

1. Перейдите в **Infrastructure & Operations** > **Hosts**.
2. Выберите запись хоста.
3. Перейдите в  > **Настройки хоста** > **Общие**.
4. Найдите **Log Monitoring** в списке поддерживаемых технологий.
5. Включите **Включить на этом хосте**.

## Обогащение логов

Как встроенная функция, OneAgent автоматически обогащает логи, добавляя контекст топологии, сохраняя информацию о трассировках и определяя уровни серьёзности. Подробнее см. Автоматическое обогащение логов.

## Альтернативы приёму через OneAgent

Вы можете использовать следующие альтернативы OneAgent для мониторинга данных логов:

* API приёма логов: сбор логов через API, когда невозможно установить OneAgent.
* Dynatrace Extensions: используйте настраиваемые дополнения для приёма логов и расширения наблюдаемости.
* Syslog: потоковая передача, контроль и управление файлами логов из различных системных компонентов.

## Приём логов недавнего прошлого при включении модуля логов

При включении мониторинга логов модуль логов Dynatrace принимает ограниченный объём данных недавнего прошлого из файлов логов.

* Если модуль логов может обнаружить поддерживаемый формат временной метки, он принимает последние 15 минут логов.
* Если формат временной метки не может быть определён, модуль логов принимает последние 10 МиБ файла.

Приём логов недавнего прошлого начинается, когда модуль логов впервые пытается прочитать файл логов. Это означает, что начальная точка может различаться для разных модулей логов в вашей среде. Кроме того, интервалы приёма модуля логов и время распространения конфигурации (от среды к модулю логов) влияют на фактическое время начала. Это встроенная функция, и никакая конфигурация не требуется.

## Файлы логов с устаревшими изменениями

Файлы логов, время последнего изменения которых превышает 7 дней, не отслеживаются OneAgent. Это имеет следующие последствия:

* Данные логов из этих файлов не принимаются.
* При добавлении нового содержимого файл рассматривается как новый. Если содержимое не содержит временных меток, OneAgent принимает последние 10 МиБ файла, что может включать ранее принятые данные. Это может привести к дублированию записей.
