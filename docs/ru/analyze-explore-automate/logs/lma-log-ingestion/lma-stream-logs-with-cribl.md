# Потоковая передача журналов с помощью Cribl

* Latest Dynatrace
* How-to guide
* 3-min read
* Published Jun 12, 2025

Вы можете отправлять журналы, метрики и трассировки в Dynatrace с помощью Cribl Stream через OpenTelemetry Protocol (OTLP) или отправлять только журналы с помощью Cribl Stream через HTTP и приём по API.

[Dynatrace Collector](/docs/ingest-from/opentelemetry/collector "Learn about the Dynatrace OTel Collector.") предлагает различные возможности приёма и преобразования данных, что делает его универсальным инструментом для обработки данных журналов из различных источников.

## Ключевые возможности

* **Несколько типов конечных точек:** подключение к Dynatrace Cloud (SaaS), ActiveGate или указание URL конечной точки вручную.
* **Защищённая аутентификация:** использование токенов API Dynatrace для безопасной передачи данных.
* **Постоянная очередь:** буферизация данных во время сбоев подключения для предотвращения потери данных.
* **Пользовательские HTTP-заголовки:** добавление информации для отслеживания или метаданных к передаваемым журналам.
* **Пересылка журналов, метрик и трассировок:** отправка всех типов телеметрических данных в Dynatrace.

## Развёртывание интеграции с использованием OpenTelemetry

Настройка прямой интеграции телеметрических данных через Cribl Stream OTLP Destination занимает всего несколько простых шагов:

1. **Получите ключ API для приёма телеметрических данных.**

   * Сгенерируйте новый токен с соответствующей областью действия. За подробностями обратитесь к документации Dynatrace.
2. **Настройте Cribl Stream OTLP Destination.**

   * В Cribl Stream перейдите в **Data > Destinations** и добавьте новое назначение **Dynatrace OTLP**.
   * Настройте конечную точку Dynatrace (SaaS или ActiveGate).
   * Укажите идентификатор среды Dynatrace и токен доступа API.
3. **Настройте маршрутизацию телеметрических данных.**

   * Создайте маршруты в Cribl Stream для направления телеметрических данных в Dynatrace OTLP Destination.
   * Разверните конфигурацию для начала отправки данных.
4. **Обрабатывайте входящие данные с помощью Dynatrace OpenPipeline.**

   * Обогащайте и контекстуализируйте данные.
   * Извлекайте метрики или создавайте бизнес-события из журналов, метрик и трассировок.

Дополнительные сведения о конфигурации см. в [документации по продукту Criblï»¿](https://docs.cribl.io/stream/destinations-dynatrace-otlp/).

## Развёртывание интеграции с использованием HTTP и API

Настройка прямой интеграции журналов через Cribl Stream HTTP Destination занимает всего несколько простых шагов:

1. **Получите ключ API для приёма журналов.**

   * Сгенерируйте новый токен с соответствующей областью действия. За подробностями обратитесь к документации Dynatrace.
2. **Настройте Cribl Stream HTTP Destination.**

   * В Cribl Stream перейдите в **Data > Destinations** и добавьте новое назначение **Dynatrace HTTP**.
   * Выберите тип конечной точки (Cloud, ActiveGate или Manual).
   * Укажите идентификатор среды Dynatrace и токен доступа API.
3. **Настройте маршрутизацию данных журналов.**

   * Создайте маршруты в Cribl Stream для направления данных журналов в Dynatrace HTTP Destination.
   * Разверните конфигурацию для начала отправки журналов.
4. **Обрабатывайте входящие журналы и события с помощью Dynatrace OpenPipeline.**

   * Обогащайте и контекстуализируйте данные.
   * Извлекайте метрики или создавайте бизнес-события из журналов.

Дополнительные сведения о конфигурации см. в [документации по продукту Criblï»¿](https://docs.cribl.io/stream/destinations-dynatrace-http/).
