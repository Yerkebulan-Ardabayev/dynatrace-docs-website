На этой странице описаны ограничения, применяемые при выполнении запросов и просмотре журналов в ![Logs](https://dt-cdn.net/images/logs-256-ae0a9ca67f.png "Logs") **Logs**, а также способы изменения ограничений по умолчанию.

## Строки результатов запроса (**Record limit**)

По умолчанию ![Logs](https://dt-cdn.net/images/logs-256-ae0a9ca67f.png "Logs") **Logs** отображает не более 1000 строк в качестве результата вашего запроса.

Вы можете увеличить значение **Record limit** до 50 000 строк, чтобы просматривать больше результатов в рамках одного запроса.

## Объём сканируемых данных за запрос (**Read data limit**)

![Logs](https://dt-cdn.net/images/logs-256-ae0a9ca67f.png "Logs") **Logs** считывает не более 500 ГБ данных за один запрос. Выполнение запроса прекращается после достижения этого ограничения.

Вы можете изменить значение **Read data limit** до нужного вам значения для сканирования большего объёма данных, однако установить неограниченное значение невозможно.

## Размер результирующих данных (**Result size limit**)

По умолчанию размер результирующих данных, возвращаемых запросом, ограничен 100 МБ.

Вы можете уменьшить значение **Result size limit** и выбрать значение от 1 до 100 МБ в соответствии с вашими потребностями.

## Настройка ограничений

Чтобы настроить ограничения для ваших запросов в ![Logs](https://dt-cdn.net/images/logs-256-ae0a9ca67f.png "Logs") **Logs**:

1. Перейдите в ![Logs](https://dt-cdn.net/images/logs-256-ae0a9ca67f.png "Logs") **Logs**.
2. В правом верхнем углу, рядом с кнопкой **Run query**, выберите  (**Actions menu**) >  **App settings**.
3. В панели **App settings** настройте нужные ограничения.
