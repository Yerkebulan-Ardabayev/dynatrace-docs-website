# Фильтрация с помощью фасетов


* Последняя версия Dynatrace
* Практическое руководство
* Чтение: 3 мин
* Опубликовано 15 июня 2025 г.

Фасеты -- это быстрые фильтры для данных логов.
Они соответствуют парам ключ-значение атрибутов логов, обнаруженных в вашей среде, и сгруппированы по категориям фасетов.
Наиболее важные идентификаторы полей DQL сгруппированы по умолчанию в предопределённых категориях.
Фасеты также помогают оценить объём данных логов, соответствующих каждому атрибуту.

## Фильтрация данных с помощью фасетов

Чтобы запросить логи в вашей среде с помощью фасетов

1. Перейдите в ![Logs](https://dt-cdn.net/images/logs-256-ae0a9ca67f.png "Logs") **Logs**.
2. Разверните группы фасетов, такие как "Core" или "Log source".
3. Разверните фасет, относящийся к вашему запросу, например "Status".
4. Выберите значения, относящиеся к вашему запросу, например "Error" и "Warning".
5. Обратите внимание на то, как фильтры генерируются в поле фильтра.
6. Выберите столько фасетов, сколько необходимо для вашего запроса.

   Внутри одного фасета выбранные значения объединяются с помощью оператора `OR`. Это означает, что будут включены логи, соответствующие любому из выбранных значений для этого фасета.
   Между различными фасетами значения объединяются с помощью оператора `AND`. Это означает, что логи должны соответствовать хотя бы одному значению из каждого выбранного фасета, чтобы быть включёнными.
7. Нажмите **Run query**, чтобы увидеть логи из вашей среды на основе вашего фильтра.

## Оценка объёма данных с помощью фасетов

Чтобы получить представление о том, сколько логов с определёнными атрибутами существует в вашей среде

1. Перейдите в ![Logs](https://dt-cdn.net/images/logs-256-ae0a9ca67f.png "Logs") **Logs**.
2. Разверните группы фасетов и фасеты, относящиеся к вашему запросу.
3. Посмотрите на приблизительные значения для каждого значения.

Числа, отображаемые для каждого значения фасета, представляют приблизительное количество логов на основе фильтров вашего последнего запроса. Если вы видите символ '~', это означает, что Dynatrace использует выборку при чтении данных логов для повышения отзывчивости.

Пример:

* Вы запрашиваете логи ошибок из пространства имён Kubernetes "astroshop" со следующими фильтрами: `k8s.namespace.name = "astroshop"` `status = "Error"`
* После выполнения запроса посмотрите на фасет `k8s.container.name`
* Вы увидите только имена контейнеров для логов, поступающих из пространства имён Kubernetes "astroshop" со статусом "Error"
* Число, отображаемое рядом с каждым именем контейнера, показывает приблизительное количество логов с пространством имён Kubernetes "astroshop", статусом "Error" и соответствующим именем контейнера

## Управление всеми фасетами

Чтобы управлять тем, какие фасеты вашей среды отображаются

1. Перейдите в ![Logs](https://dt-cdn.net/images/logs-256-ae0a9ca67f.png "Logs") **Logs**.
2. Нажмите на **Facets edit icon** рядом с панелью поиска фасетов.
3. Выберите или отмените выбор элементов из предопределённых категорий.

   * Установите флажок категории, чтобы выбрать или отменить выбор всех фасетов в категории.
   * Нажмите, чтобы просмотреть фасеты в категории и выбрать или отменить их выбор.
4. Нажмите **Save**.

## Возврат к настройкам по умолчанию

Если вы ранее изменяли фасеты, чтобы вернуться к настройкам по умолчанию для фасетов в вашей среде

1. Перейдите в ![Logs](https://dt-cdn.net/images/logs-256-ae0a9ca67f.png "Logs") **Logs**.
2. Нажмите на **Facets edit icon** рядом с панелью поиска фасетов.
3. Нажмите **Reset to default**.
4. Нажмите **Save**.
