Log Monitoring Classic

Dynatrace Log Monitoring позволяет преобразовывать входящие данные журналов для лучшего понимания, анализа или дальнейшей обработки на основе создаваемых вами правил обработки журналов.

Информация может записываться в самых разных форматах в зависимости от приложения или процесса, операционной системы и других факторов. Обработка журналов предлагает централизованный и гибкий способ извлечения ценности из необработанных строк журнала.

Например, с помощью обработки журналов можно извлекать числовые значения из строк журнала, преобразовывать их в метрики на платформе Dynatrace и включать в дашборды и обнаружение проблем.

Обработка журналов не влияет на потребление DDU при приёме журналов.

## Правила обработки журналов

Перейдите в **Settings** > **Log Monitoring** > **Processing**, чтобы просмотреть действующие правила обработки журналов, изменить порядок существующих правил и создать новые. Правила выполняются в том порядке, в котором они перечислены, сверху вниз. Этот порядок критически важен, поскольку предшествующее правило может влиять на данные журнала, используемые последующим правилом в его определении.

Разверните раздел **Details**, чтобы просмотреть определение правила. Правило обработки журнала состоит из следующих элементов:

* **Rule name** — Имя правила
* **Matcher** — Условие соответствия
* **Rule definition** — Определение правила

Любое правило можно включить или выключить в столбце **Active**.

## Встроенные правила

По умолчанию обработка журналов включает множество включённых встроенных правил, отвечающих за очистку или нормализацию данных журналов. Название каждого встроенного правила начинается с `[Built-in]`.

Эти правила нельзя изменять напрямую, однако их можно отключать, копировать их определения и создавать новые правила со своими изменениями.

## Добавление правила обработки журнала

Чтобы создать правило обработки журнала

1. Перейдите в **Settings** > **Log Monitoring** > **Processing**.
2. Выберите **Add rule**.
3. Введите имя правила обработки журнала.
4. Укажите запрос журнала в разделе **Matcher**.
   Поисковый запрос по журналу сужает доступные данные журнала для выполнения этого конкретного правила. Это тот же поисковый запрос, который используется в [поисковом запросе средства просмотра журналов](analyze-log-data/log-viewer.md#query-syntax "Узнайте, как использовать средство просмотра журналов Dynatrace для анализа данных журналов.").

   Сопоставление на основе предыдущих правил не поддерживается

   Условие соответствия работает с исходным набором данных до применения каких-либо правил обработки. Сопоставление записей, изменённых предшествующими правилами, не поддерживается. Например, изменённое поле в правиле 1, используемое для сопоставления в правиле 2, будет содержать исходное значение этого поля и не будет использовать изменённое поле из правила 1.
5. Укажите определение правила обработки.
   Определение правила обработки — это инструкция о том, как Dynatrace должна преобразовывать или изменять данные журнала, отфильтрованные **Log query**.

   Определение правила создаётся с использованием команд обработки журналов, функций и сопоставления с образцом, что позволяет добавлять, преобразовывать или удалять входящие записи журналов. Это даёт вам полный контроль над тем, как данные журналов представляются в Dynatrace Log Monitoring.
6. Проверьте правило обработки журнала.
   Можно проверить определение правила, загрузив образец журнала или вставив фрагмент образца журнала вручную в текстовое поле **Log sample**.

   1. Выберите образец журнала.

      * Если вы загружаете образец журнала, данные для тестирования правила — это результат совпадения с **Log query**.
      * Если вы вводите фрагмент данных журнала вручную, убедитесь, что он в формате JSON. Любые текстовые данные журнала должны быть вставлены в поле `content` JSON.
   2. Запустите тест.

      Выберите **Test the rule** и просмотрите результат в текстовом поле **Test result**.
7. Выберите **Save changes**.
