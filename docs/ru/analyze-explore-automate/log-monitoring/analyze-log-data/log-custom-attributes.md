# Пользовательские атрибуты журналов (Logs Classic)

# Пользовательские атрибуты журналов (Logs Classic)

* Учебное руководство
* Время чтения: 3 мин
* Обновлено 03 июля 2023 г.

Log Monitoring Classic

Dynatrace [автоматически определяет атрибуты](../../logs/lma-log-ingestion/lma-log-ingestion-via-api/lma-log-data-transformation.md#autoattributes "API получения журналов автоматически преобразует данные журналов в выходные значения для атрибута loglevel.") поступающих данных журналов. **Доступные атрибуты** позволяют быстро фильтровать данные таблицы результатов по конкретному атрибуту данных журнала.

Вы также можете определять собственные пользовательские атрибуты данных журналов, соответствующие вашему конкретному формату данных журнала. Аналогично автоматически обнаруживаемым атрибутам журналов, ваши пользовательские атрибуты журналов извлекаются из данных журнала при поступлении и становятся доступными в Dynatrace.

Вы можете использовать их в качестве фильтров в средстве просмотра журналов (параметры таблицы и атрибуты подробных сведений о записи журнала), в качестве измерений при создании метрик журналов и в качестве свойств при создании событий журналов.

Эти пользовательские атрибуты должны соответствовать атрибутам журнала в поступающих данных журнала, иначе они будут проигнорированы.

Dynatrace версии 1.226+

Dynatrace автоматически распознаёт атрибуты журнала, видимые в сведениях журнала и параметрах таблицы.

## Создание пользовательского атрибута журнала

1. Перейдите в **Settings** > **Log Monitoring** > **Custom attributes**.
2. Нажмите **Add custom attribute**.
3. Выберите **Log Monitoring** > **Custom attributes**, затем нажмите **Add custom attribute**.
4. Включите параметр **Show attribute values in side bar**. Это обеспечивает индексирование системой значений атрибутов и поиск журналов по ним.
5. Введите **Key**.
   Правила для ключей:

   * Ключ может содержать только буквенно-цифровые символы, символы подчёркивания ('\_'), дефисы ('-'), точки ('.') и двоеточия (':').
   * Ключ не может начинаться с дефиса.
   * Все символы должны быть из латинского алфавита без диакритических знаков; символы, такие как 'ö', не допускаются.

Вы можете проверить, входят ли пользовательские атрибуты, которые планируете добавить, в список [атрибутов типа «ключ-значение»](../acquire-log-data/logs-classic-ingestion-api.md#semantics "Узнайте, как Dynatrace получает данные журналов и каковы возможные ограничения такого получения.") для автоматического обнаружения. Если они там присутствуют, добавлять их в пользовательский список не нужно.

Dynatrace Log Monitoring позволяет определять пользовательские атрибуты индексируемых данных журнала для поступающих данных журнала.

## Пример

В данном примере вы выполните вызов API для получения журнала с JSON, содержащим следующие атрибуты журнала:

```
{



"timestamp": "2021-07-29T10:54:40.962165022Z",



"level": "error",



"source": "Skynet",



"application.id": "PaymentService-Prod",



"message": "PaymentService-Prod failure.",



"data": {}



}
```

Затем вы создадите пользовательский атрибут журнала и используете его для создания метрики журнала и события журнала.

1. Выполните вызов API.
2. Создайте пользовательский атрибут.

   ![Creating a custom log attribute.](https://dt-cdn.net/images/lm-custom-att-set-custom-att-1129-96179dda7e.png)
3. Просмотрите атрибут в средстве просмотра журналов. (Подробнее см. в разделе [Средство просмотра журналов (Logs Classic)](log-viewer.md "Узнайте, как использовать средство просмотра журналов Dynatrace для анализа данных журналов.").)

   * Проверьте раздел **Available attributes**

     Пользовательский атрибут показан как доступный фильтр атрибутов в средстве просмотра журналов

     ![Available attributes-based filter in a log viewer.](https://dt-cdn.net/images/lm-custom-att-lv-facets-r-1493-e08c666d5b.png)
   * Проверьте параметры таблицы

     Пользовательский атрибут отображается в параметрах таблицы в средстве просмотра журналов

     ![Custom attribute shown in table options in the log viewer.](https://dt-cdn.net/images/lm-custom-att-lv-table-options-r-352-1192136876.png)
   * Проверьте сведения о записи/элементе журнала

     Пользовательский атрибут отображается в записи журнала в средстве просмотра журналов в разделе дополнительных атрибутов событий

     ![Custom attribute shown in log record in a log viewer under additional event attributes.](https://dt-cdn.net/images/lm-custom-att-lv-details-r-1172-19c628f888.png)
