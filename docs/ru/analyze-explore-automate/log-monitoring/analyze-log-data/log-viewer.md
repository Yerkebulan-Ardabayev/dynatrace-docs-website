# Просмотр логов (Logs Classic)


* Пояснение

Log Monitoring Classic

Средство просмотра логов позволяет просматривать логи за определённый период времени, используя обнаруженные аспекты содержимого логов. Вы можете использовать **Available attributes** для сужения области просмотра и фокусировки на конкретном аспекте содержимого логов.

Чтобы открыть средство просмотра логов, перейдите в ![Logs and Events](https://dt-cdn.net/images/logs-and-events-512-4b43bbadbe.png "Logs and Events") **Logs & Events Classic**. Средство просмотра логов состоит из четырёх разделов:

## Поиск

В поле **Filter by** можно настроить фильтры для сужения списка событий логов, отображаемых в таблице результатов. Выберите **Advanced query**, чтобы редактировать запрос вручную.

* При отображённом **Filter by** (по умолчанию) фильтр работает в режиме автодополнения, в котором вы выбираете из набора обнаруженных полей данных логов для фильтрации результатов (ограничение — 10 различных атрибутов). Для фильтров с одинаковыми атрибутами достаточно выполнения хотя бы одного условия. Для фильтров с разными атрибутами все условия должны быть выполнены. В режиме автодополнения выбранные фильтры и **Available attributes** синхронизируются автоматически.

  При использовании поискового запроса в простом режиме:

  + Если искомые имена атрибутов совпадают, применяется оператор **OR**. Результаты поиска включают элементы, соответствующие любому из указанных имён атрибутов. Это обеспечивает более широкий поиск, допуская вариации в именах атрибутов.
  + Если искомые имена атрибутов различаются, применяется оператор **AND**. Результаты поиска включают только элементы, одновременно соответствующие **всем** указанным именам атрибутов. Это сужает поиск до элементов, соответствующих нескольким критериям.
* При выбранном **Advanced query** вы можете задать более сложные критерии для событий логов, используя комбинации ключевых слов, фраз, логических операторов и скобок (ограничение — 10 различных атрибутов). Язык поисковых запросов Dynatrace обеспечивает полную гибкость при поиске по содержимому логов. Вы можете использовать поле ввода запроса для быстрого текстового поиска по содержимому данных логов. Любая строка, введённая в текстовое поле запроса без указания атрибута данных логов, будет рассматриваться как простой текстовый поиск по содержимому данных логов.

При фильтрации по содержимому логов можно использовать только полные токены. Содержимое логов разбивается на токены в соответствии с правилами [сегментации текста Unicode](https://unicode.org/reports/tr29/). В качестве альтернативы можно использовать подстановочный знак \* в конце токена.

Вы можете включать и выключать **Advanced query** для переключения между режимами автодополнения и расширенным. Dynatrace преобразует фильтры автодополнения в запрос и наоборот, при условии что запрос в расширенном режиме поддаётся преобразованию. Некоторые сложные запросы с логическими операторами не могут быть преобразованы в фильтры автодополнения, и в этом случае переключение в режим автодополнения становится недоступным.

В расширенном режиме можно выполнить пустой запрос для получения нефильтрованных данных логов.

## Диаграмма

Диаграмма логов представляет собой гистограмму событий логов во времени, которая даёт быстрый обзор логов и их уровней серьёзности в выбранном временном интервале.

## Таблица результатов

Таблица результатов под диаграммой отображает события логов, соответствующие заданному запросу и фильтру в выбранном временном интервале. Каждая строка таблицы представляет запись лога и может быть развёрнута для отображения подробных данных лога. Отображаются первые 100 совпадающих записей, но вы можете просмотреть больше результатов, выбрав **Show 100 more** в нижней части таблицы.

По умолчанию принятые записи логов сортируются по временной метке, а затем в порядке, сохранённом в источнике лога, где источник лога — это удалённый процесс, записывающий в конечную точку REST API, или удалённый процесс, на котором обнаружены логи.

По умолчанию средство просмотра логов отображает максимум 1 000 событий логов. Если вы не видите ожидаемых результатов, выполните более точный запрос или сузьте временной интервал для получения более сфокусированных данных логов.

Чтобы показать или скрыть определённые столбцы в таблице результатов

1. Выберите **Format table**.
   Это отобразит список сгенерированных Dynatrace и зарезервированных атрибутов логов, которые можно добавить в таблицу результатов для наглядности и использования в качестве измерений при создании метрики лога. Например, вы можете использовать атрибут `dt.entity.process_group` для отображения экземпляра группы процессов, для которого произошло событие лога.
2. Установите или снимите флажки, чтобы показать или скрыть соответствующие столбцы в таблице.

Чтобы экспортировать данные таблицы

1. Выберите **Actions**.
2. Выберите **Download table (JSON)** или **Download table (CSV)** в зависимости от нужного формата.
   Хотя ваш поисковый запрос может вернуть более 1 000 записей логов, таблица результатов отобразит только первые 1 000 записей. Соответственно, экспортированные данные таблицы будут содержать только 1 000 записей логов, видимых в таблице. Экспортированные записи логов будут включать полные данные для каждой записи, даже если они не отображаются в столбце таблицы.

## Доступные атрибуты

**Available attributes** (отображаются слева от таблицы) предоставляют обзор и возможность фильтрации данных логов. **Available attributes** — это автоматически обнаруженные атрибуты данных, представленных в таблице. Вы можете использовать их для быстрой фильтрации данных таблицы результатов по конкретному атрибуту данных логов. Каждый доступный атрибут отображает до десяти наиболее популярных значений для этого атрибута. Чтобы отфильтровать все значения определённого атрибута, создайте и выполните запрос в средстве просмотра логов.

## Подробности лога

Уникальные атрибуты данных логов (атрибуты с высокой кардинальностью), такие как `span_id` и `trace_id`, генерируют избыточно длинные списки доступных атрибутов, которые могут повлиять на производительность средства просмотра логов. Поэтому они не отображаются в **Available attributes**. Тем не менее вы по-прежнему можете использовать их в расширенном поисковом запросе.
