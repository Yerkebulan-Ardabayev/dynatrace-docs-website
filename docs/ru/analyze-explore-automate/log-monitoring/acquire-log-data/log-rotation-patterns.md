# Паттерны ротации логов (Logs Classic)


* Пояснение
* Чтение: 3 мин
* Обновлено 18 января 2023 г.

Log Monitoring Classic

Dynatrace отслеживает паттерны ротации лог-файлов. Эта функциональность обеспечивает полноту процесса чтения файлов, даже если OneAgent временно отключён или приложение аварийно завершило работу. Процесс мониторинга имеет два измерения:

* [Отслеживание ротации по имени файла](log-rotation-patterns.md#filenametracking "Узнайте о поддерживаемых паттернах ротации логов")
* [Отслеживание ротации по типу генерации лога](log-rotation-patterns.md#gennametracking "Узнайте о поддерживаемых паттернах ротации логов")

## Отслеживание ротации по имени файла

Как правило, ротированные лог-файлы содержат специфические идентификаторы ротации, встроенные в имена файлов. Идентификаторами ротации могут служить добавленная метка времени или счётчик. Например:

* Активный лог-файл без идентификатора ротации:
  `/var/log/application.log`
* Ротированные/архивные лог-файлы с добавленными идентификаторами в виде метки даты:
  `/var/log/application.log.20220509`
  `/var/log/application.log.20220508`
  …
  `/var/log/application.log.20220503`
* Ротированные/архивные лог-файлы с добавленными идентификаторами в виде счётчика:
  `/var/log/application.log.1`
  `/var/log/application.log.2`

### Идентификатор ротации и имя лога

В зависимости от типа файла Dynatrace может заменять идентификатор ротации символом `#` в процессе автообнаружения логов.

#### Пример замены идентификатора ротации символом решётки

Активный лог-файл:
`/var/log/application20220509_060000.log`

Ротированные/архивные лог-файлы:
`/var/log/application20220509_030000.log`
`/var/log/application20220508_000000.log`
`/var/log/application20220508_210000.log`
…
`/var/log/application20220503_090000.log`

определяются как `/var/log/application#_#.log`

### Поддерживаемые идентификаторы ротации

Типы данных, которые могут быть заменены символом `#`, включают:

* Десятичные числа любой длины
* Шестнадцатеричные числа длиной не менее восьми цифр
* Глобально уникальные идентификаторы (GUID) — 128-битные текстовые строки, представляющие идентификационный номер (ID)

Чтобы быть включённым в мониторинг, ротированный лог-файл должен содержать идентификатор ротации, которому **и** предшествует, **и** следует один из предопределённых разделителей, включая `.` (точка), `-` (дефис), `_` (нижнее подчёркивание), `()` (скобки) или `[]` (квадратные скобки).
В сценарии, когда идентификатором ротации является десятичное число, достаточно, чтобы ему либо предшествовал, **либо** следовал предопределённый разделитель.

Помимо явных идентификаторов ротации, видимых под символом `#`, Dynatrace неявно поддерживает мониторинг ротированных файлов, которые:

* Содержат десятичный суффикс, добавленный в конец имени лога
* Содержат десятичный инфикс вместо одного из предопределённых разделителей (`.`, `-`, `_`, `()`, `[]`)

#### Примеры обнаруженных ротированных лог-файлов с неявными идентификаторами

Активный лог-файл:
`/var/log/application.log`
Dynatrace обнаруживает файл с именем '/var/log/application.log' и идентифицирует ротированные файлы во всех приведённых ниже примерах:

`/var/log/application.log.1`
`/var/log/application.log.2`
…
`/var/log/application.log.9`
или
`/var/log/application.20220509.120000.log`
`/var/log/application.20220509.060000.log`
`/var/log/application.20220508.120000.log`
…
`/var/log/application.20220503.120000.log`
или
`/var/log/application.20220509.log.1`
`/var/log/application.20220509.log.2`
`/var/log/application.20220508.log.1`
…
`/var/log/application.20220503.log.1`
`/var/log/application.20220503.log.2`

## Отслеживание ротации по типу генерации лога

Dynatrace поддерживает основные типы генерации логов, включая:

* Ротированная генерация логов по правилу **'Index'** (Индекс), при которой приложение открывает файлы последовательно и присваивает им индексные номера.
* Ротированная генерация логов по правилу **'Rename and replace'** (Переименование и замена), при которой приложение ведёт запись в файл с одним и тем же именем. Если критерий ротации выполнен (например, достигнут требуемый размер файла), открывается новый файл с тем же именем, а старый переименовывается.
* Ротированная генерация логов по правилу **'Copy and truncate'** (Копирование и усечение), при которой приложение ведёт запись в файл с одним и тем же именем, однако сама ротация выполняется внешним процессом, который после выполнения критерия ротации копирует содержимое файла в другое место и очищает содержимое исходного (активного) файла. Примером здесь служит процесс 'logrotate' в Linux.
