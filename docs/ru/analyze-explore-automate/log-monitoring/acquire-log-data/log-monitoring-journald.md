Log Monitoring Classic

OneAgent версии 1.307+

Модуль журналов умеет читать, обрабатывать и загружать записи из systemd-journald — централизованной службы ведения журналов в Linux. События и записи журналов обычно хранятся в файлах в каталоге `/var/log/journal/'machine-id'/`, где `machine-id` — длинная строка цифр, идентифицирующая машину. Эти файлы хранятся в бинарном сжатом формате, управляемом движком systemd-journald для контроля размера файлов, общего объёма, хранения, ротации и т. д. в соответствии с его настройками.

## Принцип работы Journald

Чтобы избежать проблем, связанных с ручным отслеживанием файлов, таких как ротация, распаковка или разбор, модуль журналов использует системные вызовы API для доступа ко всей функциональности. Это достигается путём открытия системной библиотеки `libsystemd.so` и её зависимостей, отображения выбранных функций из библиотеки и создания объекта контекста для взаимодействия с системой Journald. Таким образом, модуль журналов становится клиентом Journald и может использовать разнообразные предоставляемые им функции.

В начале работы модуль журналов открывает обработчик по умолчанию в расположении контекста Journald. Конкретный путь не указывается — обработчик открывается по своему стандартному расположению. Затем агент считывает записи поочерёдно, отслеживая текущую позицию.

## Включение поддержки Journald

Чтобы включить обнаружение журналов Journald, выполните следующие шаги:

1. Перейдите в **Settings** > **Log Monitoring** > **Log module feature flags**.
2. Выберите **Enable Journald log detector**.
3. Выберите **Save changes**.

Включение флага функции обязательно для активации данной возможности. Без него журналы Journald не будут появляться в Dynatrace даже при настроенных правилах приёма.

## Настройка приёма журналов из Journald

Вы можете включить приём журналов из Journald, либо настроив правило приёма Journald по умолчанию (ingest rule), либо настроив загрузку в хранилище тенанта, например `kubelet.service` как `Journald.unit` из Journald.

### Включение правила приёма по умолчанию

Для новых аккаунтов правило приёма Journald по умолчанию уже включено, и настройка, описанная в этом разделе, не требуется.

Выполните следующие шаги для настройки правила приёма Journald по умолчанию:

1. Перейдите в **Settings** > **Log Monitoring** > **Log ingest rules**.
2. Включите правило **[Built-in] Ingest Journald logs** для приёма всех журналов Journald.
3. Выберите **Save changes**.

### Создание пользовательского правила приёма

Выполните следующие шаги для настройки загрузки в хранилище тенанта для журналов `kubelet.service` из Journald:

1. Перейдите в **Settings** > **Log Monitoring** > **Log ingest rules**.
2. Выберите **Add rule** и укажите заголовок для вашей конфигурации.
3. Выберите **Add matcher**. Это первое условие сопоставления для двух указанных групп процессов.
4. Из списка **Attribute** выберите **Journald Unit**.
5. Выберите **Add value** и введите `kubelet.service`.
6. Выберите **Add matcher** снова для сопоставления указанного источника данных журналов.
7. Из списка **Attribute** выберите **Log source**.
8. Выберите **Add value** и введите **Journald** в качестве значения.
9. Выберите **Save changes**.
