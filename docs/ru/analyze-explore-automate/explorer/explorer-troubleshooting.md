# Вопросы и ответы по Data Explorer


Эта страница относится к Data Explorer — классической функции Dynatrace, тесно интегрированной с классическими информационными панелями, созданными с помощью [![Dashboards Classic](https://dt-cdn.net/images/dashboards-classic-512-15764940e8.png "Dashboards Classic") **Dashboards Classic**](../dashboards-classic.md "Узнайте, как создавать, управлять и использовать Dynatrace Dashboards Classic."). Перейдите на последнюю версию Dynatrace, чтобы воспользоваться новейшими возможностями изучения данных.

Почему я вижу не все серии своей метрики?

* *По умолчанию* на каждую метрику отображается `20` серий. Следовательно, некоторые серии могут отсутствовать в Data Explorer. Чтобы гарантировать отображение нужных данных серий, используйте более конкретные фильтры, например фильтр по зоне управления или имени сущности.
* *Максимальное* количество отображаемых серий на метрику составляет `100`. Обратите внимание, что это ограничение действует, даже если вы удалите [преобразование **limit**](../../dynatrace-api/environment-api/metric-v2/metric-selector.md#limit "Настройка селектора метрик для Metric v2 API.") из селектора метрик на вкладке **Code**.

Если данные серий для выражения метрики отсутствуют, см. раздел [Почему результат моего выражения метрики пуст?](../../dynatrace-api/environment-api/metric-v2/metric-faq.md#empty-result-metric-expression "Часто задаваемые вопросы об API метрик v2.").

Почему некоторые ячейки таблицы пусты, хотя в них должны быть значения?

Первопричина этой проблемы часто та же, что и для вопроса [Почему я вижу не все серии своей метрики?](#missing-series)

Серии метрик ограничены определённым числом.

Предположим, вы запрашиваете `builtin:host.cpu.usage` и `builtin:host.cpu.idle` с разбивкой по `dt.entity.host`. Для обеих метрик по умолчанию запрашиваются топ 100 хостов. Но топ 100 хостов для метрики `builtin:host.cpu.usage`, вероятно, отличается от топ 100 хостов для метрики `builtin:host.cpu.idle`, что приводит к пустым ячейкам в таблице для некоторых хостов.

Почему в Data Explorer отображаются другие значения доступности, чем на страницах хостов и процессов?

Метрики `builtin:host.availability` и `builtin:pgi.availability` основаны на данных временных рядов, предоставляемых OneAgent, тогда как значения доступности на страницах хостов и процессов вычисляются по специальным событиям. Поэтому значения могут расходиться.

В будущем доступность, отображаемая на страницах хостов и процессов, также будет основана на метриках доступности.

Почему значения на графике падают в текущий момент времени?

Если на графиках наблюдается вводящее в заблуждение падение значений в конце (текущее время), можно вручную скорректировать временной диапазон, вычтя один или два последних временных интервала.

Временная корректировка временного диапазона:

1. Откройте селектор временного диапазона.
2. Вычтите один-два интервала из конца временного диапазона. Например, измените `-2h` на `-2h-2m`.

Установка скорректированного временного диапазона в качестве временного диапазона по умолчанию для информационной панели:

1. Перейдите в режим редактирования информационной панели.
2. В разделе **Edit dashboard** выберите вкладку **Settings**.
3. Включите **Default timeframe**.
4. Выберите и отредактируйте временной диапазон. Например, измените `-2h` на `-2h-2m`.
5. Нажмите **Done**.

Обсуждение этой проблемы см. в записи [Correction required on Dashboard Charts showing dip at current time](https://community.dynatrace.com/t5/Dynatrace-product-ideas/RFE-Correction-required-onDashboard-Charts-showing-dip-at/idi-p/144070) в сообществе Dynatrace.

Для получения дополнительной информации о временных диапазонах см. раздел Настройки временного диапазона и зоны управления информационных панелей Dynatrace.
