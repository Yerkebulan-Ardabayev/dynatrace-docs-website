# Отправка уведомлений в Slack при возникновении проблем

* Последнее Dynatrace
* Учебник
* 4-минутное чтение
* Обновлено 19 октября 2025 года

Проблемы автоматически открываются Dynatrace, когда обнаруживаются аномалии или условия оповещения в вашей среде.
В ![Workflows](https://dt-cdn.net/images/workflows-1024-b5708f3cf9.webp "Workflows") **Workflows** создайте простой рабочий процесс, который слушает проблемы и автоматически отправляет уведомления в Slack при возникновении новой проблемы.
Этот гид показывает вам, как это сделать.

## Что вы узнаете

В этом учебнике вы узнаете, как оповестить свою команду в режиме реального времени, автоматически отправляя сообщения с подробностями новой проблемы в определенный канал Slack.

В кратком обзоре вы:

1. [Создадите простой рабочий процесс](../workflows/simple-workflow.md#create-simple-workflow "Создайте и запустите простой рабочий процесс.").
2. Добавите триггер события для [проблем Davis](../workflows/trigger/event-trigger.md#davis-problem-trigger "Руководство по созданию триггеров автоматизации рабочих процессов в Dynatrace Workflows.").
3. Настройте сообщение в Slack.
4. Сохраните и запустите рабочий процесс, чтобы получить уведомления по электронной почте.
5. Проверьте, что ваш рабочий процесс работает как ожидается.

## Предварительные условия

* У вас должно быть разрешение на настройку и запуск простого рабочего процесса.
  Например, разрешение, предоставленное по умолчанию, для [стандартного пользователя](../../manage/identity-access-management/permission-management/default-policies.md "Dynatrace справочник по умолчанию").
* Вы должны выбрать необходимые разрешения в [настройках авторизации](../workflows/security.md#authorization-settings "Руководство по аспектам безопасности автоматизации рабочих процессов в Dynatrace Workflows").

  + Вы должны разрешить необходимые разрешения для

    - Доступа к ![Workflows](https://dt-cdn.net/images/workflows-1024-b5708f3cf9.webp "Workflows") **Workflows**.
    - Создания и выполнения рабочего процесса.
      Для получения дополнительной информации см. [настройки авторизации](../workflows/security.md#user-permission "Руководство по аспектам безопасности автоматизации рабочих процессов в Dynatrace Workflows").
* Вы настроили интеграцию с Slack.

## Шаги

1. [Создайте простой рабочий процесс](../workflows/simple-workflow.md#create-simple-workflow "Создайте и запустите простой рабочий процесс.").

   1. Перейдите к ![Workflows](https://dt-cdn.net/images/workflows-1024-b5708f3cf9.webp "Workflows") **Workflows**.
   2. Выберите ![Add](https://dt-cdn.net/images/dashboards-app-menu-plus-7e9b7c3547.svg "Add") **Рабочий процесс** в правом верхнем углу страницы.
   3. Выберите название рабочего процесса.
      По умолчанию оно равно `Untitled workflow`, и введите осмысленное имя.
      Тип рабочего процесса по умолчанию установлен в простой рабочий процесс.
2. Добавьте триггер события для [проблем Davis](../workflows/trigger/event-trigger.md#davis-problem-trigger "Руководство по созданию триггеров автоматизации рабочих процессов в Dynatrace Workflows.").

   1. В разделе **Выберите триггер** выберите триггер [проблемы Davis](../workflows/trigger/event-trigger.md#davis-problem-trigger "Руководство по созданию триггеров автоматизации рабочих процессов в Dynatrace Workflows.").
   2. Установите **Состояние проблемы** в **активное или закрытое**.
      Этот вариант означает, что проблема может быть как активной, так и закрытой.
      Это настройка вызывает запуск рабочего процесса дважды, один раз, когда проблема становится активной, и снова, когда она закрывается.
   3. В выпадающем списке **Категория события** выберите **Выбрать все**.
   4. Необязательно выберите **Запрос прошлых событий**, чтобы увидеть последние события проблем, которые бы вызвали этот рабочий процесс.
   5. Необязательно введите **Теги сущностей** или **Дополнительный пользовательский запрос фильтра**, чтобы запустить рабочий процесс только для соответствующих проблем.
3. Настройте сообщение в Slack.

   1. Выберите ![Add](https://dt-cdn.net/images/dashboards-app-menu-plus-7e9b7c3547.svg "Add") **Добавить задачу** на узле триггера.
   2. В разделе **Выберите действие** выберите ![Slack Connector](https://dt-cdn.net/images/slack-for-workflows1-257-4ad7b09fd3.png "Slack Connector") **Отправить сообщение** тип действия.
      Дайте действию осмысленное название.
   3. Выберите предварительно настроенное подключение к Slack.
   4. Выберите подключение к Slack из выпадающего списка **Подключение**.
   5. Выберите канал Slack для вашего сообщения из выпадающего списка **Канал**.
   6. В поле **Сообщение** введите следующее:

      ```
      {



      "blocks": [



      {



      "type": "header",



      "text":).



      "type": "plain_text",



      "text": "{{ ':white_check_mark:' if event()['event.status'] == 'CLOSED' else ':warning:' }} {{ 'RESOLVED' if event()['event.status'] == 'CLOSED' else 'OPEN' }} - {{ event()['event.name']}}",



      "emoji": true



      },



      {



      "type": "section",



      "text":.



      "type": "mrkdwn",



      "text": "- *Ссылка на проблему*: <{{ environment().url }}/ui/intent/dynatrace.davis.problems/view-problem#%7B%22event.id%22%3A%22{{ event()['event.id'] }}%22,%22event.kind%22%3A%22{{event()['event.kind']}}%22%7D|{{ event()['display_id'] }}> \n- *Пострадавшие сущности:* `{{ event()['affected_entity_ids'] }}`\n- *Продолжительность проблемы:* `{{ (event().get('resolved_problem_duration', 0) | int) / 1000000 / 1000 / 60 }} минуты"



      },



      {



      "type": "section",



      "text":.



      "type": "mrkdwn",



      "text": {{ ('>' ~ event()['event.description']) | replace('\n', '\n>') | to_json }}



      },



      {



      "type": "divider"



      },



      {



      "type": "section",



      "text":.



      "type": "mrkdwn",



      "text": "*Ссылка на рабочий процесс*: <{{ environment().url }}/ui/apps/dynatrace.automations/workflows/{{ execution().workflow.id }}|Рабочий процесс>"



      },



      ]



      }
      ```

      Эта конфигурация использует контекстные заполнители событий, чтобы заполнить сообщение в Slack подробностями проблемы динамически.

      Триггер проблем Davis возвращает запись проблемы.
      Вы можете использовать любое поле из записи проблемы, хранящейся в `dt.davis.problems`, в сообщении Slack.
4. Сохраните и запустите рабочий процесс, чтобы отправить уведомления в Slack.

   1. Выберите **Создать черновик**.
   2. Выберите **Развернуть**.
   3. Выберите **Запустить**, чтобы увидеть выбранное событие проблемы, которое будет использоваться для рабочего процесса.
5. Проверьте, что ваш рабочий процесс работает как ожидается:

   1. Перейдите к вашему рабочему процессу.
   2. Выберите **Запустить**.
   3. Выберите **Запустить** еще раз, чтобы выполнить рабочий процесс.

      Журналы выполнения недоступны для простого рабочего процесса.
      Если возникает ошибка, вы можете найти подробности об ошибке в правом углу в панели задач.

## Вывод

Вы создали простой рабочий процесс, который отправляет сообщения в Slack, когда проблемы открываются или закрываются.
Эта настройка помогает обеспечить, чтобы ваша команда была сразу же проинформирована о критических проблемах в вашей среде.

Вы можете расширить этот рабочий процесс, добавив:

* Условия для обработки конкретных категорий или уровней серьезности проблем.
* Автоматические шаги для исправления в ваш рабочий процесс.

Этот рабочий процесс является отличной отправной точкой для автоматизации реагирования на инциденты и улучшения оперативной осведомленности.
