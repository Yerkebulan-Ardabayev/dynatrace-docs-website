# Диаграмма-индикатор (Meter bar)


* Latest Dynatrace

Когда использовать визуализацию meter bar

Используйте meter bar для визуализации одного числового значения в виде индикатора прогресса.

## Примеры

### Пример 1

![Визуализация Meter Bar: пример 1](https://dt-cdn.net/images/meter-bar-example-01-478-739cd09225.png)

Визуализация meter bar выше основана на следующем запросе, который вычисляет `CPU` как среднее использование ЦП хоста. В настройках **Data mapping** значение **Meter value** затем устанавливается как `CPU`.

```
timeseries avg(dt.host.cpu.usage)


| fieldsAdd CPU = arrayAvg(`avg(dt.host.cpu.usage)`)


| fieldsKeep CPU
```

### Пример 2

![Визуализация Meter Bar: пример 2](https://dt-cdn.net/images/meter-bar-example-02-474-3cf8cba784.png)

Визуализация meter bar выше основана на следующем запросе, который вычисляет `pctActive` как процент от общего числа проблем, которые открыты. В настройках **Data mapping** значение **Meter value** затем устанавливается как `pctActive`.

```
fetch dt.davis.problems


| summarize count = count(), by:{event.status}


| summarize { active=toDouble(takeAny(if(event.status=="ACTIVE", count))), closed=toDouble(takeAny(if(event.status=="CLOSED", count)))}


| fieldsAdd total = active + closed


| fieldsAdd pctActive = (active / total) * 100


| fieldsKeep pctActive
```

## Взаимодействие с диаграммой

### Взаимодействие при выборе

Когда вы выбираете значение на диаграмме и закрепляете отображаемую всплывающую подсказку, вы можете навести на неё курсор для отображения меню параметров, специфичных для выбора.

Доступные взаимодействия зависят от вашего запроса и визуализации. Например, если вы выбираете хост на линейной диаграмме и наводите курсор на всплывающую подсказку, вы увидите меню с такими пунктами, как:

* **Copy name** — копирование имени выбранного хоста.
* **Fields** — раздел с подменю для каждого поля запроса. Подменю поля предлагает специфичные для поля параметры, такие как:

  + **Copy value** — копирование значения поля. Также отображает тип поля.
  + **Hide** — скрытие поля на диаграмме.
  + **Explain value** — использование ИИ для объяснения поля.
  + **Add command to query** — раздел команд, специфичных для поля, которые можно автоматически добавить к вашему запросу.
  + Также может быть указано рекомендуемое приложение.
* **Visual options** — открытие панели редактирования для изменения параметров визуализации выбранного элемента.
* **Set color** — открытие панели редактирования для изменения цвета выбранного элемента.
* ![Infrastructure & Operations](https://dt-cdn.net/images/infrasctucture-operations-highresolution-1025-07d1bc45b5.png "Infrastructure & Operations") **Go to host** — открытие выбранного элемента в ![Infrastructure & Operations](https://dt-cdn.net/images/infrasctucture-operations-highresolution-1025-07d1bc45b5.png "Infrastructure & Operations") **Infrastructure & Operations**.

  В общем случае, если есть рекомендуемые приложения для открытия выбранного элемента, меню предлагает прямые ссылки на эти приложения, а затем пункт **Open with** для выбора другого целевого приложения.
* **Open with** — подробнее см. [Детализация и навигация](../drilldowns-and-navigation.md "Детализация из дашбордов и блокнотов с помощью ссылок на основе интентов или URL-адресов.").

## Заголовок

Используйте поле заголовка в верхней части панели параметров (изначально `Untitled tile` или `Untitled section`), чтобы добавить заголовок к плитке дашборда или разделу блокнота.

* Вы можете использовать эмодзи, такие как ð, ð и â¤ï¸.
* Вы можете использовать переменные.

**Пример:**

1. Определите переменные `Status` и `Emoji` в вашем дашборде.
2. Установите заголовок `Current $Emoji status is $Status`.
3. Установите `Status` в значение `Good`.
4. Установите `Emoji` в значение `ð`.

Заголовок будет отображаться как `Current ð status is Good`.

## Сопоставление данных

Раздел сопоставления данных показывает, как столбец вашего результата сопоставлен с визуализацией.

Общие правила настройки сопоставления данных (раскрыть)

Разверните раздел **Data mapping** в настройках визуализации, чтобы увидеть, как данные в вашем результате сопоставлены с визуализацией, и при необходимости изменить эти настройки.

* Обязательные поля отмечены звёздочкой (`*`). Пример:

  ![Пример сопоставления данных: линейная диаграмма](https://dt-cdn.net/images/example-data-mapping-line-chart-1-732-eb9cca00d8.png)
* Типы данных отображаются рядом с именами полей в выпадающих списках и сопоставленных полях.
* Единицы измерения отображаются, когда назначена только одна единица.
* Поля результатов сгруппированы в **Suitable** и **Unsuitable**. Поля отмечаются как неподходящие, если они не могут быть использованы для отображения данных в визуализации. Пример:

  ![Пример сопоставления данных: линейная диаграмма, выпадающий список Time](https://dt-cdn.net/images/example-data-mapping-line-chart-2-763-9cc593d261.png)
* Автоматическое применение настроек сопоставления данных по умолчанию:

  Dynatrace версии 1.319+

  + Уже существующие плитки и разделы считаются определёнными пользователем. Их конфигурации сопоставления данных не обновляются автоматически.
  + Для вновь созданных плиток и разделов применяются настройки сопоставления данных по умолчанию. Если вы не изменяете эти настройки вручную, они могут измениться, если новое выполнение плитки/раздела изменит результаты и появятся недостающие поля или новые поля, лучше подходящие для сопоставления данных.

### Настройки сопоставления данных для конкретной визуализации

Для диаграммы-индикатора раздел сопоставления данных включает:

* **Meter value**: числовое значение, которое вы хотите представить в виде индикатора.

## Meter bar

### Показать метку

Чтобы отобразить строку над полосой, включите **Show label** и введите строку.

### Показать значок

Чтобы отобразить значок над полосой, включите **Show icon** и выберите значок из списка. Когда **Show label** и **Show icon** выбраны одновременно, значок отображается перед меткой.

### Минимальное значение

Определяет, где начинается полоса индикатора слева.

* **Auto** — начинает индикатор с `0`
* **Custom** — указывает другую начальную точку индикатора

### Максимальное значение

Определяет, где заканчивается полоса индикатора справа.

* **Auto**

  + Если значение индикатора меньше `100`, правый конец индикатора = `100`
  + Если значение индикатора больше `100`, правый конец индикатора равен значению индикатора
* **Custom** — указывает другую конечную точку индикатора

## Цвет

* Чтобы установить основной цвет полосы, выберите нужный цвет из меню **Bar color** (или откройте меню **Bar color** и введите HEX-код цвета).
* Вы можете добавить один или несколько **Custom colors** для определения цветов, отображаемых при достижении индикатором определённых значений.

  Для каждого пользовательского цвета введите значение индикатора и установите соответствующий цвет, который будет использоваться при достижении этого значения. Вертикальная линия отображается на полосе для каждого значения пользовательского цвета.

  Пример

  Предположим, вы хотите, чтобы ваш индикатор оставался зелёным в обычном состоянии, но становился жёлтым при достижении `80` и красным при достижении `90`:

  + Установите **Bar color** в нужный оттенок зелёного
  + Выберите  **Color** и добавьте строку пользовательского цвета со значением `80` и выбранным оттенком жёлтого. Если значение индикатора достигнет `80`, полоса станет жёлтой.
  + Выберите  **Color** снова и добавьте строку пользовательского цвета со значением `90` и выбранным оттенком красного. Если значение индикатора достигнет `90`, полоса станет красной.

## Единицы измерения и форматы

Чтобы переопределить единицы измерения и форматы по умолчанию в визуализации дашборда или блокнота:

1. Выберите  для редактирования плитки визуализации.
2. Выберите вкладку **Visual**.
3. Выберите **Units and formats**.
4. Выберите **Override**.
5. Выберите  **Override**
6. В выпадающем списке выберите элемент, для которого вы хотите добавить переопределение единицы измерения.

   Это числовой столбец базового результата DQL, поэтому он варьируется в зависимости от запроса. Например:

   * Запрос `fetch events` возвращает события. Выпадающий список здесь позволяет выбрать числовое поле (например, `transfer_size`) из результатов.
   * Запрос `timeseries avg(dt.host.cpu.usage)` возвращает один временной ряд для `avg(dt.host.cpu.usage)`. Этот временной ряд будет единственным доступным вариантом в списке.
7. Определите переопределение.

   * **Default unit**: базовая единица, в которой были собраны значения. Она равна `None`, если не была включена в результат DQL, или автоматически определяется единицей, переданной из результата DQL. Это поле не приводит к конвертации.
   * **Displayed unit**: после определения единицы по умолчанию вы можете использовать **Displayed unit** для конвертации. Например, если результат DQL определил ваше числовое значение в `Bytes`, **Displayed unit** теперь предлагает подходящий список конвертаций байтов, таких как `Kilobyte` и `Megabyte`. В отличие от **Default unit**, **Displayed unit** всегда является числовой конвертацией.
   * **Decimals** отображает количество десятичных знаков (степень точности) по умолчанию. Чтобы увидеть это в действии, измените выбор **Decimals** и наблюдайте за изменением визуализации.
   * **Suffix** отображает суффикс, который будет показан после единицы измерения. Чтобы увидеть это в действии, введите строку и наблюдайте за изменением визуализации. Когда вы не находите нужную единицу измерения, вы можете использовать **Suffix** для отображения нужной единицы.
8. Включите **Abbreviate large numbers**, если вы хотите отображать большие числа в сокращённой форме. Например, `1053` становится `1.1K`.

Чтобы сбросить к настройкам по умолчанию (отменить переопределение для выбранного элемента), выберите значок корзины  рядом с элементом.

Пример для дашбордов

В этом примере используется линейная диаграмма, но параметры применимы к другим визуализациям.

1. В ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** создайте дашборд.
2. Выберите  и в разделе **Snippets** меню выберите **Metrics** > **Chart average CPU across all hosts**.
3. В панели редактирования раздела выберите вкладку **Visual** и выберите **Line**.
4. Выберите **Units and formats**.

   ![Выберите Units and formats](https://dt-cdn.net/images/units-example-proc-01-1450-635695f6b8.png)
5. Выберите  **Override**.

   ![Выберите Add Override.](https://dt-cdn.net/images/units-example-proc-02-1450-e1805870b1.png)
6. В выпадающем списке выберите метрику, для которой вы хотите добавить переопределение. В этом примере есть только одна метрика для выбора.
7. Определите переопределение для отображаемой метрики. Вы можете наблюдать за изменениями на оси Y диаграммы.

   * **Default unit** отображает `Percent (%)`, что является единицей по умолчанию для выбранной метрики. Попробуйте другую настройку, например `One`, чтобы вместо этого отображать результат как долю от 1.
   * **Displayed unit** отображает `Auto`. Вы можете изменить её на другую единицу, например `One`, чтобы вместо этого отображать результат как долю от 1.

     Поддерживаются только линейные и статические конвертации. Например, вы не можете конвертировать `Degree Celsius(°C)` в `Degree Fahrenheit(°F)` или `Usd(US$)` в `Eur(€)`.
   * **Decimals** отображает количество десятичных знаков (степень точности) по умолчанию. Чтобы увидеть это в действии, измените выбор **Decimals** и наблюдайте за изменением визуализации.

     Например, измените это:

     ![Настройка Decimals до изменения](https://dt-cdn.net/images/units-example-proc-03-1454-6a84518802.png)

     На это:

     ![Настройка Decimals после изменения](https://dt-cdn.net/images/units-example-proc-04-1450-03cc893027.png)
   * **Suffix** отображает необязательный суффикс, который будет показан после единицы измерения. Чтобы увидеть это в действии, введите строку и наблюдайте за изменением визуализации.

Чтобы сбросить к настройкам по умолчанию (отменить переопределение для выбранной метрики), выберите значок корзины  рядом с метрикой.
