Когда использовать визуализацию в виде шкалы

Используйте шкалу для отображения одного числового значения в виде индикатора.

## Примеры

### Пример 1

![Визуализация шкалы: пример 1](https://dt-cdn.net/images/gauge-example-01-479-24a2a01700.png)

Визуализация шкалы выше основана на следующем запросе, который вычисляет `CPU` как среднее использование процессора хоста. В настройках **Data mapping** параметр **Gauge value** установлен как `CPU`.

```
timeseries avg(dt.host.cpu.usage)



| fieldsAdd CPU = arrayAvg(`avg(dt.host.cpu.usage)`)



| fieldsKeep CPU
```

### Пример 2

![Визуализация шкалы: пример 2](https://dt-cdn.net/images/gauge-example-02-476-681bd11f53.png)

Визуализация шкалы выше основана на следующем запросе, который вычисляет `pctActive` как процент от общего числа открытых проблем. В настройках **Data mapping** параметр **Gauge value** установлен как `pctActive`.

```
fetch dt.davis.problems



| summarize count = count(), by:{event.status}



| summarize { active=toDouble(takeAny(if(event.status=="ACTIVE", count))), closed=toDouble(takeAny(if(event.status=="CLOSED", count)))}



| fieldsAdd total = active + closed



| fieldsAdd pctActive = (active / total) * 100



| fieldsKeep pctActive
```

## Заголовок

Используйте поле заголовка в верхней части панели настроек (изначально `Untitled tile` или `Untitled section`) для добавления заголовка к плитке дашборда или секции notebook.

* Вы можете использовать эмодзи, например, ð, ð и â¤ï¸.
* Вы можете использовать переменные.

**Пример:**

1. Определите переменные `Status` и `Emoji` в вашем дашборде.
2. Установите заголовок `Current $Emoji status is $Status`.
3. Установите значение `Status` в `Good`.
4. Установите значение `Emoji` в `ð`.

Заголовок будет отображён как `Current ð status is Good`.

## Сопоставление данных

В секции сопоставления данных показано, как столбец вашего результата сопоставлен с визуализацией.

Общие правила настройки сопоставления данных

Разверните секцию **Data mapping** в настройках визуализации, чтобы увидеть, как данные в вашем результате сопоставлены с визуализацией, и при необходимости скорректировать эти настройки.

* Обязательные поля отмечены звёздочкой (`*`). Пример:

  ![Пример сопоставления данных: линейная диаграмма](https://dt-cdn.net/images/example-data-mapping-line-chart-1-732-eb9cca00d8.png)
* Типы данных отображаются рядом с именами полей в выпадающих списках и сопоставленных полях.
* Единицы измерения отображаются, если назначена только одна.
* Поля результата сгруппированы в **Suitable** и **Unsuitable**. Поля помечаются как непригодные, если они не могут быть использованы для отображения данных в визуализации. Пример:

  ![Пример сопоставления данных: линейная диаграмма, выпадающий список Time](https://dt-cdn.net/images/example-data-mapping-line-chart-2-763-9cc593d261.png)
* Автоматическое применение настроек сопоставления данных по умолчанию:

  Dynatrace версии 1.319+

  + Существующие плитки и секции считаются определёнными пользователем. Их конфигурации сопоставления данных не обновляются автоматически.
  + Вновь созданные плитки и секции применяют настройки сопоставления данных по умолчанию. Если вы не измените эти настройки вручную, они могут измениться при повторном выполнении плитки/секции, если результаты изменятся и появятся отсутствующие или более подходящие поля.

### Настройки сопоставления данных для конкретной визуализации

Для диаграммы-шкалы секция сопоставления данных включает:

* **Gauge value**: числовое значение, которое вы хотите представить в виде шкалы.

## Полоса шкалы

### Отображение метки

Чтобы отобразить строку в центре шкалы, включите **Show label** и введите строку.

### Минимальное значение

Определяет, где шкала начинается слева.

* **Auto** -- шкала начинается с `0`
* **Custom** -- задаёт другую начальную точку шкалы

### Максимальное значение

Определяет, где шкала заканчивается справа.

* **Auto**

  + Если значение шкалы меньше `100`, правый конец шкалы = `100`
  + Если значение шкалы больше `100`, правый конец шкалы равен значению шкалы
* **Custom** -- задаёт другую конечную точку шкалы

## Цвет

* Чтобы задать основной цвет шкалы, выберите нужный цвет из меню **Bar color** (или откройте меню **Bar color** и введите hex-код цвета).
* Вы можете добавить один или несколько **Custom colors**, чтобы определить цвета, отображаемые при достижении шкалой определённых значений.

  Для каждого пользовательского цвета введите значение шкалы и установите соответствующий цвет, который будет использоваться при достижении этого значения. На шкале отображается вертикальная линия для каждого значения пользовательского цвета.

  Пример

  Предположим, вы хотите, чтобы шкала оставалась зелёной в обычном состоянии, но становилась жёлтой при достижении `80` и красной при достижении `90`:

  + Установите **Bar color** в нужный оттенок зелёного
  + Выберите **Color** и добавьте строку пользовательского цвета со значением `80` и нужным оттенком жёлтого. Если значение шкалы достигнет `80`, шкала станет жёлтой.
  + Выберите **Color** ещё раз и добавьте строку пользовательского цвета со значением `90` и нужным оттенком красного. Если значение шкалы достигнет `90`, шкала станет красной.

## Единицы и форматы

Чтобы переопределить единицы и форматы по умолчанию в визуализации дашборда или notebook:

1. Выберите  для редактирования плитки визуализации.
2. Перейдите на вкладку **Visual**.
3. Выберите **Units and formats**.
4. Выберите **Override**.
5. Выберите  **Override**
6. В выпадающем списке выберите элемент, для которого вы хотите добавить переопределение единиц.

   Это числовой столбец базового результата DQL, поэтому он варьируется в зависимости от запроса. Например:

   * Запрос `fetch events` возвращает события. Выпадающий список позволяет выбрать числовое поле (например, `transfer_size`) из результатов.
   * Запрос `timeseries avg(dt.host.cpu.usage)` возвращает один временной ряд для `avg(dt.host.cpu.usage)`. Этот временной ряд является единственным доступным вариантом в списке.
7. Определите переопределение.

   * **Default unit**: базовая единица, в которой были получены значения. Это `None`, если она не была включена в результат DQL, или определяется автоматически единицей, переданной из результата DQL. Это поле не приводит к конвертации.
   * **Displayed unit**: после определения единицы по умолчанию вы можете использовать **Displayed unit** для конвертации. Например, если результат DQL определил ваше числовое значение как `Bytes`, **Displayed unit** предложит подходящий список конвертаций байтов, например `Kilobyte` и `Megabyte`. В отличие от **Default unit**, **Displayed unit** всегда выполняет числовую конвертацию.
   * **Decimals** отображает количество десятичных знаков (степень точности) по умолчанию. Чтобы увидеть результат, измените выбор **Decimals** и наблюдайте за изменением в визуализации.
   * **Suffix** отображает суффикс, добавляемый после единицы. Чтобы увидеть результат, введите строку и наблюдайте за изменением в визуализации. Если вы не нашли нужную единицу, вы можете использовать **Suffix** для отображения нужной единицы.
8. Включите **Abbreviate large numbers**, если хотите отображать большие числа в сокращённой форме. Например, `1053` станет `1.1K`.

Чтобы сбросить настройки по умолчанию (отменить переопределение для выбранного элемента), выберите значок корзины  рядом с элементом.

Пример для дашбордов

В этом примере используется линейная диаграмма, но настройки применимы и к другим визуализациям.

1. В ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** создайте дашборд.
2. Выберите  и в секции **Snippets** меню выберите **Metrics** > **Chart average CPU across all hosts**.
3. В панели редактирования секции выберите вкладку **Visual** и выберите **Line**.
4. Выберите **Units and formats**.

   ![Выбор Units and formats](https://dt-cdn.net/images/units-example-proc-01-1450-635695f6b8.png)
5. Выберите  **Override**.

   ![Выбор Add Override.](https://dt-cdn.net/images/units-example-proc-02-1450-e1805870b1.png)
6. В выпадающем списке выберите метрику, для которой вы хотите добавить переопределение. В этом примере доступна только одна метрика.
7. Определите переопределение для отображаемой метрики. Вы можете наблюдать за изменениями на оси Y диаграммы.

   * **Default unit** отображает `Percent (%)` -- единицу по умолчанию для выбранной метрики. Попробуйте другую настройку, например `One`, чтобы отобразить результат как дробь от 1.
   * **Displayed unit** отображает `Auto`. Вы можете изменить его на другую единицу, например `One`, чтобы отобразить результат как дробь от 1.

     Поддерживаются только линейные и статические конвертации. Например, нельзя конвертировать `Degree Celsius(°C)` в `Degree Fahrenheit(°F)` или `Usd(US$)` в `Eur(€)`.
   * **Decimals** отображает количество десятичных знаков (степень точности) по умолчанию. Чтобы увидеть результат, измените выбор **Decimals** и наблюдайте за изменением в визуализации.

Чтобы сбросить настройки по умолчанию (отменить переопределение для выбранной метрики), выберите значок корзины  рядом с метрикой.
