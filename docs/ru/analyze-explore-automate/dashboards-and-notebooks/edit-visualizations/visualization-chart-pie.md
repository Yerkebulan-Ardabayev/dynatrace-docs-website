# Круговая диаграмма

# Круговая диаграмма

* Актуальная версия Dynatrace
* Руководство
* Время чтения: 1 мин
* Опубликовано 08 июля 2022

## Пример

Когда использовать круговую диаграмму

Используйте круговую диаграмму:

* Когда у вас есть лишь несколько (не более 5–7) чётко различимых категорий/сегментов в данных. В таких случаях круговая диаграмма проста и легко читается.
* Когда вы хотите показать пропорции или процентные доли от целого. Например, отображение подов по фазам.
* Когда вы хотите подчеркнуть структуру данных. Например, можно использовать круговую диаграмму для иллюстрации распределения ошибок по типам.

![Пример круговой диаграммы](https://dt-cdn.net/images/pie-chart-642-aa7d7a76a4.png)

Приведённая выше круговая диаграмма построена на основе следующего запроса.

```
fetch logs



| summarize count(), by:{loglevel}
```

## Взаимодействие с диаграммой

### Взаимодействие при выборе

Когда вы выбираете значение на диаграмме и закрепляете отображаемую всплывающую подсказку, вы можете навести курсор на подсказку, чтобы отобразить меню с опциями для выбранного элемента.

Доступные взаимодействия зависят от вашего запроса и визуализации. Например, если вы выбираете хост на линейной диаграмме и наводите курсор на подсказку, вы увидите меню с элементами, такими как:

* **Copy name** — копировать имя выбранного хоста.
* **Fields** — раздел с подменю для каждого поля запроса. Подменю поля предлагает специфичные для поля опции, такие как:

  + **Copy value** — копировать значение поля. Также отображает тип поля.
  + **Hide** — скрыть поле на диаграмме.
  + **Explain value** — использовать ИИ для объяснения поля.
  + **Add command to query** — раздел специфичных для поля команд, которые можно автоматически добавить к запросу.
  + Также может быть указано рекомендованное приложение.
* **Visual options** — открывает панель редактирования для изменения параметров визуализации выбранного элемента.
* **Set color** — открывает панель редактирования для изменения цвета выбранного элемента.
* ![Infrastructure & Operations](https://dt-cdn.net/images/infrasctucture-operations-highresolution-1025-07d1bc45b5.png "Infrastructure & Operations") **Go to host** — открывает выбранный элемент в ![Infrastructure & Operations](https://dt-cdn.net/images/infrasctucture-operations-highresolution-1025-07d1bc45b5.png "Infrastructure & Operations") **Infrastructure & Operations**.

  В общем случае, если есть рекомендованные приложения для открытия выбранного элемента, меню предлагает прямые ссылки на эти приложения, а затем опцию **Open with** для выбора другого целевого приложения.
* **Open with** — подробнее см. в разделе [Переходы и навигация](../drilldowns-and-navigation.md "Детализация из Dashboards и Notebooks с помощью ссылок на основе намерений или URL-адресов.").

## Заголовок

Используйте поле заголовка в верхней части панели параметров (изначально `Untitled tile` или `Untitled section`), чтобы добавить заголовок к плитке дашборда или секции ноутбука.

* Можно использовать эмодзи, такие как 🟢 и 🔴 и ❤️.
* Можно использовать переменные.

**Пример:**

1. Определите переменные `Status` и `Emoji` в вашем дашборде.
2. Установите заголовок `Current $Emoji status is $Status`.
3. Установите значение `Status` равным `Good`.
4. Установите значение `Emoji` равным `🟢`.

Заголовок будет отображаться как `Current 🟢 status is Good`.

## Визуализация

Если вы не уверены, что выбрали правильную визуализацию, используйте [селектор визуализации](../edit-visualizations.md#select-visualization "Создание, редактирование и просмотр визуализаций на дашбордах и в ноутбуках Dynatrace."), чтобы попробовать различные визуализации.

## Сопоставление данных

Раздел сопоставления данных показывает, как столбец вашего результата сопоставлен с визуализацией.

Развернуть для общих правил настройки сопоставления данных

Разверните раздел **Data mapping** в настройках визуализации, чтобы увидеть, как данные вашего результата сопоставлены с визуализацией, и при необходимости скорректировать эти настройки.

* Обязательные поля отмечены звёздочкой (`*`). Пример:

  ![Пример сопоставления данных: линейная диаграмма](https://dt-cdn.net/images/example-data-mapping-line-chart-1-732-eb9cca00d8.png)
* Типы данных отображаются рядом с именами полей в выпадающих списках и сопоставленных полях.
* Единицы измерения отображаются, если назначена только одна.
* Поля результатов группируются в категории **Suitable** и **Unsuitable**. Поля помечаются как неподходящие, если их нельзя использовать для отображения данных в визуализации. Пример:

  ![Пример сопоставления данных: линейная диаграмма, выпадающий список Time](https://dt-cdn.net/images/example-data-mapping-line-chart-2-763-9cc593d261.png)
* Автоматическое применение настроек сопоставления данных по умолчанию:

  Dynatrace версии 1.319+

  + Уже существующие плитки и секции считаются пользовательскими. Их конфигурации сопоставления данных не обновляются автоматически.
  + Вновь созданные плитки и секции применяют настройки сопоставления данных по умолчанию. Если вы не изменяете эти настройки вручную, они могут измениться при новом выполнении плитки/секции, если результаты изменились и появились недостающие или более подходящие поля.

### Настройки сопоставления данных для конкретной визуализации

Для круговой диаграммы раздел сопоставления данных включает:

* **Values**: значение, отображаемое в размере сегмента.
* **Categories**: столбец результата, отображаемый как сегмент круговой диаграммы.

## Параметры круговой диаграммы

Чтобы быстро ознакомиться с параметрами и определить, что подходит лучше всего, включайте и выключайте параметры и сразу наблюдайте эффект на диаграмме. Например, лучше ли выглядит с подписью или без? Включите и выключите эту опцию и посмотрите сами.

### Цветовые палитры

Вы можете выбрать цветовую палитру из списка в разделе **Color palettes**.

### Сегмент

Установите **Slice** в единицу, по которой вы хотите разделить диаграмму. **Slice** и **Value** вместе определяют, как результаты запроса отображаются на диаграмме.

### Значение

Установите **Value** в значение, определяющее размер сегментов. **Slice** и **Value** вместе определяют, как результаты запроса отображаются на диаграмме.

### Диаграмма

Используйте **Hide labels**, чтобы показать или скрыть подписи на диаграмме.

### Положение легенды

Выберите, где отображать легенду визуализации:

* **Automatic** — Dynatrace выбирает подходящее расположение
* **Hidden** — легенда не отображается
* **Bottom** — легенда отображается внизу визуализации
* **Right** — легенда отображается справа от визуализации

### Общее значение

Используйте **Relative values (%)**, чтобы отображать значения диаграммы как абсолютные или относительные значения (проценты от общего).

### Объединение сегментов

Используйте настройки **Merge slices** для определения способа объединения сегментов на диаграмме.

* **Absolute**

  Например, если у вас три сегмента со следующими значениями:

  + Сегмент 1: `40`
  + Сегмент 2: `5`
  + Сегмент 3: `5`

  Если вы выберете **Absolute** и установите **Absolute limit** в `10`, сегменты 2 и 3 будут объединены в одну группу с подписью, заданной в **Group label**.
* **Relative (%)**

  Например, если у вас три сегмента со следующими значениями:

  + Сегмент 1 — `40` (80% от общего)
  + Сегмент 2 — `5` (10% от общего)
  + Сегмент 3 — `5` (10% от общего)

  Если вы выберете **Relative (%)** и установите **Relative limit** в `20` (20%), сегменты 2 и 3 (каждый из которых ниже относительного значения 20%) будут объединены в одну группу с подписью, заданной в **Group label**.
* **Number of slices**

  Например, если у вас три сегмента со следующими значениями:

  + Сегмент 1: `40`
  + Сегмент 2: `5`
  + Сегмент 3: `5`

  Если вы выберете **Number of slices** и установите **Slice limit** в `2`, первые два сегмента будут отображаться отдельно, а остальные будут объединены в одну группу с подписью, заданной в **Group label**.

### Добавить переопределение цвета

Чтобы переопределить цвет элемента диаграммы, выберите **Add color override**, выберите элемент из списка, а затем установите **Color** в цвет, который вы хотите использовать для выбранного элемента.

## Лимиты запросов

Используйте раздел **Query limits** для проверки и настройки лимитов запросов Grail для каждой секции ноутбука или плитки дашборда. Эти настройки определяют максимальные лимиты при получении данных. Превышение любого лимита приведёт к генерации предупреждения.

Плитки дашбордов и секции ноутбуков, созданные в Dynatrace ранее версии 1.296, не затрагиваются. Эти существующие плитки/секции будут возвращать те же результаты, что и раньше.

* **Read data limit (GB)**

  Лимит в гигабайтах для объёма данных, которые будут просканированы при чтении.
* **Record limit**

  Максимальное количество записей результата, которое вернёт данный запрос. По умолчанию: 1 000 записей. Чтобы увидеть больше записей, нужно увеличить значение **Record limit**.

  + Если ваш запрос не содержит `limit`, например

    ```
    fetch logs
    ```

    применяется значение **Record limit**. По умолчанию вы увидите до 1 000 записей.
  + Если ваш запрос также включает `limit`, например

    ```
    fetch logs



    | limit 2000
    ```

    применяется меньшее из двух значений (либо `limit` в вашем запросе, либо **Record limit** в веб-интерфейсе).

    В приведённом выше примере вы всё равно увидите только 1 000 записей, если не увеличите значение **Record limit**.
* **Result size limit**

  Максимальное количество байт результата, которое вернёт данный запрос. Для лучшей производительности при типичных запросах и небольших документах значение по умолчанию составляет 1 МБ.
* **Sampling (Logs and Spans only)**

  Приводит к выборке подмножества записей Log или Span.

## Пороговые значения

### Настройка пороговых значений

Чтобы настроить пороговые значения в визуализации дашборда или ноутбука

1. Выберите  для редактирования плитки визуализации.
2. Выберите **Thresholds**.

   * В ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** выберите вкладку **Visual** в боковой панели, а затем выберите **Thresholds**.
   * В ![Notebooks](https://dt-cdn.net/images/notebooks-768-046137830a.webp "Notebooks") **Notebooks** выберите **Thresholds** в боковой панели.
3. Выберите  **Threshold**.
4. Определите пороговые значения. Для каждого диапазона:

   * Выберите цвет для отображения этого диапазона
   * Выберите оператор для определения порогового значения этого диапазона
   * Введите статическое значение для сравнения (с использованием выбранного оператора) с возвращённым значением
   * Введите подпись для связи с определённым диапазоном.

     + Подписи применяются только к диаграммам с осью Y, таким как диаграммы временных рядов и категориальные столбчатые диаграммы.
     + Подписи нельзя определить для таблиц и диаграмм одного значения.

Пример настроек пороговых значений в Dashboards

Этот пример использует столбчатую диаграмму, но параметры применимы и к другим визуализациям.

1. В ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** создайте дашборд.
2. Выберите  и в разделе **Snippets** меню выберите **Metrics** > **Chart top 10 hosts by CPU usage**.
3. На панели редактирования секции выберите вкладку **Visual** и выберите **Bar**.
4. Выберите **Thresholds**.
5. Выберите  **Threshold**.

   Отображается пустой набор полей порогового значения.
6. Определите пороговые значения для отображаемой метрики. Вы можете наблюдать изменения на оси Y диаграммы.

   В этом примере мы определяем три диапазона загрузки CPU с соответствующими цветами и подписями.

   Вы можете увидеть диапазоны, отображаемые на оси Y и во всплывающей подсказке.

Для сброса к значениям по умолчанию (отмена настроек пороговых значений) выберите значок корзины  рядом с элементом.

### Редактирование пороговых значений

1. Выберите  для редактирования плитки визуализации.
2. Выберите **Thresholds**.

   * В ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** выберите вкладку **Visual** в боковой панели, а затем выберите **Thresholds**.
   * В ![Notebooks](https://dt-cdn.net/images/notebooks-768-046137830a.webp "Notebooks") **Notebooks** выберите **Thresholds** в боковой панели.
3. С этого момента вы можете выполнять следующие действия (разверните строки для подробностей):

   Добавить пороговые значения

   Чтобы добавить пороговые значения (если выбранная визуализация поддерживает дополнительные пороговые значения), выберите  **Threshold**.

   Отключить пороговые значения

   Чтобы отключить (скрыть) существующие пороговые значения, используйте переключатель. Никакие настройки не теряются. Вы можете снова включить их, если передумаете.

   Удалить пороговые значения

   Чтобы удалить существующие пороговые значения, выберите значок корзины .

   Редактировать настройки пороговых значений

   Чтобы изменить существующие настройки пороговых значений, просто отредактируйте поля.

   Добавить диапазон

   Чтобы добавить диапазон к существующим пороговым значениям, выберите  **Add range**.

   Удалить диапазон

   Чтобы удалить диапазон из существующих пороговых значений, выберите кнопку удаления в этой строке.

## Единицы измерения и форматы

Чтобы переопределить единицы измерения и форматы по умолчанию в визуализации дашборда или ноутбука

1. Выберите  для редактирования плитки визуализации.
2. Выберите вкладку **Visual**.
3. Выберите **Units and formats**.
4. Выберите **Override**.
5. Выберите  **Override**
6. В выпадающем списке выберите элемент, для которого вы хотите добавить переопределение единицы измерения.

   Это числовой столбец базового результата DQL, поэтому он зависит от запроса. Например:

   * Запрос `fetch events` возвращает события. Выпадающий список здесь позволяет выбрать числовое поле (например, `transfer_size`) из результатов.
   * Запрос `timeseries avg(dt.host.cpu.usage)` возвращает один временной ряд для `avg(dt.host.cpu.usage)`. Этот временной ряд является единственным доступным вариантом в списке.
7. Определите переопределение.

   * **Default unit**: базовая единица, в которой были зафиксированы значения. Это `None`, если она не была включена в результат DQL, или автоматически определяется единицей из результата DQL. Это поле не приводит к преобразованию.
   * **Displayed unit**: после определения единицы по умолчанию вы можете использовать **Displayed unit** для преобразования. Например, если результат DQL определил ваше числовое значение как `Bytes`, **Displayed unit** теперь предлагает подходящий список преобразований байтов, таких как `Kilobyte` и `Megabyte`. В отличие от **Default unit**, **Displayed unit** всегда является числовым преобразованием.
   * **Decimals** отображает количество десятичных знаков по умолчанию (степень точности). Чтобы увидеть это в действии, измените параметр **Decimals** и наблюдайте за изменением визуализации.
   * **Suffix** отображает суффикс после единицы измерения. Чтобы увидеть это в действии, введите строку и наблюдайте за изменением визуализации. Когда вы не находите нужную единицу, можно использовать **Suffix** для отображения желаемой единицы.
8. Включите **Abbreviate large numbers**, если хотите отображать большие числа в сокращённом виде. Например, `1053` становится `1.1K`.

Для сброса к значениям по умолчанию (отмена настроек переопределения для выбранного элемента) выберите значок корзины  рядом с элементом.

Пример для дашбордов

Этот пример использует линейную диаграмму, но параметры применимы и к другим визуализациям.

1. В ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** создайте дашборд.
2. Выберите  и в разделе **Snippets** меню выберите **Metrics** > **Chart average CPU across all hosts**.
3. На панели редактирования секции выберите вкладку **Visual** и выберите **Line**.
4. Выберите **Units and formats**.

   ![Выберите Units and formats](https://dt-cdn.net/images/units-example-proc-01-1450-635695f6b8.png)
5. Выберите  **Override**.

   ![Выберите Add Override.](https://dt-cdn.net/images/units-example-proc-02-1450-e1805870b1.png)
6. В выпадающем списке выберите метрику, для которой вы хотите добавить переопределение. В этом примере доступна только одна метрика.
7. Определите переопределение для отображаемой метрики. Вы можете наблюдать изменения на оси Y диаграммы.

   * **Default unit** отображает `Percent (%)` — единицу по умолчанию для выбранной метрики. Попробуйте другую настройку, например `One`, чтобы вместо этого отобразить результат как долю от 1.
   * **Displayed unit** отображает `Auto`. Вы можете изменить его на другую единицу, например `One`, чтобы вместо этого отобразить результат как долю от 1.

     Поддерживаются только линейные и статические преобразования. Например, нельзя преобразовать `Degree Celsius(°C)` в `Degree Fahrenheit(°F)` или `Usd(US$)` в `Eur(€)`.
   * **Decimals** отображает количество десятичных знаков по умолчанию (степень точности). Чтобы увидеть это в действии, измените параметр **Decimals** и наблюдайте за изменением визуализации.

   * **Suffix** отображает необязательный суффикс после единицы измерения. Чтобы увидеть это в действии, введите строку и наблюдайте за изменением визуализации.

Для сброса к значениям по умолчанию (отмена настроек переопределения для выбранной метрики) выберите значок корзины  рядом с метрикой.
