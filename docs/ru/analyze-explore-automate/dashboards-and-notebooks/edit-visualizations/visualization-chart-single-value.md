# Визуализация одиночного значения

* Последняя версия Dynatrace
* Руководство
* Чтение: 2 мин
* Обновлено 17 сентября 2025 г.

Когда использовать визуализацию одиночного значения

Используйте визуализацию одиночного значения, когда вы хотите:

* Показать одно агрегированное измерение и его тренд во времени.
* Отобразить важные бизнес-ключевые показатели эффективности (KPI).
* Отобразить критически важные целевые уровни обслуживания (SLO), связанные с инфраструктурой.

```

| sort arrayAvg(value), direction:"descending"
```

* **Trend value**: `Relative`
* **Show sparkline**: `On`
* **Sparkline**: `sparkline`

## Взаимодействие с диаграммой

### Взаимодействие при выборе

Когда вы выбираете значение на диаграмме и закрепляете отображаемую всплывающую подсказку, вы можете навести курсор на неё для отображения меню с параметрами, специфичными для выбора.

Доступные взаимодействия с диаграммой зависят от вашего запроса и визуализации. Например, если вы выберете хост на линейной диаграмме и наведёте курсор на всплывающую подсказку, вы увидите меню с такими пунктами, как:

* **Copy name** — копирование имени выбранного хоста.
* **Fields** — раздел с подменю для каждого поля запроса. Подменю поля предлагает специфичные для поля параметры, такие как:

  + **Copy value** — копирование значения поля. Также отображается тип поля.
  + **Hide** — скрытие поля на диаграмме.
  + **Explain value** — использование ИИ для объяснения поля.
  + **Add command to query** — раздел с командами, специфичными для поля, которые можно автоматически добавить в запрос.
  + Также может отображаться рекомендуемое приложение.
* **Visual options** — открывает панель редактирования для изменения параметров визуализации выбранного элемента.
* **Set color** — открывает панель редактирования для изменения цвета выбранного элемента.
* ![Infrastructure & Operations](https://dt-cdn.net/images/infrasctucture-operations-highresolution-1025-07d1bc45b5.png "Infrastructure & Operations") **Go to host** — открывает выбранный элемент в ![Infrastructure & Operations](https://dt-cdn.net/images/infrasctucture-operations-highresolution-1025-07d1bc45b5.png "Infrastructure & Operations") **Infrastructure & Operations**.

  В общем случае, если есть рекомендуемые приложения для открытия выбранного элемента, меню предлагает прямые ссылки на эти приложения, а затем опцию **Open with** для выбора другого целевого приложения.
* **Open with** — подробности см. в разделе [Детализация и навигация](../drilldowns-and-navigation.md "Drill down from Dashboards and Notebooks using links based on intents or URLs.").

## Отображение данных

Раздел отображения данных показывает, как столбец вашего результата сопоставляется с визуализацией.

Общие правила настроек отображения данных

Разверните раздел **Data mapping** в настройках визуализации, чтобы увидеть, как данные вашего результата сопоставляются с визуализацией, и при необходимости скорректировать эти настройки.

* Обязательные поля отмечены звёздочкой (`*`). Пример:

  ![Пример отображения данных: линейная диаграмма](https://dt-cdn.net/images/example-data-mapping-line-chart-1-732-eb9cca00d8.png)
* Типы данных отображаются рядом с именами полей в выпадающих списках и сопоставленных полях.
* Единицы измерения отображаются, когда назначена только одна.
* Поля результата группируются на **Suitable** и **Unsuitable**. Поля помечаются как неподходящие, если их нельзя использовать для отображения данных в визуализации. Пример:

  ![Пример отображения данных: линейная диаграмма, выпадающий список Time](https://dt-cdn.net/images/example-data-mapping-line-chart-2-763-9cc593d261.png)
* Автоматическое применение настроек отображения данных по умолчанию:

  Dynatrace версии 1.319+

  + Уже существующие плитки и секции считаются определёнными пользователем. Их конфигурации отображения данных не обновляются автоматически.
  + Вновь созданные плитки и секции применяют настройки отображения данных по умолчанию. Если вы не изменяете эти настройки вручную, они могут измениться, если новое выполнение плитки/секции изменит результаты и появятся отсутствующие поля или новые поля, лучше подходящие для отображения данных.

### Настройки отображения данных для конкретной визуализации

Для диаграммы одиночного значения раздел отображения данных включает:

* **Single value**: столбец вашего результата, отображаемый в визуализации одиночного значения.
* **Sparkline**: значение, отражаемое в спарклайне визуализации одиночного значения. Это также влияет на тренд, который рассчитывается как разница между первым и последним элементами временного ряда, выбранного в качестве спарклайна.
* **Interval**: это значение сопоставляется автоматически и не может быть изменено. Оно позволяет узнать, какие поля сопоставлены для результатов на основе временных рядов. Берётся первое доступное поле интервала из набора результатов при использовании временного ряда (также включает данные на основе makeTimeseries).
* **Trend**: выберите `Auto` для автоматического расчёта тренда на основе данных, предоставленных для спарклайна. Выберите `Custom`, чтобы указать числовое поле.

Режим сетки

Когда результат содержит несколько строк:

* Ранее в визуализации одиночного значения отображался только последний результат.
* Начиная с Dynatrace версии 1.324, визуализация одиночного значения автоматически переключается в режим сетки, где каждое значение отображается как отдельное одиночное значение. Максимальное количество значений, отображаемых в сетке, — 25.

Преимущества:

* Повышает прозрачность и предотвращает потенциально вводящие в заблуждение интерпретации.
* Нет необходимости в нескольких визуализациях связанных результатов. Все данные отображаются в едином формате на одной сетке.

Если вы предпочитаете предыдущее поведение (без сетки), вы можете применить команды DQL [limit](../../../platform/grail/dynatrace-query-language/commands/ordering-commands.md#limit "DQL ordering commands") и [sort](../../../platform/grail/dynatrace-query-language/commands/ordering-commands.md#sort "DQL ordering commands"), используя [DQL](../../../platform/grail/dynatrace-query-language/commands.md "A list of DQL commands.") или [интерфейс Explore](../explore-data.md "Explore your data with our point-and-click interface.").

## Параметры одиночного значения

Чтобы быстро ознакомиться с параметрами и решить, что лучше всего подходит, включайте и выключайте параметры и сразу наблюдайте эффект на диаграмме. Например, лучше ли выглядит с меткой или без? Включите и выключите эту опцию и убедитесь сами.

### Общие

Используйте эти параметры для задания общих настроек отображения вашей визуализации.

* **Show label** указывает, отображать ли метку на визуализации.

  + **Label** — текст метки, которую вы хотите показать, например `World ð trend`. Этот параметр конфигурации отображается только при включённом **Show label**.

    Чтобы задать метку через DQL с использованием псевдонима, включите **Show label**, но не задавайте **Label**.
* **Format** указывает, форматировать ли значения на визуализации.
* **Record field** определяет поле, отображаемое на визуализации.
* **Alignment** задаёт выравнивание по левому краю, по центру или по правому краю для отображаемых элементов визуализации.
* **Icon** определяет иконку, отображаемую перед значением.

### Тренд и спарклайн

Используйте эти параметры для задания настроек тренда и спарклайна вашей визуализации.

* **Show trend** включает или выключает отображение иконки тренда и значения.

  Следующий параметр конфигурации отображается только при включённом **Show trend**:

  + **Trend value** указывает, отображать ли абсолютное или процентное значение тренда.
* **Show sparkline** включает или выключает отображение спарклайна.

  Следующие параметры конфигурации отображаются только при включённом **Show sparkline**:

  + **Sparkline color** определяет цвет спарклайна.
  + **Sparkline variant** указывает, отображать спарклайн в виде линии или области.
  + **Record field** определяет поле записи, отображаемое в виде спарклайна на визуализации.
  + **Show ticks** указывает, отображать ли значения вдоль оси X.

## Лимиты запросов

Используйте раздел **Query limits** для проверки и настройки лимитов запросов Grail для каждой секции ноутбука или плитки дашборда. Эти настройки определяют максимальные лимиты при получении данных. Превышение любого лимита генерирует предупреждение.

Плитки дашбордов и секции ноутбуков, созданные в Dynatrace ранее версии 1.296, не затрагиваются. Эти существующие плитки/секции будут возвращать те же результаты, что и раньше.

* **Read data limit (GB)**

  Лимит в гигабайтах для объёма данных, которые будут сканированы при чтении.
* **Record limit**

  Максимальное количество записей результатов, которое вернёт этот запрос. По умолчанию: 1 000 записей. Чтобы увидеть больше записей, необходимо увеличить значение **Record limit**.

  + Если в вашем запросе нет `limit`, например

    ```
    fetch logs
    ```

    применяется значение **Record limit**. По умолчанию вы увидите до 1 000 записей.
  + Если ваш запрос также включает `limit`, например

    ```
    fetch logs



    | limit 2000
    ```

    применяется меньшее из двух значений (либо `limit` в вашем запросе, либо **Record limit** в веб-интерфейсе).

    В приведённом выше примере вы всё равно увидите только 1 000 записей, если не увеличите значение **Record limit**.
* **Result size limit**

  Максимальное количество байт результата, которое вернёт этот запрос. Для лучшей производительности при типичных запросах и небольших документах значение по умолчанию установлено в 1 МБ.
* **Sampling (только для логов и спанов)**

  Приводит к выборке подмножества записей логов или спанов.

## Единицы измерения и форматы

Чтобы переопределить единицы измерения и форматы по умолчанию в визуализации дашборда или ноутбука

1. Выберите  для редактирования плитки визуализации.
2. Выберите вкладку **Visual**.
3. Выберите **Units and formats**.
4. Выберите **Override**.
5. Выберите  **Override**
6. В выпадающем списке выберите элемент, для которого вы хотите добавить переопределение единицы.

   Это числовой столбец базового результата DQL, поэтому он варьируется в зависимости от запроса. Например:

   * Запрос `fetch events` возвращает события. Выпадающий список здесь позволяет выбрать числовое поле (например, `transfer_size`) из результатов.
   * Запрос `timeseries avg(dt.host.cpu.usage)` возвращает один временной ряд для `avg(dt.host.cpu.usage)`. Этот временной ряд является единственным доступным вариантом в списке.
7. Определите переопределение.

   * **Default unit**: базовая единица, в которой были записаны значения. Это `None`, если она не была включена в результат DQL, или она автоматически определяется единицей, переданной из результата DQL. Это поле не приводит к конвертации.
   * **Displayed unit**: после определения единицы по умолчанию вы можете использовать **Displayed unit** для конвертации. Например, если результат DQL определил ваше числовое значение как `Bytes`, **Displayed unit** теперь предлагает подходящий список конвертаций байтов, таких как `Kilobyte` и `Megabyte`. В отличие от **Default unit**, **Displayed unit** всегда является числовой конвертацией.
   * **Decimals** отображает количество десятичных знаков по умолчанию (степень точности). Чтобы увидеть в действии, измените выбор **Decimals** и наблюдайте изменение в визуализации.
   * **Suffix** отображает суффикс, добавляемый после единицы. Чтобы увидеть в действии, введите строку и наблюдайте изменение в визуализации. Когда вы не находите нужную единицу, вы можете использовать **Suffix** для отображения желаемой единицы.
8. Включите **Abbreviate large numbers**, если хотите отображать большие числа в сокращённом виде. Например, `1053` становится `1.1K`.

Чтобы сбросить настройки по умолчанию (отменить переопределения для выбранного элемента), выберите иконку корзины  рядом с элементом.

Пример для дашбордов

Этот пример использует линейную диаграмму, но параметры применимы к другим визуализациям.

1. В ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** создайте дашборд.
2. Выберите  и в разделе **Snippets** меню выберите **Metrics** > **Chart average CPU across all hosts**.
3. В панели редактирования секции выберите вкладку **Visual** и выберите **Line**.
4. Выберите **Units and formats**.

   ![Выбор Units and formats](https://dt-cdn.net/images/units-example-proc-01-1450-635695f6b8.png)
5. Выберите  **Override**.

   ![Выбор Add Override.](https://dt-cdn.net/images/units-example-proc-02-1450-e1805870b1.png)
6. В выпадающем списке выберите метрику, для которой вы хотите добавить переопределение. В этом примере доступна только одна метрика.
7. Определите переопределение для отображаемой метрики. Вы можете наблюдать изменения на оси Y диаграммы.

   * **Default unit** отображает `Percent (%)`, что является единицей по умолчанию для выбранной метрики. Попробуйте другую настройку, например `One`, чтобы отобразить результат как долю от 1.
   * **Displayed unit** отображает `Auto`. Вы можете изменить её на другую единицу, например `One`, чтобы отобразить результат как долю от 1.

     Поддерживаются только линейные и статические конвертации. Например, нельзя конвертировать `Degree Celsius(°C)` в `Degree Fahrenheit(°F)` или конвертировать `Usd(US$)` в `Eur(€)`.
   * **Decimals** отображает количество десятичных знаков по умолчанию (степень точности). Чтобы увидеть в действии, измените выбор **Decimals** и наблюдайте изменение в визуализации.

     Например, измените это:

     ![Настройка Decimals до](https://dt-cdn.net/images/units-example-proc-03-1454-6a84518802.png)

     На это:

     ![Настройка Decimals после](https://dt-cdn.net/images/units-example-proc-04-1450-03cc893027.png)
   * **Suffix** отображает необязательный суффикс, добавляемый после единицы. Чтобы увидеть в действии, введите строку и наблюдайте изменение в визуализации.

Чтобы сбросить настройки по умолчанию (отменить переопределения для выбранной метрики), выберите иконку корзины  рядом с метрикой.

## Пороговые значения

### Настройка пороговых значений

Чтобы настроить пороговые значения в визуализации дашборда или ноутбука

1. Выберите  для редактирования плитки визуализации.
2. Выберите **Thresholds**.

   * В ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** выберите вкладку **Visual** на боковой панели, затем выберите **Thresholds**.
   * В ![Notebooks](https://dt-cdn.net/images/notebooks-768-046137830a.webp "Notebooks") **Notebooks** выберите **Thresholds** на боковой панели.
3. Выберите  **Threshold**.
4. Определите пороговые значения. Для каждого диапазона:

   * Выберите цвет для отображения этого диапазона
   * Выберите оператор для определения порогового значения этого диапазона
   * Введите статическое значение для сравнения (с использованием выбранного оператора) с возвращаемым значением
   * Введите метку для связи с определённым диапазоном.

     + Метки применяются только к диаграммам с осью Y, таким как диаграммы временных рядов и категориальная столбчатая диаграмма.
     + Метки не могут быть определены для таблиц и диаграммы одиночного значения.

Пример настройки пороговых значений в Dashboards

Этот пример использует столбчатую диаграмму, но параметры применимы к другим визуализациям.

1. В ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** создайте дашборд.
2. Выберите  и в разделе **Snippets** меню выберите **Metrics** > **Chart top 10 hosts by CPU usage**.
3. В панели редактирования секции выберите вкладку **Visual** и выберите **Bar**.
4. Выберите **Thresholds**.
5. Выберите  **Threshold**.

   Отображается пустой набор полей пороговых значений.
6. Определите пороговые значения для отображаемой метрики. Вы можете наблюдать изменения на оси Y диаграммы.

   В этом примере мы определяем три диапазона использования ЦП с соответствующими цветами и метками.

   Вы можете видеть диапазоны, отображаемые на оси Y и во всплывающей подсказке.

Чтобы сбросить настройки по умолчанию (отменить настройки пороговых значений), выберите иконку корзины  рядом с элементом.

### Редактирование пороговых значений

1. Выберите  для редактирования плитки визуализации.
2. Выберите **Thresholds**.

   * В ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** выберите вкладку **Visual** на боковой панели, затем выберите **Thresholds**.
   * В ![Notebooks](https://dt-cdn.net/images/notebooks-768-046137830a.webp "Notebooks") **Notebooks** выберите **Thresholds** на боковой панели.
3. С этого момента вы можете выполнить следующие действия (разверните строки для подробностей):

   Добавить пороговые значения

   Чтобы добавить пороговые значения (если выбранная визуализация поддерживает дополнительные пороговые значения), выберите  **Threshold**.

   Отключить пороговые значения

   Чтобы отключить (скрыть) существующие пороговые значения, используйте переключатель. Настройки не теряются. Вы можете включить их снова, если передумаете.

   Удалить пороговые значения

   Чтобы удалить существующие пороговые значения, выберите иконку корзины .

   Редактировать настройки пороговых значений

   Чтобы изменить существующие настройки пороговых значений, просто отредактируйте поля.

   Добавить диапазон

   Чтобы добавить диапазон к существующим пороговым значениям, выберите  **Add range**.

   Удалить диапазон

   Чтобы удалить диапазон из существующих пороговых значений, выберите кнопку удаления в этой строке.
