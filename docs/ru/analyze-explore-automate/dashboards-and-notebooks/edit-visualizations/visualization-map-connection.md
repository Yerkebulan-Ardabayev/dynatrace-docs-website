# Визуализация карты соединений


* Latest Dynatrace
* Практическое руководство
* Чтение: 4 мин
* Опубликовано 3 апреля 2025 г.

Dashboards версия 1.311+ Notebooks версия 1.311+

Когда использовать

Используйте карту соединений для визуализации связей между точками на карте. Примеры включают сетевой трафик между серверами или дата-центрами, а также авиамаршруты для оптимизации расписания и улучшения клиентского опыта.

## Примеры

Чтобы попробовать пример

1. Создайте DQL-плитку в ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** или DQL-секцию в ![Notebooks](https://dt-cdn.net/images/notebooks-768-046137830a.webp "Notebooks") **Notebooks**.
2. Скопируйте данные примера и вставьте их в поле редактирования DQL.
3. Выполните запрос.
4. Выберите визуализацию и поэкспериментируйте с настройками визуализации.

## Взаимодействие с графиком

### Взаимодействие при выделении

Когда вы выбираете значение на графике и закрепляете отображаемую всплывающую подсказку, вы можете навести курсор на неё для отображения меню с опциями, специфичными для выделения.

Доступные взаимодействия зависят от вашего запроса и визуализации. Например, если вы выберете хост на линейном графике и наведёте курсор на всплывающую подсказку, вы увидите меню с элементами, такими как:

* **Copy name** — копировать имя выбранного хоста.
* **Fields** — раздел с подменю для каждого поля запроса. Подменю поля предлагает специфичные для поля опции, такие как:

  + **Copy value** — копировать значение поля. Также отображает тип поля.
  + **Hide** — скрыть поле на графике.
  + **Explain value** — использовать ИИ для объяснения поля.
  + **Add command to query** — раздел команд, специфичных для поля, которые можно автоматически добавить к запросу.
  + Также может быть указано рекомендованное приложение.
* **Visual options** — открывает панель редактирования для изменения параметров визуализации выбранного элемента.
* **Set color** — открывает панель редактирования для изменения цвета выбранного элемента.
* ![Infrastructure & Operations](https://dt-cdn.net/images/infrasctucture-operations-highresolution-1025-07d1bc45b5.png "Infrastructure & Operations") **Go to host** — открывает выделение в ![Infrastructure & Operations](https://dt-cdn.net/images/infrasctucture-operations-highresolution-1025-07d1bc45b5.png "Infrastructure & Operations") **Infrastructure & Operations**.

  В целом, если есть рекомендованные приложения для открытия выбранного элемента, меню предлагает прямые ссылки на эти приложения, за которыми следует опция **Open with** для выбора другого целевого приложения.
* **Open with** — подробности см. в разделе [Переходы и навигация](../drilldowns-and-navigation.md "Drill down from Dashboards and Notebooks using links based on intents or URLs.").

Если вы не уверены, что выбрали правильную визуализацию, используйте [селектор визуализации](../edit-visualizations.md#select-visualization "Create, edit, and view visualizations on your Dynatrace dashboards and notebooks.") для выбора различных визуализаций.

## Вид

* **Масштаб по умолчанию**

  Установите уровень масштабирования по умолчанию для карты, выбрав один из следующих вариантов:

  + **Data**: автоматически настраивает уровень масштабирования для отображения всех точек данных в пределах видимой области карты.
  + **World**: устанавливает уровень масштабирования для отображения всего мира.
  + **Custom**: позволяет указать координаты центра карты и установить пользовательский уровень масштабирования.
* **Show country regions**

  Включите, чтобы отображать границы регионов внутри стран.

## Сопоставление данных

Раздел сопоставления данных показывает, как столбец вашего результата сопоставлен с визуализацией.

Развернуть для общих правил настройки сопоставления данных

Разверните раздел **Data mapping** в настройках визуализации, чтобы увидеть, как данные вашего результата сопоставлены с визуализацией, и при необходимости скорректировать эти настройки.

* Обязательные поля отмечены звёздочкой (`*`). Пример:

  ![Пример сопоставления данных: линейный график](https://dt-cdn.net/images/example-data-mapping-line-chart-1-732-eb9cca00d8.png)
* Типы данных отображаются рядом с именами полей в выпадающих списках и сопоставленных полях.
* Единицы измерения отображаются, если назначена только одна.
* Поля результата группируются в **Suitable** и **Unsuitable**. Поля помечаются как неподходящие, если они не могут быть использованы для отображения данных в визуализации. Пример:

  ![Пример сопоставления данных: линейный график, выпадающий список Time](https://dt-cdn.net/images/example-data-mapping-line-chart-2-763-9cc593d261.png)
* Автоматическое применение настроек сопоставления данных по умолчанию:

  Dynatrace версия 1.319+

  + Существующие плитки и секции считаются пользовательскими. Их конфигурации сопоставления данных не обновляются автоматически.
  + Вновь созданные плитки и секции применяют настройки сопоставления данных по умолчанию. Если вы не изменяете эти настройки вручную, они могут измениться, если новое выполнение плитки/секции изменит результаты и появятся отсутствующие или новые поля, лучше подходящие для сопоставления данных.

### Настройки сопоставления данных для визуализации

* **Latitude**: выберите значение из вашего запроса для использования в качестве широты каждой точки на карте.
* **Longitude**: выберите значение из вашего запроса для использования в качестве долготы каждой точки на карте.
* **Color value**: выберите значение из вашего запроса для использования в качестве цвета каждой точки на карте.

## Линия

* **Line type**: выберите форму линии соединения.

  + **Linear**: прямая линия между двумя точками.
  + **Smooth**: изогнутая линия между двумя точками.
* **Stroke style**: выберите стиль линии соединения.

  + **Solid**: непрерывная линия между двумя точками.
  + **Dashed**: пунктирная линия между двумя точками.
* **Direction indicator**: где отображать индикатор направления на линии соединения.

  + **Start**: отображает индикатор направления в начале линии соединения.
  + **End**: отображает индикатор направления в конце линии соединения.
  + **Both**: отображает индикатор направления на обоих концах линии соединения.
* **Thickness**: толщина (`1`-`64` пикс.) линии соединения.

## Легенда и всплывающая подсказка

* **Show custom fields**: для отображения пользовательских полей (имя и значение) при наведении на область карты включите **Show custom fields** и выберите каждое пользовательское поле для отображения.
* **Text truncation**: определяет способ обрезки текста, когда полный текст не может быть отображён.

  + **A...**: обрезка с правого конца текста (когда правый конец менее важен)
  + **A...B**: обрезка из середины текста (когда середина менее важна)
  + **...B**: обрезка с левого конца текста (когда левый конец менее важен)
* **Show legend**: для отображения легенды карты включите **Show legend** и выберите **Position** легенды:

  + **Auto**: выбирает подходящее расположение на основе размера карты и доступного пространства.
  + **Bottom**: отображает легенду под картой.
  + **Right**: отображает легенду справа от карты.

## Цвет

* **Line colors**

  Выберите способ окраски линий соединения:

  + **Color palette**: отображает все линии соединения в оттенке из выбранной цветовой палитры. Оттенок, используемый для каждой линии соединения, соответствует значению **Color value** относительно других областей.

    Пример

    Если значения **Color value**, возвращённые вашим запросом, находятся в диапазоне от `0` до `100`

    - Линия соединения со значением около `0` имеет оттенок из правого конца палитры.
    - Линия соединения со значением около `100` имеет оттенок из левого конца палитры.
  + **Single-color**: отображает все линии соединения одним цветом. Выберите цвет из списка или введите HEX-код цвета.
  + **Custom colors**: отображает каждую линию соединения пользовательским цветом, определённым вами.

    Для каждого пользовательского цвета, который вы хотите добавить

    1. Выберите  **Color**.
    2. Введите значение, оператор и цвет для этого значения и оператора.

    Пример

    Предположим, вы хотите окрасить линии соединения по трём уровням **Color value**:

    - Зелёный, если **Color value** меньше `4 000`
    - Жёлтый, если **Color value** достигает или превышает порог `4 000`
    - Красный, если **Color value** достигает или превышает порог `5 000`

    Для настройки

    - Выберите  **Color** и добавьте строку пользовательского цвета со значением `0`, оператором `>=` и нужным оттенком зелёного. Если **Color value** равно `0` или выше, линия соединения будет зелёной.
    - Выберите  **Color** и добавьте строку пользовательского цвета со значением `4 000`, оператором `>=` и нужным оттенком жёлтого. Если **Color value** равно `4 000` или выше, линия соединения будет жёлтой.
    - Выберите  **Color** и добавьте строку пользовательского цвета со значением `5 000`, оператором `>=` и нужным оттенком красного. Если **Color value** равно `5 000` или выше, линия соединения будет красной.

## Лимиты запросов

Используйте раздел **Query limits** для проверки и настройки лимитов запросов Grail для секции ноутбука или плитки дашборда. Эти настройки определяют максимальные лимиты при получении данных. Превышение любого лимита приведёт к генерации предупреждения.

Плитки дашбордов и секции ноутбуков, созданные в Dynatrace версии ранее 1.296, не затрагиваются. Существующие плитки/секции будут возвращать те же результаты, что и ранее.

* **Read data limit (GB)**

  Лимит в гигабайтах для объёма данных, которые будут просканированы при чтении.
* **Record limit**

  Максимальное количество записей результата, которое вернёт этот запрос. По умолчанию: 1 000 записей. Чтобы увидеть больше записей, необходимо увеличить значение **Record limit**.

  + Если ваш запрос не содержит `limit`, например

    ```
    fetch logs
    ```

    применяется значение **Record limit**. По умолчанию вы увидите до 1 000 записей.
  + Если ваш запрос также включает `limit`, например

    ```
    fetch logs


    | limit 2000
    ```

    применяется меньшее из двух значений (либо `limit` в вашем запросе, либо **Record limit** в веб-интерфейсе).

    В приведённом выше примере вы всё равно увидите только 1 000 записей, если не увеличите значение **Record limit**.
* **Result size limit**

  Максимальное количество байтов результата, которое вернёт этот запрос. Для лучшей производительности при типичных запросах и документах меньшего размера значение по умолчанию установлено на 1 МБ.
* **Sampling (только Logs и Spans)**

  Приводит к выборке подмножества записей Log или Span.

## Единицы измерения и форматы

Для переопределения единиц измерения и форматов по умолчанию в визуализации дашборда или ноутбука

1. Выберите  для редактирования плитки визуализации.
2. Выберите вкладку **Visual**.
3. Выберите **Units and formats**.
4. Выберите **Override**.
5. Выберите  **Override**
6. В выпадающем списке выберите элемент, для которого вы хотите добавить переопределение единицы измерения.

   Это числовой столбец базового результата DQL, поэтому он зависит от запроса. Например:

   * Запрос `fetch events` возвращает события. Выпадающий список здесь позволяет выбрать числовое поле (например, `transfer_size`) из результатов.
   * Запрос `timeseries avg(dt.host.cpu.usage)` возвращает одну временную серию для `avg(dt.host.cpu.usage)`. Эта временная серия является единственным доступным вариантом в списке.
7. Определите переопределение.

   * **Default unit**: базовая единица измерения, в которой были получены значения. Это `None`, если она не была включена в результат DQL, или автоматически определяется единицей, переданной из результата DQL. Это поле не приводит к конвертации.
   * **Displayed unit**: после определения единицы измерения по умолчанию вы можете использовать **Displayed unit** для конвертации. Например, если результат DQL определил числовое значение как `Bytes`, **Displayed unit** предлагает подходящий список конвертаций байтов, такой как `Kilobyte` и `Megabyte`. В отличие от **Default unit**, **Displayed unit** всегда является числовой конвертацией.
   * **Decimals** отображает количество десятичных знаков (степень точности) по умолчанию. Чтобы увидеть это в действии, измените выбор **Decimals** и наблюдайте за изменением в визуализации.
   * **Suffix** отображает суффикс после единицы измерения. Чтобы увидеть это в действии, введите строку и наблюдайте за изменением в визуализации. Когда вы не можете найти нужную единицу измерения, используйте **Suffix** для отображения желаемой единицы.
8. Включите **Abbreviate large numbers**, если хотите отображать большие числа в сокращённом виде. Например, `1053` станет `1.1K`.

Для сброса к значениям по умолчанию (отмены настроек переопределения для выбранного элемента) выберите значок корзины  рядом с элементом.

Пример для дашбордов

В этом примере используется линейный график, но параметры применимы к другим визуализациям.

1. В ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** создайте дашборд.
2. Выберите  и в разделе **Snippets** меню выберите **Metrics** > **Chart average CPU across all hosts**.
3. В панели редактирования секции выберите вкладку **Visual** и выберите **Line**.
4. Выберите **Units and formats**.

   ![Выбор Units and formats](https://dt-cdn.net/images/units-example-proc-01-1450-635695f6b8.png)
5. Выберите  **Override**.

   ![Выбор Add Override.](https://dt-cdn.net/images/units-example-proc-02-1450-e1805870b1.png)
6. В выпадающем списке выберите метрику, для которой вы хотите добавить переопределение. В этом примере доступна только одна метрика.
7. Определите переопределение для отображаемой метрики. Вы можете наблюдать изменения на оси Y графика.

   * **Default unit** отображает `Percent (%)`, что является единицей измерения по умолчанию для выбранной метрики. Попробуйте другую настройку, например `One`, чтобы отобразить результат как долю от 1.
   * **Displayed unit** отображает `Auto`. Вы можете изменить его на другую единицу, например `One`, чтобы отобразить результат как долю от 1.

     Поддерживаются только линейные и статические преобразования. Например, нельзя преобразовать `Degree Celsius(°C)` в `Degree Fahrenheit(°F)` или `Usd(US$)` в `Eur(€)`.
   * **Decimals** отображает количество десятичных знаков (степень точности) по умолчанию. Чтобы увидеть это в действии, измените выбор **Decimals** и наблюдайте за изменением в визуализации.

     Например, измените это:

     ![Настройка Decimals до](https://dt-cdn.net/images/units-example-proc-03-1454-6a84518802.png)

     На это:

     ![Настройка Decimals после](https://dt-cdn.net/images/units-example-proc-04-1450-03cc893027.png)
   * **Suffix** отображает необязательный суффикс после единицы измерения. Чтобы увидеть это в действии, введите строку и наблюдайте за изменением в визуализации.

Для сброса к значениям по умолчанию (отмены настроек переопределения для выбранной метрики) выберите значок корзины  рядом с метрикой.
