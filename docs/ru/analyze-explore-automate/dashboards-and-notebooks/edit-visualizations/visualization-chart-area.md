Когда использовать визуализацию диаграммы с областями

Используйте визуализацию диаграммы с областями:

* Когда вы хотите показать, как различные категории вносят вклад в общий тренд с течением времени, подчёркивая величину изменений в каждой категории.
* Когда нужно сравнить вклад различных категорий с течением времени — это помогает визуализировать долю каждой категории в определённый момент времени.
* Когда нужно показать кумулятивные данные, такие как количество сессий или доход с течением времени. Диаграммы с областями эффективны для отображения суммы/роста значений.

## Взаимодействие с диаграммой

Для доступа к инструментам диаграммы:

* Когда пространство ограничено, наведите курсор на диаграмму и выберите **Инструменты диаграммы**, чтобы открыть меню инструментов.
* Когда достаточно места, инструменты диаграммы отображаются на панели инструментов.

  ![Панель инструментов временных рядов — полная](https://dt-cdn.net/images/chart-tool-bar-269-c1d9a7bc39.png)

  При наведении курсора на диаграмму панель инструментов по умолчанию отображается в правом нижнем углу диаграммы в свёрнутом виде.

### Параметры панели инструментов

### Правила масштабирования

Для диаграммы временных рядов (линейная, с областями, столбчатая):

* Вы не можете масштабировать диаграмму, если временной интервал задан в запросе DQL.
* ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards**

  Если вы масштабируете плитку без пользовательского временного интервала плитки, глобальный временной интервал панели мониторинга обновляется соответственно, так что временные интервалы остаются синхронизированными по всей панели мониторинга, за исключением случаев, когда плитка имеет собственный переопределяющий временной интервал.

  Если вы масштабируете плитку с пользовательским временным интервалом, но без временного интервала в запросе, выбранный временной интервал применяется к пользовательскому временному интервалу плитки, и изменённый временной интервал отражается на панели мониторинга.
* ![Notebooks](https://dt-cdn.net/images/notebooks-768-046137830a.webp "Notebooks") **Notebooks**

  Если вы масштабируете временной ряд в секции Notebooks, временной интервал секции изменяется, и изменение сохраняется в Notebooks.

### Взаимодействие с выделением

Когда вы выбираете значение на диаграмме и закрепляете отображаемую всплывающую подсказку, вы можете навести курсор на всплывающую подсказку, чтобы отобразить меню параметров, специфичных для выделения.

Доступные взаимодействия с диаграммой зависят от вашего запроса и визуализации. Например, если вы выбираете хост на линейной диаграмме и наводите курсор на всплывающую подсказку, вы увидите меню с такими пунктами, как:

* **Копировать имя** — копирует имя выбранного хоста.
* **Поля** — раздел с подменю для каждого поля запроса. Подменю поля предлагает специфичные для поля параметры, такие как:

  + **Копировать значение** — копирует значение поля. Также отображает тип поля.
  + **Скрыть** — скрывает поле на диаграмме.
  + **Объяснить значение** — использует ИИ для объяснения поля.
  + **Добавить команду к запросу** — раздел команд, специфичных для поля, которые можно автоматически добавить к вашему запросу.
  + Также может быть указано рекомендуемое приложение.
* **Визуальные параметры** — открывает панель редактирования, чтобы вы могли изменить параметры визуализации для выбранного элемента.
* **Установить цвет** — открывает панель редактирования, чтобы вы могли изменить цвет выбранного элемента.
* ![Infrastructure & Operations](https://dt-cdn.net/images/infrasctucture-operations-highresolution-1025-07d1bc45b5.png "Infrastructure & Operations") **Перейти к хосту** — открывает выбранный элемент в ![Infrastructure & Operations](https://dt-cdn.net/images/infrasctucture-operations-highresolution-1025-07d1bc45b5.png "Infrastructure & Operations") **Infrastructure & Operations**.

  В целом, если есть рекомендуемые приложения для открытия выбранного элемента, меню предлагает прямые ссылки на эти приложения, а затем опцию **Open with** для выбора другого целевого приложения.
* **Open with** — подробнее см. в разделе [Детализация и навигация](../drilldowns-and-navigation.md "Детализация из Dashboards и Notebooks с использованием ссылок на основе интентов или URL.").

Используйте поле заголовка в верхней части панели параметров (изначально `Untitled tile` или `Untitled section`), чтобы добавить заголовок к плитке панели мониторинга или секции Notebooks.

* Вы можете использовать эмодзи, такие как ð и ð и â¤ï¸.
* Вы можете использовать переменные.

**Пример:**

1. Определите переменные с именами `Status` и `Emoji` в вашей панели мониторинга.
2. Установите заголовок `Current $Emoji status is $Status`.
3. Установите `Status` в значение `Good`.
4. Установите `Emoji` в значение `ð`.

Заголовок будет отображаться как `Current ð status is Good`.

## Сопоставление данных

Раздел сопоставления данных показывает, как столбец вашего результата сопоставлен с визуализацией.

Например, при работе с DQL и временными рядами в результате есть столбец с массивом значений для каждого ряда (одна строка с одной ячейкой значений). Для временного ряда это сопоставление выполняется полностью автоматически, но раздел сопоставления данных показывает, какой базовый столбец сопоставлен.

Развернуть общие правила настройки сопоставления данных

Разверните раздел **Сопоставление данных** в настройках визуализации, чтобы увидеть, как данные в вашем результате сопоставлены с визуализацией, и при необходимости настроить эти параметры.

* Обязательные поля отмечены звёздочкой (`*`). Пример:

  ![Пример сопоставления данных: линейная диаграмма](https://dt-cdn.net/images/example-data-mapping-line-chart-1-732-eb9cca00d8.png)
* Типы данных отображаются рядом с именами полей в раскрывающихся списках и сопоставленных полях.
* Единицы измерения отображаются, когда назначена только одна.
* Поля результата сгруппированы в **Подходящие** и **Неподходящие**. Поля помечаются как неподходящие, если они не могут использоваться для отображения данных в визуализации. Пример:

  ![Пример сопоставления данных: линейная диаграмма, раскрывающийся список Время](https://dt-cdn.net/images/example-data-mapping-line-chart-2-763-9cc593d261.png)
* Автоматическое применение настроек сопоставления данных по умолчанию:

  Dynatrace версии 1.319+

  + Уже существующие плитки и секции считаются определёнными пользователем. Их конфигурации сопоставления данных не обновляются автоматически.
  + Вновь созданные плитки и секции применяют настройку сопоставления данных по умолчанию. Если вы не изменяете эти настройки вручную, они могут измениться, если новое выполнение плитки/секции изменит результаты и появятся отсутствующие поля или новые поля, лучше подходящие для сопоставления данных.

Ограничения сопоставления данных для визуализаций графиков на основе событий

Dynatrace версии 1.322+

#### Что изменилось?

Чтобы сделать сопоставление данных проще и интуитивнее, мы ограничили определённые поля в интерфейсе для диаграмм на основе событий, показывая только наиболее релевантные параметры.

* **До**: До версии Dynatrace 1.322 поля, такие как `timestamp`, `interval` и `duration`, могли быть сопоставлены с опцией «names» в интерфейсе сопоставления данных.
* **После**: Начиная с версии Dynatrace 1.322, эти поля больше недоступны для выбора в опции «names».

#### Влияние

Если ваши диаграммы ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** или ![Notebooks](https://dt-cdn.net/images/notebooks-768-046137830a.webp "Notebooks") **Notebooks** используют эти поля, затронутые плитки отобразят сообщение об ошибке, например:

`Selected field unavailable. The field "" is no longer available. Try adjusting the data, or select another field or visualization.`

#### Решение

Чтобы решить эту проблему

1. Замените существующую команду `summarize` на `makeTimeseries`.
2. Замените параметр `by: { ... 4h }` на `interval: 4h`, используя тот же интервал, что и в предыдущей конфигурации в `by`.

После применения этих изменений сопоставление данных будет правильно разрешать подходящие поля для опции «names», и ваши плитки будут работать как ожидается.

### Настройки сопоставления данных, специфичные для визуализации

Диаграмма с областями отображает одно или несколько значений с течением времени, поэтому сопоставление должно включать следующее:

* **Время**: столбец вашего результата, используемый для оси X ([timestamp](../../../platform/grail/dynatrace-query-language/data-types.md#timestamp "Список типов данных DQL.") или [timeframe](../../../platform/grail/dynatrace-query-language/data-types.md#timeframe "Список типов данных DQL.")).
* **Интервал**: это значение сопоставляется автоматически и не может быть изменено. Оно позволяет узнать, какие поля сопоставлены для результатов на основе временных рядов. Оно берёт первое доступное поле интервала из набора результатов при использовании временного ряда (также включает любые данные на основе makeTimeseries).
* **Значения**: выбор одного или нескольких значений, которые ваша диаграмма отображает с течением времени.
* **Имена**: элементы, отображаемые, например, в легенде и названиях рядов.

## Параметры диаграммы с областями

Чтобы быстро узнать о параметрах и решить, что лучше подходит, включайте и выключайте параметры и сразу смотрите эффект на диаграмме. Например, лучше ли выглядит с меткой или без? Включите и выключите этот параметр и посмотрите сами.

### Цветовые палитры

Вы можете выбрать цветовую палитру из списка в разделе **Цветовые палитры**.

Для линейной диаграммы, диаграммы с областями или столбчатой диаграммы вы можете при необходимости [переопределить выбранную цветовую палитру](#series-overrides) для любого ряда.

### Положение легенды

Выберите, где отображать легенду визуализации:

* **Автоматически** — Dynatrace выбирает подходящее расположение
* **Скрыта** — легенда не отображается
* **Внизу** — легенда отображается вдоль нижней части визуализации
* **Справа** — легенда отображается с правой стороны визуализации

### Поля легенды

Выберите, какие элементы отображать в легенде.

### Нулевые значения

**Соединять нулевые значения** — указывает, следует ли соединять нулевые значения в визуализации.

### Ось времени

Выбирает ось X.

### Левая ось

Устанавливает масштаб левой оси:

* **Логарифмический**
* **Линейный**

### Минимум

Минимум диапазона визуализации.

* **Авто** — Dynatrace автоматически выбирает подходящий минимум на основе данных (0 или минимальное значение).
* **Минимальное значение** — минимум устанавливается равным минимальному значению данных.
* **Пользовательский** — отображает поле ввода для ввода пользовательского минимального значения.

### Максимум

Максимум диапазона визуализации.

* **Авто** — Dynatrace автоматически выбирает подходящий максимум на основе данных (0 или максимальное значение).
* **Максимальное значение** — максимум устанавливается равным максимальному значению данных.
* **Пользовательский** — отображает поле ввода для ввода пользовательского максимального значения.

### Метка

Вертикальный текст, отображаемый как метка для оси Y.

### Переопределение рядов

Для линейной диаграммы, диаграммы с областями или столбчатой диаграммы вы можете при необходимости переопределить выбранную цветовую палитру для любого ряда.

Раздел **Переопределение рядов** содержит список всех настроенных переопределений.

Чтобы добавить или изменить переопределение для ряда

1. Отредактируйте плитку визуализации.
2. Выберите ряд, для которого хотите переопределить цвет палитры. Есть два способа выбрать ряд:

   * На графике выберите ряд и затем выберите **Редактировать ряд** во всплывающем окне
   * Выберите **Добавить переопределение** в разделе **Переопределение рядов** в правом нижнем углу страницы и затем выберите ряд из списка.
3. Измените настройку **Цвет** на цвет, который хотите отображать для выбранного ряда.
4. Выберите **Обновить**, чтобы сохранить изменения на панели мониторинга.

## Ограничения запросов

Используйте раздел **Ограничения запросов**, чтобы проверить и настроить ограничения запросов Grail для каждой секции Notebooks или плитки панели мониторинга. Эти настройки определяют максимальные ограничения при получении данных. Превышение любого ограничения вызовет предупреждение.

Плитки панели мониторинга и секции Notebooks, созданные в Dynatrace ранее версии 1.296, не затронуты. Эти существующие плитки/секции будут возвращать те же результаты, что и раньше.

* **Ограничение чтения данных (ГБ)**

  Ограничение в гигабайтах для объёма данных, которые будут просканированы во время чтения.
* **Ограничение записей**

  Максимальное количество записей результата, которые вернёт этот запрос. По умолчанию: 1 000 записей. Чтобы увидеть больше записей, вам нужно увеличить значение **Ограничение записей**.

  + Если ваш запрос не содержит `limit`, например

    ```
    fetch logs
    ```

    применяется значение **Ограничение записей**. По умолчанию вы увидите до 1 000 записей.
  + Если ваш запрос также включает `limit`, например

    ```
    fetch logs



    | limit 2000
    ```

    применяется меньшее из двух значений (либо `limit` в вашем запросе, либо **Ограничение записей** в веб-интерфейсе).

    В примере выше вы всё равно увидите только 1 000 записей, если не увеличите значение **Ограничение записей**.
* **Ограничение размера результата**

  Максимальное количество байтов результата, которые вернёт этот запрос. Для лучшей производительности с типичными запросами и небольшими документами значение по умолчанию установлено на 1 МБ.
* **Выборка (только для Logs и Spans)**

  Приводит к выбору подмножества записей Log или Span.

## Пороговые значения

### Настройка пороговых значений

Чтобы настроить пороговые значения в визуализации панели мониторинга или Notebooks

1. Выберите  для редактирования плитки визуализации.
2. Выберите **Пороговые значения**.

   * В ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** выберите вкладку **Визуальные** на боковой панели, а затем выберите **Пороговые значения**.
   * В ![Notebooks](https://dt-cdn.net/images/notebooks-768-046137830a.webp "Notebooks") **Notebooks** выберите **Пороговые значения** на боковой панели.
3. Выберите  **Пороговое значение**.
4. Определите пороговые значения. Для каждого диапазона:

   * Выберите цвет для отображения этого диапазона
   * Выберите оператор для определения порогового значения этого диапазона
   * Введите статическое значение для сравнения (с использованием выбранного оператора) с возвращённым значением
   * Введите метку для связи с определённым диапазоном.

     + Метки применяются только к диаграммам с осью Y, таким как диаграммы временных рядов и категориальная столбчатая диаграмма.
     + Метки не могут быть определены для таблиц и диаграммы с одним значением.

Пример настройки пороговых значений в Dashboards

В этом примере используется столбчатая диаграмма, но параметры применимы к другим визуализациям.

1. В ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** создайте панель мониторинга.
2. Выберите  и в разделе **Сниппеты** меню выберите **Метрики** > **Диаграмма топ-10 хостов по использованию CPU**.
3. На панели редактирования секции выберите вкладку **Визуальные** и выберите **Столбчатая**.
4. Выберите **Пороговые значения**.
5. Выберите  **Пороговое значение**.

   Отображается пустой набор полей пороговых значений.
6. Определите пороговые значения для отображаемой метрики. Вы можете наблюдать изменения на оси Y диаграммы.

   В этом примере мы определяем три диапазона использования CPU с соответствующими цветами и метками.

   Вы можете видеть диапазоны, отображаемые на оси Y и во всплывающей подсказке.

Чтобы сбросить к значениям по умолчанию (отменить настройки пороговых значений), выберите значок корзины  рядом с элементом.

### Редактирование пороговых значений

1. Выберите  для редактирования плитки визуализации.
2. Выберите **Пороговые значения**.

   * В ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** выберите вкладку **Визуальные** на боковой панели, а затем выберите **Пороговые значения**.
   * В ![Notebooks](https://dt-cdn.net/images/notebooks-768-046137830a.webp "Notebooks") **Notebooks** выберите **Пороговые значения** на боковой панели.
3. На этом этапе вы можете выполнить следующее (раскройте строки для подробностей):

   Добавить пороговые значения

   Чтобы добавить пороговые значения (если выбранная визуализация поддерживает дополнительные пороговые значения), выберите  **Пороговое значение**.

   Отключить пороговые значения

   Чтобы отключить (скрыть) существующие пороговые значения, используйте переключатель. Никакие настройки не теряются. Вы можете включить их обратно, если передумаете.

   Удалить пороговые значения

   Чтобы удалить существующие пороговые значения, выберите значок корзины .

   Редактировать настройки пороговых значений

   Чтобы изменить существующие настройки пороговых значений, просто отредактируйте поля.

   Добавить диапазон

   Чтобы добавить диапазон к существующим пороговым значениям, выберите  **Добавить диапазон**.

   Удалить диапазон

   Чтобы удалить диапазон из существующих пороговых значений, выберите кнопку удаления в этой строке.

## Единицы измерения и форматы

Чтобы переопределить единицы измерения и форматы по умолчанию в визуализации панели мониторинга или Notebooks

1. Выберите  для редактирования плитки визуализации.
2. Выберите вкладку **Визуальные**.
3. Выберите **Единицы измерения и форматы**.
4. Выберите **Переопределить**.
5. Выберите  **Переопределение**
6. В раскрывающемся списке выберите элемент, для которого хотите добавить переопределение единиц измерения.

   Это числовой столбец базового результата DQL, поэтому он варьируется в зависимости от запроса. Например:

   * Запрос `fetch events` возвращает события. Раскрывающийся список здесь позволяет выбрать числовое поле (например, `transfer_size`) из результатов.
   * Запрос `timeseries avg(dt.host.cpu.usage)` возвращает один временной ряд для `avg(dt.host.cpu.usage)`. Этот временной ряд затем является единственным доступным для выбора параметром в списке.
7. Определите переопределение.

   * **Единица измерения по умолчанию**: Базовая единица, в которой были получены значения. Она равна `None`, если не была включена в результат DQL, или автоматически определяется единицей, переданной из результата DQL. Это поле не приводит к конвертации.
   * **Отображаемая единица**: После определения единицы по умолчанию вы можете использовать **Отображаемую единицу** для конвертации. Например, если результат DQL определил ваше числовое значение в результате как `Bytes`, **Отображаемая единица** теперь предлагает подходящий список конвертаций байтов, таких как `Kilobyte` и `Megabyte`. В отличие от **Единицы по умолчанию**, **Отображаемая единица** всегда является числовой конвертацией.
   * **Десятичные знаки** отображает количество десятичных знаков по умолчанию (степень точности) для отображения. Чтобы увидеть это в действии, измените выбор **Десятичные знаки** и наблюдайте за изменением в визуализации.
   * **Суффикс** отображает суффикс, который будет показан после единицы измерения. Чтобы увидеть это в действии, введите строку и наблюдайте за изменением в визуализации. Когда вы не находите нужную единицу измерения, вы можете использовать **Суффикс** для отображения желаемой единицы.
8. Включите **Сокращать большие числа**, если хотите отображать большие значения в сокращённой форме. Например, `1053` становится `1.1K`.

Чтобы сбросить к значениям по умолчанию (отменить настройки переопределения для выбранного элемента), выберите значок корзины  рядом с элементом.

Пример для панелей мониторинга

В этом примере используется линейная диаграмма, но параметры применимы к другим визуализациям.

1. В ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** создайте панель мониторинга.
2. Выберите  и в разделе **Сниппеты** меню выберите **Метрики** > **Диаграмма среднего CPU по всем хостам**.
3. На панели редактирования секции выберите вкладку **Визуальные** и выберите **Линейная**.
4. Выберите **Единицы измерения и форматы**.

   ![Выберите Единицы измерения и форматы](https://dt-cdn.net/images/units-example-proc-01-1450-635695f6b8.png)
5. Выберите  **Переопределение**.

   ![Выберите Добавить переопределение.](https://dt-cdn.net/images/units-example-proc-02-1450-e1805870b1.png)
6. В раскрывающемся списке выберите метрику, для которой хотите добавить переопределение. В этом примере доступна только одна метрика для выбора.
7. Определите переопределение для отображаемой метрики. Вы можете наблюдать изменения на оси Y диаграммы.

   * **Единица по умолчанию** отображает `Percent (%)`, которая является единицей по умолчанию для выбранной метрики. Попробуйте другую настройку, например `One`, чтобы вместо этого отобразить результат как долю от 1.
   * **Отображаемая единица** отображает `Auto`. Вы можете изменить её на другую единицу, например `One`, чтобы вместо этого отобразить результат как долю от 1.

     Поддерживаются только линейные и статические конвертации. Например, вы не можете конвертировать `Degree Celsius(°C)` в `Degree Fahrenheit(°F)` или конвертировать `Usd(US$)` в `Eur(€)`.
   * **Десятичные знаки** отображает количество десятичных знаков по умолчанию (степень точности) для отображения. Чтобы увидеть это в действии, измените выбор **Десятичные знаки** и наблюдайте за изменением в визуализации.

     На это:

     ![Настройка десятичных знаков после](https://dt-cdn.net/images/units-example-proc-04-1450-03cc893027.png)
   * **Суффикс** отображает необязательный суффикс после единицы измерения. Чтобы увидеть это в действии, введите строку и наблюдайте за изменением в визуализации.

Чтобы сбросить к значениям по умолчанию (отменить настройки переопределения для выбранной метрики), выберите значок корзины  рядом с метрикой.
