Когда использовать визуализацию таблицы

Используйте визуализацию таблицы:

* Когда вы хотите получить первый образец и впечатление о данных, возвращаемых вашим запросом или кодом. Например, покажите последние 100 строк логов для вашего приложения или хоста.
* Когда вам нужно просматривать множество записей одновременно. Где плотность данных и возможность сравнивать записи одну за другой важнее, чем видеть все поля результата одновременно.
* Когда вам нужно дополнительно исследовать данные с помощью взаимодействий.

## Пример

![Пример визуализации таблицы](https://dt-cdn.net/images/table-example-01a-776-2528f83012.png)

Таблица выше основана на следующем запросе.

```
timeseries cpu=avg(dt.host.cpu.usage), by:{dt.entity.host}



| sort arrayAvg(cpu), direction:"descending"



| limit 3
```

* Заголовок установлен как **Chart average CPU across all hosts**.
* Тип визуализации установлен как **Table**.
* В разделе  **Columns**:

  + **Visibility and order**: выберите **Edit**  для настройки видимых столбцов и их порядка. В примере столбец `dt.entity.host` не выбран (для скрытия имён хостов), а три других столбца (`timeframe`, `interval` и `cpu`) выбраны.
  + **Custom column types**: столбец `cpu` (временная серия) установлен как  **Sparkline**.

## Взаимодействие с таблицей

### Взаимодействия, специфичные для таблицы

Щёлкните левой кнопкой мыши по заголовку столбца для локальной сортировки результатов по этому столбцу.

* указывает на отсутствие сортировки
* указывает на сортировку по возрастанию по этому столбцу
* указывает на сортировку по убыванию по этому столбцу

### Взаимодействия, специфичные для столбца

Щёлкните правой кнопкой мыши по заголовку столбца или наведите курсор на заголовок столбца и выберите , чтобы отобразить меню команд, специфичных для столбца. Доступные опции могут различаться для разных столбцов.

* **Move left** перемещает выбранный столбец на одну позицию влево.
* **Move right** перемещает выбранный столбец на одну позицию вправо.
* **Hide column** скрывает выбранный столбец. Чтобы снова отобразить скрытый столбец, отредактируйте таблицу и в разделе  **Columns** отредактируйте  настройки **Visibility and order** для настройки видимых столбцов и их порядка.
* **Enable line wrap** и  **Disable line wrap** определяют, нужно ли переносить текст в выбранном столбце.
* **Copy field name** копирует имя поля выбранного столбца в буфер обмена.
* Раздел **Add command to query** меню содержит связанные команды, которые можно автоматически добавить к запросу, такие как:

  + **Sort ascending** добавляет команду к запросу `sort <fieldname> asc`
  + **Sort descending** добавляет команду к запросу `sort <fieldname> desc`
* **Summarize** подсчитывает, как часто каждое из уникальных значений определённого столбца встречается в результате (например, подсчёт частоты каждого идентификатора хоста).
* **Add entity names** добавляет подстановку (результат добавляется в отдельное поле с именем `dt.entity.<entity>.name`) фактического имени сущности, связанного с данным идентификатором сущности в столбце таблицы.
* **Convert to histogram** сопоставляет числовые значения с интервалами гистограммы и показывает, как часто значения попадают в каждый интервал.

### Взаимодействия, специфичные для строки

Щёлкните левой кнопкой мыши в любом месте строки, чтобы выделить всю строку. С выделенной строкой вы можете затем щёлкнуть и перетащить внутри ячейки этой строки, чтобы выбрать именно то, что хотите скопировать (**Ctrl**+**C**) из ячейки.

### Взаимодействия, специфичные для ячейки

Щёлкните правой кнопкой мыши по ячейке или наведите курсор на ячейку и выберите , чтобы отобразить меню команд, специфичных для ячейки. Доступные опции могут различаться для разных ячеек.

* **Copy value** копирует значение ячейки в буфер обмена.
* **View in fullscreen** применяется только к строкам. Отображает содержимое ячейки в полноэкранном окне.

  + и  обеспечивают навигацию от записи к записи.
  + копирует текущее значение в буфер обмена.
  + закрывает окно и возвращает к таблице.
* Может быть указано рекомендованное приложение — например, ![Kubernetes (new)](https://dt-cdn.net/images/kubernetes-new-1024-45d3de15d1.webp "Kubernetes (new)") **View Kubernetes workload** — для быстрого доступа.
* **Open with** передаёт всю запись в диалог **Open with**, не учитывая, какое поле вы выбрали. Это обеспечивает более широкий диапазон возможных опций **Open with**.
* Раздел **Add command to query** в меню ячейки содержит команды, которые можно автоматически добавить к запросу, такие как:

  + для фильтрации результатов запроса по выбранной ячейке и соответствующему оператору (например, **Greater than** значение ячейки).
  + **Add entity names** добавляет подстановку (результат добавляется в отдельное поле с именем `dt.entity.<entity>.name`) фактического имени сущности, связанного с данным идентификатором сущности.

## Заголовок

Используйте поле заголовка в верхней части панели параметров (изначально `Untitled tile` или `Untitled section`) для добавления заголовка к плитке дашборда или секции ноутбука.

* Вы можете использовать эмодзи, такие как ð и ð и â¤ï¸.
* Вы можете использовать переменные.

**Пример:**

1. Определите переменные с именами `Status` и `Emoji` в вашем дашборде.
2. Установите заголовок `Current $Emoji status is $Status`.
3. Установите `Status` в значение `Good`.
4. Установите `Emoji` в значение `ð`.

Заголовок будет отображаться как `Current ð status is Good`.

## Визуализация

Если вы не уверены, что выбрали правильную визуализацию, используйте [селектор визуализации](../edit-visualizations.md#select-visualization "Create, edit, and view visualizations on your Dynatrace dashboards and notebooks.") для выбора различных визуализаций.

Чтобы быстро изучить опции и решить, что работает лучше всего, включайте и выключайте опции и сразу наблюдайте за эффектом на графике. Например, лучше ли выглядит с подписью или без? Включите и выключите эту опцию и убедитесь сами.

## Столбцы

* **Visibility and order**: выберите **Edit**  для настройки видимых столбцов и их порядка.

  При загрузке данных таблицы в файл (выберите  > **Download result** > **CSV**), загрузка включает только столбцы, выбранные здесь.

  Это полезно, когда, например, вы хотите загрузить CSV-файл, открыть его в электронной таблице и поделиться ей с другими, но хотите исключить нерелевантные или конфиденциальные данные из загрузки.
* **Line wrap**: определяет, нужно ли переносить строки, выходящие за пределы ширины столбца.

  + **All**: применяет перенос строк ко всем столбцам таблицы.
  + **Custom**: применяет перенос строк к выбранным столбцам таблицы.
* **Monospaced font**: определяет, нужно ли использовать моноширинный шрифт.

  + **All**: применяет моноширинный шрифт ко всем столбцам таблицы.
  + **Custom**: применяет моноширинный шрифт к выбранным столбцам таблицы.
* **Custom column types**: определяет форматирование столбцов. Выберите  **Column type** для добавления строки, затем выберите один или несколько столбцов и способ форматирования выбранных столбцов.

Результаты таблицы

Если результат таблицы содержит более 50 столбцов, будут отображены только первые 50. Если результат превышает 50 столбцов, вы увидите предупреждение. Предупреждение позволит вам **Modify visibility**, где вы можете выбрать, какие столбцы отображать, и изменить порядок их отображения.

## Ячейки

* **Density** — определяет, насколько компактно отображаются строки таблицы по вертикали.

  + **Condensed** — наиболее плотный вариант отображения, для максимизации количества отображаемых строк.
  + **Default** — средний вариант отображения с типичными промежутками между строками.
  + **Comfortable** — максимальные промежутки между строками.
* **Apply threshold color to**

  + **Value** применяет цвет порога к значению, отображаемому ячейкой.
  + **Background** применяет цвет порога к фону ячейки.
* **Show threshold in row** применяет цвет порога ко всей строке.

## Лимиты запросов

Используйте раздел **Query limits** для проверки и настройки лимитов запросов Grail для секции ноутбука или плитки дашборда. Эти настройки определяют максимальные лимиты при получении данных. Превышение любого лимита приведёт к генерации предупреждения.

Плитки дашбордов и секции ноутбуков, созданные в Dynatrace версии ранее 1.296, не затрагиваются. Существующие плитки/секции будут возвращать те же результаты, что и ранее.

* **Read data limit (GB)**

  Лимит в гигабайтах для объёма данных, которые будут просканированы при чтении.
* **Record limit**

  Максимальное количество записей результата, которое вернёт этот запрос. По умолчанию: 1 000 записей. Чтобы увидеть больше записей, необходимо увеличить значение **Record limit**.

  + Если ваш запрос не содержит `limit`, например

    ```
    fetch logs
    ```

    применяется значение **Record limit**. По умолчанию вы увидите до 1 000 записей.
  + Если ваш запрос также включает `limit`, например

    ```
    fetch logs



    | limit 2000
    ```

    применяется меньшее из двух значений (либо `limit` в вашем запросе, либо **Record limit** в веб-интерфейсе).

    В приведённом выше примере вы всё равно увидите только 1 000 записей, если не увеличите значение **Record limit**.
* **Result size limit**

  Максимальное количество байтов результата, которое вернёт этот запрос. Для лучшей производительности при типичных запросах и документах меньшего размера значение по умолчанию установлено на 1 МБ.
* **Sampling (только Logs и Spans)**

  Приводит к выборке подмножества записей Log или Span.

## Единицы измерения и форматы

Для переопределения единиц измерения и форматов по умолчанию в визуализации дашборда или ноутбука

1. Выберите  для редактирования плитки визуализации.
2. Выберите вкладку **Visual**.
3. Выберите **Units and formats**.
4. Выберите **Override**.
5. Выберите  **Override**
6. В выпадающем списке выберите элемент, для которого вы хотите добавить переопределение единицы измерения.

   Это числовой столбец базового результата DQL, поэтому он зависит от запроса. Например:

   * Запрос `fetch events` возвращает события. Выпадающий список здесь позволяет выбрать числовое поле (например, `transfer_size`) из результатов.
   * Запрос `timeseries avg(dt.host.cpu.usage)` возвращает одну временную серию для `avg(dt.host.cpu.usage)`. Эта временная серия является единственным доступным вариантом в списке.
7. Определите переопределение.

   * **Default unit**: базовая единица измерения, в которой были получены значения. Это `None`, если она не была включена в результат DQL, или автоматически определяется единицей, переданной из результата DQL. Это поле не приводит к конвертации.
   * **Displayed unit**: после определения единицы измерения по умолчанию вы можете использовать **Displayed unit** для конвертации. Например, если результат DQL определил числовое значение как `Bytes`, **Displayed unit** предлагает подходящий список конвертаций байтов, такой как `Kilobyte` и `Megabyte`. В отличие от **Default unit**, **Displayed unit** всегда является числовой конвертацией.
   * **Decimals** отображает количество десятичных знаков (степень точности) по умолчанию. Чтобы увидеть это в действии, измените выбор **Decimals** и наблюдайте за изменением в визуализации.
   * **Suffix** отображает суффикс после единицы измерения. Чтобы увидеть это в действии, введите строку и наблюдайте за изменением в визуализации. Когда вы не можете найти нужную единицу измерения, используйте **Suffix** для отображения желаемой единицы.
8. Включите **Abbreviate large numbers**, если хотите отображать большие числа в сокращённом виде. Например, `1053` станет `1.1K`.

Для сброса к значениям по умолчанию (отмены настроек переопределения для выбранного элемента) выберите значок корзины  рядом с элементом.

Пример для дашбордов

В этом примере используется линейный график, но параметры применимы к другим визуализациям.

1. В ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** создайте дашборд.
2. Выберите  и в разделе **Snippets** меню выберите **Metrics** > **Chart average CPU across all hosts**.
3. В панели редактирования секции выберите вкладку **Visual** и выберите **Line**.
4. Выберите **Units and formats**.

   ![Выбор Units and formats](https://dt-cdn.net/images/units-example-proc-01-1450-635695f6b8.png)
5. Выберите  **Override**.

   ![Выбор Add Override.](https://dt-cdn.net/images/units-example-proc-02-1450-e1805870b1.png)
6. В выпадающем списке выберите метрику, для которой вы хотите добавить переопределение. В этом примере доступна только одна метрика.
7. Определите переопределение для отображаемой метрики. Вы можете наблюдать изменения на оси Y графика.

   * **Default unit** отображает `Percent (%)`, что является единицей измерения по умолчанию для выбранной метрики. Попробуйте другую настройку, например `One`, чтобы отобразить результат как долю от 1.

   * **Decimals** отображает количество десятичных знаков (степень точности) по умолчанию. Чтобы увидеть это в действии, измените выбор **Decimals** и наблюдайте за изменением в визуализации.

     Например, измените это:

     ![Настройка Decimals до](https://dt-cdn.net/images/units-example-proc-03-1454-6a84518802.png)

     На это:

Для сброса к значениям по умолчанию (отмены настроек переопределения для выбранной метрики) выберите значок корзины  рядом с метрикой.

## Пороговые значения

### Настройка пороговых значений

Для настройки пороговых значений в визуализации дашборда или ноутбука

1. Выберите  для редактирования плитки визуализации.
2. Выберите **Thresholds**.

   * В ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** выберите вкладку **Visual** в боковой панели, затем выберите **Thresholds**.
   * В ![Notebooks](https://dt-cdn.net/images/notebooks-768-046137830a.webp "Notebooks") **Notebooks** выберите **Thresholds** в боковой панели.
3. Выберите  **Threshold**.
4. Определите пороговые значения. Для каждого диапазона:

   * Выберите цвет для отображения этого диапазона
   * Выберите оператор для определения порога этого диапазона
   * Введите статическое значение для сравнения (с использованием выбранного оператора) с возвращаемым значением
   * Введите метку для ассоциации с определённым диапазоном.

     + Метки применяются только к графикам с осью Y, таким как графики временных серий и категориальная столбчатая диаграмма.
     + Метки не могут быть определены для таблиц и графика одиночного значения.

Пример настройки пороговых значений в Dashboards

В этом примере используется столбчатая диаграмма, но параметры применимы к другим визуализациям.

1. В ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** создайте дашборд.
2. Выберите  и в разделе **Snippets** меню выберите **Metrics** > **Chart top 10 hosts by CPU usage**.
3. В панели редактирования секции выберите вкладку **Visual** и выберите **Bar**.
4. Выберите **Thresholds**.
5. Выберите  **Threshold**.

   Отображается пустой набор полей пороговых значений.
6. Определите пороговые значения для отображаемой метрики. Вы можете наблюдать изменения на оси Y графика.

   В этом примере мы определяем три диапазона использования CPU с соответствующими цветами и метками.

   Вы можете видеть диапазоны, отображаемые на оси Y и во всплывающей подсказке.

Для сброса к значениям по умолчанию (отмены настроек пороговых значений) выберите значок корзины  рядом с элементом.

### Редактирование пороговых значений

1. Выберите  для редактирования плитки визуализации.
2. Выберите **Thresholds**.

   * В ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** выберите вкладку **Visual** в боковой панели, затем выберите **Thresholds**.
   * В ![Notebooks](https://dt-cdn.net/images/notebooks-768-046137830a.webp "Notebooks") **Notebooks** выберите **Thresholds** в боковой панели.
3. Далее вы можете выполнить следующие действия (раскройте строки для подробностей):

   Добавление пороговых значений

   Чтобы добавить пороговые значения (если выбранная визуализация поддерживает дополнительные пороги), выберите  **Threshold**.

   Отключение пороговых значений

   Чтобы отключить (скрыть) существующие пороговые значения, используйте переключатель. Настройки не теряются. Вы можете включить их обратно, если передумаете.

   Удаление пороговых значений

   Чтобы удалить существующие пороговые значения, выберите значок корзины .

   Редактирование настроек пороговых значений

   Чтобы изменить существующие настройки пороговых значений, просто отредактируйте поля.

   Добавление диапазона

   Чтобы добавить диапазон к существующим пороговым значениям, выберите  **Add range**.

   Удаление диапазона

   Чтобы удалить диапазон из существующих пороговых значений, выберите кнопку удаления в этой строке.
