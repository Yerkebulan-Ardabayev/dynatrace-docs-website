# Визуализация необработанных данных


Выберите визуализацию **Raw**, чтобы увидеть фактические данные, возвращаемые запросом (а не визуальное представление данных).

![Пример визуализации необработанных данных](https://dt-cdn.net/images/raw-example-812-8a1234b934.png)

Приведённая выше визуализация необработанных данных основана на следующем запросе. Часть данных была скрыта.

```
timeseries avg=avg(dt.host.cpu.load),


max=max(dt.host.cpu.load),


min=min(dt.host.cpu.load),


by:dt.entity.host


| limit 1
```

## Заголовок

Используйте поле заголовка в верхней части панели параметров (изначально `Untitled tile` или `Untitled section`), чтобы добавить заголовок к плитке дашборда или секции блокнота.

* Можно использовать эмодзи, например, и и .
* Можно использовать переменные.

**Пример:**

1. Определите переменные `Status` и `Emoji` в вашем дашборде.
2. Задайте заголовок `Current $Emoji status is $Status`.
3. Установите значение `Status` равным `Good`.
4. Установите значение `Emoji` равным ``.

Заголовок будет отображаться как `Current  status is Good`.

## Ограничения запросов

Используйте раздел **Query limits**, чтобы проверить и настроить ограничения запросов Grail для каждой секции блокнота или плитки дашборда. Эти настройки определяют максимальные лимиты при получении данных. Превышение любого лимита приведёт к появлению предупреждения.

Плитки дашбордов и секции блокнотов, созданные в Dynatrace до версии 1.296, не затрагиваются. Эти существующие плитки/секции будут возвращать те же результаты, что и раньше.

* **Read data limit (GB)**

  Лимит в гигабайтах на объём данных, которые будут просканированы при чтении.
* **Record limit**

  Максимальное количество записей результата, которое вернёт данный запрос. По умолчанию: 1 000 записей. Чтобы увидеть больше записей, увеличьте значение **Record limit**.

  + Если в вашем запросе нет `limit`, например:

    ```
    fetch logs
    ```

    применяется значение **Record limit**. По умолчанию вы увидите до 1 000 записей.
  + Если ваш запрос также содержит `limit`, например:

    ```
    fetch logs


    | limit 2000
    ```

    применяется меньшее из двух значений (`limit` в вашем запросе или **Record limit** в веб-интерфейсе).

    В приведённом выше примере вы по-прежнему увидите только 1 000 записей, если не увеличите значение **Record limit**.
* **Result size limit**

  Максимальное количество байтов результата, которое вернёт данный запрос. Для лучшей производительности при типичных запросах и документах меньшего размера значение по умолчанию установлено в 1 МБ.
* **Sampling (только для логов и спанов)**

  Приводит к выборке подмножества записей логов или спанов.

## Единицы измерения и форматы

Чтобы переопределить единицы измерения и форматы по умолчанию в визуализации дашборда или блокнота:

1. Выберите , чтобы редактировать плитку визуализации.
2. Перейдите на вкладку **Visual**.
3. Выберите **Units and formats**.
4. Выберите **Override**.
5. Выберите  **Override**
6. В выпадающем списке выберите элемент, для которого хотите добавить переопределение единицы измерения.

   Это числовой столбец базового результата DQL, поэтому он варьируется в зависимости от запроса. Например:

   * Запрос `fetch events` возвращает события. Выпадающий список здесь позволяет выбрать числовое поле (например, `transfer_size`) из результатов.
   * Запрос `timeseries avg(dt.host.cpu.usage)` возвращает один временной ряд для `avg(dt.host.cpu.usage)`. Этот временной ряд будет единственным доступным вариантом в списке.
7. Определите переопределение.

   * **Default unit**: базовая единица измерения, в которой были получены значения. Значение `None`, если она не была включена в результат DQL, или автоматически определяется единицей, переданной из результата DQL. Это поле не приводит к конвертации.
   * **Displayed unit**: после определения единицы измерения по умолчанию вы можете использовать **Displayed unit** для конвертации. Например, если результат DQL определил числовое значение в байтах (`Bytes`), **Displayed unit** предложит подходящий список вариантов конвертации, таких как `Kilobyte` и `Megabyte`. В отличие от **Default unit**, **Displayed unit** всегда выполняет числовую конвертацию.
   * **Decimals** отображает количество десятичных знаков (степень точности) по умолчанию. Чтобы увидеть эффект, измените настройку **Decimals** и наблюдайте изменения в визуализации.
   * **Suffix** отображает суффикс, который будет показан после единицы измерения. Чтобы увидеть эффект, введите строку и наблюдайте изменения в визуализации. Если вы не нашли нужную единицу измерения, используйте **Suffix** для отображения желаемой единицы.
8. Включите **Abbreviate large numbers**, если хотите отображать большие числа в сокращённом виде. Например, `1053` станет `1.1K`.

Чтобы сбросить значения по умолчанию (отменить настройки переопределения для выбранного элемента), нажмите на значок корзины  рядом с элементом.

Пример для дашбордов

В этом примере используется линейный график, но настройки применимы и к другим визуализациям.

1. В ![Дашборды](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Дашборды") **Dashboards** создайте дашборд.
2. Выберите  и в разделе **Snippets** меню выберите **Metrics** > **Chart average CPU across all hosts**.
3. На панели редактирования секции перейдите на вкладку **Visual** и выберите **Line**.
4. Выберите **Units and formats**.

   ![Выберите Units and formats](https://dt-cdn.net/images/units-example-proc-01-1450-635695f6b8.png)
5. Выберите  **Override**.

   ![Выберите Add Override.](https://dt-cdn.net/images/units-example-proc-02-1450-e1805870b1.png)
6. В выпадающем списке выберите метрику, для которой хотите добавить переопределение. В этом примере для выбора доступна только одна метрика.
7. Определите переопределение для отображаемой метрики. Вы можете наблюдать изменения на оси Y графика.

   * **Default unit** отображает `Percent (%)` — единицу измерения по умолчанию для выбранной метрики. Попробуйте другую настройку, например `One`, чтобы отображать результат как долю от 1.
   * **Displayed unit** отображает `Auto`. Вы можете изменить её на другую единицу, например `One`, чтобы отображать результат как долю от 1.

     Поддерживаются только линейные и статические преобразования. Например, невозможно конвертировать `Degree Celsius(°C)` в `Degree Fahrenheit(°F)` или `Usd(US$)` в `Eur(€)`.
   * **Decimals** отображает количество десятичных знаков (степень точности) по умолчанию. Чтобы увидеть эффект, измените настройку **Decimals** и наблюдайте изменения в визуализации.

     Например, измените это:

     ![Настройка десятичных знаков до](https://dt-cdn.net/images/units-example-proc-03-1454-6a84518802.png)

     На это:

     ![Настройка десятичных знаков после](https://dt-cdn.net/images/units-example-proc-04-1450-03cc893027.png)
   * **Suffix** отображает необязательный суффикс, который будет показан после единицы измерения. Чтобы увидеть эффект, введите строку и наблюдайте изменения в визуализации.

Чтобы сбросить значения по умолчанию (отменить настройки переопределения для выбранной метрики), нажмите на значок корзины  рядом с метрикой.
