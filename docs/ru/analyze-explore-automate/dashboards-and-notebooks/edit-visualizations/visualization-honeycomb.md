# Визуализация «Соты»

* Последняя версия Dynatrace
* Практическое руководство
* Чтение: 2 мин
* Обновлено 7 октября 2025

Когда использовать визуализацию «Соты»

Визуализация «Соты» предлагает компактное, высокоплотное представление метрик по множеству сущностей, идеально подходящее для мониторинга инфраструктуры. Используйте её для выявления горячих точек, группировки кластеров и детализации данных.

## Примеры

### Пример 1: Использование CPU хостов

![Пример сот](https://dt-cdn.net/images/visualization-example-honeycomb-01-859-230a172dea.png)

Визуализация «Соты» выше основана на следующем запросе.

```
timeseries avg(dt.host.cpu.usage), by:{dt.entity.host}



| fieldsAdd cpu = arrayAvg(`avg(dt.host.cpu.usage)`)
```

* Наведите курсор на ячейку, чтобы увидеть подробности
* Выберите ячейку, чтобы открыть меню параметров, специфичных для ячейки

![Подсказка при наведении на ячейку сот](https://dt-cdn.net/images/honeycomb-cell-tooltip-295-61aedfcf6e.png)

## Взаимодействие с диаграммой

### Взаимодействие при выборе

При выборе значения на диаграмме и закреплении отображаемой подсказки можно навести курсор на подсказку, чтобы увидеть меню параметров, специфичных для выбранного элемента.

Доступные варианты взаимодействия зависят от вашего запроса и визуализации. Например, если вы выберете хост на линейной диаграмме и наведёте курсор на подсказку, вы увидите меню со следующими пунктами:

* **Copy name** -- копирование имени выбранного хоста.
* **Fields** -- раздел с подменю для каждого поля запроса. Подменю поля предлагает специфичные параметры, такие как:

  + **Copy value** -- копирование значения поля. Также отображает тип поля.
  + **Hide** -- скрытие поля на диаграмме.
  + **Explain value** -- использование ИИ для объяснения поля.
  + **Add command to query** -- раздел команд, специфичных для поля, которые можно автоматически добавить в запрос.
  + Также может быть указано рекомендуемое приложение.
* **Visual options** -- открытие панели редактирования для изменения параметров визуализации выбранного элемента.
* **Set color** -- открытие панели редактирования для изменения цвета выбранного элемента.
* ![Infrastructure & Operations](https://dt-cdn.net/images/infrasctucture-operations-highresolution-1025-07d1bc45b5.png "Infrastructure & Operations") **Go to host** -- открытие выбранного элемента в ![Infrastructure & Operations](https://dt-cdn.net/images/infrasctucture-operations-highresolution-1025-07d1bc45b5.png "Infrastructure & Operations") **Infrastructure & Operations**.

  Как правило, если для выбранного элемента есть рекомендуемые приложения, меню предлагает прямые ссылки на эти приложения, а также опцию **Open with** для выбора другого целевого приложения.
* **Open with** -- подробнее см. [Детализация и навигация](../drilldowns-and-navigation.md "Детализация из дашбордов и блокнотов с использованием ссылок на основе интентов или URL.").

## Заголовок

Используйте поле заголовка в верхней части панели настроек (изначально `Untitled tile` или `Untitled section`), чтобы добавить заголовок к плитке дашборда или разделу блокнота.

* Можно использовать эмодзи.
* Можно использовать переменные.

**Пример:**

1. Определите переменные с именами `Status` и `Emoji` в вашем дашборде.
2. Установите заголовок `Current $Emoji status is $Status`.
3. Установите `Status` в значение `Good`.
4. Установите `Emoji` в значение эмодзи.

Заголовок будет отображаться как `Current <эмодзи> status is Good`.

## Сопоставление данных

Раздел сопоставления данных показывает, как столбец результата сопоставляется с визуализацией.

Общие правила настройки сопоставления данных

Разверните раздел **Data mapping** в настройках визуализации, чтобы увидеть, как данные результата сопоставлены с визуализацией, и при необходимости изменить эти настройки.

* Обязательные поля отмечены звёздочкой (`*`). Пример:

  ![Пример сопоставления данных: линейная диаграмма](https://dt-cdn.net/images/example-data-mapping-line-chart-1-732-eb9cca00d8.png)
* Типы данных отображаются рядом с именами полей в раскрывающихся списках и сопоставленных полях.
* Единицы измерения отображаются, когда назначена только одна.
* Поля результатов сгруппированы в **Suitable** и **Unsuitable**. Поля помечаются как непригодные, если их нельзя использовать для отображения данных в визуализации. Пример:

  ![Пример сопоставления данных: линейная диаграмма, раскрывающийся список Time](https://dt-cdn.net/images/example-data-mapping-line-chart-2-763-9cc593d261.png)
* Автоматическое применение настроек сопоставления данных по умолчанию:

  Dynatrace версии 1.319+

  + Существующие плитки и разделы считаются определёнными пользователем. Их конфигурации сопоставления данных не обновляются автоматически.
  + Вновь созданные плитки и разделы применяют настройку сопоставления данных по умолчанию. Если вы не изменяете эти настройки вручную, они могут измениться при повторном выполнении плитки/раздела, если результаты содержат отсутствующие поля или новые поля, лучше подходящие для сопоставления данных.

### Настройки сопоставления данных для конкретной визуализации

Для визуализации «Соты» раздел сопоставления данных включает:

* **Value**: числовое/строковое поле из запроса, которое вы хотите сопоставить с ячейкой.
* **Names**: элементы, отображаемые, например, в легенде и именах серий.

## Параметры визуализации «Соты»

Чтобы быстро изучить параметры и решить, что лучше подходит, включайте и выключайте параметры и сразу наблюдайте эффект на диаграмме. Например, лучше ли выглядит с подписью или без? Включите и выключите эту опцию и убедитесь сами.

### Форма ячейки

Вы можете установить форму ячейки: **Hexagon** (классическая форма соты), **Circle** или **Square**.

### Отображение подписей

Включите, чтобы отображать подписи в ячейках.

### Расположение легенды

Выберите, где отображать легенду визуализации:

* **Automatic** -- Dynatrace выбирает подходящее расположение
* **Hidden** -- легенда не отображается
* **Bottom** -- легенда отображается внизу визуализации
* **Right** -- легенда отображается справа от визуализации

**Automatic** отдаёт приоритет размещению легенды справа от области диаграммы. При уменьшении ширины диаграммы легенда перемещается в область под диаграммой.

### Цветовой режим, цветовая палитра и пользовательские цвета

В **Color mode** выберите один из следующих цветовых режимов.

#### Цветовая палитра

Когда **Color mode** установлен в **Color palette**, отображается список **Color palette**. Выберите цветовую палитру из списка.

#### Пользовательские цвета

Когда **Color mode** установлен в **Custom colors**, отображается раздел **Custom colors**. Используйте его для сопоставления цветов с диапазонами значений. Значения, равные или превышающие определённое значение, будут отображаться определённым цветом. Для каждого диапазона вы выбираете значение и цвет.

## Ограничения запроса

Используйте раздел **Query limits** для проверки и настройки ограничений запроса Grail для раздела блокнота или плитки дашборда. Эти настройки определяют максимальные ограничения при выборке данных. Превышение любого ограничения приведёт к генерации предупреждения.

Плитки дашборда и разделы блокнота, созданные в Dynatrace до версии 1.296, не затрагиваются. Эти существующие плитки/разделы будут возвращать те же результаты, что и раньше.

* **Read data limit (GB)**

  Ограничение в гигабайтах на объём данных, которые будут сканироваться при чтении.
* **Record limit**

  Максимальное количество записей результата, которое вернёт этот запрос. По умолчанию: 1 000 записей. Чтобы увидеть больше записей, нужно увеличить значение **Record limit**.

  + Если ваш запрос не содержит `limit`, например

    ```
    fetch logs
    ```

    применяется значение **Record limit**. По умолчанию вы увидите до 1 000 записей.
  + Если ваш запрос также включает `limit`, например

    ```
    fetch logs



    | limit 2000
    ```

    применяется меньшее из двух значений (либо `limit` в запросе, либо **Record limit** в веб-интерфейсе).

    В приведённом выше примере вы по-прежнему увидите только 1 000 записей, если не увеличите значение **Record limit**.
* **Result size limit**

  Максимальный объём в байтах результата, который вернёт этот запрос. Для лучшей производительности при типичных запросах и небольших документах значение по умолчанию установлено в 1 МБ.
* **Sampling (Logs and Spans only)**

  Выборка подмножества записей Log или Span.

## Единицы измерения и форматы

Чтобы переопределить единицы измерения и форматы по умолчанию в визуализации дашборда или блокнота:

1. Выберите для редактирования плитки визуализации.
2. Выберите вкладку **Visual**.
3. Выберите **Units and formats**.
4. Выберите **Override**.
5. Выберите **Override**.
6. В раскрывающемся списке выберите элемент, для которого вы хотите добавить переопределение единицы измерения.

   Это числовой столбец базового результата DQL, поэтому он зависит от запроса. Например:

   * Запрос `fetch events` возвращает события. Раскрывающийся список позволяет выбрать числовое поле (например, `transfer_size`) из результатов.
   * Запрос `timeseries avg(dt.host.cpu.usage)` возвращает один временной ряд для `avg(dt.host.cpu.usage)`. Этот временной ряд является единственным доступным вариантом в списке.
7. Определите переопределение.

   * **Default unit**: базовая единица измерения, в которой были захвачены значения. Значение `None`, если она не была включена в результат DQL, или автоматически определяется единицей, переданной из результата DQL. Это поле не приводит к конвертации.
   * **Displayed unit**: после определения единицы измерения по умолчанию можно использовать **Displayed unit** для конвертации. Например, если результат DQL определил числовое значение как `Bytes`, **Displayed unit** предлагает подходящий список конвертаций, таких как `Kilobyte` и `Megabyte`. В отличие от **Default unit**, **Displayed unit** всегда является числовой конвертацией.
   * **Decimals** отображает количество десятичных знаков по умолчанию (степень точности). Чтобы увидеть действие, измените выбор **Decimals** и наблюдайте за изменениями в визуализации.
   * **Suffix** отображает суффикс после единицы измерения. Чтобы увидеть действие, введите строку и наблюдайте за изменениями в визуализации. Если вы не нашли нужную единицу измерения, можно использовать **Suffix** для отображения нужной единицы.
8. Включите **Abbreviate large numbers**, если хотите отображать большие числа в сокращённой форме. Например, `1053` станет `1.1K`.

Чтобы сбросить настройки по умолчанию (отменить переопределение для выбранного элемента), нажмите значок корзины рядом с элементом.

Пример для дашбордов

В этом примере используется линейная диаграмма, но параметры применимы и к другим визуализациям.

1. В ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** создайте дашборд.
2. Выберите и в разделе **Snippets** меню выберите **Metrics** > **Chart average CPU across all hosts**.
3. На панели редактирования раздела выберите вкладку **Visual** и выберите **Line**.
4. Выберите **Units and formats**.

   ![Выбор Units and formats](https://dt-cdn.net/images/units-example-proc-01-1450-635695f6b8.png)
5. Выберите **Override**.

   ![Выбор Add Override.](https://dt-cdn.net/images/units-example-proc-02-1450-e1805870b1.png)
6. В раскрывающемся списке выберите метрику, для которой вы хотите добавить переопределение. В этом примере доступна только одна метрика.
7. Определите переопределение для отображаемой метрики. Вы можете наблюдать за изменениями на оси Y диаграммы.

   * **Default unit** отображает `Percent (%)`, что является единицей по умолчанию для выбранной метрики. Попробуйте другую настройку, например `One`, чтобы вместо этого отображать результат как долю от 1.
   * **Displayed unit** отображает `Auto`. Вы можете изменить его на другую единицу, например `One`, чтобы отображать результат как долю от 1.

     Поддерживаются только линейные и статические конвертации. Например, нельзя конвертировать `Degree Celsius` в `Degree Fahrenheit` или `Usd(US$)` в `Eur`.
   * **Decimals** отображает количество десятичных знаков по умолчанию (степень точности). Чтобы увидеть действие, измените выбор **Decimals** и наблюдайте за изменениями в визуализации.

     Например, измените это:

     ![Настройка Decimals до](https://dt-cdn.net/images/units-example-proc-03-1454-6a84518802.png)

     На это:

     ![Настройка Decimals после](https://dt-cdn.net/images/units-example-proc-04-1450-03cc893027.png)
   * **Suffix** отображает необязательный суффикс после единицы измерения. Чтобы увидеть действие, введите строку и наблюдайте за изменениями в визуализации.

Чтобы сбросить настройки по умолчанию (отменить переопределение для выбранной метрики), нажмите значок корзины рядом с метрикой.
