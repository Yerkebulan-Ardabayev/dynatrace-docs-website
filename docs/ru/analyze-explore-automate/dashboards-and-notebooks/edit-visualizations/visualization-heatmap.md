# Визуализация "Тепловая карта"

* Latest Dynatrace
* Практическое руководство
* Чтение: 2 мин
* Обновлено 17 июн. 2025

Когда использовать тепловую карту

Тепловая карта предоставляет компактную и гибкую матричную визуализацию для отображения агрегированных наборов данных.

* Когда вы хотите выявить закономерности и выбросы в метриках в больших агрегированных наборах данных
* Когда вы хотите сопоставить и сравнить метрики на основе времени, числовых и категориальных данных друг с другом

## Примеры

### Пример 1

![Пример тепловой карты 1: Тепловая карта (время отклика по сервисам)](https://dt-cdn.net/images/heatmap-example-01-resp-705-31ffafba81.png)

Приведённая выше тепловая карта основана на следующем запросе.

```
timeseries response_time = avg(dt.service.request.response_time), by: { dt.entity.service }



| fields response_time, entityName(dt.entity.service), interval, timeframe, dt.entity.service



| limit 10
```

### Пример 2

![Пример тепловой карты 2: Тепловая карта (оценка уязвимости по event.status для событий безопасности)](https://dt-cdn.net/images/heatmap-example-02-vuln-715-6e2f83a092.png)

Приведённая выше тепловая карта основана на следующем запросе.

Приведённый ниже запрос был обновлён в соответствии с новой таблицей событий безопасности Grail. Полный список обновлений и необходимых действий для миграции см. в [руководстве по миграции таблиц безопасности Grail](../../../secure/threat-observability/migration.md "Описание изменений в новой таблице безопасности Grail и инструкции по миграции.").

```
fetch security.events



| summarize count = count(), by:{range(vulnerability.risk.score, 0.5), event.status}
```

## Взаимодействие с диаграммой

### Взаимодействие при выборе элементов

Когда вы выбираете значение на диаграмме и закрепляете отображаемую подсказку, вы можете навести курсор на подсказку для отображения меню с параметрами, специфичными для выбранного элемента.

Доступные взаимодействия зависят от вашего запроса и визуализации. Например, если вы выбираете хост на линейной диаграмме и наводите курсор на подсказку, вы увидите меню с такими пунктами:

* **Copy name** -- скопировать имя выбранного хоста.
* **Fields** -- раздел с подменю для каждого поля запроса. Подменю поля предлагает специфичные параметры, такие как:

  + **Copy value** -- скопировать значение поля. Также отображается тип поля.
  + **Hide** -- скрыть поле на диаграмме.
  + **Explain value** -- использовать ИИ для пояснения поля.
  + **Add command to query** -- раздел команд, специфичных для поля, которые можно автоматически добавить к запросу.
  + Также может отображаться рекомендуемое приложение.
* **Visual options** -- открывает панель редактирования для изменения параметров визуализации выбранного элемента.
* **Set color** -- открывает панель редактирования для изменения цвета выбранного элемента.
* ![Infrastructure & Operations](https://dt-cdn.net/images/infrasctucture-operations-highresolution-1025-07d1bc45b5.png "Infrastructure & Operations") **Go to host** -- открывает выбранный элемент в ![Infrastructure & Operations](https://dt-cdn.net/images/infrasctucture-operations-highresolution-1025-07d1bc45b5.png "Infrastructure & Operations") **Infrastructure & Operations**.

  Как правило, если есть рекомендуемые приложения для открытия выбранного элемента, меню предлагает прямые ссылки на эти приложения, а затем опцию **Open with** для выбора другого целевого приложения.
* **Open with** -- подробнее см. [Детализация и навигация](../drilldowns-and-navigation.md "Детализация из Dashboards и Notebooks с помощью ссылок на основе интентов или URL.").

## Заголовок

Используйте поле заголовка в верхней части панели параметров (изначально `Untitled tile` или `Untitled section`), чтобы добавить заголовок для плитки дашборда или секции ноутбука.

* Можно использовать эмодзи.
* Можно использовать переменные.

**Пример:**

1. Определите переменные `Status` и `Emoji` в дашборде.
2. Задайте заголовок `Current $Emoji status is $Status`.
3. Установите `Status` в значение `Good`.
4. Установите `Emoji` в нужное значение.

Заголовок будет отображаться соответственно.

## Визуализация

Если вы не уверены, что выбрали правильную визуализацию, используйте [селектор визуализации](../edit-visualizations.md#select-visualization "Создание, редактирование и просмотр визуализаций в дашбордах и ноутбуках Dynatrace."), чтобы попробовать различные варианты.

Чтобы быстро изучить параметры и решить, что подходит лучше всего, включайте и выключайте опции и сразу наблюдайте эффект на диаграмме. Например, выглядит ли лучше с подписью или без? Включите и выключите эту опцию и убедитесь сами.

## Сопоставление данных

Раздел сопоставления данных показывает, как столбец вашего результата сопоставлен с визуализацией.

Подробнее об общих правилах настроек сопоставления данных

Разверните раздел **Data mapping** в настройках визуализации, чтобы увидеть, как данные результата сопоставлены с визуализацией, и при необходимости изменить эти настройки.

* Обязательные поля отмечены звёздочкой (`*`). Пример:

  ![Пример сопоставления данных: линейная диаграмма](https://dt-cdn.net/images/example-data-mapping-line-chart-1-732-eb9cca00d8.png)
* Типы данных отображаются рядом с именами полей в выпадающих списках и сопоставленных полях.
* Единицы измерения отображаются, когда назначена только одна.
* Поля результата группируются на **Suitable** и **Unsuitable**. Поля помечаются как непригодные, если их нельзя использовать для отображения данных в визуализации. Пример:

  ![Пример сопоставления данных: линейная диаграмма, выпадающий список Time](https://dt-cdn.net/images/example-data-mapping-line-chart-2-763-9cc593d261.png)
* Автоматическое применение настроек сопоставления данных по умолчанию:

  Dynatrace версии 1.319+

  + Уже существующие плитки и секции считаются определёнными пользователем. Их конфигурации сопоставления данных не обновляются автоматически.
  + Вновь созданные плитки и секции применяют настройку сопоставления данных по умолчанию. Если вы не изменяете эти настройки вручную, они могут измениться при новом выполнении плитки/секции, если результаты изменились и появились отсутствующие или более подходящие поля.

### Настройки сопоставления данных, специфичные для визуализации

Раздел сопоставления данных тепловой карты включает:

* **X-axis**: выберите значение для оси X тепловой карты.

  + **Numeric**: требует вложенные начальное и конечное значения, что лучше всего достигается с помощью DQL-функции `range` совместно с командой `summarize`.
  + **Time**: требует вложенные начальную и конечную временные метки, а также поле интервала. Проще всего достигается с помощью DQL-команды `timeseries` или `makeTimeseries`.
  + **String**: требует хотя бы одно строковое значение, которое можно использовать как категорию. Достигается с помощью DQL-команды `summarize` с указанием конкретного поля для группировки или с помощью команды `by` для указания полей, по которым разбиваются серии.
* **Interval**: отображается под **X-axis**, если для оси X выбран временной диапазон; это значение сопоставляется автоматически и не может быть изменено. Оно показывает, какие поля сопоставлены для результатов на основе временных рядов. Используется первое доступное поле интервала из набора результатов при использовании временных рядов (включая данные на основе makeTimeseries).
* **Y-axis**: выберите значение для оси Y тепловой карты.

  + **Numeric**: требует вложенные начальное и конечное значения, что лучше всего достигается с помощью DQL-функции `range` совместно с командой `summarize`.
  + **Time**: требует вложенные начальную и конечную временные метки, а также поле интервала. Проще всего достигается с помощью DQL-команды `timeseries` или `makeTimeseries`.
  + **String**: требует хотя бы одно строковое значение, которое можно использовать как категорию. Достигается с помощью DQL-команды `summarize` с указанием конкретного поля для группировки или с помощью команды `by` для указания полей, по которым разбиваются серии.
* **Interval**: отображается под **Y-axis**, если для оси Y выбран временной диапазон; это значение сопоставляется автоматически и не может быть изменено. Оно показывает, какие поля сопоставлены для результатов на основе временных рядов. Используется первое доступное поле интервала из набора результатов при использовании временных рядов (включая данные на основе makeTimeseries).
* **Value field**: выберите значение (числовое или строковое поле) для отображения на тепловой карте.

## Ось X

Эти настройки определяют внешний вид оси X тепловой карты.

* **Show label**: включите, чтобы задать подпись для оси X.
* **Reverse**: включите, чтобы изменить направление оси X.
* **Position**: выберите, где отображать ось X относительно тепловой карты.
* **Tick label layout**: выберите горизонтальное или вертикальное отображение значений оси X.

## Ось Y

Эти настройки определяют внешний вид оси Y тепловой карты.

* **Show label**: включите, чтобы задать подпись для оси Y.
* **Reverse**: включите, чтобы изменить направление оси Y.
* **Position**: выберите, где отображать ось Y относительно тепловой карты.

## Легенда и подсказка

* **Show legend**: чтобы отобразить легенду тепловой карты, включите **Show legend** и выберите **Position** легенды:

  + **Auto**: выбирает подходящее расположение на основе размера тепловой карты и доступного пространства.
  + **Bottom**: отображает легенду под тепловой картой.
  + **Right**: отображает легенду справа от тепловой карты.
* **Text truncation**: определяет способ обрезки текста, когда полный текст не помещается.

  + **A...**: обрезка с правого конца текста (когда правая часть менее важна)
  + **A...B**: обрезка из середины текста (когда середина менее важна)
  + **...B**: обрезка с левого конца текста (когда левая часть менее важна)

## Цвет

Эти настройки определяют использование цвета в тепловой карте.

* **Color palette**: отображает цвета из выбранной цветовой палитры.
* **Custom colors**: отображает цвета, определённые вами.

  Для каждого пользовательского цвета, который вы хотите добавить:

  1. Выберите **Color**.
  2. Введите значение, оператор и цвет для этого значения и оператора.

## Единицы измерения и форматы

Чтобы переопределить единицы измерения и форматы по умолчанию в визуализации дашборда или ноутбука:

1. Выберите значок редактирования для плитки визуализации.
2. Выберите вкладку **Visual**.
3. Выберите **Units and formats**.
4. Выберите **Override**.
5. Выберите **Override**.
6. В выпадающем списке выберите элемент, для которого вы хотите добавить переопределение единиц.

   Это числовой столбец базового результата DQL, поэтому он зависит от запроса. Например:

   * Запрос `fetch events` возвращает события. Выпадающий список позволяет выбрать числовое поле (например, `transfer_size`) из результатов.
   * Запрос `timeseries avg(dt.host.cpu.usage)` возвращает один временной ряд для `avg(dt.host.cpu.usage)`. Этот временной ряд является единственным доступным вариантом в списке.
7. Определите переопределение.

   * **Default unit**: базовая единица, в которой были захвачены значения. Значение `None`, если она не включена в результат DQL, или определяется автоматически по единице из результата DQL. Это поле не приводит к конвертации.
   * **Displayed unit**: после определения единицы по умолчанию вы можете использовать **Displayed unit** для конвертации. Например, если результат DQL определил числовое значение как `Bytes`, **Displayed unit** предложит подходящий список конвертаций байтов, таких как `Kilobyte` и `Megabyte`. В отличие от **Default unit**, **Displayed unit** всегда является числовой конвертацией.
   * **Decimals** отображает количество десятичных знаков (степень точности) по умолчанию. Чтобы увидеть в действии, измените выбор **Decimals** и наблюдайте изменения в визуализации.
   * **Suffix** отображает суффикс после единицы измерения. Чтобы увидеть в действии, введите строку и наблюдайте изменения в визуализации. Если вы не нашли нужную единицу, используйте **Suffix** для отображения желаемой единицы.
8. Включите **Abbreviate large numbers**, если хотите отображать большие числа в сокращённой форме. Например, `1053` станет `1.1K`.

Чтобы сбросить настройки до значений по умолчанию (отменить переопределения для выбранного элемента), нажмите значок корзины рядом с элементом.

Пример для дашбордов

В этом примере используется линейная диаграмма, но параметры применимы к другим визуализациям.

1. В ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** создайте дашборд.
2. Выберите значок добавления и в разделе **Snippets** меню выберите **Metrics** > **Chart average CPU across all hosts**.
3. В панели редактирования секции выберите вкладку **Visual** и выберите **Line**.
4. Выберите **Units and formats**.

   ![Выберите Units and formats](https://dt-cdn.net/images/units-example-proc-01-1450-635695f6b8.png)
5. Выберите **Override**.

   ![Выберите Add Override.](https://dt-cdn.net/images/units-example-proc-02-1450-e1805870b1.png)
6. В выпадающем списке выберите метрику, для которой вы хотите добавить переопределение. В этом примере доступна только одна метрика.
7. Определите переопределение для отображаемой метрики. Вы можете наблюдать изменения на оси Y диаграммы.

   * **Default unit** отображает `Percent (%)`, что является единицей по умолчанию для выбранной метрики. Попробуйте другую настройку, например `One`, чтобы отображать результат как долю от 1.
   * **Displayed unit** отображает `Auto`. Вы можете изменить на другую единицу, например `One`, чтобы отображать результат как долю от 1.

     Поддерживаются только линейные и статические конвертации. Например, нельзя конвертировать `Degree Celsius` в `Degree Fahrenheit` или `Usd(US$)` в `Eur`.
   * **Decimals** отображает количество десятичных знаков (степень точности) по умолчанию. Чтобы увидеть в действии, измените выбор **Decimals** и наблюдайте изменения в визуализации.

     Например, изменить это:

     ![Настройка Decimals до](https://dt-cdn.net/images/units-example-proc-03-1454-6a84518802.png)

     На это:

     ![Настройка Decimals после](https://dt-cdn.net/images/units-example-proc-04-1450-03cc893027.png)
   * **Suffix** отображает необязательный суффикс после единицы измерения. Чтобы увидеть в действии, введите строку и наблюдайте изменения в визуализации.

Чтобы сбросить настройки до значений по умолчанию (отменить переопределения для выбранной метрики), нажмите значок корзины рядом с метрикой.
