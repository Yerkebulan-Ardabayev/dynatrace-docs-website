## Плитки дашборда

В ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** дашборд может состоять из нескольких плиток:

* [Explore](#explore): исследуйте данные, такие как логи, метрики и бизнес-события, с помощью нашего интерфейса «укажи и щёлкни».
* [Запрос](#query): отображает данные, полученные через Grail.
* [Код](#code): отображает данные, возвращённые кодом, выполненным через функции Dynatrace.
* [Markdown](#markdown): статический контент, редактируемый в формате Markdown.

### Плитка Explore

Вы можете использовать возможности Explore для исследования логов, метрик, бизнес-событий и многого другого с помощью нашего интерфейса «укажи и щёлкни». Без знания DQL или программирования вы можете создавать и начинать использовать плитки дашборда за считанные минуты.

Для получения дополнительной информации у нас есть отдельная страница Исследование данных, которая показывает, как создавать плитки Explore.

### Плитка запроса

Плитки запросов позволяют легко запрашивать данные из Grail и визуализировать результаты различными способами.

Плитка запроса состоит из поля ввода запроса, в котором вы можете написать DQL-запрос. В поле ввода запроса используйте **Ctrl**+**Space** для вызова автодополнения в любой момент.

Вы можете управлять временным интервалом запроса через выпадающий список временных интервалов. Если временной интервал определён в самом запросе, выпадающий список отключается.

### Плитка кода

Раздел кода позволяет получать данные для вашего дашборда с помощью функций Dynatrace.

### Плитка Markdown

Раздел Markdown может быть чем угодно — от небольшой заметки о чём-то на дашборде до целой страницы форматированной информации со ссылками и изображениями.

* Легко редактировать
* Привлекательный внешний вид

Чтобы вставить запросы с вашего дашборда с автодополнением, используйте **Ctrl**+**Space**.

Вы можете добавлять ссылки на другие места в вашем дашборде и за его пределами.

Синтаксис ссылок Markdown

Ссылка в Markdown — это метка и адрес в формате `label`, где:

* `label` — произвольный текст, отображаемый на ссылке в вашей плитке или разделе Markdown
* `address` указывает, что открывать при переходе по ссылке, например, веб-сайт или приложение Dynatrace

## Сценарии использования

Dashboards позволяет вам:

* Использовать готовые дашборды для мониторинга состояния вашей системы в реальном времени.
* Создавать пользовательские дашборды без усилий с помощью удобного редактора или Davis CoPilot.
* Детально анализировать данные благодаря бесшовной интеграции с другими приложениями Dynatrace.
* Использовать возможности Davis AI для обнаружения аномалий и прогнозирования непосредственно на ваших графиках.
* Искать любые типы данных по всей платформе и комбинировать их даже с внешними данными в одном представлении.
* Настраивать фильтры переменных для мониторинга различных ресурсов в рамках одного дашборда.

Эти процедуры описывают основы использования ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** и помогут вам начать настройку и создание собственных дашбордов.

### Список дашбордов

#### Список всех дашбордов

Чтобы вывести список дашбордов

1. Перейдите в ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards**.
2. На панели **Dashboards** в разделе **Last opened by you** перечислены недавно открытые дашборды.
3. Наведите курсор на название дашборда и выберите  для отображения меню доступных команд для этого дашборда. В этом примере мы отобразим меню для дашборда с названием `my dashboard 3`.

   ![Dashboards: Last opened by you: контекстное меню дашборда](https://dt-cdn.net/images/dashboards-last-opened-by-you-dashboard-specific-menu-269-08e78a134c.png)

   Команды, которые вы видите в меню, зависят от ваших разрешений для данного дашборда. Например, вы не можете переименовать чужой дашборд, если владелец не предоставил вам права на редактирование. (Но вы *можете* сделать копию общего дашборда и редактировать свою копию.)

   * **Rename** позволяет редактировать название дашборда
   * **Duplicate** создаёт копию дашборда
   * **Download** сохраняет дашборд в JSON-файл, который можно импортировать
   * **Change owner** передаёт владение дашбордом новому владельцу
   * **Move to trash** перемещает дашборд в корзину

   Значок  после названия дашборда означает, что кто-то поделился этим дашбордом с вами.
4. Чтобы отобразить таблицу всех дашбордов, к которым у вас есть доступ — ваших собственных дашбордов и всех дашбордов, которыми с вами поделились — выберите  **All dashboards**.

   * Чтобы отсортировать таблицу, выберите заголовок **Name**, **Created** или **Last modified**.
   * Чтобы отфильтровать таблицу:

     + Вы можете ввести строку поиска в строке фильтра  в верхней части таблицы.
     + Вы можете выбрать **Owned by anyone**, **Owned by me** или **Shared with me** в списке в верхней части таблицы.
   * Чтобы создать новый дашборд, выберите  **Dashboard** в верхнем левом углу.
   * Чтобы загрузить дашборд, выберите  **Upload** в верхнем левом углу.
   * Чтобы удалить дашборд, выберите  **Move to trash**. Если значка  нет для дашборда, это означает, что у вас есть разрешение на просмотр этого общего дашборда, но не на его удаление.
   * Чтобы вывести список удалённых дашбордов, выберите вкладку **Deleted** на странице **Dashboards**.

     + Чтобы восстановить удалённый дашборд, выберите  **Restore**.
     + Чтобы окончательно удалить дашборд, выберите  **Delete permanently**.

### Обновление дашборда

При первом открытии дашборда частота обновления установлена в `Off` (без автоматического обновления).

#### Обновление вручную

Чтобы обновить текущий дашборд вручную, в правом верхнем углу дашборда выберите  (в паре  ).

#### Автоматическое обновление

Чтобы обновлять текущий дашборд автоматически, в правом верхнем углу дашборда выберите  (в паре  ) и выберите частоту обновления.

* `Off` отключает автоматическое обновление
* Другие параметры обновляют дашборд с указанным интервалом

Если вы измените частоту обновления, она будет сохранена при следующем открытии дашборда.

Частое обновление может поддерживать данные в актуальном состоянии, но для сложного дашборда пересчёт при каждом обновлении может занять некоторое время. Выберите частоту обновления, которая соответствует вашим потребностям и сложности дашборда.

### Редактирование дашбордов «только для чтения»

Когда вы открываете документ (дашборд или блокнот), для которого у вас нет прав на запись, вы всё равно можете редактировать документ в рамках текущего сеанса. После завершения у вас есть два варианта:

* Сохранить изменения в новом документе
* Отменить изменения

Пример:

1. Перейдите в ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards**, откройте список готовых дашбордов и выберите дашборд **Getting started**.

   В верхнем левом углу рядом с названием документа отображается надпись  **Ready-made**.
2. Выберите плитку с круговой диаграммой, затем выберите  **Edit**.
3. Измените визуализацию с Pie на Donut.

   Теперь вам предлагаются две кнопки: **Save as new** и **Discard changes**.
4. Используйте обновлённый дашборд по мере необходимости. У вас есть полный доступ к редактированию в рамках этого сеанса.
5. Когда закончите, выберите, что делать с вашими изменениями:

   * **Save as new** — сохраняет ваши изменения в новой копии отредактированного дашборда.
   * **Discard changes** — отменяет ваши изменения и возвращает вас к нередактированному дашборду «только для чтения».

### Выбор временного интервала

Чтобы просмотреть или изменить настройки, применяемые ко всему дашборду

1. Отобразите дашборд.
2. В правом верхнем углу дашборда выберите  **Settings**.

   Отобразится панель **Settings**.
3. Просмотрите или измените настройки по мере необходимости.

   Пользовательский временной интервал

   Чтобы установить временной интервал по умолчанию для дашборда

   1. Отобразите дашборд.
   2. В правом верхнем углу дашборда выберите  **Settings**.
   3. Включите **Custom timeframe**.
   4. Выберите временной интервал по умолчанию для выбранного дашборда.

      Изменение этого параметра не приводит к немедленному обновлению временного интервала текущего дашборда. Изменение применяется только к новому сеансу работы с дашбордом (для другого пользователя или для того же пользователя при повторном открытии дашборда).

   Подробнее о временных интервалах дашборда см. в разделе [Выбор временного интервала](#dashboard-timeframe).

   Сегмент по умолчанию

   Чтобы просмотреть или изменить сегмент дашборда по умолчанию

   1. Отобразите дашборд.
   2. В правом верхнем углу дашборда выберите  **Settings**.
   3. Включите **Default segment**.
   4. Выберите сегмент по умолчанию для выбранного дашборда.

      Изменение этого параметра не приводит к немедленному обновлению сегмента текущего дашборда. Изменение применяется только к новому сеансу работы с дашбордом (для другого пользователя или для того же пользователя при повторном открытии дашборда).

   Подробнее о сегментах дашборда см. в разделе [Выбор сегментов](#segment).

   {$} Переменные

   Чтобы просмотреть или изменить переменные дашборда

   1. Отобразите дашборд.
   2. В правом верхнем углу дашборда выберите  **Settings**.

      Отобразится панель **Settings**.
   3. Выберите **Variables**.

   Подробнее о переменных дашборда см. в разделе Добавление переменной в дашборд.

Временной интервал описывает *когда* были получены данные, отображаемые на дашборде.

* При первом открытии дашборда применяется стандартный глобальный временной интервал (**Last 2 hours**).
* Если вы измените глобальный временной интервал для вашего дашборда, этот интервал будет применён при следующем открытии дашборда. Однако существуют исключения, описанные ниже.
* Вы можете [установить временной интервал по умолчанию для конкретного дашборда](#timeframe-dashboard), который будет применяться при каждом последующем открытии дашборда.
* Вы можете [установить временной интервал для конкретной плитки](#timeframe-tile). Этот параметр переопределяет глобальный временной интервал.
* Когда вы открываете дашборд по ссылке, если в ссылке указан временной интервал, применяется интервал из ссылки, а временной интервал дашборда по умолчанию игнорируется.

#### Изменение глобального временного интервала

Чтобы изменить глобальный временной интервал

1. В правом верхнем углу дашборда откройте селектор временного интервала (по умолчанию **Last 2 hours** ).
2. Выберите новый временной интервал.

   * Чтобы выбрать стандартный временной интервал, выберите один из стандартных **Relative timeframes**
   * Чтобы определить пользовательский временной интервал, задайте интервал на панели **Custom timeframe** и выберите **Apply**.

     + Используйте кнопки календаря ![Calendar](https://dt-cdn.net/images/dashboards-app-dashboard-timeframe-calendar-b05d935595.svg "Calendar") для выбора дат
     + Измените настройки **From** и **To** для точной настройки диапазона

При изменении глобального временного интервала:

* Новый временной интервал отображается в правом верхнем углу дашборда.
* Содержимое дашборда пересчитывается и отображается в соответствии с новым временным интервалом. Это переопределяет временной интервал дашборда по умолчанию.
* Это не влияет на переопределения временного интервала для отдельных плиток. Временные интервалы плиток имеют приоритет над любыми глобальными настройками.

#### Временной интервал дашборда по умолчанию

Чтобы установить временной интервал по умолчанию для дашборда

1. Отобразите дашборд.
2. В правом верхнем углу дашборда выберите  **Settings**.
3. Включите **Custom timeframe**.
4. Выберите временной интервал по умолчанию для выбранного дашборда.

   Изменение этого параметра не приводит к немедленному обновлению временного интервала текущего дашборда. Изменение применяется только к новому сеансу работы с дашбордом (для другого пользователя или для того же пользователя при повторном открытии дашборда).

#### Переопределение временного интервала для конкретной плитки

Чтобы указать пользовательский временной интервал в плитке дашборда

1. Отредактируйте плитку.
2. В панели редактирования включите **Custom timeframe**.
3. Выберите временной интервал для применения к выбранной плитке.

   Этот временной интервал переопределяет временной интервал дашборда, установленный в правом верхнем углу. С помощью этого метода дашборд может иметь несколько плиток, каждая со своим собственным временным интервалом.

Вы также можете указать пользовательский временной интервал в DQL-запросе плитки данных. Если вы используете этот метод (с временным интервалом, указанным в запросе), указанная выше настройка интерфейса отключается и используется временной интервал из запроса.

Пример указания временного интервала в DQL:

```
fetch [recordtype], from:now() - 2h


| ....
```

Подробнее об указании временного интервала в DQL см. раздел [Указание временного интервала](../../platform/grail/dynatrace-query-language/dql-guide.md#specifytimeframe "Find out how DQL works and what are DQL key concepts.") в документации DQL.

### Выбор сегментов

Для фильтрации данных вы можете указать сегменты на двух уровнях: дашборда и плитки. Выбор сегментов на уровне плитки переопределяет выбор сегментов на уровне дашборда.

Следует использовать сегменты или переменные?

#### Сегменты

Используйте сегменты, когда хотите повторно использовать их в разных дашбордах. Например, используйте сегменты для регулярных фильтров, таких как кластеры Kubernetes, пространства имён, рабочие нагрузки или поды. Сегменты автоматически применяются поверх запросов ваших плиток/разделов, поэтому вам не нужно ссылаться на них в запросах.

#### Переменные

Если вам нужен больший контроль над применением фильтра, вы можете использовать переменные.

* Переменные позволяют полностью контролировать базовый запрос или раздел Explore, определяя, где и как они применяются. Например, вы можете указать, как они связаны с другими применёнными фильтрами (**AND**, **OR**), и можете контролировать, какой оператор используется для вашего фильтра (например, `equals`, `contains`, `startsWith` и `endsWith`).
* Кроме того, используйте переменные, когда вам нужен детальный контроль над взаимозависимостью фильтров.

* Подробнее о сегментах см. Сегменты
* Для сценария использования сегментов в ![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards** см. Анализ данных мониторинга с помощью сегментов

### Запуск плитки с кодом

При открытии документа другого пользователя вы можете увидеть следующее сообщение:

`This dashboard contains custom code. It is read-only until you review the code and select "Accept and run".`

Когда вы запускаете плитку или раздел с кодом, написанным другим человеком, Dynatrace выполняет JavaScript этого человека, используя вашу учётную запись и ваши разрешения. Это мощная функция, но её необходимо использовать правильно и ответственно. JavaScript-код может получать доступ к внешним API от вашего имени (используя вашу учётную запись и разрешения).

Чтобы просмотреть код

1. Выберите **Review all code**.

   На странице **Review code** код каждой плитки отображается в отдельном блоке.
2. Просмотрите код и решите, хотите ли вы его запустить.

   Если вы хотите запустить код, вы можете одобрить его только на этот раз или на постоянной основе.

   * Чтобы запустить код только в этот раз, выберите **Accept and run**. При следующем открытии этого документа вам снова будет предложено просмотреть код перед запуском.
   * Чтобы навсегда одобрить код в этом документе, выберите **Always trust code in this document**, а затем выберите **Accept and run**.

### Скачивание результата

Чтобы скачать (экспортировать) результат текущей плитки дашборда или раздела блокнота в локальный файл

1. Выберите плитку или раздел.
2. На панели команд откройте меню  и выберите  **Download result** > [формат].

   Доступные форматы зависят от визуализации.

   * **CSV**: результат скачивается в локальный файл со значениями, разделёнными запятыми (\*.csv). Этот вариант включает форматирование таблицы, такое как видимые столбцы и форматирование единиц.
   * **CSV (raw)**: результат скачивается в локальный файл со значениями, разделёнными запятыми (\*.csv). Этот вариант включает полные неформатированные данные.
   * **JSON**: результат скачивается в локальный JSON-файл (\*.json).

   Некоторые визуализации не предоставляют возможности скачивания результата.

#### Возможная проблема при импорте CSV в Excel

При экспорте результатов в CSV-файл — будь то **CSV** или **CSV (raw)** — с переносами строк в любом столбце/поле результата и последующем открытии файла в Microsoft Excel или попытке преобразования файла в таблицу с помощью функции Excel **Data** > **Text to Columns** вы можете столкнуться с проблемой, когда переносы строк некорректно отображаются в Excel. Это вызвано ограничением Excel.

Чтобы избежать этой проблемы, используйте функцию Excel **Data** > **From Text/CSV**.

* **Access for all (view-only)**: предоставление доступа «только для просмотра» всем пользователям в вашем окружении Dynatrace.
* **Share access**: создание и поддержание списка пользователей и групп пользователей с доступом к документу.

Например, вы можете вести целевой список пользователей для постоянного доступа к документу (возможно, все в определённой группе регулярно редактируют документ) и создавать и распространять ссылки «только для просмотра» для более широкой аудитории по мере необходимости.

В любом случае вы контролируете, могут ли люди редактировать документ или только просматривать его.

### Создание дашборда

Самый быстрый и простой способ исследовать ваши данные — это наши новые плитки и разделы Explore. За несколько секунд вы можете найти и проанализировать ваши логи, метрики или бизнес-события. DQL не требуется!

### Запрос данных из Grail

Чтобы выполнить запрос к Grail

1. В правом верхнем углу дашборда выберите  **Add** >  **DQL**.

   Горячая клавиша: **Shift**+**D**

   * **Data**
   * **Visual**
2. На вкладке **Data** используйте Dynatrace Query Language (DQL) для определения вашего запроса.
3. Выберите **Run** для выполнения запроса.
4. На вкладке **Visual** выберите формат визуализации для ваших результатов.

### Добавление кода

Чтобы добавить код в дашборд

1. В правом верхнем углу дашборда выберите  **Add** >  **Code**.

   Горячая клавиша: **Shift**+**C**

   ![Dashboards: кнопка добавления плитки (плюс)](https://dt-cdn.net/images/updated-dashboards-add-tile-button-481-c21ba8f200.png)

   Чтобы ваши запросы не блокировались, попросите администратора разрешить ваши внешние источники данных, добавив их в **External requests**.

### Добавление переменной

Используйте переменные для фильтрации дашбордов, в качестве значений переменных в плитках с кодом и в качестве заполнителей в заголовках плиток и тексте плиток Markdown.

Чтобы добавить переменную в дашборд

1. В заголовке дашборда откройте меню  и выберите  **Variables**.
   Горячая клавиша: **Shift**+**V**

   Горячая клавиша: **Shift**+**V**

   ![Dashboards: кнопка добавления плитки (плюс)](https://dt-cdn.net/images/updated-dashboards-add-tile-button-481-c21ba8f200.png)

   Отобразится панель **Variable**.
2. Определите переменную.

   * **Name**: имя переменной.

     + Допускаются только буквенно-цифровые значения
     + Не начинайте имя переменной с `dt_`
   * **Type**: может быть одним из следующих:

     + **DQL**: значение возвращается из запроса, который вы вводите при определении переменной.

       - Выберите **Run** для тестирования запроса.
     + **Code**: значение возвращается из кода, который вы вводите при определении переменной.
     + **List**: список значений, разделённых запятыми (CSV).

       - Чтобы определить возможные значения, введите их (через запятую) в поле под **Data**.
       - Чтобы определить значение по умолчанию, выберите его из списка **Default value**.
       - Чтобы разрешить выбор нескольких значений одновременно, включите **Multi-select**.
     + **Free Text**: свободный текст. Вы можете ввести **Default value**.
       Ваши изменения сохраняются автоматически.
3. Установите **Display settings**.

   * Если выпадающий список переменной должен отображаться на дашборде, включите **Display as filter on dashboard**.
     Отключите его, если хотите скрыть переменную, например, когда она используется как статическое значение в плитках, но не должна отображаться пользователю дашборда.
   * Если пользователи должны иметь возможность выбирать несколько значений одновременно в выпадающем списке переменной, включите **Multi-select**.
     Отключите его, если хотите использовать только одиночные значения выпадающего списка переменной.
   * Выберите **Default value** для выпадающего списка переменной. Если вы ничего не вводите в это поле, выбирается первое доступное значение.
4. Когда закончите, выберите **< Variable** вверху, чтобы перейти к панели **Variables**, или выберите  для закрытия панели **Variables**.

Переменные в дашбордах могут быть определены с зависимостью от других переменных.

* Значение переменной пересчитывается, если её определение ссылается на другую переменную и значение другой переменной изменяется.

  Например, если значение переменной A изменяется, значение любой переменной, определение которой ссылается на переменную A, пересчитывается.
* Циклические зависимости не допускаются.

  Например, если значение переменной A зависит от значения переменной B, значение переменной B не может зависеть от значения переменной A.

Подробнее см. Добавление переменной в дашборд.

   * Подробнее о настройках обнаружения аномалий см. [Конфигурация обнаружения аномалий](../../dynatrace-intelligence/anomaly-detection/anomaly-detection-configuration.md#analyzer "How to set up an alert for missing measurements.")

   Обнаружение аномалий: автоадаптивное пороговое обнаружение аномалий

   * **Number of signal fluctuations** — сколько раз флуктуация сигнала добавляется к базовой линии для формирования фактического порога оповещения.
   * **Alert condition** — ваш выбор зависит от того, хотите ли вы получать уведомления, когда метрика выше, ниже или за пределами (выше или ниже) нормального диапазона.

   Подробнее см. [Конфигурация обнаружения аномалий](../../dynatrace-intelligence/anomaly-detection/anomaly-detection-configuration.md#analyzer "How to set up an alert for missing measurements.").

   Обнаружение аномалий: сезонное базовое обнаружение аномалий

   * **Tolerance** — чем выше допуск, тем шире полоса доверия, что приводит к меньшему количеству срабатываний.
   * **Alert condition** — ваш выбор зависит от того, хотите ли вы получать уведомления, когда метрика выше, ниже или за пределами (выше или ниже) нормального диапазона.

   Подробнее см. [Конфигурация обнаружения аномалий](../../dynatrace-intelligence/anomaly-detection/anomaly-detection-configuration.md#analyzer "How to set up an alert for missing measurements.").

   Обнаружение аномалий: обнаружение аномалий по статическому порогу

   * **Threshold** — жёсткий предел, который метрика не должна нарушать.
   * **Unit** — единица измерения значения.
   * **Alert condition** — ваш выбор зависит от того, хотите ли вы получать уведомления, когда метрика выше или ниже порогового значения.
   * **Suggest threshold** — Davis AI может помочь вам найти подходящий порог на основе исторических данных.

   Подробнее см. [Конфигурация обнаружения аномалий](../../dynatrace-intelligence/anomaly-detection/anomaly-detection-configuration.md#analyzer "How to set up an alert for missing measurements.").

   Прогнозирование: Прогноз

   * **Data points to predict** — общее количество шагов, на которое прогнозируется временной ряд. Большее количество шагов обычно приводит к менее надёжным прогнозам и более длительному времени работы анализатора.
   * **Forecast offset** — смещение начала прогноза. Например, если смещение установлено в `2`, последние две точки данных игнорируются, и для них также возвращается прогноз.

   Подробнее см. Прогнозный ИИ-анализ Dynatrace Intelligence.

   Расширенные настройки (для обнаружения аномалий)

   Вы можете использовать значения по умолчанию или включить **Show advanced properties** для тонкой настройки этих параметров.

   * **Alert on missing data** — оповещение при отсутствии данных в скользящем окне.
   * **Violating samples** — количество образцов в скользящем окне, которые должны нарушить порог для срабатывания события.
   * **Sliding window** — количество образцов, формирующих скользящее окно.
   * **Dealerting samples** — количество образцов в скользящем окне, которые должны вернуться к норме для закрытия события.
5. По умолчанию анализатор не включён. Чтобы включить его, включите переключатель в верхней части панели редактирования (переключите с **AI data analysis is not active** на **AI data analysis is active**).
6. Чтобы просмотреть результаты, выберите визуализацию **Davis AI analysis** и разверните раздел **Davis AI analysis chart** для просмотра или изменения настроек визуализации:

### Изменение значений переменных

Если дашборд имеет одну или несколько переменных, они перечислены по имени в верхнем левом углу дашборда, под названием дашборда. При изменении значений переменных содержимое дашборда пересчитывается и отображается в соответствии с новыми значениями.

Чтобы изменить значение переменной

1. В верхнем левом углу дашборда найдите имя переменной.
2. Используйте меню или поле ввода под именем переменной для изменения значения.

   * Если переменная допускает только один выбор (значение) одновременно, выберите значение, которое хотите применить к дашборду.
   * Если переменная допускает множественный выбор (значений) одновременно, установите флажок для каждого значения, которое хотите применить к дашборду. В названии меню этой переменной отображается количество выбранных значений.
   * Для переменной Free Text вы можете отредактировать текст в поле под именем переменной.

### Копирование в другой документ

Вы можете скопировать плитку дашборда в другой документ (например, в другой дашборд или блокнот).

Например, простой способ начать новый блокнот — скопировать многоразовые плитки из существующих дашбордов в новый блокнот и затем отредактировать скопированные разделы как часть нового блокнота.

Чтобы скопировать плитку дашборда в блокнот (как раздел блокнота)

1. На дашборде выберите плитку, которую хотите скопировать в блокнот.
2. Откройте меню  и выберите ![Notebooks](https://dt-cdn.net/images/notebooks-768-046137830a.webp "Notebooks") **Add to notebook**.
3. В окне **Select destination**:

   * Если вы хотите создать новый блокнот с выбранной плиткой дашборда в качестве раздела, выберите **New notebook**.
   * Если вы хотите добавить выбранную плитку дашборда как ещё один раздел в существующий блокнот, выберите существующий блокнот из списка, а затем выберите **Confirm**.

   Блокнот откроется с выбранной плиткой, скопированной в него в качестве раздела.

Чтобы скопировать плитку дашборда в другое приложение (не ![Notebooks](https://dt-cdn.net/images/notebooks-768-046137830a.webp "Notebooks") **Notebooks**), откройте меню  и выберите  **Open with**, затем выберите целевое приложение.

Подробнее см. Детализация и навигация.

### Добавление или редактирование заголовка плитки

Вы можете добавить заголовок к любой плитке дашборда (кроме плитки Markdown).

Независимо от настройки заголовка, панель заголовка автоматически создаётся при необходимости для отображения индикаторов плитки, таких как предупреждение или индикатор пользовательского временного интервала.

1. Выберите плитку и отредактируйте  её.

   * Заголовок отображается в поле в верхней части панели редактирования плитки.
   * Если у плитки нет заголовка, в поле отображается `Untitled tile`.
2. Отредактируйте заголовок. Ваши изменения отображаются по мере ввода.

   Заголовок может включать эмодзи и переменные.

   **Пример:**

   1. В вашем дашборде определите переменные с именами `Status` и `Emoji`.
   2. Установите заголовок `Current $Emoji status is $Status`.
   3. Установите переменную `Status` в значение `Good`.
   4. Установите переменную `Emoji` в значение эмодзи.

   Заголовок будет отображаться как `Current [эмодзи] status is Good` и будет автоматически обновляться при изменении значений переменных.

Чтобы удалить заголовок плитки, выберите плитку, отредактируйте  её и очистите поле заголовка.

## Макет

### Изменение размера плитки

1. Наведите курсор на плитку.
2. Выберите и перетащите  в правом нижнем углу плитки для изменения размеров плитки.

   Для помощи в выборе конкретного размера размер плитки отображается на панели команд плитки.

### Перемещение плитки

Вы можете перемещать одну плитку за раз или выбрать и переместить несколько плиток одновременно.

Чтобы переместить плитку, перетащите её на новую позицию.

* Если у плитки есть заголовок, выберите и перетащите  панель заголовка для перемещения плитки.
* Если у плитки нет заголовка, выберите плитку для отображения команд плитки, затем выберите и перетащите  на панели команд плитки для перетаскивания выбранной плитки на новое место.

  Для помощи в выравнивании плиток при перемещении отображается фоновая сетка.

Чтобы переместить несколько плиток одновременно

1. Выберите одну плитку.
2. Ctrl-щелчком выберите дополнительные плитки, которые хотите переместить.

   * Каждая выбранная плитка подсвечивается.
   * Общее количество выбранных плиток отображается на панели команд плитки.
3. Выберите и перетащите  **Move** для перетаскивания всех выбранных плиток на новое место.

### Копирование и вставка плитки

Вы можете копировать и вставлять плитки на тот же дашборд или другой дашборд.

Чтобы скопировать и вставить одну плитку

1. Выберите плитку, чтобы отобразить команды плитки.
2. На панели команд плитки выберите  **Copy to clipboard**.

   Выбранная плитка скопирована в буфер обмена.
3. Вставьте плитку (Ctrl-V).

   * Вы можете вставить скопированную плитку на текущий дашборд — в этом случае это эквивалент  **Duplicate**.
   * Вы можете переключиться на другой дашборд и вставить скопированную плитку там.

Чтобы скопировать и вставить несколько плиток одновременно

1. Щёлкните на одной плитке, чтобы выбрать её.
2. Ctrl-щелчком выберите дополнительные плитки, которые хотите скопировать и вставить.

   * Каждая выбранная плитка подсвечивается.
   * Общее количество выбранных плиток отображается на панели команд плитки.
3. На панели команд плитки выберите  **Copy to clipboard**.

   Выбранные плитки скопированы в буфер обмена.
4. Вставьте плитки (Ctrl-V) на тот же дашборд или переключитесь на другой дашборд и вставьте их там.

### Дублирование плитки

Чтобы дублировать одну плитку

2. Выберите  **Duplicate**.

   Копия плитки создаётся на текущем дашборде.

Чтобы дублировать несколько плиток одновременно

1. Щёлкните на одной плитке, чтобы выбрать её.
2. Ctrl-щелчком выберите дополнительные плитки, которые хотите дублировать.

   * Каждая выбранная плитка подсвечивается.
   * Общее количество выбранных плиток отображается на панели команд плитки.
3. На панели команд плитки выберите  **Duplicate**.

   Выбранные плитки дублируются на текущем дашборде.

### Разделение пространства плитки для вставки новой плитки

При настройке макета дашборда вы можете разделить пространство, которое плитка занимает на сетке, и вставить вторую плитку рядом с первой в том же пространстве.

* Первая плитка не меняется, за исключением того, что теперь занимает половину ширины или высоты, которую использовала до разделения.
* Вторая (новая) плитка использует другую половину этого пространства.
* Эта функция доступна только при выборе одной плитки, достаточно большой для разделения. Она недоступна при выборе нескольких плиток или если размер выбранной плитки слишком мал для разделения.

Например, давайте выберем плитку и добавим другую плитку справа от неё, чтобы две плитки делили пространство, изначально занимаемое первой плиткой.

1. Выберите плитку.
2. Наведите курсор на плитку, чтобы увидеть элементы управления разделением  — слева, справа и снизу — указывающие, где вы разделяете пространство и вставляете новую плитку.

   ![Разделение плитки дашборда для вставки другой плитки в то же пространство](https://dt-cdn.net/images/split-tiles-01-1515-c796df2c96.png)
3. В этом примере выберите элемент управления  справа. Откроется меню, аналогичное меню , которое вы обычно используете для добавления плитки на дашборд.
4. Выберите тип плитки, которую хотите добавить.
5. Отредактируйте определение новой плитки и закройте панель редактирования, чтобы увидеть изменённый макет.

   * Оригинальная плитка не изменена, за исключением того, что теперь занимает только левую половину ранее используемого пространства.
   * Новая плитка использует правую половину этого пространства.
   * Вы всё ещё можете вручную настроить размеры и положение плиток.

### Удаление плитки

Вы можете удалить одну плитку за раз или выбрать и удалить несколько плиток одновременно.

Чтобы удалить одну плитку

1. Выберите плитку, чтобы отобразить команды плитки.
2. На панели команд плитки выберите  **More actions** >  **Delete**.
3. В сообщении **Are you sure you want to delete this tile?** выберите **Cancel** или **Delete**.

Чтобы удалить несколько плиток одновременно

1. Щёлкните на одной плитке, чтобы выбрать её.
2. Ctrl-щелчком выберите дополнительные плитки, которые хотите удалить.

   * Каждая выбранная плитка подсвечивается.
   * Общее количество выбранных плиток отображается на панели команд плитки.
3. На панели команд плитки выберите  **Delete**.
4. В сообщении **Are you sure you want to delete this tile?** выберите **Cancel** или **Delete**.
