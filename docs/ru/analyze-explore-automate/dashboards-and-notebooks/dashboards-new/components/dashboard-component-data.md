# Добавление данных на панель мониторинга

* Последнее Dynatrace
* Руководство по действию
* 10-минутное чтение
* Обновлено 27 мая 2025 г.

Эта страница описывает, как добавить данные с помощью запроса Grail, и как указать пользовательский временной интервал и сегмент.

Быстрейший и простейший способ изучить ваши данные - с помощью наших новых [Explore](/docs/analyze-explore-automate/dashboards-and-notebooks/explore-data "Изучите ваши данные с помощью нашего интерфейса point-and-click.") плиток и разделов. За несколько секунд вы можете найти и проанализировать ваши журналы, метрики или бизнес-события. Не требуется DQL!

Устаревшие: поля `dt.entity.*`

Если вы видите следующее сообщение:

`Поля dt.entity.* устарели. Пожалуйста, используйте поля dt.smartscape.* вместо этого.`

мы рекомендуем использовать эквивалентное поле `dt.smartscape.*` вместо этого.

Устаревшие поля `dt.entity.*` в конечном итоге будут удалены.

## Добавление данных

Чтобы выполнить запрос Grail

1. В правом верхнем углу панели мониторинга выберите **Добавить** > **DQL**.

   Горячая клавиша: **Shift**+**D**

   ![Панели мониторинга: кнопка добавления плитки (Плюс)](https://dt-cdn.net/images/updated-dashboards-add-tile-button-481-c21ba8f200.png)

   Открывается боковая панель конфигурации справа, отображающая две вкладки:

   * **Данные**
   * **Визуализация**
2. На вкладке **Данные** используйте язык запросов [Dynatrace Query Language (DQL)](/docs/platform/grail/dynatrace-query-language "Как использовать язык запросов Dynatrace.") для определения вашего запроса.
3. Выберите **Выполнить**, чтобы выполнить запрос.
4. На вкладке **Визуализация** выберите формат визуализации для ваших результатов.

   * означает, что тип визуализации недоступен для вашего запроса.
5. Под селектором **Визуализация** разверните разделы параметров, чтобы отрегулировать настройки визуализации по мере необходимости.
6. Закройте боковую панель, когда закончите.

   Если вы хотите вернуться к этим настройкам, выберите свою плитку, чтобы отобразить их.

## Указание пользовательского временного интервала

Чтобы указать пользовательский временной интервал в плитке панели мониторинга

1. Отредактируйте плитку.
2. В панели редактирования включите **Пользовательский временной интервал**.
3. Выберите временной интервал, который будет применен к выбранной плитке.

   Этот временной интервал переопределяет временной интервал панели мониторинга, установленный в правом верхнем углу панели мониторинга. С помощью этого метода панель мониторинга может иметь несколько плиток, каждая из которых имеет свой собственный временной интервал.

Вы также можете указать пользовательский временной интервал в запросе DQL плитки данных. Если вы используете этот метод (с временным интервалом, указанным в запросе), вышеуказанная настройка интерфейса отключена, и используется временной интервал, указанный в запросе.

Пример указания временного интервала в DQL:

```
fetch [recordtype], from:now() - 2h



| ....
```

Для получения подробной информации об указании временного интервала в DQL см. [Указание временного интервала](/docs/platform/grail/dynatrace-query-language/dql-guide#specifytimeframe "Узнайте, как работает DQL и какие ключевые понятия DQL.") в документации по DQL.

## Выбор сегментов

Чтобы фильтровать данные, вы можете указать сегменты на двух уровнях: панели мониторинга и плитки. Выбор сегментов на уровне плитки переопределяет выбор сегментов на уровне панели мониторинга.

Использовать ли сегменты или переменные?

#### Сегменты

Используйте сегменты, когда вы хотите повторно использовать их в разных панелях мониторинга. Например, используйте сегменты для повторяющихся фильтров, таких как для ваших Kubernetes кластеров, пространств имен, рабочих нагрузок или подов. Сегменты автоматически применяются поверх запросов ваших плиток/разделов, поэтому вам не нужно ссылаться на них внутри.

#### Переменные

Если вам нужно больше контроля над тем, как фильтр применяется, вы можете использовать переменные.

* Переменные позволяют вам полностью контролировать основной запрос или внутри вашего раздела Explore или плитки, определяя, где и как они применяются. Например, вы можете указать, как они соединяются с другими примененными фильтрами (**AND**, **OR**) и можете контролировать, какой оператор используется для вашего фильтра (такой как `equals`, `contains`, `startsWith` и `endsWith`).
* Кроме того, используйте переменные, когда вам нужен тонкий контроль над тем, как фильтры взаимозависимы.

* Для получения подробной информации о сегментах см. [Сегменты](/docs/manage/segments "Используйте сегменты для логической структуризации и удобного фильтрования данных наблюдаемости в приложениях.")
* Для конкретного случая использования сегментов **Панели мониторинга** см. [Анализ данных мониторинга с помощью сегментов](/docs/manage/segments/getting-started/segments-getting-started-analyze-monitoring-data "Узнайте, как анализировать данные мониторинга более эффективно, используя сегменты в Панелях мониторинга.")

Чтобы выбрать сегменты на уровне плитки

1. Отобразите панель мониторинга.
2. Выберите плитку, а затем выберите  в контроле плитки, чтобы отредактировать ее.
3. В панели редактирования включите **Пользовательские сегменты**.
4. В списке **Пользовательские сегменты** выберите сегмент.  
   Если сегменту требуется дополнительный выбор значения, выберите его сейчас.
5. Чтобы добавить другой сегмент, выберите  **Сегмент**. Повторите этот шаг для каждого сегмента, который вы хотите добавить для выбранной плитки.
6. Выберите **Применить**, чтобы применить выбор и отфильтровать данные на плитке.

   * Селектор сегментов  теперь отображает имя выбранного сегмента или, если вы выбрали более одного сегмента, количество выбранных сегментов.
   * Чтобы изменить ваш выбор сегмента, выберите  снова, внесите изменения и выберите **Применить**.
   * Чтобы управлять сегментами в целом (список, создание, просмотр, редактирование, удаление), выберите  и затем выберите ссылку **Управлять сегментами**.
## Пример данных 1

```
fetch logs



| summarize loglines = count(), by:{`1m interval` = bin(timestamp, 1m), status}
```

## Пример данных 2

### Создание двух переменных

Если вы еще не создали переменную, сначала ознакомьтесь с [Добавление переменной на панель мониторинга](/docs/analyze-explore-automate/dashboards-and-notebooks/dashboards-new/components/dashboard-component-variable#add-a-variable "Добавление переменных в ваши Dynatrace панели мониторинга.").

1. Выберите **Добавить переменную** и определите первую переменную.

   * **Имя:** `Хосты`
   * **Тип:** `Запрос`
   * **Запрос:**

     ```
     fetch logs



     | summarize count(), by:{`dt.entity.host`}



     | limit 100



     | sort `count()`, direction:"descending"



     | fields `dt.entity.host`
     ```
   * **Множественный выбор:** включен
2. Выберите **Добавить переменную** и определите вторую переменную.

   * **Имя:** `Уровень журнала`
   * **Тип:** `Запрос`
   * **Запрос:**

     ```
     fetch logs, from:now() - 2h



     | summarize  count(), by:{loglevel}



     | sort `count()`, direction:"descending"



     | fields loglevel
     ```
   * **Множественный выбор:** включен

### Добавление данных

Теперь добавьте данные, ссылаясь на ранее созданные переменные `$LogLevel` и `$Hosts`, чтобы вы могли позже использовать фильтры переменных в верхней части вашей панели мониторинга для фильтрации плитки в соответствии с вашими выборами.

```
fetch logs



| filter in (loglevel, {$Loglevel})



| limit 10
```

Выберите **Выполнить запрос**.
