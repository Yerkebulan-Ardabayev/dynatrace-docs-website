Ваша среда Dynatrace может интегрироваться со средой ServiceNow с помощью коннектора ServiceNow ![ServiceNow for Workflows](https://dt-cdn.net/images/servicenow-for-workflows-257-9349ea0329.png "ServiceNow for Workflows"), позволяя автоматически создавать инциденты на основе данных мониторинга и событий.
Кроме того, вы можете получать группы из ServiceNow и импортировать их как [команды владельцев](../../../deliver/ownership/ownership-teams.md#import-teams "Определение команд с идентификаторами, описаниями, обязанностями и маршрутизацией для владения сущностями.").

### Шаг 1. Предоставьте разрешения для Workflows

Некоторые разрешения необходимы Workflows для выполнения действий от вашего имени. Другие разрешения требуются действиями, входящими в состав коннектора ServiceNow.

Для тонкой настройки разрешений, предоставленных Workflows

1. Перейдите в **Workflows** и выберите **Настройки** > **Настройки авторизации**.
2. Выберите следующие разрешения помимо общего разрешения Workflows.

   * Разрешения, необходимые для действий рабочего процесса ServiceNow:

     + `app-settings:objects:read`

Подробнее об общих разрешениях пользователей Workflows см. [Разрешения пользователей для рабочих процессов](../security.md#user-permission "Руководство по аспектам безопасности автоматизации рабочих процессов в Dynatrace Workflows").

### Шаг 2. Создайте учётные данные аутентификации ServiceNow

Войдите в свой экземпляр ServiceNow и создайте учётные данные базовой аутентификации со следующими разрешениями. Эти учётные данные понадобятся на следующем шаге.

Разрешения пользователя ServiceNow:

* Поиск, создание и обновление инцидентов (таблица incident)

### Шаг 3. Настройте подключение ServiceNow

Для каждой из ваших сред ServiceNow необходимо настроенное подключение.

Для настройки подключения

1. Перейдите в **Настройки** и выберите **Подключения** > **Коннекторы** > **ServiceNow**.
2. Нажмите ![Add](https://dt-cdn.net/images/dashboards-app-menu-plus-7e9b7c3547.svg "Add") **Подключение**
3. Опишите ваше подключение ServiceNow.

   * **Имя подключения**: Укажите понятное имя для вашего подключения.
   * **URL экземпляра ServiceNow**: Добавьте URL вашей среды ServiceNow.
   * **Тип**: Используйте базовую аутентификацию или учётные данные клиента.

     + Для **Базовой аутентификации** укажите имя пользователя и пароль.
     + Для **Учётных данных клиента** укажите идентификатор клиента и секрет клиента.
4. Нажмите **Создать**.

## Доступные действия

Следующие действия рабочих процессов доступны для интеграции с ServiceNow. Каждое действие соответствует конечной точке API ServiceNow.

## Создание инцидента ServiceNow

Чтобы создать инцидент ServiceNow в вашем рабочем процессе ![Workflows](https://dt-cdn.net/images/workflows-1024-b5708f3cf9.webp "Workflows")

1. В панели **Выберите действие** найдите ServiceNow и выберите **Создать инцидент**.
2. В действии **Создать инцидент** выберите [Подключение](#connection) к вашей среде ServiceNow.
3. Настройте поля ввода по необходимости.

   | Поле | Описание | Обязательное |
   | --- | --- | --- |
   | **Идентификатор корреляции** | Уникальный идентификатор (в большинстве случаев это ID события Dynatrace). | Нет |
   | **Инициатор** | Пользователь, который сообщает об инциденте. | Нет |
   | **Категория** | Категория инцидента. Варианты категорий получаются запросом таблицы `sys_choice` с `sysparm_query: 'name=incident^element=category^inactive=false'`. | Да |
   | **Подкатегория** | Подкатегория инцидента. Варианты подкатегорий получаются запросом таблицы `sys_choice` с `sysparm_query: 'name=incident^element=subcategory^inactive=false^dependent_value=${category}'`. | Да |
   | **Влияние** | Влияние инцидента. | Да |
   | **Срочность** | Срочность инцидента. | Да |
   | **Группа назначения** | Группа, которая будет работать над инцидентом. Варианты групп получаются запросом таблицы `sys_user_group` с `sysparm_display_value: 'all'`. | Да |
   | **Элемент конфигурации** | Затронутая сущность. | Нет |
   | **Краткое описание** | Краткое описание инцидента. | Нет |
   | **Описание** | Подробное описание инцидента. | Нет |

   Подробнее см. [официальную документацию ServiceNow](https://dt-url.net/vc039n0).

   | Поле | Описание | Обязательное |

   | **SysParm Query** | Закодированный запрос для фильтрации набора результатов | Нет |

   | **Смещение** | Количество пропускаемых записей (по умолчанию: 0) | Нет |
   | **Сортировка по возрастанию** | Имя поля для сортировки результатов по возрастанию | Нет |
