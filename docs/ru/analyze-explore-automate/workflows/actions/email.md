# Email


* Latest Dynatrace
* 3-min read

Вы можете автоматизировать отправку готовых электронных писем с помощью Email ![Email for Workflows](https://dt-cdn.net/images/email-for-workflows-new-256-f6c0e2d343.png "Email for Workflows") на основе событий и расписаний, определённых вашими рабочими процессами.

## Предоставление разрешений для Workflows

Workflows требуют определённых разрешений для выполнения действий от вашего имени. Другие разрешения необходимы для действий, поставляемых в комплекте с Email.

Для тонкой настройки разрешений, предоставленных Workflows

1. Перейдите в **Workflows** ![Workflows](https://dt-cdn.net/images/workflows-1024-b5708f3cf9.webp "Workflows") и откройте **Settings** > **Authorization settings**.
2. Выберите следующие разрешения помимо общих разрешений Workflows.

   * Разрешения, необходимые для действий **Send email**

     + `email:emails:send`

Подробнее об общих пользовательских разрешениях Workflows см. в разделе Справочник по выражениям.

Пробные среды не имеют права отправлять электронные письма с помощью **Email**.

## Отправка электронного письма с помощью Workflows

1. Перейдите в **Workflows** ![Workflows](https://dt-cdn.net/images/workflows-1024-b5708f3cf9.webp "Workflows") и выберите **![Add](https://dt-cdn.net/images/dashboards-app-menu-plus-7e9b7c3547.svg "Add") Workflow**, чтобы создать новый рабочий процесс.
2. Выберите **Trigger**, чтобы подобрать подходящий триггер для ваших нужд.
3. На боковой панели выберите один из триггеров.
4. На узле триггера выберите **Add task** ![Add](https://dt-cdn.net/images/dashboards-app-menu-plus-7e9b7c3547.svg "Add"), чтобы просмотреть доступные действия.
5. На боковой панели найдите **Email** и выберите **Send email**.
6. В поле **To** введите адреса электронной почты получателей. Также доступны поля **Cc** и **Bcc**. Для полей **To**, **Cc** и **Bcc** количество адресов ограничено 10 на каждое поле.
7. В полях **Subject** и **Message** введите содержательный текст.
8. Выберите **Run**.

Email отправляет письма с адреса `no-reply@apps.dynatrace.com`. Рекомендуется добавить этот домен в белый список, чтобы письма не попадали в папку нежелательной почты.

## Форматирование письма

Для форматирования тела письма

1. Выберите **Message**.
2. Введите текст.

   Когда вы выбираете, например, жирный стиль, текст отображается жирным шрифтом в отправленном письме.
   Предварительный просмотр без символов форматирования недоступен.
   Однако при отправке письма получатель увидит корректно отформатированный текст, а символы форматирования не будут видны.

   Вы можете использовать панель инструментов для вставки стандартных элементов, таких как заголовок, жирный текст или ссылка.

   Вложение изображения или вставка кода JavaScript в тело письма невозможны. Они будут отображаться как обычный текст.

   ![Screenshot of a workflow with an email action where the email body text can be formatted.](https://dt-cdn.net/images/umsaywsjuo-dev-apps-dynatracelabs-com-ui-apps-dynatrace-automations-workflows-new-view-draft-task-send-email-3754-473a3b4d5b.png)

## Синтаксис форматирования

### Курсив

Оберните текст одинарными звёздочками (`*вот так*`), чтобы получить курсив *вот так*.

### Жирный

Оберните текст двойными звёздочками (`**вот так**`), чтобы получить жирный текст **вот так**.

### Зачёркивание

Оберните текст двойными тильдами (~~вот так~~), чтобы получить зачёркнутый текст ~~вот так~~.

### Код

* Оберните текст обратными кавычками (`вот так`), чтобы получить моноширинный текст `вот так`.
* Оберните текст тройными обратными кавычками (```вот так```), чтобы отобразить блок кода (несколько строк `кода`).

### Заголовки

Чтобы добавить заголовок, начните строку с символа `#`, например: `# Текст вашего заголовка`.

### Горизонтальная линия

Для визуального разделения секций вашей аннотации добавьте горизонтальную линию с помощью трёх дефисов (`---`):

### Списки

Каждая строка маркированного (ненумерованного) списка начинается со звёздочки (`*`):

```
* Line 1


* Line 2
```

В качестве альтернативы можно использовать дефис (`-`):

```
- Line 1


- Line 2
```

Нумерованный список начинается с числа и точки (`1.`), за которыми следуют пробел и текст:

```
1. The first line of my procedure.


2. The second line of my procedure.


3. The third line of my procedure.
```

### Таблицы

Чтобы добавить таблицу, определите заголовки, строку форматирования столбцов и затем строки с данными для отображения

```
| Header 1 | Header2


--- | ---


content2 | content2
```

| Header 1 | Header2 |
| --- | --- |
| content2 | content2 |

### Ссылки

Чтобы создать ссылку на веб-сайт, используйте следующий формат:

`[Пример текста ссылки](https://www.example.com)`

```
Here's a link to the Dynatrace website.
```

При использовании ссылок в формате Markdown в электронных письмах поддерживаются только определённые протоколы, которые преобразуются в HTML-ссылки:

* `http`
* `https`
* `ftp`
* `ftps`
* `mailto`
* `tel`
* `sms`

Неподдерживаемые протоколы ссылок

Ссылки с неподдерживаемыми протоколами (например, `file://`) не преобразуются из Markdown в HTML-ссылки и остаются обычным текстом. В зависимости от вашего почтового клиента такие ссылки могут быть выделены или отображены как кликабельные самим клиентом.

### Перенос строки

Чтобы добавить одиночный перенос строки и начать новую строку, нажмите **Enter** в поле **Message** коннектора.

Поддержка HTML отсутствует

Этот коннектор позволяет форматировать тело сообщения только с использованием перечисленных вариантов форматирования. Поддержка HTML отсутствует.

Форматирование электронного письма отключено для сообщений размером 256 КиБ и более.

## Входные данные

| Поле | Описание | Обязательно |
| --- | --- | --- |
| **To** | Получатели письма. | Обязательно |
| **Cc** | Получатели копии письма. | Обязательно |
| **Bcc** | Получатели скрытой копии письма. | Обязательно |
| **Subject** | Тема письма. | Обязательно |
| **Content** | Содержание письма. | Обязательно |

Необходимо указать как минимум одного и максимум десять получателей в одном из полей (to, cc, bcc).
Если вы используете выражения для получателей, выражения должны возвращать список адресов электронной почты, например: `["user1@domain.com", "user2@domain.com"]`.

## Результат действия

Действие **Send email** возвращает следующий результат.

## Ожидаемое поведение 'Failed to send email. Status code 403.'

Failed to send email. Status code 403.

Это сообщение связано с отсутствием разрешений или использованием пробной среды, как описано выше в разделе [Предоставление разрешений для Workflows](#permissions).

### Отправка JSON-данных в качестве содержимого сообщения

При отправке объекта или нестроковых данных в качестве содержимого сообщения убедитесь, что они правильно отформатированы как строка.
Рекомендуемый подход — обернуть входные данные в выражение и преобразовать их в JSON-строку с помощью фильтра `to_json`:

```
{{ result("run_javascript_1") | to_json }}
```

Этот подход обеспечивает правильное форматирование данных для передачи. Дополнительную информацию см. в разделе Справочник по выражениям.

Размер сообщения ограничен 256 КиБ. Более крупные данные приведут к ошибке выполнения действия.
