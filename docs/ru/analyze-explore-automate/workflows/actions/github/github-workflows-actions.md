# Действия для GitHub Connector


* Latest Dynatrace

На этой странице описаны действия рабочих процессов GitHub, доступные для интеграции GitHub Connector ![GitHub](https://dt-cdn.net/images/github-for-workflows-new-lxjn9po-256-94579b3812.png "GitHub").

### Необходимые разрешения токена для выполнения всех действий GitHub

## Получение содержимого

Действие **Get content** возвращает содержимое существующего файла в репозитории.

### Входные данные

| Поле | Описание | Обязательность |
| --- | --- | --- |
| **Connection** | [Подключение](github-workflows-setup.md#connection "Узнайте, как настроить GitHub Connector.") к вашей среде GitHub. | Обязательно |
| **Owner** | Владелец учётной записи репозитория (частный пользователь или организация). | Обязательно |
| **Repository** | Имя репозитория без расширения `.git`. Регистр не учитывается. | Обязательно |
| **File path** | Путь к одному существующему файлу. Путь к директории не поддерживается. | Обязательно |
| **Reference** | Имя коммита, ветки или тега. Если не задано, используется ветка по умолчанию репозитория. | Необязательно |

### Выходные данные

Действие возвращает свойство `content`, содержащее текстовое содержимое существующего файла в репозитории.

### Необходимые разрешения токена

## Создание или замена файла

Действие **Create or replace file** создаёт новый файл с указанным содержимым или заменяет существующий. Затем оно коммитит изменение во вновь созданную ветку на основе указанной исходной ветки или в указанную существующую ветку.

### Входные данные

| Поле | Описание | Обязательность |
| --- | --- | --- |
| **Connection** | [Подключение](github-workflows-setup.md#connection "Узнайте, как настроить GitHub Connector.") к вашей среде GitHub. | Обязательно |
| **Owner** | Владелец учётной записи репозитория (частный пользователь или организация). | Обязательно |
| **Repository** | Имя репозитория без расширения `.git`. Регистр не учитывается. | Обязательно |
| **Create new branch** | Выберите этот вариант, если хотите внести изменение в новую ветку. | Необязательно |
| **Commit to an existing branch** | Выберите этот вариант, если собираетесь использовать существующую ветку для изменения. | Необязательно |
| **Source branch** | Исходная ветка, на основе которой создаётся новая ветка. Например: `main`. | Необязательно |
| **Branch** | Новая или существующая ветка, в которую вы хотите сделать коммит. | Обязательно |
| **File path** | Относительный путь к файлу. Путь к директории не поддерживается. Существующие файлы будут перезаписаны. | Обязательно |
| **File content** | Полное содержимое файла. Существующие файлы будут перезаписаны. | Обязательно |
| **Commit message** | Сообщение коммита git для изменения. | Обязательно |

### Выходные данные

Действие возвращает свойство `fileMetadata`, содержащее полный ответ соответствующего эндпоинта GitHub API.

## Создание pull request

### Входные данные

| Поле | Описание | Обязательность |

## Обновление issue

Действие **Update issue** обновляет существующий issue в указанном репозитории. Используйте опцию **Add Field** для указания полей для обновления.

### Входные данные

| Поле | Описание | Обязательность |
| --- | --- | --- |
| **Connection** | [Подключение](github-workflows-setup.md#connection "Узнайте, как настроить GitHub Connector.") к вашей среде GitHub. | Обязательно |
| **Owner** | Владелец учётной записи репозитория (частный пользователь или организация). | Обязательно |
| **Repository** | Имя репозитория без расширения `.git`. Регистр не учитывается. | Обязательно |
| **Issue Number** | Номер обновляемого issue. | Обязательно |
| **Title** | Новый заголовок issue. Отображается только после добавления поля. Если добавленное поле оставить пустым, оно будет обновлено пустым значением. | Необязательно |
| **Description** | Новое описание или тело issue. Отображается только после добавления поля. Если добавленное поле оставить пустым, оно будет обновлено пустым значением. | Необязательно |
| **State** | Новое состояние issue. Отображается только после добавления поля. Если добавленное поле оставить пустым, оно будет обновлено пустым значением. | Необязательно |
| **Assignees** | Новый список имён пользователей для назначения на issue. Отображается только после добавления поля. Если добавленное поле оставить пустым, оно будет обновлено пустым значением. | Необязательно |
| **Labels** | Новый список меток для связи с issue. Отображается только после добавления поля. Если добавленное поле оставить пустым, оно будет обновлено пустым значением. | Необязательно |

## Запуск рабочего процесса

Действие **Trigger workflow run** запускает рабочий процесс в указанном репозитории. Рабочий процесс должен быть настроен для запуска по событию `workflow_dispatch`. Если рабочий процесс вызывает повторно используемые рабочие процессы из других приватных репозиториев, вы можете сделать их доступными в **Settings** > **Actions** > **General** > **Access** на GitHub.

### Входные данные

| Поле | Описание | Обязательность |
| --- | --- | --- |
| **Connection** | [Подключение](github-workflows-setup.md#connection "Узнайте, как настроить GitHub Connector.") к вашей среде GitHub. | Обязательно |
| **Owner** | Владелец учётной записи репозитория (частный пользователь или организация). | Обязательно |
| **Repository** | Имя репозитория без расширения `.git`. Регистр не учитывается. | Обязательно |
| **Workflow** | Идентификатор рабочего процесса или имя файла рабочего процесса, включая расширение `.yml` или `.yaml`. | Обязательно |
| **Reference** | Имя ветки или тега для рабочего процесса. Например: `main`. | Обязательно |
| **Inputs** | Пары ключ-значение входных параметров для передачи рабочему процессу в формате JSON. | Необязательно |

### Выходные данные

Действие не возвращает результат.
