# Отправка уведомлений по электронной почте о проблемах

* Latest Dynatrace
* Руководство
* Чтение: 4 мин
* Обновлено 19 октября 2025 г.

Проблемы автоматически создаются Dynatrace при обнаружении аномалий или срабатывании условий оповещений в вашей среде.
В ![Workflows](https://dt-cdn.net/images/workflows-1024-b5708f3cf9.webp "Workflows") **Workflows** создайте простой рабочий процесс, который отслеживает проблемы и автоматически отправляет оповещения по ![Email for Workflows](https://dt-cdn.net/images/email-for-workflows-new-256-f6c0e2d343.png "Email for Workflows") электронной почте.
В этом руководстве показано, как это сделать.

![Скриншот рабочего процесса для обучения отправке уведомлений по электронной почте о проблемах Davis с помощью простого рабочего процесса.](https://dt-cdn.net/images/umsaywsjuo-dev-apps-dynatracelabs-com-ui-apps-dynatrace-automations-workflows-e45c4e16-ae41-422d-b6d6-0f8d2aa3d015-view-live-3728-0fad43a67d.png)

## Чему вы научитесь

В этом руководстве вы узнаете, как оповещать вашу команду в реальном времени, отправляя детали новой проблемы по электронной почте конкретному получателю.

Краткий обзор:

1. [Создайте простой рабочий процесс](/docs/analyze-explore-automate/workflows/simple-workflow#create-simple-workflow "Создание и запуск простого рабочего процесса.").
2. Добавьте [триггер события](/docs/analyze-explore-automate/workflows/trigger/event-trigger "Руководство по созданию триггеров событий для автоматизации рабочих процессов в Dynatrace Workflows.") для [проблем Davis](/docs/analyze-explore-automate/workflows/trigger/event-trigger#davis-problem-trigger "Руководство по созданию триггеров событий для автоматизации рабочих процессов в Dynatrace Workflows.").
3. Добавьте [уведомление по электронной почте](/docs/analyze-explore-automate/workflows/actions/email "Автоматизация отправки готовых электронных писем на основе событий и расписаний, определенных для ваших рабочих процессов.").
4. Сохраните и запустите рабочий процесс для получения уведомлений по электронной почте.
5. Проверьте, что ваш рабочий процесс работает корректно.

## Предварительные требования

* У вас должно быть разрешение на настройку и запуск простого рабочего процесса.
  Например, разрешение, предоставляемое политикой по умолчанию для [стандартного пользователя](/docs/manage/identity-access-management/permission-management/default-policies "Справочник по политикам Dynatrace по умолчанию").
* Необходимо выбрать соответствующие разрешения в [настройках авторизации](/docs/analyze-explore-automate/workflows/security#authorization-settings "Руководство по аспектам безопасности автоматизации рабочих процессов в Dynatrace Workflows").

  + Необходимо разрешить следующее:

    - Доступ к ![Workflows](https://dt-cdn.net/images/workflows-1024-b5708f3cf9.webp "Workflows") **Workflows**.
    - Создание и выполнение рабочего процесса.
      Подробнее см. в [настройках авторизации](/docs/analyze-explore-automate/workflows/security#user-permission "Руководство по аспектам безопасности автоматизации рабочих процессов в Dynatrace Workflows").

## Шаги

1. [Создайте простой рабочий процесс](/docs/analyze-explore-automate/workflows/simple-workflow#create-simple-workflow "Создание и запуск простого рабочего процесса.").

   1. Перейдите в ![Workflows](https://dt-cdn.net/images/workflows-1024-b5708f3cf9.webp "Workflows") **Workflows**.
   2. Выберите ![Add](https://dt-cdn.net/images/dashboards-app-menu-plus-7e9b7c3547.svg "Add") **Workflow** в правом верхнем углу страницы.
   3. Выберите заголовок рабочего процесса.
      По умолчанию это `Untitled workflow` -- введите понятное название.
      Тип рабочего процесса по умолчанию установлен как простой рабочий процесс.
2. Добавьте [триггер события](/docs/analyze-explore-automate/workflows/trigger/event-trigger "Руководство по созданию триггеров событий для автоматизации рабочих процессов в Dynatrace Workflows.") для [проблем Davis](/docs/analyze-explore-automate/workflows/trigger/event-trigger#davis-problem-trigger "Руководство по созданию триггеров событий для автоматизации рабочих процессов в Dynatrace Workflows.").

   1. В разделе **Select trigger** выберите [триггер проблем Davis](/docs/analyze-explore-automate/workflows/trigger/event-trigger#davis-problem-trigger "Руководство по созданию триггеров событий для автоматизации рабочих процессов в Dynatrace Workflows.").
   2. Установите **Problem state** в значение **active or closed**.
      Этот параметр означает, что проблема может быть как активной, так и закрытой.
      Эта настройка приводит к двукратному срабатыванию рабочего процесса: один раз при активации проблемы и второй раз при ее закрытии.
   3. В выпадающем списке **Event category** выберите **Select all**.
   4. Необязательно. Выберите **Query past events**, чтобы увидеть последние события проблем, которые вызвали бы этот рабочий процесс.
   5. Необязательно. Введите **Entity tags** или **Additional custom filter query**, чтобы рабочий процесс срабатывал только на релевантные проблемы.
3. Добавьте [уведомление по электронной почте](/docs/analyze-explore-automate/workflows/actions/email "Автоматизация отправки готовых электронных писем на основе событий и расписаний, определенных для ваших рабочих процессов.").

   1. Выберите ![Add](https://dt-cdn.net/images/dashboards-app-menu-plus-7e9b7c3547.svg "Add") **Add task** на узле триггера.
   2. В разделе **Choose action** выберите тип действия **Send email**.
      Дайте типу действия понятное название.
   3. Введите адрес электронной почты получателя в поле **To**.
   4. В поле **Subject** введите следующее:

      ```
      {{ event()["event.status"] }} Problem {{ event()["display_id"] }}: {{ event()["event.category"] }} {{ event()["event.name"] }}
      ```
   5. В поле **Message** введите следующее:

      ```
      {{ event()["event.description"] }}



      Go to problem: {{ environment().url }}/ui/apps/dynatrace.davis.problems/problem/{{ event()["event.id"] }}
      ```

   Эта конфигурация использует контекстные заполнители событий для динамического наполнения электронного письма релевантными деталями проблемы.

   Триггер проблем Davis возвращает запись о проблеме.
   Вы можете использовать любое поле из записи о проблеме, хранящейся в `dt.davis.problems`, в уведомлении по электронной почте.
4. Сохраните и запустите рабочий процесс для получения уведомлений по электронной почте.

   1. Выберите **Create draft**.
   2. Выберите **Deploy**.
   3. Выберите **Run**, чтобы увидеть выбранное событие проблемы, которое будет использоваться для рабочего процесса.
5. Проверьте, что ваш рабочий процесс работает корректно:

   1. Перейдите к вашему рабочему процессу.
   2. Выберите **Run**.
   3. Выберите **Run** еще раз для выполнения рабочего процесса.

      После выполнения рабочего процесса вы должны увидеть статус **Success** в верхней части редактора рабочего процесса.
      Журналы выполнения недоступны для простого рабочего процесса.
      В случае ошибки вы можете найти подробности ошибки справа в панели деталей задачи.

      ![Скриншот успешно выполненного рабочего процесса уведомлений по электронной почте.](https://dt-cdn.net/images/umsaywsjuo-dev-apps-dynatracelabs-com-ui-apps-dynatrace-automations-executions-ced3847e-34b5-4a04-9312-a79f423e32a2-3718-dfc1a7d8c5.png)

## Заключение

Вы создали простой рабочий процесс, который отправляет уведомления по электронной почте при открытии или закрытии проблем.
Эта настройка помогает обеспечить немедленное информирование вашей команды о критических проблемах в среде.

Вы можете расширить этот рабочий процесс, добавив:

* Условия для обработки определенных категорий или степеней серьезности проблем.
* Шаги автоматического устранения в рабочий процесс.

Этот рабочий процесс является отличной отправной точкой для автоматизации реагирования на инциденты и повышения операционной осведомленности.
