# Действие HTTP request для Workflows


* Latest Dynatrace
* Reference
* 1-min read
* Published Apr 02, 2024

Действие **HTTP Request** для Workflows позволяет отправлять HTTP-запросы к любому API.

## Безопасность действия HTTP Request

* Все HTTP-вызовы проверяются по глобальному списку разрешённых адресов.

Мы настоятельно не рекомендуем использовать статический заголовок Authorization, так как это может привести к утечке секрета. Используйте хранилище учётных данных для хранения данных аутентификации типа **Basic** или **Token**, либо действие Run JavaScript для реализации любого другого типа аутентификации.

## Входные данные

* **Method**: метод HTTP-запроса.
* **URL**: URL-адрес, на который направляется запрос.
* **Authentication**: действие HTTP Request поддерживает использование учётных данных из хранилища учётных данных для аутентификации Basic и Token.
  Убедитесь, что конфигурация учётных данных [разрешает доступ](../../../../common/manage/credential-vault.md#access-cv "Хранение и управление учётными данными в хранилище учётных данных.") для участника рабочего процесса (область учётных данных: `AppEngine`, доступ приложения: `Workflows app`, доступ владельца или участника как выбранного пользователя).
* **Payload**: полезная нагрузка HTTP-запроса.
  Установите соответствующий заголовок content-type.
* **Headers**: заголовки HTTP-запроса с их именем и значением.
  Обратите внимание, что любое значение заголовка доступно в мониторе рабочего процесса всем пользователям, имеющим доступ к рабочему процессу. Поэтому настоятельно не рекомендуется раскрывать секреты — используйте конфигурацию аутентификации через хранилище учётных данных.
* **Error handling**: коды HTTP-ответа, при которых задача завершается с ошибкой.
  В зависимости от сценария использования выполнение HTTP-запроса должно считаться успешным независимо от ответа.
  Поэтому отключите опцию завершения с ошибкой при определённых кодах HTTP-ответа или укажите конкретные коды, при которых задача должна завершаться с ошибкой.

## Результат

Результат действия HTTP request представляет собой структуру JSON, которая включает:

* **status\_code**: код статуса HTTP-ответа.
* **body**: необработанное тело HTTP-ответа.
* **headers**: заголовки HTTP-ответа в виде объектов, состоящих из имени и значения заголовка.
* **json**: HTTP-ответ в виде документа JSON. Поддерживаются только полезные нагрузки ответа в формате JSON.
