# Data Explorer

Классическая функция Dynatrace для запросов и визуализации метрик, интегрированная с **Dashboards Classic**. Рекомендуется обновиться до последней версии.

Возможности:
* Анализ многомерных метрик (встроенных, StatsD, Prometheus, Telegraf)
* Выбор типа визуализации
* Закрепление визуализаций на классических дашбордах

## Компоненты запросов

Каждый запрос метрик включает:

* **Имя метрики** -- выбор из списка, поиск, категории, избранное.
* **Пространственная агрегация** -- как агрегировать по измерениям: Auto, Average, Count, Maximum, Minimum, Sum, Median, Percentile 10th/75th/90th.
* **Split by** -- разделение по измерению (например, Host).
* **Sort by** -- сортировка: ASC/DESC.
* **Rate** -- None, Per second/minute/hour.
* **Filter by** -- фильтрация по измерениям и атрибутам. Можно добавить несколько фильтров.
* **Limit** -- 1, 10, 20, 100 (по умолчанию 20).
* **Global limit** -- выравнивание записей по нескольким метрикам с одинаковым Split by (для Table с 2+ метриками).
* **Default** -- значение по умолчанию.
* **Timeshift** -- сдвиг по времени (число + единица).

## Визуализация

Типы: Graph, Stacked column, Stacked area, Pie, Single value, Table, Top list, Heatmap, Honeycomb.

## Команды запросов

* **Add metric** -- новая строка запроса.
* **Duplicate** -- копирование метрики.
* **Add metric event** -- создание события метрики из запроса.
* Перетаскивание -- изменение порядка метрик.
* Переключение -- включение/выключение метрик (буква или иконка глаза).
* **Delete** -- удаление метрики.
* **Advanced mode** -- полные возможности Metrics API v2 (выражения метрик, сдвиги по временным рамкам).

## Базовые линии

Добавление доверительного интервала к графику для выявления аномалий (модель Seasonal baseline). Только для визуализации Graph.

* **Добавить**: выберите линию на графике > **Add baseline**.
* **Скрыть/показать**: через легенду графика.
* **Удалить**: выберите линию > **Remove baseline**.

Различия с событиями метрик: фиксированные параметры (Tolerance=4, Alert condition='outside'), разрешение зависит от текущего выбора.

## Коррелированные метрики

Davis AI находит метрики с похожей формой серии. Требуется: визуализация Graph + Split by по измерению.

1. Выполните запрос с Split by.
2. Выберите линию на графике > **See correlated metrics**.
3. На боковой панели выберите метрику для добавления в запрос.

## Фокус

Временное скрытие всех метрик кроме выбранной: выберите линию > **Focus**. Снять: **Remove focus**.

## Результат

* Взаимодействие с визуализацией: наведение, клики для деталей.
* **Pin to dashboard** -- закрепление на дашборде.
* **Share link** -- ссылка для других пользователей (с/без временного диапазона).
* **Export CSV** -- экспорт в CSV (для всех визуализаций кроме honeycomb и single value).
* **Copy request** -- копирование запроса для API (JSON или CSV формат).

## Автоматически расширенная фильтрация

Использует топологию Dynatrace для предоставления дополнительных фильтров, недоступных в исходной метрике. Работает на уровне плитки (выбор связей) и дашборда (автоматическое расширение).

Примеры:
* Метрика шага Synthetic, расширенная монитором Synthetic.
* Метрика хоста, расширенная экземпляром EC2 или группой хостов.

## Ограничения

* Максимум 10 метрик на визуализацию, до 100 серий на метрику.
* Максимум 4000 точек данных на плитке дашборда (автоматическое переключение разрешения).
* Data Explorer использует долгосрочные данные метрик (не данные трассировок).
