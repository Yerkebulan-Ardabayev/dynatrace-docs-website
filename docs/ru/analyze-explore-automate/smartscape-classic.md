# Визуализация вашей среды с помощью Smartscape Classic


![Smartscape](https://dt-cdn.net/images/smartscape-topology-512-dca23011f9.png "Smartscape") **Smartscape Classic** — инструмент визуализации топологии среды, работающий практически в реальном времени, является одной из самых мощных возможностей Dynatrace.

![Smartscape](https://dt-cdn.net/images/smartscape-topology-512-dca23011f9.png "Smartscape") **Smartscape Classic** с автоматическим обнаружением обеспечивает быструю и эффективную визуализацию всех топологических зависимостей в вашей инфраструктуре, процессах и сервисах:

* По вертикальной оси отображаются зависимости полного стека по всем уровням
* По горизонтальной оси визуализируются все входящие и исходящие связи вызовов внутри каждого уровня

Всего за несколько нажатий ![Smartscape](https://dt-cdn.net/images/smartscape-topology-512-dca23011f9.png "Smartscape") **Smartscape Classic** предоставляет вам доступ к подробному топологическому представлению всей вашей среды, давая больше информации и контроля над вашей средой.

* Принимайте лучшие решения, например, корректируя архитектуру сервисов или инфраструктуру для улучшения производительности приложений.
* Изучайте межуровневые и одноуровневые взаимозависимости процессов, хостов и сервисов, чтобы лучше понять, как зависимости влияют на производительность ваших приложений.
* Углубляйтесь для более чёткого понимания проблем. Например, Dynatrace может выявить проблему с зависимостями от сторонних сервисов и помочь вам понять влияние этой проблемы на производительность вашего приложения.

## Доступ к Smartscape **Smartscape Classic**

Для доступа к классическому представлению топологии среды Smartscape перейдите в ![Smartscape](https://dt-cdn.net/images/smartscape-topology-512-dca23011f9.png "Smartscape") **Smartscape Classic**.

Для начала выберите представление, которое хотите увидеть:

* Выберите **Show problems** для отображения топологии [проблем](#problems)
* Выберите **Show third-party vulnerabilities** и укажите уровни риска (можно выбрать несколько), чтобы отобразить топологию [уязвимостей сторонних компонентов](#vulnerabilities)

## Проблемы

При выборе **Show problems** каждый уровень ![Smartscape](https://dt-cdn.net/images/smartscape-topology-512-dca23011f9.png "Smartscape") **Smartscape Classic** включает отдельную вкладку с индикатором состояния. Выберите уровень, на котором хотите просмотреть топологию:

* Приложения
* Сервисы
* Процессы
* Хосты
* Центры обработки данных

Каждый уровень показывает различный тип сущности: приложение, сервис, процесс, хост или центр обработки данных.

* Для просмотра другого уровня выберите вкладку этого уровня.
* Для масштабирования используйте кнопки **+/-** в правом верхнем углу или колесо мыши.
* Для перемещения обзора нажмите и перетащите в любом месте ![Smartscape](https://dt-cdn.net/images/smartscape-topology-512-dca23011f9.png "Smartscape") **Smartscape Classic**.
* Для просмотра имени сущности наведите курсор на символ сущности. Пока имя отображается, вы можете нажать стрелку рядом с именем сущности, чтобы перейти на страницу обзора этой сущности.

В первом примере изображения ниже:

* Уровень **Hosts** отображает состояние **1/33**, что означает 33 отслеживаемых хоста, один из которых имеет проблему.
* Другие четыре уровня (**Applications**, **Services**, **Processes** и **Data centers**) в этом примере показывают только количество сущностей на каждом уровне, поскольку все сущности на этих уровнях исправны.

![Smartscape](https://dt-cdn.net/images/smartscape-topology-512-dca23011f9.png "Smartscape") **Smartscape Classic** отображает данные за последние 72 часа и поэтому не может быть скорректирован с помощью селектора временного интервала. Для просмотра зависимостей между сущностями одного уровня просто выберите соответствующую вкладку слева.

### Приложения

В этом примере выбрана вкладка **Applications**, и **12** обнаруженных приложений в этой среде отображаются справа.

![Smartscape applications](https://dt-cdn.net/images/smartscape-applications-1310-dc4704a941.png)

Dynatrace поддерживает мониторинг веб-приложений и мобильных приложений. Символ узла указывает тип приложения:

| Иконка | Значение |
| --- | --- |
| Символ Smartscape new | **Веб-приложение** |
| Символ Smartscape 2 | **Мобильное приложение** |

Между приложениями нет соединений или зависимостей, потому что в Dynatrace приложения рассматриваются с точки зрения пользователя и поэтому представляют собой только пользовательские конечные точки. Таким образом, несколько приложений могут использовать одни и те же сервисы, но приложения не могут быть напрямую связаны друг с другом.

### Сервисы

Вкладка **Services** отображает топологию всех сервисов, работающих в вашей среде.

![Smartscape services](https://dt-cdn.net/images/smartscape-services-1621-d595e2d107.png)

**Узлы**

Каждый узел представляет отдельный сервис с символом, обозначающим базовый тип сервиса.

| Иконка | Значение |
| --- | --- |
| см. пример выше | Коммерческие логотипы, такие как дельфин MySQL, перо Apache и кот Tomcat, обозначают [технологии сервисов, поддерживаемые Dynatrace](../ingest-from/technology-support.md "Найдите технические детали о поддержке Dynatrace для конкретных платформ и фреймворков разработки."). |
| Символ Smartscape 3 | Сервисы нераспознанных технологий обозначаются обобщённым символом сервиса. Обычно это [непрозрачные сервисы](../observe/application-observability/services/service-detection/service-detection-v1/service-types/opaque-services.md "Узнайте, что такое непрозрачные сервисы."). |

**Соединения**

* Стрелки на линиях соединения к выбранному сервису и от него указывают, являются ли они входящими или исходящими вызовами сервисов.
* Пунктирная линия соединения означает, что за последние два часа между двумя сервисами не было запросов, или что сервис не пересылал и не получал данные за последние два часа.
* Соединение устаревает и больше не отображается в ![Smartscape](https://dt-cdn.net/images/smartscape-topology-512-dca23011f9.png "Smartscape") **Smartscape Classic**, если оно было неактивно более 72 часов или если сервис не получал нагрузки в течение последних 72 часов.

**Фильтрация отображения**
Откройте меню обзора (**...**) в правом верхнем углу ![Smartscape](https://dt-cdn.net/images/smartscape-topology-512-dca23011f9.png "Smartscape") **Smartscape Classic**, чтобы скрыть или отобразить:

* Неактивные запросы
* Сервисы, не связанные с другими сервисами

Обратите внимание, что вы можете заметить разницу в количестве сервисов, отображаемых на вкладке **Services** страницы ![Smartscape](https://dt-cdn.net/images/smartscape-topology-512-dca23011f9.png "Smartscape") **Smartscape Classic**, по сравнению со страницей **Services**. Это связано с тем, что ![Smartscape](https://dt-cdn.net/images/smartscape-topology-512-dca23011f9.png "Smartscape") **Smartscape Classic** учитывает более широкий спектр категорий сервисов.

### Процессы

Вкладка **Processes** предоставляет визуализацию, в которой каждый узел соответствует процессу, а каждое соединение представляет TCP/IP-запрос.

* Не все процессы, работающие в вашей среде, показаны, поскольку количество работающих процессов часто очень велико. Для наглядности показаны только [наиболее важные процессы](../observe/infrastructure-observability/process-groups/basic-concepts/which-are-the-most-important-processes.md "Отображение наиболее важных процессов для мониторинга и группировки процессов.").
* Пунктирные линии показывают соединения между процессами, которые неактивны или истекли по времени.

![Smartscape processes](https://dt-cdn.net/images/smartscape-processes-1623-2321abf3d3.png)

### Хосты

Вкладка **Hosts** показывает топологию вашей инфраструктуры с точки зрения хостов.

* Соединения представляют TCP/IP-запросы.
* Хосты, физические или виртуальные машины, обозначаются коммерческими логотипами операционных систем хостов.

![Smartscape hosts](https://dt-cdn.net/images/smartscape-hosts-1659-8c03fceb20.png)

Обобщённые символы хостов используются для обозначения кандидатов на мониторинг (неизвестные хосты, обнаруженные потому, что они получают TCP/IP-запросы от отслеживаемых хостов). Мы рекомендуем установить Dynatrace OneAgent на кандидатов мониторинга в вашей среде по возможности.

| Иконка | Значение |
| --- | --- |
| Символ Smartscape 4 | кандидат на мониторинг |

Неактивные кандидаты мониторинга — это кандидаты мониторинга, которые не обменивались данными с хостом. Неактивные соединения между хостами и соединения с истёкшим временем визуализируются пунктирными линиями.

### Центры обработки данных

Вкладка **Data centers** отображает узлы, указывающие, где расположены ваши хосты. Если у вас есть физические серверы в инфраструктуре, соответствующие узлы центров обработки данных указывают города, в которых расположены центры. Они обозначаются обобщёнными иконками хостов.

Если вы используете виртуальные серверы или инфраструктуру на базе PaaS, узлы центров обработки данных будут помечены соответственно (например, центр обработки данных VMware, зона доступности AWS или регион Azure) и визуализированы соответствующими логотипами компаний.

![Smartscape datacenters](https://dt-cdn.net/images/smartscape-datacenters-1625-63d687b49e.png)

### Межуровневые связи

Для просмотра межуровневых связей выберите любую сущность в любом уровне ![Smartscape](https://dt-cdn.net/images/smartscape-topology-512-dca23011f9.png "Smartscape") **Smartscape Classic**. Вертикальные зависимости сущности отобразятся слева.

В этом примере выбрано веб-приложение под названием `easytravel-dynatrace-dev`. Слева вы можете увидеть, как зависимости приложения распространяются вниз до уровня центров обработки данных.

![Smartscape inter 1](https://dt-cdn.net/images/smartscape-inter1-1687-680a65579e.png)

Приложение `easytravel-dynatrace-dev` отмечено красным, поскольку у него проблема. Двигаясь вниз по связанным зависимостям, мы видим, что это приложение вызывает 27 сервисов Tomcat и 4 сервиса ASP/.NET. Эти сервисы работают на процессах тех же типов технологий (отмеченные красным испытывают проблемы). Вы можете видеть, что эти процессы находятся на хосте Windows под названием `lr-ws-l02v` (для просмотра этой детали наведите курсор на узел хоста). Поскольку неисправные процессы работают на этом хосте, этот хост также неисправен (обратите внимание на красную **1** на плитке **Hosts** слева).

Этот пример показывает, как можно легко обнаружить первопричину проблемы и сократить время решения проблемы, исследуя связанные сущности в ![Smartscape](https://dt-cdn.net/images/smartscape-topology-512-dca23011f9.png "Smartscape") **Smartscape Classic**.

## Уязвимости сторонних компонентов

При выборе **Show third-party vulnerabilities** в ![Smartscape](https://dt-cdn.net/images/smartscape-topology-512-dca23011f9.png "Smartscape") **Smartscape Classic** выберите **Risk level**, чтобы указать уровни риска для отображения. Можно установить несколько флажков.

* В левой части вертикальный столбец показывает количество затронутых узлов красным цветом и общее количество узлов серым цветом на уровнях процессов и хостов.
* На топологии затронутые узлы отображаются цветом, соответствующим уровню риска:

  + Тёмно-красный для `Critical`
  + Красный для `High`
  + Жёлтый для `Medium`
  + Синий для `Low`

Для просмотра уязвимостей затронутых сущностей переключитесь на уровень **Processes** или **Hosts**. (Уязвимости могут быть связаны с приложением, сервисом или центром обработки данных, но они не влияют на них напрямую. Непосредственно затронуты могут быть только процессы или хосты.)

* Для масштабирования используйте кнопки **+/-** в правом верхнем углу или колесо мыши.
* Для перемещения обзора нажмите и перетащите в любом месте ![Smartscape](https://dt-cdn.net/images/smartscape-topology-512-dca23011f9.png "Smartscape") **Smartscape Classic**.
* Для просмотра имени сущности наведите курсор на символ сущности. Пока имя отображается, вы можете нажать стрелку рядом с именем сущности, чтобы перейти на страницу обзора этой сущности.
* Для просмотра межуровневых связей выберите любую сущность. Панель слева раскроется и отобразит вертикальные зависимости сущности.

Подробнее об управлении уязвимостями сторонних компонентов см. [Уязвимости сторонних компонентов](../secure/application-security/vulnerability-analytics/third-party-vulnerabilities.md "Отслеживайте, визуализируйте, анализируйте и устраняйте уязвимости сторонних компонентов, следите за ходом устранения и создавайте правила мониторинга.").
