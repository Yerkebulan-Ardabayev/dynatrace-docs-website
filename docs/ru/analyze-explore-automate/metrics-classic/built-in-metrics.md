# Встроенные классические метрики


* Справочник
* Чтение: 3 мин
* Опубликовано 3 сентября 2020 г.

Каждая технология, поддерживаемая Dynatrace, предлагает множество «встроенных» метрик. Встроенные метрики включены в продукт из коробки, в некоторых случаях как часть встроенных расширений.

Метрики, основанные на расширениях OneAgent или ActiveGate (префикс `ext:`), и вычисляемые метрики (префикс `calc:`) являются пользовательскими метриками, а не встроенными; [потребление DDU](../../license/monitoring-consumption-classic/davis-data-units/metric-cost-calculation.md "Узнайте, как рассчитать потребление Davis data unit и затраты, связанные с мониторингом метрик.") для этих метрик может значительно варьироваться в зависимости от того, как вы используете Dynatrace.

Префикс `ext:` используется метриками из [расширений OneAgent](../../ingest-from/extensions/develop-your-extensions.md "Разрабатывайте собственные расширения в Dynatrace.") и [расширений ActiveGate](../../ingest-from/extensions/develop-your-extensions.md "Разрабатывайте собственные расширения в Dynatrace."), а также [классическими метриками для интеграции с AWS](../../ingest-from/amazon-web-services/integrate-with-aws/cloudwatch-metrics.md "Интеграция метрик из Amazon CloudWatch.").

Несмотря на схожие названия, метрики интеграции с AWS **не** основаны на расширениях.

Для просмотра всех метрик, доступных в **вашей** среде, используйте вызов API [GET metrics](../../dynatrace-api/environment-api/metric-v2/get-all-metrics.md "Получите список всех метрик, доступных в вашей среде мониторинга через Metrics v2 API."). Мы рекомендуем следующие параметры запроса:

* `pageSize=500` — чтобы получить максимально возможное количество метрик в одном ответе.
* `fields=displayName,unit,aggregationTypes,dduBillable` — чтобы получить тот же набор полей, что и в этих таблицах.
* В зависимости от того, какие метрики вы хотите запросить, одно из следующих значений для параметра **metricSelector**:

  + `metricSelector=ext:*` — чтобы получить все метрики от расширений.
  + `metricSelector=calc:*` — чтобы получить все вычисляемые метрики.
  + Опустите параметр, чтобы получить **все** метрики вашей среды.

В разделах ниже описаны несоответствия или ограничения, выявленные для встроенных метрик Dynatrace.

Метрики приложений и биллинга для мобильных и пользовательских приложений

Раздел [Метрики других приложений](#other-applications-metrics) содержит метрики, собранные для мобильных и пользовательских приложений. Эти метрики, начинающиеся с `builtin:apps.other`, собираются без указания того, является ли это мобильным или пользовательским приложением. Однако [метрики биллинга приложений](#apps), начинающиеся с `builtin:billing.apps`, разделены для этих типов приложений:

* Мобильные приложения:

  + `builtin:billing.apps.mobile.sessionsWithoutReplayByApplication`
  + `builtin:billing.apps.mobile.sessionsWithReplayByApplication`
  + `builtin:billing.apps.mobile.userActionPropertiesByMobileApplication`
* Пользовательские приложения:

  + `builtin:billing.apps.custom.sessionsWithoutReplayByApplication`
  + `builtin:billing.apps.custom.userActionPropertiesByDeviceApplication`

Метрики биллинга учитывают как оплаченные, так и неоплаченные сессии

Следующие метрики «биллинга» для подсчёта сессий фактически представляют собой сумму оплаченных **и неоплаченных** пользовательских сессий.

* `builtin:billing.apps.custom.sessionsWithoutReplayByApplication`
* `builtin:billing.apps.mobile.sessionsWithReplayByApplication`
* `builtin:billing.apps.mobile.sessionsWithoutReplayByApplication`
* `builtin:billing.apps.web.sessionsWithReplayByApplication`
* `builtin:billing.apps.web.sessionsWithoutReplayByApplication`

Если вы хотите получить только количество оплаченных сессий, установите фильтр **Type** на **Billed**.

Различные единицы измерения для метрик длительности запросов

Для аналогичных метрик длительности запросов для мобильных и пользовательских приложений используются различные единицы измерения.

`builtin:apps.other.keyUserActions.requestDuration.os` измеряется в микросекундах, тогда как другие метрики длительности запросов (`builtin:apps.other.requestTimes.osAndVersion` и `builtin:apps.other.requestTimes.osAndProvider`) измеряются в миллисекундах.

Пользовательские и встроенные метрики

Пользовательские метрики определяются или устанавливаются пользователем, тогда как встроенные метрики по умолчанию являются частью продукта. Некоторые встроенные метрики отключены по умолчанию и при включении будут потреблять [DDU](../../license/monitoring-consumption-classic/davis-data-units.md "Узнайте, как рассчитывается потребление мониторинга Dynatrace на основе Davis data units (DDU)."). Эти метрики охватывают широкий спектр поддерживаемых технологий, включая Apache Tomcat, NGINX, Couchbase, RabbitMQ, Cassandra, Jetty и многие другие.

Пользовательская метрика — это новый тип метрики с пользовательским идентификатором и единицей измерения. Семантика пользовательских метрик определяется вами и не включена в стандартную установку OneAgent. Пользовательские метрики отправляются в Dynatrace через [различные интерфейсы](../../ingest-from/extend-dynatrace/extend-metrics.md "Узнайте, как расширить наблюдаемость метрик в Dynatrace.").

Например, если вы определите новую пользовательскую метрику `Files count`, которая подсчитывает вновь созданные файлы в каталоге, эта метрика может быть собрана для одного или двух отдельных хостов. Сбор одной метрики для двух хостов приводит к двум временным рядам одного типа пользовательской метрики.

![Custom metrics](https://dt-cdn.net/images/custommetrics2-1329-59422c6592.png)

Для целей [расчёта потребления мониторинга](../../license/monitoring-consumption-classic/davis-data-units/metric-cost-calculation.md "Узнайте, как рассчитать потребление Davis data unit и затраты.") сбор одной пользовательской метрики для двух хостов считается как две отдельные пользовательские метрики.
