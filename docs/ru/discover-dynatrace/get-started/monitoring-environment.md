# Что такое среда мониторинга?


* Latest Dynatrace
* 4-min read
* Published Dec 05, 2017

Среда мониторинга Dynatrace — это место, где выполняется весь анализ производительности в Dynatrace. [Dynatrace OneAgent](../../platform/oneagent.md "Узнайте о возможностях мониторинга OneAgent.") отправляет все собранные данные мониторинга в вашу среду мониторинга для анализа. Среда мониторинга аналогична серверу анализа, который предоставляет все функциональные возможности анализа производительности приложений Dynatrace, включая все панели управления, графики, отчёты и другие инструменты.

![Dynatrace architecture](https://dt-cdn.net/images/dynatrace-architecture-1743-fab92236e8.png)

## Расположение среды мониторинга

Расположение вашей среды мониторинга зависит от типа развёртывания.

| Тип развёртывания | Расположение среды мониторинга |
| --- | --- |
| Standard SaaS | В облаке Dynatrace |
| On-premises | В вашем собственном центре обработки данных |
| Dynatrace for Government | В облаке с авторизацией FedRAMP moderate |

## Идентификатор среды

Весь внешний доступ к вашей среде мониторинга Dynatrace основан на двух типах учётных данных: *идентификаторе среды* и [токене доступа](../../manage/identity-access-management/access-tokens-and-oauth-clients/access-tokens.md "Узнайте о концепции токена доступа и его областях видимости.").

Каждая среда, которую вы отслеживаете с помощью Dynatrace, идентифицируется уникальной строкой символов — идентификатором среды. API Dynatrace активно использует идентификаторы среды, чтобы обеспечить извлечение данных мониторинга из правильных сред Dynatrace и отправку соответствующих внешних событий в нужные среды.

* В Dynatrace SaaS идентификатор вашей среды является первой частью URL-адреса вашей среды Dynatrace:

  ```
  https://{your-environment-id}.live.dynatrace.com/
  ```

  Например, для среды Dynatrace `https://abc123a.dynatrace.com` идентификатором среды является `abc123a`.

  Идентификатор среды также можно найти в [Account Management](../../manage/account-management.md "Управляйте лицензией Dynatrace, учётными записями, внедрением платформы и состоянием среды.").

  1. Перейдите в [**Account Management**](https://myaccount.dynatrace.com/). Если у вас несколько учётных записей, выберите учётную запись, которой хотите управлять.
  2. Выберите **Settings** > **Environments**.

     Идентификатор среды можно найти в столбце **Environment**.

Например, для URL-адреса `https://managed-cluster/e/abc123a` идентификатором среды является `abc123a`.

## Несколько сред мониторинга

Вы можете настроить и отслеживать несколько отдельных сред мониторинга Dynatrace с помощью одной учётной записи Dynatrace.

При настройке нескольких сред мониторинга:

* Каждая среда мониторинга использует одну и ту же базу пользователей и другие настройки учётной записи.
* Данные, собранные из каждой среды мониторинга, хранятся отдельно.
* Легко переключаться между представлениями мониторинга для отдельных сред, чтобы отслеживать производительность и доступность каждой среды.

### Сценарии использования

Существует несколько причин, по которым люди создают отдельные среды мониторинга.

Например:

* Вы можете захотеть отслеживать среды разработки, тестирования и производства вашей организации отдельно.
* Вы можете захотеть поддерживать отдельную среду мониторинга для каждого центра обработки данных вашей организации.

### Настройка нескольких сред мониторинга

Способ настройки различных сред мониторинга зависит от того, используете ли вы Dynatrace SaaS, Dynatrace Managed или Dynatrace for Government.

| Тип развёртывания | Как настроить различные среды мониторинга |
| --- | --- |
| SaaS | Свяжитесь с экспертом по продукту Dynatrace через живой чат в вашей среде Dynatrace; эксперт по продукту обеспечит вам необходимую помощь. |
| Managed | Вы можете использовать Cluster Management Console. Для получения подробной информации см. [Managed - managed-monitoring-environment](https://docs.dynatrace.com/managed/shortlink/managed-monitoring-environment). |
| Dynatrace for Government | Обратитесь к вашему представителю Dynatrace. |

### Подключение нескольких сред

При наличии нескольких сред данные мониторинга строго разделены по замыслу. Среды не получают информацию друг от друга автоматически.

Однако в некоторых сценариях может потребоваться соединить ваши среды. Например:

* Для трассировки вызовов между сервисами, отслеживаемыми в разных средах Dynatrace, можно настроить трассировку между средами. Для получения подробной информации см. [Настройка трассировки между средами](../../observe/application-observability/distributed-traces/analysis/connect-environments.md "Анализируйте запросы через границы сред.").
* Для отображения метрик из удалённых сред на панелях управления локальной среды можно настроить плитки панелей управления для нескольких сред. Для получения подробной информации см. [Создание удалённых/многосредовых панелей управления Dynatrace](../../analyze-explore-automate/dashboards-classic/dashboards/dashboards-multi-environment.md "Создавайте панели управления, отображающие данные из нескольких сред Dynatrace.").
