# Поле фильтрации


* Latest Dynatrace
* Reference
* 6-min read
* Updated on Jan 07, 2026

Поле фильтрации — это мощный компонент пользовательского интерфейса, который позволяет быстро сузить результаты и найти нужную информацию в приложениях Dynatrace. Используйте поле фильтрации, когда вам нужно выполнить поиск по определённым атрибутам (например, имя сервиса или статус), объединить несколько условий с помощью логических операторов или отфильтровать JSON-данные с использованием точечной или скобочной нотации.

Для сложных запросов, выходящих за рамки возможностей поля фильтрации, рекомендуется использовать [Dynatrace Query Language (DQL)](../../../platform/grail/dynatrace-query-language.md "How to use Dynatrace Query Language.") в [![Notebooks](https://dt-cdn.net/images/notebooks-768-046137830a.webp "Notebooks") **Notebooks**](../../../analyze-explore-automate/dashboards-and-notebooks/notebooks.md "Analyze, visualize, and share insights from your observability dataâall in one collaborative, customizable workspace.").

## Структура

![Filter field component in the Dynatrace UI.](https://dt-cdn.net/images/ui-filter-field-2018-44c7241dc1.png)

1. **Выражение фильтра**: каждый серый блок представляет собой одно полное выражение критерия фильтрации.
2. **Ключ**: первая часть выражения фильтра — это ключ или категория фильтра.
3. **Оператор сравнения**: показывает тип сравнения.
4. **Значение**: конкретное значение, которое вы ищете.
5. **Логический оператор**: соединяет несколько выражений фильтра. `AND` используется автоматически, если вы не добавите `OR`.

## Как использовать фильтры

* Чтобы создать выражение фильтра, просто начните вводить текст в поле фильтрации. Сразу откроется выпадающий список с подсказками.
* В зависимости от реализации вы можете применить фильтр:

  + Внесением изменений в поле фильтрации (динамическая фильтрация)
  + Нажатием кнопки **Apply** (пакетная фильтрация)
  + С помощью сочетаний клавиш: Windows **Ctrl+Enter** или macOS **Cmd+Enter**
* Чтобы очистить всё поле фильтрации, нажмите `X` в правой части поля.
* Чтобы удалить отдельный фильтр, наведите курсор на выражение фильтра и нажмите `X` для этого выражения.

### Использование подсказок фильтра

Для удобства поле фильтрации предлагает релевантные подсказки по мере ввода.

* Описания в правой части подсказки предоставляют контекст.
* Чтобы применить подсказку, выберите её из списка (щёлкните или коснитесь), либо используйте клавиши со стрелками вверх и вниз для навигации по списку, а затем нажмите **Enter**.

## Общий синтаксис

* Используйте пробел в качестве разделителя между ключами, операторами сравнения, значениями и выражениями фильтра.
* Для чисел используйте десятичную точку `.` в качестве разделителя дробной части.
* Для массивов используйте знак равенства `key = value`.

Примечание

Некоторые функции могут быть недоступны из-за ограничений производительности или источника данных. Например, поле фильтрации в целом поддерживает [подстановочные знаки](#wildcards) (обозначаются звёздочкой `*`) как в начале (заканчивается на), так и в конце (начинается с) значения. Однако некоторые реализации, например основанные на API классического селектора сущностей, могут не поддерживать вариант «заканчивается на».

### Поиск

* Используйте `* ~ searchterm` для поиска по всем данным.
* Несколько поисковых запросов можно комбинировать с помощью `AND` или `OR`. Подробности см. в разделе [Логические операторы](#filter-field-logical-operators). В следующем примере результат включает все данные с терминами `paymentError` или `timeout` в любом поле.

  `* ~ paymentError OR * ~ timeout`
* Поисковые запросы можно комбинировать с фильтрами с помощью `AND`. Подробности см. в разделе [Логические операторы](#filter-field-logical-operators). В следующем примере результат включает все данные из `payment-service` с термином `paymentError` в любом поле.

  `service = payment-service AND * ~ paymentError`

### Логические операторы

Поле фильтрации поддерживает логические операторы `AND` и `OR`. Если вы не указываете оператор, выражения фильтра автоматически соединяются неявным `AND`, который является логическим оператором по умолчанию между любыми двумя выражениями.

### Группировка выражений фильтра

Используйте круглые скобки `()` для логической группировки выражений фильтра. Например, `key = value OR (key2 = value key3 = value)`.

### Экранирование значений

Пробел, `*`, `,`, `(`, `)`, `!`, `<`, `>`, `=`, `"`, `$`, `:`, `[`, `]`, `\` и `~` являются специальными символами в поле фильтрации. Чтобы использовать специальные символы как часть ключа или значения, их необходимо экранировать.

### Чувствительность к регистру

Строковые значения не чувствительны к регистру для всех операторов.

### Подстановочные знаки

Подстановочный знак соответствует любому символу в значении. Подстановочные знаки обозначаются звёздочкой `*`.

Если значение содержит **специальные символы**, такие как пробелы, значение необходимо экранировать, чтобы оно было правильно интерпретировано. В таких случаях подстановочные знаки можно использовать **снаружи или внутри** экранированного значения. Примеры ниже:

Предположим, вы хотите отфильтровать сервис с именем "Payment Service". Фильтр будет выглядеть следующим образом:

`Name = "Payment Service"`

Это гарантирует, что вся строка "Payment Service" рассматривается как единое значение, включая пробел между "Payment" и "Service".

Если вы хотите использовать подстановочные знаки со значением, содержащим пробелы, вы можете сделать это так:

`Name = *"Payment Service"*`

Или так:

`Name = "*Payment Service*"`

Этот фильтр будет соответствовать любому имени сервиса, содержащему "Payment Service" в любом месте.

Подстановочные знаки не поддерживаются внутри оператора `in ()`. Оператор `in ()` выполняет фильтрацию по точным совпадениям и не интерпретирует `*` как подстановочный знак.
Если вы хотите использовать `*` как литеральный символ, его нужно экранировать с помощью обратной косой черты `\`. Например, чтобы найти строку `/visit*/_settings`, используйте `key = "/visit\*/_settings"`.
Подробности см. в разделе [Экранирование значений](#escaping-values) в выражениях фильтра.

### Переменные

Переменные выступают в качестве заполнителей и позволяют динамически выбирать нужные значения. Для ссылки на переменную начните значение с символа `$`. Например: `k8s.cluster.name = $cluster`.

#### Переменные и подстановочные знаки

Используйте подстановочный знак `*` для фильтрации данных, которые начинаются с, заканчиваются на или содержат значение переменной. Вы можете разместить `*` перед, после или с обеих сторон переменной. Позиция `*` определяет поведение фильтра. Подробности см. в разделе [Подстановочные знаки](#wildcards).

При использовании переменной, например `$cluster`, позиция `*` относительно переменной определяет поведение сопоставления:

* **Заканчивается значением переменной**

  + `k8s.cluster.name = *$cluster` соответствует любому значению, которое заканчивается значением `$cluster`.
  + **Пример:** Если `$cluster` равно *myClusterName*, это соответствует *prod-myClusterName*.
* **Начинается со значения переменной**

  + `k8s.cluster.name = $cluster*` соответствует любому значению, которое начинается со значения `$cluster`.
  + **Пример:** Соответствует *myClusterName-east*.
* **Содержит значение переменной**

  + `k8s.cluster.name = *$cluster*` соответствует любому значению, которое содержит значение `$cluster` в любом месте строки.
  + **Пример:** Соответствует *prod-myClusterName-east*.
