# Стартовые панели

* Последняя версия Dynatrace
* Чтение: 4 мин
* Опубликовано 18 сент. 2024

Полный спектр сценариев использования Dynatrace может показаться сложным, с непростой кривой обучения для понимания того, какие элементы нужны людям в их повседневной работе с наблюдаемостью в Dynatrace.

Чтобы упростить задачу, используйте приложение **Launcher** для создания настраиваемых стартовых панелей (launchpads) — настраиваемых начальных страниц, предназначенных для повышения удобства и эффективности работы. Когда пользователи начинают работу с Dynatrace со стартовой панели, они сосредоточены на том, что хотят видеть в первую очередь, и имеют быструю навигацию к наиболее важным для них компонентам Dynatrace.

* Координация проекта с большой группой людей
* Управление развёртыванием Dynatrace в масштабах всей организации

## Начало работы со стартовыми панелями

1. В приложении **Launcher** выберите **Launchpad**, чтобы добавить новую стартовую панель.
2. Выберите **Customize**.
3. Выберите тип компонента и начните его настройку. Компоненты, которые можно добавить на стартовую панель:

   * Ссылки — коллекция ссылок на приложения и документы Dynatrace, а также общие ссылки на внутренние или внешние ресурсы
   * Markdown — плитка с текстом в формате Markdown, включая ссылки на веб-страницы и изображения
   * Карточки — для каждой карточки можно добавить краткое описание, необязательное изображение и кнопку для перехода к целевому объекту
4. После создания стартовой панели выберите **Done**.

## Домашние стартовые панели

Домашние стартовые панели выступают в роли индивидуальных начальных страниц, обеспечивая персонализированный опыт при работе с Dynatrace, для фокусировки и быстрого доступа к наиболее необходимым элементам.

### Настройка домашних стартовых панелей для групп пользователей

Чтобы настроить домашние стартовые панели для групп пользователей:

1. Перейдите в **Settings**.
2. Выберите **General**.
3. Выберите **Launcher** > **Home launchpad**.
4. Выберите **Add home launchpad**.
5. Выберите стартовую панель из списка.
6. Начните вводить текст, чтобы **Select a user group**, или выберите **Everyone**, чтобы изменить начальную страницу для всех пользователей.
7. Выберите **Save**.

После настройки домашней стартовой панели она будет отмечена как **Home** и загружаться как начальная страница при входе в Dynatrace. Доступ к **Getting started with Dynatrace**, стартовой панели по умолчанию, сохраняется на странице обзора стартовых панелей.

![Список стартовых панелей](https://dt-cdn.net/images/launchpads-list-2208-f213920787.png)

### Настройка персональной домашней стартовой панели

Чтобы настроить персональную домашнюю стартовую панель:

1. Выберите **Browse all**.
   Откроется страница обзора стартовых панелей.
2. Выберите стартовую панель, на которую хотите переключиться.
3. Выберите > **Set as Home**
4. При следующем входе в Dynatrace ваша домашняя стартовая панель станет новой начальной страницей.

![Установить как домашнюю](https://dt-cdn.net/images/set-as-home-2642-cfef19cf7b.png)

Установка домашней стартовой панели для себя переопределит любую домашнюю стартовую панель, установленную администратором. Доступ к домашним стартовым панелям, установленным администратором, сохранится в разделе **Suggested** на странице обзора стартовых панелей.

Приоритет домашней стартовой панели

1. Ваша персональная домашняя стартовая панель, установленная как **Home**
2. Домашняя стартовая панель, установленная администратором, с наивысшим рангом
3. Другие домашние стартовые панели, установленные администратором, с более низким рангом и другими группами, в которых вы состоите
4. **Getting started with Dynatrace**, домашняя стартовая панель по умолчанию

Если домашняя стартовая панель не загружается (например, из-за проблем с правами доступа), откроется следующая по приоритету.

## Общие действия для стартовых панелей

Выберите значок меню на стартовой панели или на странице обзора. Оттуда вы можете:

* **Rename** — переименовать стартовую панель.
* **Make a copy** — начать создание новой стартовой панели на основе существующей.
* **Delete** — удалить стартовую панель.
* **Download** — скачать стартовую панель, чтобы перенести её между средами. См. [Скачивание и загрузка стартовой панели](#move).
* **Add to Dock** — добавить в панель быстрого доступа.

## Настройка стартовой панели

Чтобы настроить стартовую панель, откройте её и выберите **Customize**.

* Наведите курсор на компонент, который хотите настроить, и выберите > **Edit**. Также можно просто выбрать компонент в режиме **Customize**.
* Наведите курсор на компонент и выберите значок перемещения, чтобы перетащить компонент на новую позицию.

![Перемещение компонента стартовой панели](https://dt-cdn.net/images/drag-1888-0f03edd34a.png)

### Добавление значка стартовой панели

Добавление значка поможет вашей команде понять, на какой стартовой панели они находятся, и облегчит выбор нужной панели из списка.

![Загрузка значка стартовой панели](https://dt-cdn.net/images/launchpads-icon-upload-2442-75e4e4057e.png)

Чтобы добавить значок стартовой панели:

1. Выберите **Customize**
2. Выберите **Upload icon**
3. Выберите изображение для использования в качестве значка стартовой панели.
4. Выберите **Save**

Ограничения размера файлов

* 1 МБ для изображений JPG, PNG и WEBP
* 10 КБ для изображений SVG

## Скачивание и загрузка стартовой панели

Чтобы перенести стартовую панель между средами:

1. Выберите > **Download**, чтобы скачать стартовую панель.
2. Перейдите в целевую среду, выберите **Launchpads** > **Upload** и загрузите JSON-файл, скачанный на предыдущем шаге.

## Общий доступ к стартовым панелям

Если вы являетесь владельцем стартовой панели, вы можете поделиться ею с другими пользователями в вашей среде Dynatrace.

![Общий доступ к стартовой панели](https://dt-cdn.net/images/share-2226-f40006e092.png)

Подробнее о предоставлении общего доступа к документам Dynatrace (включая стартовые панели) см. в разделе [Общий доступ к документам](share.md "Предоставьте общий доступ к документам Dynatrace (дашбордам, ноутбукам и стартовым панелям) другим пользователям Dynatrace в вашей компании.").

## API для стартовых панелей

Приложение **Launcher** использует API [сервиса документов](https://dt-url.net/x403ua9) для сохранения стартовых панелей как документов.

Документы не зависят от содержимого, но имеют общие метаданные: уникальный идентификатор, имя и описание.

Чтобы различать разные типы документов и делать их видимыми для соответствующих приложений, необходим правильный атрибут документа `type`, в данном случае: `launchpad`. Этот атрибут также можно использовать для запросов к API, как показано в следующем примере.

```
https://environment/platform/document/v1/documents?filter=type='launchpad'
```

### Доступ к полной документации сервиса документов

Чтобы просмотреть полную документацию API сервиса документов:

1. Перейдите на страницу [Document service](https://dt-url.net/x403ua9) сайта [Dynatrace Developer](https://developer.dynatrace.com/).
2. В разделе **Related links** выберите ссылку **Swagger API**.

   Возможно, потребуется войти в вашу среду Dynatrace.
