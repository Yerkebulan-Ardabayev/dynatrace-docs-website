* Последняя Dynatrace
* 3 мин. чтения

Для корректной работы компонентов Dynatrace Operator в кластере Kubernetes они должны иметь возможность взаимодействовать как с кластером Dynatrace, так и с кластером Kubernetes.

Компоненты Dynatrace Operator доступны через определённые порты и обращаются к различным ресурсам внутри и за пределами кластера Kubernetes. Для получения подробной информации о том, к каким ресурсам осуществляется доступ внутри кластера Kubernetes, см. справочную страницу RBAC-разрешения оператора.

## Входящий трафик

1

[Проверки жизнеспособности (liveness probes)](https://dt-url.net/dh03q2c) используются Kubernetes для проверки корректной работы контейнера. При неудачном запросе контейнер будет перезапущен. [Проверки готовности (readiness probes)](https://dt-url.net/ml23qbl) используются Kubernetes для проверки готовности пода принимать трафик.

2

[Конечные точки метрик](https://dt-url.net/t543q6q) предоставляют дополнительные метрики в формате Prometheus.

Для EdgeConnect и OneAgent входящий трафик не принимается.

## Исходящий трафик

Компоненты Dynatrace Operator должны обращаться как к кластеру Kubernetes, так и к ресурсам за пределами кластера для корректной работы. Все ресурсы в пространстве имён Dynatrace Operator (по умолчанию — `dynatrace`) должны иметь возможность разрешать DNS-запросы.

В зависимости от вашей конфигурации, порт по умолчанию может отличаться от `TCP 443`.
