# Extensions


## Что такое Extensions?

Extensions -- это модульные пакеты, которые определяют, как Dynatrace собирает и структурирует данные телеметрии из внешних источников. Каждое расширение выполняется контроллером Extension Execution Controller (EEC), который работает в вашей среде.

## Как работают Extensions?

Extensions позволяют расширить возможности Dynatrace за пределы встроенных функций мониторинга. Они позволяют собирать данные телеметрии, такие как метрики, события и логи, из внешних технологий.

С помощью Extensions вы можете:

* Получать данные из технологий, которые не поддерживаются изначально.
* Создавать индивидуальную логику мониторинга для вашей среды.
* Визуализировать и анализировать пользовательские данные.

## Зачем использовать Extensions?

Extensions полезны, когда вам необходимо мониторить технологию или сервис, который Dynatrace не обнаруживает автоматически. Они обеспечивают расширенную настройку и наблюдаемость практически для любого источника данных.

### Ключевые преимущества

* Автоматическое распространение на ActiveGate и OneAgent со встроенным механизмом отказоустойчивости.
* Полное моделирование топологии, включая пользовательские сущности и маппинг связей.
* Богатые метаданные метрик для последовательного контекстно-зависимого мониторинга.
* Возможность создания пользовательских расширений, адаптированных под ваши нужды:

  + Декларативный, удобочитаемый формат YAML (для поддерживаемых источников данных программирование не требуется).
  + Программируемые расширения с использованием Python для гибкости.

### Сценарии использования

* Мониторинг на основе SNMP: мониторинг сетевых устройств.
* Мониторинг на основе SQL: мониторинг SQL-баз данных различных производителей.
* Мониторинг на основе Prometheus: мониторинг экспортеров Prometheus.
* Мониторинг на основе WMI: мониторинг устройств Windows.
* Мониторинг на основе JMX: получение данных из JMX MBeans.
