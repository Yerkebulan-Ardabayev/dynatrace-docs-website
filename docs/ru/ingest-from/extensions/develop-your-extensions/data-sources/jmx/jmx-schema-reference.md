# Справочник по источнику данных JMX

* Последняя версия Dynatrace
* Справочник
* 4 мин. чтения
* Обновлено 15 сентября 2025 г.

Это общее описание YAML-файла расширения на основе источника данных JMX, а также способов объявления метрик и измерений, которые вы хотите собирать с помощью расширения.

## Значения метрик

Значение метрики может поступать из различных источников.

Наиболее распространённый источник — числовой атрибут JMX MBean:

```
metrics:



- key: com.example.somekey



value: attribute:ThreadCount
```

Этот код ищет атрибут с именем `ThreadCount`. Возвращаемое значение должно быть числовым (любой подкласс `java.lang.Number`, такой как `Integer`, `Long`, `Double`) или `boolean` (преобразуется в `0` для `false` и `1` для `true`).

JMX позволяет определять атрибуты со сложными нечисловыми типами. Можно извлечь числовое значение из такого нечислового атрибута. Для этого необходимо указать, какие методы или поля следует использовать.

Например:

```
metrics:



- key: com.example.somekey



value:



attribute: SomeNonNumericAttribute



accessor: getSomeNumericValue()
```

Подробное описание синтаксиса см. в разделе [синтаксис accessor](#accessor-syntax) ниже.

Особый случай — использование постоянного значения вместо запроса атрибута:

```
metrics:



- key: com.example.somekey



value: const:1
```

Если `query` соответствует одному MBean, эта метрика всегда будет возвращать значение `1`. Это можно использовать для фиксации самого факта наличия конкретного MBean. Если запрос соответствует нескольким MBean, эта метрика будет возвращать значение, соответствующее количеству совпавших MBean.

## Пользовательские измерения

Каждое пользовательское измерение состоит из постоянного `key` и `value`. Значение может поступать из различных источников.

Простейший случай — задать значение измерения как постоянную строку:

```
dimensions:



- key: k1



value: const:constant_value
```

Это создаст метрику, где измерение `k1` всегда имеет значение `constant_value`.

Значение свойства ключа объектного имени MBean может использоваться в качестве значения измерения:

```
query: java.lang:type=GarbageCollector,name=*



dimensions:



- key: k1



value: property:name
```

Это создаст метрику, где измерение `k1` соответствует значению свойства ключа `name`. Например, для MBean `java.lang:type=GarbageCollector,name=YoungGen` будет создана метрика, где измерение `k1` имеет значение `YoungGen`.

Если в процессе есть 3 разных сборщика мусора, будут созданы метрики с 3 разными значениями измерений, которые можно отображать на графиках независимо.

Атрибут MBean также может использоваться в качестве значения измерения.

```
query: java.lang:type=Compilation



dimensions:



- key: k1



value: attribute:Name
```

Это создаст метрику, где измерение `k1` соответствует значению атрибута `Name`. В настоящее время поддерживаются только неизменяемые атрибуты. Атрибут для конкретного MBean запрашивается только один раз, при первом обнаружении MBean агентом OneAgent.

Аналогично значениям метрик, можно извлечь значение измерения из сложного атрибута с помощью выражения accessor:

```
query: java.lang:type=Compilation



dimensions:



- key: k1



value:



attribute: SomeAttribute



accessor: getName()
```

Этот код ищет атрибут `SomeAttribute`, вызывает для него метод `getName` и использует возвращённое значение в качестве значения измерения `k1`.

## Синтаксис accessor

| Accessor | Описание |
| --- | --- |
| `getSomeNumericValue()` | Вызов метода `getSomeNumericValue` без параметров. |
| `getSomeNumericValue` | Скобки необязательны для методов без параметров. |
| `getA().getB()` | Вызов метода `getA`, затем для возвращённого значения вызов метода `getB`. |
| `getA(1)` | Вызов метода `getA` с целочисленным аргументом `1`. |
| `getA("x")` | Вызов метода `getA` со строковым аргументом `x`. |
| `getA(1, "x")` | Вызов метода `getA` с двумя аргументами. |
| `getA()[1]` | Вызов метода `getA`, затем из возвращённого значения извлечение элемента с индексом 1. |

## Переменные расширения

### Использование переменных расширений для фильтрации MBean

Переменные расширений можно использовать для того, чтобы пользователи расширения могли отслеживать только определённые MBean:

```
vars:



- id: gc_name_filter



displayName: Garbage Collector Name



type: text



jmx:



groups:



- group: jvm



subgroups:



- subgroup: basic



query: java.lang:type=GarbageCollector



queryFilters:



- field: name



filter: var:gc_name_filter



dimensions:



- key: k1



value: property:name



metrics:



- key: com.example.jmx.var



type: count



value: attribute:CollectionTime
```

Этот код создаёт переменную с внутренним именем `gc_name_filter` и отображаемым именем `Garbage Collector Name` в пользовательском интерфейсе. Значение переменной будет использоваться для выбора конкретного MBean. Например, если значение переменной — `YoungGen`, то полный запрос объектного имени будет `java.lang:type=GarbageCollector,name=YoungGen`.

Каждая конфигурация мониторинга может задать конкретное значение для этой переменной. Чтобы несколько конфигураций мониторинга с разными значениями переменных не смешивались в пользовательском интерфейсе, рекомендуется также добавлять измерение для свойства `name`, как показано выше.

### Использование переменных расширений в качестве измерений

Переменные расширений также могут использоваться напрямую в качестве значений пользовательских измерений метрик.

```
vars:



- id: my_variable



displayName: My Variable



type: text



jmx:



groups:



- group: jvm



subgroups:



- subgroup: variable as dimension value



query: "java.lang:type=Threading"



dimensions:



- key: my_dimension



value: var:my_variable



metrics:



- key: com.example.jmx-reference.var-dimension



type: gauge



value: attribute:ThreadCount
```

Этот код ссылается на переменную `my_variable` и добавляет дополнительное измерение к метрике. Значение переменной будет использоваться в качестве содержимого измерения. Например, если значение — `My Value`, добавленное измерение будет `my_dimension="My Value"`.

## Агрегация значений gauge

В приведённом выше примере мы получаем количество потоков из одного MBean. Но в некоторых случаях JMX-запрос может вернуть несколько совпадающих бинов. Сбор результатов предоставит слишком много информации. Вместо этого полезнее вычислить минимальное, максимальное или среднее значение для понимания диапазона.

Именно для этой цели мы предоставляем `gauge_statcounter`, который работает как замена `gauge`. В отличие от обычного `gauge`, который суммирует значения, `gauge_statcounter` включает отдельные значения, такие как:

* `min`: минимальное значение метрики
* `max`: максимальное значение метрики
* `sum`: сумма значений метрики при сборе
* `count`: количество проходов сбора данных

Обратите внимание, что среднее значение можно легко получить, разделив сумму на количество.

```
- query: Catalina:type=Manager,host=*,context=*



dimensions:



- key: host



value: attribute:host



metrics:



- key: metric_activeSessions_1752841036351



value: attribute:activeSessions



type: gauge_statcounter
```
