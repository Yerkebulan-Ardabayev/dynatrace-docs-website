## Специфическое сопоставление Dynatrace

Dynatrace сопоставляет отдельные инструменты OpenTelemetry со следующими типами метрик Dynatrace:

| Инструмент | с темпоральностью | сопоставляется с Dynatrace |
| --- | --- | --- |
| Counter | Delta | Counter |
| Counter | Cumulative | Н/Д |
| Gauge | Н/Д | Gauge |
| Explicit bucket histogram [1](#fn-1-1-def) | Delta | Histogram |
| Exponential histogram [2](#fn-1-2-def) | Delta | Exponential histogram |
| UpDownCounter | Delta | Counter |
| UpDownCounter | Cumulative | Gauge |
| Summary | Н/Д | Н/Д |

1

Гистограммы с явными корзинами поддерживаются начиная с версии Dynatrace 1.300+.

2

Для экспоненциальных гистограмм Dynatrace загружает `min|max|sum|count` гистограммы, но не загружает корзины.

### Загрузка атрибутов

OpenTelemetry поддерживает атрибуты на разных уровнях в запросе метрик OpenTelemetry (то есть на уровне точек данных, областей и ресурсов).
Поскольку на стороне Dynatrace атрибуты сохраняются в развёрнутом виде, могут возникать конфликты имён, если атрибуты на разных уровнях имеют одинаковое имя.

Для разрешения таких конфликтов имён Dynatrace применяет следующий порядок приоритета для выбора загружаемого атрибута:

1. Атрибуты точек данных
2. Атрибуты области
3. Атрибуты ресурса

Например, если атрибут точки данных и атрибут области имеют одинаковое имя, приоритет будет у значения точки данных.
Аналогично, если атрибут области и атрибут ресурса имеют одинаковое имя, Dynatrace загрузит значение атрибута области.

### Темпоральность агрегации

Бэкенд Dynatrace работает исключительно с дельта-значениями и требует соответствующей темпоральности агрегации.
Убедитесь, что ваш экспортёр метрик настроен соответствующим образом, или установите переменную окружения [`OTEL_EXPORTER_OTLP_METRICS_TEMPORALITY_PREFERENCE`](https://opentelemetry.io/docs/specs/otel/metrics/sdk_exporters/otlp/) в значение `DELTA`.

Примеры настройки темпоральности для каждого отдельного языка см. в [пошаговых руководствах по интеграции](../../walkthroughs.md "Learn how to integrate and ingest OpenTelemetry data (traces, metrics, and logs) into Dynatrace.").

### Ключи метрик

* Ключ метрики состоит из секций, разделённых точками (например, `dt.metrics`).
* Ключ метрики может содержать строчные и прописные буквы, цифры, дефисы (`-`) и подчёркивания (`_`).
* Ключ метрики должен начинаться с буквы.
* Ключ метрики не должен содержать нелатинские символы (такие как `a`, `o` и `u`).
* К ключу метрики может автоматически добавляться суффикс в зависимости от полезной нагрузки (например, `.count` для счётчиков и `.gauge` для измерителей).

Если вы используете символы, которые являются недопустимыми согласно приведённым выше правилам, они будут заменены на подчёркивания.
Если ваш ключ метрики не содержит хотя бы одного допустимого символа, точка данных будет отброшена.

### Ключи измерений

* Измерения сопоставимы с атрибутами спанов или ресурсов.
* Ключ измерения может содержать только строчные буквы (не прописные), цифры, дефисы (`-`), точки (`.`) и подчёркивания (`_`).
* Ключ измерения должен начинаться с буквы.
* Ключ измерения не должен содержать нелатинские символы (такие как `a`, `o` и `u`).
* Ключ измерения может быть в формате `key.key-section`.
* Вы можете указать до 50 измерений.
* Если один и тот же ключ измерения указан несколько раз в одной полезной нагрузке, принимается только первое встретившееся значение.

Если вы используете символы, которые являются недопустимыми согласно приведённым выше правилам, они будут заменены на подчёркивания.
Если ваш ключ измерения не содержит хотя бы одного допустимого символа, ключ будет отброшен.

Значения измерений должны передаваться в виде строки, логического значения или целого числа.

Dynatrace не поддерживает нестроковые измерения и будет преобразовывать логические значения и целые числа в строки при загрузке.

Если используется любой другой тип, всё измерение целиком будет отброшено.

### Гистограммы

Для экспоненциальных гистограмм Dynatrace загружает `min|max|sum|count` гистограммы, но не загружает корзины.

Если произойдёт любое из описанного ниже, API загрузки OpenTelemetry вернёт ответ `400` или `200 с частичным успехом`.

* Кумулятивные гистограммы не загружаются (аналогично кумулятивным счётчикам).
* Точки данных гистограммы без суммы не загружаются. Это происходит при записи отрицательных значений.
* Корзины гистограммы не отсортированы.
* Граничные значения корзин гистограммы `NaN` или `Infinite` являются недопустимыми.

API загрузки OpenTelemetry в Dynatrace возвращает HTTP `400` только тогда, когда все метрики в запросе OTLP являются недопустимыми.

### Сводки

Dynatrace не поддерживает метрики типа Summary.

Метрики Summary существуют в протоколе OpenTelemetry (OTLP) только для совместимости с другими форматами.
Приложения, использующие официальные SDK OpenTelemetry, не могут создавать метрики Summary.
