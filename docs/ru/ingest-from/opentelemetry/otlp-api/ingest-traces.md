# Ingest OTLP traces


* Последняя версия Dynatrace
* Справочник
* 1 мин. чтения
* Обновлено 15 июл. 2024 г.

Следующие ограничения применяются к запросам на приём трассировок OpenTelemetry и к принятым спанам.

| Тип | Ограничение | Описание |
| --- | --- | --- |
| Время окончания спана | 60 минут в прошлом | Минимальное значение временной метки окончания спана на момент приёма |
| Время окончания спана | 10 минут в будущем | Максимальное значение временной метки окончания спана на момент приёма |
| Количество атрибутов спана | 128[1](#fn-1-1-def) | Максимальное количество атрибутов на один спан |
| Количество событий спана | 128[1](#fn-1-1-def) | Максимальное количество событий на один спан |
| Количество атрибутов события | 128[1](#fn-1-1-def) | Максимальное количество атрибутов на одно событие спана |
| Количество ссылок спана | 128[1](#fn-1-1-def) | Максимальное количество ссылок на один спан |
| Количество атрибутов ссылки | 128[1](#fn-1-1-def) | Максимальное количество атрибутов на одну ссылку спана |
| Размер запроса | 8 МБ | Максимальный размер OTLP-запроса для приёма трассировок через ActiveGate (несжатые данные) |
| Размер запроса (gzip) | 8 МБ | Максимальный размер OTLP-запроса для приёма трассировок через ActiveGate (сжатые данные) |

1

Типичное ограничение OpenTelemetry SDK. Dynatrace не устанавливает это ограничение.
