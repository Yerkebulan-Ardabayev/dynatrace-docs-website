Запуск ActiveGate на выделенной системе

Для оптимальной производительности и повышенной безопасности рекомендуется устанавливать и запускать ActiveGate на выделенной системе.
Использование ActiveGate на выделенной системе не только минимизирует риск компрометации данных аутентификации ActiveGate, но и снижает возможность злонамеренного изменения конфигурации.

## Требования к оборудованию

Вам нужна машина, выделенная для ActiveGate, со следующими характеристиками:

* 4 ГБ свободного дискового пространства для установки, конфигурации и логов ActiveGate и расширений для целей автоматического обновления.
* 4 ГБ для кэшированных установщиков ActiveGate и OneAgent и образов контейнеров — если их потребуется хранить.
* Пространство для файлов дампов — если их потребуется хранить. Эта функция по умолчанию отключена, но может быть включена в конфигурации ActiveGate. Максимальный размер пространства хранения [настраивается](../../../../observe/application-observability/profiling-and-optimization/memory-dump-analysis/configure-an-activegate-for-memory-dump-storage.md "Узнайте, как включить хранение дампов памяти на ActiveGate.") — 100 ГБ по умолчанию.
* 600 МБ + 1,5 ГБ (буфер) свободного дискового пространства для файла персистентности повторной передачи логов Extension Execution Controller.
* Пространство для загрузок расширений — в зависимости от используемых расширений.
* 2 ГБ оперативной памяти (рекомендуется 4 ГБ).
* 1 двухъядерный процессор.

Для крупных сред может потребоваться машина с дополнительными ресурсами ЦП и памяти.

## Подробности персистентности

Механизм надёжности обеспечивает персистентность логов Extension Execution Controller (EEC) в случае недоступности ActiveGate или OneAgent, проблем с сетью или перегрузки приёма данных EEC. Это минимизирует пробелы в покрытии логов.

### Общая информация

* Для постоянного хранения данных требуется 2136 МБ свободного дискового пространства:

  + 600 МБ свободного дискового пространства для использования механизмом надёжности
  + 1,5 ГБ свободного дискового пространства для использования в качестве буфера
* Требование проверяется периодически, и если оно не выполнено, персистентность будет отключена и приём логов будет осуществляться без механизма надёжности.
* Том используется пропорционально нагрузке приёма логов.
* Если требование не может быть выполнено на хосте, вы можете изменить конфигурацию персистентности логов. Дополнительную информацию см. в разделе [Конфигурация персистентности](#persistence_config).

### Конфигурация

Файл конфигурации Windows: `C:\ProgramData\dynatrace\remotepluginmodule\agent\conf\extensionsuser.conf`

Файл конфигурации Linux: `/var/lib/dynatrace/remotepluginmodule/agent/conf/extensionsuser.conf`

## Системные требования

* ActiveGate поддерживает только операционные системы, работающие на архитектуре x86-64 (64-разрядная Intel/AMD).
* Убедитесь, что у вас правильная [конфигурация сетевых портов](../../supported-connectivity-schemes-for-activegates.md "Узнайте о приоритетах подключения между типами ActiveGate, а также о приоритетах между ActiveGate и OneAgent.").
* Установка ActiveGate не поддерживается на Windows с отключённой учётной записью `NT AUTHORITY\LocalService`.

## Руководство по выбору размера для AWS

В следующей таблице представлены требования к размеру экземпляра машины в зависимости от количества OneAgent, взаимодействующих с ActiveGate. На каждом хосте OneAgent выполняет восемь задач мониторинга:

* Мониторинг инфраструктуры
* Мониторинг логов
* Полностековый мониторинг 3 экземпляров Apache Tomcat
* Полностековый мониторинг 2 экземпляров Apache HTTP Server
* Мониторинг расширений

Реальное количество хостов может отличаться в зависимости от мониторируемых технологий в вашей среде. Рекомендуется, чтобы машина, на которой работает ActiveGate, не превышала 50% загрузки ЦП и 80% памяти. Кроме того, следует учитывать, что ActiveGate могут быть недоступны во время обновлений, перезапусков или кратковременных проблем со связью. Для обеспечения высокой доступности работающие ActiveGate должны иметь возможность принять на себя трафик недоступных ActiveGate.
