Запуск ActiveGate на выделенной системе

Для оптимальной производительности и повышенной безопасности мы рекомендуем устанавливать и запускать ActiveGate на выделенной системе.
Использование ActiveGate на выделенной системе не только минимизирует риск компрометации данных аутентификации ActiveGate, но и снижает возможность вредоносного манипулирования конфигурацией.

Подробную информацию о характеристиках пропускной способности логов на Environmental ActiveGate для API приёма логов см. на странице [Лимиты по умолчанию Log Management and Analytics](../../../../analyze-explore-automate/logs/lma-limits.md "Лимиты по умолчанию для последней версии Dynatrace Log Management and Analytics.").

## Аппаратные требования

Вам необходима машина, выделенная для ActiveGate, которая имеет:

* 4 ГБ свободного дискового пространства для установки, конфигурации и логов ActiveGate и расширений для целей автоматического обновления.
* 4 ГБ для кэшированных установщиков ActiveGate и OneAgent и образов контейнеров — если такие потребуется хранить.
* Пространство для файлов дампов — если такие потребуется хранить. Эта функциональность выключена по умолчанию, но может быть включена в конфигурации ActiveGate. Максимальный размер пространства хранения [настраивается](../../../../observe/application-observability/profiling-and-optimization/memory-dump-analysis/configure-an-activegate-for-memory-dump-storage.md "Узнайте, как включить хранение дампов памяти на ActiveGate.") — по умолчанию 100 ГБ.
* 600 МБ + 1,5 ГБ (буфер) свободного дискового пространства для файла сохранения данных при повторной передаче логов Extension Execution Controller.
* Пространство для загрузки расширений — в зависимости от используемых расширений.
* 2 ГБ оперативной памяти (рекомендуется 4 ГБ).
* 1 двухъядерный процессор.

Для больших сред вам может потребоваться машина с дополнительным процессором и памятью.

## Детали сохранения данных

Механизм надёжности обеспечивает сохранение логов Extension Execution Controller (EEC) в случае недоступности ActiveGate или OneAgent, сетевых проблем или перегрузки приёма данных EEC. Это минимизирует пробелы в покрытии логов.

### Общая информация

* Для сохранения данных на диске требуется 2136 МБ свободного дискового пространства:

  + 600 МБ свободного дискового пространства для использования механизмом надёжности
  + 1,5 ГБ свободного дискового пространства для использования в качестве буфера
* Требование проверяется периодически, и если оно не выполнено, сохранение данных будет отключено и приём логов будет передаваться без механизма надёжности.
* Объём используется пропорционально нагрузке приёма логов.
* Если требование не может быть выполнено на хосте, вы можете изменить конфигурацию сохранения логов. Подробности см. в [Конфигурация сохранения данных](#persistence_config).

### Конфигурация

Файл конфигурации Windows: `C:\ProgramData\dynatrace\remotepluginmodule\agent\conf\extensionsuser.conf`

Файл конфигурации Linux: `/var/lib/dynatrace/remotepluginmodule/agent/conf/extensionsuser.conf`

## Системные требования

* Убедитесь, что у вас правильная [конфигурация сетевых портов](../../supported-connectivity-schemes-for-activegates.md "Узнайте о приоритетах подключения между типами ActiveGate, а также о приоритетах между ActiveGate и OneAgent.").
* Ваша операционная система должна поддерживать не менее 500 000 открытых файлов для пользователя `dtuserag`.
  Чтобы посмотреть системный лимит, выполните следующую команду:

  ```
  [user@host]# cat /proc/sys/fs/file-max
  ```

  Также возможно, что вы столкнулись с проблемой [слишком большого количества открытых файлов в Linux](../../../dynatrace-activegate.md#too-many-open-files-in-linux "Основные концепции, связанные с ActiveGate.").
* Ваша операционная система должна иметь не менее 20 000 процессов, доступных пользователю `dtuserag`.
  Чтобы посмотреть системный лимит, выполните следующую команду:

  ```
  [user@host]# cat /proc/sys/kernel/pid_max
  ```
* Установщик ActiveGate для Linux не поддерживает ACL (списки контроля доступа). Правила ACL могут запрещать доступ к директориям и файлам, созданным установщиком, что приведёт к невозможности запуска ActiveGate. Если вы используете ACL, правила, связанные с директориями установки, определёнными в следующих параметрах, должны быть отключены:

  ```
  INSTALL=



  CONFIG=



  LOG=



  TEMP=



  PACKAGES_DIR=
  ```

## Руководство по определению размеров

В следующей таблице представлены требования к размеру экземпляра машины в зависимости от количества OneAgent, взаимодействующих с ActiveGate. На каждом хосте OneAgent выполняет восемь задач мониторинга:

* Мониторинг инфраструктуры
* Мониторинг логов
* Full-stack мониторинг 3 экземпляров Apache Tomcat
* Full-stack мониторинг 2 экземпляров Apache HTTP Server
* Мониторинг расширений

Реальное количество хостов может отличаться в зависимости от мониторируемых технологий в вашей среде. Рекомендуется, чтобы машина, на которой работает ActiveGate, не превышала 50% загрузки процессора и 80% памяти. Кроме того, необходимо учитывать, что ActiveGate может быть неработоспособен во время обновлений, перезапусков или кратковременных проблем со связью. Для обеспечения высокой доступности работающие ActiveGate должны иметь возможность принять на себя трафик недоступных ActiveGate.
