# Назначения и функциональность ActiveGate


* Последняя версия Dynatrace
* Чтение: 4 мин
* Обновлено 10 мая 2023 г.

ActiveGate может использоваться для трёх различных сценариев, которые мы называем **назначениями**:

* [Маршрутизация трафика OneAgent к Dynatrace, мониторинг облачных окружений или мониторинг удалённых технологий с помощью расширений](capabilities/routing-monitoring-purpose.md "Узнайте о возможностях и применении ActiveGate для маршрутизации и мониторинга.")
* [Запуск Synthetic-мониторов из частного расположения](capabilities/synthetic-purpose.md "ActiveGate, настроенный для синтетического мониторинга внутренних и внешних ресурсов из частных Synthetic-расположений")
* [Установка модуля zRemote для мониторинга z/OS](capabilities/zremote-purpose.md "Узнайте об установке модуля zRemote для мониторинга z/OS.")

Каждое назначение сопровождается отдельным набором функциональных [модулей](configuration/configure-activegate.md#modules "Узнайте, какие свойства ActiveGate можно настроить в зависимости от ваших потребностей и требований."). Смешивать модули разных назначений не следует — такая реконфигурация не поддерживается.

## Функциональность, доступная для ActiveGate с назначением «маршрутизация и мониторинг»

1

API-эндпоинт приёма журналов поддерживается на контейнеризованном ActiveGate в ряде [поддерживаемых вариантов Kubernetes](../technology-support.md#kubernetes "Найдите технические сведения о поддержке Dynatrace для конкретных платформ и фреймворков разработки.") при развёртывании с помощью Dynatrace Operator. Поскольку ActiveGate использует файловые буферы, рекомендуется применять [постоянное хранилище](../setup-on-k8s/guides/deployment-and-configuration/activegate-pvc.md "Настройте постоянное хранилище для контейнеризованного ActiveGate в качестве временного хранилища для принятых данных.") во избежание потери данных при перезапусках или перепланировании.
