# Назначения и функциональность ActiveGate


ActiveGate может использоваться для трёх различных сценариев, которые мы называем **назначениями**:

* Маршрутизация трафика OneAgent к Dynatrace, мониторинг облачных окружений или мониторинг удалённых технологий с помощью расширений
* Запуск Synthetic-мониторов из частного расположения
* Установка модуля zRemote для мониторинга z/OS

Каждое назначение сопровождается отдельным набором функциональных модулей. Смешивать модули разных назначений не следует — такая реконфигурация не поддерживается.

1

API-эндпоинт приёма журналов поддерживается на контейнеризованном ActiveGate в ряде [поддерживаемых вариантов Kubernetes](../technology-support.md#kubernetes "Найдите технические сведения о поддержке Dynatrace для конкретных платформ и фреймворков разработки.") при развёртывании с помощью Dynatrace Operator. Поскольку ActiveGate использует файловые буферы, рекомендуется применять постоянное хранилище во избежание потери данных при перезапусках или перепланировании.
