# Безопасность OneAgent на AIX


* Latest Dynatrace
* Reference
* 3-min read
* Published Nov 12, 2020

Для полной автоматизации мониторинга операционных систем, процессов и сетевых интерфейсов Dynatrace требует привилегированного доступа к операционной системе как во время установки, так и во время работы.

OneAgent проходит обширное тестирование, чтобы обеспечить минимальное влияние на производительность вашей системы и [соответствие высочайшим стандартам безопасности](../../../../../../common/manage/data-privacy-and-security.md "Узнайте, как Dynatrace применяет различные меры безопасности для защиты персональных данных.").

## Разрешения

### Установка

Установщику OneAgent требуются права администратора для:

* Добавления записи для OneAgent в `/etc/rc.shutdown`
* Регистрации службы OneAgent в системном init с помощью команды `mkitab`

### Работа

OneAgent требует права администратора для:

* Доступа к списку открытых сокетов для каждого процесса.
* Доступа к списку библиотек, загруженных для каждого процесса.
* Доступа к имени и пути исполняемого файла для каждого процесса.
* Доступа к параметрам командной строки для каждого процесса.
* Мониторинга сетевого трафика.
* Чтения файлов конфигурации приложений.
* Загрузки расширения ядра `oneagentkmod` для обеспечения автоматического внедрения в процессы.

## Изменения операционной системы

* Расширение ядра `oneagentkmod` загружается при запуске службы OneAgent.

## Изменённые файлы

* В `/etc/rc.shutdown` и `/etc/inittab` добавляется запись для службы `oneagent`.

## Добавленные файлы

### Установка

Установщик OneAgent добавляет в вашу систему следующие файлы:

* Бинарные файлы и файлы конфигурации OneAgent сохраняются в `/opt/dynatrace/oneagent`. Обратите внимание, что расположение можно изменить с помощью параметра [INSTALL\_PATH](customize-oneagent-installation-on-aix.md#installation-path "Узнайте, как использовать установщик AIX с параметрами командной строки.").

### Работа

OneAgent добавляет в вашу систему следующие файлы:

* Временные файлы и конфигурация среды выполнения OneAgent сохраняются в `/var/lib/dynatrace/oneagent/runtime`.
* Постоянная конфигурация OneAgent сохраняется в `/var/lib/dynatrace/oneagent/config`.
* Большие данные среды выполнения, такие как дампы памяти, сохраняются в `/var/lib/dynatrace/oneagent/datastorage`. Обратите внимание, что расположение больших данных среды выполнения можно изменить с помощью параметра [DATA\_STORAGE](customize-oneagent-installation-on-aix.md#data-storage "Узнайте, как использовать установщик AIX с параметрами командной строки.").

## Системные журналы, загружаемые OneAgent

OneAgent загружает определённые системные журналы, чтобы Dynatrace мог диагностировать проблемы, которые могут быть вызваны условиями в вашей среде. Чаще всего такие проблемы связаны с глубоким мониторингом или установками, выполняемыми в виде автоматических обновлений.
Системные журналы, загружаемые OneAgent на AIX:

* `/etc/security/limits`
* `/var/adm/ras/errlog`
* `/var/log/kern`
* `/var/log/syslog`

Отзыв доступа к системным журналам

Чтобы отозвать доступ к системным журналам, используйте команду `oneagentctl` с параметром `--set-system-logs-access-enabled`, установленным в значение `false`.
Для получения дополнительной информации см. [Конфигурация OneAgent через интерфейс командной строки](../../../oneagent-configuration-via-command-line-interface.md "Узнайте, как выполнять некоторые задачи конфигурации OneAgent без необходимости повторной установки OneAgent.").

## Глобально доступные для записи каталоги

Структура каталогов OneAgent содержит глобально доступные для записи каталоги (права доступа `1777`). Изменение этих прав доступа пользователями не поддерживается.

### Механизм внедрения OneAgent

Такие права доступа на выбранный набор каталогов необходимы для успешного внедрения OneAgent в процессы на отслеживаемых хостах. Когда OneAgent внедряется в процесс, модуль кода, отвечающий за внедрение, работает в контексте исходного внедряемого процесса. Следовательно, пользователям, под которыми выполняются эти процессы, необходимо разрешение на запись в структуру каталогов OneAgent — это и является причиной глобальных прав на запись, которые это обеспечивают.

Аналогично, некоторые файлы журналов требуют глобальных прав на запись (`666`), чтобы приложения, работающие под различными пользователями, могли записывать в них данные.

### Системная безопасность

Нам известно, что глобальные права на чтение и запись в каталогах OneAgent отмечаются эвристиками сканирования безопасности, однако мы можем заверить вас, что они полностью безопасны.

* Мы сводим к минимуму количество глобально доступных для записи каталогов.
* Для всех этих каталогов установлен sticky bit (фактические права доступа `1777`). Только владелец файла, владелец каталога или пользователь root могут изменять файлы в каталоге. Это стандартная практика, делающая права доступа более надёжными. Она также используется для каталога `/tmp`, чтобы предотвратить удаление или перемещение файлов других пользователей обычными пользователями.
