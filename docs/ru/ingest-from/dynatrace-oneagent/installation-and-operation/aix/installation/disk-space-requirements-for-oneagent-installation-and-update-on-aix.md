## Механизм устаревания файлов OneAgent

OneAgent в режиме мониторинга Full-Stack использует встроенный механизм устаревания для поддержания разумного размера файлов OneAgent, включая файлы логов и данные среды выполнения. Для получения дополнительной информации см. [Механизм устаревания файлов OneAgent](../../../oneagent-aging-mechanism.md "Узнайте, как OneAgent удаляет старые файлы для минимизации использования дискового пространства.").

## Пространство, необходимое для обновлений

При расчёте пространства, необходимого для обновлений, учитывается размер файла установщика и размер процесса установки, то есть пространство, необходимое для развёртывания файлов OneAgent.

|  |  |
| --- | --- |
| Размер файла установщика | ~120 МБ |
| Размер процесса установки | ~1,1 ГБ |

Дополнительное пространство, необходимое для обновлений, рассчитывается с 10% запасом:

`(размер файла установщика + размер процесса установки) * 1,1`

### Размер процесса установки

Размер процесса установки рассчитывается следующим образом:

`3 * размер файла установщика + размер установки`

Размер установщика необходимо умножить на 3 для учёта:

* Загруженного файла установщика (`/var/lib/dynatrace/oneagent`)
* Архива, отделённого от скрипта установщика (`/opt/dynatrace/oneagent`)
* Распакованного внешнего TAR-архива (`/opt/dynatrace/oneagent`)

С точки зрения требований к пространству нет реальной разницы между ручной установкой новой версии (при уже установленной старой версии), автоматическим обновлением и обновлениями, инициируемыми перезапуском контейнера OneAgent. Во всех этих случаях процесс установки выполняется одинаково. Различается лишь метод, которым инициируется обновление.

## Подробности хранения данных

Механизм надёжности обеспечивает сохранение логов Extension Execution Controller (EEC) в случае недоступности ActiveGate или OneAgent, проблем с сетью или перегрузки EEC при приёме данных. Это минимизирует пробелы в покрытии логов.

### Общая информация

* Для постоянного хранения данных требуется 2136 МБ свободного дискового пространства:

  + 600 МБ свободного дискового пространства для механизма надёжности
  + 1,5 ГБ свободного дискового пространства для буфера
* Требование проверяется периодически, и если оно не выполнено, хранение будет отключено, а приём логов будет осуществляться без механизма надёжности.
* Объём используется пропорционально нагрузке приёма логов.
* Если требование не может быть выполнено на хосте, вы можете изменить конфигурацию хранения логов. Для получения дополнительной информации см. [Настройка хранения данных](#persistence_config).

### Конфигурация

Файл конфигурации Windows: `C:\ProgramData\dynatrace\remotepluginmodule\agent\conf\extensionsuser.conf`

Файл конфигурации Linux: `/var/lib/dynatrace/remotepluginmodule/agent/conf/extensionsuser.conf`
