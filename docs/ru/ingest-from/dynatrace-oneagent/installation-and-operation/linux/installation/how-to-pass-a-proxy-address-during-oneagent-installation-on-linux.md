# Как передать адрес прокси при установке OneAgent на Linux


* Последняя версия Dynatrace
* Время чтения: 1 мин
* Опубликовано 19 сентября 2018 г.

Установщик OneAgent распознаёт параметры `--set-proxy` (рекомендуется начиная с версии 1.185) или `PROXY`. Значением этих параметров является адрес прокси-сервера. Добавьте номер порта через двоеточие, например `172.1.1.128:8080`. Для прокси с аутентификацией можно указать имя пользователя и пароль следующим образом: `username:password@172.1.1.128:8080`, при этом имя пользователя и пароль должны быть закодированы в формате URL. Также поддерживаются адреса IPv6.

Имена параметров чувствительны к регистру, поэтому используйте `ПРОПИСНЫЕ БУКВЫ` для имён параметров.

## Передача адреса прокси в установщик

Предположим, вы используете сервер openSUSE, скачали установщик OneAgent в каталог `/tmp`, а IP-адрес вашего прокси — `10.1.1.5`. В этом случае начните установку следующим образом:

```
cd /tmp


chmod +x Dynatrace-OneAgent-Linux-0.5.0-20140217-175809.sh


su -c 'Dynatrace-OneAgent-Linux-0.5.0-20140217-175809.sh --set-proxy=10.1.1.5'
```

## Изменение прокси после установки

Если вам необходимо изменить адрес прокси после установки, используйте параметр `--set-proxy` в [интерфейсе командной строки OneAgent](../../../oneagent-configuration-via-command-line-interface.md "Узнайте, как выполнять некоторые задачи настройки OneAgent без необходимости его переустановки.").
