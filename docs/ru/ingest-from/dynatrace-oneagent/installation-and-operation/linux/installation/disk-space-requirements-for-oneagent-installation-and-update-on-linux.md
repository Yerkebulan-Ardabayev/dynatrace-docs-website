# Файлы OneAgent и требования к дисковому пространству в Linux

* Latest Dynatrace
* 4-min read
* Updated on Jun 25, 2025

На этой странице представлена информация о структуре каталогов OneAgent и требованиях к дисковому пространству для полной установки и обновления OneAgent. Обратите внимание, что точные значения могут различаться в зависимости от версии OneAgent.

## Режим Full-stack и Infrastructure Monitoring

Одинаковые требования к дисковому пространству применяются как к режиму Full-stack, так и к режиму Infrastructure Monitoring.

Подробнее см. ## Пространство, необходимое для обновлений

При расчете пространства, необходимого для обновлений, учитывается размер файла установщика и размер процесса установки, то есть пространство, необходимое для развертывания файлов OneAgent.

Приведенные ниже значения являются примерами и должны использоваться как общее руководство. Для точного размера установки см. **Размер установки** в разделе [Каталоги OneAgent и их размеры](#sizes).

|  | **Linux (x86)** | **Linux (ppcle)** | **Linux (s390)** |
| --- | --- | --- | --- |
| Размер файла установщика | ~260 МБ | ~90 МБ | ~80 МБ |
| Размер процесса установки | ~2.1 ГБ | ~770 МБ | ~720 МБ |

Дополнительное пространство для обновлений рассчитывается с использованием 10% запаса безопасности:

`(размер файла установщика + размер процесса установки) * 1.1`

### Размер процесса установки

Размер процесса установки рассчитывается следующим образом:

`3 * размер файла установщика + размер установки`

Размер установщика необходимо умножить на 3, чтобы учесть:

* Загруженный файл установщика (`/var/lib/dynatrace/oneagent`)
* Архив, отделенный от скрипта установщика (`/opt/dynatrace/oneagent`)
* Распакованный внешний TAR (`/opt/dynatrace/oneagent`)

С точки зрения требований к пространству нет реальной разницы между ручной установкой новой версии (когда старая версия уже установлена), автоматическим обновлением и обновлениями, инициируемыми перезапуском контейнера OneAgent. Во всех этих случаях процесс установки выполняется одинаково. Различается лишь метод, которым инициируется обновление.

## Детали хранилища

Механизм надежности обеспечивает сохранность журналов Extension Execution Controller (EEC) в случае недоступности ActiveGate или OneAgent, сетевых проблем или перегрузки EEC при загрузке данных. Это минимизирует пробелы в покрытии журналов.

### Общая информация

* Для постоянного хранения данных требуется 2136 МБ свободного дискового пространства:

  + 600 МБ свободного дискового пространства для использования механизмом надежности
  + 1.5 ГБ свободного дискового пространства для использования в качестве буфера
* Требование проверяется периодически, и если оно не выполнено, хранилище будет отключено, а загрузка журналов будет передаваться без механизма надежности.
* Объем используется пропорционально нагрузке загрузки журналов.
* Если требование не может быть выполнено на хосте, вы можете изменить конфигурацию хранилища журналов. Подробнее см. [Конфигурация хранилища](#persistence_config).

### Конфигурация

Файл конфигурации Windows: `C:\ProgramData\dynatrace\remotepluginmodule\agent\conf\extensionsuser.conf`

Файл конфигурации Linux: `/var/lib/dynatrace/remotepluginmodule/agent/conf/extensionsuser.conf`
