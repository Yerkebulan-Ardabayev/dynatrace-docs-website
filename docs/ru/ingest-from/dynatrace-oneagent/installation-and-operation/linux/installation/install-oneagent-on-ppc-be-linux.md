# Установка OneAgent на PPC BE Linux


* Последняя версия Dynatrace
* Чтение займёт 3 минуты
* Обновлено 22 января 2026 г.

Чтобы установить Dynatrace OneAgent при наличии развёртывания Dynatrace SaaS, перейдите на [Dynatrace.com](https://www.dynatrace.com) и выполните **Вход** с помощью имени пользователя и пароля, полученных от Dynatrace в письме с подтверждением регистрации. Если у вас развёртывание Dynatrace Managed, [откройте Cluster Management Console и выберите среду](https://docs.dynatrace.com/managed/shortlink/managed-monitoring-environment), которую хотите отслеживать. Затем выполните шаги установки, приведённые ниже.

## Требования

* Вам необходимы [разрешения](linux-non-privileged.md "Узнайте, когда Dynatrace OneAgent требует привилегий root на Linux.") для следующих действий:

  + Создание каталога, в который вы хотите установить OneAgent
  + Изменение настроек брандмауэра (необходимо только в том случае, если ваша внутренняя политика маршрутизации может препятствовать соединению программного обеспечения Dynatrace с Интернетом).
  + Перезапуск служб приложений
* Также необходимо проверить [требования к дисковому пространству](../../aix/installation/disk-space-requirements-for-oneagent-installation-and-update-on-aix.md "Узнайте о требованиях к дисковому пространству для установки OneAgent на AIX.").
* Все хосты, которые нужно отслеживать, должны иметь возможность отправлять данные в кластер Dynatrace. В зависимости от типа вашего развёртывания Dynatrace (SaaS или Managed), а также от конфигурации сети и параметров безопасности вы можете либо обеспечить прямой доступ к кластеру Dynatrace, либо [настроить ActiveGate](../../../../dynatrace-activegate.md "Ознакомьтесь с основными концепциями, связанными с ActiveGate.").

* На PPC BE Linux OneAgent поддерживает только Java и Apache/IHS.
* Для установки OneAgent на PPC BE Linux доступ с правами root не требуется.
* OneAgent можно установить в любой каталог.

### Разрешение подключений через брандмауэр

Убедитесь, что настройки брандмауэра разрешают соединение с Dynatrace.
В зависимости от политики брандмауэра может потребоваться явно разрешить определённые исходящие подключения. **Удалённые адреса Dynatrace, которые необходимо добавить в список разрешений, указаны на странице установки OneAgent.**

## Установка

1. В Dynatrace Hub выберите **OneAgent**.
2. Выберите **Настройка** > **Linux**.
3. В списке выберите тип установщика **PowerPC (BE)**. Скопируйте команду из текстового поля **Используйте эту команду на целевом хосте для загрузки OneAgent для Linux PowerPC (BE)**.
4. Войдите на хост PPC BE Linux и выполните команду, скопированную из Dynatrace.

   * Если вы планируете загружать Dynatrace OneAgent непосредственно на сервер, учтите, что устаревшие или отсутствующие библиотеки (например, сертификаты CA или OpenSSL) предотвратят загрузку.
   * Dynatrace использует зашифрованные соединения. OpenSSL необходим для того, чтобы `wget` мог получить доступ к серверу. Вы также можете загрузить ZIP-архив OneAgent, нажав на ссылку **Загрузить установщик OneAgent** в нижней части страницы и сохранив скрипт установки в любое место — это позволяет обойти команду `wget`.
5. В файловой системе создайте папку для установки OneAgent. Например, `/opt/dynatrace/oneagent`.
6. Распакуйте ZIP-архив OneAgent в созданную папку.

   Все отслеживаемые приложения должны иметь возможность читать библиотеку OneAgent. Убедитесь, что разрешения это позволяют.
7. Теперь у вас есть два варианта: отслеживать все приложения на хосте или только одно приложение.

   Все приложения

   Одно приложение

   Чтобы автоматически отслеживать все приложения на хосте, включите компонент `liboneagentproc.so` OneAgent. Он находится в системном каталоге библиотек (`/lib` или `/lib64` в зависимости от архитектуры) по пути `/etc/ld.so.preload`.

   ## Готово!

   Отлично, настройка завершена! Теперь вы можете ознакомиться с новой средой мониторинга.

   Вы можете получить доступ к среде мониторинга в любое время, перейдя на сайт Dynatrace и нажав **Войти** в правом верхнем углу.

   И ещё одно: чтобы отслеживать процессы, их необходимо перезапустить. В любой момент вы можете проверить, какие процессы не отслеживаются и требуют перезапуска. Просто перейдите в **Deployment Status**, переключитесь на вкладку **All hosts** или **Recently connected hosts** и разверните интересующий вас хост.

   Чтобы отслеживать одно приложение, его необходимо сначала перезапустить. Добавьте следующие команды перед командой запуска приложения:

   ```
   DT_HOME=<installation directory>


   export DT_HOME


   LD_PRELOAD=$DT_HOME/agent/<system library>/liboneagentproc.so


   export LD_PRELOAD
   ```

   Где:

   * `<installation directory>` — каталог, в который установлен OneAgent
   * `<system library>` — `/lib` или `/lib64` в зависимости от архитектуры

   ## Готово!

   Отлично, настройка завершена! Теперь вы можете ознакомиться с новой средой мониторинга. Если у вас развёртывание SaaS, вы можете получить доступ к среде мониторинга в любое время, перейдя на [Dynatrace.com](https://www.dynatrace.com) и нажав кнопку **Войти** в правом верхнем углу. Если у вас развёртывание Dynatrace Managed, вы можете [получить доступ к среде мониторинга через Cluster Management Console](https://docs.dynatrace.com/managed/shortlink/managed-monitoring-environment).
