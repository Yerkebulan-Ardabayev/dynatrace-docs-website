OneAgent, установленный в режиме full-stack, имеет встроенный настраиваемый механизм автообновления.

Обзор обновления OneAgent, включая мониторинг обновлений и создание окон обновления, см. в разделе Обновление OneAgent.

## Настройка обновлений OneAgent

Вы можете настроить обновление OneAgent глобально, для группы хостов и для отдельного хоста.

Настройка глобального обновления OneAgent

Когда автоматические обновления включены глобально, при появлении новой версии OneAgent все установленные экземпляры OneAgent — за исключением тех, для которых автообновление отключено на уровне группы хостов или отдельного хоста — автоматически загрузят обновление и обновят свои двоичные файлы и файлы конфигурации.

Настройки автоматического обновления на уровне группы хостов и отдельного хоста переопределяют глобальные настройки.

1. Перейдите в **Settings** > **Updates** > **OneAgent updates**.
2. Выберите один из вариантов обновления:

   * **Automatic updates at earliest convenience**
     Автоматическое обновление всех OneAgent независимо от окон обновления.
   * **Automatic updates during update windows**
     Автоматическое обновление всех OneAgent в выбранном окне обновления.

     + При выборе этого параметра отображается список доступных окон обновления. Выберите одно из них.
     + Для настройки нового окна обновления OneAgent перейдите в **Settings** > **Updates** > **Update windows for OneAgent updates**.
   * **No automatic updates**
     Не обновлять OneAgent автоматически.

Ручной запуск **Update now to target version** обновит все хосты с выбранной комбинацией ОС и архитектуры независимо от их статуса автоматического обновления.

Настройка обновления OneAgent для группы хостов

Настройки обновления OneAgent на уровне группы хостов переопределяют глобальные настройки и переопределяются настройками на уровне отдельного хоста.

1. Откройте страницу **Host group settings**.
   Варианты доступа:

   * Перейдите в **Settings** > **Monitoring** > **Monitoring overview**, найдите любой хост в нужной группе хостов и выберите имя группы хостов (не имя хоста).
   * Перейдите в ![Hosts](https://dt-cdn.net/images/hosts-512-59f5d2dd7f.png "Hosts") **Hosts Classic**, откройте страницу хоста и разверните раздел **Properties and tags**. Если хост принадлежит группе, ссылка на неё отображается под **Host group**.
   * Перейдите в **Deployment Status** > **OneAgents**. Если хост принадлежит группе хостов, ссылка на страницу настроек группы хостов отображается под именем хоста.
2. На странице **Host group settings** выберите **OneAgent updates** в левой части страницы.
3. Выберите один из вариантов обновления:

   * **Inherit global update settings**
     Следовать глобальным настройкам обновления OneAgent для этой группы хостов.

     + Текущая глобальная настройка отображается в скобках в этой строке.
     + Для перехода на страницу глобальной конфигурации выберите ссылку `global`.
   * **Automatic updates at earliest convenience**
     Автоматическое обновление всех OneAgent в этой группе хостов независимо от окон обновления. Глобальные настройки обновления игнорируются.
   * **Automatic updates during update windows**
     Автоматическое обновление всех OneAgent в этой группе хостов в выбранном окне обновления. Глобальные настройки обновления игнорируются.

     + При выборе этого параметра отображается список доступных окон обновления. Выберите одно из них.
     + Для настройки нового окна обновления OneAgent перейдите в **Settings** > **Updates** > **Update windows for OneAgent updates**.
   * **No automatic updates**
     Не обновлять OneAgent в этой группе хостов автоматически. Глобальные настройки обновления игнорируются.

Ручной запуск **Update now to target version** обновит все хосты с выбранной комбинацией ОС и архитектуры независимо от их статуса автоматического обновления.

Настройка обновления OneAgent на уровне хоста

Настройки обновления OneAgent на уровне хоста переопределяют настройки обновления OneAgent на глобальном уровне и на уровне группы хостов.

1. Откройте страницу **Host** для хоста, который вы хотите настроить.
   Варианты доступа:

   * Перейдите в ![Hosts](https://dt-cdn.net/images/hosts-512-59f5d2dd7f.png "Hosts") **Hosts Classic** и выберите хост.
   * Перейдите в **Settings** > **Monitoring** > **Monitoring overview**, выберите вкладку **Hosts**, а затем выберите хост.
2. На странице **Host** откройте меню (**...**) и выберите **Settings**.
3. Выберите **OneAgent updates** в левой части страницы.
4. Выберите один из вариантов обновления:

   * **Inherit...**
     Следовать настройкам обновления группы хостов или глобальным настройкам для этого OneAgent.

     + Если выбранный хост принадлежит группе хостов, текущая настройка группы хостов отображается в скобках в этой строке, а имя группы является ссылкой на страницу настроек группы хостов.
     + Если выбранный хост не принадлежит группе хостов, глобальная настройка отображается в скобках в этой строке, а «global» является ссылкой на страницу глобальной конфигурации (**Settings** > **Updates** > **OneAgent updates**).
   * **Automatic updates at earliest convenience**
     Автоматическое обновление этого OneAgent независимо от окон обновления. Настройки обновления группы хостов игнорируются.
   * **Automatic updates during update windows**
     Автоматическое обновление этого OneAgent в выбранном окне обновления. Глобальные настройки обновления игнорируются.

     + При выборе этого параметра отображается список доступных окон обновления. Выберите одно из них.
     + Для настройки нового окна обновления OneAgent перейдите в **Settings** > **Updates** > **Update windows for OneAgent updates**.
   * **No automatic updates**
     Не обновлять этот OneAgent автоматически. Глобальные настройки обновления игнорируются.

Ручное обновление OneAgent на отдельных хостах

Для ручного обновления OneAgent, работающего на отдельном хосте:

1. Перейдите в **Settings > Monitoring > Monitoring overview**.
2. Выберите вкладку **Hosts**.
3. Для обновления OneAgent или загрузки последней версии выберите **Update** рядом с именем нужного хоста.

   Кнопка **Update** отображается только в том случае, если установленная версия OneAgent на конкретном хосте устарела и это OneAgent в режиме full-stack. Эта кнопка не отображается для PaaS и standalone OneAgent.
4. Выберите **Update now**.
   Если кнопка **Update now** неактивна, у вас нет разрешений на загрузку установщика.

Также можно загрузить последнюю версию установщика OneAgent, вручную скопировать её на целевой хост и выполнить установку непосредственно на целевом хосте.

Выбор версии OneAgent для установки на новых хостах

Чтобы указать, какая версия OneAgent будет автоматически устанавливаться на всех новых хостах:

1. Перейдите в **Settings** > **Updates** > **OneAgent updates**.
2. В разделе **Update mode** выберите **No automatic updates**, чтобы отключить автоматическое обновление OneAgent.

   Подробнее об отключении автоматического обновления OneAgent для PaaS/Kubernetes см. в разделе Параметры DynaKube для Dynatrace Operator на Kubernetes/OpenShift.
3. В разделе **Target version** выберите версию OneAgent для установки на всех новых хостах.

Выбранная версия OneAgent также используется для интеграций PaaS.

Выбор целевой версии OneAgent для обновления

Чтобы выбрать, до какой версии обновить OneAgent:

1. Перейдите в **Settings** > **Updates** > **OneAgent updates**.
2. В списке **Target version** вы можете указать конкретную версию по номеру версии OneAgent или выбрать относительную целевую версию:

   * **Latest stable version**
     Самая последняя стабильная версия OneAgent, доступная в вашей среде. Фактический номер версии отображается в скобках.
   * **Previous stable version**
     Стабильная версия перед **Latest stable version**. Соответствующий номер версии отображается в скобках. Номер версии OneAgent увеличивается на *два* с каждым выпуском, поэтому этот номер будет на *два* меньше номера последней версии.
   * **Older stable version**
     Стабильная версия перед **Previous stable version**. Соответствующий номер версии отображается в скобках. Номер версии OneAgent увеличивается на *два* с каждым выпуском, поэтому этот номер будет на *четыре* меньше номера последней версии.

* **Specific OneAgent version**

Целевая версия используется для:

* Автоматических обновлений
* Автоматических обновлений во время окон обслуживания
* Ручных обновлений при выборе версии для обновления

Вы можете задать целевую версию и режим обновления на уровнях:

* **Уровень среды**
  Влияет на все OneAgent в среде. Также используется для Deployment API.
* **Группа хостов**
  Влияет на все OneAgent в группе хостов. Переопределяет уровень среды. Не влияет на Deployment API.
* **Хост**
  Влияет на OneAgent на этом хосте. Переопределяет конфигурацию на уровне группы хостов и среды. Не влияет на Deployment API.

Если вы выберете более старую версию, чем текущая развёрнутая версия, вы не сможете понизить версию OneAgent. Необходимо будет установить более новую версию поверх существующей версии OneAgent.

## Системные требования

### Дисковое пространство

Подробности см. в разделе Файлы OneAgent и требования к дисковому пространству

### Свободная память

Для обновления OneAgent на хосте требуется 200 МБ свободной памяти.

## Проверка установленной версии OneAgent

Используйте один из следующих способов для проверки текущей установленной версии OneAgent.

### Интерфейс командной строки OneAgent

Запустите `oneagentctl` с параметром `--version`. Подробнее см. в разделе [Настройка OneAgent через интерфейс командной строки](../../../oneagent-configuration-via-command-line-interface.md#display-oneagent-version "Узнайте, как выполнить некоторые задачи настройки OneAgent без необходимости переустановки OneAgent.").

### Обзор хоста

1. Перейдите в ![Hosts](https://dt-cdn.net/images/hosts-512-59f5d2dd7f.png "Hosts") **Hosts Classic**.
2. Нажмите на интересующий вас хост.
3. Разверните **Properties** под именем хоста. Установленная версия OneAgent включена в список свойств.

### Статус развёртывания

1. Перейдите в **Deployment Status**.
2. Нажмите на вкладку **All hosts** или **Recently connected hosts**.
3. Разверните запись интересующего вас хоста. Установленная версия OneAgent включена в отображаемую информацию.
