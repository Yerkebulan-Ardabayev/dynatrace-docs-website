# Файлы OneAgent и требования к дисковому пространству в Windows


На этой странице приведена информация о структуре каталогов OneAgent и требованиях к дисковому пространству для полной установки и обновления OneAgent. Обратите внимание, что точные значения могут варьироваться в зависимости от версии OneAgent.

## Режим Full-stack и Infrastructure Monitoring

Одинаковые требования к дисковому пространству применяются как для режима Full-stack, так и для режима Infrastructure Monitoring.

## Каталоги OneAgent и их размеры

|  |  | Каталог по умолчанию | Можно изменить? |
| --- | --- | --- | --- |
| Размер установки (после удаления временных файлов установки) | ~775 МБ | `%PROGRAMFILES%\dynatrace\oneagent` | Да [1](#fn-1-1-def) |
| Постоянная конфигурация | ~10 МБ | `%PROGRAMDATA%\dynatrace\oneagent\agent\config` | Нет |
| Временные файлы, конфигурация времени выполнения | 200 МБ | `%PROGRAMDATA%\dynatrace\oneagent\agent\runtime` | Нет |
| Логи | 1 ГБ | `%PROGRAMDATA%\dynatrace\oneagent\log` | Да [2](#fn-1-2-def) |
| Отчеты об аварийных остановках, дампы памяти | 3 ГБ | `%PROGRAMDATA%\dynatrace\oneagent\datastorage` | Да [3](#fn-1-3-def) |
| Хранилище аналитики логов | ~1 ГБ [4](#fn-1-4-def) | `%PROGRAMDATA%\dynatrace\oneagent\datastorage\loganalytics` | Да [3](#fn-1-3-def) |
| Файл хранения повторной передачи логов EEC | 600 МБ + 1,5 ГБ буфер | `%PROGRAMDATA%\dynatrace\oneagent\agent\runtime\extensions\persistence` | Да [5](#fn-1-5-def) [6](#fn-1-6-def) |
| Дополнительное пространство для обновлений | ~3,4 ГБ | См. [Пространство, необходимое для установки и обновлений](#updates) |  |
| **Итого** | **~11,5 ГБ** |  |  |

1

Используйте параметр установки [INSTALL\_PATH](customize-oneagent-installation-on-windows.md#installation-path "Узнайте, как использовать установщик OneAgent для Windows.").

2

Используйте параметр установки [LOG\_PATH](customize-oneagent-installation-on-windows.md#log-path "Узнайте, как использовать установщик OneAgent для Windows.").

3

Используйте параметр установки [DATA\_STORAGE](customize-oneagent-installation-on-windows.md#data-storage "Узнайте, как использовать установщик OneAgent для Windows.").

4

Размер зависит от количества полученных логов.

5

## Механизм устаревания файлов OneAgent

OneAgent в режиме мониторинга full-stack использует встроенный механизм устаревания для обеспечения поддержания разумного размера файлов OneAgent, включая файлы логов и данные времени выполнения. Для получения дополнительной информации см. Механизм устаревания файлов OneAgent.

## Пространство, необходимое для обновлений

При расчете пространства, необходимого для обновлений, мы учитываем размер сжатого EXE-установщика, извлеченного MSI-пакета и размер процесса установки (пространство, необходимое для развертывания файлов OneAgent).

|  | Размер | Описание | Удаляется после | Используется | Путь |
| --- | --- | --- | --- | --- | --- |
| 1 | ~130 МБ | EXE-установщик, загруженный AutoUpdate | Перезапуска OneAgent или установки следующей версии [1](#fn-2-1-def) | AutoUpdate | `%PROGRAMDATA%\dynatrace\oneagent` |
| 2 | ~750 МБ | Извлеченный MSI-установщик | Окончания установки | Установка OneAgent | `%PROGRAMDATA%\dynatrace\oneagent` |
| 3 | ~750 МБ | Дополнительное хранилище для временных файлов установки | Окончания установки | Установка OneAgent | `%APPDATA%` |
| 4 | ~750 МБ | Копия MSI-установщика, сохраненная Windows | Установки следующей версии | Установщик Windows | `%WINDIR%\Installer` |
| 5 | ~750 МБ | Установленные файлы | Установки следующей версии | Установка OneAgent | `[Путь установки]` |
| Итого | ~3130 МБ |  |  |  |  |

1

Если вы загрузите EXE-установщик вручную, процесс установки его не удалит.

Дополнительное пространство, необходимое для установки и обновлений, рассчитывается с 10-процентным запасом:

`(размер процесса установки [2+3+4+5] + размер файлов установщика [1]) * 1.1`

Необходимое дисковое пространство для установки и обновления OneAgent на одном диске составляет ~3,4 ГБ, рассчитанное по этой формуле.

Обратите внимание, что EXE-установщик сжат, что объясняет разницу в размере по сравнению с MSI-установщиком.

С точки зрения требований к пространству, нет существенной разницы между ручной установкой новой версии (когда более старая версия уже установлена), автоматическим обновлением и обновлениями, инициированными перезапуском контейнера OneAgent. Во всех этих случаях процесс установки выполняется одинаково. Отличается только способ инициации обновления.

## Детали хранения

Механизм надежности обеспечивает сохранение логов Extension Execution Controller (EEC) в случае, если ActiveGate или OneAgent недоступен, возникают проблемы с сетью или EEC испытывает перегрузку при приеме данных. Это минимизирует пробелы в покрытии логами.

### Общая информация

* Постоянное хранение данных требует 2136 МБ свободного дискового пространства:

  + 600 МБ свободного дискового пространства для использования механизмом надежности
  + 1,5 ГБ свободного дискового пространства в качестве буфера
* Требование проверяется периодически, и если оно не выполнено, хранение будет отключено, а прием логов будет осуществляться без механизма надежности.
* Объем используется пропорционально нагрузке приема логов.
* Если требование не может быть выполнено на хосте, вы можете изменить конфигурацию хранения логов. Для получения дополнительной информации см. [Конфигурация хранения](#persistence_config).

### Конфигурация

Файл конфигурации Windows: `C:\ProgramData\dynatrace\remotepluginmodule\agent\conf\extensionsuser.conf`

Файл конфигурации Linux: `/var/lib/dynatrace/remotepluginmodule/agent/conf/extensionsuser.conf`
