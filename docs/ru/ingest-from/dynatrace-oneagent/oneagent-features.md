При создании окружения OneAgent поставляется с большим набором функций, активированных по умолчанию. Тем не менее, вам всегда необходимо явно активировать функции, добавленные новыми версиями OneAgent, а также функции с *добровольной активацией* (opt-in), такие как автоматическое обогащение записей журналов идентификатором трассировки (trace ID).

Новые функции OneAgent полностью поддерживаются и протестированы сразу после их появления.

* Для новых окружений вы можете активировать или деактивировать функции OneAgent (opt-in) в зависимости от конкретного сценария использования.
* Для существующих окружений вновь добавленные функции OneAgent по-прежнему требуют явной активации пользователями с разрешением **Write settings** (`settings.write`) в таблице схемы Settings API - OneAgent features.

## Область действия

Новые функции OneAgent обычно включаются глобально (для всего окружения). Однако вы можете включить или отключить их в рамках области действия группы процессов. Настройки группы процессов переопределяют глобальные настройки для той же функции OneAgent.

## Сценарии использования

* Поскольку новые функции постоянно добавляются в OneAgent, вам необходимо явно включать их в существующих окружениях мониторинга, чтобы избежать неожиданных изменений.
* Вы можете отключить определённые функции OneAgent на детальном уровне при устранении проблем. Это может быть полезно для определения первопричины проблемы вплоть до конкретной функции. Подробнее см. в разделе Устранение проблем глубокого мониторинга OneAgent.

## Веб-интерфейс или API

* Вы можете включать или отключать функции OneAgent через веб-интерфейс Dynatrace на странице **OneAgent features** на соответствующем уровне (глобальном или группы процессов).
* Вы можете использовать Settings API Dynatrace для выполнения тех же настроек через API-вызовы.

### Настройка через веб-интерфейс

Для включения или отключения функций OneAgent через веб-интерфейс Dynatrace используйте страницу **OneAgent features** на соответствующем уровне.

Каждая функция на странице **OneAgent features** содержит список требований:

* **Min. OneAgent version** указывает минимальную требуемую версию OneAgent. Вы можете включить функцию, только если версия OneAgent соответствует требованию.
* **Requires restart** указывает, требуется ли перезапуск процесса перед тем, как функция начнёт работать.

### Глобальная настройка

Чтобы включить или отключить функцию OneAgent глобально

1. Перейдите в **Settings** и выберите **Preferences** > **OneAgent features**.
2. Найдите функцию в списке и включите или отключите переключатель **Enabled**.

### Настройка группы процессов

Чтобы включить или отключить функцию OneAgent для группы процессов

1. Перейдите в **Smartscape Topology**.
2. Выберите **Processes**.
3. На карте топологии наведите указатель на процесс и выберите значок ссылки, чтобы перейти на страницу деталей процесса. Например:

   ![Ссылка из топологии Smartscape на страницу деталей процесса](https://dt-cdn.net/images/link-to-process-page-377-da0282e435.png)
4. На странице деталей процесса выберите  > **Settings**.
5. На странице **Process group settings** выберите вкладку **OneAgent features**.
6. Выберите **Add override**, чтобы добавить настройку, специфичную для процесса, которая переопределяет настройку окружения.
7. Введите строку поиска в поле **Feature**, чтобы найти и выбрать функцию.
8. Установите переключатели по необходимости для переопределения, а затем выберите **Save changes**.

### Настройка через API

С помощью Settings API вы можете:

* **Включать/отключать** настройки функций OneAgent.
* **Переопределять область действия** функции OneAgent, чтобы задать другую настройку для конкретной группы процессов или процесса.
* **Экспортировать** текущую конфигурацию из окружения.

Для использования API вам потребуется токен доступа с областями **Read settings** (`settings.read`) и **Write settings** (`settings.write`). Чтобы узнать, как его получить, см. [Создание токена доступа](../../dynatrace-api/basics/dynatrace-api-authentication.md#create-token "Узнайте, как пройти аутентификацию для использования Dynatrace API.").

Для использования Settings API

1. Чтобы узнать формат JSON, необходимый для отправки вашей конфигурации, используйте конечную точку Get a schema. Идентификатор схемы — `builtin:oneagent.features`.

   Пример: JSON-полезная нагрузка для отключения обогащения контекста журналов Java для неструктурированных журналов для конкретной группы процессов.

   ```
   [


   {


   "schemaId":  "builtin:oneagent.features",


   "scope": "PROCESS_GROUP-1",


   "value": {


   "enabled": false,


   "key": "JAVA_LOG_ENRICHMENT_UNSTRUCTURED"


   }


   }


   ]
   ```

   Вы также можете изменять настройки, применяемые ко всему окружению, используя область действия environment.

   Пример: JSON-полезная нагрузка для глобального включения поддержки Kafka Streams

   Dynatrace версии 1.244+

   ```
   [


   {


   "schemaId":  "builtin:oneagent.features",


   "scope": "Environment",


   "value": {


   "enabled": true,


   "key": "JAVA_KAFKA_STREAMS"


   }


   }


   ]
   ```
2. Для отправки конфигурации используйте конечную точку Post an object.
