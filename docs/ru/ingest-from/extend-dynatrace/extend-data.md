# Обогащение загруженных данных полями, специфичными для Dynatrace

# Обогащение загруженных данных полями, специфичными для Dynatrace

* Последняя версия Dynatrace
* Чтение: 4 мин
* Опубликовано 26 апр. 2021

В отличие от автоматической загрузки с помощью OneAgent, данные, отправленные напрямую в ActiveGate (например, через API загрузки), не обогащаются автоматически информацией, связанной с хостом. Это может повлечь дополнительные расходы, поскольку не будут учтены возможные включённые [квоты DDU](../../license/monitoring-consumption-classic/davis-data-units.md "Узнайте, как рассчитывается потребление мониторинга Dynatrace на основе единиц данных Davis (DDU).").

Различные варианты развёртывания Dynatrace предоставляют несколько файлов свойств в стиле Java и JSON-файлов с наборами атрибутов, которые вы можете использовать для обогащения ваших запросов к Dynatrace и обеспечения правильного сопоставления данных с вашей инфраструктурой.

## Директория обогащения

Dynatrace использует следующие директории для предоставления файлов `.json` и `.properties` с данными обогащения:

* В Unix: `/var/lib/dynatrace/enrichment`
* В Windows: `%ProgramData%\dynatrace\enrichment`

## Dynatrace OneAgent

Стандартная установка OneAgent предоставляет следующие файлы с информацией о хосте в [директории обогащения](#enrichment-directory):

* `dt_host_metadata.json`
* `dt_host_metadata.properties`

Оба файла содержат одинаковый набор атрибутов ресурсов на уровне хоста, которые OneAgent использует для обогащения артефактов мониторинга для данного хоста. Сюда также входят теги и свойства в формате «ключ-значение», заданные через [oneagentctl](../../observe/infrastructure-observability/hosts/configuration/define-tags-and-metadata-for-hosts.md "Узнайте, как задавать теги и дополнительные свойства для отслеживаемого хоста.") или через [удалённое управление конфигурацией](../bulk-configuration.md "Выполняйте настройку OneAgent и ActiveGate на хостах со страницы статуса развёртывания или массово с помощью Dynatrace API.").

Если вы передаёте телеметрические данные по каналам, отличным от OneAgent (например, с использованием OpenTelemetry), важно вручную обогатить телеметрические данные этими атрибутами для обеспечения правильной привязки к хосту.

Для получения дополнительной информации см. [Атрибуты ресурсов на уровне хоста](../../platform/oneagent/resource-attributes.md#host-level-resource-attributes "Любой сигнал, использующий данный ресурс, такой как хост или группа процессов, обогащается определёнными атрибутами, поступающими от ресурса.").

Пример загрузки JSON-файла см. в [примере на Python](#python-example) ниже.

## Dynatrace Kubernetes Operator

Следующие файлы доступны в [директории обогащения](#enrichment-directory) при включённом обогащении метаданными.

* `dt_metadata.json`
* `dt_metadata.properties`

Оба файла содержат одинаковые данные в разных форматах. Пример загрузки JSON-файла см. в [примере на Python](#python-example) ниже.

Инструкции по настройке см. в руководстве [Настройка директории обогащения](../setup-on-k8s/guides/metadata-automation/metadata-enrichment.md "Обогащение метаданными в Dynatrace Operator добавляет контекст к подам Kubernetes, прикрепляя соответствующие метаданные к сущностям, таким как поды, хосты и процессы, для улучшения наблюдаемости.").

Чтобы узнать, как обогатить все [телеметрические данные, поступающие из рабочих нагрузок Kubernetes, метками и аннотациями Kubernetes](../setup-on-k8s/guides/metadata-automation/k8s-metadata-telemetry-enrichment.md "Руководства по обогащению телеметрии в Kubernetes").

## Виртуальные файлы OneAgent

Когда OneAgent осуществляет мониторинг вашего приложения, ваше приложение также может получить доступ к следующим виртуальным файлам:

* `dt_metadata_e617c525669e072eebe3d0f08212e8f2.json`
* `dt_metadata_e617c525669e072eebe3d0f08212e8f2.properties`

Эти файлы не относятся к директории обогащения и физически не существуют в вашей файловой системе, а предоставляются инструментированием OneAgent. Оба файла возвращают одинаковые данные, и расширение файла (`.json`/`.properties`) определяет только формат вывода.

В контексте [Kubernetes Operator](#dynatrace-kubernetes-operator) виртуальный файл также содержит атрибуты файлов `dt_metadata.{json,properties}`.

### Как получить доступ к виртуальным файлам

1. Используйте стандартную функцию чтения файлов вашей языковой платформы для открытия и чтения одного из этих файлов:

   * `dt_metadata_e617c525669e072eebe3d0f08212e8f2.json`
   * `dt_metadata_e617c525669e072eebe3d0f08212e8f2.properties`

   Выбранное расширение файла имеет значение для шага 4. Используйте только имя файла и не указывайте дополнительный путь.
2. Содержимое файла — это одна текстовая строка с абсолютным путём к файлу (считайте его эфемерным значением и не сохраняйте, не храните и не кэшируйте этот путь для дальнейшего использования).
3. Откройте файл по пути, полученному на предыдущем шаге, и прочитайте всё его содержимое.
4. Формат содержимого, полученного при предыдущем чтении, будет соответствовать расширению файла, выбранному на шаге 1 (свойства в стиле Java или JSON).

Если шаг 1 возвращает ошибку «файл не найден», убедитесь, что ваше приложение инструментировано OneAgent.

### Ограничения

* Поддерживается для процессов с полным мониторингом (full-stack) и глубоким мониторингом только приложений (application-only).
* Проверки `stat` и другие проверки `if (exists)` для этих файлов не работают. Эти проверки не требуются для работы механизма.
* Системные вызовы (`syscalls`), используемые напрямую для доступа к файлам, не поддерживаются. Это также означает, что приложения на Go, используемые для загрузки метрик, не поддерживаются, если вы не используете OneAgent SDK, как описано в руководстве [Инструментирование приложения на Go с помощью OpenTelemetry](../opentelemetry/walkthroughs/go.md "Узнайте, как инструментировать приложение на Go с использованием OpenTelemetry и Dynatrace.").

## Пример на Python

Следующий пример показывает, как загрузить информацию обогащения в формате JSON на Python в Unix.

```
# Initialize dictionary variable



enrich_attrs = dict()



# Iterate over the potential data files and try reading them



for name in ["dt_metadata_e617c525669e072eebe3d0f08212e8f2.json", "/var/lib/dynatrace/enrichment/dt_metadata.json", "/var/lib/dynatrace/enrichment/dt_host_metadata.json"]:



try:



data = ''



with open(name) as f:



data = json.load(f if name.startswith("/var") else open(f.read()))



enrich_attrs.update(data)



except:



pass # An exception indicates the file was not available



# Use enrich_attrs here to enrich your requests to Dynatrace.



# For example, when instrumenting with OpenTelemetry, add the data as resource attributes.
```

Пример кода инициализирует пустой словарь для импортированных атрибутов. Затем он выполняет итерацию по массиву имён файлов `.json` и загружает содержимое каждого файла как JSON-документ, добавляя ключи в словарь. Исключения при работе с файлами указывают на то, что конкретный файл недоступен, и игнорируются.
