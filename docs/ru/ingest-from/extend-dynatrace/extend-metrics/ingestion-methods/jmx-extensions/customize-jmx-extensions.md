# Настройка расширений JMX

* Latest Dynatrace
* Чтение: 10 мин
* Опубликовано 16 февраля 2020 г.

Расширения JMX определяются JSON-файлами.

Определение расширения состоит из трёх основных элементов: `metadata`, `metrics` и `UI`.

```
The basic format is as follows:



{



"version": "1.0",



"name": "custom.jmx.hornetq",



"type": "JMX",



"entity": "PROCESS_GROUP_INSTANCE",



"metricGroup": "tech.HornetQ",



"configUI" : {



"displayName": "HornetQ JMX"



},



"metrics": [ ],



"ui": {



"keycharts" : [ ],



"charts": [ ]



}



}
```

## Метаданные

Каждое расширение JMX имеет следующие обязательные свойства:

| Поле | Тип | Описание |
| --- | --- | --- |
| `version` | String | Версия расширения в формате `d.dd`, должна обновляться при каждом обновлении определения расширения |
| `name` | String | Уникальное имя расширения в формате Java-пакета. Имена пользовательских расширений JMX должны следовать правилу `custom.jmx.name`. Допускаются только буквы, цифры и символы "-", "\_". Например, `custom.jmx.newPlugin-Ver2` |
| `type` | String | Всегда используйте `JMX` |
| `entity` | String | Всегда используйте `PROCESS_GROUP_INSTANCE` |
| `metricGroup` | String | Группа метрик используется для группировки пользовательских метрик в иерархическое пространство имён, куда могут вносить вклад различные источники, например несколько расширений. Кроме того, группа метрик становится основной частью ключа метрики. Поэтому после определения она не может быть изменена. Допустимые символы: буквы, цифры и символы "-", "\_". |
| `configUI.displayName` | String | Удобочитаемое имя расширения. Это имя отображается на странице **Monitoring extensions** в Dynatrace после загрузки расширения. |

## Метрики

Эта часть JSON определяет, какие метрики собираются расширением. Каждая метрика определяется JSON в формате, аналогичном следующему:

```
{



"timeseries": {



"key": "Queue.ConsumerCount",



"unit": "Count",



"displayname": "Queue Consumer Count",



"dimensions": [



"rx_pid"



]



},



"alert_settings": [



{



"alert_id": "too_many_consumers",



"event_type": "PERFORMANCE_EVENT",



"event_name": "Too many consumers",



"description": "The {metricname} of {severity} is {alert_condition} the threshold of {threshold}",



"threshold": 35.0,



"alert_condition": "ABOVE",



"samples":5,



"violating_samples":3,



"dealerting_samples":5,



"value_extractor": "MAX"



}



],



"source": {



"domain": "org.hornetq",



"keyProperties": {



"type": "Queue"



},



"attribute": "ConsumerCount"



}



}
```

### Временные ряды

Эта часть определяет метаданные метрики.

| Поле | Тип | Описание |
| --- | --- | --- |
| `key` | String | Имя метрики. Должно быть уникальным в пределах данного расширения. Допускаются только буквы, цифры и символы "-", "\_". |
| `unit` | String | Единица измерения метрики. Должна быть одной из доступных единиц, описанных ниже |
| `dimensions` | Массив строк | Должен содержать `rx_pid` по индексу `0`. Это обеспечивает привязку JMX-атрибутов к системному идентификатору процесса (PID) как измерению. Дополнительные измерения можно использовать, например, для предоставления одной метрики на каждое значение ключевого свойства JMX `ObjectName` (например, `QueueName`, `ThreadPoolName` или `ConnectionPoolName`). Допускаются только буквы, цифры и символы "-", "\_". |
| `displayname` | String | Отображаемое имя метрики в Dynatrace. Это поле обязательно. Оно должно отличаться от ключа метрики. |

Доступные единицы измерения:
`NanoSecond`, `MicroSecond`, `MilliSecond`, `Second`, `Byte`, `KiloByte`, `MegaByte`, `BytePerSecond`, `BytePerMinute`, `KiloBytePerSecond`, `KiloBytePerMinute`, `MegaBytePerSecond`, `MegaBytePerMinute`, `Count`, `PerSecond`, `PerMinute`

### Настройки оповещений

Эта часть определяет конфигурацию одного или нескольких оповещений для заданного временного ряда.

| Поле | Тип | Описание |
| --- | --- | --- |
| `alert_id` | String | Уникальный идентификатор оповещения. Допускаются только буквы, цифры и символы "-", "\_". |
| `event_type` | String | Допустимые типы: `PERFORMANCE_EVENT`, `ERROR_EVENT`, `AVAILABILITY_EVENT`. |
| `description` | String | Описание определяет сообщение оповещения; могут использоваться следующие фрагменты кода: `{threshold}` — значение пользовательского порога, которое было нарушено, `{severity}` — нарушающее значение, `{entityname}` — отображаемое имя сущности, где метрика была нарушена, `{violating_samples}` — количество нарушающих выборок, приведших к событию, `{dimensions}` — строка с нарушающими измерениями метрики, `{alert_condition}` — строка, показывающая, было ли нарушение выше или ниже порога |
| `event_name` | String | Имя события, отображаемое на страницах интерфейса. |
| `threshold` | Float | Значение порога. |
| `alert_condition` | String | ABOVE или BELOW. |
| `samples` | Integer | Размер «окна», в котором подсчитываются violating\_samples. |
| `violating_samples` | Integer | Количество нарушающих выборок, вызывающих оповещение. |
| `dealerting_samples` | Integer | Количество не нарушающих выборок, деактивирующих оповещение. |
| `value_extractor` | String | Dynatrace собирает значение каждые 10 секунд, но отправляет только одно агрегированное значение в минуту. Этот параметр определяет, как агрегировать эти 10-секундные значения. Возможные значения: `MIN`, `MAX`, `SUM`, `COUNT`, `AVG`, `MEDIAN`, `P90`. Значение по умолчанию — `AVG` |

### Источник

Эта часть определяет, как метрика собирается с помощью JMX. Следующие атрибуты обязательны для всех метрик:

| Поле | Тип | Описание |
| --- | --- | --- |
| `domain` | String | Доменное имя MBean |
| `keyProperties` | Пары ключ-значение | Ключевые свойства MBean. Значения могут содержать подстановочные символы (`*`) |
| `attribute` | String | Имя атрибута, содержащего значение метрики. |

Необязательные атрибуты:

| Поле | Тип | Описание |
| --- | --- | --- |
| `attributePath` | String | См. [CompositeData](#compositedata) |
| `allowAdditionalKeys` | Boolean | Если false, ключевые свойства должны совпадать точно. Дополнительные ключи в имени приведут к несоответствию. Если true, дополнительные ключевые свойства помимо указанных в "keyProperties" допускаются и игнорируются. |
| `calculateDelta` | bool | Если true, вычисляется изменение значений заданного атрибута. Значение = attribute(t) - attribute(t-1). Полезно для монотонно возрастающих значений. |
| `calculateRate` | bool | Если true, вычисляется скорость изменения в секунду. Используется совместно с calculateDelta для преобразования абсолютного атрибута (например, количество запросов) в скорость (например, запросов в секунду). Значение = attribute / интервал запроса |
| `aggregation` | String | Используется для агрегации нескольких значений, если более одного MBean соответствует фильтру домена и ключевых свойств. Возможные значения: SUM, AVG, MIN, MAX |
| `splitting` | Object | Задать [разделение](#splitting) |

#### Разделение

Разделения могут использоваться для определения дополнительного измерения метрики. Это измерение должно быть определено в свойстве `dimension` временного ряда и в свойстве `splitting` источника.

```
"splitting": {



"name": "name",



"type": "keyProperty",



"keyProperty": "name"



}
```

Для определения нескольких разделений используйте следующий формат:

```
"splittings":[



{



"name":"name",



"type":"keyProperty",



"keyProperty":"name"



},



{



"name":"context",



"type":"keyProperty",



"keyProperty":"context"



}



]
```

Для каждого разделения необходимы следующие атрибуты:

| Поле | Тип | Описание |
| --- | --- | --- |
| `name` | String | Должно совпадать с именем измерения, определённым для временного ряда |
| `type` | String | Всегда должно быть `keyProperty` |
| `keyProperty` | String | Определяет, какое ключевое свойство `ObjectName` MBean используется для разделения. |

#### Пример метрики с дополнительным разделением

Следующий пример показывает, как определить метрику, предоставляющую несколько временных рядов в рамках одной метрики:

```
{



"timeseries": {



"key": "XY.Size",



"unit": "Count",



"displayname": "Queue Consumer Count",



"dimensions": [



"rx_pid",



"name"



]



}



"source": {



"domain": "com.sample",



"keyProperties": {



"type": "XY",



"name": "*"



},



"attribute": "Size",



"splitting": {



"name": "name",



"type": "keyProperty",



"keyProperty": "name"



}



}



}
```

Например, `MBeans com.sample:type=XY,name=A и com.sample:type=XY,name=B` приведут к двум временным рядам: первый для "A" и второй для "B".

#### CompositeData

Для извлечения значений отдельных ключей, возвращаемых атрибутом как тип `CompositeData`, необходимо использовать механизм `attributePath` и указать на нужный ключ.

Например, допустим, вы хотите извлечь значение `used` из атрибута `HeapMemoryUsage`. `HeapMemoryUsage` — это тип `CompositeData`, возвращающий следующий список пар значение-ключ:

```
{



committed: integer,



init: integer,



max: integer,



used: integer



}
```

Определите метрику, указывающую на MBean с атрибутом `HeapMemoryUsage`, и в разделе `source` укажите `attributePath` на ключ `used`. Например:

```
"source": {



"domain": "java.lang",



"keyProperties": {



"type": "Memory",



},



"attribute": "HeapMemoryUsage",



"attributePath": "get(\"used\")"



}
```

## Конфигурация интерфейса

Эта часть JSON определяет, как метрики отображаются на графиках на странице процесса. Она содержит обязательный раздел charts и необязательный раздел keycharts. Каждый раздел имеет одинаковый формат и выглядит следующим образом:

```
{



"ui": {



"keymetrics" : [



{



"key" : "requestCount",



"aggregation" : "avg",



"mergeaggregation" : "sum",



"displayname" : "Requests"



}



],



"keycharts" : [ ],



"charts": [ ]



}



}
```

Раздел `keymetrics` полностью необязателен и позволяет определить до двух метрик, которые должны быть частью инфографики процесса. Он имеет следующие атрибуты:

| Поле | Тип | Описание |
| --- | --- | --- |
| `key` | String | Ключ временного ряда для размещения в инфографике. Допускаются только буквы, цифры и символы "-", "\_". |
| `aggregation` | String | Агрегация определяет метод агрегации минутных значений при работе с более длительными временными рамками. Dynatrace собирает значение каждые 10 секунд, но отправляет только одно агрегированное значение в минуту. Этот параметр определяет, как агрегировать эти 10-секундные значения. |
| `mergeaggregation` | String | Если метрика содержит несколько измерений, определяет, как агрегировать значения измерений в одно измерение. |
| `displayname` | String | Имя для отображения в инфографике |

Каждый раздел графиков имеет одинаковый формат и выглядит следующим образом:

```
{



"group": "Section Name",



"title": "Chart Name",



"series": [



{



"key": "MetricName",



"aggregation": "avg",



"displayname": "Display name for metric",



"seriestype": "area"



},



{



"key": "Other Metric Name",



"aggregation": "avg",



"displayname": "Display name for metric",



"color": "rgba(42, 182, 244, 0.6)",



"seriestype": "area"



}



]



}
```

Раздел charts описывает, как отображать каждую метрику в разделе подробностей на странице процесса (доступен при выборе **Further details**).

Оба раздела позволяют определить массив графиков. График имеет следующие обязательные атрибуты:

| Поле | Тип | Описание |
| --- | --- | --- |
| `group` | String | Имя раздела, в который должен быть помещён график |
| `title` | String | Имя графика |
| `series` | Array | Массив определений временных рядов и графиков. Один график может содержать несколько метрик. |

Серия имеет следующие атрибуты:

| Поле | Тип | Описание |
| --- | --- | --- |
| `key` | String | Ключ временного ряда для отображения на графике |
| `displayname` | String | Отображаемое имя метрики. Перезаписывает displayname метрики. По умолчанию: displayname метрики. |
| `aggregation` | String | Способ агрегации нескольких минутных значений на графиках при просмотре более длительного временного периода. Возможные значения: `SUM`, `AVG`, `MIN`, `MAX` |
| `mergeaggregation` | String | Ключевые графики не показывают несколько измерений. Если метрика содержит несколько измерений, определяет, как агрегировать значения измерений в одно измерение. |
| `color` | String | HTML-нотация цвета (RGB или RGBA). |
| `seriestype` | String | Тип графика. Возможные значения: `line`, `area` и `bar` |
| `rightaxis` | Boolean | Если true, метрика будет размещена на правой оси вместо левой. Обратите внимание, что Dynatrace поддерживает двухосевые графики. |
| `stacked` | Boolean | Если true, несколько метрик накладываются друг на друга. Работает только для графиков типа area и bar. |
