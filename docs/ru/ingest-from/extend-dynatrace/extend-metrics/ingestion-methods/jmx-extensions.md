# Расширения JMX


* Latest Dynatrace
* Чтение: 7 мин.
* Опубликовано 19 июля 2017 г.

JMX ([Java Management Extensions](https://www.oracle.com/technetwork/java/javase/tech/javamanagement-140525.html)) идеально подходит для мониторинга приложений, созданных на Java. С помощью Dynatrace вы можете отслеживать любую метрику в вашей JVM, которая доступна через MBean.

## Режим мониторинга инфраструктуры

Расширения JMX и PMI также доступны в [режиме мониторинга инфраструктуры](../../../../platform/oneagent/monitoring-modes/monitoring-modes.md "Узнайте больше о доступных режимах мониторинга при использовании OneAgent."). Вы можете использовать их для мониторинга любого компонента инфраструктуры и вспомогательного сервиса, написанного на Java, и получать все собранные метрики в Dynatrace.

## Создание расширения JMX

Dynatrace версии 1.162 и новее

Dynatrace версии 1.161 и старше

1. Войдите в Dynatrace и перейдите в **Settings** > **Monitoring** > **Monitored technologies** и нажмите **Add new technology monitoring**. Из четырёх доступных вариантов один — **Monitor Java or WebSphere based technologies**. Нажмите **Add JMX/PMI extension**.
2. Нажмите **Use JMX/PMI extensions editor**.
3. Введите имя расширения. Можно использовать только буквы `A-Z, a-z`, цифры `0-9` или `-/_`.
4. Добавьте источник метрик.

* Выберите одну из доступных технологий.
* На основе вашего выбора редактор предложит Java-процесс в качестве источника метрик. Это сузит список доступных MBean или модулей PMI, которые вы будете использовать на следующем шаге. Технология, лежащая в основе процесса, определяет, будете ли вы просматривать JMX MBean или модули PMI. Нажмите **Add metrics source**, чтобы использовать предложенный источник. Нажмите ![Edit](https://dt-cdn.net/images/edit-icon-a083c62c49.svg "Edit"), чтобы выбрать другой процесс. При его выборе вы можете сузить список доступных процессов, выбрав конкретный хост или зону управления.

Выбор процесса, хоста или зоны управления не ограничивает область мониторинга расширения. Расширения JMX/PMI мониторят все Java-процессы, доступные OneAgent. Метрика, определённая для одного и того же MBean и атрибута, будет также доступна в других Java-процессах вашего приложения, что может быстро исчерпать лимит пользовательских метрик.

Вы также можете выбрать отдельные хосты, на которых будет выполняться ваше новое расширение JMX, вместо его глобального запуска по всей среде.

1. Сразу после добавления расширения JMX перейдите в **Settings** > **Monitored technologies** > вкладка **Custom extensions** и отключите его глобально.
2. Перейдите на страницы отдельных хостов, откройте настройки хоста, найдите ваше расширение JMX и включите его.

5. Когда список метрик загрузится, нажмите кнопку **Add metric**, чтобы начать создание расширения.

   #### Домен

   Выберите домен.

   #### Выбор ключевых свойств

   Ключевые свойства фильтруют список доступных MBean. Обратите внимание, что значения свойств могут содержать подстановочные знаки. Выберите **Exact match**, чтобы отфильтровать список MBean только по записям, точно соответствующим значениям ключевых свойств.

   #### Выбор MBean

   Выберите один MBean из списка или оставьте опцию **All matching MBeans**. Если совпадает более одного MBean, это приведёт к агрегации атрибутов из всех совпадающих MBean в одну метрику. Тип агрегации вы определите ниже.

   #### Выбор атрибута

   В списке доступны только числовые атрибуты. Чтобы увидеть другие атрибуты, например атрибуты типа `String`, выберите **Include non-numeric attributes**. Обратите внимание, что выбранные атрибуты должны быть преобразуемы в тип double или boolean. В противном случае OneAgent не будет собирать данные такой метрики.

   #### Отображаемое имя

   Вы можете задать пользовательское отображаемое имя для метрики.

   #### Единица измерения

   Выберите единицу измерения метрики.

   #### Агрегация

   Выберите метод агрегации, если вы извлекаете атрибут из нескольких совпадающих MBean.

   #### Расчёт

   **Delta** вычисляет изменение значений данного атрибута. **Rate** вычисляет скорость изменений в секунду. Когда вы выбираете оба варианта, вы можете преобразовать абсолютный атрибут (например, количество запросов) в скорость (например, запросы в секунду). Значение = атрибут / интервал запроса.

## Создание расширения PMI

Создание расширения PMI очень похоже на расширение JMX. Модуль PMI соответствует JMX MBean, а статистика — атрибутам. Все остальные настройки работают аналогично.

## Сохранение расширения

После добавления метрики вы всё ещё можете редактировать или удалить её определение. Необходимо нажать кнопку **Save**, чтобы зарегистрировать расширение и начать сбор данных из всех Java-процессов, содержащих определённые MBean. После сохранения расширения вы не можете редактировать или удалять метрики, так как это позволяет обеспечить непрерывность ваших данных. Вы всё ещё можете добавлять новые метрики. Однако в этом случае вы не сможете выбрать базовый Java-процесс, поскольку будете использовать процесс, на котором расширение уже развёрнуто.

## Настройка расширения

Расширения JMX в Dynatrace обладают широкими возможностями настройки, выходящими за рамки редактора. Вы можете загрузить ваше расширение и отредактировать его JSON-файл, описывающий ваши JMX-метрики и способ их отображения. Справочную информацию и примеры см. в разделе [Настройка плагинов JMX](jmx-extensions/customize-jmx-extensions.md "Узнайте, как дополнительно настроить расширения JMX.").

Вам нужно создать JSON-файл, описывающий ваши JMX-метрики и способ их отображения. Справочную информацию и примеры см. в разделе [Настройка плагинов JMX](jmx-extensions/customize-jmx-extensions.md "Узнайте, как дополнительно настроить расширения JMX.").

## Загрузка расширения JMX

1. Чтобы добавить расширение JMX в Dynatrace, перейдите в **Settings** > **Monitored technologies > Custom extensions** и нажмите **Upload extension**.
2. Выберите файл `plugin.json` вашего расширения JMX.
   После успешной загрузки ваше расширение отобразится на странице **Custom extensions**. Перезапуск JVM для включения расширений JMX или применения изменений не требуется.

   Если в будущем вы внесёте изменения в расширение, не забудьте загрузить обновлённый файл `plugin.json` в Dynatrace.

## Просмотр метрик JMX в Dynatrace

Метрики JMX доступны для всех процессов на базе Java, мониторинг которых осуществляется OneAgent.

После загрузки расширения Dynatrace автоматически начинает запрашивать определённые метрики для всех Java-процессов. Чтобы найти метрики, перейдите на страницу соответствующего процесса и нажмите **Further details**.

![Метрики JMX](https://dt-cdn.net/images/jmxeditor-1-600-91b2c04b73.png)

Например, для расширения с именем `HornetQ` перейдите к процессу, который вы выбрали при создании расширения, нажмите **Further details**, а затем выберите вкладку **HornetQ**, автоматически названную по имени расширения. Там вы найдёте все определённые метрики.

![Метрики JMX](https://dt-cdn.net/images/jmxeditor-2-1211-ca87466d9e.png)

Не видите определённую метрику?

Мониторинг JMX является высокодинамичным. Если конкретная метрика не существует в вашей JVM, это не ошибка — метрика просто недоступна.

## Пользовательские метрики расширений JMX/PMI

Вы можете заметить, что количество метрик в файле `plugin.json` вашего расширения JMX/PMI отличается от количества тарифицируемых пользовательских метрик. Это связано с тем, что расширения JMX/PMI предоставляют «разбиения» (splittings), которые можно использовать для определения дополнительных измерений для создаваемых метрик. Для каждого значения выбранного свойства во время выполнения создаётся отдельная метрика временного ряда. Фактическое количество метрик временных рядов зависит от количества уникальных значений выбранного свойства.

Следующий пример показывает, как определить метрику, создающую несколько временных рядов из одной метрики:

```
"metrics": [


{


"timeseries": {


"key": "XY.Size",


"unit": "Count",


"displayname": "Queue Consumer Count",


"dimensions": [


"rx_pid",


"name"


]


}


"source": {


"domain": "com.sample",


"keyProperties": {


"type": "XY",


"name": "*"


},


"attribute": "Size",


"splitting": {


"name": "name",


"type": "keyProperty",


"keyProperty": "name"


}


}


}


]
```

В этом примере MBean `com.sample:type=XY,name=A` и `com.sample:type=XY,name=B` приведут к созданию двух метрик временных рядов (`A` и `B`).

Поскольку такие значения обычно не известны заранее и могут изменяться, это может привести к неожиданно высокому количеству [потребляемых пользовательских метрик](../../../../license/monitoring-consumption-classic/davis-data-units/metric-cost-calculation.md "Узнайте, как рассчитать потребление единиц данных Davis и затраты, связанные с отслеживаемыми метриками."). Если значения свойства известны заранее или если интересны только некоторые из них, мы рекомендуем указать одну метрику для каждого значения и не использовать разбиение.

## Лимит метрик на расширение JMX

Каждое расширение JMX может содержать не более `5 000` метрик. Этот лимит необходим для предотвращения того, чтобы неоптимальные или перегруженные метриками расширения перегружали систему мониторинга.
