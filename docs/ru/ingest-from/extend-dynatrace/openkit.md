Существует множество способов взаимодействия вашего бизнеса с клиентами в цифровом мире. Мониторинг пользовательского опыта и поведения в ваших веб-приложениях и мобильных приложениях — это отличный способ начать работу с мониторингом цифрового опыта; Dynatrace легко обнаруживает и автоматически отслеживает все точки взаимодействия с приложением с помощью OneAgent. Однако у вашего бизнеса, вероятно, есть и множество других цифровых точек взаимодействия за пределами ваших приложений, где клиенты соприкасаются с вашим брендом и которые также важны для успеха вашего бизнеса. С Dynatrace OpenKit вы получаете набор библиотек с открытым исходным кодом, позволяющих инструментировать все остальные цифровые точки взаимодействия в вашей среде — будь то традиционные приложения для богатых клиентов, умные IoT-приложения или даже навыки Alexa.

Dynatrace OpenKit — это набор библиотек с открытым исходным кодом, который предоставляет простые и лёгкие средства для инструментирования исходного кода ваших пользовательских приложений, чтобы вы могли отслеживать их производительность с помощью Dynatrace. Dynatrace OpenKit лучше всего подходит для клиент-серверных приложений, взаимодействующих через HTTP, — например, приложений с богатым клиентом, встраиваемых устройств и терминалов.

Основные преимущества OpenKit:

* Простота использования
* Отсутствие зависимостей от библиотек OneAgent
* Лёгкая переносимость на другие языки и платформы

С Dynatrace OpenKit вы можете:

* Отслеживать пользовательские сессии и действия пользователей
* Сообщать о событиях, ошибках и сбоях
* Трассировать веб-запросы в распределённые трассировки на стороне сервера
* Помечать пользовательские сессии тегами пользователей
* Поддерживать совместимость с Dynatrace

С Dynatrace OpenKit вы не можете:

* Создавать распределённые трассировки на стороне сервера. Эта функциональность предоставляется Dynatrace OneAgent SDK.
* Создавать метрики. Однако вы можете использовать Topology and Smartscape API и Metrics API для передачи метрик.

Начиная с апреля 2022 года Dynatrace больше не поддерживает TLS 1.0 и TLS 1.1 для Dynatrace SaaS Real User Monitoring (RUM). Теперь Dynatrace SaaS поддерживает только TLS 1.2+. По этой причине использование версии .NET Framework ниже 4.7 может потребовать [дополнительной настройки](https://docs.microsoft.com/en-us/dotnet/framework/network-programming/tls#if-your-app-targets-a-net-framework-version-earlier-than-47).
