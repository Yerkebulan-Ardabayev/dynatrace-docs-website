Страницы унифицированного анализа Dynatrace объединяют все данные наблюдаемости и соответствующие аналитические инструменты для эффективного анализа и устранения неполадок в контексте. При изучении метрик, событий, журналов и метаданных для проблемной, доменно-специфичной сущности вы можете найти каждый сигнал наблюдаемости, связанный с этой сущностью, на одной странице.

Страница обзора хоста -- пример страницы унифицированного анализа, доступной в большинстве сред.

## Типы страниц

Существует два типа страниц унифицированного анализа:

* **Экран списка**
  Экран списка генерируется автоматически и позволяет просматривать все экземпляры определенного типа сущности. Доступные настройки можно найти в разделе Настройки списка.
* **Экран деталей**
  Экран деталей сущности объединяет все сигналы наблюдаемости, привязанные к сущности, в контексте. Как и экран списка, экран деталей автоматически генерируется для каждой сущности в вашей среде. Доступные настройки можно найти в разделе Настройки деталей.

На странице сущности вы можете определить различные типы карточек.

Используйте карточку свойств для отображения атрибутов и тегов. По умолчанию она отображает все атрибуты, поступающие из API отслеживаемых сущностей. Дополнительную информацию см. в разделе Панель уведомлений.

Подробности конфигурации см. в разделе Карточки свойств.

![Пример карточки свойств](https://dt-cdn.net/images/properties-528-184d1764f0-528-10fef21345.png)

### Журналы

Используйте карточку журналов -- которая имеет те же функции, что и Просмотр журналов -- для отображения столбчатой диаграммы, представляющей различные вхождения журналов в выбранном временном интервале, и детальной таблицы, где каждый журнал является записью с дополнительными свойствами, такими как временная метка, статус и содержимое.

Подробности конфигурации см. в разделе Карточки журналов.

![logs-card](https://dt-cdn.net/images/screenshot-2023-03-14-at-10-16-08-624-bf7cf1200b.png)

### Сообщения

Используйте карточку сообщений для отображения информации при выполнении определенного условия. Подробности конфигурации см. в разделе Карточки сообщений. Существует два типа визуализации карточки сообщений:

* **Сообщение** -- карточка, которая просто отображает текстовую информацию.
* **Карточка** -- карточка с заголовком, описанием и доступными действиями.

Используйте карточку событий для отображения событий, связанных с указанными сущностями.

![Пример карточки событий](https://dt-cdn.net/images/screenshot-2023-03-14-at-14-12-35-571-7d4521137a.png)

### Здоровье

Используйте карточку здоровья для отображения определенных метрик в визуальном формате. По умолчанию она предоставляет быстрый обзор до шести отдельных плиток, каждая из которых представляет уникальную метрику или точку данных.

Каждая плитка реагирует на определенные события по подключенным метрикам и может принимать различные цвета:

* зеленый -- есть данные хотя бы по одной подключенной метрике
* красный -- есть открытая проблема, связанная хотя бы с одной подключенной метрикой
* серый -- есть закрытая проблема, связанная хотя бы с одной подключенной метрикой
* белый -- нет данных для этой плитки в текущем временном интервале

Подробности конфигурации см. в разделе Карточки здоровья.

![Карточка здоровья](https://dt-cdn.net/images/dee80e89-6646-420a-810d-0e7e2566677b-1640-4b6291895b.png)

## Концепции

### Действия

Действия определяют, что происходит после выбора одного из доступных вариантов в меню **More** (**...**) в правом верхнем углу каждой карточки.

Подробности конфигурации см. в разделе Действия.

### Фильтрация

Унифицированный анализ поддерживает фильтрацию сущностей по индексированным атрибутам сущностей. Вы можете включить фильтрацию для экрана списка и в контексте определенных карточек. Фильтрация сущностей может быть настроена на двух уровнях:

* На уровне страницы, где фильтрация влияет на все карточки на экране. Существуют отдельные конфигурации для экрана деталей и экрана списка.
* На уровне списка сущностей, где фильтрация влияет только на один список.

### Внедрения

Если вы хотите отображать карточки на странице без изменения их макета, см. раздел Расширение встроенных страниц унифицированного анализа.

### Исследовательский анализ

Исследовательский анализ анализирует только метрики из линейных графиков, которые находятся в группах графиков, списках сущностей и таблицах метрик. Дополнительную информацию см. в разделе Анализ причинно-следственных корреляций Davis.
