# Страницы унифицированного анализа


* Classic
* Обзор
* Чтение: 4 мин
* Опубликовано 9 марта 2023 г.

Страницы унифицированного анализа Dynatrace объединяют все данные наблюдаемости и соответствующие аналитические инструменты для эффективного анализа и устранения неполадок в контексте. При изучении метрик, событий, журналов и метаданных для проблемной, доменно-специфичной сущности вы можете найти каждый сигнал наблюдаемости, связанный с этой сущностью, на одной странице.

[Страница обзора хоста](../../../observe/infrastructure-observability/hosts/monitoring/host-monitoring.md "Мониторинг хостов с помощью Dynatrace.") -- пример страницы унифицированного анализа, доступной в большинстве сред.

## Типы страниц

Существует два типа страниц унифицированного анализа:

* **Экран списка**
  Экран списка генерируется автоматически и позволяет просматривать все экземпляры определенного типа сущности. Доступные настройки можно найти в разделе [Настройки списка](unified-analysis/unified-analysis-reference.md#list-settings "Узнайте о синтаксисе унифицированного анализа").
* **Экран деталей**
  Экран деталей сущности объединяет все сигналы наблюдаемости, привязанные к сущности, в контексте. Как и экран списка, экран деталей автоматически генерируется для каждой сущности в вашей среде. Доступные настройки можно найти в разделе [Настройки деталей](unified-analysis/unified-analysis-reference.md#details-settings "Узнайте о синтаксисе унифицированного анализа").

## Карточки

На странице сущности вы можете определить различные типы карточек.

### Группа графиков

Используйте компонент группы графиков для группировки настроенных графиков в сетку. Подробности конфигурации см. в разделе [Карточки группы графиков](unified-analysis/unified-analysis-reference.md#chart-group-cards "Узнайте о синтаксисе унифицированного анализа").

### Список сущностей

Подробности конфигурации см. в разделе [Карточки списка сущностей](unified-analysis/unified-analysis-reference.md#entities-list-cards "Узнайте о синтаксисе унифицированного анализа").

Используйте карточку таблицы метрик для отображения нескольких данных метрик в одной таблице.

Подробности конфигурации см. в разделе [Карточки таблицы метрик](unified-analysis/unified-analysis-reference.md#metric-table-cards "Узнайте о синтаксисе унифицированного анализа").

### Свойства

Используйте карточку свойств для отображения атрибутов и тегов. По умолчанию она отображает все атрибуты, поступающие из [API отслеживаемых сущностей](../../../dynatrace-api/environment-api/entity-v2.md "Узнайте об API отслеживаемых сущностей Dynatrace."). Дополнительную информацию см. в разделе [Панель уведомлений](../../../observe/infrastructure-observability/hosts/monitoring/host-monitoring.md#notifications-bar "Мониторинг хостов с помощью Dynatrace.").

Подробности конфигурации см. в разделе [Карточки свойств](unified-analysis/unified-analysis-reference.md#properties-cards "Узнайте о синтаксисе унифицированного анализа").

### Журналы

Используйте карточку журналов -- которая имеет те же функции, что и [Просмотр журналов](../../../analyze-explore-automate/log-monitoring/analyze-log-data/log-viewer.md "Узнайте, как использовать средство просмотра журналов Dynatrace для анализа данных журналов.") -- для отображения столбчатой диаграммы, представляющей различные вхождения журналов в выбранном временном интервале, и детальной таблицы, где каждый журнал является записью с дополнительными свойствами, такими как временная метка, статус и содержимое.

Подробности конфигурации см. в разделе [Карточки журналов](unified-analysis/unified-analysis-reference.md#logs-cards "Узнайте о синтаксисе унифицированного анализа").

### Сообщения

Используйте карточку сообщений для отображения информации при выполнении определенного условия. Подробности конфигурации см. в разделе [Карточки сообщений](unified-analysis/unified-analysis-reference.md#message-cards "Узнайте о синтаксисе унифицированного анализа"). Существует два типа визуализации карточки сообщений:

* **Сообщение** -- карточка, которая просто отображает текстовую информацию.
* **Карточка** -- карточка с заголовком, описанием и доступными действиями.

### События

Используйте карточку событий для отображения событий, связанных с указанными сущностями.

Подробности конфигурации см. в разделе [Карточки событий](unified-analysis/unified-analysis-reference.md#events-cards "Узнайте о синтаксисе унифицированного анализа").

### Здоровье

Используйте карточку здоровья для отображения определенных метрик в визуальном формате. По умолчанию она предоставляет быстрый обзор до шести отдельных плиток, каждая из которых представляет уникальную метрику или точку данных.

Каждая плитка реагирует на определенные события по подключенным метрикам и может принимать различные цвета:

* зеленый -- есть данные хотя бы по одной подключенной метрике
* красный -- есть открытая проблема, связанная хотя бы с одной подключенной метрикой
* серый -- есть закрытая проблема, связанная хотя бы с одной подключенной метрикой
* белый -- нет данных для этой плитки в текущем временном интервале

Подробности конфигурации см. в разделе [Карточки здоровья](unified-analysis/unified-analysis-reference.md#health-cards "Узнайте о синтаксисе унифицированного анализа").

## Концепции

### Действия

Действия определяют, что происходит после выбора одного из доступных вариантов в меню **More** (**...**) в правом верхнем углу каждой карточки.

Подробности конфигурации см. в разделе [Действия](unified-analysis/unified-analysis-reference.md#actions "Узнайте о синтаксисе унифицированного анализа").

### Фильтрация

Унифицированный анализ поддерживает фильтрацию сущностей по индексированным атрибутам сущностей. Вы можете включить фильтрацию для экрана списка и в контексте определенных карточек. Фильтрация сущностей может быть настроена на двух уровнях:

* На уровне страницы, где фильтрация влияет на все карточки на экране. Существуют отдельные конфигурации для [экрана деталей](unified-analysis/unified-analysis-reference.md#details-filters "Узнайте о синтаксисе унифицированного анализа") и [экрана списка](unified-analysis/unified-analysis-reference.md#list-filters "Узнайте о синтаксисе унифицированного анализа").
* На уровне списка сущностей, где фильтрация влияет только на один список.

### Внедрения

Если вы хотите отображать карточки на странице без изменения их макета, см. раздел [Расширение встроенных страниц унифицированного анализа](extend-unified-analysis-pages.md "Расширьте встроенную страницу унифицированного анализа дополнительными данными, полученными вашим расширением.").

### Исследовательский анализ

Исследовательский анализ анализирует только метрики из линейных графиков, которые находятся в группах графиков, списках сущностей и таблицах метрик. Дополнительную информацию см. в разделе [Анализ причинно-следственных корреляций Davis](../../../dynatrace-intelligence/reference/ai-models/causal-correlation-analysis.md "Узнайте, как анализ причинно-следственных корреляций Dynatrace Intelligence находит связанные метрики в вашей среде.").
