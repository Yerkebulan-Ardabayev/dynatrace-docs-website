# Настройка CORS в Amazon S3


* How-to guide
* 3-min read

Воспользуйтесь этими инструкциями для настройки CORS (межсайтовых HTTP-запросов) в AWS для бакетов в Amazon S3 (Simple Storage Service). Чтобы узнать больше о CORS и принципах его работы, см. [HTTP access control (CORS)](https://developer.mozilla.org/en-US/docs/Web/HTTP/Access_control_CORS).

## Введение

Современные веб-браузеры не предоставляют доступ к сведениям об ошибках в `window.onerror` для скриптов, загружаемых с других доменов. Это серьёзное ограничение снижает эффективность механизма обнаружения ошибок JavaScript в Dynatrace.

Без CORS вместо подробностей об ошибке и трассировки стека при возникновении необработанной ошибки JavaScript отображается сообщение следующего вида:

«Script error» обычно возникает, когда исключение нарушает политику одного источника браузера, то есть когда ошибка происходит в скрипте, размещённом на домене, отличном от домена текущей страницы.
Если скрипт получен с установленными HTTP-заголовками CORS, можно получить полные сведения об ошибке в большинстве браузеров, добавив атрибут `crossorigin="anonymous"` к тегу script. К сожалению, Internet Explorer 11 и Edge в настоящее время не поддерживают атрибут `crossorigin`. Также необходимо соблюдать осторожность и устанавливать атрибут `crossorigin` только для скриптов с заголовками CORS, поскольку иначе скрипт перестанет выполняться.

Чтобы получить полную видимость возникающих ошибок JavaScript, необходимо установить HTTP-заголовки CORS на межсайтовых серверах и применить атрибут `crossorigin` к тегу script. Перед выполнением этого шага необходимо настроить Amazon S3.

### Настройка Amazon S3

Первый шаг — ввести настройки CORS для ресурсов в S3. Это делается на уровне бакета. К сожалению, ввести настройки CORS для отдельных ресурсов внутри бакетов невозможно.

1. Откройте консоль Amazon S3.
2. Выберите бакет, содержащий ваши ресурсы.
3. Выберите **Permissions** (Разрешения).
4. Прокрутите вниз до раздела **Cross-origin resource sharing (CORS)** и выберите **Edit** (Редактировать).
5. Вставьте конфигурацию CORS в формате JSON.

   Пример JSON:

   ```
   [


   {


   "AllowedHeaders": [


   "*"


   ],


   "AllowedMethods": [


   "GET",


   "HEAD"


   ],


   "AllowedOrigins": [


   "*"


   ],


   "ExposeHeaders": []


   }


   ]
   ```

   Подробности см. в статье [Creating a cross-origin resource sharing (CORS) configuration](https://docs.aws.amazon.com/AmazonS3/latest/userguide/ManageCorsUsing.html).
6. Выберите **Save changes** (Сохранить изменения), чтобы сохранить конфигурацию.

### Настройка CloudFront

После настройки Amazon S3 необходимо включить заголовки CORS в CloudFront. В противном случае заголовки будут отфильтрованы и механизм не будет работать.

1. Откройте консоль CloudFront. Вы увидите окно, похожее на показанное ниже. Найдите своё распределение в списке и нажмите на ссылку **ID**. При необходимости воспользуйтесь полем поиска (выделено красным) для поиска распределения.

   ![AWS cors 3](https://dt-cdn.net/images/aws-cors-3-1652-d619805ef5.png)
2. Теперь вы должны видеть своё распределение. Нажмите вкладку **Behaviors** (Поведение). Найдите запись поведения для вашего ресурса, описанную в столбце **Path Pattern**. Выберите шаблон и нажмите **Edit** (Редактировать). Если запись поведения не существует, её необходимо создать.

   ![AWS cors 4](https://dt-cdn.net/images/aws-cors-4-968-4f0dee4542.png)
3. Теперь можно редактировать настройки поведения. Все элементы, связанные с CORS, выделены красным ниже. Сначала необходимо выбрать, какой HTTP-метод будет доступен для этого поведения. Для параметра **Allowed HTTP Methods** (Разрешённые HTTP-методы) рекомендуется выбрать второй или третий вариант и установить флажок **OPTIONS**, иначе вы не сможете использовать CORS для REST-вызовов, использующих preflight-запросы.
4. В списке **Forward Headers** (Пересылать заголовки) выберите **Allowlist** (Разрешённые). После этого появится меню **Allowlist headers** (Разрешённые заголовки). Выберите конкретный заголовок из левого списка и нажмите **Add** (Добавить). Если нужный заголовок не отображается, введите его в текстовое поле **Filter headersâ¦** и нажмите **Add custom** (Добавить пользовательский). Для CORS наиболее важны заголовки `Origin` и `Access-Control-Allow-Origin`. Остальные заголовки необязательны и зависят от того, что вы хотите достичь.
5. Подтвердите изменения, нажав **Yes, Edit** (Да, редактировать).

   ![AWS cors 5](https://dt-cdn.net/images/aws-cors-5-953-60dec24e19.png)
