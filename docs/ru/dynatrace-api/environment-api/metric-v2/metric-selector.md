Селектор метрик — это мощный инструмент для указания того, какие метрики вы хотите прочитать через запрос [GET metric data points](get-data-points.md "Чтение точек данных одной или нескольких метрик через Metrics v2 API.") или в [**Расширенном режиме** Data Explorer](../../../analyze-explore-automate/explorer/explorer-advanced-query-editor.md "Создание расширенных запросов с помощью расширенного режима Data Explorer.").

Кроме того, вы можете преобразовывать полученный набор точек данных. Эти преобразования модифицируют необработанные данные метрик.

Даже если вы создаёте селектор для использования в вызове API, мы рекомендуем создавать запрос с помощью вкладки **Code** в Data Explorer, которая предлагает встроенные инструменты (например, автодополнение) для построения запроса.

## Ограничения

* Селектор должен содержать хотя бы один ключ метрики.
* В одном запросе можно запросить точки данных до 10 метрик.

## Измерения метрик

Многие метрики Dynatrace могут быть указаны с более высокой детализацией с помощью измерений. Например, метрика **builtin:host.disk.avail** имеет два измерения:

* Первичное измерение — **Host**
* Вторичное измерение — **Disk**

Запросите метрику с помощью вызова [GET metric descriptor](get-descriptor.md "Просмотр дескриптора метрики через Metrics v2 API."), чтобы получить информацию о доступных измерениях — вы найдёте их в поле **dimensionDefinitions** дескриптора метрики.

Показать пример дескриптора

```
{


"dimensionDefinitions": [


{


"key": "dt.entity.host",


"name": "Host",


"displayName": "Host",


"index": 0,


"type": "ENTITY"


},


{


"key": "dt.entity.disk",


"name": "Disk",


"displayName": "Disk",


"index": 1,


"type": "ENTITY"


}


]


}
```

Везде, где вы видите заполнитель `<dimension>` в примерах синтаксиса, вы можете выбрать конкретное измерение метрики. Измерение можно указать по его ключу. Например, для **builtin:host.disk.avail** это **dt.entity.host** и **dt.entity.disk**.

Операции преобразования изменяют список измерений, добавляя или удаляя их. Последующие преобразования работают с изменённым списком измерений. Запросите дескриптор метрики с предшествующими преобразованиями (например, **builtin:host.disk.avail:names**), чтобы увидеть новый список доступных измерений.

### Остаточное измерение

Dynatrace сохраняет только топ X кортежей измерений (точное число зависит от метрики, агрегации, временного диапазона и других факторов). Все остальные кортежи измерений агрегируются в один, называемый *остаточным* измерением.

Если результат запроса включает это измерение, значения `dimensions` и `dimensionMap` будут `null`. Однако если `dimensionMap` вообще не содержит записи, то это не остаточное измерение, а буквальное значение `null`.

## Агрегация по времени

Объём необработанных данных в Dynatrace затрудняет их представление в удобном виде. Для улучшения читаемости Dynatrace применяет агрегацию по времени, выравнивая данные по временным интервалам. Вы можете определить метод агрегации через [преобразование **aggregation**](#aggregation).
