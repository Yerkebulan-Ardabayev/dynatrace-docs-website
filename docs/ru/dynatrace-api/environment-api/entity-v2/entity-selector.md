# Environment API v2 - Селектор сущностей


Селектор сущностей -- это мощный инструмент для указания того, какие сущности вы хотите включить в область действия вызовов Environment v2 API. Он используется в нескольких API, поэтому вам нужно изучить синтаксис только один раз и затем использовать его для множества вариантов применения.

Вы должны указать один из следующих критериев:

* [Тип сущности](#type)
* [Идентификатор сущности](#id)

Дополнительно вы можете указать следующие критерии в любой комбинации:

* [Имя сущности](#name)
* [Атрибут сущности](#attribute)
* [Тег](#tag)
* [ID зоны управления](#mzid)
* [Имя зоны управления](#mzname)
* [Состояние здоровья](#health)
* [Включение удалённых сущностей](#deletedEntities)

Если вы указываете несколько критериев, в ответ включаются только результаты, соответствующие **всем** критериям.

* Например: `type(HOST),entityName.equals(Server)`

Если текстовый ввод в критериях содержит определённые символы, такие как скобки или запятая, текст необходимо заключить в кавычки.

* Например: `type(HOST),entityName.equals("Server(prod),1")`

## Ограничения

Общая длина строки **entitySelector** ограничена 2 000 символами.

Вы можете выбрать только один тип сущности на запрос.

## Тип сущности

Тип сущности, которую вы хотите запросить.

Вы можете получить список доступных типов сущностей с помощью вызова GET all entity types.

|  |  |
| --- | --- |
| Синтаксис | `type("value")` |
| Множественные значения | Не применимо |
| Оператор значения | `EQUALS` |
| Чувствительность к регистру | Не применимо |

## Идентификатор сущности Dynatrace

Идентификатор сущности Dynatrace запрашиваемой сущности.

Чтобы указать несколько идентификаторов, разделите их запятой (`,`). Все запрашиваемые сущности должны быть одного типа.

Вы можете получить список доступных сущностей с помощью вызова GET entities list.

|  |  |
| --- | --- |
| Синтаксис | `entityId("id-1","id-2")` |
| Множественные значения | Применимо |
| Оператор значения | `EQUALS` |
| Чувствительность к регистру | Не применимо |

## Имя сущности

Имя запрашиваемой сущности.

Вы можете получить список доступных сущностей с помощью вызова GET entities list.

|  |  |
| --- | --- |
| Синтаксис | `entityName("name")` |
| Множественные значения | Не применимо |
| Оператор значения | `CONTAINS` |
| Чувствительность к регистру | Не применимо |

### Модификация "начинается с"

Изменяет оператор значения критерия имени сущности на `BEGINS WITH`.

|  |  |
| --- | --- |
| Синтаксис | `entityName.startsWith("name")` |
| Множественные значения | Не применимо |
| Оператор значения | `BEGINS WITH` |
| Чувствительность к регистру | Не применимо |

### Модификация "равно"

Изменяет оператор значения критерия имени сущности на `EQUALS`.

|  |  |
| --- | --- |
| Синтаксис | `entityName.equals("name")` |
| Множественные значения | Не применимо |
| Оператор значения | `EQUALS` |
| Чувствительность к регистру | Не применимо |

### Модификация "одно из значений"

Позволяет указать несколько значений для критерия имени сущности.

|  |  |
| --- | --- |
| Синтаксис | `entityName.in("name1","name2")` |
| Множественные значения | Применимо |
| Оператор значения | `EQUALS` |
| Чувствительность к регистру | Не применимо |

### Модификация чувствительности к регистру

По умолчанию при оценке имён сущностей регистр не учитывается. Вы можете сделать критерий более строгим, используя модификацию `caseSensitive`. Она принимает любой критерий имени сущности в качестве аргумента (включая уже изменённые модификаторами `.startsWith` или `.equals`) и оценивает значения с учётом регистра.

|  |  |
| --- | --- |
| Синтаксис | `caseSensitive(<entity name criterion>)` |
| Множественные аргументы | Не применимо |

## Атрибут сущности

Пара "имя атрибута -- значение атрибута", которой должна обладать запрашиваемая сущность.

Чтобы получить список доступных атрибутов, выполните вызов GET entity type и проверьте поле **properties**. Вы можете использовать атрибуты со значениями, которые могут быть представлены в виде строки.

|  |  |
| --- | --- |
| Синтаксис | `<attribute name>("attribute value")` |
| Множественные значения | Применимо |
| Оператор значения | `EQUALS` |
| Чувствительность к регистру имени атрибута | Не применимо |
| Чувствительность к регистру значения атрибута | Применимо |

### Модификация "существует"

Изменяет оператор критерия атрибута сущности на `EXISTS`. В этом случае условие выбирает сущности, у которых есть данный атрибут, независимо от его значения.

|  |  |
| --- | --- |
| Синтаксис | `<attribute name>.exists()` |
| Оператор | `EXISTS` |

## Тег

Тег запрашиваемых сущностей. Теги в форматах `[context]key:value`, `key:value` и `value` определяются и разбираются автоматически. Если тег, состоящий только из значения, содержит двоеточие (`:`), вы должны экранировать двоеточие обратной косой чертой (`\`). В противном случае тег будет разобран как тег формата `key:value`.

Чтобы указать несколько тегов, разделите их запятой (`,`). Сущность с **любым** из указанных тегов включается в ответ.

Вы можете получить список доступных тегов с помощью вызовов GET custom tags и GET auto-tags.

|  |  |
| --- | --- |
| Синтаксис | `tag("[context]key1:value-1","key2:value-2","value-3")` |
| Множественные значения | Применимо |
| Оператор значения | `EQUALS` |
| Чувствительность к регистру | Применимо |

## ID зоны управления

Идентификатор зоны управления, к которой принадлежат запрашиваемые сущности.

Чтобы указать несколько идентификаторов, разделите их запятой (`,`).

Вы можете получить список доступных зон управления с помощью вызова GET all management zones.

|  |  |
| --- | --- |
| Синтаксис | `mzId("123456789","987654321")` |
| Множественные значения | Применимо |
| Оператор значения | `EQUALS` |
| Чувствительность к регистру | н/д |

## Имя зоны управления

Имя зоны управления, к которой принадлежат запрашиваемые сущности.

Чтобы указать несколько имён, разделите их запятой (`,`).

Вы можете получить список доступных зон управления с помощью вызова GET all management zones.

|  |  |
| --- | --- |
| Синтаксис | `mzName("name-1","name-2")` |
| Множественные значения | Применимо |
| Оператор значения | `EQUALS` |
| Чувствительность к регистру | Применимо |

## Состояние здоровья

Состояние здоровья запрашиваемых сущностей. Возможные значения: `HEALTHY` и `UNHEALTHY`.

|  |  |
| --- | --- |
| Синтаксис | `healthState("HEALTHY")` |
| Множественные значения | Не применимо |
| Оператор значения | `EQUALS` |
| Чувствительность к регистру | Применимо |

## Время первого обнаружения

Метка времени (миллисекунды UTC), когда сущность была впервые обнаружена.

## Включение удалённых сущностей

Включение или исключение сущностей, помеченных как удалённые. Это ограничено следующими типами сущностей:

* `HTTP_CHECK`
* `SYNTHETIC_TEST`
* `EXTERNAL_SYNTHETIC_TEST`
* `MULTIPROTOCOL_MONITOR`
* `APPLICATION`
* `MOBILE_APPLICATION`
* `CUSTOM_APPLICATION`
* `DCRUM_APPLICATION`
* `CUSTOM_DEVICE`

## Связи

Связи, которыми должна обладать запрашиваемая сущность.

Чтобы получить список доступных связей, выполните вызов GET entity type и проверьте поля **fromRelationships** и **toRelationships**.

## Инверсия критерия

Вы можете использовать модификацию `not` для инверсии любого критерия, кроме **type**. Она принимает критерий в качестве аргумента и инвертирует условие. Например, критерий `not(tag("Infrastructure:Linux"))` выбирает сущности, которые *не* имеют тега **Infrastructure:Linux**.

Вы можете использовать инвертированные критерии как часть сложных селекторов, так же как и любые другие критерии.

|  |  |
| --- | --- |
| Синтаксис | `not (<criterion>)` |
| Множественные аргументы | Не применимо |
| Примечание | Не поддерживает критерии [**type**](#type). |
