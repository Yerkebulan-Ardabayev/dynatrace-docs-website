# Расчёт потребления мониторинга инфраструктуры (DPS)


* Последняя версия Dynatrace
* Пояснение
* Чтение: 1 мин
* Обновлено 10 декабря 2025 г.

Dynatrace OneAgent может быть настроен в режиме мониторинга инфраструктуры, который обеспечивает комплексный мониторинг физических и виртуальных хостов.

В дополнение ко всем функциям [Foundation & Discovery](foundation-and-discovery.md "Узнайте, как рассчитывается потребление возможности Dynatrace Foundation & Discovery DPS.") мониторинг инфраструктуры также включает детальные метрики производительности процессов, метрики производительности дисков, анализ сетевого взаимодействия между процессами и анализ памяти по процессам.
Расширения Dynatrace могут быть включены на хостах в режиме мониторинга инфраструктуры и могут потреблять [пользовательские точки данных метрик](infrastructure-monitoring.md#infra-metrics "Узнайте, как рассчитывается потребление возможности Dynatrace Infrastructure Monitoring DPS.") и [Log Analytics](../log-analytics.md "Узнайте, как рассчитывается потребление Dynatrace Log Analytics в модели подписки Dynatrace Platform.").

### Часы хостов

Единицей измерения потребления мониторинга хостов в режиме мониторинга инфраструктуры является час хоста (host-hour).
Каждый экземпляр Dynatrace OneAgent, установленный и работающий на экземпляре операционной системы (развёрнутом, например, на физической или виртуальной машине) с включённым режимом мониторинга инфраструктуры, потребляет часы хостов.
Чем дольше хост находится под мониторингом, тем больше часов хостов потребляется.
Потребление не зависит от объёма памяти хоста.

### Гранулярность биллинга для потребления часов хостов

Dynatrace создан для динамичных облачных сред, где хосты и сервисы быстро создаются и уничтожаются.
Поэтому гранулярность биллинга для потребления часов хостов рассчитывается в 15-минутных интервалах.
Когда хост находится под мониторингом менее 15 минут в течение интервала, потребление часов хостов округляется до 15 минут перед расчётом.

На рисунке ниже показано, как рассчитывается потребление часов хостов для каждого хоста в 15-минутных интервалах.

### Метрики

Этот раздел предполагает, что вы использовали рекомендованные Dynatrace варианты развёртывания.
При пользовательском развёртывании включённые метрики могут работать как описано, но это не гарантируется Dynatrace.

Мониторинг инфраструктуры Dynatrace включает метрики мониторинга инфраструктуры и [другие встроенные метрики](../metrics/dps-metrics-ingest.md#billable-and-non-billable-metrics "Узнайте, как рассчитывается потребление возможности Metrics - Ingest & Process DPS.").
Эти метрики включены и никогда не тарифицируются.

Мониторинг инфраструктуры также включает определённое количество пользовательских точек данных метрик.
Каждый хост добавляет 1500 пользовательских точек данных метрик в каждом 15-минутном интервале.
Включённые точки данных метрик, не использованные в 15-минутном интервале, в котором они были предоставлены, не переносятся на последующие интервалы.
Включённые точки данных метрик вашей среды автоматически применяются к метрикам, исходящим от хостов, мониторируемых OneAgent в режиме мониторинга инфраструктуры.
Это применяется к пользовательским метрикам, как описано в таблице ниже.

Точки данных метрик, превышающие включённый объём, тарифицируются как:

* Если на вашей карте тарифов существует Metrics powered by Grail, они тарифицируются как [Metrics powered by Grail](../metrics.md "Узнайте, как рассчитывается потребление Dynatrace Metrics powered by Grail в модели подписки Dynatrace Platform.").
* Если на вашей карте тарифов нет Metrics powered by Grail, они тарифицируются как [Custom Metrics Classic](../platform-extensions/custom-metrics-classic.md "Узнайте, как рассчитывается потребление возможности Dynatrace Custom Metrics Classic DPS.").

1

Точки данных метрик, отправленные через Dynatrace Collector, включены только при наличии [обзора Metrics powered by Grail (DPS)](../metrics.md "Узнайте, как рассчитывается потребление Dynatrace Metrics powered by Grail в модели подписки Dynatrace Platform.") на вашей карте тарифов.

2

Точки данных метрик, отправленные через OpenTelemetry Collector, включены только при наличии [обзора Metrics powered by Grail (DPS)](../metrics.md "Узнайте, как рассчитывается потребление Dynatrace Metrics powered by Grail в модели подписки Dynatrace Platform.") на вашей карте тарифов и при условии, что OpenTelemetry Collector настроен как описано в [Обогащение запросов OTLP данными Kubernetes](../../../ingest-from/opentelemetry/collector/use-cases/kubernetes/k8s-enrich.md "Настройте OpenTelemetry Collector для обогащения запросов OTLP данными Kubernetes.").

На рисунке 2 выше показан включённый объём пользовательских точек данных метрик для четырёх 15-минутных интервалов.

### Пример расчёта включённых пользовательских точек данных метрик

* Первый 15-минутный интервал: `1 (мониторируемых хостов) x 1500 (точек данных метрик) = 1500 включённых пользовательских точек данных метрик`
* Второй 15-минутный интервал: `2 (мониторируемых хостов) x 1500 (точек данных метрик) = 3000 включённых пользовательских точек данных метрик`
* Третий 15-минутный интервал: `1 (мониторируемых хостов) x 1500 (точек данных метрик) = 1500 включённых пользовательских точек данных метрик`
* Четвёртый 15-минутный интервал: `1 (мониторируемых хостов) x 1500 (точек данных метрик) = 1500 включённых пользовательских точек данных метрик`

### Как потребляются пользовательские точки данных метрик в режиме мониторинга инфраструктуры

Потребление пользовательских точек данных метрик принимает различные формы.
Одинаковое количество пользовательских точек данных метрик может быть потреблено:

* Несколькими высокочастотными метриками или множеством низкочастотных метрик.
* Равномерно в нескольких 15-минутных интервалах или целиком за одну минуту.
* Всеми хостами в режиме мониторинга инфраструктуры, подмножеством хостов или одним хостом.

## Детали потребления: Инфраструктура

Dynatrace предоставляет встроенные метрики использования, которые помогают вам понять и проанализировать потребление мониторинга инфраструктуры вашей организацией.
Чтобы использовать эти метрики, в ![Data Explorer](https://dt-cdn.net/images/data-explorer-512-743267b1fc.png "Data Explorer") **Data Explorer** введите `DPS` в поле **Search**.
Эти метрики также доступны через Environment API и в Account Management (**Usage summary** > **Infrastructure Monitoring** > **Actions** > **View details**).

Ниже приведён список метрик, которые можно использовать для мониторинга деталей потребления мониторинга инфраструктуры.

(DPS) Использование биллинга мониторинга инфраструктуры
:   Ключ: `builtin:billing.infrastructure_monitoring.usage`

    Измерение: Count

    Разрешение: 15 мин

    Описание: Общее количество часов хостов, потреблённых в режиме мониторинга инфраструктуры.

(DPS) Использование биллинга мониторинга инфраструктуры по хостам
:   Ключ: `builtin:billing.infrastructure_monitoring.usage_per_host`

    Измерение: Host (`dt.entity.host`)

    Разрешение: 15 мин

    Описание: Потреблённые часы хостов в режиме мониторинга инфраструктуры по хостам.

(DPS) Всего точек данных метрик, сообщённых хостами инфраструктуры
:   Ключ: `builtin:billing.infrastructure_monitoring.metric_data_points.ingested`

    Измерение: Count

    Разрешение: 15 мин

    Описание: Количество точек данных метрик, потреблённых всеми хостами в режиме мониторинга инфраструктуры.

(DPS) Точки данных метрик, сообщённые и разделённые по хостам инфраструктуры
:   Ключ: `builtin:billing.infrastructure_monitoring.metric_data_points.ingested_by_host`

    Измерение: Host (`dt.entity.host`)

    Разрешение: 15 мин

    Описание: Количество точек данных метрик, разделённых по хостам в режиме мониторинга инфраструктуры.

(DPS) Доступные включённые точки данных метрик для хостов мониторинга инфраструктуры
:   Ключ: `builtin:billing.infrastructure_monitoring.metric_data_points.included`

    Измерение: Count

    Разрешение: 15 мин

    Описание: Общее количество включённых точек данных метрик, вычитаемых из точек данных метрик, сообщённых всеми хостами мониторинга инфраструктуры.

(DPS) Потреблённые включённые точки данных метрик для хостов инфраструктуры
:   Ключ: `builtin:billing.infrastructure_monitoring.metric_data_points.included_used`

    Измерение: Count

    Разрешение: 15 мин

    Описание: Количество потреблённых включённых точек данных метрик для хостов мониторинга инфраструктуры.

(DPS) Всего тарифицированных точек данных метрик для хостов инфраструктуры
:   Ключ: `builtin:billing.custom_metrics_classic.usage.infrastructure_hosts`

    Измерение: Count

    Разрешение: 15 мин

    Описание: Количество тарифицированных точек данных метрик для всех хостов мониторинга инфраструктуры.

### Мониторинг потребления часов хостов

Вы можете отслеживать общее потребление часов хостов для различных интервалов (15 мин, час, день или неделя) за любой выбранный временной период с помощью метрики "(DPS) Infrastructure Monitoring billing usage".
В приведённом ниже примере показано, что 5 хостов мониторились, что привело к общему потреблению 5 часов хостов за каждый час.

![Infrastructure Monitoring (DPS)](https://dt-cdn.net/images/image011-1099-50bac22dd0.png)

Вы можете разделить общее потребление часов хостов с помощью метрики "(DPS) Infrastructure Monitoring billing usage per host".
В приведённом ниже примере показан список всех хостов, сообщивших о потреблении.

![Infrastructure Monitoring (DPS)](https://dt-cdn.net/images/image013-1099-cb1470f609.png)

### Мониторинг потребления метрик для хостов мониторинга инфраструктуры

Используйте метрику "(DPS) Total metric data points billed for Infrastructure hosts" для мониторинга количества тарифицированных точек данных метрик для хостов мониторинга инфраструктуры, как показано в примере ниже.

![Infrastructure Monitoring (DPS)](https://dt-cdn.net/images/image015-1101-5c4b7620a4.png)

Для управления бюджетом метрик вы можете отслеживать количество доступных включённых точек данных метрик в сравнении с общим количеством потреблённых точек данных метрик, используя две метрики: "(DPS) Available included metric data points for Infrastructure hosts" и "(DPS) Total metric data points reported by Infrastructure hosts".
В приведённом ниже примере показано, что было потреблено больше точек данных метрик, чем включено для этих хостов мониторинга инфраструктуры.

![Infrastructure Monitoring (DPS)](https://dt-cdn.net/images/image017-1100-7b133be12d.png)

Вы можете использовать метрику "(DPS) Metric data points reported and split by Infrastructure hosts" для отслеживания количества потреблённых точек данных метрик по каждому хосту мониторинга инфраструктуры.
Разделённое представление помогает обнаружить хосты, потребляющие наибольшее количество точек данных метрик.
В приведённом ниже примере показано, что один из хостов мониторинга инфраструктуры сообщил значительно больше точек данных метрик, чем остальные.

![Infrastructure Monitoring (DPS)](https://dt-cdn.net/images/image019-1768-c4ed044b93.png)
