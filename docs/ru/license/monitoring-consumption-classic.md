# Dynatrace classic licensing


Потребление мониторинга при классическом лицензировании Dynatrace основано на различных единицах мониторинга, которые потребляет ваша среда Dynatrace по мере использования возможностей платформы Dynatrace вашей организацией. Если у вас есть подписка Dynatrace Platform Subscription, обратитесь к документации Dynatrace Platform Subscription (DPS), the licensing model for all Dynatrace capabilities.").

Чтобы начать использование Dynatrace, [свяжитесь с отделом продаж Dynatrace](https://www.dynatrace.com/contact/). Ваш торговый представитель предоставит вам дополнительные сведения.

Данная страница предназначена исключительно для информационных целей. К любому использованию продуктов или услуг Dynatrace применяются условия бесплатного пробного предложения Dynatrace и/или вашей лицензии Dynatrace.

### Жизненный цикл лицензии Dynatrace

Ознакомьтесь с жизненным циклом лицензий Dynatrace SaaS и узнайте, как лицензии влияют на потребление сервисов Dynatrace.### Мониторинг приложений и инфраструктуры (единицы хостов)

Каждый хост, отслеживаемый OneAgent в вашей среде, потребляет единицы хостов.### Единицы данных Davis (DDU)

Единицы данных Davis (DDU) обеспечивают простой способ лицензирования определённых возможностей (пользовательские метрики, трассировки, мониторинг логов и пользовательские события) на платформе Dynatrace..")### Мониторинг цифрового опыта (единицы DEM)

Dynatrace Synthetic Monitoring, Real User Monitoring и Session Replay потребляют единицы DEM.### Мониторинг безопасности приложений (единицы ASU)

Dynatrace Application Security потребляет единицы Application Security (ASU).
