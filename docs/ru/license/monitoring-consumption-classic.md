# Dynatrace classic licensing

# Классическое лицензирование Dynatrace

* Classic
* Чтение: 1 мин
* Опубликовано 30 марта 2021 г.

Потребление мониторинга при классическом лицензировании Dynatrace основано на различных единицах мониторинга, которые потребляет ваша среда Dynatrace по мере использования возможностей платформы Dynatrace вашей организацией. Если у вас есть подписка Dynatrace Platform Subscription, обратитесь к [документации Dynatrace Platform Subscription (DPS)](/docs/license "About Dynatrace Platform Subscription (DPS), the licensing model for all Dynatrace capabilities.").

Чтобы начать использование Dynatrace, [свяжитесь с отделом продаж Dynatrace](https://www.dynatrace.com/contact/). Ваш торговый представитель предоставит вам дополнительные сведения.

Данная страница предназначена исключительно для информационных целей. К любому использованию продуктов или услуг Dynatrace применяются условия бесплатного пробного предложения Dynatrace и/или вашей лицензии Dynatrace.

[### Жизненный цикл лицензии Dynatrace

Ознакомьтесь с жизненным циклом лицензий Dynatrace SaaS и узнайте, как лицензии влияют на потребление сервисов Dynatrace.](/docs/license/license-lifecycle "Understand your Dynatrace DPS or Classic license lifecycle, and how it affects your consumption of Dynatrace services.")[### Мониторинг приложений и инфраструктуры (единицы хостов)

Каждый хост, отслеживаемый OneAgent в вашей среде, потребляет единицы хостов.](/docs/license/monitoring-consumption-classic/application-and-infrastructure-monitoring "Understand how Dynatrace application and infrastructure monitoring consumption is calculated based on host units.")[### Единицы данных Davis (DDU)

Единицы данных Davis (DDU) обеспечивают простой способ лицензирования определённых возможностей (пользовательские метрики, трассировки, мониторинг логов и пользовательские события) на платформе Dynatrace.](/docs/license/monitoring-consumption-classic/davis-data-units "Understand how Dynatrace monitoring consumption is calculated based on Davis data units (DDU).")[### Мониторинг цифрового опыта (единицы DEM)

Dynatrace Synthetic Monitoring, Real User Monitoring и Session Replay потребляют единицы DEM.](/docs/license/monitoring-consumption-classic/digital-experience-monitoring-units "Understand how Dynatrace Digital Experience Monitoring consumption is calculated based on DEM units.")[### Мониторинг безопасности приложений (единицы ASU)

Dynatrace Application Security потребляет единицы Application Security (ASU).](/docs/license/monitoring-consumption-classic/application-security-units "Understand how Dynatrace Application Security and Runecast SPM consumption are calculated.")
