# Анализ производительности

* Classic
* Руководство
* Чтение: 6 мин
* Опубликовано 19 июля 2017

Раздел **Performance analysis** отображает ряд метрик производительности вашего приложения. Просто разверните раздел **Performance analysis** инфографики на [странице обзора приложения](/docs/observe/digital-experience/web-applications/analyze-and-use/introduction-to-application-overview "Ознакомьтесь с обзором опций анализа, доступных на странице обзора приложения."), чтобы увидеть параметры анализа производительности.

![Performance analysis](https://dt-cdn.net/images/performance-analysis-2346-018950fbc4.png)

## Работа с инфографикой

Каждая область инфографики в верхней части является кликабельной и предоставляет доступ к более детальной информации по каждой метрике. При выборе части инфографики левый раздел непосредственно под ней отображает различные данные, соответствующие выбранной части.

Левая часть инфографики анализа производительности показывает разбивки трафика вашего приложения по измерениям: тип браузера, тип пользователя и географический регион. В центре вы можете видеть различные типы действий, рейтинг Apdex и раздел ошибок, а в правой части отображаются разделы ресурсов и сервисов. По умолчанию в каждом разделе отображается главный результат для каждого измерения. Отдельные разделы инфографики кратко описаны ниже.

Топ браузеров

Если вы выберете раздел **Top browser** инфографики, вы увидите **Browser breakdown** слева под инфографикой. Здесь показан трафик по настольным браузерам, мобильным браузерам, [синтетическому мониторингу](/docs/observe/digital-experience/synthetic-monitoring "Узнайте о синтетическом мониторинге и о том, как создать монитор для одного URL, кликпас браузера или HTTP-монитор.") и другим категориям.

Выбрав **Analyze performance** в правом нижнем углу раздела **Browser breakdown**, вы перейдёте на [страницу многомерного анализа](/docs/observe/digital-experience/web-applications/analyze-and-use/multi-dimensional-analysis "Узнайте, как Dynatrace Real User Monitoring позволяет глубоко анализировать действия пользователей по множеству измерений."), где можно выполнить многомерный анализ с точки зрения **Browsers**.

![Browsers](https://dt-cdn.net/images/browsers-2278-df044b9e0a.png)

Топ типов пользователей

Выбрав **Top user type**, в разделе **User type** под инфографикой вы можете увидеть количество действий в минуту, действия Load и XHR, а также ошибки JavaScript для **Real users** по сравнению с **Robots** (например, Googlebot) и [**Synthetic**](/docs/observe/digital-experience/synthetic-monitoring "Узнайте о синтетическом мониторинге и о том, как создать монитор для одного URL, кликпас браузера или HTTP-монитор."). Полоса в верхней части раздела **User type** является визуальным представлением этого распределения.

Выбрав **Analyze performance** в правом нижнем углу раздела **Browser breakdown**, вы перейдёте на [страницу многомерного анализа](/docs/observe/digital-experience/web-applications/analyze-and-use/multi-dimensional-analysis "Узнайте, как Dynatrace Real User Monitoring позволяет глубоко анализировать действия пользователей по множеству измерений."), где можно выполнить многомерный анализ с точки зрения **User type**.

![User types](https://dt-cdn.net/images/user-types-2289-a96765e8cd.png)

Разбивка по геолокации

Выбрав **Geolocation breakdown**, вы можете просмотреть географические местоположения с наибольшим трафиком. Дополнительные сведения доступны на [карте мира](/docs/observe/digital-experience/web-applications/analyze-and-use/world-map-view "Узнайте, как представление карты мира предоставляет информацию о рейтингах Apdex, действиях пользователей, длительности действий и ошибках JavaScript."), к которой можно перейти, выбрав **View full world map**.

Выбрав **Analyze performance** в правом нижнем углу этого раздела, вы перейдёте на [страницу многомерного анализа](/docs/observe/digital-experience/web-applications/analyze-and-use/multi-dimensional-analysis "Узнайте, как Dynatrace Real User Monitoring позволяет глубоко анализировать действия пользователей по множеству измерений."), где можно выполнить многомерный анализ с точки зрения **Geolocations**.

![Geolocations](https://dt-cdn.net/images/geolocations-2335-e51f88186e.png)

Действия Load, XHR и пользовательские действия

[Действия пользователей](/docs/observe/digital-experience/rum-concepts/user-actions "Узнайте, что такое действия пользователей и как они помогают понять, что пользователи делают с вашим приложением.") разделены на действия **Load** и действия **XHR** (или пользовательские действия, если они доступны).

Если вы выберете часть инфографики, посвящённую действиям пользователей, вы сможете увидеть **Impact of user actions on performance**. Отображаемые графики показывают, среди прочего, метрику [Visually complete](/docs/observe/digital-experience/rum-concepts/user-action-metrics "Узнайте, какие метрики Dynatrace рассчитывает для действий пользователей и что означает каждая метрика.") для действий загрузки и [Response end](/docs/observe/digital-experience/rum-concepts/user-action-metrics "Узнайте, какие метрики Dynatrace рассчитывает для действий пользователей и что означает каждая метрика.") для действий XHR.

Вы также можете использовать селектор метрик для отображения **Slowest 10%**, **Median** или **Fastest 10%** действий пользователей.

Выбрав **Analyze performance** в правом нижнем углу этого раздела, вы можете [анализировать производительность приложения и действия пользователей по нескольким измерениям](/docs/observe/digital-experience/web-applications/analyze-and-use/multi-dimensional-analysis "Узнайте, как Dynatrace Real User Monitoring позволяет глубоко анализировать действия пользователей по множеству измерений.").

![Performance analysis](https://dt-cdn.net/images/performance-analysis-marked-2346-a63d69b1ca.png)

Рейтинг Apdex

Dynatrace использует [рейтинги Apdex](/docs/observe/digital-experience/rum-concepts/scores-and-ratings/apdex-ratings "Узнайте, как Dynatrace использует Apdex для измерения удовлетворённости пользователей производительностью приложения.") для расчёта удовлетворённости пользователей конкретными приложениями. Выбрав **Apdex rating**, вы можете просмотреть удовлетворённость пользователей в указанном временном диапазоне.

![Apdex](https://dt-cdn.net/images/apdex-2275-4f37729a71.png)

Ресурсы

Выберите **Resources** в инфографике для просмотра сведений о ресурсах, от которых зависит ваше приложение. Dynatrace использует имена хостов провайдеров загружаемых ресурсов для категоризации ресурсов контента на **3rd party resources**, **CDN resources** или **1st party resources**. Вы можете исследовать длительность действий по типу ресурса, а также сравнить её с предыдущим временным диапазоном.

На вкладке **1st party resources** вы также можете искать тренды во внутренних ресурсах — ресурсы **Scripts**, **Images** и **CSS** отслеживаются и отображаются отдельно.

Из этого раздела вы также можете перейти к [настройкам обнаружения провайдеров](/docs/observe/digital-experience/web-applications/additional-configuration/configure-third-party-and-cdn-content-detection-web "Определите вручную сторонних провайдеров и CDN-провайдеров наряду с автоматически обнаруженными провайдерами для ваших веб-приложений."), где можно вручную добавить провайдера.

![Resources](https://dt-cdn.net/images/resources-2279-8542dfb13e.png)

Сервисы

Выберите раздел **Services** инфографики для просмотра сервисов, поддерживающих ваше приложение, в разделе **Called services**. Если вы выберете **View service flow** в правом нижнем углу этого раздела, вы сможете [просмотреть последовательность вызовов сервисов, инициированных каждым сервисом](/docs/observe/application-observability/services-classic/service-flow "Узнайте, как Dynatrace может помочь отследить последовательность вызовов сервисов, инициированных каждым запросом сервиса в вашей среде."). Dynatrace позволяет [получить доступ к потоку сервисов как для приложения, так и для действия пользователя](/docs/observe/digital-experience/web-applications/analyze-and-use/service-flows-for-applications-and-user-actions "Узнайте, как получить доступ к потокам сервисов для приложений и действий пользователей.").

![Services](https://dt-cdn.net/images/services-2255-23e1ba2133.png)

### Доступность приложения

Если у вас настроен один или несколько [синтетических мониторов](/docs/observe/digital-experience/web-applications/initial-setup/define-your-applications-via-the-my-web-application-placeholder "Узнайте, как определить ваши приложения с использованием предлагаемого, ручного подхода или правил обнаружения приложений.") для проверки доступности вашего приложения, вы можете видеть доступность приложения непосредственно на странице обзора приложения в разделе **Availability**. График доступности показывает агрегированное представление всех синтетических мониторов выбранного приложения. Вы можете выбрать **View full details** для просмотра подробностей, связанных с простоями, и перейти непосредственно к деталям синтетического монитора (ваши настройки [временного диапазона анализа](/docs/analyze-explore-automate/dashboards-classic/dashboards/dashboard-timeframe "Узнайте о временном диапазоне дашборда Dynatrace и настройках зон управления.") сохранятся) для дальнейшего анализа.

![Availability](https://dt-cdn.net/images/availability-2274-52399f442d.png)

## Составные метрики времени отклика

Вы можете определить, например, является ли проблемой для вашего приложения время до получения первого байта, или следует сосредоточиться на воспринимаемой пользователем производительности загрузки, на которую указывают Speed index и Visually complete.

Выбрав **Analyze performance** в правом нижнем углу этого раздела, вы можете [анализировать производительность приложения и действия пользователей по нескольким измерениям](/docs/observe/digital-experience/web-applications/analyze-and-use/multi-dimensional-analysis "Узнайте, как Dynatrace Real User Monitoring позволяет глубоко анализировать действия пользователей по множеству измерений.").

## Основные ошибки

Список **Top errors** отображает наиболее часто возникающие [ошибки](/docs/observe/digital-experience/rum-concepts/user-and-error-events#error "Узнайте о пользовательских и ошибочных событиях и типах пользовательских и ошибочных событий, фиксируемых Dynatrace.") по типу: **JavaScript**, **Request** или **Custom**. Вы можете [настроить ошибки](/docs/observe/digital-experience/web-applications/additional-configuration/configure-errors "Настройте приложение для захвата или игнорирования ошибок запросов, пользовательских ошибок и ошибок JavaScript.") по типу или индивидуально.

Чтобы просмотреть **Top errors**

1. Перейдите в **Web**.
2. Выберите приложение, которое хотите проанализировать, и прокрутите вниз до **Top errors**.

## Топ-3 действия пользователей

Список **Top 3 user actions** показывает три самых медленных действия пользователей. Действия сортируются по **приоритету** (действия с высоким приоритетом отображаются первыми) и **общему затраченному времени** (длительность, умноженная на количество действий). Метрика **общего затраченного времени** ценна тем, что учитывает частоту вызова действия. Например, медленное действие, вызываемое часто, оказывает большее влияние на производительность, чем медленное действие, которое вызывается редко.

Если вы выберете действие пользователя, вы будете перенаправлены на [страницу обзора действия пользователя](/docs/observe/digital-experience/web-applications/analyze-and-use/analyze-individual-user-actions "Узнайте, как получить доступ к страницам деталей действий пользователей и анализировать действия пользователей."), где можно дополнительно проанализировать конкретное действие.

Если вы выберете **View full details** в правом нижнем углу раздела, вы можете [анализировать производительность приложения и действия пользователей по нескольким измерениям](/docs/observe/digital-experience/web-applications/analyze-and-use/multi-dimensional-analysis "Узнайте, как Dynatrace Real User Monitoring позволяет глубоко анализировать действия пользователей по множеству измерений.").
