# Анализ отдельных действий пользователя

* Classic
* Практическое руководство
* Чтение: 3 мин
* Опубликовано 19 июля 2017

Страницы с детальной информацией о действиях пользователя обеспечивают быстрый доступ ко всем релевантным данным, предоставляя всю необходимую информацию для понимания того, что влияет на производительность каждого действия пользователя.

Чтобы перейти к страницам с детальной информацией о действиях пользователя:

1. Перейдите в раздел **Web**.
2. Выберите приложение.
3. Прокрутите вниз до раздела **Top user actions** и выберите **View full details**.
4. Выберите действие из списка **Key user actions** или **Top 100 user actions**.

В зависимости от того, какое действие пользователя вы хотите проанализировать, вы также можете напрямую выбрать одно из действий, показанных в разделе **Top user actions**.

На странице с детальной информацией о действии пользователя выбирайте области инфографики в верхней части страницы для перехода к соответствующему разделу, связанному с каждой сводной метрикой, или введите строку в поле **Filter user types** для просмотра данных, специфичных для реальных пользователей или синтетического мониторинга.

## Диаграмма разбивки факторов и каскадный анализ

При анализе действий пользователя один из главных вопросов — на каком уровне потребляется наибольшее время отклика. Больше времени было потрачено на фронтенде (в основном в браузере), в сети или на сервере? Диаграмма **Contributor breakdown** даёт быстрый обзор времени, затраченного на фронтенде, в сети и на сервере. Для полного [каскадного анализа](/docs/observe/digital-experience/web-applications/analyze-and-use/waterfall-analysis "Узнайте, как анализировать все данные мониторинга действий пользователя с помощью каскадного анализа.") отдельных действий пользователя выберите **View analysis in waterfall chart**, чтобы увидеть, какие ресурсы влияют на продолжительность действия.

![User actions](https://dt-cdn.net/images/actions-2285-9af9ca267d.png)

## Рейтинг Apdex

Dynatrace использует [рейтинги Apdex](/docs/observe/digital-experience/rum-concepts/scores-and-ratings/apdex-ratings "Узнайте, как Dynatrace использует Apdex для измерения удовлетворённости пользователей производительностью приложения.") для расчёта удовлетворённости пользователей. Выбрав **Apdex rating**, вы можете просмотреть удовлетворённость пользователей за указанный период времени для конкретного действия пользователя.

![Apdex](https://dt-cdn.net/images/apdex-actions-2291-8bcf39ab96.png)

## Ошибки

Выберите **Errors** для просмотра анализа [ошибок](/docs/observe/digital-experience/rum-concepts/user-and-error-events#error "Узнайте о событиях пользователей и ошибок, а также о типах событий, фиксируемых Dynatrace.") по двум измерениям: тип ошибки (JavaScript, запрос и пользовательские ошибки) и происхождение (собственные, сторонние или CDN).

Вы также можете просмотреть наиболее часто возникающие ошибки JavaScript в этом действии пользователя за определённый [период времени](/docs/analyze-explore-automate/dashboards-classic/dashboards/dashboard-timeframe "Узнайте о настройках периода времени и зоны управления на панели инструментов Dynatrace."). Выберите **Analyze errors** для перехода на [страницу многомерного анализа](/docs/observe/digital-experience/web-applications/analyze-and-use/multi-dimensional-analysis "Узнайте, как мониторинг реальных пользователей Dynatrace позволяет глубоко анализировать действия пользователей по множеству измерений."), где вы можете выполнить многомерный анализ с точки зрения **Errors** в сочетании с измерениями типа, контекста или происхождения. На этой странице, выбрав конкретную ошибку из списка **Error**, вы можете перейти на [страницу детальной информации об ошибке](/docs/observe/digital-experience/session-segmentation/new-user-sessions#error-details-page "Узнайте о сегментации пользовательских сеансов и атрибутах фильтрации.").
