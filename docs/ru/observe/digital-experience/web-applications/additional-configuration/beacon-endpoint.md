# Настройка конечной точки beacon для веб-приложений

# Настройка конечной точки beacon для веб-приложений

* Classic
* Практическое руководство
* Чтение: 6 мин
* Обновлено 21 ноября 2025 г.

RUM JavaScript отправляет RUM-beacon для передачи собранных данных в Dynatrace. По умолчанию конечная точка beacon зависит от метода внедрения, используемого для вашего приложения.

* **Приложения с автоматическим внедрением:** Когда [RUM JavaScript внедряется автоматически](../initial-setup/rum-injection.md "Настройте автоматическое внедрение RUM JavaScript на страницы ваших приложений"), beacon отправляются обратно на ваш веб-сервер или сервер приложений с использованием корне-относительного URL, где последний сегмент пути имеет префикс `rb_` (например, `/rb_xxxxxxxxxx` или `/myapplication/rb_xxxxxxxxxx`). Конечная точка beacon предоставляется OneAgent, который перехватывает и пересылает RUM-beacon.
* **Безагентные приложения:** Если вы выбрали [безагентный мониторинг](../initial-setup/set-up-agentless-real-user-monitoring.md "Настройте безагентный мониторинг для ваших веб-приложений."), данные отправляются на конечную точку beacon, которая является частью инфраструктуры Dynatrace SaaS.

Обычно менять конечную точку beacon по умолчанию не требуется, но существуют определённые сценарии, в которых может потребоваться альтернативная конфигурация конечной точки beacon. Например:

* Если ваша инфраструктура блокирует beacon приложения с автоматическим внедрением из-за их URL-пути по умолчанию.
* Если вы хотите, чтобы трафик мониторинга приложения с автоматическим внедрением обходил ваш CDN.
* Если вы предпочитаете, чтобы RUM-beacon не обрабатывались на веб-сервере или сервере приложений, на котором размещено ваше приложение.

В следующих разделах описаны альтернативные конфигурации конечной точки beacon, которые позволяют учесть эти и подобные ограничения.

Конфигурации конечной точки beacon, описанные на этой странице, не влияют на корреляцию между действиями пользователей и распределёнными трассировками. Однако проверьте раздел [Поддержка технологий](../../../../ingest-from/technology-support.md#rum-auto-injection "Технические подробности о поддержке Dynatrace конкретных платформ и фреймворков разработки."), чтобы узнать, поддерживается ли мониторинг реальных пользователей для вашей технологии.

## Приложение с автоматическим внедрением: изменение URL конечной точки beacon

В зависимости от вашей инфраструктуры и её конфигурации beacon могут не проходить по автоматически выбранному URL-пути и, следовательно, не обрабатываться OneAgent. Для решения этой проблемы можно изменить часть URL конечной точки beacon, которая предшествует префиксу `rb_`.

Чтобы изменить URL конечной точки beacon для приложения с автоматическим внедрением:

1. Перейдите в раздел **Web**.
2. Выберите приложение, которое хотите настроить.
3. В правом верхнем углу страницы обзора приложения выберите **Ещё** (**...**) > **Редактировать**.
4. В настройках приложения выберите **Общие настройки** > **Конечная точка beacon**.
5. В выпадающем списке **Тип** выберите **OneAgent**.
6. В поле **URL** введите относительный URL конечной точки beacon.

Обратите внимание, что нельзя удалить сегмент пути с префиксом `rb_`, так как он необходим OneAgent для идентификации RUM-beacon.

#### Примеры

Следующие примеры предполагают, что по умолчанию RUM-beacon отправляются на `/rb_bf12345abc`.

* **Корне-относительный URL:** Если задать в поле **URL** значение `/custompath`, beacon будут отправляться на `/custompath/rb_bf12345abc`.
* **Относительный URL:** Если задать в поле **URL** значение `./`, то URL, по которому RUM JavaScript отправляет beacon, будет относительным к текущей странице. Например:

  + Если текущая страница — `/shop/index.html`, beacon отправляются на `/shop/rb_bf12345abc`.
  + Если текущая страница — `/account/dashboard/`, beacon отправляются на `/account/dashboard/rb_bf12345abc`.

## Приложение с автоматическим внедрением: отправка beacon в инфраструктуру Dynatrace SaaS

Если вы хотите, чтобы RUM-beacon приложения с автоматическим внедрением обрабатывались инфраструктурой Dynatrace SaaS вместо OneAgent, выполните следующие шаги.

1. Перейдите в раздел **Web**.
2. Выберите приложение, которое хотите настроить.
3. В правом верхнем углу страницы обзора приложения выберите **Ещё** (**...**) > **Редактировать**.
4. В настройках приложения выберите **Общие настройки** > **Конечная точка beacon**.
5. В выпадающем списке **Тип** выберите **Cluster ActiveGate**.

Для этой конфигурации Dynatrace применяет [список разрешённых источников beacon](configure-beacon-domain-allowlist.md "Укажите источники, из которых должны приниматься кросс-доменные RUM-beacon.") к RUM-beacon вашего приложения.

## Приложение с автоматическим внедрением: отправка beacon на другой веб-сервер

По умолчанию RUM-beacon приложения с автоматическим внедрением обрабатываются одной из групп процессов, на которой размещено ваше приложение. В качестве альтернативы beacon могут обрабатываться на любом другом инструментированном веб-сервере или сервере приложений из списка технологий в разделе [Поддержка технологий — мониторинг реальных пользователей — веб-серверы и приложения](../../../../ingest-from/technology-support.md#rum-auto-injection "Технические подробности о поддержке Dynatrace конкретных платформ и фреймворков разработки.").

Чтобы отправить beacon приложения с автоматическим внедрением на другой инструментированный сервер:

1. Во внедрённом RUM JavaScript найдите `reportUrl` и скопируйте последний сегмент URL-пути, который имеет префикс `rb_`.
2. Добавьте это значение к URL инструментированного веб-сервера или сервера приложений.

   Например, если последний сегмент `reportUrl` — `/rb_abcdefghi`, а URL сервера — `http://www.my-server.com`, результирующий URL конечной точки beacon будет `http://www.my-server.com/rb_abcdefghi`.
3. Перейдите в раздел **Web**.
4. Выберите приложение, которое хотите настроить.
5. В правом верхнем углу страницы обзора приложения выберите **Ещё** (**...**) > **Редактировать**.
6. В настройках приложения выберите **Общие настройки** > **Конечная точка beacon**.
7. В выпадающем списке **Тип** выберите **OneAgent**.
8. В поле **URL** введите конечную точку beacon, определённую на шаге 2.
9. Включите **Отправлять данные beacon через CORS**.

Для этой конфигурации Dynatrace применяет [список разрешённых источников beacon](configure-beacon-domain-allowlist.md "Укажите источники, из которых должны приниматься кросс-доменные RUM-beacon.") к RUM-beacon вашего приложения.

## Безагентное приложение: отправка beacon на веб-сервер

Вместо использования инфраструктуры Dynatrace SaaS в качестве конечной точки beacon для безагентного приложения можно использовать любой инструментированный веб-сервер или сервер приложений из списка технологий в разделе [Поддержка технологий — мониторинг реальных пользователей — веб-серверы и приложения](../../../../ingest-from/technology-support.md#rum-auto-injection "Технические подробности о поддержке Dynatrace конкретных платформ и фреймворков разработки.").

Чтобы отправить beacon безагентного приложения на инструментированный сервер:

1. Перейдите в раздел **Web**.
2. Выберите любое приложение с **автоматическим внедрением**, которое не использует одну из пользовательских конфигураций конечной точки beacon, описанных на этой странице.

   На этом шаге выбирайте не безагентное приложение, которое вы настраиваете, а другое приложение с **автоматическим внедрением**, имеющее конфигурацию конечной точки beacon по умолчанию. Если у вас нет приложения с автоматическим внедрением, временно создайте его, как описано в разделе [Определение приложений для мониторинга реальных пользователей | Подход с правилами обнаружения приложений](../initial-setup/define-your-applications-via-the-my-web-application-placeholder.md#application-detection-rules "Узнайте, как определить свои приложения, используя рекомендуемый, ручной подход или подход с правилами обнаружения приложений."). Затем вы можете [удалить](delete-application-web.md "Удалите веб-приложения через веб-интерфейс Dynatrace или API.") это временное приложение.
3. В правом верхнем углу страницы обзора приложения выберите **Ещё** (**...**) > **Редактировать**.
4. В настройках приложения выберите **Внедрение** > **Ручная вставка**.
5. В разделе **OneAgent JavaScript tag** найдите `reportUrl` в предоставленном фрагменте кода и скопируйте его значение.
6. Добавьте значение `reportUrl` к URL инструментированного веб-сервера или сервера приложений.

   Например, если значение `reportUrl` — `/rb_abcdefghi`, а URL сервера — `http://www.my-server.com`, результирующий URL конечной точки beacon будет `http://www.my-server.com/rb_abcdefghi`.
7. Перейдите в раздел **Web**.
8. Выберите безагентное приложение, которое хотите настроить.
9. В правом верхнем углу страницы обзора приложения выберите **Ещё** (**...**) > **Редактировать**.
10. В настройках приложения выберите **Общие настройки** > **Конечная точка beacon**.
11. В выпадающем списке **Тип** выберите **OneAgent**.
12. В поле **URL** введите конечную точку beacon, определённую на шаге 6.
13. Включите **Отправлять данные beacon через CORS**.

Для этой конфигурации Dynatrace применяет [список разрешённых источников beacon](configure-beacon-domain-allowlist.md "Укажите источники, из которых должны приниматься кросс-доменные RUM-beacon.") к RUM-beacon вашего приложения (то же самое выполняется для конфигурации конечной точки beacon по умолчанию для всех безагентных приложений).
