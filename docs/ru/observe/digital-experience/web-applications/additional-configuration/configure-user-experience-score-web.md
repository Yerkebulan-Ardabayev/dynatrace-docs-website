# Изменение пороговых значений оценки пользовательского опыта для веб-приложений


* How-to guide
* Чтение займёт 1 минуту
* Опубликовано 27 января 2023 г.

[Оценка пользовательского опыта](../../rum-concepts/scores-and-ratings/user-experience-score.md "Оценка пользовательского опыта — это метрика, используемая для классификации пользовательских сессий как «Неудовлетворительные», «Приемлемые» или «Удовлетворительные».") — это важная метрика, отражающая общую производительность, удобство использования и обнаруженные ошибки каждой сессии. Вы можете настроить пороговые значения оценки пользовательского опыта для ваших приложений.

Чтобы настроить пороговые значения оценки пользовательского опыта

1. Перейдите в **Настройки** > **Мониторинг веб и мобильных приложений** > **Оценка пользовательского опыта**.

   ![User experience score settings](https://dt-cdn.net/images/user-xp-score-settings-2292-03a4586d9e.png)
2. Необязательно Включите параметр **Если последнее действие пользователя в сессии классифицировано как «Неудовлетворительное», классифицировать всю сессию как «Неудовлетворительную»**, чтобы оценивать неудовлетворительную сессию по её последнему действию.
3. Необязательно Включите параметр **Учитывать rage-клики / rage-тапы при расчёте оценки**, чтобы включить [rage-события](../../rum-concepts/user-and-error-events.md#rage-event "Узнайте о событиях пользователя и ошибках, а также о типах таких событий, фиксируемых Dynatrace.") при определении оценки пользовательского опыта.
4. В поле **Порог неудовлетворительного пользовательского опыта** введите процент **Неудовлетворительных** действий пользователя, при превышении которого сессия считается **Неудовлетворительной**.
5. В поле **Порог удовлетворительного пользовательского опыта** введите минимальный процент **Удовлетворительных** действий пользователя, необходимый для признания сессии **Удовлетворительной**.
