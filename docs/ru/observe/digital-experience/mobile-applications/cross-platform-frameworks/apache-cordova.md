# Инструментирование мобильных приложений с помощью плагина Dynatrace Cordova

* Classic
* How-to guide
* 5-min read
* Updated on Oct 12, 2022

Гибридные мобильные приложения сочетают преимущества нативных мобильных приложений, такие как доступ к оборудованию телефона и распространение через магазин приложений, с гибкостью и платформенной независимостью современных веб-технологий.

Плагин Dynatrace Cordova позволяет инструментировать приложения, построенные на основе Apache Cordova. Это включает приложения, использующие Capacitor, Ionic или IBM Mobile First. Плагин Cordova упрощает мониторинг гибридных приложений с помощью следующих технологий:

* OneAgent for Mobile, который отслеживает нативную часть вашего приложения
* RUM JavaScript, который отслеживает производительность веб-части вашего приложения

Подробную техническую документацию см. на странице [плагина Dynatrace Cordova](https://www.npmjs.com/package/@dynatrace/cordova-plugin) на сайте npm.

## Настройка мониторинга гибридного мобильного приложения

1. В Dynatrace перейдите в **Mobile**.
2. Выберите **Create mobile app**.
3. Введите имя для вашего приложения и выберите **Create mobile app**. Откроется страница настроек приложения.
4. В настройках приложения выберите **Instrumentation wizard** и выберите платформу вашего мобильного приложения, например, **Cordova**.

   ![Создание приложения в Dynatrace](https://dt-cdn.net/images/setup-wizard-1764-763c621790.png)
5. Введите команду `cordova plugin add @dynatrace/cordova-plugin --save` в корневом каталоге вашего проекта на базе Cordova для установки плагина Dynatrace Cordova.
6. Выберите **Monitor the web view** для включения мониторинга веб-представления.
7. Выберите **Download dynatrace.config.js** и добавьте загруженный файл в корень рабочего пространства вашего приложения Cordova рядом с файлом `config.xml`.
8. Соберите проект с помощью `cordova build android` или `cordova build ios`.

При сборке приложения для Android или iOS плагин Cordova автоматически инструментирует ваше приложение с помощью OneAgent for Mobile, который отслеживает нативную часть вашего гибридного приложения. Это обеспечивает видимость запусков приложения, пользовательских действий, отчётов о сбоях и метаданных устройства, таких как модель и версия ОС. Данные мониторинга, собранные OneAgent for Mobile, инкапсулируются в мобильном приложении в Dynatrace.

Для захвата пользовательских действий в веб-представлениях Dynatrace использует [RUM JavaScript](../../web-applications/initial-setup/snippet-formats.md "Выберите формат фрагмента RUM JavaScript, который лучше всего подходит для вашего конкретного сценария использования"), который также используется для всех веб-приложений, [настроенных для безагентного мониторинга Dynatrace](../../web-applications/initial-setup/set-up-agentless-real-user-monitoring.md "Настройте безагентный мониторинг для ваших веб-приложений."). Плагин Cordova автоматически внедряет RUM JavaScript непосредственно в ваши HTML-источники. Данные мониторинга, собранные RUM JavaScript, инкапсулируются в веб-приложении в Dynatrace. Это приложение получает все данные мониторинга и генерирует токен API, чтобы плагин Cordova мог получать RUM JavaScript из REST API.

![Инструментирование приложения Cordova](https://dt-cdn.net/images/instrumentcordovaapp-1913-f1c90c1409.png)

## Начало мониторинга и анализа вашего приложения

1. Запустите ваше гибридное приложение в эмуляторе или на тестовом устройстве и пройдите сценарии использования для генерации трафика, который Dynatrace сможет отслеживать. По завершении закройте приложение, чтобы убедиться, что все данные отправлены в Dynatrace. Обратите внимание, что OneAgent for Mobile может кэшировать данные до 2 минут.
2. В Dynatrace перейдите в **Mobile**.
3. Выберите ваше новое гибридное приложение из списка приложений.

   Все метрики анализа производительности, относящиеся к вашему приложению, отображаются на странице обзора вашего гибридного приложения. Раздел **Hybrid app** отображает общие сведения о производительности веб-части вашего приложения.

   ![Анализ мобильных приложений](https://dt-cdn.net/images/analyze-mobile-app-1954-6c4199ccdd.png)
4. Выберите **View web application** для анализа данных производительности, которые Dynatrace собирает через внедрённый RUM JavaScript.

   Страница обзора веб-приложения содержит пользовательские действия, захваченные в веб-представлении, поэтому здесь вы найдёте большую часть пользовательских действий вашего приложения.

   ![Анализ веб-приложений](https://dt-cdn.net/images/analyze-web-app-1954-d5e81e0ff9.png)

   Данные производительности разделены между двумя представлениями приложений. Однако Dynatrace объединяет все обнаруженные пользовательские действия из этих представлений приложений в одного пользователя и одну пользовательскую сессию.
5. Перейдите в ![Session Segmentation](https://dt-cdn.net/images/session-segmentation-512-5278e8fa16.png "Session Segmentation") **Session Segmentation** для доступа к странице **User sessions**. В поле **Filtered by** введите **Application** и выберите ваше гибридное мобильное приложение, что позволит сосредоточить анализ на вашем вновь созданном приложении.
6. Выберите пользователя по умолчанию, созданного для вашего гибридного приложения, из столбца **User**. Обратите внимание, что этот пользователь посещал два приложения — мобильное и веб-приложение.

   ![Анализ пользователя](https://dt-cdn.net/images/analyze-user-1950-e23a310611.png)
7. Выберите пользовательскую сессию. Вы должны увидеть, что все пользовательские действия, захваченные в мобильной и веб-части вашего гибридного приложения, были объединены в одну сессию вместе с метаданными, которые позволяют Dynatrace показывать, какие пользовательские действия были захвачены OneAgent for Mobile, а какие — RUM JavaScript.

   Следующая диаграмма показывает последовательность пользовательских действий, время их возникновения в ходе пользовательской сессии и время ожидания между действиями.

   ![Объединённая пользовательская сессия](https://dt-cdn.net/images/joined-user-session-1571-0684cc8268.png)

Отсюда вы можете продолжить анализ пользовательского опыта. Вы можете добавить дополнительные фильтры, чтобы сосредоточиться на сессиях, которые хотите рассмотреть подробнее, или перейти к каскадным диаграммам пользовательских действий веб- и мобильных сессий вашего приложения.
