# Настройка ключевых действий пользователей для мобильных приложений


* Classic
* How-to guide
* 1-min read
* Published Jan 30, 2023

Большинство приложений включают действия пользователей, которые особенно важны для успеха вашего цифрового бизнеса. Примерами таких действий являются регистрация, оформление заказа и поиск товаров. Подобные [ключевые действия пользователей](../../rum-concepts/user-actions.md#key-user-actions "Узнайте, что такое действия пользователей и как они помогают понять, что пользователи делают с вашим приложением.") могут занимать больше времени, чем другие, или к ним может предъявляться требование о выполнении в более короткие сроки.

С помощью функции ключевых действий пользователей вы можете [настраивать пороговые значения Apdex](../../rum-concepts/scores-and-ratings/apdex-ratings.md#key-user-actions "Узнайте, как Dynatrace использует Apdex для измерения удовлетворённости пользователей производительностью приложения.") для каждого из этих действий. Вы можете использовать эту функцию для мониторинга ключевых действий с помощью специальной плитки на дашборде и отслеживания исторических тенденций.

## Отмечение действия пользователя как ключевого

1. Перейдите в раздел **Frontend**.
2. Выберите приложение и прокрутите вниз до **Top 3 user actions** или **Top 3 actions**.
3. Выберите **View full details** или **Analyze performance**.
4. Прокрутите вниз до списка действий пользователей и выберите нужное действие.
   Откроется страница сведений о действии пользователя.
5. В правом верхнем углу страницы сведений о действии пользователя выберите **Mark as key user action**.

Вы можете определить до 500 ключевых действий пользователей на среду для всех ваших приложений и до 100 ключевых действий на приложение.

При достижении максимального предела ключевых действий пользователей рассмотрите возможность использования вычисляемых метрик (доступны для [веб-](../../web-applications/additional-configuration/rum-calculated-metrics-web.md "Создавайте вычисляемые метрики и пользовательские диаграммы на их основе для ваших веб-приложений."), [мобильных](rum-calculated-metrics-mobile.md "Создавайте вычисляемые метрики и пользовательские диаграммы на их основе для ваших мобильных приложений.") и [пользовательских приложений](../../custom-applications/additional-configuration/rum-calculated-metrics-custom.md "Создавайте вычисляемые метрики и пользовательские диаграммы на их основе для ваших пользовательских приложений.")), которые предлагают аналогичные возможности.

## Закрепление ключевого действия пользователя на дашборде

[Dashboards Classic](../../../../analyze-explore-automate/dashboards-classic.md "Узнайте, как создавать, управлять и использовать Dynatrace Dashboards Classic.")

1. Перейдите в раздел **Frontend**.
2. Выберите приложение и прокрутите вниз до **Top 3 user actions** или **Top 3 actions**.
3. Выберите **View full details** или **Analyze performance**.
4. Найдите нужное ключевое действие пользователя и выберите его.
   Откроется страница сведений о действии пользователя.
5. В верхней части страницы сведений о действии пользователя выберите **Pin to dashboard**. Подробнее см. в разделе [Закрепление плиток на дашборде](../../../../analyze-explore-automate/dashboards-classic/charts-and-tiles/pin-tiles-to-your-dashboard.md "Узнайте, как закрепить плитки на ваших дашбордах.").

## Настройка рейтинга Apdex для ключевого действия пользователя

Для изменения пороговых значений Apdex для ключевого действия пользователя:

1. Перейдите в раздел **Frontend**.
2. Выберите приложение и прокрутите вниз до **Top 3 user actions** или **Top 3 actions**.
3. Выберите **View full details** или **Analyze performance**.
4. Найдите нужное ключевое действие пользователя и выберите его.
   Откроется страница сведений о действии пользователя.
5. В правом верхнем углу страницы сведений о действии пользователя выполните одно из следующих действий:

   * Для веб-приложений выберите **More** (**…**) > **Edit** > **Key performance metric**.
   * Для мобильных и пользовательских приложений выберите ![Expand](https://dt-cdn.net/images/expandbutton-40-e1f11ff81d.png "Expand") (**Expand**) > **Edit** > **Key performance metric**.
6. Используйте ползунок для настройки пороговых значений Apdex.
