# Метрики действий пользователя


* Classic
* Explanation
* 8-min read
* Updated on Mar 20, 2023

Для веб-приложений Dynatrace вычисляет множество метрик [действий пользователя](user-actions.md "Узнайте, что такое действия пользователя и как они помогают понять, что пользователи делают с вашим приложением.").

Core web vitals

Следующие метрики являются основными веб-показателями (core web vitals) и доступны только для загрузочных действий:

* **Largest contentful paint**
* **Cumulative layout shift**
* **First input delay**

## Метрики контрольных точек

Метрики контрольных точек представляют события, происходящие в определённый момент времени в ходе действия пользователя. При этом они измеряются в секундах или миллисекундах, поскольку рассчитываются относительно времени начала действия пользователя. Иными словами, время начала действия пользователя служит точкой отсчёта, относительно которой измеряется время для метрик контрольных точек. Например, значение 47 мс для **Navigation start** означает, что это событие произошло через 47 мс после времени начала действия.

1

**Speed index** и **Visually complete** доступны только в браузерах, поддерживающих [mutation observers](https://developers.google.com/web/updates/2012/02/Detect-DOM-changes-with-Mutation-Observers): Microsoft Internet Explorer 11, Microsoft Edge 15+, Firefox 57+ и Google Chrome 61+.

2

**First paint** поддерживается во всех браузерах, кроме Internet Explorer.

3

**First contentful paint** доступен только для [браузеров на основе Chromium](https://www.chromium.org/Home).

4

**Largest contentful paint** доступен для браузеров на основе Chromium. Измеряется с помощью [API, предоставленных Google](https://web.dev/lcp/#measure-lcp-in-javascript).

5

Для браузеров на основе Chromium **First input delay** измеряется с помощью [API, предоставленных Google](https://web.dev/fid/#measure-fid-in-javascript). Для Firefox, Safari и Internet Explorer 9+ метрика измеряется с использованием реализации Dynatrace, которая прослушивает события типа `click`, `mousedown`, `keydown` или `touchstart` и вычисляет задержку с момента регистрации таких событий JavaScript-кодом RUM до момента их срабатывания.

## Метрики фаз

В отличие от метрик контрольных точек, метрики фаз подразумевают продолжительность.

1

Вместо метрики [**Total blocking time**](https://web.dev/tbt/) можно использовать метрику **Long tasks** — от введения метрики Total blocking time было решено отказаться из-за ряда технических ограничений. Также обратите внимание, что метрика **Blocking**, отображаемая в схеме водопада, не связана с **Total blocking time** и доступна только для определённых запросов.

## Ключевые метрики производительности

Из перечисленных выше метрик Dynatrace выделил ряд ключевых. Ключевые метрики производительности позволяют выбрать оптимальную метрику пользовательского опыта для каждого из ваших приложений. Эти ключевые метрики предоставляют ценные сведения о производительности, способствующие успеху вашего цифрового бизнеса. Ключевые метрики производительности включают:

* User action duration
* Visually complete
* Speed index
* DOM interactive
* Load event end
* Load event start

* HTML downloaded / Response end
* Time to first byte
* Largest contentful paint
* Cumulative layout shift[1](#fn-3-1-def)
* First input delay (только RUM web)

1

**Cumulative layout shift** — это оценка, измеряющая неожиданное смещение видимых элементов веб-страницы. Доступна для [браузеров на основе Chromium](https://www.chromium.org/Home) и измеряется с помощью [API, предоставленных Google](https://web.dev/cls/#measure-cls-in-javascript).

Dynatrace позволяет [выбрать подходящую метрику производительности для каждого условия приложения](../web-applications/analyze-and-use/work-with-key-performance-metrics.md "Узнайте, как использовать правильные ключевые метрики производительности для оптимизации данных пользовательского опыта для каждого из ваших приложений.").

## Отсутствующие измерения метрик

Измерения метрик могут отсутствовать в следующих случаях:

#### Visually complete

* Во время загрузочного действия пользователь переходит по другой ссылке, которая инициирует новое загрузочное действие до завершения исходного события загрузки страницы. Второе загрузочное действие прерывает вычисление **Visually complete** первого загрузочного действия, поэтому измерение недоступно.
  В более ранней версии **Visually complete** Dynatrace использовал продолжительность действия пользователя в качестве резервного измерения. Это искажало временные ряды, поэтому такие случаи теперь исключаются для обеспечения более точных агрегатов.
* Поздний редирект срабатывает непосредственно после загрузочного действия.
* В ходе вычисления достигается [тайм-аут Visually complete](../web-applications/analyze-and-use/how-to-use-visually-complete-and-speed-index-metrics.md#config-enhanced-vc "Узнайте, как использовать метрики «Visually complete» и «Speed index».").

#### Core web vitals

* API браузера не срабатывает и не предоставляет измерение метрики до окончания действия пользователя (см. метрику **User action duration**).
  В более ранних версиях Dynatrace Real User Monitoring эти значения отправлялись после завершения действия пользователя. Такие значения не были информативными и зачастую не были репрезентативными. Например, **First input delay (FID)** измеряет скорость отклика при загрузке страницы. Основным фактором высокого значения FID является выполнение JavaScript. Поэтому если пользователь ждал минуту после загрузки страницы, прежде чем выбрать новую ссылку, большая часть или всё выполнение JavaScript уже произошло бы, и полученное небольшое измерение FID не было бы репрезентативным значением для реальной загрузки страницы. Такие измерения искажали общий агрегат метрики.
* Браузер пользователя не поддерживает API, используемые для измерения core web vitals. Подробности о том, какие браузеры поддерживают Core web vitals, см. в разделе [Метрики контрольных точек](#milestone-metrics).
* Страница загружается или перезагружается в неактивной вкладке, и API браузера не сообщает значения.
* API браузера возвращает отрицательные значения.
