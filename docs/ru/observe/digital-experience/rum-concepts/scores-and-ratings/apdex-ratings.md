# Рейтинги Apdex

# Рейтинги Apdex

* Classic
* Описание
* Чтение: 3 мин
* Обновлено 27 января 2023

Dynatrace рассчитывает [рейтинги Apdex](https://en.wikipedia.org/wiki/Apdex), чтобы предоставить вам единую метрику, которая характеризует производительность вашего приложения и ошибки, влияющие на пользовательский опыт.

Apdex рассчитывается для каждого отдельного [действия пользователя](../user-actions.md "Узнайте, что такое действия пользователя и как они помогают понять, что пользователи делают с вашим приложением.") и для каждого приложения в целом. Таким образом, он обеспечивает быстрое понимание пользовательского опыта в вашем приложении.

Рейтинги Apdex по умолчанию в Dynatrace основаны на порогах, специфичных для приложения.

| Оценка | Уровень производительности |
| --- | --- |
| 0,94–1,0 | Отличный |
| 0,85–0,94 | Хороший |
| 0,7–0,85 | Удовлетворительный |
| 0,5–0,7 | Низкий |
| < 0,5 | Неприемлемый |

Рейтинги Apdex можно использовать как эталоны для сравнения двух приложений во времени, даже если пороги времени, настроенные для этих двух приложений, могут отличаться.

Приложения обычно состоят из множества различных типов действий пользователя. Например, может быть допустимо, чтобы сложный поиск занимал до 6 секунд, в то время как загрузка домашней страницы того же приложения должна занимать менее 2 секунд для обеспечения удовлетворённости пользователей. Такие различия можно учесть, настроив различные пороги Apdex для разных типов действий пользователя.

Дополнительную информацию о стандарте Apdex см. в разделе [Справочные материалы Apdex](https://www.apdex.org/index.php/documents/).

## Настройка параметров Apdex в Dynatrace

Вы можете настроить рейтинги Apdex в Dynatrace в соответствии с конкретными требованиями вашего приложения. После настройки они предоставляют быстрый и удобный способ оценки производительности всех отслеживаемых действий пользователя: значение `1.0` — идеально; значения ниже `0.5` — неприемлемы. Рекомендуется определить соответствующие пороги времени удовлетворённости пользователей и конфигурации влияния ошибок для каждого отслеживаемого действия пользователя.

Вы можете настроить параметры Apdex для вашего приложения и его ключевых действий пользователя.

### Настройка параметров Apdex для приложений

Вы можете изменить пороги Apdex для ваших [веб-приложений](../../web-applications/additional-configuration/configure-apdex-web.md "Настройте пороги производительности удовлетворённости пользователей для вашего веб-приложения и его ключевых действий."), [мобильных приложений](../../mobile-applications/additional-configuration/configure-apdex-mobile.md "Настройте пороги производительности удовлетворённости пользователей для вашего мобильного приложения и его ключевых действий.") и [пользовательских приложений](../../custom-applications/additional-configuration/configure-apdex-custom.md "Настройте пороги производительности удовлетворённости пользователей для вашего пользовательского приложения и его ключевых действий.").

### Настройка параметров Apdex для ключевых действий пользователя

Вы также можете изменить пороги Apdex для конкретного ключевого действия пользователя для ваших [веб-приложений](../../web-applications/additional-configuration/configure-key-user-actions-web.md "Отметьте действие пользователя как ключевое и настройте рейтинг Apdex для ключевых действий пользователя ваших веб-приложений."), [мобильных приложений](../../mobile-applications/additional-configuration/configure-key-user-actions-mobile.md "Отметьте действие пользователя как ключевое и настройте рейтинг Apdex для ключевых действий пользователя ваших мобильных приложений.") и [пользовательских приложений](../../custom-applications/additional-configuration/configure-key-user-actions-custom.md "Отметьте действие пользователя как ключевое и настройте рейтинг Apdex для ключевых действий пользователя ваших пользовательских приложений.").

## Влияние ошибок на Apdex

Действия пользователя с [ошибками](../user-and-error-events.md#error "Узнайте о событиях пользователей и ошибок, а также о типах событий, фиксируемых Dynatrace.") JavaScript оцениваются как **Frustrated** (раздражённые). Иногда действия пользователя, которые на самом деле быстрые и ниже порога Apdex, отображаются красным цветом и оцениваются как **Frustrated**, потому что некоторые из действий пользователя содержат ошибки JavaScript.

То же самое относится к ошибкам запросов. Код ответа HTTP, нарушение CSP или запрос ресурса, настроенный в параметрах для захвата как ошибка, приводит к действию пользователя с оценкой **Frustrated**.

Вы можете исключить ошибки из расчётов Apdex. Подробнее см. [Настройка обнаружения ошибок для веб-приложений](../../web-applications/additional-configuration/configure-errors.md "Настройте ваше приложение для захвата или игнорирования ошибок запросов, пользовательских ошибок и ошибок JavaScript."), [Настройка параметров Apdex для мобильных приложений](../../mobile-applications/additional-configuration/configure-apdex-mobile.md "Настройте пороги производительности удовлетворённости пользователей для вашего мобильного приложения и его ключевых действий.") и [Настройка параметров Apdex для пользовательских приложений](../../custom-applications/additional-configuration/configure-apdex-custom.md "Настройте пороги производительности удовлетворённости пользователей для вашего пользовательского приложения и его ключевых действий.").

Dynatrace позволяет легко анализировать Apdex вашего приложения по различным измерениям. Вы можете проверить рейтинг Apdex для конкретного действия пользователя, местоположения и приложения, а также просмотреть рейтинг Apdex для каждого действия пользователя в рамках одного пользовательского сеанса.
