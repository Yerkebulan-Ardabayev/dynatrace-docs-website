# Обзор оповещений синтетического мониторинга


* Latest Dynatrace
* 2-min read

Синтетический монитор создаёт проблему, когда выполняется одно из следующих условий:

* Отслеживаемая система недоступна. В этом случае монитор сообщает о проблеме с доступностью. Проблема создаётся после [заданного числа последовательных неудачных выполнений](synthetic-app/create-and-configure-an-http-monitor.md#outage-and-performance "Узнайте, как настроить и отредактировать HTTP-монитор для проверки производительности и доступности вашего сайта.").

  Отслеживаемая система считается недоступной, если:

  + Система не отвечает.
  + Возникают проблемы с сертификатами (для браузерных и HTTP-мониторов).
  + Нарушаются ограничения (для всех типов мониторов). Ограничения можно задавать при настройке или редактировании мониторов.
* Нарушается заданный порог производительности, то есть отслеживаемая система доступна, но работает медленно. В этом случае монитор сообщает о проблеме с производительностью.

## Обработка сбоев и пороги производительности

Чтобы настроить обработку сбоев и задать пороги производительности, перейдите в ![Synthetic](https://dt-cdn.net/images/synthetic-new-256-1ddb35ac79.png "Synthetic") **Synthetic**, выберите монитор, а затем выберите **Edit**.

Не забудьте нажать **Save** для сохранения настроек монитора.

### Обработка сбоев

В разделе **Outage handling** на вкладке **Outage and performance** можно настроить условия для создания проблемы. Доступны два варианта:

* **Generate a problem and send an alert when the monitor is unavailable at all configured locations (global outage)**. Вы можете указать, сколько последовательных выполнений должны завершиться неудачей, чтобы была создана проблема.
* **Generate a problem and send an alert when the monitor is unavailable for one or more consecutive runs at any location** (локальный сбой). Вы можете указать:

  + Минимальное количество локаций, которые должны быть затронуты, чтобы была создана проблема.
  + Сколько последовательных выполнений должны завершиться неудачей, чтобы была создана проблема.

  Этот параметр доступен только при наличии как минимум двух назначенных локаций.

Для браузерных мониторов можно дополнительно выбрать **Automatic retry on error** — это настраивает монитор для предотвращения ложных срабатываний путём выполнения ещё одной попытки после первой неудачи. Ошибка фиксируется только после неудачи при второй попытке.

### Порог производительности для монитора в целом

В разделе **Performance thresholds** на вкладке **Outage and performance** можно задать порог производительности для монитора в целом. Монитор создаёт проблему, когда сумма длительностей всех шагов превышает этот порог.

Dynatrace создаёт новую проблему, если синтетический монитор превышает любой из пороговых значений общей длительности производительности в трёх из пяти последних выполнений в данной локации, если только для синтетического монитора не открыто окно технического обслуживания. В одну проблему может быть включено несколько локаций с тремя подобными нарушениями. Проблема закрывается, если ни один порог производительности не нарушается в пяти последних выполнениях в каждой из ранее затронутых локаций.

### Пороги производительности для отдельных шагов

На вкладке **Steps** можно задать пороги производительности для отдельных шагов. Монитор создаёт проблему, когда хотя бы один шаг с заданным порогом нарушает этот порог.

## Проблемы

Вы можете отслеживать проблемы в:

* [![Problems app - new](https://dt-cdn.net/images/dynatrace-davis-new-256-340162f8c6.webp "Problems app - new") **Problems**](../../../dynatrace-intelligence/davis-problems-app.md "Используйте приложение Problems для быстрого определения первопричины инцидентов в вашей среде.")
* ![Synthetic](https://dt-cdn.net/images/synthetic-new-256-1ddb35ac79.png "Synthetic") **Synthetic** напрямую. Мониторы с проблемами имеют красный значок в столбце **Problems**. Красный значок также отображает количество проблем для данного монитора. В списке мониторов выберите монитор, проблемы которого вы хотите изучить. Откроется панель предварительного просмотра справа. Оттуда вы можете, например, выбрать **View details** > **Analyze executions** и изучить выполнения, в которых были обнаружены проблемы.
