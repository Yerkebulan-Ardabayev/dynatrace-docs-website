# Управление частными локациями

* Последняя версия Dynatrace
* Практическое руководство
* Обновлено 12 декабря 2025 г.

Для просмотра и управления частными локациями перейдите на вкладку **Private locations** в ![Synthetic](https://dt-cdn.net/images/synthetic-new-256-1ddb35ac79.png "Synthetic") **Synthetic**.

## Состояние работоспособности

В столбце **Health status** списка **Private Synthetic locations** вы можете видеть состояние работоспособности ActiveGate для каждой частной локации в вашей среде. Столбец содержит три подстолбца:

* **Health alerts** — оповещения отображают причины проблем с работоспособностью ActiveGate. Оповещения также выделяются цветовыми индикаторами в зависимости от серьёзности проблем:

  + Жёлтый:

    - Ёмкость — использование ёмкости хотя бы одного типа монитора превышает 80%.
    - Нет резервирования — к локации назначен только один ActiveGate. Резервная копия отсутствует на случай отключения ActiveGate.
    - Совместимость — хотя бы один ActiveGate не поддерживает хотя бы один тип монитора, назначенный локации. Для получения подробностей наведите курсор на (**Warning**) в столбце **Assigned monitors**.
  + Красный:

    - Ёмкость — использование ёмкости хотя бы одного типа монитора превышает 90%.
    - Offline AG — хотя бы один ActiveGate отключён.
    - Offline location — частная локация отключена. Это происходит, когда к частной локации не назначены ActiveGate.
    - Discarded — за последние 30 минут произошло хотя бы одно отклонённое выполнение монитора.
* **ActiveGates** — количество ActiveGate, назначенных локации, по их состоянию работоспособности. Состояние работоспособности обозначается одним из следующих цветов:

  + Зелёный — нет ActiveGate с проблемами.
  + Жёлтый — один или более ActiveGate имеют проблемы, которые ещё не являются серьёзными, но могут потребовать внимания.
  + Красный — один или более ActiveGate имеют серьёзные проблемы, требующие немедленного внимания.
* **Assigned monitors** — тип и количество мониторов Synthetic, назначенных локации. Количество HTTP-мониторов также включает мониторы HTTP High Resource.

## Просмотр деталей локации и редактирование локации

Выберите локацию, чтобы просмотреть её детали и отредактировать. Для классических локаций вы можете видеть ActiveGate, назначенные локациям. Они отображаются красным цветом, когда Synthetic engine или сам ActiveGate находится в автономном режиме; столбец **Status** показывает соответствующее сообщение.

Вы можете увидеть **ActiveGate ID** в столбце, который скрыт по умолчанию. Чтобы отобразить его, нажмите значок столбцов > **ActiveGate ID** > **Apply**.

Вы можете добавлять или удалять ActiveGate со страницы деталей локации. Вы также можете внести другие изменения и нажать **Save** для их сохранения.

### Диагностика

Карточка **Diagnostics** отображает состояние работоспособности для каждого типа монитора.

* **Capacity usage** — использование ёмкости для типа монитора. Если использование ёмкости менее 80%, в столбце отображается `OK`, а при отсутствии данных отображается `0`.
* **Compatibility** — количество ActiveGate, поддерживающих тип монитора. Например, если к локации назначено три ActiveGate, но только два из них поддерживают данный тип монитора, вы увидите `2/3`.
* **Monitors** — количество мониторов по типам, назначенных локации.
* **Executions/hour** — количество выполнений в час для типа монитора.
* **ActiveGates** — количество ActiveGate, назначенных локации. Сюда также входит количество ActiveGate, находящихся в автономном режиме. Например, если к локации назначено три ActiveGate, но один из них находится в автономном режиме, вы увидите `1/3`.

Для **Capacity usage**, **Compatibility** и **ActiveGates** состояние работоспособности дополнительно обозначается одним из следующих цветов:

* Зелёный — нет проблем.
* Жёлтый — есть проблемы, которые ещё не являются серьёзными, но могут потребовать внимания.
* Красный — есть серьёзные проблемы, требующие немедленного внимания.

### Метрики состояния работоспособности

Метрики состояния работоспособности каждого типа монитора доступны для построения графиков и оповещений. Метрики для построения графиков и оповещений о состоянии работоспособности каждого типа монитора доступны в [![Notebooks](https://dt-cdn.net/images/notebooks-768-046137830a.webp "Notebooks") **Notebooks**](/docs/analyze-explore-automate/dashboards-and-notebooks/notebooks "Анализируйте, визуализируйте и делитесь аналитикой из ваших данных наблюдаемости — всё в одном совместном настраиваемом рабочем пространстве."). Например, вы можете выбрать следующие метрики для построения графиков и оповещений:

* Synthetic - HTTP - Engine Utilization
* Synthetic - HTTP High Resource - Engine Utilization
* Synthetic - Browser - Engine Utilization

Мы настоятельно рекомендуем разбивать эти метрики по локации для получения точной картины состояния работоспособности локации.

## Обработка отключений локации

Для каждой локации включите соответствующие переключатели, чтобы генерировать проблемы, когда локация или любой из её движков ActiveGate недоступен. В настоящее время доступны следующие переключатели:

* Вы можете генерировать проблему, когда вся частная локация недоступна (все ActiveGate отключены) или если у локации отсутствует возможность, необходимая для выполнения данного типа монитора.
* Вы можете генерировать проблему, когда отдельный ActiveGate в этой локации отключён.

Например, предположим, что у вашей локации два ActiveGate, и вы включили оба переключателя проблем. При недоступности локации вы увидите три проблемы — одну для всей локации и по одной для каждого отключённого ActiveGate.

Контейнеризированные локации

Для контейнеризированных локаций вы можете генерировать проблему только тогда, когда вся частная локация недоступна (все ActiveGate отключены) или если у локации отсутствует возможность, необходимая для выполнения данного типа монитора.
