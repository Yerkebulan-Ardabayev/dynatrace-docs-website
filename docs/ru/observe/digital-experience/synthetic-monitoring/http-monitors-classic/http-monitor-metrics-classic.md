# Метрики HTTP-монитора


* Classic
* Пояснение
* Время чтения: 1 мин
* Опубликовано 04 декабря 2020 г.

Следующие метрики фиксируются и отображаются для синтетических [HTTP-мониторов](../general-information/types-of-synthetic-monitors.md "Узнайте о типах синтетических мониторов Dynatrace."):

Время ответа
:   Измеряет производительность HTTP-монитора или события.
    **Время ответа** — это сумма **времени поиска DNS**, **времени установки TCP-соединения**, **времени TLS-рукопожатия**, **времени ожидания** и **времени загрузки**.

Время поиска DNS
:   Время, затраченное на разрешение имени хоста. Если из-за перенаправления выполняется несколько DNS-запросов, указывается суммарное время всех запросов.

Время установки TCP-соединения
:   Время, затраченное на установку TCP-соединения. Если из-за перенаправления устанавливается несколько TCP-соединений, указывается суммарное время всех попыток.

Время TLS-рукопожатия
:   Время, затраченное на завершение TLS-рукопожатия.

Время ожидания
:   Время, затраченное сервером на ответ первым байтом. Равно **времени до первого байта** минус (**время поиска DNS** + **время установки TCP-соединения** + **время TLS-рукопожатия**).
    Обратите внимание: **время до первого байта** — это промежуток от начала запроса до получения первого байта ответа.

Время загрузки
:   Время, затраченное на загрузку HTTP-ответа.
    Время загрузки может быть равно 0, если ответ умещается в одном пакете. Время загрузки также может быть равно 0 при медленной сети — в таких случаях задержка сети учитывается во **времени ожидания**.
