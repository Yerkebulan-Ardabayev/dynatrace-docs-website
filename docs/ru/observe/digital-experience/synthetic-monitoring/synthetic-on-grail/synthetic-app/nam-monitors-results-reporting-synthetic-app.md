# Отчётность по результатам NAM-мониторов


* Latest Dynatrace
* Reference
* 2-min read
* Updated on Nov 03, 2025

Выполнение монитора по запросу

[Выполнение монитора по запросу](on-demand-executions.md "Узнайте о том, как выполнять запуск по запросу.") через веб-интерфейс доступно только по лицензии DPS. Срок хранения данных о выполнении монитора составляет 35 дней.
Для пользователей без лицензии DPS выполнение монитора возможно через [API](../../../../../dynatrace-api/environment-api/synthetic-v2/synthetic-monitor-execution.md "Просмотр результатов выполнения Synthetic-мониторов через Synthetic v2 API."). Вы можете получить доступ к результатам за последние 6 часов.

Чтобы открыть страницу отчётности для вашего монитора, выберите **View details** в правом верхнем углу панели предварительного просмотра.

Страница отчётности по умолчанию состоит из следующих элементов:

* Плитки с краткой информацией о глобальных сбоях, активных проблемах, количестве затронутых локаций, доступности, общем времени простоя и продолжительности.
* **Availability** — график доступности во времени. Доступность для NAM-мониторов рассчитывается путём сравнения частоты успешных запросов с заданными [ограничениями](../../network-availability-monitors/create-a-nam-monitor.md#script-configuration-constraints "Узнайте, как настроить NAM-монитор для проверки производительности и доступности вашего сайта."). Период, когда доступность составляет 0%, называется сбоем.

  Вы можете сузить временной диапазон графика, чтобы он охватывал только период сбоя. Это позволяет выявить, например, запросы, негативно влияющие на доступность. По умолчанию график доступности свёрнут, если за весь временной диапазон он показывает 0% или 100%. Чтобы открыть его, просто выберите **Availability**.
* **Success rate** — график частоты успешных запросов во времени, где частота успешных запросов — это процент всех выполненных запросов, завершившихся успешно.

  + Ознакомьтесь с [условиями](../../network-availability-monitors/create-a-nam-monitor.md#script-configuration-constraints "Узнайте, как настроить NAM-монитор для проверки производительности и доступности вашего сайта."), которым должен соответствовать запрос, чтобы считаться успешным.
  + Выберите кнопку **Show requests** под графиком, чтобы увидеть изменения доступности, такие как средняя производительность, средняя доступность и частота успешных пакетов (этот столбец доступен только для ICMP).
* **Performance** — график производительности во времени, где производительность рассчитывается как среднее значение для всех успешных запросов. Обратите внимание, что метрики производительности не отображаются для неудачных запросов.
* **Changes** — журнал истории, в котором отображается:

  + Когда монитор был создан и кем.
  + Кто изменил свойства монитора, когда было внесено изменение и какие свойства были изменены в выбранном временном диапазоне.
