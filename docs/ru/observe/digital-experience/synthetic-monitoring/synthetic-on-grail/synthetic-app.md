# Приложение Synthetic

Создание и управление синтетическими мониторами.

## Предварительные условия

### Разрешения

| Разрешение | Описание |
|---|---|
| geolocation:locations:search | Запрос данных о частных местоположениях |
| storage:buckets:read | Запрос данных из Grail |
| storage:entities:read | Запрос сущностей из Grail |
| storage:metrics:read | Запрос метрик из Grail |
| storage:events:read | Запрос событий из Grail |
| storage:user.events:read | Запрос пользовательских событий |
| storage:files:read | Запрос файлов |
| document:documents:write | Создание/обновление наборов фильтров |
| document:documents:read | Чтение наборов фильтров |
| document:documents:delete | Удаление наборов фильтров |

## Установка

Убедитесь, что приложение [установлено](../../../../manage/hub.md#install).

## Начало работы

### Таблица мониторов

По умолчанию: сначала мониторы с проблемами, затем включённые, затем отключённые — в алфавитном порядке. Настройка столбцов через **Столбцы**.

Столбцы: Имя, Создано, Последнее изменение, ID, Последнее выполнение, Тип, Frequency, Доступность, Продолжительность.

#### Предварительный просмотр

Выберите имя монитора для панели предварительного просмотра: состояние доступности, активные проблемы, затронутые местоположения, время простоя, свойства. Действия: редактировать, удалить, включить/отключить, отчёты.

### Фильтры

Категории фильтров: Тип, Проблемы, Внешнее приложение, Статус, Местоположения, Профиль устройства, Последний редактор, Теги, Период редактирования, Провайдер, Частота.

Внутри категории — логика `OR`. Между категориями — логика `AND`.

**Наборы фильтров**: сохранение комбинаций фильтров с возможностью установки по умолчанию.

### Создание мониторов

**+Новый монитор** > выберите тип:
* **Браузер** — создание в Synthetic Classic. См. Создание монитора браузера.
* **HTTP** — создание в **Synthetic**. См. [Создание HTTP-монитора](synthetic-app/create-and-configure-an-http-monitor.md).
* **Доступность сети (NAM)** — создание в **Synthetic**. См. [Создание NAM-монитора](synthetic-app/create-a-nam-monitor-synthetic-app.md).

### Расчёт времени простоя

Время мониторинга разделено на минутные точки данных. Все точки между первым неудачным и первым успешным выполнением считаются простоем.

Общее время простоя = первое выполнение "вниз" + все последующие точки до "вверх".
