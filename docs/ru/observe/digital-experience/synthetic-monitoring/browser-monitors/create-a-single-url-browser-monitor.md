# Создание монитора браузера для одного URL

# Создание монитора браузера для одного URL

* Classic
* Практическое руководство
* Чтение: 3 мин
* Обновлено 19 августа 2025 г.

Синтетический мониторинг позволяет создавать два типа мониторов браузера — для одного URL и для цепочки кликов (clickpath) — для проверки доступности и производительности вашего веб-приложения через регулярные интервалы. Мониторы браузера для одного URL выполняют тесты доступности одной страницы вашего веб-сайта или веб-приложения. Также доступна возможность проверки производительности.

## Создание монитора браузера для одного URL

1. Перейдите в **Synthetic Classic**.
2. Нажмите **Create a synthetic monitor** в правом верхнем углу > **Create a browser monitor**.
3. На странице «Configure a browser monitor» введите **URL**, который вы хотите мониторить, и используйте **Name** по умолчанию или укажите собственное.

   Для повышения безопасности синтетического мониторинга Dynatrace блокирует отправку запросов мониторами на локальный хост (например, `localhost` или `127.0.0.1`).
4. Нажмите **Add tag**, чтобы применить вручную созданные теги к монитору. Вы можете выбрать из предложений автодополнения по мере ввода или создать собственные. (После создания монитора вы можете управлять тегами на странице [Synthetic details](../analysis-and-alerting/synthetic-details-for-browser-monitors.md "Анализ результатов монитора браузера и цепочки кликов на странице Synthetic details.").
5. [Настройте монитор](configure-browser-monitors.md "Узнайте о настройке мониторов браузера и цепочек кликов.") соответствующим образом, включая выбор профиля эмулируемого устройства, аутентификацию и другие параметры.

   ![Настройка монитора браузера](https://dt-cdn.net/images/configurebrowsermonitor1-1665-749a574062.png)

   Аутентификация через веб-форму больше не поддерживается для мониторов браузера с одним URL. Вместо этого вы можете создать мониторы [цепочки кликов браузера](record-a-browser-clickpath.md "Узнайте, как записать цепочку кликов браузера для мониторинга доступности и производительности вашего приложения.") для тестовых сценариев, требующих входа через веб-форму. Ранее настроенные мониторы для одного URL будут работать как прежде, но мы рекомендуем перезаписать их как цепочки кликов, чтобы чётко разделить каждый шаг процесса входа.

   Перезапись необходима, если вы хотите изменить какую-либо часть конфигурации монитора. Вы больше не можете сохранять изменения в их текущем формате.

   Начиная с версии Dynatrace 1.324+, мониторы для одного URL с входом через веб-форму будут автоматически обновлены с добавлением свободного шага JavaScript для поддержки процесса входа.
6. Нажмите **Next**, чтобы продолжить настройку — выберите местоположения мониторинга и частоту. Подробности см. в разделе [Настройка мониторов браузера](configure-browser-monitors.md "Узнайте о настройке мониторов браузера и цепочек кликов.").

   ![Частота мониторинга и местоположения](https://dt-cdn.net/images/syntheticfrequencylocations2-2223-297d2c4656.png)
7. Нажмите **Next** внизу страницы, чтобы просмотреть сводку монитора.
8. На странице «Summary» вы можете просмотреть и изменить конфигурацию (**Change URL or name**; **Change configuration**).

   ![Сводка монитора браузера для одного URL](https://dt-cdn.net/images/summarysingleurl-2246-6f4d67b798.png)
9. Внизу страницы нажмите **Create browser monitor**. В течение нескольких минут вы [получите данные мониторинга](#view-the-analytics-of-a-browser-monitor) для нового монитора браузера.

Хотя вы не можете воспроизвести монитор браузера для одного URL локально, вы можете [выполнить его по требованию](../general-information/on-demand-executions.md "Выполнение синтетических мониторов по требованию из публичных или частных местоположений") из назначенных местоположений.

## Просмотр аналитики монитора браузера

1. Перейдите в **Synthetic Classic**.
2. При желании выберите **Browser** в левом меню для фильтрации по мониторам браузера для одного URL.
3. Из списка мониторов выберите монитор браузера, который хотите изучить. Вы будете перенаправлены на страницу [Synthetic details](../analysis-and-alerting/synthetic-details-for-browser-monitors.md "Анализ результатов монитора браузера и цепочки кликов на странице Synthetic details.") для данного монитора браузера.

Страница подробностей каждого монитора предоставляет детальные результаты, например метрики доступности и обнаруженные проблемы.

## Отключение или удаление монитора браузера

Мониторы включены по умолчанию при создании.

Отключение синтетического монитора приостанавливает дальнейшие выполнения, но сохраняет монитор и его данные измерений. Все открытые проблемы производительности и доступности истекают по тайм-ауту при отключении монитора (подробности см. в разделе [Расчёты синтетического мониторинга](../general-information/synthetic-calculations.md "Понимание расчётов метрик синтетического мониторинга.")). Удаление удаляет монитор и связанные с ним данные измерений; это действие необратимо. Перед удалением монитора рекомендуется сначала отключить его и убедиться, что данные измерений вам больше не нужны.

Чтобы отключить или удалить монитор

1. Перейдите в **Synthetic Classic**.
2. Выберите отображение мониторов в формате списка.
3. Установите флажок рядом с монитором, который хотите удалить или отключить.
4. Нажмите **Delete** или **Disable** в нижнем левом углу.

   ![Отключение монитора браузера](https://dt-cdn.net/images/disabledeletemonitor1-891-3c3e768323.png)

Вы также можете отключить или удалить монитор со страницы [подробностей](../analysis-and-alerting/synthetic-details-for-browser-monitors.md "Анализ результатов монитора браузера и цепочки кликов на странице Synthetic details.").

1. Перейдите в **Synthetic Classic**.
2. Выберите интересующий вас монитор.
3. Нажмите кнопку **Browse** (**...**) и выберите **Disable** или **Delete**.

[Выполнение синтетического монитора может быть отключено во время окна обслуживания](../general-information/synthetic-calculations.md#m-windows-availability "Понимание расчётов метрик синтетического мониторинга.") в настройках окна обслуживания.
