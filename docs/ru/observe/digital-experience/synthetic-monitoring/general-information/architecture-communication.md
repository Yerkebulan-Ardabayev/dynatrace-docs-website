## Компоненты архитектуры

В потоке коммуникации Synthetic задействованы следующие компоненты:

* **Кластер Dynatrace** — хранит конфигурацию Synthetic-мониторов, планирует выполнение мониторов для частных расположений и при необходимости отправляет конфигурацию мониторов в публичную инфраструктуру.
* **Частное расположение** — группа Synthetic ActiveGates, назначенных вашему монитору. Кластер Dynatrace планирует выполнения для частного расположения.
* **ActiveGate** — компонент, используемый движком Synthetic для регистрации и взаимодействия с кластером Dynatrace.
* **Публичная инфраструктура Synthetic** — инфраструктура, отвечающая за планирование и конфигурацию публичных расположений.
* **Публичное расположение** — группа движков Synthetic, назначенных вашему монитору. Публичная инфраструктура Synthetic планирует выполнения для публичного расположения.
* **Движок Synthetic** — движок, отвечающий за выполнение мониторинга.
* **Beacon forwarder** — перенаправляет результаты от движков Synthetic в кластер Dynatrace.

## Коммуникация и поток данных

В зависимости от используемого расположения поток данных варьируется. При помощи приведённой выше диаграммы ознакомьтесь с тем, как работает коммуникация для частных и публичных расположений, и как токены задействованы в этом процессе.

Каждый номер соответствует шагу на приведённой выше диаграмме.

Коммуникация через частное расположение

Коммуникация через публичное расположение

1. **Кластер Dynatrace — ActiveGate**

   Кластер предоставляет конфигурацию, определения мониторов и планирует выполнения. Данные шифруются через [токен ActiveGate](../../../../ingest-from/dynatrace-activegate/activegate-security.md "Защитите ActiveGates с помощью выделенных токенов.").
2. **Движок Synthetic — ActiveGate**

   Движок Synthetic запрашивает конфигурацию, определения мониторов и задания на выполнение от кластеров через ActiveGate. Данные шифруются через `synthetic token`, который создаётся при запуске ActiveGate и хранится в файле `synthetic.token` в папке `config`. Этот файл токена совместно используется ActiveGate и движком Synthetic.
3. **Движок Synthetic — beacon forwarder**

   Движок Synthetic отправляет результаты выполнений. Данные шифруются через токен `synthetic monitors`. Кластер Dynatrace генерирует этот токен на уровне тенанта, и он обновляется каждые 24 часа. Раз в день удаляются токены, возраст которых превышает 48 часов. Токен передаётся движку вместе с деталями выполнения, а коммуникация защищена с помощью [токена ActiveGate](../../../../ingest-from/dynatrace-activegate/activegate-security.md "Защитите ActiveGates с помощью выделенных токенов.").
4. **Beacon forwarder — кластер Dynatrace**

   Beacon forwarder перенаправляет результаты, отправленные движком, в кластер. Данные шифруются через [токен ActiveGate](../../../../ingest-from/dynatrace-activegate/activegate-security.md "Защитите ActiveGates с помощью выделенных токенов.").

5. **Кластер Dynatrace — публичная инфраструктура Synthetic**

   Кластер предоставляет определения мониторов, сведения об окнах обслуживания и состояние тенанта, включая состояние лицензии. Данные шифруются через токен `synthetic monitors`. Кластер Dynatrace генерирует этот токен на уровне тенанта, и он обновляется каждые 24 часа. Раз в день удаляются токены, возраст которых превышает 48 часов. Токен передаётся движку вместе с деталями выполнения, а коммуникация защищена с помощью [токена ActiveGate](../../../../ingest-from/dynatrace-activegate/activegate-security.md "Защитите ActiveGates с помощью выделенных токенов.").
6. **Движок Synthetic — публичная инфраструктура**

   Движок Synthetic запрашивает конфигурацию, определения мониторов и задания на выполнение из публичной инфраструктуры и передаёт диагностические данные. Данные защищены на сетевом уровне.
7. **Движок Synthetic — beacon forwarder**

   Движок Synthetic отправляет результаты выполнений. Данные шифруются через токен `synthetic monitors`. Кластер Dynatrace генерирует этот токен на уровне тенанта, и он обновляется каждые 24 часа. Раз в день удаляются токены, возраст которых превышает 48 часов. Токен передаётся движку вместе с деталями выполнения, а коммуникация защищена с помощью [токена ActiveGate](../../../../ingest-from/dynatrace-activegate/activegate-security.md "Защитите ActiveGates с помощью выделенных токенов.").
