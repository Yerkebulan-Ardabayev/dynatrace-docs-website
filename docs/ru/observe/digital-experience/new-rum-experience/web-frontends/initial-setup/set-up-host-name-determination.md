# Настройка определения имени хоста в новом интерфейсе RUM


Часть URL-адреса, содержащая хост, необходима для двух специфических функций RUM — применения правил обнаружения фронтенда и автоматического обнаружения домена RUM-куки. Однако в средах с такими компонентами, как обратные прокси или балансировщики нагрузки, исходный хост может быть перезаписан до того, как запрос достигнет инструментированного уровня, использующего [технологию с поддержкой RUM](../../../../../ingest-from/technology-support.md#rum-auto-injection "Технические подробности о поддержке конкретных платформ и фреймворков в Dynatrace."). Для решения этой проблемы OneAgent предоставляет функцию определения имени хоста.

## Добавление заголовка для определения имени хоста

Такие компоненты, как обратные прокси и балансировщики нагрузки, можно настроить на передачу исходной информации о хосте на бэкенд через заголовок запроса. Распространённые заголовки для этой цели — [`X-Forwarded-Host`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/X-Forwarded-Host) и стандартизированный заголовок [`Forwarded`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Forwarded).
OneAgent может извлекать исходный хост из одного из этих заголовков или из любого проприетарного заголовка, следующего тому же формату, что и `X-Forwarded-Host`: `<host>` для портов по умолчанию или `<host>:<port>` в остальных случаях. Также поддерживается разделённый запятыми список таких записей — предполагается, что первая запись представляет исходное имя хоста.

Некоторые компоненты добавляют один из этих заголовков по умолчанию. Если ваш компонент этого не делает, необходимо настроить его на включение такого заголовка для использования функции определения имени хоста.

## Настройка определения имени хоста

OneAgent использует список заголовков запросов для определения имени хоста. Заголовки в списке обрабатываются по порядку: заголовки в начале списка имеют приоритет. Как только OneAgent обнаруживает совпадающий заголовок в запросе, он прекращает проверку оставшихся заголовков. После того как вы убедились, что неинструментированный компонент добавляет подходящий заголовок, возможно, потребуется добавить этот заголовок в список.

Чтобы настроить заголовки, используемые для определения имени хоста:

1. Перейдите в ![Settings](https://dt-cdn.net/images/settings-icon-256-38e1321b51.webp "Settings") **Settings** > **Collect and capture** > **Real User Monitoring** > **Frontend** > **Host name determination**.
2. Если заголовок, добавляемый вашим компонентом, отсутствует в списке, выберите **Add HTTP header** и введите имя заголовка.
3. Вы можете изменить порядок записей, выбрав и удерживая ![Drag handle](https://dt-cdn.net/images/drag-handle-turquoise-600-1aa0e5ea00.svg "Drag handle") рядом с именем заголовка, затем перетащив его вверх или вниз. Убедитесь, что новая запись находится выше заголовка `Host`, который присутствует в каждом запросе HTTP/1.1.

## Пример

Если исходное имя хоста, запрошенное браузером, — `www.example.com`, а неинструментированный обратный прокси перенаправляет запрос на внутренний хост, например `backend.com`, исходная информация о хосте теряется. В этом случае прокси должен добавить заголовок, сохраняющий исходное имя хоста, чтобы OneAgent мог его определить, как показано на диаграмме ниже.
