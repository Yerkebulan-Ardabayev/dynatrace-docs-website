# Настройка источника кода Real User Monitoring в New RUM Experience

* Последняя версия Dynatrace
* Практическое руководство
* Обновлено 28 января 2026 г.

В большинстве сценариев RUM браузер выполняет как минимум один дополнительный запрос кода мониторинга:

* Все форматы сниппетов, кроме [встроенного кода](snippet-formats.md#inline-code "Узнайте, как выбрать формат JavaScript-сниппета RUM, наиболее подходящий для вашего сценария использования в New RUM Experience."), включают код мониторинга как внешний ресурс, который запрашивается браузером отдельно.
* Код мониторинга для Session Replay Classic всегда запрашивается отдельно, даже если используется формат встроенного кода.

Источник кода Real User Monitoring определяет, откуда загружается код мониторинга и какой URL используется для этих запросов.

## Поведение по умолчанию

По умолчанию применяется источник кода мониторинга, подходящий для большинства сред, который варьируется в зависимости от метода инструментации вашего фронтенда. В этом разделе подробно описывается поведение по умолчанию, а в следующем -- параметры конфигурации для сценариев, требующих настройки.

### Безагентный мониторинг

Если вы выбрали [безагентный мониторинг](set-up-agentless-monitoring.md "Узнайте, как настроить безагентный RUM для ваших веб-фронтендов в New RUM Experience."), код мониторинга запрашивается из Dynatrace CDN. В случае формата сниппета [JavaScript tag](snippet-formats.md#js-tag "Узнайте, как выбрать формат JavaScript-сниппета RUM, наиболее подходящий для вашего сценария использования в New RUM Experience.") имя файла заканчивается на `_complete.js` и содержит идентификатор фронтенда (например, `7cab1abeacdfe1_complete.js`). Для всех остальных форматов имя начинается с `ruxitagent_` и содержит информацию об активных модулях кода и версии кода мониторинга (например, `ruxitagent_ICA1589ENPQRTUVXfghqru_10307250124095659.js`). Имя файла кода мониторинга Session Replay Classic начинается с `ruxitagent_` (например, `ruxitagent_D_10307250124095659.js`) для всех форматов сниппетов.

### Автоматическое внедрение

Если [JavaScript RUM внедряется автоматически](set-up-auto-injected-frontend.md "Узнайте, как настроить автоматически внедряемый веб-фронтенд в New RUM Experience."), как код мониторинга RUM, так и код Session Replay Classic запрашиваются с вашего веб-сервера или сервера приложений с использованием относительного URL от корня, где имя файла начинается с `ruxitagentjs_` и содержит информацию об активных модулях кода и версии кода мониторинга. Точный путь URL зависит от технологии:

* **Java или IIS:** Имя файла добавляется к корневому контексту, если он доступен, например: `/contextroot/ruxitagentjs_ICA1589ENPQRTUVXfghqru_10307250124095659.js`.
* **Другие технологии или когда корневой контекст недоступен:** Имя файла добавляется к корню, например: `/ruxitagentjs_ICA1589ENPQRTUVXfghqru_10307250124095659.js`. Это также значение по умолчанию, если вы вручную вставляете JavaScript RUM, как описано в [Использование ручной вставки для страниц автоматически внедряемого фронтенда](configure-auto-injection.md#manual-insertion-using-oneagent "Настройка автоматического внедрения JavaScript RUM на страницы ваших фронтендов в New RUM Experience.").

### Ручная вставка для страниц автоматически внедряемого фронтенда

Если [JavaScript RUM вставляется вручную, несмотря на то что ваши группы процессов инструментированы OneAgent](configure-auto-injection.md#manual-insertion-using-oneagent "Настройка автоматического внедрения JavaScript RUM на страницы ваших фронтендов в New RUM Experience."), код мониторинга, как и при автоматическом внедрении, запрашивается с вашего веб-сервера или сервера приложений с использованием относительного URL от корня, где имя файла начинается с `ruxitagentjs_`. Единственным исключением является формат сниппета JavaScript tag, при котором код мониторинга RUM запрашивается из CDN, а имя файла заканчивается на `_complete.js`. Код мониторинга Session Replay Classic запрашивается с вашего веб-сервера или сервера приложений и имеет имя файла, начинающееся с `ruxitagentjs_`, для всех форматов сниппетов.

## Параметры конфигурации

В определённых сценариях может потребоваться использовать альтернативную конфигурацию источника кода мониторинга. Например:

* Если ваша инфраструктура блокирует запросы кода мониторинга автоматически внедряемого фронтенда из-за их URL-пути по умолчанию.
* Если вы предпочитаете, чтобы запросы кода мониторинга не обрабатывались веб-сервером или сервером приложений, на котором размещено ваше приложение.
* Если вы хотите предотвратить блокировку кода мониторинга блокировщиками рекламы.

Следующие параметры конфигурации позволяют решить эти задачи.

### Автоматически внедряемый фронтенд: Изменение URL-пути кода мониторинга

В зависимости от вашей инфраструктуры и её конфигурации может оказаться, что запросы кода мониторинга не могут пройти с автоматически выбранным URL-путём и, следовательно, не могут быть обработаны OneAgent. Чтобы решить эту проблему, вы можете изменить часть URL-пути, которая идёт перед префиксом `ruxitagentjs_`.

Чтобы изменить URL-путь кода мониторинга RUM для автоматически внедряемого фронтенда

1. Перейдите в ![Experience Vitals](https://dt-cdn.net/images/experience-vitals-256-9999590b55.png "Experience Vitals") **Experience Vitals** > **Overview**.
2. Нажмите **Web**, чтобы просмотреть все веб-фронтенды.
3. Выберите фронтенд, который хотите настроить.
4. На вкладке **Settings** выберите **Automatic injection**.
5. В выпадающем списке **Real User Monitoring code source** выберите **OneAgent**.
6. В поле **Specify path for RUM monitoring code** введите относительный URL-путь кода мониторинга.
7. Нажмите **Save**.

Обратите внимание, что вы не можете убрать сегмент пути с префиксом `ruxitagentjs_`, который необходим для идентификации запроса как запроса кода мониторинга.

#### Примеры

Следующие примеры предполагают, что код мониторинга по умолчанию запрашивается из `/ruxitagentjs_ICA1589ENPQRTUVXfghqru_10307250124095659.js`.

* **Абсолютный URL от корня:** Если вы настроите путь `/custompath`, код мониторинга будет запрашиваться из `/custompath/ruxitagentjs_ICA1589ENPQRTUVXfghqru_10307250124095659.js`.
* **Относительный URL:** Если вы настроите путь `./`, URL запроса кода мониторинга будет относительным к текущей странице. Например:

  + Если текущая страница -- `/shop/index.html`, код мониторинга будет запрашиваться из `/shop/ruxitagentjs_ICA1589ENPQRTUVXfghqru_10307250124095659.js`.
  + Если текущая страница -- `/account/dashboard/`, код мониторинга будет запрашиваться из `/account/dashboard/ruxitagentjs_ICA1589ENPQRTUVXfghqru_10307250124095659.js`.

Эта конфигурация действует не только для автоматического внедрения, но и для [ручной вставки для страниц автоматически внедряемого фронтенда](configure-auto-injection.md#manual-insertion-using-oneagent "Настройка автоматического внедрения JavaScript RUM на страницы ваших фронтендов в New RUM Experience."). Единственным исключением является запрос `_complete.js` для формата сниппета JavaScript, который по-прежнему будет направляться на Dynatrace CDN.

### Автоматически внедряемый фронтенд: Запрос кода мониторинга из Dynatrace CDN

Обратите внимание, что все подключённые ActiveGate должны работать на ActiveGate версии 1.310+ в течение как минимум 30 дней, прежде чем эта функция станет доступной.

Если вы хотите, чтобы код мониторинга для автоматически внедряемого фронтенда запрашивался из Dynatrace CDN вместо OneAgent, выполните следующие шаги.

1. Перейдите в ![Experience Vitals](https://dt-cdn.net/images/experience-vitals-256-9999590b55.png "Experience Vitals") **Experience Vitals** > **Overview**.
2. Нажмите **Web**, чтобы просмотреть все веб-фронтенды.
3. Выберите фронтенд, который хотите настроить.
4. На вкладке **Settings** выберите **Automatic injection**.
5. В выпадающем списке **Real User Monitoring code source** выберите **CDN**.
6. Нажмите **Save**.

Эта конфигурация действует как для автоматического внедрения, так и для [ручной вставки для страниц автоматически внедряемого фронтенда](configure-auto-injection.md#manual-insertion-using-oneagent "Настройка автоматического внедрения JavaScript RUM на страницы ваших фронтендов в New RUM Experience."). При её использовании имена файлов, ранее начинавшиеся с `ruxitagentjs_`, будут начинаться с `ruxitagent_`.

### Настройка пользовательского префикса имени файла кода мониторинга

После изменения префикса имени файла кода мониторинга вы можете временно наблюдать сокращение объёма собираемых данных RUM. Поэтому мы рекомендуем избегать частых изменений этой настройки.

По умолчанию имя файла кода мониторинга имеет префикс `ruxitagent` или `ruxitagentjs`, если не используется формат сниппета JavaScript tag. Альтернативно, вы можете указать пользовательский префикс, который будет использоваться вместо стандартного как для безагентных, так и для автоматически внедряемых фронтендов, а также для кода мониторинга RUM и Session Replay Classic.

Чтобы указать пользовательский префикс имени файла кода мониторинга

1. Перейдите в ![Settings](https://dt-cdn.net/images/settings-icon-256-38e1321b51.webp "Settings") **Settings** > **Collect and capture** > **Real User Monitoring** > **Frontend** > **RUM monitoring code filename**.
2. В поле **Custom filename prefix** введите желаемый пользовательский префикс.
3. Нажмите **Save changes**.
