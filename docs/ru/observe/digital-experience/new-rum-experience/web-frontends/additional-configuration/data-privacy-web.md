# Настройка параметров конфиденциальности данных для веб-фронтендов

# Настройка параметров конфиденциальности данных для веб-фронтендов

* Последняя версия Dynatrace
* Практическое руководство
* Опубликовано 15 января 2026 г.

Обеспечение соответствия требованиям GDPR и другим нормативным актам о защите данных имеет решающее значение для успеха вашего цифрового бизнеса. New RUM Experience предоставляет полный набор параметров, которые помогают защитить конфиденциальность персональных данных ваших клиентов при мониторинге веб-фронтендов.

Чтобы получить доступ к настройкам конфиденциальности данных для веб-фронтендов:

1. Перейдите в ![Experience Vitals](https://dt-cdn.net/images/experience-vitals-256-9999590b55.png "Experience Vitals") **Experience Vitals** > **Overview**.
2. Выберите  **Web** для просмотра всех веб-фронтендов.
3. Выберите фронтенд, который хотите настроить.
4. На вкладке **Settings** выберите **Data privacy**.

Ознакомьтесь с разделами ниже для подробного описания каждой настройки.

## Маскировка персональных данных в URI

Чтобы получить доступ к этой настройке, выберите **Data Privacy** > **General**.

Отключено по умолчанию

Dynatrace собирает полные URI запросов, отправляемых из настольных и мобильных браузеров, а также URI запросов, отправляемых и получаемых в рамках мониторируемых серверных процессов. URI могут содержать персональные данные, такие как имена пользователей, пароли или идентификаторы.

Когда функция **Mask personal data in URIs** включена, Dynatrace обнаруживает персональные данные — адреса электронной почты, IBAN, номера платёжных карт, IP-адреса, UUID и другие идентификаторы — в URI, заголовках, сообщениях исключений и данных, собранных для атрибутов запросов. Эти данные маскируются при сохранении путём замены строкой `<masked>`. Значения параметров запроса также заменяются строкой `<masked>`. Идентификаторы и числа должны содержать не менее 5 десятичных или шестнадцатеричных цифр для маскировки.

Примеры маскировки URI

В результате персональные данные, содержащиеся в URI, маскируются в пользовательских событиях.

## Использование постоянных cookie для отслеживания пользователей

Чтобы получить доступ к этой настройке, выберите **Data Privacy** > **General**.

Отключено по умолчанию

Настройка **Use persistent cookies for user tracking** позволяет включить или отключить использование постоянных cookie для идентификации возвращающихся пользователей.

Когда эта функция включена, JavaScript-код RUM устанавливает постоянный cookie в браузерах конечных пользователей, указывающий, что браузер ранее использовался для доступа к вашему фронтенду. Когда функция отключена, RUM Classic больше не может связывать сессии одного пользователя при перезапуске браузера. Узнайте, [как мы храним этот cookie](../../../../../manage/data-privacy-and-security/data-privacy/cookies.md#cookie-storage "Узнайте об использовании cookie первой стороны в Dynatrace.").

## Режим сбора данных и согласия

Чтобы получить доступ к этой настройке, выберите **Data Privacy** > **General**.

Отключено по умолчанию

Чтобы предоставить конечным пользователям возможность решать, следует ли отслеживать их активность, включите режим согласия (opt-in).

По умолчанию RUM автоматически создаёт [cookie](../../../../../manage/data-privacy-and-security/data-privacy/cookies.md#dynatrace-rum-cookies "Узнайте об использовании cookie первой стороны в Dynatrace."). Когда функция **Data-collection and opt-in mode** включена, ни OneAgent, ни JavaScript-код RUM не устанавливают cookie, и JavaScript-код RUM не собирает никаких данных. После того как конечный пользователь принимает вашу политику использования cookie, вы можете активировать RUM для этого пользователя с помощью вызова JavaScript API [`dtrum.enable()`](https://www.dynatrace.com/support/doc/javascriptapi/doc/types/dtrum.html#enable). С помощью вызова [`dtrum.disable()`](https://www.dynatrace.com/support/doc/javascriptapi/doc/types/dtrum.html#disable) вы можете реализовать диалоговое окно, позволяющее конечным пользователям прекратить отправку данных мониторинга в Dynatrace даже после того, как они ранее дали согласие и `dtrum.enable()` уже был вызван.

Оба вызова API действуют как для RUM Classic, так и для New RUM Experience, поэтому [новый JavaScript API](../new-javascript-api.md "Узнайте, как настроить мониторинг веб-фронтендов в New RUM Experience с помощью JavaScript API.") на данный момент не предоставляет эквивалентных методов.

## Соответствие настройке браузера «Не отслеживать»

Чтобы получить доступ к этой настройке, выберите **Data Privacy** > **General**.

Включено по умолчанию

Ещё один способ защиты конфиденциальности конечных пользователей — функция «Do Not Track» (не отслеживать). Когда пользователь включает эту функцию, его браузер добавляет HTTP-заголовок `DNT` ко всем исходящим веб-запросам. Этот заголовок указывает, что пользователь предпочитает не быть отслеживаемым.

После включения функции **Comply with "Do Not Track" browser settings** вы можете выбрать один из двух вариантов:

* **Capture anonymous user sessions for "Do Not Track"-enabled browsers**: при обнаружении заголовка `DNT` Dynatrace собирает данные RUM, но исключает всю персональную информацию, которая может привести к идентификации пользователя.

  + IP-адрес маскируется.
  + В RUM Classic информация [тега пользователя](../../../rum-concepts/user-and-error-events.md#user-tagging "Узнайте о пользовательских событиях и событиях ошибок, а также о типах событий, фиксируемых Dynatrace.") не отправляется.
  + В New RUM Experience [идентификатор пользователя](../../concepts/data-model.md#user-sessions "Ознакомьтесь с моделью данных, лежащей в основе New RUM Experience.") не отправляется, если он был добавлен с помощью [JavaScript API RUM Classic](https://www.dynatrace.com/support/doc/javascriptapi/doc/types/dtrum.html#identifyuser). Обратите внимание, что идентификатор пользователя по-прежнему будет отправляться, если он был добавлен с помощью [нового JavaScript API](https://www.dynatrace.com/support/doc/javascriptapi/doc-latest/functions/Types.dynatrace.identifyUser.html)

  При включённой настройке [**User tracking**](#user-tracking) Dynatrace по-прежнему устанавливает постоянный cookie для обнаружения возвращающихся пользователей.
* **Turn Real User Monitoring off for "Do Not Track"-enabled browsers**: при обнаружении заголовка `DNT` Dynatrace не собирает никаких данных из браузеров с включённой настройкой «Do Not Track».

Если вы отключите функцию **Comply with "Do Not Track" browser settings**, Dynatrace игнорирует настройку браузера «Do Not Track» и заголовок `DNT`.

Параметр **Comply with "Do Not Track" browser settings** — **Capture anonymous user sessions for "Do Not Track"-enabled browsers** включён по умолчанию для всех сред и фронтендов.

## Маскировка IP-адресов и GPS-координат конечных пользователей

Чтобы получить доступ к этой настройке, выберите **Data privacy** > **IP masking**.

Включено по умолчанию

Для определения региона, из которого конечные пользователи обращаются к веб- и мобильным фронтендам, Dynatrace собирает их IP-адреса. GPS-координаты собираются только для мобильных фронтендов.

Когда параметр **Mask end-user IP addresses and GPS coordinates** включён, IP-адреса маскируются на [конечной точке биконов](../initial-setup/configure-beacon-endpoint.md "Узнайте, как настроить конечную точку биконов для веб-фронтендов в соответствии с вашими требованиями."). Последний октет мониторируемых IPv4-адресов и последние 80 бит IPv6-адресов заменяются нулями. [Геолокация](../../../rum-concepts/detection-of-ip-addresses-locations-and-user-agents.md#geolocations "Dynatrace определяет IP-адреса и геолокации — город, регион и страну, а также браузеры, устройства и операционные системы.") выполняется с использованием замаскированных IP-адресов.
