# Действия пользователей в веб-фронтендах


* Latest Dynatrace
* Пояснение
* Обновлено 05 мар. 2026 г.

Действие пользователя представляет значимую операцию во фронтенде и её последствия. На этой странице объясняется концепция для веб-фронтендов в новом опыте RUM и чем она отличается от RUM Classic.

## Типы действий пользователей

Действия пользователей в веб-фронтендах можно разделить на три типа:

* **Навигационные действия пользователя:** При возникновении программной навигации, результатом которой является смена представления, автоматически создаётся новое действие пользователя. Навигации обычно инициируются [взаимодействием пользователя](../../concepts/data-model.md#user-interactions "Get familiar with the data model at the heart of the New RUM Experience."), однако взаимодействие пользователя не является обязательным условием для создания действия пользователя.
* **Запросные действия пользователя:** Если за кликом пользователя следует XHR- или fetch-запрос, создаётся действие пользователя.
* **API-действия пользователя:** Действия пользователя также можно создавать с помощью [`dynatrace.userAction.create()`](https://docs.dynatrace.com/javascriptapi/doc-latest/interfaces/Types.UserActions.html) в JavaScript API. Для инициирования действия пользователя этим способом предварительные требования отсутствуют.

## Как завершаются действия пользователей

Действие пользователя завершается автоматически при соблюдении определённых условий, либо его можно закрыть вручную через API.

### Поведение завершения по умолчанию

Пока действие пользователя активно, RUM JavaScript отслеживает XHR- и fetch-запросы, мутации DOM и тайминги ресурсов, чтобы определить момент завершения действия. Действие пользователя завершается при выполнении одного из следующих условий:

* Действие пользователя достигает продолжительности 50 с.
* Пользователь покидает страницу.
* Создаётся новое действие пользователя. В этом случае текущее действие пользователя прерывается и связывается с новым через атрибут `user_action.interrupted_by_instance_id`.
* Когда в течение последних 100 мс выполняются все следующие условия:

  + Не произошло XHR- или fetch-запросов.
  + Не произошло мутаций DOM.
  + Ни один новый запрос не был добавлен в [W3C Resource Timing API](https://www.w3.org/TR/resource-timing/). Обратите внимание, что запросы добавляются только после их завершения.

### Завершение действий пользователей через API

Закрывать действия пользователей через API можно с помощью [`UserActionTracker.finish()`](https://docs.dynatrace.com/javascriptapi/doc-latest/interfaces/Types.UserActionTracker.html).

По умолчанию действия пользователей, созданные через API, следуют поведению завершения по умолчанию. Чтобы отключить автозакрытие, используйте [`UserActionTracker.autoClose()`](https://docs.dynatrace.com/javascriptapi/doc-latest/interfaces/Types.UserActionTracker.html#autocloseautoclose-1) или установите опцию `autoClose` в значение `false` при [создании действия пользователя](https://docs.dynatrace.com/javascriptapi/doc-latest/interfaces/Types.UserActions.html#create-1).

## Пример

На приведённой ниже диаграмме показан жизненный цикл действия пользователя на примере запросного действия пользователя.

* Пользователь кликает в каком-либо месте страницы.
* Если в течение 30 мс происходит XHR- или fetch-запрос, начинается новое действие пользователя.
* Пока в течение 100 мс после предыдущего отчёта поступают сообщения о мутациях DOM, XHR- или fetch-запросах или таймингах ресурсов, действие пользователя продлевается.
* Если более 100 мс не поступает ни одного отчёта, действие пользователя закрывается и отправляется.
