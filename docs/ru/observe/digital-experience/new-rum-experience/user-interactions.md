# Взаимодействия пользователей

* Latest Dynatrace

* Обновлено 21 января 2026 г.

Ранний доступ

Новый опыт RUM (New RUM Experience) позволяет фиксировать [взаимодействия пользователей](concepts/data-model.md#user-interactions "Ознакомьтесь с моделью данных, лежащей в основе New RUM Experience."), такие как клики и прокрутка на веб-интерфейсах, а также события касания на мобильных интерфейсах.

Взаимодействия пользователей в настоящее время находятся в режиме раннего доступа и доступны для следующих технологий:

* Web
* iOS native
* Android native
* SwiftUI
* Jetpack Compose

## Часто задаваемые вопросы по раннему доступу

Какова стоимость приёма взаимодействий пользователей?

Дополнительная плата в период раннего доступа к взаимодействиям пользователей не взимается.

Какова стоимость запросов по взаимодействиям пользователей?

Запросы к необработанным данным DEM в настоящее время находятся в режиме раннего доступа. Дополнительная плата в этот период не взимается. Подробнее см. [Расчёт потребления Digital Experience Monitoring (DEM) — Запросы](../../../license/capabilities/digital-experience-monitoring-query-retain/queries-dem.md "Узнайте, как потребление запросов DQL, связанных с DEM, учитывается и тарифицируется до и после раннего доступа.").

Каков период хранения данных для взаимодействий пользователей?

Как и другие пользовательские события, взаимодействия пользователей имеют [период хранения данных по умолчанию](../../../manage/data-privacy-and-security/data-privacy/data-retention-periods.md#new-rum-user-events-and-sessions "Проверьте сроки хранения для различных типов данных.") в 35 дней. Вы можете расширить период хранения данных, присоединившись к [предварительному просмотру расширенного хранения для RUM и Synthetic](../../../../common/whats-new/preview-releases.md#extended-retention-for-rum-and-synthetic "Узнайте о наших предварительных версиях и о том, как принять в них участие.").

## Типы взаимодействий пользователей

В следующей таблице представлен обзор доступных типов взаимодействий пользователей. Подробную спецификацию см. в разделе [Взаимодействие пользователя](../../../semantic-dictionary/model/rum/user-events/user-interactions.md) в Семантическом словаре.

Тип взаимодействия

Описание

Тип интерфейса

Клик (Click)

Когда пользователь кликает или касается элемента HTML. Обратите внимание, что касание элемента HTML фиксируется как клик.

Web

Касание (Touch)

Когда пользователь касается мобильного компонента.

Mobile

Нажатие клавиши (Key press)

Когда пользователь нажимает комбинацию клавиш с `Ctrl`, `Alt` или `Meta` (`Windows` или `Cmd`), либо нажимает одну из отдельных клавиш `Esc`, `Enter`, `Tab` и `Space`.

Web

Масштабирование/Изменение размера (Zoom/Resize)

Когда пользователь масштабирует изображение или изменяет размер области просмотра.

Web

Фокус/Расфокусировка (Focus/Blur)

Когда пользователь фокусируется на элементе HTML или покидает его. Это взаимодействие фиксируется только при наличии атрибутов `data-dt-focus` или `data-dt-blur` соответственно на элементе HTML.

Web

Изменение (Change)

Когда пользователь изменяет значение в поле формы, например устанавливая флажок.

Web

Прокрутка (Scroll)

Когда пользователь прокручивает страницу или представление.

Web

Перетаскивание (Drag/Drop)

Когда пользователь начинает или завершает операцию перетаскивания. Также фиксируются допустимые внешние действия сброса, например перетаскивание файла в область загрузки. В обоих случаях записывается позиция.

Web

Наведение (Mouseover)

Когда пользователь наводит курсор на элемент HTML. Это взаимодействие фиксируется только при наличии атрибута `data-dt-mouse-over=<ms>` на элементе HTML.

Web

## Активация захвата взаимодействий пользователей

Чтобы захватывать взаимодействия пользователей для веб- или мобильного интерфейса:

1. Перейдите в ![Experience Vitals](https://dt-cdn.net/images/experience-vitals-256-9999590b55.png "Experience Vitals") **Experience Vitals** > **Overview**.
2. Выберите **Frontends** для просмотра всех интерфейсов.
3. Выберите интерфейс, который хотите настроить.
4. Перейдите на вкладку **Settings**.
5. В разделе **Enablement and cost control** включите **User Interactions**.

## Анализ взаимодействий пользователей

Существует два основных способа анализа взаимодействий пользователей:

* Приложение [![Users & Sessions](https://dt-cdn.net/images/users-sessions-149-f84e0b9b20.png "Users & Sessions") **Users & Sessions**](users-and-sessions.md "Приложение Users & Sessions предоставляет информацию об отдельных пользовательских путях и паттернах поведения.") позволяет просматривать все взаимодействия пользователей, произошедшие в ходе пользовательской сессии. Это особенно полезно для команд поддержки клиентов и разработчиков, которым необходимо понять, что произошло при устранении проблем клиентов или ошибок. После фильтрации сессий по заданным критериям и выбора одной из них все связанные взаимодействия пользователей перечисляются и отображаются на временной шкале.
* Анализ взаимодействий пользователей с помощью DQL поддерживает широкий спектр поведенческих сценариев использования для менеджеров по продукту и владельцев продуктов. В этих сценариях взаимодействия пользователей нередко появляются рядом с навигационными событиями. Примеры извлечения поведенческих данных с помощью DQL см. в разделе [Примеры DQL](use-cases/dql-examples.md#behavioral-insights "Анализируйте и исследуйте данные RUM с помощью DQL.").
