# Каскадная диаграмма загрузки страницы

* Актуальная версия Dynatrace
* Справочник
* Чтение: 7 мин
* Опубликовано 14 янв. 2025 г.

Каскадная диаграмма загрузки страницы в ![Experience Vitals](https://dt-cdn.net/images/experience-vitals-256-9999590b55.png "Experience Vitals") **Experience Vitals** предоставляет мощные возможности визуализации и анализа для понимания производительности веб-приложения на уровне [экземпляра страницы](../web-frontends/concepts/pages-views-and-navigations.md#pages-and-page-instances "Understand how pages, views, and navigations are defined for web frontends the New RUM Experience."). Эта функция помогает анализировать отдельные загрузки страниц, выявлять узкие места производительности и оптимизировать пользовательский опыт путём изучения паттернов загрузки ресурсов и метрик тайминга.

![Example page load waterfall](https://dt-cdn.net/images/rum-page-load-waterfall-1915-f8def7fc9c.png)

## Доступ к анализу каскадной диаграммы

Чтобы перейти к каскадной диаграмме загрузки страницы:

1. Перейдите в ![Experience Vitals](https://dt-cdn.net/images/experience-vitals-256-9999590b55.png "Experience Vitals") **Experience Vitals**.
2. Выберите веб-фронтенд в представлении **Explorer** и нажмите **Analyze performance** или выберите фронтенд непосредственно в представлении **Performance**.
3. Перейдите в раздел **Pages** и детализируйте конкретную страницу.
4. Нажмите **Perform waterfall analysis**.

## Таблица экземпляров загрузки страниц

Таблица **Page load instances** отображает список отдельных загрузок страниц, зафиксированных в течение выбранного периода времени, с метаданными, счётчиками и метриками.

Доступные столбцы:

* Start time: Временная метка начала загрузки страницы
* Country: Страна
* Browser and browser version: Браузер и версия браузера
* Errors: Количество ошибок, возникших в течение всего экземпляра страницы (не только во время загрузки страницы)
* Long tasks: Количество длительных задач, которые могли заблокировать основной поток в течение всего экземпляра страницы (не только во время загрузки страницы)
* Core Web Vitals и тайминги навигации W3C страницы

## Визуализация каскадной диаграммы

При выборе экземпляра загрузки страницы открывается панель каскадной диаграммы. На каскадной диаграмме отображается каждый отдельный запрос экземпляра загрузки страницы с его типом, именем, таймингами, метаданными, диаграммой Ганта с аннотациями метрик тайминга экземпляра и отдельными фазами.

Щелчок по любому запросу в каскадной диаграмме открывает подробную информацию о запросе и дополнительные варианты детализации. При наведении курсора на записи диаграммы Ганта отображаются сведения о фазе.

Сценарии использования

Детализация

### Выявление медленных ресурсов

Используйте каскадную диаграмму для выявления ресурсов, загрузка которых занимает слишком много времени:

* Сортируйте экземпляры страниц по длительности или конкретным метрикам, таким как TTFB или LCP
* Выбирайте экземпляры с низкой производительностью
* Изучайте каскадную диаграмму для поиска ресурсов, влияющих на медленное время загрузки
* Обращайте внимание на ресурсы с длительным временем ответа или блокирующим рендеринг поведением

### Сравнение производительности в различных условиях

Фильтруйте экземпляры загрузки страниц по:

* **Типу и версии браузера**: выявление проблем производительности, специфичных для конкретного браузера
* **Географическому расположению**: понимание вариаций производительности по регионам
* **Условиям ошибок**: сравнение успешных загрузок с неудавшимися
* **Периодам времени**: анализ тенденций производительности с течением времени

* **Сведения о сессии**: откройте полную сессию в дополнении, предоставляемом ![Users & Sessions](https://dt-cdn.net/images/users-sessions-149-f84e0b9b20.png "Users & Sessions") **Users & Sessions**, чтобы увидеть загрузку страницы в контексте всего пользовательского пути.
* **Связывание трассировок запросов**: получите доступ к подробной информации о трассировке для анализа производительности на стороне сервера в ![Distributed Tracing](https://dt-cdn.net/images/distributed-tracing-4ed13d1274.svg "Distributed Tracing") **Distributed Tracing**.
* **Анализ ошибок**: для неудавшихся запросов вы можете углубиться в анализ в ![Error Inspector](https://dt-cdn.net/images/error-inspector-256-c849b18430.png "Error Inspector") **Error Inspector**.
* **Готовые материалы по ресурсам**: для отдельных запросов вы можете детализировать предоставленный готовый дашборд. Это позволяет более глубоко изучить отдельные ресурсы.
* **Запрос**: откройте запрос каскадной диаграммы в ![Notebooks](https://dt-cdn.net/images/notebooks-768-046137830a.webp "Notebooks") **Notebooks** для расширенных запросов и пользовательского анализа с помощью DQL.
