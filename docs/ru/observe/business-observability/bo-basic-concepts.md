# Основные концепции Dynatrace Business Observability

# Основные концепции Dynatrace Business Observability

* Последняя версия Dynatrace
* Пояснение
* Чтение: 3 мин
* Опубликовано 24 октября 2022 г.

Бизнес-события — важная концепция, лежащая в основе Dynatrace Business Observability. Бизнес-события генерируют данные бизнес-уровня для поддержки важных сценариев использования.

## Бизнес-событие

Событие — это действие или явление, которое происходит внутри системы или сервиса, например, когда клиент завершает опрос об удовлетворённости или когда пользователь нажимает кнопку подписки. Когда происходит событие, система генерирует сигнал.
Событие становится **бизнес**-событием, если оно генерирует **данные бизнес-уровня**. Dynatrace обрабатывает бизнес-события отдельно от данных наблюдаемости, чтобы обеспечить качество данных бизнес-уровня.

## Данные бизнес-уровня

Данные бизнес-уровня — это точные данные, которые не зависят от выборок при формировании базовых показателей, выявлении тенденций или оповещениях об аномалиях с высокой статистической точностью. Они опираются на полные и точные данные, позволяя бизнесу получать информацию о производительности системы и аномалиях в режиме реального времени. Бизнес-события предоставляют полные и точные данные для поддержки требовательных бизнес-сценариев, а также ИТ-сценариев, требующих абсолютной точности.

Данные бизнес-уровня:

* **Полные**
  Все события фиксируются без выборки.
* **Полностью доступные**
  Все события можно запрашивать без экстраполяции.
* **В реальном времени**
  События доступны сразу после их возникновения.
* **Неограниченные**
  Нет ограничений на количество генерируемых событий.
* **Детализированные**
  События не агрегируются по мере устаревания.
* **Долгосрочные**
  Период хранения событий — до десяти лет.
* **Топологически осведомлённые**
  События хранятся с учётом топологического контекста.
* **Доступные из нескольких источников**
  Фиксируемые события включают те, которые генерируются OneAgent, RUM, журналами и внешними источниками.
* **Унифицированные**
  События хранятся в едином хранилище данных (Grail) и доступны через единый язык запросов — [DQL](/docs/platform/grail/dynatrace-query-language "Как использовать Dynatrace Query Language.").

Данные бизнес-уровня часто необходимы для принятия бизнес-решений и составления отчётов, где точность критически важна. В отличие от этого, типичная ИТ-отчётность достигает статистической точности через выборку и экстраполяцию.

## Сценарии использования бизнес-событий

* Данные бизнес-событий можно использовать для отслеживания финансовой информации, например, выручки от продаж и нормы прибыли. Сезонные и долгосрочные тенденции можно использовать для прогнозирования спроса. Сезонные тенденции легко преобразуются в долгосрочные без экстраполяции — благодаря десятилетнему сроку хранения.
* Бизнес-события можно использовать для отслеживания сложных бизнес-процессов, таких как управление заказами и их выполнение, для извлечения и сегментации финансовых деталей из платёжного шлюза, а также для обогащения данных Dynatrace внешними бизнес-данными, например отзывами пользователей, полученными через решения для анализа мнений клиентов (VoC).
* Бизнес-события могут, например, использоваться автопроизводителем для лучшего понимания пути клиента в онлайн-конфигураторах автомобилей, даже если этот путь охватывает несколько недель или месяцев и включает несколько точек соприкосновения. Руководители бизнеса могут выявлять места, где пользователи застревают, экспериментировать с подходами для повышения достижимости целей и точнее таргетировать конкретные демографические сегменты аудитории.

## Поток данных бизнес-событий

Поток данных бизнес-событий в Dynatrace состоит из трёх этапов:

1. **Захват**

   Данные бизнес-событий могут приниматься из OneAgent, JavaScript API для RUM и внешних источников.
2. **Обработка**

   Конвейеры обрабатывают входящие бизнес-события для улучшения анализа и отчётности. Варианты обработки включают фильтрацию, разбор, обогащение, преобразование или назначение периода хранения.
3. **Анализ**

   Данные, хранящиеся в Grail™, можно изучать с помощью Dynatrace Query Language — [DQL](/docs/platform/grail/dynatrace-query-language "Как использовать Dynatrace Query Language.") — для обнаружения закономерностей, выявления аномалий и выбросов, формирования отчётов по тенденциям и многого другого. Запросы DQL также используются для построения дашбордов, графиков, метрик и отчётов. С помощью DQL можно экспортировать выбранные данные во внешние инструменты.
