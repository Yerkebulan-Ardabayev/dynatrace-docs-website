# Business reporting


* Latest Dynatrace
* Tutorial
* 3-min read
* Published Feb 19, 2025

Отслеживайте бизнес-метрики, KPI и SLO в режиме реального времени, автоматически в контексте ИТ-инфраструктуры и сервисов, чтобы обеспечить сотрудничество между бизнес- и ИТ-командами. Приоритизируйте ИТ-инвестиции и автоматизируйте устранение последствий на основе измеримого влияния на бизнес.

## Введение

Используйте бизнес-события для захвата ключевых бизнес-метрик и преобразования их в бизнес-KPI. Бизнес-события — это особый класс данных, хорошо подходящий для требовательных бизнес-сценариев. Получайте видимость бизнес-состояния в режиме реального времени и проводите исследовательскую аналитику для оптимизации бизнес-результатов и понимания бизнес-аномалий.

Бизнес-события могут поступать от OneAgent, RUM, файлов логов и внешних систем через API. Dynatrace автоматически обогащает бизнес-события топологическим контекстом Smartscape, связывая бизнес-метрики с поддерживающими их ИТ-системами. OneAgent может захватывать бизнес-события из полезных нагрузок работающих приложений без изменений кода, приоритизируя бизнес-данные для обеспечения точности без потерь, которой требуют многие бизнес-сценарии.

## Целевая аудитория

Эта статья предназначена для бизнес-аналитиков и менеджеров процессов, которые понимают, как работает их бизнес в режиме реального времени, и стремятся к оптимальному выполнению бизнес-процессов. Необходимо базовое знание принципов захвата бизнес-событий и некоторые знания предметной области анализируемого бизнеса.

## Варианты использования

### Определение метрик и измерений для отчётности

Зачастую бизнес-KPI представляют собой функцию нескольких точек данных. Например, для отчётности по KPI срока выполнения поставщика нужно вычесть дату запроса заказа из даты доставки заказа. Даты принимаются как бизнес-события, затем преобразуются правилом обработки в KPI срока выполнения. В отчёты стоит включить соответствующие измерения, например имя поставщика и продукт, для добавления важной детализации. Эти измерения берутся из полей бизнес-события.

### Определение места захвата бизнес-событий

OneAgent упрощает захват бизнес-событий с помощью простых шагов настройки без написания пользовательского кода. Если приложение можно инструментировать с помощью OneAgent, это предпочтительный вариант.

Файлы логов — широко используемый источник, особенно если приложение уже записывает в них соответствующие бизнес-данные.

Бизнес-события из внешних систем можно принимать через API.

Для захвата бизнес-событий из действий пользователей можно использовать Dynatrace real user monitoring (RUM).

### Определение правил обработки

Правила обработки помогают фильтровать, анализировать и преобразовывать бизнес-данные. Можно добавлять новые вычисляемые поля, маскировать или удалять конфиденциальные данные, определять политики хранения посредством назначения бакетов и многое другое.

Создайте и поделитесь своим [дашбордом](../../analyze-explore-automate/dashboards-and-notebooks/dashboards-new.md "Создавайте интерактивные настраиваемые представления для визуализации, анализа и обмена данными наблюдаемости в режиме реального времени.").

После сохранения в Grail данные бизнес-событий можно интерактивно запрашивать и анализировать с помощью [DQL](../../platform/grail/dynatrace-query-language.md "Как использовать Dynatrace Query Language."). DQL служит отправной точкой для анализа, будь то Notebooks, Dashboards или DQL Query API. Результаты запросов можно использовать интерактивно или закреплять на дашборде в виде диаграмм, плиток или таблиц.

### Измерение и оптимизация бизнес-ценности каналов продаж агрегаторов

Захватывайте бизнес-события из API агрегаторов для извлечения бизнес-KPI, таких как отправленные предложения, бронирования и средний доход. Разбивайте эти бизнес-KPI на соответствующие когорты или измерения: например, партнёр, продукт и география. Добавляйте соответствующие ИТ-метрики, такие как время ответа API и ошибки API.

### Восстановление потерянного бизнеса с помощью автоматизированного взаимодействия с клиентами

Отслеживайте критические ошибки в нижней части воронки конверсии, которые часто вынуждают пользователей прерывать свой путь. Сегментируйте пользователей по потенциальной или реальной ценности для клиента: сумма в корзине, история покупок или статус лояльности. Создайте Workflow для автоматизации соответствующего ответного взаимодействия, например предложения стимула для возвращения или скидки на будущую покупку.

### Снижение затрат колл-центра с помощью оповещений о бизнес-исключениях

Извлекайте бизнес-сообщения об ошибках (например, нет в наличии, карта отклонена, неизвестный SKU продукта, неверный номер счёта и т. д.) из полезных нагрузок работающих приложений. Определяйте пороги оповещений и профили для уведомления ИТ- и бизнес-команд по мере необходимости.

### Отслеживание долгосрочных сезонных тенденций

Для отчётности по тенденциям захватывайте соответствующие бизнес-метрики как бизнес-события. Убедитесь, что назначенные бакеты бизнес-событий имеют характеристики хранения, необходимые для достижения целей анализа и отчётности.
