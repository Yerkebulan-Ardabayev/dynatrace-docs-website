Dynatrace автоматически обогащает бизнес-события дополнительным контекстом для улучшения анализа и обеспечения возможности детального изучения. В частности, Dynatrace добавляет определённые свойства к вашим бизнес-событиям, например, информацию о геолокации, операционной системе, приложении и многом другом. Затем вы можете использовать Dynatrace Query Language (DQL) для анализа ваших бизнес-событий.

## Обогащение бизнес-событий OneAgent и внешних источников

Поля, определённые в таблице ниже, могут использоваться для обогащения бизнес-событий. Некоторые поля добавляются автоматически в зависимости от их источника (OneAgent или API).

1

Не является обогащающим полем как таковым, но требуется внутренней схемой Dynatrace.

2

Если исходная временная метка недоступна, она заполняется во время приёма данных.

3

Устанавливается OneAgent и требуется спецификацией CloudEvents, но не является обязательным для внутренней схемы Dynatrace.

## Обогащение бизнес-событий RUM

Бизнес-события, [передаваемые через RUM](bo-events-capturing.md#report-business-event-rum "Сбор бизнес-событий для Dynatrace Business Observability."), обогащаются полями, релевантными для устройства и пользователя, такими как геолокация, тип устройства, браузер и приложение. Следующие поля добавляются автоматически в зависимости от типа приложения — web или mobile.

### Поля верхнего уровня

Поля верхнего уровня содержат общую релевантную информацию для всех событий и точек данных.

1

Если исходная временная метка недоступна, она заполняется во время приёма данных.

2

Unix-временная метка в наносекундах.

### Поля событий

Пространство имён `event` содержит общую идентификационную, классификационную информацию и контекст событий в Dynatrace.

1

Не обогащается автоматически; устанавливается через API при передаче бизнес-события.

2

Полное доменное имя для web, bundle приложения для mobile и идентификатор приложения для OpenKit.

### Поля Dynatrace RUM

Пространство имён `dt.rum` содержит специфические для Dynatrace RUM поля.

1

Также известен как внутренний идентификатор пользователя или идентификатор посетителя.

2

При наличии.

### Поля устройства

Пространство имён `device` содержит информацию об устройстве, на котором запущено приложение.

### Поля операционной системы

Пространство имён `os` содержит информацию об операционной системе, под управлением которой работает приложение.

### Поля браузера

Пространство имён `browser` содержит информацию о браузере, в котором запущено приложение.

1

Возможные значения: `desktop`, `mobile`, `tablet`, `robot`, `other`.

### Поля клиента

Пространство имён `client` содержит информацию об инициаторе сетевого соединения.

### Поля приложения

Пространство имён `app` содержит информацию о приложении, отправляющем бизнес-событие.

### Поля геолокации

Пространство имён `geo` содержит информацию о геолокации.
