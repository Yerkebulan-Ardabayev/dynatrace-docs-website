# Расширение Disk Analytics

# Расширение Disk Analytics

* Latest Dynatrace
* Extension
* Обновлено 18 декабря 2025 г.

Получите детальную видимость локальных хранилищ данных на хостах Linux, где установлен OneAgent.

## Начало работы

### Обзор

Расширение Disk Analytics позволяет проверять и анализировать физические и логические локальные устройства хранения данных, такие как диски, разделы, тома и программные или аппаратные RAID-массивы на машинах Linux, где установлен OneAgent.

* Проверка аппаратного обеспечения дисков, а не только точек монтирования
* Обнаружение проблем, связанных с дисками, непосредственно из Dynatrace
* Подробные метрики для всех разделов, экземпляров программных или аппаратных RAID-массивов и логических томов. Детали включают точки монтирования, тип файловой системы, размеры разделов и статус шифрования
* Для устройств с назначенной точкой монтирования отображается статистика файловой системы

### Требования

* OneAgent 1.233+
* Только хосты Linux; см. [Поддерживаемые технологии OneAgent для Linux](/docs/ingest-from/technology-support "Технические подробности о поддержке Dynatrace для конкретных платформ и фреймворков разработки.")
* Только локальные диски (сетевые диски не поддерживаются)
* Только развёртывания OneAgent Full-stack. Развёртывания PaaS OneAgent не поддерживают расширение Disk Analytics

## Активация и настройка

Перед включением расширения необходимо его установить: в Dynatrace Hub выберите и установите **Disk Analytics**.

После добавления этого расширения в вашу среду вам также необходимо включить сбор данных в настройках на уровне хоста или группы хостов. Название настройки — **Disk Analytics Extension**. Это обеспечивает полный контроль над пользовательскими метриками и потреблением DDU.

После включения сбора данных на целевых хостах Linux вы можете просматривать ключевые метрики для всех локальных экземпляров дисков и далее проверять любой из экземпляров дисков на выбранных хостах, используя страницу хоста.

Классическая страница хоста не отображает эту информацию, но информация также доступна через Data Explorer.

### Включение для группы хостов

Если вы включите расширение для группы хостов, эта настройка переопределяет любую настройку на уровне среды, но, в свою очередь, может быть переопределена настройкой на уровне хоста для любого хоста в группе хостов.

Чтобы включить или отключить расширение для всех хостов Linux в группе хостов:

1. Перейдите к ![Deployment Status](https://dt-cdn.net/images/deploy-status-512-c91e319ae5.png "Deployment Status") **Deployment Status** > **OneAgents**.
2. На странице **OneAgent deployment** отключите **Show new OneAgent deployments**.
3. В поле **Filter by** введите **Host group**, а затем выберите группу хостов, которую хотите настроить, из выпадающего списка.

   Список хостов теперь отфильтрован по выбранной группе хостов. Каждый перечисленный хост имеет ссылку **Host group:** `<group name>`, где `<group name>` — это имя группы хостов, которую вы хотите настроить.

   Свойство **Host group** не отображается, если выбранный хост не принадлежит ни к одной группе хостов.
4. Выберите имя группы хостов в любой строке.

   Поскольку вы выполнили фильтрацию по группе хостов, все отображаемые хосты относятся к одной и той же группе хостов.

5. В настройках группы хостов выберите **Disk Analytics Extension**.
6. На странице **Disk Analytics extension** для выбранной группы хостов включите или отключите расширение по мере необходимости.

### Включение для хоста

Если вы включите расширение для хоста, это переопределяет любую настройку на уровне группы хостов или среды.

Чтобы включить или отключить расширение на одном хосте Linux:

1. Перейдите к ![Hosts](https://dt-cdn.net/images/hosts-512-59f5d2dd7f.png "Hosts") **Hosts Classic**.
2. Найдите и выберите ваш хост для отображения обзорной страницы хоста.
3. В правом верхнем углу обзорной страницы хоста выберите **More** (**...**) > **Settings**.

4. В настройках хоста выберите **Disk Analytics Extension** и включите или отключите расширение по мере необходимости для выбранного хоста.

## Подробности

### Метрики

1. Перейдите к ![Hosts](https://dt-cdn.net/images/hosts-512-59f5d2dd7f.png "Hosts") **Hosts Classic** и выберите хост, на котором вы хотите проанализировать локальные устройства хранения.
2. Выберите вкладку **Linux disks**, чтобы прокрутить страницу вниз до раздела **Linux disks**.

### Лицензирование и стоимость

Метрики расширения Disk Analytics потребляют единицы данных Davis (DDU) для выбранных хостов. Для каждого дискового устройства отправляется одна точка данных в минуту. Дисковое устройство включает каждую точку монтирования, том, раздел или экземпляр RAID-массива.

## Наборы функций

При активации расширения с помощью [конфигурации мониторинга](#monitoring-configuration) вы можете ограничить мониторинг одним из наборов функций. Для корректной работы расширение должно собрать хотя бы одну метрику после активации.

В сильно сегментированных сетях наборы функций могут отражать сегменты вашей среды. Затем, при создании конфигурации мониторинга, вы можете выбрать набор функций и соответствующую группу ActiveGate, которая может подключаться к этому конкретному сегменту.

Все метрики, не отнесённые к какому-либо набору функций, считаются набором по умолчанию и всегда отправляются.

Метрика наследует набор функций подгруппы, которая, в свою очередь, наследует набор функций группы. Кроме того, набор функций, определённый на уровне метрики, переопределяет набор функций, определённый на уровне подгруппы, который, в свою очередь, переопределяет набор функций, определённый на уровне группы.
