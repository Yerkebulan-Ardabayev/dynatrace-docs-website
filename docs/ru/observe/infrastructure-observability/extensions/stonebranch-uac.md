# Расширение Stonebranch Universal Automation Center


* Latest Dynatrace
* Расширение
* Опубликовано 12 декабря 2025

Мониторинг Stonebranch Universal Automation Center (UAC) через OpenTelemetry.

## Начало работы

### Обзор

Это расширение собирает метрики UAC через Dynatrace ActiveGate и преобразует их в представление, ориентированное на UC, внутри Dynatrace. Оно обнаруживает Universal Controllers, серверы OMS, Universal Agents, рабочие процессы и бизнес-сервисы из меток метрик и отображает их в топологии с выделенными экранами Unified Analysis и примерами панелей мониторинга.

Специалисты IT Ops, SRE и платформенные команды могут использовать это расширение для:

* Отслеживания выполнения задач и результатов рабочих процессов.
* Мониторинга состояния контроллера, JVM и процессов.
* Наблюдения за потреблением лицензий UAC и лимитами ёмкости.
* Получения специфических оповещений UAC при проблемах с OMS, агентами или лицензиями.

Всё это выполняется через сбор метрик — без необходимости написания пользовательских скриптов.

### Сценарии использования

* Обзор всей среды UAC: получите обзор одного или нескольких Universal Controllers в едином представлении внутри Dynatrace.
* Мониторинг состояния контроллера и OMS: раннее обнаружение проблем с узлами контроллера или OMS с помощью метрик статуса и подключения.
* Видимость рабочих нагрузок автоматизации: мониторинг количества выполнений, активных экземпляров и распределения результатов по рабочим процессам и типам задач.
* Управление лицензиями и ёмкостью: отслеживание использования агентов, узлов, выполнений и задач относительно лицензионных квот для предотвращения превышения лимитов.
* Представления по рабочим процессам и агентам: используйте экраны сущностей рабочих процессов и Universal Agent для понимания того, где выполняются задачи, как они работают и к каким бизнес-сервисам относятся.
* Интегрированное устранение неполадок внутри Dynatrace: используйте панели мониторинга, экраны сущностей и оповещения для быстрой изоляции проблем в логике автоматизации, контроллере, агентах или базовой инфраструктуре.

### Информация о совместимости

* Universal Controller: 7.9.0.0+
* Universal Agent: 7.9.0.0+

## Активация и настройка

1. Установите и активируйте расширение из Dynatrace Hub.
2. Настройте конечную точку метрик. Укажите расширение на конечную точку метрик Prometheus вашего Universal Controller.
3. Настройте аутентификацию. Убедитесь, что пользователь UC (или эквивалентный сервисный пользователь) имеет роль `ops_service`.
4. рекомендуется Включите метки рабочих процессов и бизнес-контекста. Для более глубокого понимания рабочих процессов, агентов и бизнес-сервисов настройте дополнительные метки в `uc.properties`, чтобы они появлялись в метриках, таких как `uc_history_total`.

   Например

   * `task_name` (имя рабочего процесса/задачи)
   * `security_business_services` (имена бизнес-сервисов)
   * `agent_id`
   * `cluster_node_id`

   Эти метки используются в маппинге топологии (рабочие процессы, бизнес-сервисы, Universal Agents), на панелях мониторинга рабочих процессов/агентов и на экранах сущностей.

   Для получения дополнительных сведений см.:

   * Документация Stonebranch по свойствам: [Properties](https://stonebranchdocs.atlassian.net/wiki/spaces/UC79/pages/1614542260/Properties).
   * Детали реализации и примеры: [Dynatrace Observability for UAC - Implementation Guide](https://stonebranchdocs.atlassian.net/wiki/spaces/UE/pages/1949892609/Dynatrace+Observability+for+UAC+-+Implementation+Guide).

## Детали

### Загрузка метрик

Расширение использует Dynatrace ActiveGate для сбора метрик в формате Prometheus с одного или нескольких Universal Controllers. Метрики сгруппированы в наборы функций, включая

* Метрики Universal Controller

  + Рабочая нагрузка и история
  + Пул подключений к базе данных
* Стандартные метрики процессов

  Основаны на метриках вроде `process_cpu_seconds_total`, `process_virtual_memory_bytes`.
  Примеры включают

  + Время CPU
  + Виртуальная и резидентная память
  + Время запуска процесса
  + Открытые/максимальные файловые дескрипторы
* Метрики JVM

  + Использование кучи и не-кучи, пулы памяти
  + Использование и ёмкость буферного пула
  + Время сборки мусора (GC)
  + Количество и состояния потоков
  + Статистика загрузки классов
  + Информация о среде выполнения JVM
* Метрики Universal Agent и OMS

  + Статус агента
  + Статус OMS
  + Серверная сессия
  + Время последнего подключения
* Метрики лицензий и ёмкости

  + Текущее количество используемых распределённых агентов
  + Максимальное количество распределённых агентов, которые могут использоваться
  + Используемые / максимальные агенты z/OS
  + Используемые / максимальные узлы кластера
  + Используемые / максимальные ежемесячные выполнения
  + Используемые / максимальные определения задач
  + `uc_monthly_executions` для подсчёта выполнений за текущий месяц

Эти метрики обеспечивают высокоуровневый обзор UAC.

### Топология и модель сущностей

Расширение отображает измерения метрик в сущности и связи Dynatrace, позволяя визуально навигировать по ландшафту автоматизации. Например, вы можете перейти от контроллера к рабочим процессам, выполняющимся на нём, и далее к агентам, которые их выполняют.

Типы сущностей

* Universal Controller
* OMS
* Workflows (Рабочие процессы)
* Business Services (Бизнес-сервисы)
* Universal Agent

### Экраны Unified Analysis и панели мониторинга

Расширение поставляется с

* Обзорной панелью мониторинга Universal Controller для высокоуровневого мониторинга
* Экранами Unified Analysis для

  + Universal Controller
  + Workflow (Рабочий процесс)
  + Universal Agent

Примеры представлений включают

* Распределение истории по всему UC по типу и имени задачи
* Активные экземпляры задач по статусу
* Количество ежемесячных выполнений
* Статус экземпляров рабочих процессов и группировка по бизнес-сервисам
* Распределение задач Universal Agent по типу и статусу

### Встроенные оповещения

Расширение включает предопределённые конфигурации оповещений, которые сохранены как шаблоны оповещений. Они доступны для

* Доступности сервера OMS
* Доступности узла кластера UC
* Состояния пула подключений к базе данных
* Различных оповещений, связанных с лицензией UC

Эти оповещения основаны исключительно на метриках и могут быть настроены или расширены в Dynatrace по мере необходимости.

### Лицензирование и стоимость

Использование расширения бесплатно. Оплата взимается только за данные, которые расширение загружает. Детали потребления лицензии зависят от используемой модели лицензирования. Это может быть [классическое лицензирование Dynatrace](../../../license/monitoring-consumption-classic.md "Understand how Dynatrace monitoring consumption is calculated for classic licensing.") или модель [Dynatrace Platform Subscription (DPS)](../../../license.md "About Dynatrace Platform Subscription (DPS), the licensing model for all Dynatrace capabilities.").

#### Метрики

Потребление лицензии основано на количестве загруженных точек данных метрик. Приблизительная годовая оценка при включении всех наборов функций показана ниже.

```
(


(


( 1 * number of UC Cluster Nodes )


+ ( 46 * number of Universal Controllers )


+ ( 1 * number of OMS )


+ ( 4 * number of Universal Agents )


+ ( 5 * number of unique Task Names )


)


) * 60 minutes * 24 hours * 365 days per year
```

#### Классическое лицензирование

В модели классического лицензирования Dynatrace загрузка метрик потребляет [Davis Data Units (DDU)](../../../license/monitoring-consumption-classic/davis-data-units.md "Understand how Dynatrace monitoring consumption is calculated based on Davis data units (DDU).") со скоростью 0,001 DDU за точку данных метрики.
Чтобы оценить годовое потребление DDU, возьмите результат приведённой выше формулы для годовых точек данных и умножьте его на 0,001.

## Часто задаваемые вопросы

Собирает ли это расширение трассировки или логи?

Нет. Это расширение ориентировано на мониторинг на основе метрик и топологию. Трассировки и логи могут быть добавлены отдельно с использованием возможностей OpenTelemetry или пересылки логов UAC.

Можно ли настраивать панели мониторинга и оповещения?

Да. Поставляемые примеры и шаблоны оповещений являются отправной точкой. Вы можете клонировать и адаптировать их в Dynatrace.

Почему некоторые плитки не работают?

Если некоторые плитки не отображают данные, проверьте, имеет ли пользователь доступ на чтение конечной точки метрик Universal Controller или включены ли необязательные метрики в вашем `uc.properties`.
