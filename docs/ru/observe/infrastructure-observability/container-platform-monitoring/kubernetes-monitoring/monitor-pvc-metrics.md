# Мониторинг запросов на постоянные тома в Kubernetes/OpenShift

# Мониторинг запросов на постоянные тома в Kubernetes/OpenShift

* Классическая версия
* Чтение: 2 мин.
* Обновлено 09 июля 2024 г.

На этой странице описаны минимальные требования к версии для базовых возможностей мониторинга запросов на постоянные тома.

Dynatrace версии 1.294 +

ActiveGate версии 1.289+

В Kubernetes постоянные данные хранятся в [запросах на постоянные тома (PVC)](https://dt-url.net/n403y11). Dynatrace предоставляет необходимые сведения о ёмкости ваших запросов на постоянные тома.

* Dynatrace предоставляет предустановленные дашборды **Kubernetes persistent volume claims**, позволяющие анализировать запросы на постоянные тома по общей ёмкости, использованию, оставшемуся свободному месту и темпам роста.
* Шаблоны для пользовательских оповещений позволяют получать уведомления о соответствующих проблемах, например об исчерпании свободного места в запросах на постоянные тома или об их необычном росте.

Чтобы начать мониторинг запросов на постоянные тома, см. раздел ниже.

## Включение

Мониторинг PVC является встроенной функцией и не требует ручного включения.

## Разрешения

Убедитесь, что в Kubernetes ClusterRole включены правило `get` и ресурсы `nodes/metrics`. Если вы осуществляете мониторинг PVC с помощью ActiveGate, работающего вне кластера, вам также потребуется разрешение `nodes/proxy`.

Пример:

![Пример разрешений PVC](https://dt-cdn.net/images/screenshot-1-1215-23c529f37c.png)

## Дашборды

Dynatrace предоставляет предварительно настроенный дашборд, охватывающий следующие варианты использования:

* Отображение пространств имён с наибольшим / наименьшим объёмом свободного места
* Отображение пространств имён с наибольшим темпом роста
* Отображение топ-10 списка самых крупных PVC
* Отображение ёмкости / использования по пространству имён

![Пример дашборда PVC](https://dt-cdn.net/images/kubernetes-persistent-volume-claims-dashboard-1357-3d62f7e8b3.png)
