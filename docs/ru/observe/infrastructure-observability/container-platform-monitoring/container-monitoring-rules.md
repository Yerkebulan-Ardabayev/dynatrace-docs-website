# Правила мониторинга контейнеров

* Classic
* How-to guide
* 2-min read
* Published Jun 25, 2021

Для управления всеми настройками, связанными с контейнерами, следуйте приведённым ниже инструкциям.

## Определение правил мониторинга контейнеров

Правила мониторинга контейнеров определяют, включён ли глубокий мониторинг для процессов, выполняющихся в контейнерах.

**Добавление правила мониторинга**

1. Перейдите в **Settings** > **Processes and containers** > **Container monitoring rules**.
2. Выберите **Add item**.
3. Выберите **Mode**: нужно ли Dynatrace отслеживать данный контейнер.
4. Выберите **Container property** для сравнения с вашим правилом.
5. Выберите оператор условия (например, `begins with`)
6. Задайте значение условия.
7. Выберите **Save changes**, чтобы сохранить конфигурацию и добавить новое правило в список правил мониторинга контейнеров.

**Редактирование пользовательского правила мониторинга**

1. Перейдите в **Settings** > **Processes and containers** > **Container monitoring rules**.
2. Найдите правило и внесите необходимые изменения.

   * Чтобы изменить условия правила, раскройте раздел **Details** и внесите нужные изменения
   * Чтобы удалить правило, выберите соответствующий элемент в столбце **Delete**
   * Чтобы отключить или включить правило, измените положение переключателя в столбце **Enabled**
   * Чтобы переместить правило вверх или вниз по списку, в столбце **Order** выберите и перетащите правило на новую позицию

     Правила мониторинга контейнеров выполняются в том порядке, в котором они отображаются в списке.
3. Выберите **Save changes**.

## Включение и отключение правил мониторинга контейнеров

Существует две категории правил мониторинга контейнеров, которые можно включать или отключать по умолчанию:

* Пользовательские правила — созданные вами
* Встроенные правила — определённые Dynatrace

**Включение/отключение пользовательского правила мониторинга**

1. Перейдите в **Settings** > **Processes and containers** > **Container monitoring rules**.
2. Найдите правило и измените положение переключателя в столбце **Enabled**.
3. Выберите **Save changes**, чтобы сохранить конфигурацию.

**Включение/отключение встроенных правил мониторинга**

1. Перейдите в **Settings** > **Processes and containers** > **Built-in container monitoring rules**.
2. Включите или отключите следующие правила по мере необходимости:

   * Do not monitor containers where Kubernetes container name equals `POD`
   * Do not monitor containers where Docker stripped image name contains `pause-amd64`
   * Do not monitor containers where Kubernetes namespaces equals `openshift-sdn`
3. Выберите **Save changes**, чтобы сохранить конфигурацию.

Встроенные правила включены по умолчанию. Вы можете отключить их, но не можете редактировать.

## Ограничения

* Правила мониторинга контейнеров действуют только при установке OneAgent на хостах.
* Интеграции только на уровне приложения без полной установки OneAgent не поддерживают правила мониторинга. Однако в таких ситуациях сами интеграции фактически обеспечивают такой же уровень контроля над настройкой мониторинга контейнеров.
* В Kubernetes правила мониторинга контейнеров применяются только к режиму внедрения `classicFullStack`.

  Правила мониторинга контейнеров игнорируются для режимов внедрения на основе webhook (`cloudNativeFullStack` или `applicationMonitoring`). Для этих режимов используйте вариант конфигурации на основе аннотаций, как описано в [Configure monitoring for namespaces and pods](../../../ingest-from/setup-on-k8s/guides/deployment-and-configuration/monitoring-and-instrumentation/annotate.md "Configure monitoring for namespaces and pods").
