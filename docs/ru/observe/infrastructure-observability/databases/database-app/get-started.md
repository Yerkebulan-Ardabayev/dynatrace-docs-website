# Начало работы с мониторингом баз данных

* Последняя версия Dynatrace
* Практическое руководство
* Опубликовано 20 января 2026 г.

Мониторинг баз данных в Dynatrace начинается с установки приложения ![Databases](https://dt-cdn.net/images/dynatrace-database-256-1afe08286e.webp "Databases") **Databases** из **Hub**. Затем можно использовать SQL-расширения для сбора метрик производительности, аналитики запросов и пользовательских SQL-данных, добавляя конфигурации мониторинга для каждого экземпляра.

## Установка приложения Databases

1. Перейдите в **Hub**.
2. Найдите приложение ![Databases](https://dt-cdn.net/images/dynatrace-database-256-1afe08286e.webp "Databases") **Databases**, откройте обзор приложения и нажмите **Open**.
3. Убедитесь, что ваша среда соответствует необходимым [разрешениям](/docs/observe/infrastructure-observability/databases/database-app#permissions "Приложение Databases предоставляет обзор всех баз данных, отслеживаемых с помощью Extensions Framework 2.0.").

## Настройка мониторинга баз данных в Dynatrace

Ознакомьтесь с доступными SQL-расширениями ниже, чтобы узнать, как настроить каждое из них для мониторинга баз данных. Эти расширения обеспечивают метрики производительности, аналитику запросов и пользовательский SQL-мониторинг без необходимости установки агентов на хостах баз данных. Чтобы начать мониторинг экземпляров баз данных, следуйте шагам активации и настройки для каждого расширения.
