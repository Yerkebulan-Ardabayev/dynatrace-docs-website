Приложение **Databases** — мониторинг, анализ и оптимизация баз данных. Детальная информация о производительности, состоянии и конфигурации. Интеграция с экосистемой Dynatrace для сквозной наблюдаемости.

## Предварительные требования

* **ActiveGate**: назначьте группы ActiveGate для подключения к серверам баз данных.
* **Компоненты**: установите необходимые драйверы (например, JDBC) на ActiveGate.
* **Сеть**: проверьте подключение и правила файрвола. Создайте учётную запись мониторинга с нужными разрешениями.

### Разрешения

| Разрешение | Описание |
|---|---|
| davis:analyzers:execute | Анализатор Davis для проблем |
| settings:objects:read/write | Settings V2 для Ownership и SLO |
| state:user-app-states:read/write | Состояние UI |
| storage:buckets:read | Grail buckets |
| storage:entities:read | Сущности из Grail |
| storage:events:read | События из Grail |
| storage:logs:read | Логи из Grail |
| storage:metrics:read | Метрики из Grail |

Для строгого контроля доступа — политика ограничения бакета `default_database_monitoring`:

```
ALLOW storage:buckets:read WHERE storage:bucket-name = 'default_database_monitoring'
```

### Установка

1. Установите **Databases** из Dynatrace Hub.
2. Настройте мониторинг экземпляров. См. [Начало работы с мониторингом баз данных](database-app/get-started.md).
3. Добавьте учётные данные и конечные точки.

## Начало работы

### Вкладка Overview

Высокоуровневая сводка: состояние баз данных (здоровые/нездоровые), метрики хостов.

### Вкладка Explorer

Список отслеживаемых экземпляров с перспективами:
* **Состояние** — проблемы, обнаруженные Dynatrace Intelligence.
* **Утилизация** — CPU, память, сессии.
* **Детали хоста** — метрики хоста и переход в Infrastructure & Operations.

Выберите экземпляр для доступа к панели **Entity details** со всеми метриками расширения.

### Производительность запросов

Анализ ресурсоёмких запросов: фильтрация по Time, CPU, Disk, Waits. Планы выполнения для оптимизации.

## Концепции

### Health score

Предопределённая метрика оценивает доступность, производительность, конфигурацию и использование ресурсов.

### Оповещения

* **Health alert** — критические события, инициирующие расследование.
* **Warning signal** — некритические предупреждения.

Вкладка **Alert templates** — предопределённые шаблоны для популярных СУБД.

### Инсайты

Выявление закономерностей, аномалий и тенденций на основе встроенной экспертизы. Рекомендации по устранению проблем.

### Развёртывание

Поддержка: On-premises, Cloud (AWS RDS и др.), Hybrid.

## Сценарии использования

* Мониторинг состояния баз данных в реальном времени.
* Анализ и оптимизация запросов (медленные запросы, индексация, кэширование).
* Обнаружение и устранение проблем с помощью ИИ.
* Сквозная наблюдаемость через Grail, Smartscape, Dynatrace Intelligence.
