# Поддержка переменных привязки SQL

* Классическая версия
* Практическое руководство
* 4 мин. чтения
* Обновлено 26 апреля 2023 г.

Переменные привязки (bind variables) — это способ параметризации SQL-операторов, при котором операторы содержат вопросительные знаки или параметры в своих условиях **where**, например:

* SQL Server: `select count (*) from report where tenant = @tenant`
* Java JDBC: `select count (*) from report where tenant = ?`

Переменные привязки позволяют серверу базы данных подготовить оператор один раз и выполнять его многократно без повторного разбора и анализа.

Переменные привязки неприменимы к операторам, использующим литералы, например:

```
select count (*) from report where tenant = 'xxxx'
```

Такие операторы не могут быть параметризованы и повторно разбираются и анализируются сервером базы данных при каждом выполнении.

Переменные привязки создают высокую нагрузку на сеть и хранилище. Дополнительные сведения о поддержке переменных привязки и её доступности см. в разделе [Часто задаваемые вопросы](#faq) ниже.

## Включение захвата переменных привязки SQL

Чтобы включить захват значений привязки SQL:

1. Перейдите в **Settings** > **Server-side service monitoring** > **Deep monitoring**.
2. Разверните раздел **Database** и включите/отключите **Capture SQL bind values** на глобальном уровне.
3. Необязательно: чтобы переопределить глобальные настройки, перейдите в **Process group override**.

   1. Чтобы добавить переопределение, выберите **Add process group override** и укажите нужную группу процессов.
   2. Необязательно: чтобы выбрать конкретный процесс из выбранной группы процессов, выберите его из выпадающего списка.
   3. Выберите **Add**.
4. Нажмите **Save changes**.

![Capture SQL bind values](https://dt-cdn.net/images/capture-sql-bind-values-942-067033b12f.png)

Независимо от того, включаете ли вы этот параметр в масштабе всего сайта или для отдельных групп процессов, вы можете использовать Dynatrace OneAgent для захвата значений переменных привязки. Это применимо к следующим технологиям:

* ADO.net
* JDBC
* Фреймворки баз данных PHP

Если массив, возвращаемый `executeBatch()`, содержит более одного элемента, что указывает на выполнение нескольких команд, Dynatrace маскирует значения переменных привязки для обеспечения конфиденциальности данных. Это связано с тем, что различные выполнения `executeBatch()` могут агрегировать несколько команд, что требует маскировки значений переменных привязки во избежание раскрытия конфиденциальной информации.

## Часто задаваемые вопросы

Переменные привязки недоступны в моей среде Dynatrace. Как получить эту функцию?

Эта функция доступна в средах Dynatrace, лицензированных по [Dynatrace Platform Subscription (DPS)](/docs/license "О Dynatrace Platform Subscription (DPS), модели лицензирования для всех возможностей Dynatrace."). Если у вас есть классическая лицензия Dynatrace, измените план подписки на лицензию Dynatrace Platform Subscription (DPS), чтобы использовать переменные привязки.

Влечёт ли захват переменных привязки негативные последствия?

Эта функция может захватывать большой объём конфиденциальных данных, поэтому следует тщательно обдумать её использование.
Вы также можете замаскировать или удалить части захваченных данных с помощью [OpenPipeline](/docs/platform/openpipeline "Масштабируйте обработку данных платформы Dynatrace с помощью Dynatrace OpenPipeline.").

Кроме того, эта функция может захватывать большой объём данных в абсолютном выражении.
Активное использование этой функции означает, что включённый объём трассировки для ваших приложений с Full-Stack мониторингом будет исчерпан быстрее.
Иными словами, активное использование может снизить частоту захвата трассировок.
Чтобы снизить этот эффект, следует либо использовать эту функцию только для устранения неполадок, либо подключить [Extended trace ingest](/docs/license/capabilities/app-infra-observability/full-stack-monitoring#extend-trace-ingest "Узнайте о тарификации возможности Dynatrace Full-Stack Monitoring DPS.") для обеспечения возросшего объёма данных.
