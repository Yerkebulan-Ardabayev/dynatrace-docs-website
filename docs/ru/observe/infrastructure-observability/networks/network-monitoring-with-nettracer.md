# Расширенный мониторинг сети

* Практическое руководство
* Чтение: 5 мин
* Обновлено 3 окт. 2025

Расширьте мониторинг сети с помощью метрик сетевого трафика на контейнеризованных хостах Linux.

С добавлением сетевых метрик к вашим контейнеризованным хостам анализ первопричин Davis будет использовать их и расширять анализ для обеспечения видимости проблем, связанных с сетью. Интенсивный сетевой трафик на определённых узлах является сигналом о необходимости масштабирования кластера.

## NetTracer

NetTracer — это инструмент с открытым исходным кодом для отслеживания событий TCP и сбора метрик сетевых подключений в Linux. Он состоит из двух частей:

* BPF-программа, используемая для сбора данных
* Бинарный файл, который представляет данные в структурированном или полуструктурированном формате

Преимущества:

* Может отслеживать TCP-события: **connect**, **accept** и **close**
* Может собирать метрики по каждому отслеживаемому подключению
* Высокопроизводительное приложение (написано на C и C++)
* Не зависит от версии и конфигурации ядра (ядро Linux 4.15 и выше)
* Проект с открытым исходным кодом ([NetTracer](https://github.com/dynatrace-oss/nettracer-bpf))

NetTracer определяет TCP-подключение IPv4 и IPv6 по адресу и порту источника, адресу и порту назначения, PID взаимодействующего процесса и сетевому пространству имён.

Используя это определение TCP-подключения, он собирает следующие метрики:

* Отправленные байты
* Полученные байты
* Отправленные пакеты
* Полученные пакеты
* Повторно переданные пакеты
* Время кругового обхода (в микросекундах)
* Дисперсия времени кругового обхода (не используется в анализе Dynatrace)

По умолчанию NetTracer входит в состав каждой установки OneAgent в виде бинарного файла `oneagentnettracer` и может быть включён через веб-интерфейс Dynatrace.

## Поддерживаемые платформы NetTracer

NetTracer официально поддерживает версии ядра Linux 4.15 и выше, но другие компоненты Dynatrace, сосуществующие с NetTracer на конкретном хосте, имеют специфические требования и поддерживаются на определённых дистрибутивах Linux. В следующей таблице перечислены протестированные и наиболее безопасные дистрибутивы Linux для использования NetTracer с Dynatrace.

## Включение NetTracer

При включении OneAgent будет использовать NetTracer для сбора сетевых данных из контейнеров, но только для хостов Linux.

Чтобы включить NetTracer на конкретном хосте Linux

1. Перейдите в ![Hosts](https://dt-cdn.net/images/hosts-512-59f5d2dd7f.png "Hosts") **Hosts Classic** и выберите ваш хост Linux.
2. На странице обзора хоста нажмите **More** (**...**) > **Settings** в правом верхнем углу страницы.
3. На странице **Host settings** выберите **NetTracer traffic** и включите **Enable NetTracer traffic network monitoring**.

Чтобы включить NetTracer глобально на всех ваших хостах Linux

1. Перейдите в **Settings** > **Network & Discovery** > **NetTracer traffic**.
2. Включите **Enable NetTracer traffic network monitoring**.

Для корректной работы NetTracer необходимо, чтобы OneAgent был установлен в режиме Full-Stack или Infrastructure monitoring, поскольку эти режимы включают функцию мониторинга сети. Если OneAgent установлен в ограниченном режиме (например, Discovery monitoring), NetTracer может работать некорректно. Подробнее см. [Режимы мониторинга OneAgent](../../../platform/oneagent/monitoring-modes/monitoring-modes.md "Узнайте больше о доступных режимах мониторинга при использовании OneAgent.").

## Встроенные метрики для NetTracer

| Ключ метрики | Название и описание | Единица измерения | Агрегации |
| --- | --- | --- | --- |
| builtin:tech.nettracer.bytes\_rx | Полученные байты. Количество полученных байтов | Byte | autoavgcountmaxminsum |
| builtin:tech.nettracer.bytes\_tx | Переданные байты. Количество переданных байтов | Byte | autoavgcountmaxminsum |
| builtin:tech.nettracer.pkts\_retr | Повторно переданные пакеты. Количество повторно переданных пакетов | Count | autovalue |
| builtin:tech.nettracer.pkts\_rx | Полученные пакеты. Количество полученных пакетов | Count | autovalue |
| builtin:tech.nettracer.pkts\_tx | Переданные пакеты. Количество переданных пакетов | Count | autovalue |
| builtin:tech.nettracer.retr\_percentage | Повторные передачи. Процент повторно переданных пакетов | Percent (%) | autoavgmaxmin |
| builtin:tech.nettracer.rtt | Время кругового обхода. Время кругового обхода в миллисекундах. Агрегирует данные из активных сессий | Millisecond | autoavgcountmaxminsum |
| builtin:tech.nettracer.traffic | Сетевой трафик. Суммарный входящий и исходящий сетевой трафик в битах в секунду | bit/s | autovalue |
| builtin:tech.nettracer.traffic\_rx | Входящий трафик. Входящий сетевой трафик в битах в секунду | bit/s | autovalue |
| builtin:tech.nettracer.traffic\_tx | Исходящий трафик. Исходящий сетевой трафик в битах в секунду | bit/s | autovalue |

### Вычисляемые метрики для NetTracer

Следующие метрики, доступные для NetTracer, являются вычисляемыми:

* `builtin:tech.nettracer.retr_percentage` (Повторные передачи)

  Повторные передачи = повторно переданные пакеты / (повторно переданные пакеты + переданные пакеты) x 100
* `builtin:tech.nettracer.traffic_rx` (Входящий трафик)

  Входящий трафик = (сумма полученных байтов \* 8) в секунду
* `builtin:tech.nettracer.traffic_tx` (Исходящий трафик)

  Исходящий трафик = (сумма переданных байтов:sum \* 8) в секунду
* `builtin:tech.nettracer.traffic` (Сетевой трафик)

  Сетевой трафик = ((сумма полученных байтов + сумма переданных байтов) \* 8) в секунду

## Измерения для NetTracer

### Измерения контейнеров для NetTracer

Если процесс выполняется в контейнере, добавляются следующие измерения:

* `dt.entity.container_group_instance`
* `dt.entity.container_group`

Дополнительные измерения контейнеров добавляются в зависимости от типа развёртывания.

## Где можно увидеть данные NetTracer?

После сбора данные NetTracer доступны в виде метрик по всей системе Dynatrace.

* **Data Explorer**: вы можете использовать метрики в [Data Explorer](../../../analyze-explore-automate/explorer.md "Запрос метрик и преобразование результатов для получения нужной информации.") для создания графиков и дашбордов, отображающих интересующие вас данные.
* **Страница экземпляра группы процессов**: перейдите на страницу экземпляра группы процессов и выберите вкладку **Networking**.

  ![Страница экземпляра группы процессов — сетевые данные](https://dt-cdn.net/images/pgi-page-networking-details-2172-bcb6d64191.png)
* **Обзор хоста**: перейдите на страницу обзора хоста и прокрутите вниз до раздела **Network analysis**.

  ![Network analysis](https://dt-cdn.net/images/lde68092-dev-apps-dynatracelabs-com-ui-apps-dynatrace-classic-hosts-ui-entity-host-b8ec70b7dc022ec8-gtf-2h-gf-all-sessionid-4aquod8c7d1hqbf8-2-2913-bca5600276.png)

Разделы **Networking** и **Network analysis** содержат данные NetTracer в сочетании с другими сетевыми данными, проанализированными для этого хоста. NetTracer собирает данные для контейнеризованных процессов, в то время как Network Agent — для нативных (т.е. неконтейнеризованных) процессов.

## Характеристики NetTracer

* Из модуля NetTracer `ebpf` отслеживаются только 4096 TCP-подключений.
* Информация о портах прослушивания требует активного TCP-подключения.
