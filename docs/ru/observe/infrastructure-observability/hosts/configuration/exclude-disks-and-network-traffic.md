OneAgent автоматически обнаруживает и отслеживает точки монтирования и сетевой трафик хоста, однако вы можете исключить отдельные точки монтирования или сетевой трафик из мониторинга.

## Исключение дисков

Используйте настройки **Disk options** для создания правил исключения дисков из мониторинга.

* Некоторые файловые системы (например, `autofs`, `proc`, `cgroup`, `tmpfs`) всегда исключаются, поскольку их мониторинг не представляет практической ценности.
* Можно создавать правила исключения дисков на уровне среды, группы хостов или отдельного хоста. Исключения, заданные на более низком уровне, переопределяют исключения более высокого уровня (например, исключения на уровне хоста переопределяют исключения на уровне среды).

Чтобы создать правило исключения дисков:

1. Перейдите на страницу **Disk options** для нужного уровня:

   * Среда

     Перейдите в **Settings** > **Preferences** > **Disk options**.
   * Группа хостов

     1. Перейдите в **Deployment Status** > **OneAgents**.
     2. Отфильтруйте таблицу по `Host group` и выберите группу хостов, для которой нужно создать правило исключения диска.
     3. Для любого из перечисленных хостов (все они входят в выбранную группу) выберите ссылку **Host group**.
     4. Выберите **Disk options**.
   * Хост

     1. Перейдите в ![Hosts](https://dt-cdn.net/images/hosts-512-59f5d2dd7f.png "Hosts") **Hosts Classic** и выберите хост, для которого нужно создать правило исключения диска.
     2. На странице обзора хоста выберите  > **Settings**.
     3. Выберите **Disk options**.

   Находясь на странице **Disk options**, шаги по созданию правила исключения диска одинаковы. Принципиальное различие состоит в уровне, к которому применяется правило: среда, группа хостов или хост.
2. Выберите **Add item**.
3. Выберите **Operating system** исключаемого диска.
4. В поле **Disk or mount point path** укажите путь к точке монтирования диска, который следует исключить из мониторинга.

   **Примеры:**

   * `/mnt/my_disk`
   * `/staff/emp1`
   * `C:\`
   * `/staff/*`
   * `/disk*`

   * Пути к точкам монтирования чувствительны к регистру
   * Подстановочный знак в `/staff/*` означает исключение всех дочерних папок `/staff`
   * Подстановочный знак в `/disk*` означает исключение всех точек монтирования, начинающихся с `/disk` (например, `/disk1`, `/disk99` и `/diskabc`)
5. В поле **File system type** укажите тип файловой системы, которую следует исключить из мониторинга.

   **Примеры:**

   * `ext4`
   * `ext3`
   * `btrfs`
   * `ext*`

   * Типы файловых систем чувствительны к регистру
   * Подстановочный знак в `ext*` означает исключение соответствующих файловых систем (например, типов `ext4` и `ext3`)
6. Нажмите **Save changes**.

## Исключение сетевого трафика

Используйте настройки **Exclude network traffic** для исключения трафика на определённых сетевых интерфейсах или хостах из мониторинга.

### Исключение сетевого интерфейса (NIC)

Весь сетевой трафик со всех выбранных сетевых интерфейсов (NIC) исключается из мониторинга.

1. На странице **Exclude network traffic** в разделе **Exclude NIC** выберите **Add item**.
2. В поле **Operating system** укажите операционную систему сетевого интерфейса.
3. В поле **Name** введите имя сетевого интерфейса в указанной операционной системе.
4. Нажмите **Save changes**.

### Исключение IP-адреса

Весь сетевой трафик со всех выбранных IP-адресов хостов исключается из расчёта показателей связности (остальные метрики по-прежнему рассчитываются). Это может быть полезно, например, для предотвращения ложных предупреждений о связности.

Чтобы исключить IP-адрес из расчётов связности:

1. На странице **Exclude network traffic** в разделе **Exclude IP** выберите **Add item**.
2. Введите IP-адрес, трафик которого нужно исключить из расчётов связности. Подстановочные знаки и диапазоны не допускаются.
3. Нажмите **Save changes**.

## Диски, не отслеживаемые OneAgent

Следующие диски не отслеживаются OneAgent. Чтобы исключить дополнительный тип файловой системы или имя точки монтирования, см. раздел [Исключение дисков](#disk-options).
