Приложение **Clouds** предназначено для команд облачных операций и SRE. Мониторинг состояния, устранение неполадок и оптимизация мультиоблачных сред.

* Подключение облачных аккаунтов и анализ инвентаря ресурсов за минуты.
* Метрики, события, логи, трассировки, метаданные и топология в одном представлении.
* Оповещения на основе ИИ для сокращения времени устранения неполадок.
* Использование облачных тегов для маршрутизации уведомлений, владельцев и распределения затрат.
* Готовые дашборды.

Данные хранятся в Grail с поддержкой DQL в **Notebooks**, **Dashboards** и **Workflows**.

## Предварительные требования

Приложение **Clouds** устанавливается автоматически.

### Подключения

**Новые подключения (AWS, Azure preview)**:
* Среда Dynatrace на базе Grail и AppEngine в регионе AWS или Azure.
* Лицензия DPS с возможностями: Metrics/Logs/Events powered by Grail.

**Классические подключения (AWS, Azure, GCP)**:
* Среда Dynatrace на базе Grail и AppEngine.
* Доступно для DPS и классической лицензии.

### Разрешения

| Разрешение | Описание |
|---|---|
| settings:objects:read | Чтение настроек владения |
| unified-analysis:screen-definition:read | Конфигурация экрана деталей |
| davis:analyzers:execute | Выполнение анализаторов Davis |
| hub:catalog:read | Чтение каталога Hub |
| app-settings:objects:read | Чтение настроек приложения |
| app-settings:objects:write | Запись настроек приложения |
| slo:slos:read | Чтение SLO |
| state:user-app-states:read/write | Чтение/запись состояния приложения |
| document:documents:read | Чтение документов и дашбордов |

## Начало работы

Встроенный мастер подключения проведёт через все шаги. Для новых пользователей AWS рекомендуются новые подключения вместо классических.

**Новые подключения**:
- [Создание подключения AWS](clouds-app/create-aws-connection.md)
- [Создание подключения Azure](clouds-app/create-azure-connection.md)

**Классические подключения**: **Create connection** > выберите провайдера (Classic connections). Данные доступны на вкладке **Explorer (Classic connections)**.

### Overview (Новые подключения)

Начальная страница со сводкой состояния сервисов AWS/Azure. Плитки сервисов, индикаторы здоровья, готовые дашборды.

### Explorer (Новые подключения)

Анализ облачных сервисов с фильтрацией по тегам, статусу оповещений и атрибутам. Перспективы: **Health**, **Utilization**, **Metadata**. Детали сервиса: метрики, логи, события, конфигурация, топология.

### Оповещения (Новые подключения)

Готовые оповещения о состоянии и предупреждающие сигналы. Настройка через **Settings** > **Analyze and alert** > **Alerts** > **Cloud services**.

**Шаблоны оповещений**: предопределённые шаблоны для создания пользовательских оповещений. **Clouds** > выберите шаблон > **New Alert**.

### Explorer (Классические подключения)

Вкладка для данных от классических подключений AWS, Azure и GCP. Существующие классические подключения продолжают работать.

## Концепции

### Новые vs классические подключения

| | Новые | Классические |
|---|---|---|
| Провайдеры | AWS, Azure (preview) | AWS, Azure, GCP |
| Хранение | Grail | Legacy |
| Вкладки | Overview New, Explorer New | Explorer (Classic) |
| Рекомендация | Для новых пользователей AWS | Для существующих интеграций |

Не рекомендуется настраивать оба типа подключений для одного AWS-аккаунта.

### Перспективы: Health, Utilization, Metadata

* **Health** — состояние работоспособности, оповещения.
* **Utilization** — метрики использования (для вычислительных сервисов).
* **Metadata** — дополнительная информация, облачные теги.

**Режим проблемы** — при переходе от конкретной проблемы в **Problems** выделяются релевантные метрики и сужается временной интервал.

### Готовые дашборды

Доступны через **Dashboards** (поиск по `aws`) или через вкладку **Overview New** в **Clouds**.

### Сегменты

Сегменты для фильтрации данных по первичным полям Grail (например, `aws.account.id`, `aws.region`).

## FAQ
