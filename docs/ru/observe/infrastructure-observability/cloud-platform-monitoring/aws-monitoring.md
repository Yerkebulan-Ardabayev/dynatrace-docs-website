Dynatrace автоматически обнаруживает, устанавливает базовые показатели и интеллектуально отслеживает облачные среды Amazon Web Services.

## Мониторинг и обнаружение проблем

Мониторинг AWS в Dynatrace выполняет автоматический анализ влияния на бизнес на основе ИИ, обнаружение проблем и анализ первопричин для всех сервисов AWS на основе всех метрик, публикуемых в Amazon CloudWatch. Этот анализ может быть дополнительно расширен за счёт полного стекового мониторинга, обеспечиваемого OneAgent, развёрнутым на вычислительных ресурсах, таких как Amazon EC2, AWS Lambda и Kubernetes. Все сущности также могут быть подвергнуты всестороннему анализу журналов.

В дополнение к вышесказанному, Dynatrace обеспечивает понимание нагрузки на применяемые ресурсы AWS и динамики среды по регионам.

Мониторинг AWS доступен для всех клиентов Dynatrace независимо от того, является ли их кластер Dynatrace SaaS-based или Managed.

## Простота использования и удобная визуализация

Мониторинг Amazon Web Services поставляется с согласованными готовыми метриками, дашбордами и оповещениями сразу после включения мониторинга.

Измерения и анализ основных сервисов AWS визуализируются на специально разработанном инфографическом дашборде (см. галерею). Удобные отчёты и дашборды также доступны для анализа влияния на бизнес и карт топологии.

## Гибкость

Пользователи Dynatrace могут выбирать, какие сервисы и экземпляры отслеживаются и как осуществляется этот процесс. Этот выбор может быть сделан в рамках начальной настройки мониторинга или в любой последующий момент. Это также применяется к новым сервисам, включённым в облаке AWS или добавленным компанией Amazon.

Для каждого из поддерживаемых сервисов можно выбирать метрики мониторинга. Аналогично, все дашборды для мониторинга AWS можно клонировать и легко настраивать по мере необходимости.

## Гибридные среды

Отслеживание сервисов и зависимостей не ограничивается AWS. Благодаря Dynatrace Intelligence и её способности обрабатывать данные из всех типов сред, Dynatrace оптимизирована для мониторинга гибридных сред, включая бизнес-приложения, охватывающие несколько облачных и виртуализационных платформ, таких как Microsoft Azure, Google Cloud, VMware, Kubernetes, Openshift и локальную инфраструктуру.

## Обработка журналов с помощью парсеров технологических пакетов

Через OpenPipeline вы можете использовать и настраивать технологические пакеты. Технологический пакет — это библиотека парсеров (правил обработки), которые обрабатывают журналы из различных технологий, таких как Java, .NET и Microsoft IIS.

Парсеры помогают улучшить фильтрацию, устранение неисправностей, метрики, оповещения и дашборды, эффективно извлекая уровни журналов и соответствующие атрибуты. Вы также можете использовать технологические пакеты для структурирования журналов от технологий, которые не поддерживаются Dynatrace из коробки.
