# Расширенный сетевой мониторинг


* Классическая версия
* Практическое руководство
* 5 мин. чтения
* Обновлено 3 октября 2025 г.

Расширение сетевого мониторинга с помощью метрик сетевого трафика в контейнерных Linux-хостах.

С добавлением сетевых метрик к вашим контейнерным хостам анализ первопричин Davis® будет использовать их и расширит анализ для обеспечения видимости сетевых проблем. Интенсивный сетевой трафик на отдельных узлах является признаком того, что следует рассмотреть масштабирование кластера.

## NetTracer

NetTracer — это инструмент с открытым исходным кодом для трассировки TCP-событий и сбора метрик сетевых соединений в Linux. Он состоит из двух частей:

* BPF-программа для сбора данных
* Бинарный файл, представляющий данные в структурированном или полуструктурированном формате

Преимущества:

* Может трассировать TCP-события: **connect**, **accept** и **close**
* Может собирать метрики о каждом отслеживаемом соединении
* Высокопроизводительное приложение (написано на C и C++)
* Независимо от версии и конфигурации ядра (Linux kernel 4.15 и выше)
* Проект с открытым исходным кодом ([NetTracer](https://github.com/dynatrace-oss/nettracer-bpf))

NetTracer определяет IPv4 и IPv6 TCP-соединение по адресу и порту источника, адресу и порту назначения, PID взаимодействующего процесса и сетевому пространству имён.

Используя это определение TCP-соединения, он собирает следующие метрики:

* Отправленные байты
* Полученные байты
* Отправленные пакеты
* Полученные пакеты
* Повторно переданные пакеты
* Время кругового обхода (Round-Trip Time, в микросекундах)
* Дисперсия времени кругового обхода (не используется в анализе Dynatrace)

По умолчанию NetTracer включён как бинарный файл `oneagentnettracer` с каждой установкой OneAgent и может быть включён через веб-интерфейс Dynatrace.

## Поддерживаемые платформы NetTracer

NetTracer официально поддерживает версии ядра Linux 4.15 и выше, но другие компоненты Dynatrace, сосуществующие с NetTracer на конкретном хосте, имеют особые требования и поддерживаются на определённых дистрибутивах Linux. В следующей таблице перечислены протестированные и наиболее безопасные дистрибутивы Linux для использования NetTracer с Dynatrace.

## Включение NetTracer

При включении OneAgent будет использовать NetTracer для сбора сетевых данных из контейнеров, но только для Linux-хостов.

Чтобы включить NetTracer на конкретном Linux-хосте

1. Перейдите в ![Hosts](https://dt-cdn.net/images/hosts-512-59f5d2dd7f.png "Hosts") **Hosts Classic** и выберите ваш Linux-хост.
2. На странице обзора хоста выберите **Дополнительно** (**…**) > **Настройки** в правом верхнем углу страницы.
3. На странице **Настройки хоста** выберите **Трафик NetTracer** и включите **Включить сетевой мониторинг трафика NetTracer**.

Чтобы включить NetTracer глобально на всех ваших Linux-хостах

1. Перейдите в **Настройки** > **Сеть и обнаружение** > **Трафик NetTracer**.
2. Включите **Включить сетевой мониторинг трафика NetTracer**.

Для корректной работы NetTracer OneAgent должен быть установлен в режиме Full-Stack или Infrastructure мониторинга, так как эти режимы включают функцию сетевого мониторинга. Если OneAgent установлен в ограниченном режиме (например, Discovery), NetTracer может работать некорректно. Подробнее см. [Режимы мониторинга OneAgent](../../../platform/oneagent/monitoring-modes/monitoring-modes.md "Узнайте больше о доступных режимах мониторинга при использовании OneAgent.").

## Встроенные метрики NetTracer

| Ключ метрики | Название и описание | Единица | Агрегации |
| --- | --- | --- | --- |
| builtin:tech.nettracer.bytes\_rx | Полученные байты — количество полученных байтов | Байт | auto avg count max min sum |
| builtin:tech.nettracer.bytes\_tx | Переданные байты — количество переданных байтов | Байт | auto avg count max min sum |
| builtin:tech.nettracer.pkts\_retr | Повторно переданные пакеты — количество повторно переданных пакетов | Количество | auto value |
| builtin:tech.nettracer.pkts\_rx | Полученные пакеты — количество полученных пакетов | Количество | auto value |
| builtin:tech.nettracer.pkts\_tx | Переданные пакеты — количество переданных пакетов | Количество | auto value |
| builtin:tech.nettracer.retr\_percentage | Повторная передача — процент повторно переданных пакетов | Процент (%) | auto avg max min |
| builtin:tech.nettracer.rtt | Время кругового обхода — время кругового обхода в миллисекундах. Агрегирует данные из активных сессий | Миллисекунда | auto avg count max min sum |
| builtin:tech.nettracer.traffic | Сетевой трафик — суммарный входящий и исходящий сетевой трафик в битах в секунду | бит/с | auto value |
| builtin:tech.nettracer.traffic\_rx | Входящий трафик — входящий сетевой трафик в битах в секунду | бит/с | auto value |
| builtin:tech.nettracer.traffic\_tx | Исходящий трафик — исходящий сетевой трафик в битах в секунду | бит/с | auto value |

### Вычисляемые метрики NetTracer

Следующие метрики, доступные для NetTracer, являются вычисляемыми:

* `builtin:tech.nettracer.retr_percentage` (Повторная передача)

  Повторная передача = повторно переданные пакеты / (повторно переданные пакеты + переданные пакеты) × 100
* `builtin:tech.nettracer.traffic_rx` (Входящий трафик)

  Входящий трафик = (сумма полученных байтов \* 8) в секунду
* `builtin:tech.nettracer.traffic_tx` (Исходящий трафик)

  Исходящий трафик = (сумма переданных байтов \* 8) в секунду
* `builtin:tech.nettracer.traffic` (Сетевой трафик)

  Сетевой трафик = ((сумма полученных байтов + сумма переданных байтов) \* 8) в секунду

## Размерности NetTracer

### Размерности контейнеров для NetTracer

Если процесс выполняется в контейнере, добавляются следующие размерности:

* `dt.entity.container_group_instance`
* `dt.entity.container_group`

Дополнительные размерности контейнеров добавляются в зависимости от типа развёртывания.

## Где можно увидеть данные NetTracer?

После сбора данные NetTracer доступны как метрики по всему Dynatrace.

* **Data Explorer**: Вы можете использовать метрики в [Data Explorer](../../../analyze-explore-automate/explorer.md "Запрос метрик и трансформация результатов для получения нужной аналитики.") для создания диаграмм и панелей, отображающих интересующие вас данные.
* **Страница экземпляра группы процессов**: Перейдите на страницу экземпляра группы процессов и выберите вкладку **Сеть**.

  ![Страница экземпляра группы процессов — Детали сети](https://dt-cdn.net/images/pgi-page-networking-details-2172-bcb6d64191.png)
* **Обзор хоста**: Перейдите на страницу обзора хоста и прокрутите до раздела **Анализ сети**.

  ![Анализ сети](https://dt-cdn.net/images/lde68092-dev-apps-dynatracelabs-com-ui-apps-dynatrace-classic-hosts-ui-entity-host-b8ec70b7dc022ec8-gtf-2h-gf-all-sessionid-4aquod8c7d1hqbf8-2-2913-bca5600276.png)

Разделы **Сеть** и **Анализ сети** содержат данные NetTracer в сочетании с другими сетевыми данными, проанализированными для данного хоста. NetTracer собирает данные для контейнерных процессов, а Network Agent — для нативных (т.е. неконтейнерных) процессов.

## Характеристики NetTracer

* Из модуля NetTracer `ebpf` отслеживается не более 4096 TCP-соединений.
* Информация о прослушиваемых портах требует активного TCP-соединения.
