# Ограничение мониторинга инфраструктуры VMware с помощью разрешений


* Classic
* Практическое руководство
* Опубликовано 10 фев. 2021 г.

Следующее относится только к VMware. Для других платформ виртуализации достаточно установить OneAgent для мониторинга виртуализированных хостов, поскольку мониторинг уровней управления виртуализацией поддерживается только для VMware.

При настройке мониторинга VMware вы можете управлять инфраструктурными элементами (такими как хосты и виртуальные машины), которые Dynatrace должен отслеживать, устанавливая или изменяя разрешения в vCenter.

Смотрите приведённые ниже примеры с подробностями о том, как управлять правами доступа пользователей для ваших виртуальных машин.

## Предварительные требования

* Роль администратора с доступом к управлению разрешениями в vCenter

## Мониторинг всех виртуальных машин в пуле ресурсов

Для мониторинга всех виртуальных машин в пуле ресурсов (или любом другом «родительском» объекте в иерархии инфраструктуры VMware) необходимо

Назначить пользователям права доступа только для чтения к вашим хостам VMware

Для назначения пользователям прав доступа только для чтения к вашим хостам VMware

1. В vCenter перейдите в представление хоста.
2. Выберите **Permissions**, затем нажмите знак `+`.
3. Выберите имя пользователя, а затем роль **Read-only**.
4. Нажмите **OK** для сохранения изменений.

Поскольку виртуальная машина может впоследствии перенестись на другой хост, мы рекомендуем включить мониторинг Dynatrace (добавить права доступа только для чтения) на всех хостах, на которые может перенестись ваша виртуальная машина.

Назначить пользователям права доступа только для чтения к пулу ресурсов

Для назначения пользователям прав доступа только для чтения к пулу ресурсов

1. В vCenter перейдите в представление пула ресурсов.
2. Выберите **Permissions**, затем нажмите знак `+`.
3. Выберите имя пользователя, а затем роль **Read-only**.
4. Выберите **Propagate to children**.
5. Нажмите **OK** для сохранения изменений.

После назначения пользователям прав доступа только для чтения к вашим хостам VMware и пулу ресурсов, хосты и виртуальные машины, которым вы предоставили разрешения, будут видны в Dynatrace.

## Исключение отдельной виртуальной машины из мониторинга

Чтобы исключить отдельную виртуальную машину из мониторинга Dynatrace, например виртуальную машину, унаследовавшую права только для чтения от родительского объекта, необходимо удалить права доступа пользователя на чтение для соответствующей виртуальной машины.

Для удаления прав доступа пользователя на чтение для виртуальной машины

1. В vCenter перейдите в представление виртуальной машины.
2. Выберите **Permissions**, затем нажмите знак `+`.
3. Выберите имя пользователя, а затем выберите **No access** в качестве роли.
4. Нажмите **OK** для сохранения изменений.

## Добавление отдельной виртуальной машины в мониторинг

Чтобы добавить отдельную виртуальную машину в мониторинг Dynatrace, необходимо назначить пользователям права доступа только для чтения к соответствующей виртуальной машине.

Для назначения пользователям прав доступа только для чтения к виртуальной машине

1. В vCenter перейдите в представление виртуальной машины.
2. Выберите **Permissions**, затем нажмите знак `+`.
3. Выберите имя пользователя, а затем роль **Read-only**.
4. Нажмите **OK** для сохранения изменений.
