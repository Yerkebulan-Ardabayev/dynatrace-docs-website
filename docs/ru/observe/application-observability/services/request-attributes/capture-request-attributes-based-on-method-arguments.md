Dynatrace позволяет создавать атрибуты запроса на основе аргументов метода.

## Создание атрибутов запроса

Чтобы создать атрибут запроса на основе аргумента метода

1. Перейдите в **Settings** > **Server-side service monitoring** > **Request attributes**.
2. Выберите **Define a new request attribute**.
3. Укажите уникальное **Request attribute name**. Вы можете переименовать атрибут в любой момент в будущем.
4. Выберите **Add new data source**.
5. Необязательно: определите область действия атрибута запроса.
6. В списке **Request attribute source** выберите **Java method parameter(s)**, **.NET method parameter(s)** или **PHP method parameter(s)**.
7. Нажмите кнопку **Select method sources**, чтобы открыть мастер классов. Здесь вы можете выбрать класс метода, на основе аргумента которого хотите задать атрибут.
8. Выберите группу процессов, содержащую интересующие вас классы или интерфейсы, и нажмите **Continue**.
9. Найдите класс, содержащий интересующий вас метод. Начните вводить имя класса и нажмите кнопку **Search**. Заполнение списка может занять несколько секунд.
10. Выберите класс из отображенного списка. Если список не содержит нужного класса, уточните строку поиска.

    ![Атрибуты запроса .NET](https://dt-cdn.net/images/request-attributes-dotnet-1-1326-b3cbc95405.png)
11. Наконец, выберите один или несколько методов, параметры которых хотите захватить, а затем нажмите **Finish**.
12. Выбранные методы отображаются в правиле захвата аргументов метода (см. ниже). Для каждого метода в раскрывающемся списке под **Capture** выберите аргумент или возвращаемое значение метода, которое хотите захватить.

    ![Атрибуты запроса .NET](https://dt-cdn.net/images/request-attributes-dotnet-2-1325-6354f7c2ab.png)
13. Вы всегда можете расширить список или удалить методы позже. После сохранения перезапустите процессы, к которым применяется это правило.

### Примечания

* Помимо аргументов и возвращаемых значений любого метода, вы также можете захватывать количество вхождений. При захвате вхождений следует использовать только одно правило метода. Dynatrace будет подсчитывать количество вызовов этого метода в рамках одного запроса.
* Вы можете [захватывать не примитивные объекты и объекты, не являющиеся строками](#numerical). Однако это может привести к увеличению накладных расходов на производительность. Некоторые объекты могут непреднамеренно изменять свое состояние в зависимости от вызванного метода, поэтому используйте эту опцию с осторожностью.
* Базовые классы Java (например, `javax.` и `java.`) или классы .NET (например, `System.`) выдают предупреждение **Not instrumentable**. Эти классы нельзя использовать, поскольку их инструментация означала бы инструментацию значительной части вашей среды.

## Что дальше?

Как только ваши сервисы начнут вызывать соответствующие методы, вы увидите атрибут запроса на странице распределенных трассировок.

![Атрибуты запроса .NET](https://dt-cdn.net/images/request-attributes-net-1801-7073b46dd3.png)

Представление дерева на уровне кода также содержит эти методы. Это представление показывает, какое значение было при каждом конкретном вызове метода -- на случай, если метод вызывается несколько раз с разными значениями.

## Постобработка

В большинстве случаев захваченное значение будет содержать именно то, что вам нужно. Однако вам может не понадобиться все значение целиком или даже каждое значение. С помощью постобработки вы можете манипулировать захваченным значением.

Разверните параметр **Optionally restrict or process the captured parameter(s) further**, чтобы увидеть шаги обработки. Шаги выполняются в представленном порядке -- каждый шаг применяется к результату предыдущего шага.

Вам не нужно применять все шаги. Каждый шаг становится активным после того, как вы укажете значение для него или выберете соответствующий флажок.

![Параметры постобработки для определения правила атрибута запроса](https://dt-cdn.net/images/post5steps-1353-6e753964a1.png)

Шаг 1 позволяет извлечь часть из результирующей строки на основе символов-разделителей.

Шаг 2 может разделить захваченное значение на несколько значений на основе символа-разделителя.

Шаг 3 удаляет пробелы.

Шаг 4 позволяет отфильтровать захваченные значения, не соответствующие заданному критерию.

Шаг 5 позволяет извлечь часть из результирующей строки на основе [регулярного выражения](../../../../manage/tags-and-metadata/reference/regular-expressions-in-dynatrace.md "Learn how to use regular expressions in the context of Dynatrace.").

## Числовые значения и агрегации

Вы можете захотеть захватить аргумент метода, который вызывается несколько раз. Иногда вас интересует только конкретное значение. В других случаях вы можете захотеть только подсчитать количество выполнений или вычислить среднее значение захваченных значений. В других ситуациях интересующий аргумент может быть сложным объектом, и вас на самом деле интересует только один аспект этого объекта.

Для решения таких ситуаций вы можете использовать параметры **Data type** и **Aggregation on request** при определении или настройке атрибута запроса.

### Агрегации

Агрегация применяется не только в рамках одного источника данных (например, правила метода), но и между несколькими источниками данных. Порядок определяется:

* Порядком правил источников данных
* Порядком правил методов в рамках источника данных метода
* Порядком выполнения методов в вашем приложении (когда метод выполняется несколько раз)

#### Текстовые атрибуты

Для текстовых атрибутов вы можете выбрать между следующими параметрами:

* Первое вхождение
* Последнее вхождение
* Набор уникальных значений

Параметр **Set of distinct values** позволяет использовать все уникальные значения, найденные в запросе, для фильтрации. Например, если запрос захватывает атрибут запроса с именем `Product` и для этого атрибута захвачены два значения: `Book` и `Video`. Вы можете фильтровать по любому из этих значений, чтобы найти нужный запрос.

#### Числовые атрибуты

Для числовых значений вы можете использовать следующие агрегации:

* Минимум захваченных значений
* Максимум захваченных значений
* Среднее захваченных значений
* Сумма захваченных значений
* Подсчет вхождений
* Подсчет уникальных значений

Если вы выберете один из этих параметров, тип данных автоматически изменится на integer. Эти настройки полезны при подсчете чего-либо.

## Глубокий доступ к объектам

Интересующее вас значение может быть недоступно в виде простого аргумента, а является частью сложного объекта-аргумента или даже переменной-членом класса, который вы рассматриваете. Такие значения все равно можно получить. Вы можете получить доступ не только к значениям аргументов и возвращаемым значениям, но и к самому объекту. Всякий раз, когда элемент для захвата (будь то аргумент или объект) является сложным объектом, у вас есть возможность определить метод (цепочку методов), обеспечивающий глубокий доступ к объекту. В приведенном ниже примере используется метод `getBookingCode`. Вы даже можете выполнить цепочку, например, `getBookingCode().getCustomerCode()`.

![Атрибуты запроса](https://dt-cdn.net/images/deep-object-access-1157-bad06a137b.png)

Функция глубокого доступа к объектам вводит в ваше приложение новый код, который должен быть выполнен. Это может изменить состояние вашего приложения или повлиять на производительность. Используйте эту функцию с осторожностью.

### Ограничения

К глубокому доступу к объектам применяются следующие ограничения:

* Вы можете получить доступ только к одному полю за раз. Если вы обращаетесь к полю, оно должно быть в начале цепочки.
* Параметры методов не допускаются.
* Необходимо использовать допустимую нотацию Java, .NET или PHP.
