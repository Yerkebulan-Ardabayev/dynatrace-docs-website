Эта тема посвящена странице ![Services Classic](https://dt-cdn.net/images/services-classic-f58502bd22.svg "Services Classic") **Services Classic** и представляет обновленный дизайн классической страницы. Ознакомьтесь с приложением [![Services](https://dt-cdn.net/hub/logos/services.png "Services") **Services**](../services/services-app.md "Контролируйте состояние, производительность и ресурсы сервисов с помощью приложения Services.") для получения новейшего опыта.

Все сервисы, обнаруженные Dynatrace в вашей среде, отображаются на странице ![Services Classic](https://dt-cdn.net/images/services-classic-f58502bd22.svg "Services Classic") **Services Classic**. Вы можете проанализировать каждый сервис и получить информацию вплоть до уровня кода.

![Список сервисов](https://dt-cdn.net/images/services-page-1504-133c9f1b72.png)

Как перейти:

1. Перейдите в ![Services Classic](https://dt-cdn.net/images/services-classic-f58502bd22.svg "Services Classic") **Services Classic**.
2. Выберите имя сервиса в списке, чтобы перейти на страницу обзора этого сервиса.

![Обзор сервиса с новым дизайном Unified Analysis](https://dt-cdn.net/images/service-overview-new-design-1768-fe1ab99c85.png)

На каждой странице **Сервиса** представлена наиболее важная информация о данном сервисе. Все ключевые метрики сервиса отображаются на одной странице, которая разделена на несколько логических разделов. Другие панели страницы обзора сервиса показывают производительность сервиса и служат точками входа для более глубокого анализа.

## Панель уведомлений

Панель уведомлений сервиса дает вам краткий обзор состояния сервиса. Выберите элемент уведомления для отображения дополнительной информации.

### Свойства и теги

Выберите **Properties and tags** на панели уведомлений для отображения панели **Properties and tags**, которая содержит метаданные о выбранном сервисе:

* **Tags** -- список тегов, применённых к сервису.
  Выберите **Add Tag** для добавления тега к метаданным сервиса.
* **Properties** -- список различных свойств сервиса, таких как имя приложения, тип сервиса, технологии и зоны управления.

### Проблемы

* На панели уведомлений **Problems** отображает активные и закрытые проблемы, связанные с выбранным сервисом.
* Выберите **Problems** на панели уведомлений для отображения панели **Problems**, которая содержит список проблем.

  + Выберите проблему для отображения подробностей.
  + Выберите **Go to problems** для перехода на страницу [Problems](../../../dynatrace-intelligence.md "Ознакомьтесь с возможностями Dynatrace Intelligence."), отфильтрованную по выбранному сервису.

### SLO

* На панели уведомлений **SLOs** отображает текущее количество [SLO](../../../deliver/service-level-objectives-classic.md "Мониторинг и оповещение о целевых уровнях обслуживания с помощью Dynatrace в Service-Level Objectives Classic."), связанных с выбранным сервисом.
* Выберите **SLOs** на панели уведомлений для отображения панели **Service-level objectives**, которая содержит список SLO, прямо или косвенно связанных с сервисом.

#### Напрямую связанные SLO

* SLO напрямую связан с сервисом, когда [селектор сущности](../../../dynatrace-api/environment-api/entity-v2/entity-selector.md "Настройка селектора сущностей для конечных точек Environment API.") SLO соответствует следующим критериям:

  + Тип сущности установлен в `"SERVICE"`.
  + Идентификатор сущности установлен в идентификатор сервиса.
* Чтобы видеть только SLO, напрямую связанные с сервисом, убедитесь, что включена опция **Show only directly connected SLOs**.

#### Косвенно связанные SLO

* SLO не связан с сервисом напрямую, когда в [селекторе сущности](../../../dynatrace-api/environment-api/entity-v2/entity-selector.md "Настройка селектора сущностей для конечных точек Environment API.") SLO не указан идентификатор сущности.

  Пример: когда указаны общие значения, такие как `type("SERVICE"),tag("slo")`, запрос возвращает все SLO для всех сервисов, включая текущий сервис.
* Чтобы видеть SLO, не связанные с сервисом напрямую, отключите опцию **Show only directly connected SLOs**.

#### Параметры

* Разверните **Details** для просмотра графика соответствующих метрик SLO.
* В **Actions** выберите

  + **View in Data Explorer** для [просмотра метрик SLO в Data Explorer](../../../deliver/service-level-objectives-classic/configure-and-monitor-slo.md#explorer "Создание, настройка и мониторинг целевых уровней обслуживания с помощью Dynatrace.").
  + **Pin to Dashboard** для [закрепления SLO на панели мониторинга](../../../deliver/service-level-objectives-classic/configure-and-monitor-slo.md#dash "Создание, настройка и мониторинг целевых уровней обслуживания с помощью Dynatrace.").
  + **SLO definition** для редактирования SLO в **Service-level objective definitions**.
  + **Clone** для [клонирования SLO](../../../deliver/service-level-objectives-classic/configure-and-monitor-slo.md#clone "Создание, настройка и мониторинг целевых уровней обслуживания с помощью Dynatrace.").
  + **Create alert** для [создания оповещения для SLO](../../../deliver/service-level-objectives-classic/configure-and-monitor-slo.md#alerts "Создание, настройка и мониторинг целевых уровней обслуживания с помощью Dynatrace.").

#### Нет SLO

Если SLO не найдены, вы можете

* Выбрать другой временной интервал в правом верхнем углу.

  ![Выбор временного интервала: панель меню](https://dt-cdn.net/images/timeframe-selector-menu-bar-264-8193110c8c.png)
* Выбрать **Add SLO** для создания SLO в [мастере SLO](../../../deliver/service-level-objectives-classic/configure-and-monitor-slo.md#wizard "Создание, настройка и мониторинг целевых уровней обслуживания с помощью Dynatrace.").

### Владельцы

Выберите **Owners** на панели уведомлений для отображения панели **Ownership**, которая содержит список [владельцев](../../../deliver/ownership.md "Сопоставьте владение командами с отслеживаемыми сущностями для улучшения сотрудничества, распределения задач, реагирования на инциденты и уязвимости, а также управления уровнем обслуживания.") выбранного сервиса.

* Выберите  для получения дополнительной информации о текущем владельце.
* Чтобы добавить владельца, выберите **Add a new Ownership tag**.

## Производительность

### Обзор сервиса

Вы можете настроить карточку **Service overview** для фокусировки на различных метриках производительности сервиса.

* В **Service overview** вы можете

  + Просмотреть сервис в Smartscape, выбрав **Smartscape view** .
  + Проанализировать **Response time hotspots**, **Details of failures** и горячие точки методов.
  + Сравнить показатели производительности запросов сервиса, такие как время ответа, ошибки, CPU и нагрузка, на основе различных временных интервалов.
* Для каждой метрики вы можете выбрать **More** (**...**) и

  + Проанализировать метрику в Data Explorer.
  + Создать событие метрики.
  + Закрепить метрику на классической панели мониторинга. Подробнее см. [Закрепление плиток на панели мониторинга](../../../analyze-explore-automate/dashboards-classic/charts-and-tiles/pin-tiles-to-your-dashboard.md "Узнайте, как закреплять плитки на панелях мониторинга.").

### Конечные точки

Unified service

Карточка **Endpoints** предлагает обзор отслеживаемых конечных точек сервиса.

* Для анализа метрик конечной точки в столбце **Details** выберите .
* Для анализа распределенных трассировок конечной точки в столбце **Actions** выберите **More** (**...**) > **View distributed traces**.

Доступность данных зависит от метрик конечных точек, которые потребляют DDU. Чтобы начать мониторинг конечных точек, см. [Управление мониторингом конечных точек](../services/service-detection/service-detection-v1/service-types/unified-service.md#endpoint-monitoring "Определяйте сервисы на основе сигналов наблюдаемости, поступающих через API приема трассировок.").

### Метрики сервисной сетки

Unified service

Если в вашем приложении обнаружена сервисная сетка, для соответствующего сервиса отображаются метрики сервисной сетки. [Логи и метрики ISTIO](../../infrastructure-observability/container-platform-monitoring/kubernetes-monitoring/monitor-istio-metrics.md "Прием метрик Istio и обнаружение топологии") обогащаются идентификатором сервисной сетки, помещая данные в контекст. Обратите внимание, что метрики сервисной сетки создаются из принятых трассировок сервисной сетки.

### Ключевые запросы

Карточка **Key requests** предлагает обзор ключевых запросов, найденных для сервиса в выбранном временном интервале. Вы можете быстро получить доступ к анализу других запросов, связанных с сервисом, таких как:

* [**View all requests**](../services-classic.md#charts "Узнайте о классическом мониторинге сервисов Dynatrace")
* [**Top web requests**](../multidimensional-analysis/top-web-requests.md "Узнайте, как анализировать все веб-запросы по всем вашим сервисам с помощью Dynatrace.")
* [**View outliers**](response-time-distribution-and-outlier-analysis.md#outlier-analysis "Получите представление о распределении времени ответа по всем запросам, включая аномально высокие или аномально низкие.")

### Топология

В карточке **Topology** вы можете узнать

* Какие сервисы вызывают данный сервис и какие вызываются им.
  Выберите **Related services** для понимания взаимосвязей сервиса. Разверните **Details** для просмотра графика соответствующих метрик сервиса. Вы можете быстро получить доступ к дополнительным вариантам анализа, таким как [**View backtrace**](service-backtrace.md "Отследите последовательность вызовов сервиса до действия в браузере, которое инициировало цепочку вызовов.") и [**Service flow**](service-flow.md "Узнайте, как Dynatrace помогает отслеживать последовательность вызовов сервисов, инициированных каждым запросом сервиса в вашей среде.").
* На каких процессах и хостах работает сервис.
  Разверните **Details** для просмотра графика соответствующих метрик процесса. Выберите имя процесса для более глубокого анализа.
* Какие HTTP-мониторы и (мобильные) приложения вызывают данный сервис.
  Выберите имя вызывающей сущности для более глубокого анализа.

![Топология - Обзор сервиса](https://dt-cdn.net/images/topology-services-2886-fc9dda7f82.png)

### Распределенные трассировки

Карточка **Distributed traces** предоставляет обзор наиболее свежих трассировок в зависимости от выбранного временного интервала. Выберите **Full search** для прямого доступа к [обзору распределенных трассировок для сервиса](../distributed-traces/analysis/get-started.md "Начните работу с анализом распределенных трассировок в Dynatrace.").

### События

Содержит список [событий](../../../dynatrace-intelligence/root-cause-analysis/event-analysis-and-correlation.md "Получите представление о разделе событий на каждой странице обзора хоста, процесса и сервиса."), влияющих на сервис в текущем временном интервале.

### Связанные логи

Содержит список [логов](../../../analyze-explore-automate/logs.md "Log Management and Analytics обеспечивает унифицированный подход к контролю и изучению данных логов в Dynatrace."), связанных с сервисом в текущем временном интервале.

* Для анализа всех логов связанного сервиса выберите **Go to logs** .
* Для анализа конкретного лога разверните **Details**. Если для строки лога найдена трассировка или пользовательская сессия, вы можете получить к ней прямой доступ из этого представления.
