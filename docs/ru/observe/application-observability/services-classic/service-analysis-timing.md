# Тайминги анализа сервисов

* Classic
* Reference
* 5-min read
* Updated on Jul 24, 2023

Анализ сервисов работает с множеством различных таймингов, описывающих поведение сервиса. В таблице ниже представлен обзор таких таймингов. Тайминги варьируются в зависимости от типа анализа:

* DT: [Распределённые трассировки](../distributed-traces/use-cases/segment-request.md#pp-code-level-tab "Повысьте производительность распределённой системы, сегментируя запросы с медленным временем отклика через Service flow и анализируя их распределённые трассировки.").
* RT: [Время отклика](service-response-time-hotspots.md "Определите действия, потребляющие наибольшее время отклика для каждого сервиса.").
* SF: [Service flow](service-flow.md "Узнайте, как Dynatrace помогает отслеживать последовательность вызовов сервисов, инициируемых каждым запросом к сервису в вашей среде.").
* MH: [Горячие точки методов](service-response-time-hotspots.md "Определите действия, потребляющие наибольшее время отклика для каждого сервиса.").

Независимо от того, какой анализ вы выполняете, не гарантируется, что вы увидите все перечисленные здесь тайминги. Фактически отображаемые тайминги полностью зависят от того, как запущены ваши сервисы. Например, если распределённая трассировка полностью выполняется на одном хосте без какого-либо сетевого взаимодействия, сетевые тайминги отображаться не будут.
