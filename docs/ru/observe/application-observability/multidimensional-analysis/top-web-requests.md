# Top web requests

# Top web requests

* Classic
* How-to guide
* 4-min read
* Updated on Sep 13, 2022

Гибкое представление **Top web requests** позволяет анализировать наиболее популярные веб-запросы ко всем вашим сервисам и находить запросы, поступающие с определённых URL-адресов или направленные к ним. Это представление легко настраивается и служит удобной отправной точкой для углублённого анализа ваших сервисов.

Чтобы перейти на страницу анализа **Top web requests**

1. Перейдите в **Multidimensional Analysis**.
2. Нажмите плитку **Top web requests**.

![Top web requests page](https://dt-cdn.net/images/top-web-requests-page-3518-5c4cc7fb02.png)

## Настройка представления

На странице **Top web requests** перечислены все веб-запросы, произошедшие за выбранный период времени и в рамках выбранной зоны управления. В разделе **Configure view** доступны несколько параметров фильтрации. Представление обновляется автоматически по мере изменения параметров.

После настройки представления его можно сохранить для быстрого доступа в будущем. Нажмите **Save view** и укажите название.

Также можно сохранить конфигурацию как вычисляемую метрику сервиса и использовать её как любую другую метрику в Dynatrace (например, для [оповещения](/docs/dynatrace-intelligence/anomaly-detection/metric-events "Узнайте о метрических событиях в Dynatrace")).

Данные таблицы можно экспортировать в файл формата CSV (значения, разделённые запятыми).

1. В правом нижнем углу страницы выберите **Show export menu** ![More](https://dt-cdn.net/images/more-icon-01c8b008ca.svg "More").

   ![Show export menu](https://dt-cdn.net/images/show-export-menu-107-2a8a76c9a2.png)
2. Выберите **Export visible data** или **Export table data**.

## Просмотр

* График на странице **Top web requests** отображает 15 наиболее популярных измерений (все остальные измерения агрегируются в одно), а таблица под ним содержит до 85 дополнительных измерений, что в общей сложности даёт 100 отображаемых измерений. Представление мгновенно адаптируется к изменениям, вносимым в панели **Configure view**.
* График использует [данные трассировок и запросов](/docs/observe/application-observability/multidimensional-analysis#data-source "Настройте представление многомерного анализа и сохраните его как вычисляемую метрику."), которые имеют разные периоды хранения. Для временных диапазонов, содержащих данные старше 10 дней, можно включить переключатель **Show data retention**, чтобы непосредственно на графике понять, какие данные доступны за какой период.
* В столбце **Actions** таблицы можно выбрать:

  + **Filter** ![Filter](https://dt-cdn.net/images/filter-icon-41ddd02d66.svg "Filter") для фильтрации представления по указанному измерению.
  + **More** (**...**) для доступа к дополнительным параметрам анализа из [меню **Analyze**](/docs/observe/application-observability/services-classic/context-specific-drill-down#analyze-menu "Узнайте об удобной навигации и фильтрации для анализа сервисов.").

## Примеры использования

Ниже приведены некоторые сценарии использования представления **Top web requests**.

### Поиск конкретного URL

С помощью представления **Top web requests** можно сосредоточить анализ на запросах, поступающих с определённого URL или направленных к нему.

1. Укажите `{URL:Host}{Relative-URL}` в поле **Split by dimension**, чтобы разделить запросы по URL-адресам.
2. Теперь отфильтруйте запросы по конкретному URL. В поле **Filter requests** выберите тип фильтра `Web request URL` и укажите шаблон URL. Обратите внимание, что этот фильтр использует оператор **CONTAINS**.

### Поиск запросов с кодами ошибок в ответе

Можно легко сузить список запросов до тех, которые вернули код ошибки (то есть любой код в диапазоне **4xx**). После настройки интересующих вас измерений поместите курсор в поле **Filter requests** и выберите тип фильтра **HTTP response code** (можно начать вводить текст, чтобы сузить предложения). Затем выберите критерий **4xx** и дождитесь завершения обработки данных.

### Поиск HTTP-запросов POST

Существует два варианта просмотра запросов POST (или любого другого HTTP-метода): (1) задать измерение HTTP-метода или (2) отфильтровать запросы по HTTP-методу.

#### Измерение

В этом случае все запросы будут сгруппированы по HTTP-методу и по любому другому указанному измерению. Просто укажите `{HTTP-Method}` в поле **Split by dimension**.

#### Фильтр

В этом случае все запросы, кроме POST, будут отфильтрованы, что позволит сосредоточить анализ. Просто укажите `HTTP method` и выберите значение `POST` в поле **Filter requests**.
