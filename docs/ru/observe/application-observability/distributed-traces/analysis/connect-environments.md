# Настройка межсредового трассирования

# Настройка межсредового трассирования

* Classic
* How-to guide
* 6 мин. чтения
* Обновлено 04 июня 2024 г.

В сценариях с несколькими средами пулы данных разделены. Если запрос затрагивает сервисы, мониторинг которых ведётся в разных средах Dynatrace, их трассировки не коррелируются автоматически.

Для корреляции и отслеживания таких трассировок вы можете настроить соединение между средами Dynatrace и включить переключатели заголовков ответа и координированной выборки. После этого поток сервисов отображает связь между подключёнными средами, распределённые трассировки содержат ссылки на подключённую удалённую среду, и вы получаете доступ к возможностям анализа Davis AI.

## Конфигурация

### Перед началом работы

Ограничения

* Межсредовое трассирование ограничено трассировками запросов, которые могут передавать информацию о заголовках ответа и контексте трассировки из принимающей среды, например HTTP или синхронные запросы.
* IBM z/OS (CICS, IMS и Java) и AWS Lambda не поддерживают межсредовое трассирование.

Предварительные условия

* Убедитесь, что между кластерами может быть установлено прямое сетевое соединение по HTTP/HTTPS.

Эквиваленты API

Приведённые ниже процедуры используют веб-интерфейс Dynatrace. Для выполнения аналогичных задач через API см.:

* [API токенов доступа](../../../../dynatrace-api/environment-api/tokens-v2/api-tokens.md "Управление токенами аутентификации Dynatrace API.") — для создания токена в удалённой среде
* [API удалённых сред](../../../../dynatrace-api/configuration-api/remote-environments.md "Управление конфигурациями удалённых сред Dynatrace через API конфигурации Dynatrace.") — для создания ссылки на удалённую среду из локальной среды

### Шаги

1. Создание токена доступа

С помощью этой процедуры вы получите токен доступа из удалённой среды, который потребуется на остальных шагах.

Чтобы создать токен доступа в удалённой среде Dynatrace

1. Войдите в удалённую среду.

   * Это среда, из которой вы будете получать данные.
   * Если вы не можете войти в удалённую среду, кто-либо с доступом к ней может выполнить эту процедуру за вас.
2. Перейдите в **Access Tokens**.
3. Выберите **Generate new token**.
4. Введите имя токена.
5. Найдите и выберите следующие области действия:

   * **Look up a single trace** (`traces.lookup`)
   * **Fetch data from a remote environment** (`RestRequestForwarding`)
6. Выберите **Generate token**.
   Это создаст токен, который даёт вашей локальной среде право проверять наличие трассировки в удалённой среде и извлекать данные из неё.
7. Выберите **Copy**, затем вставьте токен в безопасное место.
   Это длинная строка, которую вам нужно будет скопировать и вставить обратно в Dynatrace позже.

2. Добавление удалённой среды

Чтобы добавить удалённую среду Dynatrace в список доступных удалённых сред

1. Войдите в свою локальную среду Dynatrace.
2. Перейдите в **Settings**.
3. Выберите **Integration** > **Remote environments**.
4. Выберите **Connect environment**.
5. Определите удалённую среду, из которой ваша локальная среда будет получать данные, и затем выберите **Save changes**.

   * **Name** — имя, под которым удалённая среда будет отображаться в вашей текущей среде Dynatrace. Это произвольный текст. Он не влияет на удалённую среду.
   * **Remote environment URI**

     + Для Dynatrace SaaS формат должен быть следующим:

       `https://<ENVIRONMENTID>.live.dynatrace.com/`

       Замените `<ENVIRONMENTID>` на ваш реальный идентификатор среды.
     + Для Dynatrace Managed допускается любой URI.
     + Чтобы подключить среду Dynatrace (SaaS-развёртывание) к развёртыванию Dynatrace Managed через URI за пределами домена `dynatrace-managed.com`, обратитесь к специалисту Dynatrace через чат в вашей среде Dynatrace.
   * **Network scope**

     + `External`: удалённая среда находится в другой сети. Глобально настроенные параметры прокси используются при их наличии. Это область действия по умолчанию.
     + `Internal`: удалённая среда находится в той же сети. Глобально настроенные параметры прокси не используются.
     + `Cluster`: удалённая среда находится в том же кластере. Запрос отправляется на `localhost`.

     Dynatrace SaaS может использовать только область действия `External`.
   * **Token** — токен, сгенерированный на предыдущем шаге. Он должен включать область действия **Look up a single trace** (`traces.lookup`).
   * **Test connection** — проверяет соединение из вашей текущей среды с удалённой средой.

     Убедитесь, что вы получили сообщение `connection successfully established`, прежде чем продолжить.

3. Включение функций

Включите следующие функции в каждой среде из группы, которую вы хотите подключить:

* **Координированная выборка**

  Вы можете повысить согласованность межсредового трассирования с помощью координированной выборки. При включении переключателя выборка координируется между средами с помощью идентификатора трассировки W3C.

  1. Перейдите в **Settings** > **Preferences** > **OneAgent features**.
  2. Настройте распространение контекста трассировки W3C в соответствии с вашими предпочтениями.
     Вы можете отправлять HTTP-заголовки контекста трассировки W3C или заголовки gRPC.
  3. Найдите и включите **Cross-environment tracing - Coordinated sampling**.
* **Заголовки ответа**

  Вы можете включить идентификацию и отслеживание межсредовых вызовов, включая идентификатор среды и идентификатор трассировки в заголовках HTTP-ответов. Если вызывающая среда, получающая ответ, имеет данную среду в списке доверенных удалённых сред, назначение вызова связывается.

  1. Перейдите в **Settings** и выберите **Preferences** > **OneAgent features**.
  2. Найдите и включите **Cross-environment tracing - Environment and transaction IDs in HTTP response headers**.

Теперь, когда вы подключили свои среды, вы можете анализировать трассировки в перекрёстно связанных удалённых средах.

[Сообщение `Verification failed, please check your settings: Constraints violated.` при добавлении удалённой среды](https://dt-url.net/t903mr6)

## Анализ перекрёстно связанных удалённых сред

После настройки соединения между средами доступны следующие возможности анализа.

### Обзор сервиса

Dynatrace обнаруживает запросы к подключённым средам Dynatrace и использует доступную информацию для создания ассоциированного веб-сервиса или сервиса веб-запросов **Remote environment**.

![Обзор сервиса удалённой среды](https://dt-cdn.net/images/cross-environment-4-1608-af340752cb.jpeg)

* Исходящие вызовы к подключённым средам отображаются в **Smartscape** и **Service flow**.

  ![Сервис удалённой среды в топологии Smartscape](https://dt-cdn.net/images/cross-environment-1-1202-a48e0448d7.jpeg)
* Все сервисы, мониторинг которых ведётся удалённой средой и которые участвуют в запросе, перечислены на странице **Remote environments**. Чтобы открыть её, выберите **Open environment** на странице сервиса **Remote environment**.

### Поток сервисов

**Service flow** распознаёт вызовы удалённых сред. Поскольку мониторинг запросов выполняется другой средой Dynatrace, при выборе вызова удалённой среды доступна специальная возможность детализации. На боковой панели выберите **Open environment** для перехода на страницу **Remote environments**.

![Удалённая среда в анализе потока сервисов](https://dt-cdn.net/images/cross-environment-2-1531-85c4a673fc.jpeg)

### Распределённые трассировки

* Когда запрос направляется в подключённую удалённую среду, Dynatrace отображает имя удалённой среды в дереве вызовов.

  Чтобы отследить трассировку в удалённой среде

  1. Выберите запрос удалённой среды.
  2. На вкладке **Summary** выберите **Open environment**.
* Если соединение между средами настроено правильно, Dynatrace автоматически отображает вызов **Aggregated requests**. Он агрегирует все данные обнаруженных запросов в каждой подключённой удалённой среде.

  Если вы настроили соединение с удалённой средой, но не видите запросов в ней, вы можете запустить ручной поиск трассировки в настроенных средах. В правом верхнем углу страницы трассировки выберите **Find in remote environments**.

### Davis AI

После того как Dynatrace создаст сервис удалённой среды, проблемы и события коррелируются с затронутым сервисом удалённой среды.

Вы можете использовать возможности [Davis AI](../../../../dynatrace-intelligence.md "Ознакомьтесь с возможностями Dynatrace Intelligence.") для понимания того, какие запросы завершаются с ошибкой, и обнаружения первопричины проблем в удалённой среде.

![Анализ проблем Davis AI для сервиса удалённой среды](https://dt-cdn.net/images/cross-environment-3-1600-8b1ae112a1.jpeg)
