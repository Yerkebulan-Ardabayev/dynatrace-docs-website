Вы можете использовать ![Live Debugger](https://dt-cdn.net/images/live-debugger-256-b934a2bad5.png "Live Debugger") **Live Debugger** непосредственно из вашей интегрированной среды разработки (IDE). В настоящее время Dynatrace поддерживает Visual Studio Code и пакет JetBrains.

## Требования

### Visual Studio Code

* Минимальная поддерживаемая версия: 1.85.0+

### IDE JetBrains

* Минимальная поддерживаемая версия: 2024.1
* Рекомендуемая версия: 2025.1+ для наилучшего опыта

JetBrains часто обновляет версии своей платформы. Мы рекомендуем обновлять IDE до последней версии, чтобы обеспечить совместимость с плагином Live Debugger.

## Расширение Visual Studio Code

Следуйте приведенным ниже шагам для настройки и использования расширения **Observability for Developers by Dynatrace** в Visual Studio Code.

### 1. Настройка расширения

1. В Visual Studio Code установите расширение [Observability for Developers by Dynatrace](https://dt-url.net/6v03quu).
2. На боковой панели выберите (**Dynatrace Debugger**). Панель конфигурации **Dynatrace Debugger** откроется — обычно слева в вашей IDE.
3. Выберите **Log in** для входа в Dynatrace.
4. В разделе **Environment** выберите среду, которую вы хотите отлаживать. Список сред откроется вверху, отображая все среды Dynatrace, в которых у вас есть права на Live Debugging. Выберите среду из списка.
5. Выберите **Customize session filters** для настройки сеанса отладки. Отображаются только экземпляры с включенным Live Debugging. Выберите фильтры для экземпляров, которые вы хотите отлаживать, и нажмите **Set**.

### 2. Добавление точки останова Live Debugging

Вам необходимо добавить [точку останова](breakpoints.md "Learn how to add breakpoints, view available statuses, set breakpoint levels, and more.") Live Debugging в Visual Studio Code.

Щелкните правой кнопкой мыши на номере строки кода и выберите **Add Live Debugging Breakpoint**. Точка останова применяется к экземплярам, соответствующим фильтрам и среде, настроенным на предыдущих шагах.

### 3. Просмотр данных

Когда точки останова срабатывают, данные поступают в панель **Dynatrace snapshots** в Visual Studio Code. Выберите **Open the Snapshot Panel** в панели конфигурации **Dynatrace Debugger**, а затем выберите снимок для просмотра локальных переменных, стека вызовов, информации о процессе и данных трассировки.

### 4. Обновление конфигурации сеанса отладки (необязательно)

При необходимости вы можете получить доступ ко всем параметрам конфигурации сеанса через боковую панель Visual Studio Code. Выберите (**Dynatrace Debugger**), чтобы открыть панель конфигурации **Dynatrace Debugger**.

### 5. Управление точками останова (необязательно)

* Чтобы добавить точку останова, щелкните правой кнопкой мыши на номере строки кода и выберите **Add Live Debugging Breakpoint**.
* Чтобы открыть контекстное меню точки останова, щелкните правой кнопкой мыши на значке точки останова. Отобразятся следующие параметры:

  + **Breakpoint Status**
  + **Disable Breakpoint**
  + **Edit Breakpoint**
  + **Remove Breakpoint**
* Чтобы просмотреть список точек останова, выберите (**Dynatrace Debugger**) на боковой панели вашей IDE и раскройте раздел **Live debugging breakpoints**. Выберите точку останова в этом списке для просмотра её статуса и редактора.

## Плагин JetBrains

Выполните приведенные ниже шаги для настройки и использования плагина **Observability for Developers by Dynatrace** в IDE JetBrains.

### 1. Настройка плагина

1. Установите плагин [Observability for Developers by Dynatrace](https://dt-url.net/pf23qui) и перезапустите IDE JetBrains.
2. На панели инструментов (обычно расположена слева) выберите (**Observability for Developers**). Панель **Observability for Developers** откроется — обычно в нижней части IDE JetBrains.
3. Выберите **Log in** для входа в Dynatrace.
4. Раскройте выпадающий список в заголовке панели и выберите среду, которую вы хотите отлаживать. Список сред отображает все среды Dynatrace, в которых у вас есть права на Live Debugging.
5. Выберите **Filter Applications** в заголовке панели для настройки сеанса отладки.
   Отображаются только экземпляры с включенным Live Debugging. Выберите фильтры для экземпляров, которые вы хотите отлаживать, и нажмите **Set**.

### 2. Добавление точки останова Live Debugging

Вам необходимо добавить [точку останова](breakpoints.md "Learn how to add breakpoints, view available statuses, set breakpoint levels, and more.") Live Debugging в вашей IDE JetBrains.

Выберите нужный номер строки кода, затем выберите **Live Debugging Breakpoint**. Точка останова применяется к экземплярам, соответствующим фильтрам и среде, настроенным на предыдущих шагах.

### 3. Просмотр данных

Когда точки останова срабатывают, данные поступают в панель **Observability for Developers**. Выберите снимок для просмотра локальных переменных, стека вызовов, информации о процессе и данных трассировки.

### 4. Обновление конфигурации сеанса отладки (необязательно)

При необходимости вы можете получить доступ ко всем параметрам конфигурации сеанса через заголовок панели **Observability for Developers** в вашей IDE JetBrains. Откройте панель и наведите курсор на заголовок, чтобы увидеть параметры конфигурации.

### 5. Управление точками останова (необязательно)

* Чтобы добавить точку останова, выберите нужный номер строки кода, затем выберите **Live Debugging Breakpoint** из списка.
* Вы можете взаимодействовать с точками останова непосредственно из кода. Выберите значок точки останова, чтобы открыть редактор точки останова.
* Выберите **More** в редакторе точки останова для доступа к расширенным параметрам редактирования и просмотра всех точек останова в этом сеансе.
