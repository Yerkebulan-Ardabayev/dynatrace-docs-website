**Неблокирующие точки останова** Dynatrace позволяют OneAgent, развёрнутому в вашем приложении, получать данные из любого места в вашем коде, давая возможность наблюдать текущее состояние приложения и быстро находить ошибки в продакшене без остановки приложения.

## Добавление точек останова

Чтобы добавить точку останова в ![Live Debugger](https://dt-cdn.net/images/live-debugger-256-b934a2bad5.png "Live Debugger") **Live Debugger**, нажмите на область слева от номера нужной строки кода.

## Статус точки останова

После установки неблокирующей точки останова ей присваивается статус (**Active**, **Pending**, **Disabled**, **Error** или **Warning**). Для просмотра дополнительной информации о статусе нажмите правой кнопкой мыши на точку останова и выберите **Status**.

Для получения дополнительной информации вы можете выбрать индикатор статуса точки останова или прочитать подробнее на этой странице.

Разверните любой из следующих разделов для получения сводки о статусе точки останова.

Active

Статус `Active` возникает, когда одно или несколько ваших приложений применили точку останова и ошибок не обнаружено.
В большинстве случаев после перехода точки останова в активное состояние вы увидите собранные снимки при следующем выполнении строки кода.
Если вы не видите поступающих снимков, это может быть связано с одной из следующих причин:

* **Неправильное приложение**
  Вы не вызываете нужную строку кода в нужном экземпляре приложения.
* **Долго выполняющаяся функция**
  Вы установили точку останова на долго выполняющейся функции. В этой среде выполнения точки останова применяются только для вызовов функций, выполненных после создания точки останова.

Pending

Статус `Pending` указывает на то, что точка останова не была применена ни к одному из ваших приложений. Это может означать одно из следующего:

* **Нет экземпляров приложения**
  Точку останова не удалось применить, поскольку ни один экземпляр приложения не соответствует текущему фильтру. Это может означать одно из следующего:

  + OneAgent не установлен.
  + OneAgent установлен, но приложение не может подключиться к сервису Dynatrace.
  + Приложение может подключиться к сервису Dynatrace, но исключено текущим определением фильтра.
  + Приложение в данный момент не запущено и будет запущено в будущем.
* **Неправильный исходный файл**
  Исходный файл, использованный для установки точки останова, не загружен ни в одном из приложений в текущем фильтре.
* **Нет кода**
  Вы установили точку останова на строке, с которой не связан исполняемый код.
* **(JVM) Нет отладочной информации**
  Вы скомпилировали классы без отладочной информации. Нажмите здесь для получения дополнительной информации.
* **(Node) Нет Source Maps**
  Вы используете транспилированное приложение без включения source maps. Попробуйте использовать минимальный уровень транспиляции или убедитесь, что source maps добавлены в развёрнутое приложение. Подробнее см. в разделе исходного кода.
* **(Node) Код в зависимости**
  Вы отлаживаете пакет, развёрнутый как зависимость. Это требует соответствующей настройки репозитория исходного кода.
* **(Node) Другая структура файлов**
  Пути к файлам изменены между репозиторием исходного кода и развёртыванием. Это требует соответствующей настройки репозитория исходного кода.

Disabled

Статус `Disabled` возникает, когда точка останова была отключена из-за ограничений. К ним относятся ограничения, установленные пользователем для конкретной точки останова (например, ограничение по времени, ограничение по количеству срабатываний).

* Для получения дополнительной информации о причине отключения точки останова нажмите правой кнопкой мыши на точку останова и выберите **Status**.
* Для повторного включения точки останова (путём сброса счётчика ограничений) нажмите правой кнопкой мыши на точку останова и выберите **Enable**.

Error

Статус `Error` возникает, когда одно или несколько ваших приложений сообщили об ошибке при обработке, применении или выполнении точки останова.

Сообщения `Error` документируются в ![Live Debugger](https://dt-cdn.net/images/live-debugger-256-b934a2bad5.png "Live Debugger") **Live Debugger** или IDE, примеры включают:

* **Исходный файл изменён**
  Dynatrace проверяет, что исходный файл, который вы видите в нашей IDE, является тем файлом, который вы развёртываете в своём приложении. Если версия файла неверна, точка останова не будет установлена. Если вы используете обнаружение коммитов исходного кода, вы увидите правильный git-коммит на странице экземпляров приложения.
* **Сбор данных точки останова занял слишком много времени**
  Dynatrace использует защитный механизм на случай, если одна точка останова слишком долго собирает данные из вашего приложения.
* **(Node) Запуск с отладчиком**
  Вы используете ![Live Debugger](https://dt-cdn.net/images/live-debugger-256-b934a2bad5.png "Live Debugger") **Live Debugger** параллельно с другим отладчиком, таким как WebStorm Debugger.

Warning

`Warning` указывает на то, что с точкой останова возникли некоторые проблемы, и Dynatrace прилагает все усилия для доставки запрошенных вами данных.

Сообщения `Warning` документируются в ![Live Debugger](https://dt-cdn.net/images/live-debugger-256-b934a2bad5.png "Live Debugger") **Live Debugger** или IDE, примеры включают:

* **Сбор данных точки останова выполняется с выборкой из-за ограничения частоты**
  Dynatrace использует встроенный механизм ограничения частоты для предотвращения влияния точек останова, установленных в часто выполняемых участках кода, на производительность приложения. Это сообщение указывает, что лимит частоты был достигнут и точка останова была ограничена для соответствующего экземпляра приложения. Сбор данных выполняется с выборкой для предотвращения влияния на производительность вашего приложения.
* **(JVM) Исходный файл не найден**
  Dynatrace использует хеширование исходных файлов для проверки того, что вы отлаживаете правильную версию файлов. В большинстве языков на основе JVM включайте исходный код в ваши архивы Jar/War/Ear.

* **Глубина сбора**: насколько глубоко выполнять сбор данных.

* **Ширина сбора**: сколько элементов собирать из векторов, массивов, списков и т.д.

## Ограничения точек останова

Вы можете установить ограничения для отдельных точек останова, чтобы ограничить объём собираемых данных. При достижении ограничений точка останова будет автоматически отключена. После отключения она не будет собирать дополнительные данные. Для повторного включения точки останова нажмите правой кнопкой мыши на неё и выберите **Enable**.

Ограничения могут быть установлены на основе:

* Времени (например, 1 час, 24 часа, неделя)
* Количества срабатываний (количество раз, когда точка останова активируется)

Значения ограничений точки останова по умолчанию:

* Ограничение по времени: 7 дней
* Ограничение по количеству срабатываний: 100
* Уровень сбора: Medium

## Условия точек останова

Условные точки останова позволяют ограничить данные, собираемые OneAgent. Данные будут собираться только тогда, когда определённое выражение принимает значение true. Это позволяет отлаживать конкретные сценарии и ограничивать количество получаемых сообщений.
Чтобы задать условие, нажмите правой кнопкой мыши на точку останова и выберите **Edit**.

Типы условий:

* Простые условия
  С помощью простого условия вы можете сравнить переменную с определённым значением (или другой переменной).
* Расширенные условия
  С помощью расширенного условия вы можете определить более сложное условие, используя логические параметры. Используйте `&&` для оператора AND, `||` для оператора OR и `(` и `)` для группировки.
