# Защита данных в Dynatrace

* Latest Dynatrace
* 4-min read
* Published May 17, 2018

Мы стремимся обеспечить безопасность, конфиденциальность и доступность платформы Dynatrace Software Intelligence Platform. Безопасность ваших данных и цифрового опыта является для нас ключевым приоритетом, и мы прилагаем все усилия для ответственной защиты и обработки ваших данных. Надлежащая забота о данных лежит в основе нашего бизнеса.

Dynatrace обеспечивает вам полный контроль над вашими данными. С соответствующими настройками по умолчанию именно вы определяете, какими данными делиться с Dynatrace. Для предотвращения несанкционированного доступа, поддержания точности данных и обеспечения корректного использования информации Dynatrace имеет физические, технические и организационные процедуры для защиты ваших данных с помощью нескольких уровней защиты.

## Стратегия минимизации данных

Dynatrace предоставляет вам возможность следовать принципу минимизации данных с целью сокращения обработки данных до строгого минимума. Вы контролируете данные, отправляемые в Dynatrace, с помощью встроенных механизмов, включая [правила маскировки данных](../configuration/configure-global-privacy-settings.md "Узнайте, как настроить маскировку конфиденциальности данных для IP-адресов конечных пользователей, геолокаций и имён пользовательских действий."), [правила именования пользовательских действий](../../../observe/digital-experience/web-applications/initial-setup/create-custom-names-for-user-actions.md "Настройте автоматически генерируемые имена пользовательских действий для ваших веб-приложений."), [API анонимизации](../../../dynatrace-api/environment-api/anonymization.md "Узнайте, как выполнить требования GDPR с помощью Dynatrace API для удаления пользовательских данных.") и другие. Мы обеспечиваем вам возможность получить полную ценность продуктов и сервисов Dynatrace без необходимости передачи персональных данных (за исключением целей аутентификации) в Dynatrace.

## Конфиденциальность и безопасность по дизайну

Dynatrace ценит вашу конфиденциальность и доверие, поэтому мы внедрили концепции конфиденциальности и безопасности по дизайну. Эти концепции прозрачно дают вам возможность контролировать ваши данные и делать конкретный выбор в отношении конфиденциальности. Dynatrace помогает вам в принятии этих решений и стремится обеспечить встраивание компонентов конфиденциальности и безопасности во все части продукта с самого начала. Таким образом, конфиденциальность становится неотъемлемой частью платформы Dynatrace Software Intelligence Platform без ущерба для основной функциональности. Кроме того, наш подход «конфиденциальность и безопасность по дизайну» обеспечивает лучшие в своём классе [настройки конфиденциальности и безопасности по умолчанию](../configuration/configure-global-privacy-settings.md "Узнайте, как настроить маскировку конфиденциальности данных для IP-адресов конечных пользователей, геолокаций и имён пользовательских действий.") прямо из коробки.

## Автоматическое тестирование

Для обеспечения лучшей в своём классе конфиденциальности Dynatrace опирается на обширное автоматическое тестирование всей новой и существующей функциональности продукта. Непрерывное автоматическое тестирование обеспечивает эффективность мер конфиденциальности, развёрнутых на каждом этапе жизненного цикла продукта Dynatrace.

Наши автоматические тесты:

* Адаптируются и выполняются для каждого нового релиза (каждые две недели)
* Регулярно пересматриваются для обеспечения 100% покрытия тестами (каждые два месяца)
* Проверяются в рамках ежегодного обзора структуры конфиденциальности

## Обучение сотрудников

В Dynatrace мы признаём, что наши высокие стандарты конфиденциальности и безопасности данных зависят от осведомлённости и вовлечённости каждого сотрудника Dynatrace. Благодаря культуре осведомлённости о рисках конфиденциальности и безопасности, целенаправленным принципам конфиденциальности данных и регулярным индивидуальным тренингам по кибербезопасности и конфиденциальности мы помогаем нашим сотрудникам поддерживать наше лидерство на рынке в области конфиденциальности и безопасности. Легкодоступные каналы помощи и партнёрства по конфиденциальности данных по всей компании дополнительно укрепляют культуру конфиденциальности и безопасности данных в Dynatrace.

## Контроль доступа

Вы контролируете, имеет ли Dynatrace доступ к вашим данным. Только пользователи, являющиеся членами административных групп среды мониторинга, которые строго контролируются вами, могут предоставлять или отзывать доступ к собранным данным. Для получения информации о том, какие возможности управления доступом доступны в Dynatrace, см. [Разрешения на основе ролей](../../identity-access-management/permission-management/role-based-permissions.md "Разрешения на основе ролей").

Когда вам нужна наша поддержка, авторизованным сотрудникам Dynatrace, а именно представителям второго и третьего уровня поддержки, может быть предоставлен доступ для просмотра ваших данных. Такой доступ ограничен строгой политикой «необходимости доступа». Каждый доступ регистрируется, снабжается временной меткой и предоставляется вам в автоматическом режиме через наш REST API. Все авторизованные сотрудники Dynatrace связаны строгими соглашениями о конфиденциальности.

## Мониторинг и аудит-логирование

Все системы, управляемые Dynatrace, подвергаются мониторингу состояния и безопасности, аудит-логированию и автоматическому анализу системных логов. Когда запрашивается поддержка Dynatrace для удалённых установок, доступ к вашим системам записывается в [аудит-логи](../configuration/audit-logs-api.md "Узнайте, как управлять аудит-логами с помощью API.").

## Управление хранением данных

Для выполнения сервисов мониторинга Dynatrace хранит только те данные, которые вы решите предоставить, и сохраняет их в вашем кластере Dynatrace. Dynatrace автоматически удаляет данные, которые старше настроенных [периодов хранения](data-retention-periods.md "Проверьте сроки хранения для различных типов данных."). Dynatrace также предоставляет функциональность [адаптивного хранения данных](adaptive-data-retention.md "Узнайте, как время хранения данных в хранилищах транзакций, Session Replay и мониторинга логов корректируется.") для периодического увеличения или уменьшения времени хранения данных при превышении квоты хранилища вашей среды.

## Соответствие законам о защите данных

Dynatrace предоставляет гибкие [настройки конфиденциальности данных](../configuration/configure-global-privacy-settings.md "Узнайте, как настроить маскировку конфиденциальности данных для IP-адресов конечных пользователей, геолокаций и имён пользовательских действий."), которые позволяют вам точно настроить конфигурацию среды и приложений в соответствии с региональными законами о конфиденциальности.

## Средства контроля безопасности

Dynatrace также предоставляет многоуровневые [средства контроля безопасности](../data-security/data-security-controls.md "Узнайте о средствах контроля безопасности данных и операционной безопасности.") для обеспечения сохранности ваших данных.
