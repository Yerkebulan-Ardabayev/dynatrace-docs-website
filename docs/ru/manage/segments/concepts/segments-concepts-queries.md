# Сегменты в DQL-запросах


* Latest Dynatrace
* Explanation
* 2-min read
* Published Mar 29, 2023

Сегменты предназначены для использования в качестве контекста для DQL-запросов. Подобно **временным диапазонам**, сегменты непосредственно влияют на объём данных, сканируемых и возвращаемых для заданного запроса.

## Основные термины

Grail
:   Grail — это хранилище данных Dynatrace, разработанное специально для данных наблюдаемости. Оно выступает единым унифицированным хранилищем для журналов, метрик, трасс, событий и многого другого.

Dynatrace Query Language (DQL)
:   Dynatrace Query Language (DQL) — это мощный инструмент для изучения данных и выявления закономерностей, обнаружения аномалий и выбросов, создания статистических моделей и многого другого на основе данных, хранящихся в Dynatrace Grail.

## Сегменты в DQL-запросах

Grail служит основным бэкендом для приложений на платформе Dynatrace. Данные, отображаемые в этих приложениях, извлекаются из Grail с помощью DQL-запросов.

* В более технических приложениях, таких как **Notebooks** и **Dashboards**, вы можете непосредственно составлять и изменять DQL-запросы.
* Другие приложения обновляют данные на экране в ответ на взаимодействие пользователя с простыми элементами веб-интерфейса, отправляя DQL-запросы для получения данных в фоновом режиме.

Сегменты выступают в роли необязательного контекста для DQL-запросов, внедряя определённые пользователем, предварительно сконфигурированные условия фильтрации.

В ходе выполнения запроса Grail оценивает только релевантные условия переданных сегментов на основе целевого объекта данных запроса. Запрос `fetch logs` будет рассматривать условия фильтрации только для журналов, запрос `timeseries` — только для метрик и так далее.

* Несколько условий для одного объекта данных в одном сегменте объединяются с помощью оператора OR.
* Условия из нескольких разных сегментов объединяются с помощью оператора AND, формируя пересечение параллельно выбранных сегментов.

Подробнее о том, как сегменты передаются в контексте запроса DQL Query API, см. в разделе [ExecuteRequest](https://dt-url.net/6h03utf).
