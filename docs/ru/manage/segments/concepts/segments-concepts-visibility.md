# Видимость сегментов


* Latest Dynatrace
* Explanation
* 1-min read
* Published Mar 29, 2023

Параметр **Visibility** определяет, кто видит сегмент в своём списке сегментов.

* **Unlisted** — эти сегменты видны только в списке сегментов владельца. Это предотвращает излишнее засорение списков сегментов. Данный параметр установлен по умолчанию.
* **Anyone in the environment** — эти сегменты отображаются в списке сегментов всех пользователей.

![segments query](https://dt-cdn.net/images/segments-6-1000-e8d8694fe0.png)

Важно понимать, что параметр **Visibility** не влияет на общий доступ к сегментам.

* Скрытые сегменты по-прежнему могут быть доступны другим пользователям через ссылки в приложениях, например в общих записных книжках и дашбордах. Все пользователи имеют доступ к сегментам только для чтения. Это упрощает совместную работу с сегментами.
* Разрешения в политиках IAM определяют, кто может настраивать сегменты, видимые всем пользователям.
* Сами сегменты не содержат никаких данных. Все запросы, с сегментами или без них, всегда соблюдают разрешения доступа к данным, установленные политиками IAM.

## Разрешения для сегментов

Независимо от настроенной видимости, доступ к любому сегменту можно получить с разрешением `storage:filter-segments:read`. Это гарантирует, что даже скрытые сегменты, на которые могут ссылаться в общей записной книжке, смогут использоваться любым пользователем, имеющим доступ к этой записной книжке.

Дополнительные сведения о разрешениях для сегментов см. в разделе [Выражения политик IAM](../../identity-access-management/permission-management/manage-user-permissions-policies/advanced/iam-policystatements.md#storage-filter-segments-read "Полный справочник политик IAM и соответствующих условий для всех сервисов Dynatrace.").
