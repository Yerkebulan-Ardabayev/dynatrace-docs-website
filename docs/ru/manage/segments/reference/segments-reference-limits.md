# Ограничения сегментов

* Latest Dynatrace
* Reference
* 1-min read
* Updated on Oct 21, 2025

## Общие сведения

* 10 000 сегментов на среду
* 20 блоков включения на сегмент
* 1 блок включения на тип (например, Logs, Metrics, Spans и т. д.)
* Минимум 1 выражение на условие фильтра
* Максимум 10 выражений на условие фильтра
* 10 000 значений в выпадающем списке переменных
* 10 сегментов на запрос

### Классические сущности

Для классических сущностей действуют различные ограничения. Эти ограничения не применяются к Smartscape на Grail.

* 1 дополнительное включение на тип классической сущности + связь (только для включений сущностей)
* Условия для связанных классических сущностей могут быть пустыми
* 100 выбранных значений переменной на сегмент

## Условия блока включения

### Все данные

* Поддерживаются только поля и значения типов `STRING` и `ARRAY_OR_STRINGS`
* Поддерживаются только операторы `=` и `in()`

  + Можно использовать подстановочные знаки `*` для совпадений `starts-with`, `contains` и `ends-with`. Подробнее см. в разделе [Wildcards](../../../discover-dynatrace/get-started/dynatrace-ui/ui-filter-field.md#wildcards "The filter field is a powerful tool that allows you to quickly find relevant information or narrow down results within apps.").
  + В условиях для `starts-with` после имён переменных допускаются символы звёздочки (например, `foo = $bar*`)

### Классические сущности

Для классических сущностей действуют различные ограничения. Эти ограничения не применяются к Smartscape на Grail.

* Допускаются только выбранные свойства (предлагаемые в поле фильтра)
* `starts-with`, `contains` и `ends-with` не поддерживаются

  + Единственное исключение — `entity.name`, которое поддерживает `starts-with`
  + Для остальных свойств поддерживается только `equals`
