# Сегментирование журналов по бакетам


Настройте сегмент для использования в качестве удобного фильтра журналов по бакетам Grail для оптимизации производительности запросов и потребления лицензий.

## Для кого предназначена эта статья

Эта статья предназначена для администраторов, управляющих разделением данных в бакетах Grail, а также для опытных пользователей, стремящихся к более эффективным запросам журналов.

## Что вы узнаете

В этой статье вы узнаете, как создать новый сегмент для использования в качестве фильтра бакетов журналов при запросах. Вы также узнаете разницу между одноразовыми сегментами со статическими условиями и динамическими сегментами с использованием переменных.

Предварительные условия

* Среда Dynatrace SaaS на базе Grail и AppEngine.
* Наличие разрешений `storage:filter-segments:write` и `storage:filter-segments:read`. Сведения о настройке разрешений см. в разделе [Permissions in Grail](../../../platform/grail/organize-data/assign-permissions-in-grail.md "Узнайте, как назначать разрешения для бакетов и таблиц в Grail.").

Ключевые термины

Бакет Grail
:   Журналы могут храниться в разных бакетах Grail. Бакеты позволяют улучшить производительность запросов за счёт сокращения времени выполнения и области чтения данных.

Одноразовый сегмент
:   Сегмент, настроенный со статическими условиями для одноразового сценария.

Динамический сегмент
:   Сегмент, настроенный с динамическими условиями с использованием переменных.

### Шаг 1: Создание сегмента для одного бакета журналов

В ряде случаев может быть желательно иметь сегмент для одного бакета. В следующем примере показано, как это сделать путём фильтрации журналов бакета `default_logs`.

![Сегмент logs_default bucket](https://dt-cdn.net/images/segments-log-bucket1-2528-0fcb794b9b.png)

1. Перейдите в ![Segments](https://dt-cdn.net/images/segments-256-8e66310720.webp "Segments") **Segments** и выберите **Segment** для добавления нового сегмента
2. Введите имя сегмента, например "Log bucket"
3. Выберите **Add from data types** > **Logs**, чтобы включить журналы в сегмент
4. В поле **Type to filter** выберите `dt.system.bucket`
5. Укажите определённый бакет для фильтрации (например, `dt.system.bucket = default\_logs`)
6. Выберите **Save**

Успешно настроенные сегменты отображаются в списке сегментов. Выберите > **Edit**, чтобы изменить сегмент.

### Шаг 2: Добавление переменной для фильтрации по любому бакету журналов

Добавление переменной в сегмент для динамической фильтрации по нескольким бакетам журналов вместо одного делает сегмент универсально применимым.

![Сегмент с переменной для бакета журналов](https://dt-cdn.net/images/segments-log-bucket2-2524-201b656143.png)

1. Выберите **Variable**
2. Выполните запрос списка бакетов журналов, отсортированных по алфавиту

   ```
   fetch dt.system.buckets


   | filter dt.system.table == "logs"


   | fields bucket=name


   | sort bucket
   ```
3. Выберите **Run query**
4. Выберите **Done** для завершения настройки переменной
5. Настройте условие включения журналов для использования переменной `$bucket` (`dt.system.bucket = $bucket`)
6. Выберите **Save**

Успешно настроенные переменные отображаются над существующими блоками включения сегмента. Выберите **Edit variable(s)** для изменения переменных.

### Шаг 3: Использование сегмента для анализа журналов по бакетам

Вы можете анализировать журналы в различных приложениях. Для выполнения запросов журналов в ![Notebooks](https://dt-cdn.net/images/notebooks-768-046137830a.webp "Notebooks") **Notebooks**:

1. Перейдите в ![Notebooks](https://dt-cdn.net/images/notebooks-768-046137830a.webp "Notebooks") **Notebooks** и выберите **Notebook**
2. Выберите **Logs**
3. Откройте селектор сегментов и в разделе **Filter by segments** выберите ранее созданный сегмент для бакетов журналов
4. В списке **Select an option** выберите один или несколько бакетов журналов для фильтрации
5. Выберите **Apply** для завершения выбора сегмента
6. Выберите **Run query** для выполнения запроса журналов выбранных бакетов

![Выбор сегмента](https://dt-cdn.net/images/segments-12-2142-b2a14173b1.png)
