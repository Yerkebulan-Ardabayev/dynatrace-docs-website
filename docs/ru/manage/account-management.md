# Управление учётной записью


* Latest Dynatrace
* Описание

Dynatrace Account Management предоставляет единое место для:

* Управления лицензиями и подписками Dynatrace
* Управления пользователями Dynatrace и доступом SSO (только для SaaS-развёртываний; для Managed-учётных записей это выполняется в настройках кластера)
* Мониторинга внедрения платформы Dynatrace и состояния среды

## Доступ

### Веб-интерфейс

Чтобы открыть **Account Management** через веб-интерфейс:

* **Latest Dynatrace**: выберите своё имя пользователя в левом нижнем углу, а затем выберите **Account Management**.
* **Previous Dynatrace**: выберите значок пользователя в правом верхнем углу, а затем выберите **Account settings**.

  ![Отображение меню пользователя](https://dt-cdn.net/images/user-menu-86-9ab2cd2961.png)

### Адрес в браузере

Для прямого доступа к Account Management из адресной строки браузера перейдите по адресу [https://myaccount.dynatrace.com/](https://myaccount.dynatrace.com/).

### Разрешения

Информацию о разрешениях, необходимых для доступа к Account Management, см. в разделе [Разрешения Account Management](identity-access-management/permission-management/account-management-permissions.md "Разрешения Dynatrace, необходимые для доступа к Account Management").

## Навигация верхнего уровня

Верхний уровень Account Management содержит три вкладки для управления вашей информацией.

### Мои учётные записи

![Account Management: Мои учётные записи](https://dt-cdn.net/images/account-management-my-accounts-569-b7d835311e.png)

Вкладка **My accounts** отображает ваши учётные записи.

* Выберите учётную запись, которой вы хотите управлять.
* После выбора учётной записи вы можете добавить страницу учётной записи в закладки для прямого доступа в следующий раз.

Чтобы вернуться к вкладке **My accounts** позже, откройте меню в правом верхнем углу и выберите **My accounts**.

### Мой профиль

![Account Management: Мой профиль](https://dt-cdn.net/images/account-management-my-profile-571-20958f7605.png)

Используйте вкладку **My profile** для обновления вашего профиля пользователя и параметров уведомлений.

* **My profile** — информация о вас, такая как имя, должность и местоположение.

  Настройка часового пояса здесь носит информационный характер и не влияет на какие-либо функции Dynatrace, кроме генерации отчётов для [подписок на панели мониторинга в Dashboards Classic](../analyze-explore-automate/dashboards-classic/dashboards/subscribe-to-dashboard-reports.md "Узнайте, как подписаться на отчёты, создаваемые на основе панелей мониторинга Dynatrace.").
* **Environment notifications** — список сред, для которых вы можете включить уведомления по электронной почте о сбоях.

Чтобы вернуться к вкладке **My profile** позже, откройте меню в правом верхнем углу и выберите имя вашей учётной записи.

### Мои токены платформы

![Account Management: Мои токены платформы](https://dt-cdn.net/images/account-management-my-platform-tokens-578-bcbc9af2e1.png)

Токены платформы позволяют взаимодействовать с платформой Dynatrace через API для внешних интеграций или скриптов. Подробнее см. [Токены платформы](identity-access-management/access-tokens-and-oauth-clients/platform-tokens.md "Создавайте персонализированные токены платформы для доступа к сервисам Dynatrace через API в контексте вашего пользователя.").

Чтобы вернуться к вкладке **My platform tokens** позже, откройте меню в правом верхнем углу и выберите **Platform tokens**.

## Навигация на уровне учётной записи

После выбора учётной записи на вкладке **My accounts** в **Account Management** отображается панель меню на большом экране.

* На маленьком экране панель меню сворачивается в компактное ![Меню](https://dt-cdn.net/images/account-management-icon-hamburger-c275e8015b.svg "Меню") меню с теми же опциями.
* Доступные опции меню зависят от типа развёртывания, разрешений пользователя и модели лицензирования.

#### Примеры

![Пример панели меню Account Management: Лицензия](https://dt-cdn.net/images/account-management-header-license-736-374ccfea23.png)

![Пример панели меню Account Management: Подписка](https://dt-cdn.net/images/account-management-header-subscription-870-da77a8f47b.png)

### Главная

Выберите **Home** из любого места в **Account Management**, чтобы вернуться на главную страницу выбранной учётной записи.

### Лицензия / Подписка

* **License** отображается для классических контрактов
* **Subscription** отображается для DPS

Откройте меню **License** или **Subscription**, чтобы:

* Просмотреть использование лицензии/подписки и историю потребления
* Просмотреть прогноз будущего использования

* Просмотреть все уведомления о квотах лицензий для SaaS и Managed лицензий

#### Обзор

* **License** > **Overview**

  Потребление основано на различных типах единиц мониторинга, которые расходуются вашими средами Dynatrace при мониторинге ваших приложений и связанных сервисов.
* **Subscription** > **Overview**

  Dynatrace Platform Subscription: потребление основано на различных типах единиц мониторинга, которые расходуются вашими средами Dynatrace при мониторинге ваших приложений и связанных сервисов. Отображаемые значения представляют вычеты из вашего годового обязательства.

### Управление удостоверениями и доступом

Для клиентов Dynatrace SaaS и для развёртываний Dynatrace Platform Subscription, использующих модель SaaS, откройте меню **Identity & access management** (IAM) для управления:

* Группами пользователей и разрешениями
* Пользователями и группами с SAML в Dynatrace SaaS
* Пользователями и группами с SCIM в Dynatrace SaaS
* Токенами API OAuth для Account Management API

IAM для развёртываний Dynatrace Managed управляется внутри каждого кластера. Подробнее см. [Managed - managed-user-groups](https://docs.dynatrace.com/managed/shortlink/managed-user-groups).

#### Пользователи

Используйте **Identity & access management** > **People** для приглашения и назначения пользователей в группы для управления разрешениями и доступом Dynatrace.

#### Группы

Используйте **Identity & access management** > **Group management** для создания групп пользователей для управления разрешениями и доступом Dynatrace для пользователей, назначенных в эти группы.

#### Политики

Используйте **Identity & access management** > **Policy management** для управления политиками Dynatrace.

#### Верификация домена

Используйте **Identity & access management** > **Domain verification** для создания и верификации доменов учётных записей для использования в конфигурациях SSO, таких как SAML или SCIM.

#### Настройка SAML

Используйте **Identity & access management** > **SAML configuration** для настройки аутентификации пользователей для нескольких доменов. Если вы хотите использовать корпоративные учётные данные для аутентификации в Dynatrace SaaS, вы можете настроить SAML для делегирования аутентификации вашему поставщику удостоверений. В качестве предварительного условия необходимо подтвердить право собственности на ваш домен, добавив запись ресурса в ваш домен.

#### Настройка SCIM

Используйте **Identity & access management** > **SCIM configuration** для управления пользовательскими удостоверениями в облачных приложениях и сервисах. SCIM используется для автоматизации обмена пользовательскими удостоверениями между различными доменами и системами. Необходимо добавить и подтвердить право собственности на домен, к которому принадлежат ваши пользователи, перед генерацией токенов и завершением настройки SCIM.

#### Клиенты API OAuth

Используйте **Identity & access management** > **OAuth clients** для настройки и управления клиентами API OAuth учётной записи. Account Management API помогает управлять вашей учётной записью и её пользователями. Например, вы можете управлять доступом к средам Dynatrace, создавая группы с различными уровнями доступа и затем связывая эти группы с пользователями.

### Lens

Функция **Lens** в Account Management предоставляет информацию о внедрении платформы и её состоянии, помогая понять, как Dynatrace используется в вашей организации, и рекомендует области для оптимизации.

Подробнее о Lens см. [Lens](account-management/lens.md "Получите информацию о внедрении и состоянии Dynatrace.").

#### Внедрение

Страница **Lens** > **Adoption** предоставляет информацию о внедрении платформы и её состоянии:

* Активность пользователей
* Какие технологии мониторятся в Dynatrace
* Как развёрнуты мониторируемые хосты (тип лицензии, тип платформы)
* Сколько синтетических тестов выполняется
* Сколько интеграций уведомлений о проблемах настроено

Подробнее см. [Внедрение](account-management/lens.md#adoption "Получите информацию о внедрении и состоянии Dynatrace.").

#### Среды

**Lens** > **Environments** предлагает:

* Обзор количества технологий, обнаруженных и мониторируемых Dynatrace с течением времени.
* Сводку сред в вашем развёртывании Dynatrace с фокусом на хосты.
* Сводку количества мониторируемых хостов, развёрнутых в облаке.
* Сводку количества публичных и частных синтетических тестов, выполняющихся в ваших средах.
* Общую сводку количества определённых токенов API, настроенных интеграций уведомлений о проблемах, используемых интеграций PaaS и определённых экспортов сессий.

Подробнее см. [Среды](account-management/lens.md#environments "Получите информацию о внедрении и состоянии Dynatrace.").

### Настройки

Используйте **Settings** для указания общей информации об учётной записи.

Подробнее см. [Настройки](account-management/settings.md "Настройте контактную информацию учётной записи, данные для выставления счетов и доставки.").

### Контактная информация

Используйте **Settings** > **Contact information** для указания контактной информации учётной записи, данных для выставления счетов и доставки.

### Среды

Используйте **Settings** > **Environments** для отображения списка сред и, с помощью **Action** > **Edit environment**, изменения имени или часового пояса выбранной среды.

Настройка часового пояса здесь носит информационный характер и не влияет на какие-либо функции Dynatrace, кроме генерации отчётов для [подписок на панели мониторинга в Dashboards Classic](../analyze-explore-automate/dashboards-classic/dashboards/subscribe-to-dashboard-reports.md "Узнайте, как подписаться на отчёты, создаваемые на основе панелей мониторинга Dynatrace.").

* Тикеты поддержки
