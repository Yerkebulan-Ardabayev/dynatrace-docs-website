System for Cross-domain Identity Management (SCIM) -- это открытый стандарт для автоматизации обмена информацией об идентификации пользователей между доменами идентификации или ИТ-системами. Вы можете настроить Dynatrace SaaS для автоматического получения информации об идентификации пользователей от поставщика удостоверений (IdP) вашей организации через SCIM.

## Преимущества использования SCIM с Dynatrace SSO

* Пользователи в вашем IdP будут автоматически синхронизированы с Dynatrace, поэтому нет необходимости приглашать их вручную

  + если пользователь отключён или удалён из IdP, он также будет отключён в Dynatrace
* Группы в вашем IdP будут автоматически синхронизированы с Dynatrace

  + Все группы, созданные через SCIM, будут иметь тип `SCIM` в Account Management
* Пользователи будут назначены в группы в Dynatrace, отражая взаимосвязи внутри IdP

## Замечания и ограничения SCIM

* Синхронизация SCIM является односторонней -- от IdP к Dynatrace. Это связано с тем, что технически IdP является владельцем как пользователей, так и групп, и изменения в Dynatrace никогда не будут перенесены в IdP.
* Dynatrace принимает только пользователей с доменами, которые были верифицированы в рамках учётной записи
* Группы типа `SCIM` управляются только через SCIM и не могут быть удалены или изменены через Account Management
* Разрешения для групп SCIM по-прежнему необходимо назначать вручную в Dynatrace

  + Для программного управления разрешениями можно использовать [Account Management API](../../../dynatrace-api/account-management-api.md "Изучите конечные точки Account Management API.")
* Если ваш SCIM-клиент не поддерживает динамическое изменение внешнего идентификатора, смена email/логина приведёт к прекращению работы интеграции SCIM для таких пользователей. Запросы будут отклонены из-за недействительного имени пользователя (в SCIM это email)
* Область пользователей и групп, синхронизируемых в Dynatrace через SCIM, может быть сужена в приложении SCIM вашего IdP
* Пользователи в группах SCIM не отображаются в веб-интерфейсе при предоставлении общего доступа к документу (панели мониторинга или блокноту) определённым пользователям или группам, если вы не добавите этих пользователей в локальные группы Dynatrace.
* Пользователь не может быть назначен учётной записи, если учётная запись уже достигла [максимального количества назначенных пользователей](../iam-limits.md "Ограничения IAM для Dynatrace SaaS").
* Пользователь не может быть добавлен в группу, если он уже достиг [максимального количества групп, в которые может быть назначен](../iam-limits.md "Ограничения IAM для Dynatrace SaaS").

## Требования и поддерживаемые функции SCIM

* Dynatrace поддерживает **SCIM 2.0** и операции **GET**, **POST**, **PATCH**, **PUT** и **DELETE** для ресурсов **User** и **Group**.
* Для аутентификации SCIM требует **Bearer token** в заголовке **Authorization**.
* SCIM настраивается для областей учётной записи и домена. Для учётной записи требуется как минимум одна верификация владения доменом.
* Через SCIM с Dynatrace могут быть синхронизированы только пользователи, домены электронной почты которых были верифицированы на предмет владения.
* Обязательные и поддерживаемые атрибуты SCIM:

  | Атрибут SCIM | Тип |
  | --- | --- |
  | userName | формат email |
  | name.givenName | string |
  | name.familyName | string |
  | active | boolean |
* **userName** должен быть постоянным. Dynatrace не поддерживает смену email пользователя.

## Верификация владения доменом

Прежде чем приступить к настройке SCIM, необходимо подтвердить владение доменом. Верификация основана на проверке DNS TXT-записи.

Для учётной записи достаточно верифицировать домен один раз. Если домен был верифицирован для SAML, он будет действителен и для SCIM.

Чтобы верифицировать владение доменом

1. Перейдите в [**Account Management**](https://myaccount.dynatrace.com/). Если у вас несколько учётных записей, выберите учётную запись, которой хотите управлять.
2. Перейдите в **Identity & access management** > **Domain verification**.
3. В разделе **Add domain** введите домен (например, `mycompanyname.com`), для которого вы хотите настроить SCIM, и нажмите **Add**, чтобы добавить его в таблицу **Pending domains**.

   Несколько доменов

   Если пользователи в вашей организации используют более одного домена для входа (например, `@mycompanyname.com` и `@uk.mycompanyname.com`), вы можете добавить дополнительные домены в отдельных строках и начать их верификацию параллельно. Введите каждый домен в отдельной строке.
4. Для каждого домена, который вы верифицируете, в таблице **Pending domains** выберите **Copy** (в столбце **Value**) и добавьте скопированную TXT-запись ресурса в DNS-конфигурацию вашего домена.
5. Для каждого домена, который вы верифицируете, в таблице **Pending domains** выберите **Actions** > **Verify**, чтобы Dynatrace мог проверить, что запись была добавлена в DNS вашего домена.

   Время распространения

   Обычно для распространения записи через систему DNS и доступности значения для проверки Dynatrace требуется несколько минут. В некоторых случаях это может занять до 24 часов.
6. Каждый верифицированный домен добавляется в таблицу **Verified domains**.

## Получение конечной точки Dynatrace SCIM и создание секретного токена

Используйте эту процедуру для получения базового URL Dynatrace SCIM для вашей учётной записи и создания секретного токена.

Токен отображается только один раз во время генерации. Скопируйте и вставьте его в безопасное место. Если вы потеряете его, вам придётся сгенерировать новый и заменить его в вашем приложении.

1. Перейдите в [**Account Management**](https://myaccount.dynatrace.com/) > **Identity & access management** > **SCIM configuration**.
2. В разделе **Generate new token** при необходимости введите **Description** (описание) токена (или вы можете добавить описание для токена позже).
3. Нажмите **Generate token**
4. Рядом с **Token value** нажмите **Copy**, чтобы скопировать токен в буфер обмена, а затем вставьте его в безопасное место для дальнейшего использования.

   Dynatrace SCIM поддерживает только аутентификацию Bearer Token. Пример:

   | Заголовок | Значение |
   | --- | --- |
   | Authorization | Bearer <DYNATRACE_TOKEN_PLACEHOLDER> |

Конечная точка SCIM, необходимая для настройки SCIM в вашем приложении, добавляется в **List of tokens**.

## Часто задаваемые вопросы

Поддерживается ли смена адреса электронной почты?

Нет, адрес электронной почты не может быть изменён через SCIM. Любые запросы на обновление адреса электронной почты игнорируются.

Сколько токенов аутентификации SCIM можно сгенерировать?

Вы можете сгенерировать до 10 токенов аутентификации SCIM. По соображениям безопасности мы настоятельно рекомендуем удалить все неиспользуемые токены.

Можно ли подготовить пользователей из разных доменов электронной почты?

Да, вы можете подготовить столько пользователей из разных доменов электронной почты, сколько необходимо, при условии, что все их домены были верифицированы. Вы можете верифицировать несколько доменов для одной учётной записи.

Можно ли создать более одной группы с одинаковым именем?

Нет, некоторые IdP, такие как Azure AD, допускают несколько групп с одинаковым именем в одной учётной записи. Однако для Dynatrace SSO каждое имя группы в учётной записи должно быть уникальным.

Можно ли установить `description` группы во время подготовки?

Нет, мы ограничены спецификацией протокола SCIM, поэтому дополнительные поля, такие как `description`, не могут быть установлены во время подготовки.

### Azure

Я случайно удалил пользователя, синхронизированного через SCIM, в моей учётной записи Dynatrace. Можно ли повторно синхронизировать пользователя через SCIM в Azure?

Да, вы можете перезапустить синхронизацию SCIM. Для этого перейдите на вкладку **Overview** приложения SCIM и выберите **Restart provisioning**.
