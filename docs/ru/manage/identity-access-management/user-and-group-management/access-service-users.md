# Сервисные пользователи

* Latest Dynatrace

* 3 мин. чтения
* Обновлено 05 дек. 2025 г.

Администратор или пользователь, входящий в группу с разрешением `View and manage users and groups`, может выполнять перечисленные здесь операции по управлению сервисными пользователями.

## Что такое сервисный пользователь?

Сервисный пользователь — это неинтерактивный пользователь: он не может войти в Dynatrace и не связан с реальным человеком. У него есть собственные разрешения на управление идентификацией и доступом, назначенные напрямую.

Сервисного пользователя можно выбрать в качестве исполнителя [рабочего процесса](/docs/analyze-explore-automate/workflows/security#service-users "Руководство по аспектам безопасности автоматизации рабочих процессов в Dynatrace Workflows") или [пользовательского оповещения](/docs/dynatrace-intelligence/anomaly-detection/anomaly-detection-app#service-user "Изучите конфигурации обнаружения аномалий с помощью приложения Anomaly Detection.").

## Сценарий использования

Когда рабочие процессы и пользовательские оповещения выполняются в контексте сервисного пользователя, они становятся независимыми от статуса пользователя, который их поддерживает. Это хорошо подходит для отдельных отделов, производственных сценариев или любых совместных усилий, где зависимость от реального пользователя может помешать работе. Использование сервисного пользователя также повышает безопасность действий, выполняемых исполнителем.

## Просмотр и редактирование сервисных пользователей

1. Перейдите в [**Account Management**](https://myaccount.dynatrace.com/). Если у вас несколько учётных записей, выберите нужную.
2. Перейдите в **Identity & access management** > **Service users**.

   В таблице отображаются определённые сервисные пользователи.
3. В столбце **Actions** для нужного пользователя выберите > **Edit**.
4. На странице **Edit service user** можно:

   * Изменить **Name**
   * Изменить **Description**

**Примечание**: Изменить адрес электронной почты сервисного пользователя нельзя. Он является идентификатором, используемым в операторах политик. См. [Создание политик на основе сервисного пользователя](#policy).

5. В разделе **Select group** можно добавить или удалить выбранные группы. Сервисные пользователи получают разрешения через выбранные группы.
6. Выберите **Save**.

## Создание сервисного пользователя

1. На странице **Service users** выберите **Add service user**.
2. На странице **Create service user** введите следующие сведения о сервисном пользователе.

   * **Name**
   * Необязательное **Description**

     **Совет**: Убедитесь, что оба значения понятны администраторам среды, чтобы они понимали назначение сервисного пользователя.
     Адрес электронной почты сервисного пользователя создаётся автоматически и не может быть изменён. Он является идентификатором, используемым в [операторах политик](#policy).
3. В разделе **Assign permissions** выберите, назначать ли разрешения **Through existing groups** или **Directly**.

   * **Through existing groups**: Выберите одну или несколько существующих групп, разрешения которых вы хотите назначить этому сервисному пользователю.
   * **Directly**: Сделайте собственный выбор разрешений. Новая группа будет создана автоматически.

     1. На вкладке **Assign permissions** выберите **Permission**, **Scope** и **Boundaries** для назначения этому сервисному пользователю.
     2. На вкладке **View permissions** проверьте сведения о выбранном разрешении.
4. Выберите **Create**.

## Разрешения пользователей на использование сервисных пользователей в качестве исполнителей

Пользователь, желающий использовать сервисного пользователя в качестве исполнителя рабочего процесса или пользовательского оповещения, должен иметь разрешение `iam:service-users:use`.

Это разрешение предоставляется следующими политиками по умолчанию:

* [Dynatrace Pro User](/docs/manage/identity-access-management/permission-management/default-policies#DynatraceAccessProUser "Справочник по политикам Dynatrace по умолчанию")
* [Dynatrace Admin User](/docs/manage/identity-access-management/permission-management/default-policies#DynatraceAccessAdminUser "Справочник по политикам Dynatrace по умолчанию")

## Создание политик на основе сервисного пользователя

Для более детального контроля над использованием сервисных пользователей можно создать политику, разрешающую пользователям использовать только конкретных сервисных пользователей в качестве исполнителей рабочих процессов или пользовательских оповещений.

Для этого создайте политику с `iam:service-users:use`, установленным в условие `iam:service-user-email`.

Например:

```
ALLOW iam:service-users:use



WHERE iam:service-user-email IN ("be820735-3114-4d40-9c44-dfa18fa62be9@service.sso.dynatrace.com");
```

Такие политики безопасны. Невозможно изменить идентификатор сервисного пользователя или назначить ему пользовательский адрес электронной почты.
