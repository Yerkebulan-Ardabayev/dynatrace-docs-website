![Settings](https://dt-cdn.net/images/settings-icon-256-38e1321b51.webp "Settings") **Settings** — это предустановленное приложение Dynatrace, которое позволяет вам легко управлять всеми настройками вашей среды в одном месте.

Оно служит центральной точкой входа для общесистемных конфигураций. Вы можете использовать его для управления тем, как ваши данные собираются, обрабатываются, хранятся и анализируются.

## Предварительные требования

* Для доступа к ![Settings](https://dt-cdn.net/images/settings-icon-256-38e1321b51.webp "Settings") **Settings**:

  Пользователь должен принадлежать к группе, привязанной к политике по умолчанию Standard user.
* Для изменения ![Settings](https://dt-cdn.net/images/settings-icon-256-38e1321b51.webp "Settings") **Settings**:

  Пользователь должен принадлежать к группе, привязанной к политике по умолчанию Pro user.

Обратите внимание, что доступ к выбранным настройкам может быть дополнительно ограничен с помощью специализированных политик IAM, при которых пользователи имеют доступ только к тем настройкам, на которые им назначены разрешения. Для получения дополнительной информации см. [Разрешения](#permissions).

## Сценарии использования

С помощью ![Settings](https://dt-cdn.net/images/settings-icon-256-38e1321b51.webp "Settings") **Settings** вы можете:

* Быстро находить и обнаруживать все настройки среды, связанные со сбором, обработкой и анализом данных.
* Получать доступ к подмножеству этой конфигурации или к представлению только для чтения в зависимости от разрешений пользователя.

## Основные концепции

Ознакомьтесь со следующими концепциями, чтобы понять, как работают настройки Dynatrace.

### Область действия и иерархия настроек

Многие настройки могут быть заданы для различных областей действия (для всей среды мониторинга или для конкретной сущности). Область действия по умолчанию — глобальная (вся среда мониторинга).

Наиболее конкретная настройка всегда имеет приоритет. Например, конфигурация на уровне хоста переопределяет конфигурацию на уровне группы хостов, а конфигурация на уровне группы хостов, в свою очередь, переопределяет конфигурацию на уровне среды.

### Доступ к настройкам

Доступ к настройкам контролируется через политики IAM. Политики позволяют создавать гибкий и детализированный доступ к конфигурациям, при котором пользователи имеют доступ только к тем настройкам, на которые им назначены разрешения. Дополнительные разрешения для вступления политик в силу не требуются. Политики предоставляют доступ к конфигурациям как через приложение ![Settings](https://dt-cdn.net/images/settings-icon-256-38e1321b51.webp "Settings") **Settings**, так и через Settings API, если иное не настроено на уровне схемы.

Если вам необходимо настроить детализированный доступ к определённым сущностям, вы можете сделать это, например, через группу хостов или контекст безопасности.

Чтобы узнать, как настроить политики доступа для настроек, см. [Справочник политик IAM](identity-access-management/permission-management/manage-user-permissions-policies/advanced/iam-policystatements.md#settings "Полный справочник политик IAM и соответствующих условий для всех сервисов Dynatrace.").

### Настройки отдельных приложений Dynatrace

![Settings](https://dt-cdn.net/images/settings-icon-256-38e1321b51.webp "Settings") **Settings** является расширяемым и автоматически отображает страницы настроек, установленные через другие приложения.

### Программное управление настройками

Вы можете управлять настройками Dynatrace централизованно, со всеми преимуществами системы контроля версий, такими как история изменений, проверки, утверждение и программная генерация настроек.

Dynatrace предоставляет два варианта:

#### Dynatrace Settings API

Вы можете управлять настройками Dynatrace программно с помощью Settings API. Этот API позволяет создавать, обновлять, получать и удалять объекты настроек в Dynatrace. Вот как можно к этому подойти.

Настройки передаются в Dynatrace в виде JSON-полезной нагрузки для объекта настроек. Структура JSON зависит от конкретной функции и определяется схемами настроек.

Для получения дополнительной информации см. Settings API.

#### Monaco

Dynatrace Monaco CLI обеспечивает общую поддержку любой схемы Settings 2.0, доступной в вашей среде. Monaco — это подход Dynatrace к конфигурации как коду. Он позволяет управлять средой мониторинга через конфигурационные файлы, предлагая ряд функций для оптимизации процесса.

Для получения дополнительной информации см. Обзор конфигурации как кода через Monaco.
