Вы можете запустить полностью автоматизированное устранение неполадок OneAgent для сред Dynatrace SaaS и Managed, начиная с версии Dynatrace 1.208.

Рабочий процесс позволяет вам:

* Автоматически определить проблему, связанную с OneAgent, в высокодинамичных средах в определённый момент времени
* Легко собрать диагностические данные для конкретной сущности и автоматически получить возможные решения для обнаруженных аномалий
* Быстро решить распространённые проблемы самостоятельно, сократив время, затрачиваемое на диагностику
* Напрямую предоставить службе поддержки Dynatrace все необходимые детали для диагностики проблемы

Диагностика через командную строку

Если у вас нет доступа к Dynatrace или вы хотите автоматизировать сбор диагностических данных с помощью скриптов, вы можете использовать команду `oneagentctl` для сбора части полных диагностических данных непосредственно на хосте, где установлен OneAgent. Дополнительные сведения см. в разделе [Создание архива поддержки](../../../../ingest-from/dynatrace-oneagent/oneagent-configuration-via-command-line-interface.md#create-support-archive "Узнайте, как выполнять некоторые задачи настройки OneAgent без необходимости переустановки.").

## Требования

* Среда Dynatrace SaaS: версия Dynatrace 1.202+
* Среда Dynatrace Managed: версия Dynatrace 1.208+
* Разрешение среды **View sensitive request data**

## Автоматический анализ

Эта процедура описывает стандартный процесс: Dynatrace собирает диагностические данные для хоста или процесса и сразу анализирует их.

Если вы предпочитаете собрать и просмотреть данные перед их отправкой в Dynatrace для анализа вручную, см. раздел [Сбор и локальный просмотр](#collect-and-review-locally).

1. Перейдите в ![Hosts](https://dt-cdn.net/images/hosts-512-59f5d2dd7f.png "Hosts") **Hosts Classic**.
2. Выберите хост, для которого требуется устранить неполадки.
3. Запустите диагностику на уровне хоста или процесса.

   * Уровень хоста: на странице хоста откройте меню обзора (**…**) и выберите **Run OneAgent diagnostics**.
     Пример выбора пункта меню

     ![OneAgent diagnostics: menu item](https://dt-cdn.net/images/diagnostics-menu-item2-950-0243641987.png)
   * Уровень процесса: на странице хоста откройте страницу процесса, а затем выберите **Run OneAgent diagnostics** в меню обзора (**…**) на странице процесса.
4. На странице **Run Dynatrace OneAgent diagnostics** кратко опишите, что работает не так, как ожидалось, с вашей точки зрения.

   Пример страницы диагностики OneAgent

   ![OneAgent diagnostics: run](https://dt-cdn.net/images/run-diagnostics-1548-2f9988d21a.png)
5. Необязательно По умолчанию для анализа собираются данные за 24 часа (1 день). Если вам нужно больше данных, выберите ссылку **Advanced options**, измените количество дней и выберите **Apply**.
6. Выберите **Start analysis**.

### Что происходит дальше

Dynatrace выполняет следующее:

* Собирает диагностические данные за последние 24 часа (если вы не изменили значение по умолчанию) для затронутого хоста или процесса
* Сохраняет собранные диагностические данные
* Загружает диагностические данные в корзину S3 в регионе AWS вашей среды для дальнейшего анализа

Столбец **State** описывает текущую фазу процесса.

**State** не обновляется автоматически. Выберите **Refresh**, чтобы проверить изменение состояния.

Collecting (Сбор данных)

Идёт сбор данных.
Во время сбора данных вы можете:

* **Refresh** — обновить страницу для проверки прогресса.
* **Cancel** — отменить сбор диагностических данных.

Collected (Данные собраны)

Dynatrace завершил сбор диагностических данных.
После сбора данных вы можете:

* **Analyze** — отправить собранные данные в Dynatrace для анализа.
* **Download** — скачать собранные данные локально для проверки.
* **Delete** — удалить запись, включая собранные диагностические данные.

Sending in progress (Отправка)

Диагностические данные передаются в Dynatrace для анализа.
Во время отправки данных вы можете:

* **Refresh** — обновить страницу для проверки прогресса.
* **Download** — скачать собранные диагностические данные.
* **Delete** — удалить запись, включая собранные диагностические данные.

Sent to Dynatrace cloud (Отправлено в облако Dynatrace)

Диагностические данные переданы в Dynatrace для анализа.

Analyzing (Анализ)

Dynatrace анализирует диагностические данные.
Во время анализа данных вы можете:

* **Refresh** — обновить страницу для проверки прогресса.
* **Download** — скачать собранные диагностические данные.
* **Delete** — удалить запись, включая собранные диагностические данные.

Analyzed (Анализ завершён)

Анализ завершён. Количество связанных предупреждений отображается в скобках.
После анализа вы можете:

* **Download** — скачать собранные диагностические данные.
* **Delete** — удалить запись, включая собранные диагностические данные.

Delete in progress (Удаление)

Диагностические данные удаляются. Во время удаления данных вы можете:

* **Refresh** — обновить страницу для проверки прогресса.

Deleted (Удалено)

Диагностические данные удалены. Dynatrace сохраняет только небольшой набор информации о том, кто, когда, где и зачем собирал диагностические данные.

Canceled (Отменено)

Процесс диагностики был отменён вручную до его завершения.

### Просмотр результатов анализа Dynatrace

Когда анализ завершён, Dynatrace отправляет результаты обратно в вашу среду. Если найдено потенциальное решение, Dynatrace отображает его в разделе **Alerts**.

## Сбор и локальный просмотр

Эта процедура описывает, как собрать диагностические данные для хоста или процесса локально. Используйте этот вариант, если вы предпочитаете собрать и просмотреть данные перед их отправкой в Dynatrace для анализа вручную.

Если вместо этого вы хотите собрать данные и автоматически отправить их в Dynatrace для анализа, см. раздел [Автоматический анализ](#analyze-automatically).

1. Перейдите в ![Hosts](https://dt-cdn.net/images/hosts-512-59f5d2dd7f.png "Hosts") **Hosts Classic**.
2. Выберите хост, для которого требуется устранить неполадки.
3. Запустите диагностику на уровне хоста или процесса.

   * Уровень хоста: на странице хоста откройте меню обзора (**…**) и выберите **Run OneAgent diagnostics**.
     Пример

     ![OneAgent diagnostics: menu item](https://dt-cdn.net/images/diagnostics-menu-item2-950-0243641987.png)
   * Уровень процесса: на странице хоста откройте страницу процесса, а затем выберите **Run OneAgent diagnostics** в меню обзора (**…**) на странице процесса.
4. На странице **Run Dynatrace OneAgent diagnostics** кратко опишите, что работает не так, как ожидалось, с вашей точки зрения.

   Пример страницы диагностики OneAgent

   ![OneAgent diagnostics: run](https://dt-cdn.net/images/run-diagnostics-1548-2f9988d21a.png)
5. Выберите ссылку **Advanced options**.
6. Выберите **and store locally**.

   * Здесь же вы можете изменить количество дней сбора данных (по умолчанию = `1 day`).
7. Выберите **Apply**.
8. Выберите **Start collection**, чтобы собрать диагностические данные и сохранить их локально.

Теперь Dynatrace:

* Собирает диагностические данные за последние 24 часа (если вы не изменили значение по умолчанию) для затронутого хоста или процесса
* Сохраняет собранные диагностические данные

Столбец **State** описывает текущую фазу процесса.

**State** не обновляется автоматически. Выберите **Refresh**, чтобы проверить изменение состояния.

Collecting (Сбор данных)

Идёт сбор данных.
Во время сбора данных вы можете:

* **Refresh** — обновить страницу для проверки прогресса.
* **Cancel** — отменить сбор диагностических данных.

Collected (Данные собраны)

Dynatrace завершил сбор диагностических данных.
После сбора данных вы можете:

* **Analyze** — отправить собранные данные в Dynatrace для анализа.
* **Download** — скачать собранные данные локально для проверки.
* **Delete** — удалить запись, включая собранные диагностические данные.

### Что делать с собранными данными

Теперь, когда данные собраны, вы можете:

* **Download** — скачать собранные данные.

  + Вы можете просмотреть данные. См. раздел [Содержимое диагностических данных](#contents) для обзора содержимого загрузки.
  + Вы можете добавить данные к своему обращению в службу поддержки.
* **Analyze** — проанализировать данные.
* **Delete** — удалить запись, включая собранные диагностические данные.

### Устранение неполадок OneAgent в изолированных средах Dynatrace Managed

В изолированной (air-gapped) среде Dynatrace Managed:

1. Используйте опцию **Store locally** в разделе **Advanced options**, как описано выше.
2. После сбора диагностических данных вы можете добавить данные к своему обращению в службу поддержки.
3. Dynatrace затем может получить диагностические данные из вашего обращения, проанализировать их и предоставить автоматическую обратную связь службе поддержки Dynatrace об обнаруженных аномалиях.

На протяжении всего этого процесса применяются и регистрируются строгие меры защиты конфиденциальности данных.

## Обзор диагностики OneAgent

Для обзора всех запусков диагностики OneAgent в вашей среде

1. Перейдите в ![Hosts](https://dt-cdn.net/images/hosts-512-59f5d2dd7f.png "Hosts") **Hosts Classic**.
2. Откройте меню обзора (**…**) и выберите **OneAgent diagnostics overview**.

Страница **Dynatrace OneAgent diagnostics overview** отображает все действия по диагностике OneAgent в вашей среде. Разверните любую запись, чтобы увидеть подробности. Если данные не были удалены (все диагностические данные автоматически удаляются через 30 дней), вы можете скачать или удалить их.

## Конфиденциальность данных

Для соблюдения региональных требований по защите данных и конфиденциальности Dynatrace выполняет следующее:

* Маскирует некоторые персональные данные перед сохранением архива поддержки в Cassandra и загрузкой его в корзину AWS S3. Например, IBAN и учётные данные URI заменяются на `<masked>`. Однако некоторые персональные данные могут остаться незамаскированными.
* Записывает сообщения в журнал аудита при создании, анализе, доступе и удалении архивов поддержки для обеспечения прозрачности использования архивов.
* Предоставляет доступ к архивам поддержки OneAgent только пользователям, имеющим [разрешение среды](../../../../manage/identity-access-management/permission-management/role-based-permissions.md#environment "Разрешения на основе ролей") **View sensitive request data**.
* Автоматически удаляет все диагностические данные через 30 дней после их сбора.

  Это относится к данным в вашей среде Dynatrace. Вы также можете удалить собранные диагностические данные раньше.

## Устранение неполадок

Статус: `Collecting of the diagnostic data wasn't possible within 20 minutes.`

* Если Dynatrace не может собрать диагностические данные в течение 20 минут, система автоматически повторяет попытку.
* Если повторная попытка также не удалась, мы рекомендуем связаться с экспертом по продукту Dynatrace через чат.

`State` выглядит зависшим.

**State** не обновляется автоматически. Выберите **Refresh**, чтобы проверить изменение состояния.

## Часто задаваемые вопросы

Могу ли я получить доступ к S3 напрямую или использовать собственный S3?

Нет, вы не можете получить доступ к S3 напрямую или использовать свой собственный.

Диагностика OneAgent загружает диагностические данные в корзину Dynatrace S3, настроенную для среды/кластера компанией Dynatrace. Используемая корзина S3 зависит от расположения среды/кластера.
