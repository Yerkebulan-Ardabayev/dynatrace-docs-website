# Обеспечение соответствия требованиям с помощью Security Posture Management


* Актуальная версия Dynatrace

Early Adopter

В этом руководстве вы узнаете, как Security Posture Management может помочь вам обеспечить соответствие [руководствам по укреплению безопасности и стандартам соответствия нормативным требованиям](../application-security/spm.md#support "Оценивайте, управляйте и принимайте меры по устранению неправильных конфигураций и нарушений руководств по укреплению безопасности и стандартов соответствия нормативным требованиям.").

## Целевая аудитория

Это руководство предназначено для инженеров по безопасности (Security Ops), DevOps, DevSecOps и инженеров по надёжности (SRE).

## Сценарий

* Ваша организация должна следовать лучшим отраслевым практикам или нормативным требованиям.
* Новые рабочие нагрузки постоянно добавляются или удаляются из вашей среды.

## Цель

* Получить немедленное представление об общем состоянии безопасности вашей контролируемой среды.
* Легко обнаруживать и устранять проблемы безопасности и неправильные конфигурации.
* Убедиться, что ваша среда настроена безопасно и эффективно.
* Повысить общую надёжность системы.
* Обеспечить соответствие стандартам безопасности.

## Результат

* Кластеры Kubernetes активно оцениваются с помощью Kubernetes Security Posture Management на соответствие стандартам и лучшим практикам безопасности.
* Неправильные конфигурации и нарушения стандартов обнаруживаются непрерывно.

## Предварительные требования

* Развёртывание Kubernetes Security Posture Management
* Установите ![xSPM](https://dt-cdn.net/images/security-posture-management-highresolution-1024-83a748ecdd.png "xSPM") **Security Posture Management**

  1. Покажите как

  1. В Dynatrace откройте **Hub**.
  2. Найдите ![xSPM](https://dt-cdn.net/images/security-posture-management-highresolution-1024-83a748ecdd.png "xSPM") **Security Posture Management** и нажмите **Install**.

## Начало работы

1. Просмотр результатов

Откройте [![xSPM](https://dt-cdn.net/images/security-posture-management-highresolution-1024-83a748ecdd.png "xSPM") **Security Posture Management**](../xspm.md "Обнаруживайте, управляйте и принимайте меры по выявленным проблемам безопасности и соответствия.") ![xSPM](https://dt-cdn.net/images/security-posture-management-highresolution-1024-83a748ecdd.png "xSPM") и просмотрите:

* [Результаты оценки](../xspm/concepts.md#concept-results "Концепции, специфичные для приложения Dynatrace Security Posture Management.")
* Статус соответствия

  + [Для всей среды](../xspm/assess-coverage.md#standards "Просмотрите охват Security Posture Management ваших систем на одной странице.")
  + [Для конкретных систем](../xspm/assess-coverage.md#systems "Просмотрите охват Security Posture Management ваших систем на одной странице.")

2. Поиск релевантной информации

Используйте фильтрацию и сортировку для получения представления о проблемах в вашей среде.

3. Получение инсайтов

Определите, какие правила актуальны на основе контекстных инсайтов.

4. Определение стратегии соответствия

1. Исправьте проблемы конфигурации для [правил с наивысшим приоритетом](../xspm/concepts.md#concept-severity "Концепции, специфичные для приложения Dynatrace Security Posture Management.") в узком контексте (например, на одном кластере).
2. Контролируйте соответствие и работу в течение некоторого времени.
3. Если всё работает нормально, распространите исправление на другие среды.

5. Определение мониторинга для соответствия

Есть два способа:

В приложении

Через DQL-запросы

Используйте ![xSPM](https://dt-cdn.net/images/security-posture-management-highresolution-1024-83a748ecdd.png "xSPM") **Security Posture Management** для контроля того, что:

1. [Процент соответствия стандарту](../xspm/assess-coverage.md#standards "Просмотрите охват Security Posture Management ваших систем на одной странице.") увеличивается.
2. Общее количество оценённых ресурсов с результатом `Passed` увеличивается.

Для удобства управления и сравнения результатов вы можете [скачать результаты в виде CSV-файла](../xspm/share-findings.md#dwld "Взаимодействуйте с другими приложениями для дополнительных инсайтов и делитесь результатами с заинтересованными сторонами.").

Используйте [![Investigations](https://dt-cdn.net/images/security-investigator-256-93f6c187d9.png "Investigations") **Investigations**](../investigations.md "Объединяйте функциональность Grail для расследований на основе доказательств, включая разрешение инцидентов, анализ корневых причин и поиск угроз.") или [![Notebooks](https://dt-cdn.net/images/notebooks-768-046137830a.webp "Notebooks") **Notebooks**](../../analyze-explore-automate/dashboards-and-notebooks/notebooks.md "Анализируйте, визуализируйте и делитесь инсайтами из данных наблюдаемости — всё в одном совместном настраиваемом рабочем пространстве.") для контроля:

1. Истории результатов по стандарту или правилу, по кластеру или среде.
2. Того, что результат оценки ресурса для определённого набора правил остаётся `Passed`.

   Список примеров DQL на основе событий соответствия, которые вы можете использовать, см. в разделе [Запросы событий соответствия](../threat-observability/dql-examples.md#compliance "Примеры DQL для данных безопасности на базе Grail.").

6. Создание уведомлений

Создайте рабочий процесс для отправки оповещения в выбранный вами канал, если ранее пройденное правило (`Passed`) становится непройденным (`Failed`).
