* Актуальная версия Dynatrace
* Руководство

Early Adopter

В этом руководстве вы узнаете, как [Security Posture Management](/docs/secure/application-security/spm "Оценивайте, управляйте и принимайте меры по устранению неправильных конфигураций и нарушений руководств по укреплению безопасности и стандартов соответствия нормативным требованиям.") может помочь вам обеспечить соответствие [руководствам по укреплению безопасности и стандартам соответствия нормативным требованиям](/docs/secure/application-security/spm#support "Оценивайте, управляйте и принимайте меры по устранению неправильных конфигураций и нарушений руководств по укреплению безопасности и стандартов соответствия нормативным требованиям.").

## Целевая аудитория

Это руководство предназначено для инженеров по безопасности (Security Ops), DevOps, DevSecOps и инженеров по надёжности (SRE).

## Сценарий

* Ваша организация должна следовать лучшим отраслевым практикам или нормативным требованиям.
* Новые рабочие нагрузки постоянно добавляются или удаляются из вашей среды.

## Цель

* Получить немедленное представление об общем состоянии безопасности вашей контролируемой среды.
* Легко обнаруживать и устранять проблемы безопасности и неправильные конфигурации.
* Убедиться, что ваша среда настроена безопасно и эффективно.
* Повысить общую надёжность системы.
* Обеспечить соответствие стандартам безопасности.

## Результат

* Кластеры Kubernetes активно оцениваются с помощью Kubernetes Security Posture Management на соответствие стандартам и лучшим практикам безопасности.
* Неправильные конфигурации и нарушения стандартов обнаруживаются непрерывно.

## Предварительные требования

* [Развёртывание Kubernetes Security Posture Management](/docs/ingest-from/setup-on-k8s/deployment/security-posture-management "Настройка и включение Security Posture Management в Kubernetes.")
* Установите ![xSPM](https://dt-cdn.net/images/security-posture-management-highresolution-1024-83a748ecdd.png "xSPM") **Security Posture Management**

  1. Покажите как

  1. В Dynatrace откройте [**Hub**](/docs/manage/hub "Информация о Dynatrace Hub.").
  2. Найдите ![xSPM](https://dt-cdn.net/images/security-posture-management-highresolution-1024-83a748ecdd.png "xSPM") **Security Posture Management** и нажмите **Install**.

## Начало работы

1. Просмотр результатов

Откройте [![xSPM](https://dt-cdn.net/images/security-posture-management-highresolution-1024-83a748ecdd.png "xSPM") **Security Posture Management**](/docs/secure/xspm "Обнаруживайте, управляйте и принимайте меры по выявленным проблемам безопасности и соответствия.") ![xSPM](https://dt-cdn.net/images/security-posture-management-highresolution-1024-83a748ecdd.png "xSPM") и просмотрите:

* [Результаты оценки](/docs/secure/xspm/concepts#concept-results "Концепции, специфичные для приложения Dynatrace Security Posture Management.")
* Статус соответствия

  + [Для всей среды](/docs/secure/xspm/assess-coverage#standards "Просмотрите охват Security Posture Management ваших систем на одной странице.")
  + [Для конкретных систем](/docs/secure/xspm/assess-coverage#systems "Просмотрите охват Security Posture Management ваших систем на одной странице.")

2. Поиск релевантной информации

Используйте [фильтрацию и сортировку](/docs/secure/xspm/review-findings "Ищите релевантную информацию для эффективного анализа проблем безопасности и соответствия.") для получения представления о проблемах в вашей среде.

3. Получение инсайтов

Определите, какие правила актуальны [на основе контекстных инсайтов](/docs/secure/xspm/gain-insights "Углубитесь в результаты, которые помогут исправить неправильные конфигурации и нарушения соответствия.").

4. Определение стратегии соответствия

1. Исправьте проблемы конфигурации для [правил с наивысшим приоритетом](/docs/secure/xspm/concepts#concept-severity "Концепции, специфичные для приложения Dynatrace Security Posture Management.") в узком контексте (например, на одном кластере).
2. Контролируйте соответствие и работу в течение некоторого времени.
3. Если всё работает нормально, распространите исправление на другие среды.

5. Определение мониторинга для соответствия

Есть два способа:

В приложении

Через DQL-запросы

Используйте ![xSPM](https://dt-cdn.net/images/security-posture-management-highresolution-1024-83a748ecdd.png "xSPM") **Security Posture Management** для контроля того, что:

1. [Процент соответствия стандарту](/docs/secure/xspm/assess-coverage#standards "Просмотрите охват Security Posture Management ваших систем на одной странице.") увеличивается.
2. Общее количество оценённых ресурсов с результатом `Passed` увеличивается.

Для удобства управления и сравнения результатов вы можете [скачать результаты в виде CSV-файла](/docs/secure/xspm/share-findings#dwld "Взаимодействуйте с другими приложениями для дополнительных инсайтов и делитесь результатами с заинтересованными сторонами.").

Используйте [![Investigations](https://dt-cdn.net/images/security-investigator-256-93f6c187d9.png "Investigations") **Investigations**](/docs/secure/investigations "Объединяйте функциональность Grail для расследований на основе доказательств, включая разрешение инцидентов, анализ корневых причин и поиск угроз.") или [![Notebooks](https://dt-cdn.net/images/notebooks-768-046137830a.webp "Notebooks") **Notebooks**](/docs/analyze-explore-automate/dashboards-and-notebooks/notebooks "Анализируйте, визуализируйте и делитесь инсайтами из данных наблюдаемости — всё в одном совместном настраиваемом рабочем пространстве.") для контроля:

1. Истории результатов по стандарту или правилу, по кластеру или среде.
2. Того, что результат оценки ресурса для определённого набора правил остаётся `Passed`.

   Список примеров DQL на основе событий соответствия, которые вы можете использовать, см. в разделе [Запросы событий соответствия](/docs/secure/threat-observability/dql-examples#compliance "Примеры DQL для данных безопасности на базе Grail.").

6. Создание уведомлений

[Создайте рабочий процесс](/docs/analyze-explore-automate/workflows/actions "Используйте готовые действия Dynatrace для ваших рабочих процессов и интегрируйте Dynatrace с системами третьих сторон.") для отправки оповещения в выбранный вами канал, если ранее пройденное правило (`Passed`) становится непройденным (`Failed`).
