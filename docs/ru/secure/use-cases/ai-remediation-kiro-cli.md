## Обзор

Не каждую облачную ошибку конфигурации нужно исправлять. Dynatrace помогает SRE-инженерам сосредоточиться на важном, проверяя облачные ошибки конфигурации с использованием контекста среды выполнения и оптимизируя устранение с помощью [Kiro CLI](https://kiro.dev/cli/). Этот сценарий использования демонстрирует, как приоритизировать и устранять облачные ошибки конфигурации, влияющие на производственные приложения. Он поддерживает два пути реализации в зависимости от того, где и как вы хотите сортировать проблемы -- оба работают на основе интеграций Dynatrace и Kiro CLI.

## Проблема

В сложных облачных средах количество находок высокой и критической важности, которые необходимо обработать, очень велико. SRE-инженеры не могут обработать их все и нуждаются в эффективном способе устранения тех ошибок конфигурации, которые создают риски для их сред и приводят к проблемам с соответствием требованиям. Облачные ошибки конфигурации могут не оказывать прямого влияния на производственные приложения. Для более эффективной сортировки и устранения необходима дополнительная валидация и контекст.

## Решение

Dynatrace интегрируется с продуктами AWS, такими как [AWS Security Hub](https://aws.amazon.com/de/security-hub/), чтобы обогатить их находки дополнительным контекстом среды выполнения и проверить их влияние на производственную среду.

* Подтвержденные находки требуют немедленного исправления, которое можно автоматизировать с помощью Kiro CLI.
* Неподтвержденные находки можно подавить, чтобы убрать их из списка активных проблем.

Это снижает шум и нагрузку на SRE-инженеров, обеспечивая устранение только релевантных ошибок конфигурации.

Вы можете реализовать это решение двумя способами в зависимости от того, где вы хотите сортировать облачные ошибки конфигурации:

* [**Сортировка на основе Kiro CLI**](#kiro-cli-triaging): использует Kiro CLI в качестве основной точки взаимодействия, подключенной к серверам Dynatrace и (нативно) AWS MCP. Пользователи взаимодействуют с Kiro CLI, чтобы сначала получить основные облачные ошибки конфигурации, контекстуализировать и приоритизировать их с помощью контекста Dynatrace, а затем устранить.
* [**Сортировка на основе Dynatrace**](#dynatrace-triaging): использует ![Workflows](https://dt-cdn.net/images/workflows-1024-b5708f3cf9.webp "Workflows") **Workflows** в качестве основного механизма сортировки с действием генеративного ИИ Dynatrace Intelligence в рабочем процессе для обобщения результатов в тикет Jira. Остальная часть процесса выполняется в Kiro CLI, как в первом сценарии.

* [Запросите доступ к Dynatrace MCP Server](../../../common/whats-new/preview-releases.md#mcp-server "Узнайте о наших Preview-релизах и о том, как принять в них участие.").

## Сортировка на основе Kiro CLI

В этом сценарии SRE-инженер взаимодействует с Kiro CLI для выполнения действий по валидации и устранению с помощью Dynatrace MCP.

### Как это работает (сортировка на основе Kiro CLI)

![Сортировка на основе Kiro CLI](https://dt-cdn.net/images/dd7782f2-aaa7-47db-a2ba-be0d9701e734-1920-c3a5adeea6.png)

1. Обнаружение

AWS Security Hub выявляет облачные ошибки конфигурации.

2. Проверка

* SRE-инженер взаимодействует с Kiro CLI для извлечения основных находок.
* Агент Dynatrace, вызываемый Kiro CLI, использует MCP-сервер Dynatrace для проверки того, влияют ли эти находки на производственные приложения.

3. Устранение

* SRE-инженер устраняет подтвержденные находки через Kiro CLI; остальные подавляются.
* Находки разрешаются и проверяются в AWS Security Hub.

### Начало работы (сортировка на основе Kiro CLI)

Для начала выполните следующие шаги.

1. Скачайте файл [`dynatrace-security-agent.json`](https://dt-url.net/ov0301d) и поместите его в `.kiro/agents/`. Он содержит конфигурацию Dynatrace MCP с соответствующим платформенным токеном Dynatrace API и ссылку на файл инструкций.
   Убедитесь, что вы заменили:

   * `<DYNATRACE_TENANT>` на вашу среду, например `mytenant.apps.dynatrace.com`.
   * `<DYNATRACE_PLATFORM_TOKEN>` на токен Dynatrace, сгенерированный в разделе [предварительные требования](#dt).
2. Скачайте файл [`instructions.md`](https://dt-url.net/cp2300s) и поместите его в тот же каталог `.kiro/agents/`, рядом с файлом конфигурации агента.

## Сортировка на основе Dynatrace

В этом сценарии начальная валидация и сортировка автоматизированы в Dynatrace Workflows, а SRE-инженер затем работает с тикетами Jira для выполнения автоматизированного устранения с помощью Kiro CLI.

### Как это работает (сортировка на основе Dynatrace)

![Сортировка на основе Dynatrace](https://dt-cdn.net/images/kiro-cli-2-2088-7ca80c5d12.png)

1. Обнаружение

AWS Security Hub выявляет облачные ошибки конфигурации.

2. Проверка

* Интеграция AWS Security Hub загружает находки в Dynatrace.
* Рабочий процесс Dynatrace анализирует новые находки критической/высокой степени риска и выполняет автоматическую проверку с использованием действия генеративного ИИ Dynatrace Intelligence в рабочем процессе.

3. Устранение

* Тикет Jira создается автоматически с обобщением результатов проверки.
* SRE-инженер использует Kiro CLI для работы с находками Jira. Подтвержденные находки устраняются, а неподтвержденные могут быть подавлены.
* Находки разрешаются в AWS Security Hub и больше не отображаются в панели основных находок в Dynatrace.

### Начало работы (сортировка на основе Dynatrace)

Для начала выполните следующие шаги.

1. Скачайте файл [`dynatrace-security-agent.json`](https://dt-url.net/ov0301d) и поместите его в `.kiro/agents/`. Он содержит конфигурацию Dynatrace MCP с соответствующим платформенным токеном Dynatrace API и ссылку на файл инструкций.
   Убедитесь, что вы заменили:

   * `<DYNATRACE_TENANT>` на вашу среду, например `mytenant.apps.dynatrace.com`.
   * `<DYNATRACE_PLATFORM_TOKEN>` на токен Dynatrace, сгенерированный в разделе [предварительные требования](#dt).
2. Скачайте файл [`instructions.md`](https://dt-url.net/cp2300s) и поместите его в тот же каталог `.kiro/agents/`, рядом с файлом конфигурации агента.
3. Необязательно. Настройте удаленный MCP для Jira как часть агента Dynatrace или как глобальную настройку Kiro (`.kiro/settings/mcp.json`). Подробнее см. [Model context protocol (MCP)](https://kiro.dev/docs/cli/mcp/).
4. Разверните [рабочий процесс сортировки Dynatrace для находок AWS Security Hub](https://dt-url.net/a9032pe).

* Отредактируйте конфигурацию, чтобы установить нужные значения параметров Jira (такие как ключ проекта, тип задачи, ответственный, автор и метки), которые будут использоваться действием создания тикета Jira.
* Настройте периодическое выполнение рабочего процесса по расписанию.
