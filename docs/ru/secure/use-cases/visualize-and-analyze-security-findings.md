# Визуализация и анализ результатов проверки безопасности


* Последняя версия Dynatrace
* Учебное руководство
* Обновлено 07 октября 2025 г.

Организации используют множество продуктов и инструментов безопасности, которые формируют результаты проверки безопасности в различных форматах данных. Разрозненный доступ к этим данным существенно усложняет работу аналитиков безопасности, вынужденных тратить значительные ручные усилия на формирование единой картины состояния безопасности.

В этом контексте вы можете:

* Получать результаты проверки безопасности от ваших инструментов безопасности (см. [Интеграции безопасности](../threat-observability/security-events-ingest.md "Получение внешних данных безопасности в Grail.")) или события обнаружения уязвимостей из процессов, отслеживаемых Dynatrace (см. [События сторонних библиотек](../vulnerabilities/concepts.md#tpv-events "Концепции, специфичные для приложения Dynatrace Vulnerabilities.")), и сопоставлять их со [Словарём семантики Dynatrace](https://dt-url.net/3q03pb0), что обеспечивает единообразный доступ к событиям из различных инструментов с помощью DQL.
* Просматривать и анализировать результаты проверки безопасности по продуктам и инструментам с помощью наших дашбордов, которые также могут служить хорошей основой для дальнейшей визуальной кастомизации в соответствии с требованиями вашей организации к анализу состояния безопасности и созданию отчётов.
* [Запрашивать](../../platform/grail/dynatrace-query-language.md "Использование Dynatrace Query Language.") поступившие данные в наших специализированных приложениях.

## Целевая аудитория

Аналитики безопасности и руководители, ответственные за анализ и отчётность о состоянии безопасности организации.

Ключевые сценарии использования:

* Получение единого представления всех результатов проверки безопасности
* Приоритизация результатов проверки безопасности по продуктам
* Определение наиболее пострадавших ресурсов

## Предварительные условия

В зависимости от источника поступления данных выполните соответствующую настройку:

* **Сторонние продукты безопасности**: [Настройте поддерживаемую интеграцию](../threat-observability/security-events-ingest.md "Получение внешних данных безопасности в Grail.").
* **Окружения, отслеживаемые Dynatrace**: [Включите обнаружение уязвимостей сторонних компонентов](../application-security/vulnerability-analytics.md#tpv-detection "Мониторинг, визуализация, анализ и устранение уязвимостей сторонних и кодовых компонентов, отслеживание прогресса устранения и создание правил мониторинга.") для получения событий обнаружения уязвимостей из сторонних библиотек.

## Начало работы

1. Визуализация

1. Откройте [![Dashboards](https://dt-cdn.net/images/dashboards-512-b1f1e9690b.png "Dashboards") **Dashboards**](../../analyze-explore-automate/dashboards-and-notebooks/dashboards-new.md "Создание интерактивных, настраиваемых представлений для визуализации, анализа и обмена данными наблюдаемости в режиме реального времени.") и перейдите в раздел **Ready-made**.

   Готовые дашборды доступны для интеграций со сторонними инструментами. События обнаружения уязвимостей из окружений, отслеживаемых Dynatrace, доступны через DQL, но в настоящее время не включены в эти дашборды.
2. Найдите и выберите **Security findings** для нужной интеграции.

Пример результата:

![dashboard sample result](https://dt-cdn.net/images/2025-05-07-15-06-056-1903-93aa46287c.png)

2. Анализ

Откройте [![Notebooks](https://dt-cdn.net/images/notebooks-768-046137830a.webp "Notebooks") **Notebooks**](../../analyze-explore-automate/dashboards-and-notebooks/notebooks.md "Анализ, визуализация и обмен аналитическими данными о наблюдаемости — всё в одном совместном настраиваемом рабочем пространстве.") или [![Investigations](https://dt-cdn.net/images/security-investigator-256-93f6c187d9.png "Investigations") **Investigations**](../investigations.md "Объединение функций Grail для расследований на основе доказательств, включая устранение инцидентов, анализ первопричин и поиск угроз."), чтобы [запросить](../../platform/grail/dynatrace-query-language.md "Использование Dynatrace Query Language.") поступившие данные, используя формат данных из [Словаря семантики](https://dt-url.net/3q03pb0).

Для лучшего понимания того, как строить запросы, см. [Примеры DQL-запросов для поступивших событий](../threat-observability/dql-examples.md#ingested "Примеры DQL для данных безопасности на основе Grail.").
