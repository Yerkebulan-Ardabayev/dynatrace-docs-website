# Обогащение индикаторов угроз с помощью AbuseIPDB

* Последняя версия Dynatrace
* Практическое руководство

Обогащайте индикаторы угроз с помощью AbuseIPDB и анализируйте их в Dynatrace.

## Начало работы

### Обзор

Интеграция Dynatrace с [AbuseIPDB](https://www.abuseipdb.com/) улучшает оповещения и расследования обнаружений, предоставляя ценный контекст для анализа угроз. Это помогает организациям бороться с онлайн-злоупотреблениями, включая кибератаки, рассылку спама и другие вредоносные действия.

Обогащая наблюдаемость данными о репутации из AbuseIPDB, вы можете проводить более эффективные расследования безопасности, автоматизировать классификацию оповещений и снижать шум за счёт приоритизации с учётом угроз. Это оптимизирует реагирование на инциденты и повышает общий уровень безопасности.

### Сценарии использования

После настройки интеграции с AbuseIPDB вы можете использовать данные об угрозах для обогащения наблюдаемых объектов, таких как IP-адреса.

### Требования

Ниже приведены требования для [AbuseIPDB](#abuseipdb) и [Dynatrace](#dt).

#### Требования AbuseIPDB

Зарегистрируйтесь в AbuseIPDB и создайте ключ API v2.

#### Требования Dynatrace

Необходимы следующие разрешения IAM:

* `app-engine:apps:run`
* `app-settings:objects:read`
* `document:documents:read`
* `settings:objects:read`
* `storage:system:read`
* `security-intelligence:enrichments:run`

Для выполнения действия обогащения в рабочем процессе все перечисленные выше разрешения также должны быть включены в ![Workflows](https://dt-cdn.net/images/workflows-1024-b5708f3cf9.webp "Workflows") **Workflows**.

1. Перейдите в меню настроек в правом верхнем углу ![Workflows](https://dt-cdn.net/images/workflows-1024-b5708f3cf9.webp "Workflows") **Workflows** и выберите **Authorization settings**.
2. В разделе **Secondary permissions** найдите и выберите перечисленные выше разрешения.
3. Выберите **Save**.

## Активация и настройка

1. В Dynatrace откройте [**Hub**](/docs/manage/hub "Информация о Dynatrace Hub.").
2. Найдите **AbuseIPDB** и выберите **Install**.
3. Выберите **Set up**, затем выберите **Configure new connection**.
4. Следуйте инструкциям на экране для настройки подключения с использованием ключа API, полученного в разделе **Предварительные условия**.

   Разрешённые исходящие подключения автоматически расширяются на `api.abuseipdb.com`.

   1. Как настроить исходящие подключения

   1. В **Settings** перейдите в **General** > **External requests**.
   2. В разделе **Allowlist** выберите **New host pattern** и добавьте домен.
5. Протестируйте подключение, чтобы убедиться в правильности конфигурации, и сохраните его.

## Подробности

### Как это работает

![Механизм AbuseIPDB](https://dt-cdn.net/images/image-20250325-091933-2537-ee7bd898c5.png)

1. Установка и настройка приложения

Интеграция Dynatrace с AbuseIPDB — это приложение, которое можно установить из [**Hub**](/docs/manage/hub "Информация о Dynatrace Hub.").

Приложение предоставляет действие рабочего процесса для обогащения наблюдаемых объектов в ![Workflows](https://dt-cdn.net/images/workflows-1024-b5708f3cf9.webp "Workflows") **Workflows**.

Для предотвращения случайных изменений или удалений в разных средах настройка подключения теперь включает контроль доступа на основе владельца. Это обеспечивает надёжную автоматизацию, предотвращает неожиданную потерю конфигурации и соответствует требованиям минимального доступа.

Подробнее о совместном использовании и разрешениях см. в разделе [Контроль доступа для коннекторов](/docs/analyze-explore-automate/workflows/actions/access-control "Отображение, просмотр, создание и совместное использование подключений для коннекторов Dynatrace.").

2. Обогащение наблюдаемых объектов

Различные приложения-потребители могут выполнять обогащение наблюдаемых объектов по запросу, например, через [действие рабочего процесса](#workflow).

Dynatrace обращается к AbuseIPDB для выполнения обогащения наблюдаемого объекта.

Поля геолокации в результатах обогащения получены от провайдера и могут отличаться от геолокации, используемой в Dynatrace.
Подробнее см. в разделе [FAQ: Различия в геолокации](#ti-geo).

3. Использование данных об угрозах

Контекст данных об угрозах отображается в приложениях-потребителях или в ![Workflows](https://dt-cdn.net/images/workflows-1024-b5708f3cf9.webp "Workflows") **Workflows**, помогая вам принимать более обоснованные решения.

## Часто задаваемые вопросы

### Какие типы наблюдаемых объектов поддерживаются в настоящее время?

Поддерживаемые наблюдаемые объекты: IP-адреса (другие типы появятся в ближайшее время).

### Как эта интеграция повлияет на квоты моего API AbuseIPDB?

Для каждого нового обогащения наблюдаемого объекта выполняется один вызов API.

### Почему геолокация в результатах обогащения отличается от геолокации в Dynatrace?

Поля геолокации в результатах обогащения (такие как `geo.country.iso_code`, `geo.country.name`, а также город/координаты при наличии) предоставляются непосредственно внешним провайдером (в данном случае AbuseIPDB).

Эти значения отражают данные геолокации AbuseIPDB и могут отличаться от геолокации, используемой в функциях Dynatrace (таких как мониторинг реальных пользователей или геолокация на уровне платформы).

Различия могут возникать из-за разных баз данных, циклов обновления или правил сопоставления.
