# Обогащение индикаторов угроз с помощью VirusTotal


* Latest Dynatrace
* How-to guide
* Updated on Feb 23, 2026

Обогащайте индикаторы угроз с помощью VirusTotal и анализируйте их в Dynatrace.

## Начало работы

### Обзор

Интеграция Dynatrace с [VirusTotal](https://www.virustotal.com) привносит контекст разведки угроз в оповещения и расследования обнаружений, помогая организациям бороться со злоупотреблениями в сети, такими как кибератаки, спам и другие вредоносные действия.

Благодаря обогащению индикаторов репутацией, сформированной на основе информации об угрозах от VirusTotal, вы можете проводить более эффективные расследования безопасности и автоматизировать сортировку оповещений, снижая шум за счет приоритизации с учетом угроз.

### Сценарии использования

### Требования

Ниже приведены требования для [VirusTotal](#virustotal) и [Dynatrace](#dt).

#### Требования VirusTotal

Зарегистрируйтесь в VirusTotal и создайте ключ API v3.

#### Требования Dynatrace

Необходимы следующие разрешения IAM:

* `app-engine:apps:run`
* `app-settings:objects:read`
* `document:documents:read`
* `settings:objects:read`
* `storage:system:read`
* `security-intelligence:enrichments:run`

Для выполнения действия обогащения в рабочем процессе все указанные выше разрешения также должны быть включены в ![Workflows](https://dt-cdn.net/images/workflows-1024-b5708f3cf9.webp "Workflows") **Workflows**.

1. Перейдите в меню настроек  в правом верхнем углу ![Workflows](https://dt-cdn.net/images/workflows-1024-b5708f3cf9.webp "Workflows") **Workflows** и выберите **Authorization settings**.
2. В разделе **Secondary permissions** найдите и выберите перечисленные выше разрешения.
3. Выберите **Save**.

## Активация и настройка

1. В Dynatrace откройте [**Hub**](../../../manage/hub.md "Информация о Dynatrace Hub.").
2. Найдите **VirusTotal** и выберите **Install**.
3. Выберите **Set up**, затем выберите **Configure new connection**.
4. Следуйте инструкциям на экране для настройки подключения с использованием ключа API, полученного в разделе [Предварительные требования](#prereq).

   Разрешенные исходящие соединения автоматически дополняются доменом `www.virustotal.com`.

   1. Как настроить исходящие соединения

   1. В **Settings** перейдите в **General** > **External requests**.
   2. В разделе **Allowlist** выберите **New host pattern** и добавьте домен.
5. Протестируйте подключение, чтобы убедиться в правильности конфигурации, и сохраните его.

## Как это работает

![how it works](https://dt-cdn.net/images/image-20250418-083906-2216-9527a8fea8.png)

1. Установка и настройка приложения

Интеграция Dynatrace с VirusTotal -- это приложение, которое можно установить из [**Hub**](../../../manage/hub.md "Информация о Dynatrace Hub.").

Приложение предоставляет действие рабочего процесса для обогащения индикаторов в ![Workflows](https://dt-cdn.net/images/workflows-1024-b5708f3cf9.webp "Workflows") **Workflows**.

Для предотвращения случайных изменений или удалений в разных средах настройка подключения теперь включает контроль доступа на основе владельца. Это обеспечивает надежную автоматизацию, исключает неожиданную потерю конфигурации и соответствует требованиям минимального доступа.

Подробнее о совместном использовании и разрешениях см. [Контроль доступа для Connectors](../../../analyze-explore-automate/workflows/actions/access-control.md "Отображение, просмотр, создание и предоставление общего доступа к подключениям для Dynatrace Connectors.").

2. Обогащение индикаторов

Различные приложения-потребители могут выполнять обогащение индикаторов по запросу, например, через [действие рабочего процесса](#workflow).

Dynatrace обращается к VirusTotal для выполнения обогащения индикаторов.

Поля геолокации в результатах обогащения предоставляются провайдером и могут отличаться от геолокации, используемой в Dynatrace.
Подробнее см. [FAQ: Различия в геолокации](#ti-geo).

3. Использование данных разведки угроз

Контекст разведки угроз отображается в приложениях-потребителях или в ![Workflows](https://dt-cdn.net/images/workflows-1024-b5708f3cf9.webp "Workflows") **Workflows**, помогая вам принимать более обоснованные решения.

## Часто задаваемые вопросы

### Какие типы индикаторов в настоящее время поддерживаются?

Поддерживаемые типы индикаторов: IP-адреса (поддержка других типов ожидается в ближайшее время).

### Как эта интеграция повлияет на мои квоты API VirusTotal?

Для каждого нового обогащения индикатора выполняется один вызов API.

### Почему геолокация в результатах обогащения отличается от данных Dynatrace?

Поля геолокации в результатах обогащения (такие как `geo.country.iso_code`, `geo.country.name`, а также город/координаты при их наличии) предоставляются непосредственно внешним провайдером (в данном случае VirusTotal).

Эти значения отражают данные геолокации VirusTotal и могут отличаться от геолокации, используемой в функциях Dynatrace (таких как Real User Monitoring или геолокация на уровне платформы).

Различия могут возникать из-за использования разных баз данных, циклов обновления или правил сопоставления.
