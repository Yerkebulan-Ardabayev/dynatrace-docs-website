## Начало работы

### Обзор

Интеграция Dynatrace с Qualys позволяет пользователям объединять и контекстуализировать результаты анализа уязвимостей из различных инструментов и продуктов DevSecOps, обеспечивая централизованную приоритизацию, визуализацию и автоматизацию работы с результатами безопасности.

[Платформа Qualys Enterprise TruRisk](https://www.qualys.com/enterprise-trurisk-platform) предлагает ряд продуктов, включая [Vulnerability Management, Detection, & Response (VMDR)](https://www.qualys.com/apps/vulnerability-management-detection-response), который помогает обнаруживать и приоритизировать уязвимости для устранения на хостах. Платформа Dynatrace наблюдает за соответствующими приложениями и сервисами, связанными с этими хостами. Импорт и обогащение результатов анализа уязвимостей контекстом среды выполнения помогает пользователям лучше сосредоточиться на основных рисках, влияющих на их продуктивные приложения.

### Требования

Ниже приведены требования для [Qualys](#qualys) и [Dynatrace](#dt).

#### Требования Qualys

Для аутентификации с помощью имени пользователя и пароля учётная запись пользователя должна иметь следующие разрешения:

* **Модуль Vulnerability Management**

  + Роль пользователя: **Reader** с включённым доступом к API
  + Группы ресурсов: доступ к соответствующим группам ресурсов или ко всем группам ресурсов
* **Модуль Administration** (необходим для импорта журнала аудита):

  + Достаточно роли пользователя типа **Reader**.
  + Для сбора журналов активности роль также должна включать:

    - Включённый доступ к API
    - Доступ к модулю Administration (**Action Log Permissions** > **Action Log Access**)

#### Требования Dynatrace

* ActiveGate версии 1.310+, который должен:

  + Поддерживать фреймворк Extensions 2.0
  + Иметь доступ к конечным точкам API Qualys
* Разрешения: список необходимых разрешений можно найти, перейдя в  **Hub**, выбрав ![Extensions](https://dt-cdn.net/images/dynatrace-extensions-256-9cb05e0f55.png "Extensions") **Extensions** и просмотрев **Technical information**.
* Сгенерируйте токен доступа с областью `openpipeline.events_security` и сохраните его для дальнейшего использования. Подробнее см. [Dynatrace API — Токены и аутентификация](../../../dynatrace-api/basics/dynatrace-api-authentication.md "Узнайте, как пройти аутентификацию для использования Dynatrace API.").

## Активация и настройка

1. В Dynatrace найдите **Qualys** и нажмите **Install**.
2. Следуйте инструкциям на экране для настройки расширения.
3. Проверьте конфигурацию, выполнив следующие запросы в [![Notebooks](https://dt-cdn.net/images/notebooks-768-046137830a.webp "Notebooks") **Notebooks**](../../../analyze-explore-automate/dashboards-and-notebooks/notebooks.md "Анализируйте, визуализируйте и делитесь данными наблюдаемости — всё в одном совместном настраиваемом рабочем пространстве."):

   * Для журналов аудита:

     ```
     fetch logs


     | filter log.source=="Qualys"
     ```
   * Для событий обнаружения:

     ```
     fetch security.events


     | filter dt.system.bucket == "default_securityevents"


     | filter event.provider=="Qualys"


     AND event.type=="VULNERABILITY_FINDING"
     ```
   * Для событий сканирования:

     ```
     fetch security.events


     | filter dt.system.bucket == "default_securityevents"


     | filter event.provider=="Qualys"


     AND event.type=="VULNERABILITY_SCAN"
     ```
4. После установки и начала работы расширения вы можете получить доступ к нему и управлять им в Dynatrace через ![Extensions](https://dt-cdn.net/images/dynatrace-extensions-256-9cb05e0f55.png "Extensions") **Extensions**. Подробнее см. [О расширениях](../../../ingest-from/extensions/concepts.md "Узнайте больше о концепции расширений Dynatrace.").

## Подробности

### Как это работает

![как это работает](https://dt-cdn.net/images/image-20251202-165058-2086-c9122593e9.png)

Интеграция Dynatrace с Qualys — это [расширение](../../../ingest-from/extensions.md "Узнайте, как создавать и управлять расширениями Dynatrace."), работающее на Dynatrace ActiveGate. После включения и настройки расширения Dynatrace для Qualys:

1. Оно периодически собирает результаты безопасности и журналы аудита с помощью [REST API Qualys](https://docs.qualys.com/en/vm/api/scans/index.htm#t=get_started%2Fget_started.htm).
2. Полученные данные импортируются в Dynatrace и сопоставляются с [Семантическим словарём Dynatrace](../../../semantic-dictionary/model/security-events.md "Познакомьтесь с моделями Семантического словаря, связанными с событиями безопасности.").
3. Данные хранятся в бакете `default_securityevents` (подробнее см. [Встроенные бакеты Grail](../../../platform/grail/organize-data.md#built-in-grail-buckets "Информация о модели данных Grail, состоящей из бакетов, таблиц и представлений.")).

## Часто задаваемые вопросы

### Какая модель данных используется для событий безопасности и журналов аудита, поступающих от интеграции с Qualys?

* [**События обнаружения уязвимостей**](../../../semantic-dictionary/model/security-events.md#vulnerability-finding-events "Познакомьтесь с моделями Семантического словаря, связанными с событиями безопасности.") хранят отдельные результаты обнаружения уязвимостей, сообщённые Qualys для затронутых артефактов и компонентов.
* [**События сканирования уязвимостей**](../../../semantic-dictionary/model/security-events.md#vulnerability-scan-events "Познакомьтесь с моделями Семантического словаря, связанными с событиями безопасности.") указывают на охват сканирования для отдельных артефактов.
* [**Журналы аудита**](../../../semantic-dictionary/model/log.md#audit-logs "Познакомьтесь с моделями Семантического словаря, связанными с аналитикой журналов.") представляют журналы активности пользователей в Qualys.

### Какие результаты безопасности Qualys импортируются в Dynatrace?

* Результаты Qualys VMDR импортируются по умолчанию.
* При первом запуске импорта интеграция импортирует все результаты, обновлённые за последние `m` часов, где `m` — интервал первого импорта, настроенный в конфигурации мониторинга.
* Если расширение настроено на импорт данных с интервалом `n` часов, то при каждом запуске расширения будут импортированы все результаты анализа уязвимостей, обновлённые за последние `n` часов.
* Если новые или обновлённые результаты не обнаружены, данные импортированы не будут.

### Какие поля расширения добавляются поверх основных полей событий, импортированных из Qualys?

* Пространство имён `qualys` добавляется для извлечения нескольких специфичных для Qualys атрибутов для удобства пользователя поверх исходного JSON, который хранится в поле `event.original_content`.

Примеры полей:

* `qualys.host.asset_id`: идентификатор, присвоенный всем ресурсам в подписке, который не отображается в VMDR.
* `qualys.host.tracking_method`: способ отслеживания хостового ресурса (например, Cloud Agent, IP, DNS).
* `qualys.detection.first_found`: когда обнаружение было впервые найдено на ресурсе.
* `qualys.detection.last_found`: когда обнаружение было последний раз найдено на ресурсе.
* `qualys.detection.times_found`: сколько раз это обнаружение было найдено.
* `qualys.detection.qds_factors`: карта с подробностями о том, что повлияло на показатель Qualys Detection Score.

### Какие типы ресурсов Qualys поддерживаются Dynatrace для контекстуализации среды выполнения?

`HOST`: все результаты Qualys VMDR, полученные при оценке хостов, сопоставляются со значением HOST в поле object.type, и добавляются пространства имён хоста с соответствующими полями:

* `host.name`: имя хоста (если доступно).
* `host.ip`: IP-адрес хостового ресурса.
* `host.fqdn`: полное доменное имя хостового ресурса (если доступно).

### Как нормализуется оценка рисков для результатов Qualys?

Уровни и оценки рисков Dynatrace сопоставлены с исходными уровнями серьёзности Qualys.

* `dt.security.risk.score` сопоставляется из предоставленной Qualys оценки серьёзности в статические оценки.
* `dt.security.risk.level` сопоставляется из оценки серьёзности Qualys и исходных значений в finding.score.

Qualys Detection Score (QDS) имеет диапазон от 1 до 100. Для сопоставления с 10-балльной шкалой оценки рисков безопасности Dynatrace значение QDS делится на 10.

| `dt.security.risk.score` (сопоставлено из `finding.score`) | `dt.security.risk.level` (сопоставлено из `dt.security.risk.score`) |
| --- | --- |
| 9.0-10.0 | CRITICAL |
| 7.0-8.9 | HIGH |
| 4.0-6.9 | MEDIUM |
| 0.1-3.9 | LOW |
