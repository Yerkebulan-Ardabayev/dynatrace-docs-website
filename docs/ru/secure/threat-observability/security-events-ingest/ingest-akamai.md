Загрузка журналов безопасности и событий Akamai в Dynatrace в качестве событий безопасности.

## Начало работы

### Обзор

Интеграция Dynatrace с [Akamai](https://dt-url.net/3403xsr) позволяет объединять и контекстуализировать результаты анализа безопасности из различных инструментов и продуктов, обеспечивая централизованную приоритизацию, визуализацию и автоматизацию.

Продукты Akamai генерируют события безопасности и обнаруживают подозрительную сетевую активность. Dynatrace наблюдает за сущностями среды выполнения, защищёнными этими продуктами. Загрузка событий безопасности из продуктов Akamai помогает пользователям анализировать эти журналы и результаты в контексте их рабочих сред.

## Активация и настройка

1. В Dynatrace найдите **Akamai** и выберите **Install**.
2. Следуйте инструкциям на экране для настройки расширения.
3. Проверьте конфигурацию, выполнив следующие запросы в [![Notebooks](https://dt-cdn.net/images/notebooks-768-046137830a.webp "Notebooks") **Notebooks**](../../../analyze-explore-automate/dashboards-and-notebooks/notebooks.md "Анализируйте, визуализируйте и делитесь выводами из данных наблюдаемости в едином настраиваемом рабочем пространстве."):

   * Для журналов безопасности:

     ```
     fetch logs


     | filter log.source=="Akamai SIEM"
     ```
   * Для событий обнаружения (если расширение настроено для извлечения событий обнаружения):

     ```
     fetch security.events


     | filter dt.system.bucket == "default_securityevents"


     | filter event.provider=="Akamai"
     ```
4. После установки и запуска расширения вы можете получить к нему доступ и управлять им в Dynatrace через ![Extensions](https://dt-cdn.net/images/dynatrace-extensions-256-9cb05e0f55.png "Extensions") **Extensions**. Подробнее см. [О расширениях](../../../ingest-from/extensions/concepts.md "Узнайте больше о концепции расширений Dynatrace.").

## Подробности

### Как это работает

![как это работает](https://dt-cdn.net/images/image-20250212-154515-3094-1f68916ee3.png)

Интеграция Dynatrace с Akamai представляет собой [расширение](../../../ingest-from/extensions.md "Узнайте, как создавать и управлять расширениями Dynatrace."), работающее на Dynatrace ActiveGate. После включения и настройки расширения Dynatrace Akamai

1. Оно периодически обращается к [Akamai SIEM API](https://dt-url.net/wbe3xyb) и получает события безопасности.
2. Необработанные данные загружаются в Dynatrace в виде журналов. Если настроено извлечение событий безопасности, события обнаружения загружаются дополнительно к журналам и сопоставляются с [семантическим словарём Dynatrace](https://dt-url.net/z1c3xsm).
3. Данные хранятся следующим образом:

   * Журналы хранятся в корзине `default_logs`
   * События безопасности хранятся в корзине `default_securityevents`

   Подробнее см. [Встроенные корзины Grail](../../../platform/grail/organize-data.md#built-in-grail-buckets "Информация о модели данных Grail, состоящей из корзин, таблиц и представлений.").

## Наборы функций

При активации расширения с помощью [конфигурации мониторинга](#monitoring-configuration) вы можете ограничить мониторинг одним из наборов функций. Для корректной работы расширение должно собрать хотя бы одну метрику после активации.

В сильно сегментированных сетях наборы функций могут отражать сегменты вашей среды. Затем при создании конфигурации мониторинга вы можете выбрать набор функций и соответствующую группу ActiveGate, которая может подключаться к данному сегменту.

Все метрики, не отнесённые к какому-либо набору функций, считаются набором по умолчанию и всегда передаются.

Метрика наследует набор функций подгруппы, которая, в свою очередь, наследует набор функций группы. Также набор функций, определённый на уровне метрики, переопределяет набор функций, определённый на уровне подгруппы, который, в свою очередь, переопределяет набор функций, определённый на уровне группы.

self-monitoring

| Название метрики | Ключ метрики | Описание |
| --- | --- | --- |
| Загруженные журналы | sfm.akamai-siem.ingested.logs | Количество записей журнала, загруженных расширением. |
| Загруженные журналы (байты) | sfm.akamai-siem.ingested.logs\_bytes | Объём байтов, загруженных расширением в виде журналов. |
| Загруженные события безопасности | sfm.akamai-siem.ingested.security\_events | Количество событий безопасности, загруженных расширением. |
| Загруженные события безопасности (байты) | sfm.akamai-siem.ingested.security\_events\_bytes | Объём байтов, загруженных расширением в виде событий безопасности. |

## Часто задаваемые вопросы

### Какая модель данных используется для журналов безопасности и событий, поступающих из интеграции Akamai SIEM?

* [Журналы](../../../semantic-dictionary/model/log.md "Ознакомьтесь с моделями семантического словаря, связанными с Log Analytics.") — все данные Akamai SIEM загружаются в виде журналов. Журнал следует базовой схеме для журналов с несколькими соответствующими расширениями пространств имён, такими как [`geo`](../../../semantic-dictionary/fields.md#geolocation "Ознакомьтесь со списком глобальных полей, имеющих чётко определённое семантическое значение в Dynatrace."), [`http`](../../../semantic-dictionary/fields.md#http "Ознакомьтесь со списком глобальных полей, имеющих чётко определённое семантическое значение в Dynatrace.") и [`url`](../../../semantic-dictionary/fields.md#url "Ознакомьтесь со списком глобальных полей, имеющих чётко определённое семантическое значение в Dynatrace.").
* [События обнаружения](../../../semantic-dictionary/model/security-events.md#detection-finding-events "Ознакомьтесь с моделями семантического словаря, связанными с событиями безопасности.")

### Какие поля расширения добавлены поверх основных полей событий, загружаемых из Akamai?

* Пространство имён [`geo`](../../../semantic-dictionary/fields.md#geolocation "Ознакомьтесь со списком глобальных полей, имеющих чётко определённое семантическое значение в Dynatrace.") сопоставляет соответствующую информацию о геолокации субъекта, обнаруженного в журнале.
* Пространство имён [`http`](../../../semantic-dictionary/fields.md#http "Ознакомьтесь со списком глобальных полей, имеющих чётко определённое семантическое значение в Dynatrace.") сопоставляет соответствующие поля HTTP-запроса из отслеживаемой транзакции.
* Пространство имён [`url`](../../../semantic-dictionary/fields.md#url "Ознакомьтесь со списком глобальных полей, имеющих чётко определённое семантическое значение в Dynatrace.") сопоставляет соответствующее веб-приложение/URL, к которому был осуществлён доступ в качестве цели отслеживаемой транзакции.
* Пространство имён `akamai` извлекает несколько специфических для Akamai полей для удобства пользователя поверх исходного содержимого JSON, которое хранится в поле `log.content`.

Некоторые извлечённые поля, которые могут быть полезны:

* `akamai.config.id`
* `akamai.attackdata.*`

### Какие метрики автоматически извлекаются расширением Akamai?

| Ключ метрики | Описание |
| --- | --- |
| `log.akamai-siem.volumetric-activity` | Количество событий, соответствующих объёмной активности, такой как превышение частоты запросов или DoS-атаки. |
| `log.akamai-siem.deny_count` | Количество событий, в которых действие правила — блокировка запроса (deny). |
| `log.akamai-siem.alert_count` | Количество событий, в которых действие правила — разрешение запроса и запись предупреждения (alert). |
| `log.akamai-siem.monitor_count` | Количество событий с типом действия правила monitor. |
| `log.akamai-siem.total-events` | Общее количество событий, обработанных из Akamai SIEM, независимо от типа атаки или серьёзности. |
| `log.akamai-siem.slow-posts` | Количество событий, соответствующих атаке slow POST, которая пытается заблокировать сайт с помощью крайне медленных запросов и ответов. |
| `log.akamai-siem.targeted-web-attacks` | Количество событий, соответствующих специализированным атакам на веб-приложения, таким как SQL-инъекции, PHP-инъекции, инъекции команд и межсайтовый скриптинг. |
| `log.akamai-siem.generic-web-attacks` | Количество событий, соответствующих общим атакам на веб-приложения. К ним относятся ключевые слова: `Trojan`, `Web attack tool`, `Web protocol attack` и `Web platform attack`. |
