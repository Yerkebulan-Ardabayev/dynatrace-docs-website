# Оценка покрытия

* Latest Dynatrace
* How-to guide
* Обновлено 13 января 2026 г.

Доступны следующие варианты оценки покрытия.

## Просмотр покрытия систем

На странице **Overview** кольцевая диаграмма **System coverage** помогает определить процент контролируемых систем, на которых включено [Security Posture Management](../application-security/spm.md "Оценка, управление и принятие мер в отношении ошибок конфигурации и нарушений руководств по обеспечению безопасности и стандартов соответствия нормативным требованиям.").

![system coverage](https://dt-cdn.net/images/2024-11-08-08-15-45-387-114cb209db.png)

Включение Security Posture Management на ваших системах позволяет просматривать результаты в приложении и в полной мере использовать его [возможности](../xspm.md#about "Обнаружение, управление и принятие мер в отношении выявленных проблем безопасности и соответствия.").

Если впоследствии вы отключите Security Posture Management на своей системе, вы сохраните доступ к ранее созданным результатам оценки и сможете запрашивать [события соответствия](../threat-observability/dql-examples.md#compliance "Примеры DQL для данных безопасности на основе Grail.") из этой системы в [![Investigations](https://dt-cdn.net/images/security-investigator-256-93f6c187d9.png "Investigations") **Investigations**](../investigations.md "Объединение функций Grail для расследований на основе фактов, включая разрешение инцидентов, анализ основных причин и поиск угроз.") и [![Notebooks](https://dt-cdn.net/images/notebooks-768-046137830a.webp "Notebooks") **Notebooks**](../../analyze-explore-automate/dashboards-and-notebooks/notebooks.md "Анализ, визуализация и обмен результатами наблюдаемости — всё в одном совместном, настраиваемом рабочем пространстве.").

### Улучшение покрытия систем

Для улучшения покрытия [Security Posture Management](../application-security/spm.md "Оценка, управление и принятие мер в отношении ошибок конфигурации и нарушений руководств по обеспечению безопасности и стандартов соответствия нормативным требованиям.") на ваших системах

1. На странице **Overview** в таблице **My systems** найдите системы с меткой `Not enabled`.
2. Для каждой системы, которую вы хотите охватить, выберите **Enable SPM**.
3. На открывшейся странице **Settings** включите **Enable Security Posture Management**.

## Просмотр статуса соответствия по окружениям

На странице **Overview** карточки стандартов соответствия помогают определить статус соответствия по каждому окружению для каждого [поддерживаемого стандарта соответствия](../application-security/spm.md#support "Оценка, управление и принятие мер в отношении ошибок конфигурации и нарушений руководств по обеспечению безопасности и стандартов соответствия нормативным требованиям."). Вы можете просмотреть

* Процент пройденных правил
* Общее количество правил и сколько из них [пройдены, требуют ручной проверки и не пройдены](concepts.md#concept-results "Понятия, специфичные для приложения Dynatrace Security Posture Management.")
* Количество правил по [уровню серьёзности](concepts.md#concept-severity "Понятия, специфичные для приложения Dynatrace Security Posture Management.") (критический, высокий, средний и низкий)

![compliance status](https://dt-cdn.net/images/2024-11-04-21-06-16-1477-6f03aab3d1.png)

Выберите карточку, чтобы перейти на страницу **Assessment results** и просмотреть результаты, отфильтрованные по соответствующему стандарту.

### Улучшение покрытия соответствия

Kubernetes Security Posture Management (KSPM)

Вы можете улучшить покрытие соответствия для каждого окружения, включив соответствующие стандарты.

1. Откройте меню настроек в правом верхнем углу ![xSPM](https://dt-cdn.net/images/security-posture-management-highresolution-1024-83a748ecdd.png "xSPM") **Security Posture Management**.
2. На открывшейся странице **Settings** включите нужные стандарты.

   Стандарт CIS включён по умолчанию и не может быть отключён.

Кроме того, вы можете включить стандарты непосредственно из **Settings**.

1. Перейдите в **Settings** > **Analyze and alert** > **Application Security** > **Security Posture Management**.
2. Включите нужные стандарты.

Любые изменения конфигурации вступят в силу после следующего анализа соответствия.

## Просмотр статуса соответствия по системам

На странице **Overview** таблица **My systems** помогает просматривать результаты и статус соответствия для каждой из ваших контролируемых систем. Вы можете просмотреть

* Общее количество [не пройденных, требующих ручной проверки и пройденных правил](concepts.md#concept-results "Понятия, специфичные для приложения Dynatrace Security Posture Management.")
* Общее количество не пройденных правил по [уровню серьёзности](concepts.md#concept-severity "Понятия, специфичные для приложения Dynatrace Security Posture Management.")
* Статус соответствия
* Время завершения последней оценки

![system list](https://dt-cdn.net/images/2024-11-04-22-02-43-1839-4166f65292.png)

Выберите систему, чтобы перейти на страницу **Assessment results** и просмотреть результаты, отфильтрованные по этой системе. Если для системы [нет результатов](#no-results), навигация невозможна.

## Часто задаваемые вопросы

### Почему для моей системы нет результатов?

Ниже приведены возможные причины, по которым на странице **Overview** в таблице **My systems** для системы не отображаются результаты.

* Security Posture Management не включён на этой системе. В этом случае система помечена как `Not enabled`. Выберите **Enable SPM**, чтобы перейти на страницу настроек этой системы и включить Security Posture Management.

* Первоначальная оценка ещё выполняется. В этом случае система помечена как `No data`. Пожалуйста, подождите около часа для завершения оценки. Подробнее о механизме оценки см. в разделе [Как это работает](../application-security/spm.md#mechanism "Оценка, управление и принятие мер в отношении ошибок конфигурации и нарушений руководств по обеспечению безопасности и стандартов соответствия нормативным требованиям.").
