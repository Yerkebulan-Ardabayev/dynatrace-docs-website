# Получение аналитических данных


* Latest Dynatrace

На странице **Assessment results** при выборе правила открывается боковое окно, которое позволяет:

**Просмотреть ключевую информацию:**

* [Правило](#rule)
* [Связанные системные ресурсы](#resources)
* [Конфигурация ресурса](#configuration)

**Инициировать углублённый анализ:**

* [Объяснение оценки](#explain)

Этот контекст может помочь вам устранить проблемы в вашей системе.

## Информация о правиле

На вкладке **Rule details** вы можете:

* Просмотреть релевантные сведения о правиле, такие как:

  + Версия правила, использованная для оценки
  + Серьёзность правила
  + Стандарт соответствия, издавший правило
* Перейти к исходному документационному ресурсу для получения полного описания правила

  ![Информация о правиле](https://dt-cdn.net/images/2024-11-17-18-05-08-896-899014266d.png)

## Информация о ресурсах

На вкладке **Assessed resources** вы можете изучить ресурсы, которые должны соответствовать правилу. Это может помочь вам определить ресурсы, затронутые нарушением соответствия или критически важной для безопасности неправильной конфигурацией.

Оцениваемые ресурсы со статусом `Not relevant` по умолчанию отфильтровываются.

Вы можете:

* Просматривать систему, к которой применяется правило, вместе со всеми системными ресурсами
* Использовать панель фильтров для фильтрации ресурсов, которые:

  + Требуют вашего внимания (`Failed`)
  + Соответствуют требованиям (`Passed`)
  + Не могут быть автоматически оценены (`Manual`)
  + Не соответствуют определённым критериям для оценки правила (`Not relevant`)
* Использовать строку поиска для поиска конкретного ресурса (полное или частичное совпадение)
* Наведите указатель мыши на диаграмму системы, чтобы увидеть статус соответствия системных ресурсов

  ![Статус соответствия узла](https://dt-cdn.net/images/2024-11-13-06-06-14-471-e22dd6056b.png)
* Определить тип ресурса и дату последней оценки в разделе **Rule assessment** > **Resource**

  ![Тип ресурса](https://dt-cdn.net/images/2024-11-17-18-13-15-537-4dcd8fcd4b.png)

## Информация о конфигурации

На вкладке **Assessed resources** перейдите в раздел **Rule assessment** > **Relevant configuration properties** для получения информации о конфигурации ресурса.

Это может помочь вам определить текущую неправильную конфигурацию для выбранного ресурса, которая способствует нарушению правила.

* **Пример**: Правило со статусом `Failed` `1.2.19 Ensure that the --audit-log-maxsize argument is set to 100 or as appropriate` сообщает, что на узле плоскости управления не настроен параметр `--audit-log-maxsize`. Это означает, что в случае необходимости проведения расследования безопасности не гарантируется наличие достаточного количества логов для проведения расследования из-за этой неправильной конфигурации.

  ![Пример правила 1](https://dt-cdn.net/images/2024-11-17-18-32-42-446-e03c9462e5.png)

Для правил со статусом `Manual`, где автоматическая оценка невозможна, предоставляются подсказки о том, какая информация необходима для завершения оценки.

Подробнее о правилах `manual` см. [Результаты](concepts.md#concept-results "Концепции, специфичные для приложения Dynatrace Security Posture Management.").

* **Пример**: Правило со статусом `Manual` `The Kubernetes kubelet staticPodPath must not enable static pods` сообщает, что Dynatrace не может проверить конфигурацию, поскольку отсутствует Kubernetes Node Configuration Collector.

  ![Пример отсутствия NCC](https://dt-cdn.net/images/2024-11-17-19-11-42-473-231173eb9b.png)

## Объяснение оценки

Для использования этой функции генеративного ИИ убедитесь, что:

* Генеративный ИИ Dynatrace Intelligence включён для вашей среды. См. [Включение генеративного ИИ Dynatrace Intelligence](../../dynatrace-intelligence/copilot/copilot-getting-started.md#enable-davis-copilot "Узнайте, как настроить агентный и генеративный ИИ Dynatrace Intelligence.").
* У вас есть разрешение на его использование. См. [Права пользователя](../../dynatrace-intelligence/copilot/copilot-getting-started.md#davis-copilot-user-permissions "Узнайте, как настроить агентный и генеративный ИИ Dynatrace Intelligence.").

Генеративный ИИ Dynatrace Intelligence может предоставлять контекстные объяснения оценок правил на понятном языке для ускорения понимания и устранения проблем.

Для доступа к функциональности:

1. В [![xSPM](https://dt-cdn.net/images/security-posture-management-highresolution-1024-83a748ecdd.png "xSPM") **Security Posture Management**](../xspm.md "Обнаружение, управление и принятие мер по результатам оценки безопасности и соответствия.") на странице **Assessment results** выберите правило.
2. На вкладке **Assessed resources** выберите **Explain assessment**.

При выборе генеративный ИИ Dynatrace Intelligence анализирует технические детали оценки правила и генерирует структурированную сводку, которая может включать:

* **Что означает оценка**: Объясняет правило соответствия (например, требования CIS Kubernetes Benchmark) и его назначение для обеспечения безопасности сред.
* **Почему это важно**: Выделяет уровни серьёзности и приоритет, а также описывает потенциальные последствия, такие как неправильные конфигурации, ведущие к несанкционированному доступу, повышению привилегий или компрометации кластера.
* **Как устранить**: При возможности рекомендует шаги по устранению, такие как обновление разрешений файлов, применение исправлений конфигурации, перезапуск сервисов или включение непрерывного мониторинга кластеров и облачных аккаунтов.

Структура и глубина объяснения генеративного ИИ могут варьироваться в зависимости от типа правила и доступного контекста. Хотя генеративный ИИ Dynatrace Intelligence стремится предоставить подробные аналитические данные, не все оценки будут включать каждый из перечисленных выше элементов.

Объяснения генеративного ИИ Dynatrace Intelligence адаптированы к характеру каждой оценки правила, предоставляя релевантные и практически применимые аналитические данные, которые ускоряют сортировку и поддерживают принятие обоснованных решений, даже для пользователей без глубоких знаний в области безопасности.
