* Последняя Dynatrace

Благодаря возможностям Grail и [данным безопасности](threat-observability/concepts.md#security-data "Основные концепции, связанные с Threat Observability"), поступающим в него, можно реализовать различные сценарии анализа данных, отчётности и автоматизации.

## Реагирование на инциденты

* Анализ логов доступа Amazon API Gateway с помощью Investigations
* Анализ логов AWS CloudTrail с помощью Investigations
* Обнаружение угроз для ваших секретов AWS с помощью Investigations
* Операционализация результатов DQL-запросов с помощью Investigations
* Более быстрое решение инцидентов с шаблонами Investigations
* Ускорение реагирования на инциденты с помощью контрольного времени Dynatrace Investigations
* Поиск угроз и криминалистика

## Оркестрация безопасности

* Автоматизация и оркестрация обнаруженных уязвимостей
* Автоматизированная сортировка оповещений об угрозах
* Обнаружение угроз в облачных средах с помощью рабочих процессов
* Приём и обработка пользовательских обнаружений безопасности
* Автоматизация уведомлений CSPM

## Анализ состояния безопасности

* Обнаружение пробелов в покрытии обнаружений безопасности.")
* Мониторинг подозрительной активности входа в систему с помощью Dynatrace
* Контекстуализация обнаружений контейнеров во время выполнения
* Соответствие требованиям с помощью Security Posture Management
* Визуализация и анализ обнаружений безопасности

## Автоматизация на базе ИИ

* Автоматизация устранения уязвимостей с помощью Atlassian Rovo Dev и Dynatrace
* Автоматизация устранения уязвимостей с помощью GitHub Copilot и Dynatrace
* Автоматизация сортировки и устранения облачных неправильных конфигураций с помощью Kiro CLI и Dynatrace
