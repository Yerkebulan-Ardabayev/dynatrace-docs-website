# Кластер Dynatrace Managed

Руководство по управлению кластером.

## Архитектура кластера

```
┌─────────────────────────────────────────┐
│            Load Balancer                │
└────────────────┬────────────────────────┘
                 │
    ┌────────────┼────────────┐
    │            │            │
┌───▼───┐   ┌───▼───┐   ┌───▼───┐
│ Node 1│   │ Node 2│   │ Node 3│
│ Active│   │ Active│   │ Active│
└───────┘   └───────┘   └───────┘
```

## Добавление узла

1. Установите Dynatrace Managed на новый сервер
2. При установке укажите "Join existing cluster"
3. Введите IP первого узла кластера
4. Подтвердите в CMC

## Управление узлами

### Просмотр состояния

```bash
# На любом узле
/opt/dynatrace-managed/cluster/status.sh
```

### Удаление узла

1. Откройте CMC
2. **Cluster → Nodes**
3. Выберите узел → **Remove**

!!! warning "Внимание"
    Минимум 3 узла для production!

## Масштабирование

| Размер кластера | Узлов | RAM/узел | Хостов |
|-----------------|-------|----------|--------|
| Small | 3 | 32 GB | до 100 |
| Medium | 5 | 64 GB | до 500 |
| Large | 7+ | 128 GB | 1000+ |

## Балансировка нагрузки

Рекомендуемые решения:
- HAProxy
- NGINX
- F5 Big-IP
- AWS ALB / Azure LB
