# Troubleshooting RUM Classic for web applications

# Troubleshooting RUM Classic for web applications

* Reference
* 2-min read
* Updated on Jul 07, 2023

This page explains what to do when Real User Monitoring Classic (RUM Classic) isn't working in your environment.

Before performing any troubleshooting, try checking the [health check page](/managed/observe/digital-experience/rum-classic/web-applications/initial-setup/app-health-check "The Application health check page allows you to analyze the health of your application, see which RUM JavaScript versions are currently in use, or confirm that the RUM JavaScript is injected correctly.") for your application.

## General troubleshooting

For general troubleshooting, refer to [Web applications: Issues with RUM JavaScript﻿](https://dt-url.net/up03l19) in the Dynatrace Community.

## User-action-related troubleshooting

To troubleshoot issues related to user actions, refer to the following pages in the Dynatrace Community.

* [User action data is missing﻿](https://dt-url.net/zqa3lef)
* [Web request isn't linked to its user action﻿](https://dt-url.net/io63ls2)
* [Sessions are randomly split into shorter sessions﻿](https://dt-url.net/cb23ley)
* [Source actions are missing for subdomain navigations﻿](https://dt-url.net/xi43lit)

## Metadata-related troubleshooting

If you've encountered a metadata-related issue, check [Web applications: Sessions are not correctly tagged or metadata is missing﻿](https://dt-url.net/rp83l5k) in the Dynatrace Community.
