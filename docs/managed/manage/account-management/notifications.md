# Notification center

# Notification center

* Explanation
* Published May 29, 2026

Account Management sends notifications when spending thresholds are approached or when unexpected consumption changes are detected.
This page describes how notifications work across [Budget alerts](/managed/manage-your-costs/control/budgets "Learn how to configure budgets in Dynatrace.") and [Cost monitors](/managed/manage-your-costs/control/cost-monitors "Learn how to use the Cost Monitors feature to make forecasts and cost events.").

## Overview

Two features generate notifications in Account Management.

* **Budget alerts** notify you when actual or forecasted consumption reaches a configured threshold.
* **Cost monitors** notify you when forecast conditions or daily consumption anomalies are detected.

Both features send notifications through the **Notification Center** and by email.
Cost monitors additionally support API-based notifications.

## Use cases

* Understand where notifications appear in Account Management.
* Verify which permissions are required to receive and open notifications.
* Decide whether to integrate cost monitor notifications through API.
* Compare budget and cost monitor notification behavior before configuring thresholds.

## Notification channels

| Channel | Available for |

### Notification center

**Notification Center** provides a history of all notifications sent from Account Management, including budget and cost monitor events.

To open **Notification Center**, select  > **Notification Center** in the Account Management toolbar.

The table displays events detected during the current commitment period, with the most recent events at the top.
The table includes:

* **Date**: When the event was detected; use the **Timeframe** filter to limit results based on date.
* **Message**: Event text generated by Dynatrace; use the **Search** filter to filter results based on specific text strings.
* **Event type**: Filter by the specific type of event.
* **Levels**: Select to view events by severity.
* **Environments**: Select an individual environment name to filter for events of a specific environment, or select **Account** to filter for events at the account level.
* **Capability**: Indicates the capability that the event relates to. Select a specific capability to filter the table, or select **DPS Cost Total** to view all forecast events.

### Email notifications

You can configure when email notifications are sent and who receives them.
For instructions on configuring email recipients for each alert type, see the relevant page:

* [Cost monitor email notifications](/managed/manage-your-costs/control/cost-monitors#email-notifications "Learn how to use the Cost Monitors feature to make forecasts and cost events.")
* [Budget alert email notifications](/managed/manage-your-costs/control/budgets#notification "Learn how to configure budgets in Dynatrace.")

Notifications are sent from `no-reply@dynatrace.com`.

Notification emails include links to Account Management.
Recipients need **View account** permission to open linked pages.

To configure notification settings, your Dynatrace user needs the **View and manage account and billing information** account permission.

### API notifications

API notifications are available for cost monitor events. For details, see [Notifications API - POST filter notifications](/managed/dynatrace-api/account-management-api/post-notifications "List notifications for your account.").
