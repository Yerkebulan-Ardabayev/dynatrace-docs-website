# New installation

# New installation

* How-to guide
* 1-min read
* Published Jan 28, 2020

When you introduce Dynatrace to your system for the first time, we recommend that you add network zones straight away.

If you're already using Dynatrace, refer to [Migration](/managed/manage/network-zones/migration "Start using network zones in an existing Dynatrace environment."), which is a different procedure.

1. [Analyze](/managed/manage/network-zones/new-installation/analyze "Find out how to analyze your network in preparation for network zones implementation.")
2. [Plan](/managed/manage/network-zones/new-installation/plan "Plan network zones that reflect your network topology.")
3. [Deploy](/managed/manage/network-zones/new-installation/deploy "Find out how to deploy network zones for an existing Dynatrace environment.")
4. [Verify](/managed/manage/network-zones/new-installation/verify "Learn how to proof that network zones implementation was successful.")
