447@@@## Непривилегированный режим
449@@@### **`NON_ROOT_MODE`**
451@@@**Значение по умолчанию**: `1` (OneAgent версии 1.193+. Для более ранних версий `0`)
453@@@При установке OneAgent в непривилегированном режиме повышенные привилегии нужно предоставить OneAgent только во время установки. Повышенные привилегии сбрасываются, как только OneAgent развёрнут.
455@@@Начиная с версии 1.193, OneAgent по умолчанию устанавливается в непривилегированном режиме. Существующие установки не переключаются в непривилегированный режим.
457@@@Чтобы переключить установленный OneAgent в непривилегированный режим, необходимо вручную добавить параметр `NON_ROOT_MODE=1` к команде установки. Пример:
458@@@`sudo /bin/sh Dynatrace-Agent-Linux-1.0.0.sh NON_ROOT_MODE=1`
459@@@Чтобы вернуть установщик в режим по умолчанию для последующих обновлений, запустите его с `NON_ROOT_MODE=0`.
461@@@Обратите внимание, что непривилегированный режим требует возможностей ядра Linux, доступных в этих версиях:
463@@@* Версия ядра Linux 2.6.26+ для установки OneAgent без привилегий root.
464@@@* Версия ядра Linux 4.3+ (рекомендуется systemd версии 221+) для автоматических обновлений и полноценной работы OneAgent без привилегий root.
465@@@  Дополнительные сведения см. в [Непривилегированный режим Linux](/managed/ingest-from/dynatrace-oneagent/installation-and-operation/linux/installation/linux-non-privileged "Узнайте, когда Dynatrace OneAgent требует привилегий root на Linux.")
467@@@### **`DISABLE_ROOT_FALLBACK`**
469@@@**Значение по умолчанию**: `0`
471@@@Используется вместе с параметром `NON_ROOT_MODE` для блокировки уровня разрешений суперпользователя для OneAgent, работающего в непривилегированном режиме. Привилегии root требуются для автоматических обновлений и отдельных операций на версиях ядра между 2.6.26 и 4.3, то есть версиях без поддержки Linux ambient capabilities.
473@@@`sudo /bin/sh Dynatrace-OneAgent-Linux-1.0.0.sh NON_ROOT_MODE=1 DISABLE_ROOT_FALLBACK=1`
475@@@Чтобы вернуть установщик к использованию уровня разрешений суперпользователя для последующих обновлений, запустите его с `DISABLE_ROOT_FALLBACK=0`.
477@@@Дополнительные сведения см. в [требованиях к разрешениям для установки и эксплуатации OneAgent на Linux](/managed/ingest-from/dynatrace-oneagent/installation-and-operation/linux/installation/linux-non-privileged "Узнайте, когда Dynatrace OneAgent требует привилегий root на Linux.").

482@@@## Указание непривилегированного пользователя и группы
484@@@### **`USER`**
486@@@**Значение по умолчанию**: `dtuser`
488@@@Указывает имя непривилегированного пользователя, от имени которого работают непривилегированные процессы OneAgent. Непривилегированными являются процессы, которым не нужны привилегии root. Эти процессы на Linux называются `Network OneAgent` и `Plugin OneAgent`.
490@@@* По умолчанию установщик Dynatrace использует `dtuser` в качестве имени непривилегированного пользователя.
491@@@* Если задан параметр `USER=<username>`, установщик использует `<username>` в качестве имени непривилегированного пользователя.
493@@@В любом случае установщик Dynatrace проверяет, существует ли уже в системе требуемый пользователь (`dtuser` или пользователь, заданный параметром `USER`).
495@@@* Если пользователь и группа с одинаковым именем оба существуют и для этого пользователя данная группа задана как основная, этот пользователь используется для запуска сетевого и плагинного модулей OneAgent.
496@@@* Если пользователь не существует, установщик Dynatrace создаёт этого пользователя и группу и затем запускает эти непривилегированные процессы от имени нового пользователя.
497@@@* Если пользователь существует в системе, но не имеет группы с тем же именем, заданной как основная, установка прерывается; чтобы использовать группу с другим именем, необходимо использовать параметр `GROUP`.
499@@@Требования к строке `USER`:
501@@@* Может содержать только буквенно-цифровые символы, дефис `-`, подчёркивание `_` и точку `.`
502@@@* Минимальная длина 3 символа
503@@@* Максимальная длина 32 символа
504@@@* Не может быть строкой [идентификатора пользователя](https://man7.org/linux/man-pages/man7/credentials.7.html)
506@@@### **`GROUP`**
508@@@**Значение по умолчанию**: `dtuser`
510@@@Может использоваться только вместе с параметром `USER` и служит для указания основной группы пользователя, переданного через параметр `USER`. Если вы не указываете параметр `GROUP`, установщик предполагает, что она совпадает с `USER`, как для существующих, так и для несуществующих пользователей. Если вы указываете группу с помощью параметра `GROUP`, а пользователь не существует, установщик создаёт пользователя и назначает ему указанную группу. Параметр `GROUP` также используется для указания непривилегированного пользователя, принадлежащего определённой группе с именем, отличным от имени пользователя. Для усиления безопасности вашей системы мы настоятельно рекомендуем использовать выделенную группу пользователей для запуска процессов OneAgent.
512@@@Требования к строке `GROUP`:
514@@@* Может содержать только буквенно-цифровые символы, дефис `-`, подчёркивание `_` и точку `.`
515@@@* Минимальная длина 3 символа
516@@@* Максимальная длина 32 символа
517@@@* Не может быть строкой [идентификатора группы](https://man7.org/linux/man-pages/man7/credentials.7.html)
525@@@Сведения о механизме автообновления OneAgent см. в [Обновление OneAgent на Linux](/managed/ingest-from/dynatrace-oneagent/installation-and-operation/linux/operation/update-oneagent-on-linux "Узнайте о различных способах обновления OneAgent на Linux.").
