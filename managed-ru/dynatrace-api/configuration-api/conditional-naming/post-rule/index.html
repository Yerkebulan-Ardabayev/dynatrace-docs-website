    Conditional naming API - POST a new naming rule¶

    Conditional naming API - POST a new naming rule¶

    • Справочник
    • Опубликовано 23 апр. 2020 г.

    Создаёт новое правило условного именования.

    Запрос принимает и возвращает данные в формате application/json.
    POST ManagedDynatrace for Government https://{your-domain}/e/{your-environment-id}/api/config/v1/conditionalNaming/{type}
    POST Environment ActiveGate https://{your-activegate-domain}/e/{your-environment-id}/api/config/v1/conditionalNaming/{type}

    Аутентификация¶

    Для выполнения этого запроса нужен токен доступа с областью действия WriteConfig.

    О том, как получить и использовать токен, читай в разделе Tokens and authentication.

    Параметры¶

    Модели JSON для конкретного type модели приведены в разделе JSON models.

    Параметр Тип Описание Расположение Обязательный
    type string Тип правила, определяется типом сущностей Dynatrace, к которым применяется правило. Элемент может принимать следующие значения * processGroup * host * service path Обязательный
    body ConditionalNamingRule Тело запроса JSON. Содержит параметры нового правила именования. Не указывай id правила. Значение поля type должно совпадать со значением параметра пути type. body Необязательный

    Ответ¶

    Запрос возвращает краткое представление вновь созданного правила.

    Коды ответа¶

    Код Тип Описание
    201 EntityShortRepresentation Успех. Правило именования создано. Ответ содержит ID нового правила
    400 ErrorEnvelope Ошибка. Входные данные некорректны.

    id string ID сущности Dynatrace.

    Проверка полезной нагрузки¶

    Рекомендуется проверять полезную нагрузку перед отправкой её в реальном запросе. Код ответа 204 означает, что полезная нагрузка корректна.

    Запрос принимает полезную нагрузку application/json.
    POST ManagedDynatrace for Government https://{your-domain}/e/{your-environment-id}/api/config/v1/conditionalNaming/{type}/validator
    POST Environment ActiveGate https://{your-activegate-domain}/e/{your-environment-id}/api/config/v1/conditionalNaming/{type}/validator

    Аутентификация¶

    Для выполнения этого запроса нужен токен доступа с областью действия WriteConfig.

    Подробнее о том, как получить и использовать токен, см. Tokens and authentication.

    Ответ¶

    Коды ответа¶

    Код Тип Описание

    400 ErrorEnvelope Ошибка. Входные данные некорректны.
