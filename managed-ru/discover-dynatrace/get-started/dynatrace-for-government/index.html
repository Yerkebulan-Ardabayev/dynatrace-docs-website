    Dynatrace для государственных учреждений¶

    Dynatrace для государственных учреждений¶

    • Практическое руководство
    • 10 минут на чтение
    • Обновлено 15 мая 2026 г.

    Dynatrace for Government, это версия Dynatrace Managed, сертифицированная по FedRAMP и предназначенная для федеральных, региональных и муниципальных агентств.

    Dynatrace, это платформа программного интеллекта для мониторинга, которая упрощает сложность корпоративного облака и ускоряет цифровую трансформацию. Работая на основе Davis® (движка причинно-следственного анализа Dynatrace AI) и полной автоматизации, платформа Dynatrace «всё в одном» даёт ответы, а не просто данные, о приложениях и инфраструктуре. Она также отслеживает опыт конечных пользователей. Dynatrace модернизирует и автоматизирует операции корпоративного облака, ускоряет выпуск более качественного программного обеспечения и обеспечивает оптимальный цифровой опыт для клиентов организации.

    Dynatrace бесшовно объединяет мониторинг инфраструктуры и облака, производительности приложений и цифрового опыта в единое автоматизированное решение на основе AI. Dynatrace помогает добиваться результатов в производительности, предоставляя командам разработки, эксплуатации и бизнеса общую платформу и метрики. Таким образом, Dynatrace может служить единым «источником истины» для организации.

    Интеграция¶

    Dynatrace for Government входит в облачную экосистему корпоративного агентства. Он интегрируется с ключевыми компонентами для поддержки динамической оркестрации облака, включая Amazon Web Services (AWS), Azure, Google Cloud, VMware Tanzu Application Service, Red Hat OpenShift и Kubernetes. В этих средах Dynatrace автоматически запускает и отслеживает весь стек. Он отслеживает все приложения и контейнеры в стеке, включая рабочие нагрузки, которые охватывают несколько облачных и гибридных сред.

    Тесная интеграция с облачными платформами помогает упростить разработку и эксплуатацию, повысить прозрачность и улучшить понимание ситуации в гибридных, мультиоблачных средах.

    Шаг 1 Создайте учётную запись Dynatrace¶

    Встретьтесь с командой Dynatrace, чтобы настроить среду Dynatrace for Government. Нужно интегрировать существующую конфигурацию единого входа (SSO). См. Поддерживаемые технологии SSO, чтобы вместе с командой Dynatrace for Government выполнить интеграцию.

    Шаг 2 Разверните Dynatrace¶

    Чтобы загрузить и установить OneAgent на хосте

    1. Перейдите в раздел Deploy Dynatrace.
    2. Выберите Start installation, затем выберите платформу, на которую нужно установить OneAgent.

    Выбор платформы OneAgent 3. Вставьте PaaS-токен в поле Download token или выберите Create token, чтобы сгенерировать новый токен Deployment API.

    Скопируйте токен и сохраните его в надёжном месте, поскольку получить к нему доступ повторно не получится. 4. Введите или выберите нужные параметры

    • Architecture (только для Linux)
    • Monitoring mode Доступны варианты Full-Stack, Infrastructure или Discovery. Если используется бесплатная пробная версия Dynatrace, выберите Full-Stack, чтобы увидеть все возможности наблюдения Dynatrace. Режим мониторинга всегда можно изменить после установки.
    • В разделе Optional parameters добавьте Custom host name для более удобной идентификации. Остальные параметры выходят за рамки этого руководства.

    Параметры развёртывания OneAgent 5. Загрузите OneAgent. Используйте предоставленную команду интерфейса командной строки (CLI) или выберите Download. 6. Проверьте подпись. Используйте предоставленную команду CLI. (Примечание: только для Linux и AIX.) 7. Установите OneAgent. Используйте предоставленную команду CLI или запустите исполняемый файл, выбрав его в графическом интерфейсе. Следуйте шагам, описанным в установщике.

    При установке через графический интерфейс добавьте следующие параметры на экране Optional: advanced command-line settings: --set-monitoring-mode=fullstack --set-app-log-content-access=true 8. Когда установщик покажет сообщение Congratulations! Dynatrace OneAgent was successfully installed!, OneAgent запущен на хосте. Выберите Finish, чтобы закрыть установщик. 9. Поскольку OneAgent не может внедряться в уже запущенные процессы, нужно перезапустить все процессы, которые должен отслеживать OneAgent. 10. Чтобы убедиться, что OneAgent отслеживает хост, откройте Dynatrace и перейдите в Infrastructure & Operations > Host. Если всё работает как ожидается, имя хоста появится в таблице Hosts. Пример показан на снимке экрана ниже.

    OneAgent теперь настроен и отслеживает хост. См. Начало работы, чтобы продолжить первое знакомство с Dynatrace.

    Шаг 3 Отслеживайте хост и его процессы¶

    При выборе имени хоста Dynatrace показывает всё, что уже известно об этом хосте.

    Ниже перечислены лишь некоторые данные, которые можно найти на экране обзора хоста.

    Свойства и теги¶

    Выберите Properties and tags на панели уведомлений, чтобы открыть панель Properties and tags, которая отображает метаданные о выбранном хосте:

    • Tags содержит список тегов, применённых к хосту в данный момент. Выберите Add Tag, чтобы добавить тег к метаданным хоста.
    • Properties содержит список различных свойств хоста, таких как версия OneAgent, версия ОС, режим мониторинга, IP-адреса и management zones.

    Проблемы¶

    • На панели уведомлений вкладка Problems показывает активные и закрытые проблемы, связанные с выбранным хостом.

    • Выберите Problems на панели уведомлений, чтобы открыть панель Problems, в которой перечислены проблемы.

    • Выберите проблему, чтобы отобразить подробности.

    • Выберите Go to problems, чтобы перейти на страницу Problems, отфильтрованную по выбранному хосту.

    Уязвимости¶

    • На панели уведомлений вкладка Vulnerabilities показывает основные обнаруженные уязвимости, затрагивающие выбранный хост.

    • Выберите Vulnerabilities на панели уведомлений, чтобы открыть панель Vulnerabilities, в которой перечислены наиболее серьёзные уязвимости сторонних библиотек и уязвимости на уровне кода, связанные с этим хостом.

    • Выберите уязвимость в списке, чтобы посмотреть подробности и понять серьёзность и влияние уязвимости в вашей среде.

    • Чтобы увидеть полный список обнаруженных уязвимостей, затрагивающих этот хост, выберите Show all third-party vulnerabilities/Show all code-level vulnerabilities.

    Доступность¶

    • На панели уведомлений вкладка Availability показывает процент времени, в течение которого хост был онлайн и отвечал на запросы. Dynatrace обнаруживает и показывает завершения работы операционной системы (включая перезагрузки) и периоды, когда хост находится офлайн (например, если хост неожиданно выключился).
    • Выберите Availability на панели уведомлений, чтобы открыть панель Host availability, на которой отображается график доступности хоста во времени.

    Производительность хоста¶

    Перейдите в раздел Host performance, чтобы быстро получить представление по ключевым метрикам: CPU, памяти и сети с разными агрегациями метрик для выбранного периода времени. Просмотр временной шкалы позволяет одновременно точно определять выбранные аномалии на всех графиках метрик, что упрощает понимание взаимосвязей между различными компонентами инфраструктуры в конкретный момент времени.

    Легко проверить максимальные или минимальные пики потребления ресурсов, поскольку каждый график метрик позволяет выбрать разную агрегацию. Вместо метрик по умолчанию можно также отображать пользовательские метрики, что позволяет изучить конкретные взаимосвязи между метриками, которые могут быть критичны для конкретной конфигурации хоста.

    Использование этих графиков

    Выберите в правом верхнем углу графика, чтобы:

    • Show in Data Explorer, открывает Data Explorer для связанного запроса, что позволяет посмотреть связанный запрос, изучить данные более подробно, настроить параметры графика и закрепить график на собственной панели мониторинга.
    • Pin to dashboard, закрепляет копию выбранного графика на любой классической панели мониторинга, которую можно редактировать. Например, если определённые хосты особенно важны для бизнеса, можно создать панель мониторинга, предназначенную только для отслеживания этих хостов, а затем закрепить на ней графики со страниц обзора хостов, практически без ввода текста. Подробности см. в разделе Pin tiles to your dashboard.

    Снимки экземпляров процессов¶

    OneAgent версии 1.237+

    Раздел Process instance snapshots предлагает дополнительные сведения о наиболее ресурсоёмких процессах, запущенных на хосте, и о процессах, определённых для мониторинга Process availability.

    Process instance snapshot

    Единичный снимок экземпляра процесса, это набор данных мониторинга для процессов. Он содержит данные о CPU usage (%), Memory usage (B), Incoming network traffic (KB) и Outgoing network traffic (KB) процесса, измеренные с интервалом в одну минуту. Один снимок содержит 20 минут данных мониторинга: 10 минут до триггера и 10 минут после триггера. Каждый хост может передавать эти метрики не более 60 минут в сутки. Процесс учитывается для снимка, если потребление им CPU, памяти или сети превышает 1%.

    Снимок экземпляра процесса запускается при высоком потреблении CPU, памяти или сети на хосте. Снимок процесса также можно запросить вручную. Выберите в правом верхнем углу раздела и выберите Request process snapshot now. Дождитесь сообщения, подтверждающего успешный запуск снимка. Данные снимка процесса должны появиться после перезагрузки страницы в течение 90 секунд.

    Кроме того, для процессов, определённых для мониторинга Process availability, снимок показывает, как вели себя процессы до того, как они исчезли, и появились ли они снова в течение 10 минут.

    Включение снимков экземпляров процессов¶

    Снимки экземпляров процессов можно включить на уровне хоста или на уровне среды.

    • Чтобы включить их на уровне среды, перейдите в Settings, выберите Processes and containers > Process instance snapshots и включите Enable process instance snapshots.
    • Чтобы задать правило на уровне хоста, перейдите на страницу обзора хоста, выберите , перейдите в Settings, выберите Process instance snapshots и включите Enable process instance snapshots.
    • Чтобы задать правило на уровне группы хостов, перейдите на страницу группы хостов по адресу https://your-environment/ui/settings/HOST_GROUP-NAME, выберите Process instance snapshots и включите Enable process instance snapshots.

    На той же странице настроек можно также снизить лимит процессов, отображаемых в одном снимке. Максимальное/стандартное значение, 100 процессов.

    Анализ дисков¶

    Чтобы выявить узкие места в производительности диска, перейди в раздел Disk analysis, где отображаются все точки монтирования для систем Linux и все тома для Windows. С первого взгляда видны использование дискового пространства и метрики пропускной способности, а также другие выбранные метрики диска, что позволяет быстро выявить любые проблемы с производительностью диска.

    • На странице узла можно фильтровать диски по имени диска, чтобы сосредоточиться на выбранном диске.
    • Разверни запись диска, чтобы увидеть подробности о выбранном диске. Для каждого экземпляра диска отображаются отдельные подробные метрики производительности, что упрощает выявление любого ресурса диска, работающего не оптимально.

    Каждая точка монтирования (Linux) или том (Windows) имеет собственные метрики производительности в дополнение к объединённым метрикам. Это значительно упрощает обнаружение медленного или нестабильного диска. Оповещения можно настроить как для отдельных дисков, так и для объединённых графиков.

    Используй эти графики

    Выбери значок в правом верхнем углу графика, чтобы:

    • Show in Data Explorer, открывает Data Explorer для связанного запроса, позволяя просмотреть сам запрос, изучить данные более подробно, настроить параметры графика и закрепить график на собственной панели мониторинга.
    • Pin to dashboard, закрепляет копию выбранного графика на любой классической панели мониторинга, которую можно редактировать. Например, если определённые узлы особенно важны для бизнеса, можно создать панель мониторинга, предназначенную только для наблюдения за этими узлами, а затем закрепить на ней графики со страниц обзора этих узлов, практически без ручного ввода. Подробнее см. Pin tiles to your dashboard.

    Disk monitoring¶

    Disk monitoring, это запланированная задача, которая собирает метрики диска от агента. Она запускается автоматически, когда OS agent обнаруживает диск.

    Доступные настройки для дисков:

    • Disk options: можно создавать правила исключений, чтобы убирать определённые диски из списка мониторинга, и обнаруживать дублирующиеся диски. Подробнее см. ниже или в разделе Disk options.
    • Disk anomaly detection rules: можно настроить обнаружение аномалий узла, включая пороги проблем и событий, под свои конкретные задачи. Подробнее см. Host anomaly detection.
    • Anomaly detection for infrastructure: Disk: эти настройки позволяют настроить чувствительность обнаружения, задать пороги оповещений или отключить оповещения для дисков. Подробнее см. Anomaly detection.

    Отключение мониторинга для отдельных дисков¶

    Задай фильтр исключения, чтобы избежать проблем со специальными точками монтирования:

    1. Перейди в Settings > Preferences > Disk options.
    2. Выбери Add item, чтобы исключить диск из списка мониторинга.
    3. Укажи операционную систему, путь диска и тип файловой системы.
    4. Выбери Save changes.

    Дедупликация дисков¶

    Дедупликация дисков помогает снизить избыточность в монтированиях Network File System (NFS) на узле. В некоторых случаях один и тот же общий ресурс NFS может быть примонтирован несколько раз в разных точках монтирования или с разными параметрами. Это может привести к дублирующимся записям, влияющим на точность мониторинга. Дедупликация дисков выявляет и удаляет эти избыточные монтирования, обеспечивая более чистые и точные данные о диске.

    Система обнаруживает дублирующиеся диски, сравнивая их IP-адреса и доступное или используемое пространство. Если два или более диска имеют одинаковый IP-адрес и метрики использования, они считаются дубликатами и соответствующим образом дедуплицируются.

    Дедупликацию дисков можно включить или отключить в Settings > Preferences > Disk options.

    Имперсонация пользователя в Windows¶

    В Windows OS agent имперсонирует вошедшего в систему пользователя для сбора метрик диска с общих ресурсов NFS. Агент использует имперсонированный контекст безопасности для запроса метрик диска у сервера NFS.

    Network analysis¶

    Чтобы быстро выявлять проблемы, связанные с сетью, перейди в раздел Network analysis, в котором перечислены все сетевые интерфейсы и объединённые метрики по всем из них, а также отдельные метрики по каждому сетевому интерфейсу.

    Используй этот раздел, чтобы:

    • Выявлять потерю пакетов, пакеты с ошибками и другие сетевые проблемы
    • Искать сетевые интерфейсы по имени сети
    • Выявлять узкие места сети вплоть до конкретного адаптера

    Используй эти графики

    Выбери значок в правом верхнем углу графика, чтобы:

    • Show in Data Explorer, открывает Data Explorer для связанного запроса, позволяя просмотреть сам запрос, изучить данные более подробно, настроить параметры графика и закрепить график на собственной панели мониторинга.
    • Pin to dashboard, закрепляет копию выбранного графика на любой классической панели мониторинга, которую можно редактировать. Например, если определённые узлы особенно важны для бизнеса, можно создать панель мониторинга, предназначенную только для наблюдения за этими узлами, а затем закрепить на ней графики со страниц обзора этих узлов, практически без ручного ввода. Подробнее см. Pin tiles to your dashboard.

    События¶

    Раздел событий отображает недавние события хоста, которые сгенерировал Davis AI, с наглядным представлением временной шкалы для быстрого выявления критических событий. Временная шкала интерактивна, фильтрует события вокруг определённого момента, что упрощает выделение конкретного события. Кроме того, разные типы событий выделены цветом для более лёгкой и быстрой идентификации и просмотра.

    • Show single card, открывает карточку Events для выбранного хоста.

    Логи¶

    Временная шкала просмотрщика логов интерактивна, позволяет выбирать глобальный интервал времени. Используй её, чтобы выявить проблемы вокруг определённого события в логе и посмотреть, как это связано с производительностью хоста или процессами.

    • Выбор времени на всей странице хоста будет соответствовать тому, что выбрано в просмотрщике логов. Таким образом, лог ошибки можно легко сопоставить с метриками производительности хоста или метриками процессов в момент, когда произошла ошибка в логе. Тот же выбор временного интервала появится на карточке события.
    • Логи можно фильтровать по группе процессов, статусу, уровню лога и другим параметрам, что позволяет искать, например, только логи ошибок или логи по определённому процессу.

    Используй эту информацию

    Выбери в правом верхнем углу раздела Logs, чтобы:

    • Go to Log Viewer, открывает страницу Log Viewer, отфильтрованную по выбранному хосту.
    • Create metric, открывает страницу Log metrics со значением Query, установленным на выбранный хост.

    Шаг 4 Настройка уведомления о проблеме¶

    Dynatrace предлагает несколько готовых интеграций, которые автоматически отправляют уведомления о проблемах Dynatrace в сторонние системы обмена сообщениями или управления инцидентами. Если сторонняя система не поддерживается готовой интеграцией, можно настроить интеграцию по электронной почте. При таком подходе Dynatrace отправляет письмо каждый раз, когда обнаруживает в среде проблему, которая затрагивает реальных пользователей.

    1. Перейди в Settings и выбери Integration > Problem notifications.
    2. Выбери Add notification.

    Add an email problem notification 3. Выбери Email из доступных типов уведомлений. 4. Настрой уведомление:

    • Введи Display name для этой интеграции. Это произвольное имя появится в таблице Problem notifications после завершения настройки.
    • В разделе To выбери Add recipient, чтобы добавить адрес электронной почты, который должен получать уведомления. Чтобы добавить больше получателей, выбирай Add recipient для каждого дополнительного адреса To.
    • В поле CC выбери Add recipient, чтобы добавить дополнительные адреса электронной почты, которые должны получать уведомления.
    • В поле BCC выбери Add recipient, чтобы добавить дополнительные адреса электронной почты, которые должны получать уведомления, но не должны отображаться в заголовке письма.
    • В поле Subject введи текст или вставь плейсхолдеры, которые Dynatrace автоматически заполняет соответствующими деталями проблемы, такими как ID проблемы, состояние или влияние.

    • В поле Body настрой сообщение уведомления о проблеме, которое будет отображаться в теле писем с уведомлениями о проблеме.

      Плейсхолдеры

      Раздел Available placeholders на экране настройки перечисляет плейсхолдеры, которые можно использовать для этой интеграции. Плейсхолдеры автоматически заменяются на фактические значения в сообщении. * Выбери профиль оповещений для фильтрации ленты проблем. 5. Выбери Send test notification, чтобы убедиться, что интеграция с электронной почтой работает. 6. Save changes.

    Шаг 5 Создание первой дашборды¶

    Чтобы создать дашборд

    1. Перейди в Dashboards.
    2. Выбери Create Dashboard.
    3. Введи имя дашборда и выбери Create. Новая дашборда откроется в режиме редактирования.
    4. Чтобы добавить плитку, перетащи её из панели Tiles на дашборд. Например, перетащи плитку Host health на новую дашборду.

    Drag host health tile to dashboard 5. Выбери Done. Дашборд теперь отображается так, как он выглядит для тебя и людей, с которыми ты им поделился. (Другие люди не увидят кнопку Edit, если им не дать право на редактирование.)

    Чтобы закрепить график с экрана обзора хоста на новой дашборде

    1. Перейди в Hosts.
    2. Найди и выбери свой хост в таблице.
    3. На экране обзора хоста, в разделе Host performance, найди график CPU (central processing unit) usage.
    4. Выбери > Pin to dashboard.

    Pin CPU usage chart to dashboard 5. Выбери Pin. Это закрепит копию графика CPU usage на дашборде. 6. Выбери Open dashboard, чтобы вернуться к дашборде. Дашборд откроется в режиме редактирования с выбранной новой плиткой. 7. Выбери Done, чтобы сохранить изменения и отобразить обновлённую дашборду.

    Попробуй с помощью той же процедуры добавить ещё несколько плиток на дашборд.
