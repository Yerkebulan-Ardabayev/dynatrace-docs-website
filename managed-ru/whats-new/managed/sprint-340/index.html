    Что нового в Dynatrace Managed 1.340¶

    • Примечания к выпуску

    • Начало развёртывания 08 июня 2026 г.

    На этой странице представлены новые функции, изменения и исправления ошибок в Dynatrace Managed версии 1.340. Здесь содержится:

    • Обновления функций: 4
    • Критические изменения: 2
    • Исправления и обслуживание: 6

    Infrastructure Observability

    Обновлённая страница Network interface для более быстрого анализа¶

    Рады представить полностью переработанную страницу Network interface, которая обеспечивает более широкую видимость, удобную навигацию и более интуитивный процесс мониторинга производительности сети.

    Infrastructure Observability

    Обновлённая страница Disk для более быстрого анализа¶

    Рады представить полностью переработанную страницу Disk, которая обеспечивает более широкую видимость, удобную навигацию и более интуитивный процесс мониторинга производительности инфраструктуры.

    Platform

    Настраиваемое хранение и объём хранилища логов для каждого окружения¶

    Администраторы Dynatrace Managed теперь могут настраивать срок хранения логов и лимиты хранилища для каждого окружения напрямую в интерфейсе CMC в разделе настроек Environments. Срок хранения логов можно задать от 1 до 90 дней (с шагом в сутки), а объём хранилища можно ограничить конкретным значением в ГиБ или оставить неограниченным. Ранее срок хранения логов был фиксированным и составлял 35 дней без возможности его изменить. Изменения срока хранения применяются к новым записям логов начиная с момента изменения, а ежедневная очистка обеспечивает соблюдение настроенного периода.

    Platform

    Поддержка произвольного текста для фильтров, не относящихся к сущностям, в Data Explorer¶

    Поле Filter by в Data Explorer теперь принимает произвольный текст для фильтров, не относящихся к сущностям. По мере ввода текста появляется вариант с произвольным текстом с мгновенной обратной связью.

    Platform | Platform Services

    Обновление Elasticsearch до версии 8.19¶

    Узлы Elasticsearch обновлены до версии 8.19, что даёт исправления ошибок и уязвимостей, а также позволяет использовать улучшения производительности, добавленные в этом обновлении. Ручное вмешательство пользователя или простой не требуются: обновление происходит через поэтапные (rolling) обновления в рамках обычных обновлений версий.

    Критические изменения¶

    Application Observability

    Предстоящее принудительное применение версии окончания жизненного цикла OneAgent¶

    Начиная с версии Dynatrace Managed 1.342, кластер будет отклонять подключения от версий OneAgent 1.141 и более ранних.

    Может потребоваться действие:

    Если используются версии OneAgent 1.141 или более ранние, нужно обновиться до поддерживаемой версии OneAgent перед обновлением до версии Dynatrace Managed 1.342, чтобы избежать потери данных и проблем с подключением, а также воспользоваться улучшенной безопасностью и функциями, недоступными в более ранних версиях OneAgent.

    Подробнее см. Объявления об окончании жизненного цикла.

    Infrastructure Observability | Infrastructure & Operations

    Устаревание API автообновления ActiveGate¶

    API настройки автообновления ActiveGate объявлен устаревшим. Вместо него используйте API Settings 2.0 с идентификатором схемы deployment.activegate.updates.

    Исправления и обслуживание¶

    Устранённые проблемы в этом выпуске¶

    • Исправлена проблема, при которой настройки Deep monitoring / Process group monitoring, отключающие мониторинг, игнорировались для автономных/только-приложение и вручную внедрённых OneAgent. (PS-42728)
    • Исправлена проблема, при которой отображалось сообщение о том, что Python 3.14 не поддерживается в Linux. Сообщение было некорректным: фактический мониторинг процесса работает начиная с OneAgent версии 1.329 и выше. (MGD-11388)
    • Исправлена проблема, из-за которой RUM JavaScript добавлял заголовки к межисточниковым (cross-origin) запросам, если тег base указывает на другой источник, что приводило к потенциально нарушенным предварительным (preflight) запросам. (DEM-26180)
    • Исправлена проблема, приводившая к формированию последовательных визитов, если куки не удавалось установить. (DEM-25392)
    • Исправлена проблема, из-за которой инфографика на подстранице пользовательского устройства Technology & Processes иногда не отображалась из-за некорректной обработки отсутствующих данных. (DAQ-24380)
