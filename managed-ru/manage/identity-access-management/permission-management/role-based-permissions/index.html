    Разрешения на основе ролей¶

    Разрешения на основе ролей¶

    • Reference
    • 2-min read
    • Published Jul 01, 2024

    Миграция ролей в политики

    Рекомендовано

    Dynatrace версии 1.252+

    Начиная с Dynatrace версии 1.252, управлять доступом на основе ролей можно с помощью политик. Подробнее см. в разделе Обновление разрешений на основе ролей до политик Dynatrace IAM.

    Перейдите в Account Management — дополнительные сведения см. в разделе Account Management.

    Разрешения уровня учётной записи¶

    Dynatrace предоставляет следующие разрешения уровня учётной записи.

    • View account: разрешает доступ к учётной записи для просмотра данных окружения (часы хостов, сессии, синтетические мониторы) и ссылок на документацию Dynatrace. Также разрешает просмотр и создание обращений в поддержку и доступ к форуму Dynatrace Community. Доступ к биллингу и управлению пользователями/группами отсутствует.
    • View and manage account and billing information: разрешает доступ к данным об оплате (реквизиты банковской карты), данным биллинга (счета) и контактной информации (контактные данные компании).
    • View and manage users and groups: разрешает доступ к управлению пользователями (добавление, редактирование, удаление из группы) и группами (создание, редактирование, удаление).

    Разрешения уровня окружения¶

    Dynatrace предоставляет следующие разрешения уровня окружения. Выберите все применимые:

    • View environment: разрешает доступ к окружению только для чтения. Одного этого разрешения недостаточно для изменения настроек или установки OneAgent.

    Разрешение View environment необходимо для любого другого разрешения уровня окружения, поэтому при выборе любого другого разрешения уровня окружения оно выбирается автоматически.

    • View sensitive request data: разрешает просмотр потенциальных персональных данных, собираемых Dynatrace, включая загрузку дампов памяти. Пользователи без этого разрешения видят, что точка данных существует, но персональные данные маскируются звёздочками (*****). Также позволяет вручную запускать дампы памяти.

    • View logs: разрешает доступ к конфиденциальным данным файлов журналов на вкладке Logs.

    Роль View logs обходит существующий условный доступ, определённый в границах политик через разрешение storage logs. Чтобы соблюдать эти условия, рекомендуется выразить те же разрешения View logs в операторе политики, используя эквивалентное разрешение просмотра журналов окружения.

    • Replay session data with masking: разрешает воспроизведение записанных пользовательских сессий с применением правил маскирования воспроизведения во время воспроизведения. Данные, маскированные при записи, никогда не захватываются и, следовательно, всегда маскируются при воспроизведении.

    • Replay session data without masking: разрешает воспроизведение записанных пользовательских сессий без применения правил маскирования воспроизведения. Данные, маскированные при записи, всегда маскируются при воспроизведении.

    • Install OneAgent: разрешает загрузку OneAgent и его установку на хостах. Для изменения/редактирования настроек необходимо предоставить разрешение Manage monitoring settings.

    • Manage monitoring settings: разрешает изменение всех настроек окружения. Для установки OneAgent необходимо предоставить разрешение Install OneAgent.

    • Manage capturing of sensitive request data: разрешает настройку правил захвата атрибутов запросов. Их можно использовать для захвата таких элементов, как HTTP-заголовки или параметры POST, для хранения, фильтрации и поиска. Также позволяет вручную запускать дампы памяти.

    • Manage support tickets: разрешает доступ ко всем обращениям в поддержку, созданным для данного окружения.

    • Manage security problems: разрешает просмотр уязвимостей, выявленных Dynatrace Application Security, и управление ими.

    • View security problems: разрешает просмотр (но не управление) уязвимостей, выявленных Dynatrace Application Security.

    Подробные сведения о разрешениях Application Security см. в разделе Точная настройка разрешений.

    Разрешения уровня зоны управления¶

    Dynatrace предоставляет следующие разрешения уровня зоны управления. Выберите все применимые:

    • View environment: разрешает доступ только для чтения к объектам в зоне управления. Для изменения/редактирования настроек необходимо предоставить разрешение Change monitoring settings.

    Разрешение View environment необходимо для любого другого разрешения уровня зоны управления, поэтому при выборе любого другого разрешения уровня зоны управления оно выбирается автоматически.

    • View sensitive request data: разрешает просмотр потенциальных персональных данных, собираемых Dynatrace, для объектов в зоне управления. Пользователи без этого разрешения видят, что точка данных существует, но персональные данные маскируются звёздочками (*****) — см. также раздел Разрешения уровня окружения выше.

    • View logs: разрешает доступ к конфиденциальным данным файлов журналов на вкладке Logs для хостов, явно включённых в зону управления. Обратите внимание, что недостаточно предоставить доступ уровня зоны управления к группам хостов, которым принадлежат эти хосты — см. раздел Правила зоны управления.

    • Replay session data with masking: разрешает воспроизведение записанных пользовательских сессий с применением правил маскирования воспроизведения во время воспроизведения. Данные, маскированные при записи, никогда не захватываются и, следовательно, всегда маскируются при воспроизведении.

    • Replay session data without masking: разрешает воспроизведение записанных пользовательских сессий без применения правил маскирования воспроизведения. Данные, маскированные при записи, всегда маскируются при воспроизведении.

    Для работы разрешений Session Replay в рамках зоны управления пользователю также необходим доступ к соответствующим приложениям.

    • Если пользовательская сессия охватывает несколько приложений, не все из которых назначены зоне управления, пользователи всё равно могут видеть и воспроизводить сессию. Однако пользовательские действия, связанные с приложением, к которому доступ отсутствует, маскируются, а соответствующая часть воспроизведения не отображается.
    • Кнопки воспроизведения недоступны для пользователей, имеющих доступ к приложениям, но не имеющих разрешения на воспроизведение сессий.

    Подробнее о зонах управления см. в разделе Зоны управления.

    • Manage monitoring settings: разрешает изменение настроек объектов в зоне управления, например создание или редактирование синтетических мониторов. Также предоставляет доступ к некоторым пунктам глобального меню настроек, но разрешает вносить изменения только для назначенных зон управления. Например, профили оповещения о проблемах можно создавать и изменять только для конкретной зоны управления.

    • Manage security problems: разрешает просмотр уязвимостей, выявленных Dynatrace Application Security, и управление ими.

    • View security problems: разрешает просмотр (но не управление) уязвимостей, выявленных Dynatrace Application Security.

    Подробные сведения о разрешениях Application Security см. в разделе Точная настройка разрешений.

    Соотношение разрешений окружения и зоны управления¶

    При предоставлении любого разрешения, кроме View environment, на уровне окружения разрешение View environment также автоматически включается для окружения. Аналогично, при предоставлении любого разрешения, кроме View environment, на уровне зоны управления разрешение View environment автоматически включается для зоны управления.

    Зоны управления предназначены для обеспечения целевого и ограниченного доступа к определённым объектам в окружении. Если необходимо предоставить разрешение пользователям, работающим в зоне управления, рекомендуется использовать разрешения уровня зоны управления. Любое разрешение, предоставленное на уровне окружения, заменяет и дополняет разрешения уровня зоны управления. Иными словами, разрешения зоны управления не могут ограничивать разрешения, уже предоставленные на уровне окружения.

    Рассмотрим пример зоны управления, содержащей три хоста из пяти в окружении. Если предоставить разрешение View logs для зоны управления, наблюдатели увидят вкладку Logs с информацией по трём хостам в зоне управления. Однако если убрать это разрешение на уровне зоны управления и предоставить его на уровне окружения, пользователи смогут:

    • Получить доступ ко всем зонам управления через фильтр зон управления в строке меню.
    • Видеть вкладку Logs для всех пяти хостов в окружении при просмотре всех зон управления.
    • Видеть вкладку Logs для трёх хостов в назначенной зоне управления при переключении на неё.
