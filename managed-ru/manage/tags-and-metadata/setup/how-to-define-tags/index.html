    Определение и применение тегов¶

    Определение и применение тегов¶

    • How-to guide
    • 14-min read
    • Published Jul 19, 2017

    Используйте теги для организации данных мониторинга и анализа на основе связанных объектов во всех окружениях. Теги упрощают поиск связанных сервисов, групп процессов и хостов. Они также облегчают сбор связанных метрик в значимые группы для анализа. Теги можно использовать для всех типов объектов в окружении в целях оповещения, построения диаграмм и управления окнами обслуживания. Подробнее об определении окон обслуживания с помощью тегов см. раздел Как определить окно обслуживания.

    Теги можно применять вручную или автоматически.

    • Ручное тегирование хорошо подходит для небольших задач, где создание правил тегирования нецелесообразно при наличии лишь нескольких статичных объектов.
    • Автоматическое тегирование на основе правил отлично подходит для динамичных или крупных окружений, где ручное тегирование может быть непрактично. Определение правил автоматического тегирования требует начальных усилий, однако значительно проще и надёжнее попыток вручную поддерживать теги для тысяч объектов в постоянно меняющемся окружении.

    Ручное тегирование¶

    В небольших окружениях теги можно настроить вручную, следуя шагам ниже.

    Просмотр всех тегов, применённых вручную¶

    Чтобы вывести список всех тегов, применённых вручную к любым объектам:

    1. Перейдите в Settings и выберите Tags > Manually applied tags.

    2. Необязательно. В поле Filter by name введите строку фильтрации для отображения всех вручную применённых тегов, соответствующих вашему запросу:

    3. Введите частичную строку для поиска всех тегов, которые начинаются с, заканчиваются на или содержат строку (bc соответствует bcd, abc и abcd)

    4. Используйте двоеточие для поиска пары «ключ:значение»
    5. В крупном окружении может потребоваться фильтрация для сокращения списка до управляемого размера или использование основанного на правилах автоматического тегирования.

    Чтобы вывести список всех тегов, применённых вручную к конкретному объекту (приложению, монитору браузера, HTTP-монитору, сервису, хосту или процессу):

    1. Откройте страницу конкретного объекта (например, страницу сведений о хосте).

    2. Теги, применённые к этому объекту, перечислены под заголовком страницы.

    3. Можно просматривать, добавлять и удалять теги, применённые вручную, непосредственно со страницы объекта.

    4. Теги, применённые автоматически, также перечислены, но их нельзя добавлять или удалять напрямую — они применяются на основе правил тегирования, определённых для автоматического тегирования.

    Ручное применение или снятие тега¶

    Теги можно применять и снимать вручную:

    • Со страницы Manually applied tags — удобнее для применения или снятия тегов на нескольких объектах
    • Со страницы объекта — удобнее при работе с конкретным объектом, например с одним хостом

    Ручное тегирование нескольких объектов¶

    Чтобы вручную применить или снять тег для нескольких объектов:

    1. Перейдите в Settings и выберите Tags > Manually applied tags.
    2. Найдите и выберите существующий тег или создайте новый.

    Чтобы найти и выбрать существующий тег:

    1. В поле Filter by name введите строку фильтрации для отображения всех вручную применённых тегов:

      • Введите частичную строку для поиска тегов, которые начинаются с, заканчиваются на или содержат строку
      • Используйте двоеточие для поиска пары «ключ:значение»
      • Найдя нужный тег, выберите его имя. Отобразится таблица компонентов, к которым можно применить теги вручную.

    Чтобы создать новый тег:

    1. В поле Key (required) введите имя нового тега.
    2. Необязательно. В поле Value введите значение (для создания пары «ключ:значение»).
    3. Нажмите Add. Тег и его необязательное значение будут созданы, после чего отобразится таблица компонентов, к которым можно применить новый тег.

    4. По умолчанию в таблице отображаются приложения.

    5. Если применяется существующий тег, выбрано значение Tagged, а установленные флажки указывают приложения, к которым тег уже применён

    6. Если создаётся новый тег, выбрано значение Untagged и ни один флажок не установлен (поскольку тег ещё не существует)
    7. Необязательно. Измените Applications на All types или другой конкретный тип компонента в зависимости от того, к каким компонентам нужно применить тег.

    Типы компонентов, к которым можно применять теги вручную:

    • Applications
    • Browser monitors
    • HTTP monitors
    • Services
    • Hosts
    • ESXi
    • AWS
    • Azure Scale sets
    • Processes
    • Process groups
    • Custom devices
    • Custom device groups
    • Необязательно. Измените Tagged на All или Untagged.

    Параметры All/Tagged/Untagged:

    • All — показывать все компоненты выбранного типа, независимо от наличия выбранного тега
    • Tagged — показывать все компоненты выбранного типа, к которым уже применён выбранный тег
    • Untagged — все компоненты выбранного типа, к которым выбранный тег ещё не применён
    • Необязательно. Введите значение в поле редактирования.

    Значение — для пары «ключ:значение». 7. Установите или снимите флажок для каждого компонента, к которому нужно применить или снять выбранный тег.

    • Установите флажок Tag component для применения тега ко всем совпадающим компонентам
    • Снимите флажок Tag component для удаления тега со всех совпадающих компонентов

    Ручное тегирование одного объекта¶

    Чтобы вручную применить или снять тег для одного объекта (приложения, монитора браузера, HTTP-монитора, сервиса, хоста или процесса), удобнее начать со страницы объекта.

    Например, чтобы вручную добавить тег одному хосту:

    1. Перейдите в Hosts.
    2. Выберите хост, который нужно тегировать.

    3. Разверните раздел Properties and tags под заголовком страницы для просмотра всех тегов, применённых к хосту вручную или автоматически.

    4. Теги, применённые вручную, имеют кнопку удаления, что означает возможность их удаления вручную.

    5. Теги, применённые автоматически, не имеют кнопки удаления. При наведении на автоматически применённый тег появляется всплывающая подсказка с ключом и значением тега, за которыми следует фраза (applied by tagging rule).
    6. Нажмите Add tag.

    7. В появившемся окне введите обязательный Key, необязательно введите Value и нажмите Add.

    8. Если вместо этого нужно тегировать несколько объектов (например, хостов), нажмите Tag settings в всплывающем окне для перехода на страницу Manually applied tags, а затем следуйте инструкциям выше по ручному тегированию нескольких объектов.

    Автоматическое тегирование¶

    Автоматическое тегирование основано на правилах:

    • Теги автоматически применяются к новым объектам, соответствующим определённым вами правилам.
    • Автоматически применённые теги нельзя удалять вручную с отдельных сервисов, групп процессов, экземпляров групп процессов, приложений или хостов.

    Автоматическое тегирование объектов в окружении можно также настроить с помощью:

    • Topology and Smartscape API
    • Переменных окружения
    • Файла конфигурации автотегирования хоста (только для хостов)

    Как определить автоматически применяемый тег¶

    Чтобы автоматически применять тег:

    1. Перейдите в Settings и выберите Tags > Automatically applied tags.
    2. Нажмите Create tag.
    3. Введите имя нового тега в поле Tag name.
    4. Нажмите Add new rule.
    5. Укажите необязательное Optional tag value. Это значение будет отображаться рядом с именем тега после : и используется для предоставления более точной информации. Значением может быть свойство объекта (например, сервиса или группы процессов), к которому применяется тег. Если выбранное свойство недоступно, значение тега останется пустым. Обратите внимание, что заполнители необязательных значений тегов не работают для правил на основе entity selector.
    6. В списке Rule applies to выберите тип объекта, к которому нужно применить правило (например, Services, Process groups или Hosts).
    7. Необязательно. Ограничьте правило конкретными объектами, задав предоставленные свойства с помощью соответствующих списков.
    8. Можно напрямую определить условие, которому объект должен соответствовать перед применением тега. Для добавления дополнительных условий нажмите Add condition. Условия проверяют конкретные значения любого свойства, доступного в раскрывающемся списке Conditions слева. Пример правила для тега с именем ocp-project показан на изображении ниже.

    Страница создания правила для нового тега. 9. Правило также можно распространить на базовые объекты (например, группы процессов и хосты в случае сервисов, или хосты в случае групп процессов), установив соответствующие флажки в нижней части правила (см. изображение выше). 10. Нажмите кнопку Preview для проверки результатов конкретного правила. Обратите внимание: для получения тега объект должен соответствовать всем указанным условиям правила. 11. Нажмите Create rule. Для каждого тега можно определить несколько правил. Правила выполняются по порядку. Определённое правило можно редактировать, удалять или активировать/деактивировать в любое время с помощью переключателя Disable/Enable. 12. Для сохранения тега нажмите Save changes в правом нижнем углу страницы.

    При выборе тега отображаются правила, определённые для него, а также объекты, соответствующие всем определённым правилам, в области Matching entities. Автоматически применённые теги применяются ко всем существующим и вновь обнаруженным объектам (сервисам, группам процессов и другим). Обратите внимание, что до применения нового тега может пройти до одной минуты. После применения тега к объекту он отображается на странице этого объекта. Например, для сервисов каждый новый тег перечислен на странице обзора сервиса в разделе Properties and tags. Тег ocp-project, определённый в примере выше, отображается на странице обзора ocp AuthenticationService в разделе Properties and tags с дополнительным значением demo-live-1 (см. изображение ниже).

    Каждое правило тегирования является самостоятельным и оценивается независимо от других существующих правил.

    Регулярные выражения¶

    Можно создавать условия для автоматически применяемых тегов на основе регулярных выражений. На шаге 8 выше выберите нужное свойство в раскрывающемся списке Conditions слева. Затем выберите contains regex в следующем раскрывающемся списке и введите регулярное выражение в текстовое поле.

    Автоматически применённые теги нельзя использовать в условиях для автоматизированного тегирования на основе правил.

    Свойства сервисов и групп процессов, доступные для тегирования¶

    Свойства сервисов и групп процессов, доступные для тегирования, зависят от типа технологии.

    Чтобы узнать, какие свойства предоставляет сервис:

    1. Перейдите в Services.
    2. Выберите сервис, который нужно тегировать.
    3. Выберите Properties and tags для отображения доступных свойств.

    Чтобы узнать, какие свойства предоставляет группа процессов:

    1. Перейдите в Hosts.
    2. Выберите хост, содержащий нужную группу процессов.
    3. Выберите Properties and tags для отображения доступных свойств.

    Примеры использования тегов¶

    Теги сервисов, групп процессов и хостов можно использовать различными способами. Ниже приведены два примера.

    Сужение области анализа¶

    Теги можно использовать для сужения анализа до определённого набора сервисов.

    1. Перейдите в Services.
    2. В поле Filtered by выберите Tag, затем введите и выберите имя тега. Повторите для каждого тега, по которому нужно фильтровать таблицу.

    После выбора группы связанных сервисов по тегам легко сосредоточить анализ на этих сервисах. Например:

    Dashboards Classic

    1. Отфильтруйте страницу Services по одному или нескольким тегам.
    2. Нажмите кнопку Pin to dashboard для добавления плитки мониторинга работоспособности на дашборд для сервисов, соответствующих выбранным тегам.

    3. С дашборда теперь можно:

    4. Наблюдать за плиткой для мониторинга выбранных сервисов с первого взгляда

    5. Нажать на плитку для перехода к странице Services, отфильтрованной по тегам из шага 1.

    Обеспечение эффективной маршрутизации уведомлений о проблемах¶

    Теги также можно использовать для эффективной маршрутизации уведомлений о проблемах ответственным членам команды.

    Например, предположим, что вы определили тег TeamBoston для всех объектов, за которые отвечает эта команда. Чтобы члены команды получали уведомления о проблемах:

    1. Используйте этот тег для определения профиля оповещений.
    2. Перейдите в Settings и выберите Integration > Problem notifications.
    3. Нажмите Set up notifications.
    4. Выберите подходящую систему управления инцидентами или канал совместной работы команды.
    5. Выберите профиль оповещений (из шага 1) в списке Alerting profile.

    При следующей отправке уведомления о проблеме Dynatrace проверит, несут ли затронутые сервисы свойства, определённые в тегах сервисов. Когда критически важные части окружения затронуты обнаруженной проблемой, соответствующее уведомление будет доставлено соответствующим командам.
