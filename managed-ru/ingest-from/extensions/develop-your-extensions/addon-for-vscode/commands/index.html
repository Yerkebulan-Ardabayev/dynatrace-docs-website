    Справочник по командам¶

    Справочник по командам¶

    • Справочник
    • Чтение: 11 мин
    • Опубликовано 16 июня 2025 г.

    Все основные рабочие процессы, автоматизированные дополнением Dynatrace Extensions, доступны как команды VS Code из палитры команд редактора. Чтобы открыть палитру команд, нажмите клавишу F1. Там находятся все доступные команды с префиксом Dynatrace extensions.

    Обзор команд¶

    Команда Описание
    Initialize workspace Регистрация и инициализация рабочего пространства. Первый шаг для начала работы
    Load schemas Загрузка файлов схем и настройка проверки манифеста
    Generate certificates Генерация разработческих сертификатов для подписи расширений
    Distribute certificate Распространение корневого сертификата на компоненты, которые его используют
    Build Сборка расширения и создание подписанного пакета
    Upload Загрузка последнего пакета расширения в окружение Dynatrace
    Activate Активация версии текущего расширения
    Create documentation Генерация файла README.md с описанием расширения и его содержимого
    Create a dashboard Генерация лендингового дашборда для включения в расширение
    Create alert Генерация событий метрик для оповещений для включения в расширение
    Convert JMX Конвертация расширения Java Management Extensions (JMX) в платформу 2.0

    Initialize workspace¶

    Эта команда инициализирует новое или существующее рабочее пространство для разработки Extension 2.0 и регистрирует его в дополнении VS Code. В процессе инициализации создаётся начальная структура папок и шаблонные файлы расширения, чтобы любой проект начинался корректно.

    Подробнее об этой команде см. в руководстве по началу работы.

    Предварительные требования¶

    • В окне VS Code должно быть открыто рабочее пространство или папка.
    • Требуется подключение к окружению Dynatrace.

    Шаги рабочего процесса¶

    1. Выделяет внутреннее хранилище для проекта.
    2. Запускает рабочий процесс из Load schemas. Этот шаг можно пропустить, если рабочее пространство уже зарегистрировано в Dynatrace Extensions.

    3. Связывает сертификаты с рабочим пространством. Доступны варианты:

    4. Use existing: проверяет настройки разработческого сертификата и ключа.

    5. Generate new ones: запускает рабочий процесс из Generate certificates.
    6. Регистрирует рабочее пространство в Dynatrace Extensions.
    7. Создаёт папку проекта и файлы на основе выбранного шаблона.

    Шаблоны проектов¶

    Extension 2.0¶

    Это выбор по умолчанию: если все содержимое уже есть и нужно зарегистрировать рабочее пространство в дополнении, или если начинается новое расширение с нуля. Создаётся папка расширения и манифест с минимально обязательными данными для любого расширения.

    Python extension 2.0¶

    Этот вариант создаёт новое расширение, использующее источник данных Python. В процессе настройки также загружается и устанавливается модуль Dynatrace Extensions SDK для Python.

    JMX 1.0 conversion¶

    Источник данных JMX теперь доступен в Extensions, и пришло время конвертировать расширения JMX из платформы 1.0. Этот тип проекта предложит указать расширение JMX 1.0 (из локального файла или тенанта) и конвертирует его в новую платформу с созданием манифеста.

    Existing 2.0 extension¶

    Нужно отредактировать уже развёрнутое расширение или посмотреть, как выглядит содержимое расширения, собранного Dynatrace? Этот вариант загружает пакет расширения 2.0 из тенанта и распаковывает его в рабочее пространство.

    Load schemas¶

    Эта команда загружает файлы схем определённой версии из подключённого окружения Dynatrace и настраивает проверку манифеста расширения. Если в проекте уже есть манифест расширения, он обновляется до выбранной версии.

    Предварительные требования¶

    • Требуется подключение к окружению Dynatrace.

    Шаги рабочего процесса¶

    1. Запрашивается список доступных версий схем в окружении Dynatrace, затем предлагается выбрать нужную версию.

    2. Файлы выбранной версии схемы загружаются и сохраняются в глобальном хранилище Visual Studio Code.

    3. Если файлы уже загружались ранее, появится запрос с возможностью пропустить этот шаг.

    4. Настройки рабочего пространства обновляются для включения проверки манифеста расширения по выбранной схеме.

    5. Если в рабочем пространстве есть файл манифеста, он будет обновлён до выбранной минимальной версии.

    Generate certificates¶

    Эта команда генерирует все учётные данные, необходимые для подписи и проверки Extensions.

    Предварительные требования¶

    • В окне VS Code должно быть открыто рабочее пространство или папка.

    Шаги рабочего процесса¶

    1. Генерируется пара ключей RSA для создания корневого сертификата.
    2. Генерируется ещё одна пара ключей RSA, которая используется для создания разработческого сертификата; корневой сертификат добавляется как удостоверяющий центр для этих учётных данных.

    3. Разработческий сертификат создаётся из пары ключей RSA, а корневой сертификат добавляется как удостоверяющий центр для этого сертификата.

    4. Примечание: все параметры, используемые при генерации сертификатов, можно настроить в настройках.

    5. Все промежуточные файлы сохраняются в хранилище рабочего пространства VS Code, а настройки учётных данных для этого рабочего пространства обновляются путями к сгенерированным файлам. Это выполняется в файле ./vscode/settings.json.
    6. Запускается рабочий процесс из Distribute certificate.

    Distribute certificate¶

    Эта команда загружает корневой сертификат рабочего пространства в хранилище учётных данных Dynatrace. Если дополнительно обнаруживаются локально установленные OneAgent или ActiveGate, сертификат загружается и в них.

    Предварительные требования¶

    • В окне VS Code должно быть открыто рабочее пространство или папка.
    • Требуется подключение к окружению Dynatrace.
    • Настройка dynatraceExtensions.rootOrCaCertificate должна быть задана глобально или для рабочего пространства.

    Шаги рабочего процесса¶

    1. Дополнение проверяет, связан ли с рабочим пространством идентификатор записи в хранилище учётных данных Dynatrace, и запрашивает, следует ли перезаписать запись.

    2. При выборе перезаписи запись обновляется новым файлом.

    3. В противном случае рабочий процесс продолжается с созданием новой записи в хранилище учётных данных.
    4. Предлагается указать имя учётных данных и необязательное описание, после чего файл загружается с этими данными.

    5. Проверяется наличие локальных путей OneAgent и ActiveGate, и предлагается также загрузить сертификат в эти места.

    6. В Windows:

      • OneAgent: %PROGRAMDATA%\dynatrace\oneagent\agent\config\certificates
      • ActiveGate: %PROGRAMDATA%\dynatrace\remotepluginmodule\agent\conf\certificates

      • В Linux:

      • OneAgent: /var/lib/dynatrace/oneagent/agent/config/certificates

      • ActiveGate: <CONFIG>/remotepluginmodule/agent/conf/certificates/

    Этот шаг требует запуска VS Code с правами администратора. Например, в Windows используйте Run As Administrator.

    Build¶

    Эта команда собирает расширение и упаковывает его в подписанный ZIP-архив, который помещается в папку dist рабочего пространства.

    Предварительные требования¶

    • В окне VS Code должно быть открыто зарегистрированное рабочее пространство.
    • С рабочим пространством должны быть связаны разработческие сертификаты.
    • В манифесте расширения не должно быть ошибок.

    Шаги рабочего процесса¶

    1. Версия расширения берётся из манифеста. При подключении к окружению Dynatrace версия проверяется на конфликты и при необходимости автоматически увеличивается.
    2. Манифест расширения и ресурсы упаковываются в ZIP-архив, подписанный разработческими учётными данными. Полученная подпись и архив добавляются в итоговый ZIP-файл, представляющий пакет расширения.
    3. При подключении к окружению Dynatrace пакет проверяется по нему. Ошибки проверки отображаются в Output Channel (в окне редактора), после чего рабочий процесс завершается. В папку dist рабочего пространства перемещаются только допустимые пакеты.
    4. Запускается рабочий процесс из Upload.

    Upload¶

    Эта команда загружает последний пакет из папки dist рабочего пространства в подключённое окружение.

    Предварительные требования¶

    • В окне VS Code должно быть открыто зарегистрированное рабочее пространство.
    • Требуется подключение к окружению Dynatrace.
    • В папке dist рабочего пространства должен находиться ZIP-архив.

    Шаги рабочего процесса¶

    1. Проверяется возможность загрузки путём запроса количества уже развёрнутых версий расширения.

    2. Если версий слишком много, предлагается удалить самую старую.

    3. Дополнение пытается удалить самую старую версию; если это не удаётся (например, с ней могут быть связаны конфигурации мониторинга), предлагается выбрать другую версию для удаления.
    4. Если загрузка возможна, расширение загружается.
    5. Запускается рабочий процесс из Activate для активации этой версии.

    Activate¶

    Эта команда активирует версию расширения рабочего пространства.

    Предварительные требования¶

    • В окне VS Code должно быть открыто зарегистрированное рабочее пространство.
    • Требуется подключение к окружению Dynatrace.

    Шаги рабочего процесса¶

    1. Имя расширения считывается из манифеста рабочего пространства.

    2. В окружении Dynatrace запрашиваются доступные версии расширения, и предлагается выбрать, какую из них активировать.

    3. Этот шаг пропускается, если команда вызвана из другого рабочего процесса, например из Upload.

    4. Выбранная версия активируется в окружении Dynatrace.

    Create documentation¶

    Эта команда автоматически генерирует подробную документацию в файле README.md, анализируя пакет расширения и его содержимое.

    Предварительные требования¶

    • В окне VS Code должно быть открыто зарегистрированное рабочее пространство.

    Шаги рабочего процесса¶

    1. Команда просматривает манифест расширения и извлекает все универсальные топологические сущности.
    2. Затем из манифеста извлекаются метрики.
    3. Затем из манифеста извлекаются дашборды.
    4. После этого оповещения преобразуются в понятные сводки.
    5. Метрики сопоставляются с наборами функций и связываются с определёнными сущностями.
    6. Создаётся файл readme; при необходимости отсутствующие сведения пропускаются.

    Create a dashboard¶

    Эта команда просматривает манифест расширения и создаёт обзорный дашборд, служащий лендинговой страницей расширения.

    Предварительные требования¶

    • В окне VS Code должно быть открыто зарегистрированное рабочее пространство.
    • Манифест расширения должен содержать как минимум определение топологии.

    Шаги рабочего процесса¶

    1. Команда извлекает универсальные типы сущностей, определённые в манифесте, и все связанные с ними метрики. Для каждого типа сущностей берётся одна или две первые метрики.

    2. Дашборд создаётся из шаблона, который содержит следующее:

    3. Плитки с одним значением, показывающие количество уникальных отслеживаемых сущностей.

    4. Список ссылок в формате markdown, предоставляющих быстрый переход к экрану единого анализа каждой сущности.
    5. Таблицы для каждого типа сущностей и графики на основе связанных метрик.
    6. Дашборд сохраняется в файл ./extension/dashboards/overview_dashboard.json, а манифест расширения обновляется для включения ссылки на этот ресурс.
    7. Рабочий процесс завершается запросом о загрузке дашборда в окружение Dynatrace.

    Дашборд будет автоматически загружен в процессе развёртывания расширения. Этот последний шаг предлагается, если нужен предварительный просмотр ресурса до развёртывания расширения.

    Create alert¶

    Эта команда создаёт событие метрики на основе метрик, определённых в манифесте расширения.

    Предварительные требования¶

    • В окне VS Code должно быть открыто зарегистрированное рабочее пространство.
    • Манифест расширения должен содержать метаданные метрик.

    Шаги рабочего процесса¶

    1. Команда разбирает манифест расширения и отображает список всех доступных метрик. Выберите одну для продолжения.
    2. Затем предлагается ввести заголовок для оповещения.
    3. Нужно выбрать, срабатывает ли порог, когда метрика превышает (above) или опускается ниже (below) заданного уровня.
    4. Наконец, укажите фактическое значение порога оповещения.
    5. Команда завершается записью JSON-файла оповещения в папку ./extension/alerts и обновлением манифеста расширения для включения созданного оповещения.

    Convert JMX¶

    Эта команда конвертирует существующее расширение JMX в Extension 2.0.

    Шаги рабочего процесса¶

    1. Предлагается выбрать способ загрузки расширения JMX:

    2. Locally: перейдите в файловой системе и выберите JSON- или ZIP-файл, содержащий расширение JMX.

    3. Remotely: просмотрите расширения JMX, доступные на подключённом тенанте.
    4. Если технология процесса не определяется автоматически, предлагается выбрать её вручную.
    5. Запрашивается, включать ли данные на странице сведений о хосте.
    6. Рабочий процесс обрабатывает JSON-файл расширения JMX и конвертирует его в эквивалентный манифест расширения 2.0.
    7. Рабочий процесс сохраняет сгенерированный манифест в extension/extension.yaml или запрашивает место сохранения, если эта папка отсутствует в рабочем пространстве.

    Ознакомьтесь с этим рабочим процессом подробнее в руководстве по конвертации JMX 1.0.
