    Сетевой трафик¶

    Сетевой трафик¶

    • Чтение: 3 мин
    • Обновлено 02 января 2026 г.

    Чтобы компоненты Dynatrace Operator работали правильно в кластере Kubernetes, они должны иметь возможность взаимодействовать как с кластером Dynatrace, так и с кластером Kubernetes.

    Компоненты Dynatrace Operator доступны через определённые порты и обращаются к различным ресурсам внутри и снаружи кластера Kubernetes. Подробнее о том, к каким ресурсам выполняется обращение внутри кластера Kubernetes, см. справочную страницу Разрешения RBAC для Operator.

    Входящий трафик¶

    Источник Назначение Порт Примечание
    kubelet Dynatrace Operator /healthz TCP 10080 Liveness probe 1
    Prometheus metrics scraper Необязательно Dynatrace Operator /metrics TCP 8080 Адрес метрик 2
    kubelet Dynatrace Webhook /healthz TCP 10080 Liveness/Readiness probe 1
    kube-apiserver Dynatrace Webhook /inject, /label-ns, /validate* TCP 8443 Dynamic Admission Controller
    Prometheus metrics scraper Необязательно Dynatrace Webhook /metrics TCP 8080 Адрес метрик 2
    kubelet Dynatrace Operator CSI driver server container /healthz TCP 9808 Liveness probe 1
    kubelet Dynatrace Operator CSI driver provisioner container /healthz TCP 10090 Liveness probe 1
    Prometheus metrics scraper Необязательно Dynatrace Operator CSI driver server container /metrics TCP 8080 Адрес метрик 2
    Prometheus metrics scraper Необязательно Dynatrace Operator CSI driver provisioner container /metrics TCP 8090 Адрес метрик 2
    kubelet ActiveGate /rest/health TCP 9999 Readiness probe 1
    kubelet Extension Execution Controller /readyz TCP 14599 Readiness probe 1
    Application pods ActiveGate /* TCP 9999 Порт HTTPS по умолчанию
    Application pods ActiveGate /* TCP 9998 Порт HTTP по умолчанию, приём данных, доступ к API
    Application pods Dynatrace Collector Порты приёма телеметрии Приём данных телеметрии
    kubelet SQL Extension Executor container /health/live TCP 8080 Liveness probe 1
    kubelet SQL Extension Executor container /health/ready TCP 8080 Readiness probe 1

    1

    Проверки Liveness используются Kubernetes для проверки того, что контейнер работает правильно. Если запрос не выполняется, контейнер будет перезапущен. Проверки Readiness используются Kubernetes для проверки того, что под готов принимать трафик.

    2

    Конечные точки метрик выдают дополнительные метрики в формате Prometheus.

    Входящий трафик для EdgeConnect и OneAgent не принимается.

    Исходящий трафик¶

    Компоненты Dynatrace Operator должны обращаться как к кластеру Kubernetes, так и к ресурсам за пределами кластера, чтобы работать правильно. Все ресурсы в пространстве имён Dynatrace Operator, где пространством имён по умолчанию является dynatrace, должны иметь возможность разрешать DNS-запросы.

    В зависимости от вашей конфигурации порт по умолчанию может отличаться от TCP 443.

    Источник Назначение Порт Примечание
    * Dynatrace Operator * Dynatrace Webhook * Dynatrace Operator CSI driver * ActiveGate * Extension Execution Controller kube-dns TCP 53, UDP 53 1 Разрешение имён хостов для обнаружения сервисов
    Dynatrace Operator Dynatrace server TCP 443 1 Настройка на стороне сервера 2
    Dynatrace Operator kube-apiserver TCP 443 1 Управление жизненным циклом компонентов
    Dynatrace Webhook kube-apiserver TCP 443 1 Запросы Mutating/Validating/Conversion
    Dynatrace Operator CSI driver Dynatrace server TCP 443 1 Расположение по умолчанию для двоичных файлов модуля кода 2
    Dynatrace Operator CSI driver kube-apiserver TCP 443 1 Обработка томов CSI
    Dynatrace Operator CSI driver private registry TCP 443 1 Необязательно Взаимодействие с приватным реестром для доступа к модулям кода 3
    ActiveGate Communication endpoints 4 TCP 443, TCP 9999 1 Информация наблюдаемости 2
    ActiveGate kube-apiserver TCP 443 1 Сбор ресурсов
    ActiveGate Application Pods Prometheus Exporter port 1 Сбор метрик
    OneAgent Communication endpoints 4 TCP 443, TCP 9999 1 Информация наблюдаемости 2
    EdgeConnect Dynatrace server TCP 443 1 Настройка на стороне сервера 2
    EdgeConnect kube-apiserver TCP 443 1 Необязательно Взаимодействия с Workflow 5
    Extension Execution Controller ActiveGate TCP 443 1 Настройка расширений и данные телеметрии 2

    1

    В зависимости от вашей конфигурации порт может отличаться от значения по умолчанию.

    2

    Взаимодействие с хостами должно быть разрешено в соответствии с настройкой в пользовательских ресурсах DynaKube (apiUrl) или EdgeConnect (apiServer). Для обеспечения корректного подключения в качестве резервных могут использоваться разные конечные точки взаимодействия.

    3

    Требуется только при использовании поля codeModulesImage.

    4

    Поддерживаемые схемы подключения для ActiveGate

    5

    Требуется только при включённой Kubernetes Automation.
