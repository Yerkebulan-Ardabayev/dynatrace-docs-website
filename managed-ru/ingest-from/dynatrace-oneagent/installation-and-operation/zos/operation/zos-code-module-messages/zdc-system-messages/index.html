    Сообщения модулей z/OS - системные сообщения zDC¶

    • Чтение: 63 мин
    • Обновлено 10 февраля 2026 г.

    ZDC000I¶

    • Полное сообщение - ZDC000I INITIALIZATION @@ FOR zDC @@ VER @@.@@.@@
    • Пояснение - Это информационное сообщение указывает, что инициализация zDC началась или завершилась. Отображается идентификатор подсистемы и версия продукта.
    • Действие системы - Обработка продолжается.
    • Реакция пользователя - Нет

    ZDC001E¶

    • Полное сообщение - ZDC001E AN ERROR HAS OCCURRED DURING LOAD FOR MODULE @@
    • Пояснение - При запуске системы zDC была предпринята попытка загрузить все модули обработки в память. Загрузка модуля, указанного в этом сообщении (@@), завершилась неудачей.
    • Действие системы - zDC завершает работу.
    • Реакция пользователя - Убедитесь, что операторы //STEPLIB или //JOBLIB DD в JCL, запускающем zDC, указывают на правильные наборы данных, и что эти наборы данных содержат нужный выпуск zDC.

    ZDC002E¶

    • Полное сообщение - ZDC002E MODULE=@@ R15=@@ R1=@@
    • Пояснение - Это сообщение следует за ZDC001 и содержит имя модуля, а также содержимое регистров R15 и R1 по завершении операции LPAD.
    • Действие системы - zDC завершает работу.
    • Реакция пользователя - Сохраните информацию из этого сообщения, если потребуется обратиться к системному программисту или специалисту по продуктам Dynatrace через чат. Обратитесь к специалисту по продуктам Dynatrace через чат в вашем окружении.

    ZDC003E¶

    • Полное сообщение - ZDC003E COMMON AREA STORAGE NOT AVAILABLE FOR MODULE @@
    • Пояснение - Это сообщение указывает, что попытка загрузить модуль, указанный в сообщении, в расширенную общую память завершилась ошибкой из-за нехватки доступной памяти для размещения модуля.
    • Действие системы - zDC завершает работу.
    • Реакция пользователя - Увеличьте объём общей памяти, в частности субпула 241, и снова запустите zDC.

    ZDC004E¶

    • Полное сообщение - ZDC004E AN ERROR HAS OCCURRED DURING BLDL FOR MODULE @@
    • Пояснение - При запуске системы zDC была предпринята попытка загрузить все модули обработки в память. Операция BLDL для загрузки модуля, указанного в сообщении, завершилась неудачей.
    • Действие системы - zDC завершает работу.
    • Реакция пользователя - Убедитесь, что операторы //STEPLIB или //JOBLIB DD в JCL, запускающем zDC, указывают на правильные наборы данных, и что эти наборы данных содержат нужный выпуск zDC.

    ZDC005E¶

    • Полное сообщение - ZDC005E MODULE=@@ R15=@@ R1=@@
    • Пояснение - Это сообщение следует за ZDC004 и содержит имя модуля, а также содержимое регистров R15 и R1 по завершении операции BLDL.
    • Действие системы - zDC завершает работу.
    • Реакция пользователя - Сохраните информацию из этого сообщения, если потребуется обратиться к системному программисту или специалисту по продуктам Dynatrace через чат. Обратитесь к специалисту по продуктам Dynatrace через чат в вашем окружении.

    ZDC006E¶

    • Полное сообщение - ZDC006E SUBSYSTEM @@ ALREADY EXISTS AND IS ACTIVE
    • Пояснение - Это сообщение указывает, что 4-символьный идентификатор подсистемы, указанный в наборе данных параметров //SYSIN DD, уже существует в системе и является активной подсистемой zDC.
    • Действие системы - zDC завершает работу.
    • Реакция пользователя - Убедитесь, что указанный идентификатор подсистемы уникален в вашей системе. Затем перезапустите zDC или остановите экземпляр zDC с этим именем.

    ZDC007E¶

    • Полное сообщение - ZDC007E ECSA STORAGE NOT AVAILABLE FOR SUBSYSTEM @@, R15=@@
    • Пояснение - Это сообщение указывает, что попытка выделить память для управляющего блока подсистемы завершилась ошибкой. В расширенном субпуле CSA 241 недостаточно памяти выше граничной линии для этих блоков.
    • Действие системы - zDC завершает работу.
    • Реакция пользователя - Увеличьте объём памяти расширенного субпула CSA 241 и перезапустите zDC.

    ZDC008I¶

    • Полное сообщение - ZDC008I MODULE P/T DATE TIME VERSION ADDRESS LENGTH
    • Пояснение - Это сообщение отображается только при запуске zDC. В нём указаны столбцы, отображаемые далее в нескольких сообщениях ZDC009: имя модуля, наибольший номер отслеживания проблем, дата и время последней сборки модуля, его версия и адрес основной памяти, по которому он загружен.
    • Действие системы - Обработка продолжается.
    • Реакция пользователя - Нет

    ZDC009I¶

    • Полное сообщение - ZDC009I @@ @@ @@ @@ @@ @@ @@
    • Пояснение - В этом сообщении перечислен один модуль, загруженный zDC. Описание столбцов см. в сообщении ZDC008.
    • Действие системы - Обработка продолжается.
    • Реакция пользователя - Нет

    ZDC010E¶

    • Полное сообщение - ZDC010E SDUMPX TAKEN SUCCESSFULLY FOR zDC @@ ABEND
    • Пояснение - Это сообщение выводится на системную консоль подпрограммами дампа zDC. В нём указан код аварийного завершения.
    • Действие системы - Выполнение завершается.
    • Реакция пользователя - Подготовьте дамп для изучения системным персоналом или передачи специалисту по продуктам Dynatrace через чат. Обратитесь к специалисту по продуктам Dynatrace через чат в вашем окружении. Повторно запустите zDC.

    ZDC011E¶

    • Полное сообщение - ZDC011E SDUMPX FAILED FOR zDC @@ ABEND. RETCODE=@@/@@
    • Пояснение - Это сообщение выводится на системную консоль подпрограммами дампа zDC. Оно указывает, что попытка получить системный дамп завершилась ошибкой. В сообщении отображены код возврата и код причины сбоя.
    • Действие системы - Выполнение завершается.
    • Реакция пользователя - Изучите коды возврата и причины или передайте это сообщение специалисту по продуктам Dynatrace через чат. Обратитесь к специалисту по продуктам Dynatrace через чат в вашем окружении. Повторно запустите zDC.

    ZDC012E¶

    • Полное сообщение - ZDC012E @@ ABEND DUMP IN DYNATRACE zDC @@
    • Пояснение - Это сообщение является заголовком для всех макросов SDUMPX, выдаваемых zDC. Заголовок указывает код аварийного завершения, ставший причиной создания дампа.
    • Действие системы - Выполнение завершается.
    • Реакция пользователя - Убедитесь, что дамп сохранён и может быть передан системным программистам или специалисту по продуктам Dynatrace через чат. Обратитесь к специалисту по продуктам Dynatrace через чат в вашем окружении. Повторно запустите zDC.

    ZDC014E¶

    • Полное сообщение - ZDC014E ZDCLOGER VSAM @@ ERROR
    • Пояснение - Это сообщение указывает, что запрос журнала VSAM zDC вернул ненулевой код возврата. В сообщении содержится имя запроса.
    • Действие системы - Задача журналирования завершается, но выполнение zDC продолжается.
    • Реакция пользователя - Изучите последующие сообщения, содержащие подробную информацию о характере ошибки. Сохраните эту информацию для системных программистов или специалиста по продуктам Dynatrace через чат. Попробуйте повторно подключить и открыть журнал (команда MVS: Modify ZdcStcName,LOG OPEN). Обратитесь к специалисту по продуктам Dynatrace через чат в вашем окружении.

    ZDC015E¶

    • Полное сообщение - ZDC015E ZDCLOGER VSAM RPL FEEDBACK WORD @@

    • Действие системы - Задача журналирования завершается, но выполнение zDC продолжается.

Сохраните эту информацию для системных программистов или специалиста по продуктам Dynatrace через чат. Попробуйте повторно подключить и открыть журнал (команда MVS: Modify ZdcStcName,LOG OPEN). Обратитесь к специалисту по продуктам Dynatrace через чат в вашем окружении.

    ZDC016E¶

    • Полное сообщение - ZDC016E ZDCLOGER VSAM ACB ERROR FLAG @@
    • Пояснение - Это сообщение содержит значение флага ошибки VSAM ACB после операции VSAM OPEN или CLOSE над набором данных журнала.
    • Действие системы - Задача журналирования завершается, но выполнение zDC продолжается.
    • Реакция пользователя - Изучите последующие сообщения, содержащие подробную информацию о характере ошибки. Сохраните эту информацию для системных программистов или специалиста по продуктам Dynatrace через чат. Войдите в систему управления ISPF zDC. Затем повторно подключите и откройте журнал. Обратитесь к специалисту по продуктам Dynatrace через чат в вашем окружении.

    ZDC017E¶

    • Полное сообщение - ZDC017E SRB ABEND @@ AT DISPLACEMENT @@ INTO ZDCSRBRT
    • Пояснение - Это сообщение указывает, что произошло аварийное завершение по смещению, указанному в подпрограмме SRB ZDCSRBRT.
    • Действие системы - Выполнение завершается.
    • Реакция пользователя - Если дамп создан, сохраните его для системного персонала или передайте специалисту по продуктам Dynatrace через чат. Обратитесь к специалисту по продуктам Dynatrace через чат в вашем окружении. Повторно запустите zDC.

    ZDC018E¶

    • Полное сообщение - ZDC018E EXTENDED CSA STORAGE NOT AVAILABLE FOR SRB ROUTINE
    • Пояснение - Это сообщение указывает, что операция STORAGE OBTAIN завершилась ошибкой: расширенная память CSA из субпула 241 недоступна.
    • Действие системы - Выполнение завершается.
    • Реакция пользователя - Сообщите об этой ситуации системному программисту, которому может потребоваться увеличить объём расширенной памяти CSA в системе.

    ZDC019E¶

    • Полное сообщение - ZDC019E zDC INITIALIZATION FAILED. RETCODE=@@/@@
    • Пояснение - Это сообщение указывает, что подпрограммы инициализации завершились ошибкой. В нём также отображены коды возврата и причины, описывающие сбой.
    • Действие системы - Выполнение zDC завершается.
    • Реакция пользователя - Изучите набор данных SYSPRINT на наличие других сообщений, которые могут быть связаны со сбоем. Обратитесь к специалисту по продуктам Dynatrace через чат в вашем окружении.

    ZDC026I¶

    • Полное сообщение - ZDC026I STOP COMMAND ACCEPTED
    • Пояснение - Это сообщение подтверждает, что команда STOP, введённая оператором, принята zDC и начата обработка завершения задания.
    • Действие системы - Выполнение завершается.
    • Реакция пользователя - Нет

    ZDC027I¶

    • Полное сообщение - ZDC027I MODIFY COMMAND RECEIVED
    • Пояснение - Это сообщение подтверждает, что команда MODIFY, введённая оператором, получена zDC и обрабатывается. Ошибки, возникшие при синтаксическом разборе или выполнении команды, будут указаны позднее.
    • Действие системы - Выполнение продолжается.
    • Реакция пользователя - Нет

    ZDC028I¶

    • Полное сообщение - ZDC028I @@
    • Пояснение - Это сообщение содержит копию полной команды, введённой оператором на консоли. Отображаются только первые 120 байт команды.
    • Действие системы - Выполнение продолжается.
    • Реакция пользователя - Нет

    ZDC029E¶

    • Полное сообщение - ZDC029E TCPSP subtask unavailable to process shutdown request
    • Пояснение - Это сообщение указывает, что ECB подзадачи TCPSP занят и недоступен для получения запроса на завершение работы.
    • Действие системы - Выполнение завершается.
    • Реакция пользователя - Если дамп создан, сохраните его для системного персонала или передайте специалисту по продуктам Dynatrace через чат. Повторно запустите zDC. Обратитесь к специалисту по продуктам Dynatrace через чат в вашем окружении.

    ZDC033E¶

    • Полное сообщение - ZDC033E INVALID COMMAND. PLEASE REENTER
    • Пояснение - Команда, введённая на операторской консоли, недопустима и не была обработана.
    • Действие системы - Команда игнорируется, обработка продолжается.
    • Реакция пользователя - Если возможно, введите допустимую команду. Если команда была верной, обратитесь к специалисту по продуктам Dynatrace через чат. Обратитесь к специалисту по продуктам Dynatrace через чат в вашем окружении.

    ZDC035I¶

    • Полное сообщение - ZDC035I @@ ARMED FOR AUTOMATIC RESTART
    • Пояснение - Это сообщение указывает, что выбрана обработка ARM и элемент, указанный в сообщении, вооружён впервые. Это означает, что выполнение не является результатом автоматического перезапуска.
    • Действие системы - Обработка продолжается.
    • Реакция пользователя - Нет

    ZDC036I¶

    • Полное сообщение - ZDC036I @@ RESTARTED DUE TO PREVIOUS ABEND. RSNCD=@@
    • Пояснение - Это сообщение указывает, что данный zDC был автоматически перезапущен системой. Код причины указывает, был ли изменён JCL или команда START (RSNCD=108) или нет (RSNCD=104).
    • Действие системы - Обработка продолжается.
    • Реакция пользователя - Нет

    ZDC037E¶

    • Полное сообщение - ZDC037E @@ IXCARM REGISTER FAILURE. RETCD=@@. RSNCD=@@
    • Пояснение - Попытка выдать макрос IXCARM привела к коду возврата больше 4. Это означает, что запрос не выполнен. Фактические коды возврата и причины отображены в этом сообщении.
    • Действие системы - Обработка продолжается, но без поддержки ARM.
    • Реакция пользователя - Полный список возможных кодов возврата и причин см. в z/OS MVS Sysplex Services Reference. Исправьте ошибки среды или сообщите о проблеме специалисту по продуктам Dynatrace через чат. Обратитесь к специалисту по продуктам Dynatrace через чат в вашем окружении.

    ZDC038E¶

    • Полное сообщение - ZDC038E @@ IXCARM READY FAILURE. RETCD=@@. RSNCD=@@
    • Пояснение - Попытка выдать макрос IXCARM привела к коду возврата больше 4. Это означает, что запрос не выполнен. Фактические коды возврата и причины отображены в этом сообщении.
    • Действие системы - Обработка продолжается, но без поддержки ARM.
    • Реакция пользователя - Полный список возможных кодов возврата и причин см. в z/OS MVS Sysplex Services Reference. Исправьте ошибки среды или сообщите о проблеме специалисту по продуктам Dynatrace через чат в вашем окружении Dynatrace.

    ZDC039I¶

    • Полное сообщение - ZDC039I @@ AUTOMATIC RESTART NOT SUPORTED ON THIS SYSTEM
    • Пояснение - Попытка выдать макрос IXCARM привела к коду возврата, указывающему, что поддержка ARM недоступна в данной системе или может быть доступна, но остановлена.
    • Действие системы - Обработка продолжается, но без поддержки ARM.
    • Реакция пользователя - Полный список возможных кодов возврата и причин см. в z/OS MVS Sysplex Services Reference. Возможно, потребуется выполнить команду оператора SETXCF START,POLICY,TYPE=ARM для активации обработки ARM.

    ZDC040I¶

    • Полное сообщение - ZDC040I @@ DEREGISTERED FOR AUTOMATIC RESTART
    • Пояснение - Это сообщение указывает, что был выдан макрос IXCARM, чтобы сервер перешёл к нормальному завершению задания.
    • Действие системы - Обработка продолжается.
    • Реакция пользователя - Нет
