    Configure and monitor service-level objectives with Dynatrace¶

    Configure and monitor service-level objectives with Dynatrace¶

    • How-to guide
    • 15-min read
    • Updated on Mar 25, 2024

    Обзор SLO¶

    Список определённых целей уровня обслуживания (SLO) в окружении мониторинга Dynatrace на странице Service-level objectives отображает важную информацию: текущий статус, бюджет ошибок и скорость сжигания, целевое и предупредительное значения, количество открытых проблем из общего числа проблем для селектора сущностей SLO, а также временной диапазон для оценки SLO.

    Анализ проблем¶

    Если с SLO связаны открытые проблемы, значение в столбце Open/total problems для данного SLO отмечается красным предупреждающим символом. Выберите значение для перехода на страницу Problems, отфильтрованную по соответствующему селектору сущностей. Подробнее об анализе проблем см. в разделе Davis® AI.

    Сведения о SLO¶

    Раскройте Details SLO для получения дополнительной информации, например:

    • Метрические и Entity-селекторы SLO
    • График, отображающий оценку SLO во времени

    • Табличный вид последних 10 оценённых SLO, относящихся к определённому типу сущности. Переключитесь на табличный вид, чтобы выяснить, например, точное значение, негативно влияющее на результат агрегированной оценки SLO, и связанную с ним сущность. Кроме того, вы можете:

    • Сортировать табличный вид по статусу в порядке возрастания или убывания

    • Выбирать любую сущность для получения дополнительной информации на соответствующей странице сущности

    По умолчанию каждый SLO оценивается в соответствии с заданным временным диапазоном, но для анализа «что если» с другими временными диапазонами или для ретроспективного просмотра можно временно переключиться на глобальный временной диапазон.

    Настройка цели уровня обслуживания¶

    Для настройки новой цели уровня обслуживания используйте мастер SLO для выбора из набора преднастроенных шаблонов Dynatrace для распространённых вариантов использования. Кроме того, можно создать собственные определения SLO.

    Добавление SLO с помощью мастера¶

    В Dynatrace перейдите в Service-Level Objectives, выберите Add new SLO и пройдите следующие шаги мастера SLO.

    Шаг 1 Выбор индикаторов¶

    • Выберите нужный SLO:

    • SLO Service-level availability (доступность на уровне сервиса), где доступность на уровне сервиса измеряется как отношение числа успешных вызовов сервиса к общему числу вызовов.

    • SLO Service-method availability (доступность метода сервиса), где доступность метода сервиса измеряется как отношение числа успешных вызовов ключевых запросов к общему числу вызовов ключевых запросов.

    • SLO Service performance (производительность сервиса), где коэффициент производительности сервиса измеряется как отношение числа good (хороших) минут к общему числу total минут с помощью выражения метрики.

      good минуты — это минуты, в течение которых задержка ответа ниже заданного порога. + SLO User experience (пользовательский опыт), основанный на измерении Apdex: процент пользователей со статусом SATISFIED от общего числа пользователей веб- или мобильного приложения. + SLO Mobile crash-free users (пользователи мобильного приложения без сбоев), измеряющий процент пользователей без сбоев в ваших мобильных приложениях. + SLO Synthetic availability (доступность синтетических мониторов), представляющий процент успешных выполнений синтетического монитора от общего числа выполнений.

    Подробнее о вариантах использования см. в разделе Примеры конфигурации определений целей уровня обслуживания. * Введите SLO name (имя SLO). * Введите Metric name (имя метрики), например my_new_slo. Оно будет использовано для создания ключей метрик:

    • Статус SLO, например func:slo.my_new_slo.
    • Бюджет ошибок SLO, например func:slo.errorBudget.my_new_slo.
    • Нормализованный бюджет ошибок SLO, например func:slo.normalizedErrorBudget.my_new_slo.
    • Скорость сжигания бюджета ошибок SLO, например func:slo.errorBudgetBurnRate.my_new_slo.

    Эти ключи метрик можно использовать для построения графиков на всех страницах, допускающих применение метрик, например в Data Explorer.

    После создания SLO:

    • Ключи метрик изменить нельзя.
    • Просмотреть ключи метрик можно в сведениях о SLO.
    • Необязательно Настройте метрики SLI, которые нужно добавить в ваш SLO.

    Шаг 2 Определение фильтра¶

    В селекторе временного диапазона прокрутите вниз для выбора значения временного диапазона для вашего SLO.

    • Для выбора одного из существующих значений нажмите Presets.
    • Для создания собственного значения временного диапазона нажмите Custom.

    Селектор сущностей соответствует синтаксису запросов REST API Dynatrace. Можно использовать фильтры по ID/имени зоны управления, тегам, имени/ID/типу сущности, состоянию работоспособности или их комбинации. Для зон управления можно выбирать из списка доступных зон.

    После ввода нужных селекторов сущностей можно проверить отсутствие ошибок, нажав Preview рядом с панелью селектора сущностей.

    Шаг 3 Добавление критериев успеха¶

    Задайте целевой процент (Failure) и предупредительный процент (Warning).

    Предупредительный процент должен находиться между 100% и вашим целевым процентом SLO, чтобы быть эффективным. Например, если ваш целевой процент SLO составляет 99,00%, для получения раннего предупреждения (обозначаемого жёлтым состоянием) нужно задать предупредительный процент между 99,00% и 100%.

    Для отслеживания скорости сжигания бюджета ошибок сервисом относительно SLO убедитесь, что Error budget burn rate включён, и задайте пороговые значения для slow-burn (медленного) и fast burn (быстрого) сжигания.

    Шаг 4 Оценка¶

    После ввода значений критериев успеха нажмите Evaluate для оценки SLO на основе введённых значений.

    Если всё корректно и ошибок нет, нажмите Create для сохранения конфигурации и добавления нового SLO.

    После завершения настройки только что созданная цель уровня обслуживания появится на странице SLOs.

    Создание собственного SLO¶

    Для настройки собственной цели уровня обслуживания перейдите в Settings, выберите Cloud Automation > Definition и нажмите Add new SLO.

    Редактирование SLO¶

    Для редактирования SLO в Dynatrace перейдите в Service-Level Objectives, найдите нужный SLO и в столбце Actions выберите More (…) > SLO definition.

    Нормализация бюджета ошибок¶

    Для просмотра нормализованного бюджета ошибок для всех SLO перейдите в Settings > Cloud Automation > Setup и включите Normalize error budget.

    Возьмём, например, SLO с целевым значением 95% и текущим статусом SLO 96%. При включённой нормализации оставшийся бюджет ошибок вычисляется как (статус − цель) ÷ (100 − цель) × 100, например, (96% − 95%) ÷ (100% − 95%) × 100.

    Скорость сжигания бюджета ошибок¶

    Скорость сжигания бюджета ошибок показывает, насколько быстро сервис потребляет бюджет ошибок относительно SLO. Например:

    • Скорость сжигания 1 означает, что сервис потребил 100% бюджета ошибок за временной диапазон SLO.
    • Скорость сжигания 2 означает, что сервис потребил вдвое больше бюджета ошибок за временной диапазон SLO.

    Скорость сжигания рассчитывается либо за последний час (если выбран временной диапазон SLO), либо за значение глобального временного диапазона (если временной диапазон SLO не выбран).

    Настройка скорости сжигания бюджета ошибок¶

    Для отображения скорости потребления бюджета ошибок сервисом можно включить визуализацию скорости сжигания — либо из мастера, либо со страницы настроек при создании SLO.

    В любой момент можно изменить пороговое значение или отключить визуализацию скорости сжигания в SLO definition вашего SLO.

    Визуализация скорости сжигания бюджета ошибок¶

    После настройки скорости сжигания бюджета ошибок есть несколько мест в окружении, где можно её просматривать:

    • На странице обзора SLO в столбце бюджета ошибок:

    • Жёлтая иконка медленного сжигания отображается, когда значение скорости сжигания находится между 1 и порогом быстрого сжигания, заданным при создании SLO.

    • Красная иконка быстрого сжигания отображается, когда скорость сжигания больше или равна порогу быстрого сжигания, заданному при создании SLO.

    Если визуализация скорости сжигания включена, но иконка не отображается, скорость сжигания ниже 1. * В сведениях о SLO. * В Data Explorer. * На дашборде, если вы закрепили SLO на нём.

    Настройка оповещений¶

    Можно настроить два типа оповещений:

    • Оповещения SLO отправляются, когда статус SLO опускается ниже целевого значения.
    • Оповещения о скорости сжигания отправляются, когда бюджет ошибок SLO уменьшается с определённой скоростью.

    Оповещения можно создавать только на основе метрических событий за последний час. Если задать пороговое значение 10 для оповещения о скорости сжигания, оповещение будет сгенерировано, когда скорость сжигания превысит 10 за последний час.

    • Для настройки оповещения SLO:

    • Перейдите в Service-Level Objectives, найдите нужный SLO и выберите More (…) > Create alert.

    • Для Select alert type выберите Status.
    • Назовите оповещение и задайте пороговое значение. Если значение не задано, порог заполняется существующим целевым значением SLO.

    • Выберите Create alert.

    • Для настройки оповещения о скорости сжигания:

    • Перейдите в Service-Level Objectives, найдите нужный SLO и выберите More (…) > Create alert.

    • Для Select alert type выберите Burn rate.
    • Назовите оповещение и задайте пороговое значение скорости сжигания.
    • Выберите Create alert.

    Только что созданное оповещение SLO или скорости сжигания появится на странице Metric events, где его можно настроить дополнительно. Подробнее см. в разделе Метрические события.

    Добавление SLO в зоны управления¶

    SLO, не принадлежащие ни одной зоне управления, видны всем пользователям. При добавлении SLO в зону управления его смогут видеть на странице обзора Service-level objectives только пользователи с доступом к этой зоне управления.

    • Для добавления SLO в зону управления:

    • Перейдите в Settings.

    • Выберите Cloud Automation > Definition.
    • Выберите Add new SLO.
    • В поле Entity selector добавьте имя или ID зоны управления.

    • После ввода всех данных SLO нажмите Save changes для сохранения конфигурации.

    • Для добавления существующего SLO в зону управления см. раздел Редактирование SLO.

    • Для просмотра SLO, принадлежащих конкретной зоне управления, выберите зону управления с помощью кнопки фильтра в строке меню.
    • Для просмотра глобальных SLO независимо от выбранного фильтра зоны управления включите Show global SLOs. Глобальные SLO — это SLO, видимые всем пользователям независимо от их разрешений на зоны управления.

    Подробнее об управлении доступом к SLO в вашем окружении через разрешения см. в разделе Просмотр и редактирование SLO на основе уровней разрешений.

    Закрепление SLO на дашборде¶

    После определения целей SLO можно добавить их на дашборд для визуализации текущего статуса и оставшихся бюджетов ошибок.

    1. В Dynatrace перейдите в Service-Level Objectives, найдите нужный SLO и в столбце Actions выберите More (…) > Pin to your dashboard.
    2. В списке выберите существующий дашборд или Create new dashboard, затем нажмите Pin.
    3. Выберите Open dashboard для открытия дашборда в режиме редактирования с выбранной плиткой SLO.
    4. При необходимости измените конфигурацию плитки.
    5. Выберите Done.

    Плитка SLO на дашборде¶

    По умолчанию плитка SLO оценивает и фильтрует временной диапазон SLO вместо выбранного глобального. Можно переопределить временной диапазон в конфигурации плитки для сравнения глобального и диапазона SLO. Подробнее см. в разделе Просмотр и добавление плиток SLO на дашборде на основе уровней разрешений.

    Плитка SLO на дашборде отображает Status, Error budget и Target.

    • Если с SLO связаны проблемы, в нижнем левом углу плитки отображается красный восклицательный знак.
    • Если скорость сжигания бюджета ошибок включена и скорость сжигания больше единицы, перед значением бюджета ошибок отображается иконка индикатора состояния. Цвет иконки зависит от порога SLO.

    Плитка SLO обновляется автоматически. Частота обновления плитки зависит от применяемого временного диапазона.

    При уменьшении размера плитка SLO будет отображать меньше информации. Выбор плитки SLO осуществляет переход к обзору SLO, отфильтрованному по выбранному SLO.

    Редактирование и просмотр плитки SLO на дашборде¶

    Выберите View details в меню в правом верхнем углу плитки для просмотра сведений о SLO.

    Выберите Edit tile в меню в правом верхнем углу плитки для настройки в панели Service-level objective.

    • Select an SLO: выберите SLO для отображения в этой плитке.
    • Title: заголовок, отображаемый вверху плитки. По умолчанию заголовок берётся из имени выбранного SLO. Используйте Title Size для настройки размера шрифта.
    • Max shown decimals: максимальное число отображаемых знаков после запятой (0–10). Знаки после запятой усекаются, а не округляются. Число отображаемых знаков автоматически подстраивается под размер плитки.

    Доступные переключатели:

    • Show metric names: отображать ли Status, Error budget и Target под значениями в плитке.
    • Show legend: отображать ли легенду для цветов статуса: зелёный — Good, жёлтый — Warning, красный — Bad, серый — No Data.
    • Show problems indicator: отображать ли индикатор проблем (красный восклицательный знак) в плитке.
    • Colorize based on status: закрашивать ли весь фон плитки (вместо значений) в зависимости от статуса. При включении этой опции легенда скрывается.
    • Title Size: задайте размер шрифта для Title: Inherit (по умолчанию), Small, Medium или Large.

    Информация, отображаемая в плитке, зависит от её размера.

    • Вся информация отображается, если размер плитки 8x4 или больше.
    • При меньшем размере плитки Open/total problems, Error budget, скорость сжигания бюджета ошибок и Target скрываются.
    • При ширине менее 4 заголовок скрывается.

    Фильтры плитки SLO на дашборде¶

    Для переопределения настроек дашборда для выбранной плитки можно использовать следующие фильтры.

    • Custom timeframe: по умолчанию Last 1 week.
    • Custom management zone: по умолчанию All.

    Как Custom timeframe, так и Custom management zone можно задавать в глобальных селекторах, в Tile filters на боковой панели дашборда или в Edit tile. Подробнее см. в разделе Настройки временного диапазона и зоны управления дашборда Dynatrace.

    Плитку можно настроить для запроса удалённого окружения. Подробнее см. в разделе Создание удалённых/мультиокружных дашбордов Dynatrace.

    Если плитка имеет пользовательские фильтры, отображается иконка фильтра. При наведении курсора отображается список фильтров: Timeframe, Management zone и настройки удалённого окружения.

    Сравнение плиток SLO и Data Explorer¶

    Плитки SLO и Data Explorer могут показывать разные значения.

    • Значения в плитках Data Explorer округляются
    • Значения в плитках SLO усекаются

    Разрешения плиток SLO на дашборде¶

    Вы можете видеть глобальные SLO. Глобальный SLO виден всем пользователям и не принадлежит ни одной зоне управления. Подробнее см. в разделе добавление SLO в зону управления.

    Также можно видеть SLO, к которым у вас есть разрешение на доступ.

    • При редактировании SLO, если у вас нет доступа к определённой зоне управления, SLO в этой зоне управления не отображаются в списке Select an SLO.
    • Если открыть существующий дашборд с SLO, к которым у вас нет доступа, плитки SLO отображают сообщение об отказе в разрешении.

    Если дашборд имеет анонимную ссылку доступа, разрешения устанавливаются пользователем, создавшим ссылку. Все пользователи видят связанный дашборд так же, как его видит создатель ссылки.

    Визуализация статуса SLO по цвету¶

    После закрепления SLO на дашборде статус SLO указывается в плитке с помощью сочетания текста и цвета текста.

    Для включения дополнительной раскраски фона SLO в соответствии со статусом:

    1. В Dynatrace перейдите в Dashboards и откройте дашборд.
    2. Выберите Edit.
    3. Выберите плитку SLO, которую нужно раскрасить.
    4. В панели Service-level objective справа включите Colorize based on status.
    5. Выберите Done.

    После этого фоновый цвет плитки SLO (вместо текста) будет автоматически изменяться в соответствии со статусом SLO:

    Цвет плитки Статус
    Зелёный Good
    Жёлтый Warning
    Красный Bad

    Клонирование SLO¶

    Клонирование SLO позволяет создать новый SLO, повторно используя конфигурацию существующего.

    Для клонирования SLO:

    1. Перейдите в Service-Level Objectives.
    2. Выберите SLO, который нужно клонировать, и нажмите Action > Clone.
      Страница Add new SLO будет предзаполнена настройками клонируемого SLO.
    3. При необходимости измените настройки нового SLO и нажмите Create.

    Просмотр метрик в Data Explorer¶

    Для запроса и построения графиков метрик перейдите к нужному SLO и выберите Actions > View in Data Explorer. Подробнее см. в разделе Data Explorer.

    Ограничения¶

    • Data Explorer показывает ключи метрик; преобразования или фильтры не отображаются.

    Оповещения Davis¶

    Dynatrace Davis® обеспечивает быстрые уведомления об обнаруженных аномалиях вместе с описанием первопричин. Если ваш SLO стал красным, скорее всего, Davis уже поднял проблему по базовым метрикам, показывая первопричину.

    Davis не предоставляет оповещений о нарушениях целевых значений SLO, но предоставляет их для базовых метрик и сущностей SLO.

    Устранение неполадок¶

    Ниже приведены решения проблем, с которыми столкнулись некоторые пользователи.

    • Service-level objective (SLO) list takes a lot of time to load
    • Service-level objective (SLO): One resulting value is required, but the SLO definition delivers more than one
    • Service-level objective (SLO): Timeframe starts before the metric was created
    • Service-level objective (SLO): No management zone selected
