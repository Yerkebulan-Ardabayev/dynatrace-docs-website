    Hardware requirements for Dynatrace Monaco CLI¶

    Hardware requirements for Dynatrace Monaco CLI¶

    • Reference
    • 5-min read
    • Updated on Sep 12, 2023

    На этой странице представлен обзор аппаратных требований Dynatrace Configuration as Code через Monaco.

    Требования к памяти¶

    Основное аппаратное требование Dynatrace Monaco CLI — оперативная память.

    При выполнении в контейнере с жёстким ограничением памяти или на оборудовании с ограниченными ресурсами убедитесь в наличии достаточно высокого лимита памяти или доступного пространства подкачки (swap).

    Необходимый объём памяти определяется преимущественно количеством конфигураций и размером JSON-файлов шаблонов. В следующих таблицах приведены ориентировочные требования к памяти для проектов разных размеров; вашему проекту может потребоваться больше или меньше памяти.

    Обратите внимание, что CLI версии 2.9.0 и ранее имеет более высокие требования к памяти.

    Для более крупных проектов используйте следующие правила оценки необходимой памяти:

    1. Доступная память должна быть примерно в 2,5 раза больше места, занимаемого конфигурациями на диске.
    2. На каждую конфигурацию проекта требуется около 15 КБ памяти.

    Оба правила лучше применимы к крупным проектам, где накладные расходы памяти, не связанные с конфигурацией, менее заметны.

    Ограничения памяти¶

    Мягкое ограничение памяти для Dynatrace Monaco CLI можно настроить с помощью переменной окружения GOMEMLIMIT.

    GOMEMLIMIT — числовое значение с необязательным суффиксом единицы измерения: B, KiB, MiB, GiB или TiB. При отсутствии суффикса значение считается в байтах (B).

    Например, для установки лимита в 3 гибибайта: GOMEMLIMIT=3GiB.

    Linux/macOS

    Windows

    GOMEMLIMIT=3GiB monaco deploy manifest.yaml

    $env:GOMEMLIMIT=3GiB



monaco deploy manifest.yaml

    Это мягкое ограничение: при необходимости оно может быть превышено. Однако слишком низкий лимит увеличит время выполнения, так как будет тратиться больше времени на освобождение памяти при его превышении. Оценочные требования к памяти см. в таблице выше.

    Различия в версиях CLI¶

    • Dynatrace Monaco CLI версии 2.8.0+: применяется лимит памяти по умолчанию 2 ГБ. Для изменения задайте GOMEMLIMIT.
    • Dynatrace Monaco CLI версии 2.7.0 и ранее: лимит памяти по умолчанию не применяется. Dynatrace Monaco CLI может потреблять избыточный объём памяти. Для установки лимита задайте GOMEMLIMIT.

    Влияние CPU на время развёртывания¶

    Доступный CPU преимущественно влияет на время развёртывания и, в отличие от памяти, не создаёт жёстких ограничений: на оборудовании с ограниченными ресурсами развёртывания просто занимают больше времени.

    Например, развёртывание в контейнере с одним vCPU занимает значительно больше времени, чем с несколькими vCPU или при запуске непосредственно на системе с многоядерным процессором.

    В следующих таблицах представлен приблизительный обзор влияния числа конфигураций на время развёртывания. Это лишь иллюстрирует зависимость между размером проекта и временем развёртывания и может не совпадать с реальным временем на вашем оборудовании.

    • В Dynatrace Monaco CLI версии 2.7.0+: время развёртывания измерено для параллельных развёртываний, которые доступны по умолчанию.
    • В Dynatrace Monaco CLI версии 2.6.0 и ранее: развёртывание полностью последовательное и значительно медленнее, чем в новых версиях.
