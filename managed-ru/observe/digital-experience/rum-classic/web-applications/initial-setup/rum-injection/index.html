    Настройка автоматического внедрения в RUM Classic¶

    Настройка автоматического внедрения в RUM Classic¶

    • Практическое руководство
    • Обновлено 01 июля 2025 г.

    После развёртывания OneAgent в режиме полного мониторинга стека (full-stack monitoring) на хосте, он по умолчанию автоматически внедряет RUM JavaScript в HTML веб-приложений для технологий, перечисленных в разделе Поддержка технологий - Real User Monitoring - веб-серверы и приложения. Чтобы инициировать внедрение, достаточно перезапустить процессы веб-сервера после установки OneAgent.

    По умолчанию внедрение следует набору встроенных правил автоматического внедрения, но бывают сценарии, в которых требуется определить пользовательское правило внедрения для достижения внедрения. Пользовательские правила внедрения также поддерживают исключение отдельных страниц из внедрения, что позволяет настроить ручную вставку для страниц приложения с автоматическим внедрением.

    RUM JavaScript отвечает за сбор данных RUM из веб-браузеров пользователей. Он отправляет эти данные в виде XHR POST-запросов на конечную точку beacon, которая выполняет первичную проверку перед пересылкой данных в кластер Dynatrace. Конечная точка beacon по умолчанию в сценариях автоматического внедрения предоставляется OneAgent, инструментирующим веб-сервер или сервер приложений, но возможны и альтернативные настройки, см. Настройка конечной точки beacon для веб-приложений в RUM Classic.

    Фрагмент RUM JavaScript, внедряемый в HTML приложения, может отличаться по формату, каждый из которых предназначен для удовлетворения конкретных требований, описанных в разделе Выбор формата фрагмента в RUM Classic. Для настройки формата, используемого при автоматическом внедрении, см. Настройка формата фрагмента.

    OneAgent внедряет RUM JavaScript на каждую страницу на основе текущей конфигурации всякий раз, когда инструментированный веб-сервер или сервер приложений доставляет страницу. Однако политика кэширования страницы определяет, как часто у OneAgent появляется возможность выполнить это внедрение. Для решения этой проблемы OneAgent включает функцию, оптимизирующую заголовки управления кэшем. Подробнее о том, как работает эта функция, и о её ограничениях см. в разделе Оптимизации заголовков управления кэшем.

    Если нет доступа к веб-серверу, на котором размещено приложение, и поэтому нет возможности установить OneAgent, можно отслеживать приложение с помощью агентless RUM.

    Встроенные правила автоматического внедрения¶

    OneAgent обнаруживает ответы, содержащие HTML-контент, и внедряет RUM JavaScript в раздел head каждой страницы. Он не изменяет другие ресурсы, такие как изображения, файлы CSS, документы JSON или XML, либо простой текст.

    OneAgent пытается внедрить RUM JavaScript как первый скрипт на странице, что помогает обеспечить надёжные результаты мониторинга. Внедрение работает только с корректным HTML, соответствующим основным передовым практикам, например, все элементы должны быть правильно открыты и закрыты. OneAgent следует указанным ниже встроенным правилам, чтобы определить наиболее подходящее место в HTML-документе для внедрения RUM JavaScript.

    Спецификация правил автоматического внедрения¶

    Название Условие Действие Отменяет
    <!DOCTYPE> Тег <!DOCTYPE> не является <!DOCTYPE html>. Прервать и не внедрять. Обратите внимание, что допускается только одна декларация Doctype.
    initial tag Перед <html> встречается тег, не являющийся одним из следующих: * <!DOCTYPE ...> * <html> * <link> * <meta> * <script> * <style> Прервать без внедрения.
    Встречается спецификация <?xml ...?>. Игнорировать и продолжить сканирование документа. non- tag non- tag initial tag
    tag Встречается тег <html>. Если потенциальная точка внедрения найдена ранее, внедрить туда и не продолжать сканирование. Если найдено несколько потенциальных точек внедрения, используется самая ранняя. В противном случае продолжить сканирование документа
