    Анализ сессий пользователей¶

    Анализ сессий пользователей¶

    • How-to guide
    • 21-min read
    • Updated on Oct 12, 2023

    Dynatrace версии 1.224+

    Анализ отдельных сессий пользователей бывает полезен в конкретных ситуациях, однако зачастую он не даёт полной картины. Пользователи вашего приложения ведут себя непредсказуемо, преследуют разные цели, находятся в разных географических регионах и используют самые разные комбинации устройств, операционных систем и браузеров.

    Dynatrace поддерживает сегментацию сессий пользователей с помощью мощного механизма фильтрации. Анализ сессий пользователей Dynatrace позволяет разбивать сессии вашего приложения на значимые сегменты по общим характеристикам: операционной системе, типу браузера, местоположению или пользовательскому тегу. Например, вы можете сегментировать анализ по типам браузеров: настольный, мобильный или синтетический. Это позволяет детально изучить агрегированные результаты и выявить проблемы производительности, затрагивающие лишь небольшую часть пользователей.

    Анализ сессии пользователя¶

    Для анализа сессии пользователя:

    1. Перейдите в Session Segmentation.
    2. Нажмите на текстовое поле в верхней части страницы (см. 1 в примере ниже) и выберите один из доступных атрибутов фильтрации. После выбора атрибута отобразится список доступных значений для этого фильтра.

    ### Доступные атрибуты фильтрации

    Категория атрибута Атрибут Описание
    Приложения Application Выберите имя приложения для анализа.
    Application type Укажите, сессии каких приложений анализировать: веб, мобильных или пользовательских.
    Browser monitor Выберите имя браузерного монитора для синтетического мониторинга.
    Браузер Browser type Фильтрация сессий по типу браузера: настольный, планшетный, мобильный, синтетический агент или боты.
    Browser family Фильтрация сессий по используемому браузеру.
    Browser version Используйте этот атрибут для фильтрации по конкретному браузеру и версии.
    Screen width Фильтрация сессий по ширине экрана.
    Screen height Фильтрация сессий по высоте экрана.
    Операционная система Operating system family Фильтрация сессий по семейству ОС (Windows, Linux, iOS и т.д.).
    Operating system version Выберите конкретную версию ОС.
    Местоположение Continent Фильтрация сессий по континенту, откуда они поступают.
    Country Фильтрация сессий по стране.
    Region Фильтрация сессий по географическому региону.
    City Фильтрация сессий по городу.
    Мобильные устройства Application version Фильтрация сессий по версии мобильного приложения.
    Crashes Выберите Yes или No для фильтрации сессий, в которых произошёл или не произошёл сбой.
    Device Фильтрация сессий по типу мобильного устройства.
    Manufacturer Фильтрация сессий по производителю мобильного устройства.
    Rooted or jailbroken Выберите Yes или No для фильтрации сессий с разблокированными (root/jailbreak) или оригинальными устройствами. Если статус устройства неизвестен, сессии имеют значение null и не отображаются в результатах. Пользовательские приложения всегда сообщают неизвестный или неопределённый статус.
    Пользователь User tag Выберите пользовательский тег для анализа сессий конкретного пользователя.
    Internal user ID Фильтрация сессий по уникальному идентификатору пользователя.
    User type Выбор типа пользователей: боты, синтетические пользователи или реальные пользователи.
    New user Выберите Yes или No для фильтрации по новым или возвращающимся пользователям.
    Сессия Live Выберите Yes или No для отображения текущих или завершённых сессий.
    Session Replay Выберите Yes или No для отображения сессий с Session Replay или без него.
    Bounced Выберите Yes или No для фильтрации отказных сессий (сессии, немедленно прерванные пользователем). Высокий процент отказов нежелателен.
    Converted Выберите Yes или No для анализа сессий, в которых была или не была достигнута цель конверсии.
    Session conversion count Фильтрация сессий по количеству достижений любой из целей конверсии.
    Conversion goal Выберите конкретную цель конверсии для просмотра сессий, в которых она была достигнута.
    Has errors Выберите Yes или No для фильтрации сессий с ошибками или без.
    Error count Укажите диапазон количества ошибок. Используйте для фокусировки на сессиях с более чем определённым числом ошибок (если верхняя граница не указана), менее определённого значения (если нижняя граница не указана) или в указанном диапазоне.
    Error type Укажите тип ошибок: request, reported, custom или JavaScript.
    IP Фильтрация сессий по IP-адресу.
    IPS Фильтрация сессий по интернет-провайдеру.
    Duration Укажите длительность сессии в минутах.
    Свойства сессии Session date properties Фильтрация по свойству-дате сессии.
    Session double properties Фильтрация по свойству-числу с плавающей запятой сессии.
    Session long properties Фильтрация по свойству-целому числу сессии.
    Session string properties Фильтрация по строковому свойству сессии.
    Действия пользователя User action count Укажите диапазон количества действий пользователя в сессии. Помогает выявить сессии с большим числом действий.
    User action name Укажите действие пользователя для анализа всех сессий, содержащих хотя бы одно такое действие.
    User action date properties Фильтрация по свойству-дате действия пользователя.
    User action double properties Фильтрация по свойству-числу с плавающей запятой действия пользователя.
    User action string properties Фильтрация по строковому свойству действия пользователя.
    User action long properties Фильтрация по свойству-целому числу действия пользователя.
    Страницы Page name Отображение сессий, в которых пользователь посетил указанную страницу.
    Page group Отображение сессий, в которых пользователь посетил страницу из указанной группы.
    Удобство использования Rage click count Укажите диапазон количества rage click.
    User experience score Отображение сессий с оценкой пользовательского опыта: Satisfying, Tolerable или Frustrating.
    Rage tap count Укажите диапазон количества rage tap в мобильных сессиях.

    В фильтрах оператор тильды (~) не работает как ключевое слово LIKE в USQL. 3. Выберите интересующее вас значение атрибута. Некоторые атрибуты предоставляют текстовые поля для произвольного поиска. Вы также можете выбрать несколько значений одного атрибута — это работает как оператор OR для данного атрибута. 4. Повторите процесс для нужных атрибутов. После определения фильтра нажмите в любом месте за пределами текстового поля.

    Результат отображает первые 500 сессий, упорядоченных по времени начала (от новых к старым). Для изменения порядка сортировки кликните по заголовку столбца. 5. Выберите временную метку сессии (см. 3 в примере ниже) для перехода на страницу сведений о сессии. Или выберите имя пользователя (см. 2 в примере ниже) для перехода на страницу этого пользователя.

    Используйте выбор временного интервала в строке меню для настройки анализируемого периода.

    Сессии пользователей, которые являются текущими в пределах выбранного интервала, отображаются в списке сессий. Поиск по атрибуту Analysis over time, напротив, учитывает только сессии, начавшиеся в пределах заданного интервала.

    Например, если интервал задан с 12:00 до 12:05, а сессия, начавшаяся в 11:55, продолжается в этот период, она будет показана в списке, но не будет учтена в поиске Analysis over time, поскольку начата до заданного интервала.

    Детализация с помощью анализа¶

    Панель анализа расположена слева на странице User sessions. Панель содержит готовые результаты анализа и различные визуализации для разных атрибутов. Например, выберите категорию Application versions, чтобы увидеть, какая версия вашего приложения имеет больше сессий, или категорию Applications, чтобы просмотреть агрегированные данные по каждому приложению.

    Вы можете выбрать любой из результатов анализа и применить его как фильтр, нажав Apply selection as a filter в нижнем левом углу страницы.

    Фокус на сессиях отдельного пользователя¶

    Вы можете сфокусироваться на сессиях конкретного пользователя. Выберите пользователя в столбце User, чтобы перейти на страницу обзора этого пользователя.

    Для поиска сессий конкретного пользователя выберите User tag в поле Filter by и выберите нужного пользователя. Например, для отображения сессий пользователя Zara добавьте фильтр User tag: Zara. Затем выберите Zara в столбце User для перехода на страницу обзора этого пользователя.

    Для настройки тегирования пользователей в зависимости от типа приложения см.:

    • Веб-приложения
    • Мобильные приложения:
    • Android
    • iOS
    • Пользовательские приложения: OpenKit

    Просмотр сведений об ошибках¶

    Вы также можете использовать анализ сессий для получения подробной информации об ошибках в вашем приложении.

    Для просмотра сведений об ошибке:

    1. Перейдите в Session Segmentation.
    2. В поле Filter by задайте Error type одно из следующих значений в зависимости от типа приложения:
    Тип ошибки Описание Web Mobile Custom
    Request error Обнаружена браузером и OneAgent на серверах Применимо Применимо Применимо
    Reported error Сообщается вручную через специальный метод API Не применимо Применимо Применимо
    Custom error Сообщается вручную через RUM JavaScript API Применимо Не применимо Не применимо
    JavaScript error Исключения JavaScript в браузере Применимо Не применимо Не применимо
    1. Выберите интересующую вас сессию для открытия страницы сведений о ней.
    2. В разделе Events and actions разверните действие пользователя, содержащее ошибку, и выберите Perform waterfall analysis.

    3. Выполните одно из следующих действий в зависимости от типа приложения:

    4. Веб-приложения: на странице Waterfall analysis выберите плитку Error, а затем выберите ошибку. Откроется страница сведений об ошибке.

    5. Мобильные и пользовательские приложения: прокрутите до раздела Web request errors или Reported errors и выберите ошибку. Откроется страница сведений об ошибке.

    Страница сведений об ошибке предоставляет подробную информацию: предполагаемое количество ошибок, провайдер (для request error), технология (для reported error) и другое. На странице также перечислены затронутые сессии и действия пользователей. Разбивка по распределению отображает сведения об относительной частоте операционных систем, версий ОС, версий приложений и устройств, а разбивка по странам показывает все затронутые страны и соответствующий уровень ошибок.

    Проверка сессий с rage events¶

    Когда приложение не реагирует достаточно быстро, текстовая метка выглядит как кнопка или переключатель скрыт под другим элементом, пользователи могут многократно нажимать на затронутый элемент управления. Dynatrace обнаруживает такое поведение как rage event: rage click или rage tap.

    Для просмотра сессий с rage events:

    1. Перейдите в Session Segmentation.
    2. Задайте Filter by значение Rage click count: >= 1 или Rage tap count: >= 1.
    3. Выберите интересующую вас сессию для открытия страницы сведений о ней.
    4. Прокрутите до раздела Events and actions и разверните rage click или rage tap, чтобы просмотреть подробности.

    Анализ сбоев¶

    Мобильные и пользовательские приложения

    Когда сессия пользователя заканчивается сбоем, вы можете использовать анализ сессий для просмотра полной последовательности действий пользователя, предшествовавших сбою. Также вы можете открыть отчёт о сбое для получения информации на уровне кода и быстрого выявления первопричин.

    1. Перейдите в Session Segmentation.

    2. Задайте следующие фильтры в поле Filter by:

    3. Application type: Mobile для мобильных сессий или Application type: Custom для пользовательских приложений

    4. Crashes: Yes для сессий, завершившихся сбоем
    5. Мобильные приложения: Session Replay: Yes для сессий, записанных с Session Replay on crashes для Android или iOS
    6. Выберите интересующую вас сессию для открытия её страницы сведений.
    7. Мобильные приложения: для просмотра записи Session Replay перейдите на вкладку Session Replay и нажмите Play. Последнее событие сессии является сбоем, отображаемым красной точкой на временной шкале. Используйте элементы управления Session Replay для детального анализа сбоя.
    8. Для просмотра всех действий и событий пользователя, предшествовавших сбою, прокрутите до раздела Events and actions.
    9. Для просмотра отчёта о сбое разверните событие сбоя и выберите Open crash details. Отчёт предоставляет информацию об устройстве пользователя и трассировку стека. Также вы можете анализировать группы сбоев для мобильных и пользовательских приложений или загрузить трассировку стека.

    Анализ сессии в нескольких доменах¶

    По техническим и соображениям безопасности анализ одной сессии пользователя в разных доменах невозможен.

    Если пользователь посещает два совершенно разных домена в рамках одной «сессии», а оба домена инструментированы Dynatrace, вы увидите две отдельные сессии:

    1. Первая сессия начинается с загрузки страницы первого домена и заканчивается, когда пользователь переходит по ссылке на страницу другого домена.
    2. Вторая сессия начинается с первой загрузки страницы на втором домене и заканчивается при выполнении любого из критериев завершения сессии.

    Причины: куки не могут быть общими для разных доменов (за исключением поддоменов) и это является ограничением безопасности браузеров.

    Анализ в реальном времени¶

    • Доступ к текущим сессиям через плитку Live user activity на дашборде.
    • Перейдите на одну из страниц проблем и выберите See user sessions sample.
    • Затронутые сессии можно также открыть непосредственно со страницы проблемы.
    • Отказы и конверсии доступны для завершённых сессий.

    Примеры¶

    Фильтрация сессий по длительности

    Вы можете фильтровать сессии по длительности: дольше или короче определённого значения или в определённом диапазоне.

    Поиск всех сессий, содержащих хотя бы одно из указанных действий пользователя

    Вы можете искать все сессии пользователей, которые содержат хотя бы одно из нескольких указанных действий в своих clickpaths.

    Создание сложных фильтров

    Пример фильтра, охватывающего сессии, соответствующие следующим критериям:

    • Application type: Web и Browser Type: Desktop Browser. Пользователь посетил веб-приложение через настольный браузер.
    • Duration 60s. Длительность сессии не менее 1 минуты.
    • Action count = 5. Пользователь выполнил пять действий в ходе сессии.
    • User action name: test. В сессии присутствовало действие пользователя test.

    Просмотр категорий на панели анализа

    Для каждой категории на панели анализа имеется отдельный раздел с визуализациями и результатами для этой категории. Например, Application type отображает текущее распределение, распределение во времени и географическое распределение различных типов приложений.

    Вы можете выбрать любой результат и применить его как фильтр, нажав Apply selection as a filter в нижнем левом углу страницы.
