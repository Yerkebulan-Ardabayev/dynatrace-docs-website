    Script mode для настройки браузерных мониторов¶

    Script mode для настройки браузерных мониторов¶

    • How-to guide
    • 2-min read
    • Published Sep 19, 2018

    Помимо стандартной настройки в интерфейсе, вы можете использовать режим Script для настройки мониторов типа clickpath или по одному URL. В этом режиме доступен базовый JSON-скрипт монитора. Для опытных пользователей Synthetic это значительно упрощает работу и ускоряет создание и управление clickpath-ами. Используйте редактор скриптов, чтобы быстро находить конкретные события (шаги), изменять локаторы по всему скрипту или редактировать части clickpath-а без повторной записи.

    Режимы не ограничены: вы можете переключаться между интерфейсным режимом и режимом скрипта, нажав переключатель Clickpath/Script.

    Доступ к скрипту clickpath-а¶

    Чтобы изменить браузерный монитор-clickpath в режиме скрипта:

    1. Перейдите в Synthetic Classic.
    2. Выберите нужный монитор-clickpath.
    3. Нажмите кнопку Browse (...) и выберите Edit.
    4. Нажмите вкладку Recorded clickpath в меню настроек монитора слева.
    5. Нажмите Script вверху.

    Доступ к скрипту монитора по одному URL¶

    Чтобы изменить браузерный монитор по одному URL в режиме скрипта:

    1. Перейдите в Synthetic Classic.
    2. Выберите нужный монитор по одному URL.
    3. Нажмите кнопку Browse (...) и выберите Edit.
    4. Нажмите вкладку Monitor script в меню настроек монитора слева.

    Редактирование скрипта¶

    Вы можете скачать скрипт (Download script as .json) или скопировать его из редактора и отредактировать в любом текстовом редакторе. Также можно редактировать скрипт прямо в браузере. Это даёт следующие преимущества:

    • Автодополнение: нажмите Ctrl+Spacebar, чтобы увидеть список подсказок.
    • Подсветка синтаксиса упрощает написание кода скрипта.
    • Мгновенная проверка ошибок: редактор сразу отображает предупреждение при любой ошибке. Наведите курсор на ошибку, чтобы узнать причину и получить рекомендацию по исправлению. Сохранение изменений невозможно до устранения всех ошибок.

    Обратите внимание: все специальные символы и переносы строк необходимо экранировать обратным слешем (например, \n для новой строки, \" для двойной кавычки, \t для табуляции).

    Изменения в скриптах clickpath-ов можно воспроизвести без сохранения: нажмите Play back clickpath.

    Главный объект скрипта¶

    Содержит скрипт монитора.

    Элемент Тип Описание Обязательный
    configuration ScriptConfig Настройка монитора Опциональный
    type string Тип монитора. Возможные значения: * clickpath для мониторов-clickpath * availability для браузерных мониторов по одному URL. Для таких мониторов разрешено только одно событие типа navigate. Обязательный
    version string Версия скрипта: используйте значение 1.0. Обязательный
    events Массив из: navigateEvent interactionEvent javaScriptEvent selectOptionEvent cookieEvent keystrokesEvent Шаги clickpath-а: первый шаг всегда должен быть типа navigate. Обратите внимание: событие не равнозначно действию. Действиями называются только события, генерирующие веб-запросы, поэтому в вашем скрипте может быть меньше действий, чем событий. Синтетические действия (аналог пользовательских действий в Real User Monitoring) хранят данные о производительности, собранные при воспроизведении событий clickpath-а. Фактический JSON-объект события зависит от его типа: * navigateEvent — содержит событие Navigate. * interactionEvent — содержит событие Click или Tap. * javaScriptEvent — содержит событие JavaScript. * selectOptionEvent — содержит событие Select option. * cookieEvent — содержит событие Cookie. * keystrokesEvent — содержит событие Keystroke. Обязательный

    Объект ScriptConfig¶

    Содержит настройку монитора.

    Элемент Тип Описание Обязательный
    userAgent string User agent запроса Опциональный
    device customDevice или predefinedDevice Эмулируемое устройство монитора: содержит параметры пользовательского устройства или имя и ориентацию преднастроенного устройства. Если не задано, используется преднастроенное устройство Desktop. Опциональный
    bandwidth bandwidthOptions или predefinedBandwidth Эмулируемые сетевые условия монитора. Если не задано, используется полная доступная пропускная способность. Опциональный
    requestHeaders requestHeader[] Список HTTP-заголовков, отправляемых с запросами монитора Опциональный
    cookies requestCookie[] Список cookie-файлов, создаваемых для монитора. Они добавляются перед выполнением первого шага. Опциональный

    Объект customDevice¶

    Содержит параметры эмулируемого устройства монитора.

    Элемент Тип Описание Обязательный
    mobile boolean Признак мобильного устройства. Укажите true для мобильных устройств или false для настольного компьютера или ноутбука. Обязательный
    touchEnabled boolean Признак сенсорного экрана. Укажите true, если устройство использует сенсорный экран. В этом случае событие взаимодействия можно задать как tap. Обязательный
    width integer Ширина экрана в пикселях. Максимально допустимое значение: 1920. Обязательный
    height integer Высота экрана в пикселях. Максимально допустимое значение: 1080. Обязательный
    scaleFactor integer Коэффициент масштабирования пикселей устройства Опциональный

    Объект predefinedDevice¶

    Содержит одну из преднастроенных эмуляций устройств.

    Элемент Тип Описание Обязательный
    deviceName string Имя преднастроенного устройства. При редактировании в браузере нажмите Ctrl+Spacebar для просмотра списка доступных устройств. Обязательный
    orientation string Ориентация устройства: portrait или landscape. Для настольных компьютеров и ноутбуков ориентация portrait не допускается. Обязательный

    Объект requestHeader¶

    Содержит список HTTP-заголовков, отправляемых с запросами монитора.

    Следующие заголовки не разрешены:

    • user-agent
    • cookie
    Элемент Тип Описание Обязательный
    name string Имя HTTP-заголовка Обязательный
    value string Значение HTTP-заголовка Обязательный

    Объект requestCookie¶

    Содержит список cookie-файлов, создаваемых для монитора.

    Каждый cookie-файл должен быть уникальным в пределах списка. Однако тот же cookie можно использовать повторно в другом событии.

    Элемент Тип Описание Обязательный
    name string Имя cookie-файла. Следующие имена не разрешены: * dtCookie * dtLatC * dtPC * rxVisitor * rxlatency * rxpc * rxsession * rxvt Обязательный
    value string Значение cookie-файла. Следующие символы не допускаются: ;,\". Обязательный
    domain string Домен cookie-файла Обязательный
    path string Путь к cookie-файлу Опциональный

    Объект navigateEvent¶

    Содержит событие Navigate.

    Элемент Тип Описание Обязательный
    type string Тип синтетического события. В данном случае тип события: navigate. Обязательный
    url string URL для перехода Обязательный
    description string Краткое описание события, отображаемое в интерфейсе Обязательный
    wait waitCondition Условие ожидания для события: определяет, как долго Dynatrace будет ждать перед выполнением следующего действия. Опциональный
    validate validationType[] Правило валидации для события: помогает проверить, загружает ли браузерный монитор ожидаемое содержимое или элемент страницы. Опциональный
    target targetType Вкладка, в которой должна открыться страница Опциональный
    authentication plainAuthenticationType или secureAuthenticationType Учётные данные для обхода браузерного диалога входа Опциональный

    Содержит событие Cookie.

    Элемент Тип Описание Обязательный
    type string cookie Обязательный
    description string Краткое описание события, отображаемое в интерфейсе Обязательный
    cookies requestCookie[] Список cookie-файлов, создаваемых во время события. Каждый cookie должен быть уникальным в пределах списка. Однако тот же cookie можно использовать повторно в другом событии. Обязательный

    Содержит событие Keystroke.

    Элемент Тип Описание Обязательный
    type string keystrokes Обязательный
    description string Краткое описание события, отображаемое в интерфейсе Обязательный
    textValue string Вводимый текст Обязательный
    masked boolean Указывает, зашифровано ли значение textValue (true) или нет (false). Обязательный
    simulateBlurEvent boolean Определяет, следует ли размывать текстовое поле при потере фокуса. Укажите true, чтобы применить blur к textValue. Обязательный
    wait waitCondition Условие ожидания для события: определяет, как долго Dynatrace будет ждать перед выполнением следующего действия. Опциональный
    validate validationType[] Правило валидации для события: помогает проверить, загружает ли браузерный монитор ожидаемое содержимое или элемент страницы. Опциональный
    target targetType Объект для ввода текста Опциональный

    Объект waitCondition¶

    Содержит условие ожидания для события.

    Элемент Тип Описание Обязательный
    waitFor string Время ожидания перед запуском следующего события. Возможные значения: * page_complete — ждать полной загрузки страницы. * network — ждать завершения фоновой сетевой активности. * next_action — ждать следующего действия. * time — ждать определённый период времени. * validation — ждать появления конкретного элемента. Обязательный
    milliseconds integer Время ожидания в миллисекундах. Максимально допустимое значение: 60000. Обязательный для типа time. Неприменимо в остальных случаях
    timeoutInMilliseconds integer Максимальное время ожидания появления элемента в миллисекундах. При превышении действие помечается как неудачное. Максимально допустимое значение: 60000. Обязательный для типа validation. Неприменимо в остальных случаях
    validation validationType[] Элемент для ожидания Обязательный для типа validation. Неприменимо в остальных случаях

    Объект validationType¶

    Содержит правило валидации для события или условие ожидания.

    Элемент Тип Описание Обязательный
    type string Цель валидации: * content_match — проверить страницу на наличие конкретного содержимого. Не допускается для валидации внутри условия ожидания. * element_match — проверить страницу на наличие конкретного элемента. Обязательный
    match string Содержимое для поиска на странице. Допускаются регулярные выражения. В этом случае задайте isRegex как true. Обязательный для content_match. Опциональный для element_match
    isRegex boolean Определяет, является ли match обычным текстом (false) или регулярным выражением (true). Опциональный
    failIfFound boolean Условие валидации: * false — валидация успешна, если указанное содержимое/элемент найден. * true — валидация неуспешна, если указанное содержимое/элемент найден. Обязательный
    target targetType Элемент для поиска на странице Обязательный для element_match. Опциональный для content_match

    Объект plainAuthenticationType¶

    Открытые учётные данные для обхода браузерного диалога входа в событии Navigate.

    Элемент Тип Описание Обязательный
    type string Тип аутентификации: http_authentication или webform. Тип webform не разрешён в мониторах-clickpath. Опциональный
    username string Имя пользователя для входа. Обязательный
    password string Пароль для входа. Содержит не сам пароль, а его уникальный идентификатор; сам пароль хранится в Dynatrace и находится по идентификатору. При изменении идентификатора сохранённый пароль становится недоступным. Для смены пароля задайте свойство masked как false и введите новый пароль. Обязательный
    masked boolean Признак маскированного пароля. true означает, что пароль зашифрован и сохранён, а поле password содержит идентификатор для шифрования пароля. Задайте false для ввода нового пароля. Опциональный

    Объект secureAuthenticationType¶

    Учётные данные для обхода браузерного диалога входа в событии Navigate.

    Элемент Тип Описание Обязательный
    type string Тип аутентификации: http_authentication, webform или kerberos. Тип webform не разрешён в мониторах-clickpath. Опциональный
    credential credentialType Объект учётных данных, содержащий имя пользователя и пароль. Обязательный
    domain string Домен Kerberos. Обязательный для kerberos
    authServerAllowlist string Список разрешённых серверов для аутентификации Kerberos. Допускаются подстановочные символы. Подробности см. в документации Chrome Enterprise. Обязательный для kerberos

    credentialField string Тип аутентификации: username или password. Опциональный
