    Метрики потока сервисов¶

    Метрики потока сервисов¶

    • 4-min read
    • Updated on Aug 18, 2022

    При выборе сервиса в потоке сервисов путь к этому сервису — начиная с сервиса, инициировавшего поток, — выделяется синим цветом (см. изображение ниже). Кроме того, стрелки, составляющие путь, обогащены метриками.

    Обзор потока сервисов — вклад в время отклика

    Помимо вклада в время отклика, можно также просматривать Average response time и количество Requests, инициированных каждым сервисом, включённым в поток сервисов. Просто выберите сервис для его развёртывания. После выбора можно просмотреть дополнительные сведения, такие как Avg. time spent in called services и количество Failed requests (см. изображение выше, правый нижний угол).

    Каждый сервис также показывает Response time contribution, рассчитываемый по следующей формуле:

    Вклад в время отклика

    Расхождение значений

    Отображаемое значение вклада в время отклика может отличаться от фактического математического результата, поскольку расчёты основаны на исходных данных и затем округляются.

    В приведённом выше примере потока сервисов они обозначены следующим образом:

    • 1 — Average response time service
    • 2 — Call percentage

    Доля запросов, вызывающих целевой сервис, среди всех запросов, инициированных сервисом. * 3 — Call per request

    Среднее количество вызовов целевого сервиса, включённых в каждый запрос. * 4 — Average response time overall

    Предположим, что поток сервисов показывает следующие значения запросов для вашего сервиса:

    • Average response time = 134ms
    • Call percentage = 6%
    • Calls per request = 1.7

    тогда:

    • Response time contribution = (Average response time) × (Call percentage) × (Calls per request)
    • Response time contribution = 134 × 0.06 × 1.7

    В данном случае Response time contribution = 14%

    Если выбрать Filter service flow для сервиса, представление отфильтруется, чтобы показывать только те запросы, которые вызывают этот сервис. Call percentage тогда вырастет до 100%. Этот набор запросов является подмножеством анализируемого, и поскольку он вносит вклад в каждый запрос в новом представлении, contribution будет выше.

    Детали запросов на боковой панели потока сервисов¶

    Правая часть страницы Service flow содержит панель с деталями запросов к сервису, инициировавшему поток сервисов (easyTravel Customer Frontend на изображении ниже), включающую:

    • Вкладку Passing transactions. Эта вкладка показывает пропускную способность сервиса и его производительность — время отклика, количество запросов и др. Выберите Show more для просмотра дополнительных параметров анализа. Анализ включает все запросы, исходящие от сервиса.
    • Вкладку Infrastructure. Эта вкладка показывает, что easyTravel Customer Frontend работает на двух хостах, и распределение пропускной способности между ними. Выберите Apply as filter для фильтрации потока сервисов — отображаются только вызовы, исходящие от выбранного хоста.

    Поток сервисов — боковая панель

    При выборе сервиса в потоке сервисов нижняя плитка содержит детали выбранного сервиса (на изображении ниже — easyTravel-Business).

    • Верхняя плитка показывает детали в контексте выбранного сервиса. Количество вызовов, исходящих от easyTravel Customer Frontend и последовательно вызывающих easyTravel-Business, составляет 8 490 из 127 000. Выберите Filter service flow для отображения только вызовов, исходящих от easyTravel Customer Frontend и последовательно достигающих easyTravel-Business. Дополнительные параметры анализа также фокусируются на этих вызовах.
    • Между плитками можно видеть список всех промежуточных сервисов и/или прокси между easyTravel Customer Frontend и easyTravel-Business. Можно развернуть каждый раздел и просмотреть его детали, выбрав его.
    • Новая плитка внизу показывает детали вызовов к easyTravel-Business, исходящих от easyTravel Customer Frontend. Дополнительные параметры анализа также фокусируются на этих вызовах.

    Поток сервисов — боковая панель адаптируется к выбранному сервису

    Параметры анализа для конкретного сервиса¶

    В зависимости от типа сервиса нижняя плитка боковой панели адаптируется, предоставляя быстрый доступ к конкретным параметрам анализа.

    • Remote environments Для запросов к подключённым окружениям можно перейти на страницу Remote environments в соответствующем удалённом окружении, выбрав Open environment.
    • Databases Для запросов из баз данных выберите View database statements для использования анализа топ операторов базы данных.

    Дополнительные параметры анализа¶

    • Distributed traces Анализируйте детальную цепочку вызовов на уровне методов.
    • Analyze backtrace Исследуйте последовательность вызовов сервисов, ведущую к конкретному запросу к сервису.
    • View response time Просматривайте распределение времени отклика по различным функциям сервиса (например, использование базы данных, выполнение кода и т.д.).
    • Analyze outliers Просматривайте распределение времени отклика запросов к сервису в конкретном временном периоде.

    Можно указать дополнительные параметры фильтрации для сужения области анализа. О том, как это сделать, см. в разделе Фильтрация потока сервисов. Все эти фильтры также будут применены к дальнейшему анализу.
