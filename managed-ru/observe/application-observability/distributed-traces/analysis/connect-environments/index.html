    Настройка межсредовой трассировки¶

    Настройка межсредовой трассировки¶

    • How-to guide
    • 6-min read
    • Updated on Jun 04, 2024

    В сценариях с несколькими окружениями пулы данных разделены. Если запрос затрагивает сервисы, отслеживаемые в разных окружениях Dynatrace, его трассировки не коррелируются автоматически.

    Чтобы коррелировать и отслеживать такие трассировки, можно настроить соединение между окружениями Dynatrace и включить переключатели заголовков ответов и скоординированной выборки. Поток сервисов тогда отображает связь между подключёнными окружениями, распределённые трассировки ссылаются на подключённое удалённое окружение, и вы получаете доступ к возможностям анализа Davis® AI.

    Настройка¶

    Перед началом¶

    Ограничения

    • Межсредовая трассировка ограничена трассировками запросов, которые могут передавать информацию о заголовках ответов и контексте трассировки из принимающего окружения, например HTTP или синхронных запросов.
    • IBM z/OS (CICS, IMS и Java) и AWS Lambda не поддерживают межсредовую трассировку.

    Предварительные требования

    • Убедитесь, что прямое сетевое соединение между кластерами может быть установлено через HTTP/HTTPS.

    Эквиваленты API

    Следующие процедуры используют веб-интерфейс Dynatrace. Для выполнения аналогичных задач через API см.:

    • API токенов доступа — для создания токена в удалённом окружении
    • API удалённых окружений — для создания ссылки на удалённое окружение из локального окружения

    Шаги¶

    1. Создайте токен доступа

    С помощью этой процедуры вы получаете токен доступа из удалённого окружения, который понадобится на других шагах.

    Чтобы создать токен доступа в удалённом окружении Dynatrace

    1. Войдите в удалённое окружение.

    2. Это окружение, из которого вы извлекаете данные.

    3. Если вы не можете войти в удалённое окружение, эту процедуру может выполнить пользователь с доступом к нему.
    4. Перейдите в Access Tokens.
    5. Выберите Generate new token.
    6. Введите имя токена.

    7. Найдите и выберите следующие области доступа:

    8. Look up a single trace (traces.lookup)

    9. Fetch data from a remote environment (RestRequestForwarding)
    10. Выберите Generate token.
      Создаётся токен, предоставляющий вашему локальному окружению разрешение на проверку существования трассировки в удалённом окружении и извлечение данных из него.

    11. Выберите Copy и вставьте токен в безопасное место.
      Это длинная строка, которую нужно скопировать и вставить обратно в Dynatrace позже.

    12. Добавьте удалённое окружение

    Чтобы добавить удалённое окружение Dynatrace в список доступных удалённых окружений

    1. Войдите в своё локальное окружение Dynatrace.
    2. Перейдите в Settings.
    3. Выберите Integration > Remote environments.
    4. Выберите Connect environment.

    5. Определите удалённое окружение, из которого ваше локальное окружение извлекает данные, и выберите Save changes.

    6. Name — имя, под которым удалённое окружение будет отображаться в вашем текущем окружении Dynatrace. Это произвольный текст, не влияющий на удалённое окружение.

    7. Remote environment URI

      • Для Dynatrace SaaS формат должен быть следующим:

      https://<ENVIRONMENTID>.live.dynatrace.com/

      Замените <ENVIRONMENTID> фактическим идентификатором окружения. + Для Dynatrace Managed допускается любой URI. + Чтобы подключить окружение Dynatrace (развёртывание SaaS) к развёртыванию Dynatrace Managed через URI, находящийся за пределами домена dynatrace-managed.com, обратитесь к эксперту по продуктам Dynatrace через чат в вашем окружении Dynatrace. * Network scope

      • External: удалённое окружение находится в другой сети. Если настроены глобальные параметры прокси, они используются. Это область по умолчанию.
      • Internal: удалённое окружение находится в той же сети. Глобально настроенные параметры прокси не используются.
      • Cluster: удалённое окружение находится в том же кластере. Запрос выполняется к localhost.

      Dynatrace SaaS может использовать только область External. * Token — токен, созданный в предыдущей процедуре. Он должен включать область Look up a single trace (traces.lookup). * Test connection — проверяет соединение из вашего текущего окружения с удалённым.

      Убедитесь, что получено сообщение connection successfully established, прежде чем продолжать.

    8. Включите функции

    Включите следующие функции в каждом окружении группы, которую нужно соединить:

    • Coordinated sampling

    Вы можете повысить согласованность межсредовой трассировки с помощью скоординированной выборки. При включении этого переключателя выборка координируется между окружениями с использованием W3C trace ID.

    1. Перейдите в Settings > Preferences > OneAgent features.
    2. Настройте распространение контекста трассировки W3C в соответствии с вашими предпочтениями.
      Вы можете отправлять HTTP-заголовки контекста трассировки W3C или заголовки gRPC.
    3. Найдите и включите Cross-environment tracing - Coordinated sampling.
    4. Response headers

    Вы можете включить идентификацию и отслеживание межсредовых вызовов, включая идентификатор окружения и идентификатор трассировки в HTTP-заголовках ответов. Если вызывающее окружение, получающее ответ, настроило данное окружение как доверенное удалённое, назначение вызова связывается.

    1. Перейдите в Settings и выберите Preferences > OneAgent features.
    2. Найдите и включите Cross-environment tracing - Environment and transaction IDs in HTTP response headers.

    После соединения окружений вы можете анализировать трассировки в межсвязанных удалённых окружениях.

    Устранение неполадок¶

    Сообщение Verification failed, please check your settings: Constraints violated. при добавлении удалённого окружения

    Анализ межсвязанных удалённых окружений¶

    После настройки соединения между окружениями доступны следующие специальные параметры анализа.

    Обзор сервиса¶

    Dynatrace обнаруживает запросы к подключённым окружениям Dynatrace и использует доступную информацию для создания связанного сервиса веб-запросов или веб-запросов Remote environment.

    Обзор сервиса удалённого окружения

    • Исходящие вызовы к подключённым окружениям отображаются в Smartscape и Service flow.

    Сервис удалённого окружения в топологии Smartscape * Все сервисы, отслеживаемые удалённым окружением и участвующие в запросе, перечислены на странице Remote environments. Чтобы открыть её, выберите Open environment на странице сервиса Remote environment.

    Поток сервисов¶

    Service flow распознаёт вызовы удалённых окружений. Поскольку запросы отслеживаются другим окружением Dynatrace, при выборе вызова удалённого окружения доступен специальный вариант детализации. На боковой панели выберите Open environment, чтобы перейти на страницу Remote environments.

    Распределённые трассировки¶

    • Когда запрос поступает в подключённое удалённое окружение, Dynatrace отображает имя удалённого окружения в дереве вызовов.

    Чтобы проследить трассировку в удалённом окружении

    1. Выберите запрос к удалённому окружению.
    2. На вкладке Summary выберите Open environment.
    3. Если соединение окружений настроено правильно, Dynatrace автоматически отображает Aggregated requests. Эти данные агрегируют все обнаруженные запросы в каждом подключённом удалённом окружении.

    Если вы настроили соединение с удалённым окружением, но запросы в нём не отображаются, можно запустить ручной поиск трассировки в настроенных окружениях. В правом верхнем углу страницы трассировки выберите Find in remote environments.

    Davis AI¶

    После создания Dynatrace сервиса удалённого окружения проблемы и события коррелируются с затронутым сервисом удалённого окружения.

    Вы можете использовать возможности Davis AI для понимания того, какие запросы завершаются с ошибкой, и выявления первопричины проблем в удалённом окружении.
