    Начало работы с распределёнными трассировками¶

    • How-to guide
    • 6-min read
    • Updated on Aug 13, 2024

    Distributed Traces Classic предоставляет набор инструментов анализа для получения детальной информации о транзакциях в вашем окружении. Он сочетает видимость на уровне кода, информацию о топологии и метаданные с высочайшим уровнем детализации и точности данных. В этой статье вы ознакомитесь с функциональностью Distributed Traces Classic и тем, как использовать её для анализа запросов в вашем окружении.

    Быстрый старт¶

    Чтобы начать анализ распределённой трассировки

    1. Перейдите в Distributed Traces.
    2. Необязательно: используйте диаграмму Overview, чтобы понять количество запросов и время отклика за выбранный временной период. Хранение данных

    Диаграмма использует данные трассировок и запросов с разными периодами хранения. Для временных периодов, содержащих данные старше 10 дней, можно включить переключатель Show data retention, чтобы лучше понять, какие данные доступны за какой период непосредственно на диаграмме. 3. Необязательно: в таблице разверните детали трассировки, которую нужно проанализировать, для просмотра её топологии, атрибутов, информации HTTP и временных показателей. 4. Выберите имя распределённой трассировки для углублённой аналитики. 5. Выберите временной сегмент, чтобы развернуть вкладки с подробностями.

    Обзор¶

    Чтобы настроить представление распределённых трассировок в вашем окружении

    1. Перейдите в Distributed Traces.

    Обзор распределённых трассировок 2. Настройте представление с помощью фильтров. Для фильтрации распределённых трассировок по

    • Методу приёма данных

      • Выберите PurePaths, чтобы просмотреть распределённые трассировки PurePath®, захваченные OneAgent.
      • Выберите Ingested traces, чтобы просмотреть распределённые трассировки, инструментированные другими библиотеками.
      • Сервису

      Перейдите в Filter requests > Service name и введите имя сервиса. Обратите внимание, что это представление также доступно через Services > More (…) > Distributed traces для соответствующего сервиса.

    Экспорт данных обзора¶

    Вы можете экспортировать данные таблицы обзора распределённых трассировок в файл CSV.

    1. Перейдите в Distributed Traces.
    2. Необязательно: уточните результаты таблицы, изменив настройки представления или воспользовавшись полем поиска над таблицей.

    Таблица отображает до 3 000 наиболее актуальных трассировок, захваченных за выбранный временной период в пределах выбранной зоны управления. В зависимости от временного периода и настроенного представления в списке выделяется наиболее важный узел трассировки.

    Поиск узла

    Чтобы найти конкретный узел трассировки

    1. Перейдите в Distributed Traces.
    2. Отфильтруйте таблицу по идентификатору трассировки.
    3. Выберите имя трассировки.
    4. В представлении Trace найдите нужный узел.
    5. В правом нижнем углу страницы выберите меню Show export.

    Меню Show export 4. Выберите один из следующих вариантов.

    Export table data Все данные таблицы, включая трассировки, не отображаемые на текущей странице Все доступные поля, связанные с распределёнными трассировками Все 3 000 наиболее актуальных трассировок

    Анализ отдельной трассировки¶

    Представление анализа распределённой трассировки предоставляет сквозную визуализацию запросов в виде каскадной диаграммы для одной трассировки.

    Чтобы перейти к представлению отдельной трассировки

    1. Перейдите в Distributed Traces.
    2. Выберите имя распределённой трассировки в таблице.
    3. Необязательно: перейдите на вкладку Execution breakdown, чтобы узнать распределение времени трассировки.
    4. Просмотрите каскадную диаграмму, чтобы узнать, какие сервисы проходит трассировка и в каком порядке.
    5. Выберите запрос, чтобы увидеть последующие взаимодействия.

    Используйте цвета и позиции горизонтальных полос на диаграмме, чтобы определить, какие запросы выполнялись синхронно (параллельно). Они показывают как последовательность, так и время отклика каждого запроса.

    Распределённые трассировки — обзор трассировки 6. Выберите временной сегмент для углублённого анализа трассировки на вкладках с подробностями вплоть до уровня кода. 7. Выберите вкладку с подробностями для продолжения анализа.

    Часто задаваемые вопросы¶

    Что означает [declared unavailable by agent]?

    [declared unavailable by agent] — это значение-заполнитель, отображаемое в представлении трассировки, когда метаданные недоступны для захвата OneAgent, например из-за отсутствия видимости данных или ограничений в их передаче (например, технология не поддерживается OneAgent). Рекомендуется обратиться к специалисту по продуктам Dynatrace через чат в вашем окружении Dynatrace для разбора конкретной ситуации.
