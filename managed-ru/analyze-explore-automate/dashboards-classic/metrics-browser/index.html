    Браузер метрик¶

    Браузер метрик¶

    • Практическое руководство
    • Чтение: 5 мин
    • Обновлено 1 августа 2022 г.
    • Early Access

    Перейдите в Metrics, чтобы открыть браузер метрик (Metrics) — удобный инструмент для просмотра всех метрик, доступных в вашей среде мониторинга, и быстрого создания визуализации для отдельной метрики.

    • Dashboards Classic: визуализацию можно закрепить на классическом дашборде.

    Фильтрация и сортировка таблицы¶

    • По умолчанию таблица метрик отфильтрована для отображения только тех метрик, последний отчёт по которым поступил после начала выбранного временного диапазона (например, Last 2 hours). Отключите Only show metrics reported after the start of the selected timeframe, чтобы видеть все метрики независимо от времени последнего отчёта.

    • Установите Favorites в значение Any, Yes или No, чтобы фильтровать таблицу по метрикам, добавленным в избранное.

    • Чтобы добавить метрику в избранное, нажмите звёздочку в столбце Favorite таблицы. Избранные метрики по умолчанию сортируются в начало таблицы.

    • Чтобы удалить метрику из избранного, снова нажмите звёздочку.

    • Строка фильтров (Filtered by) содержит следующие параметры:

    • Text — выберите Text, введите строку фильтра и нажмите Enter, чтобы отобразить метрики, содержащие строку в полях Metric name, Metric key или Description.

    • Tag — выберите Tag, введите строку фильтра и выберите соответствующий тег, чтобы отобразить метрики с этим тегом в поле Tags.
    • Unit — выберите Unit, введите строку фильтра и выберите соответствующую единицу измерения (например, Percent), чтобы отобразить метрики с выбранной единицей.
    • Favorites — выберите Favorites, затем Yes (только избранные метрики) или No (только неизбранные метрики).

    • Dimension — выберите Dimension, введите строку фильтра и выберите соответствующее измерение метрики (например, Host), чтобы отобразить метрики с выбранным измерением.

      При объединении фильтров они применяются по условию И. Например, если задать Dimension: Host и Text: usage, браузер метрик отобразит все метрики с измерением Host, в которых также присутствует строка usage в полях Metric name, Metric key или Description. * Нажмите заголовок столбца Favorite, Name или Key, чтобы отсортировать по выбранному столбцу. * Нажмите в столбце Details, чтобы просмотреть подробную информацию о метрике.

    Просмотр подробных сведений о метрике¶

    Разверните Details для любой метрики (строки) в таблице, чтобы увидеть её подробные сведения и визуализацию за выбранный временной диапазон.

    Подробные сведения о метрике в браузере метрик

    Поле Описание
    Metric name Имя метрики в пользовательском интерфейсе.
    Metric key Полный ключ метрики. Если применялось преобразование, оно отражается в ключе метрики.
    Entity type Тип объекта для данной метрики.
    Description Краткое описание метрики.
    Tags Теги для дополнительной группировки метрик.
    Created Метка времени создания метрики.
    Last written Метка времени последней записи метрики.
    DDU billing Относится ли метрика к потреблению единиц данных Davis (DDU).
    Unit Единица измерения метрики.
    Minimum value Известное нижнее граничное значение метрики.
    Maximum value Известное верхнее граничное значение метрики.
    Default aggregation Агрегация по умолчанию для данной метрики.
    Aggregations Список допустимых агрегаций для данной метрики.
    Dimensions Детальное деление метрики (например, группа процессов и идентификатор процесса для метрики, связанной с процессом).
    Transformations Возможные операторы преобразования.

    Нотация порядка величины¶

    В этом примере значения отображаются в миллионах. Это нотация порядка величины (7.5M означает «приблизительно 7,5 миллиона», а не «ровно 7,5 миллиона»).

    Порядок величины выбирается автоматически в зависимости от размера значений. Например, та же метрика за более короткий временной диапазон может отображаться в значениях k, а не M.

    Пример значений в нотации порядка величины в браузере метрик.

    Примеры нотации порядка величины в Dynatrace:

    Обозначение Множитель Значение
    k 10^3 кило, тысяча
    M 10^6 мега, миллион
    G 10^9 гига, миллиард
    T 10^12 тера, триллион

    Подробности см. в разделе Нотация порядка величины.

    Добавление метрики на дашборд¶

    Dashboards Classic

    Чтобы добавить метрику на классический дашборд

    1. Нажмите в столбце Details, чтобы просмотреть подробные сведения о метрике.
    2. Нажмите Create chart, чтобы открыть метрику в Data Explorer.
    3. Настройте запрос и параметры визуализации по необходимости. Подробности см. в разделах Data Explorer или Быстрый старт Data Explorer.
    4. Нажмите Pin to dashboard.
    5. Укажите целевой дашборд и заголовок метрики.
    6. Нажмите Pin.

    Метаданные для пользовательских метрик¶

    Чтобы предоставить контекстную информацию для пользовательских метрик (например, определить единицу измерения, задать отображаемые имена, описания или даже нижнее и верхнее граничные значения и данные, релевантные для Davis), можно создать метаданные метрики для каждого ключа метрики.

    После предоставления эта информация становится частью дескриптора метрики и может запрашиваться через API, а также использоваться в браузере метрик и Data Explorer.

    • Предоставление метаданных через Settings 2.0 доступно только для пользовательских/бессхемных метрик, но не для встроенных/зарегистрированных в коде метрик.
    • Метаданные, предоставленные для конкретного ключа метрики, существуют параллельно с самими временными рядами метрики. Метаданные можно создать ещё до первого приёма точек данных.

    Настройка через API¶

    Метаданные метрик полностью настраиваются через API. Подробности см. в разделе Метаданные пользовательских метрик.

    Настройка через веб-интерфейс¶

    Чтобы настроить метаданные метрики для пользовательских метрик через веб-интерфейс

    1. Перейдите в Metrics.
    2. Нажмите в столбце Details, чтобы просмотреть подробные сведения о метрике.
    3. Нажмите Edit metadata.
      Если кнопка Edit metadata в развёрнутой строке отсутствует, выбрана не пользовательская метрика.

    4. На странице Metric metadata отредактируйте метаданные выбранной метрики.
      Если страница отображает значок замка, у вас нет необходимых прав на запись для редактирования метаданных выбранной метрики.

    5. Display name — введите понятное имя метрики.

    6. Description — введите произвольное описание метрики: как она измеряется и т. п.
    7. Unit — выберите единицу измерения из списка.

    8. Metric properties

      • Minimum value — нижняя граница метрики.
      • Maximum value — верхняя граница метрики.
      • Root cause relevant — метрика, релевантная для корневой причины, является сильным индикатором неисправного компонента.
      • Impact relevant — метрика, релевантная для влияния, сильно зависит от других метрик и изменяется из-за изменения базовой метрики корневой причины.

      • Value type — тип значения метрики. Доступные варианты:

      • score: метрика оценки — метрика, в которой высокие значения указывают на хорошее состояние, а низкие — на проблему. Пример: показатель успешности.

      • error: метрика ошибок — метрика, в которой высокие значения указывают на проблему, а низкие — на хорошее состояние. Пример: количество ошибок.
      • Latency — задержка метрики в минутах. Указывает, сколько времени проходит до появления нового принятого значения метрики в Dynatrace.
      • Metric dimensions — чтобы добавить измерение, нажмите Add dimension и введите Dimension key и Display name для измерения.
      • Tags — чтобы добавить тег, нажмите Add tag и введите тег.
      • Source entity type — для настройки типа объекта метрики. Редактируется только если метрика поступает через конечную точку API.
      • Нажмите Save changes, чтобы сохранить изменения.
