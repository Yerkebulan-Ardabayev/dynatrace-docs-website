    Классические дашборды

    Ниже описаны плитки, которые можно добавлять на дашборды.

    Советы по редактированию

    • Чтобы создать несколько похожих плиток, начните с одной удачной плитки, клонируйте её и затем отредактируйте клоны.

    Клонировать плитку на текущий дашборд

    1. Перейдите в Dashboards.
    2. Выберите название дашборда, чтобы открыть его.

    3. Нажмите Edit в правом верхнем углу дашборда. Дашборд откроется в режиме редактирования.

      • Если кнопка Edit отсутствует, у вас нет прав на редактирование этого дашборда.

      • Выберите плитку, которую хотите клонировать (добавить копию на текущий дашборд), и нажмите Clone.

      • Чтобы клонировать несколько плиток на текущий дашборд, выделите нужные плитки прямоугольником выделения и нажмите Clone x tiles.

      • Отредактируйте клонированные плитки по необходимости.

    Клонировать плитку на другой дашборд

    1. Перейдите в Dashboards.
    2. Выберите название дашборда, чтобы открыть его.

    3. Нажмите Edit в правом верхнем углу дашборда. Дашборд откроется в режиме редактирования.

      • Если кнопка Edit отсутствует, у вас нет прав на редактирование этого дашборда.

      • Выберите плитку, которую хотите клонировать на другой дашборд, и нажмите Clone to.

      • Чтобы клонировать несколько плиток на другой дашборд, выделите нужные плитки прямоугольником выделения и нажмите Clone x to.

      • В всплывающем окне Where do you want to pin to? выберите существующий дашборд или нажмите Create new dashboard.
      • Нажмите Pin, чтобы добавить выбранные плитки на дашборд.
      • Отредактируйте клонированные плитки по необходимости.

    4. Чтобы создать несколько похожих дашбордов с одинаковым набором плиток, начните с одного удачного дашборда, клонируйте его, а затем отредактируйте клоны дашбордов и плиток.

    Клонировать существующий дашборд

    1. Перейдите в Dashboards.

    2. В таблице дашбордов нажмите More (⋯) > Clone для нужного дашборда.

      • Копия откроется в режиме редактирования.
      • Исходный дашборд не изменится.

    Визуализации¶

    Используйте плитки визуализации для создания наглядных представлений запросов Data Explorer, которые можно закрепить на дашбордах.

    Типы визуализаций¶

    Dynatrace предлагает следующие типы визуализаций:

    • График
    • Столбчатая диаграмма с накоплением
    • Диаграмма площадей с накоплением
    • Круговая диаграмма
    • Единственное значение
    • Таблица
    • Топ-список
    • Тепловая карта
    • Соты

    Интерактивность¶

    Интерактивность плиток визуализации варьируется в зависимости от типа, однако все они обладают общими возможностями.

    • Наведите курсор на элемент (например, линию или сектор круговой диаграммы), чтобы увидеть подробности во всплывающей подсказке.
    • Щёлкните по элементу и выберите кнопку во всплывающей подсказке для перехода к детальному просмотру. Например, в визуализации, где для каждого хоста отображается отдельная линия, щёлкните по интересующей точке линии, а затем нажмите кнопку View host для перехода на страницу выбранного хоста.
    • Легенда активна: нажмите на запись в легенде, чтобы показать или скрыть соответствующий элемент на визуализации.

    • У всех плиток есть меню в правом верхнем углу:

    • Configure tile in Data Explorer — открывает плитку в Data Explorer, где можно настроить запрос и тип визуализации.

    • Edit tile (при наличии прав редактирования) — открывает дашборд в режиме редактирования с выбранной плиткой.

    Настройка¶

    Настройка плитки визуализации из редактора дашборда

    1. Откройте дашборд и нажмите Edit.
    2. Перетащите плитку визуализации из панели Edit dashboard, вкладки Tiles, на дашборд.
    3. Нажмите Configure tile, чтобы открыть плитку в Data Explorer.

    4. Настройте запрос для плитки. Финальную версию не нужно создавать сразу — запрос можно строить постепенно.

    5. Нажимайте Run query после каждого изменения запроса, чтобы видеть результаты.

    6. При желании выберите другой тип визуализации и оцените, что лучше подходит для вашего запроса.
    7. Набор доступных визуальных настроек зависит от запроса и типа визуализации.

    8. Завершив определение запроса:

    9. Чтобы сохранить изменения в исходной плитке и дашборде, нажмите Save changes to dashboard.

    10. Чтобы сохранить изменения в другой плитке или на другом дашборде, нажмите Pin to dashboard и укажите дашборд назначения и заголовок плитки.
    11. Нажмите Open dashboard.
    12. При желании измените заголовок плитки.
    13. При желании выберите пользовательский временной диапазон.
    14. При желании выберите пользовательскую зону управления.
    15. При желании выберите окружение.

    Настройка плитки визуализации из Data Explorer

    1. Перейдите в Data Explorer.

    2. Настройте запрос для плитки. Финальную версию не нужно создавать сразу — запрос можно строить постепенно.

    3. Нажимайте Run query после каждого изменения запроса, чтобы видеть результаты.

    4. Пробуйте разные типы визуализации, чтобы найти наиболее подходящий для вашего запроса.

    5. Набор доступных визуальных настроек зависит от запроса и типа визуализации. Подробнее:

      • График
      • Столбчатая диаграмма с накоплением
      • Диаграмма площадей с накоплением
      • Круговая диаграмма
      • Единственное значение
      • Таблица
      • Топ-список
      • Тепловая карта
      • Соты
      • Некоторые визуализации (например, тепловые карты) могут отображать только одну метрику. Чтобы отобразить другую метрику, выберите букву рядом с нужной метрикой. В этом примере мы изменили бы выбор с A (CPU usage %) на B (Memory used %).

      Изменение выбранной метрики

      Другие визуализации (например, таблицы) могут отображать несколько метрик. Чтобы изменить набор отображаемых метрик, выбирайте буквы рядом с их названиями. В приведённом примере можно изменить выбор с A (только CPU usage %) на A и B (CPU usage % и Memory used %). 3. Когда результаты запроса вас устраивают, нажмите Pin to dashboard и укажите дашборд назначения и заголовок плитки. 4. Нажмите Open dashboard, чтобы увидеть плитку на дашборде. 5. При желании измените заголовок плитки. 6. При желании выберите пользовательский временной диапазон. 7. При желании выберите пользовательскую зону управления. 8. При желании выберите окружение.

    Контекст¶

    Используйте контекстные плитки (Header, Markdown и Image), чтобы поясять содержимое дашборда и добавлять графику, например логотипы компаний. Это особенно важно при создании дашбордов для совместного использования.

    Заголовок (Header)¶

    Используйте плитку-заголовок, чтобы разместить жирную метку над другой плиткой.

    Markdown¶

    Используйте одну или несколько плиток Markdown для описания дашборда: его назначения, способа использования и т. д.

    Ограничения: 1000 символов на одну плитку Markdown.

    Синтаксис Markdown

    Заголовки¶

    Поддерживаются уровни заголовков Markdown от # до ######.

    Горизонтальные линии¶

    Используйте ***, ___ или --- на отдельной строке для добавления горизонтальной разделительной линии между секциями плитки.

    Переносы строк¶

    Оставьте два пробела в конце строки для принудительного переноса строки.

    Выделение¶

    Используйте **текст** или __текст__ для отображения жирного текста.

    Списки¶

    Можно создавать нумерованные (1.) или маркированные (*) списки Markdown, а также их комбинации с вложенностью.

    Ссылки¶

    Если у вас есть избранные страницы Dynatrace или внешние сайты, вы можете добавлять ссылки на них прямо с дашбордов.

    Ссылка состоит из двух частей:

    • Текст ссылки в квадратных скобках: [ и ]. Произвольный текст между квадратными скобками, например [Dynatrace].

    • Адрес ссылки в круглых скобках: ( и ). Адрес задаётся двумя способами:

    • Абсолютный URL. Например, для ссылки на:

      https://www.example.com/

      поместите полный URL в круглые скобки:

      (https://www.example.com/)

      получится определение ссылки вида:

      [Example](https://www.example.com/)
      + Якорная ссылка на страницу Dynatrace (включая другие дашборды). Чтобы получить якорь, откройте целевую страницу в Dynatrace и скопируйте всё, что находится в адресной строке браузера после домена и слеша. Если полный URL целевой страницы:

      https://myenvironment.live.dynatrace.com/ui/deploymentstatus/oneagents?gtf=-2h&gf=all

      то текст якоря:

      ui/deploymentstatus/oneagents?gtf=-2h&gf=all

      и полное определение ссылки выглядит примерно так:

      [Ссылка на статус развёртывания](ui/deploymentstatus/oneagents?gtf=-2h&gf=all)

      Аналогично, если добавить следующий пример в плитку Markdown на одном из ваших дашбордов, ссылка откроет таблицу хостов Dynatrace:

      [Хосты](#newhosts;gtf=-2h;gf=all)

    Изображение (Image)¶

    Добавляйте изображения на дашборды, чтобы улучшить их внешний вид и адаптировать для презентаций.

    Поддерживаемые форматы: JPG/JPEG, GIF, PNG, WEBP, TIFF, BMP, SVG

    Изображение можно загрузить или указать ссылку на него.

    В отличие от других типов плиток, плитка с изображением не обновляется автоматически. Для обновления изображений необходимо обновить дашборд вручную.

    Загрузить изображение

    Чтобы загрузить изображение и отобразить его на дашборде:

    1. Перейдите в Dashboards.
    2. Выберите название дашборда, чтобы открыть его.

    3. Нажмите Edit в правом верхнем углу дашборда. Дашборд откроется в режиме редактирования.

    4. Если кнопка Edit отсутствует, у вас нет прав на редактирование этого дашборда.

    5. Перетащите плитку Image в нужное место.
    6. На панели Image выберите вкладку Upload an image.
    7. Нажмите Upload image.
    8. Найдите и выберите файл изображения для отображения в плитке.
    9. Измените размер и положение плитки по необходимости.

    Указать изображение по URL

    Чтобы указать изображение по URL, сначала необходимо добавить URL в список разрешённых, после чего на него можно ссылаться из плитки с изображением.

    Добавить URL в список разрешённых

    1. Перейдите в Settings и выберите Dashboards > Allowed URL pattern rules.
    2. Нажмите Add item.

    3. Задайте параметр Rule, который определяет способ обработки этой записи в списке разрешённых.

    4. Starts with — разрешить любое изображение, URL которого начинается с содержимого поля Pattern.

    5. Exact — разрешить конкретное изображение, URL которого точно совпадает с содержимым поля Pattern.

    6. Задайте параметр Pattern:

    7. Чтобы задать начало URL, введите достаточно символов, чтобы любые совпадающие URL изображений подходили для ваших дашбордов. Пример: введите https://example.com/images/, чтобы разрешить любое изображение, URL которого начинается с https://example.com/images/, например https://example.com/images/image-x.jpg и https://example.com/images/my-picture.svg

    8. Чтобы задать точный URL, введите полный URL изображения. Пример: введите https://example.com/images/my-image-file-name.jpg, чтобы разрешить только это изображение.
    9. Нажмите Save changes, чтобы добавить указанное правило в список разрешённых.

    Добавить плитку с изображением

    1. Перейдите в Dashboards.
    2. Выберите название дашборда, чтобы открыть его.

    3. Нажмите Edit в правом верхнем углу дашборда. Дашборд откроется в режиме редактирования.

    4. Если кнопка Edit отсутствует, у вас нет прав на редактирование этого дашборда.

    5. Перетащите плитку Image в нужное место.
    6. На панели Image выберите вкладку Add image URL.
    7. Введите URL файла изображения для отображения в плитке. Он должен соответствовать одному из правил в списке разрешённых.
    8. Измените размер и положение плитки по необходимости.

    Инфраструктура¶

    Работоспособность хостов (Host health)¶

    Отображает количество хостов (экземпляров операционных систем, физических и виртуальных) в вашей среде в сравнении с числом хостов, на которых в данный момент зафиксированы проблемы. Каждый экземпляр хоста соответствует установленному в среде Dynatrace OneAgent.

    Детализация¶

    Детализация для плиток работоспособности

    • На плитке работоспособности (например, Host health, Service health, Application health), где отображаются зелёные и красные элементы (хосты, сервисы, приложения), наведите курсор на проблемный (красный) элемент, чтобы увидеть его идентификатор.
    • Для перехода к проблемному (красному) элементу щёлкните по красному шестиугольнику и нажмите кнопку View…. В этом примере плитки Synthetic monitor health мы выбрали красный элемент, что активировало кнопку детализации View test для него.

    Переход к проблемной сущности с плитки работоспособности * Переход по здоровому (зелёному) элементу невозможен. Однако с любой плитки можно открыть меню в её правом верхнем углу и выбрать View details, чтобы перейти на соответствующую страницу Dynatrace. Например, View details с плитки Synthetic monitor health открывает таблицу Synthetic monitors.

    Открыть меню плитки

    Выбрать «View details» в меню плитки

    Настройка¶

    Настройка данного типа плитки из редактора дашборда

    1. Перетащите плитку Host health из панели Edit dashboard, вкладки Tiles, на дашборд.
    2. При желании выберите, показывать ли визуализацию на плитке.
    3. При желании выберите пользовательский временной диапазон.
    4. При желании выберите пользовательскую зону управления.
    5. При желании выберите окружение.

    Закрепить плитку на дашборде с заданными фильтрами

    1. Перейдите в Hosts.
    2. Задайте фильтры в поле Filtered by.

    3. Нажмите Pin to dashboard, выберите целевой дашборд и нажмите Pin, чтобы добавить плитку Hosts с выбранными фильтрами. После этого можно задать любые из описанных выше параметров.

    4. Плитка будет отражать все заданные фильтры и параметры.

    5. При следующем открытии плитки на дашборде страница Hosts откроется с уже применёнными фильтрами и параметрами.

    Сетевые метрики (Network metrics)¶

    Отображает показатели сетевого здоровья для трафика, проходящего через отслеживаемые хосты. Показывает текущий объём трафика и качество связи как для новых соединений (Connectivity), так и для установленных сессий (Retransmissions).

    Детализация¶

    • В меню плитки выберите View details, чтобы открыть страницу Host networking.

    Настройка¶

    Настройка данного типа плитки из редактора дашборда

    1. При желании выберите пользовательский временной диапазон.
    2. При желании выберите пользовательскую зону управления.
    3. При желании выберите окружение.

    Состояние сети (Network status)¶

    Отображает текущий сетевой трафик через отслеживаемые хосты. Показывает объём трафика, количество узлов (Talkers), обменивающихся трафиком, и количество узлов с проблемами производительности (Processes и Hosts).

    Детализация¶

    • В меню плитки выберите View details, чтобы открыть страницу Host networking.

    Настройка¶

    Настройка данного типа плитки из редактора дашборда

    1. Перетащите плитку Network status из панели Edit dashboard, вкладки Tiles, на дашборд.
    2. При желании выберите, показывать ли визуализацию на плитке.
    3. При желании выберите пользовательский временной диапазон.

    • На плитке работоспособности (например, Host health, Service health, Application health), где отображаются зелёные и красные элементы (хосты, сервисы, приложения), наведите курсор на проблемный (красный) элемент, чтобы увидеть его идентификатор.
    • Для перехода к проблемному (красному) элементу щёлкните по красному шестиугольнику и нажмите кнопку View…. В этом примере плитки Synthetic monitor health мы выбрали красный элемент, что активировало кнопку детализации View test для него.

    Переход к проблемной сущности с плитки работоспособности * Переход по здоровому (зелёному) элементу невозможен. Однако с любой плитки можно открыть меню в её правом верхнем углу и выбрать View details, чтобы перейти на соответствующую страницу Dynatrace. Например, View details с плитки Synthetic monitor health открывает таблицу Synthetic monitors.

    Карта мира (World map)¶

    Отображает географическую карту выбранной метрики для выбранного приложения.

    Временной диапазон плитки с картой мира всегда составляет Last 2 hours, независимо от глобального или установленного на дашборде диапазона. Если необходим другой период, откройте полную карту мира и измените временной диапазон там.

    Детализация¶

    • Чтобы открыть полноразмерную карту мира для выбранной метрики и местоположения, щёлкните по плитке или выберите View details в меню плитки.

    Настройка¶

    Настройка данного типа плитки из редактора дашборда

    1. Перетащите плитку World map из панели Edit dashboard, вкладки Tiles, на дашборд.
    2. Выберите приложение (или Most active application) из списка.
    3. Выберите геолокацию.
    4. Выберите метрику для отображения на карте. Доступно две категории; можно выбрать только один элемент.

    Метрика производительности на основе действий пользователей:

    • Apdex
    • User actions
    • Load actions
    • XHR actions
    • Custom actions
    • Ошибки

    Метрика поведения на основе сессий или пользователей:

    • Active sessions
    • Active users
    • Actions per session
    • Bounce rate
    • Session duration

    Закрепить плитку на дашборде с заданными фильтрами

    1. Перейдите в Custom Applications, Frontend, Mobile или Web в зависимости от типа приложения.
    2. Выберите приложение.
    3. На инфографике нажмите View geolocation breakdown.
    4. В разделе Geolocation breakdown, который появится под инфографикой, нажмите View full world map.
    5. В строке фильтров задайте фильтр Location, чтобы приблизить карту к этому местоположению.
    6. Нажмите кнопку метрики (Apdex, User actions, Load actions, XHR actions, Custom actions или Errors), чтобы сфокусировать карту на этой метрике.
    7. Нажмите Pin to dashboard, выберите целевой дашборд и нажмите Pin.

    Устранение неполадок:

    • Если данные на карте мира не отображаются, возможно, потребуется сопоставить внутренние IP-адреса с местоположениями для веб-приложений, мобильных приложений и пользовательских приложений.
    • Помните, что временной диапазон плитки с картой мира всегда равен Last 2 hours, независимо от глобального или установленного на дашборде диапазона.

    • Для перехода к проблемному (красному) элементу щёлкните по красному шестиугольнику и нажмите кнопку View…. В этом примере плитки Synthetic monitor health мы выбрали красный элемент, что активировало кнопку детализации View test для него.

    Переход к проблемной сущности с плитки работоспособности * Переход по здоровому (зелёному) элементу невозможен. Однако с любой плитки можно открыть меню в её правом верхнем углу и выбрать View details, чтобы перейти на соответствующую страницу Dynatrace. Например, View details с плитки Synthetic monitor health открывает таблицу Synthetic monitors.

    Проблемы (Problems)¶

    Отображает количество отслеживаемых и активных проблем в сравнении с числом решённых проблем в вашей среде. При наличии активных проблем их счётчик отображается красным.

    Детализация¶

    • В меню плитки выберите View details, чтобы открыть ленту Problems.

    Настройка¶

    Настройка данного типа плитки из редактора дашборда

    1. Перетащите плитку Problems из панели Edit dashboard, вкладки Tiles, на дашборд.
    2. При желании выберите пользовательский временной диапазон.
    3. При желании выберите пользовательскую зону управления.
    4. При желании выберите окружение.

    Smartscape¶

    Синтетически визуализирует компоненты вашей среды на основе анализа Smartscape. Отображение плитки циклически переключается между пятью слоями Smartscape: приложения, сервисы, процессы ОС, хосты и центры обработки данных. Для каждого слоя плитка показывает общее количество объектов в вашей среде и, красным цветом, количество объектов, на которых в данный момент зафиксированы проблемы.

    Детализация¶

    • В меню плитки выберите View details, чтобы открыть топологическое представление Smartscape на уровне сервисов.

    Настройка¶

    Настройка данного типа плитки из редактора дашборда

    1. Перетащите плитку Smartscape из панели Edit dashboard, вкладки Tiles, на дашборд.
    2. При желании выберите пользовательский временной диапазон.
    3. При желании выберите пользовательскую зону управления.
    4. При желании выберите окружение.

    Таблица журнального запроса (Log query table)¶

    Отображает события журнала, соответствующие заданному запросу к журналам. В табличной плитке отображаются только столбцы, настроенные в средстве просмотра журналов для данного запроса.

    Настройка¶

    Закрепить данный тип плитки на дашборде с заданными фильтрами

    1. Перейдите в Logs.
    2. В Log Viewer введите запрос к журналу, который должна отражать плитка.
    3. При желании нажмите Table options, чтобы указать отображаемые столбцы таблицы.
    4. Нажмите Pin to dashboard, выберите целевой дашборд и нажмите Pin.

    5. Нажмите Open dashboard, чтобы увидеть новую плитку на дашборде.

    6. Плитка отображает результаты запроса, заданного в Log Viewer.

    7. Отображаются только столбцы, указанные в Table options.

    Настройка данного типа плитки из редактора дашборда

    1. Перетащите плитку Log query table из панели Edit dashboard, вкладки Tiles, на дашборд.
    2. Нажмите Configure tile, чтобы открыть Log Viewer.
    3. Введите запрос к журналу.
    4. При желании нажмите Table options, чтобы указать отображаемые столбцы таблицы.
    5. Нажмите Save changes to dashboard, чтобы вернуться к редактированию плитки Log query table на дашборде.
    6. При желании выберите пользовательский временной диапазон.
    7. При желании выберите пользовательскую зону управления.
    8. При желании выберите окружение.
