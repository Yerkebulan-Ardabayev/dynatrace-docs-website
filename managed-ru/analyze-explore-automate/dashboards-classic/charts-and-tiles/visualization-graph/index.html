    Настройка и использование визуализации «График» в Dynatrace¶

    • Практическое руководство
    • Чтение: 2 мин
    • Опубликовано 13 декабря 2021 г.

    Dashboards Classic

    Чтобы визуализировать результаты запроса в виде графика, выберите Graph из списка над определением запроса в левом верхнем углу страницы.

    На графике можно отобразить до 20 рядов на метрику.

    Пример, закреплённый на дашборде в виде плитки¶

    График с двумя метриками: плитка дашборда

    Смена визуализации¶

    При переключении между визуализациями учтите, что некоторые параметры настройки характерны для конкретной визуализации.

    • Если выбрать визуализацию, настроить для неё Settings, а затем переключиться на другую визуализацию, часть настроек первой визуализации может быть проигнорирована, так как они неприменимы к новой. Значок информации в списке визуализаций предупредит об этой возможности.

    Data Explorer: предупреждение о смене визуализации * При возврате к исходной визуализации может потребоваться повторная настройка некоторых параметров.

    Изменение набора метрик¶

    Визуализация может отображать любой набор метрик в запросе с несколькими метриками.

    Чтобы включать или отключать метрики, выберите букву рядом с нужной метрикой или нажмите значок глаза.

    Data Explorer: переключение метрики

    Базовые линии¶

    Чтобы помочь в обнаружении аномалий, можно использовать базовые линии для добавления доверительной полосы к линии метрики на графике. Тогда будет видно, когда значение выходит за пределы доверительной полосы. Расчёт базовой линии основан на модели Seasonal baseline, которая используется для создания событий метрик для обнаружения аномалий.

    • Базовые линии применяются только к визуализации Graph.
    • Базовые линии не добавляются к плитке дашборда при закреплении диаграммы.
    • Временной диапазон для вычисления базовой линии определяется текущим выбранным разрешением:
    Диапазон разрешения Примеры разрешений Временной диапазон базовой линии
    разрешение < 5 минут + 1 минута предыдущие 14 дней
    5 минут ≥ разрешение < 1 час + 5 минут + 10 минут + 30 минут предыдущие 28 дней
    1 час ≥ разрешение < 1 день + 1 час + 6 часов + 12 часов 400 дней
    разрешение ≥ 1 день + 1 день + 1 неделя + 1 месяц 5 лет

    Добавление базовой линии¶

    1. На графике выберите линию метрики, для которой хотите добавить базовую линию.
    2. Во всплывающем окне нажмите Add baseline.
    3. Подождите, пока рассчитывается базовая линия (Loading). После расчёта график будет перерисован с отображением базовой линии для выбранной метрики.

    Скрытие и отображение базовой линии¶

    Базовые линии отображаются отдельно в легенде диаграммы. Например, если добавить базовую линию для метрики CPU usage % в визуализации Graph, легенда будет содержать CPU usage % и CPU usage % - baseline. Выберите элементы легенды, чтобы включить или отключить их отображение.

    Удаление базовой линии¶

    1. На графике выберите линию метрики, для которой хотите удалить базовую линию.
    2. Во всплывающем окне нажмите Remove baseline. График будет перерисован без базовой линии.

    Сравнение с базовыми линиями событий метрик¶

    Вы можете заметить различия между базовыми линиями в Data Explorer и событиями метрик. Эти функции используют разные подходы, соответствующие их разным задачам. Конфигурация Data Explorer фиксированная, конфигурация событий метрик — настраиваемая.

    Data Explorer Metric events
    Samples 5 Настраиваемое
    Violating samples 3 Настраиваемое
    Dealerting samples 5 Настраиваемое
    Alert on no data false Настраиваемое
    Tolerance (влияет на ширину доверительной полосы) 4 Настраиваемое (диапазон: 0.1 до 10)
    Resolution (влияет на детализацию) Настраиваемое 1 минута
    Training time Мгновенное Ежедневное

    Подробнее о сезонных базовых линиях см. в разделе Seasonal baseline.

    Часто задаваемые вопросы о базовых линиях¶

    Как рассчитывается базовая линия?

    Расчёт базовой линии основан на модели сезонной базовой линии, которая используется для создания событий метрик для обнаружения аномалий. Подробнее о работе модели см. в разделе Seasonal baseline.

    Почему базовая линия отличается от предварительного просмотра модели сезонной базовой линии?

    Хотя базовая линия основана на модели сезонной базовой линии, существуют несколько причин, по которым результирующие базовые линии могут отличаться:

    • Разрешение: Базовая линия в Data Explorer определяется из данных в зависимости от текущего выбранного разрешения, как описано выше. Модель сезонной базовой линии для конфигурации событий метрик всегда обучается на данных с разрешением 1 минута. Если разрешения отличаются, результирующие базовые линии также будут отличаться, поскольку значения метрик тоже различаются.
    • Временной диапазон базовой линии: Временной диапазон для вычисления базовой линии определяется текущим выбранным разрешением, как описано выше. Поскольку конфигурации событий метрик всегда используют данные с разрешением 1 минута, временной диапазон обучения может отличаться, что также может приводить к различным базовым линиям.

    • Параметры модели: Базовая линия в Data Explorer использует фиксированные параметры по умолчанию для обучения модели:

    • Tolerance = 4

    • Условие оповещения = 'Alert if metric is outside'
    • Нет оповещения при отсутствии данных Если в конфигурации событий метрик параметры отличаются, результирующая базовая линия также может отличаться.

    Коррелированные метрики¶

    Davis® от Dynatrace учитывает предметно-специфические знания и топологию при вычислении связанных сигналов наблюдаемости. Davis ранжирует наиболее релевантные сигналы в начале списка, а оценка Davis для каждого обнаруженного сигнала указывает на степень его соответствия поведению эталонного сигнала в выбранном временном диапазоне. Подробнее о Davis® AI.

    Добавление коррелированных метрик¶

    Обратите внимание, что этот параметр доступен только при использовании Split by по измерению в запросе.

    1. Перейдите в Data Explorer (стандартный или расширенный режим), создайте запрос для ряда метрик с группировкой по связанному измерению и отобразите его в визуализации Graph.

    Коррелированные метрики доступны только при:

    • Выборе визуализации Graph
    • Указании запроса с Split by по измерению, связанному с выбранным рядом данных

    Попробуйте этот пример:

    Коррелированные метрики: запрос: стандартный режим

    В Advanced mode это выглядит так:

    Коррелированные метрики: запрос: расширенный режим

    builtin:host.cpu.usage:splitBy("dt.entity.host"):sort(value(auto,descending)):limit(20)
    2. Нажмите Run query для построения графика. 3. Выберите (нажмите) линию на графике, чтобы открыть всплывающее окно с доступными параметрами. 4. Во всплывающем окне нажмите See correlated metrics.

    На боковой панели Davis for Correlation analysis будет показан список метрик, которые, по результатам корреляционного анализа Davis AI, коррелируют с выбранным рядом. Корреляция определяется формой ряда, а не значениями.

    Что отображает анализатор?

    • Reference signal — ряд данных, выбранный на графике. Формы других рядов метрик сравниваются с формой этого ряда.

    • Connected signals — другие ряды метрик со схожей формой, отсортированные от наибольшего сходства к наименьшему. Чем больше сходство формы, тем теснее корреляция.

      Для каждой коррелированной метрики анализатор отображает:

      • Имя метрики
      • Измерение
      • ID объекта

      Корреляции иногда группируются. 5. На боковой панели выберите любую метрику из списка, чтобы автоматически добавить её к текущему запросу.

    • Можно добавить несколько коррелированных метрик к запросу

    • Можно добавить одну и ту же метрику несколько раз, а затем отредактировать запрос
    • После добавления коррелированных метрик нажмите Run query для обновления графика.

    Часто задаваемые вопросы о коррелированных метриках¶

    Что означает «корреляция» в данном контексте?

    Для определения корреляции анализатор проверяет форму рядов данных, а не значения. Два ряда с очень похожей формой считаются коррелированными.

    Почему параметр «See correlated metrics» недоступен?

    Возможные причины отсутствия параметра «See correlated metrics»:

    • Не выбрана визуализация Graph
    • В запросе не задан Split by по измерению
    • Запрос не запущен для построения графика
    • Линия на графике не выбрана (не нажата)

    Что означает «No connected signals found»?

    Если отображается No connected signals found, возможные причины:

    • Слишком малая дисперсия в выборке (например, метрика, представляющая прямую линию)
    • Слишком мало точек данных в выборке (например, при очень коротком временном диапазоне)

    Фокус¶

    Чтобы временно убрать лишнее с графика и сосредоточиться на одном ряде метрики, можно скрыть всё, кроме выбранного ряда.

    • Фокус применяется только к визуализации Graph.
    • Фокус не изменяет запрос и не влияет на плитку дашборда при закреплении диаграммы.

    Фокус на ряде метрики¶

    1. На линейном графике выберите линию метрики, на которой хотите сосредоточиться.
    2. Во всплывающем окне нажмите Focus.

    График будет перерисован с отображением только выбранной метрики.

    Снятие фокуса¶

    1. На графике выберите линию метрики, на которой установлен фокус.
    2. Во всплывающем окне нажмите Remove focus.

    График будет перерисован с отображением всех метрик.

    Настройки¶

    Раздел Settings — один из разворачиваемых разделов на правой панели Data Explorer. Содержимое раздела Settings может варьироваться в зависимости от выбранной визуализации.

    Data Explorer: раздел Settings

    Разрешение¶

    Разрешение — это детализация оси X (времени) визуализации.

    • Чтобы Data Explorer автоматически выбирал подходящее разрешение для выбранного временного диапазона, в разделе Settings установите Resolution в Auto
    • Чтобы задать конкретное разрешение, выберите его из списка

    Некоторые разрешения недоступны для определённых временных диапазонов. Если выбрана несовместимая комбинация временного диапазона и разрешения, Dynatrace автоматически подберёт разрешение и выведет пояснительное сообщение, например: Auto-resolution applied. Resolution value of [6 hours] applied. Selected timeframe doesn't allow for [5 minutes] resolution. Чтобы переопределить автоматическое разрешение, выберите другое из списка.

    Умное разрешение на плитках дашборда

    Во избежание проблем с производительностью на плитках дашборда, созданных с помощью Data Explorer, максимальное количество точек данных для запроса на плитке составляет 4 000. На основе выбранного временного диапазона и заданного пользовательского разрешения Dynatrace оценивает количество точек данных для результата запроса. Если прогнозируемое количество точек данных превышает 4 000, Dynatrace автоматически переключается на разрешение, достаточно высокое для поддержания количества точек ниже 4 000.

    Обратите внимание, что это не применяется к визуализациям в самом Data Explorer, где допускается более 4 000 точек данных. Ограничение распространяется только на плитки дашборда, созданные с помощью Data Explorer, где комбинация разрешения и временного диапазона, выбранная на дашборде, даёт более 4 000 точек данных.

    Определяет, отображать ли легенду под визуализацией.

    Обратите внимание, что легенда интерактивна: можно выбрать элемент легенды, чтобы включить или отключить отображение соответствующего элемента.

    Настройки по метрике¶

    Раздел Settings также отображает параметры визуализации для каждой метрики, выбранной в запросе.

    Переименование метрики¶

    Можно изменить имя метрики, отображаемое на диаграмме и в её легенде. Определение запроса сохранит исходное имя метрики.

    1. В разделе Settings выберите для метрики, которую хотите переименовать.
    2. Отредактируйте имя и выберите галочку для сохранения нового имени.

    Изменение режима диаграммы¶

    Чтобы изменить режим диаграммы для метрики, в разделе Settings выберите новый режим из списка.

    • Line
    • Column
    • Area

    Изменение цветовой палитры¶

    Чтобы изменить цветовую палитру для метрики, в разделе Settings выберите новую палитру из списка.

    Data Explorer: список цветовых палитр метрик

    Единица измерения и формат¶

    Используйте параметры Unit и Format для определения способа отображения данных. При экспорте в CSV-файл параметры Unit и Format также применяются к экспортируемым значениям.

    Единица измерения¶

    Используйте параметр Unit для задания единицы измерения, в которой отображается метрика.

    • None = Единица измерения не отображается
    • Auto = Dynatrace выбирает подходящую единицу измерения
    • Другие варианты задают конкретную единицу для отображения. Доступные параметры зависят от единицы метрики. Для временной метрики, например, предлагаются альтернативные единицы отображения времени.
    • Чтобы добавить пользовательскую единицу/суффикс, введите строку в поле Unit и выберите её из списка.
    • В Advanced mode можно использовать :setUnit(<unit>) для выбора из более широкого диапазона единиц.

    Примеры обозначений порядков величин в Dynatrace:

    Обозначение Множитель Значение
    k 10^3 кило, тысяча
    M 10^6 мега, миллион
    G 10^9 гига, миллиард
    T 10^12 тера, триллион

    Формат¶

    Используйте параметр Format для настройки количества десятичных знаков, отображаемых для выбранной метрики.

    • None = Без форматирования.
    • Auto = Dynatrace выбирает подходящий формат. Например, там, где None отображал бы 5.062357754177517 %, Auto отображает 5.06 %.
    • Другие варианты задают количество десятичных знаков: 0, 0.0, 0.00, 0.000

    Примеры¶

    Пример: байты

    Когда базовая единица метрики — байты:

    • При Unit = Auto Dynatrace автоматически выражает результаты в удобочитаемой единице, в данном случае GiB.

    Единица на основе байт может иметь двоичную или десятичную основу, что определяет, выберет ли Dynatrace, например, GiB или GB. Если в метрике не указана основа, используется десятичная. * Если автоматически выбранная единица не подходит, можно явно задать единицу (Unit = B, KiB, MiB или GiB). * Для просмотра исходных данных (без конвертации) установите Unit в None и смотрите результаты в базовой единице метрики (в данном случае байтах).

    Пример: доллары и центы

    Когда базовая единица метрики — доллары и центы:

    • Для небольших значений, чтобы видеть результаты в точных долларах и центах: установите Unit = Auto и Format = 0.00 (два десятичных знака для центов).
    • Для больших значений, чтобы видеть результаты в тысячах, миллионах или миллиардах долларов без центов: установите Unit = k (thousand), M (million) или G (billion) и Format = 0 (ничего после десятичной точки).

    Пример: точные счётчики

    Когда базовая единица метрики — счётчик:

    • Для отображения точного значения:

    • Установите Unit = Auto

    • Установите Format = None

    • Для приблизительного значения:

    • Установите Unit = k (thousand), M (million), G (billion) или T (trillion) в зависимости от порядка величин

    • Установите Format = 0.0, 0.00 или 0.0000 в зависимости от нужного количества десятичных знаков

    Пример: пороги

    При задании значений порогов:

    • Если выбрать Unit (например, MiB), параметры Threshold будут подготовлены для соответствия выбранной единице, и достаточно будет ввести значения порогов без указания MiB.
    • Если установить Unit = Auto (чтобы Dynatrace автоматически масштабировал отображение), значения Threshold всё равно нужно задавать в конкретной единице, например в байтах.

    Добавление переопределения цвета¶

    Чтобы принудительно задать другой цвет (переопределить цветовую палитру) для конкретного ряда, например выбранного хоста

    1. В разделе Settings нажмите Add color override
    2. Выберите ряд из списка
    3. Выберите цвет переопределения для этого ряда

    Оси¶

    Раздел Axes — один из разворачиваемых разделов на правой панели Data Explorer. В разделе Axes можно управлять отображением оси X и каждой оси Y вашей визуализации.

    Data Explorer: раздел Axes

    Пошаговое руководство по настройке осей

    В этом руководстве вы добавляете несколько метрик к визуализации и настраиваете параметры осей в разделе Axes. (В примере используется визуализация Graph, но те же настройки применяются к другим визуализациям с осями.)

    1. Перейдите в Data Explorer.

    2. В Data Explorer добавьте метрику builtin:host.mem.used, разбитую по Host, и нажмите Run query.

    3. Ось X, расположенная вдоль нижней части графика, отображает текущий временной диапазон, заданный селектором временного диапазона.

      • По умолчанию ось X не имеет имени, но его можно задать: в разделе Axes найдите X axis, выберите в строке оси X, измените имя (например, на Time) и нажмите галочку для сохранения.
      • Чтобы показать или скрыть ось X, выберите в строке оси X.

      • Ось Y по умолчанию расположена слева от графика.

      • На диаграмме может быть несколько осей Y. Первая автоматически получает метку Y axis 1 в разделе Axes. В данном примере она отображает использованную память в ГБ, соответствующую первой добавленной метрике Memory used (builtin:host.mem.used).

      • Как и для оси X, можно задать имя и показать/скрыть ось Y: найдите Y axis 1 в разделе Axes и используйте соответствующие кнопки.
      • Чтобы переместить ось Y в правую часть диаграммы, измените Position с Left на Right.
      • Чтобы задать диапазон значений оси Y, измените Min, Max с Auto, Auto на числовые значения. Например, установите Min, Max для этой метрики (Memory used) = 5, 10 для отображения значений в этом диапазоне.

    Одна ось Y 3. Нажмите Add metric, добавьте метрику builtin:host.cpu.usage с разбивкой по Host и снова нажмите Run query.

    • Ось Y для второй метрики (CPU usage %) отображается в правой части графика и показывает процент загрузки CPU. (Если на предыдущем шаге была перемещена ось Y, обе оси теперь расположены с одной стороны.)

    • В разделе Axes появится новый раздел Y axis 2.

    Именование оси¶

    Чтобы задать имя оси

    1. В разделе Axes найдите ось, которую хотите переименовать.
    2. Нажмите рядом с именем оси.
    3. Введите новое имя и нажмите галочку для сохранения.
      Имя отображается вертикально рядом с осью Y и горизонтально под осью X.

    Скрытие и отображение оси¶

    Чтобы скрыть ось

    1. В разделе Axes найдите ось, которую хотите скрыть.
    2. Нажмите.
      Чтобы снова показать ось, нажмите ещё раз.

    Задание положения оси¶

    Чтобы указать сторону диаграммы, на которой отображается ось Y

    1. В разделе Axes найдите ось Y, которую хотите переместить.
    2. Установите Position в Left или Right.

    Задание минимума и максимума оси¶

    По умолчанию минимальное и максимальное значения оси задаются автоматически.

    Чтобы задать пользовательские минимальное и максимальное значения оси

    1. В разделе Axes найдите ось, для которой хотите задать границы.
    2. Измените Min, Max с Auto, Auto на пару значений через запятую, соответствующих значениям на выбранной оси.

    Пороговые значения¶

    Раздел Threshold — один из разворачиваемых разделов на правой панели Data Explorer. Содержимое этого раздела может варьироваться в зависимости от выбранной визуализации. Используйте настройки порогов для улучшения визуализаций и плиток.

    Data Explorer: раздел Threshold

    Устанавливайте значения порогов после выбора Unit:

    • Если сначала выбрать Unit, параметры порогов будут подготовлены для соответствия выбранной единице.
    • Если изменить Unit после задания значений порогов, значения порогов не будут автоматически пересчитаны.
